title: GitHub Actions 作业的最大执行时间
---

我一直在使用GitHub Actions来自动化翻译我的博客文章。最初，我尝试在一个任务中翻译所有文章，待所有翻译完成后，再将更改一次性提交回仓库。

我乐观地去睡觉了，期待这个过程能顺利完成。然而，八小时后醒来，却发现了以下错误：

> 在 runner GitHub Actions 12 上运行的作业已超过 360 分钟的最大执行时间。

这意味着6个小时的翻译工作都白费了，因为提交只在最后时刻进行。

为了解决这个问题，我调整了工作流程，改为每处理完10个文件就提交一次更改。

此外，我还采用了多线程编程技术，将总翻译时间从6小时大幅缩短至约1小时。

GitHub Actions 提供了极大的灵活性。它支持多工作流作业，使得任务可以分开执行。一些作业可以在每次提交时触发，而另一些则可以通过不同的事件来触发。
