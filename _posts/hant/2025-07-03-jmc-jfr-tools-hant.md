JDK Mission Control (JMC) 是一套開源的高階工具組，專為管理、監控、分析及診斷在 Java 虛擬機（JVM）上運行的 Java 應用程式而設計。它尤其以能夠以極低效能開銷收集詳細運行時資訊而聞名，使其適合在生產環境中使用。

JMC 的核心與 **JDK Flight Recorder (JFR)** 緊密整合，JFR 是一個直接內建於 JVM 的強大分析與事件收集框架。JFR 持續收集關於 JVM 及應用程式行為的大量數據，包括執行緒活動、記憶體分配、垃圾回收及 I/O 操作。JMC 則作為分析與視覺化這些豐富數據集的主要工具。

**JDK Mission Control 的主要面向與功能包括：**

* **低開銷分析：** 與許多會引入顯著開銷的傳統分析器不同，JMC 透過 JFR 設計為最小化對運行中應用的影響，使其可安全用於生產環境。
* **即時監控（JMX 控制台）：** JMC 包含一個 JMX（Java 管理擴展）控制台，允許對 JVM 和 Java 應用程式進行即時監控與管理。您可以查看各種指標，甚至可以在運行時更改某些 JVM 屬性。
* **詳細數據分析：** JMC 提供一套全面的工具來分析 JFR 收集的數據。這包括：
    * **程式碼效能分析：** 識別瓶頸及應用程式耗費最多時間的區域。
    * **記憶體管理分析：** 了解物件分配模式、垃圾回收行為及潛在的記憶體洩漏（使用如 JOverflow 等工具進行堆積傾印分析）。
    * **執行緒與並行分析：** 視覺化執行緒互動、識別競爭條件及診斷死結。
    * **I/O 與延遲分析：** 深入了解 I/O 操作與延遲問題。
* **自動化分析與規則：** JMC 提供自動化分析功能，具備預定義規則與啟發式方法，以檢測應用程式中常見的效能與功能問題，並提供調校建議與解釋。
* **視覺化工具：** 它提供多種數據的視覺化呈現，包括：
    * 堆疊追蹤圖（火焰圖、熱力圖、依賴關係視圖）。
    * 執行緒時間軸。
    * 各種指標的圖表與表格。
* **可擴展性：** JMC 支援外掛程式，允許使用者擴展其功能並創建自訂視覺化或分析工具。
* **生產環境就緒：** 其低開銷與全面的數據收集，使其成為在生產環境中診斷問題與優化 Java 應用程式的寶貴工具。

本質上，JDK Mission Control 使開發者與運維團隊能夠深入洞察其 Java 應用程式的行為，從而有效識別並解決效能問題、記憶體問題及其他運行時異常。
