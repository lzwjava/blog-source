### GitHub Actions 使用限制

GitHub Actions 根據您的方案（個人帳戶或組織）設有分級限制。這些限制包括私有儲存庫中 GitHub 託管運行器的每月免費分鐘數以及產品的儲存空間。公開儲存庫和自託管運行器不會消耗這些分鐘數。分鐘數限制每月重置（儲存空間則持續計算）。如果您設有有效的付款方式，超額使用將自動計費；否則，在達到限制後作業將被阻擋。

#### 各方案包含的分鐘數與儲存空間

| 方案                          | 儲存空間 | 分鐘數 (每月) |
|-------------------------------|---------|---------------------|
| GitHub Free (個人/組織)   | 500 MB | 2,000              |
| GitHub Pro (個人)        | 1 GB   | 3,000              |
| GitHub Team (組織)            | 2 GB   | 3,000              |
| GitHub Enterprise Cloud (組織)| 50 GB  | 50,000             |

- **分鐘數**：計算 GitHub 託管運行器上的總作業運行時間（失敗作業的部分時間亦計入）。適用倍率：Linux (1倍)、Windows (2倍)、macOS (10倍)。對儲存庫具有寫入權限的任何人都會使用儲存庫擁有者的額度。
- **儲存空間**：基於產品儲存空間的 GB-小時計算（例如上傳/下載）。日誌和摘要不計入。

#### 超額計費
若超出配額：
- **分鐘數**：按額外分鐘數收費（標準 Linux 運行器的美元費率：$0.008/分鐘；因倍率關係，Windows/macOS 費用更高）。
- **儲存空間**：按 GB-月計費（總 GB-小時 / 每月 744 小時，無條件進位）。
- 範例：在 Team 方案中，超額使用 5,000 分鐘（3,000 Linux + 2,000 Windows）約需花費 $56。

若未設定付款方式，使用量將在達到配額時停止。您看到的錯誤訊息（「近期帳戶付款失敗或需要提高您的支出限額」）意味著付款被拒或您的**支出限額（預算）** 限制了超額使用。

#### 支出限額（預算）
- 這是用於控制 Actions（及其他計量功能）每月總支出的可選閾值。
- 若設定過低，會在達到配額前阻擋作業。
- **檢查/提高方法**：
  1. 前往 GitHub 設定 > 帳單與方案。
  2. 在「預算與提醒」下，檢視或編輯您的 GitHub Actions 預算。
  3. 提高限額、新增付款方式，或完全移除預算（無限制超額使用，但需透過提醒監控）。
- 遷移的帳戶可能從舊有限制自動生成預算 — 請根據使用情況更新它們。

如需完整詳情，請在帳戶的帳單儀表板中查看即時使用情況。
