以下是一些按類別整理的常用編程提示，適用於各種編程任務。這些提示常在使用 IDE、程式碼編輯器或與 AI 編程助手互動時，用於撰寫程式碼、重構、變更變數名稱等相關活動。

### 撰寫程式碼
這些提示用於生成新的程式碼片段或實現：
- 「編寫一個函數來[執行特定任務]。」  
  *範例：「編寫一個函數來計算陣列的總和。」*
- 「建立一個用於[特定目的]的類別。」  
  *範例：「建立一個用於管理用戶檔案的類別。」*
- 「生成程式碼以[實現特定功能]。」  
  *範例：「生成程式碼以從 API 獲取資料。」*

### 重構
這些提示專注於改善現有程式碼的結構、可讀性或效率，而不改變其行為：
- 「重構此程式碼以提升可讀性。」  
  *範例：「重構此程式碼，使用更清晰的變數名稱和一致的格式。」*
- 「透過將此函數拆分為多個小函數來簡化它。」  
  *範例：「透過將驗證邏輯提取到獨立方法中來簡化此函數。」*
- 「優化此程式碼以提升效能。」  
  *範例：「透過減少嵌套迴圈來優化此程式碼。」*

### 變更變數名稱
這些提示旨在透過更新識別符號為更具描述性或一致的名稱，來提升程式碼清晰度：
- 「將變數『[變數名稱]』重新命名為更具描述性的名稱。」  
  *範例：「將變數『x』重新命名為『userAge』。」*
- 「將函數名稱『[函數名稱]』更改為更能反映其用途的名稱。」  
  *範例：「將函數名稱『func』更改為『calculateTotalPrice』。」*
- 「更新所有變數名稱以遵循[特定命名約定]。」  
  *範例：「更新所有變數名稱以遵循駝峰式命名約定。」*

### 除錯
這些提示有助於識別和解決程式碼中的問題：
- 「找出並修復此程式碼中的錯誤。」  
  *範例：「找出並修復導致此空指標異常的錯誤。」*
- 「對此函數進行除錯以處理[特定邊界情況]。」  
  *範例：「對此函數進行除錯以處理空輸入陣列。」*

### 添加註釋/文件
這些提示透過添加解釋或正式文件來提升程式碼的可維護性：
- 「為此程式碼添加註釋以解釋其功能。」  
  *範例：「為此程式碼添加註釋以描述所使用的排序演算法。」*
- 「為此類別/方法生成文件。」  
  *範例：「為此 User 類別生成文件。」*

### 程式碼轉譯
這些提示用於在不同程式語言之間轉換程式碼：
- 「將此[來源語言]程式碼轉譯為[目標語言]。」  
  *範例：「將此 Python 程式碼轉譯為 Java。」*
- 「將此[語言]函數轉換為[另一語言]。」  
  *範例：「將此 JavaScript 函數轉換為 TypeScript。」*

### 程式碼審查
這些提示用於徵求回饋或識別潛在改進點：
- 「審查此程式碼並提出改進建議。」  
  *範例：「審查此程式碼並提出降低複雜度的方法。」*
- 「檢查此程式碼是否存在潛在安全問題。」  
  *範例：「檢查此程式碼是否存在 SQL 注入漏洞。」*

### 生成測試
這些提示用於建立測試案例以驗證程式碼正確性：
- 「為此函數編寫單元測試。」  
  *範例：「為此 calculateDiscount 函數編寫單元測試。」*
- 「建立測試案例以驗證此方法。」  
  *範例：「建立測試案例以驗證此方法處理負數輸入的情況。」*

### 對話式提示
這些是更貼近自然語言的提示，常與 AI 助手一起使用：
- 「你能幫我編寫一個腳本來[執行任務]嗎？」  
  *範例：「你能幫我編寫一個腳本來重新命名目錄中的檔案嗎？」*
- 「我需要能[執行特定操作]的程式碼。」  
  *範例：「我需要能讀取 CSV 檔案並排序資料的程式碼。」*
- 「展示一個[使用特定功能或 API]的範例。」  
  *範例：「展示一個在 JavaScript 中使用 Fetch API 的範例。」*

這些提示為常見編程任務提供了多功能的基礎框架。您可以根據專案的具體情境或需求，填入詳細資訊（例如函數名稱、語言或限制條件）來調整使用。無論是編寫新程式碼、優化現有程式庫，還是向 AI 工具尋求協助，這些模板都能涵蓋廣泛的應用場景。
