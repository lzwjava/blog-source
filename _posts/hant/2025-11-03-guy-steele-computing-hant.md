小蓋·路易斯·史提爾（Guy Lewis Steele Jr.），在程式設計圈內親切地被稱為 GLS，是電腦科學史上最具影響力的人物之一。作為語言設計、實作與理論基礎的大師，史提爾從 Lisp 方言到企業級程式語言的各個領域都留下了不可磨滅的印記。他的作品體現了優雅、實用與一絲奇思妙想——常融入令駭客同好會心一笑的雙關語和文字遊戲。歷經超過五十年的職業生涯，史提爾的貢獻影響了我們對程式語言的思考、建構與演進方式，使複雜的概念既易於理解又強大實用。

## 早年生活與教育
史提爾於 1954 年 10 月 2 日出生於密蘇里州，在一個重視知識探索的家庭中成長。他於 1975 年獲得加州理工學院的數學學士學位，隨後於 1977 年和 1980 年分別獲得麻省理工學院的電腦科學碩士及博士學位。他的博士論文《資料類型作為格：程式語言設計的新途徑》為他對類型系統和形式語義學的興趣奠定了早期基礎——這些概念將貫穿他的整個職業生涯。

在麻省理工學院，史提爾受到傑拉德·傑伊·蘇斯曼等學術巨擘的指導，沉浸在人工智慧實驗室的駭客文化中。這個充滿早期人工智慧實驗和 Lisp 機器的環境，點燃了他對表達力強、簡約語言的終身熱情。

## Lisp 與 Scheme 的先驅工作
史提爾的突破性成就出現在 1970 年代末期，他成為 **Scheme** 的關鍵架構師。Scheme 是 Lisp 的一種方言，專為簡潔性和可擴展性而設計。該語言於 1975 年與蘇斯曼共同開發（並在 1978 年的 Lambda 論文中正式化），Scheme 剔除了 Lisp 的歷史包袱，創造出一種潔淨、函數式的語言，非常適合教學、研究和符號運算。其詞法作用域、一級 Continuation 和尾呼叫最佳化成為現代函數式程式語言的標誌。

Scheme 的影響深遠：它啟發了如 Clojure、Racket 等語言，甚至影響了 JavaScript 和 Python 的某些方面。史提爾的角色不僅限於設計；他實作了早期版本，並合著了《Scheme 修訂報告》，確保了該語言歷經多次修訂直至 2013 年的 R7RS。他的格言——「簡單的事情應該簡單，複雜的事情應該可能」——概括了 Scheme 的哲學。

## 從學術到業界：T、Java 及其他
史提爾的職業生涯無縫連接了學術界與產業界。1980 年代，在麻省理工學院的電腦科學實驗室，他共同領導了 **T 程式語言**（後更名為 Argus）的開發，這是 Scheme 的一個物件導向擴展，用於分散式系統。這項工作預示了他在並行與平行計算方面的專業知識。

1990 年，史提爾加入 **Sun Microsystems**，成為 **Java 程式語言** 的主要設計者之一。他負責定義 Java 的語法與語義，確保其具備可移植性、安全性和物件導向特性——這對 Java 爆炸性成長為企業級主流語言至關重要。他在 OOPSLA '98 上的演講「Growing a Language」，講述了 Java 的演進過程，是迭代設計的典範，強調了語言如何透過實際使用而非自上而下的指令來「成長」。

後來，在 **Oracle**（於 2010 年收購 Sun 後），史提爾參與了 **Fortran** 的標準化工作，並透過 **Parallel Fortress** 項目探索平行程式設計——這是一個用於科學計算的高效能語言。他也涉足領域特定語言，例如在其業餘項目 Common Lisp Music 系統中用於音樂記譜的語言。

## 理論貢獻與著作
史提爾不僅是建構者，更是思想家。他的開創性論文《揭穿「昂貴程序呼叫」迷思》（1977）挑戰了關於最佳化的既有假設，影響了編譯器設計。他合著了經典著作如《Common Lisp: The Language》（1984, 1990），該書為 Common Lisp 的商業應用制定了標準。

他的寫作充滿幽默感：Lambda 論文中充斥著雙關語（例如，「Hairy Control」，意指複雜的解析和 Continuation）。史提爾的演講，如「是時候迎接新的程式語言了」或他與蘇斯曼共同獲得 2020 年 ACM 圖靈獎（表彰其在程式語言設計方面的貢獻）時的獲獎感言，都融合了嚴謹與機智。

## 獎項、遺產與個人風格
史提爾的榮譽包括 ACM 程式語言設計與實作獎（1996）、IEEE 約翰·馮·諾依曼獎章（2010）以及前述的圖靈獎——被譽為「計算界的諾貝爾獎」。他是 ACM、IEEE 和美國科學促進會會士，並任職於美國國家工程院。

在程式碼之外，史提爾是一位博學家：鋼琴家、合唱團員和出色的雙關語高手。他的電子郵件簽名曾寫道：「在建構語言中，如同在生活中，旅程本身就是回報。」（Conlanging？那是指人造語言，這是他的另一個愛好。）如今他已年近七旬，依然活躍，就語言演進提供諮詢，並偶爾在推特上發布隱晦的格言。

在框架膨脹的時代，史提爾提醒我們，偉大的軟體始於簡單而優美的想法。正如他在 2008 年的一次訪談中所說：「抽象的目的不是為了顯得聰明，而是為了清晰。」他的遺產是什麼？一個讓程式設計感覺更像詩歌而非苦役的世界。
