GraalVM 是一個**高效能、多語言運行環境**，讓您能在單一環境中運行以多種程式語言編寫的應用程式，例如 Java、JavaScript、Python、Ruby、R 以及基於 LLVM 框架的語言（如 C 和 C++）。它由 Oracle 開發，基於 Java HotSpot 虛擬機器。

以下是其關鍵特性與優勢的詳細說明：

**1. 多語言支援：**

* GraalVM 能實現不同程式語言間的無縫互操作性。您可以將一種語言編寫的程式碼嵌入到另一種語言的應用程式中。例如，您可以讓 Java 應用程式執行 Python 腳本或 Node.js 模組。
* 這為在單一應用程式中發揮不同語言的優勢開闢了可能性。

**2. 原生映像生成：**

* GraalVM 最顯著的功能之一，是能夠將 Java 應用程式透過預先編譯（AOT）轉換為**原生可執行檔**。
* 此過程消除了運行應用程式時對傳統 JVM 的需求。生成的原生映像包含應用程式運行所需的一切，包括必要的運行時組件。
* **原生映像的優勢：**
    * **更快的啟動速度：** 原生可執行檔幾乎能瞬間啟動，這對雲端原生應用和微服務至關重要。
    * **更低的記憶體佔用：** 與在 JVM 上運行相比，原生映像通常消耗的記憶體顯著減少。
    * **縮小的攻擊面：** 透過排除未使用的程式碼和 JIT 編譯基礎設施，原生映像可以提升安全性。
    * **更小的部署體積：** 原生可執行檔通常體積更小，易於封裝和部署。

**3. 高效能：**

* GraalVM 內含一個名為 Graal 的先進優化編譯器，能顯著提升應用程式的效能，包括那些在 JVM 上運行的應用。
* 對於多語言應用程式，GraalVM 旨在提供與語言專用運行時相當或更優的效能。

**4. 可嵌入性：**

* GraalVM 可嵌入到其他應用程式中。例如，您可以將 GraalVM 的 JavaScript 引擎整合到 Java 應用程式中，以提供腳本功能。

**5. 使用情境：**

* **微服務：** 原生映像是為雲端環境構建快速啟動且資源高效的微服務的理想選擇。
* **無伺服器函數：** 其快速啟動特性使 GraalVM 成為解決冷啟動問題的無伺服器函數的絕佳選擇。
* **多語言應用程式：** 需要利用多種程式語言程式碼的應用程式。
* **提升現有 Java 應用程式效能：** 透過在標準 JVM 中使用 Graal 編譯器作為 JIT 編譯器。
* **創建語言運行時與工具：** GraalVM 內的 Truffle 框架讓開發者能為新興或現有程式語言構建高效能的直譯器。

**6. 開源：**

* GraalVM 社群版是一個開源專案，使其能被廣泛用戶和開發者使用。此外還有提供額外功能與支援的企業版。

總而言之，GraalVM 是一個多功能且強大的運行時環境，在效能、資源效率以及多程式語言協作能力方面提供顯著優勢。其原生映像生成功能對於現代雲端原生開發尤其具有吸引力。
