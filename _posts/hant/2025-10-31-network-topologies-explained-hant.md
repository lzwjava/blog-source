### 網絡拓撲結構詳解

這段筆記看似是中國高等教育自學考試（廣東省）「計算機網絡技術」（課程代碼 02141）的自學應試精簡筆記。此考試體系專為在職成人提供高等教育文憑途徑。筆記重點在於**網絡拓撲結構**，即電腦與設備在網絡中進行通訊的基本佈局方式。拓撲可分為**物理拓撲**（實體線纜與硬件的佈局）與**邏輯拓撲**（數據在網絡中的傳輸路徑，與實體佈局無關）。

本質上，拓撲結構定義了節點（如電腦、打印機、伺服器等設備）之間的連接與互動方式。選擇合適的拓撲將影響可靠性、成本、擴展性及故障排查難易度。以下將根據筆記內容，詳細說明四種常見拓撲類型，包含其關鍵特性、優缺點及實際應用案例，並以文字圖示輔助說明。

#### 1. **星型拓撲**
   - **說明**：所有設備直接連接到中央的 hub、switch 或 router（如車輪輻條）。設備間傳輸數據需先經過中央節點，再轉送至目標設備。
   - **關鍵特性**（摘自筆記）：中央集線器；易於管理。
   - **優點**：
     - 增減設備時不影響網絡運作。
     - 故障隔離：單一線路故障僅影響該設備。
     - 高效能，數據碰撞機率低。
   - **缺點**：
     - 單點故障風險：中央節點失效將導致全網癱瘓。
     - 線纜用量較其他拓撲多。
   - **文字圖示**：
     ```
         設備 A ----+
                       |
         設備 B ----+---- 集線器/交換器
                       |
         設備 C ----+
     ```
   - **應用場景**：多數家庭/辦公室區域網（如透過 Wi-Fi router 組建的 Ethernet 網絡）。

#### 2. **匯流排拓撲**
   - **說明**：所有設備連接至單一共享線纜（即「匯流排」）作為骨幹。數據沿線纜傳輸，所有設備皆可讀取，但僅目標設備會處理數據。
   - **關鍵特性**（摘自筆記）：單一線纜；結構簡單但易生碰撞（多設備同時傳輸導致數據衝突）。
   - **優點**：
     - 成本低且架設簡易（線纜用量最少）。
     - 適用小型網絡。
   - **缺點**：
     - 易發生碰撞，長距離傳輸易信號衰減。
     - 故障排查困難；線纜斷裂或短路可能導致全網中斷。
     - 已不適用現代高速網絡需求。
   - **文字圖示**：
     ```
     設備 A ----- 設備 B ----- 設備 C
                  (共享線纜/匯流排)
     ```
   - **應用場景**：早期 Ethernet 網絡或細同軸電纜佈線（如 10BASE2）；現已罕見。

#### 3. **環型拓撲**
   - **說明**：設備形成封閉迴路（環狀），每台設備僅與相鄰兩台連接。數據沿單一方向（或雙環結構中的雙向）繞環傳輸，經過每台設備直至抵達目標。
   - **關鍵特性**（摘自筆記）：循環數據流；每台設備連接至下一台。
   - **優點**：
     - 無數據碰撞（數據有專屬傳輸路徑）。
     - 所有設備存取權平等；效能穩定可預測。
     - 適用 token-passing 通訊協定（如僅持有「token」的設備可發送數據）。
   - **缺點**：
     - 單點斷線或設備故障即可能中斷全網（雙環結構除外）。
     - 增減設備需暫停網絡運作。
     - 大型環網故障排查困難。
   - **文字圖示**：
     ```
           設備 A
            /     \
     設備 D        設備 B
            \     /
           設備 C
     (數據流向：A → B → C → D → A)
     ```
   - **應用場景**：Token Ring 網絡（IBM 舊標準）或高可靠性環境的光纖佈線（如 FDDI）。

#### 4. **網狀拓撲**
   - **說明**：每台設備直接連接到其他所有設備（全網狀），或至少與多台設備連接（部分網狀）。此結構創造多條數據傳輸路徑。
   - **關鍵特性**（摘自筆記）：所有設備互連；可靠性高但結構複雜。
   - **優點**：
     - 極高可靠性：多重路徑確保單一故障不影響全網。
     - 高冗餘與容錯能力。
     - 適用高流量或關鍵系統。
   - **缺點**：
     - 成本高昂（線纜與連接端口需求大；擴展性差——n 台設備需 n(n-1)/2 條連接）。
     - 安裝、設定與維護複雜。
     - 小型網絡過度浪費。
   - **文字圖示**（4 台設備部分網狀）：
     ```
     設備 A ─── 設備 B
       │         │
       └───┬─── 設備 C
           │
       設備 D ───┘
     (全網狀需增補 A-D、B-D 等連接)
     ```
   - **應用場景**：無線網狀網絡（如 Google Nest 等智能家居設備）、骨幹網絡或軍用通訊系統等高韌性需求場景。

### 拓撲比較表
為便於快速參考，以下為各拓撲的對照摘要表：

| 拓撲類型 | 線纜需求 | 可靠性 | 管理難易度 | 適用場景 | 常見缺點 |
|----------|---------------|-------------|--------------------|----------|-----------------|
| **星型** | 高（集中佈線） | 中（依賴中央節點） | 高 | 中小型區域網 | 中央節點故障 |
| **匯流排** | 低（單一線纜） | 低 | 中 | 極小型/簡易網絡 | 碰撞與線路中斷 |
| **環型** | 中（環狀佈線） | 中（單向傳輸） | 中 | 權杖傳遞網絡 | 單點中斷風險 |
| **網狀** | 極高 | 極高 | 低 | 關鍵/高冗餘系統 | 成本與複雜度 |

### 應試準備要點
- **混合拓撲**：實際網絡常結合多種拓撲（如星型-匯流排混合結構）。
- **現代趨勢**：無線技術普及後，邏輯拓撲（如 Wi-Fi 的類星型運作）較純物理拓撲更常見。
- **考試重點**：掌握各拓撲優缺點、故障模式及相關通訊協定（如匯流排拓撲用 CSMA/CD 處理碰撞）。
