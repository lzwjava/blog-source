# lzwjava.github.io

歡迎來到我的個人部落格！這裡，我分享我的想法、經歷和各種主題的知識。

## 主要改進

這個部落格相比於使用 jekyll-theme-cayman 的標準 Jekyll 部落格，包含了幾項增強功能：

*   **AI 翻譯：**利用先進的語言模型進行準確且上下文相關的翻譯，擴展內容的全球可訪問性。
*   **XeLaTeX PDF 生成：** 整合 XeLaTeX 以產生高品質的印刷準備 PDF，適用於離線閱讀和分享。
*   **Google Cloud 文本轉語音：** 利用 Google Cloud 的文本轉語音服務生成文章的音鑑版本，改善視障用戶和偏愛音鑑內容的用戶的可訪問性。
*   **增強 CSS 設計：** 具有精緻且自定義的 CSS 設計，提供視覺上吸引力和用戶友好的體驗。
*   **MathJax 支持：** 實現 MathJax 來渲染複雜的數學表達式和方程，使技術內容更易訪問。
*   **夜間模式：** 提供夜間模式選項以減少疲勞和在低光條件下提高可讀性。
*   **靈活的文章選擇：** 提供各種文章選擇選項，例如按類別或標籤過濾，以增強導航。
*   **定期更新：** 確保博客的庫和依賴項目保持最新，以實現最佳性能和安全性。
*   **`awesome-cv` 整合：** 使用 `awesome-cv` 直接從博客生成專業的簡歷。
*   **RSS 訂閱支持：** 提供 RSS 訂閱，允許用戶訂閱博客。
*   **雙語內容：** 支持中文和英語內容以滿足多元化的受眾。
*   **GitHub Workflow 自動化：** 實現 GitHub Actions，用於自動構建、測試和部署，確保開發流程的順暢。
*   **自動翻譯工作流：** 使用 GitHub Actions 自動將新的或更新的文章翻譯成多種語言。
*   **EPUB 支持：** 將 Markdown 轉換為 EPUB 以供電子書閱讀器使用。

## 開始使用

要設置本地 Jekyll 環境，請遵循以下步驟：

```shell
gem install jekyll bundler

jekyll new myblog

cd myblog

bundle install

bundle exec jekyll serve

bundle exec jekyll serve --draft

bundle add webrick

jekyll serve
```

## Markdown 樣式

要自定義語法高亮，使用 `rougify`：

```shell
rougify help style

rougify style github > _sass/syntax.css

rougify style gruvbox.dark > _sass/syntax.css

rougify style base16.monokai.dark > _sass/syntax.css
```

## 音頻和 PDF 流水線

要生成音頻和 PDF 文件，使用以下命令：

```bash
conda activate google-cloud-env

python scripts/audio_pipeline.py --task posts --n 10

python scripts/audio_pipeline.py --task pages

python scripts/audio_pipeline.py --task notes

python scripts/audio_pipeline.py --task notes --n 100

python scripts/pdf_pipeline.py --task pages

python scripts/pdf_pipeline.py --task posts --n 10

python scripts/pdf_pipeline.py --task notes

python scripts/generate_notes_link.py

python scripts/auto-ss-config/upload_configs.py

python scripts/epub_book.py notes

python scripts/epub_book.py _posts/en

python scripts/epub_book.py _posts/ja
```

## 腳本

要創建新的草稿文章，使用：

```
python scripts/draft.py <name>
```

## 內容指南

*   `-en`：表示英文文章。
*   `-zh`：表示中文文章。

`_posts` 和 `pages` 目錄支持 `-en` 和 `-zh` 後綴。

`notes` 目錄主要包含英文內容，供個人學習使用，不需要翻譯。
