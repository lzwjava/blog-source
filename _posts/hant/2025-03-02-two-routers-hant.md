要設定您的兩台 TP-Link AX3000 (TL-XDR 3050) 路由器與數據機組成 **數據機 → 有線 → 路由器1 → 無線 → 路由器2** 的配置，請按照以下逐步指南操作。您的數據機已具備 WiFi 功能且 IP 地址為 `192.168.1.1`。我們將配置路由器1作為通過有線連接到數據機的主路由器，並配置路由器2以無線方式擴展網絡。

以下是實現此配置的方法：

---

### **步驟 1：了解設定**
- **數據機**：提供互聯網接入並擁有自身的 WiFi (IP: `192.168.1.1`)。
- **路由器1**：將通過有線連接到數據機，並作為您網絡的主要路由器。
- **路由器2**：將無線連接到路由器1以擴展網絡覆蓋範圍。

您提到了幾種模式（有線 AP 橋接、無線 AP 橋接、DHCP、寬頻連接）。我們將對路由器1使用 **DHCP** 模式以從數據機獲取互聯網連接，並對路由器2使用 **無線 AP 橋接**（或類似的模式，如 WDS/中繼器）以無線連接到路由器1。

---

### **步驟 2：設定路由器1**
1. **將路由器1連接到數據機**：
   - 取一條以太網線，一端插入數據機的 **LAN 端口**。
   - 另一端插入路由器1的 **WAN (互聯網) 端口**。

2. **存取路由器1的網頁介面**：
   - 將電腦或智能手機連接到路由器1的預設 WiFi 網絡（檢查路由器標籤上的預設 SSID 和密碼）或使用以太網線連接。
   - 打開網頁瀏覽器，輸入 `http://tplinkwifi.net` 或 `192.168.0.1`（TP-Link 路由器的預設 IP）。
   - 使用預設憑據登入（通常用戶名和密碼均為 `admin`），除非您已更改。

3. **配置路由器1**：
   - **互聯網連接**：
     - 前往 **快速設定** 或 **互聯網** 設定部分。
     - 選擇 **DHCP** 模式。這允許路由器1自動從數據機獲取 IP 地址（可能在 `192.168.1.x` 範圍內）。
   - **WiFi 設定**：
     - 為路由器1的 WiFi 設定一個獨特的 **SSID**（網絡名稱）和一個強效的 **密碼**。
     - 保存這些詳細信息，因為路由器2需要它們來進行無線連接。
   - **LAN 設定**：
     - 確保路由器1的 LAN IP 與數據機的 IP 不同。預設情況下，它可能是 `192.168.0.1`，這沒問題，因為數據機是 `192.168.1.1`。
     - 確認路由器1上已啟用 **DHCP**。這允許路由器1為連接到它的設備（包括路由器2）分配 IP 地址（例如 `192.168.0.x`）。
   - 保存設定並在提示時重啟路由器1。

---

### **步驟 3：將路由器2設定為無線橋接**
1. **存取路由器2的網頁介面**：
   - 將電腦或智能手機連接到路由器2的預設 WiFi 網絡或通過以太網連接。
   - 打開網頁瀏覽器，輸入 `http://tplinkwifi.net` 或 `192.168.0.1`。
   - 使用預設憑據（或您的自訂憑據）登入。

2. **在無線橋接模式下配置路由器2**：
   - 在設定中尋找類似 **無線 AP 橋接**、**WDS** 或 **中繼器** 的模式（可能在 **操作模式** 或 **無線** 設定下）。
   - 選擇 **無線 AP 橋接**（或 WDS/中繼器，如果可用）。
   - **連接到路由器1的 WiFi**：
     - 掃描可用的網絡並選擇路由器1的 SSID。
     - 輸入路由器1的 WiFi 密碼。
     - 為確保兼容性，請確保路由器2使用與路由器1相同的無線頻道（例如，如果路由器1使用頻道 6，則將路由器2設定為頻道 6）。
   - **LAN IP 設定**：
     - 更改路由器2的 LAN IP 以避免與路由器1衝突。例如，將其設定為 `192.168.0.2`（因為路由器1可能是 `192.168.0.1`）。
     - 在路由器2上 **停用 DHCP**。路由器1將處理所有設備的 IP 分配。
   - 保存設定並重啟路由器2。它現在應該無線連接到路由器1。

---

### **步驟 4：測試設定**
1. **驗證路由器2的連接**：
   - 重啟後，檢查路由器2的介面以確認它已連接到路由器1的 WiFi。
2. **將設備連接到路由器2**：
   - 使用智能手機、筆記本電腦或其他設備連接到路由器2的 WiFi（根據模式，它可能使用與路由器1相同的 SSID）。
   - 檢查設備是否從路由器1獲取 IP 地址（例如 `192.168.0.x`）。
   - 通過瀏覽網站測試互聯網訪問。

---

### **最終配置摘要**
- **數據機**：IP `192.168.1.1`，提供互聯網和 WiFi。
- **路由器1**：
  - WAN：通過有線連接到數據機，設定為 **DHCP**（從數據機獲取 IP，例如 `192.168.1.x`）。
  - LAN：IP `192.168.0.1`，**啟用 DHCP** 以分配 IP 給設備。
  - WiFi：自訂 SSID 和密碼。
- **路由器2**：
  - 模式：**無線 AP 橋接**（或 WDS/中繼器），無線連接到路由器1的 WiFi。
  - LAN IP：`192.168.0.2`，**停用 DHCP**。

---

### **疑難排解提示**
- **路由器2無法無線連接**：
  - 在路由器2的配置中，仔細檢查路由器1的 SSID、密碼和頻道設定。
  - 如果 **無線 AP 橋接** 不可用，請嘗試 **WDS** 或 **中繼器** 模式。請參閱 TP-Link AX3000 手冊以了解支援的模式。
- **路由器2沒有互聯網**：
  - 確保路由器1的 DHCP 正常運作，並且路由器2的 DHCP 已關閉。
  - 驗證路由器2是否成功橋接到路由器1。
- **雙重 NAT 問題**：
  - 由於您的數據機具有 WiFi 功能且路由器1充當路由器，您可能會遇到雙重 NAT（數據機和路由器1都分配 IP）。對於基本使用來說，這通常沒問題，但可能會影響某些應用程序（例如遊戲）。如果需要，將數據機設定為 **橋接模式**（請參閱數據機手冊），但這可能會停用其 WiFi 功能。

通過此設定，路由器2將從路由器1無線擴展您的網絡，連接到任一路由器的設備都應能通過數據機訪問互聯網。如果您需要進一步協助，請告訴我！

如果您決定為第一個路由器（路由器1）使用固定（靜態）IP 地址，而不是依賴 DHCP，您需要手動配置其 WAN 設定以與數據機通訊。以下是您應該為 IP 地址、DNS、子網路遮罩和閘道配置的內容：

---

### **假設**
- 您的數據機 IP 地址為 `192.168.1.1`，並使用子網路 `192.168.1.0/24`（子網路遮罩 `255.255.255.0`）。
- 數據機有一個 DHCP 範圍（例如 `192.168.1.2` 到 `192.168.1.100`），用於為連接的設備分配 IP 地址。

如果您的數據機使用不同的 IP 或子網路，請相應調整數值。

---

### **路由器1的 WAN 設定**
這些設定配置路由器1如何連接到數據機：

1. **IP 地址**
   - 在數據機的子網路內（`192.168.1.0/24`）選擇一個靜態 IP，但 **在數據機的 DHCP 範圍之外** 以避免衝突。
   - 例如：如果數據機的 DHCP 範圍是 `192.168.1.2` 到 `192.168.1.100`，您可以使用 **`192.168.1.101`**。
   - 這確保路由器1在數據機的網絡上擁有唯一的固定地址。

2. **子網路遮罩**
   - 將其設定為與數據機的子網路遮罩匹配，通常為 **`255.255.255.0`**。
   - 這將網絡範圍定義為 `192.168.1.0` 到 `192.168.1.255`。

3. **預設閘道**
   - 將其設定為數據機的 IP 地址，即 **`192.168.1.1`**。
   - 閘道是路由器1用於訪問互聯網的設備（您的數據機）。

4. **DNS 伺服器**
   - 您有兩個選擇：
     - 使用數據機的 IP：**`192.168.1.1`**。這讓數據機處理 DNS 請求。
     - 使用公共 DNS 伺服器：**`8.8.8.8`**（Google）和 **`8.8.4.4`**（Google 次要）。這些伺服器可靠且通常更快。
   - 建議：從 `192.168.1.1` 開始。如果遇到 DNS 問題，請切換到 `8.8.8.8` 和 `8.8.4.4`。

---

### **配置範例**
您的路由器1 WAN 設定可能如下所示：

- **IP 地址**：`192.168.1.101`
- **子網路遮罩**：`255.255.255.0`
- **預設閘道**：`192.168.1.1`
- **DNS 伺服器**：`192.168.1.1`（或 `8.8.8.8` 和 `8.8.4.4`）

---

### **補充說明**
- **檢查數據機設定**：登入您的數據機（通常在 `192.168.1.1`）以確認其 IP、子網路遮罩和 DHCP 範圍。如果需要，調整路由器1的靜態 IP。
- **路由器1的 LAN 設定**：為避免衝突，將路由器1的 LAN 網絡配置在不同的子網路上，例如 `192.168.0.1`，子網路遮罩為 `255.255.255.0`。在路由器1上啟用 DHCP 以分配 IP（例如 `192.168.0.2` 到 `192.168.0.254`）給連接到它的設備。
- **測試**：應用這些設定後，確保路由器1可以訪問互聯網。如果不可以，請仔細檢查 IP（無衝突）、閘道和 DNS 數值。

此配置為路由器1在數據機的網絡上提供了一個固定 IP，確保了穩定的連接性。如果您需要有關其他路由器或特定數據機詳細信息的幫助，請告訴我！

---

聽起來您已經為路由器1設定了靜態 WAN IP 並配置了其 LAN 的 DHCP，但您的筆記本電腦仍然無法連接到網絡。讓我們逐步進行疑難排解，找出原因並修復它。

---

### 步驟 1：驗證路由器1的 WAN 連接
您的 WAN 設定如下：

- **IP 地址**：`192.168.1.101`
- **子網路遮罩**：`255.255.255.0`
- **預設閘道**：`192.168.1.1`
- **DNS 伺服器**：`192.168.1.1`（或 `8.8.8.8` 和 `8.8.4.4`）

這些設定表明路由器1連接到 IP 為 `192.168.1.1` 的數據機或另一個路由器。以下是需要檢查的內容：

- **路由器1的互聯網訪問**：
  - 登入路由器1的網頁介面（根據您下面的 LAN 設定，可能位於 `192.168.0.1`）。
  - 前往 **狀態** 或 **WAN** 部分，查看路由器1是否顯示活躍的互聯網連接。如果沒有，問題可能出在路由器1和數據機之間。

- **IP 衝突**：
  - 確保數據機網絡上沒有其他設備使用 `192.168.1.101`。如果數據機的 DHCP 範圍包含此 IP，可能會導致衝突。登入數據機（在 `192.168.1.1`）並檢查其 DHCP 設定或已連接設備列表。如果 `192.168.1.101` 在數據機的 DHCP 範圍內，請將路由器1的 WAN IP 更改為該範圍之外的某個 IP（例如 `192.168.1.200`），或將其從數據機的 DHCP 池中排除。

- **數據機連接性**：
  - 確認以太網線已插入數據機的 **LAN 端口** 和路由器1的 **WAN 端口**。如果沒有正確設定，路由器1將無法連接到互聯網。

---

### 步驟 2：檢查路由器1的 LAN 和 DHCP 設定
您的 LAN 和 DHCP 設定如下：

- **LAN IP**：`192.168.0.1`
- **子網路遮罩**：`255.255.255.0`
- **啟用 DHCP**：是
- **IP 地址範圍**：`192.168.0.2` 到 `192.168.0.254`
- **閘道**：`192.168.0.1`
- **DNS 伺服器**：`192.168.0.1`

這些設定看起來沒問題，但讓我們確保它們正常運作：

- **DHCP 功能**：
  - 啟用 DHCP 後，您的筆記本電腦應自動獲取一個介於 `192.168.0.2` 和 `192.168.0.254` 之間的 IP 地址，閘道為 `192.168.0.1`。如果沒有，DHCP 可能無法正常運作。

- **DNS 配置**：
  - 將 DNS 伺服器設定為 `192.168.0.1` 意味著路由器1為您的筆記本電腦處理 DNS 請求。確保路由器1配置為將這些請求轉發到上游 DNS 伺服器（如 `192.168.1.1` 或 `8.8.8.8`）。這通常是自動的，但請在路由器1的設定中仔細檢查。或者，您可以直接將 DHCP DNS 設定為 `8.8.8.8` 和 `8.8.4.4`，以繞過路由器1進行 DNS 解析。

---

### 步驟 3：測試筆記本電腦的連接
既然您的筆記本電腦無法連接，讓我們診斷一下：

- **連接類型**：
  - 您使用的是 WiFi 還是以太網？如果是 WiFi，請確保您連接到的是路由器1的 SSID（不是數據機的）。如果是以太網，請確認網線已插入路由器1的 LAN 端口之一。

- **檢查筆記本電腦的 IP 地址**：
  - 在您的筆記本電腦上，打開 **命令提示字元**（Windows）或 **終端機**（macOS/Linux）：
    - Windows：輸入 `ipconfig` 並按 Enter。
    - macOS/Linux：輸入 `ifconfig` 或 `ip addr` 並按 Enter。
  - 尋找您的網絡適配器的 IP 地址。它應該是類似 `192.168.0.x` 的格式（其中 `x` 介於 2 和 254 之間），子網路遮罩為 `255.255.255.0`，閘道為 `192.168.0.1`。
  - **如果沒有 IP 地址**：
    - 您的筆記本電腦可能無法從 DHCP 獲取 IP。確保其網絡設定設為“自動獲取 IP 地址”。嘗試更新 IP：
      - Windows：運行 `ipconfig /release` 然後 `ipconfig /renew`。
    - 如果仍然失敗，可能是路由器1上的 DHCP 或與其的連接出現問題。

- **Ping 路由器1**：
  - 從您的筆記本電腦運行 `ping 192.168.0.1`。如果您收到回覆，表示您的筆記本電腦已連接到路由器1，但可能無法訪問互聯網。如果超時，則筆記本電腦無法連接到路由器1。

---

### 步驟 4：常見問題與修復
以下是一些可能的原因：

- **網線問題**：
  - 如果使用以太網，請嘗試更換網線或路由器1上的 LAN 端口。故障的網線或端口可能導致連接問題。

- **WiFi 問題**：
  - 如果使用 WiFi，請仔細檢查密碼並確保路由器1的 WiFi 已啟用。信號干擾或配置錯誤的 SSID 也可能導致問題。

- **路由器1-數據機鏈路**：
  - 如果路由器1無法獲取互聯網連接，數據機可能需要特定設定（例如 MAC 地址克隆）。請檢查數據機的手冊或設定以了解相關信息。

- **雙重 NAT**：
  - 您的設定中，數據機（位於 `192.168.1.1`）和路由器1（位於 `192.168.0.1`）都充當路由器。這不應阻止基本連接，但將數據機設定為 **橋接模式**（如果可能）可以簡化設定。

- **防火牆**：
  - 路由器1的防火牆可能阻止了您的筆記本電腦。暫時停用任何自訂防火牆規則進行測試。

---

### 步驟 5：必要時重設
如果所有方法都無效，請重設路由器1並重新配置：

- **恢復出廠設定**：
  - 按住路由器1的重設按鈕 10 秒，然後按照您列出的設定重新配置 WAN 和 DHCP。

- **嘗試動態 WAN**：
  - 不要使用靜態 WAN IP，將路由器1的 WAN 設定為 **DHCP**，看看它是否可以自動從數據機獲取 IP。如果這樣可行，則靜態 IP（`192.168.1.101`）可能是問題所在。

---

### 最後提示
- **數據機 DHCP 範圍**：確認 `192.168.1.101` 在數據機的 DHCP 範圍之外以避免衝突。
- **韌體**：檢查路由器1的網頁介面是否有韌體更新。
- **筆記本電腦驅動程序**：確保您的筆記本電腦網絡適配器驅動程序是最新的。

按照這些步驟操作，您應該能找出筆記本電腦無法連接的原因。如果您仍然卡住，請告訴我您在每個步驟中的發現——我將幫助您進一步縮小範圍！
