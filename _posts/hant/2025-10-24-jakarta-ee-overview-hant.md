Jakarta EE 是一套用於開發企業級 Java 應用程式的開源平台。它提供了一系列規範、API 和工具，擴展了核心 Java SE（標準版）的功能，以處理複雜任務，例如 Web 服務、分散式計算、雲原生部署和可擴展的伺服器端應用程式。

### 簡要歷史
- 最初稱為 Java 2 Platform, Enterprise Edition (J2EE)，後來更名為 Java EE，由 Oracle/Sun Microsystems 開發。
- 2017 年，Oracle 將相關技術捐贈給 Eclipse Foundation，導致其重新命名為 Jakarta EE（以印尼雅加達命名，首屆 Eclipse Java 會議在此舉行）。
- 這一轉變使其成為完全開源和社區驅動的項目，促進了更快的創新和更廣泛的採用。

### 主要特點與用途
- **元件**：包含用於 Servlet 的 API、用於資料存取的 JPA（Java Persistence API）、用於管理依賴關係的 CDI（Contexts and Dependency Injection）、用於 RESTful Web 服務的 JAX-RS 等。
- **使用案例**：適用於構建穩健、安全且可移植的企業軟體，例如電子商務平台、金融系統和微服務架構。
- **現狀**：截至 2025 年，Jakarta EE 11 是最新的主要版本，強調雲原生兼容性、JSON 處理和改進的安全性。它受到 Open Liberty、Payara 和 WildFly 等伺服器的支援。
