讓我們深入探討一些實用的 Windows 網路指令，您可以在命令提示字元 (CMD) 中執行這些指令來進行故障排除、收集資訊或管理網路。我將解釋幾個常用的指令，例如 `getmac`，並補充其他可能派上用場的指令。您不需要是技術達人——這些指令一旦上手就相當簡單明瞭。

首先，要使用這些指令，請開啟命令提示字元：按下 `Windows 鍵 + R`，輸入 `cmd`，然後按 Enter。很簡單對吧？

### 1. `getmac` - 查詢 MAC 位址
`getmac` 指令會顯示您網路介面卡的媒體存取控制 (MAC) 位址——這些是您裝置在網路上的唯一識別碼。使用方法如下：

- 輸入 `getmac` 後按 Enter。
- 您會看到每個網路介面卡（例如乙太網路或 Wi-Fi）的 MAC 位址清單。
- 加上 `-v` 參數（`getmac -v`）可啟用詳細模式，提供額外資訊如介面卡名稱和傳輸類型（例如乙太網路或無線網路）。

範例輸出可能如下：
```
實體位址          傳輸名稱
=================== ==========================================================
00-14-22-01-23-45   \Device\Tcpip_{12345678-ABCD-1234-EF56-7890ABCDEF12}
```
「實體位址」就是您的 MAC 位址。這在網路故障排除或設定路由器的 MAC 過濾時非常實用。

### 2. `ipconfig` - 檢查 IP 設定
這是查詢網路資訊的常用指令：
- 輸入 `ipconfig` 後按 Enter，可查看基本資訊如 IP 位址、子網路遮罩和預設閘道。
- 使用 `ipconfig /all` 可取得完整詳細資訊，包括 DNS 伺服器、DHCP 狀態，以及——沒錯——您的 MAC 位址。

這個指令非常適合用來判斷裝置是否正確連線，或是否存在 IP 衝突。

### 3. `ping` - 測試連線能力
想檢查是否能連線到其他裝置或網站嗎？
- 輸入 `ping [位址]`（例如 `ping google.com` 或 `ping 8.8.8.8`）。
- 它會發送幾個封包並告訴您是否收到回覆，以及所需時間（以毫秒為單位）。

如果顯示「要求等候逾時」，表示有東西阻擋了連線——可能是防火牆、故障的伺服器，或是您自己的網路問題。

### 4. `tracert` - 追蹤路由路徑
這是「追蹤路由」的縮寫，會顯示資料傳送到目的地的路徑：
- 輸入 `tracert [位址]`（例如 `tracert google.com`）。
- 它會列出沿途的每個躍點（路由器或伺服器）及其回應時間。

這個指令有助於找出連線變慢或中斷的位置。

### 5. `netstat` - 檢視網路活動
這個指令用於檢查當前網路狀態：
- 輸入 `netstat -a` 可查看所有作用中連線和監聽中的連接埠。
- 加上 `-n` 參數（`netstat -an`）會顯示 IP 位址而非主機名稱——速度更快且更原始。

這就像一扇窗，讓您看到電腦正在與誰通訊——對於偵測可疑活動非常有用。

### 6. `nslookup` - 深入解析 DNS
如果網站無法載入，這個指令可以檢查網域名稱系統 (DNS) 解析：
- 輸入 `nslookup [網域]`（例如 `nslookup google.com`）。
- 它會傳回該網域對應的 IP 位址，顯示您的 DNS 是否正常運作。

### 讓操作更順暢的小技巧
- 如果指令需要更高權限，請以系統管理員身分執行 CMD（在命令提示字元上按右鍵 >「以系統管理員身分執行」）。
- 使用 `|`（管道符號）結合指令進行篩選——例如 `getmac | find "00-14"` 來搜尋特定 MAC 位址。
- 如果遇到困難，輸入 `[指令] /?`（例如 `ipconfig /?`）可取得該指令的說明。
