- Satya Nadella 提到了 Jevons 悖論。值得學習。

- 王寅：人工智能中沒有「智能」，神經網絡中沒有「神經」，機器學習中沒有「學習」，深度學習中沒有「深度」。這個領域真正有效的東西叫做「微積分」。所以我更喜歡把這個領域稱為「可微計算」，而建模的過程稱為「可微編程」。

- 王寅：機器學習確實非常有用，甚至可以說是美麗的理論，因為它只是微積分的改頭換面！這是牛頓、萊布尼茨的古老而偉大的理論，以更簡單、優雅和強大的形式呈現。機器學習基本上是使用微積分來導出和拟合一些函數，而深度學習是拟合更複雜的函數。

- 目前，大型語言模型無法過濾文件語言，如 YAML 或 Python。然而，現實世界中的大量信息都是以這種方式組織的。這意味著我們可以使用文件來訓練大型語言模型。

- 為了訓練大型語言模型，我們可以開發一個系統來找到精確匹配。也許我們可以將 KMP（Knuth-Morris-Pratt）搜索算法與變換器架構結合起來，以增強搜索能力。

- 沒有技術秘密。開源將揭示所有被嚴密保護的秘密。

- 人工智能將影響許多工具，包括間接的工具。有人說他們不需要 Figma 來繪製原型，他們會直接編寫代碼。我認為 Postman 也會類似；人們會直接使用 Python 或其他腳本來調用或測試 API。

- 我們不使用 Postman 或 Figma 的一個原因是，它們的功能無法通過文本生成。它們也缺乏一個命令 + K 快捷鍵來觸發組件替換。

- 用戶界面在人工智能時代成為了一個障礙。為什麼要將 Postman 升級為人工智能測試應用程序，當我們可以直接使用 Python 的 requests 庫或其他編程語言來測試代碼，因為後者將由人工智能驅動？

- 為什麼要將 Figma 升級為人工智能創建 UI，當代碼生成的 UI，由人工智能增強，提供了一種更直接和潛在強大的方法？

- 大型語言模型將首先改變文本相關的應用程序，如 Google、搜索引擎、文本編輯器和寫作工具、Quizlet、Zendesk、DeepL、Medium、WordPress、Trello、Asana、Gmail、GitHub、Goodreads、Duolingo 和 Feedly。

- 相反，大型語言模型不太可能革命化技術，如 Git、Linux、ffmpeg、手機、硬件、瀏覽器、操作系統或語音和視頻通話。這些技術是基於代碼的，它們的代碼不容易由人工智能生成，與 API 測試工具如 Postman 相反。

- 代碼更多的技術很難被人工智能革命化，如 OpenOffice、MySQL、Mozilla Firefox、Chromium、VLC Media Player、Qt Framework、LLVM/Clang 和 GNOME。如果人工智能能幫助製作這些技術，它們就不會被取代。人工智能應該幫助製作更好的技術，為此，人工智能需要更多的計算能力來生成相同數量的代碼。

- 大型語言模型有兩種方式帶來變化：首先，通過改變平台或軟件中的內容或數據，例如在 TikTok 等應用程序中進行內容翻譯；其次，通過直接取代某些軟件或平台，例如 Postman 或 Google 搜索，包括 Google 翻譯。

- 人工智能音頻工具有兩種方式帶來變化：首先，通過改變平台或軟件中的內容或數據，例如為 Audiable 生成有聲書；其次，通過直接取代某些軟件或平台，例如 Sing 歌曲應用程序，因為人工智能現在可以執行人類所做的相同任務，使人們更容易唱歌作為愛好。

- 有幾種方法可以衡量人工智能對當前軟件或平台的影響。一種方法是衡量人工智能可以生成或改進的數據或內容的數量，無論是部分還是完全。另一種方法是衡量人工智能可以編寫或改進的代碼的數量，無論是部分還是完全。這意味著我們使用人工智能生成的內容來改進當前平台。此外，人工智能可以幫助發明新的軟件和平台。

- 將會有三種類型的產品，生成式人工智能產品、使用生成式人工智能產品 API 的產品和其他產品。

- 一個產品想法是使用人工智能從社交平台如 Reddit、GitHub Trending、Twitter Trending、Quora Trending 和 Zhihu Trending 積累即時信息、新聞或更新。用戶可以使用提示自定義鏈接或甚至添加特定的社交帳戶。
