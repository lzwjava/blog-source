- Satya Nadella 提到了 Jevons 悖論，值得學習。

- 王垠：人工智能沒有「智能」，神經網絡沒有「神經」，機器學習沒有「學習」，深度學習沒有「深度」。深度學習裏根本沒有「深度」，真正起作用的是「微積分」。所以我更願意稱這個領域為「可微計算」，而構建模型的過程叫做「可微編程」。

- 王垠：機器學習確實是一個非常有用，甚至可以說很美的理論，因為它本質上就是換了個包裝的微積分！它是牛頓、萊布尼茨那個古老而偉大的理論，以一種更簡單、優雅而強大的形式呈現。機器學習基本上就是用微積分來推導和擬合一些函數，而深度學習就是擬合更複雜的函數。

- 目前，大型語言模型無法根據 YAML 或 Python 等檔案語言進行篩選。然而，現實世界中很大一部分資訊都是這樣組織的。這意味着我們可以用檔案來訓練大型語言模型。

- 為了訓練大型語言模型，我們可以開發一個查找精確匹配項的系統。也許我們可以將 KMP（Knuth-Morris-Pratt）搜尋演算法與 Transformer 架構結合起來，以增強搜尋能力。

- 並沒有什麼技術秘密。開源會揭露所有被嚴密守護的秘密。

- AI 會影響很多工具，包括間接的工具。有人說他們不需要 Figma 來繪製原型，他們會直接編碼。我認為 Postman 也會類似；人們會直接使用 Python 或其他腳本調用或測試 API。

- 我們在 AI 時代不使用 Postman 或 Figma 的一個原因是，它們的功能無法通過文字生成。它們也缺乏 command + K 快捷鍵來觸發組件替換。

- 在 AI 時代，使用者介面正在成為一個障礙。為什麼要升級 Postman 以使其支援 AI 來測試應用程式，當我們可以直接使用 Python 的 requests 函式庫或其他程式語言來測試程式碼時，後者將由 AI 驅動？

- 為什麼要升級 Figma 以使其支援 AI 來創建 UI，當基於程式碼的 UI 生成（由 AI 增強）提供更直接且潛在更強大的方法時？

- 大型語言模型首先會改變與文字相關的應用程式，例如 Google、搜尋引擎、文字編輯器和寫作工具、Quizlet、Zendesk、DeepL、Medium、WordPress、Trello、Asana、Gmail、GitHub、Goodreads、Duolingo 和 Feedly。

- 相反，大型語言模型不太可能徹底改變 Git、Linux、ffmpeg、行動電話、硬體、瀏覽器、作業系統或語音和視訊通話等技術。這些技術以程式碼為中心，它們的程式碼不容易由 AI 生成，不像 Postman 等 API 測試工具。
