在強大的程式碼助手（如 Claude Code）中，典型的**工具呼叫**或**動作**主要圍繞讓它能夠與真實的編程環境互動，並執行人類開發者會完成的任務。

這些工具呼叫可分為以下幾個主要類別：

### 1. 檔案系統與程式碼庫互動

這些功能對於助手理解專案、讀取現有程式碼及進行修改至關重要。

| 工具/動作 | 典型使用場景 |
| :--- | :--- |
| **`read_file`**（或類似 `@filename` 的檔案參考） | 讀取特定檔案的內容以理解其邏輯、依賴關係或上下文。 |
| **`list_directory`** | 獲取檔案結構或目錄中的檔案列表，以識別相關模組或尋找特定檔案。 |
| **`edit_file` / `write_file`** | 實現解決方案、重構、新增功能或修復程式錯誤的核心動作。 |
| **`create_file`** | 撰寫新檔案，例如新的測試檔案、配置檔案或新元件。 |
| **`search_files`** | 在程式碼庫中尋找包含特定字串的所有檔案（例如函數名稱、類別名稱或錯誤訊息）。 |

### 2. 執行與除錯

為了驗證工作成果、修復錯誤並獲取即時反饋，助手需要執行指令。

| 工具/動作 | 典型使用場景 |
| :--- | :--- |
| **`bash` / `run_command`** | 執行 shell 指令，例如運行建置（`npm build`）、運行測試（`pytest`、`npm test`）、程式碼檢查（`eslint`）或執行腳本。 |
| **`code_interpreter`** | 在沙盒環境中運行小段程式碼，以快速計算、測試庫函數或分析小型數據集。 |

### 3. 版本控制（Git）

具備代理能力的編程助手可以透過與 Git 互動來管理開發工作流程。

| 工具/動作 | 典型使用場景 |
| :--- | :--- |
| **`git status`** | 檢查工作目錄的當前狀態。 |
| **`git diff`** | 查看已進行的變更以審查自身工作。 |
| **`git commit`** | 將變更儲存到儲存庫，通常會附帶自動生成的提交訊息。 |
| **`git checkout`** | 切換分支或捨棄未提交的變更。 |
| **`create_pull_request`** | 自動化功能或修復的最後一步。 |

### 4. 外部資訊

為了獲取最新資訊、文件或連接外部服務，會使用專門工具。

| 工具/動作 | 典型使用場景 |
| :--- | :--- |
| **`web_search`** | 查詢最新語法、閱讀官方 API 文件或尋找特定錯誤的疑難排解指南。 |
| **Custom APIs / MCP Servers** | 使用 Model Context Protocol (MCP) 連接外部服務，例如從 Google Drive 獲取數據、與資料庫（PostgreSQL）互動，或向通訊工具（Slack）發送訊息。 |

本質上，這些「工具呼叫」讓大型語言模型（LLM）從文字生成器轉變為完整的**代理**，能夠觀察環境（讀取檔案、執行指令）、思考（規劃工作）並行動（編輯檔案、提交程式碼）。
