Java 的歷史豐富而多面向，自問世以來已歷經重大演變。以下回顧其發展歷程，以及開發者在不同時期如何運用它：

**開端：Sun Microsystems（1990年代初）**

Java 的故事始於 **1991年6月** 的 **Sun Microsystems**，由 **James Gosling** 主導的「**Green**」專案拉開序幕。最初目標是為消費性電子設備（如智慧電視和機頂盒）建立程式語言。這些設備資源有限，需要一種簡單、穩健、可移植且安全的語言。該語言最初以 Gosling 辦公室外的橡樹命名為「**Oak**」。

**關鍵設計原則：**

* **簡單性：** 比 C++ 更易學易用。
* **物件導向：** 遵循現代程式設計典範。
* **穩健性：** 透過自動記憶體管理（垃圾回收）等特性實現可靠性。
* **平台獨立性（「一次編寫，隨處執行」）：** 這是革命性概念。Java 程式碼不直接編譯為特定處理器的機器碼，而是編譯成稱為 **位元組碼** 的中間格式。這種位元組碼在 **Java 虛擬機器（JVM）** 上運行，而 JVM 可在各種作業系統和硬體上實現。
* **安全性：** 設計時包含安全功能，防止惡意程式碼損害系統。

**網路崛起（1990年代中期）：**

**1993年**，Green 團隊意識到他們的語言在蓬勃發展的 **全球資訊網** 中的潛力。他們建立了名為「**WebRunner**」（後更名為「**HotJava**」）的原型網頁瀏覽器，能夠運行嵌入網頁中的小型程式（稱為 **Applet**）。「**Java**」這個名稱於 **1995年** 正式選定。

**開發者應用：**

* **早期網頁開發者：** 使用 Java Applet 為當時主要為靜態的網站添加互動元素和多媒體功能，為網際網路帶來全新動態體驗。

**Java 進軍 Nokia 手機與行動時代（1990年代末至2000年代）：**

隨著手機功能日益複雜，需要一個能運行應用程式和遊戲的平台。**Java Micro Edition (J2ME)** 是專為資源受限設備量身打造的 Java 版本，成為該領域的領先技術。

**開發者應用：**

* **行動遊戲與應用開發者：** 運用 J2ME 為功能型手機（包括許多 **Nokia** 設備）建立龐大的遊戲和應用生態系統。這些應用通常打包為 **JAR（Java 歸檔）** 檔案。這讓開發者能一次編寫遊戲和應用程式，並在各種相容手機上運行，在當時是一大優勢。2002年發布的 Nokia 3410 是早期支援 Java 的 Nokia 手機之一。

**桌面端的 Java（1990年代末至今）：**

「一次編寫，隨處執行」的承諾使 Java 成為開發跨平台桌面應用程式的理想選擇。**Java Standard Edition (Java SE)** 為此提供了必要的函式庫和工具。

**開發者應用：**

* **桌面應用開發者：** 使用單一代碼庫為不同作業系統（Windows、Linux、macOS）建置各種 Java 應用程式。**Swing** 及後來的 **JavaFX** 等技術被用於建立圖形使用者介面。雖然隨著原生平台開發和其他跨平台框架的興起，Java 在通用桌面應用程式的主導地位有所減弱，但在企業應用和開發工具等特定領域仍舉足輕重。

**伺服器端的 Java（1990年代末至今）：**

Java 迅速成為企業應用程式開發和伺服器端程式設計的主導力量。**Java Enterprise Edition (Java EE)**（現為 **Jakarta EE**）提供了一套完整的 API 和框架，用於建置可擴展且穩健的伺服器端應用程式。

**開發者應用：**

* **企業應用開發者：** 使用 Java EE 為大型組織建置複雜的 Web 應用程式、業務邏輯和中介軟體。**Servlets**、**JSP（JavaServer Pages）**、**EJB（Enterprise JavaBeans）**、**Spring** 和 **Hibernate** 等框架變得極受歡迎，簡化了企業級軟體的開發。Java 的可靠性、可擴展性和龐大生態系統，使其成為許多企業的首選。

**Java 與 Android（2000年代至今）：**

**2008年** **Android** 的發布標誌著 Java 歷史的另一重要篇章。最初，Android 的軟體開發套件（SDK）主要使用 **Java** 作為建置應用程式的程式語言。

**開發者應用：**

* **Android 應用開發者：** 運用其 Java 知識為快速成長的 Android 平台建立應用程式。這導致 Java 開發者數量急遽增加，並使 Java 在行動領域得到廣泛應用。雖然 Google 後來推出了 **Kotlin** 作為 Android 開發的官方支援語言，但 Java 仍然是 Android 生態系統的重要組成部分，許多現有應用程式仍使用 Java 或 Java 函式庫開發。

**時代演進：**

自首次發布以來，Java 經歷了無數次更新和改進，新增功能並提升效能。Java 社區過程（JCP）的建立旨在讓社區參與平台的演進。

**關鍵里程碑：**

* **1998年：Java 2** 引入重大增強功能，包括 Swing GUI 工具包和 Collections Framework。
* **2004年：Java 5** 帶來泛型、註解和自動裝箱等主要語言特性。
* **2006年：Sun Microsystems 將 Java 平台大部分開源**，促成了 OpenJDK 專案。
* **2010年：Oracle 收購 Sun Microsystems**，接管 Java 的維護工作。
* 後續版本（Java 6、7、8 及之後版本）持續引入新功能、效能改進，並支援現代開發典範（如 Java 8 引入 Lambda 表達式支援函數式程式設計）。

**現況：**

Java 至今仍是全球最受歡迎且廣泛使用的程式語言之一。它為各種領域的眾多應用程式提供動力，從企業系統和網頁伺服器到 Android 應用程式和嵌入式設備。儘管新的語言和平臺不斷湧現，但 Java 龐大的社區、成熟的生態系統和持續的演進，確保了它在軟體開發領域的持續相關性。如今的開發者將 Java 用於各種任務，利用其穩健的功能和豐富的函式庫來建置可擴展、可靠且跨平台的應用程式。
