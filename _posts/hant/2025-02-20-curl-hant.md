這是一個結合多個 `curl` 範例的 bash 腳本：

```bash
#!/bin/bash

# 基本 GET 請求
echo "基本 GET 請求："
curl http://example.com

# 帶自訂標頭的 GET 請求
echo "\n帶自訂標頭的 GET 請求："
curl -H "User-Agent: MyUserAgent/1.0" http://example.com

# POST 請求
echo "\nPOST 請求："
curl -X POST -d "key1=value1&key2=value2" http://example.com/post

# 帶 JSON 資料的 POST 請求
echo "\n帶 JSON 資料的 POST 請求："
curl -X POST -H "Content-Type: application/json" -d "{\"key1\":\"value1\",\"key2\":\"value2\"}" http://example.com/post

# 使用代理伺服器
echo "\n使用代理伺服器："
curl -x http://proxy.example.com:8080 http://example.com

# 身份驗證
echo "\n基本身份驗證："
curl -u username:password http://example.com/auth

# 下載檔案
echo "\n下載檔案："
curl -O http://example.com/file.zip

# 跟隨重新導向
echo "\n跟隨重新導向："
curl -L http://example.com/redirect

# 將輸出儲存至檔案
echo "\n將輸出儲存至檔案："
curl http://example.com -o output.txt

# 自訂 HTTP 方法
echo "\n自訂 HTTP 方法 (DELETE)："
curl -X DELETE http://example.com/delete/resource

# 詳細輸出
echo "\n詳細輸出："
curl -v http://example.com

# 連線逾時
echo "\n連線逾時："
curl --connect-timeout 5 http://example.com

echo "\n腳本執行完成。"
```

將此腳本儲存為 `.sh` 副檔名，例如 `curl_examples.sh`。如果您在類 Unix 環境或 Windows Subsystem for Linux (WSL) 中執行，請確保將其設為可執行：

```sh
chmod +x curl_examples.sh
```

然後，您可以執行它：

```sh
./curl_examples.sh
```

**Windows 注意事項：**
- 如果您在 Windows Cmd 或 PowerShell 中執行此腳本，您需要使用 `bash` 或 `wsl`（如果您已安裝 WSL），或確保 `curl` 在您的 PATH 中以便直接執行：
  - `bash curl_examples.sh` 或 `wsl ./curl_examples.sh`
- 請記住，此腳本使用 bash 語法，因此除非透過 WSL 或 bash 環境執行，否則無法原生相容於 Windows Command Prompt 或 PowerShell。但 `curl` 指令本身與 Windows 版本的 `curl` 相容。
