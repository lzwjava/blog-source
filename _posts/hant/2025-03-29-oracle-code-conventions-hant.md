## 1. 檔案組織

### 1.1 檔案名稱
- 原始碼檔案必須使用 `.java` 副檔名
- 原始碼檔案名稱必須與檔案中的公開類別名稱完全一致
- 範例：
  - `MyClass.java` 包含名為 `MyClass` 的公開類別
  - `CircleShape.java` 包含名為 `CircleShape` 的公開類別

### 1.2 檔案結構
1. 開頭註解
2. 套件宣告
3. 匯入陳述式
4. 類別與介面宣告

## 2. 格式化與縮排

### 2.1 縮排
- 使用 4 個空格進行縮排（不使用定位字元）
- 續行縮排 8 個空格
- 大括號應垂直對齊

### 2.2 行長度
- 建議最大行長度為 80 個字元
- 對於較長的陳述式，應在更高邏輯層級處斷行

### 2.3 換行
- 在運算子之前換行
- 新行應與同層級表達式的開頭對齊

## 3. 註解

### 3.1 檔案註解
- 每個原始碼檔案應以註解區塊開頭：
  ```java
  /*
   * 類別名稱
   * 
   * 版本資訊
   * 
   * 日期
   * 
   * 版權聲明
   */
  ```

### 3.2 實作註解
- 多行註解使用 `/* */`
- 單行註解使用 `//`
- 註解應解釋「為什麼」，而非「是什麼」

### 3.3 文件註解
- 對類別、介面、方法使用 Javadoc 風格的註解
- 包含：
  - 簡要描述
  - 方法參數的 `@param`
  - 回傳值的 `@return`
  - 異常的 `@throws`

## 4. 宣告

### 4.1 每行數量
- 每行一個宣告
- 建議：
  ```java
  int level;        // 正確
  int size;         // 正確
  
  // 避免：
  int level, size;  // 不建議
  ```

### 4.2 初始化
- 盡可能在宣告時初始化變數
- 將相關宣告分組在一起

## 5. 陳述式

### 5.1 簡單陳述式
- 每行一個陳述式
- 逗號後使用空格
- 運算子周圍使用空格

### 5.2 複合陳述式
- 即使單行區塊，也應在 `if`、`else`、`for`、`while`、`do` 陳述式中使用大括號

### 5.3 回傳陳述式
- 優先使用明確回傳
- 在 `return` 後避免不必要的 `else`

## 6. 命名慣例

### 6.1 套件名稱
- 全部小寫
- 唯一前綴，通常是反轉的公司網域
- 範例：`com.company.project`

### 6.2 類別名稱
- 大駝峰式命名
- 名詞或名詞片語
- 範例：`DataProcessor`、`HttpClient`

### 6.3 介面名稱
- 類似類別名稱
- 考慮描述行為
- 範例：`Runnable`、`Serializable`

### 6.4 方法名稱
- 小駝峰式命名
- 動詞或動詞片語
- 範例：`calculateTotal()`、`sendMessage()`

### 6.5 變數名稱
- 小駝峰式命名
- 有意義且具描述性
- 除臨時變數外，避免使用單字母名稱

### 6.6 常數名稱
- 全大寫並使用底線分隔
- 範例：`MAX_VALUE`、`COLOR_RED`

## 7. 程式設計實踐

### 7.1 可見性
- 使用最低可能的可見性
- 優先使用 `private` 而非 `public`
- 對類別欄位使用存取方法

### 7.2 變數作用域
- 在最小可能的作用域中宣告變數
- 在接近首次使用處初始化變數

### 7.3 異常處理
- 異常中應始終包含訊息
- 捕獲特定異常
- 適當使用 try-catch-finally

## 8. 最佳實踐

### 8.1 避免魔術數字
- 使用命名常數而非字面數字
- 提高程式碼可讀性與可維護性

### 8.2 優先使用不可變性
- 對不變的變數使用 `final`
- 盡可能建立不可變類別

## 結論
這些慣例旨在提高 Java 專案的程式碼可讀性、可維護性與一致性。
