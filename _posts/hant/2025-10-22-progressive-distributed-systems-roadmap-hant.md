### **第一階段：基礎知識——理解「分散式」的含義**

在深入學習 Paxos 或 Raft 之前，必須深刻理解*分散式*的含義以及會出現哪些問題。

**核心概念**

* 什麼是分散式系統（與集中式系統的區別）？
* 網絡通訊：延遲、頻寬、可靠性。
* 故障類型：崩潰、網絡分區、拜占庭行為。
* 一致性、可用性、分區容錯性（CAP 定理）。

**推薦閱讀**

* Jeff Hodges 的《給新手的分散式系統筆記》
* Martin Kleppmann 的《數據密集型應用系統設計》（前 5 章）
* 《Google SRE 手冊》（第 1–2 章，關於可靠性與故障思維）

**實作練習**

* 使用 Python 或 Go 編寫一個簡單的 TCP socket 客戶端-伺服器程式。
* 使用 Docker 模擬網絡分區（`tc netem`）。

### **第二階段：通訊與協調**

現在專注於*節點如何在不可靠的網絡中通訊與協調*。

**核心主題**

* RPC（遠程過程呼叫）與訊息傳遞。
* 序列化格式（JSON、Protobuf、Avro）。
* 分散式系統中的時間：Lamport 時間戳、向量時鐘。
* 領導選舉（Bully 算法、基於 Raft 的選舉）。
* Gossip 與流行病協議。

**專案實作**

* 實現一個簡易訊息佇列或聊天系統。
* 加入 Lamport 時間戳來排序事件。

**推薦資源**

* Leslie Lamport 的《分散式系統中的事件排序與時鐘》（1978）
* MIT 6.824 課程第 1–3 講影片。

### **第三階段：數據分區與複製**

這是實現可擴展性與容錯能力的核心。

**概念**

* 數據分片 / 分區：基於雜湊、基於範圍。
* 複製模型：主從複製、多主複製、基於法定人數。
* 複製延遲與一致性級別（強一致性、最終一致性、因果一致性）。

**實作練習**

* 使用雜湊分區建立一個簡單的分散式鍵值儲存系統。
* 模擬動態添加/移除節點。

**學習現有系統**

* Dynamo（Amazon）
* Cassandra
* MongoDB 分片
* 一致性雜湊

**論文閱讀**

* 《Dynamo：Amazon 的高可用鍵值儲存系統》（2007）

### **第四階段：共識與協調（Paxos、Raft、ZAB）**

共識是分散式系統的**核心**——在不可靠的節點間就某個值達成一致。

**核心協議**

* Paxos（與 Multi-Paxos）
* Raft（較易理解）
* Viewstamped Replication
* ZAB（ZooKeeper 使用）

**資源**

* Lamport 的《Paxos 簡化版》
* Raft 論文《尋找易理解的共識算法》
* MIT 6.824 的 Raft 實驗

**實作練習**

* 使用 Go 或 Python 實現 Raft。
* 使用 etcd 或 ZooKeeper 來協調微服務。

### **第五階段：容錯、恢復與成員管理**

**專案實作**
