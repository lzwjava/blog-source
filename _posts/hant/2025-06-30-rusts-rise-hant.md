Rust 是一種現代系統程式語言，憑藉其對安全性、效能與並行處理的重點關注而廣受歡迎。它的發展歷程是一段充滿創新與社群驅動的精彩旅程，主要由 Mozilla 發起並培育。

### 早期開端（2006-2009 年）

Rust 的起源可追溯至 **2006 年**，由 Mozilla 開發者 **Graydon Hoare** 的個人專案開始。由於對軟體頻繁崩潰（尤其因大樓電梯故障）感到挫折，Hoare 決心創建一種能克服 C 與 C++ 等語言常見記憶體管理與配置問題的語言。他的目標是打造既具備傳統系統語言底層控制力與高效能，又能避免常見記憶體錯誤與安全漏洞的程式語言。據說「Rust」這名稱的靈感來自一種「為生存而過度設計」的真菌群，呼應了這門語言對健壯性的重視。

在最初幾年，Rust 由 Hoare 利用業餘時間開發，且主要僅在 Mozilla 內部流傳。早期編譯器採用 OCaml 編寫，語言特性探索了顯式物件導向程式設計，以及用於追蹤變數狀態的型別狀態系統。

### Mozilla 資助與開源化（2009-2012 年）

**2009 年**，Mozilla 正式認可 Rust 的潛力並開始資助此專案。像 Brendan Eich 等高階主管看到了運用 Rust 打造更安全網頁瀏覽器引擎的契機。這使得包括 Patrick Walton、Niko Matsakis 與 Felix Klock 在內的一批工程師加入 Hoare 的團隊。

此階段標誌著重大轉變：
* **自舉編譯器**：基於 LLVM，團隊開始以 Rust 自身重寫 Rust 編譯器，這是實現語言獨立性與成熟度的關鍵一步。
* **所有權系統的引入**：約在 **2010 年**左右，Rust 所有權系統的基礎概念逐漸成形，此系統是實現無垃圾回收機制下記憶體安全保證的核心。

**2010 年**，Rust 以開源專案形式釋出，向更廣泛的社群開放其開發過程。

### 演進與成熟期（2012-2015 年）

在 1.0 版釋出前的數年間，語言經歷了大幅甚至可謂激進的變革。開發團隊致力於精煉 Rust 的核心功能並確保其穩定性。關鍵進展包括：
* **移除型別狀態與垃圾回收器**：初始的型別狀態系統被移除，且至關重要的是，實驗性垃圾回收器於 **2013 年**被逐步淘汰，轉而支持演進中的所有權系統。此決策對確立 Rust 作為高效能、零成本抽象語言的身份至關重要。
* **記憶體管理機制的鞏固**：所有權系統連同借用與生命週期機制，逐步擴充並鞏固，以在編譯時期防堵記憶體相關錯誤。
* **多元語言影響**：Rust 的設計汲取了多種程式設計範式的靈感，借鑒了 C++（底層效能）、腳本語言（如 Cargo 套件管理）與函數式程式設計（型別系統）等概念。
* **為 1.0 版聚焦穩定性**：此期間團隊全力專注於敲定語言功能，為提供向後相容性的穩定 1.0 版釋出做準備。

### Rust 1.0 與後續發展（2015 年至今）

**2015 年 5 月 15 日**，在 Mozilla 正式資助六年後，首個穩定版本 **Rust 1.0** 正式發布。這是歷史性的一刻，標誌著 Rust 已準備好投入生產環境，並展現了對穩定性的承諾。

自 1.0 版以來，Rust 呈現以下發展：
* **社群快速成長**：編譯器迅速累積了數千名貢獻者，套件管理網站 Crates.io 上的第三方函式庫數量也大幅增長。
* **廣泛業界採用**：亞馬遜雲端運算（AWS）、微軟、Google 與 Meta 等大型企業紛紛採用 Rust，應用於高效能後端系統、作業系統元件、網路服務與區塊鏈基礎架構。Mozilla 自身也使用 Rust 重寫了 Firefox 引擎的部份組件（如 CSS 引擎 Stylo），從而實現效能提升。
* **Rust 基金會成立**：**2021 年**，Rust 基金會正式成立，使語言治理制度化，確保其持續發展並在 Mozilla 起源之外保持獨立性。
* **透過「版本」持續演進**：Rust 運用「版本」機制來管理重大語言變更，同時不破壞現有程式碼，使語言能在維持向後相容性的前提下持續演化。

時至今日，Rust 以其媲美 C/C++ 的效能，結合無與倫比的記憶體安全與並行處理保證，而備受認可，成為系統程式設計、網頁開發（尤其結合 WebAssembly）、嵌入式系統等領域中極具價值的語言。
