作為一個使用 Emacs 半年的用戶，我簡直無法想像自己以前如何編碼，把手從標準位置移開去點擊滑鼠或按箭頭鍵，這讓人覺得非常繁瑣和難以忍受。現在，當我告訴我的朋友我設定了 Alt+P 和 Alt+N 快捷鍵來快速在 XML 文件和圖形佈局之間切換，他們的回應只是「好吧」，暗示使用滑鼠切換也可以。對我來說，這簡直是噩夢；這樣子太慢了！如果你是 Emacs 用戶，你會理解的...

這篇文章描述了構建高效 Eclipse 「編輯」環境的簡單技巧。基本上，你的手可以保持在標準位置，讓你以最大效率編碼！

最重要的是安裝 Emacs+ 插件。參見「Emacs+: Eclipse 中的 Emacs 體驗」。

要有效利用代碼助手，你需要讓它能由任何字符觸發，並防止在按下空格或 = 時自動完成。我建議從 CSDN 下載這個 jar 文件。有了它，加上快速的 Google 搜索，你可以輕鬆導入套件。

接下來，我們來自定義一些快捷鍵：

1）將 Alt+P 綁定到「上一個子標籤」，將 Alt+N 綁定到「下一個子標籤」。

子標籤是編輯器下方的標籤欄，例如在編輯 XML 文件時的「圖形佈局」和「XML」標籤。這樣你就可以即時查看佈局。

2）將 Ctrl+C, Ctrl+C 綁定到「運行」。

這是從 sbcl 的配置中複製過來的。預設是 Ctrl+F11，這樣對於頻繁使用的快捷鍵來說太遠了，讓 Emacs 用戶感到不適！我愚蠢地按了Ctrl+F11幾天才改變它。

3）將 Ctrl+X, Ctrl+O 綁定到「下一個視圖」。在 Windows 和編輯文本時。

這樣你就可以從編輯器即時跳到控制台，當編寫 Java 代碼時。

4）將 Ctrl+X, O 綁定到「下一個編輯器」。在 Windows 和編輯文本時。

這樣你就可以快速在 Java 文件之間切換。

5）將 Ctrl+Q 綁定到「快速修復」。

這樣，當你輸入 @string/xx，並且光標在 xx 上時，按下 Ctrl+Q 然後 Enter 就會即時跳轉到 string.xml，並且光標位於 `<string name="xx">TODO</string>` 中的 TODO。

6）將 Ctrl+Shift+W 綁定到「關閉」（在窗口中）並移除原始綁定（關閉所有）。
原始的關閉快捷鍵是 Ctrl+W，這與我們在瀏覽器、聊天框和文件資源管理器中的習慣一致。然而，它與 Emacs 的剪切命令發生衝突。實際上，按住 Ctrl+Shift+W 一秒鐘可以關閉許多文件。因此，將 Ctrl+Shift+W 從「關閉所有」改為「關閉」不會失去任何東西。

有一個問題：安裝 Emacs+ 後，編碼時出現代碼助手，按上下箭頭鍵不會選擇代碼助手的候選項；而會上下移動正在編輯的代碼。預設是用 F2 激活它。激活代碼助手會將焦點放在代碼助手上，但然後你只能使用上下箭頭鍵選擇。如果我們能使用 Ctrl+P 和 Ctrl+N，編碼將會非常快！然而，問題是安裝了 Emacs+ 插件後，儘管 Eclipse 變得更像 Emacs，但這個功能丟失了。在清潔的 Eclipse 中，選擇 Emacs（而不是 Emacs+）鍵盤佈局，你可以在代碼助手出現時使用 Ctrl+N 和 Ctrl+P 選擇完成項。有人在 Stack Overflow 上問過這個問題，但還沒有答案。

如果我們能使用 Ctrl+P 和 Ctrl+N 選擇完成項，那將會非常棒！
