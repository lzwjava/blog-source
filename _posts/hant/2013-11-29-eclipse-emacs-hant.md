作為一個使用了半年的 Emacs 用戶，我簡直無法想像自己以前是怎麼編寫程式的，將手從標準位置移開來點擊滑鼠或按下箭頭鍵，感到疏忙和難受。現在當我告訴我的朋友我設置了 Alt+P 和 Alt+N 快捷鍵以快速在 XML 文件和圖形佈局之間切換時，他們的回應只是「好吧」，暗示使用滑鼠切換也是可以的。

對我來說，這是噩夢；這 simply 不夠快！如果你是一個 Emacs 用戶，你會明白的...

這篇文章描述了一些簡單的技術，用來構建一個快速的 Eclipse「編輯」環境。基本上，你的雙手可以保持在標準位置，允許你以最大效率編寫程式碼！

最重要的事是安裝 Emacs+ 插件。請參閱「Emacs+: Eclipse 中的 Emacs 體驗」。

要充分利用程式碼助手，您需要啟用它以由任何字符觸發，並防止在按下空格或 = 時自動完成。我建議從 CSDN 下載這個 jar 文件。有了它，和快速搜索 Google，你可以在短時間內匯入套件。

接下來，讓我們自定義一些快捷鍵：

1）將 Alt+P 綁定到「上一個子選項卡」，並將 Alt+N 綁定到「下一個子選項卡」。

子選項卡是編輯器下方的選項卡圖，例如在編輯 XML 文件時的「圖形佈局」和「XML」選項卡。這允許你立即查看佈局。

2）將 Ctrl+C, Ctrl+C 綁定到「運行」。

這是從 sbcl 的配置中複製的。默認是 Ctrl+F11，對於這樣一個經常使用的快捷鍵來說，這太遠了，讓 Emacs 用戶感到不適！我愚蠢地按了幾天的 Ctrl+F11 才改變它。

3）將 Ctrl+X, Ctrl+O 綁定到「下一個視圖」。在 Windows 和編輯文字中。

這允許你在編寫 Java 程式碼時，立即從編輯器跳轉到控制台。

4）將 Ctrl+X, O 綁定到「下一個編輯器」。在 Windows 和編輯文字中。

這允許你快速在 Java 文件之間切換。

5）將 Ctrl+Q 綁定到「快速修復」。

這樣，當你輸入 `@string/xx`，並將光標放在 `xx` 上時，按下 Ctrl+Q 然後 Enter，會立即跳轉到 `string.xml`，光標位於 `<string name=\"xx\">TODO</string>` 中的 `TODO`。

6）將 Ctrl+Shift+W 綁定到「關閉」（在 Windows 中）並移除原始綁定（關閉全部）。
原始的關閉快捷鍵是 Ctrl+W，這與我們在瀏覽器、聊天框和文件資料夾中的習慣一致。然而，它與 Emacs 的剪下命令衝突。實際上，按下 Ctrl+Shift+W 可以在幾秒內關閉許多文件。因此，將 Ctrl+Shift+W 從「關閉全部」更改為「關閉」不會失去任何東西。

有一個問題：安裝 Emacs+ 之後，在編輯程式碼並出現程式碼助手時，按上下箭頭鍵不會選擇程式碼助手的候選項；相反，它會在正在編輯的程式碼中上下移動。默認情況下是通過 F2 激活它。激活程式碼助手使焦點轉到程式碼助手，然後你只能使用上下箭頭鍵來選擇。如果我們可以使用 Ctrl+P 和 Ctrl+N，編寫程式碼將會非常快！但問題是，安裝 Emacs+ 插件後，當 Eclipse 變得更像 Emacs 時，這個功能就消失了。在乾淨的 Eclipse 中，選擇 Emacs（不是 Emacs+Scheme）鍵盤佈局，你可以使用 Ctrl+N 和 Ctrl+P 選擇完成項目，當程式碼助手出現時。有人在 Stack Overflow 上詢問過這個問題，但還沒有答案。

如果我們能夠使用 Ctrl+P 和 Ctrl+N 來選擇完成項，那將會非常棒！
