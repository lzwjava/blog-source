GitHub 的**個人存取權杖 (PAT)** 就像是您自行生成的密碼，用於安全存取 GitHub 的 API 和資源。它之所以被廣泛使用，是因為比起在各處輸入 GitHub 密碼，它更靈活且更安全。

以下是**明智使用**它的方法及其**主要功能**：

---

## PAT 的主要功能

1. **Git 指令的身份驗證**
   在推送/拉取時，您可以使用 PAT 來代替輸入 GitHub 用戶名和密碼。這尤其重要，因為 GitHub 已禁用了對 HTTPS Git 操作的密碼驗證。

2. **API 存取**
   如果您編寫的腳本、工具或 CI/CD 流水線需要呼叫 GitHub API，PAT 可讓您安全地進行身份驗證並發出請求（例如創建 issue、獲取儲存庫或管理工作流）。

3. **具有範圍的存取控制**
   您可以精確決定權杖的權限（例如，對儲存庫的唯讀存取，或用於推送代碼的寫入存取）。這有助於在權杖洩露時將損害降至最低。

4. **自動化與整合**
   像 GitHub Actions、Jenkins 或本地腳本等工具可以使用 PAT 來獲取依賴項、發布套件或更新儲存庫。

5. **SSH 的替代方案**
   有些人更喜歡使用 PAT，而不是設置 SSH 金鑰——在腳本或環境之間共享更簡單。

---

## 如何明智地使用 PAT

* **限制範圍**：僅授予您實際需要的權限。例如，如果您只需要拉取私有儲存庫，則不要給予寫入權限。
* **設定過期時間**：對於大多數任務，請使用短期權杖（例如 30 或 90 天）。定期更新它們，而不是永久保留一個。
* **安全儲存**：不要將您的 PAT 硬編碼在腳本中或將其推送到 GitHub。請使用環境變數、`.gitignore` 檔案或密碼管理器（例如 GitHub Actions Secrets、1Password、AWS Secrets Manager）。
* **定期輪換**：撤銷您不再使用的舊權杖。這有助於保持您的安全衛生清潔。
* **使用細粒度權杖（較新）**：GitHub 現在推薦使用細粒度 PAT，您可以逐個儲存庫控制存取，而不是廣泛的全局存取。
* **日常工作中優先使用 SSH**：PAT 最適合用於自動化和 API 使用。對於您自己的筆記本電腦編碼，SSH 金鑰通常更安全且長期來看更便捷。
