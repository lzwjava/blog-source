- **在獲取代碼庫存取權之前**，請同事要 `pom.xml` 文件並嘗試下載 Nexus 依賴項。

- 沒有實際代碼但有 Nexus 圖書館，我們可以實際反編譯 JAR 或實驗圖書館。我們可以做很多事情。跳出思維框架。

- 更喜歡在 Sonatype Nexus Repository 的 `settings.yaml` 中 **配置** 使用者令牌。

- 不要等待 ServiceNow 要求。在擁有存取權之前，考慮你可以做什麼。

- 許多工作可以在開始工作之前完成。可以提前熟悉幾乎所有事情。一切都有開源替代代碼或材料。

- 工作通常涉及特定設置、稍微不同的代碼邏輯以及存取權或密碼來完成工作。

- **預見** 專案的下一階段。考慮當前專案將如何進展以及用戶將需要或期望什麼。

- 考慮所有可能的方法來完成工作、取得進展，並在獲得存取權或其他人提供的材料之前做好準備。

- 不要滿足於現狀；完成工作並盡可能自動化。

- 使用 Python 的 `requests` 圖書館來替代 Postman。

- Windows 和 PowerShell 也是可行的選項。常用的命令僅限於幾十個。學好它們。

- 考慮工程師如何在這個專案上工作 5 或 10 年，並考慮如何快速實現相似的效率。

- 使用筆記和日誌目錄。仔細複製控制台或前端頁面的日誌以進行詳細分析。

- 考慮如何將這些知識傳達給新加入團隊的成員。

- 考慮如果每個人都盡力而為，我們有 50 位這樣的工程師，我們可以實現什麼。

- 可以從內部 Nexus 存儲庫輕鬆下載的圖書館可以手動下載並放置在 `.m2` 存儲庫中。

- 讀取 `pom.xml` 文件以確定 Maven 編譯目標和專案所需的 JDK 版本。

- 而不是搜索 Jira 或 Confluence，直接使用員工 ID 來查找他們的頁面並了解他們的活動。

- 如果我們遇到錯誤，例如 `npm run build` 在大型構建管道中有問題，我們可以單獨運行它以確保它在運行整個管道之前工作。這使得調試循環更快更小。
