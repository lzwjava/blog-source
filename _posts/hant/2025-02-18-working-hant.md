- 在獲取代碼庫的存取權之前，請同事要取 `pom.xml` 文件並嘗試下載 Nexus 依賴項。

- 即使沒有實際的代碼，但有 Nexus 圖書館，我們可以實際反編譯 JAR 或與圖書館進行實驗。我們可以做很多事情。想法要超出框架。

- 最好在 Sonatype Nexus Repository 中的 `settings.yaml` 中配置使用者令牌。

- 不要等待存取請求票據的批准。在你擁有存取權之前，考慮你可以做什麼。

- 許多工作可以在你開始工作之前完成。可以提前熟悉幾乎所有的東西。一切都有開源的替代代碼或材料。

- 工作通常涉及特定的設置、稍微不同的代碼邏輯以及存取權或密碼來完成工作。

- 預見項目的下一階段。考慮當前項目如何進展以及用戶需要或期望什麼。

- 考慮所有可能的方法來完成工作、取得進展，並在獲得存取權或他人提供的材料之前做好準備。

- 不要滿足於現狀；完成工作並自動化盡可能多的工作。

- 使用 Python 的 `requests` 圖書館來替代 Postman。

- Windows 和 PowerShell 也是可行的選項。常用的命令僅限於幾十個。學好它們。

- 考慮工程師如何在這個項目上工作 5 或 10 年，並考慮如何快速實現相似的效率。

- 使用筆記和日誌目錄。仔細從控制台或前端頁面複製日誌以進行詳細分析。

- 考慮如何將這些知識傳達給新加入的團隊成員。

- 考慮如果每個人都盡力而為，我們有 50 位這樣的工程師，我們能夠達成什麼。

- 可以從內部 Nexus 存儲庫輕鬆下載的圖書館可以手動下載並放置在 `.m2` 存儲庫中。

- 讀取 `pom.xml` 文件以確定 Maven 編譯目標和項目所需的 JDK 版本。

- 而不是在 Jira 或 Confluence 中搜索，直接構建 URL 到個人資料頁面並定位頁面。

- 如果我們遇到錯誤，例如 `npm run build` 在大型構建管道中有問題，我們可以單獨運行它以確保它在運行整個管道之前工作。這使得調試循環更快且更小。

- 你職業生涯中會反覆遇到這些問題或類似的錯誤，所以確保你記錄它們。即使在 AI 時代，這種做法仍然有用。有時候，大型語言模型提供的答案並不完全準確，需要你驗證它們。
