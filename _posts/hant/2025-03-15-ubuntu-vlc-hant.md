若要在您的 Ubuntu 系統上安裝 VLC 媒體播放器，您可以使用終端機（指令行）或 Ubuntu 軟體中心（圖形介面）。以下是兩種方法的詳細說明。

---

### **方法一：使用終端機安裝 VLC**

此方法快速高效，特別適合熟悉指令行的使用者。請按照以下步驟操作：

1. **開啟終端機**  
   - 在 Ubuntu 應用程式選單中搜尋「Terminal」，或按 `Ctrl + Alt + T` 快捷鍵開啟。

2. **更新套件清單**  
   - 安裝軟體前，最好先確保您的套件清單是最新的。執行此指令：  
     ```bash
     sudo apt update
     ```  
   - 出現提示時輸入密碼。此指令會重新整理可用軟體套件清單。

3. **安裝 VLC**  
   - 套件清單更新後，執行以下指令安裝 VLC：  
     ```bash
     sudo apt install vlc
     ```  
   - 您需要再次輸入密碼。系統可能會要求您輸入 `y` 並按 Enter 鍵確認安裝。這將下載並安裝 VLC 及其必要依賴項。

4. **驗證安裝**  
   - 安裝完成後，可透過輸入以下指令檢查 VLC 是否成功安裝：  
     ```bash
     vlc
     ```  
   - 這應該會啟動 VLC 媒體播放器。若成功開啟，表示安裝成功。  
   - 或者，您可以執行以下指令檢查 VLC 版本：  
     ```bash
     vlc --version
     ```  
   - 這將顯示類似「VLC media player 3.0.11.1 Vetinari」的資訊（版本號可能有所不同）。

5. **選項：測試 VLC**  
   - 為確保 VLC 運作正常，請嘗試播放媒體檔案。例如，若桌面上有 MP4 檔案，可對其右擊並選擇「使用 VLC 媒體播放器開啟」。若成功播放，表示 VLC 功能完整。

---

### **方法二：使用 Ubuntu 軟體中心安裝 VLC**

若您偏好圖形介面而非終端機，請使用 Ubuntu 軟體中心：

1. **開啟 Ubuntu 軟體中心**  
   - 在應用程式選單中搜尋「Ubuntu Software」並點擊開啟。

2. **搜尋 VLC**  
   - 在軟體中心中，點擊搜尋圖示（通常是放大鏡），並在搜尋欄中輸入「VLC」。

3. **安裝 VLC**  
   - 在搜尋結果中找到「VLC media player」，點擊後再點擊「Install」按鈕。您可能需要輸入密碼以授權安裝。

4. **啟動 VLC**  
   - 安裝完成後，您可以直接從軟體中心點擊「Open」（若可用）啟動 VLC，或在應用程式選單中搜尋「VLC」來啟動。

---

### **補充說明**

- **保持 VLC 更新**  
   - 若您使用終端機安裝 VLC，可定期執行以下指令來保持更新：  
     ```bash
     sudo apt update
     sudo apt upgrade
     ```  
   - 這會更新 VLC 及其他系統套件。若使用軟體中心，更新通常會透過相同介面管理。

- **外掛與編解碼器**  
   - 透過 `sudo apt install vlc` 進行的預設 VLC 安裝已包含基本外掛（如 `vlc-plugin-base`）和編解碼器，因此應能直接播放大多數常見媒體格式（例如 MP4、AVI）。若遇到特定檔案類型問題，後續可安裝額外編解碼器，例如使用以下指令安裝 `ubuntu-restricted-extras`：  
     ```bash
     sudo apt install ubuntu-restricted-extras
     ```

- **透過 Snap 替代安裝**  
   - VLC 也提供 Snap 套件版本。若偏好此方法，請執行：  
     ```bash
     sudo snap install vlc
     ```  
   - 但對多數使用者而言，建議使用 `apt` 方法，因其與系統整合度較佳。

---

透過任一種方法，您都能在 Ubuntu 系統上安裝並使用 VLC。終端機方法對指令行使用者更快速，而軟體中心則適合偏好圖形化操作的使用者。祝您媒體播放愉快！

若您在 Ubuntu 的 VLC 中播放 MKV 檔案時遇到錯誤——特別是關於缺少 MPEG-4 AAC 和 H.265 解碼器的訊息——請按照以下步驟解決。此問題通常因 Ubuntu 基於授權限制未預設包含某些專有編解碼器所致。安裝額外的編解碼器套件應能解決問題。

---

### **解決方案**

#### **步驟一：安裝 `ubuntu-restricted-extras` 套件**
此套件提供多種編解碼器支援，包括 MPEG-4 AAC（音訊）和 H.265（HEVC）視訊，這些很可能在您當前的 VLC 安裝中缺失。

1. **開啟終端機**  
   - 在 Ubuntu 應用程式選單中搜尋「Terminal」或按 `Ctrl + Alt + T`。

2. **更新套件清單**  
   - 執行此指令以確保系統套件清單是最新的：  
     ```bash
     sudo apt update
     ```

3. **安裝 `ubuntu-restricted-extras`**  
   - 執行以下指令：  
     ```bash
     sudo apt install ubuntu-restricted-extras
     ```  
   - 您可能需要輸入密碼。安裝過程中，系統可能會提示您接受某些元件的最終使用者授權協議（EULA）——請按照螢幕指示同意並繼續。

4. **重新啟動 VLC**  
   - 若 VLC 已開啟，請先關閉後重新開啟，然後再次嘗試播放 MKV 檔案。

---

#### **步驟二：安裝額外編解碼器套件（若需要）**
若完成步驟一後錯誤仍存在，請安裝提供 H.265 及其他編解碼器額外支援的特定套件。

1. **安裝 `libde265-0` 和 `libavcodec-extra`**  
   - 執行此指令以安裝 H.265 解碼庫及額外編解碼器支援：  
     ```bash
     sudo apt install libde265-0 libavcodec-extra
     ```

2. **重新啟動 VLC**  
   - 關閉並重新開啟 VLC，然後再次嘗試播放 MKV 檔案。

---

#### **步驟三：額外疑難排解（若需要）**
若問題仍未解決，請嘗試以下額外步驟：

- **檢查 VLC 版本**  
  - 確保您使用的是最新版 VLC。透過以下指令檢查當前版本：  
    ```bash
    vlc --version
    ```  
  - 若版本過舊，可透過執行以下指令更新：  
    ```bash
    sudo apt update
    sudo apt upgrade vlc
    ```

- **測試其他 MKV 檔案**  
  - 播放不同的 MKV 檔案，確認問題是否僅限於單一檔案。若其他檔案可正常播放，則有問題的檔案可能已損毀或使用了不支援的編碼。

- **查看 VLC 日誌獲取詳細資訊**  
  - 進階疑難排解可透過終端機以詳細模式執行 VLC：  
    ```bash
    vlc --verbose 2
    ```  
  - 嘗試播放檔案並查看有關缺失編解碼器的詳細錯誤訊息。

---

### **補充說明**
- **問題成因**：MKV 是一種容器格式，可使用多種編解碼器，例如 MPEG-4 AAC 用於音訊、H.265 用於視訊。由於專有軟體的法律限制，Ubuntu 的預設 VLC 安裝可能缺少這些編解碼器。
- **硬體加速**：若您的系統支援 H.265 硬體解碼（例如透過現代 GPU），請確保顯示驅動程式已更新。但此錯誤提示缺少軟體解碼器，因此硬體加速可能非主要問題。
- **後續步驟**：若以上方法均無效，考慮重新安裝 VLC 或在 Ubuntu 或 VLC 論壇尋求協助，並提供您的系統詳情及特定 MKV 檔案資訊。

---

按照這些步驟操作，您應能修復缺失解碼器的錯誤，並成功在 VLC 中播放 MKV 檔案。請從步驟一開始，這是最直接且最可能解決問題的方法。
