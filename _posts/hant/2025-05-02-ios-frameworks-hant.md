除了 UIKit，iOS 系統還提供了許多其他核心框架，開發者通常會運用這些框架來建置應用程式。這些框架處理的範疇遠不止於使用者介面元素。以下是一些最常用的框架：

1.  **Foundation：** 這是基礎框架，提供基本資料型別（如字串、數字、陣列、字典）、集合型別、作業系統服務和核心程式設計模式。它是許多其他框架的基石，幾乎總是會被導入 iOS 專案中。它包含如用於網路請求的 `URLSession`、用於檔案系統操作的 `FileManager`，以及用於管理應用程式內通知的 `NotificationCenter` 等核心元件。

2.  **Core Data：** 此框架提供了一個強大且靈活的方式來管理應用程式中的模型層物件。它允許您持久化資料、管理物件圖譜並處理資料同步。雖然常被描述為資料庫技術，但它更準確地說是一個物件圖譜管理框架，可以使用不同的持久化儲存方式，例如 SQLite、二進位檔案或記憶體儲存。

3.  **Core Animation：** 此框架用於建立流暢、高效能的動畫和視覺效果。它與 UIKit（或在 macOS 上的 AppKit）協同工作來渲染動畫內容。您可以使用它來為視圖、圖層和其他圖形元素添加動畫，從而建立複雜的過場和效果，而無需直接操作像素。

4.  **Core Graphics：** 也稱為 Quartz 2D，這是一個基於 C 語言的繪圖引擎，提供低階的 2D 渲染功能。它允許您高精度地繪製點、線、路徑、形狀和圖像。雖然 UIKit 提供了更高階的繪圖抽象，但 Core Graphics 常用於自訂繪圖操作、建立漸層、管理繪圖情境以及直接處理圖形基元。

5.  **AVFoundation：** 此框架提供了一套全面的工具，用於處理基於時間的視聽媒體。您可以使用它來播放、錄製和編輯音訊與影片。它讓您能夠控制相機、麥克風、播放引擎和媒體資產，對於涉及多媒體內容的應用程式至關重要。

6.  **Core Location：** 此框架允許您的應用程式存取裝置上的定位服務，例如使用者當前的的地理位置、海拔和航向。它可以使用 GPS、Wi-Fi 和行動網路訊號來確定位置，並支援區域監控和地理圍欄等功能。

7.  **UserNotifications：** 此框架用於排程和處理本地與遠端通知。它允許您的應用程式在使用者未主動使用應用時，也能通知他們重要事件，例如顯示警示、播放聲音或更新應用程式圖示上的標記。

8.  **SwiftUI：** 雖然 UIKit 是使用命令式方法建置使用者介面的傳統框架，但 SwiftUI 是 Apple 推出的較新宣告式框架，用於在所有 Apple 平台上建置使用者介面。它為 UI 開發提供了不同的典範，並且在現代 iOS 開發中日益普及，常與 UIKit 並用或作為其替代方案，特別是在新專案或新功能中。

這些框架，連同許多其他如 Metal（用於圖形加速）、SpriteKit（用於 2D 遊戲）、SceneKit（用於 3D 圖形）和 Core ML（用於機器學習）等框架，共同構成了強大的生態系統，讓 iOS 開發者能夠利用它們來創造多樣化且功能豐富的應用程式。
