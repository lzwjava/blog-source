### Java 核心

1. Java 中 OOP 的四個主要原則是什麼？  
   答案：四個主要原則是封裝、繼承、多型與抽象。封裝隱藏物件的內部狀態，繼承允許類別繼承，多型支援方法覆寫與重載，抽象則提供一種方式來呈現基本特徵而不包含背景細節。

2. 解釋 Java 中泛型的用途並提供範例。  
   答案：泛型允許型別參數化，實現程式碼重用與型別安全。例如，`ArrayList<T>` 使用型別參數 `T` 來儲存任何型別的元素。

3. 如何在 Java 中建立執行緒，其生命週期為何？  
   答案：可透過繼承 `Thread` 或實作 `Runnable` 來建立執行緒。生命週期包括新建、可執行、執行中、阻塞、等待、計時等待與終止狀態。

4. 描述 JVM 管理的不同記憶體區域。  
   答案：JVM 管理堆積、堆疊、方法區、原生方法堆疊與程式計數器暫存器。堆積儲存物件，而每個執行緒擁有自己的堆疊用於區域變數與方法呼叫。

5. Java 中受檢異常與非受檢異常有何區別？  
   答案：受檢異常必須被宣告或捕獲，而非受檢異常在編譯時不檢查。範例包括 `IOException`（受檢）與 `NullPointerException`（非受檢）。

6. 如何在 Java 中實作序列化，其重要性為何？  
   答案：序列化透過實作 `Serializable` 介面來實現。其重要性在於儲存與恢復物件狀態，適用於網路傳輸與持久化。

7. 比較 Java 集合框架中的 ArrayList 與 LinkedList。  
   答案：`ArrayList` 適用於快速存取與遍歷，而 `LinkedList` 更適合插入與刪除操作。`ArrayList` 使用連續記憶體，而 `LinkedList` 使用帶有指標的節點。

8. Java 中的 lambda 表達式是什麼，它們與函數介面有何關聯？  
   答案：Lambda 表達式提供簡潔方式來表示單一方法介面（函數介面）。它們用於實作如 `Runnable` 或 `Comparator` 等函數介面。

9. 解釋 Java Stream API 中的主要操作。  
   答案：Stream API 包括中介操作（如 `map`、`filter`）與終端操作（如 `forEach`、`collect`）。它們允許對集合進行函數式風格操作。

10. 如何在 Java 中使用反射來在執行時檢查類別？  
    答案：反射允許使用 `Class.forName()`、`getMethods()` 與 `getField()` 來檢查類別、方法與欄位。它用於動態行為與框架開發。

### Spring 生態系統

1. 什麼是 Spring IoC 容器，其運作原理為何？  
   答案：IoC 容器管理 bean 及其生命週期。它使用依賴注入來管理依賴關係，降低耦合度。

2. 解釋 Spring Boot 自動配置。  
   答案：自動配置根據 classpath 依賴自動配置 bean，簡化設定並減少樣板程式碼。

3. Spring Data JPA 如何簡化資料存取？  
   答案：Spring Data JPA 提供包含 CRUD 操作與查詢方法的儲存庫，抽象化資料庫互動。

4. Spring Security 的用途是什麼？  
   答案：Spring Security 提供認證與授權機制，保護應用程式免受未經授權的存取。

5. 描述 Spring MVC 在網頁應用程式中的角色。  
   答案：Spring MVC 處理網頁請求，將 URL 映射到控制器，並管理視圖與模型以回應網頁請求。

6. 什麼是 Spring Cloud 及其主要元件？  
   答案：Spring Cloud 提供建置雲端原生應用程式的工具，包括服務發現（Eureka）、斷路器（Hystrix）與 API 閘道。

7. Spring AOP 如何增強應用程式功能？  
   答案：AOP 允許將橫切關注點（如日誌記錄與交易管理）與業務邏輯分離，使用切面與通知。

8. 什麼是 Spring Boot Actuator，其功能為何？  
   答案：Actuator 提供用於監控與管理應用程式的端點，例如健康檢查、指標與環境資訊。

9. 解釋 Spring 設定檔的用途。  
   答案：設定檔允許針對不同環境（如開發、生產）使用不同配置，實現環境特定設定。

10. Spring Boot starters 如何簡化依賴管理？  
    答案：Starters 包含特定功能所需的所有依賴，減少手動管理依賴的需求。

### 並行與多執行緒

1. 什麼是 Java 中的死結，如何避免？  
   答案：死結發生於執行緒無限期相互等待。可透過避免循環等待與使用逾時來避免。

2. 解釋 Java 中的 Executor 框架。  
   答案：Executor 框架管理執行緒執行，提供執行緒池與任務排程。

3. Callable 與 Runnable 有何區別？  
   答案：Callable 可回傳結果並拋出異常，而 Runnable 不能，使 Callable 更適合需回傳結果的任務。

4. 描述 Java 記憶體模型。  
   答案：Java 記憶體模型定義執行緒如何存取變數，確保跨處理器操作的可見性與順序性。

5. Java 中的 volatile 關鍵字是什麼，何時應使用？  
   答案：Volatile 確保對變數的變更對所有執行緒可見，用於多執行緒環境以防止快取問題。

6. 如何在多執行緒應用程式中防止競爭條件？  
   答案：使用同步、鎖定或原子操作來確保對共享資源的獨佔存取。

7. 解釋讀寫鎖的概念。  
   答案：讀寫鎖允許多個讀取者或單一寫入者，透過允許共享存取來提升並行性。

8. 什麼是 CountDownLatch，其如何運用？  
   答案：CountDownLatch 允許一個執行緒等待一組執行緒完成，用於協調執行緒執行。

9. 描述 Java 中的鎖分段。  
   答案：鎖分段將鎖分割為多個部分（條紋），允許對不同部分的並行存取，減少競爭。

10. 如何在 Java 中處理執行緒中斷？  
    答案：執行緒可檢查中斷狀態並拋出 `InterruptedException`，實現優雅終止。

1. Nginx 常用於哪些用途？

   答案：負載平衡器將流量分發到多台伺服器，而反向代理將請求轉發到後端伺服器，通常提供快取與安全性。

3. 什麼是 HAProxy，為何使用？  
   答案：HAProxy 是一種高可用性負載平衡器與代理伺服器，用於管理與分發網路連線。

4. 如何在網頁伺服器上設定 SSL/TLS？  
   答案：SSL/TLS 透過取得憑證並設定 HTTPS 監聽器來配置，加密傳輸中的資料。

5. 什麼是伺服器端快取，如何實作？  
   答案：伺服器端快取將常用資料儲存在記憶體中，使用如 Varnish 或 Redis 等工具實作以提升效能。

6. 解釋網頁伺服器中記錄的重要性。  
   答案：記錄有助於監控伺服器活動、疑難排解問題與審計安全性，使用如 ELK Stack 等工具進行分析。

7. 保護網頁伺服器的最佳實踐有哪些？  
   答案：最佳實踐包括使用安全標頭、保持軟體更新與設定防火牆以抵禦威脅。

8. 如何在負載平衡中處理會話持久性？  
   答案：會話持久性可透過黏性會話或會話複製實現，確保使用者會話保持一致。

9. 什麼是 SSL 卸載，其好處為何？  
   答案：SSL 卸載在負載平衡器上解密 SSL/TLS 流量，減少伺服器負載並提升效能。

10. 描述水平擴展網頁伺服器的過程。  
    答案：水平擴展涉及增加更多伺服器以處理增加的負載，透過負載平衡器與自動擴展群組管理。

   答案：垃圾回收調優調整 JVM 參數以優化記憶體管理，減少暫停並提升效能。

4. Java 應用程式中哪些快取策略有效？  
   答案：策略包括本地快取、分散式快取（如 Redis）與快取過期策略，以平衡效能與一致性。

   答案：技術包括使用 CDN、優化資料中心與壓縮資料以最小化傳輸時間。
