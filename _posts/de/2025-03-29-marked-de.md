Um von Kramdown zu Marked in Ihrem Jekyll-Blog zu wechseln, müssen Sie einige Änderungen an Ihrer Jekyll-Konfiguration und Ihrem Setup vornehmen. Marked ist eine JavaScript-Bibliothek zum Parsen und Kompilieren von Markdown und kann in Verbindung mit MathJax für ein besseres Rendering von mathematischen Ausdrücken verwendet werden. Hier ist eine Schritt-für-Schritt-Anleitung für den Wechsel:

### Schritt 1: Marked installieren
Zuerst müssen Sie die Marked-Bibliothek in Ihr Jekyll-Projekt einbinden. Sie können dies tun, indem Sie sie im `<head>`-Bereich Ihrer Layout-Datei (z.B. `default.html`) hinzufügen.

```html
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>
```

### Schritt 2: Kramdown deaktivieren
Deaktivieren Sie in Ihrer `_config.yml`-Datei Kramdown, indem Sie die `markdown`-Option auf `none` setzen oder sie einfach entfernen, falls sie auf Kramdown gesetzt ist.

```yaml
markdown: none
```

### Schritt 3: Marked mit MathJax konfigurieren
Sie müssen Marked für die Arbeit mit MathJax konfigurieren. Erstellen Sie eine JavaScript-Datei (z.B. `marked-mathjax.js`) in Ihrem Jekyll-Projekt und fügen Sie den folgenden Code ein:

```javascript
document.addEventListener("DOMContentLoaded", function() {
  // Markdown-Inhalt mit Marked rendern
  const elements = document.querySelectorAll('.markdown-content');
  elements.forEach(element => {
    const html = marked(element.textContent);
    element.innerHTML = html;
  });

  // MathJax rendern
  MathJax.typesetPromise().then(() => {
    console.log('MathJax rendering complete!');
  }).catch((err) => console.log('Typeset failed:', err));
});
```

Binden Sie dieses Skript in Ihrer Layout-Datei ein, nachdem Sie Marked und MathJax eingebunden haben:

```html
<script src="https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-mml-chtml.js"></script>
<script src="{{ '/assets/js/marked-mathjax.js' | relative_url }}"></script>
```

### Schritt 4: Ihren Markdown-Inhalt aktualisieren
Stellen Sie sicher, dass Ihr Markdown-Inhalt in einen Container mit der Klasse `markdown-content` eingebettet ist, damit das JavaScript ihn zum Rendern ansteuern kann.

```html
<div class="markdown-content">
  <!-- Ihr Markdown-Inhalt hier -->
</div>
```

### Schritt 5: Ihr Setup testen
Erstellen Sie Ihre Jekyll-Website und führen Sie sie lokal aus, um sicherzustellen, dass Marked Ihren Markdown-Code korrekt rendert und MathJax mathematische Ausdrücke korrekt anzeigt.

```bash
bundle exec jekyll serve
```

### Zusätzliche Hinweise
- Stellen Sie sicher, dass Ihre MathJax-Konfiguration mit Ihrem Markdown-Inhalt kompatibel ist. Möglicherweise müssen Sie die Konfiguration basierend auf Ihren spezifischen Anforderungen anpassen.
- Wenn Sie auf Probleme stoßen, überprüfen Sie die Browser-Konsole auf Fehler und passen Sie Ihren JavaScript-Code entsprechend an.

Wenn Sie diese Schritte befolgen, sollten Sie in der Lage sein, von Kramdown zu Marked in Ihrem Jekyll-Blog zu wechseln und von der besseren MathJax-Unterstützung für mathematische Ausdrücke zu profitieren.
