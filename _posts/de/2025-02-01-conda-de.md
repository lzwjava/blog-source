## Probleme

Siehe diesen Link: [https://www.reddit.com/r/MachineLearning/comments/myl8yn/d_is_anaconda_worth_the_trouble/](https://www.reddit.com/r/MachineLearning/comments/myl8yn/d_is_anaconda_worth_the_trouble/).

Ich hatte Probleme mit conda, und andere haben ähnliche Probleme diskutiert.

Mit der Zeit wird man sich der verschiedenen Umgebungseinstellungen nicht mehr bewusst. Daher musste ich die anderen löschen. Jetzt nutze ich hauptsächlich `Python 3.10.9` und gelegentlich `Python 3.11.11`. Selbst `Python 3.11.11` nutze ich selten. Persönlich benötige ich nicht zu viele Umgebungen. Ich brauche nur eine oder zwei, um das zu betreiben, was ich derzeit untersuche oder lerne.

Ich bevorzuge eine einfachere Lösung. Wenn ich zwei Projekte habe, die sich gegenseitig beeinträchtigen, würde ich lieber eines davon verwenden und auf die Ausführung des anderen verzichten.

Zum Beispiel ist es ein echter Aufwand, auf das folgende Problem zu stoßen:

```bash
conda config --set proxy_servers.http $HTTP_PROXY
conda config --set proxy_servers.https $HTTP_PROXY
```

Obwohl die Einrichtung eines Proxys für conda anfangs eine Beschleunigung zu bieten schien, entschied ich mich letztlich dafür, es zu entfernen.

## Entfernung

```bash
rm -rf ~/anaconda3
rm -rf ~/.condarc
rm -rf ~/Library/Caches/pip
# update ~/.zprofile
# update ~/.bash_profile
```

`~/.bash_profile`:

```
# >>> conda initialize >>>
# !! Der Inhalt innerhalb dieses Blocks wird von 'conda init' verwaltet !!
__conda_setup="$('/Users/lzwjava/anaconda3/bin/conda' 'shell.bash' 'hook' 2> /dev/null)"
if [ $? -eq 0 ]; then
    eval "$__conda_setup"
else
    if [ -f "/Users/lzwjava/anaconda3/etc/profile.d/conda.sh" ]; then
        . "/Users/lzwjava/anaconda3/etc/profile.d/conda.sh"
    else
        export PATH="/Users/lzwjava/anaconda3/bin:$PATH"
    fi
fi
unset __conda_setup
# <<< conda initialize <<<
```

Und es zeigt immer noch:

```bash
% which conda
conda () {
	\local cmd="${1-__missing__}"
	case "$cmd" in
		(activate | deactivate) __conda_activate "$@" ;;
		(install | update | upgrade | remove | uninstall) __conda_exe "$@" || \return
			__conda_reactivate ;;
		(*) __conda_exe "$@" ;;
	esac
}
```

Nach dem Neustart von iTerm2 entschied ich mich, zur Python-Installation von Homebrew zu wechseln. Es schien, als gäbe es mehrere Python-Installationen auf meinem Computer, und ich hatte bereits sechs, wie unten gezeigt.
