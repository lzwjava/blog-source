Hier sind einige häufig verwendete Prompts für verschiedene Programmieraufgaben, nach Kategorien geordnet. Diese Prompts werden häufig beim Schreiben von Code, Refactoring, Ändern von Variablennamen und bei anderen damit verbundenen Tätigkeiten verwendet, insbesondere in Kontexten wie IDEs, Code-Editoren oder bei der Interaktion mit KI-Coding-Assistenten.

### Code Schreiben
Diese Prompts werden verwendet, um neue Code-Snippets oder Implementierungen zu generieren:
- "Schreibe eine Funktion, um [spezifische Aufgabe auszuführen]."  
  *Beispiel: "Schreibe eine Funktion, um die Summe eines Arrays zu berechnen."*
- "Erstelle eine Klasse für [spezifischen Zweck]."  
  *Beispiel: "Erstelle eine Klasse zur Verwaltung eines Benutzerprofils."*
- "Generiere Code, um [bestimmte Funktionalität zu erreichen]."  
  *Beispiel: "Generiere Code, um Daten von einer API abzurufen."*

### Refactoring
Diese Prompts konzentrieren sich darauf, die Struktur, Lesbarkeit oder Effizienz von vorhandenem Code zu verbessern, ohne dessen Verhalten zu ändern:
- "Refaktoriere diesen Code, um die Lesbarkeit zu verbessern."  
  *Beispiel: "Refaktoriere diesen Code, um klarere Variablennamen und konsistente Formatierung zu verwenden."*
- "Vereinfache diese Funktion, indem du sie in kleinere Funktionen aufteilst."  
  *Beispiel: "Vereinfache diese Funktion, indem du die Validierungslogik in eine separate Methode auslagerst."*
- "Optimiere diesen Code für eine bessere Performance."  
  *Beispiel: "Optimiere diesen Code, indem du verschachtelte Schleifen reduzierst."*

### Variablennamen ändern
Diese Prompts zielen darauf ab, die Codeklarheit zu verbessern, indem Bezeichner aussagekräftiger oder konsistenter gestaltet werden:
- "Benenne die Variable '[Variable_name]' in einen aussagekräftigeren Namen um."  
  *Beispiel: "Benenne die Variable 'x' in 'userAge' um."*
- "Ändere den Funktionsnamen '[Funktion_name]', um dessen Zweck besser widerzuspiegeln."  
  *Beispiel: "Ändere den Funktionsnamen 'func' in 'calculateTotalPrice'."*
- "Aktualisiere alle Variablennamen, um [spezifischer Namenskonvention] zu folgen."  
  *Beispiel: "Aktualisiere alle Variablennamen, um der camelCase-Konvention zu folgen."*

### Debugging
Diese Prompts helfen dabei, Probleme im Code zu identifizieren und zu beheben:
- "Finde und behebe den Fehler in diesem Code."  
  *Beispiel: "Finde und behebe den Fehler, der diese Nullpointer-Exception verursacht."*
- "Debugge diese Funktion, um [spezifische Randfälle] zu behandeln."  
  *Beispiel: "Debugge diese Funktion, um leere Eingabearrays zu behandeln."*

### Kommentare/Dokumentation hinzufügen
Diese Prompts verbessern die Code-Wartbarkeit, indem Erklärungen oder formale Dokumentation hinzugefügt werden:
- "Füge diesem Code Kommentare hinzu, um seine Funktionalität zu erklären."  
  *Beispiel: "Füge diesem Code Kommentare hinzu, um den verwendeten Sortieralgorithmus zu beschreiben."*
- "Generiere Dokumentation für diese Klasse/diese Methode."  
  *Beispiel: "Generiere Dokumentation für diese User-Klasse."*

### Code-Übersetzung
Diese Prompts konvertieren Code zwischen Programmiersprachen:
- "Übersetze diesen [Quellsprache]-Code in [Zielsprache]."  
  *Beispiel: "Übersetze diesen Python-Code in Java."*
- "Konvertiere diese [Sprache]-Funktion in [andere Sprache]."  
  *Beispiel: "Konvertiere diese JavaScript-Funktion in TypeScript."*

### Code-Review
Diese Prompts holen Feedback ein oder identifizieren potenzielle Verbesserungen:
- "Überprüfe diesen Code und schlage Verbesserungen vor."  
  *Beispiel: "Überprüfe diesen Code und schlage Wege zur Reduzierung der Komplexität vor."*
- "Prüfe diesen Code auf potenzielle Sicherheitsprobleme."  
  *Beispiel: "Prüfe diesen Code auf SQL-Injection-Schwachstellen."*

### Tests generieren
Diese Prompts erstellen Testfälle, um die Korrektheit des Codes zu überprüfen:
- "Schreibe Unit-Tests für diese Funktion."  
  *Beispiel: "Schreibe Unit-Tests für diese calculateDiscount-Funktion."*
- "Erstelle einen Testfall, um diese Methode zu verifizieren."  
  *Beispiel: "Erstelle einen Testfall, um zu verifizieren, dass diese Methode negative Eingaben behandelt."*

### Konversationelle Prompts
Dies sind natürlichsprachlichere Prompts, die häufig mit KI-Assistenten verwendet werden:
- "Kannst du mir helfen, ein Skript zu schreiben, um [Aufgabe auszuführen]?"  
  *Beispiel: "Kannst du mir helfen, ein Skript zu schreiben, um Dateien in einem Verzeichnis umzubenennen?"*
- "Ich brauche Code, der [bestimmte Dinge tut]."  
  *Beispiel: "Ich brauche Code, der eine CSV-Datei liest und die Daten sortiert."*
- "Zeige mir ein Beispiel für [die Verwendung eines bestimmten Features oder einer API]."  
  *Beispiel: "Zeige mir ein Beispiel für die Verwendung der Fetch API in JavaScript."*

Diese Prompts bieten eine vielseitige Grundlage für gängige Programmieraufgaben. Sie können angepasst werden, indem spezifische Details (z.B. Funktionsnamen, Sprachen oder Einschränkungen) je nach Kontext oder den Anforderungen Ihres Projekts ausgefüllt werden. Egal, ob Sie neuen Code schreiben, eine bestehende Codebasis verfeinern oder Hilfe von einem KI-Tool suchen, diese Vorlagen sollten ein breites Spektrum an Szenarien abdecken.
