- Satya Nadella erwähnte den Jevons-Paradoxon. Es lohnt sich, ihn zu lernen.

- Yin Wang: Es gibt keine "Intelligenz" in künstlicher Intelligenz, kein "neuronal" in neuronalem Netzwerk, kein "Lernen" in maschinellem Lernen und keine "Tiefe" in Deep Learning. Es gibt keine "Tiefe" in Deep Learning. Was wirklich in diesem Bereich funktioniert, nennt man "Calculus". Daher bevorzuge ich es, diesen Bereich "differenzierbare Berechnung" zu nennen, und der Prozess des Modellbaus wird "differenzierbare Programmierung" genannt.

- Yin Wang: Maschinelles Lernen ist wirklich nützlich, man könnte sogar sagen, eine schöne Theorie, weil es einfach Calculus nach einem Makeover ist! Es ist die alte und große Theorie von Newton, Leibniz, in einer einfacheren, eleganteren und mächtigeren Form. Maschinelles Lernen ist im Wesentlichen die Verwendung von Calculus, um einige Funktionen abzuleiten und anzupassen, und Deep Learning ist das Anpassen komplexerer Funktionen.

- Derzeit können große Sprachmodelle keine Dateisprachen wie YAML oder Python filtern. Ein erheblicher Teil der Informationen in der realen Welt ist jedoch auf diese Weise organisiert. Das bedeutet, dass wir große Sprachmodelle mit Dateien trainieren könnten.

- Für das Training großer Sprachmodelle könnten wir ein System entwickeln, das exakte Übereinstimmungen findet. Vielleicht können wir den KMP (Knuth-Morris-Pratt)-Suchalgorithmus mit der Transformer-Architektur kombinieren, um die Suchfähigkeiten zu verbessern.

- Es gibt keine technologischen Geheimnisse. Open Source wird alle Geheimnisse offenlegen, die streng gehütet werden.

- KI wird viele Werkzeuge beeinflussen, einschließlich indirekter. Menschen sagen, sie würden Figma nicht mehr benötigen, um Prototypen zu zeichnen, sie würden direkt zum Code gehen. Ich denke, Postman wird ähnlich sein; Menschen werden direkt Python oder andere Skripte verwenden, um APIs aufzurufen oder zu testen.

- Ein Grund, warum wir Postman oder Figma im Zeitalter der KI nicht verwenden, ist, dass deren Funktionalitäten nicht durch Text generiert werden können. Sie verfügen auch nicht über eine Befehl + K-Tastenkombination, um die Komponentenersetzung auszulösen.

- Benutzeroberflächen werden im Zeitalter der KI zu einer Barriere. Warum sollte Postman für das Testen von Anwendungen KI-gestützt werden, wenn wir direkt die Python-Bibliothek "requests" oder andere Programmiersprachen verwenden können, um Code zu testen, da Letztere KI-gestützt sein wird?

- Warum sollte Figma für die Erstellung von Benutzeroberflächen KI-gestützt werden, wenn die KI-gestützte, codebasierte Erstellung von Benutzeroberflächen einen direkteren und potenziell mächtigeren Ansatz bietet?

- LLMs werden textbezogene Anwendungen wie Google, Suchmaschinen, Texteditoren und Schreibwerkzeuge, Quizlet, Zendesk, DeepL, Medium, WordPress, Trello, Asana, Gmail, GitHub, Goodreads, Duolingo und Feedly zuerst verändern.

- Umgekehrt ist es unwahrscheinlich, dass LLMs Technologien wie Git, Linux, ffmpeg, Mobiltelefone, Hardware, Browser, Betriebssysteme oder Sprach- und Videotelefonate revolutionieren. Diese Technologien sind codezentriert, und ihr Code kann nicht so leicht von KI generiert werden wie API-Testwerkzeuge wie Postman.

- Technologien mit mehr Code sind schwer von KI zu revolutionieren, wie OpenOffice, MySQL, Mozilla Firefox, Chromium, VLC Media Player, Qt Framework, LLVM/Clang und GNOME. Wenn KI helfen könnte, diese Technologien zu machen, würden sie nicht ersetzt. KI sollte helfen, bessere Technologien zu machen, und um das zu tun, wird KI mehr Rechenleistung benötigen, um die gleiche Größenordnung an Code zu generieren.

- Es gibt zwei Möglichkeiten, wie LLMs Veränderungen mit sich bringen können: Erstens, indem sie den Inhalt oder die Daten innerhalb einer Plattform oder Software ändern, wie z.B. die Inhaltsübersetzung in Apps wie TikTok; zweitens, indem sie bestimmte Software oder Plattformen direkt ersetzen, wie Postman oder Google Search, einschließlich Google Translate.

- Es gibt zwei Möglichkeiten, wie KI-Audio-Werkzeuge Veränderungen mit sich bringen können: Erstens, indem sie den Inhalt oder die Daten innerhalb einer Plattform oder Software ändern, wie z.B. das Generieren von Hörbüchern für Audible; zweitens, indem sie bestimmte Software oder Plattformen direkt ersetzen, z.B. die Sing-Songs-App, da KI nun die gleichen Aufgaben ausführen kann wie Menschen, was es den Menschen leichter macht, Lieder als Hobby zu singen.

- Es gibt mehrere Möglichkeiten, wie KI den aktuellen Software- oder Plattformen beeinflussen kann. Eine Möglichkeit besteht darin, zu messen, wie viel Daten oder Inhalt von KI teilweise oder vollständig generiert oder verbessert werden kann. Eine weitere Möglichkeit besteht darin, zu messen, wie viel Code von KI teilweise oder vollständig geschrieben oder verbessert werden kann. Das bedeutet, dass wir das, was KI generiert, verwenden, um aktuelle Plattformen zu verbessern. Zusätzlich kann KI helfen, neue Software und Plattformen zu erfinden.

- Es wird drei Arten von Produkten geben: Generative KI-Produkte, Produkte, die die APIs von Generative KI-Produkten verwenden, und andere Produkte.

- Eine Produktidee besteht darin, KI zu verwenden, um Echtzeitinformationen, Nachrichten oder Updates von sozialen Plattformen wie Reddit, GitHub Trending, Twitter Trending, Quora Trending und Zhihu Trending zu sammeln. Benutzer können Prompts verwenden, um den Feed anzupassen.
