```bash
npm install --save-dev \
  "babel-core@6.13.0" \
  "babel-loader@6.2.5" \
  "babel-plugin-transform-runtime@6.12.0" \
  "babel-preset-es2015@6.13.2" \
  "babel-runtime@6.11.6"
```

### Konfiguration
Erstellen Sie eine `.babelrc`-Datei im Projektverzeichnis mit:

```json
{
  "presets": ["es2015"],
  "plugins": ["transform-runtime"]
}
```

Konfigurieren Sie dann Ihre `webpack.config.js` wie folgt:

```javascript
module.exports = {
  entry: './index.js',
  output: {
    filename: 'bundle.js'
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'babel-loader',
        options: {
          configFile: './.babelrc'
        }
      }
    ]
  }
};
```

Schreiben Sie Ihren ES2015-JavaScript-Code in Dateien wie `index.js`, und führen Sie `npx webpack` aus, um ihn in `bundle.js` zu kompilieren, das ältere Browser verstehen können.

#### Konfigurationsdetails
Die Konfiguration umfasst zwei Hauptdateien: `.babelrc` für Babel-Einstellungen und `webpack.config.js` für die Integration in webpack.

##### `.babelrc`-Datei
Erstellen oder bearbeiten Sie `.babelrc` im Projektverzeichnis mit folgendem Inhalt:

```json
{
  "presets": ["es2015"],
  "plugins": ["transform-runtime"]
}
```

- **Presets**: `"es2015"` entspricht `babel-preset-es2015@6.13.2`, wodurch alle ES2015-Transformationen in ES5 aktiviert werden, wie z. B. Pfeilfunktionen, let/const und Klassen.
- **Plugins**: `"transform-runtime"` verwendet `babel-plugin-transform-runtime@6.12.0`, um Babel-Helfer zu externalisieren, die auf `babel-runtime@6.11.6` verweisen, wodurch die Größe des kompilierten Codes reduziert wird, indem die Duplikation von Helfern über Dateien hinweg vermieden wird.

Diese Konfiguration stellt sicher, dass ES2015-Code transpiliert und für die Laufzeitoptimierung optimiert wird.

##### Webpack-Konfiguration
Für webpack erstellen oder aktualisieren Sie `webpack.config.js` wie folgt:

```javascript
module.exports = {
  entry: './index.js',
  output: {
    filename: 'bundle.js'
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'babel-loader',
        options: {
          configFile: './.babelrc'
        }
      }
    ]
  }
};
```

- **Entry und Output**: Gibt `index.js` als Einstiegspunkt an und gibt `bundle.js` aus.
- **Module Rules**: Verwendet `babel-loader@6.2.5` für `.js`-Dateien, zeigt auf `.babelrc` für Optionen, sodass Babel die Dateien vor dem Bündeln verarbeitet.

Diese Einrichtung nutzt das Modulsystem von webpack, um JavaScript-Dateien zu verarbeiten und sie vor dem Bündeln zu kompilieren.

Das `transform-runtime`-Plugin stellt sicher, dass Helfer gemeinsam genutzt werden, wodurch die Größe des Bündels reduziert wird, was besonders für große Projekte nützlich ist.
