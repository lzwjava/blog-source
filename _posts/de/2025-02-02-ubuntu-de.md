## Installation

Ich habe zuvor über [Wie man einen Computer baut](./computer-en) geschrieben. Versuchen wir etwas anderes. Heute installiere ich Ubuntu 24.04 auf einem Lenovo Xiaoxing 14IAH8 Laptop, der mit vorinstalliertem Windows 10 geliefert wurde.

```bash
sudo dd if=ubuntu-24.04.1-desktop-amd64.iso of=/dev/rdisk6 bs=1m
```

Es ist notwendig, in die Einstellungen zu gehen und die Geräteverschlüsselung zu deaktivieren, da der Ubuntu-Installer dies erfordert.

Wenn die Geräteverschlüsselung aktiviert und der sichere Start im BIOS deaktiviert ist, fordert Windows zur Eingabe der BitLocker-Passphrase zur Wiederherstellung auf.

Daher ist es notwendig, zuerst den sicheren Start im BIOS zu aktivieren, sich bei Windows anzumelden und dann in die Einstellungen zu gehen, um die Geräteverschlüsselung zu deaktivieren. Danach kann der sichere Start wieder deaktiviert werden, um den Ubuntu-Installer zu starten.

## Hängenbleiben

Wenn Ubuntu den Dienst `gdm.service` (GNOME Display Manager) startet und hängen bleibt, deutet dies in der Regel auf ein Problem mit der grafischen Benutzeroberfläche (GUI) oder dem Display-Manager selbst hin. Hier sind einige Schritte, die Sie zur Fehlerbehebung und möglichen Behebung des Problems unternehmen können:

1. **Zugriff auf ein virtuelles Terminal:**
   - Wenn Ihr System auf dem GDM-Bildschirm hängen bleibt, können Sie versuchen, auf ein Terminal zuzugreifen, um das Problem zu diagnostizieren.
   - Drücken Sie `Strg + Alt + F2` (oder `F3`, `F4` usw.), um zu einem virtuellen Terminal zu wechseln.

2. **Anmelden:**
   - Melden Sie sich mit Ihrem Benutzernamen und Passwort an.

3. **Systemprotokolle überprüfen:**
   - Verwenden Sie den folgenden Befehl, um die Systemprotokolle auf Fehler oder Warnungen zu überprüfen, die die Ursache des Problems aufzeigen könnten:
     ```bash
     sudo journalctl -xe
     ```

4. **GDM-Dienst neu starten:**
   - Versuchen Sie, den GDM-Dienst neu zu starten, um zu sehen, ob das Problem dadurch behoben wird:
     ```bash
     sudo systemctl restart gdm
     ```

5. **System neu starten:**
   - Starten Sie Ihr System nach dem Vornehmen von Änderungen neu, um diese anzuwenden:
     ```bash
     sudo reboot
     ```
