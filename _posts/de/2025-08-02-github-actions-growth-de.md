Basierend auf den neuesten verfügbaren Daten aus den Octoverse-Berichten von GitHub ist die Anzahl der in GitHub Actions ausgeführten Workflow-Jobs im Laufe der Jahre aufgrund der zunehmenden Akzeptanz bei Entwicklern und des wachsenden Automatisierungsbedarfs erheblich gestiegen.

Im Jahr 2022 (das letzte Jahr, in dem die Job-Anzahl explizit berichtet wurde) führte GitHub Actions etwa 263 Millionen Workflow-Jobs pro Monat aus. Dies entspricht:
- Etwa 8,77 Millionen Jobs pro Tag (263 Millionen / 30 Tage).
- Etwa 365.000 Jobs pro Stunde (8,77 Millionen / 24 Stunden).

Seitdem ist die Gesamtnutzung erheblich gestiegen. Die Minuten in GitHub Actions (ein Indikator für Aktivität) stiegen von 7,3 Milliarden im Jahr 2023 auf 10,54 Milliarden im Jahr 2024, was einem jährlichen Anstieg von 30 % entspricht. Die Anzahl der Entwickler auf GitHub stieg ebenfalls von etwa 94 Millionen im Jahr 2022 auf über 150 Millionen bis Mitte 2025. Unter der Annahme eines ähnlichen Wachstums des Job-Aufkommens liegt die tägliche Zahl im Jahr 2025 wahrscheinlich im Bereich von mehreren zehn Millionen (z. B. 10-15 Millionen pro Tag) und die stündliche Zahl im Bereich von mehreren hunderttausend bis niedrigen Millionen (z. B. 400.000-600.000 pro Stunde).

Dies sind Schätzungen für die gesamte globale Nutzung über alle Repositorys hinweg (öffentlich und privat). Genaue Zahlen für 2025 sind nicht öffentlich verfügbar, aber die Trends deuten darauf hin, dass sie im Bereich von mehreren zehn Millionen pro Tag liegen.
