## **Überblick über WinSCP**
WinSCP (Windows Secure Copy) ist ein kostenloser, quelloffener Dateiübertragungs-Client für Windows. Er ermöglicht Benutzern die sichere Übertragung von Dateien zwischen einem lokalen Windows-Rechner und einem Remote-Server oder zwischen zwei Remote-Servern. Seine grafische Benutzeroberfläche (GUI) macht ihn für Anfänger zugänglich, während seine Scripting- und Automatisierungsfunktionen fortgeschrittene Benutzer ansprechen. Er wird häufig zur Verwaltung von Dateien auf Web-Servern, Cloud-Speicher oder anderen Remote-Systemen verwendet.

---

## **Hauptfunktionen**
- **Protokollunterstützung**: SFTP, FTP, FTPS, SCP, WebDAV und Amazon S3.
- **Grafische Benutzeroberfläche**: Dual-Pane-Dateimanager mit Drag-and-Drop-Unterstützung.
- **Sicherheit**: Unterstützt SSH-Key-Authentifizierung, Verschlüsselung und sichere Dateiübertragungen.
- **Automatisierung**: Scripting und Command-Line-Interface für Stapeldatei-Übertragungen.
- **Synchronisierung**: Automatische Synchronisierung von lokalen und Remote-Verzeichnissen.
- **Integration**: Nahtlose Integration mit PuTTY für SSH-Terminal-Zugriff.
- **Anpassung**: Umfangreiche Optionen für Übertragungseinstellungen, Dateiberechtigungen und mehr.
- **Portabler Modus**: Kann von einem USB-Laufwerk aus ohne Installation ausgeführt werden.
- **Mehrsprachig**: Unterstützt mehrere Sprachen für globale Zugänglichkeit.

---

## **Systemanforderungen**
- **Betriebssystem**: Windows XP oder neuer (Windows 10/11 empfohlen).
- **Festplattenspeicher**: Etwa 20 MB für die Installation.
- **Abhängigkeiten**: Keine für die Grundfunktionalität; PuTTY wird für die SSH-Key-Verwaltung empfohlen.
- **Internet**: Erforderlich für die Verbindung zu Remote-Servern.

## **Installation**

### **WinSCP herunterladen und installieren**
1. **WinSCP herunterladen**:
   - Besuchen Sie die offizielle WinSCP-Website: [https://winscp.net/](https://winscp.net/).
   - Laden Sie die neueste Version herunter (z.B. WinSCP 6.3.4 Stand Mai 2025).
   - Wählen Sie zwischen dem **Installer** (für die meisten Benutzer empfohlen) oder der **portablen Ausführung** (für USB-Laufwerke).

2. **Installer ausführen**:
   - Doppelklicken Sie auf die heruntergeladene `.exe`-Datei.
   - Folgen Sie dem Setup-Assistenten:
     - Wählen Sie die Sprache (standardmäßig Englisch).
     - Wählen Sie den Installationstyp: **Standard** (installiert in Program Files) oder **Portable** (keine Änderungen an der Registry).
     - Wählen Sie den Oberflächenstil: **Commander** (Dual-Pane, ähnlich wie Norton Commander) oder **Explorer** (ähnlich wie Windows Explorer).
   - Schließen Sie die Installation ab.

3. **Installation überprüfen**:
   - Starten Sie WinSCP über das Startmenü oder die Desktop-Verknüpfung.
   - Stellen Sie sicher, dass die Anwendung ohne Fehler öffnet.

## **Erste Schritte**

### **Ersteinrichtung und Konfiguration**
1. **WinSCP starten**:
   - Öffnen Sie WinSCP, um auf den **Login**-Dialog zuzugreifen.

2. **Eine neue Sitzung konfigurieren**:
   - **Dateiprotokoll**: Wählen Sie SFTP (am sichersten), FTP oder ein anderes Protokoll.
   - **Host-Name**: Geben Sie die IP-Adresse oder Domain des Servers ein (z.B. `example.com`).
   - **Port-Nummer**: Standard ist 22 für SFTP/SCP, 21 für FTP.
   - **Benutzername**: Geben Sie Ihren Server-Benutzernamen ein.
   - **Passwort**: Geben Sie Ihr Passwort ein oder lassen Sie es für schlüsselbasierte Authentifizierung leer.
   - **Sitzung speichern**: Klicken Sie auf "Save", um die Verbindungsdetails für die zukünftige Verwendung zu speichern.

3. **Erweiterte Einstellungen** (Optional):
   - Klicken Sie im Login-Dialog auf **Advanced**, um zu konfigurieren:
     - **SSH-Schlüssel**: Importieren Sie private Schlüssel für eine sichere Authentifizierung.
     - **Übertragungseinstellungen**: Legen Sie den Standard-Übertragungsmodus fest (binär, Text oder automatisch).
     - **Verzeichniseinstellungen**: Geben Sie Standard-Local- und Remote-Verzeichnisse an.

4. **Verbinden**:
   - Klicken Sie auf **Login**, um eine Verbindung zum Server herzustellen.
   - Akzeptieren Sie den Host-Key des Servers, wenn Sie dazu aufgefordert werden (überprüfen Sie ihn bei Unklarheiten bei Ihrem Server-Administrator).

## **WinSCP als Dateimanager verwenden**

### **Überblick über die Oberfläche**
WinSCP bietet zwei Oberflächenstile:
- **Commander-Oberfläche**: Dual-Pane-Ansicht mit lokalen Dateien links und Remote-Dateien rechts. Ideal für Power-User.
- **Explorer-Oberfläche**: Single-Pane-Ansicht, die nur den Remote-Server anzeigt, ähnlich wie der Windows Explorer.

**Wichtige Oberflächenelemente**:
- **Symbolleiste**: Schnellzugriff auf häufige Aufgaben (verbinden, trennen, neue Sitzung).
- **Datei-Panels**: Zeigen lokale und Remote-Verzeichnisse an.
- **Statusleiste**: Zeigt Verbindungsstatus, Übertragungsfortschritt und Fehler an.
- **Kontextmenüs**: Rechtsklick auf Dateien/Ordner für Optionen wie Hochladen, Herunterladen, Löschen oder Umbenennen.

### **Verbinden mit einem Remote-Server**
1. Öffnen Sie WinSCP und wählen Sie eine gespeicherte Sitzung aus oder erstellen Sie eine neue.
2. Geben Sie die Anmeldedaten ein oder wählen Sie einen SSH-Schlüssel aus.
3. Klicken Sie auf **Login**, um eine Verbindung herzustellen.
4. Wenn erfolgreich, erscheint das Dateisystem des Remote-Servers in der Oberfläche.

### **Navigieren und Verwalten von Dateien**
- **Navigation**:
  - Verwenden Sie die Datei-Panels, um lokale und Remote-Verzeichnisse zu durchsuchen.
  - Doppelklicken Sie auf Ordner, um sie zu öffnen.
  - Verwenden Sie die **Zurück**- und **Vorwärts**-Buttons oder die Brotkrümelnavigation.
- **Dateioperationen**:
  - **Kopieren/Verschieben**: Ziehen Sie Dateien zwischen den Panels oder verwenden Sie die Kontextmenü-Optionen.
  - **Löschen**: Rechtsklick und **Delete** auswählen oder `Entf` drücken.
  - **Umbenennen**: Rechtsklick und **Rename** auswählen oder `F2` drücken.
  - **Ordner erstellen**: Rechtsklick und **New > Directory** auswählen.
  - **Berechtigungen**: Rechtsklick und **Properties** auswählen, um Dateiberechtigungen zu ändern (z.B. 755 für Verzeichnisse, 644 für Dateien auf Linux-Servern).
- **Suchen**: Verwenden Sie `Strg+F`, um Dateien im aktuellen Verzeichnis zu finden.

### **Dateien übertragen**
1. **Hochladen**:
   - Ziehen Sie Dateien vom lokalen Panel in das Remote-Panel.
   - Oder rechtsklicken Sie auf eine Datei und wählen Sie **Upload**.
   - Wählen Sie den Übertragungsmodus:
     - **Binär**: Für Bilder, ausführbare Dateien oder Archive.
     - **Text**: Für Textdateien, um Zeilenumbruch-Konvertierungen zu behandeln.
     - **Automatisch**: WinSCP entscheidet basierend auf dem Dateityp.
2. **Herunterladen**:
   - Ziehen Sie Dateien vom Remote-Panel in das lokale Panel.
   - Oder rechtsklicken Sie und wählen Sie **Download**.
3. **Übertragungseinstellungen**:
   - Konfigurieren Sie unter **Preferences > Transfer**:
     - Geschwindigkeitsbegrenzungen festlegen.
     - Resume-Unterstützung für unterbrochene Übertragungen aktivieren.
     - Zeitstempel oder Berechtigungen beibehalten.

### **Dateien synchronisieren**
Die Synchronisierungsfunktion von WinSCP hält lokale und Remote-Verzeichnisse synchron.
1. Gehen Sie zu **Commands > Synchronize**.
2. Wählen Sie:
   - **Local**: Lokales Verzeichnis an Remote anpassen.
   - **Remote**: Remote-Verzeichnis an lokal anpassen.
   - **Mirror**: Beide Verzeichnisse identisch halten.
3. Wählen Sie die zu vergleichenden Verzeichnisse aus.
4. Vorschau der Änderungen anzeigen und bestätigen, um zu synchronisieren.

## **Erweiterte Funktionen**

### **Automatisierung und Scripting**
WinSCP unterstützt Automatisierung für wiederkehrende Aufgaben:
- **Batch-Skripte**:
  - Schreiben Sie Skripte mit der WinSCP-Kommandozeilenschnittstelle (z.B. `winscp.com`).
  - Beispiel: Automatisieren Sie nächtliche Backups.
    ```bash
    open sftp://user:password@host
    get /remote/path/* C:\local\path\
    exit
    ```
  - Als `.txt`-Datei speichern und mit `winscp.com /script=script.txt` ausführen.
- **.NET Assembly**:
  - Verwenden Sie die .NET-Assembly von WinSCP für die Integration mit PowerShell oder C#.
  - Beispiel: Automatisieren Sie Dateiübertragungen in einem PowerShell-Skript.

### **Benutzerdefinierte Befehle**
- Erstellen Sie benutzerdefinierte Befehle für wiederkehrende Aufgaben (z.B. Komprimieren von Dateien auf dem Server).
- Gehen Sie zu **Commands > Custom Commands > Add**.
- Beispiel: Führen Sie `tar -czf archive.tar.gz !` aus, um ausgewählte Dateien zu komprimieren.

### **Integration mit PuTTY**
- Verknüpfen Sie WinSCP mit PuTTY für SSH-Terminal-Zugriff:
  - Geben Sie unter **Preferences > Integration** den Pfad zu `putty.exe` an.
  - Verwenden Sie **Open in PuTTY** in der Symbolleiste, um eine Terminal-Sitzung mit denselben Anmeldedaten zu starten.

### **Verschlüsselung und Sicherheit**
- **SSH-Schlüssel**:
  - Generieren Sie Schlüssel mit PuTTYgen oder importieren Sie vorhandene Schlüssel.
  - Konfigurieren Sie diese unter **Advanced > SSH > Authentication**.
- **Verschlüsselung**: Verschlüsseln Sie Dateien vor der Übertragung mit der integrierten Verschlüsselung von WinSCP.
- **Master-Passwort**: Schützen Sie gespeicherte Sitzungspasswörter unter **Preferences > Security**.

## **Tipps und Best Practices**
- **Sitzungen speichern**: Speichern Sie häufig verwendete Verbindungen, um Zeit zu sparen.
- **Binärmodus verwenden**: Für Nicht-Text-Dateien, um Beschädigungen zu vermeiden.
- **Backup vor dem Löschen**: Überprüfen Sie Dateien immer, bevor Sie sie auf dem Server löschen.
- **Berechtigungen prüfen**: Stellen Sie korrekte Dateiberechtigungen für Web-Server sicher (z.B. 644 für Dateien, 755 für Verzeichnisse).
- **Regelmäßig aktualisieren**: Halten Sie WinSCP für Sicherheitspatches und neue Funktionen aktuell.
- **Filter verwenden**: Schließen Sie unnötige Dateien (z.B. `.git`) während Übertragungen aus.

## **Behebung häufiger Probleme**
1. **Verbindung abgelehnt**:
   - Überprüfen Sie Host, Port und Anmeldedaten.
   - Prüfen Sie, ob die Firewall des Servers Verbindungen erlaubt.
2. **Authentifizierung fehlgeschlagen**:
   - Stellen Sie sicher, dass Benutzername/Passwort oder SSH-Schlüssel korrekt sind.
   - Überprüfen Sie das Schlüsselformat (WinSCP unterstützt PuTTYs `.ppk`-Format).
3. **Langsame Übertragungen**:
   - Überprüfen Sie die Netzwerkstabilität.
   - Passen Sie die Übertragungseinstellungen an (z.B. Komprimierung für schnellere Übertragungen deaktivieren).
4. **Zugriff verweigert**:
   - Bestätigen Sie die Benutzerberechtigungen auf dem Server.
   - Verwenden Sie **Properties**, um Datei-/Ordnerberechtigungen anzupassen.
5. **Host-Key-Überprüfung**:
   - Akzeptieren Sie den Host-Key nur, wenn Sie dem Server vertrauen.
   - Kontaktieren Sie den Server-Administrator, um den Key-Fingerprint zu überprüfen.
