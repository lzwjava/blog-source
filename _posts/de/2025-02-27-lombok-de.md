Um Lombok in Java zu verwenden, müssen Sie es in Ihrem Projekt einrichten, um Ihren Code durch die Reduzierung von Boilerplate-Code wie Getter, Setter und Konstruktoren zu vereinfachen. Hier ist eine Schritt-für-Schritt-Anleitung:

Zuerst müssen Sie Lombok als Abhängigkeit hinzufügen. Wenn Sie Maven verwenden, fügen Sie dies in Ihre `pom.xml` ein:

```xml
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <version>1.18.32</version> <!-- Prüfen Sie auf die neueste Version -->
    <scope>provided</scope>
</dependency>
```

Für Gradle fügen Sie dies zu Ihrer `build.gradle` hinzu:

```gradle
provided 'org.projectlombok:lombok:1.18.32'
// Fügen Sie dies ebenfalls hinzu, um die Annotation Processing zu gewährleisten
annotationProcessor 'org.projectlombok:lombok:1.18.32'
```

Als Nächstes müssen Sie, wenn Sie eine IDE wie IntelliJ IDEA oder Eclipse verwenden, die Annotation Processing aktivieren und das Lombok-Plugin installieren:
- **IntelliJ**: Gehen Sie zu `Datei > Einstellungen > Plugins`, suchen Sie nach "Lombok", installieren Sie es und starten Sie die IDE neu. Stellen Sie dann sicher, dass `Annotation Processing aktivieren` unter `Einstellungen > Build, Ausführung, Bereitstellung > Compiler > Annotation Processors` aktiviert ist.
- **Eclipse**: Laden Sie die Lombok JAR von der offiziellen Website (projectlombok.org) herunter, führen Sie sie mit `java -jar lombok.jar` aus und folgen Sie den Anweisungen des Installers, um sie auf Ihre Eclipse-Installation zu verweisen.

Nun können Sie beginnen, Lombok-Annotationen in Ihrem Java-Code zu verwenden. Hier ist ein einfaches Beispiel:

```java
import lombok.Getter;
import lombok.Setter;
import lombok.NoArgsConstructor;
import lombok.AllArgsConstructor;

@Getter
@Setter
@NoArgsConstructor
@AllArgsConstructor
public class Person {
    private String name;
    private int age;
}
```

In diesem Beispiel:
- `@Getter` und `@Setter` generieren automatisch Getter- und Setter-Methoden für `name` und `age`.
- `@NoArgsConstructor` erstellt einen parameterlosen Konstruktor.
- `@AllArgsConstructor` erstellt einen Konstruktor mit allen Feldern als Parameter.

Wenn Sie dies kompilieren oder ausführen, verarbeitet Lombok diese Annotationen und fügt den Boilerplate-Code für Sie hinzu. Sie können es wie folgt testen:

```java
public class Main {
    public static void main(String[] args) {
        Person person = new Person("Alice", 30);
        System.out.println(person.getName()); // Gibt aus: Alice
        person.setAge(31);
        System.out.println(person.getAge()); // Gibt aus: 31
    }
}
```

Einige weitere nützliche, beliebte Annotationen:
- `@Data`: Eine Abkürzung für `@Getter`, `@Setter`, `@ToString`, `@EqualsAndHashCode` und einen Konstruktor für erforderliche Felder (wie finale Felder).
- `@Builder`: Bietet ein Builder-Pattern für die Objekterstellung.
- `@Slf4j`: Fügt einen Logger (z.B. von SLF4J) zu Ihrer Klasse hinzu.

Zum Beispiel mit `@Data` und `@Builder`:

```java
import lombok.Data;
import lombok.Builder;

@Data
@Builder
public class User {
    private String username;
    private String email;
}

// Verwendung
public class Main {
    public static void main(String[] args) {
        User user = User.builder()
                        .username("john_doe")
                        .email("john@example.com")
                        .build();
        System.out.println(user); // Gibt aus: User(username=john_doe, email=john@example.com)
    }
}
```

Beachten Sie:
- Prüfen Sie immer die neueste Version von Lombok auf Maven Central oder der offiziellen Website.
- Wenn Sie in einem Team arbeiten, stellen Sie sicher, dass alle Teammitglieder Lombok in ihrer IDE eingerichtet haben, um Verwirrung zu vermeiden, da der generierte Code nicht im Quelltext sichtbar ist.
