## Was ist PuTTY?

PuTTY ist ein vielseitiger Terminalemulator und Netzwerkclient, der von Simon Tatham entwickelt wurde, primär für Windows, aber auch für Linux und macOS verfügbar. Es unterstützt mehrere Protokolle, darunter:

- **SSH (Secure Shell)**: Für sicheren Remote-Zugriff auf Server.
- **Telnet**: Für unverschlüsselte Verbindungen (weniger verbreitet aufgrund von Sicherheitsbedenken).
- **Rlogin**: Für Remote-Login auf Unix-Systemen.
- **SCP und SFTP**: Für sichere Dateiübertragungen.
- **Raw Socket-Verbindungen**: Für direkte Netzwerkkommunikation.
- **Serielle Verbindungen**: Für die Interaktion mit Hardware wie Routern.

PuTTY ist leichtgewichtig, hochgradig konfigurierbar und enthält Tools wie PuTTYgen (zum Erzeugen von SSH-Schlüsseln) und PSFTP/PSCP (für Dateiübertragungen). Es wird häufig von Systemadministratoren, Entwicklern und IT-Fachleuten zur Verwaltung von Remote-Servern verwendet.[](https://en.wikipedia.org/wiki/PuTTY)

## PuTTY herunterladen und installieren

### Schritt 1: PuTTY herunterladen
1. Besuchen Sie die offizielle PuTTY-Website unter [www.putty.org](https://www.putty.org/) oder die Download-Seite unter [chiark.greenend.org.uk/~sgtatham/putty/latest.html](https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html).[](https://www.putty.org/)
2. Wählen Sie das passende Installationsprogramm:
   - **32-Bit oder 64-Bit MSI-Installer**: Wählen Sie `putty-<version>-installer.msi` basierend auf Ihrem System (64-Bit wird für moderne PCs empfohlen).[](https://its.gmu.edu/knowledge-base/how-to-install-putty-ssh-for-windows/)
   - **Einzelne Binärdateien**: Laden Sie `putty.exe`, `puttygen.exe`, `psftp.exe` usw. herunter, wenn Sie kein Installationsprogramm verwenden möchten.
3. Wenn Sie sich über Ihren Systemtyp unsicher sind, ist die 32-Bit-Version universell kompatibel.[](https://www.ssh.com/academy/ssh/putty/windows/install)

### Schritt 2: PuTTY installieren
1. Doppelklicken Sie auf die heruntergeladene `.msi`-Datei, um den PuTTY Setup-Assistenten zu starten.
2. Klicken Sie auf **Weiter** auf dem Begrüßungsbildschirm.
3. Akzeptieren Sie den standardmäßigen Zielordner (normalerweise `C:\Program Files\PuTTY\`) oder wählen Sie einen benutzerdefinierten Speicherort, dann klicken Sie auf **Weiter**.[](https://www.techtarget.com/searchsecurity/tutorial/How-to-use-PuTTY-for-SSH-key-based-authentication)
4. Wählen Sie die gewünschten Funktionen (z.B. Erstellen einer Desktop-Verknüpfung) und klicken Sie auf **Installieren**.[](https://www.ssh.com/academy/ssh/putty/windows/install)
5. Falls Sie von der Windows-Benutzerkontensteuerung dazu aufgefordert werden, klicken Sie auf **Ja**.
6. Sobald die Installation abgeschlossen ist, klicken Sie auf **Fertig stellen**.[](https://www.ssh.com/academy/ssh/putty/windows/install)

### Schritt 3: Installation überprüfen
- Wenn Sie eine Desktop-Verknüpfung erstellt haben, doppelklicken Sie auf das PuTTY-Symbol.
- Alternativ suchen Sie nach "PuTTY" im Windows-Startmenü oder navigieren Sie zu `C:\Program Files\PuTTY\putty.exe`.[](https://its.gmu.edu/knowledge-base/how-to-install-putty-ssh-for-windows/)

## Grundlegende Konfiguration und Verbindung zu einem Server

### PuTTY starten
1. Öffnen Sie PuTTY über das Startmenü oder die Desktop-Verknüpfung. Dies öffnet das Fenster **PuTTY-Konfiguration**, welches enthält:
   - Einen linken Bereich für Konfigurationskategorien.
   - Einen mittleren Bereich zur Eingabe der Verbindungsdetails.
   - Einen Bereich unten rechts zum Speichern von Sitzungsprofilen.[](https://www.ssh.com/academy/ssh/putty/windows)

### Verbindung zu einem Server herstellen
1. **Host-Details eingeben**:
   - Tragen Sie im Feld **Host Name (or IP address)** den Domainnamen des Servers (z.B. `students.example.edu`) oder die IP-Adresse (z.B. `192.168.1.1`) ein.
   - Stellen Sie sicher, dass der **Port** auf `22` für SSH (Standard) oder den von Ihrem Server-Administrator angegebenen Port gesetzt ist.[](https://www.wikihow.com/Use-Putty-in-Windows)
2. **Verbindungstyp auswählen**:
   - Wählen Sie **SSH** (Standard) für sichere Verbindungen. Andere Optionen sind Telnet, Rlogin, Raw oder Serial.[](https://www.wikihow.com/Use-Putty-in-Windows)
3. **Sitzung speichern** (optional):
   - Geben Sie einen Namen im Feld **Gespeicherte Sitzungen** ein (z.B. `MeinServer`).
   - Klicken Sie auf **Speichern**, um die Konfiguration für die zukünftige Verwendung zu speichern.[](https://www.cuit.columbia.edu/putty)
4. **Verbinden**:
   - Klicken Sie auf **Öffnen**, um die Verbindung zu initiieren.
   - Wenn Sie sich zum ersten Mal verbinden, erscheint möglicherweise eine **PuTTY-Sicherheitswarnung**, die darauf hinweist, dass der Host-Schlüssel des Servers nicht zwischengespeichert ist. Klicken Sie auf **Ja**, um dem Server zu vertrauen und fortzufahren.[](https://www.ssh.com/academy/ssh/putty/windows)

### Anmelden
- Geben Sie Ihren Benutzernamen und Ihr Passwort ein, wenn Sie im Terminalfenster dazu aufgefordert werden.
- Wenn eine schlüsselbasierte Authentifizierung erforderlich ist, konfigurieren Sie diese wie im Abschnitt [Verwendung von PuTTYgen](#verwendung-von-puttygen-für-die-ssh-schlüssel-authentifizierung) beschrieben.[](https://www.wikihow.com/Use-Putty-in-Windows)

## Erweiterte Konfigurationsoptionen

Das Konfigurationsfenster von PuTTY bietet umfangreiche Optionen zur Anpassung. Im Folgenden sind die wichtigsten Kategorien und Einstellungen aufgeführt:

### Sitzung
- **Gespeicherte Sitzungen**: Laden, speichern oder löschen Sie Sitzungsprofile für schnellen Zugriff.
- **Fenster beim Beenden schließen**: Wählen Sie, ob sich das Terminal nach dem Trennen der Verbindung automatisch schließt (`Immer`, `Nie` oder `Nur bei sauberem Beenden`).[](https://www.ssh.com/academy/ssh/putty/windows)

### Terminal
- **Auto-Wrap-Modus**: Aktivieren Sie diese Option, um lange Textzeilen in die nächste Zeile umzubrechen. Wenn deaktiviert, wird Text, der die Fensterbreite überschreitet, möglicherweise nicht angezeigt.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
- **Scrollback-Zeilen**: Erhöhen Sie die Anzahl der im Puffer gespeicherten Zeilen (Standard ist 200), um mehr Terminalausgabe anzuzeigen.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
- **Keepalives**: Setzen Sie einen Wert (in Sekunden), um Null-Pakete zu senden und Verbindungsabbrüche aufgrund von Inaktivität zu verhindern.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)

### Fenster
- **Erscheinungsbild**:
   - Ändern Sie die Schriftarteinstellungen (z.B. Größe, Stil) unter **Fenster > Erscheinungsbild**.
   - Ändern Sie die Farben unter **Fenster > Farben**, um die Lesbarkeit zu verbessern (stellen Sie ausreichenden Kontrast zwischen Vorder- und Hintergrund sicher).[](https://www.cuit.columbia.edu/putty)
- **Größenänderungsverhalten**: Passen Sie an, wie das Terminal auf Fenstergrößenänderungen reagiert (z.B. Schriftgröße skalieren oder Terminalabmessungen anpassen).[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)

### Verbindung
- **Proxy-Einstellungen**: Konfigurieren Sie Proxys, wenn Sie sich über ein Unternehmensnetzwerk verbinden.
- **Automatische Anmeldung**: Geben Sie einen Benutzernamen an, um die Anmeldeaufforderung zu umgehen.[](https://www.puttygen.com/download-putty)
- **SSH-Einstellungen**:
  - Wählen Sie die bevorzugte SSH-Protokollversion (SSH-2 wird aus Sicherheitsgründen empfohlen).
  - Wählen Sie Verschlüsselungschiffren (z.B. AES, 3DES) und Schlüsselaustauschalgorithmen.[](https://en.wikipedia.org/wiki/PuTTY)

### Sitzungen speichern und laden
1. Nach dem Konfigurieren der Einstellungen kehren Sie zur Kategorie **Sitzung** zurück.
2. Geben Sie einen Namen in das Feld **Gespeicherte Sitzungen** ein und klicken Sie auf **Speichern**.
3. Um eine gespeicherte Sitzung zu laden, wählen Sie sie aus der Liste aus, klicken Sie auf **Laden** und dann auf **Öffnen**.[](https://www.cuit.columbia.edu/putty)

## Verwendung von PuTTYgen für die SSH-Schlüssel-Authentifizierung

Die SSH-schlüsselbasierte Authentifizierung ist sicherer als die passwortbasierte Authentifizierung. PuTTYgen erzeugt Schlüsselpaare im `.ppk`-Format von PuTTY.

### Schritt 1: Schlüsselpaar erzeugen
1. Öffnen Sie **PuTTYgen** über das Startmenü oder `C:\Program Files\PuTTY\puttygen.exe`.
2. Wählen Sie einen Schlüsseltyp (z.B. **RSA**, Standard, oder **ECDSA**) und eine Bit-Länge (z.B. 2048 oder 4096 für RSA).
3. Klicken Sie auf **Generieren** und bewegen Sie Ihre Maus, um Zufälligkeit zu erzeugen, bis der Fortschrittsbalken vollständig ist.
4. Optional: Legen Sie eine **Passphrase** für zusätzliche Sicherheit fest.
5. Speichern Sie den privaten Schlüssel (`.ppk`-Datei) an einem sicheren Ort.
6. Kopieren Sie den öffentlichen Schlüssel, der im Feld "Public key for pasting into OpenSSH authorized_keys file" angezeigt wird.[](https://www.wikihow.com/Use-Putty-in-Windows)

### Schritt 2: Server konfigurieren
1. Verbinden Sie sich mit dem Remote-Server über die passwortbasierte Authentifizierung (oder eine andere Methode).
2. Erstellen oder bearbeiten Sie die Datei `~/.ssh/authorized_keys`:
   - Führen Sie `mkdir ~/.ssh && chmod 700 ~/.ssh` aus, um das `.ssh`-Verzeichnis zu erstellen.
   - Öffnen Sie die `authorized_keys`-Datei mit einem Texteditor (z.B. `nano ~/.ssh/authorized_keys`).
   - Fügen Sie den öffentlichen Schlüssel ein und speichern Sie die Datei.
   - Setzen Sie die Berechtigungen mit `chmod 600 ~/.ssh/authorized_keys`.[](https://www.techtarget.com/searchsecurity/tutorial/How-to-use-PuTTY-for-SSH-key-based-authentication)
3. Melden Sie sich vom Server ab.

### Schritt 3: PuTTY für die Schlüsselauthentifizierung konfigurieren
1. Öffnen Sie PuTTY und laden Sie Ihre gespeicherte Sitzung.
2. Navigieren Sie zu **Verbindung > SSH > Auth > Credentials**.
3. Klicken Sie auf **Durchsuchen** neben "Private key file for authentication" und wählen Sie Ihre `.ppk`-Datei aus.
4. Kehren Sie zur Kategorie **Sitzung** zurück, speichern Sie die aktualisierte Sitzung und klicken Sie auf **Öffnen**.
5. Geben Sie Ihre Passphrase (falls gesetzt) ein, wenn Sie dazu aufgefordert werden. Sie sollten sich jetzt ohne Passwort verbinden.[](https://www.wikihow.com/Use-Putty-in-Windows)

## Dateiübertragungen mit PSFTP und PSCP

PuTTY enthält **PSFTP** (für SFTP) und **PSCP** (für SCP) für sichere Dateiübertragungen.

### PSFTP verwenden
1. Öffnen Sie **PSFTP** über das Startmenü oder `C:\Program Files\PuTTY\psftp.exe`.
2. Geben Sie `open <Hostname>` ein (z.B. `open students.example.edu`) und drücken Sie die Eingabetaste.
3. Melden Sie sich mit Ihren Anmeldedaten oder Ihrem privaten Schlüssel an.
4. Verwenden Sie Befehle wie:
   - `ls` oder `dir`: Dateien auflisten.
   - `get <Dateiname>`: Eine Datei herunterladen.
   - `put <Dateiname>`: Eine Datei hochladen.
   - `cd <Verzeichnis>`: Verzeichnis wechseln.
   - `quit`: PSFTP beenden.[](https://www.wikihow.com/Use-Putty-in-Windows)

### PSCP verwenden
1. Öffnen Sie eine Eingabeaufforderung oder PowerShell.
2. Navigieren Sie zum PuTTY-Installationsverzeichnis (z.B. `cd C:\Program Files\PuTTY`).
3. Führen Sie Befehle wie folgt aus:
   - `pscp <lokale_Datei> <Benutzername>@<Hostname>:<Remote_Pfad>` zum Hochladen.
   - `pscp <Benutzername>@<Hostname>:<Remote_Datei> <lokaler_Pfad>` zum Herunterladen.
   - Fügen Sie `-i <private_key.ppk>` für die schlüsselbasierte Authentifizierung hinzu.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)

### Hinweis
- PSFTP ist interaktiv, während PSCP auf der Befehlszeile basiert.
- Für grafische Dateiübertragungen sollten Sie Alternativen wie WinSCP in Betracht ziehen, die PuTTY-Sitzungen importieren können.[](https://www.ssh.com/academy/ssh/putty)

## Portweiterleitung und Tunneling

PuTTY unterstützt **SSH-Tunneling** (Portweiterleitung), um sicher auf Dienste auf Remote-Servern zuzugreifen.

### Lokale Portweiterleitung
1. Gehen Sie in PuTTY zu **Verbindung > SSH > Tunnels**.
2. Geben Sie einen **Quellport** (z.B. `8080`) auf Ihrem lokalen Rechner ein.
3. Geben Sie das **Ziel** (z.B. `remote_host:80`) für den Remote-Dienst ein.
4. Wählen Sie **Lokal** und klicken Sie auf **Hinzufügen**.
5. Speichern Sie die Sitzung und verbinden Sie sich.
6. Greifen Sie lokal auf den Dienst zu (z.B. `http://localhost:8080`).[](https://www.ssh.com/academy/ssh/putty/windows)

### Remote-Portweiterleitung
- Ähnlich wie die lokale Weiterleitung, aber wählen Sie **Remote** und geben Sie einen Port auf dem Remote-Server an, der an Ihren lokalen Rechner weitergeleitet werden soll.

### Dynamische Portweiterleitung
- Wählen Sie **Dynamisch** und geben Sie einen lokalen Port an (z.B. `8080`). Dies erstellt einen SOCKS-Proxy für dynamisches Routing (nützlich für das Browsen über den Server).[](https://rushax.com/what-is-putty-a-comprehensive-guide/)

## X11-Forwarding für grafische Anwendungen

PuTTY unterstützt **X11-Forwarding**, um grafische Anwendungen auf einem Remote-Server auszuführen und sie lokal anzuzeigen.

1. Installieren Sie einen X-Server unter Windows (z.B. **XMing**, eine kostenlose Option).
2. Gehen Sie in PuTTY zu **Verbindung > SSH > X11**.
3. Aktivieren Sie das Kontrollkästchen **Enable X11 forwarding**.
4. Setzen Sie **X display location** auf `localhost:0.0`.
5. Speichern Sie die Sitzung und verbinden Sie sich.
6. Führen Sie grafische Befehle (z.B. `xterm`) auf dem Server aus, und die Oberfläche wird lokal über XMing angezeigt.[](https://www.ssh.com/academy/ssh/putty/windows)

## Fehlerbehebung bei häufigen Problemen

1. **Fehler "Zugriff verweigert"**:
   - Überprüfen Sie Benutzername, Passwort oder privaten Schlüssel.
   - Stellen Sie sicher, dass der SSH-Zugriff auf dem Server aktiviert ist (z.B. überprüfen Sie die Einstellungen im Hosting-Control-Panel).[](https://www.hostinger.com/uk/tutorials/how-to-use-putty-ssh)
   - Prüfen Sie, ob der Server nur schlüsselbasierte Authentifizierung erfordert.

2. **"Verbindung durch Peer zurückgesetzt"**:
   - Aktivieren Sie **Keepalives** in den **Verbindungs**-Einstellungen, um Timeouts zu verhindern.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
   - Überprüfen Sie Firewall- oder Netzwerkgeräteeinstellungen, die inaktive Verbindungen schließen könnten.

3. **"Host-Schlüssel des Servers nicht zwischengespeichert"**:
   - Dies ist bei Erstverbindungen normal. Klicken Sie auf **Ja**, um den Schlüssel zu cachen. Wenn dies wiederholt auftritt, überprüfen Sie die Authentizität des Servers, um einen Man-in-the-Middle-Angriff auszuschließen.[](https://www.ssh.com/academy/ssh/putty/windows)

4. **Warnung vor fehlgeschlagenen Anmeldeversuchen**:
   - Setzen Sie Ihr Passwort zurück oder prüfen Sie auf Brute-Force-Schutz auf dem Server.
   - Kontaktieren Sie Ihren Server-Administrator, falls Sie gesperrt sind.[](https://www.hostinger.com/uk/tutorials/how-to-use-putty-ssh)

5. **PuTTY-Fenster schließt sich unerwartet**:
   - Überprüfen Sie die Server-Protokolle auf Fehler (z.B. falsche Berechtigungen für `~/.ssh/authorized_keys`).
   - Stellen Sie sicher, dass der richtige private Schlüssel geladen ist.[](https://www.hostinger.com/uk/tutorials/how-to-use-putty-ssh)

## Tipps und Best Practices

- **Sitzungen speichern**: Speichern Sie immer Sitzungsprofile, um das erneute Eingeben von Einstellungen zu vermeiden.[](https://www.cuit.columbia.edu/putty)
- **Schlüsselbasierte Authentifizierung verwenden**: Sie ist sicherer als Passwörter und verhindert Brute-Force-Angriffe.[](https://www.techtarget.com/searchsecurity/tutorial/How-to-use-PuTTY-for-SSH-key-based-authentication)
- **Private Schlüssel sichern**: Bewahren Sie `.ppk`-Dateien an einem sicheren Ort auf und verwenden Sie eine Passphrase.[](https://www.ssh.com/academy/ssh/putty)
- **Erscheinungsbild anpassen**: Passen Sie Schriftarten und Farben für eine bessere Lesbarkeit an, insbesondere bei langen Sitzungen.[](https://www.cuit.columbia.edu/putty)
- **Keepalives aktivieren**: Verhindern Sie Verbindungsabbrüche während Leerlaufphasen.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
- **Konfigurationen sichern**: Exportieren Sie gespeicherte Sitzungen aus der Windows-Registrierung (`HKEY_CURRENT_USER\Software\SimonTatham\PuTTY\Sessions`) für Backup oder Migration.
- **Regelmäßig aktualisieren**: Laden Sie die neueste PuTTY-Version herunter, um Sicherheitspatches und neue Funktionen zu erhalten (z.B. elliptische Kurvenkryptographie in neueren Versionen).[](https://www.ssh.com/academy/ssh/putty)

## Alternativen zu PuTTY

Während PuTTY zuverlässig ist, bieten moderne Alternativen erweiterte Funktionen:
- **OpenSSH**: In Windows 10/11, Linux und macOS integriert. Befehlszeilenbasiert, aber robust.[](https://phoenixnap.com/kb/putty-alternative)
- **WinSCP**: Ein grafischer Dateiübertragungs-Client, der sich in PuTTY integrieren lässt.[](https://www.ssh.com/academy/ssh/putty)
- **MobaXterm**: Bietet eine Oberfläche mit Tabs, integrierten X-Server und SFTP-Unterstützung.[](https://rushax.com/what-is-putty-a-comprehensive-guide/)
- **Bitvise SSH Client**: Unterstützt GUI, Skripting und One-Click-Tunneling.[](https://phoenixnap.com/kb/putty-alternative)
- **Termius**: Plattformübergreifend mit SSH-Schlüsselverwaltung und einer modernen Oberfläche.[](https://phoenixnap.com/kb/putty-alternative)
