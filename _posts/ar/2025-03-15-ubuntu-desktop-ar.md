يبدو أن نظام Ubuntu الخاص بك قد لا يكون لديه واجهة مستخدم رسومية (GUI) مثبتة، مما يفسر سبب عدم رؤية متصفح الملفات (غالبًا ما يُطلق عليه مدير الملفات في Linux) أو أي نوع من "الباحث". قد يحدث هذا إذا قمت بتثبيت إصدار خادم من Ubuntu، أو اخترت تثبيتًا بحد أدنى، أو إذا كانت هناك مشكلة تمنع تحميل الواجهة الرسومية. لا تقلق—يمكننا إصلاح هذا عن طريق تثبيت واجهة رسومية مثل GNOME (بيئة سطح المكتب الافتراضية لأوبنتو) أو خيار آخر إذا كنت تفضل. إليك كيفية القيام بذلك خطوة بخطوة.

### الخطوة 1: تحديث نظامك
قبل تثبيت أي شيء، من الجيد التأكد من أن قائمة الحزم لديك محدثة وأن نظامك تم تحديثه بالكامل. افتح طرفية وقم بتشغيل هذه الأوامر:

```bash
sudo apt update
```
هذا يقوم بتحديث قائمة الحزم المتاحة.

```bash
sudo apt upgrade
```
هذا يقوم بتحديث جميع الحزم المثبتة حاليًا إلى أحدث إصداراتها. إذا تمت مطالبتك، اضغط `Y` للتأكيد.

### الخطوة 2: تثبيت بيئة سطح المكتب GNOME
بما أنك ذكرت رغبتك في واجهة رسومية "مثل GNOME أو غيرها"، فلنبدأ بـ GNOME، حيث إنه الافتراضي لسطح مكتب Ubuntu ويتضمن مدير ملفات يسمى Nautilus. لتثبيته، قم بتشغيل:

```bash
sudo apt install ubuntu-desktop
```
يقوم هذا الأمر بتثبيت بيئة سطح مكتب GNOME الكاملة، جنبًا إلى جنب مع التطبيقات القياسية مثل مدير الملفات ومتصفح الويب والمزيد. قد تستغرق عملية التثبيت بضع دقائق، اعتمادًا على سرعة الإنترنت والنظام لديك. إذا طُلب منك اختيار مدير عرض (مثل GDM أو LightDM)، يمكنك عادةً الالتزام بالخيار الافتراضي (GDM لـ GNOME) بالضغط على Enter.

### الخطوة 3: إعادة تشغيل النظام
بعد انتهاء التثبيت، أعد تشغيل النظام لبدء الواجهة الرسومية:

```bash
sudo reboot
```
بمجرد إعادة تشغيل النظام، يجب أن ترى شاشة تسجيل دخول رسومية. سجّل الدخول، وسيكون لديك سطح مكتب يعمل بكامل طاقته مع مدير ملفات (Nautilus) يسمح لك بتصفح الملفات.

### ماذا لو لم تنجح؟
إذا لم تظهر الواجهة الرسومية بعد إعادة التشغيل، فمن المحتمل وجود مشكلة في مدير العرض أو برامج تشغيل الرسومات. يمكنك محاولة بدء مدير العرض يدويًا للاختبار:

```bash
sudo systemctl start gdm
```
إذا ظهرت شاشة تسجيل الدخول، فهذا رائع! لضمان بدئها تلقائيًا في كل إقلاع، قم بتشغيل:

```bash
sudo systemctl enable gdm
```
إذا لم تظهر الواجهة الرسومية بعد، فتحقق من الأخطاء عن طريق مراجعة سجلات النظام باستخدام:

```bash
journalctl -b
```
ابحث عن الرسائل المتعلقة بـ `gdm` أو `Xorg` أو برامج تشغيل الرسومات، ولا تتردد في طلب المزيد من المساعدة مع هذه التفاصيل.

### خيارات واجهة مستخدم رسومية بديلة
إذا كنت تفضل واجهة رسومية مختلفة بدلاً من GNOME، فإليك بعض البدائل الشائعة:
-   **KDE Plasma** (أنيقة وقابلة للتخصيص):
    ```bash
    sudo apt install kubuntu-desktop
    ```
-   **XFCE** (خفيفة الوزن وسريعة):
    ```bash
    sudo apt install xubuntu-desktop
    ```
-   **LXDE** (خفيفة الوزن جدًا، جيدة للأجهزة القديمة):
    ```bash
    sudo apt install lubuntu-desktop
    ```
بعد تثبيت أي من هذه، أعد التشغيل باستخدام `sudo reboot`، وستحصل على بيئة سطح مكتب مع مدير ملفات (مثل Dolphin لـ KDE، أو Thunar لـ XFCE).

### لماذا لم تكن لدي واجهة رسومية؟
قد لا تكون قد رأيت واجهة رسومية لأن:
-   قمت بتثبيت Ubuntu Server، والذي لا يتضمن سطح مكتب افتراضيًا.
-   اخترت تثبيتًا بحد أدنى أثناء الإعداد، متخطيًا الواجهة الرسومية.
-   هناك احتمال نادر أن الواجهة الرسومية مثبتة ولكنها لا تبدأ بسبب مشكلة في التكوين أو الأجهزة.

يجب أن يؤدي تثبيت `ubuntu-desktop` كما هو موضح إلى حل هذه المشكلة إذا كانت الواجهة الرسومية مفقودة. إذا كنت تشك في وجود مشكلة في واجهة رسومية موجودة، فأخبرني، ويمكننا استكشاف الأخطاء وإصلاحها بشكل أكبر!

### الملخص
لتثبيت GNOME والحصول على متصفح ملفات على نظام Ubuntu الخاص بك:
1. `sudo apt update`
2. `sudo apt upgrade`
3. `sudo apt install ubuntu-desktop`
4. `sudo reboot`

ستحصل قريبًا على واجهة رسومية مع مدير ملفات جاهز للاستخدام. استمتع بسطح مكتب Ubuntu الخاص بك!
