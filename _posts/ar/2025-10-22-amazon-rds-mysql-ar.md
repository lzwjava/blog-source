Amazon RDS (Relational Database Service) for MySQL هي خدمة قاعدة بيانات مُدارة تجعل من السهل إعداد وتشغيل وتوسيع قواعد بيانات MySQL في السحابة. فهي تتولى المهام الروتينية مثل النسخ الاحتياطي والتحديثات والتوسع، حتى تتمكن من التركيز على تطبيقك. يرشدك هذا الدليل خلال الأساسيات: إنشاء مثيل قاعدة بيانات MySQL، والاتصال به، وإجراء عمليات بسيطة. سنستخدم وحدة تحكم إدارة AWS للتبسيط.

**ملاحظة حول التكاليف:** تقدم AWS Free Tier استخدامًا محدودًا للحسابات الجديدة، ولكنك ستتحمل رسومًا للموارد التي تتجاوز ذلك الحد. احرص دائمًا على حذف الموارد عند الانتهاء لتجنب الفواتير غير المتوقعة. للاستخدام في بيئة الإنتاج، اتبع أفضل ممارسات الأمان مثل استخدام VPCs والتشفير والوصول بأقل صلاحيات.

## المتطلبات الأساسية
- حساب AWS (سجل في [aws.amazon.com](https://aws.amazon.com) إذا لزم الأمر).
- معرفة أساسية بوحدة تحكم AWS وMySQL.
- لاختبار الاتصال الآمن، سننشئ مثيل EC2 في نفس VPC (السحابة الخاصة الافتراضية). حدد عنوان IP العام الخاص بك (على سبيل المثال، عبر [checkip.amazonaws.com](https://checkip.amazonaws.com)) للوصول عبر SSH.
- اختر منطقة AWS قريبة منك (مثل US East (N. Virginia)).

**أفضل ممارسة:** استخدم مثيل قاعدة بيانات خاص في VPC لتقييد الوصول إلى الموارد الموثوقة فقط. مكّن SSL/TLS للاتصالات المشفرة.

## الخطوة 1: إنشاء مثيل EC2 للاتصال
يقوم هذا بإعداد خادم Linux بسيط للاتصال بمثيل قاعدة البيانات الخاص بك.

1.  سجل الدخول إلى [وحدة تحكم إدارة AWS](https://console.aws.amazon.com) وافتح وحدة تحكم EC2.
2.  اختر منطقتك.
3.  انقر فوق **Launch instance**.
4.  قم بالتكوين:
    -   **الاسم:** `ec2-database-connect`.
    -   **AMI:** Amazon Linux 2023 (مؤهل للطبقة المجانية).
    -   **نوع المثيل:** t3.micro (مؤهل للطبقة المجانية).
    -   **زوج المفاتيح:** أنشئ أو اختر زوج مفاتيح موجود للوصول عبر SSH.
    -   **إعدادات الشبكة:** تحرير > اسمح بمرور SSH من **My IP** (أو عنوان IP المحدد الخاص بك، مثل `192.0.2.1/32`). تجنب `0.0.0.0/0` للأمان.
    -   اترك الإعدادات الافتراضية للتخزين والعلامات.
5.  انقر فوق **Launch instance**.
6.  لاحظ معرف المثيل، واسم النطاق لعنوان IPv4 العام، واسم زوج المفاتيح من تفاصيل المثيل.
7.  انتظر حتى تظهر الحالة **Running** (من 2 إلى 5 دقائق).

**نصيحة أمنية:** قيد الوصول عبر SSH لعنوان IP الخاص بك فقط. قم بتنزيل ملف زوج المفاتيح (.pem) بطريقة آمنة.

## الخطوة 2: إنشاء مثيل قاعدة بيانات MySQL
استخدم "Easy create" للإعداد السريع بالإعدادات الافتراضية.

1.  افتح [وحدة تحكم RDS](https://console.aws.amazon.com/rds/).
2.  اختر نفس منطقة مثيل EC2 الخاص بك.
3.  في جزء التنقل، انقر فوق **Databases** > **Create database**.
4.  اختر **Easy create**.
5.  ضمن **Configuration**:
    -   نوع المحرك: **MySQL**.
    -   القوالب: **Free tier** (أو **Sandbox** للحسابات المدفوعة).
    -   معرّف مثيل قاعدة البيانات: `database-test1` (أو الاسم الذي تختاره).
    -   اسم المستخدم الرئيسي: `admin` (أو مخصص).
    -   كلمة المرور الرئيسية: أنشئ تلقائيًا أو عيّن كلمة مرور قوية (سجلها في مكان آمن).
6.  (اختياري) ضمن **Connectivity**، اختر **Connect to an EC2 compute resource** واختر مثيل EC2 الخاص بك لتسهيل الإعداد.
7.  انقر فوق **Create database**.
8.  اعرض النافذة المنبثقة لبيانات الاعتماد (اسم المستخدم/كلمة المرور) — احفظها، حيث لا يمكن استرداد كلمة المرور لاحقًا.
9.  انتظر حتى تتغير الحالة إلى **Available** (حتى 10-20 دقيقة). لاحظ **Endpoint** (اسم النطاق) والمنفذ (الافتراضي: 3306) من علامة التبويب **Connectivity & security**.

**أفضل ممارسة:** للاستخدام في بيئة الإنتاج، استخدم "Standard create" لتخصيص VPC، والنسخ الاحتياطي (تمكين التلقائي)، والتخزين. مكّن حماية الحذف و multi-AZ لتوفر عالي.

## الخطوة 3: الاتصال بمثيل قاعدة البيانات
اتصل من مثيل EC2 الخاص بك باستخدام عميل MySQL.

1.  اتصل عبر SSH بمثيل EC2 الخاص بك:
    ```
    ssh -i /path/to/your-key-pair.pem ec2-user@your-ec2-public-dns
    ```
    (استبدل بالتفاصيل الخاصة بك؛ على سبيل المثال، `ssh -i ec2-database-connect-key-pair.pem ec2-user@ec2-12-345-678-90.compute-1.amazonaws.com`.)

2.  على مثيل EC2، حدّث الحزم:
    ```
    sudo dnf update -y
    ```

3.  ثبّت عميل MySQL:
    ```
    sudo dnf install mariadb105 -y
    ```

4.  اتصل بقاعدة البيانات:
    ```
    mysql -h your-db-endpoint -P 3306 -u admin -p
    ```
    أدخل كلمة المرور الرئيسية عند الطلب.

إذا نجح الاتصال، سترى موجه MySQL (`mysql>`).

**استكشاف الأخطاء وإصلاحها:** تأكد من أن مجموعات الأمان تسمح بمرور الحركة الواردة على المنفذ 3306 من مثيل EC2. للاتصالات الخارجية، اجعل قاعدة البيانات عامة (غير موصى به) أو استخدم خوادم Bastion / VPN.

**نصيحة أمنية:** استخدم `--ssl-mode=REQUIRED` للاتصالات المشفرة: `mysql -h endpoint -P 3306 -u admin -p --ssl-mode=REQUIRED`.

## الخطوة 4: الاستخدام الأساسي
بمجرد الاتصال، يمكنك تشغيل أوامر SQL. أمثلة:

-   عرض قواعد البيانات: `SHOW DATABASES;`
-   إنشاء قاعدة بيانات نموذجية: `CREATE DATABASE myapp;`
-   استخدامها: `USE myapp;`
-   إنشاء جدول:
    ```
    CREATE TABLE users (
      id INT AUTO_INCREMENT PRIMARY KEY,
      name VARCHAR(50),
      email VARCHAR(100)
    );
    ```
-   إدراج بيانات: `INSERT INTO users (name, email) VALUES ('Alice', 'alice@example.com');`
-   استعلام: `SELECT * FROM users;`
-   خروج: `EXIT;`

من التطبيقات (مثل Python، Node.js)، استخدم برامج تشغيل MySQL القياسية مع Endpoint والمنفذ واسم المستخدم وكلمة المرور كسلاسل اتصال.

**أفضل ممارسة:** راقب الأداء عبر CloudWatch، ومكّن النسخ الاحتياطي التلقائي، وقم بالتوسع رأسيًا (حجم المثيل) أو أفقيًا (نسخ للقراءة) حسب الحاجة.

## التنظيف: حذف الموارد
لإيقاف الرسوم:

1.  **حذف مثيل EC2:** وحدة تحكم EC2 > حدد المثيل > **Instance state** > **Terminate** > تأكيد.
2.  **حذف مثيل قاعدة البيانات:** وحدة تحكم RDS > حدد قاعدة البيانات > **Actions** > **Delete** > ألغِ تحديد إنشاء لقطة > تأكيد.

تحقق من وحدة تحكم الفوترة في AWS لأي موارد قيد التشغيل.

لمواضيع أكثر تقدمًا مثل عمليات الترحيل، ضبط الأداء، أو استخدام API/CLI، راجع وثائق AWS.
