Gradle هو أداة قوية لأتمتة البناء تُستخدم بشكل شائع لمشاريع Java وKotlin وAndroid، لكنها مرنة بما يكفي للغات الأخرى أيضًا. إليك دليلًا مناسبًا للمبتدئين للبدء واستخدام Gradle بفعالية:

---

### 1. **تثبيت Gradle**
قبل أن تتمكن من استخدام Gradle، تحتاج إلى تثبيته:
- **نظامي Mac/Linux (باستخدام Homebrew)**:
  - شغّل `brew install gradle` في طرفيتك.
- **نظام Windows (أو التثبيت اليدوي)**:
  - نزِّل ثنائي Gradle من [gradle.org](https://gradle.org/install/).
  - استخرجه إلى مجلد (مثل `C:\Gradle` أو `/opt/gradle`).
  - أضف دليل `bin` الخاص بـ Gradle (مثل `C:\Gradle\gradle-8.x\bin`) إلى متغير بيئة PATH في نظامك.
- **التحقق من التثبيت**:
  - افتح طرفية وشغّل `gradle -v`. يجب أن ترى إصدار Gradle والتفاصيل الخاصة به.

بدلاً من ذلك، إذا كنت تعمل على مشروع به **Gradle Wrapper** (شائع في المشاريع المشتركة)، فلن تحتاج إلى تثبيت Gradle بشكل عام—ما عليك سوى استخدام `./gradlew` (Linux/Mac) أو `gradlew.bat` (Windows) في دليل المشروع.

---

### 2. **فهم أساسيات Gradle**
يستخدم Gradle نصًا **Groovy** أو **Kotlin** (عادة `build.gradle` أو `build.gradle.kts`) لتعريف كيفية بناء مشروعك. إليك ما تحتاج إلى معرفته:
- **المشاريع**: يمكن أن يحتوي بناء Gradle على مشروع واحد أو أكثر (مثل تطبيق واحد أو إعداد متعدد الوحدات).
- **المهام**: هذه هي الإجراءات التي ينفذها Gradle، مثل `compileJava` أو `test` أو `build`.
- **التبعيات**: يدير Gradle المكتبات التي يحتاجها مشروعك (مثلًا من Maven Central أو مستودع Google).

---

### 3. **إنشاء مشروع Gradle بسيط**
لنقم بإعداد مشروع Java أساسي لرؤية Gradle في العمل:
1. **إنشاء مجلد المشروع**:
   - أنشئ دليلاً (مثل `my-gradle-project`) وانتقل إليه في طرفيتك.
2. **تهيئة Gradle**:
   - شغّل `gradle init`.
   - اتبع المطالبات: اختر "application"، و"Java"، و"Groovy" (أو Kotlin) لبرنامج بناء النص.
   - يؤدي هذا إلى إنشاء هيكل أساسي مع ملف `build.gradle` ونموذج للكود.
3. **استكشاف الملف `build.gradle` المُنشأ**:
   ```groovy
   plugins {
       id 'java'
       id 'application'
   }

   repositories {
       mavenCentral()
   }

   dependencies {
       implementation 'org.slf4j:slf4j-api:1.7.36'
   }

   application {
       mainClass = 'com.example.App'  // اضبط بناءً على الحزمة الخاصة بك
   }
   ```
   - `plugins`: يضيف دعمًا لـ Java وتشغيل التطبيق.
   - `repositories`: المكان الذي يبحث فيه Gradle عن التبعيات (مثل Maven Central).
   - `dependencies`: المكتبات التي يستخدمها مشروعك.
   - `application`: يحدد الفئة الرئيسية للتشغيل.

4. **تشغيل المهام**:
   - بناء المشروع: `gradle build`.
   - تشغيل التطبيق: `gradle run`.
   - سرد المهام المتاحة: `gradle tasks`.

---

### 4. **أوامر Gradle الشائعة**
إليك بعض الأوامر التي ستستخدمها غالبًا:
- `gradle build`: يترجم ويحزم مشروعك.
- `gradle clean`: يحذف دليل `build` للبدء من جديد.
- `gradle test`: يشغّل اختبارات مشروعك.
- `gradle dependencies`: يعرض شجرة التبعيات.
- `./gradlew <task>`: يستخدم Gradle Wrapper الخاص بالمشروع بدلاً من التثبيت العام.

---

### 5. **إضافة التبعيات**
لاستخدام المكتبات الخارجية، قم بتحرير `build.gradle`. على سبيل المثال، لإضافة Gson (مكتبة تحليل JSON):
```groovy
dependencies {
    implementation 'com.google.code.gson:gson:2.10.1'
}
```
ثم شغّل `gradle build` لتنزيلها.

---

### 6. **تخصيص البناء الخاص بك**
يمكنك إضافة مهام مخصصة أو تعديل الإعدادات في `build.gradle`. مثال على مهمة مخصصة:
```groovy
task hello {
    doLast {
        println 'Hello, Gradle!'
    }
}
```
شغّلها باستخدام `gradle hello`.

---

### 7. **استخدام Gradle Wrapper (موصى به للفرق)**
يضمن Gradle Wrapper أن الجميع يستخدمون نفس إصدار Gradle:
- إنشاؤه: `gradle wrapper`.
- استخدامه: `./gradlew build` (أو `gradlew.bat build` على Windows).

---

### 8. **نصائح للنجاح**
- **التوثيق**: تحقق من [الوثائق الرسمية لـ Gradle](https://docs.gradle.org/current/userguide/userguide.html) للتعمق أكثر.
- **ذاكرة التخزين المؤقت**: يقوم Gradle بتخزين التبعيات ومخرجات المهام مؤقتًا، لذا تصبح عمليات البناء أسرع بمرور الوقت.
- **الإضافات**: استكشف الإضافات (مثل تلك الخاصة بـ Spring Boot أو Android أو جودة الكود) لتوسيع الوظائف.

---

### مثال: بناء وتشغيل تطبيق Java
إذا كان ملف `src/main/java/com/example/App.java` يبدو هكذا:
```java
package com.example;
public class App {
    public static void main(String[] args) {
        System.out.println("Hello from Gradle!");
    }
}
```
شغّل `gradle run`، وسترى الناتج.
