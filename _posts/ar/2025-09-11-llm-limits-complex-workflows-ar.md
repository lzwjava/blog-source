أنت محق تمامًا! نماذج اللغة الكبيرة (بما في ذلك أنا) تتفوق في **المهام الصغيرة والمركزة**—مثل شرح مفهوم واحد، أو إنشاء مقتطف كود قصير، أو تصحيح خطأ محدد—ولكنها تواجه صعوبة في **سير العمل المعقدة متعددة الخطوات** التي تتضمن:

1.  **تهيئات متعددة مترابطة** (مثل ملفات Maven `pom.xml`، وسيطات وكيل Jacoco، تكامل اختبارات Python).
2.  **تبعيات ذات حالة** (مثل "قم أولاً بتحديث `module-A/pom.xml`، ثم شغل `mvn test` بهذه الأعلام، ثم ادمج التقارير").
3.  **تفاعلات بين الأنظمة** (مثل Java + Python + أدوات CI مثل Jenkins/GitHub Actions).
4.  **التبديل بين السياقات** (مثل "انسَ الآن جزء Java—كيف يمكننا تحليل Jacoco XML في Python؟").

---

### **لماذا تفشل نماذج اللغة الكبيرة في سير العمل المعقدة**
1.  **لا يوجد ذاكرة مستمرة**: نحن لا "نتذكر" الخطوات السابقة إلا إذا ذكرتنا بها صراحةً (وحتى في هذه الحالة، نطاقات السياق محدودة).
2.  **التفكير الخطي**: ننشئ الردود خطوة بخطوة لكن لا يمكننا تتبع التبعيات ديناميكيًا (مثل "هل قمت بتشغيل `mvn clean` قبل الوكيل؟").
3.  **التحسين المفرط للإجابات الفردية**: تم تدريبنا على إعطاء إجابة *واحدة* كاملة، وليس توجيهًا تكراريًا.
4.  **لا توجد بيئة تنفيذ**: لا يمكننا تشغيل أوامر للتحقق مما إذا كان `jacoco:prepare-agent` قد عمل بالفعل في إعداداتك.

### **نهج أفضل: "قسّم وتحقق"**
لمشاكل مثل **Jacoco + Maven متعدد الوحدات + اختبارات Python**، قم بتقسيمها إلى **خطوات ذرية** وتحقق من كل واحدة. مثال:

#### **الخطوة 1: التحقق من إعداد وكيل Jacoco (جانب Java)**
```bash
# في وحدة واحدة (مثل module-A):
mvn clean test -Pcoverage
```
-   **تحقق**: هل الملف `target/site/jacoco/index.html` موجود؟
-   **تصحيح**: إذا لم يكن موجودًا، شارع مقتطف `pom.xml` الخاص بـ Jacoco وسجل الخطأ.

#### **الخطوة 2: إنشاء تقرير مجمع (متعدد الوحدات)**
```bash
mvn jacoco:report-aggregate
```
-   **تحقق**: هل تم إنشاء `target/site/jacoco-aggregate/index.html`؟
-   **تصحيح**: إذا كان مفقودًا، تحقق من مسارات `<module>` في ملف `pom.xml` الرئيسي.

#### **الخطوة 3: التصدير لـ Python (XML/CSV)**
```bash
# تحويل Jacoco الثنائي إلى XML
java -jar jacoco-cli.jar report jacoco.exec --xml output.xml
```
-   **تحقق**: هل يحتوي `output.xml` على وسوم `<package>`؟
-   **تصحيح**: إذا كان فارغًا، تأكد من أن الاختبارات قد شغلت مع الوكيل (`-javaagent:jacocoagent.jar`).

#### **الخطوة 4: التكامل مع Python**
```python
import xml.etree.ElementTree as ET
tree = ET.parse("output.xml")
coverage = tree.find(".//counter[@type='LINE'][@covered]")
print(f"Covered lines: {coverage.get('covered')}/{coverage.get('missed')}")
```
-   **تحقق**: هل ينتج النصيخ الأرقام المتوقعة؟
-   **تصحيح**: إذا لم يحدث ذلك، شارع بنية XML وخطأ Python.

### **كيفية العمل مع نماذج اللغة الكبيرة بفعالية**
1.  **عزل المشكلة**:
    - *"ها هو ملف `pom.xml` الخاص بي للوحدة module-A. لماذا لا يضيف `jacoco:prepare-agent` وسيط VM؟"*
    - *(أفضل من: "ساعدني في إعداد Jacoco لـ 5 وحدات + Python.")*

2.  **شارك الحد الأدنى من السياق القابل للتكرار**:
    - سجلات الأخطاء، مقاطع التهيئة (وليس الملفات بأكملها)، و**ما جربته**.

3.  **التصحيح التكراري**:
    - **أنت**: "الخطوة 1 فشلت مع `No tests run`. ها هو السجل."
    - **النموذج اللغوي**: "أضف أهداف `<test>` إلى `maven-surefire-plugin` الخاص بك."
    - **أنت**: "الآن الخطوة 2 تعمل، لكن التقرير فارغ..."

4.  **استخدمنا في "المهام الجزئية"**:
    - *"اكتب نصيحة Python لتحليل Jacoco XML والفشل إذا كانت التغطية < 80%."*
    - *"اشرح وسم `<execution>` في Maven لـ Jacoco."*

### **أدوات لأتمتة سير العمل المعقدة**
بما أن نماذج اللغة الكبيرة لا يمكنها إدارة الحالة، استخدم:
-   **نصوص Shell** (مثل `setup_coverage.sh` لربط أوامر Maven/Python).
-   **Makefiles** (عرّف `make test-coverage` لتشغيل جميع الخطوات).
-   **قوالب CI** (أنابيب GitHub Actions/Jenkins بخطوات واضحة).
-   **Docker** (حاوِئ البيئة لتجنب مشاكل "يعمل على جهازي").

### **الخلاصة الرئيسية**
**نماذج اللغة الكبيرة مثل صفحة `man` فائقة القوة أو Stack Overflow على منشطات**—رائعة في **الأجزاء الفردية**، ولكن ليس لتنسيق الأنظمة بأكملها. للمهام المعقدة:
1.  **قسّمها**.
2.  **تحقق من كل خطوة**.
3.  **اطرح أسئلة مستهدفة**.

هل تريد مني الخوض بشكل أعمق في أي من الخطوات المذكورة أعلاه (مثل تحليل Jacoco XML في Python أو إعدادات Maven متعددة الوحدات)؟
