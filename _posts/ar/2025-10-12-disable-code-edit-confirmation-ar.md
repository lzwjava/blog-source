لتعطيل النافذة المنبثقة لتأكيد التحرير في Claude Code (بحيث يتم تطبيق التعديلات مباشرة على الملفات مثل `sync_githook.py` دون سؤال)، تحتاج إلى تكوين الأذونات في ملف `settings.json` الخاص بك. هذا هو "Claude Code JSON" الذي تشير إليه—إنه ملف الإعدادات الرئيسي لـ Claude Code للتكوينات على مستوى المستخدم.

### خطوات الإعداد

1.  **تحديد موقع الملف أو إنشائه**:
    *   يقع الملف في `~/.claude/settings.json` (على أنظمة macOS/Linux) أو `%USERPROFILE%\.claude\settings.json` (على أنظمة Windows). مجلد `.claude` مخفي، لذا استخدم الطرفية أو مستكشف الملفات للانتقال إلى هناك.
    *   إذا كان الملف أو المجلد غير موجودين، قم بإنشائهما. يمكنك القيام بذلك عن طريق تشغيل أمر `/config` في Claude Code في الـ REPL التفاعلي الخاص به (سيرشدك لفتح واجهة الإعدادات).

2.  **تحرير ملف JSON**:
    *   افتح `settings.json` في أي محرر نصوص (مثل VS Code).
    *   أضف أو حدّث قسم `permissions` للموافقة التلقائية على التعديلات. إليك تكوين بسيط لتعطيل المطالبات لجميع إجراءات التحرير:

      ```json:disable-run
      {
        "permissions": {
          "defaultMode": "acceptEdits",
          "allow": [
            "Edit(*)",
            "Write(*)",
            "MultiEdit(*)"
          ],
          "ask": []
        }
      }
      ```

      *   **ما يفعله هذا التكوين**:
        *   `"defaultMode": "acceptEdits"`: يضع الوضع الافتراضي العام لقبول التعديلات دون مطالبة.
        *   `"allow"`: يمنح الموافقة المسبقة لأدوات محددة مثل `Edit` (لتحرير ملف واحد)، و `Write` (لكتابة المحتوى)، و `MultiEdit` (للتغييرات المجمعة) لأي ملفات (الرمز `*` يعني جميع الملفات).
        *   `"ask": []`: يضمن عدم ظهور مطالبة لأي إجراء ما لم يتم رفضه صراحة في مكان آخر.
      *   احفظ الملف. التغييرات سارية المفعول فورًا في جلسة Claude Code التالية.

3.  **اختباره**:
    *   أعد تشغيل Claude Code أو أعِد تحميل الجلسة.
    *   اطلب منه تحرير ملف (مثل "Edit sync_githook.py to add a comment"). يجب أن يطبق التغيير دون ظهور النافذة المنبثقة.
    *   إذا كنت تريد إعدادات خاصة بمشروع معين (على سبيل المثال، لهذا المستودع فقط)، فأنشئ `.claude/settings.json` في المجلد الجذري لمشروعك بدلاً من ذلك.

4.  **تعديلات اختيارية**:
    *   للتحكم الدقيق (على سبيل المثال، السماح بالتحرير في مجلدات معينة فقط): غيّر `"Edit(*)"` إلى `"Edit(./src/**)"`.
    *   إذا استمرت ظهور المطالبات، شغّل `/allowed-tools` في الـ REPL للتحقق من القواعد الحالية، أو استخدم `/config للتعديل عبر واجهة المستخدم.
    *   لأسباب أمنية، تجنب قواعد `allow` واسعة النطاق—ابدأ بشيء بسيط وقم بالتوسع لاحقًا.

هذا من شأنه أن يجعل سير عملك أسرع دون مقاطعات. إذا كنت في بيئة عمل مؤسسية، تحقق من مسؤول النظام، حيث قد تتجاوز السياسات المدارة الإعدادات الخاصة بالمستخدم.
