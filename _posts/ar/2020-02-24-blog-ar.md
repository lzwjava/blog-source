# lzwjava.github.io

مرحبًا بكم في مدونتي الشخصية! هنا أشارك أفكاري، تجاربي ومعلومي على مجموعة متنوعة من المواضيع.

## تحسينات رئيسية

تضمن هذا المدونة تحسينات عدة مقارنة بـ مدونات Jekyll القياسية باستخدام jekyll-theme-cayman:

*   **ترجمة مفعَّلة بإقتران الذكاء الاصطناعي:** تستفيد من نماذج لغات متقدمة للحصول على ترجمات دقة ودقيقة من الناحية السياقية، مما يوسع إمكانية الوصول للمحتوى إلى جمهور عالمي.
*   **إنشاء PDF باستخدام XeLaTeX:** يدمج XeLaTeX لإنتاج PDFات عالية الجودة جاهزة للطباعة لمشاهدتها وقراءتها offline.
*   **Google Cloud Text-to-Speech:** يستخدم Google Cloud's Text-to-Speech service لإنشاء نسخ صوتية للمقالات، مما يحسِّن إمكانية الوصول للمكفوفين والذين يفضلون المحتوى الصوتي.
*   **تسليط CSS متقدم:** يحتوي على تصميم CSS مخصص ومصقول لفصل تجربة المستخدم والجمالية.
*   **دعم MathJax:** يدمج MathJax لتقديم التعبيرات الرياضية والتساويات، مما يجعل المحتوى الفني أكثر إمكانية الوصول.
*   **الوضع الليلي:** يتضمن خيار الوضع الليلي لخفض الإجهاد على العين والإعطاء قابلية قراءة أفضل في ظروف ضوئية ضعيفة.
*   **اختيار مقالات مرن:** يقدم مجموعة من خيارات اختيار المقالات، مثل تصفية حسب الفئة أو التسمية، لتحسين التنقل.
*   **تحديثات منتظمة:** يضمن أن مكتبة المدونة والمكونات قيد التحديث لاختيار الأداء وأمان.
*   **دمج `awesome-cv`:** يستخدم `awesome-cv` لإنشاء CVات مهنية مباشرة من المدونة.
*   **دعم RSS Feed:** يقدم RSS feeds عبر `feed.xml`، مما يسمح للمستخدمين بإشتراك في المدونة.
*   **محتوى ثنائي اللغة:** يدعم المحتوى باللغتين الصينية والإنجليزية لاستضافة جمهور متنوع.
*   **GitHub Workflow Automation:** يدمج GitHub Actions لعملية بناء، اختبار، وتوزيع، مما يضمن عملية تطوير متسق.
*   **عملية ترجمة تلقائية:** يترجم المقالات الجديدة أو المتحديثة تلقائياً إلى عدة لغات باستخدام GitHub Actions.
*   **دعم EPUB:** يحوّل Markdown إلى EPUB لمشاهدة الكتب الإلكترونية.

## بدء العمل

لتكوين بيئة Jekyll محلية، اتبع هذه الخطوات:

```shell
gem install jekyll bundler

jekyll new myblog

cd myblog

bundle install

bundle exec jekyll serve

bundle exec jekyll serve --draft

bundle add webrick

jekyll serve
```

## تعديل Markdown

لتخصيص إضاءة النص، استخدم `rougify`:

```shell
rougify help style

rougify style github > _sass/syntax.css

rougify style gruvbox.dark > _sass/syntax.css

rougify style base16.monokai.dark > _sass/syntax.css
```

## pipelines الصوت و PDF

لإنشاء ملفات الصوت و PDF، استخدم الأوامر التالية:

```bash
conda activate google-cloud-env

python scripts/audio_pipeline.py --task posts --n 10

python scripts/audio_pipeline.py --task pages

python scripts/audio_pipeline.py --task notes

python scripts/audio_pipeline.py --task notes --n 100

python scripts/pdf_pipeline.py --task pages

python scripts/pdf_pipeline.py --task posts --n 10

python scripts/pdf_pipeline.py --task notes

python scripts/generate_notes_link.py

python scripts/auto-ss-config/upload_configs.py

python scripts/epub_book.py notes

python scripts/epub_book.py _posts/en

python scripts/epub_book.py _posts/ja
```

## Scripts

لإنشاء مسودة مضمون جديد، استخدم:

```
python scripts/draft.py <name>
```

## دليل المحتوى

*   `-en`: يحدد المقالات باللغة الإنجليزية.
*   `-zh`: يحدد المقالات باللغة الصينية.

الدليلان `_posts` و `pages` يدعمان كل من suffixes `-en` و `-zh`.

الدليل `notes` يحتوي بشكل أساسي على محتوى باللغة الإنجليزية لأغراض التعلم الشخصي ولا يتطلب ترجمات.
