### بدائل تعديل الكود المصدري لـ DirB

إذا لم يكن تعديل الكود المصدري (مثل `variables.h`) خيارًا — ربما بسبب إعداد للقراءة فقط، أو سياسة معينة، أو مجرد تفضيل — إليك طرقًا مباشرة لتشغيل DirB دون الحاجة إلى الترجمة. هذه الطرق تستفيد من الحزم المبنية مسبقًا أو إعادة التطبيقات. سأركز على لينكس (بناءً على مسار البناء الخاص بك)، لكن لاحظ أن هناك تعديلات خاصة بنظام التشغيل.

#### 1. **التثبيت عبر مدير الحزم (الأسهل لنظم Debian/Ubuntu/Kali)**
   DirB متاح كحزمة مسبقة الترجمة في مستودعات العديد من التوزيعات، لذا لا حاجة لتغييرات في المصدر أو البناء.
   - **في Kali Linux** (مُوصى به لأدوات اختبار الاختراق مثل هذه):
     ```
     sudo apt update
     sudo apt install dirb
     ```
     - إنه مدعوم رسميًا ومُحافظ عليه في مستودعات كالي. [صفحة أدوات كالي](https://www.kali.org/tools/dirb/)
   - **في Ubuntu/Debian**:
     ```
     sudo apt update
     sudo apt install dirb
     ```
     - إذا لم يتم العثور عليه (الإصدارات القديمة قد لا تحتويه)، فعّل مستودع universe: `sudo add-apt-repository universe && sudo apt update`.
   - **التحقق**: شغّل `dirb --help` بعد التثبيت. قوائم الكلمات موجودة في `/usr/share/dirb/wordlists/`.
   - **لماذا تعمل هذه الطريقة**: الحزم تعالج جميع الإصلاحات (بما في ذلك التعريفات المتعددة) في المصدر.

   إذا كنت تستخدم توزيعة مختلفة:
   - **Fedora/RHEL**: `sudo dnf install dirb` (إذا كانت في مستودعات EPEL؛ أضف EPEL إذا لزم الأمر: `sudo dnf install epel-release`).
   - **Arch**: `sudo pacman -S dirb`.

#### 2. **استخدام إعادة تنفيذ بلغة Python (عبر المنصات، لا حاجة لترجمة C)**
   DirB الأصلي مبني بلغة C وصعب البناء، ولكن هناك إصدار حديث بلغة Python مطابق وظيفيًا (أو أفضل) ويُثبت عبر pip — دون الحاجة لتعديل المصدر.
   - **المستودع**: [ct-Open-Source/dirb على GitHub](https://github.com/ct-Open-Source/dirb)
   - **التثبيت**:
     ```
     pip install git+https://github.com/ct-Open-Source/dirb.git
     ```
     - أو استنساخه: `git clone https://github.com/ct-Open-Source/dirb.git && cd dirb && pip install .`
   - **الاستخدام**: مثل الأصلي، على سبيل المثال: `dirb https://example.com /usr/share/wordlists/dirb/common.txt`.
   - **الإيجابيات**: يُحافظ عليه بنشاط، أسرع في بعض الحالات، يعمل على Mac/Windows/Linux. يتضمن إضافات مثل المسح العودي.
   - **السلبيات**: ليس البرنامج "الرسمي"، لكن الناتج مطابق تقريبًا.

#### 3. **تنزيل البرنامج الثنائي المسبق البناء (التوفر محدود)**
   - إصدارات SourceForge الرسمية هي في الغالب أرشيفات مصدرية (مثل dirb222.tar.gz)، والتي لا تزال تتطلب البناء.
   - لا توجد برامج ثنائية عالمية، ولكن إذا كنت تستخدم Kali/Parrot OS، فإن طريقة الحزمة هي الأفضل.
   - كحل بديل: إذا كان لديك Docker، اسحب صورة Kali وشغّل DirB داخلها:
     ```
     docker run -it kalilinux/kali-rolling dirb http://example.com
     ```
     - لا حاجة للتثبيت؛ الحاويات مبنية مسبقًا.

#### مقارنة سريعة

| الطريقة              | دعم أنظمة التشغيل   | الجهد | ملاحظات |
|---------------------|---------------------|--------|-------|
| **Apt/DNF/Pacman** | لينكس (Debian/Fedora/Arch) | منخفض | مسبق الترجمة؛ تتضمن قوائم الكلمات. |
| **إصدار Python**    | أي (يدعم Python 3+) | منخفض | بديل حديث؛ يعتمد على pip فقط. |
| **Docker**         | أي (مع Docker)    | متوسط | معزول؛ لا يتطلب تثبيت على المضيف. |

إذا لم تنجح أي من هذه الطرق (على سبيل المثال، بيئة مقيدة)، ففكر في بدائل مثل Gobuster (`go install github.com/OJ/gobuster/v3@latest`) — فهو مبني بلغة Go، ويُترجم بسهولة دون مشاكل، ويقوم بعملية اكتشاف الدلائل المماثلة.

على أي نظام تشغيل أنت؟ هذا سيساعد في التحديد. إذا واجهت مشاكل، شارك الخطأ/الناتج.
