### تجهيز الملفات باستخدام الأنماط في Git

أمر `git add` في Git يقبل مسارات الملفات وأنماط التجميع (مثل `*.java`) لتجهيز الملفات المطابقة. يمكنك تشغيل أوامر `git add` متعددة لتجهيز مجموعات مختلفة من الملفات، ثم إيداعها بشكل منفصل. هذا يسمح بالتجهيز الانتقائي للإيداعات المجمعة (مثلاً، إيداع واحد لملفات Java، وآخر لبقية الملفات).

#### مثال أساسي: تجهيز وإيداع ملفات Java أولاً
1. **تجهيز جميع ملفات .java**:
   ```bash
   git add "*.java"
   ```
   - هذا يُجهز جميع الملفات غير المتعقبة أو المعدلة التي تنتهي بـ `.java` في الدليل الحالي والأدلة الفرعية.
   - استخدم علامات الاقتباس المزدوجة (`"`) لمنع مشاكل توسعة shell في العديد من الأنظمة (مثل Bash).

2. **إيداع ملفات Java المُجهزة**:
   ```bash
   git commit -m "Add Java files"
   ```
   - هذا يُودع فقط الملفات المُجهزة (ملفات Java).

3. **تجهيز كل شيء آخر**:
   ```bash
   git add .
   ```
   - هذا يُجهز جميع الملفات المتبقية غير المتعقبة/المعدلة (بما في ذلك محتويات الدليل الحالي).

4. **إيداع الملفات المتبقية**:
   ```bash
   git commit -m "Add other files"
   ```

- **ملاحظات**:
  - بعد الإيداع الأول، `git add .` لن يعيد تجهيز الملفات التي تم إيداعها مسبقاً؛ فهو يؤثر فقط على التغييرات غير المُجهزة.
  - تحقق من `git status` بعد كل أمر add للتحقق مما تم تجهيزه.
  - إذا كان لديك أدلة متداخلة، `*.java` تلتقط الملفات على أي عمق.

#### التعامل مع الاستثناءات أو المزيد من الأنماط
إذا كنت تقصد شيئاً مثل تضمين `*.java` ولكن استبعاد غيرها في خطوة واحدة، فإن `git add` في Git لا يدعم النفي المباشر مثل `"!*.java"`. (بناء الجملة في مثالك `"!*.java"` غير صالح في أوامر Git). بدلاً من ذلك، استخدم عدة أوامر `git add` كما هو مذكور أعلاه، أو:

- استخدم مسارات محددة: `git add file1.txt file2.md` للتحديد اليدوي.
- للاستبعاد المتقدم: استخدم مؤقتاً ملف `.gitignore` لإخفاء الملفات، قم بإضافة ما تريد، ثم استعد ملف `.gitignore`. أو استخدم `git add -A` (يضيف كل شيء) ولكن اقترنه بالتجهيز الانتقائي.
- الإضافة التفاعلية: `git add -p` للتجهيز جزءاً جزءاً.
- للمعاينة المسبقة للتغييرات: `git add --dry-run "*.java"` يُظهر ما سيتم تجهيزه دون تنفيذه.

#### نصائح للإيداعات المنفصلة
- راجع دائماً باستخدام `git diff --cached` قبل الإيداع.
- للمستودعات الكبيرة، فكر في استخدام `git add --all` أو `git add -A` لتجهيز جميع التغييرات دفعة واحدة، ثم قم بإلغاء تجهيز ما تريد انتقائياً إذا لزم الأمر باستخدام `git reset HEAD <file>`.
- إذا لم تعمل الأنماط، تأكد من استخدام علامات الاقتباس وأن shell الخاص بك يدعم أنماط التجميع (مثلاً، في Windows، استخدم Git Bash). لحساسية حالة الأحرف، يحترم Git الإعدادات في نظامك (انظر `core.ignorecase`).

إذا كان هذا لا يتطابق مع ما تقصده (مثلاً، إذا كنت تقصد أنماط الاستبعاد كما في `.gitignore`)، قدم المزيد من التفاصيل!
