TortoiseGit هو امتداد لقشرة Windows يوفر واجهة سهلة الاستخدام لـ Git، نظام التحكم في الإصدارات الشهير. وهو يندمج مباشرة في مستكشف Windows، مما يسمح لك بتنفيذ عمليات Git عبر القوائم السياقية. يغطي هذا الدليل تثبيت وتكوين واستخدام TortoiseGit على Windows، إلى جانب المهام الشائعة وأفضل الممارسات.

### نظرة عامة على TortoiseGit
تم بناء TortoiseGit على أساس msysGit (Git لنظام Windows) ويوفر واجهة رسومية لأوامر Git. إنه مثالي للمستخدمين الذين يفضلون واجهة المستخدم الرسومية (GUI) على عمليات Git عبر سطر الأوامر. تشمل الميزات الرئيسية:
- التكامل مع القائمة السياقية في مستكشف Windows.
- أدوات المقارنة المرئية (Visual diff) والدمج.
- دعم سير عمل Git الشائعة (الالتزام، الدفع، السحب، الفروع، الدمج، إلخ).
- التكامل مع أنظمة تتبع المشكلات والأدوات الخارجية الشهيرة.

### المتطلبات الأساسية
قبل تثبيت TortoiseGit، تأكد من توفر:
- نظام تشغيل Windows (يُوصى بـ Windows 10 أو إصدار أحدث).
- تثبيت Git for Windows (يعتمد TortoiseGit عليه).
- فهم أساسي لمفاهيم Git (المستودعات، الالتزامات، الفروع، إلخ).

### التثبيت
1. **تثبيت Git for Windows**:
   - حمّل أحدث إصدار من [Git for Windows](https://gitforwindows.org/) أو [Git SCM](https://git-scm.com/downloads).
   - شغّل المثبت واتبع التعليمات. الإعدادات الموصى بها:
     - استخدم المحرر الافتراضي (مثل Notepad) أو اختر واحدًا مثل VS Code.
     - اختر "Use Git from the Windows Command Prompt" لسهولة الوصول.
     - اختر "OpenSSL" لنقل HTTPS.
     - اختر "Checkout as-is, commit as-is" لنهايات الأسطر (ما لم تكن تعمل مع فرق عبر منصات متعددة).
   - أكمل التثبيت.

2. **تثبيت TortoiseGit**:
   - حمّل أحدث إصدار من [الموقع الرسمي لـ TortoiseGit](https://tortoisegit.org/download/).
   - شغّل المثبت:
     - اختر اللغة والمكونات الافتراضية.
     - تأكد من اكتشاف Git for Windows (سيطلب TortoiseGit إذا كان مفقودًا).
     - قم بتثبيت TortoiseGitPlink (موصى به لـ SSH) إذا لزم الأمر.
   - أعد تشغيل الكمبيوتر إذا طُلب منك ذلك.

3. **التحقق من التثبيت**:
   - انقر بزر الماوس الأيمن في أي مجلد في مستكشف Windows. يجب أن ترى خيارات TortoiseGit مثل "Git Clone" و "Git Create Repository here"، إلخ.

### التهيئة الأولية
بعد التثبيت، قم بتكوين TortoiseGit لتفاصيل المستخدم والتفضيلات الخاصة بك:
1. **تعيين معلومات المستخدم**:
   - انقر بزر الماوس الأيمن في مجلد، وحدد **TortoiseGit > Settings**.
   - في نافذة الإعدادات، انتقل إلى **Git > Config**.
   - أدخل اسمك وبريدك الإلكتروني (نفس المستخدم في GitHub أو GitLab، إلخ):
     ```
     Name: اسمك
     Email: بريدك.الإلكتروني@example.com
     ```
   - انقر على **Apply** ثم **OK**.

2. **تكوين SSH (اختياري)**:
   - إذا كنت تستخدم SSH للمستودعات البعيدة، قم بإعداد مفتاح SSH:
     - افتح **PuTTYgen** (مثبت مع TortoiseGit).
     - أنشئ زوج مفاتيح SSH جديدًا، واحفظ المفتاح الخاص، وانسخ المفتاح العام.
     - أضف المفتاح العام إلى خدمة استضافة Git الخاصة بك (مثل GitHub أو GitLab).
     - في إعدادات TortoiseGit، ضمن **Git > Remote**، حدد TortoiseGitPlink كعميل SSH.

3. **تعيين أدوات المقارنة والدمج**:
   - في **TortoiseGit > Settings > Diff Viewer**، اختر أداة مثل TortoiseGitMerge (الافتراضي) أو أداة خارجية مثل Beyond Compare.
   - للدمج، قم بالتكوين تحت **Merge Tool** (يُوصى بـ TortoiseGitMerge للمبتدئين).

### الاستخدام الأساسي
فيما يلي عمليات TortoiseGit الشائعة، التي يمكن الوصول إليها عبر القوائم السياقية بالنقر بزر الماوس الأيمن في مستكشف Windows.

#### 1. **استنساخ مستودع (Clone)**
   - انقر بزر الماوس الأيمن في مجلد وحدد **Git Clone**.
   - في مربع الحوار:
     - أدخل عنوان URL للمستودع (مثال: `https://github.com/username/repo.git`).
     - حدد المجلد المحلي للمستودع.
     - اختياريًا، حدد فرعًا أو حمّل مفاتيح SSH.
   - انقر على **OK** لاستنساخ المستودع.

#### 2. **إنشاء مستودع جديد**
   - انتقل إلى مجلد، انقر بزر الماوس الأيمن، وحدد **Git Create Repository here**.
   - حدد "Make it Bare" إذا كنت تنشئ مستودعًا على جانب الخادم (نادر للاستخدام المحلي).
   - انقر على **OK**. سيتم إنشاء مجلد `.git`، مما يؤدي إلى تهيئة المستودع.

#### 3. **إيداع التغييرات (Commit)**
   - أضف ملفات إلى مجلد مستودعك.
   - انقر بزر الماوس الأيمن على المجلد أو الملفات المحددة، ثم اختر **Git Commit -> "main"** (أو الفرع الحالي).
   - في مربع حوار الإيداع:
     - أدخل رسالة إيداع تصف التغييرات.
     - حدد الملفات لتضمينها في منطقة الانتظار (Stage) (مربعات الاختيار).
     - انقر على **OK** أو **Commit & Push** لدفع التغييرات إلى المستودع البعيد.

#### 4. **دفع التغييرات (Push)**
   - بعد الإيداع، انقر بزر الماوس الأيمن وحدد **TortoiseGit > Push**.
   - اختر المستودع البعيد والفرع.
   - قم بالمصادقة إذا طُلب منك (اسم المستخدم/كلمة المرور لـ HTTPS أو مفتاح SSH).
   - انقر على **OK** للدفع.

#### 5. **سحب التغييرات (Pull)**
   - لتحديث مستودعك المحلي بالتغييرات من المستودع البعيد، انقر بزر الماوس الأيمن وحدد **TortoiseGit > Pull**.
   - حدد الفرع البعيد وانقر على **OK**.
   - قم بحل التعارضات إذا طُلب منك (استخدم أداة الدمج).

#### 6. **إنشاء والتبديل بين الفروع**
   - انقر بزر الماوس الأيمن وحدد **TortoiseGit > Create Branch**.
   - أدخل اسم فرع وانقر على **OK**.
   - للتبديل بين الفروع، انقر بزر الماوس الأيمن وحدد **TortoiseGit > Switch/Checkout**، ثم اختر الفرع.

#### 7. **عرض السجل (History)**
   - انقر بزر الماوس الأيمن وحدد **TortoiseGit > Show Log**.
   - اعرض سجل الالتزامات، بما في ذلك المؤلف والتاريخ والرسائل.
   - انقر بزر الماوس الأيمن على أي إيداع لعرض التغييرات أو التراجع عنه أو اختياره انتقائيًا (cherry-pick).

#### 8. **حل تعارضات الدمج**
   - أثناء السحب أو الدمج، إذا حدثت تعارضات، سيخطرك TortoiseGit.
   - انقر بزر الماوس الأيمن على الملفات المتعارضة وحدد **TortoiseGit > Resolve**.
   - استخدم أداة الدمج لتحرير التعارضات يدويًا، ثم حددها كمحلولة.
   - قم بإيداع التغييرات المحلولة.

### ميزات متقدمة
1. **تخزين التغييرات مؤقتًا (Stashing)**:
   - لحفظ التغييرات غير المودعة مؤقتًا، انقر بزر الماوس الأيمن وحدد **TortoiseGit > Stash Save**.
   - لاسترجاع التغييرات المخزنة مؤقتًا، حدد **TortoiseGit > Stash Pop**.

2. **إعادة الكتابة (Rebasing)**:
   - انقر بزر الماوس الأيمن وحدد **TortoiseGit > Rebase**.
   - اختر الفرع الذي تريد إعادة الكتابة عليه واتبع التعليمات لإعادة ترتيب أو ضم الالتزامات.

3. **وحدات فرعية (Submodules)**:
   - لإدارة الوحدات الفرعية، انقر بزر الماوس الأيمن وحدد **TortoiseGit > Submodule Update** أو **Add**.
   - قم بتكوين إعدادات الوحدات الفرعية في إعدادات TortoiseGit.

4. **التقسيم الثنائي (Bisecting)**:
   - للعثور على إيداع أدخل خطأ، استخدم **TortoiseGit > Bisect Start**.
   - حدد الالتزامات على أنها "جيدة" أو "سيئة" لتضييق نطاق الالتزام المسبب للمشكلة.

### أفضل الممارسات
- **الالتزام كثيرًا**: قم بعمليات إيداع صغيرة ومتكررة برسائل واضحة.
- **السحب بانتظام**: حافظ على تحديث مستودعك المحلي لتجنب التعارضات.
- **استخدم الفروع**: أنشئ فروعًا للميزات الجديدة للحفاظ على استقرار الفرع الرئيسي.
- **احتفظ بنسخ احتياطية لمفاتيح SSH**: قم بتخزين مفاتيح SSH بشكل آمن واحتفظ بنسخ احتياطية منها.
- **راجع التغييرات**: استخدم عارض المقارنة لمراجعة التغييرات قبل الإيداع.

### استكشاف الأخطاء وإصلاحها
- **مشاكل المصادقة**: تأكد من تكوين مفاتيح SSH أو بيانات الاعتماد بشكل صحيح في خدمة استضافة Git الخاصة بك.
- **تعارضات الدمج**: استخدم TortoiseGitMerge لحل التعارضات بصريًا. احتفظ بنسخ احتياطية من الملفات قبل الدمج.
- **القائمة السياقية مفقودة**: تحقق من تثبيت TortoiseGit بشكل صحيح وأن تكامل مستكشف Windows مفعل في الإعدادات.
- **أداء بطيء**: عطّل الرموز الزائدة غير الضرورية في **TortoiseGit > Settings > Icon Overlays**.

### مثال على سير العمل
فيما يلي مثال على سير العمل للمساهمة في مشروع:

# مثال على سير عمل TortoiseGit

## 1. استنساخ المستودع
- انقر بزر الماوس الأيمن في مجلد، وحدد **Git Clone**.
- أدخل عنوان URL للمستودع والمجلد المحلي.
- انقر على **OK**.

## 2. إنشاء فرع للميزة
- انقر بزر الماوس الأيمن، وحدد **TortoiseGit > Create Branch**.
- سمِّ الفرع (مثال: `feature/add-login`).
- انقر على **OK** وانتقل إلى الفرع الجديد.

## 3. إجراء التغييرات
- حرر الملفات في مجلد المستودع.
- انقر بزر الماوس الأيمن، وحدد **Git Commit -> "feature/add-login"**.
- اكتب رسالة إيداع وقم بتضمين الملفات في منطقة الانتظار.
- انقر على **Commit**.

## 4. دفع التغييرات
- انقر بزر الماوس الأيمن، وحدد **TortoiseGit > Push**.
- اختر الفرع البعيد (مثال: `origin/feature/add-login`).
- انقر على **OK**.

## 5. إنشاء طلب سحب (Pull Request)
- على خدمة استضافة Git الخاصة بك (مثل GitHub)، أنشئ طلب سحب من فرعك إلى الفرع الرئيسي.
- انتظر المراجعة والدمج.

## 6. تحديث المستودع المحلي
- انتقل إلى الفرع الرئيسي (**TortoiseGit > Switch/Checkout**).
- اسحب التحديثات (**TortoiseGit > Pull**).
