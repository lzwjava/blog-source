### كيفية إصلاح خطأ "Cannot uninstall typing-extensions None"

عندما واجهت هذا الخطأ أثناء تثبيت الحزمة باستخدام pip، كان السبب مشكلة في إلغاء تثبيت حزمة `typing-extensions`. تنص رسالة الخطأ على:

```
error: uninstall-no-record-file
× Cannot uninstall typing-extensions None
╰─> The package's contents are unknown: no RECORD file was found for typing-extensions.
```

يحدث هذا الخطأ لأن pip يحاول إلغاء تثبيت تثبيت موجود لـ `typing-extensions` (على الأرجح لاستبداله بإصدار مطلوب من قبل الحزم التي تقوم بتثبيتها)، لكنه لا يستطيع المتابعة لأن ملف RECORD مفقود. ملف RECORD هو ملف بيانات وصفية تستخدمه pip لتتبع الملفات المثبتة بواسطة الحزمة، لذا بدونه، لا تعرف pip كيفية إلغاء تثبيت `typing-extensions` بشكل صحيح. كما يوفر سجل التثبيت تلميحًا مفيدًا:

```
hint: You might be able to recover from this via: pip install --force-reinstall --no-deps typing-extensions==4.14.0
```

يوجد أدناه حل خطوة بخطوة لحل هذه المشكلة وإكمال تثبيت الحزمة بنجاح.

### الخطوة 2: إصلاح تثبيت `typing-extensions`
لحل هذه المشكلة، تحتاج إلى إصلاح تثبيت `typing-extensions` المعطوب. الأمر المقترح من التلميح هو أفضل نهج:

```bash

```

#### ما يفعله هذا الأمر:
- **`pip install`**: يثبت الحزمة المحددة.
- **`--force-reinstall`**: يجبر pip على إعادة تثبيت `typing-extensions`، حتى لو كان موجودًا بالفعل، متجاوزًا التثبيت الحالي.
- **`--no-deps`**: يمنع pip من تثبيت أي تبعيات لـ `typing-extensions`. نظرًا لأن `typing-extensions` هي حزمة Python خالصة مستقلة بدون تبعيات، فإن هذا العلم يضمن إعادة تثبيت نظيفة دون التأثير على الحزم الأخرى.
- **`typing-extensions==4.14.0`**: يحدد الإصدار 4.14.0، والذي من المحتمل أن يكون الإصدار الذي كان pip يحاول تثبيته عندما حدث الخطأ.

سيؤدي تشغيل هذا الأمر إلى:
- إعادة تثبيت `typing-extensions` الإصدار 4.14.0.
- إنشاء ملف RECORD صحيح في دليل الحزمة (عادة في `site-packages`).
- إصلاح التثبيت التالف أو غير المكتمل.

#### قم بتشغيل الأمر:
افتح طرفيتك وقم بتنفيذ:

```bash

```
