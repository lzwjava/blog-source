## دليل شامل حول لغة XML

**ما هي لغة XML؟**

XML هي لغة ترميز مصممة لنقل البيانات. وهي قابلة للقراءة من قبل البشر والآلات على حد سواء. الغرض الأساسي منها هو هيكلة البيانات وتخزينها ونقلها. على عكس لغة HTML المصممة لعرض البيانات، تركز لغة XML على وصف البيانات نفسها.

**الخصائص الرئيسية لـ XML:**

*   **قابلة للتوسيع:** يمكنك تعريف علاماتك الخاصة لوصف بياناتك.
*   **وصفية ذاتيًا:** العلامات المستخدمة تصف معنى البيانات.
*   **هيكل هرمي:** تحتوي مستندات XML على هيكل يشبه الشجرة مع عنصر جذر واحد.
*   **مستقلة عن المنصة:** يمكن معالجة XML بواسطة أي نظام يفهم تنسيق XML.
*   **قائمة على النص:** مستندات XML هي ملفات نصية عادية، مما يجعل إنشاؤها وقراءتها وتحريرها أمرًا سهلاً.
*   **مشكّلة بشكل صحيح:** يجب أن تلتزم مستندات XML بقواعد صارمة للبناء النحوي لتُعتبر مشكّلة بشكل صحيح.

**المفاهيم الأساسية لـ XML:**

*   **العناصر:** اللبنات الأساسية لمستند XML. يتم تعريفها بواسطة علامات البداية والنهاية (على سبيل المثال، `<book>`، `</book>`). يمكن أن تحتوي العناصر على محتوى نصي، أو عناصر أخرى، أو مزيج من الاثنين معًا.
*   **السمات:** تقدم معلومات إضافية عن العنصر. تظهر داخل علامة البداية وتتكون من زوج اسم-قيمة (على سبيل المثال، `<book genre="fiction">`).
*   **العلامات:** كلمات أساسية محاطة بأقواس زاوية (`<>`). علامات البداية تمثل بداية العنصر، وعلامات النهاية (بشرطة مائلة للأمام) تمثل نهاية العنصر.
*   **العنصر الجذر:** يجب أن يكون لكل مستند XML عنصر واحد من المستوى الأعلى يحتوي على جميع العناصر الأخرى.
*   **العناصر المتداخلة:** يمكن تداخل العناصر داخل عناصر أخرى لإنشاء هيكل هرمي.
*   **العناصر الفارغة:** يمكن تمثيل العناصر التي لا تحتوي على محتوى بعلامة واحدة (على سبيل المثال، `<br />`) أو بعلامتي بداية ونهاية بدون أي شيء بينهما (`<br></br>`).
*   **إعلان XML (اختياري ولكنه موصى به):** يمكن أن يكون السطر الأول من مستند XML إعلانًا لـ XML يحدد إصدار XML والتشفير (على سبيل المثال، `<?xml version="1.0" encoding="UTF-8"?>`).
*   **التعليقات:** تُستخدم لإضافة ملاحظات توضيحية داخل مستند XML. يتم إحاطتها بـ `<!-- -->`.
*   **الكيانات:** تمثل أحرفًا خاصة أو كتل نصية قابلة لإعادة الاستخدام. تشمل الكيانات المحددة مسبقًا `&lt;` (<)، `&gt;` (>)، `&amp;` (&)، `&apos;` (')، و `&quot;` (").

**مواصفات XML:**

تحافظ منظمة World Wide Web Consortium (W3C) على المواصفات الخاصة بـ XML والتقنيات ذات الصلة. تتضمن بعض مواصفات XML الرئيسية:

*   **XML 1.0 (و XML 1.1):** المواصفات الأساسية التي تحدد بناء الجملة وهيكل مستندات XML. إصدار XML 1.0 هو الإصدار الأكثر اعتمادًا على نطاق واسع.
*   **XML Schema (XSD):** لغة لتعريف هيكل وأنواع بيانات مستندات XML. توفر طريقة أكثر قوة وتعبيرية للتحقق من صحة XML من تعريفات نوع المستند (DTDs).
*   **تعريف نوع المستند (DTD):** لغة مخطط أقدم تُستخدم لتعريف هيكل مستندات XML. بينما لا تزال تُصادف في بعض الأحيان، يُفضل استخدام XSD بشكل عام لميزاته المتقدمة.
*   **XPath:** لغة لاستعلام العقد وتحديدها داخل مستند XML.
*   **XSLT (لغة Stylesheet القابلة للتوسيع - التحويلات):** لغة لتحويل مستندات XML إلى تنسيقات أخرى (على سبيل المثال، HTML، نص عادي، تنسيقات XML أخرى).
*   **مساحات الأسماء في XML:** توفر طريقة لتجنب تعارض التسمية عند دمج مستندات XML من مصادر مختلفة.

**XML Schema (XSD) وتحديد الهيكل:**

يعد XML Schema حاسمًا لتعريف الهيكل والمحتوى الصالحين لمستندات XML. يسمح لك بتحديد:

*   العناصر التي يمكن أن تظهر في المستند.
*   السمات التي يمكن أن تحتويها العناصر.
*   ترتيب وعدد العناصر الفرعية داخل العنصر الأصلي.
*   أنواع بيانات العناصر والسمات.
*   القيود على قيم العناصر والسمات.

**`<sequence>` في XML Schema:**

عنصر `<sequence>` هو مُركِّب يُستخدم داخل تعريفات النوع المعقد في XML Schema. يشير إلى أن العناصر الفرعية بداخله **يجب أن تظهر بالترتيب المحدد**.

**بناء الجملة:**

```xml
<xs:complexType name="TypeName">
  <xs:sequence>
    <xs:element name="element1" type="xs:string" minOccurs="1" maxOccurs="1"/>
    <xs:element name="element2" type="xs:integer" minOccurs="0" maxOccurs="unbounded"/>
    <xs:element name="element3" type="xs:date" minOccurs="1" maxOccurs="1"/>
  </xs:sequence>
</xs:complexType>
```

في هذا المثال، يجب أن يحتوي أي عنصر XML يتوافق مع النوع المعقد `TypeName` على:

1.  عنصر `<element1>` (من نوع string) يظهر مرة واحدة بالضبط.
2.  صفر أو أكثر من عناصر `<element2>` (من نوع integer) تظهر بالتسلسل بعد `<element1>`.
3.  عنصر `<element3>` (من نوع date) يظهر مرة واحدة بالضبط بعد جميع عناصر `<element2>`.

**سمات `minOccurs` و `maxOccurs`:**

تُستخدم السمات `minOccurs` و `maxOccurs` داخل تعريفات العناصر (عادةً داخل مُركِّب `<sequence>` أو `<choice>` أو `<all>`) في XML Schema لتحديد الحد الأدنى والحد الأقصى لعدد المرات التي يمكن أن يظهر فيها العنصر.

*   **`minOccurs`:**
    *   يحدد الحد الأدنى لعدد المرات التي يجب أن يظهر فيها العنصر.
    *   القيمة الافتراضية هي `1`.
    *   تشير القيمة `0` إلى أن العنصر اختياري.
    *   يشير العدد الصحيح الموجب إلى الحد الأدنى المطلوب للتكرارات.

*   **`maxOccurs`:**
    *   يحدد الحد الأقصى لعدد المرات التي يمكن أن يظهر فيها العنصر.
    *   القيمة الافتراضية هي `1`.
    *   يشير العدد الصحيح الموجب إلى الحد الأقصى المسموح به للتكرارات.
    *   تشير القيمة `unbounded` إلى أن العنصر يمكن أن يظهر أي عدد من المرات (صفر أو أكثر إذا كان `minOccurs` هو 0، واحد أو أكثر إذا كان `minOccurs` هو 1، إلخ).

**كيف يعمل Sequence مع `minOccurs` و `maxOccurs`:**

عندما تكون العناصر داخل `<sequence>`، فإن السمات `minOccurs` و `maxOccurs` على كل عنصر فردي تحدد عدد التكرارات المسموح بها لهذا العنصر المحدد *في هذا الموضع المحدد داخل التسلسل*. يجب الحفاظ على ترتيب العناصر كما هو محدد في `<sequence>` في مستند XML الم实例.

**مثال:**

ضع في اعتبارك مقتطف XML Schema التالي:

```xml
<xs:complexType name="OrderType">
  <xs:sequence>
    <xs:element name="orderId" type="xs:ID" minOccurs="1" maxOccurs="1"/>
    <xs:element name="customer" type="xs:string" minOccurs="1" maxOccurs="1"/>
    <xs:element name="item" type="xs:string" minOccurs="0" maxOccurs="unbounded"/>
    <xs:element name="orderDate" type="xs:date" minOccurs="1" maxOccurs="1"/>
  </xs:sequence>
</xs:complexType>
```

سيكون لمستند XML الذي يتوافق مع هذا المخطط الهيكل التالي:

1.  عنصر `<orderId>` واحد بالضبط.
2.  عنصر `<customer>` واحد بالضبط، يظهر مباشرة بعد `<orderId>`.
3.  صفر أو أكثر من عناصر `<item>`، تظهر بالتسلسل بعد `<customer>`.
4.  عنصر `<orderDate>` واحد بالضبط، يظهر بعد جميع عناصر `<item>`.

**مستند XML صالح:**

```xml
<order>
  <orderId>ORD123</orderId>
  <customer>John Doe</customer>
  <item>Product A</item>
  <item>Product B</item>
  <orderDate>2025-03-29</orderDate>
</order>
```

**مستند XML صالح آخر:**

```xml
<order>
  <orderId>ORD456</orderId>
  <customer>Jane Smith</customer>
  <orderDate>2025-03-28</orderDate>
</order>
```

**مستند XML غير صالح (الترتيب خاطئ):**

```xml
<order>
  <customer>John Doe</customer>
  <orderId>ORD123</orderId>
  <orderDate>2025-03-29</orderDate>
</order>
```

**مستند XML غير صالح (`orderId` مفقود):**

```xml
<order>
  <customer>John Doe</customer>
  <item>Product A</item>
  <orderDate>2025-03-29</orderDate>
</order>
**باختصار:**

*   XML هي لغة ترميز متعددة الاستخدامات لهيكلة البيانات ونقلها.
*   تعتمد على بناء نحوي محدد جيدًا وهيكل هرمي.
*   يوفر XML Schema (XSD) طريقة قوية لتعريف هيكل ومحتوى مستندات XML.
*   يفرض المُركِّب `<sequence>` في XSD ترتيبًا محددًا للعناصر الفرعية.
*   تحدد السمات `minOccurs` و `maxOccurs` الحد الأدنى والحد الأقصى لعدد المرات التي يمكن أن يظهر فيها العنصر داخل سياقه المحدد (غالبًا داخل `<sequence>`).

يوفر هذا الدليل أساسًا متينًا لفهم لغة XML ومفاهيمها الأساسية، لا سيما دور `<sequence>` و `minOccurs` و `maxOccurs` في تعريف هيكل مستند XML باستخدام XML Schema. تذكر الرجوع إلى مواصفات W3C الرسمية للحصول على أحدث المعلومات وأكثرها تفصيلاً.
