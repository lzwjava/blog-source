### **الخيار 1: تشغيل `MultiMark.java` مباشرة باستخدام أمر `java` (إصدار Java 11+)**
بدءًا من إصدار Java 11، يمكنك تشغيل ملف مصدر Java فردي مباشرة دون الحاجة إلى تجميعه بشكل صريح أولاً، مما يجعله أقرب من حيث الشعور إلى تشغيل نص برمجي في Python.

1. **تأكد من تثبيت Java**
   تحقق مما إذا كان لديك إصدار Java 11 أو أحدث مثبتًا:
   ```bash
   java --version
   ```
   إذا لم تكن Java مثبتة أو كان الإصدار قديمًا، قم بتثبيت إصدار متوافق (مثل OpenJDK):
   ```bash
   sudo apt update
   sudo apt install openjdk-17-jdk
   ```

2. **شغّل ملف Java مباشرة**
   انتقل إلى الدليل الذي يحتوي على `MultiMark.java`:
   ```bash
   cd ~/projects/lzwjava.github.io/scripts/benchmark
   ```
   شغّل ملف Java مباشرة باستخدام:
   ```bash
   java MultiMark.java
   ```
   يقوم هذا الأمر بتجميع وتنفيذ ملف `MultiMark.java` في خطوة واحدة. سيظهر الناتج (مثل نوى المعالج، أعداد الخيوط، والأوقات) في الطرفية.

### **الخيار 2: إنشاء نص برمجي Shell للتيسير**
لجعل تشغيل `MultiMark.java` أشبه بتشغيل نص برمجي في Python (مثل `./multimark`)، يمكنك إنشاء غلاف نص برمجي shell.

1. **أنشئ نص برمجي Shell**
   أنشئ ملفًا باسم `multimark` (بدون امتداد) في دليل `scripts/benchmark`:
   ```bash
   nano ~/projects/lzwjava.github.io/scripts/benchmark/multimark
   ```
   أضف المحتوى التالي:
   ```bash
   #!/bin/bash
   java MultiMark.java
   ```
   احفظ الملف واخرج (`Ctrl+O`، `Enter`، `Ctrl+X` في nano).

2. **اجعل النص البرمجي قابلاً للتنفيذ**
   عيّن أذونات التنفيذ:
   ```bash
   chmod +x multimark
   ```

3. **شغّل النص البرمجي**
   الآن يمكنك تشغيل البرنامج مثل نص برمجي في Python:
   ```bash
   ./multimark
   ```
   سيقوم هذا بتنفيذ `java MultiMark.java` في الخلفية.

إذا كنت تفضل أو تحتاج إلى تجميع ملف Java بشكل صريح (مثلًا للإصدارات القديمة من Java أو للتحقق من أخطاء التجميع)، اتبع هذه الخطوات:

### **ملاحظات**
- **إصدار Java**: يستخدم الكود مكتبات Java قياسية (`java.util.*` و `System.nanoTime`)، لذا يجب أن يعمل مع إصدار Java 8 أو أحدث. تتطلب خاصية التشغيل المباشر (`java MultiMark.java`) إصدار Java 11+.
- **الأداء**: يقوم برنامج `MultiMark.java` بقياس أداء الفرز باستخدام خيوط متعددة. لن يؤثر تشغيله مباشرة أو عبر نص برمجي على أدائه، ولكن تأكد من أن نظامك يحتوي على موارد كافية (المعالج/الذاكرة) لأعداد الخيوط التي يتم اختبارها (حتى 64 خيطًا).
- **سياق الدليل**: بما أنك أظهرت `ls scripts/benchmark/`، افترضت أن `MultiMark.java` موجود في ذلك الدليل. اضبط المسارات إذا كان في مكان آخر.
- **معالجة الأخطاء**: إذا واجهت أخطاء (مثل `java: command not found`)، فتأكد من تثبيت Java وإضافتها إلى `PATH` الخاص بك. إذا فشل البرنامج (بسبب حدود الذاكرة أو الخيوط على سبيل المثال)، فتحقق من `stack trace` للحصول على التفاصيل.

إذا كنت تريد دمج هذا في سير عمل مع نصوصك البرمجية في Python (`executormark.py`, `multimark.py`, `threadmark.py`)، فأعلمني، ويمكنني اقتراح طرق لدمجها (مثل استدعاء برنامج Java من Python).
