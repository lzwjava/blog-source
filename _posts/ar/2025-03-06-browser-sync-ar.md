BrowserSync هو أداة رائعة لتبسيط تطوير الواجهة الأمامية من خلال مزامنة إجراءات المتصفح، وإعادة تحميل الملفات تلقائيًا، وتوفير خادم تطوير محلي. إليك دليل خطوة بخطوة حول كيفية استخدام BrowserSync في سير عمل الواجهة الأمامية:

---

### 1. **تثبيت Node.js و npm**
BrowserSync هي أداة تعتمد على Node.js، لذا ستحتاج إلى تثبيت Node.js و npm (مدير حزم Node). إذا لم تكن قد قمت بتثبيتهما بعد:
- قم بالتحميل والتثبيت من [nodejs.org](https://nodejs.org/).
- تحقق من التثبيت:
  ```bash
  node -v
  npm -v
  ```

---

### 2. **تثبيت BrowserSync**
يمكنك تثبيت BrowserSync بشكل عام (عالمي) أو محليًا في مشروعك.

#### الخيار 1: التثبيت العام
شغّل هذا الأمر في طرفيتك لتثبيت BrowserSync بشكل عام:
```bash
npm install -g browser-sync
```
هذا يسمح لك باستخدام أمر `browser-sync` من أي مكان.

#### الخيار 2: التثبيت المحلي (موصى به للمشاريع)
إذا كنت تفضل إبقاء التبعيات مرتبطة بمشروع معين:
```bash
npm install browser-sync --save-dev
```
هذا يضيف BrowserSync إلى مجلد `node_modules` الخاص بمشروعك ويسرده في ملف `package.json`.

---

### 3. **بدء استخدام BrowserSync**
اعتمادًا على إعداداتك، يمكنك استخدام BrowserSync بطرق مختلفة:

#### الاستخدام الأساسي (الملفات الثابتة)
إذا كنت تعمل مع ملفات HTML و CSS و JS ثابتة، انتقل إلى مجلد مشروعك وشغّل:
```bash
browser-sync start --server --files "*.html, css/*.css, js/*.js"
```
- `--server`: يشغّل خادمًا محليًا (يخدم الملفات من الدليل الحالي).
- `--files`: يراقب هذه الملفات للتغييرات ويعيد تحميل المتصفح تلقائيًا.

على سبيل المثال، إذا كان هيكل مجلدك كالتالي:
```
my-project/
├── index.html
├── css/
│   └── style.css
└── js/
    └── script.js
```
تشغيل الأمر أعلاه سوف:
- يبدأ خادمًا على `http://localhost:3000` (المنفذ الافتراضي).
- يفتح متصفحك الافتراضي.
- يعيد تحميل الصفحة每当 يتغير `index.html` أو `style.css` أو `script.js`.

#### وضع الوكيل (Proxy) (لخادم موجود مسبقًا)
إذا كنت تستخدم خادم backend (مثل Node.js أو PHP أو Python)، استخدم خيار الوكيل:
```bash
browser-sync start --proxy "http://localhost:8000" --files "*.html, css/*.css, js/*.js"
```
- `--proxy`: يعمل كوسيط لطلبات الخادم الحالي الخاص بك (استبدل `http://localhost:8000` مع عنوان URL لخادمك).
- سيقوم BrowserSync بإضافة ميزاته (مثل إعادة التحميل التلقائي) فوقه.

---

### 4. **الميزات الرئيسية**
بمجرد أن يعمل BrowserSync، ستحصل على:
- **إعادة التحميل المباشر (Live Reloading)**: التغييرات في الملفات المراقبة تؤدي إلى تحديث فوري للمتصفح.
- **المزامنة عبر الأجهزة**: حركة التمرير، والنقرات، وإدخالات النماذج تتم مزامنتها عبر جميع الأجهزة المتصلة (مثل سطح المكتب، الهاتف، اللوحي).
- **لوحة التحكم (UI Dashboard)**: الوصول إلى لوحة تحكم على `http://localhost:3001` (الافتراضي) لضبط الإعدادات أو إيقاف الخادم.

---

### 5. **التكامل مع أدوات البناء (اختياري)**
إذا كنت تستخدم أدوات مثل Gulp أو Webpack، فيمكن لـ BrowserSync التكامل معها بسلاسة.

#### مثال مع Gulp
قم بتثبيت Gulp و BrowserSync:
```bash
npm install --save-dev gulp browser-sync
```
أنشئ ملف `gulpfile.js`:
```javascript
const gulp = require('gulp');
const browserSync = require('browser-sync').create();

gulp.task('serve', function() {
  browserSync.init({
    server: './', // يخدم الملفات من الدليل الحالي
  });

  gulp.watch(['*.html', 'css/*.css', 'js/*.js']).on('change', browserSync.reload);
});

gulp.task('default', gulp.series('serve'));
```
شغله:
```bash
gulp
```

---

### 6. **الأوامر والخيارات الشائعة**
- تغيير المنفذ: `--port 8080`
- تعطيل الفتح التلقائي: `--no-open`
- مراقبة ملفات محددة: `--files "path/to/files/*"`
- القائمة الكاملة: شغّل `browser-sync --help`

---

### 7. **استكشاف الأخطاء وإصلاحها**
- **المنفذ قيد الاستخدام**: غيّر المنفذ باستخدام `--port <number>`.
- **لا يوجد إعادة تحميل**: تأكد من أن مسارات الملفات في `--files` تطابق هيكل مشروعك.
- **أذونات**: إذا كان مثبتًا بشكل عام، قد تحتاج إلى `sudo` (مثل `sudo npm install -g browser-sync` على Linux/macOS).
