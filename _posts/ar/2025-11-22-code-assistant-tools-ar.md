المكالمات **الأدوات** أو **الإجراءات** النموذجية في مساعد برمجة قوي مثل Claude Code تتمحور حول تمكينه من التفاعل مع بيئة برمجة واقعية وتنفيذ المهام التي يؤديها المطور البشري.

تندرج هذه المكالمات الأدوات تحت فئات رئيسية قليلة:

### 1. التفاعل مع نظام الملفات والقاعدة البرمجية

هذه الأدوات أساسية لكي يفهم المساعد المشروع، ويقرأ الكود الموجود، ويُجري تغييرات.

| الأداة/الإجراء | حالة الاستخدام النموذجية |
| :--- | :--- |
| **`read_file`** (أو إشارة إلى ملف مثل `@اسم_الملف`) | استرجاع محتوى ملف معين لفهم المنطق أو التبعيات أو السياق الخاص به. |
| **`list_directory`** | الحصول على هيكل الملفات أو قائمة الملفات في دليل لتحديد الوحدات ذات الصلة أو العثور على ملف معين. |
| **`edit_file` / `write_file`** | الإجراء الأساسي لتنفيذ حل، أو إعادة هيكلة الكود، أو إضافة ميزة، أو إصلاح خطأ في الكود. |
| **`create_file`** | كتابة ملفات جديدة، مثل ملف اختبار جديد، أو ملف إعدادات، أو مكون جديد. |
| **`search_files`** | العثور على جميع الملفات في قاعدة الكود التي تحتوي على سلسلة نصية محددة (مثل اسم دالة، أو اسم صنف، أو رسالة خطأ). |

### 2. التنفيذ وتصحيح الأخطاء

للتحقق من عمله، وإصلاح الأخطاء، والحصول على ملاحظات فورية، يحتاج المساعد إلى تشغيل أوامر.

| الأداة/الإجراء | حالة الاستخدام النموذجية |
| :--- | :--- |
| **`bash` / `run_command`** | تنفيذ أوامر الطرفية مثل تشغيل عملية البناء (`npm build`)، أو تشغيل الاختبارات (`pytest`، `npm test`)، أو فحص الكود (`eslint`)، أو تشغيل سكريبت. |
| **`code_interpreter`** | تشغيل snippet صغير من الكود في بيئة معزولة لحساب شيء ما بسرعة، أو اختبار دالة في مكتبة، أو تحليل مجموعة بيانات صغيرة. |

### 3. التحكم في الإصدارات (Git)

يمكن لمساعد البرمجة الذكي إدارة سير عمل التطوير من خلال التفاعل مع Git.

| الأداة/الإجراء | حالة الاستخدام النموذجية |
| :--- | :--- |
| **`git status`** | التحقق من الحالة الحالية لدليل العمل. |
| **`git diff`** | رؤية التغييرات التي تم إجراؤها لمراجعة عمله الخاص. |
| **`git commit`** | حفظ التغييرات في المستودع، غالبًا مع رسالة commit مُنشأة تلقائيًا. |
| **`git checkout`** | تبديل الفروع أو تجاهل التغييرات غير المُحفوظة في commit. |
| **`create_pull_request`** | أتمتة الخطوة النهائية لميزة أو إصلاح. |

### 4. المعلومات الخارجية

للحصول على معلومات محدثة، أو وثائق، أو للاتصال بالخدمات الخارجية، يتم استخدام أدوات متخصصة.

| الأداة/الإجراء | حالة الاستخدام النموذجية |
| :--- | :--- |
| **`web_search`** | البحث عن أحدث صيغة تركيبية، أو قراءة وثائق API الرسمية، أو العثور على أدوات استكشاف الأخطاء وإصلاحها لخطأ معين. |
| **واجهات برمجة تطبيقات مخصصة / خوادم MCP** | الاتصال بالخدمات الخارجية باستخدام Model Context Protocol (MCP)، مثل جلب البيانات من Google Drive، أو التفاعل مع قاعدة بيانات (PostgreSQL)، أو إرسال رسالة إلى أداة اتصال (Slack). |

في جوهر الأمر، تسمح هذه "مكالمات الأدوات" لنموذج اللغة الكبير (LLM) بالتحول من مُنشئ نصوص إلى **وكيل** كامل يمكنه مراقبة بيئته (قراءة الملفات، تشغيل الأوامر)، والتفكير (تخطيط العمل)، والتصرف (تحرير الملفات، إيداع الكود).

هل ترغب في رؤية مثال تفصيلي خطوة بخطوة لكيفية استخدام مساعد الكود لهذه الأدوات لإصلاح خطأ بسيط؟
