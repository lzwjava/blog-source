### النقاط الرئيسية
- يبدو من المرجح أنه لاستخدام `org.joda.time`، تحتاج إلى إضافته إلى مشروع Java الخاص بك واستيراد فئاته لإجراء عمليات التاريخ والوقت.
- تشير الأبحاث إلى تضمين المكتبة عبر Maven بالإصدار 2.13.1 أو تنزيل ملف JAR، ثم استخدام فئات مثل `DateTime` و `LocalDate`.
- تميل الأدلة إلى أن Joda-Time مفيدة للتعامل مع المناطق الزمنية وأنظمة التقويم والفترات الزمنية، مع أمثلة مثل إنشاء كائنات التاريخ وتعديلها.

### ما هي Joda-Time وكيفية إعدادها
Joda-Time هي مكتبة للتعامل مع التواريخ والأوقات في Java، كانت مفيدة بشكل خاص قبل Java 8، حيث تقدم واجهة برمجة تطبيقات بديهية لتحل محل فئات `Date` و `Calendar` الأقدم والأقل أمانًا في بيئات الخيوط المتعددة. لاستخدامها، قم أولاً بتضمين المكتبة في مشروعك. إذا كنت تستخدم Maven، أضف التبعية التالية إلى ملف `pom.xml` الخاص بك:
```xml
<dependency>
    <groupId>joda-time</groupId>
    <artifactId>joda-time</artifactId>
    <version>2.13.1</version>
</dependency>
```
بدلاً من ذلك، يمكنك تنزيل ملف JAR من [هذا الموقع](https://www.joda.org/joda-time/download.html) وإضافته إلى classpath مشروعك، كما في Eclipse عن طريق إنشاء مجلد "libs" وربط ملف JAR عبر خصائص المشروع.

### أمثلة الاستخدام الأساسي
بمجرد الإعداد، قم باستيراد فئات مثل `org.joda.time.DateTime` أو `org.joda.time.LocalDate`. إليك بعض الأمثلة:
- إنشاء تاريخ ووقت حاليين: `DateTime now = new DateTime();`
- الوصول إلى الحقول: `int year = now.getYear(); String monthName = now.monthOfYear().getAsText();`
- التعديل: `DateTime future = now.plusDays(5);`

### الميزات المتقدمة
تدعم Joda-Time المناطق الزمنية (مثل `DateTimeZone.forID("America/New_York")`) وأنظمة التقويم المختلفة (مثل التقويم القبطي عبر `CopticChronology.getInstance()`). كما تتعامل مع الفترات والمدة الزمنية، مثل `Interval interval = new Interval(startDt, endDt);`.

من التفاصيل غير المتوقعة أن Joda-Time تعتبر مشروعًا "مكتملًا"، حيث يُوصى بحزمة `java.time` في Java 8 للمشاريع الجديدة، لكنها لا تزال ذات صلة بالأنظمة القديمة أو الاحتياجات المحددة.

### ملاحظة المسح: دليل شامل لاستخدام `org.joda.time`

#### مقدمة إلى Joda-Time
Joda-Time، التي طورها joda.org، هي مكتبة مستخدمة على نطاق واسع لمعالجة التاريخ والوقت، خاصة قبل إصدار Java 8. تعالج مشاكل التصميم في فئتي `Date` و `Calendar` في Java، مثل مخاوف أمان الخيوط، من خلال استخدام فئات غير قابلة للتغيير. قبل Java 8، لم تكن فئة `Date` و `SimpleDateFormatter` آمنة للاستخدام في بيئات الخيوط المتعددة، وكانت العمليات مثل إزاحة اليوم/الشهر/السنة غير بديهية (مثل بدء الأيام من 0، والأشهر من 1، مما يتطلب استخدام `Calendar`). تقدم Joda-Time واجهة برمجة تطبيقات نظيفة وسلسة وتدعم ثمانية أنظمة تقويم، مقارنة بنظامي Java (الميلادي والإمبراطوري الياباني). بعد Java 8، يعتبر مؤلفو Joda-Time أنها اكتملت إلى حد كبير، ويوصون بالانتقال إلى `java.time` (JSR-310) للمشاريع الجديدة، لكنها تظل ذات صلة بالأنظمة القديمة أو حالات الاستخدام المحددة.

#### إعداد Joda-Time
لاستخدام Joda-Time، يجب أولاً تضمينها في مشروع Java الخاص بك. أحدث إصدار اعتبارًا من 3 مارس 2025 هو 2.13.1، مما يضمن الاستقرار والتوافق مع JDK 1.5 أو إصدار أحدث. لمستخدمي Maven، أضف التبعية التالية إلى ملف `pom.xml` الخاص بك:
```xml
<dependency>
    <groupId>joda-time</groupId>
    <artifactId>joda-time</artifactId>
    <version>2.13.1</version>
</dependency>
```
يمكن العثور على هذا في [مستودع Maven](https://mvnrepository.com/artifact/joda-time/joda-time). للمشاريع التي لا تستخدم Maven، قم بتنزيل ملف `.tar.gz` من [هذا الموقع](https://www.joda.org/joda-time/download.html)، واستخرجه، ثم أضف `joda-time-2.13.1.jar` إلى classpath مشروعك. على سبيل المثال، في Eclipse، أنشئ مجلد "libs"، وانسخ ملف JAR فيه، واربطه عبر Properties -> Java Build Path -> Libraries -> Add Jars. اختبر الإعداد باستخدام `DateTime test = new DateTime();` لضمان الوظيفة.
