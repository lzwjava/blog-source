### ما هو Git Cherry-Pick؟

أمر `git cherry-pick` هو أمر يسمح لك بتطبيق التغييرات من commit محدد (أو مجموعة من commits) على الفرع الحالي الخاص بك، مما ينشئ commit جديدًا يحتوي على تلك التغييرات. يكون هذا الأمر مفيدًا عندما تريد اختيار إصلاح أو ميزة معينة من فرع ما دون دمج الفرع بأكمله. على عكس الدمج، الذي يدمج كل السجل، فإن `cherry-pick` ينسخ التغييرات بشكل انتقائي.

### كيف يعمل Git Cherry-Pick داخليًا

1.  **اختيار الـ Commit**: تقوم بتحديد hash الـ commit (على سبيل المثال، عبر `git log` للعثور عليه) الذي يحتوي على التغييرات التي تريدها.
2.  **تطبيق التغييرات**: يأخذ Git الـ diff (التغييرات الفعلية في الكود) من ذلك الـ commit ويحاول تطبيقه على دليل العمل الخاص بالفرع الحالي، باستخدام منطق الدمج ثلاثي الاتجاهات في Git حيثما أمكن ذلك.
3.  **إنشاء Commit جديد**: إذا نجحت العملية، يتم إنشاء commit جديد على فرعك يحتوي على:
    *   نفس التغييرات الموجودة في الـ commit الأصلي.
    *   hash commit جديد (لأنه commit جديد).
    *   رسالة commit تشير إلى الأصل (على سبيل المثال، عبر علامة `-x` لإلحاق "cherry picked from commit <hash>").

    إذا كانت هناك تعارضات في الدمج (على سبيل المثال، إذا كان الفرع الهدف يحتوي على تغييرات غير متوافقة)، يتوقف Git ويسمح لك بحلها يدويًا قبل عمل الـ commit.

لا تغير هذه العملية السجل التاريخي للـ commit الأصلي — إنها تنسخ التغييرات فقط. هذا يجعل `cherry-pick` رائعًا لـ backporting إصلاحات الأخطاء أو تطبيق الميزات بشكل انتقائي.

### خطوات الاستخدام الأساسية

1.  تأكد من أنك على الفرع الهدف (انتقل إليه باستخدام `git checkout <branch>`).
2.  ابحث عن hash الـ commit الذي تريده (على سبيل المثال، `git log --oneline`).
3.  قم بتنفيذ: `git cherry-pick <commit-hash>`
    *   لعدة commits: `git cherry-pick <hash1> <hash2>`
    *   مع خيارات مثل `-n` (لا يقوم بعمل commit، للتجهيز) أو `-x` (للإشارة إلى الـ commit الأصلي).
4.  إذا نشأت تعارضات، قم بتحرير الملفات، ثم جهزها باستخدام `git add`، ثم `git cherry-pick --continue`.
5.  إذا أردت إلغاء العملية في منتصف الطريق: `git cherry-pick --abort`.

### مثال توضيحي

لنفترض أن لديك فرع ميزة به commits بالترتيب A -> B -> C، وتريد أن تقوم بـ cherry-pick للـ commit B على فرع main.

*   على فرع main: `git checkout main`
*   `git cherry-pick B` (استبدل B بـ hash الـ commit الفعلي)

هذا يضيف التغييرات من B كـ commit جديد D على فرع main، دون جلب A أو C. يبقى الـ commit الأصلي B في فرع الميزة.

### محاذيث ونصائح

*   تجنب استخدام `cherry-pick` على عمليات الدمج (commits لها أكثر من parent)، لأن ذلك قد يؤدي إلى مشاكل — استخدم `git cherry-pick -m 1 <merge-commit>` لتحديد parent.
*   لا يحافظ الأمر على البيانات الوصفية للـ commit مثل تاريخ المؤلف، لكن `git cherry-pick --edit` يسمح لك بتعديل الرسالة.
*   للتغييرات المتسلسلة، قم بـ cherry-pick لعدة commits بالترتيب لتجنب فقدان التبعيات.
*   دائمًا اختبر الكود بعد استخدام `cherry-pick`، لأنه قد يتسبب في ظهور أخطاء إذا اختلفت السياقات.

لمزيد من التفاصيل، توفر وثائق Git الرسمية إرشادات شاملة: https://git-scm.com/docs/git-cherry-pick. إذا واجهتك مشاكل محددة، شارك ناتج أمر `git log` الخاص بك!
