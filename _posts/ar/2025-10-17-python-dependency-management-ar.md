### ما هو Poetry؟

Poetry هو أداة لإدارة التبعيات والتعبئة في Python. يسمح لك بالإعلان عن المكتبات التي يعتمد عليها مشروعك، ويتعامل مع تثبيتها وتحديثها وحلها تلقائيًا مع ضمان بيئات قابلة للتكرار.

### الهدف الرئيسي
يبسط Poetry سير عمل مشاريع Python من خلال إدارة التبعيات عبر ملف تكوين واحد (`pyproject.toml`)، وإنشاء ملف قفل للتثبيتات المتسقة عبر الأجهزة، ودعم بناء المشاريع للتوزيع (مثل التوزيع إلى PyPI). تم تصميمه ليحل محل الأدوات المجزأة مثل `pip` و `pipenv` و `setuptools` بطريقة موحدة وسهلة الاستخدام.

### الميزات الرئيسية
- **حل التبعيات**: يثبت التبعيات ويحددها تلقائيًا، ويحل التعارضات بذكاء.
- **ملف القفل**: يولد `poetry.lock` لتثبيت الإصدارات الدقيقة للبناء القابل للتكرار.
- **البيئات الافتراضية**: ينشئ بيئات معزولة لكل مشروع لتجنب التلوث العالمي.
- **التعبئة**: يبني عجلات (wheels) أو حزم مصدرية (sdists) لتسهيل التوزيع.
- **دعم منصات متعددة**: يعمل على Linux و macOS و Windows.
- **دمج Shell**: إكمال تلقائي للأوامر (Tab completion) لـ Bash و Fish و Zsh.
- **متوافق مع Python 3.9+**: يتطلب إصدارات Python حديثة.

### التثبيت
ثبّت Poetry في بيئة معزولة باستخدام `pipx` (موصى به للوصول العالمي دون تعارضات):

1. ثبّت `pipx` إذا لزم الأمر (عبر مدير الحزم الخاص بنظام التشغيل أو `python -m pip install --user pipx`).
2. شغّل: `pipx install poetry`.

للتحديث: `pipx upgrade poetry`.  
لإلغاء التثبيت: `pipx uninstall poetry`.

مكّن إكمال الأوامر في Shell (مثال لـ Bash: `poetry completions bash >> ~/.bash_completion`).

### الاستخدام الأساسي
تهيئة مشروع جديد: `poetry init`.  
إضافة تبعية: `poetry add requests`.  
التثبيت من ملف القفل: `poetry install`.  
تشغيل النصوص البرمجية: `poetry run python script.py`.  
بناء الحزمة: `poetry build`.

للحصول على الأوامر الكاملة، راجع الوثائق.
