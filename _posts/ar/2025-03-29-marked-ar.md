للالنتقال من استخدام Kramdown إلى Marked في مدونة Jekyll الخاصة بك، ستحتاج إلى إجراء بعض التغييرات على إعدادات وإعداد Jekyll. Marked هي مكتبة JavaScript لتحليل وتجميع Markdown، ويمكن استخدامها بالتزامن مع MathJax لعرض رياضي أفضل. إليك دليل خطوة بخطوة لإجراء هذا التغيير:

### الخطوة 1: تثبيت Marked
أولاً، تحتاج إلى تضمين مكتبة Marked في مشروع Jekyll الخاص بك. يمكنك القيام بذلك عن طريق إضافتها إلى قسم `<head>` في ملف التخطيط الخاص بك (مثل `default.html`).

```html
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>
```

### الخطوة 2: تعطيل Kramdown
في ملف `_config.yml` الخاص بك، عطل Kramdown عن طريق تعيين خيار `markdown` على `none` أو ببساطة قم بإزالته إذا كان مضبوطًا على Kramdown.

```yaml
markdown: none
```

### الخطوة 3: تكوين Marked مع MathJax
تحتاج إلى تكوين Marked للعمل مع MathJax. أنشئ ملف JavaScript (مثل `marked-mathjax.js`) في مشروع Jekyll الخاص بك وقم بتضمين الكود التالي:

```javascript
document.addEventListener("DOMContentLoaded", function() {
  // عرض محتوى Markdown باستخدام Marked
  const elements = document.querySelectorAll('.markdown-content');
  elements.forEach(element => {
    const html = marked(element.textContent);
    element.innerHTML = html;
  });

  // عرض MathJax
  MathJax.typesetPromise().then(() => {
    console.log('MathJax rendering complete!');
  }).catch((err) => console.log('Typeset failed:', err));
});
```

قم بتضمين هذا السكريبت في ملف التخطيط الخاص بك بعد تضمين Marked و MathJax:

```html
<script src="https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-mml-chtml.js"></script>
<script src="{{ '/assets/js/marked-mathjax.js' | relative_url }}"></script>
```

### الخطوة 4: تحديث محتوى Markdown الخاص بك
تأكد من أن محتوى Markdown الخاص بك مغلف بحاوية تحتوي على الصنف `markdown-content` حتى يتمكن JavaScript من استهدافه للعرض.

```html
<div class="markdown-content">
  <!-- محتوى Markdown الخاص بك هنا -->
</div>
```

### الخطوة 5: اختبار إعداداتك
قم ببناء وتشغيل موقع Jekyll الخاص بك محليًا للتأكد من أن Marked يعرض Markdown الخاص بك بشكل صحيح وأن MathJax يعبر عن التعبيرات الرياضية بشكل صحيح.

```bash
bundle exec jekyll serve
```

### ملاحظات إضافية
- تأكد من أن تكوين MathJax متوافق مع محتوى Markdown الخاص بك. قد تحتاج إلى ضبط التكوين بناءً على احتياجاتك الخاصة.
- إذا واجهت أي مشاكل، تحقق من console المتصفح للبحث عن الأخطاء وقم بتعديل كود JavaScript الخاص بك وفقًا لذلك.

باتباع هذه الخطوات، يجب أن تكون قادرًا على الانتقال من Kramdown إلى Marked في مدونة Jekyll الخاصة بك والاستفادة من دعم MathJax الأفضل للتعبيرات الرياضية.
