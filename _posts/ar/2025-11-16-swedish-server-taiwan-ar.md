|---------------------------------|-------------------------------|-----------------------------------------------------|-----------------------------------|
| `mirror.accum.se` (سابقاً)     | مُوجّه → أعطاك تلقائياً **مرآة آسيوية** (دائماً تقريباً `ftp.acc.umu.se` **عبر** الخلفية في **اليابان** أو **سنغافورة**، أو أحياناً مرآة هونغ كونغ مباشرة) | الصين → (محلي) → وكيل تايوان → **مرآة ويكيميديا لمنطقة آسيا والمحيط الهادئ** (كتمون منخفض، نطاق ترددي عالٍ) | 20–60 ميجابايت/ثانية ممكنة بسهولة |
| `ftp.acc.umu.se` (الآن)         | يفرض الخادم **الأوروبي/السويدي** | الصين → وكيل تايوان → عبور المحيط الهادئ مرتين → أوروبا (السويد) → العودة | عادة 1–6 ميجابايت/ثانية، أحياناً تهبط لأقل من 1 ميجابايت/ثانية |

باختصار:  
عندما استخدمت `mirror.accum.se`، رأى الموجه الخاص بهم عنوان IP الخروج التايواني الخاص بك وأرسلك بذكاء إلى أقرب/أسرع مرآة (غالباً اليابان أو سنغافورة).  
الآن أنت تفرض الخادم السويدي، لذا يجب على حركة مرورك أن تسلك طريق قوانغتشو → تايوان → الولايات المتحدة/الكابلات تحت البحر → السويد → العودة بنفس الطريق الطويل. وقت الرحلة ذهاباً وإياباً يقفز من ~30 مللي ثانية إلى ~280–350 مللي ثانية، ويتم تخفيض النطاق الترددي بشكل كبير.

### إصلاحات سريعة (اختر واحدة)

1.  **الأفضل والأسرع**: التبديل إلى مرآة في شرق آسيا (مُوصى به)
    ```bash
    wget -c https://ftp.kaist.ac.kr/wikimedia/dumps/enwiki/20251101/enwiki-20251101-pages-articles2.xml-p41243p151573.bz2
    wget -c https://ftp.kaist.ac.kr/wikimedia/dumps/enwiki/20251101/enwiki-20251101-pages-articles3.xml-p151574p311329.bz2
    ```
    من وكيل تايوان، ستحصل بسهولة على 30–80 ميجابايت/ثانية.

2.  العودة إلى الموجه (أصبح ذكياً مرة أخرى)
    ```bash
    wget -c https://mirror.accum.se/mirror/wikimedia.org/dumps/enwiki/20251101/...
    ```
    سيعطيك تلقائياً أسرع مرآة متاحة لعنوان IP التايواني الخاص بك.

3.  إذا كانت قواعد Clash تسمح بذلك، أضف قاعدة لتجاوز الوكيل لـ `*.wikimedia.org` وقم بالتحميل محلياً (أسرع خيار ممكن، 100+ ميجابايت/ثانية من مرايا Tsinghua/SJTU).
