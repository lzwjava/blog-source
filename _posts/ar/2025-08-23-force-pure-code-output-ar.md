فهمت — تريد من Claude Code (وسير عمل "الموجه" الخاصة بك التي تستدعيه) إخراج **الكود فقط**، مع **عدم وجود تعليقات داخلية أو توثيق مضمّن أو شروحات**. إليك ثلاث طرق موثوقة لفرض ذلك، من الأكثر مرونة إلى الأكثر صرامة. بدون جداول — فقط خطوات مباشرة.

---

# 1) ضع القاعدة في `CLAUDE.md` (عالمي أو لكل مشروع)

Claude Code يحمل تلقائيًا ملفات "الذاكرة" ويتابعها عبر الجلسات. أضف قاعدة قصيرة وصريحة:

**عالمي (لجميع المشاريع):** `~/.claude/CLAUDE.md`
**لكل مشروع (يتم تتبعه):** `./CLAUDE.md`

```md
# قواعد الإخراج
- عند إنشاء الكود، أخرج الكود فقط.
- لا تدرج تعليقات داخلية، أو تعليقات كتلية، أو توثيقًا مضمّنًا، أو نثرًا توضيحيًا إلا إذا طلبت ذلك صراحة.
- اجعل الرسائل موجزة؛ إذا كانت هناك حاجة لشرح، ضعه بعد كتلة الكود وفقط إذا طلبت ذلك.
```

نصيحة: يمكنك إدارة هذه الذكريات باستخدام `/memory` وإضافة ذكريات جديدة بسرعة عن طريق بدء الرسالة بـ `#` في Claude Code. ([Anthropic][1])

---

# 2) اصنع **نمط إخراج مخصص** يمنع التعليقات (أقوى)

أنماط الإخراج تعدل **النص التوجيهي للنظام** في Claude Code. أنشئ واحدًا وانتقل إليه:

**الإنشاء:**

```
/output-style:new أريد نمط إخراج يخرج دائمًا كودًا بدون أي تعليقات أو توثيق مضمّن، ولا شروحات.
```

هذا ينشئ ملفًا تحت `~/.claude/output-styles/…`. يمكنك تحريره لضمان أن القاعدة لا لبس فيها:

```md
---
name: كود فقط (بدون تعليقات)
description: أخرج دائمًا كودًا بدون تعليقات/توثيق مضمّن؛ لا تقدم شروحات إلا إذا طُلب منك.
---
# التعليمات
- أنت تبرمج في وضع "الكود فقط".
- لا تدرج أبدًا تعليقات داخلية، أو تعليقات كتلية، أو توثيقًا مضمّنًا.
- لا تضيف نثرًا قبل الكود أو بعده إلا إذا طُلب منك صراحة.

## السلوكيات المحددة
- إذا كان المهمة تتضمن اختبارات، أخرج ملف(ات) الاختبار ككود فقط.
- إذا كانت هناك حاجة لتوضيح، اسأل بإيجاز، بدون شروحات.
```

**التفعيل:**

```
/output-style "كود فقط (بدون تعليقات)"
```

(أنماط الإخراج تغير النص التوجيهي للنظام مباشرة؛ يمكنك التبديل باستخدام `/output-style` في أي وقت.) ([Anthropic][2])

# 3) ألحق نصًا توجيهيًا إضافيًا للنظام أثناء التشغيل (الأقوى / لكل جلسة)

إذا قمت بتشغيل Claude Code من سطر الأوامر أو نص برمجي، أضف:

```bash
claude --append-system-prompt "أخرج دائمًا الكود فقط. لا تعليقات داخلية، لا تعليقات كتلية، لا توثيق مضمّن، لا شروحات إلا إذا طُلبت منك صراحة."
```

يمكنك أيضًا تعيين هذا عبر `/config` إذا كنت تفضل إعدادات مشروع دائمة. (تلاحظ Anthropic أنه يمكنك استخدام `--append-system-prompt` لتعديل سلوك Claude Code.) ([Anthropic][3])

---

## أين تضع القواعد الشاملة لـ "الموجه"

* إذا كان "موجه Claude Code" الخاص بك هو مجرد جلسات Claude Code العادية عبر المستودعات، فافضل **الذاكرة العالمية** (`~/.claude/CLAUDE.md`) أو **نمط الإخراج المخصص** حتى تنتقل القاعدة إلى كل مكان. ([Anthropic][1])
* إذا قمت بالإطلاق عبر نص برمجي، أدخل علم `--append-system-prompt` بحيث ترث كل عملية التشغيل القاعدة. ([Anthropic][3])

---

## حواجز حماية إضافية (اختيارية ولكن مفيدة)

* في ذاكرة المشروع، أضف سطر **"المخالفات"** حتى يقوم النموذج بالفحص الذاتي: "إذا كنت على وشك إخراج تعليقات/توثيق مضمّن، توقف وأزلهم."
* إذا أردت أحيانًا التعليقات، أضف تجاوزًا لمرة واحدة في نصك التوجيهي: "هذه المرة فقط، قم بتضمين تعليقات موجزة."

هذا كل شيء. إذا أخبرتني كيف تبدأ Claude Code (سطح المكتب، VS Code، JetBrains، أو الطرفية)، يمكنني أن أعطيك مقتطفًا جاهزًا لهذا الإعداد.
