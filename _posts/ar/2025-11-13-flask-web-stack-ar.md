### إطار العمل الأساسي للويب والأدوات المساعدة
- **Flask==2.0.3**: إطار عمل ويب خفيف الوزن من نوع WSGI لبناء تطبيقات الويب وواجهات برمجة التطبيقات (APIs) في بايثون. يوفر التوجيه، والقوالب، ومعالجة الطلبات.
- **click==8.0.4**: مكتبة لإنشاء واجهات سطر الأوامر (CLI) بأوامر قابلة للتركيب؛ تُستخدم غالبًا مع Flask للنصوص البرمجية أو أدوات CLI.
- **gunicorn==20.1.0**: خادم HTTP من نوع WSGI لنشر تطبيقات Flask في بيئة الإنتاج؛ يدعم عمالًا متعددين للتزامن.
- **Werkzeug==2.0.3**: مكتبة أدوات مساعدة شاملة لـ WSGI؛ تشغّل معالجة الطلبات/الاستجابات، والتصحيح، والتوجيه في Flask.
- **Jinja2==3.0.3**: محرك القوالب لعرض HTML/قوالب ديناميكية في تطبيقات Flask.
- **itsdangerous==2.0.1**: يساعد في توقيع وتسلسل البيانات بشكل آمن (مثل الرموز، ملفات تعريف الارتباط) لمنع العبث.
- **MarkupSafe==2.0.1**: يقوم بعزل السلاسل النصية لإخراج HTML آمن في قوالب Jinja2 لمنع هجمات XSS.
- **python-dotenv==0.19.2**: يقوم بتحميل متغيرات البيئة من ملف `.env` إلى `os.environ` لإدارة التكوين.

### REST API والامتدادات
- **flask-restx==0.5.1**: امتداد لـ Flask يضيف دعم Swagger/OpenAPI، والتحقق من صحة الإدخال/الإخراج، وتقسيم المساحات الاسمية لبناء واجهات برمجة تطبيقات RESTful.
- **Flask-Cors==3.0.10**: يتعامل مع عناوين الاستجابة لـ Cross-Origin Resource Sharing (CORS) للسماح بطلبات cross-domain في واجهات برمجة تطبيقات Flask.
- **Flask-JWT-Extended==4.4.1**: يدير JSON Web Tokens (JWT) للمصادقة؛ يدعم الرموز المميزة للوصول/التحديث، والقوائم السوداء، والمطالبات.
- **aniso8601==9.0.1**: يحلل سلاسل التاريخ/الوقت من نوع ISO 8601؛ يُستخدم من قبل flask-restx للتعامل مع datetime في نماذج/وثائق API.
- **jsonschema==4.0.0**: يتحقق من صحة بيانات JSON مقابل تعريفات JSON Schema؛ يتكامل مع flask-restx للتحقق من صحة حمولة API.

### قاعدة البيانات و ORM
- **Flask-SQLAlchemy==2.5.1**: يدمج ORM من SQLAlchemy مع Flask؛ يبسط التفاعلات مع قاعدة البيانات، والنماذج، والجلسات.
- **SQLAlchemy==1.4.46**: مجموعة أدوات SQL و Object Relational Mapper (ORM) لتجريد قاعدة البيانات، والاستعلام، والترحيل.
- **greenlet==2.0.1**: coroutines خفيفة الوزن لخيوط التنفيذ الخضراء؛ مطلوبة من قبل SQLAlchemy لدعم البرمجة غير المتزامنة (على الرغم من عدم استخدامها هنا).
- **Flask-Migrate**: امتداد للتعامل مع ترحيل مخطط قاعدة البيانات باستخدام Alembic؛ يتكامل مع Flask-SQLAlchemy.
- **pytz==2022.6**: يوفر تعريفات ومعالجة للنطاقات الزمنية؛ يُستخدم من قبل SQLAlchemy/Flask للتعامل مع أوقات datetime الواعية بالنطاق الزمني.

### HTTP والشبكات
- **requests==2.27.1**: عميل HTTP بسيط لإجراء مكالمات API (على سبيل المثال، إلى خدمات خارجية مثل OpenAI/Anthropic).
- **certifi==2022.12.7**: مجموعة من شهادات الجذر للتحقق من اتصالات SSL/TLS في الطلبات.
- **charset-normalizer~=2.0.0**: يكتشف ترميزات الأحرف في النص؛ يُستخدم من قبل requests لفك ترميز الاستجابة.
- **idna==3.4**: يدعم Internationalized Domain Names in Applications (IDNA) للتعامل مع عناوين URL.
- **urllib3==1.26.13**: مكتبة عميل HTTP مع تجميع اتصالات وSSL؛ المحرك الأساسي لـ requests.

### المصادقة والأمان
- **PyJWT==2.4.0**: ينفذ JSON Web Tokens لترميز/فك ترميز JWTs؛ يُستخدم من قبل Flask-JWT-Extended.

### معالجة البيانات
- **pandas==1.1.5**: مكتبة تحليل البيانات لمعالجة البيانات المنظمة (DataFrames)； مفيدة لمعالجة مدخلات/مخرجات API أو السجلات.

### تكاملات الذكاء الاصطناعي/التعلم الآلي
- **openai==0.8.0**: العميل الرسمي لـ OpenAI API؛ يسمح باستدعاء نماذج مثل GPT للإكمال، والتضمينات، وما إلى ذلك.
- **anthropic==0.28.0**: عميل لـ Anthropic API (على سبيل المثال، نماذج Claude)؛ مشابه لـ OpenAI للتفاعل مع النماذج اللغوية الكبيرة (LLMs).

### المراقبة والتسجيل
- **prometheus_client==0.14.1**: يولد مقاييس بتنسيق Prometheus لمراقبة أداء التطبيق (مثل زمن انتقال الطلبات، الأخطاء).
- **logstash-formatter**: يقوم بتنسيق رسائل السجل بتنسيق Logstash JSON للت兼容ية مع مجموعة ELK (Elasticsearch, Logstash, Kibana).
- **concurrent-log-handler**: معالج ملفات دوار يدعم التسجيل المتزامن من عمليات/خيوط متعددة دون تلف.

### قائمة انتظار المهام
- **rq**: قائمة انتظار وظائف بسيطة لـ Python باستخدام Redis؛ يقوم بإدراج المهام في الخلفية (مثل معالجة API غير المتزامنة) مع عمال.
