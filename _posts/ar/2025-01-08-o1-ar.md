واجهت مؤخرًا مشكلة مع حزمة `fontawesome5` في تدفق GitHub. الحل الذي قدمه 4o-mini (باستخدام TeX Live 2021 و `tlmgr install fontawesome5`) لم يعمل، لكن 4o اقترح نهجًا أفضل (باستخدام TeX Live 2023 واستخدام `tlmgr` لتثبيت `fontawesome5`)، وهو ما أحاوله حاليًا. لم يحل المشكلة تمامًا، لكن التبديل إلى TeX Live 2023 بدلًا من 2021 أحدث تحسنًا كبيرًا.

---

## إجابة 4o-mini

الخطأ الذي تراه مرتبط بـ `tlmgr` (مدير TeX Live) الذي لم يتم تهيئته بشكل صحيح في وضع المستخدم. على وجه التحديد، تشير الرسالة `tlmgr: user mode not initialized, please read the documentation!` إلى أن `tlmgr` لم يتم إعداده بشكل صحيح في بيئة المستخدم.

إليك تفصيل لكيفية إصلاحه:

### إصلاح خطأ وضع المستخدم في `tlmgr`

1. **تعيين `tlmgr` إلى وضع المستخدم بشكل صحيح**:
   تحتاج إلى إخبار `tlmgr` بشكل صريح بالعمل في وضع المستخدم والتأكد من تهيئته بشكل صحيح. للقيام بذلك في البرنامج النصي الخاص بك، استخدم الخطوات التالية:

   - تأكد من أنك تستخدم `tlmgr` في وضع المستخدم عن طريق تعيين متغير البيئة الصحيح.
   - يمكنك أيضًا تهيئة دليل TeX Live المخصص للمستخدم قبل استخدام `tlmgr`.

2. **تهيئة `tlmgr` في وضع المستخدم**:
   قبل تشغيل أوامر `tlmgr`، تأكد من تهيئة `tlmgr` باستخدام الأمر التالي:
   ```bash
   tlmgr init-usertree
   ```

   سيؤدي هذا إلى إنشاء شجرة TeX Live المخصصة للمستخدم في `~/texmf`. بمجرد التهيئة، يمكنك بعد ذلك تشغيل أوامر `tlmgr update` و `install`.

3. **تعديل سير عمل GitHub Actions**:
   قم بتحديث ملف سير عمل GitHub Actions لتضمين خطوة التهيئة. إليك مثال لكيفية تعديل السير العمل:

   ```yaml
   - name: Set tlmgr to user mode
     run: |
       # تهيئة دليل TeX Live المخصص للمستخدم
       tlmgr init-usertree
       tlmgr update --self
       tlmgr install fontawesome5
   ```

### مثال كامل:

إذا كنت تقوم بتشغيل هذا كجزء من سير عمل أكبر في GitHub Actions، يمكنك دمج هذه التغييرات في قسم الوظيفة ذات الصلة:

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Install TeX Live (xelatex)
        run: |
          sudo apt-get update
          sudo apt-get install -y texlive-xetex
          tlmgr init-usertree  # تهيئة شجرة TeX Live المخصصة للمستخدم
          tlmgr update --self  # تحديث tlmgr نفسه
          tlmgr install fontawesome5  # تثبيت الحزمة المطلوبة
```

### شرح الأوامر:

- `tlmgr init-usertree`: يقوم بتهيئة شجرة TeX Live المخصصة للمستخدم (عادةً ما تكون موجودة في `~/texmf`).
- `tlmgr update --self`: يقوم بتحديث حزمة `tlmgr` نفسها إلى أحدث إصدار.
- `tlmgr install fontawesome5`: يقوم بتثبيت حزمة `fontawesome5`.

### ملاحظات إضافية:

- **الأذونات**: تأكد من أن المستخدم الذي يقوم بتشغيل سير عمل GitHub Actions لديه إذن لتثبيت وتعديل الملفات في دليل TeX Live المخصص للمستخدم.
- **متغيرات البيئة**: إذا كنت تعمل مع مسارات مخصصة، قد تحتاج إلى تعديل `TEXMFSYSVAR` أو `TEXMFHOME` للإشارة إلى الأدلة المحددة لديك.

من خلال التأكد من تهيئة `tlmgr` في وضع المستخدم وإعداده بشكل صحيح في سير عمل GitHub Actions الخاص بك، يجب أن يتم حل الخطأ. أخبرني إذا كان هذا يعمل أو إذا واجهت أي مشاكل أخرى!

### إجابة o1

أفهم أنك تواجه مشاكل مع `tlmgr` (مدير TeX Live) في سير عمل GitHub Actions الخاص بك، خاصة عند محاولة تثبيت حزمة `fontawesome5`. رسالة الخطأ:

```
TLPDB: not a directory, not loading: /home/runner/texmf
tlmgr: user mode not initialized, please read the documentation!
Error: Process completed with exit code 2.
```

تشير إلى أن `tlmgr` غير قادر على العمل في "وضع المستخدم" لأن الأدلة والتكوينات اللازمة لم يتم إعدادها بشكل صحيح. غالبًا ما تحدث هذه المشكلة عند استخدام تثبيتات TeX Live المقدمة من مديري الحزم مثل `apt-get`، والتي يمكن أن تكون محدودة الوظائف مقارنة بتثبيت TeX Live الكامل من TUG (مجموعة مستخدمي TeX).

لحل هذه المشكلة، أوصي **بتثبيت TeX Live 2023 يدويًا** داخل سير عمل GitHub Actions الخاص بك. يضمن هذا النهج أن `tlmgr` لديه وظائف كاملة ويمكنه تثبيت حزم إضافية مثل `fontawesome5` دون مشاكل.

### الحل خطوة بخطوة:

1. **إزالة تثبيت TeX Live من النظام:**
   - تجنب استخدام `apt-get install texlive-xetex` لأنه يوفر تثبيتًا جزئيًا لـ TeX Live مع وظائف محدودة لـ `tlmgr`.
   
2. **تثبيت TeX Live 2023 يدويًا:**
   - قم بتنزيل وتثبيت TeX Live مباشرة من TUG.
   
3. **تكوين متغير بيئة PATH:**
   - تأكد من أن ملفات TeX Live الثنائية قابلة للوصول عن طريق تحديث PATH.

4. **استخدام `tlmgr` لتثبيت الحزم المطلوبة:**
   - قم بتحديث `tlmgr` وقم بتثبيت `fontawesome5`.

5. **تحسين سير العمل باستخدام التخزين المؤقت (اختياري):**
   - قم بتخزين تثبيت TeX Live مؤقتًا لتسريع عمليات سير العمل المستقبلية.

### سير عمل GitHub Actions المحدث:

إليك نسخة محدثة من سير عمل GitHub Actions الخاص بك والتي تتضمن التثبيت اليدوي لـ TeX Live 2023 وإعداد `tlmgr` بشكل صحيح لتثبيت حزمة `fontawesome5`.

```yaml
name: Deploy Jekyll with GitHub Pages dependencies preinstalled

on:
  push:
    branches: ["master"]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  awesome-cv-copy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up Make
        run: sudo apt-get update && sudo apt-get install -y make

      - name: Cache TeX Live
        uses: actions/cache@v3
        with:
          path: /usr/local/texlive/2023
          key: ${{ runner.os }}-texlive-2023
          restore-keys: |
            ${{ runner.os }}-texlive-

      - name: Install TeX Live 2023
        if: steps.cache-texlive.outputs.cache-hit != 'true'
        run: |
          # تثبيت التبعيات اللازمة لتثبيت TeX Live
          sudo apt-get update
          sudo apt-get install -y perl wget xz-utils

          # تنزيل مثبت TeX Live
          wget http://mirror.ctan.org/systems/texlive/tlnet/install-tl-unx.tar.gz
          tar -xzf install-tl-unx.tar.gz
          cd install-tl-*/

          # إنشاء ملف texlive.profile للتثبيت التلقائي
          cat <<EOF > texlive.profile
          selected_scheme scheme-full
          TEXDIR /usr/local/texlive/2023
          TEXMFCONFIG ~/.texlive2023/texmf-config
          TEXMFHOME ~/texmf
          TEXMFLOCAL /usr/local/texlive/2023/texmf-local
          TEXMFSYSCONFIG /usr/local/texlive/2023/texmf-config
          TEXMFSYSVAR /usr/local/texlive/2023/texmf-var
          binary_x86_64-linux 1
          collection-basic 1
          collection-latex 1
          collection-fontsrecommended 1
          EOF

          # تشغيل المثبت باستخدام الملف الشخصي
          sudo ./install-tl --profile=texlive.profile

      - name: Add TeX Live to PATH
        run: echo "/usr/local/texlive/2023/bin/x86_64-linux" >> $GITHUB_PATH

      - name: Update tlmgr and Install fontawesome5
        run: |
          tlmgr update --self
          tlmgr install fontawesome5

      - name: Run make awesome-cv-copy
        run: make awesome-cv-copy

  build:
    runs-on: ubuntu-latest
    needs: awesome-cv-copy
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Setup Python 3
        uses: actions/setup-python@v4
        with:
          python-version: "3.x"

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install python-frontmatter

      - name: Generate notes links
        run: |
          python scripts/generate_notes_link.py

      - name: Setup Pages
        uses: actions/configure-pages@v5

      - name: Build with Jekyll
        uses: actions/jekyll-build-pages@v1
        with:
          source: ./
          destination: ./_site

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### شرح مفصل:

1. **التحقق من المستودع:**
   - يستخدم إجراء `actions/checkout@v4` لاستنساخ مستودعك.

2. **إعداد Make:**
   - يقوم بتثبيت `make` باستخدام `apt-get`، مما يضمن توافره لبناء سيرتك الذاتية.

3. **تخزين TeX Live مؤقتًا:**
   - يستخدم `actions/cache@v3` لتخزين تثبيت TeX Live الموجود في `/usr/local/texlive/2023`. هذا يقلل بشكل كبير من وقت التثبيت لعمليات سير العمل المستقبلية.

4. **تثبيت TeX Live 2023:**
   - **التنفيذ الشرطي:** الشرط `if: steps.cache-texlive.outputs.cache-hit != 'true'` يضمن أن TeX Live يتم تثبيته فقط إذا لم يتم تخزينه مؤقتًا بالفعل.
   - **تثبيت التبعيات:** يقوم بتثبيت التبعيات اللازمة مثل Perl و `wget` و `xz-utils`.
   - **تنزيل واستخراج المثبت:** يقوم بتنزيل أحدث مثبت لـ TeX Live من CTAN واستخراجه.
   - **إنشاء ملف التثبيت التلقائي:** يتم إنشاء ملف `texlive.profile` لأتمتة عملية التثبيت. يحدد هذا الملف:
     - **`selected_scheme scheme-full`:** يقوم بتثبيت توزيعة TeX Live الكاملة. يمكنك تعديل هذا إلى مخطط بسيط إذا كنت تريد.
     - **الأدلة:** يحدد مكان تثبيت TeX Live وتكويناته.
     - **المجموعات:** يختار المجموعات الأساسية مثل `basic` و `latex` و `fontsrecommended`.
   - **تشغيل المثبت:** يقوم بتشغيل المثبت باستخدام الملف الشخصي المحدد، مما يؤدي إلى تثبيت TeX Live 2023 في `/usr/local/texlive/2023`.

5. **إضافة TeX Live إلى PATH:**
   - يقوم بتحديث متغير بيئة `PATH` عن طريق إضافة دليل TeX Live الثنائي. هذا يضمن أن `tlmgr` وأوامر TeX Live الأخرى تكون قابلة للوصول في الخطوات اللاحقة.

6. **تحديث tlmgr وتثبيت `fontawesome5`:**
   - **تحديث `tlmgr`:** يضمن أن `tlmgr` نفسه محدث.
   - **تثبيت `fontawesome5`:** يستخدم `tlmgr` لتثبيت حزمة `fontawesome5`، مما يحل المشكلة الأولية.

7. **تشغيل أمر Make:**
   - يقوم بتنفيذ `make awesome-cv-copy` لبناء سيرتك الذاتية باستخدام إعداد TeX Live المثبت.

8. **وظائف البناء والنشر:**
   - تظل هذه الوظائف كما هي تقريبًا ولكنها تضمن أن تثبيت TeX Live متاح طوال سير العمل بسبب تحديث PATH.

### توصيات إضافية:

- **وقت تثبيت TeX Live:**
  - **التخزين المؤقت:** تضمين خطوة `actions/cache@v3` يساعد في تقليل وقت التثبيت الطويل عن طريق تخزين TeX Live مؤقتًا بعد التشغيل الأول. تأكد من أن مفتاح التخزين المؤقت (`${{ runner.os }}-texlive-2023`) يعكس إصدار TeX Live بدقة لتجنب عدم تطابق التخزين المؤقت.
  
- **تخصيص تثبيت TeX Live:**
  - إذا كنت لا تحتاج إلى توزيعة TeX Live الكاملة، يمكنك تعديل ملف `texlive.profile` لتثبيت مجموعات محددة فقط، مما يقلل من وقت التثبيت واستخدام القرص. على سبيل المثال، استبدل `scheme-full` بـ `scheme-basic` وحدد المجموعات الضرورية فقط.

- **التحقق من التثبيت:**
  - للتأكد من أن TeX Live مثبت بشكل صحيح، يمكنك إضافة خطوة تحقق:
    ```yaml
    - name: Verify TeX Live Installation
      run: |
        tex --version
        tlmgr --version
        kpsewhich fontawesome5.sty
    ```
    يمكن أن تساعد هذه الخطوة في تشخيص أي مشاكل عن طريق التأكد من أن أوامر TeX Live قابلة للوصول وأن `fontawesome5.sty` مثبتة.

- **التعامل مع PATH بشكل صحيح:**
  - استخدام `echo "/usr/local/texlive/2023/bin/x86_64-linux" >> $GITHUB_PATH` يضمن أن ملفات TeX Live الثنائية تتم إضافتها إلى PATH لجميع الخطوات اللاحقة في الوظيفة. هذا أمر بالغ الأهمية لكي تعمل أوامر `tlmgr` و LaTeX بشكل صحيح.

### نصائح استكشاف الأخطاء وإصلاحها:

1. **مشاكل عدم تطابق التخزين المؤقت:**
   - تأكد من أن مفتاح التخزين المؤقت يعكس إصدار TeX Live بدقة. إذا قمت بتحديث TeX Live أو تغيير مسار التثبيت، قم بتحديث مفتاح التخزين المؤقت وفقًا لذلك.

2. **فشل التثبيت:**
   - تحقق من سجلات GitHub Actions لأي أخطاء أثناء تثبيت TeX Live. تشمل المشاكل الشائعة مشاكل الشبكة عند تنزيل المثبت أو أذونات غير كافية.

3. **أخطاء تثبيت الحزم:**
   - إذا فشل `tlmgr` في تثبيت `fontawesome5`، تحقق من أن اسم الحزمة صحيح وأنها متوفرة في مستودع TeX Live.

4. **مشاكل PATH:**
   - تأكد من تحديث PATH بشكل صحيح. يمكنك إضافة خطوة لطباعة PATH لأغراض التصحيح:
     ```yaml
     - name: Print PATH
       run: echo $PATH
     ```

5. **قيود مساحة القرص:**
   - لدى عداء GitHub Actions مساحة قرص محدودة. يمكن أن يستهلك تثبيت TeX Live الكامل مساحة كبيرة. إذا واجهت مشاكل في مساحة القرص، ففكر في تثبيت إعداد TeX Live بسيط.
