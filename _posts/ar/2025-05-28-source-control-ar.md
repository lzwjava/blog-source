يتمتع Visual Studio Code (VS Code) بدعم مدمج قوي للتحكم في المصدر، مما يسمح للمطورين بإدارة أنظمة التحكم في الإصدار (VCS) مثل Git و Mercurial وغيرها مباشرة داخل المحرر. يقدم هذا الدليل نظرة شاملة حول استخدام التحكم في المصدر في VS Code، مع التركيز على Git، باعتباره نظام التحكم في الإصدار الأكثر استخدامًا، ويغطي الإعداد والميزات الرئيسية وسير العمل والاستخدام المتقدم.

### نظرة عامة على التحكم في المصدر في VS Code
توفر طريقة عرض "التحكم في المصدر" في VS Code واجهة بديهية للتفاعل مع أنظمة التحكم في الإصدار. وهي تتكامل مع Git افتراضيًا وتدعم الإضافات لأنظمة التحكم في الإصدار الأخرى. تعرض طريقة عرض "التحكم في المصدر" التغييرات، وتسمح بالتدرج والإيداع والتفرع والدمج، والمزيد، كل ذلك دون مغادرة المحرر. فيما يلي دليل خطوة بخطوة للاستفادة من التحكم في المصدر بفعالية.

### 1. **إعداد التحكم في المصدر في VS Code**
لاستخدام التحكم في المصدر، تحتاج إلى تثبيت Git وتهيئة مستودع. إليك كيفية إعداده:

#### المتطلبات الأساسية
- **تثبيت Git**: قم بتنزيل وتثبيت Git من [git-scm.com](https://git-scm.com/). تحقق من التثبيت بتشغيل `git --version` في الطرفية.
- **تهيئة Git**:
  ```bash
  git config --global user.name "Your Name"
  git config --global user.email "your.email@example.com"
  ```
- **تثبيت VS Code**: تأكد من أن لديك أحدث إصدار من VS Code مثبتًا من [code.visualstudio.com](https://code.visualstudio.com/).

#### تهيئة مستودع Git
1. افتح مجلد مشروع في VS Code.
2. افتح الطرفية (Ctrl+` أو Cmd+` على macOS) وقم بتشغيل:
   ```bash
   git init
   ```
   يؤدي هذا إلى إنشاء مجلد `.git` في مشروعك، مما يهيئه كمستودع Git.
3. بدلاً من ذلك، استنسخ مستودعًا موجودًا:
   ```bash
   git clone <رابط-المستودع>
   ```
   ثم افتح المجلد المستنسخ في VS Code.

#### تمكين طريقة عرض التحكم في المصدر
- افتح طريقة عرض "التحكم في المصدر" بالنقر على أيقونة التحكم في المصدر في شريط النشاط (الأيقونة الثالثة من الأعلى، التي تشبه الفرع) أو بالضغط على `Ctrl+Shift+G` (Windows/Linux) أو `Cmd+Shift+G` (macOS).
- إذا تم اكتشاف مستودع Git، ستعرض VS Code طريقة عرض "التحكم في المصدر" مع خيارات لإدارة التغييرات.

### 2. **استخدام طريقة عرض التحكم في المصدر**
تعتبر طريقة عرض "التحكم في المصدر" المركز الرئيسي لمهام التحكم في الإصدار. وهي تعرض:
- **التغييرات**: الملفات المعدلة أو المضافة أو المحذوفة منذ آخر إيداع.
- **التغييرات المُعدّة**: الملفات الجاهزة للإيداع.
- **مربع رسالة الإيداع**: حيث تدخل رسائل الإيداع.
- **الإجراءات**: أزرار للإيداع، التحديث، والمزيد.

#### سير العمل الشائع
1. **إجراء التغييرات**: قم بتحرير الملفات في مشروعك. تكتشف VS Code التغييرات تلقائيًا وتدرجها تحت "التغييرات" في طريقة عرض "التحكم في المصدر".
2. **إعداد التغييرات**:
   - انقر على أيقونة `+` بجانب الملف لإعداده، أو استخدم خيار `Stage All Changes` (قائمة النقاط الثلاث > Stage All Changes).
   - يجهز الإعداد التغييرات للإيداع التالي.
3. **كتابة رسالة إيداع**:
   - أدخل رسالة وصفية في مربع النص في أعلى طريقة عرض "التحكم في المصدر".
   - مثال: `Add user authentication feature`.
4. **إيداع التغييرات**:
   - انقر على أيقونة علامة الاختيار أو اضغط على `Ctrl+Enter` (Windows/Linux) أو `Cmd+Enter` (macOS) لإيداع التغييرات المُعدّة.
   - استخدم قائمة النقاط الثلاث للاختيار بين `Commit All`، أو `Commit Staged`، أو `Commit All and Push`.
5. **الدفع إلى المستودع البعيد**:
   - إذا كنت متصلاً بمستودع بعيد (مثل GitHub)، ادفع التغييرات باستخدام خيار `Push` في قائمة النقاط الثلاث أو قم بتشغيل `git push` في الطرفية.

### 3. **الميزات الرئيسية للتحكم في المصدر في VS Code**
توفر VS Code عدة ميزات لتبسيط التحكم في الإصدار:

#### طريقة عرض المقارنة (Diff View)
- انقر على ملف تحت "التغييرات" لفتح طريقة عرض مقارنة جنبًا إلى جنب، تُظهر التعديلات مقارنة بآخر إيداع.
- استخدم الإجراءات المضمنة لإعداد أو تجاهل أسطر محددة.

#### إدارة الفروع
- تبديل الفروع: انقر على اسم الفرع في شريط الحالة أسفل اليسار أو استخدم قائمة الفروع في طريقة عرض "التحكم في المصدر" (النقاط الثلاث > Branch > Checkout to...).
- إنشاء فرع جديد: حدد `Create Branch` من قائمة الفروع، أدخل اسمًا، وقم بالتأكيد.
- دمج الفروع: استخدم `Branch > Merge Branch` وحدد الفرع المطلوب دمجه مع الفرع الحالي.

#### السحب وجلب التغييرات (Pull and Fetch)
- **سحب (Pull)**: زامن التغييرات من المستودع البعيد باستخدام خيار `Pull` في قائمة النقاط الثلاث.
- **جلب (Fetch)**: استرجع التغييرات البعيدة دون دمج باستخدام `Fetch`.

#### حل التعارضات
- عند الدمج أو السحب، قد تنشأ تعارضات. تبرز VS Code التعارضات في الملفات وتوفر واجهة حل تعارضات مدمجة:
  - اختر `Accept Current Change`، أو `Accept Incoming Change`، أو `Accept Both Changes`، أو حرر الملف يدويًا.
  - قم بإعداد الملف المُحلّ ثم أودعه.

#### إضافة Git Lens
للحصول على ميزات Git متقدمة، قم بتثبيت إضافة **GitLens**:
- عرض سجل الإيداعات، وحواشي المسؤولية (blame)، وتغييرات الملفات.
- الوصول إلى رؤى المستودع مثل آخر الإيداعات والتخزينات المؤقتة (stashes).
- قم بالتثبيت عبر طريقة عرض "الإضافات" (`Ctrl+Shift+X` أو `Cmd+Shift+X`).

### 4. **الاستخدام المتقدم**
#### التخزين المؤقت للتغييرات (Stashing)
- احفظ التغييرات غير المودعة مؤقتًا:
  - انتقل إلى قائمة النقاط الثلاث > Stash > Stash.
  - طبق أو استرجع التخزينات لاحقًا عبر نفس القائمة.
- مفيد لتبديل الفروع دون إيداع عمل غير مكتمل.

#### أوامر Git في الطرفية
- للمهام غير المدعومة مباشرة في الواجهة، استخدم الطرفية المدمجة:
  ```bash
  git rebase <الفرع>
  git cherry-pick <الإيداع>
  git log --oneline
  ```

#### تخصيص التحكم في المصدر
- **الإعدادات**: اضبط سلوك التحكم في المصدر في إعدادات VS Code (`Ctrl+,` أو `Cmd+,`):
  - `git.autoRepositoryDetection`: تمكين/تعطيل الكشف التلقائي عن مستودعات Git.
  - `git.enableSmartCommit`: إيداع جميع التغييرات عندما لا تكون هناك ملفات مُعدّة.
- **موفرو SCM**: قم بتثبيت إضافات لأنظمة تحكم إصدار أخرى مثل Mercurial أو SVN.

#### التكامل مع GitHub
- استخدم إضافة **GitHub Pull Requests and Issues** لإدارة طلبات السحب (PRs) والمشكلات مباشرة في VS Code.
- قم بالمصادقة مع GitHub عبر قائمة "الحسابات" (في الزاوية اليسرى السفلية) للدفع/السحب من مستودعات GitHub.

### 5. **سير عمل نموذجي: إنشاء ودفع فرع ميزة**
إليك مثالًا عمليًا لسير عمل Git شائع في VS Code:


# سير عمل Git نموذجي في VS Code

## خطوات إنشاء ودفع فرع ميزة

1. **إنشاء فرع جديد**:
   - في طريقة عرض "التحكم في المصدر"، انقر على اسم الفرع في شريط الحالة أو استخدم قائمة النقاط الثلاث > Branch > Create Branch.
   - سمّ الفرع، على سبيل المثال، `feature/add-login`.
   - ينتقل VS Code إلى الفرع الجديد.

2. **إجراء وإعداد التغييرات**:
   - حرر الملفات (على سبيل المثال، أضف مكون تسجيل دخول إلى `src/Login.js`).
   - في طريقة عرض "التحكم في المصدر"، ستظهر الملفات تحت "التغييرات."
   - انقر على أيقونة `+` لإعداد التغييرات أو حدد "Stage All Changes."

3. **إيداع التغييرات**:
   - أدخل رسالة إيداع، على سبيل المثال، `Add login component`.
   - انقر على علامة الاختيار أو اضغط على `Ctrl+Enter` (Windows/Linux) أو `Cmd+Enter` (macOS) للإيداع.

4. **دفع الفرع**:
   - إذا لم يكن هناك مستودع بعيد، أضف واحدًا:
     ```bash
     git remote add origin <رابط-المستودع>
     ```
   - ادفع الفرع: قائمة النقاط الثلاث > Push، أو شغّل:
     ```bash
     git push -u origin feature/add-login
     ```

5. **إنشاء طلب سحب (Pull Request)**:
   - إذا كنت تستخدم GitHub، افتح المستودع في متصفح أو استخدم إضافة GitHub Pull Requests لإنشاء طلب سحب.
   - اربط طلب السحب بفرع `feature/add-login`.

## نصائح
- اسحب التحديثات بانتظام من الفرع الرئيسي لتجنب التعارضات.
- استخدم رسائل إيداع وصفية لتحسين التعاون.
- قم بتثبيت GitLens للحصول على عرض محسن لسجل الإيداعات وحواشي المسؤولية.


### 6. **استكشاف الأخطاء وإصلاحها الشائعة**
- **لم يتم اكتشاف Git**: تأكد من تثبيت Git وإضافته إلى مسار النظام (PATH). أعد تشغيل VS Code بعد التثبيت.
- **أخطاء المصادقة**: قم بتكوين مفاتيح SSH أو استخدم رمز وصول شخصي لـ GitHub/GitLab. قم بالمصادقة عبر قائمة "الحسابات".
- **تعارضات الدمج**: استخدم أدوات حل التعارضات المدمجة أو حرر الملفات يدويًا، ثم أعدها وأودعها.
- **تباطؤ الأداء**: عطّل `git.autoRepositoryDetection` للمشاريع الكبيرة أو استخدم GitLens لعمليات Git محسنة.

### 7. **أفضل الممارسات**
- **أودع كثيرًا**: قم بعمل إيداعات صغيرة ومنطقية لتتبع التقدم.
- **استخدم رسائل وصفية**: اتبع اصطلاحًا مثل `<type>(<scope>): <description>` (على سبيل المثال، `feat(auth): add login endpoint`).
- **اسحب بانتظام**: حافظ على المزامنة مع المستودع البعيد لتقليل التعارضات.
- **احتفظ بنسخ احتياطية للتخزينات المؤقتة**: قبل تطبيق التخزينات المؤقتة، تأكد من أنها لن تستبدل تغييرات مهمة.
- **استفد من الإضافات**: استخدم GitLens أو Git History أو Git Graph للحصول على وظائف محسنة.
