GitHub Copilot هو مساعد ترميز مدعوم بالذكاء الاصطناعي طورته GitHub وOpenAI، مصمم لتعزيز إنتاجية المطورين في Visual Studio Code (VS Code). يوفر إكمالًا للكود يدرك السياق، ودردشة باللغة الطبيعية، وإمكانيات تحرير متقدمة. يرشدك هذا الدليل خلال إعداد Copilot، واستكشاف ميزاته، وتحسين سير عملك باستخدام النصائح وأفضل الممارسات.

## 1. مقدمة إلى GitHub Copilot
يعمل GitHub Copilot كمبرمج زوجي بالذكاء الاصطناعي، حيث يقدم اقتراحات كود في الوقت الفعلي، ويجيب على أسئلة البرمجة، وأتمتة المهام المتكررة. يعمل بواسطة نماذج لغوية كبيرة تم تدريبها على مجموعات بيانات ضخمة من الكود العام، مما يمكنه من دعم العديد من لغات البرمجة وأطر العمل، بما في ذلك Python وJavaScript وTypeScript وRuby وGo وC# وC++.

تشمل الميزات الرئيسية:
- **إكمال الكود**: يقترح مقاطع كود أثناء الكتابة، من أسطر مفردة إلى دوال كاملة.
- **دردشة Copilot**: يسمح باستعلامات اللغة الطبيعية لشرح الكود، أو إنشاء مقاطع، أو تصحيح الأخطاء.
- **وضع الوكيل**: يؤتمت مهام البرمجة متعددة الخطوات، مثل إعادة الهيكلة أو إنشاء التطبيقات.
- **تعليمات مخصصة**: يعدل الاقتراحات لتتناسب مع أسلوبك في البرمجة أو متطلبات مشروعك.

## 2. إعداد GitHub Copilot في VS Code

### المتطلبات الأساسية
- **VS Code**: حمل ونصب Visual Studio Code من [الموقع الرسمي](https://code.visualstudio.com/). تأكد من أن لديك إصدارًا متوافقًا (أي إصدار حديث يدعم Copilot).
- **حساب GitHub**: تحتاج إلى حساب GitHub مع إمكانية الوصول إلى Copilot. الخيارات تشمل:
  - **Copilot مجاني**: إكمالات وتفاعلات دردشة محدودة شهريًا.
  - **Copilot Pro/Pro+**: خطط مدفوعة بحدود أعلى وميزات متقدمة.
  - **الوصول التنظيمي**: إذا وفرته مؤسستك، تحقق من مسؤولك للحصول على تفاصيل الوصول.
- **اتصال بالإنترنت**: يتطلب Copilot اتصالاً نشطًا لتقديم الاقتراحات.

### خطوات التثبيت
1. **افتح VS Code**:
   شغل Visual Studio Code على جهازك.

2. **ثبّت امتداد GitHub Copilot**:
   - اذهب إلى عرض **الامتدادات** (Ctrl+Shift+X أو Cmd+Shift+X على macOS).
   - ابحث عن "GitHub Copilot" في سوق الامتدادات.
   - انقر على **تثبيت** لامتداد GitHub Copilot الرسمي. هذا يثبّت تلقائيًا امتداد Copilot Chat أيضًا.

3. **سجّل الدخول إلى GitHub**:
   - بعد التثبيت، قد تظهر مطالبة في شريط حالة VS Code (الزاوية اليمنى السفلى) لإعداد Copilot.
   - انقر على أيقونة Copilot واختر **تسجيل الدخول** للمصادقة باستخدام حساب GitHub الخاص بك.
   - إذا لم تظهر أي مطالبة، افتح لوحة الأوامر (Ctrl+Shift+P أو Cmd+Shift+P) وشغل الأمر `GitHub Copilot: Sign in`.
   - اتبع عملية المصادقة المستندة إلى المتصفح، من خلال نسخ الرمز المقدم من VS Code إلى GitHub.

4. **تحقق من التفعيل**:
   - بمجرد تسجيل الدخول، يجب أن تتحول أيقونة Copilot في شريط الحالة إلى اللون الأخضر، مما يشير إلى الحالة النشطة.
   - إذا لم يكن لديك اشتراك في Copilot، سيتم تسجيلك في الخطة المجانية لـ Copilot مع استخدام شهري محدود.

5. **اختياري: عطّل إرسال بيانات الاستخدام**:
   - افتراضيًا، يجمع Copilot بيانات الاستخدام. لتعطيله، اذهب إلى **الإعدادات** (Ctrl+, أو Cmd+,)، ابحث عن `telemetry.telemetryLevel`، وعينها إلى `off`. بدلاً من ذلك، اضبط إعدادات Copilot المحددة تحت `GitHub Copilot Settings`.

> **ملاحظة**: إذا عطلت مؤسستك Copilot Chat أو قيدت الميزات، اتصل بمسؤولك. لاستكشاف الأخطاء وإصلاحها، راجع [دليل استكشاف أخطاء GitHub Copilot وإصلاحها](https://docs.github.com/en/copilot/troubleshooting).[](https://code.visualstudio.com/docs/copilot/setup)

## 3. الميزات الأساسية لـ GitHub Copilot في VS Code

### 3.1 إكمال الكود
يقترح Copilot الكود أثناء الكتابة، من أسطر مفردة إلى دوال أو فئات كاملة، بناءً على سياق الكود الخاص بك وبنية الملف.
- **كيف يعمل**:
  - ابدأ بالكتابة بلغة مدعومة (مثل JavaScript، Python، C#).
  - يعرض Copilot الاقتراحات في نص "شبح" رمادي.
  - اضغط **Tab** لقبول الاقتراح أو استمر في الكتابة لتجاهله.
  - استخدم **Alt+]** (التالي) أو **Alt+[** (السابق) للتنقل بين الاقتراحات المتعددة.
- **مثال**:
  ```javascript
  // Calculate factorial of a number
  function factorial(n) {
  ```
  قد يقترح Copilot:
  ```javascript
  if (n === 0) return 1;
  return n * factorial(n - 1);
  }
  ```
  اضغط **Tab** لقبول الاقتراح.

- **نصائح**:
  - استخدم أسماء دوال أو تعليقات وصافية لتوجيه Copilot (مثال: `// Sort an array in ascending order`).
  - للاقتراحات المتعددة، مرر مؤشر الفأرة فوق الاقتراح لفتح لوحة الإكمال (Ctrl+Enter) لعرض جميع الخيارات.

### 3.2 دردشة Copilot
تسمح لك دردشة Copilot بالتفاعل مع Copilot باستخدام اللغة الطبيعية لطرح الأسئلة، أو إنشاء كود، أو تصحيح الأخطاء.
- **الوصول إلى الدردشة**:
  - افتح **عرض الدردشة** من شريط النشاط أو استخدم **Ctrl+Alt+I** (Windows/Linux) أو **Cmd+Ctrl+I** (macOS).
  - بدلاً من ذلك، استخدم **الدردشة المضمنة** (Ctrl+I أو Cmd+I) مباشرة في المحرر للاستفسارات الخاصة بالسياق.
- **حالات الاستخدام**:
  - **شرح الكود**: حدد مقطع كود، افتح الدردشة المضمنة، واكتب `explain this code`.
  - **إنشاء كود**: اكتب `write a Python function to reverse a string` في عرض الدردشة.
  - **تصحيح الأخطاء**: الصق رسالة خطأ في الدردشة واطلب إصلاحًا.
- **مثال**:
  في عرض الدردشة، اكتب:
  ```
  What is recursion?
  ```
  سيجيب Copilot بشرح مفصل، غالبًا ما يتضمن أمثلة كود بـ Markdown.

- **أوامر الشرطة المائلة**:
  استخدم أوامر مثل `/explain`، `/doc`، `/fix`، `/tests`، أو `/optimize` لتحديد المهام. على سبيل المثال:
  ```
  /explain this function
  ```
  مع تحديد دالة سينشئ شرحًا مفصلاً.

### 3.3 وضع الوكيل (معاينة)
يمكن وضع الوكيل Copilot من التعامل ذاتيًا مع مهام البرمجة متعددة الخطوات، مثل إنشاء التطبيقات، أو إعادة هيكلة الكود، أو كتابة الاختبارات.
- **كيفية الاستخدام**:
  - افتح **عرض تعديلات Copilot** في VS Code Insiders أو Stable (إذا كان متاحًا).
  - اختر **الوكيل** من القائمة المنسدلة للأوضاع.
  - أدخل مطالبة، مثال: `Create a React form component with name and email fields`.
  - يحلل Copilot قاعدة الكود الخاصة بك، ويقترح تعديلات، ويمكنه تشغيل أوامر الطرفية أو الاختبارات.
- **القدرات**:
  - ينشئ كودًا عبر ملفات متعددة.
  - يراقب الأخطاء ويكرر لإصلاح المشكلات.
  - يدمج مكتبات جديدة أو يهاجر الكود إلى أطر عمل حديثة.

> **ملاحظة**: وضع الوكيل تجريدي ويعمل بشكل أفضل في المستودعات الأصغر. شارك ملاحظاتك عبر مستودع VS Code على GitHub.[](https://code.visualstudio.com/blogs/2025/02/24/introducing-copilot-agent-mode)

### 3.4 تعليمات مخصصة
خصص Copilot ليتوافق مع أسلوبك في البرمجة أو متطلبات مشروعك.
- **الإعداد**:
  - أنشئ ملف `.github/copilot-instructions.md` في مساحة العمل الخاصة بك.
  - أضف تعليمات بـ Markdown، مثال: `Use snake_case for Python variable names`.
  - مكّن التعليمات المخصصة في **الإعدادات** > **GitHub** > **Copilot** > **Enable custom instructions** (VS Code 17.12 أو أحدث).
- **مثال**:
  ```markdown
  # Copilot Custom Instructions
  - Use camelCase for JavaScript variables.
  - Prefer async/await over .then() for promises.
  ```
  سيقوم Copilot بتخصيص الاقتراحات وفقًا لهذه التفضيلات.

### 3.5 سياق مساحة العمل مع @workspace
استخدم الأمر `@workspace` للاستعلام عن قاعدة الكود بالكامل.
- **مثال**:
  في عرض الدردشة، اكتب:
  ```
  @workspace Where is the database connection string configured?
  ```
  سيبحث Copilot في مساحة العمل الخاصة بك ويشير إلى الملفات ذات الصلة.

### 3.6 اقتراحات التعديل التالية (معاينة)
يتوقع Copilot ويقترح التعديل المنطقي التالي بناءً على تغييراتك الحديثة.
- **كيف يعمل**:
  - أثناء تعديل الكود، يسلط Copilot الضوء على التعديلات المحتملة التالية.
  - اقبل الاقتراحات باستخدام **Tab** أو عدلها عبر الدردشة المضمنة.
- **حالة الاستخدام**: مثالي لإعادة الهيكلة التكرارية أو إكمال تغييرات الكود ذات الصلة.

## 4. نصائح وحيل لتحسين استخدام Copilot

### 4.1 اكتب مطالبات فعالة
- كن محددًا: بدلاً من `write a function`، جرب `write a Python function to sort a list of dictionaries by the 'age' key`.
- وفر السياق: أدرج تفاصيل إطار العمل أو المكتبة (مثال: `use React hooks`).
- استخدم التعليقات: اكتب `// Generate a REST API endpoint in Express` لتوجيه الإكمالات.

### 4.2 استفد من السياق
- **الرجوع إلى الملفات/الرموز**: استخدم `#filename`، `#folder`، أو `#symbol` في مطالبات الدردشة لتحديد نطاق السياق.
  ```
  /explain #src/utils.js
  ```
- **السحب والإفلات**: اسحب الملفات أو علامات تبويب المحرر إلى مطالبة الدردشة للحصول على السياق.
- **إرفاق الصور**: في VS Code 17.14 Preview 1 أو أحدث، أرفق لقطات الشاشة لتوضيح المشكلات (مثال: أخطاء واجهة المستخدم).

### 4.3 استخدم أوامر الشرطة المائلة
- `/doc`: أنشئ توثيقًا لدالة.
- `/fix`: اقترح إصلاحات للأخطاء.
- `/tests`: أنشئ اختبارات وحدة للكود المحدد.
- مثال:
  ```
  /tests Generate Jest tests for this function
  ```

### 4.4 احفظ المطالبات وأعد استخدامها
- أنشئ ملف `.prompt.md` في `.github/prompts/` لتخزين المطالبات القابلة لإعادة الاستخدام.
- مثال:
  ```markdown
  # React Component Prompt
  Generate a React functional component with Tailwind CSS styling. Ask for component name and props if not provided.
  ```
- أرفق المطالبة في الدردشة لإعادة استخدامها عبر المشاريع.

### 4.5 اختر النموذج المناسب
- يدعم Copilot نماذج لغوية متعددة (مثال: GPT-4o، Claude Sonnet).
- اختر النماذج في القائمة المنسدلة لعرض الدردشة للبرمجة الأسرع أو التفكير الأعمق.
- للمهام المعقدة، قد يؤدي Claude Sonnet أداءً أفضل في وضع الوكيل.[](https://code.visualstudio.com/blogs/2025/02/24/introducing-copilot-agent-mode)

### 4.6 فهرسة مساحة العمل
- مكّن فهرسة مساحة العمل لعمليات بحث كود أسرع وأكثر دقة.
- استخدم فهرسًا بعيدًا لمستودعات GitHub أو فهرسًا محليًا لقواعد الكود الكبيرة.

## 5. أفضل الممارسات
- **راجع الاقتراحات**: تحقق دائمًا من اقتراحات Copilot للتأكد من دقتها ومواءمتها لمعايير مشروعك.
- **اجمع مع IntelliCode**: في Visual Studio، يكمل Copilot IntelliCode لتحسين الإكمالات.[](https://devblogs.microsoft.com/visualstudio/github-copilot-in-visual-studio-2022/)
- **تحقق من الأمان**: قد يقترح Copilot كودًا به ثغرات أمنية. راجع الاقتراحات، خاصة في المشاريع الحساسة، وتحقق من سياسات مؤسستك.[](https://medium.com/%40codebob75/how-to-use-copilot-in-visual-studio-a-step-by-step-guide-b2a5db3b54ba)
- **استخدم أسماء ذات معنى**: تحسن أسماء المتغيرات والدوال الوصفية جودة الاقتراحات.
- **كرر مع الدردشة**: صقل المطالبات إذا كانت الاقتراحات الأولية غير مستهدفة.
- **راقب حدود الاستخدام**: مع Copilot المجاني، تتبع إكمالاتك الشهرية وتفاعلات الدردشة عبر إعدادات حساب GitHub أو شارة Copilot في VS Code.[](https://learn.microsoft.com/en-us/visualstudio/ide/copilot-free-plan?view=vs-2022)

## 6. استكشاف المشكلات الشائعة وإصلاحها
- **Copilot غير نشط**: تأكد من أنك مسجل الدخول بحساب GitHub لديه وصول إلى Copilot. حدّث بيانات الاعتماد عبر القائمة المنسدلة لشريط حالة Copilot.
- **لا توجد اقتراحات**: تحقق من اتصالك بالإنترنت أو انتقل إلى لغة مدعومة. اضبط الإعدادات تحت **Tools** > **Options** > **GitHub Copilot**.
- **وظائف محدودة**: إذا وصلت إلى حد الاستخدام المجاني لـ Copilot، ستعود إلى اقتراحات IntelliCode. ترقّ إلى خطة مدفوعة أو تحقق من حالتك.[](https://learn.microsoft.com/en-us/visualstudio/ide/copilot-free-plan?view=vs-2022)
- **مشكلات الشبكة**: راجع [دليل استكشاف أخطاء GitHub Copilot وإصلاحها](https://docs.github.com/en/copilot/troubleshooting).

## 7. حالات الاستخدام المتقدمة
- **استعلامات قاعدة البيانات**: اطلب من Copilot إنشاء استعلامات SQL (مثال: `Write a SQL query to join two tables`).
- **تطوير API**: اطلب كود نقطة نهاية API (مثال: `Create a Flask route to handle POST requests`).
- **اختبار الوحدة**: استخدم `/tests` لإنشاء اختبارات لأطر عمل مثل Jest أو Pytest.
- **إعادة الهيكلة**: استخدم وضع الوكيل لإعادة هيكلة الكود عبر الملفات (مثال: `Migrate this jQuery code to React`).
