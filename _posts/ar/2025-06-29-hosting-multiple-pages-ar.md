نعم، إنشاء منظمة على GitHub هو طريقة فعّالة لاستضافة مواقع إضافية لـ GitHub Pages مع نطاقات فرعية فريدة من نوع `*.github.io`، حيث يمكن لكل منظمة أن يكون لها موقعها الخاص `organization.github.io`، منفصل عن موقعك الشخصي `username.github.io`. إليك كيفية القيام بذلك وما تحتاج إلى معرفته:

### النقاط الرئيسية
- **قيود الحساب الشخصي**: يمكن لكل حساب مستخدم على GitHub أن يحتوي على موقع مستخدم واحد فقط، يستضاف على `username.github.io`، ومرتبط بمستودع يحمل اسم `username.github.io`. إذا كنت قد استخدمت هذا بالفعل لحسابك الشخصي، فلا يمكنك إنشاء نطاق فرعي آخر `*.github.io` تحت نفس الحساب.
- **مواقع المنظمات**: يمكن لكل منظمة على GitHub أن يكون لها موقعها الخاص للمنظمة، يستضاف على `organization.github.io`، عن طريق إنشاء مستودع يحمل اسم `organization.github.io`. هذا يسمح لك بإنشاء نطاقات فرعية إضافية `*.github.io` من خلال إعداد منظمات متعددة.
- **مواقع المشاريع**: يمكن لكل من حسابات المستخدمين والمنظمات استضافة مواقع متعددة للمشاريع (مثل `username.github.io/project` أو `organization.github.io/project`) من مستودعات أخرى، ولكن هذه تكون مسارات فرعية وليست نطاقات فرعية. إذا كنت تريد تحديدًا نطاقات فرعية مميزة (مثل `sub.example.github.io`)، فلا يمكنك تحقيق هذا مباشرةً مع GitHub Pages بدون نطاق مخصص، لأن GitHub لا تدعم النطاقات الفرعية المخصصة مثل `sub.example.github.io` تحت النطاق `github.io`.

### خطوات إنشاء منظمة على GitHub للحصول على نطاقات فرعية إضافية `*.github.io`
1. **إنشاء منظمة على GitHub**:
   - اذهب إلى GitHub وقم بتسجيل الدخول باستخدام حسابك.
   - انقر على أيقونة "+" في الزاوية العلوية اليمنى واختر **New organization**.
   - اتبع التعليمات لإعداد المنظمة، واختر اسمًا فريدًا (مثل `myorg`). هذا الاسم سيحدد النطاق الفرعي (مثل `myorg.github.io`).
   - ملاحظة: يمكن إنشاء المنظمات مجانًا، ولكن بعض الميزات (مثل المستودعات الخاصة) قد تتطلب خطة مدفوعة، اعتمادًا على احتياجاتك. GitHub Pages للمستودعات العامة متاحة مع GitHub Free.

2. **إنشاء مستودع GitHub Pages الخاص بالمنظمة**:
   - في المنظمة الجديدة، أنشئ مستودعًا يحمل اسم `myorg.github.io` تمامًا (استبدل `myorg` باسم منظمتك).
   - هذا المستودع سيستضيف موقع المنظمة، الذي يمكن الوصول إليه على `https://myorg.github.io`.

3. **إعداد GitHub Pages**:
   - اذهب إلى علامة تبويب **Settings** في مستودع `myorg.github.io`.
   - مرر لأسفل إلى قسم **Pages**.
   - تحت **Source**، اختر الفرع الذي تريد نشره (مثل `main` أو `gh-pages`) واحفظ.
   - بمجرد التهيئة، سيكون الموقع نشطًا على `https://myorg.github.io` بعد بضع دقائق.

4. **إضافة المحتوى**:
   - أضف ملف `index.html` أو استخدم مولد مواقع ثابت مثل Jekyll إلى فرع النشر في المستودع.
   - قم بإجراء commit و push للتغييرات. على سبيل المثال:
     ```bash
     git clone https://github.com/myorg/myorg.github.io
     cd myorg.github.io
     echo "Hello World" > index.html
     git add --all
     git commit -m "Initial commit"
     git push origin main
     ```
   - قم بزيارة `https://myorg.github.io` للتحقق من أن الموقع نشط.

5. **كرر للحصول على نطاقات فرعية إضافية**:
   - أنشئ منظمات إضافية (مثل `myorg2`, `myorg3`) وكرر العملية للحصول على `myorg2.github.io`, `myorg3.github.io`، إلخ.
   - يمكن لكل منظمة أن تمتلك نطاقًا فرعيًا واحدًا `*.github.io`، مما يسمح لك بإنشاء عدد غير محدود من النطاقات الفرعية كما لديك منظمات.

### القيود والاعتبارات
- **النطاقات الفرعية المخصصة على `github.io`**: لا يمكنك إنشاء نطاقات فرعية مثل `sub.myorg.github.io` مباشرةً مع GitHub Pages. النطاق `github.io` تديره GitHub، ويدعم فقط `username.github.io` أو `organization.github.io`. لاستخدام نطاقات فرعية مخصصة (مثل `blog.example.com`)، يجب أن تمتلك نطاقًا مخصصًا وتقوم بتهيئة إعدادات DNS (سجلات CNAME) لتشير إلى `myorg.github.io`.
- **مستودع واحد لكل نطاق فرعي**: كل نطاق فرعي `*.github.io` مرتبط بمستودع واحد (`username.github.io` أو `organization.github.io`). لا يمكنك تقديم نطاقات فرعية متعددة من مستودع واحد بدون نطاق مخصص وخدمات استضافة أو وساطة إضافية.
- **عبء الإدارة**: تتطلب كل منظمة إدارة منفصلة (مثل الأعضاء، الأذونات، الفواتير). تأكد من أنك مرتاح لإدارة منظمات متعددة.
- **DNS والنطاقات المخصصة**: إذا كنت تريد استخدام نطاق مخصص (مثل `example.com` أو `sub.example.com`) بدلاً من `*.github.io`، يمكنك تهيئته في إعدادات **Pages** الخاصة بالمستودع وإضافة سجل CNAME مع موفر DNS الخاص بك. على سبيل المثال، وجّه `sub.example.com` إلى `myorg.github.io`. تأكد من التحقق من النطاق لتجنب مخاطر الاستيلاء.
- **المستودعات الخاصة**: GitHub Pages للمستودعات الخاصة تتطلب خطط GitHub Pro أو Team أو Enterprise. إذا كنت تستخدم الخطط المجانية، تأكد من أن مستودع `myorg.github.io` الخاص بك عام.

### بديل للنطاقات الفرعية المتعددة
إذا كان هدفك هو الحصول على نطاقات فرعية متعددة (مثل `blog.example.com`, `shop.example.com`) تحت نطاق مخصص واحد، يمكنك:
1. شراء نطاق مخصص (مثل `example.com`) من مسجل مثل Namecheap أو GoDaddy.
2. إنشاء مستودعات متعددة في منظمتك (مثل `myorg/blog`, `myorg/shop`).
3. تمكين GitHub Pages لكل مستودع، وتعيين نطاقات مخصصة مثل `blog.example.com` و `shop.example.com` في إعدادات **Pages** الخاصة بكل منها.
4. تكوين سجلات CNAME مع موفر DNS الخاص بك لتوجيه `blog.example.com` إلى `myorg.github.io` و `shop.example.com` إلى `myorg.github.io`. لاحظ أن جميع المستودعات التي لا تمتلك نطاقها المخصص الخاص ستظهر تحت النطاق المخصص للمنظمة كمسارات فرعية (مثل `example.com/blog`).
5. كن على علم أن تقديم نطاقات فرعية متعددة من مستودع واحد غير مدعوم أصلاً من قبل GitHub Pages وقد يتطلب خدمات خارجية مثل CDN أو خادم وسيط عكسي.

### مثال توضيحي
- لقد استخدمت `username.github.io` لموقعك الشخصي.
- أنشئ منظمة باسم `myorg` ومستودع `myorg.github.io` للحصول على `myorg.github.io`.
- أنشئ منظمة أخرى باسم `myorg2` ومستودع `myorg2.github.io` للحصول على `myorg2.github.io`.
- بالنسبة للنطاقات الفرعية المخصصة، امتلك `example.com`، وقم بإعداد `blog.example.com` (يشير إلى `myorg.github.io`) و `shop.example.com` (يشير إلى `myorg2.github.io`) عبر سجلات CNAME.

### ملاحظات نهائية
- إنشاء منظمات متعددة هو طريقة مباشرة للحصول على نطاقات فرعية إضافية `*.github.io` دون الحاجة إلى نطاق مخصص.
- إذا كنت تريد نطاقات فرعية حقيقية (مثل `sub.myorg.github.io`)، فستحتاج إلى نطاق مخصص وتكوين DNS، لأن `github.io` التابعة لـ GitHub لا تدعم هذا بشكل أصلي.
- تحقق دائمًا من النطاقات المخصصة لتجنب المخاطر الأمنية مثل الاستيلاء على النطاق.
- قد تستغرق تغييرات DNS حتى 24 ساعة للانتشار.
