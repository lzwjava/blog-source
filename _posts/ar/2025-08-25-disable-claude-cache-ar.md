في **Claude Code**، هذا المقتطف

```json
"cache_control": {
  "type": "ephemeral"
}
```
يعني أن الاستجابة (أو القطعة الأثرية) التي ينتجها النموذج **لا يتم تخزينها مؤقتًا لإعادة استخدامها**.

### سبب وجوده

عادةً، قد يقوم Claude Code (وغيرها من حزم Anthropic SDK) بتخزين الاستجابات مؤقتًا لتسريع الأمور إذا أرسلت نفس المطالبة مرة أخرى. ولكن عند تعيين `"cache_control": { "type": "ephemeral" }`، فإنك تخبر واجهة برمجة التطبيقات (API) بما يلي:

* **لا تحفظ هذه النتيجة في ذاكرة التخزين المؤقت**.
* **لا تعيد استخدامها** إذا جاء طلب مطابق آخر لاحقًا.
* تعامل مع الاستجابة على أنها **قصيرة العمر**—مخصصة لهذه المكالمة فقط.

### متى تستخدمه

هذا مفيد إذا:

* كان طلبك يتضمن **بيانات حساسة أو شخصية** لا تريد تخزينها مؤقتًا.
* كنت ترسل **استعلامات لمرة واحدة** التي لن تستفيد من إعادة الاستخدام.
* كنت تريد **نتيجة جديدة** في كل مرة، حتى إذا كان نص الطلب مطابقًا.

باختصار، `"ephemeral"` يجعل ناتج Claude *عابرًا*—خاصًا بهذا التفاعل، دون أي تخزين مؤقت عبر المكالمات.

هل تريد مني أيضًا أن أشرح كيف يختلف هذا عن السلوك الافتراضي لـ `cache_control` في Claude Code (مثل `type: "default"` مقابل `"none"` مقابل `"ephemeral"`)?
