### تصدير مفتاح GPG الخاص من الخادم

لتصدير مفتاحك الخاص، ستحتاج إلى الوصول إلى الخادم عبر SSH أو طرفية. مفاتيح GPG الخاصة حساسة — تعامل معها بأمان واحذف الملف المُصدَّر بعد النقل.

1. **سرد مفاتيحك السرية** (لتحديد المفتاح بواسطة البريد الإلكتروني أو معرّف المفتاح):  
   ```
   gpg --list-secret-keys
   ```  
   ابحث عن الإدخال المشابه لـ `sec   rsa4096/ABC123DEF456 2023-01-01 [SC]`. لاحظ البريد الإلكتروني أو معرّف المفتاح (مثل `user@example.com` أو `ABC123DEF456`).

2. **تصدير المفتاح الخاص** (استخدم `--armor` للنسق ASCII، الأسهل في النقل):  
   ```
   gpg --export-secret-keys --armor user@example.com > private-key.asc
   ```  
   - استبدل `user@example.com` ببريدك الإلكتروني الفعلي أو معرّف المفتاح.  
   - هذا ينشئ ملف `private-key.asc` في دليلك الحالي.  
   - إذا كان لديك مفاتيح متعددة، حدد معرّف المفتاح للدقة: `gpg --export-secret-keys --armor ABC123DEF456 > private-key.asc`.  
   - **ملاحظة أمنية**: هذا يُصدّر المفتاح *بدون* عبارة المرور الخاصة به. ستحتاج إلى إدخال عبارة المرور عند الاستيراد إذا تم المطالبة بذلك.

3. **انقل الملف بأمان** إلى حاسوبك المحلي:  
   - استخدم `scp` (من جهازك المحلي):  
     ```
     scp user@server-ip:/path/to/private-key.asc ./
     ```  
     استبدل `user@server-ip` ببيانات خادمك و `/path/to/` بالمسار الفعلي.  
   - بدائل: `rsync` للملفات الكبيرة أو SFTP لخيار واجهة المستخدم الرسومية. تجنب البريد الإلكتروني أو القنوات غير الآمنة.

4. **قم بالتنظيف على الخادم**:  
   ```
   rm private-key.asc
   ```  
   تحقق من زواله باستخدام `ls`.

### استيراد مفتاح GPG الخاص على حاسوبك المحلي

بافتراض أن لديك GPG مثبتًا (على سبيل المثال، عبر `apt install gnupg` على Linux، أو `brew install gnupg` على macOS، أو Chocolatey على Windows).

1. **استيراد المفتاح**:  
   ```
   gpg --import private-key.asc
   ```  
   - سترى ناتجًا مشابهًا لـ "gpg: key ABC123DEF456: secret key imported".  
   - إذا طُلب منك إدخال عبارة مرور أثناء الاستيراد، أدخل العبارة الأصلية.

2. **التحقق من الاستيراد**:  
   ```
   gpg --list-secret-keys
   ```  
   يجب أن يظهر مفتاحك الآن تحت "sec".

3. **اختياري: تعيين مستوى الثقة** (إذا كنت تستخدمه للتوقيع/البريد الإلكتروني):  
   ```
   gpg --edit-key user@example.com
   ```  
   - في موجه الأوامر، اكتب `trust`، اختر "ultimate" (الخيار 5)، ثم `save` ثم `quit`.  
   - هذا يعلّم المفتاح كموثوق تمامًا على جهازك المحلي.

### نصائح وتحذيرات
- **عبارة المرور**: استخدم دائمًا عبارة مرور قوية. إذا نسيتها، يصبح المفتاح غير قابل للاستخدام.
- **المفاتيح الفرعية**: إذا كان لمفتاحك مفاتيح فرعية (مثل تلك المستخدمة للتشفير)، قم بتصديرها أيضًا باستخدام `--export-secret-subkeys`.
- **النسخ الاحتياطي**: خزّن الملفات المُصدَّرة في خزنة آمنة (مثل محرك أقراص مشفّر).
- **الإصدارات**: تم الاختبار على GPG 2.x؛ ويعمل بشكل مشابه على الإصدار 1.x.
- لمزيد من التفاصيل، راجع وثائق GPG الرسمية: [GnuPG Manual](https://www.gnupg.org/documentation/manuals/gnupg/).
