لتصور فروع متعددة في مستودع Git على نظام Windows، يمكنك استخدام TortoiseGit أو أدوات أخرى لعرض رسم بياني للفروع المحددة. يعد الرسم البياني للمراجعات في TortoiseGit قويًا ولكن يمكن أن يكون مربكًا عند عرض جميع الفروع. فيما يلي خطوات لعرض رسم بياني لعدة فروع محددة باستخدام TortoiseGit، بالإضافة إلى أدوات بديلة لمرونة أكبر.

### استخدام TortoiseGit لعرض فروع متعددة
يمكن للرسم البياني للمراجعات في TortoiseGit عرض فروع متعددة، لكنه لا يسمح باختيار فروع محددة مباشرة في الواجهة. ومع ذلك، يمكنك تصفية العرض للتركيز على الفروع ذات الصلة.

1. **فتح الرسم البياني للمراجعات**:
   - انتقل إلى مجلد مستودعك في مستكشف Windows.
   - انقر بزر الماوس الأيمن على المجلد، واختر **TortoiseGit** > **Revision Graph**.
   - يعرض هذا رسمًا بيانيًا لجميع المراجع (الفروع، الوسوم، إلخ.) افتراضيًا، مما قد يسبب الفوضى إذا كان لديك العديد من الفروع.

2. **تصفية فروع محددة**:
   - في نافذة الرسم البياني للمراجعات، استخدم **خيارات التصفية** لتقليل الفوضى:
     - انتقل إلى قائمة **View** واختر **Show branchings and mergings** لتسليط الضوء على علاقات الفروع.
     - للتركيز على فروع محددة، انقر بزر الماوس الأيمن على commit واختر **Show Log** لعرض مربع حوار السجل، حيث يمكنك تبديل **View > Labels > Local branches** أو **Remote branches** لعرض المراجع ذات الصلة فقط.
   - بدلاً من ذلك، استخدم خيار **Walk Behavior > Compressed Graph** في مربع حوار السجل لتبسيط الرسم البياني، مع عرض نقاط الدمج فقط والـ commits ذات المراجع (مثل أطراف الفروع).

3. **التنقل في الرسم البياني**:
   - استخدم **نافذة النظرة العامة** للتنقل في الرسوم البيانية الكبيرة عن طريق سحب المنطقة المميزة.
   - مرر مؤشر الماوس فوق عقدة مراجعة لرؤية التفاصيل مثل التاريخ، المؤلف، والتعليقات.
   - اضغط على Ctrl وانقر على مراجعتين لمقارنتهما عبر القائمة السياقية (مثل **Compare Revisions**).

4. **القيود**:
   - يظهر الرسم البياني للمراجعات في TortoiseGit جميع الفروع ما لم يتم تصفيتها، ولا توجد خيارات مباشرة لاختيار فروع محددة فقط في عرض الرسم البياني.
   - للحصول على عرض أكثر نظافة، فكر في استخدام الأدوات البديلة المذكورة أدناه.

### أدوات بديلة لعرض فروع متعددة
إذا كانت واجهة TortoiseGit محدودة جدًا لاختيار فروع محددة، جرب هذه الأدوات، التي تقدم تحكمًا أكبر في تصور الفروع:

#### 1. **Visual Studio Code مع إضافة Git Graph**
   - **التثبيت**: حمّل Visual Studio Code وقم بتثبيت إضافة **Git Graph**.
   - **طريقة الاستخدام**:
     - افتح مستودعك في VS Code.
     - الوصول إلى عرض Git Graph من علامة تبويب Source Control أو لوحة الأوامر (`Ctrl+Shift+P`، اكتب "Git Graph").
     - حدد فروعًا محددة لعرضها في الرسم البياني بالنقر على أسماء الفروع في الواجهة.
     - يعرض الرسم البياني الـ commits والفروع والدمجات مع خطوط ملونة للوضوح.
   - **المزايا**: خفيف الوزن، مجاني، ويسمح باختيار فروع متعددة بشكل تفاعلي. يدعم مقارنة الـ commits وعمليات Git الأساسية.

#### 2. **SourceTree**
   - **التثبيت**: حمّل SourceTree (مجاني) لنظام Windows.
   - **طريقة الاستخدام**:
     - افتح مستودعك في SourceTree.
     - يعرض عرض **History** تمثيلاً بيانيًا للفروع والـ commits.
     - استخدم قائمة الفروع على اليسار لتبديل إظهار فروع محددة، والتركيز فقط على تلك التي تريد رؤيتها.
     - انقر بزر الماوس الأيمن على الفروع أو الـ commits لإجراءات مثل الدمج أو المقارنة.
   - **المزايا**: تصور واضح للفروع مع تلوين متسق وميزات تفاعلية مثل السحب والإفلات للدمج.

#### 3. **GitKraken**
   - **التثبيت**: حمّل GitKraken (مجاني للمشاريع مفتوحة المصدر، مدفوع للمستودعات الخاصة).
   - **طريقة الاستخدام**:
     - افتح مستودعك في GitKraken.
     - يعرض الرسم البياني المركزي جميع الفروع، مع خيارات لإخفاء/إظهار فروع محددة عبر قائمة الفروع.
     - انقر على تسميات الفروع للتركيز على فروع محددة أو استخدم البحث لتصفية الـ commits.
   - **المزايا**: بديهي وجذاب بصريًا، مع تلوين متسق للفروع وميزات متقدمة مثل حل التعارضات.

#### 4. **سطر الأوامر مع `git log`**
   - إذا كنت تفضل حلًا يعتمد على الطرفية، استخدم عرض الرسم البياني المدمج في Git:
     ```bash
     git log --graph --oneline --decorate --branches=<branch1> --branches=<branch2>
     ```
     استبدل `<branch1>` و `<branch2>` بأسماء الفروع التي تريد تصورها (مثل `feature1`، `feature2`). استخدم `--branches=*` لجميع الفروع أو حدد فروعًا متعددة صراحة.
   - **اسم مستعار للراحة**:
     أنشئ اسمًا مستعارًا في Git للوصول السهل:
     ```bash
     git config --global alias.tree "log --graph --oneline --decorate --branches="
     ```
     ثم شغّل `git tree feature1 feature2` لرؤية رسم بياني لهذه الفروع فقط.
   - **المزايا**: لا حاجة لبرامج إضافية، ويمكنك تحديد فروع محددة بدقة. الإخراج نصي ولكنه واضح للمستودعات الصغيرة.

### التوصيات
- **للبساطة والتكامل**: استخدم **VS Code مع Git Graph** للحصول على حل خفيف الوزن ومجاني يسمح لك باختيار فروع محددة.
- **لتصور متقدم**: استخدم **GitKraken** أو **SourceTree** للحصول على واجهة رسومية أكثر تطورًا مع اختيار تفاعلي للفروع وتلوين متسق.
- **لمستخدمي TortoiseGit**: التزم باستخدام الرسم البياني للمراجعات واستخدم عوامل التصفية مثل **Compressed Graph** أو **Show branchings and mergings** للتركيز على الفروع الرئيسية، ولكن كن على علم بقيوده في اختيار فروع محددة.
- **لمستخدمي الطرفية**: استخدم أمر `git log` مع عوامل تصفية فرعية محددة لتصور سريع ودقيق دون الحاجة إلى أدوات إضافية.

إذا كان لديك عدد كبير من الفروع، فمن المرجح أن تكون GitKraken أو SourceTree هما الخياران الأفضل للتصور الانتقائي للفروع التي تهتم بها فقط، حيث إنهما يقدمان تبديلًا بديهيًا للفروع. دعني أعلم إذا كنت بحاجة إلى مساعدة في إعداد أي من هذه الأدوات أو إلى توضيح إضافي!
