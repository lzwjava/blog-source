Postman هو منصة شائعة الاستخدام لتطوير و اختبار API التي تسهل عملية بناء، اختبار، وتوثيق APIs. يوفر واجهة مستخدم ودية و مجموعة قوية من الأدوات التي تدعم بروتوكولات مختلفة، بما في ذلك **HTTP، REST، GraphQL، WebSockets، وغيرها**. سواء كنت تعمل بمفردك أو مع فريق، يوفر Postman ميزات مثل التعاون في الوقت الفعلي، الاختبارات الآلية، وإدارة البيئة لتسهيل سير عمل API. ستقوم هذه الدليل بتوجيهك عبر الميزات الرئيسية لـ Postman وتقديم تعليمات خطوة بخطوة حول كيفية استخدامها بشكل فعال.

### الميزات الرئيسية لـ Postman

يوفر Postman مجموعة متنوعة من الميزات مصممة لجعل تطوير API أسهل وأسرع:

- **بناء الطلبات**: إنشاء وإرسال طلبات HTTP بسهولة.
- **إدارة المجموعات**: تنظيم الطلبات في مجموعات لتحسين الإدارة.
- **متغيرات البيئة**: إدارة الإعدادات للبيئات المختلفة (مثل: تطوير، مراجعة، إنتاج).
- **التحقق من الهوية**: التعامل مع طرق التحقق من الهوية بسهولة.
- **اختبار**: كتابة وإجراء اختبارات لتأكيد إجابات API.
- **التحاكي**: تقليد إجابات API لأغراض الاختبار.
- **التوثيق**: إنشاء وتوزيع وثائق API تلقائيًا.
- **التعاون**: مشاركة المجموعات والمتغيرات البيئية مع أعضاء الفريق.

سنستعرض كل هذه الميزات بالتفصيل.

---

### 1. **بناء الطلبات**
بناء الطلبات هو الوظيفة الأساسية لـ Postman، مما يسمح لك بإنشاء وإرسال طلبات HTTP بسهولة.

- **كيفية الاستخدام**:
  - افتح Postman وانقر على **جديد** > **طلب**.
  - اختر طريقة HTTP (مثل: `GET`، `POST`، `PUT`، `DELETE`) من قائمة التنزيل.
  - أدخل عنوان URL لنقطة النهاية API في شريط العنوان (مثل: `https://api.example.com/users`).
  - أضف **العنوانات** (مثل: `Content-Type: application/json`) في علامة التبويب **العنوانات**.
  - للأطرق مثل `POST` أو `PUT`، أضف الجسم الطلب في علامة التبويب **الجسم** (اختر الصيغة مثل: `JSON`، `form-data`، إلخ).
  - انقر على **إرسال** لإجراء الطلب و عرض الإجابة في اللوحة السفلى.

- **نصائح**: استخدم علامة التبويب **المعلمات** لإضافة معلمات الاستعلام (مثل: `?id=123`) إلى URL لطلبات `GET`.

---

### 2. **إدارة المجموعات**
تساعد المجموعاتك على تنظيم الطلبات ذات الصلة، مما يجعل من السهل إدارة وإجراء عدة طلبات معًا.

- **كيفية الاستخدام**:
  - انقر على **جديد** > **مجموعة** لإنشاء مجموعة جديدة.
  - أعط المجموعة اسمًا (مثل: "API المستخدم") و وصفًا اختياريًا.
  - أضف طلبات إلى المجموعة عن طريق سحبها من الشريط الجانبي أو النقر على **إضافة طلب** داخل المجموعة.
  - لإجراء المجموعة بأكملها، انقر على **...** بجانب اسم المجموعة و اختر **تعديل المجموعة**. هذا يفتح **معدل المجموعة**، حيث يمكنك تنفيذ جميع الطلبات تسلسليًا أو بشكل متزامن.

- **نصائح**: استخدم المجلدات داخل المجموعات لتنظيم الطلبات بشكل أكبر حسب الوظيفة (مثل: "التحقق من الهوية"، "إدارة المستخدم").

---

### 3. **متغيرات البيئة**
تسمح متغيرات البيئة لك بإدارة الإعدادات المختلفة (مثل: URLs الأساسية، مفاتيح API) للبيئات المختلفة دون تغيير كل طلب يدويًا.

- **كيفية الاستخدام**:
  - انقر على أيقونة **العين** في الزاوية العليا اليمنى لفتح **مدير البيئة**.
  - انقر على **إضافة** لإنشاء بيئة جديدة (مثل: "تطوير"، "إنتاج").
  - حدد أزواج المفتاح والقيمة (مثل: `base_url: https://api.example.com`) لكل بيئة.
  - في طلباتك، استخدم المتغيرات عن طريق وضعها بين قوسين مزدوجين مثل `{{base_url}}/users`.
  - قم بالتبديل بين البيئات عن طريق اختيار المطلوب من القائمة المنسدلة في الزاوية العليا اليمنى.

- **نصائح**: استخدم **متغيرات عالمية** للقيم التي تبقى ثابتة عبر البيئات، مثل مفاتيح API.

---

### 4. **التحقق من الهوية**
يسهل Postman التعامل مع طرق التحقق من الهوية المختلفة، مما يضمن الوصول الآمن إلى APIs.

- **كيفية الاستخدام**:
  - في علامة التبويب الطلب، انتقل إلى علامة التبويب **التحقق من الهوية**.
  - اختر نوع التحقق من الهوية من القائمة المنسدلة (مثل: **Basic Auth**، **Bearer Token**، **OAuth 2.0**، **API Key**).
  - أدخل بيانات الاعتماد أو الرموز المطلوبة (مثل: اسم المستخدم وكلمة المرور للتحقق من الهوية الأساسية، أو رمز للتحقق من الهوية).
  - سيضيف Postman تفاصيل التحقق من الهوية تلقائيًا إلى عنوانات الطلب.

- **مثال**:
  - للتحقق من **Bearer Token**، ألصق رمزك، وسيشمل Postmanه في عنوان `Authorization` ك `Bearer <token>`.

---

### 5. **اختبار**
يسمح إطار اختبار Postman لك بكتابة اختبارات JavaScript لتأكيد إجابات API، مما يضمن أن APIs تعمل كما هو متوقع.

- **كيفية الاستخدام**:
  - في علامة التبويب الطلب، انتقل إلى علامة التبويب **اختبارات**.
  - أكتب كود JavaScript لتأكيد الإجابة. على سبيل المثال:
    ```javascript
    pm.test("رمز الحالة هو 200", function () {
        pm.response.to.have.status(200);
    });
    ```
  - بعد إرسال الطلب، تحقق من **نتائج الاختبار** في لوحة الإجابة لمعرفة ما إذا كان الاختبارات قد نجحت أو فشلت.

- **نصائح**: استخدم قطع Postman المضمنة (مثل: "رمز الحالة هو 200"، "جسم الإجابة: فحص قيمة JSON") لإضافة اختبارات شائعة بسرعة.

### 6. **التحاكي**
يسمح التحاكي لك بتقليد إجابات API، وهو مفيد عندما لا يزال API الحقيقي قيد التطوير أو غير متاح.

- **كيفية الاستخدام**:
  - أنشئ مجموعة جديدة أو استخدم مجموعة موجودة.
  - انقر على **...** بجانب المجموعة و اختر **تحاكي المجموعة**.
  - حدد إجابات التحاكي لكل طلب في المجموعة (مثل: بيانات JSON العينة).
  - سيولد Postman URL خادم التحاكي (مثل: `https://<mock-id>.mock.pstmn.io`) يمكنك استخدامه لإرسال طلبات واستقبال إجابات محاكاة.

- **نصائح**: استخدم التحاكي لتسهيل عمل مطوري واجهات المستخدم بشكل مستقل دون انتظار الخلفية.

---

### 7. **التوثيق**
يمكن لـ Postman إنشاء وثائق API تلقائيًا بناءً على الطلبات في مجموعاتك.

- **كيفية الاستخدام**:
  - افتح مجموعة وانقر على أيقونة **...**.
  - اختر **عرض الوثائق** لإنشاء صفحة وثائق.
  - قم بتخصيص الوثائق عن طريق إضافة وصفات، أمثلة، و علامات لكل طلب.
  - شارك الوثائق عن طريق نشرها علنًا أو مشاركة الرابط مع فريقك.

- **نصائح**: احتفظ بالوثائق على يومية عن طريق تزامنها مع تغييرات المجموعة.

---

### 8. **التعاون**
يسمح ميزات التعاون لـ Postman للفريق بالعمل معًا بشكل فعال على مشاريع API.

- **كيفية الاستخدام**:
  - أنشئ **مساحة عمل فريق** عن طريق النقر على **مساحات العمل** > **إنشاء مساحة عمل**.
  - دع أعضاء الفريق إلى المساحة عبر البريد الإلكتروني أو الرابط.
  - شارك المجموعات والمتغيرات البيئية والموارد الأخرى في المساحة.
  - استخدم **التحكم في الإصدار** لتفرع المجموعات، إجراء التغييرات، ودمج التحديثات عبر طلبات سحب.

- **نصائح**: استخدم **التعليقات** على الطلبات أو المجموعات للنقاش حول التغييرات وتقديم ملاحظات مباشرة في Postman.

### نصائح إضافية لاستخدام Postman بشكل فعال

- **استخدم scripts**: استغل **Pre-request Scripts** لتكوين البيانات أو الشروط (مثل: إنشاء Timestamp) قبل إرسال الطلب.
- **شاهد APIs**: قم بإنشاء **Monitors** لإجراء المجموعات في فترات زمنية مخطط لها وتحقق من صحة API أو وقت التشغيل.
- **دمج مع CI/CD**: استخدم **Newman**، أداة سطر الأوامر لـ Postman، لإجراء المجموعات في أنابيب CI/CD (مثل: Jenkins، GitHub Actions).
- **استكشف Postman Flows**: استخدم **Flows** لبناء وتأتمتة سير عمل API المعقدة بصورًا دون كتابة كود.
- **أمن البيانات الحساسة**: احفظ المعلومات الحساسة مثل مفاتيح API في **Postman Vault** للحصول على الوصول الآمن.
