## 1. أساسيات الملف المصدر
- **تسمية الملفات**:
  - تُسمى الملفات المصدر بالاسم الحساس لحالة الأحرف للفئة الأعلى مستوى التي تحتويها
  - يجب أن تستخدم الملفات ترميز UTF-8
  - لا يُنصح باستخدام الأحرف الخاصة في أسماء الملفات

## 2. هيكل الملف المصدر
### تنظيم الملف المصدر
1. جملة package
2. جمل import
3. فئة واحدة من المستوى الأعلى بالضبط

### قواعد جمل الاستيراد
- لا يُسمح بالاستيراد باستخدام النمط العام (wildcard)
- يُسمح بالاستيراد الثابت (static imports)
- يتم تنظيم جمل الاستيراد بترتيب محدد:
  - جميع عمليات الاستيراد الثابت
  - جميع عمليات الاستيراد غير الثابت
  - يتم ترتيب جمل الاستيراد داخل كل مجموعة أبجديًا

## 3. إرشادات التنسيق

### المسافات البادئة والأقواس
- استخدم مسافتين للمسافة البادئة (ليس علامات Tab)
- تُستخدم الأقواس مع جمل `if`، `else`، `for`، `do`، و `while`، حتى للكتل ذات السطر الواحد
- يُوصى بأسلوب K&R لوضع الأقواس
  ```java
  public void method() {
    if (condition) {
      // كود البرنامج
    }
  }
  ```

### طول السطر والتفاف النص
- الحد الأقصى لطول السطر هو 100 حرف
- يُفضل فصل الأسطر عند فواصل بناء الجملة ذات المستوى الأعلى
- عند كسر سلاسل استدعاءات الدوال، يحدث الكسر قبل النقطة `.`

## 4. اصطلاحات التسمية

### القواعد العامة
- **الحزم (Packages)**: أحرف صغيرة، بدون شرطات سفلية
- **الفئات (Classes)**: UpperCamelCase
- **الدوال (Methods)**: lowerCamelCase
- **الثوابت (Constants)**: UPPER_SNAKE_CASE
- **الحقول غير الثابتة (Non-constant Fields)**: lowerCamelCase
- **المعاملات (Parameters)**: lowerCamelCase
- **المتغيرات المحلية (Local Variables)**: lowerCamelCase

### ممارسات تسمية محددة
- تجنب الاختصارات
- يجب أن تنتهي أسماء الاستثناءات بـ `Exception`
- تُسمى فئات الاختبار `TestClassName`

## 5. ممارسات البرمجة

### قواعد لغة Java
- **الاستثناءات**:
  - امسك الاستثناءات المحددة
  - تجنب كتل الالتقاط (catch) الفارغة
  - قم دائمًا بتضمين رسالة خطأ مفصلة
- **الكلمة الأساسية Final**:
  - استخدم `final` لمعاملات الدوال
  - يُفضل استخدام الكائنات غير القابلة للتغيير (immutable)
- **الشرح (Annotations)**:
  - `@Override` إلزامي للدوال التي تُعيد تعريف الدوال الأصلية (overriding)
  - استخدم الشروح القياسية بشكل مناسب

### هيكل الكود
- يُفضل التركيب (composition) على الوراثة (inheritance)
- حافظ على كون الدوال قصيرة ومركزة
- جملة واحدة في كل سطر
- تجنب التداخل العميق للشروط

## 6. التعليقات والتوثيق

### إرشادات Javadoc
- يجب أن تحتوي جميع الفوال والدوال العامة على Javadoc
- تبدأ تعليقات Javadoc بـ `/**`
- قم بتضمين الوصف، المعاملات، قيم الإرجاع، والاستثناءات
- استخدم الوسوم `@param`، `@return`، `@throws`

### التعليقات داخل السطر
- استخدمها باعتدال
- اشرح "لماذا" وليس "ماذا"
- حافظ على كون التعليقات موجزة وذات معنى

## 7. اعتبارات نظام التحكم بالإصدارات
- تجنب إيداع الكود المُعلق عليه (commented-out)
- استخدم ميزات نظام التحكم بالإصدارات بدلاً من تعليقات الكود لتتبع التغييرات

## 8. اعتبارات الأداء
- يُفضل استخدام حلقات `for` المحسنة (enhanced)
- استخدم `StringBuilder` لدمج السلاسل النصية
- أغلق الموارد باستخدام try-with-resources
- تجنب إنشاء كائنات غير ضرورية

## 9. مبادئ الاختبار
- اكتب اختبارات الوحدة (unit tests)
- استهدف تحقيق تغطية عالية للكود (high code coverage)
- استخدم JUnit للاختبار
- استخدم المحاكاة (Mock) للتبعيات الخارجية
- اختبر الحالات الحدية (edge cases) وحالات الخطأ

## 10. أدوات إضافية
- توفر Google أداة تنسيق Java وأدوات التحليل الثابت
- يمكن استخدام CheckStyle و Error Prone لفرض إرشادات النمط
