في Visual Studio Code، السلوك الافتراضي هو **وضع الإدراج** (الكتابة تقوم بإدخال النص، دفع النص الحالي للأمام).  
يتم تفعيل **وضع الاستبدال** (الكتابة تستبدل النص الحالي) عند الضغط على مفتاح **Insert** في لوحة المفاتيح — وهذا يقوم بالتبديل بين وضع الإدخال ووضع الاستبدال/الكتابة فوق.

### كيفية العودة من وضع الاستبدال (الكتابة فوق) إلى وضع الإدخال

1. ببساطة اضغط على مفتاح **Insert** مرة واحدة في لوحة المفاتيح.  
   → سيتغير شريط الحالة في الجزء السفلي الأيمن من **OVR** (كتابة فوق) إلى **INS** (إدراج).

### إذا كنت لا تريد أبدًا الدخول عن طريق الخطأ إلى وضع الكتابة فوق مرة أخرى

يمكنك تعطيل تبديل مفتاح Insert تمامًا:

#### الطريقة 1: اختصار لوحة المفاتيح (موصى به)
اضغط على `Ctrl + Shift + P` → اكتب **"Toggle Insert/Overwrite"** → سترى الأمر  
`Preferences: Toggle Insert on Paste and Overtype` أو ما يشابه.  
بشكل أكثر موثوقية، استخدم هذا الإعداد المباشر:

#### الطريقة 2: أضف هذا إلى ملف `settings.json` الخاص بك
```json
// تعطيل مفتاح Insert من تبديل وضع الكتابة فوق
"editor.occurencesHighlight": true,
"editor.overtypeMode": false,

// هذا هو الإعداد الرئيسي:
"keyboard.dispatch": "keyCode"
```

في الواقع، الحل الأفضل والأكثر شيوعًا:

```json
// تعطيل مفتاح Insert تمامًا في VS Code
{
    "key": "insert",
    "command": "-editor.action.toggleOverwrite",
    "when": "editorTextFocus"
}
```

أو ببساطة قم بإيقاف تشغيل وضع الكتابة فوق بشكل عام:

```json
"editor.occurrencesHighlight": true,
"editor.overwriteMode": false   // ليس إعدادًا حقيقيًا، تجاهله

// الإعداد الحقيقي لتعطيل تبديل الكتابة فوق:
"editor.insertSpaces": true,  // غير مرتبط

// الطريقة الصحيحة:
{
    "key": "insert",
    "command": "noop"   // أو قم بإزالة الربط الافتراضي
}
```

#### الإصلاح الدائم الأبسط (يستخدمه معظم الناس):
1. افتح الإعدادات (`Ctrl + ,`)
2. ابحث عن **"overwrite"**
3. قم بإلغاء تحديد **Editor: Overtype** (إذا كان موجودًا) أو الأفضل:
4. أضف هذا السطر إلى ملف `settings.json` الخاص بك (Ctrl + Shift + P → "Preferences: Open Settings (JSON)"):

```json
// تعطيل تبديل مفتاح Insert لوضع الكتابة فوق تمامًا
{
    "key": "insert",
    "command": "-editor.action.toggleOverwrite"
}
```

بعد إضافة ذلك، لن يقوم مفتاح Insert بأي شيء في VS Code، وسوف تبقى دائمًا في وضع الإدراج.

هذا هو الحل الأكثر شيوعًا وموثوقية والذي يستخدمه المطورون الذين يضغطون على Insert عن طريق الخطأ باستمرار (خاصة على أجهزة اللابتوب ذات لوحات المفاتيح الصغيرة في تايوان/آسيا حيث يسهل الضغط على Insert).

أخبرني إذا كنت تستخدم macOS (يستخدم مفتاحًا مختلفًا) أو تريد حل النقر على شريط الحالة!
