سأشرح أدناه كل قاعدة من ملف تكوين Checkstyle الذي قدمته، مع مثال صغير على الكود الذي **ينتهك** القاعدة ووصف موجز لسبب الإشارة إليه. تم تجميع القواعد حسب الفئة كما في وصفك، وسأحافظ على الإيجاز في الشروحات مع ضمان الوضوح. يفترض كل مثال أن القاعدة نشطة بالإعدادات الافتراضية ما لم يُذكر خلاف ذلك. بالنسبة للقواعد ذات الخصائص القابلة للتكوين (مثل الطول الأقصى للسطر)، سأستخدم القيم الافتراضية المعتادة (مثل 80 حرفًا لـ LineLength) ما لم يُلاحظ خلاف ذلك.

---

### الوحدة الأساسية: `<module name="Checker">`
وحدة `Checker` هي منسق المستوى الأعلى لجميع الفحوصات والمرشحات. تنطبق على ملفات `.java` و`.properties` و`.xml` وتضبط مستوى الشدة إلى `error`.

#### الخصائص
- **severity="error"**: يتم التعامل مع جميع الانتهاكات كأخطاء.
- **fileExtensions="java, properties, xml"**: تنطبق الفحوصات على أنواع الملفات هذه.

### مرشحات الملفات
هذه ترشح الملفات التي يتم فحصها.

1.  **BeforeExecutionExclusionFileFilter**
    - **الغرض**: يستبعد الملفات التي تطابق تعبيرًا نمطيًا (مثل `module-info.java`).
    - **مثال على الانتهاك**:
      ```java
      // module-info.java
      module com.example {
          requires java.base;
      }
      ```
    - **سبب الإشارة**: هذا الملف يطابق التعبير النمطي `module\-info\.java$` ويتم استبعاده من الفحوصات. لا يحدث أي انتهاك لهذا الملف، ولكن لا تزال الملفات الأخرى تخضع للفحص.

2.  **SuppressionFilter**
    - **الغرض**: يكبح الفحوصات بناءً على قواعد في ملف (مثل `checkstyle-suppressions.xml`).
    - **مثال على الانتهاك**: إذا قام `checkstyle-suppressions.xml` بكبح `LineLength` لملف معين، فلن يتم الإشارة إلى سطر طويل في ذلك الملف. بدون الكبح:
      ```java
      public class MyClass { // هذا السطر طويل جدًا ويتجاوز الطول الأقصى الافتراضي البالغ 80 حرفًا، مما يتسبب في خطأ.
      }
      ```
    - **سبب الإشارة**: بدون قاعدة كبح، ينتهك السطر الطويل قاعدة `LineLength`.

3.  **SuppressWarningsFilter**
    - **الغرض**: يسمح بكبح الفحوصات باستخدام `@SuppressWarnings("checkstyle:<check-name>")`.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
          int my_field; // ينتهك MemberName (ليس camelCase)
      }
      ```
      ```java
      @SuppressWarnings("checkstyle:MemberName")
      public class MyClass {
          int my_field; // لا يوجد انتهاك بسبب الكبح
      }
      ```
    - **سبب الإشارة**: بدون الكبح، `my_field` تنتهك `MemberName` (تتوقع camelCase، مثل `myField`).

### فحوصات متنوعة
هذه تنطبق على الخصائص العامة للملف.

4.  **JavadocPackage**
    - **الغرض**: يضمن أن كل حزمة تحتوي على `package-info.java` مع Javadoc.
    - **مثال على الانتهاك**:
      ```java
      // com/example/package-info.java (مفقود أو بدون Javadoc)
      package com.example;
      ```
    - **سبب الإشارة**: تعليق Javadoc مفقود (مثل `/** Package description */`).

5.  **NewlineAtEndOfFile**
    - **الغرض**: يضمن أن الملفات تنتهي بسطر جديد.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {} // لا يوجد سطر جديد في النهاية
      ```
    - **سبب الإشارة**: ينتهي الملف بدون حرف سطر جديد.

6.  **Translation**
    - **الغرض**: يتحقق من أن ملفات `.properties` الخاصة بالتدويل تحتوي على مفاتيح متسقة.
    - **مثال على الانتهاك**:
      ```properties
      # messages.properties
      key1=Hello
      key2=World
      ```
      ```properties
      # messages_fr.properties
      key1=Bonjour
      # المفتاح key2 مفقود
      ```
    - **سبب الإشارة**: `messages_fr.properties` تفتقر إلى `key2`، الموجود في `messages.properties`.

### فحوصات الحجم
هذه تفرض حدودًا على أطوال الملفات والأسطر.

7.  **FileLength**
    - **الغرض**: يحدد العدد الإجمالي للأسطر في الملف (عادة 2000 سطر افتراضيًا).
    - **مثال على الانتهاك**: ملف Java يحتوي على 2001 سطر.
    - **سبب الإشارة**: يتجاوز الحد الأقصى الافتراضي للأسطر.

8.  **LineLength**
    - **الغرض**: يضمن ألا تتجاوز الأسطر الطول الأقصى (80 حرفًا افتراضيًا).
    - **مثال على الانتهاك**:
      ```java
      public class MyClass { public void myMethodWithVeryLongNameToExceedEightyCharactersInALine() {} }
      ```
    - **سبب الإشارة**: السطر يتجاوز 80 حرفًا.

#### فحوصات Javadoc
هذه تفرض تعليقات Javadoc صحيحة.

12. **InvalidJavadocPosition**
    - **الغرض**: يضمن أن تعليقات Javadoc تكون قبل الأصناف/الأساليب، وليس في مكان آخر.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
          /** هذا تعليق Javadoc في غير محله */
          int x;
      }
      ```
    - **سبب الإشارة**: تعليق Javadoc ليس قبل تصريح صنف/أسلوب.

13. **JavadocMethod**
    - **الغرض**: يفحص الأساليب للتأكد من وجود Javadoc صحيح (المعاملات، الإرجاع، الاستثناءات).
    - **مثال على الانتهاك**:
      ```java
      public int add(int a, int b) { return a + b; }
      ```
    - **سبب الإشارة**: Javadoc مفقود للأسلوب العام.

14. **JavadocType**
    - **الغرض**: يضمن أن الأصناف/الواجهات/التمددات تحتوي على Javadoc.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {}
      ```
    - **سبب الإشارة**: Javadoc مفقود للصنف.

15. **JavadocVariable**
    - **الغرض**: يتطلب Javadoc للحقول العامة/المحمية.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
          public int x;
      }
      ```
    - **سبب الإشارة**: Javadoc مفقود للحقل العام.

16. **JavadocStyle**
    - **الغرض**: يفرض نمط Javadoc (مثل HTML صالح، عدم وجود تعليقات مشوهة).
    - **مثال على الانتهاك**:
      ```java
      /** النقطة النهائية مفقودة في النهاية */
      public class MyClass {}
      ```
    - **سبب الإشارة**: تعليق Javadoc يفتقر إلى نقطة في النهاية.

17. **MissingJavadocMethod**
    - **الغرض**: يشير إلى الأساليب المفقودة فيها Javadoc.
    - **مثال على الانتهاك**:
      ```java
      public void myMethod() {}
      ```
    - **سبب الإشارة**: الأسلوب العام يفتقر إلى Javadoc.

#### تسميات التسمية
هذه تفرض أنماط تسمية متسقة.

18. **ConstantName**
    - **الغرض**: الثوابت (`static final`) يجب أن تكون `UPPER_CASE`.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
          static final int myConstant = 42;
      }
      ```
    - **سبب الإشارة**: `myConstant` يجب أن تكون `MY_CONSTANT`.

19. **LocalFinalVariableName**
    - **الغرض**: المتغيرات المحلية `final` يجب أن تكون `camelCase`.
    - **مثال على الانتهاك**:
      ```java
      public void myMethod() {
          final int MY_VAR = 1;
      }
      ```
    - **سبب الإشارة**: `MY_VAR` يجب أن تكون `myVar`.

20. **LocalVariableName**
    - **الغرض**: المتغيرات المحلية يجب أن تكون `camelCase`.
    - **مثال على الانتهاك**:
      ```java
      public void myMethod() {
          int MY_VAR = 1;
      }
      ```
    - **سبب الإشارة**: `MY_VAR` يجب أن تكون `myVar`.

21. **MemberName**
    - **الغرض**: حقول النموذج (Instance fields) يجب أن تكون `camelCase`.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
          int my_field;
      }
      ```
    - **سبب الإشارة**: `my_field` يجب أن تكون `myField`.

22. **MethodName**
    - **الغرض**: الأساليب يجب أن تكون `camelCase`.
    - **مثال على الانتهاك**:
      ```java
      public void MyMethod() {}
      ```
    - **سبب الإشارة**: `MyMethod` يجب أن تكون `myMethod`.

23. **PackageName**
    - **الغرض**: الحزم يجب أن تكون بأحرف صغيرة مع نقاط (مثل `com.example`).
    - **مثال على الانتهاك**:
      ```java
      package com.Example;
      ```
    - **سبب الإشارة**: `Example` يجب أن تكون `example`.

24. **ParameterName**
    - **الغرض**: معاملات الأسلوب يجب أن تكون `camelCase`.
    - **مثال على الانتهاك**:
      ```java
      public void myMethod(int MY_PARAM) {}
      ```
    - **سبب الإشارة**: `MY_PARAM` يجب أن تكون `myParam`.

25. **StaticVariableName**
    - **الغرض**: الحقول الساكنة (غير النهائية) يجب أن تتبع نمط تسمية محدد.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
          static int MY_FIELD;
      }
      ```
    - **سبب الإشارة**: `MY_FIELD` يجب أن تكون `myField` (بافتراض camelCase).

26. **TypeName**
    - **الغرض**: أسماء الأصناف/الواجهات/التمددات يجب أن تكون `UpperCamelCase`.
    - **مثال على الانتهاك**:
      ```java
      public class myClass {}
      ```
    - **سبب الإشارة**: `myClass` يجب أن تكون `MyClass`.

#### فحوصات الاستيراد
هذه تنظم تعليمات الاستيراد.

27. **AvoidStarImport**
    - **الغرض**: يمنع استيراد النمط العام (مثل `import java.util.*`).
    - **مثال على الانتهاك**:
      ```java
      import java.util.*;
      ```
    - **سبب الإشارة**: يستخدم `*` بدلاً من الاستيرادات المحددة (مثل `import java.util.List`).

28. **IllegalImport**
    - **الغرض**: يحظر الاستيراد من حزم مقيدة (مثل `sun.*`).
    - **مثال على الانتهاك**:
      ```java
      import sun.misc.Unsafe;
      ```
    - **سبب الإشارة**: `sun.misc.Unsafe` موجود في حزمة مقيدة.

29. **RedundantImport**
    - **الغرض**: يشير إلى استيرادات مكررة أو غير ضرورية.
    - **مثال على الانتهاك**:
      ```java
      import java.util.List;
      import java.util.List;
      ```
    - **سبب الإشارة**: استيراد مكرر لـ `List`.

30. **UnusedImports**
    - **الغرض**: يكتشف الاستيرادات غير المستخدمة.
    - **مثال على الانتهاك**:
      ```java
      import java.util.List;
      public class MyClass {}
      ```
    - **سبب الإشارة**: تم استيراد `List` ولكن لم يتم استخدامه.

#### فحوصات المسافات البيضاء
هذه تفرض استخدامًا متسقًا للمسافات البيضاء في الكود.

33. **EmptyForIteratorPad**
    - **الغرض**: يتحقق من المسافة البادئة في مكررات حلقة `for` الفارغة.
    - **مثال على الانتهاك**:
      ```java
      for (int i = 0; ; i++) {}
      ```
    - **سبب الإشارة**: قسم المكرر الفارغ يجب أن يحتوي على مسافة (مثل `for (int i = 0; ; i++)`).

34. **GenericWhitespace**
    - **الغرض**: يضمن التباعد حول الأنواع العامة (مثل `List<String>`).
    - **مثال على الانتهاك**:
      ```java
      List<String>list;
      ```
    - **سبب الإشارة**: لا توجد مسافة بين `>` و `list`.

35. **MethodParamPad**
    - **الغرض**: يتحقق من المسافة قبل قوائم معاملات الأسلوب.
    - **مثال على الانتهاك**:
      ```java
      public void myMethod (int x) {}
      ```
    - **سبب الإشارة**: المسافة قبل `(int x)` غير صحيحة.

36. **NoWhitespaceAfter**
    - **الغرض**: يمنع المسافات البيضاء بعد رموز معينة (مثل `++`).
    - **مثال على الانتهاك**:
      ```java
      int x = y ++ ;
      ```
    - **سبب الإشارة**: مسافة بعد `++`.

37. **NoWhitespaceBefore**
    - **الغرض**: يمنع المسافات البيضاء قبل رموز معينة (مثل `;`).
    - **مثال على الانتهاك**:
      ```java
      int x = 1 ;
      ```
    - **سبب الإشارة**: مسافة قبل `;`.

38. **OperatorWrap**
    - **الغرض**: يضمن أن تكون العوامل في السطر الصحيح.
    - **مثال على الانتهاك**:
      ```java
      int x = 1 +
          2;
      ```
    - **سبب الإشارة**: `+` يجب أن تكون في نهاية السطر الأول.

39. **ParenPad**
    - **الغرض**: يتحقق من المسافة داخل الأقواس.
    - **مثال على الانتهاك**:
      ```java
      if ( x == y ) {}
      ```
    - **سبب الإشارة**: المسافات داخل `(` و `)` غير صحيحة.

40. **TypecastParenPad**
    - **الغرض**: يضمن المسافة في تحويلات النوع.
    - **مثال على الانتهاك**:
      ```java
      Object o = ( String ) obj;
      ```
    - **سبب الإشارة**: المسافات داخل `( String )` غير صحيحة.

41. **WhitespaceAfter**
    - **الغرض**: يتطلب مسافات بيضاء بعد رموز معينة (مثل الفواصل).
    - **مثال على الانتهاك**:
      ```java
      int[] arr = {1,2,3};
      ```
    - **سبب الإشارة**: مسافة مفقودة بعد الفواصل.

42. **WhitespaceAround**
    - **الغرض**: يضمن المسافات البيضاء حول العوامل/الكلمات المفتاحية.
    - **مثال على الانتهاك**:
      ```java
      if(x==y) {}
      ```
    - **سبب الإشارة**: مسافات مفقودة حول `==` و `if`.

43. **ModifierOrder**

    - **مثال على الانتهاك**:

44. **RedundantModifier**

    - **مثال على الانتهاك**:
      ```java
      public final class MyClass {
          public final void myMethod() {}
      }
      ```

    - **سبب الإشارة**: `final` على أسلوب في صنف `final` هو زائد عن الحاجة.

#### فحوصات الكتل
هذه تفرض الاستخدام الصحيح للكتل البرمجية.

45. **AvoidNestedBlocks**
    - **الغرض**: يمنع الكتل المتداخلة غير الضرورية.
    - **مثال على الانتهاك**:
      ```java
      public void myMethod() {
          { int x = 1; }
      }
      ```
    - **سبب الإشارة**: كتلة متداخلة غير ضرورية.

46. **EmptyBlock**
    - **الغرض**: يشير إلى الكتل الفارغة.
    - **مثال على الانتهاك**:
      ```java
      if (x == 1) {}
      ```
    - **سبب الإشارة**: كتلة `if` فارغة.

47. **LeftCurly**
    - **الغرض**: يضمن وضع الأقواس المتعرجة الافتتاحية في المكان الصحيح.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass
      {
      }
      ```
    - **سبب الإشارة**: `{` يجب أن تكون على نفس سطر `class`.

48. **NeedBraces**
    - **الغرض**: يتطلب أقواسًا متعرجة للكتل ذات العبارة المفردة.
    - **مثال على الانتهاك**:
      ```java
      if (x == 1) y = 2;
      ```
    - **سبب الإشارة**: أقواس متعرجة مفقودة؛ يجب أن تكون `{ y = 2; }`.

49. **RightCurly**
    - **الغرض**: يضمن وضع الأقواس المتعرجة الختامية في المكان الصحيح.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
      }
      ```
    - **سبب الإشارة**: `}` يجب أن تكون على سطر جديد (اعتمادًا على النمط).

#### فحوصات مشاكل البرمجة
هذه تحدد مشاكل البرمجة الشائعة.

50. **EmptyStatement**
    - **الغرض**: يشير إلى العبارات الفارغة.
    - **مثال على الانتهاك**:
      ```java
      int x = 1;; // فاصلة منقوطة إضافية
      ```
    - **سبب الإشارة**: `;` الإضافية تنشئ عبارة فارغة.

51. **EqualsHashCode**
    - **الغرض**: يضمن أن يتم تجاوز `equals()` و `hashCode()` معًا.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
          @Override
          public boolean equals(Object o) { return true; }
      }
      ```
    - **سبب الإشارة**: تجاوز `hashCode()` مفقود.

52. **HiddenField**
    - **الغرض**: يكتشف الحقول التي يحجبها متغيرات محلية/معاملات.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
          int x;
          public void setX(int x) { this.x = x; }
      }
      ```
    - **سبب الإشارة**: المعامل `x` يحجب الحقل `x`.

53. **IllegalInstantiation**
    - **الغرض**: يمنع إنشاء كائنات من أصناف معينة.
    - **مثال على الانتهاك**:
      ```java
      String s = new String("test");
      ```
    - **سبب الإشارة**: إنشاء كائن غير ضروري لـ `String`.

54. **InnerAssignment**
    - **الغرض**: يمنع التعيينات داخل العبارات.
    - **مثال على الانتهاك**:
      ```java
      if (x = 1) {}
      ```
    - **سبب الإشارة**: التعيين `x = 1` داخل العبارة.

55. **MagicNumber**
    - **الغرض**: يشير إلى القيم الرقمية الثابتة المضمنة في الكود.
    - **مثال على الانتهاك**:
      ```java
      int x = 42;
      ```
    - **سبب الإشارة**: `42` يجب أن تكون ثابتًا مسمى (مثل `static final int MY_CONST = 42;`).

56. **MissingSwitchDefault**
    - **الغرض**: يتطلب حالة `default` في عبارات `switch`.
    - **مثال على الانتهاك**:
      ```java
      switch (x) {
          case 1: break;
      }
      ```
    - **سبب الإشارة**: حالة `default` مفقودة.

57. **MultipleVariableDeclarations**
    - **الغرض**: يمنع التصريح عن متغيرات متعددة في تصريح واحد.
    - **مثال على الانتهاك**:
      ```java
      int x, y;
      ```
    - **سبب الإشارة**: يجب أن يكون `int x; int y;`.

58. **SimplifyBooleanExpression**
    - **الغرض**: يشير إلى العبارات المنطقية المعقدة.
    - **مثال على الانتهاك**:
      ```java
      if (x == true) {}
      ```
    - **سبب الإشارة**: يجب أن تكون `if (x)`.

59. **SimplifyBooleanReturn**
    - **الغرض**: يبسط عبارات الإرجاع المنطقية.
    - **مثال على الانتهاك**:
      ```java
      if (x) return true; else return false;
      ```
    - **سبب الإشارة**: يجب أن تكون `return x;`.

#### فحوصات تصميم الأصناف
هذه تفرض تصميمًا جيدًا للأصناف.

60. **DesignForExtension**
    - **الغرض**: يضمن أن الأصناف غير النهائية تحتوي على أساليب محمية/مجردة.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
          public void myMethod() {}
      }
      ```
    - **سبب الإشارة**: الصنف غير النهائي يحتوي على أسلوب غير محمي/مجرد.

61. **FinalClass**
    - **الغرض**: يشير إلى الأصناف ذات البنائين الخاصين كمرشحة لأن تكون `final`.
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
          private MyClass() {}
      }
      ```
    - **سبب الإشارة**: يجب أن تكون `final` لأنه لا يمكن تمديدها.

62. **HideUtilityClassConstructor**
    - **الغرض**: يضمن أن أصناف الأدوات المساعدة تحتوي على بنائين خاصين.
    - **مثال على الانتهاك**:
      ```java
      public class MyUtils {
          public static void doSomething() {}
      }
      ```
    - **سبب الإشارة**: بنائي خاص مفقود لصنف الأدوات المساعدة.

63. **InterfaceIsType**
    - **الغرض**: يمنع الواجهات الواصمة (بدون أساليب).
    - **مثال على الانتهاك**:
      ```java
      public interface MyMarker {}
      ```
    - **سبب الإشارة**: الواجهة لا تحتوي على أساليب.

64. **VisibilityModifier**
    - **الغرض**: يفرض وضوح رؤية مناسب للحقول (يفضل private مع محصلات/معينات).
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
          public int x;
      }
      ```
    - **سبب الإشارة**: الحقل `x` يجب أن يكون `private` مع وسائل وصول.

#### فحوصات متنوعة
فحوصات إضافية لجودة الكود.

65. **ArrayTypeStyle**
    - **الغرض**: يفرض نمط تصريح متسق للمصفوفات (`int[]` مقابل `int []`).
    - **مثال على الانتهاك**:
      ```java
      int x[];
      ```
    - **سبب الإشارة**: يجب أن تكون `int[] x`.

66. **FinalParameters**
    - **الغرض**: يتطلب أن تكون معاملات الأسلوب `final` حيثما أمكن.
    - **مثال على الانتهاك**:
      ```java
      public void myMethod(int x) {}
      ```
    - **سبب الإشارة**: المعامل `x` يجب أن يكون `final int x`.

67. **TodoComment**
    - **الغرض**: يشير إلى تعليقات `TODO`.
    - **مثال على الانتهاك**:
      ```java
      // TODO: إصلاح هذا
      public void myMethod() {}
      ```
    - **سبب الإشارة**: تم اكتشاف تعليق `TODO`.

68. **UpperEll**
    - **الغرض**: يضمن استخدام `L` للقيم الحرفية الطويلة (وليس `l`).
    - **مثال على الانتهاك**:
      ```java
      long x = 100l;
      ```
    - **سبب الإشارة**: `100l` يجب أن تكون `100L`.

#### مرشحات الكبح (داخل TreeWalker)
هذه تسمح بالكبح داخل شجرة بناء الجملة (AST).

69. **SuppressionXpathFilter**
    - **الغرض**: يكبح الفحوصات باستخدام تعبيرات XPath من ملف (مثل `checkstyle-xpath-suppressions.xml`).
    - **مثال على الانتهاك**: مشابه لـ `SuppressionFilter` ولكنه يستخدم XPath (مثل كبح `JavadocMethod` لأسلوب معين).

70. **SuppressWarningsHolder**
    - **الغرض**: يدعم `@SuppressWarnings("checkstyle:...")` في شجرة بناء الجملة (AST).
    - **مثال على الانتهاك**:
      ```java
      public class MyClass {
          int MY_FIELD; // ينتهك MemberName
      }
      ```
      ```java
      @SuppressWarnings("checkstyle:MemberName")
      public class MyClass {
          int MY_FIELD; // لا يوجد انتهاك
      }
      ```
    - **سبب الإشارة**: بدون الكبح، `MY_FIELD` تنتهك `MemberName`.

### ملاحظات رئيسية
- **القدرة على التكوين**: قواعد مثل `LineLength` أو `MethodLength` يمكن أن تحتوي على عتبات مخصصة (مثل `max="120"` لـ `LineLength`).
- **الكبح**: استخدم `@SuppressWarnings("checkstyle:<check-name>")` أو ملفات الكبح لتجاوز فحوصات محددة.
- **التكامل**: شغّل مع Maven/Gradle أو إضافات بيئة التطوير؛ يتم الإبلاغ عن الأخطاء أثناء عملية البناء.
- **فحص الرأس**: غير نشط ما لم تتم إزالته من التعليق وتكوينه باستخدام `checkstyle.header.file`.

إذا كنت بحاجة إلى أمثلة تكوين محددة (مثل تغيير `LineLength` إلى 120)، أو إصلاحات للانتهاكات، أو تفاصيل التكامل، فأعلمني!
