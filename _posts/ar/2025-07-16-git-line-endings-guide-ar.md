`git config core.autocrlf` يتحكم في كيفية تعامل Git مع نهايات الأسطر عند استخراج الملفات من المستودع وعند إعادة إيداعها. هذه الإعدادات مهمة لأن نظام Windows يستخدم **CRLF** (`\r\n`) لنهايات الأسطر، بينما تستخدم أنظمة Linux وmacOS **LF** (`\n`). بدون معالجة مناسبة، قد تتبدل تنسيقات الملفات اعتمادًا على من يقوم بتعديلها، مما يؤدي إلى وجود اختلافات "غير مرغوب فيها" (dirty diffs) وتغييرات غير ضرورية.

---

### آلية العمل

* **عند استخراج (قراءة)** ملف، قد يحول Git نهايات الأسطر لتتوافق مع اصطلاح نظام التشغيل الخاص بك (بحيث تظهر طبيعية في محرر النصوص الخاص بك).
* **عند إيداع (كتابة)** ملف، قد يحول Git نهايات الأسطر مرة أخرى إلى LF (المعيار متعدد المنصات في المستودعات).

---

### الإعدادات

يمكنك ضبطه باستخدام:

```bash
git config --global core.autocrlf <value>
```

القيم الشائعة:

* `true`
  حول **LF → CRLF** عند الاستخراج، و **CRLF → LF** عند الإيداع.
  الأفضل **لمطوري Windows** الذين يقومون بتحرير الكود باستخدام محررات نصوص Windows.

* `input`
  **لا** تعدل الملفات عند الاستخراج، ولكن حول **CRLF → LF** عند الإيداع.
  الأفضل **لمطوري Linux/macOS** لضمان أن المستودعات تستخدم LF فقط.

* `false`
  لا يوجد تحويل على الإطلاق. يقوم Git بتخزين واستخراج الملفات كما هي تمامًا.
  الأفضل إذا كان فريقك/محررك يتعامل مع نهايات الأسطر بشكل متناسق.

---

### التوصيات النموذجية

* **Windows**:

  ```bash
  git config --global core.autocrlf true
  ```
* **Linux/macOS**:

  ```bash
  git config --global core.autocrlf input
  ```

بالنسبة للفرق المختلطة، من الجيد أيضًا إضافة ملف `.gitattributes` في المستود لفرض نهايات أسطر متناسقة، على سبيل المثال:

```
* text=auto
```

---

هل تريد مني أيضًا أن أعرض لك **إعداد `.gitattributes` الموصى به** والذي يعمل بشكل جيد مع الفرق متعددة المنصات (حتى لا تعتمد فقط على `core.autocrlf`)?
