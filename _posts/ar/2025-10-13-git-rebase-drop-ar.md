نعم — تمامًا.

عندما تستخدم **`drop`** أثناء `git rebase -i` التفاعلي، فهذا يعني **"إزالة هذا commit بالكامل من السجل."**

إليك ما يحدث:

* سيتخطى Git **تطبيق** ذلك commit أثناء إعادة تطبيق الآخرين.
* سيبدو الفرع الناتج وكأن ذلك commit **لم يوجد أبدًا** — كلا من تغييراته ورسالة commit الخاصة به ستختفي.
* سيتم仍然 تطبيق الـ commits التالية (إذا لم تكن تعتمد على التغييرات التي تمت إزالتها).

مثال:

```bash
git rebase -i HEAD~4
```

قد ترى شيئًا مثل:

```
pick a1b2c3d إصلاح خطأ مطبعي
pick b2c3d4e إضافة README
pick c3d4e5f تنفيذ تسجيل الدخول
pick d4e5f6g إضافة سجل التصحيح
```

إذا قمت بتغيير السطر الأخير إلى:

```
drop d4e5f6g إضافة سجل التصحيح
```

فبعد الحفظ وإغلاق المحرر، سيقوم Git بإعادة قاعدة الفرع و**استبعاد ذلك commit**.

إذن نعم — "drop" = **حذف الـ commit من سجل الفرع.**

هل تريد مني أن أشرح كيف يختلف هذا عن `revert` (الذي *يلغي* commit ولكنه يحتفظ بسجل له)؟
