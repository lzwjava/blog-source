لتثبيت Arduino IDE على Ubuntu Linux، اتبع الخطوات التالية:

1. **تحميل Arduino IDE**:
   - قم بزيارة الموقع الرسمي لـ Arduino على [https://www.arduino.cc/en/software](https://www.arduino.cc/en/software).
   - ضمن قسم "Arduino IDE"، اختر خيار التحميل "Linux 64 bits" (أو الإصدار المناسب لنظامك، مثل Linux ARM لـ Raspberry Pi).

2. **استخراج الملف الذي تم تحميله**:
   - الملف الذي تم تحميله هو أرشيف مضغوط `.tar.xz` (مثال: `arduino-1.8.19-linux64.tar.xz`).
   - افتح طرفية وانتقل إلى دليل التحميل (مثال: `cd ~/Downloads`).
   - استخرج الملف باستخدام:
     ```bash
     tar -xf arduino-*.tar.xz
     ```
   - هذا ينشئ مجلدًا مثل `arduino-1.8.19`.

3. **النقل إلى دليل مناسب (اختياري)**:
   - للوصول على مستوى النظام، انقل المجلد المستخرج إلى `/opt` (يتطلب صلاحيات superuser):
     ```bash
     sudo mv arduino-1.8.19 /opt/arduino
     ```

4. **تشغيل سكريبت التثبيت**:
   - انتقل إلى مجلد Arduino:
     ```bash
     cd /opt/arduino
     ```
   - شغل سكريبت التثبيت:
     ```bash
     sudo ./install.sh
     ```
   - هذا ينشئ اختصارًا على سطح المكتب ويجهز الأذونات اللازمة.

5. **إضافة مستخدمك إلى مجموعة Dialout**:
   - للوصول إلى لوحة Arduino عبر المنفذ التسلسلي، أضف مستخدمك إلى مجموعة `dialout`:
     ```bash
     sudo usermod -a -G dialout $USER
     ```
   - سجل الخروج ثم العودة، أو أعد التشغيل حتى يصبح تغيير المجموعة فعالاً.

6. **تشغيل Arduino IDE**:
   - إذا قمت بتشغيل سكريبت `install.sh`، يمكنك تشغيل الـ IDE من قائمة التطبيقات أو بالبحث عن "Arduino IDE."
   - بدلاً من ذلك، يمكنك تشغيله مباشرة من الطرفية:
     ```bash
     /opt/arduino/arduino
     ```

7. **تحديث أذونات لوحة Arduino (إذا لزم الأمر)**:
   - عند توصيل Arduino الخاص بك، تأكد من اكتشافه (مثال: `/dev/ttyACM0` أو `/dev/ttyUSB0`).
   - إذا واجهت مشاكل في الأذونات، عيّن الأذونات الصحيحة:
     ```bash
     sudo chmod a+rw /dev/ttyACM0
     ```
   - استبدل `/dev/ttyACM0` بالمنفذ المناسب.

8. **التحقق من التثبيت**:
   - صل لوحة Arduino الخاصة بك عبر USB.
   - افتح الـ IDE، واذهب إلى **Tools > Board** واختر موديل Arduino الخاص بك (مثال: Arduino Uno).
   - اذهب إلى **Tools > Port** واختر المنفذ (مثال: `/dev/ttyACM0`).
   - حاول رفع رسم بياني عينة (مثال: **File > Examples > 01.Basics > Blink**) لتأكيد أن كل شيء يعمل.

**بديل: التثبيت عبر Snap أو Flatpak**:
- **Snap**:
  ```bash
  sudo snap install arduino
  ```
- **Flatpak**:
  ```bash
  sudo flatpak install flathub cc.arduino.ArduinoIDE
  ```
- هذه الطرق تبسط التثبيت ولكن قد يكون لها قيود مع الأذونات أو اكتشاف اللوحة.

**ملاحظات**:
- تأكد من تثبيت Java، لأن Arduino IDE يتطلبها. قم بتثبيتها إذا لزم الأمر:
  ```bash
  sudo apt update
  sudo apt install default-jre
  ```
- للحصول على أحدث إصدار أو استكشاف الأخطاء وإصلاحها، تحقق من منتديات Arduino أو [التوثيق الرسمي](https://www.arduino.cc/en/Guide/Linux).

إذا واجهت مشاكل، أعلمني بإصدار Ubuntu أو الخطأ المحدد للحصول على مساعدة مخصصة!

بما أن Arduino Uno الخاص بك يتم اكتشافه في Arduino IDE على Ubuntu وتريد مثالاً بسيطًا دون استخدام لوحة توصيل أو مكونات خارجية، إليك رسم بياني أساسي يومض LED المدمج على Arduino Uno (المتصل بالمنفذ الرقمي 13). هذا لا يتطلب أي أجهزة إضافية.

```cpp
void setup() {
  // put your setup code here, to run once:
  pinMode(LED_BUILTIN, OUTPUT); // Set the built-in LED pin (pin 13) as output
}

void loop() {
  // put your main code here, to run repeatedly:
  digitalWrite(LED_BUILTIN, HIGH); // Turn the LED on
  delay(1000);                     // Wait for 1 second
  digitalWrite(LED_BUILTIN, LOW);  // Turn the LED off
  delay(1000);                     // Wait for 1 second
}
```

### كيفية تشغيله:
1. افتح Arduino IDE.
2. انسخ والصق الكود أعلاه في الـ IDE.
3. تأكد من أن Arduino Uno الخاص بك متصل عبر USB وتم اكتشافه:
   - اذهب إلى **Tools > Board** واختر **Arduino Uno**.
   - اذهب إلى **Tools > Port** واختر المنفذ (مثال: `/dev/ttyACM0` أو `/dev/ttyUSB0`).
4. انقر على زر **Upload** (أيقونة السهم الأيمن) لترجمة وتحميل الرسم البياني إلى Arduino Uno الخاص بك.
5. بمجرد التحميل، يجب أن يومض LED المدمج (الموسوم بـ "L" على اللوحة) مضاءً لمدة ثانية واحدة، ثم مطفأً لمدة ثانية واحدة، بشكل متكرر.

### ما يفعله:
- في `setup()`، تقوم دالة `pinMode` بتكوين منفذ LED المدمج كمخرج.
- في `loop()`، تقوم دالة `digitalWrite` بتشغيل LED (`HIGH`) وإيقافه (`LOW`)، مع توقف `delay` لمدة ثانية واحدة بين كل حالة، مما يخلق تأثير وميض.

### ملاحظات:
- لا حاجة لمكونات خارجية لأن Arduino Uno يحتوي على LED مدمج على المنفذ 13.
- إذا لم يومض LED، تحقق مرة أخرى من اختيار المنفذ أو تأكد من أن اللوحة متصلة بشكل صحيح. يمكنك أيضًا فتح **Serial Monitor** (Tools > Serial Monitor) لتصحيح الأخطاء إذا أضفت `Serial.begin(9600);` في `setup()` وطباعة رسائل مثل `Serial.println("Blink");` في `loop()`.
- إذا كنت تريد تعديل سرعة الوميض، اضبط قيم `delay(1000)` (مثال: `delay(500)` للوميض أسرع).

أعلمني إذا واجهت مشاكل أو تريد مثالاً بسيطًا آخر!

تشير رسالة الخطأ إلى مشكلة في الأذونات عند محاولة تحميل الرسم البياني إلى Arduino Uno الخاص بك على Ubuntu. لا يمكن لـ Arduino IDE الوصول إلى المنفذ التسلسلي `/dev/ttyUSB0` بسبب أذونات غير كافية. إليك كيفية إصلاح ذلك:

### الحل: إصلاح أذونات المنفذ التسلسلي

1. **أضف مستخدمك إلى مجموعة `dialout`**:
   - تتحكم مجموعة `dialout` عادةً في الوصول إلى المنافذ التسلسلية في Ubuntu. أضف مستخدمك إلى هذه المجموعة:
     ```bash
     sudo usermod -a -G dialout $USER
     ```
   - بعد تشغيل هذا، سجل الخروج ثم العودة، أو أعد تشغيل نظامك لتصبح التغييرات فعالة:
     ```bash
     sudo reboot
     ```

2. **تحقق من المنفذ**:
   - بعد إعادة التشغيل، أعد توصيل Arduino Uno وافتح Arduino IDE.
   - اذهب إلى **Tools > Port** وتأكد من اختيار `/dev/ttyUSB0` (أو `/dev/ttyACM0` لبعض لوحات Arduino). إذا لم يظهر أي منفذ، تحقق من اتصال USB أو جرب كبل/منفذ مختلف.

3. **تغيير أذونات المنفذ مؤقتًا (اختياري)**:
   - إذا استمرت المشكلة بعد إضافة مستخدمك إلى مجموعة `dialout`، يمكنك تعيين أذونات المنفذ يدويًا (هذا حل مؤقت، حيث تتم إعادة تعيين الأذونات عند إعادة التشغيل):
     ```bash
     sudo chmod a+rw /dev/ttyUSB0
     ```
   - استبدل `/dev/ttyUSB0` بالمنفذ الصحيح إذا كان مختلفًا (مثال: `/dev/ttyACM0`).

4. **حاول التحميل مرة أخرى**:
   - في Arduino IDE، انقر على زر **Upload** لتحميل الرسم البياني الخاص بك (مثال: رسم وميض LED من قبل).
   - إذا تم التحميل بنجاح، يجب أن يبدأ LED المدمج على Arduino Uno الخاص بك في الوميض.

### استكشاف الأخطاء وإصلاحها الإضافي

- **تحقق من لوحة Arduino والكبل**:
  - تأكد من أن Arduino Uno متصل بشكل صحيح عبر USB ومعترف به من قبل Ubuntu. شغل هذا الأمر لسرد الأجهزة المتصلة:
    ```bash
    ls /dev/tty*
    ```
    ابحث عن `/dev/ttyUSB0` أو `/dev/ttyACM0` عندما تكون اللوحة موصولة.
  - جرب كبل USB أو منفذ مختلفًا إذا لم يتم اكتشاف اللوحة.

- **تحقق من إعدادات اللوحة والمنفذ**:
  - في Arduino IDE، تأكد من تعيين **Tools > Board** إلى **Arduino Uno**.
  - إذا لم يظهر `/dev/ttyUSB0`، جرب **Tools > Port** بعد فصل اللوحة وإعادة توصيلها.

- **تشغيل Arduino IDE كـ Root (حل مؤقت)**:
  - إذا لم تنجح الخطوات أعلاه، يمكنك تشغيل الـ IDE بصلاحيات superuser لتجاوز مشاكل الأذونات (غير موصى به على المدى الطويل):
    ```bash
    sudo arduino
    ```
    استبدل `arduino` بالمسار إلى ملف Arduino IDE القابل للتنفيذ إذا لم يكن في PATH الخاص بنظامك (مثال: `/opt/arduino/arduino`).

- **تحقق من البرامج المتضاربة**:
  - يمكن لبرامج أخرى (مثال: `brltty`، برنامج تشغيل شاشة برايل) أن تتداخل مع المنافذ التسلسلية. أزله إذا كان مثبتًا:
    ```bash
    sudo apt remove brltty
    ```

- **مشاكل برامج التشغيل**:
  - تستخدم بعض نسخ Arduino المقلدة شريحة CH340/CH341، والتي تتطلب برنامج تشغيل. قم بتثبيته إذا لزم الأمر:
    ```bash
    sudo apt install linux-modules-extra-$(uname -r)
    ```
    ثم، أعد توصيل اللوحة.

### تأكيد الإصلاح
بمجرد حل مشكلة الأذونات، حاول تحميل رسم الوميض السابق مرة أخرى:
```cpp
void setup() {
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_BUILTIN, HIGH);
  delay(1000);
  digitalWrite(LED_BUILTIN, LOW);
  delay(1000);
}
```

إذا كنت لا تزال تواجه مشاكل، شارك رسالة الخطأ الجديدة أو شغل `ls -l /dev/ttyUSB0` في الطرفية وقدم الناتج للمساعدة في التشخيص further.

يبدو أنك تواجه مشكلة أثناء محاولة تحميل رسم بياني إلى Arduino Uno باستخدام Arduino IDE على Ubuntu. رسالة الخطأ التي قدمتها هي:

```
avrdude: ser_open(): can't open device "/dev/ttyUSB0": Permission denied
Failed uploading: uploading error: exit status 1
```

يشير هذا الخطأ إلى أن Arduino IDE لا يمتلك الأذونات اللازمة للوصول إلى المنفذ التسلسلي (`/dev/ttyUSB0`) حيث يتم توصيل Arduino Uno الخاص بك. هذه مشكلة شائعة على أنظمة Linux مثل Ubuntu بسبب كيفية إدارة الوصول إلى المنفذ التسلسلي. أدناه دليل خطوة بخطوة لإصلاح هذه المشكلة.

### الحل: إصلاح مشكلة أذونات المنفذ التسلسلي

#### 1. أضف مستخدمك إلى مجموعة `dialout`
في Ubuntu، تتحكم مجموعة `dialout` في الوصول إلى المنافذ التسلسلية. لحل مشكلة الأذونات، تحتاج إلى إضافة مستخدمك إلى هذه المجموعة.

- افتح طرفية بالضغط على `Ctrl + Alt + T`.
- شغل الأمر التالي:
  ```bash
  sudo usermod -a -G dialout $USER
  ```
- يضيف هذا الأمر مستخدمك الحالي إلى مجموعة `dialout`. يستخدم متغير `$USER` اسم المستخدم الخاص بك تلقائيًا.

#### 2. سجل الخروج ثم العودة
حتى يصبح تغيير المجموعة فعالاً، تحتاج إلى تسجيل الخروج من الجلسة الحالية ثم العودة. بدلاً من ذلك، يمكنك إعادة تشغيل نظامك.

- **لتسجيل الخروج ثم العودة:**
  - احفظ أي عمل مفتوح.
  - انقر على اسم المستخدم في الزاوية اليمنى العليا لسطح مكتب Ubuntu واختر "Log Out."
  - سجل الدخول مرة أخرى باسم المستخدم وكلمة المرور الخاصة بك.
- **لإعادة التشغيل (اختياري):**
  - شغل هذا الأمر في الطرفية:
    ```bash
    sudo reboot
    ```
  - انتظر حتى يعيد نظامك التشغيل.

#### 3. تحقق من المنفذ التسلسلي
بعد تسجيل الدخول مرة أخرى، تأكد من أن Arduino Uno الخاص بك معترف به بشكل صحيح من قبل النظام.

- أعد توصيل Arduino Uno بمنفذ USB.
- افتح Arduino IDE.
- اذهب إلى **Tools > Port** في القائمة وتحقق مما إذا كان `/dev/ttyUSB0` (أو `/dev/ttyACM0` لبعض لوحات Arduino) مدرجًا ومحددًا.
- إذا لم يظهر المنفذ:
  - افصل وأعد توصيل Arduino Uno.
  - تحقق من اتصال USB أو جرب كبل/منفذ مختلف.

#### 4. حمّل الرسم البياني مرة أخرى
مع تحديد المنفذ الصحيح، حاول تحميل الرسم البياني الخاص بك مرة أخرى:

```cpp
void setup() {
  pinMode(LED_BUILTIN, OUTPUT); // Set the built-in LED pin (pin 13) as output
}

void loop() {
  digitalWrite(LED_BUILTIN, HIGH); // Turn the LED on
  delay(1000);                     // Wait for 1 second
  digitalWrite(LED_BUILTIN, LOW);  // Turn the LED off
  delay(1000);                     // Wait for 1 second
}
```

- انقر على زر **Upload** (أيقونة السهم الأيمن) في Arduino IDE.
- إذا نجح الأمر، يجب أن يبدأ LED المدمج على Arduino Uno الخاص بك في الوميض (مضاءً لمدة ثانية واحدة، مطفأً لمدة ثانية واحدة).
