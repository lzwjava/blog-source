### نظرة عامة
تعتبر Azure SQL Database وAmazon RDS خدمتين قاعدتين للبيانات العلائقية مُدارتين بالكامل، مصممتين لتبسيط إدارة قواعد البيانات، وتوسيع نطاقها، وصيانتها. تُعد Azure SQL Database عروض PaaS من Microsoft التي تركز بشكل أساسي على SQL Server (مع خدمات مكافئة مثل Azure Database لـ MySQL وPostgreSQL لمحركات أخرى)، بينما AWS RDS هي خدمة Amazon متعددة المحركات التي تدعم SQL Server وMySQL وPostgreSQL وOracle وMariaDB ومتغيرات Aurora الخاصة. يعتمد الاختيار غالبًا على نظامك البيئي (التكامل مع Microsoft يُفضل Azure؛ البيئات متعددة السحابة أو المحركات المتنوعة تُفضل AWS)، ونوع عبء العمل، واحتياجات الترحيل. فيما يلي مقارنة جنبًا إلى جنب عبر الأبعاد الرئيسية.

| الفئة              | Azure SQL Database                                                                 | AWS RDS                                                                 |
|-----------------------|------------------------------------------------------------------------------------|-------------------------------------------------------------------------|
| **المحركات المدعومة** | بشكل أساسي SQL Server (دائمًا أحدث إصدار، مثل 2022)؛ خدمات منفصلة لـ MySQL وPostgreSQL وMariaDB. لا يوجد دعم مباشر لـ Oracle في شكل مُدار (استخدم الآلات الافتراضية). | متعددة المحركات: SQL Server (الإصدارات 2012–2019)، MySQL، PostgreSQL، Oracle، MariaDB، وAurora (متوافق مع MySQL/PostgreSQL بأداء أعلى). |

### التوصيات
- **اختر Azure SQL Database** إذا كنت في بيئة تعتمد بشكل كبير على Microsoft (مثل .NET، Active Directory)، أو تحتاج إلى أحدث ميزات SQL، أو تُعطي أولوية للتوسع بدون خادم/مرن لأحمال العمل المتغيرة.
- **اختر AWS RDS** للحصول على دعم متعدد المحركات، أو توافق SQL للإصدارات القديمة، أو إذا كنت تستخدم بالفعل AWS للحصول على خدمات سحابية أوسع.
- لعمليات ترحيل SQL Server المحددة، اختبر الأداء مع عبء العمل الخاص بك — حيث يتفوق Azure في التوسع الديناميكي، بينما يقدم RDS أجهزة متوقعة.
