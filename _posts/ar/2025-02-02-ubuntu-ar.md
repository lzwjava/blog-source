## التثبيت

كتبتُ سابقًا عن [كيفية بناء حاسوب](./computer-en). دعونا نجرب شيئًا مختلفًا. اليوم، أقوم بتثبيت Ubuntu 24.04 على حاسوب محمول Lenovo Xiaoxing 14IAH8 جاء مُثبتًا عليه Windows 10 مسبقًا.

```bash
sudo dd if=ubuntu-24.04.1-desktop-amd64.iso of=/dev/rdisk6 bs=1m
```

من الضروري الذهاب إلى الإعدادات وتعطيل تشفير الجهاز، لأن برنامج تثبيت Ubuntu يتطلب ذلك.

إذا تم تمكين تشفير الجهاز وتعطيل التمهيد الآمن في BIOS، فسيطالب Windows بكلمة مرور BitLocker للإنقاذ.

لذلك، من الضروري أولاً تمكين التمهيد الآمن في BIOS، وتسجيل الدخول إلى Windows، ثم الانتقال إلى الإعدادات لتعطيل تشفير الجهاز. بعد ذلك، يمكن تعطيل التمهيد الآمن مرة أخرى لبدء برنامج تثبيت Ubuntu.

## التعليق

عندما يبدأ Ubuntu خدمة `gdm.service` (مدير عرض GNOME) ويعلق، فهذا عادةً ما يشير إلى مشكلة في واجهة المستخدم الرسومية (GUI) أو مدير العرض نفسه. فيما يلي بعض الخطوات التي يمكنك اتخاذها لاستكشاف المشكلة وحلها:

1. **الوصول إلى محطة افتراضية:**
   - إذا تعلق نظامك في شاشة GDM، يمكنك محاولة الوصول إلى محطة لتشخيص المشكلة.
   - اضغط على `Ctrl + Alt + F2` (أو `F3`، `F4`، وما إلى ذلك) للتبديل إلى محطة افتراضية.

2. **تسجيل الدخول:**
   - سجل الدخول باستخدام اسم المستخدم وكلمة المرور الخاصين بك.

3. **التحقق من سجلات النظام:**
   - استخدم الأمر التالي للتحقق من سجلات النظام بحثًا عن أي أخطاء أو تحذيرات قد تشير إلى سبب المشكلة:
     ```bash
     sudo journalctl -xe
     ```
4. **إعادة تثبيت خدمة GDM:**
   - قم بمسح وإعادة تثبيت GDM3. سيؤدي هذا إلى إزالة ملفات التكوين وضمان تثبيت نظيف:
     ```bash
     sudo apt remove --purge gdm3
     sudo apt install gdm3
     ```

5. **إعادة تشغيل النظام:**
   - أعد تشغيل النظام لتطبيق التغييرات:
     ```bash
     sudo reboot
     ```

## LightDM

إذا فشل GDM3، يمكن استخدام LightDM كمدير عرض بديل. ومع ذلك، بعد تثبيت LightDM، قد تواجه مشكلة حيث تظهر شاشة تسجيل الدخول، ولكن يتم رفض أي كلمة مرور تدخلها على أنها غير صحيحة.

1. **تثبيت وتكوين LightDM:**
   - لتثبيت LightDM:
     ```bash
     sudo apt-get install lightdm
     ```
   - أثناء التثبيت، سيُطلب منك اختيار مدير عرض افتراضي. اختر LightDM.
