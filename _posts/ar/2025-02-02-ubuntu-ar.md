## التثبيت

كتبتُ سابقًا عن [كيفية بناء جهاز كمبيوتر](./computer-en). دعونا نجرب شيئًا مختلفًا. اليوم، أقوم بتثبيت Ubuntu 24.04 على جهاز كمبيوتر محمول Lenovo Xiaoxing 14IAH8  جاء مُثبتًا مسبقًا بنظام التشغيل Windows 10.

```bash
sudo dd if=ubuntu-24.04.1-desktop-amd64.iso of=/dev/rdisk6 bs=1m
```

من الضروري الذهاب إلى الإعدادات وتعطيل تشفير الجهاز، حيث يتطلب برنامج تثبيت Ubuntu ذلك.

إذا تم تمكين تشفير الجهاز وتعطيل التمهيد الآمن في BIOS، فسيطلب Windows عبارة مرور BitLocker للإنقاذ.

لذلك، من الضروري أولاً تمكين التمهيد الآمن في BIOS، وتسجيل الدخول إلى Windows، ثم الانتقال إلى الإعدادات لتعطيل تشفير الجهاز. بعد ذلك، يمكن تعطيل التمهيد الآمن مرة أخرى لبدء برنامج تثبيت Ubuntu.

## التعليق

عندما يبدأ Ubuntu خدمة `gdm.service` (مدير عرض GNOME) ويتعطل، فهذا عادةً ما يشير إلى مشكلة في واجهة المستخدم الرسومية (GUI) أو مدير العرض نفسه. فيما يلي بعض الخطوات التي يمكنك اتخاذها لاستكشاف المشكلة وحلها:

1. **الوصول إلى محطة افتراضية:**
   - إذا تعلق نظامك في شاشة GDM، فيمكنك محاولة الوصول إلى محطة لتشخيص المشكلة.
   - اضغط على `Ctrl + Alt + F2` (أو `F3`، `F4`، وما إلى ذلك) للتبديل إلى محطة افتراضية.

2. **تسجيل الدخول:**
   - سجل الدخول باستخدام اسم المستخدم وكلمة المرور الخاصين بك.

3. **التحقق من سجلات النظام:**
   - استخدم الأمر التالي للتحقق من سجلات النظام بحثًا عن أي أخطاء أو تحذيرات قد تشير إلى سبب المشكلة:
     ```bash
     sudo journalctl -xe
     ```

4. **إعادة تشغيل خدمة GDM:**
   - حاول إعادة تشغيل خدمة GDM لمعرفة ما إذا كان ذلك يحل المشكلة:
     ```bash
     sudo systemctl restart gdm
     ```

5. **تثبيت وتكوين LightDM:**
   - إذا استمرت المشكلة، يمكنك محاولة تثبيت مدير عرض بديل:
     ```bash
     sudo apt-get install lightdm
     ```
   - بعد تثبيت lightdm، سيطالبك النظام بتعيين مدير العرض الافتراضي.

6. **إعادة تشغيل النظام:**
   - بعد إجراء التغييرات، أعد تشغيل النظام لتطبيقها:
     ```bash
     sudo reboot
     ```
