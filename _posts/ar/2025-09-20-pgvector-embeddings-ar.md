PGVector هو **امتداد لـ PostgreSQL** يضيف دعمًا لتخزين واستعلام *التضمينات المتجهية* (مصفوفات من الأرقام، غالبًا بأبعاد تتراوح بين 128 و4096) داخل قاعدة بيانات Postgres.

يُستخدم عادةً في **تطبيقات الذكاء الاصطناعي والتعلم الآلي** حيث تحتاج إلى تخزين والبحث عن التضمينات التي تم إنشاؤها بواسطة نماذج مثل OpenAI أو Hugging Face أو مزودي التضمين الآخرين. على سبيل المثال، قد تقوم بتضمين مستندات أو مقاطع كود أو أوصاف منتجات في متجهات، ثم استخدام PGVector للعثور بسرعة على الأكثر تشابهًا مع استعلام المستخدم.

### الميزات الرئيسية

* **نوع بيانات متجهي**: يضيف نوع عمود `vector` إلى جداول Postgres.
* **بحث عن التشابه**: يدعم عمليات مثل تشابه جيب التمام أو المسافة الإقليدية أو الضرب الداخلي لمقارنة المتجهات.
* **فهارس للسرعة**: يوفر أنواع فهارس `ivfflat` و `hnsw` لإجراء *بحث تقريبي عن أقرب جار (ANN)* بكفاءة، مما يسمح باسترجاع سريع حتى مع وجود الملايين من التضمينات.
* **التكامل مع SQL**: لأنه يعمل داخل Postgres، يمكنك الجمع بين البحث المتجهي واستعلامات SQL العادية وعمليات الربط (Joins) والتصفية (Filters).

### مثال للاستخدام

```sql
-- إنشاء جدول بعمود متجهي
CREATE TABLE items (
  id bigserial PRIMARY KEY,
  embedding vector(1536)  -- يجب أن يتطابق البعد مع نموذجك
);

-- إدخال صف
INSERT INTO items (embedding) VALUES ('[0.25, 0.1, ...]');

-- البحث عن أكثر تضمين مشابه
SELECT id, embedding
FROM items
ORDER BY embedding <-> '[0.24, 0.11, ...]'
LIMIT 5;
```

هنا `<->` هو عامل التشابه.

### لماذا هو مهم

يتيح لك PGVector بناء **قاعدة بيانات متجهية كاملة مباشرة داخل Postgres**، بدلاً من استخدام قاعدة بيانات متجهية متخصصة منفصلة (مثل Pinecone أو Weaviate أو Milvus). هذا أمر جذاب إذا كنت تستخدم Postgres بالفعل وتريد الاحتفاظ بجميع البيانات والاستعلامات في مكان واحد.
