خطأ "سجّل الدخول لتأكيد أنك لست روبوتًا" في youtube-dl يحدث عادةً لأن يوتيوب يحجب الطلب، غالبًا بسبب قيود على عنوان IP، أو استخدام VPN، أو عدم وجود مصادقة. إليك خطوات حل المشكلة، بدءًا بأبسط الحلول:

1.  **تحديث youtube-dl**:
    - قد يكون الخطأ ناتجًا عن إصدار قديم من youtube-dl، حيث يقوم يوتيوب بتحديث أنظمته بشكل متكرر. قم بالتحديث إلى أحدث إصدار عن طريق تشغيل:
        ```bash
        sudo youtube-dl -U
        ```
        أو، إذا قمت بالتثبيت عبر pip:
        ```bash
        pip install --upgrade youtube-dl
        ```
    - بعد التحديث، حاول استخدام الأمر مرة أخرى:
        ```bash
        youtube-dl https://www.youtube.com/watch?v=st3mUEub99E
        ```

2.  **التبديل إلى yt-dlp (بديل موصى به)**:
    - لم يعد يتم صيانة youtube-dlp بنشاط، بينما yt-dlp، وهو فرع من youtube-dlp، أكثر موثوقية للتغييرات الحديثة في يوتيوب. قم بتثبيت yt-dlp:
        ```bash
        sudo pip install yt-dlp
        ```
        ثم استخدم:
        ```bash
        yt-dlp https://www.youtube.com/watch?v=st3mUEub99E
        ```
    - يتعامل yt-dlp مع المصادقة وقيود عنوان IP بشكل أفضل.

3.  **تعطيل VPN أو تغيير الخادم**:
    - إذا كنت تستخدم VPN، فقد يقوم يوتيوب بوضع علامة على عنوان IP الخاص بك كمشبوه. حاول تعطيل الـ VPN أو التبديل إلى خادم مختلف:
        ```bash
        yt-dlp https://www.youtube.com/watch?v=st3mUEub99E
        ```
    - أبلغ المستخدمون عن النجاح بعد فصل اتصال الـ VPN أو تبديل الخوادم.

4.  **استخدام ملفات تعريف الارتباط (Cookies) للمصادقة**:
    - قد يتطلب يوتيوب المصادقة لتجاوز فحص الروبوت. قم بتصدير ملفات تعريف الارتباط من متصفح حيث تكون مسجّل الدخول إلى يوتيوب:
        - قم بتثبيت إضافة متصفح مثل "Export Cookies" لـ Firefox أو Chrome.
        - سجّل الدخول إلى يوتيوب، وقم بتصدير ملفات تعريف الارتباط إلى ملف `cookies.txt`، واستخدمه مع:
            ```bash
            youtube-dl --cookies ~/path/to/cookies.txt https://www.youtube.com/watch?v=st3mUEub99E
            ```
            أو لـ yt-dlp:
            ```bash
            yt-dlp --cookies ~/path/to/cookies.txt https://www.youtube.com/watch?v=st3mUEub99E
            ```
        - بدلاً من ذلك، استخدم `--cookies-from-browser firefox` (أو استبدل `firefox` بـ `chrome`، `edge`، إلخ) لاستخراج ملفات تعريف الارتباط تلقائيًا:
            ```bash
            yt-dlp --cookies-from-browser firefox https://www.youtube.com/watch?v=st3mUEub99E
            ```
        - ملاحظة: تجنب استخدام حساب Google الأساسي الخاص بك لمنع التمييز المحتمل. استخدم حسابًا مؤقتًا إذا أمكن.

5.  **استخدام وكيل (Proxy)**:
    - إذا استمرت المشكلة، قد يكون عنوان IP الخاص بك محظورًا (على سبيل المثال، إذا كنت تستخدم عنوان IP لمركز بيانات). جرب وكيلًا سكنيًا لإخفاء عنوان IP الخاص بك:
        ```bash
        youtube-dl --proxy "http://proxy_address:port" https://www.youtube.com/watch?v=st3mUEub99E
        ```
        أو لـ yt-dlp:
        ```bash
        yt-dlp --proxy "http://proxy_address:port" https://www.youtube.com/watch?v=st3mUEub99E
        ```
    - من غير المرجح أن يتم وضع علامة على الوكلاء السكنيين مقارنة بوكلاء مراكز البيانات.

6.  **مسح الذاكرة المؤقتة أو تجربة شبكة مختلفة**:
    - إذا قمت بمسح سجلات الملفات أو الملفات المؤقتة مؤخرًا، فتأكد من أن youtube-dlp/yt-dlp لا يعتمد على ذاكرة تخزين مؤقت تالفة. امسح الذاكرة المؤقتة:
        ```bash
        rm -rf ~/.cache/youtube-dl
        rm -rf ~/.cache/yt-dlp
        ```
    - انتقل إلى شبكة مختلفة (مثل نقطة اتصال الهاتف المحمول) لاختبار ما إذا كانت المشكلة متعلقة بـ IP.

7.  **الإخراج المفصل (Verbose) لتصحيح الأخطاء**:
    - إذا فشلت الخطوات المذكورة أعلاه، قم بتشغيل الأمر مع إخراج مفصل لتشخيص المشكلة further:
        ```bash
        youtube-dl -v https://www.youtube.com/watch?v=st3mUEub99E
        ```
        أو لـ yt-dlp:
        ```bash
        yt-dlp -v https://www.youtube.com/watch?v=st3mUEub99E
        ```
    - شارك الناتج في منتدى ذي صلة (مثل مشاكل GitHub لـ yt-dlp) للحصول على مساعدة المجتمع.

**ملاحظات**:
- أصبح كشف الروبوت في يوتيوب أكثر صرامة، وغالبًا ما يستهدف عناوين IP لمراكز البيانات أو الطلبات غير المصادق عليها.
- يحمل استخدام ملفات تعريف الارتباط خطرًا صغيرًا يتمثل في تمييز الحساب، لذا استخدم حسابًا ثانويًا.
- إذا كان الفيديو خاصًا أو مقيدًا، فتأكد من أن حسابك لديه حق الوصول قبل استخدام ملفات تعريف الارتباط.
- إذا لم ينجح أي من هذه الحلول، ففكر في استخدام واجهات أمامية بديلة مثل Invidious أو NewPipe، على الرغم من أنها قد تكون لها حدود.

جرب هذه الخطوات بالترتيب، بدءًا من التحديث أو التبديل إلى yt-dlp، حيث أنه الإصلاح الأكثر ترجيحًا. إذا استمرت المشكلة، فأخبرني بناتج verbose أو أي تفاصيل محددة (مثل استخدام VPN، نظام التشغيل).
