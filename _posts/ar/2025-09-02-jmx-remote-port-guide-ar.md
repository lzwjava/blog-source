**وكيل JMX البعيد** هو جزء من تقنية Java Management Extensions (JMX). فهو يسمح للمسؤولين أو أدوات المراقبة بالاتصال بتطبيق Java قيد التشغيل من خارج عملية JVM، عادةً عبر بروتوكول TCP/IP. يُستخدم هذا عادةً لمراقبة تطبيقات Java وتشخيصها وإدارتها في بيئة الإنتاج.

إليك تفصيلاً لذلك:

### 1. ما هو وكيل JMX البعيد

* يعرض البيانات الداخلية لـ JVM (استخدام الذاكرة، نشاط الخيوط، جمع البيانات غير المرغوب فيها، تحميل الفئات، إلخ) كـ *Managed Beans (MBeans)*.
* يسمح لك بالاتصال عن بُعد (على سبيل المثال، عبر **jconsole**، أو **VisualVM**، أو أدوات المراقبة مثل مُصدِّرات Prometheus).
* يتم تمكينه عادةً بواسطة علامات بدء تشغيل JVM مثل:

  ```bash
  -Dcom.sun.management.jmxremote
  -Dcom.sun.management.jmxremote.port=9091
  -Dcom.sun.management.jmxremote.authenticate=false
  -Dcom.sun.management.jmxremote.ssl=false
  ```
* بمجرد التمكين، تقوم JVM بتشغيل خادم موصل RMI (Remote Method Invocation) مرتبط بالمنفذ المحدد.

### 2. حول المنفذ 9091

* الرقم **9091** نفسه ليس مميزًا؛ إنه مجرد منفذ TCP تقوم بتكوينه لاتصالات JMX.
* غالبًا ما يختار المسؤولون 9091 (أو 9010، أو 9999، إلخ) لتسهيل الاستخدام وتجنب التعارض مع المنافذ القياسية.
* عند الاستماع على `9091`، ستقبل JVM اتصالات JMX البعيدة على هذا المنفذ.
* إذا تُركت غير مؤمنة (`authenticate=false, ssl=false`)، فسيتمكن أي شخص لديه وصول إلى الشبكة من التحكم في JVM أو مراقبتها — وهو ما يمثل خطرًا أمنيًا كبيرًا.

### 3. الاستخدام النموذجي

* المراقبة باستخدام **JConsole**:

  ```bash
  jconsole host:9091
  ```
* التكامل مع أنظمة المراقبة (مثل Prometheus JMX Exporter).
* الإدارة عن بُعد أثناء استكشاف الأخطاء وإصلاحها (مثل فحص صور الذاكرة heap dumps، والخيوط threads، وسلوك جمع البيانات غير المرغوب فيها GC behavior).

### 4. الاعتبارات الأمنية

* قم دائمًا بحماية منفذ JMX باستخدام:

  * جدران الحماية للشبكة (تقييد الوصول).
  * المصادقة (ملفات `password` و `access`).
  * SSL إذا لزم الأمر.
* وإلا، فإن تعريضه على المنفذ 9091 بدون أمان يعادل منح تحكم عن بُعد على مستوى shell لـ JVM الخاصة بك.

✅ باختصار: **وكيل JMX البعيد على المنفذ 9091** يعني أن JVM الخاص بك يعرض واجهة إدارة ومراقبة عبر منفذ TCP 9091، والتي يمكن استخدامها بواسطة أدوات مثل JConsole أو Prometheus، ولكن يجب تأمينها.
