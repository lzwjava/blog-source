## 1. نظرة عامة على لغة Java

### 1.1 خصائص Java

*   **الاستقلالية عن المنصة:** يتم تجميع كود Java إلى *بايت كود*، الذي يعمل على أي جهاز به *آلة جافا الافتراضية (JVM)* — "اكتب مرة واحدة، وشغّل في أي مكان."
*   **البرمجة كائنية التوجه:** تدعم *التغليف، والوراثة، وتعدد الأشكال.*
*   **آمنة وقوية:** إدارة الذاكرة التلقائية (جمع القمامة) والتحقق القوي من النوع يقللان من الأخطاء.
*   **متعددة الخيوط:** تدعم التنفيذ المتزامن لمهام متعددة.
*   **مكتبة قياسية غنية (API):** تتضمن فئات جاهزة للرياضيات، والسلاسل النصية، والملفات، والشبكات، إلخ.

### 1.2 إصدارات Java ومكوناتها

*   **JDK (مجموعة تطوير Java):** للمطورين — تتضمن المترجم (`javac`)، و JVM، وأدوات التطوير.
*   **JRE (بيئة تشغيل Java):** للمستخدمين النهائيين — تتضمن JVM + المكتبات الأساسية.
*   **API (واجهة برمجة التطبيقات):** مكتبات الفئات المدمجة في Java، مثل `java.lang`، `java.util`، `java.io`، إلخ.

## 2. أدوات تطوير Java (IDE و CLI)

### 2.1 بيئات التطوير المتكاملة الشائعة

لأغراض الامتحان، تحتاج فقط إلى معرفة الغرض منها:

*   **Eclipse, IntelliJ IDEA, NetBeans:** تُستخدم لكتابة وتجميع وتشغيل كود Java بسهولة.

### 2.2 سير العمل باستخدام سطر الأوامر

خطوات التجميع والتشغيل النموذجية:

1.  **اكتب** الكود الخاص بك في ملف `.java`، على سبيل المثال `Hello.java`
2.  **جمّعه**:

    ```bash
    javac Hello.java
    ```

    → ينتج ملف `Hello.class` (ملف البايت كود)
3.  **شغّله**:

    ```bash
    java Hello
    ```

    (بدون لاحقة `.class` عند التشغيل)

### 2.3 مثال بسيط

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, Java!");
    }
}
```

## 3. إرشادات نمط البرمجة

### 3.1 اتفاقيات التسمية

*   **الفئات:** `CamelCase`، الحرف الأول كبير → `StudentInfo`
*   **المتغيرات والطرق:** `camelCase`، تبدأ بحرف صغير → `studentName`، `calculateScore()`
*   **الثوابت:** جميع الأحرف كبيرة مع شرطات سفلية → `MAX_SIZE`

### 3.2 المسافات البادئة والتعليقات

*   استخدم **مسافات بادئة متسقة** (4 مسافات نموذجيًا).
*   اكتب **تعليقات** واضحة:

    ```java
    // هذا تعليق من سطر واحد
    /* هذا تعليق متعدد الأسطر */
    ```

### 3.3 هيكل الكود

اتبع التجميع المنطقي وسهولة القراءة:

*   يحتوي كل ملف على **فئة عامة واحدة** لها **نفس اسم** الملف.
*   حافظ على الأسطر قصيرة وذات معنى؛ عبارة واحدة في كل سطر.

## 4. أخطاء البرمجة الشائعة وأساسيات التصحيح

### 4.1 أخطاء Syntax

يتم اكتشافها بواسطة المترجم:

*   نقطة فاصلة ناقصة `;`
*   أقواس غير متطابقة `{ }`
*   خطأ في استخدام الحروف الكبيرة والصغيرة (Java **حساسة لحالة الأحرف**)

    ```java
    System.out.println("Hi"); // صحيح
    system.out.println("Hi"); // خطأ
    ```

### 4.2 أخطاء وقت التشغيل

تحدث أثناء تشغيل البرنامج:

*   القسمة على صفر
*   فهرس مصفوفة خارج الحدود
*   استثناءات مؤشر Null

### 4.3 الأخطاء المنطقية

يعمل البرنامج ولكن يعطي مخرجات غير صحيحة.
→ استخدم **عبارات الطباعة** أو **أدوات المصحح** في بيئات التطوير المتكاملة لتتبع قيم المتغيرات خطوة بخطوة.

## 6. نموذج لسؤال امتحاني

**المثال 1:**

> أي من العبارات التالية عن Java *غير صحيحة*؟
> A. تدعم Java الاستقلالية عن المنصة من خلال البايت كود
> B. يجب دائمًا تجميع برامج Java إلى كود آلة أصلي
> C. تقوم JVM بتنفيذ البايت كود المترجم
> D. يمكن تشغيل برامج Java على أي نظام مثبت عليه JVM

**الإجابة:** B

**المثال 2:**

> ما الأمر الذي يجمّع ملف مصدر Java باسم `Test.java`؟
> A. java Test
> B. javac Test.java
> C. run Test
> D. compile Test

**الإجابة:** B
