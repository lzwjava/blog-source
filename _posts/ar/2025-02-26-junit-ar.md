JUnit هو إطار عمل شائع لاختبار تطبيقات Java. يسمح للمطورين بكتابة وتشغيل اختبارات آلية قابلة للتكرار، مما يساعد في التأكد من أن الكود يعمل كما هو متوقع. إليك دليل أساسي حول كيفية استخدام JUnit لإجراء الاختبارات:

### 1. إعداد JUnit في مشروعك

- **Maven**: أضف تبعية JUnit إلى ملف `pom.xml` الخاص بك.
  ```xml
  <dependency>
      <groupId>junit</groupId>
      <artifactId>junit</artifactId>
      <version>4.13.2</version>
      <scope>test</scope>
  </dependency>
  ```

- **Gradle**: أضف تبعية JUnit إلى ملف `build.gradle` الخاص بك.
  ```groovy
  testImplementation 'junit:junit:4.13.2'
  ```

### 2. اكتب فئة اختبار

أنشئ فئة Java جديدة لاختباراتك. من الممارسات الجيدة تسمية فئة الاختبار باسم الفئة التي تختبرها، مع إضافة لاحقة "Test".

```java
import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class MyClassTest {

    @Test
    public void testAddition() {
        MyClass myClass = new MyClass();
        int result = myClass.add(2, 3);
        assertEquals(5, result);
    }
}
```

### 3. التوضيحات (Annotations)

- `@Test`: تشير إلى أن الأسلوب هو أسلوب اختبار.
- `@Before`: يعمل قبل كل أسلوب اختبار. مفيد للإعداد.
- `@After`: يعمل بعد كل أسلوب اختبار. مفيد للتنظيف.
- `@BeforeClass`: يعمل مرة واحدة قبل أي من أساليب الاختبار في الفئة.
- `@AfterClass`: يعمل مرة واحدة بعد كل أساليب الاختبار في الفئة.

### 4. التأكيدات (Assertions)

تُستخدم التأكيدات للتحقق من أن الكود يعمل كما هو متوقع.

- `assertEquals(expected, actual)`: يتحقق من تساوي قيمتين.
- `assertTrue(condition)`: يتحقق من أن الشرط صحيح.
- `assertFalse(condition)`: يتحقق من أن الشرط خاطئ.
- `assertNull(object)`: يتحقق من أن الكائن فارغ (null).
- `assertNotNull(object)`: يتحقق من أن الكائن غير فارغ (ليس null).

### 5. تشغيل اختباراتك

يمكنك تشغيل اختباراتك باستخدام الأدوات المدمجة في بيئة التطوير المتكاملة (IDE) أو عبر سطر الأوامر باستخدام Maven أو Gradle.

- **Maven**: `mvn test`
- **Gradle**: `gradle test`

### مثال

إليك مثالًا بسيطًا لفئة وفئة الاختبار المقابلة لها:

```java
// MyClass.java
public class MyClass {
    public int add(int a, int b) {
        return a + b;
    }
}

// MyClassTest.java
import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class MyClassTest {

    @Test
    public void testAddition() {
        MyClass myClass = new MyClass();
        int result = myClass.add(2, 3);
        assertEquals(5, result);
    }
}
```

### نصائح إضافية

- **استقلالية الاختبار**: يجب أن يكون كل اختبار مستقلاً عن الآخرين. تجنب الاعتماديات بين الاختبارات.
- **تغطية الاختبار**: اسعَ لتحقيق تغطية اختبار عالية، لكن ركز على الاختبارات ذات المعنى بدلاً من مجرد زيادة نسبة التغطية.
- **المحاكاة (Mocking)**: استخدم أطر عمل للمحاكاة مثل Mockito لعزل الفئة قيد الاختبار عن تبعياتها.

باتباع هذه الخطوات، يمكنك استخدام JUnit بشكل فعال لكتابة وتشغيل اختبارات لتطبيقات Java الخاصة بك.
