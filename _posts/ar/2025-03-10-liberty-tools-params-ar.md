يمكنك تمرير معلمات JVM المخصصة—مثل `-Dversion=5.1`—من خلال إضافتها إلى تكوين تشغيل وضع المطور (dev mode) في Liberty في VS Code. عادةً، يمكنك القيام بذلك عن طريق تحرير (أو إنشاء) الملف `.vscode/launch.json` في مشروعك وتضمين المعلمة تحت خاصية مثل `"vmArgs"` (أو `"args"` إذا كان التكوين يتوقع وسيطات هدف Maven).

على سبيل المثال، إذا كنت تستخدم إضافة Liberty Tools مع وضع المطور، فقد يبدو التكوين كما يلي:

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "liberty",
      "request": "launch",
      "name": "Liberty Dev Mode",
      "goal": "dev",
      "vmArgs": "-Dversion=5.1"
    }
  ]
}
```

### الخطوات:
1. **افتح أو أنشئ تكوين التشغيل:**
   - في VS Code، افتح مشروعك.
   - انتقل إلى مجلد `.vscode` وافتح (أو أنشئ) الملف `launch.json`.

2. **أضف تكوين وضع المطور:**
   - أضف مدخلة تكوين كما هو موضح أعلاه. تتيح لك المفتاح `"vmArgs"` تحديد خيارات JVM التي سيتم تمريرها إلى الخادم عند بدء وضع المطور.

3. **احفظ وشغّل:**
   - احفظ الملف.
   - استخدم نافذة Run and Debug لبدء تكوين "Liberty Dev Mode".

إذا كنت تستخدم Maven مباشرة، يمكنك أيضًا بدء وضع المطور من سطر الأوامر باستخدام:

```bash
mvn liberty:dev -Dversion=5.1
```

هذا يمرر الخاصية المخصصة إلى JVM عند بدء تشغيل الخادم.

من خلال التكوين بهذه الطريقة، ستضمن إضافة Liberty Tools تضمين المعلمة المخصصة (`-Dversion=5.1`) في كل مرة تبدأ فيها وضع المطور.
