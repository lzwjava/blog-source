عندما تقوم بتغيير ملف `.env`، لن تقوم Docker تلقائيًا بإعادة تحميل متغيرات البيئة هذه. تحتاج إلى إعادة تشغيل الحاويات حتى تلتقط القيم الجديدة. بالنسبة لإعداد LibreChat الخاص بك، يمكنك القيام بذلك كما يلي:

1. تأكد من أنك في نفس الدليل الذي يوجد به ملف `docker-compose.yml` الخاص بك.
2. قم بتنفيذ:

```bash
docker compose down
docker compose up -d
```

سيؤدي هذا إلى إيقاف الحاويات قيد التشغيل وإزالتها، ثم إعادة إنشائها بقيم `.env` الجديدة.

---

### بديل: إعادة التشغيل فقط

إذا كنت لا تريد إيقاف كل شيء بالكامل، يمكنك إجبار الحاويات على التقاط متغيرات البيئة الجديدة عن طريق إعادة البناء:

```bash
docker compose up -d --force-recreate
```

أو إذا كنت قد قمت أيضًا بتغيير ملفات Dockerfile أو التبعيات، أضف `--build`:

```bash
docker compose up -d --build
```

---

✅ باختصار:

* بعد تحرير ملف `.env`، قم دائمًا **بإعادة إنشاء** الحاويات (`down` + `up` أو `up --force-recreate`).
* إن أمر `docker compose restart` البسيط لن يعيد تحميل قيم متغيرات البيئة الجديدة، فهو يعيد التشغيل باستخدام القيم القديمة فقط.

هل تريد أيضًا أن أريك كيفية **التحقق** من أن حاويات LibreChat الخاصة بك تستخدم قيم متغيرات البيئة الجديدة (على سبيل المثال، باستخدام `docker inspect` أو `docker exec env`)?
