لإضافة سائق Oracle JDBC إلى Squirrel SQL Client، اتبع هذه الخطوات التفصيلية. يتضمن هذا العملية تهيئة السائق وإعداد اسم مستعار للاتصال بقاعدة بيانات Oracle.

### الخطوة 1: تحديد الملف المطلوب
تحتاج إلى ملف سائق Oracle JDBC، تحديدًا `ojdbc.jar`. يحتوي هذا الملف على الفئات اللازمة لتتيح لـ Squirrel SQL Client الاتصال بقاعدة بيانات Oracle. تأكد من تنزيل هذا الملف (مثلًا من موقع Oracle أو من مدير قاعدة البيانات الخاص بك) وتعرف مكانه على نظامك.

### الخطوة 2: تشغيل Squirrel SQL Client
افتح تطبيق Squirrel SQL Client على جهازك.

### الخطوة 3: الوصول إلى علامة التبويب Drivers
على الجانب الأيسر من واجهة Squirrel SQL Client، حدد و انقر على علامة التبويب **Drivers**. تسمح هذه القسم بإدارة سائقي JDBC المتاحة للتطبيق.

### الخطوة 4: إضافة سائق جديد
- في علامة التبويب Drivers، انقر على الزر **"+"** لفتح مربع الحوار "Add Driver".

### الخطوة 5: تسمية السائق
- في حقل "Name" من مربع الحوار "Add Driver"، أدخل **Oracle Thin Driver**. هذا اسم وصفي لتحديد سائق Oracle داخل Squirrel SQL Client.

### الخطوة 6: إضافة ملف `ojdbc.jar`
- انتقل إلى علامة التبويب **Extra Class Path** داخل مربع الحوار "Add Driver".
- انقر على الزر **Add**.
- انتقل إلى موقع ملف `ojdbc.jar` على نظامك، حدده، واكد لإضافته إلى مسار الفئة للسائق.

### الخطوة 7: تحديد فئة السائق Java
- في حقل "Class Name"، أدخل فئة السائق Java: **oracle.jdbc.OracleDriver**. هذا يوضح لـ Squirrel SQL Client فئة يجب استخدامها من ملف `ojdbc.jar` لتسوية اتصالات قاعدة بيانات Oracle.

### الخطوة 8: تقديم نموذج URL
- يمكنك اختياريًا تحديد نموذج URL مثال للاتصال بقاعدة بيانات Oracle:
  - **لالاتصال عبر SID**: `jdbc:oracle:thin:@HOST[:PORT]:DB`
  - **لالاتصال عبر اسم الخدمة**: `jdbc:oracle:thin:@//HOST[:PORT]/DB`
- استبدل `HOST` و `PORT` و `DB` بقيم حقيقية عند إعداد اتصال لاحق (في تكوين الاسم المستعار).

### الخطوة 9: حفظ إعدادات السائق
- انقر على **OK** لحفظ إعدادات السائق وإغلاق مربع الحوار "Add Driver". يجب أن يظهر "Oracle Thin Driver" الآن في علامة التبويب Drivers مع علامة التحقق الخضراء، مما يشير إلى أنه تم تهيئته بشكل صحيح.

### الخطوة 10: إنشاء اسم مستعار لقاعدة بياناتك
- انتقل إلى علامة التبويب **Aliases** على الجانب الأيسر من Squirrel SQL Client.
- انقر على الزر **"+"** لفتح مربع الحوار "Add Alias".

### الخطوة 11: تهيئة الاسم المستعار
- في مربع الحوار "Add Alias":
  - **Name**: أدخل اسمًا لهذا الاتصال (مثلًا، "My Oracle DB").
  - **Driver**: حدد **Oracle Thin Driver** من قائمة التنزيل.
  - **URL**: أدخل URL الاتصال لقاعدة بيانات Oracle الخاصة بك:
    - عبر SID: `jdbc:oracle:thin:@HOST[:PORT]:DB`
    - عبر اسم الخدمة: `jdbc:oracle:thin:@//HOST[:PORT]/DB`
    - استبدل `HOST` (مثلًا localhost أو عنوان IP) و `PORT` (الافتراضي هو 1521) و `DB` (SID أو اسم الخدمة) ببيانات قاعدة البيانات الخاصة بك.
  - **Username**: أدخل اسم المستخدم لقاعدة البيانات.
  - **Password**: أدخل كلمة مرور قاعدة البيانات.

### الخطوة 12: اختبار الاتصال
- انقر على الزر **Test** في مربع الحوار "Add Alias" لتأكيد أن Squirrel SQL Client يمكن أن يتصل بقاعدة بيانات Oracle باستخدام التفاصيل المقدمة. إذا كان النجاح، ستشاهد رسالة تأكيد.

### الخطوة 13: حفظ الاسم المستعار
- إذا نجح اختبار الاتصال، انقر على **OK** لحفظ الاسم المستعار. سيظهر الاسم المستعار الجديد الآن في علامة التبويب Aliases.

### الخطوة 14: الاتصال بقاعدة البيانات
- في علامة التبويب Aliases، حدد الاسم المستعار الذي تم إنشاؤه حديثًا (مثلًا، "My Oracle DB").
- انقر على الزر **Connect** لإقامة اتصال بقاعدة بيانات Oracle. بعد الاتصال، يمكنك البدء في استعلام قاعدة البيانات باستخدام Squirrel SQL Client.

### ملخص للمعلومات الرئيسية
- **الملف المطلوب**: `ojdbc.jar`
- **فئة السائق Java**: `oracle.jdbc.OracleDriver`
- **أنموذج URL**:
  - SID: `jdbc:oracle:thin:@HOST[:PORT]:DB`
  - اسم الخدمة: `jdbc:oracle:thin:@//HOST[:PORT]/DB`

بمتابعة هذه الخطوات، ستتمكن من إضافة سائق Oracle JDBC إلى Squirrel SQL Client وتهيئة اتصال بقاعدة بيانات Oracle. تأكد من دقة جميع التفاصيل (مسار الملف، URL، بيانات الاعتماد) لتجنب مشاكل الاتصال.
