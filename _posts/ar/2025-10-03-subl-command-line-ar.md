### استخدام أمر `subl` في Sublime Text

أخبار رائعة — إن واجهة سطر الأوامر `subl` الخاصة بك مثبتة بالفعل وتعمل (كما يتضح من ناتج `--help`)! هذا هو الإصدار 4200 من Sublime Text (إصدار حديث)، و `subl` هي الواجهة الرسمية لسطر الأوامر لفتح الملفات والمجلدات والمزيد. إنها متعددة الاستخدامات بشكل كبير للمطورين والمستخدمين المتقدمين.

سأشرحها مع أمثلة عملية. قم بتشغيل هذه من أي دليل في طرفيتك. الصيغة الأساسية هي:
```
subl [arguments] [files or directories]
```

#### 1. **الاستخدام الأساسي: فتح الملفات أو المجلدات**
- **افتح الدليل الحالي** (يقوم بتحميله كمشروع/مجلد في Sublime):
  ```
  subl .
  ```
  - هذا يفتح نافذة جديدة بمحتويات المجلد الحالي.

- **افتح ملفًا محددًا**:
  ```
  subl myfile.txt
  ```
  - يفتح `myfile.txt` في النافذة الافتراضية (أو نافذة جديدة إذا أردت).

- **افتح ملفات أو مجلدات متعددة**:
  ```
  subl file1.txt file2.js ~/Documents/myproject/
  ```
  - يفتحهم جميعًا في Sublime.

- **افتح عند سطر/عمود محدد** (مفيد للانتقال إلى الأخطاء):
  ```
  subl myfile.py:42          # يفتح عند السطر 42
  subl myfile.py:42:5        # يفتح عند السطر 42، العمود 5
  ```

#### 2. **الوسائط الشائعة (من تعليمات المساعدة)**
إليك أكثر الوسائط فائدة مع أمثلة. يمكنك دمجها حسب الحاجة (مثال: `subl -n file.txt`).

- **`-n` أو `--new-window`**: افتح دائمًا في نافذة جديدة.
  ```
  subl -n myfile.txt
  ```
  - مفيد إذا أردت إبقاء جلسات Sublime الحالية منفصلة.

- **`-a` أو `--add`**: أضف ملفات/مجلدات إلى نافذة Sublime *الحالية* (إذا كانت مفتوحة بالفعل).
  ```
  subl -a newfolder/
  ```
  - هذا لا ينشئ نافذة جديدة — رائع لبناء مساحة عمل.

- **`-w` أو `--wait`**: انتظر حتى تغلق الملف(ات) في Sublime قبل أن تنتهي أمر الطرفية.
  ```
  subl -w myfile.txt
  ```
  - مفيد في النصوص البرمجية (مثال: بعد تشغيل build، افتح وانتظر المراجعة). يُفترض ضمناً عند القراءة من stdin.

- **`-b` أو `--background`**: افتح دون جلب Sublime إلى المقدمة (يبقي تركيزك على الطرفية).
  ```
  subl -b myfile.txt
  ```

- **`-s` أو `--stay`**: أبقِ Sublime في التركيز بعد إغلاق الملف (ذو صلة فقط مع `-w`).
  ```
  subl -w -s myfile.txt
  ```
  - يمنع العودة التلقائية إلى الطرفية.

- **`--project <project>`**: افتح ملف مشروع محدد في Sublime (`.sublime-project`).
  ```
  subl --project MyProject.sublime-project
  ```
  - تحفظ المشاريع مساحات العمل، الإعدادات، إلخ. أنشئ واحدًا عبر File > Save Project في Sublime.

- **`--command <command>`**: شغّل أمرًا في Sublime (مثال: إجراء plugin) دون فتح ملفات.
  ```
  subl --command "build"    # يشغل أمر build إذا كان لديك نظام build مُعد
  ```
  - تحقق من وحدة التحكم في Sublime (View > Show Console) للأوامر المتاحة.

- **`--launch-or-new-window`**: افتح نافذة جديدة فقط إذا لم يكن Sublime يعمل بالفعل.
  ```
  subl --launch-or-new-window .
  ```
  - فعال للفحص السريع دون تشويش شاشتك.

- **دعم Stdin** (تحرير الإدخال الموجه):
  ```
  echo "Hello World" | subl -   # يفتح stdin في Sublime للتحرير
  subl - > output.txt          # حرر stdin واحفظ التغييرات إلى output.txt
  ```
  - بعد التحرير في Sublime، احفظ وأغلق للمتابعة.

- **`--safe-mode`**: شغّل Sublime بدون plugins أو إعدادات مخصصة (لاستكشاف الأخطاء وإصلاحها).
  ```
  subl --safe-mode myfile.txt
  ```

- **وسائط أخرى**:
  - `-h` أو `--help`: يعرض تعليمات المساعدة التي رأيتها بالفعل.
  - `-v` أو `--version`: يطبع الإصدار (مثال: `subl -v`).

#### 3. **نصائح للاستخدام اليومي**
- **الملفات التي تبدأ بـ `-`**: إذا كان اسم ملف يشبه وسيطًا (مثال: `-config`)، استخدم `--` للفصل:
  ```
  subl -- -config.json
  ```
- **الاختصارات للراحة**: أضف اختصارات إلى ملف تعريف shell الخاص بك (`~/.zshrc` لـ zsh، أو `~/.bash_profile` لـ bash):
  ```
  alias st='subl'     # اختصار أقصر
  alias stt='subl -n' # للنوافذ/علامات التبويب الجديدة
  ```
  أعد التحميل باستخدام `source ~/.zshrc`.
- **في النصوص البرمجية/GIT**: استخدمه في hooks أو اختصارات، مثال: `git diff | subl -` لمراجعة التغييرات.
- **مقارنة بـ `open`**: الآن بعد إعداد `subl`، فهو أفضل من `open -a "Sublime Text"` لأنه يدupport الأسطر والمشاريع والميزات المتقدمة الأخرى.
- **استكشاف الأخطاء وإصلاحها**:
  - إذا توقف `subl` عن العمل، أعد تشغيل تثبيت symlink من ردي السابق.
  - للمشاريع الكبيرة، استخدم `-a` لتجنب نوافذ متعددة.
  - على macOS، إذا ظهرت مشاكل في التركيز، جرب أولاً بدون `-b`.

هذا يغطي 90% مما ستحتاجه! ابدأ بـ `subl .` في مجلد مشروع وجرّب. إذا كان لديك تدفق عمل محدد أو خطأ، شارك المزيد من التفاصيل للحصول على نصيحة مخصصة.
