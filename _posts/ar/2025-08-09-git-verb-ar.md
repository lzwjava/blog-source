لتمكين التسجيل المفصل في Git، يمكنك استخدام أعلام أو متغيرات بيئة محددة بناءً على الأمر ومستوى التفاصيل الذي تحتاجه. إليك الأساليب الرئيسية:

1. **استخدام علم `--verbose` أو `-v`**:
   تدعم العديد من أوامر Git علم `--verbose` (أو `-v`) لتقديم مخرجات أكثر تفصيلاً. على سبيل المثال:
   ```bash
   git commit --verbose
   ```
   يعرض هذا diff للتغييرات التي يتم إيداعها بالإضافة إلى المخرجات المعتادة.

2. **تمكين الوضع المفصل لأوامر محددة**:
   - بالنسبة لـ `git push`:
     ```bash
     git push --verbose
     ```
     يوفر هذا تفاصيل حول عملية الدفع، بما في ذلك المراجع التي يتم دفعها واتصال الخادم.
   - بالنسبة لـ `git fetch`:
     ```bash
     git fetch --verbose
     ```
     يعرض هذا معلومات مفصلة حول ما يتم جلبها.
   - بالنسبة لـ `git clone`:
     ```bash
     git clone --verbose <repository>
     ```
     يعرض هذا التقدم وتفاصيل إضافية أثناء الاستنساخ.

3. **تعيين متغير البيئة `GIT_TRACE`**:
   لتصحيح الأخطاء بشكل أعمق، يمكنك تمكين تتبع مخرجات Git لرؤية التفاصيل منخفضة المستوى لعمليات Git:
   ```bash
   GIT_TRACE=1 git <command>
   ```
   تنتج هذا سجلات مفصلة حول العمليات الداخلية لـ Git، مثل تنفيذ الأمر واستدعاءات النظام.

4. **تمكين مستويات تتبع محددة**:
   يمكنك استخدام متغيرات بيئة أكثر دقة لجوانب محددة من Git:
   - `GIT_TRACE_CURL`: يمكّن التسجيل المفصل لعمليات HTTP (مفيد للتفاعلات عن بُعد):
     ```bash
     GIT_TRACE_CURL=1 git <command>
     ```
   - `GIT_TRACE_PACKET`: يُظهر تفاصيل على مستوى الحزمة لبروتوكول Git:
     ```bash
     GIT_TRACE_PACKET=1 git <command>
     ```
   - `GIT_TRACE_PERFORMANCE`: يسجل بيانات الأداء:
     ```bash
     GIT_TRACE_PERFORMANCE=1 git <command>
     ```

5. **زيادة التفصيل لأوامر متعلقة بـ diff**:
   لأوامر مثل `git diff` أو `git log`، يمكنك استخدام خيارات للحصول على مزيد من التفاصيل:
   - `git diff --stat` يُظهر ملخصًا للتغييرات.
   - `git log --verbose` أو `git log -p` يتضمن diff الكامل لكل commit.

6. **التكوين العام للمخرجات المفصلة**:
   يمكنك تكوين Git ليكون دائمًا مفصلًا لأوامر معينة عن طريق تعيين أسماء مستعارة أو استخدام خيارات التكوين. على سبيل المثال:
   ```bash
   git config --global alias.logv "log --verbose"
   ```
   بعد ذلك، سيؤدي تشغيل `git logv` دائمًا إلى تضمين مخرجات مفصلة.

**مثال**:
لرؤية مخرجات مفصلة لعملية دفع مع التتبع الكامل:
```bash
GIT_TRACE=1 git push --verbose
```

**ملاحظات**:
- يمكن أن تكون المخرجات المفصلة ساحقة، خاصة مع `GIT_TRACE`، لذا استخدمها عند تصحيح أخطاء مشكلات محددة.
- تحقق من صفحات الدليل لأوامر محددة (مثل `man git-push`) لمعرفة ما إذا كانت تدعم `--verbose` أو أعلام تصحيح أخرى.
- على Windows، استخدم نفس صيغة متغير البيئة في موجه الأوامر (`set GIT_TRACE=1`) أو PowerShell (`$env:GIT_TRACE=1`).

إذا كنت بحاجة إلى تسجيل مفصل لأمر Git معين أو سيناريو محدد، دعني أعلم، ويمكنني تقديم مثال مخصص!
