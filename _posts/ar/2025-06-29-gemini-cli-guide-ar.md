فيما يلي دليل موجز لتثبيت وإعداد واستخدام **Google Gemini CLI**، وهو وكيل ذكي مفتوح المصدر يدمج إمكانيات Gemini في طرفيتك للبرمجة وأتمتة المهام والمزيد. يستند هذا الدليل إلى أحدث المعلومات المتاحة حتى 29 يونيو 2025.

---

## **دليل Google Gemini CLI**

### **ما هو Gemini CLI؟**
Gemini CLI هو أداة سطر أوامر مفتوحة المصدر طورتها Google، وتجلب قوة نموذج Gemini 2.5 Pro (بسعة سياق تصل إلى 1 مليون رمز) إلى طرفيتك. يدعم البرمجة، التصحيح، إنشاء المحتوى، أتمتة المهام، والمهام متعددة الوسائط مثل إنشاء الصور والفيديو. إنه مجاني للاستخدام مع حساب Google ويمكن دمجه مع أدوات مثل بحث Google وخوادم Model Context Protocol (MCP).

---

### **المتطلبات الأساسية**
- **Node.js**: الإصدار 18 أو أعلى. تحقق من ذلك باستخدام الأمر `node -v`. قم بالتثبيت من [nodejs.org](https://nodejs.org) إذا لزم الأمر.
- **حساب Google**: مطلوب للوصول المجاني إلى Gemini 2.5 Pro (60 طلبًا/دقيقة، 1000 طلب/يوم).
- (اختياري) **مفتاح API**: للحصول على حدود أعلى أو نماذج محددة، قم بإنشائه من [Google AI Studio](https://aistudio.google.com).
- (اختياري) **Docker**: لدمج خوادم MCP (مثل أدوات GitHub).

---

### **التثبيت**
هناك طريقتان للبدء مع Gemini CLI:

1. **التثبيت العام**:
   ```bash
   npm install -g @google/gemini-cli
   gemini
   ```
   يقوم هذا بتثبيت CLI بشكل عام وتشغيله باستخدام الأمر `gemini`.

2. **التشغيل بدون تثبيت**:
   ```bash
   npx https://github.com/google-gemini/gemini-cli
   ```
   يقوم هذا بتشغيل CLI مباشرة بدون تثبيت، وهو مثالي للاختبار.

---

### **الإعداد**
1. **تشغيل CLI**:
   - قم بتشغيل `gemini` في طرفيتك.
   - في أول تشغيل، اختر سمة (مثل ASCII، dark، light) واضغط على Enter.

2. **المصادقة**:
   - اختر **Login with Google** للوصول المجاني (موصى به لمعظم المستخدمين).
   - ستفتح نافذة متصفح؛ قم بتسجيل الدخول باستخدام حساب Google الخاص بك.
   - بدلاً من ذلك، استخدم مفتاح API:
     - قم بإنشاء مفتاح من [Google AI Studio](https://aistudio.google.com).
     - عيّنه كمتغير بيئة:
       ```bash
       export GEMINI_API_KEY=YOUR_API_KEY
       ```
     - هذا مفيد للحدود الأعلى أو الاستخدام المؤسسي.

3. **الانتقال إلى مشروعك**:
   - قم بتشغيل `gemini` في الدليل الجذري لمشروعك لتوفير سياق للمهام المتعلقة بالبرمجة.

### **الاستخدام الأساسي**
يعمل Gemini CLI في بيئة تفاعلية (REPL). اكتب الأوامر أو مطالبات اللغة الطبيعية للتفاعل مع نموذج Gemini. فيما يلي بعض المهام الشائعة:

1. **شرح الكود**:
   - انتقل إلى مجلد المشروع وشغّل:
     ```bash
     gemini
     ```
   - المطالبة: `Explain the architecture of this project` أو `Describe the main function in main.py`.
   - يقرأ CLI الملفات ويقدم شرحًا منظمًا.

2. **إنشاء الكود**:
   - المطالبة: `Create a simple to-do app in HTML, CSS, and JavaScript`.
   - يولد CLI الكود ويمكنه حفظه في ملفات إذا طُلب.

3. **التصحيح**:
   - الصق رسالة خطأ أو أثر المكدس واسأل: `What’s causing this error?`.
   - يحلل CLI الخطأ ويقترح إصلاحات، مستخدمًا بحث Google للحصول على سياق إضافي.

4. **إدارة الملفات**:
   - المطالبة: `Organize my PDF invoices by month of expenditure`.
   - يمكن لـ CLI معالجة الملفات أو تحويل التنسيقات (مثل الصور إلى PNG).

5. **دمج GitHub**:
   - استخدم خوادم MCP لمهام GitHub (مثل سرد المشكلات):
     - قم بتكوين خادم MCP لـ GitHub في `.gemini/settings.json` باستخدام رمز وصول شخصي (PAT).
     - المطالبة: `Get all open issues assigned to me in the foo/bar repo`.
   - شغّل `/mcp` لسرد خوادم وأدوات MCP المُهيأة.

6. **المهام متعددة الوسائط**:
   - أنشئ وسائط باستخدام أدوات مثل Imagen أو Veo:
     - المطالبة: `Create a short video of a cat’s adventures in Australia using Veo`.

### **الميزات الرئيسية**
- **ملفات السياق (GEMINI.md)**: أضف ملف `GEMINI.md` في الدليل الجذري لمشروعك لتعريف أنماط البرمجة، قواعد المشروع، أو التفضيلات (مثل "Use async/await for JavaScript"). يستخدم CLI هذا للحصول على استجابات مخصصة.
- **الأدوات المدمجة**:
   - `/tools`: أدرج الأدوات المتاحة (مثل بحث Google، عمليات الملفات).
   - `/compress`: لخّص سياق المحادثة لتوفير الرموز.
   - `/bug`: أبلغ عن مشكلات مباشرة إلى مستودع Gemini CLI على GitHub.
- **الوضع غير التفاعلي**: للبرمجة النصية، استخدم piping:
   ```bash
   echo "Write a Python script" | gemini
   ```
- **ذاكرة المحادثة**: احفظ سجل الجلسة باستخدام `/save <tag>` واستأنفها باستخدام `/restore <tag>`.
- **تكوين مخصص**:
   - حرر `~/.gemini/settings.json` للإعدادات العامة أو `.gemini/settings.json` في المشروع للإعدادات المحلية.
   - مثال: عيّن خوادم MCP أو سمات مخصصة.

### **نصائح وحيل**
- **ابدأ بخطط**: للمهام المعقدة، اطلب خطة أولاً: `Create a detailed implementation plan for a login system`. هذا يضمن مخرجات منظمة.
- **استخدم السياق المحلي**: ضمّن تفاصيل المشروع المحددة في `GEMINI.md` بدلاً من الاعتماد على خوادم MCP للحصول على استجابات أسرع وأكثر موثوقية.
- **التصحيح**: فعّل التسجيل المفصل باستخدام `DEBUG=true gemini` للحصول على معلومات مفصلة عن الطلبات والاستجابات.
- **راجع التغييرات**: دائمًا راجع التعديلات على الملفات أو الأوامر قبل الموافقة (اكتب `y` للتأكيد).
- **استكشف الأدوات**: شغّل `/tools` لاكتشاف الإمكانيات المدمجة مثل البحث على الويب أو حفظ الذاكرة.

### **استكشاف الأخطاء وإصلاحها**
- **مشكلات المصادقة**: تأكد من أن حساب Google أو مفتاح API الخاص بك صالح. استخدم `/auth` لتبديل الطرق.
- **حدود المعدل**: الطبقة المجانية تسمح بـ 60 طلب/دقيقة و 1000/يوم. للحدود الأعلى، استخدم مفتاح API أو Vertex AI.
- **الأخطاء**: تحقق من [دليل استكشاف الأخطاء](https://github.com/google-gemini/gemini-cli/docs/troubleshooting.md) على GitHub.
- **الاستجابات البطيئة**: CLI في مرحلة المعاينة وقد يكون بطيئًا مع استدعاءات API. قدم ملاحظات على GitHub.

### **الاستخدام المتقدم**
- **دمج خادم MCP**:
  - أنشئ خادم MCP لـ GitHub للتفاعل مع المستودعات:
    - أنشئ رمز وصول شخصي (PAT) لـ GitHub مع النطاقات اللازمة.
    - أضفه إلى `.gemini/settings.json`:
      ```json
      {
        "mcpServers": [
          {
            "name": "github",
            "url": "http://localhost:8080",
            "type": "github"
          }
        ]
      }
      ```
    - شغّل حاوية Docker لخادم MCP (انظر وثائق GitHub).
- **البرمجة النصية**: أتمت المهام عن طريق دمج Gemini CLI في البرامج النصية:
  ```bash
  gemini --non-interactive "Generate a bash script to backup my files"
  ```
- **المطالبات متعددة الوسائط**:
  - مثال: `Describe this image: path/to/image.jpg` (يتطلب نموذجًا يدعم الرؤية مثل `gemini-pro-vision`).

### **القيود**
- **مرحلة المعاينة**: Gemini CLI في مرحلة ما قبل GA، مع إمكانية وجود دعم محدود أو أخطاء.
- **ليس مفتوح المصدر بالكامل**: فقط واجهة المستخدم لـ CLI مرخصة بترخيص Apache 2.0؛ نموذج Gemini هو ملكية خاصة.
- **مشاركة الحصة**: يتم مشاركة الحدود مع Gemini Code Assist إذا تم استخدامه.
- **التسعير المستقبلي**: تسعير ما بعد مرحلة المعاينة غير واضح؛ قد تتطلب الميزات المتقدمة الدفع.
