يبدو أن `apt` لا يستطيع العثور على حزمة باسم `outline-client`. هذا يعني عادة أحد أمرين:

1.  **لم تقم بتثبيت Outline باستخدام `apt` (أي لم يتم تثبيته كحزمة .deb).** هذا هو السبب الأكثر شيوعاً لهذا الخطأ مع Outline، حيث أن العديد من المستخدمين يقومون بتثبيته عبر AppImage.
2.  **اسم الحزمة مختلف، أو أن المستودع غير مضبوط.** على الرغم من أنه احتمال ضعيف بالنسبة لـ Outline، إلا أنه احتمال وارد للبرامج الأخرى.

نظراً لمحاولتك السابقة، من المحتمل جداً أنك قمت بتثبيته كـ **AppImage**. في هذه الحالة، ستحتاج إلى إزالته يدوياً.

إليك كيفية المتابعة:

**1. تحديد موقع وحذف ملف AppImage:**

عميل Outline، عند تثبيته كـ AppImage، يكون عادةً ملفاً تنفيذياً واحداً. من المرجح أنك قمت بتنزيله في مجلد `Downloads` الخاص بك، أو ربما نقلته إلى مجلد `bin` أو `Applications` في دليل المنزل الخاص بك.

*   **المواقع الشائعة للتحقق منها:**
    *   `~/Downloads/`
    *   `~/Applications/` (إذا قمت بإنشائه)
    *   `~/bin/` (إذا قمت بإنشائه)
    *   الدليل الذي قمت بتشغيله منه في الأصل.

*   **كية العثور عليه:** يمكنك استخدام أمر `find` للبحث عنه في دليل المنزل الخاص بك:
    ```bash
    find ~ -name "Outline-Client*.AppImage" 2>/dev/null
    ```
    سيبحث هذا الأمر في دليل المنزل الخاص بك عن أي ملف يبدأ بـ "Outline-Client" وينتهي بـ ".AppImage".
    بمجرد العثور عليه، قم بحذفه:
    ```bash
    rm /path/to/Outline-Client.AppImage
    ```
    (استبدل `/path/to/` بالمسار الفعلي الذي وجده أمر `find`.)

**2. إزالة ملفات التكوين وبدء التشغيل التلقائي المتبقية:**

حتى إذا قمت بحذف ملف AppImage، فقد يكون Outline قد أنشأ بعض ملفات التكوين وإدخال بدء تشغيل تلقائي لنفسه.

*   **إزالة دليل التكوين:**
    ```bash
    rm -rf ~/.config/Outline/
    ```
    يزيل هذا الأمر مجلد `Outline` بالكامل داخل مجلد التكوين الخاص بالمستخدم.

*   **إزالة إدخال بدء التشغيل التلقائي (إذا تم إنشاؤه):**
    إذا تم ضبط Outline ليبدأ تلقائياً عند تسجيل الدخول، فقد يكون هناك ملف `.desktop` خاص به.
    ```bash
    rm -f ~/.config/autostart/Outline-Client.AppImage.desktop
    ```

*   **إزالة أيقونات سطح المكتب (إن وجدت):**
    في بعض الأحيان تقوم ملفات AppImage بإنشاء أيقونة على سطح المكتب في دليل الأيقونات المحلي الخاص بك.
    ```bash
    find ~/.local/share/icons -name "appimagekit-outline-client*.png" -delete
    ```
    سيؤدي هذا إلى حذف أي أيقونات PNG مرتبطة بـ Outline AppImage.

**3. التحقق من تكوين الشبكة واستعادته (إذا لزم الأمر):**

يمكن لـ Outline، مثل برامج VPN الأخرى، تعديل إعدادات DNS الخاصة بك. إذا واجهت مشاكل في اتصال الإنترنت بعد إزالة ملف AppImage، تحقق من ملف `/etc/resolv.conf` الخاص بك.

*   **فحص `resolv.conf`:**
    ```bash
    cat /etc/resolv.conf
    ```
    ابحث عن أسطر تبدو كما لو تمت إضافتها بواسطة Outline (على سبيل المثال، عناوين IP محددة لخوادم الأسماء التي لا تعرفها كخوادم DNS الخاصة بالموجه أو مزود خدمة الإنترنت الخاص بك).

*   **استعادة `resolv.conf` (إذا دعت الحاجة):**
    إذا بدا أن Outline قام بتعديله:
    *   **الخيار أ (موصى به لأوبونتو الحديثة):** إذا كان نظامك يستخدم `systemd-resolved` (وهو شائع في إصدارات أوبونتو الحديثة)، يمكنك عادة إصلاح هذا عن طريق إعادة إنشاء الرابط الرمزي:
        ```bash
        sudo rm /etc/resolv.conf
        sudo ln -s /run/systemd/resolve/stub-resolv.conf /etc/resolv.conf
        ```
    *   **الخيار ب (التعديل اليدوي - استخدمه إذا لم يعمل الخيار أ أو للأنظمة القديمة):**
        أولاً، قم بعمل نسخة احتياطية:
        ```bash
        sudo cp /etc/resolv.conf /etc/resolv.conf.bak
        ```
        ثم، قم بتحرير الملف:
        ```bash
        sudo nano /etc/resolv.conf
        ```
        قم بتعليق أو إزالة أي أسطر `nameserver` تمت إضافتها بواسطة Outline. يمكنك عادةً وضع خوادم DNS عامة مثل خوادم جوجل (`8.8.8.8` و `8.8.4.4`) أو خوادم Cloudflare (`1.1.1.1` و `1.0.0.1`) إذا كنت غير متأكد من خوادم DNS الخاصة بمزود خدمة الإنترنت الخاص بك.

**4. إعادة تشغيل النظام:**

بعد إكمال هذه الخطوات، يوصى بشدة بإعادة تشغيل نظام أوبونتو الخاص بك لضمان تفعيل جميع التغييرات وإنهاء جميع العمليات الخلفية المتعلقة بـ Outline بشكل كامل.

```bash
sudo reboot
```

باتباع خطوات الإزالة اليدوية هذه، يجب أن تكون قادراً على التخلص تماماً من عميل Outline VPN من نظام أوبونتو الخاص بك.
