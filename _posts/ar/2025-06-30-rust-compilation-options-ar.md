أنت تطرح سؤالاً رائعاً يتعلق بكيفية تنظيم وبناء مشاريع Rust عادةً!

عند استخدام `rustc` مباشرةً، لديك خياران رئيسيان لتحديد موقع الإخراج:

1.  **`-o <اسم_ملف_الإخراج>`**: يحدد هذا الخيار *المسار الدقيق واسم الملف* للتنفيذي أو المكتبة المترجمة.

    ```bash
    rustc scripts/rust/example.rs -o test/example
    ```

    سيقوم هذا بترجمة `example.rs` ووضع الملف التنفيذي باسم `example` مباشرة في مجلد `test/`. إذا لم يكن `test/` موجوداً، فإن `rustc` سينشئه.

2.  **`--out-dir <مجلد_الإخراج>`**: يحدد هذا الخيار *المجلد* الذي يجب على `rustc` وضع جميع ملفات الإخراج المُنشأة فيه (الملف التنفيذي/المكتبة، بالإضافة إلى أي ملفات وسيطة مثل `.d`، `.rmeta`، إلخ).

    ```bash
    rustc scripts/rust/example.rs --out-dir test/
    ```

    في هذه الحالة، سيقوم `rustc` بترجمة `example.rs` ووضع الملف التنفيذي (الذي سيُسمى `example` افتراضياً، مُشتقاً من اسم الملف المصدر) داخل مجلد `test/`.

**التوصية: استخدام Cargo للمشاريع**

بينما يمنحك `rustc` المباشر تحكماً دقيقاً، بالنسبة لمعظم مشاريع Rust، سترغب في استخدام **Cargo**. Cargo هو نظام بناء Rust ومدير الحزم، وهو يبسط إدارة التبعيات والبناء واختبار وتشغيل الكود الخاص بك.

مع Cargo، يتم عادةً وضع الملفات الثنائية الناتجة في مجلد `target` داخل جذر مشروعك. على سبيل المثال، إذا كان لديك مشروع مُعد باستخدام Cargo:

```
my_project/
├── Cargo.toml
└── src/
    └── main.rs
```

عند تشغيل `cargo build` أو `cargo run`، سيتم وضع الملف التنفيذي المترجم في `my_project/target/debug/my_project` (لنسخ التصحيح) أو `my_project/target/release/my_project` (لنسخ الإصدار).

إذا كنت تريد التحكم في مجلد `target` لـ Cargo، يمكنك:

  * **متغير البيئة `CARGO_TARGET_DIR`**:

    ```bash
    CARGO_TARGET_DIR=./test cargo build
    ```

    سيضع هذا مجلد `target` بالكامل (الذي يحتوي على `debug`، `release`، إلخ) داخل مجلد `test/` الخاص بك. سيكون الملف التنفيذي بعد ذلك في `test/debug/my_project`.

  * **خيار سطر الأوامر `--target-dir` لـ Cargo**:

    ```bash
    cargo build --target-dir test/
    ```

    هذا يفعل نفس الشيء مثل متغير البيئة.

  * **`.cargo/config.toml`**: يمكنك تعيين `target-dir` افتراضي لجميع مشاريع Cargo على جهازك أو لمساحة عمل محددة عن طريق إضافته إلى ملف `.cargo/config.toml`.

**لماذا تستخدم Cargo؟**

  * **تخطيط مشروع موحد:** يفرض Cargo هيكل مشروع مشترك، مما يسهل على مطوري Rust الآخرين فهم الكود الخاص بك.
  * **إدارة التبعيات:** يتعامل Cargo مع تنزيل وترجمة تبعيات مشروعك تلقائياً.
  * **الاختبار والمقارنة المعيارية:** يحتوي Cargo على أوامر مدمجة لتشغيل الاختبارات (`cargo test`) والمقاييس (`cargo bench`).
  * **إدارة الإصدارات:** يساعد Cargo في إنشاء نسخ الإصدار (`cargo build --release`) ونشر الصنفات إلى crates.io.

ما لم يكن لديك سبب محدد جداً لعدم القيام بذلك، يوصى بشدة بتنظيم كود Rust الخاص بك في مشروع Cargo.

**بالنظر إلى ناتج `git status` الخاص بك:**

يظهر ناتج `git status` الخاص بك أن `example` و `scripts/rust/` غير مُتتبعين. هذا يشير إلى أنك قد تقوم بترجمة `scripts/rust/example.rs` مباشرة باستخدام `rustc` خارج نطاق مشروع Cargo.

إذا كان `example` هو الملف الثنائي المترجم، فإن أمر `rustc scripts/rust/example.rs` السابق وضعه في المجلد الحالي (`lzwjava.github.io`).

لتحقيق هدفك بوضعه في `test/example` (بافتراض أن `test` هو مجلد في جذر `lzwjava.github.io`)، ستستخدم:

```bash
rustc scripts/rust/example.rs -o test/example
```

هذه هي الطريقة الأكثر مباشرة للحصول على مسار الإخراج المحدد الذي طلبته باستخدام `rustc`.
