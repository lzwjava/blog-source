JShell هو أداة REPL تفاعلية (حلقة القراءة-التقييم-الطباعة) مدمجة مع حزمة تطوير Java (JDK) بدءًا من JDK 9. تتيح لك تنفيذ مقاطع كود Java بشكل تفاعلي دون الحاجة لكتابة برنامج كامل. فيما يلي دليل موجز حول استخدام JShell:

### 1. **بدء JShell**
- **تأكد من تثبيت JDK**: تأكد من وجود JDK 9 أو إصدار أحدث. يمكنك التحقق عن طريق تشغيل `java -version` في الطرفية أو موجه الأوامر.
- **تشغيل JShell**:
  - افتح الطرفية أو موجه الأوامر.
  - اكتب `jshell` ثم اضغط Enter.
  - ستظهر رسالة مثل `jshell>` تشير إلى أن JShell جاهز للاستخدام.

### 2. **الاستخدام الأساسي**
- **تشغيل كود Java**:
  - اكتب التعبيرات أو الجمل أو التصريحات البرمجية مباشرة عند المطالبة.
  - مثال:
    ```java
    jshell> System.out.println("Hello, JShell!")
    Hello, JShell!
    ```
  - يقوم JShell بتقييم المدخلات وعرض النتيجة فورًا.

- **المتغيرات والتعبيرات**:
  - عرّف المتغيرات أو قم بتقييم التعبيرات:
    ```java
    jshell> int x = 10
    x ==> 10
    jshell> x * 2
    $2 ==> 20
    ```
  - يقوم JShell تلقائيًا بتعيين أسماء مؤقتة (مثل `$2`) لنتائج التعبيرات.

- **تحديد الدوال والفئات**:
  - يمكنك تحديد الدوال أو الفئات:
    ```java
    jshell> void sayHello() { System.out.println("Hello!"); }
    |  created method sayHello()
    jshell> sayHello()
    Hello!
    ```
    ```java
    jshell> class Test { int x = 5; }
    |  created class Test
    jshell> Test t = new Test()
    t ==> Test@7c417213
    jshell> t.x
    $5 ==> 5
    ```

### 3. **الأوامر الرئيسية**
يوفر JShell أوامر مدمجة تبدأ ب `/`. إليك بعض الأوامر الأساسية:
- **عرض كل الأكواد**: `/list` – يعرض جميع المقاطع المدخلة في الجلسة.
  ```java
  jshell> /list
  ```
- **تحرير الكود**: `/edit <id>` – يفتح محرر واجهة رسومية للمقطع ذو المعرف المحدد (من `/list`).
- **حفظ الجلسة**: `/save myfile.java` – يحفظ جميع المقاطع في ملف.
- **تحميل ملف**: `/open myfile.java` – يحمل وينفذ الكود من ملف.
- **عرض المتغيرات**: `/vars` – يعرض جميع المتغيرات المعلنة.
  ```java
  jshell> /vars
  |    int x = 10
  ```
- **عرض الدوال**: `/methods` – يعرض جميع الدوال المعرفة.
- **خروج من JShell**: `/exit` – يغلق جلسة JShell.
- **المساعدة**: `/help` – يعرض جميع الأوامر المتاحة.

### 4. **استيراد الحزم**
- يقوم JShell تلقائيًا باستيراد الحزم الشائعة (مثل `java.util`، `java.io`). لاستخدام غيرها، قم باستيرادها يدويًا:
  ```java
  jshell> import java.time.LocalDate
  jshell> LocalDate.now()
  $3 ==> 2025-06-27
  ```

### 5. **تحرير وتصحيح الكود**
- **تعديل الكود الموجود**:
  - استخدم `/list` للعثور على معرف المقطع.
  - أعد تعريفه بكتابة نسخة جديدة. يقوم JShell بالكتابة فوق التعريف القديم.
    ```java
    jshell> int x = 5
    x ==> 5
    jshell> int x = 10
    x ==> 10
    ```
- **حذف مقطع**: `/drop <id>` – يزيل مقطعًا محددًا بواسطة معرفه.

### 6. **الإكمال التلقائي**
- اضغط `Tab` للإكمال التلقائي لأسماء الفئات أو الدوال أو الأوامر.
- مثال:
  ```java
  jshell> System.out.pr<tab>
  ```
  سيقترح `println`، `print`، إلخ.

### 7. **تشغيل السكريبتات الخارجية**
- قم بتحميل وتشغيل ملف Java:
  ```java
  jshell> /open MyScript.java
  ```
- يمكنك أيضًا بدء JShell مع سكريبت:
  ```bash
  jshell MyScript.java
  ```

### 8. **تخصيص JShell**
- **سكريبتات البدء**: أنشئ ملفًا (مثل `startup.jsh`) يحتوي على استيرادات أو كود شائع، ثم قم بتحميله باستخدام `/open startup.jsh` أو اضبطه كسكريبت بدء افتراضي:
  ```bash
  jshell --start startup.jsh
  ```
- **أنماط التغذية الراجعة**: اضبط مستوى تفاصيل المخرجات باستخدام `/set feedback <mode>` (مثل `normal`، `concise`، `verbose`).

### 9. **نصائح**
- **النمذجة السريعة**: استخدم JShell لاختبار مقاطع الكود الصغيرة، أو تجربة واجهات برمجة التطبيقات (APIs)، أو تعلم تركيب Java.
- **معالجة الأخطاء**: يوفر JShell رسائل خطأ واضحة ويتيح لك تصحيح الأخطاء على الفور.
- **إدخال متعدد الأسطر**: للكود المعقد، يدعم JShell التحرير متعدد الأسطر. أنهِ السطر بشرطة مائلة للخلف (`\`) للمتابعة:
  ```java
  jshell> int sum(int a, int b) { \
      ...> return a + b; \
      ...> }
  ```

### 10. **استكشاف الأخطاء وإصلاحها**
- **عدم العثور على JShell**: تأكد من أن دليل `bin` الخاص بـ JDK موجود في مسار النظام (PATH).
- **بدء تشغيل بطيء**: في بعض الأنظمة، قد يستغرق JShell لحظة للتهيئة. تأكد من أن إصدار JDK محدث.
- **إعادة تعيين الجلسة**: استخدم `/reset` لمسح جميع المقاطع والبدء من جديد.

لمزيد من التفاصيل، راجع وثائق JDK الرسمية أو شغل `/help` في JShell. إذا كانت لديك مهام محددة أو كود تريد تجربته في JShell، فأخبرني، وسأوجهك أكثر!
