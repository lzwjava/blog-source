إليك نصًا برمجيًا من نوع bash يجمع بين عدة أمثلة لـ `curl`:

```bash
#!/bin/bash

# طلب GET أساسي
echo "طلب GET أساسي:"
curl http://example.com

# طلب GET مع رؤوس مخصصة
echo "\nطلب GET مع رؤوس مخصصة:"
curl -H "User-Agent: MyUserAgent/1.0" http://example.com

# طلب POST
echo "\nطلب POST:"
curl -X POST -d "key1=value1&key2=value2" http://example.com/post

# طلب POST ببيانات JSON
echo "\nطلب POST ببيانات JSON:"
curl -X POST -H "Content-Type: application/json" -d "{\"key1\":\"value1\",\"key2\":\"value2\"}" http://example.com/post

# استخدام وكيل
echo "\nاستخدام وكيل:"
curl -x http://proxy.example.com:8080 http://example.com

# المصادقة
echo "\nالمصادقة الأساسية:"
curl -u username:password http://example.com/auth

# تنزيل ملف
echo "\nتنزيل ملف:"
curl -O http://example.com/file.zip

# اتبع عمليات إعادة التوجيه
echo "\nاتبع عمليات إعادة التوجيه:"
curl -L http://example.com/redirect

# حفظ الناتج في ملف
echo "\nحفظ الناتج في ملف:"
curl http://example.com -o output.txt

# طريقة HTTP مخصصة
echo "\nطريقة HTTP مخصصة (DELETE):"
curl -X DELETE http://example.com/delete/resource

# ناتج مفصل
echo "\nناتج مفصل:"
curl -v http://example.com

# انتهاء مهلة الاتصال
echo "\nانتهاء مهلة الاتصال:"
curl --connect-timeout 5 http://example.com

echo "\nاكتمل تنفيذ النص البرمجي."
```

احفظ هذا النص البرمجي بامتداد `.sh`، مثل `curl_examples.sh`. تأكد من جعله قابلاً للتنفيذ إذا كنت في بيئة شبيهة بـ Unix أو تقوم بتشغيله في نظام Windows الفرعي لنظام Linux (WSL):

```sh
chmod +x curl_examples.sh
```

ثم، يمكنك تشغيله:

```sh
./curl_examples.sh
```

**ملاحظات لنظام Windows:**
- إذا كنت تشغل هذا في موجه أوامر Windows أو PowerShell، فستحتاج إلى استخدام `bash` أو `wsl` إذا كان لديك WSL مثبتًا، أو تأكد من أن `curl` موجود في مسار PATH للتنفيذ المباشر:
  - `bash curl_examples.sh` أو `wsl ./curl_examples.sh`
- تذكر أن هذا النص البرمجي يستخدم صيغة bash، لذا فهو غير متوافق أصلاً مع موجه أوامر Windows أو PowerShell ما لم يتم تشغيله عبر WSL أو بيئة bash. ومع ذلك، فإن أوامر `curl` نفسها متوافقة مع إصدار Windows من `curl`.
