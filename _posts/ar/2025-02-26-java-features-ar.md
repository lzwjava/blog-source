## 1. **الصفوف**
- **ما هو**: الصف هو الوحدة الأساسية لبناء Java، ويعمل كقالب أو مخطط لإنشاء الكائنات. يضم البيانات (الحقول) والسلوك (الطرق) التي تحدد خصائص الكائنات وأفعالها.
- **لماذا هو مهم**: الصفوف هي حجر الزاوية في نموذج برمجة Java الموجهة للأشياء (OOP). تروج للتكثيف (تجميع البيانات والطرق)، والتجريد (إخفاء تفاصيل التنفيذ)، والتقسيم، مما يجعل الكود قابل لإعادة الاستخدام وسهل الصيانة.
- **كيف يتم استخدامه**: الصفوف تنمذج الكيانات في البرنامج مثل `Person`، `Vehicle`، أو `BankAccount`. يمكن أن تشمل مصنعين، وحقول مع محددات الوصول (`public`، `private`)، وطرق لتعديل حالة الكائن.
- **الغوص العميق**:
  - يمكن أن تكون الصفوف متداخلة (الصفوف الداخلية) أو مجردة (لا يمكن أن تكون مستنثنة مباشرة).
  - تدعم الوراثة، مما يسمح لصف أن يمدد آخر ويورث خصائصه وطرقه.
- **مثال**:
  ```java
  public class Student {
      private String name;  // حقل المثال
      private int age;

      // مصنع
      public Student(String name, int age) {
          this.name = name;
          this.age = age;
      }

      // طريقة
      public void displayInfo() {
          System.out.println("Name: " + name + ", Age: " + age);
      }
  }
  ```
- **استخدام حقيقي**: يمكن أن تكون `Student` جزءًا من نظام إدارة المدرسة، مع طرق لحساب الدرجات أو تتبع الحضور.

## 2. **الأشياء**
- **ما هو**: الكائن هو مثال من الصف، يُنشأ باستخدام كلمة `new`. يمثل تحقيقًا محددًا للقالب مع حالة خاصة به.
- **لماذا هو مهم**: الأشياء تجلب الصفوف إلى الحياة، مما يسمح بتعدد الأمثلة مع بيانات فريدة. تسمح بتنمذج الأنظمة المعقدة عن طريق تمثيل الكيانات الحقيقية.
- **كيف يتم استخدامه**: الكائنات تُنشأ وتتم معالجتها عبر طرقها وحقولها. على سبيل المثال، `Student student1 = new Student("Alice", 20);` يخلق كائنًا من `Student`.
- **الغوص العميق**:
  - الكائنات تُخزن في الذاكرة العشوائية، وتخزن مراجع لها في المتغيرات.
  - Java تستخدم مرجعًا عند تمرير الكائنات، مما يعني أن التغييرات في حالة الكائن يتم انعكاسها عبر جميع المراجع.
- **مثال**:
  ```java
  Student student1 = new Student("Alice", 20);
  student1.displayInfo();  // Output: Name: Alice, Age: 20
  ```
- **استخدام حقيقي**: في نظام تجارة إلكترونية، الكائنات مثل `Order` أو `Product` تمثل المشتريات الفردية أو العناصر للبيع.

## 3. **الطرق**
- **ما هو**: الطرق هي كتل من الكود داخل الصف التي تحدد سلوك الكائنات. يمكن أن تأخذ متغيرات، وتعيد القيم، أو تؤدي أفعالًا.
- **لماذا هو مهم**: الطرق تجلب منطقًا، تقليل التكرار، وتحسين قابلية القراءة للكود. هي الطريقة الرئيسية للتفاعل مع حالة الكائن.
- **كيف يتم استخدامه**: الطرق تُستدعي على الكائنات أو بشكل ثابت على الصفوف. كل تطبيق Java يبدأ بالطريقة `public static void main(String[] args)`.
- **الغوص العميق**:
  - الطرق يمكن أن تكون متعدد الأشكال (اسم واحد، متغيرات مختلفة) أو مغطاة (تعديل في الصف الفرعي).
  - يمكن أن تكون `static` (مستوى الصف) أو مستنثنة (مستوى الكائن).
- **مثال**:
  ```java
  public class MathUtils {
      public int add(int a, int b) {
          return a + b;
      }

      public double add(double a, double b) {  // طريقة متعددة الأشكال
          return a + b;
      }
  }
  // الاستخدام
  MathUtils utils = new MathUtils();
  System.out.println(utils.add(5, 3));      // Output: 8
  System.out.println(utils.add(5.5, 3.2));  // Output: 8.7
  ```
- **استخدام حقيقي**: طريقة `withdraw` في `BankAccount` يمكن أن تحديث رصيد الحساب وتسجيل المعاملة.

## 4. **المتغيرات**
- **ما هو**: المتغيرات تخزن قيم البيانات ويجب أن تُعلن مع نوع محدد (مثل `int`، `String`، `double`).
- **لماذا هو مهم**: المتغيرات هي مكان الحفظ في الذاكرة للبيانات في البرنامج، مما يتيح إدارة الحالة والحساب.
- **كيف يتم استخدامه**: Java لديها أنواع متغيرات مختلفة:
  - **المتغيرات المحلية**: تُعلن داخل الطرق، مع نطاق محدود لتلك الطريقة.
  - **متغيرات المثال**: تُعلن في الصف، مرتبطة بكل كائن.
  - **متغيرات ثابتة**: تُعلن مع `static`، مشاركة بين جميع أمثلة الصف.
- **الغوص العميق**:
  - المتغيرات لديها قيم افتراضية (مثل `0` لـ `int`، `null` للكائنات) إذا لم يتم تهيئتها (فقط للمتغيرات المستنثنة/الثابتة).
  - Java تقيد التحديد القوي، مما يمنع التخصيص غير المتوافق دون تحويل صريح.
- **مثال**:
  ```java
  public class Counter {
      static int totalCount = 0;  // متغير ثابت
      int instanceCount;          // متغير مثال

      public void increment() {
          int localCount = 1;     // متغير محلي
          instanceCount += localCount;
          totalCount += localCount;
      }
  }
  ```
- **استخدام حقيقي**: تتبع عدد المستخدمين المتصلين (ثابت) مقابل أوقات الجلسة الفردية (مثال).

## 5. **أوامر تدفق التحكم**
- **ما هو**: أوامر تدفق التحكم تحدد مسار تنفيذ البرنامج، بما في ذلك الشرطية (`if`، `else`، `switch`) والدورات (`for`، `while`، `do-while`).
- **لماذا هو مهم**: تسمح بالاختيار والتكرار، وهما ضروريان لتنفيذ منطق معقد.
- **كيف يتم استخدامه**:
  - **الشرطية**: تنفذ الكود بناءً على الشروط المنطقية.
  - **الدورات**: تكرر على البيانات أو تكرر الأفعال حتى يتم تحقيق شرط.
- **الغوص العميق**:
  - `switch` يدعم `String` (منذ Java 7) و `enum` بالإضافة إلى الأنواع الأولية.
  - الدورات يمكن أن تكون متداخلة، وتعديل `break`/`continue` سلوكها.
- **مثال**:
  ```java
  int score = 85;
  if (score >= 90) {
      System.out.println("A");
  } else if (score >= 80) {
      System.out.println("B");
  } else {
      System.out.println("C");
  }

  for (int i = 0; i < 3; i++) {
      System.out.println("Loop iteration: " + i);
  }
  ```
- **استخدام حقيقي**: معالجة قائمة من الطلبات (`for` loop) وتطبيق الخصومات بناءً على المبلغ الإجمالي (`if`).

## 6. **الواجهة**
- **ما هو**: الواجهة هي عقد يحدد الطرق التي يجب أن تحددها الصفوف التي تنفذها. تدعم التجريد والتوريث المتعدد.
- **لماذا هو مهم**: الواجهات تتيح الارتباط الضعيف والتعدد الأشكال، مما يسمح لصفوف مختلفة أن تشارك واجهة مشتركة.
- **كيف يتم استخدامه**: الصفوف تنفذ الواجهات باستخدام كلمة `implements`. منذ Java 8، يمكن أن تحتوي الواجهات على طرق افتراضية وثابتة مع تنفيذ.
- **الغوص العميق**:
  - الطرق الافتراضية تسمح بتطور الواجهات مع الحفاظ على التوافق العكسي.
  - الواجهات الوظيفية (مع طريقة مجردة واحدة) هي مفتاح للتعبيرات اللامبدية.
- **مثال**:
  ```java
  public interface Vehicle {
      void start();
      default void stop() {  // طريقة افتراضية
          System.out.println("Vehicle stopped");
      }
  }

  public class Bike implements Vehicle {
      public void start() {
          System.out.println("Bike started");
      }
  }
  // الاستخدام
  Bike bike = new Bike();
  bike.start();  // Output: Bike started
  bike.stop();   // Output: Vehicle stopped
  ```
- **استخدام حقيقي**: واجهة `Payment` لـ `CreditCard` و `PayPal` في نظام بوابة الدفع.

## 7. **معالجة الاستثناءات**
- **ما هو**: معالجة الاستثناءات تدير الأخطاء في وقت التشغيل باستخدام `try`، `catch`، `finally`، `throw`، و `throws`.
- **لماذا هو مهم**: تضمن الاستقرار عن طريق منع التوقفات وتسمح بالاستعادة من الأخطاء مثل ملف غير موجود أو القسمة على الصفر.
- **كيف يتم استخدامه**: الكود المخاطر يذهب في كتلة `try`، الاستثناءات المحددة يتم القبض عليها في كتلة `catch`، و `finally` تنفذ الكود التنظيف.
- **الغوص العميق**:
  - الاستثناءات هي كائنات مشتقة من `Throwable` (`Error` أو `Exception`).
  - يمكن إنشاء استثناءات مخصصة عن طريق توسيع `Exception`.
- **مثال**:
  ```java
  try {
      int[] arr = new int[2];
      arr[5] = 10;  // ArrayIndexOutOfBoundsException
  } catch (ArrayIndexOutOfBoundsException e) {
      System.out.println("Index out of bounds: " + e.getMessage());
  } finally {
      System.out.println("Cleanup done");
  }
  ```
- **استخدام حقيقي**: معالجة توقيت الشبكة في تطبيق ويب.

## 8. **الأنماط**
- **ما هو**: الأنماط تسمح بالكود الآمن من حيث النوع، قابل لإعادة الاستخدام عن طريق تحديد الصفوف، الواجهات، والطرق مع أنواع.
- **لماذا هو مهم**: تكتشف الأخطاء في نوع في وقت التجميع، تقليل الأخطاء في وقت التشغيل، وتزيل الحاجة إلى التحويل.
- **كيف يتم استخدامه**: شائع في المجموعات (مثل `List<String>`) والطرق/الصفوف الأنماطية المخصصة.
- **الغوص العميق**:
  - الرموز (`? extends T`، `? super T`) تسيطر على الاختلاف في النوع.
  - إزالة نوع الأنماط (type erasure) تبعد معلومات نوع الأنماط في وقت التشغيل للتوافق العكسي.
- **مثال**:
  ```java
  public class Box<T> {
      private T content;
      public void set(T content) { this.content = content; }
      public T get() { return content; }
  }
  // الاستخدام
  Box<Integer> intBox = new Box<>();
  intBox.set(42);
  System.out.println(intBox.get());  // Output: 42
  ```
- **استخدام حقيقي**: صف `Cache<K, V>` لتخزين المفتاح/القيمة.

## 9. **التعبيرات اللامبدية**
- **ما هو**: التعبيرات اللامبدية (Java 8+) هي تمثيلات مختصرة للوظائف المجهولة، تُستخدم عادةً مع الواجهات الوظيفية.
- **لماذا هو مهم**: تسيءل الكود لتسجيل الأحداث، معالجة المجموعات، وبرمجة الوظيفة.
- **كيف يتم استخدامه**: مع الواجهات مثل `Runnable`، `Comparator`، أو المخصصة مع طريقة مجردة واحدة.
- **الغوص العميق**:
  - الصيغة: `(parameters) -> expression` أو `(parameters) -> { statements; }`.
  - تسمح بAPI التدفق لتدفق البيانات بالشكل الوظيفي.
- **مثال**:
  ```java
  List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
  names.forEach(name -> System.out.println(name.toUpperCase()));
  ```
- **استخدام حقيقي**: ترتيب قائمة من المنتجات حسب السعر باستخدام `Collections.sort(products, (p1, p2) -> p1.getPrice() - p2.getPrice())`.

## 10. **التعليقات**
- **ما هو**: التعليقات هي علامات بيانات (مثل `@Override`، `@Deprecated`) تُطبق على عناصر الكود، تُعالج في وقت التجميع أو التشغيل.
- **لماذا هو مهم**: تزويد التعليمات للمترجمين، الأطر، أو الأدوات، مما يزيد من التلقائية ويقلل من الكود المتكرر.
- **كيف يتم استخدامه**: يستخدم للتكوين (مثل `@Entity` في JPA)، الوثائق، أو فرض القواعد.
- **الغوص العميق**:
  - يمكن تعريف التعليقات المخصصة باستخدام `@interface`.
  - سياسات الاحتفاظ (`SOURCE`، `CLASS`، `RUNTIME`) تحدد عمرها.
- **مثال**:
  ```java
  public class MyClass {
      @Override
      public String toString() {
          return "Custom string";
      }

      @Deprecated
      public void oldMethod() {
          System.out.println("Old way");
      }
  }
  ```
- **استخدام حقيقي**: `@Autowired` في Spring لتسجيل التبعيات تلقائيًا.

## الميزات الأساسية الإضافية

لتعميق فهمك، إليك المزيد من الميزات الشائعة في Java مع شرحات مفصلة:

### 11. **الجداول**
- **ما هو**: الجداول هي مجموعات مرتبة ثابتة الحجم من العناصر من نفس النوع.
- **لماذا هو مهم**: توفر طريقة بسيطة، فعالة لتخزين و الوصول إلى قيم متعددة.
- **كيف يتم استخدامه**: يُعلن على النحو التالي `type[] name = new type[size];` أو يُنشئ مباشرة.
- **مثال**:
  ```java
  int[] numbers = {1, 2, 3, 4};
  System.out.println(numbers[2]);  // Output: 3
  ```
- **استخدام حقيقي**: تخزين قائمة من درجات الحرارة لأسبوع.

### 12. **الأنواع المتعددة**
- **ما هو**: الأنواع المتعددة تحدد مجموعة ثابتة من القيم الثابتة، غالبًا مع قيم أو طرق مرتبطة.
- **لماذا هو مهم**: تحسن أمان النوع وقابلية القراءة مقارنة بالثوابت الخام.
- **كيف يتم استخدامه**: يستخدم لتحديد الفئات المحددة مثل الأيام، الأحوال، أو الحالة.
- **مثال**:
  ```java
  public enum Status {
      PENDING("In progress"), APPROVED("Done"), REJECTED("Failed");
      private String desc;
      Status(String desc) { this.desc = desc; }
      public String getDesc() { return desc; }
  }
  // الاستخدام
  System.out.println(Status.APPROVED.getDesc());  // Output: Done
  ```
- **استخدام حقيقي**: تمثيل حالة الطلبات في نظام تجارة إلكترونية.

### 13. **التدفقات (Java 8+)**
- **ما هو**: التدفقات توفر نهجًا وظيفيًا لتدوير المجموعات، تدعم العمليات مثل `filter`، `map`، و `reduce`.
- **لماذا هو مهم**: تسيءل معالجة البيانات، تدعم التوازي، وتحسن تعبيرية الكود.
- **كيف يتم استخدامه**: تُنشأ من المجموعات باستخدام `.stream()` وتتم ربطها مع العمليات.
- **مثال**:
  ```java
  List<Integer> nums = Arrays.asList(1, 2, 3, 4, 5);
  int sum = nums.stream()
                .filter(n -> n % 2 == 0)
                .mapToInt(n -> n * 2)
                .sum();
  System.out.println(sum);  // Output: 12 (2*2 + 4*2)
  ```
- **استخدام حقيقي**: تجميع بيانات المبيعات حسب المنطقة.

### 14. **المصنعين**
- **ما هو**: المصنعين هي طرق خاصة تُستدعى عند إنشاء كائن، تُستخدم لتهيئة حالة الكائن.
- **لماذا هو مهم**: يضمن أن تبدأ الكائنات مع بيانات صالحة، ويقلل من أخطاء التهيئة.
- **كيف يتم استخدامه**: يُعلن باسم الصف، مع متغيرات اختيارية.
- **مثال**:
  ```java
  public class Book {
      String title;
      public Book(String title) {
          this.title = title;
      }
  }
  ```
- **استخدام حقيقي**: تهيئة كائن `User` مع اسم المستخدم وكلمة المرور.

### 15. **الوراثة**
- **ما هو**: الوراثة تسمح لصف (صف فرعي) أن يورث حقولًا وطرقًا من صف آخر (صف رئيسي) باستخدام `extends`.
- **لماذا هو مهم**: تروج لإعادة استخدام الكود وتأسيس علاقة هرمية بين الصفوف.
- **كيف يتم استخدامه**: يستخدم لإنشاء نسخ متخصصة من صف عام.
- **مثال**:
  ```java
  public class Animal {
      void eat() { System.out.println("Eating"); }
  }
  public class Dog extends Animal {
      void bark() { System.out.println("Barking"); }
  }
  // الاستخدام
  Dog dog = new Dog();
  dog.eat();  // Output: Eating
  dog.bark(); // Output: Barking
  ```
- **استخدام حقيقي**: صف `SavingsAccount` يورث من `BankAccount`.
