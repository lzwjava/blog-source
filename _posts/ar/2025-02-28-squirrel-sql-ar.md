### النقاط الرئيسية
- يبدو أن Squirrel SQL هو عميل SQL مبني على Java لربط قواعد البيانات باستخدام سائقات JDBC.
- تشير الأبحاث إلى أن عليك تثبيت Java، سائق JDBC المناسب لقاعدة البيانات الخاصة بك، وإضافة اسم مستعار مع تفاصيل الاتصال لاستخدامه.
- تشير الأدلة إلى خطوات تشمل تحميل Squirrel SQL، إعداد السائق، والاتصال عبر واجهة مستخدم مريحة.

### البدء مع Squirrel SQL
Squirrel SQL هو أداة تساعدك على إدارة واستعلام قواعد البيانات، وهي مصممة لتكون مريحة للمستخدمين الجدد في إدارة قواعد البيانات. إليك كيفية البدء:

#### التثبيت
أولا، تأكد من تثبيت Java على حاسوبك، يمكنك تحميله من [هذا الموقع](https://www.java.com/download). ثم، تحميل Squirrel SQL من [SourceForge](https://sourceforge.net/p/squirrel-sql) واتباع شريط التثبيت لتثبيته.

#### الاتصال بقاعدة البيانات
للتواصل، ستحتاج إلى سائق JDBC الخاص بقاعدة البيانات الخاصة بك (مثل MySQL، PostgreSQL). ابحث عن هذه السائقات على موقع مزود قاعدة البيانات، مثل [MySQL](https://dev.mysql.com/downloads/connector/j) أو [PostgreSQL](https://jdbc.postgresql.org/download.html). أضف السائق في Squirrel SQL تحت “عرض السائقات”، ثم أنشئ اسم مستعار مع عنوان URL لقاعدة البيانات الخاصة بك (مثل “jdbc:mysql://localhost:3306/mydatabase”)، اسم المستخدم وكلمة المرور. انقر مرتين على الاسم المستعار للاتصال.

#### استخدام الواجهة
بعد الاتصال، استخدم علامة التبويب “Objects” لتصفح بنية قاعدة البيانات وبياناتها، وعلامة التبويب “SQL” لإجراء استعلامات. كما أنها تدعم ميزات مثل استيراد البيانات وتصوير الرسوم البيانية، والتي قد تكون غير متوقعة من أداة موجهة لإدارة SQL.

### ملاحظة الاستطلاع: دليل شامل لاستخدام Squirrel SQL وربطه بقواعد البيانات

توفر هذه الملاحظة استكشافًا مفصلًا لاستخدام Squirrel SQL، عميل SQL الرسومي مبني على Java، لإدارة قواعد البيانات، مع التركيز بشكل خاص على الاتصال بقواعد البيانات. إنها توسع على التوجيهات الأولية، وتقدم نظرة شاملة ومهنية بناءً على الموارد المتاحة، مناسبة للمستخدمين الذين يبحثون عن فهم عميق.

#### مقدمة إلى Squirrel SQL
Squirrel SQL هو برنامج عميل SQL مفتوح المصدر مبني على Java، مصمم لأي قاعدة بيانات متوافقة مع JDBC، مما يسمح للمستخدمين بملاحظة الهياكل، تصفح البيانات، وإجراء أوامر SQL. يتم توزيعه تحت رخصة GNU Lesser General Public License، مما يضمن الوصولية والتكيف. نظرًا لأساسها على Java، تعمل على أي منصة مع JVM، مما يجعلها متعددة الاستخدامات لـ Windows، Linux، وmacOS.

#### عملية التثبيت
تبدأ عملية التثبيت بتأكد من تثبيت Java، حيث يتطلب Squirrel SQL على الأقل Java 6 لأصدار 3.0، على الرغم من أن الإصدارات الأحدث قد تتطلب تحديثات. يمكن للمستخدمين تحميل Java من [هذا الموقع](https://www.java.com/download). بعد ذلك، تحميل Squirrel SQL من [SourceForge](https://sourceforge.net/p/squirrel-sql)، متاحًا كملف JAR (مثل “squirrel-sql-version-install.jar”). يتضمن التثبيت تشغيل JAR مع Java واستخدام مساعد التثبيت، الذي يقدم خيارات مثل “التثبيت الأساسي” أو “التثبيت القياسي”، حيث يتضمن الأخير الإضافات المفيدة مثل إكمال الكود وتهجئة النص.

#### الاتصال بقواعد البيانات: دليل خطوة بخطوة
يتضمن الاتصال بقاعدة البيانات عدة خطوات حاسمة، كل منها يتطلب الانتباه إلى التفاصيل لضمان التكامل الناجح:

1. **الحصول على سائق JDBC**: يتطلب كل نوع قاعدة بيانات سائق JDBC محدد. على سبيل المثال، يمكن للمستخدمين MySQL تحميله من [MySQL](https://dev.mysql.com/downloads/connector/j)، PostgreSQL من [PostgreSQL](https://jdbc.postgresql.org/download.html)، وOracle من [Oracle](https://www.oracle.com/database/technologies/appdev/jdbc-downloads.html). السائق، عادةً ملف JAR، ييسر التواصل بين Squirrel SQL وقاعدة البيانات.

2. **إضافة السائق في Squirrel SQL**: افتح Squirrel SQL، انتقل إلى “Windows” > “View Drivers”، وانقر على رمز “+” لإضافة سائق جديد. اسميه (مثل “MySQL Driver”)، أدخل اسم الفئة (مثل “com.mysql.cj JDBC Driver” لأحدث إصدارات MySQL، مع الأخذ في الاعتبار الاختلافات حسب الإصدار)، وأضف مسار ملف JAR في علامة التبويب “Extra Class Path”. يشير علامة التحقق الزرقاء إلى أن السائق في مسار JVM classpath؛ يشير علامة التحقق الحمراء إلى أنه يحتاج إلى تحميله من المزود.

3. **إنشاء اسم مستعار**: من القائمة، اختر “Aliases” > “New Alias…” أو استخدم Ctrl+N. أدخل اسمًا للاسم المستعار، اختر السائق، وأدخل عنوان URL لقاعدة البيانات. تختلف صيغة URL حسب:
   - MySQL: “jdbc:mysql://hostname:port/database_name”
   - PostgreSQL: “jdbc PostgreSQL://hostname:port/database_name”
   - Oracle: “jdbc:oracle:thin:@//hostname:port/SID”
   أدخل اسم المستخدم وكلمة المرور، تأكد من صحة التفاصيل كما قدمها مدير قاعدة البيانات.

4. **إقامة الاتصال**: انقر مرتين على الاسم المستعار في نافذة “Aliases” لفتح جلسة. يدعم Squirrel SQL جلسة متعددة متزامنة، مفيدة للمقارنة بين البيانات أو مشاركة أوامر SQL عبر الاتصالات.

#### استخدام Squirrel SQL: الواجهة والميزات
بعد الاتصال، يقدم Squirrel SQL واجهة قوية للتواصل مع قاعدة البيانات:

- **علامة التبويب Objects**: هذه العلامة التبويب تسمح بتصفح كائنات قاعدة البيانات مثل الفهرس، المخططات، الجداول، المفعولات، الأطروحات، التسلسلات، الإجراءات، وUDTs. يمكن للمستخدمين التنقل في شكل شجرة، تحرير القيم، إضافة أو حذف الصفوف، واستيراد/تصدير البيانات، مما يزيد من قدرات إدارة البيانات.

- **علامة التبويب SQL**: محرر SQL، مبني على RSyntaxTextArea بواسطة fifesoft.com، يقدم تبييض النص وتدعم فتح، إنشاء، حفظ، وإجراء ملفات SQL. هو مثالي لإجراء الاستعلامات، بما في ذلك الاندماجات المعقدة، مع نتائج تعود كجداول تتضمن البيانات الوصفية.

- **الميزات الإضافية**: يتضمن Squirrel SQL الإضافات مثل إضافة بيانات Excel/CSV، إضافة DBCopy، إضافة SQL Bookmarks للمقاطع المخصصة (مثل الأوامر الشائعة SQL وDDL)، إضافة SQL Validator، وإضافة قاعدة البيانات الخاصة لـ DB2، Firebird، وDerby. إضافة Graph تبييض العلاقات بين الجداول والمفاتيح الخارجية، والتي قد تكون غير متوقعة للمستخدمين الذين يتوقعون فقط وظائف SQL الأساسية. يمكن للمستخدمين إدخال المقاطع المخصصة SQL باستخدام Ctrl+J، مما يسهل المهام المتكررة.

#### حل المشكلات والتفاصيل
قد يواجه المستخدمون مشاكل في الاتصال، والتي يمكن حلها من خلال:
- التأكد من تشغيل وخدمة قاعدة البيانات.
- التحقق من تثبيت سائق JDBC وصدق اسم الفئة، حيث قد تختلف الإصدارات (مثل سائقات MySQL القديمة التي استخدمت “com.mysql JDBC Driver”).
- التحقق من URL من الأخطاء أو المفقودات، مثل إعدادات SSL (مثل “?useSSL=false” لـ MySQL).
- الاستشارة بالوثائق الخاصة بالمزود من أجل متطلبات محددة، مثل مخازن الثقة للاتصالات الآمنة.

تدعم الواجهة ترجمات الواجهة في لغات مثل بلغارية، برتغالية برازيلية، صينية، تشيكية، فرنسية، ألمانية، إيطالية، يابانية، بولندية، إسبانية، وروسية، مما يخدم قاعدة مستخدمين عالمية.

#### رؤى مقارنة
مقارنةً بأدوات SQL الأخرى، قوة Squirrel SQL تكمن في بنية الإضافات، مما يسمح بتوسيعات محددة للمزود والتوافقية الواسعة. ومع ذلك، قد يكون التثبيت أقل مباشرة بسبب الاعتماد على Java، ويمكن أن تكون الوثائق نادرة، غالبًا تتطلب تعليمات من طرف ثالث مثل تلك الموجودة على [SqlBot](https://www.sqlbot.co/blog/squirrel-sql-review-and tutorial) للحصول على توجيهات مفصلة.

#### جدول: الخطوات الرئيسية للاتصال بـ MySQL كأمثلة
لإلحاح، إليك جدول للاتصال بـ MySQL، حالة استخدام شائعة:

| الخطوة                  | التفاصيل                                                                                     |
|-----------------------|---------------------------------------------------------------------------------------------|
| 1. تثبيت Java       | الإصدار المطلوب: على الأقل Java 6 لـ SQuirreL SQL الإصدار 3.0؛ تحميل من [هذا الموقع](https://www.java.com/download) |
| 2. تحميل SQuirreL SQL | متاح من [SourceForge](https://sourceforge.net/p/squirrel-sql) كملف JAR (مثل "squirrel-sql-version-install.jar") |
| 3. تثبيت SQuirreL SQL | استخدم مساعد التثبيت؛ اختر "التثبيت الأساسي" أو "التثبيت القياسي" مع الإضافات مثل إكمال الكود |
| 4. تعريف السائق  | إرشاد إلى ملف JDBC JAR لـ MySQL (مثل mysql-connector-java-8.0.32.jar)؛ تحقق من الحالة في نافذة السائقات (علامة التحقق الزرقاء إذا كان في مسار JVM classpath، علامة التحقق الحمراء إذا لم يكن); تحميل السائق من [MySQL](https://dev.mysql.com/downloads/connector/j) |
| 5. إنشاء اسم مستعار    | من شريط القائمة، اختر Aliases > New Alias… أو استخدم Ctrl+N؛ أدخل عنوان URL لقاعدة البيانات (مثل "jdbc:mysql://localhost:3306/mydatabase")، اسم المستخدم وكلمة المرور؛ استشارة مدير قاعدة البيانات للحصول على التفاصيل |

يوضح هذا الجدول، مشتق من التعليمات، العملية، مع ملاحظة أن النقر مرتين على الاسم المستعار يفتح جلسة للتواصل عبر علامات التبويب Objects أو SQL.

#### الخاتمة
يقدم Squirrel SQL حلًا شاملًا لإدارة قواعد البيانات، مع خطوات مفصلة للاتصال ومجموعة ميزات غنية تشمل الإضافات والتصوير. يجب على المستخدمين التأكد من إعداد مناسب، واستخدام وثائق المزود والموارد من طرف ثالث لحل المشكلات، لضمان تجربة سلسة في تعاملات قاعدة البيانات اعتبارًا من 28 فبراير 2025.
