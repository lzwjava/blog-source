### البدء مع Squirrel SQL
Squirrel SQL هو أداة تساعدك في إدارة واستعلام قواعد البيانات، ومصمم ليكون سهل الاستخدام للمبتدئين في إدارة قواعد البيانات. إليك كيفية البدء:

#### التثبيت
أولاً، تأكد من تثبيت Java على جهازك، والذي يمكنك تنزيله من [هذا الموقع](https://www.java.com/download). ثم، قم بتنزيل Squirrel SQL من [SourceForge](https://sourceforge.net/p/squirrel-sql) واتبع معالج التثبيت لإعداده.

#### الاتصال بقاعدة بيانات
للاتصال، ستحتاج إلى برنامج تشغيل JDBC الخاص بقاعدة البيانات المحددة لديك (مثل MySQL أو PostgreSQL). يمكنك العثور على هذه البرامج على موقع مورد قاعدة البيانات، مثل [MySQL](https://dev.mysql.com/downloads/connector/j) أو [PostgreSQL](https://jdbc.postgresql.org/download.html). أضف برنامج التشغيل في Squirrel SQL ضمن "View Drivers"، ثم أنشئ اسمًا مستعارًا مع عنوان URL لقاعدة البيانات (مثل "jdbc:mysql://localhost:3306/mydatabase")، واسم المستخدم، وكلمة المرور. انقر نقرًا مزدوجًا على الاسم المستعار للاتصال.

#### استخدام الواجهة
بمجرد الاتصال، استخدم علامة التبويب "Objects" لاستعراض هيكل بياناتك والبيانات، وعلامة التبويب "SQL" لتشغيل الاستعلامات. كما يدعم ميزات مثل استيراد البيانات وتصور الرسوم البيانية، وهو ما قد يكون غير متوقع لأداة تركز على إدارة SQL.

### ملاحظة مسح: دليل شامل لاستخدام Squirrel SQL والاتصال بقواعد البيانات

توفر هذه الملاحظة استكشافًا مفصلاً لاستخدام Squirrel SQL، وهو عميل SQL رسومي يعتمد على Java، لإدارة قواعد البيانات، مع التركيز بشكل خاص على الاتصال بقواعد البيانات. وهي توسع نطاق التوجيهات الأولية، وتقدم نظرة عامة شاملة واحترافية بناءً على الموارد المتاحة، ومناسبة للمستخدمين الذين يسعون إلى فهم متعمق.

#### مقدمة إلى Squirrel SQL
Squirrel SQL هو برنامج عميل SQL مفتوح المصدر مكتوب بلغة Java مصمم لأي قاعدة بيانات متوافقة مع JDBC، مما يمكن المستخدمين من عرض الهياكل واستعراض البيانات وتنفيذ أوامر SQL. يتم توزيعه بموجب ترخيص GNU Lesser General Public License، مما يضمن إمكانية الوصول والمرونة. نظرًا لأساسه بلغة Java، فهو يعمل على أي منصة تحتوي على JVM، مما يجعله متعدد الاستخدامات لمستخدمي Windows وLinux وmacOS.

#### عملية التثبيت
تبدأ عملية التثبيت بالتأكد من تثبيت Java، حيث يتطلب Squirrel SQL على الأقل Java 6 للإصدار 3.0، على الرغم من أن الإصدارات الأحدث قد تتطلب تحديثات. يمكن للمستخدمين تنزيل Java من [هذا الموقع](https://www.java.com/download). بعد ذلك، قم بتنزيل Squirrel SQL من [SourceForge](https://sourceforge.net/p/squirrel-sql)، والمتاح كملف JAR (مثل "squirrel-sql-version-install.jar"). يتضمن التثبيت تشغيل ملف JAR باستخدام Java واستخدام مساعد الإعداد، الذي يقدم خيارات مثل التثبيت "الأساسي" أو "القياسي"، حيث يتضمن الأخير إضافات مفيدة مثل إكمال الكود وإبراز بناء الجملة.

#### الاتصال بقواعد البيانات: دليل خطوة بخطوة
يتضمن الاتصال بقاعدة بيانات عدة خطوات حرجة، كل منها يتطلب الاهتمام بالتفاصيل لضمان التكامل الناجح:

1.  **الحصول على برنامج تشغيل JDBC**: كل نوع قاعدة بيانات يتطلب برنامج تشغيل JDBC محدد. على سبيل المثال، يمكن لمستخدمي MySQL التنزيل من [MySQL](https://dev.mysql.com/downloads/connector/j)، ومستخدمي PostgreSQL من [PostgreSQL](https://jdbc.postgresql.org/download.html)، ومستخدمي Oracle من [Oracle](https://www.oracle.com/database/technologies/appdev/jdbc-downloads.html). برنامج التشغيل، وهو عادةً ملف JAR، يسهل الاتصال بين Squirrel SQL وقاعدة البيانات.

2.  **إضافة برنامج التشغيل في Squirrel SQL**: افتح Squirrel SQL، وانتقل إلى "Windows" > "View Drivers"، وانقر على أيقونة "+" لإضافة برنامج تشغيل جديد. قم بتسميته (مثل "MySQL Driver")، وأدخل اسم الفئة (مثل "com.mysql.cj.jdbc.Driver" لإصدارات MySQL الحديثة، مع ملاحظة الاختلافات حسب الإصدار)، وأضف مسار ملف JAR في علامة التبويب "Extra Class Path". تشير علامة الاختيار الزرقاء إلى أن برنامج التشغيل موجود في مسار فئة JVM؛ بينما تشير علامة X الحمراء إلى أنه يحتاج إلى التنزيل من المورد.

3.  **إنشاء اسم مستعار**: من القائمة، حدد "Aliases" > "New Alias…" أو استخدم Ctrl+N. أدخل اسمًا للاسم المستعار، وحدد برنامج التشغيل، وأدخل عنوان URL لقاعدة البيانات. يختلف تنسيق عنوان URL:
    *   MySQL: "jdbc:mysql://hostname:port/database_name"
    *   PostgreSQL: "jdbc:postgresql://hostname:port/database_name"
    *   Oracle: "jdbc:oracle:thin:@//hostname:port/SID"
    قم بتقديم اسم المستخدم وكلمة المرور، مع التأكد من صحة التفاصيل كما هو مقدم من مسؤول قاعدة البيانات.

4.  **إنشاء الاتصال**: انقر نقرًا مزدوجًا على الاسم المستعار في نافذة "Aliases" لفتح جلسة. يدعم Squirrel SQL جلسات متعددة في وقت واحد، وهو أمر مفيد لمقارنة البيانات أو مشاركة عبارات SQL عبر الاتصالات.

#### استخدام Squirrel SQL: الواجهة والميزات
بمجرد الاتصال، تقدم Squirrel SQL واجهة قوية للتفاعل مع قاعدة البيانات:

*   **علامة تبويب Objects**: تتيح هذه علامة التبويب استعراض كائنات قاعدة البيانات مثل الفهارس والمخططات والجداول والمشغلات والعروض والتسلسلات والإجراءات و UDTs. يمكن للمستخدمين التنقل في شكل الشجرة، وتحرير القيم، وإدراج أو حذف الصفوف، واستيراد/تصدير البيانات، مما يعزز قدرات إدارة البيانات.

*   **علامة تبويب SQL**: محرر SQL، المبني على RSyntaxTextArea من fifesoft.com، يوفر إبرازًا للبناء النحوي ويدعم فتح وإنشاء وحفظ وتنفيذ ملفات SQL. إنه مثالي لتشغيل الاستعلامات، بما في ذلك عمليات الربط المعقدة، مع إرجاع النتائج كجداول تتضمن بيانات وصفية.

*   **ميزات إضافية**: يتضمن Squirrel SQL إضافات مثل Data Import Plugin لـ Excel/CSV، وDBCopy Plugin، وSQL Bookmarks Plugin لقوالب الكود المحددة من قبل المستخدم (مثل عبارات SQL و DDL الشائعة)، وSQL Validator Plugin، وإضافات محددة لقاعدة البيانات لـ DB2 وFirebird وDerby. تقوم إضافة Graph بتصور علاقات الجداول والمفاتيح الخارجية، وهو ما قد يكون غير متوقع للمستخدمين الذين يتوقعون وظائف SQL أساسية فقط. يمكن للمستخدمين إدراج قوالب SQL المرجعية باستخدام Ctrl+J، مما يبسط المهام المتكررة.

#### استكشاف الأخطاء وإصلاحها والاعتبارات
قد يواجه المستخدمون مشاكل في الاتصال، والتي يمكن معالجتها عن طريق:
*   التأكد من أن خادم قاعدة البيانات قيد التشغيل ويمكن الوصول إليه.
*   التحقق من تثبيت برنامج تشغيل JDBC ودقة اسم الفئة، حيث قد تختلف الإصدارات (مثل برامج تشغيل MySQL القديمة التي استخدمت "com.mysql.jdbc.Driver").
*   التحقق من عنوان URL من الأخطاء المطبعية أو المعلمات المفقودة، مثل إعدادات SSL (مثل "?useSSL=false" لـ MySQL).
*   الرجوع إلى وثائق مورد قاعدة البيانات للاطلاع على المتطلبات المحددة، مثل مخازن الثقة للاتصالات الآمنة.

تدعم الواجهة الترجمات بلغات مثل البلغارية والبرتغالية البرازيلية والصينية والتشيكية والفرنسية والألمانية والإيطالية واليابانية والبولندية والإسبانية والروسية، لتلبي قاعدة مستخدمين عالمية.
