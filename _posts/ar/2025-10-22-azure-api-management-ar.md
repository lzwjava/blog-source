### إنشاء بوابة API في Azure باستخدام إدارة API

إدارة Azure API (APIM) هي الخدمة المُدارة بالكامل التي تعمل كبوابة API، وتوفر ميزات مثل نشر API، والأمان، والتحليلات، ومداخل المطورين. فيما يلي دليل خطوة بخطوة لإنشاء مثيل APIM عبر مدخل Azure.

#### المتطلبات الأساسية
- اشتراك Azure نشط. إذا لم يكن لديك واحد، فأنشئ [حساب Azure مجاني](https://azure.microsoft.com/free/).

#### خطوات إنشاء مثيل إدارة API

1. **تسجيل الدخول إلى مدخل Azure**  
   اذهب إلى [مدخل Azure](https://portal.azure.com) وقم بتسجيل الدخول باستخدام حساب Azure الخاص بك.

2. **إنشاء مورد جديد**  
   - من قائمة مدخل Azure، حدد **إنشاء مورد**. (بدلاً من ذلك، حدد **إنشاء مورد** من الصفحة الرئيسية لـ Azure).  
   - في صفحة **إنشاء مورد**، ابحث عن وحدد **إدارة API** تحت فئة **التكامل**.  
   - انقر فوق **إنشاء**.

3. **تكوين الأساسيات**  
   في صفحة **إنشاء إدارة API**، انتقل إلى علامة التبويب **الأساسيات** واملأ التفاصيل:  
   | الإعداد              | الوصف                                                                 |
   |----------------------|-----------------------------------------------------------------------------|
   | الاشتراك         | حدد اشتراك Azure لهذا المثيل.                            |
   | مجموعة الموارد       | اختر مجموعة موارد موجودة أو أنشئ مجموعة جديدة (مثال: "APIM-RG").    |
   | المنطقة               | اختر منطقة قريبة من مستخدميك أو خدماتك الخلفية (مثال: East US).      |
   | اسم المورد        | أدخل اسمًا فريدًا (مثال: "my-apim-instance"). سيصبح هذا جزءًا من النطاق الافتراضي: `<name>.azure-api.net`. لا يمكن تغييره لاحقًا. |
   | اسم المؤسسة    | اسم مؤسستك (يُستخدم في بوابة المطورين والبريد الإلكتروني).             |
   | البريد الإلكتروني للمسؤول  | بريدك الإلكتروني لإشعارات النظام.                                        |
   | فئة التسعير         | ابدأ بـ **Basic v2** للتطوير/الاختبار (نشر سريع، ~30-40 دقيقة). تقدم المستويات الأخرى مثل Developer أو Standard المزيد من الميزات. |
   | الوحدات                | الافتراضي هو 1 للتقييم.                                                |  
   راجع المعلومة وتابع.

4. **اختياري: تكوين المراقبة والأمان**  
   في علامة التبويب **مراقبة + تأمين**:  
   - قم بتمكين الإضافات الاختيارية مثل Log Analytics للتسجيل (قد يتضمن هذا تكاليف إضافية).  
   - تجاوز هذا الآن إذا كنت قد بدأت للتو.

5. **المراجعة والإنشاء**  
   - في علامة التبويب **مراجعة + إنشاء**، تحقق من صحة إعداداتك.  
   - انقر فوق **إنشاء**. يستغرق النشر 30-40 دقيقة أو أكثر.  
   - نصيحة: ثبّت المورد في لوحة التحكم الخاصة بك لسهولة الوصول بمجرد إنشائه.

6. **التحقق من المثيل**  
   - في مدخل Azure، ابحث عن **خدمات إدارة API**.  
   - حدد المثيل الجديد الخاص بك.  
   - في صفحة **نظرة عامة**، تأكد من أن الحالة هي "متصل". أنت الآن جاهز لاستيراد واجهات برمجة التطبيقات، أو تكوين السياسات، أو إعداد بوابة المطورين.

#### الخطوات التالية
- استورد أول API لك: اتبع [البرنامج التعليمي لاستيراد ونشر API](https://learn.microsoft.com/en-us/azure/api-management/import-and-publish).  
- قم بتأمينه: أضف سياسات للمصادقة، والحد من المعدل، وما إلى ذلك.  
- نظف: إذا كنت تختبر، فاحذف مجموعة الموارد عبر المدخل لتجنب الرسوم (ابحث عن **مجموعات الموارد** > حدد المجموعة > **حذف مجموعة الموارد**).  

للحصول على النطاقات المخصصة أو التكوينات المتقدمة، راجع [توثيق APIM](https://learn.microsoft.com/en-us/azure/api-management/).
