### ما هو chatmode.md؟

في امتداد GitHub Copilot لـ Visual Studio Code (VS Code)، تُستخدم ملفات `chatmode.md` لتعريف **أوضاع الدردشة المخصصة**. تتيح لك هذه الملفات تكوين Copilot Chat لتبني شخصيات أو سلوكيات محددة لمهام مثل التخطيط، أو مراجعات الأمان، أو تنفيذ التعليمات البرمجية. يمكن لكل وضع تحديد التعليمات، والأدوات المتاحة (مثل البحث، الجلب، أو الوصول إلى مستودع GitHub)، وحتى نموذج الذكاء الاصطناعي المستخدم. هذه الميزة قيد المعاينة اعتبارًا من VS Code 1.101 وتساعد في تخصيص الردود لتحقيق الاتساق في سير العمل الخاص بك.

يتم تخزين الأوضاع المخصصة كملفات Markdown بامتداد `.chatmode.md`، إما في مساحة العمل الخاصة بك (للمشاركة مع الفريق) أو في ملف تعريف المستخدم (للاستخدام الشخصي المتكرر).

### لماذا تستخدم أوضاع الدردشة المخصصة؟
- **ردود مخصصة**: فرض إرشادات، مثل إنشاء خطط دون تحرير التعليمات البرمجية.
- **التحكم في الأدوات**: تقييد الأدوات للقراءة فقط للتخطيط أو تمكين التحرير للتنفيذ.
- **الكفاءة**: إعادة استخدام الإعدادات للأدوار الشائعة (مثل المهندس المعماري، المراجع).

### كيفية إنشاء ملف chatmode.md

1. افتح **طريقة عرض الدردشة** في VS Code:
   - انقر على أيقونة Copilot Chat في شريط العنوان، أو استخدم `Ctrl+Alt+I` (Windows/Linux) / `Cmd+Option+I` (macOS).

2. في طريقة عرض الدردشة، انقر على **Configure Chat > Modes**، ثم حدد **Create new custom chat mode file**. بدلاً من ذلك، افتح لوحة الأوامر (`Ctrl+Shift+P` / `Cmd+Shift+P`) وقم بتشغيل **Chat: New Mode File**.

3. اختر موقعًا:
   - **مساحة العمل**: يحفظ في `.github/chatmodes/` افتراضيًا (قابل للمشاركة مع فريقك). قم بتخصيص المجلدات عبر إعداد `chat.modeFilesLocations`.
   - **ملف تعريف المستخدم**: يحفظ في مجلد ملف تعريفك للمزامنة عبر الأجهزة.

4. قم بتسمية الملف (على سبيل المثال، `planning.chatmode.md`) وقم بتحريره في VS Code.

لإدارة الأوضاع الحالية، استخدم **Configure Chat > Modes** أو أمر **Chat: Configure Chat Modes**.

### هيكل الملف والصيغة

يستخدم ملف `.chatmode.md` لغة Markdown مع YAML frontmatter اختياري للبيانات الوصفية. إليك الهيكل الأساسي:

- **YAML Frontmatter** (محاط بخطوط `---`، اختياري):
  - `description`: نص قصير يظهر في العنصر النائب لإدخال الدردشة وعند التمرير فوق القائمة المنسدلة.
  - `tools`: مصفوفة بأسماء الأدوات (مثل `['fetch', 'search']`). استخدم أدوات مدمجة مثل `githubRepo` أو أدوات الامتداد؛ قم بالتكوين عبر **Configure Tools**.
  - `model`: نموذج الذكاء الاصطناعي (مثل `"Claude Sonnet 4"`). يستخدم النموذج المحدد لديك افتراضيًا.

- **النص الأساسي**: تعليمات Markdown للذكاء الاصطناعي، بما في ذلك المطالبات، الإرشادات، أو الروابط إلى الملفات الخارجية.

أولوية الأدوات: ملف المطالبة > الوضع المشار إليه > أدوات الوضع الافتراضي.

### مثال على ملف chatmode.md

هذا ينشئ وضع "تخطيط" لإنشاء خطط تنفيذ دون تغييرات في التعليمات البرمجية:

```
---
description: إنشاء خطة تنفيذ للميزات الجديدة أو إعادة هيكلة التعليمات البرمجية الحالية.
tools: ['fetch', 'githubRepo', 'search', 'usages']
model: Claude Sonnet 4
---
# تعليمات وضع التخطيط

أنت في وضع التخطيط. مهمتك هي إنشاء خطة تنفيذ لميزة جديدة أو لإعادة هيكلة التعليمات البرمجية الحالية. لا تقم بإجراء أي تعديلات على التعليمات البرمجية — فقط قم بإنشاء خطة.

تتكون الخطة من مستند Markdown بهذه الأقسام:

* **نظرة عامة**: وصف موجز للميزة أو إعادة الهيكلة.
* **المتطلبات**: قائمة بالمتطلبات.
* **خطوات التنفيذ**: خطوات مفصلة للتنفيذ.
* **الاختبار**: قائمة بالاختبارات للتحقق.
```

احفظ الملف، وسيظهر في القائمة المنسدلة لأوضاع الدردشة.

### كيفية استخدام وضع الدردشة المخصص

1. افتح **طريقة عرض الدردشة** (كما هو مذكور أعلاه).

2. في القائمة المنسدلة أعلى طريقة عرض الدردشة (بجوار محدد الوضع)، اختر وضعك المخصص (على سبيل المثال، "Planning").

3. يظهر `description` الخاص بالوضع كعنصر نائب في مربع الإدخال. اكتب مطالبتك (على سبيل المثال، "خطط لميزة مصادقة المستخدم").

4. يستجيب Copilot باتباع تعليمات وأدوات الوضع. يمكنك تبديل الأوضاع في أي وقت عبر القائمة المنسدلة — تظل الأوضاع المدمجة (Ask, Edit, Agent) متاحة.

لمزيد من الأمثلة، تحقق من مساهمات المجتمع في [مستودع Awesome Copilot](https://github.com/github/awesome-copilot/tree/main).
