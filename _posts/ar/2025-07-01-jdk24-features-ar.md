### مقدمة عن JDK 24

يُعد Java Development Kit (JDK) 24 إصدارًا مُميزًا من منصة Java SE (الإصدار القياسي)، والذي تم إصداره رسميًا في 18 مارس 2025، كتنفيذ مرجعي للإصدار 24، والمُحدد بواسطة JSR 399 في عملية مجتمع Java. وهو يُواصل نهج أوراكل للإصدارات كل ستة أشهر، حيث يُقدم مجموعة قوية من التحسينات لتعزيز إنتاجية المطورين والأداء والأمان. يتضمن JDK 24 أربعة وعشرين مقترح تحسين JDK (JEP)، وهو أعلى عدد من الميزات منذ بدء جدول الإصدارات الزمني في عام 2018، مما يجعله معلماً هاماً في تطور Java. وهو بمثابة حجر عتبة نحو JDK 25، وهو إصدار الدعم طويل الأمد (LTS) التالي والمقرر إطلاقه في سبتمبر 2025.

### حالة الدعم طويل الأمد (LTS)

إن JDK 24 **ليس** إصدار دعم طويل الأمد (LTS). إنه إصدار دعم قصير الأجل، حيث يتلقى فقط ستة أشهر من دعم Premier من أوراكل، حتى سبتمبر 2025، عندما سيحل محله JDK 25. في المقابل، تحصل إصدارات LTS مثل JDK 21 (سبتمبر 2023) و JDK 25 القادم (سبتمبر 2025) على دعم Premier لمدة خمس سنوات على الأقل، مما يجعلها مُفضلة لاستقرار المؤسسات. يحدث نمط LTS الخاص بأوراكل كل عامين، حيث كان JDK 21 هو أحدث إصدار LTS ومن المقرر أن يكون JDK 25 هو التالي.

### الإصدار والاستقرار

يُعد JDK 24 **إصدارًا مستقرًا وجاهزًا للإنتاج**، حيث وصل إلى مرحلة التوفر العام (GA) في 18 مارس 2025. الثنائيات الجاهزة للإنتاج متاحة من أوراكل بموجب شروط وأحكام أوراكل مجانية الرسوم (NFTC) ورخصة جنو العمومية (GPLv2) لـ OpenJDK، مع تتبع ثنائيات البائعين الآخرين بعد ذلك بفترة قصيرة. يتضمن الإصدار أكثر من 3000 إصلاح للأخطاء وتحسينات أصغر تتجاوز مقترحات JEP الـ 24، مما يضمن الاستقرار للاستخدام العام. ومع ذلك، كإصدار غير LTS، فإنه يهدف في المقام الأول إلى المطورين المتحمسين لاختبار الميزات الجديدة وليس للمؤسسات التي تتطلب استقرارًا طويل الأجل.

### الميزات الجديدة في JDK 24

يقدم JDK 24 أربعة وعشرين مقترح تحسين JDK (JEP)، مُصنفة إلى تحسينات المكتبة الأساسية، وتحسينات اللغة، وميزات الأمان، وتحسينات HotSpot JVM، وأدوات Java. من بين هذه المقترحات، هناك 14 ميزة دائمة، وسبع ميزات معاينة، وميزتان تجريبيتان، ووحدة حاضنة واحدة. فيما يلي بعض أبرز الميزات، مع التركيز على تلك ذات الصلة بالمطورين والنشر:

1.  **Stream Gatherers (JEP 485)** - دائمة
    - يعزز Stream API من خلال تقديم واجهة `Gatherer`، مما يسمح للمطورين بتعريف عمليات وسيطة مخصصة لخطوط أنابيب الدفق. هذا يتيح تحويلات بيانات أكثر مرونة، مكملاً لواجهة `Collector` الحالية للعمليات الطرفية.
    - مثال: تجميع الكلمات حسب الطول باستخدام `StreamGatherers.groupBy`.
    - الفائدة: يبسط معالجة الدفق المعقدة للمطورين.

2.  **Ahead-of-Time Class Loading & Linking (JEP 483)** - تجريبية
    - جزء من Project Leyden، تقلل هذه الميزة أوقات بدء تشغيل تطبيقات Java عن طريق التحميل المسبق وربط الفئات في ذاكرة تخزين مؤقت خلال مرحلة تحضيرية. يتم إعادة استخدام ذاكرة التخزين المؤقت في وقت التشغيل، متجاوزة خطوات تحميل الفئة المُكلفة.
    - الفائدة: يحسن الأداء لتطبيقات السحابة والخدمات المصغرة.

3.  **Compact Object Headers (JEP 450)** - تجريبية
    - جزء من Project Lilliput، يقلل هذا من أحجام رؤوس كائنات Java من 96-128 بت إلى 64 بت على بنى 64 بت، مما يقلل من استخدام الذاكرة المؤقتة ويحسن كفاءة الذاكرة.
    - الفائدة: يقلل البصمة التذكرية ويعزز مكانية البيانات لأداء أفضل.

4.  **Generational Shenandoah Garbage Collector (JEP 404)** - دائمة
    - ينقل الوضع التوليدي لـ Shenandoah GC من التجريبي إلى ميزة منتج، مما يحسن الإنتاجية ومرونة ذروة الحمل واستخدام الذاكرة من خلال تقسيم الكائنات إلى أجيال شابة ومسنة.
    - الفائدة: يعزز الأداء لأحمال العمل المتطلبة.

5.  **Module Import Declarations (JEP 494)** - المعاينة الثانية
    - يبسط البرمجة النمطية من خلال السماح باستيراد مباشر لجميع الحزم التي يصدرها وحدة نمطية دون الحاجة إلى ملف `module-info.java` (على سبيل المثال، `import module java.sql;`).
    - الفائدة: يقلل النفقات العامة للتطبيقات الخفيفة الوزن والنصوص البرمجية، مما يساعد المبتدئين والنماذج الأولية السريعة.

6.  **Flexible Constructor Bodies (JEP 492)** - المعاينة الثالثة
    - يسمح بعبارات في المُنشئات قبل استدعاءات `super()` أو `this()`، مما يتيح وضع منطق تهيئة الحقل بشكل أكثر طبيعية دون طرق مساعدة.
    - الفائدة: يحسن موثوقية الكود وقابليته للقراءة، خاصة بالنسبة للتوريث.

7.  **Key Derivation Function (KDF) API (JEP 487)** - معاينة
    - تقدم واجهة برمجة تطبيقات لوظائف اشتقاق المفاتيح التشفيرية مثل HMAC-based Extract-and-Expand و Argon2، لدعم تجزئة كلمات المرور الآمنة والتفاعل مع الأجهزة التشفيرية.
    - الفائدة: يعزز الأمان للتطبيقات التي تتطلب تشفيرًا متقدمًا.

8.  **Permanently Disable the Security Manager (JEP 486)** - دائمة
    - يزيل Security Manager، الذي تم إهماله في JDK 17، لأنه لم يعد الوسيلة الأساسية لتأمين تطبيقات Java (حل محله العزل القائم على الحاويات).
    - ملاحظة: قد تتطلب التطبيقات التي تعتمد على Security Manager تغييرات معمارية.

9.  **Late Barrier Expansion for G1 Garbage Collector (JEP 464)** - دائمة
    - يبسط تنفيذ الحاجز لـ G1 GC من خلال نقل التوسيع لاحقًا في خطوة تجميع البايت كود، مما يقلل وقت التجميع ويحسن القابلية للصيانة.
    - الفائدة: يعزز الأداء للتطبيقات التي تستخدم G1 GC.

10. **Quantum-Resistant Cryptography (JEP 452, 453)** - معاينة
    - تقدم Module-Lattice-Based Key Encapsulation Mechanism (ML-KEM) و Digital Signature Algorithm (ML-DSA) للحماية من هجمات الحوسبة الكمومية.
    - الفائدة: يضمن مستقبلًا آمنًا لتطبيقات Java في مواجهة الأمان ما بعد الكم.

11. **Scoped Values (JEP 480)** - المعاينة الرابعة
    - يتيح مشاركة البيانات غير القابلة للتغيير داخل وعبر الخيوط بشكل أكثر أمانًا من متغيرات الخيط المحلية، مما يحسن التعامل مع التزامن.
    - الفائدة: يبسط التفكير في الكود المتزامن.

12. **Deprecate 32-bit x86 Port (JEP 501)** - دائمة
    - يهمل منفذ 32-bit x86 لإزالته في JDK 25، مع وجود المنفذ المحايد للبنية Zero كبديل لأنظمة 32-bit.
    - الفائدة: يقلل النفقات العامة للصيانة، مع التركيز على البنى الحديثة.

13. **Vector API (JEP 489)** - الحاضنة التاسعة
    - تواصل تحسين Vector API للبرمجة SIMD، مع تحسينات للعمليات العابرة للمسارات والحسابية.
    - الفائدة: يحسن الأداء للتطبيقات المكثفة في الحساب.

14. **Linking Run-Time Images without JMODs (JEP 493)** - دائمة
    - تسمح لأداة `jlink` بإنشاء صور وقت تشغيل مخصصة دون ملفات JMOD، مما يقلل حجم JDK بنحو ~25%.
    - الفائدة: يعزز كفاءة النشر لوقت تشغيل Java المخصص.

### ملاحظات إضافية

-   **ميزات المعاينة والتجريبية**: العديد من الميزات (مثل Scoped Values، KDF API) في مراحل المعاينة أو التجريبية، مما يسمح للمطورين باختبارها وتقديم ملاحظات قبل أن تصبح دائمة في JDK 25 أو لاحقًا. قد تتغير هذه الميزات قبل التثبيت النهائي.
-   **دمج المشاريع**: يقدم JDK 24 عناصر من مشاريع OpenJDK مثل Leyden (تحسين وقت البدء)، و Lilliput (كفاءة الذاكرة)، و Panama (التشغيل البيني الأصلي)، مما يضع أساسًا للتحسينات المستقبلية.
-   **الأمان والإهمال**: تعكس ميزات مثل إزالة Security Manager وإهمال منفذ 32-bit x86 تركيز أوراكل على تحديث Java من خلال التخلص التدريجي من المكونات القديمة.

### خاتمة

يُعد JDK 24 إصدارًا مستقرًا وغنيًا بالميزات يُطور من أداء Java وإنتاجية المطورين وأمانه. على الرغم من أنه ليس إصدار LTS، فإن مقترحات JEP الأربعة والعشرين الخاصة به تقدم تحسينات كبيرة، وسيتم إقرار العديد منها في إصدار LTS القادم، JDK 25. يمكن للمطورين الاستفادة من ميزات مثل Stream Gatherers و Module Import Declarations لتحقيق مكاسب فورية في الإنتاجية، بينما تشير الميزات التجريبية مثل Compact Object Headers و Ahead-of-Time Class Loading إلى التحسين المستمر لـ Java لأحمال العمل الحديثة. للاستخدام المؤسسي، يُوصى بالالتزام بإصدارات LTS مثل JDK 21 أو انتظار JDK 25 بسبب دعمهم الممتد.
