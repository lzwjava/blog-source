لإزالة Dependabot من GitHub Actions، تحتاج إلى تعطيل قدرته على إنشاء طلبات السحب التي تشغّل سير العمل، وإزالة أي سير عمل مخصصة مرتبطة بـ Dependabot بشكل اختياري. فيما يلي الخطوات لتحقيق ذلك. لاحظ أنني أفترض أن "dependant bot" في استفسارك تشير إلى "Dependabot"، وهو أداة شائعة في GitHub لإدارة التبعيات.

---

### الخطوة 1: تعطيل تحديثات إصدار Dependabot
تقوم تحديثات إصدار Dependabot تلقائيًا بإنشاء طلبات سحب للحفاظ على تحديث تبعياتك. يمكن أن تشغّل طلبات السحب هذه سير عمل GitHub Actions. لتعطيل هذه الميزة:

- **حدد موقع ملف التكوين**: ابحث في مستودعك عن ملف باسم `dependabot.yml` في الدليل `.github`.
- **احذف الملف**: إذا كان الملف موجودًا، فاحذف ملف `dependabot.yml` واحفظ هذا التغيير. هذا يوقف Dependabot من إنشاء طلبات السحب لتحديثات الإصدار.
- **تحقق**: إذا لم يكن ملف `dependabot.yml` موجودًا، فإن تحديثات الإصدار معطلة بالفعل.

---

### الخطوة 2: تعطيل تحديثات أمان Dependabot
تقوم تحديثات أمان Dependabot بإنشاء طلبات سحب لإصلاح الثغرات الأمنية في تبعياتك، والتي يمكنها أيضًا تشغيل سير عمل GitHub Actions. لإيقاف هذا:

- **انتقل إلى إعدادات المستودع**: في مستودع GitHub الخاص بك، انقر فوق علامة التبويب **Settings**.
- **انتقل إلى إعدادات الأمان**: انتقل لأسفل إلى **Security & analysis** (أو **Code security and analysis**، اعتمادًا على واجهة GitHub الخاصة بك).
- **عطّل تحديثات الأمان**: ابحث عن **Dependabot security updates** وانقر على **Disable**.

هذا يمنع Dependabot من إنشاء طلبات سحب لإصلاحات الأمان.

---

### الخطوة 3: (اختياري) إزالة سير العمل المخصصة المرتبطة بـ Dependabot
إذا قمت بإعداد سير عمل GitHub Actions خصيصًا للتعامل مع طلبات سحب Dependabot (مثل الدمج التلقائي، أو وضع العلامات، أو استخدام بيانات وصفية لـ Dependabot)، فقد ترغب في تنظيفها:

- **تحقق من ملفات سير العمل**: ابحث في الدليل `.github/workflows` عن ملفات YAML.
- **حدد منطق Dependabot**: ابحث عن شروط مثل `if: github.event.pull_request.user.login == 'dependabot[bot]'` أو إجراءات مثل `dependabot/fetch-metadata`.
- **أزل أو عدّل**: احذف سير العمل هذه أو عدّلها لإزالة المنطق المخصص لـ Dependabot، ثم احفظ التغييرات.

هذه الخطوة اختيارية لأن تعطيل Dependabot في الخطوتين 1 و 2 سيمنعه من إنشاء طلبات سحب جديدة، مما يعني أن سير العمل الحالية لن يتم تشغيلها بواسطة Dependabot بعد الآن. ومع ذلك، فإن إزالة سير العمل هذه تضمن تنظيفًا كاملاً.

---

### ملاحظات إضافية
- **طلبات السحب الحالية**: تعطيل Dependabot لا يغلق طلبات السحب الحالية التي أنشأها. قد لا تزال هذه تشغّل سير العمل إذا تم تحديثها. يمكنك إغلاقها يدويًا إذا لزم الأمر.
- **تنبيهات Dependabot**: إذا كنت تريد أيضًا إيقاف إشعارات الثغرات الأمنية (غير المرتبطة مباشرة بـ GitHub Actions)، فانتقل إلى **Settings > Security & analysis** وعطّل **Dependabot alerts**. هذا اختياري وغير مرتبط بمشغّلات سير العمل.

---

### النتيجة
بإكمال الخطوتين 1 و 2، لن يقوم Dependabot بعد الآن بإنشاء طلبات سحب في مستودعك، مما يزيل فعليًا تأثيره على GitHub Actions. إذا كنت قد خصصت سير عمل لـ Dependabot، فإن الخطوة 3 تضمن زوال جميع الأتمتة المرتبطة به. ستعمل GitHub Actions الخاصة بك الآن بناءً على أحداث أخرى فقط، دون تأثر بـ Dependabot.
