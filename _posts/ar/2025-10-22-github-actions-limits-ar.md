### حدود استخدام GitHub Actions

يحتوي GitHub Actions على حدود متدرجة بناءً على خطتك (الحسابات الشخصية أو المؤسسات). تشمل هذه الحدود الدقائق المجانية الشهرية لمشغلات GitHub المُستضافة في المستودعات الخاصة والتخزين للقطات artifacts. لا تستهلك المستودعات العامة ومشغلات self-hosted هذه الدقائق. يتم إعادة تعيين الحدود الشهرية للدقائق (بينما التخزين مستمر). يتم فوترة التجاوزات تلقائيًا إذا كان لديك طريقة دفع صالحة؛ وإلا، يتم حظر الوظائف بعد الوصول إلى الحد.

#### الدقائق والتخزين المضمنة حسب الخطة

| الخطة                          | التخزين | الدقائق (شهريًا) |
|-------------------------------|---------|---------------------|
| GitHub Free (شخصي/مؤسسة)   | 500 ميجابايت | 2,000              |
| GitHub Pro (شخصي)        | 1 جيجابايت   | 3,000              |
| GitHub Team (مؤسسة)            | 2 جيجابايت   | 3,000              |
| GitHub Enterprise Cloud (مؤسسة)| 50 جيجابايت  | 50,000             |

- **الدقائق**: تحسب إجمالي وقت تشغيل الوظيفة على مشغلات GitHub المُستضافة (الوقت الجزئي للوظائف الفاشلة). تنطبق مضاعفات: Linux (1x)، Windows (2x)، macOS (10x). أي شخص لديه صلاحية كتابة على المستودع يستخدم بدل مالك المستودع.
- **التخزين**: يعتمد على ساعات-الجيجابايت لتخزين القطع الأثرية (مثل التحميلات/التنزيلات). لا تحتسب السجلات والملخصات.

#### فوترة التجاوزات
إذا تجاوزت الحصص:
- **الدقائق**: يتم التحصيل لكل دقيقة إضافية (أسعار USD لمشغلات Linux القياسية: 0.008 دولار/دقيقة؛ أعلى لنظامي Windows و macOS بسبب المضاعفات).
- **التخزين**: يتم التحصيل كأشهر-جيجابايت (إجمالي ساعات-الجيجابايت / 744 ساعة شهريًا، مع التقريب لأعلى).
- مثال: في خطة Team، 5000 دقيقة إضافية (3000 Linux + 2000 Windows) تكلف حوالي 56 دولارًا.

بدون طريقة دفع، يتوقف الاستخدام عند الحصة. الخطأ الذي رأيته ("فشلت مدفوعات الحساب الأخيرة أو يجب زيادة حد الإنفاق الخاص بك") يعني إما أن المدفوعات فشلت أو أن **حد الإنفاق (الميزانية)** الخاص بك يحد من التجاوزات.

#### حدود الإنفاق (الميزانيات)
- هذه عتبات اختيارية للتحكم في إجمالي الإنفاق الشهري على Actions (والميزات الأخرى ذات العدادات).
- إذا تم تعيينها منخفضة جدًا، فإنها تمنع الوظائف قبل الوصول إلى الحصص.
- **كيفية التحقق/الزيادة**:
  1. انتقل إلى إعدادات GitHub > الفوترة والخطط.
  2. ضمن "الميزانيات والتنبيهات"، اعرض أو حرر ميزانيتك لـ GitHub Actions.
  3. زِد الحد، أضف طريقة دفع، أو أزل الميزانية بالكامل (تجاوزات غير محدودة، ولكن راقب عبر التنبيهات).
- قد تحتوي الحسابات التي تم ترحيلها على ميزانيات مُنشأة تلقائيًا من الحدود القديمة — قم بتحديثها لتناسب استخدامك.

للحصول على التفاصيل الكاملة، تحقق من لوحة الفوترة لحسابك للاطلاع على الاستخدام في الوقت الفعلي.
