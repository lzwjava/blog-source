### النصائح والأنماط النموذجية في ملفات C++ لبنيامين كي
تم تحسين كود C++ الخاص بكي للسرعة (في الكتابة والتنفيذ على حد سواء)، والصحة، والحد الأدنى من تصحيح الأخطاء أثناء المسابقات. غالبًا ما تتضمن ملفاته تعليقات تعمل كتذكيرات لتجنب الأخطاء الشائعة أو لفرض عادات جيدة. فيما يلي الجوانب الرئيسية لنهجه:

1.  **هيكل كود بسيط ولكن واضح**:
    *   **النصيحة**: يشدد كي على كتابة كود موجز يكون مقروءًا بما يكفي لمدة المسابقة، حيث تفضل البرمجة التنافسية السرعة على القابلية للصيانة. تكون التعليقات قليلة ولكنها إستراتيجية، تركز على المنطق الحرج أو نقاط الخطأ المحتملة.
    *   **التعليقات النموذجية**:
        *   `// check bounds` أو `// array size`: تذكيرات للتحقق من فهارس المصفوفات أو أحجامها لتجنب أخطاء تجاوز الحدود، وهي مشكلة شائعة في C++.
        *   `// int overflow?`: تذكير للنظر فيما إذا كانت العمليات على الأعداد الصحيحة قد تتجاوز حدود `int` (مثل 2^31 - 1)، مما يشير غالبًا إلى استخدام `long long`.
        *   `// mod arithmetic`: ملاحظة لضمان معالجة الحساب المعياري بشكل صحيح، خاصة في المشكلات التي تتضمن أعدادًا كبيرة.

2.  **استخدام وحدات الماكرو والقوالب (Macros and Templates)**:
    *   **النصيحة**: يدعو كي إلى استخدام وحدات الماكرو والقوالب لتقليل الكتابة وتسريع الترميز، لكنه يحذر من الإفراط في استخدامها للحفاظ على قابلية القراءة. غالبًا ما تتضمن ملفاته قالبًا مكتوبًا مسبقًا يحتوي على أدوات شائعة (مثل الحلقات، وهياكل البيانات).
    *   **التعليقات النموذجية**:
        *   `// #define FOR(i,a,b) ...`: تعريف ماكرو حلقة مثل `FOR(i,a,b)` للتكرار من `a` إلى `b`، مع تعليق لتوضيح الغرض منه أو التحذير من سوء الاستخدام.
        *   `// careful with macro args`: تذكير لتجنب الآثار الجانبية في وسائط الماكرو (مثل `i++` في الماكرو).
        *   `// template for min/max`: تعليقات فوق دوال القالب مثل `chmin` أو `chmax` لتذكيره باستخدامها لتحديث القيم الدنيا/القصوى بكفاءة.

3.  **التركيز على تجنب الأخطاء (Bugs)**:
    *   **النصيحة**: يتضمن كود كي فحوصات لأخطاء البرمجة التنافسية الشائعة، مثل أخطاء off-by-one، أو المتغيرات غير المهيأة، أو معالجة الإدخال غير الصحيحة. غالبًا ما تبرز تعليقاته هذه المشكلات المحتملة.
    *   **التعليقات النموذجية**:
        *   `// 0-based or 1-based?`: تذكير للتأكد مما إذا كانت المشكلة تستخدم فهرسة قائمة على 0 أو على 1، خاصة لمشكلات الرسم البياني أو المصفوفات.
        *   `// init variables`: ملاحظة لضمان تهيئة جميع المتغيرات، خاصة للمصفوفات أو المجاميع.
        *   `// edge cases`: تذكير للنظر في الحالات الخاصة، مثل المدخلات الفارغة، أو حالات العنصر الواحد، أو القيم المتطرفة (مثل `n = 1` أو `n = 10^5`).

4.  **إدخال/إخراج فعال (Efficient Input/Output)**:
    *   **النصيحة**: يستخدم كي تقنيات I/O سريعة لتجنب أخطاء تجاوز المهلة (TLE)، مثل `ios::sync_with_stdio(0)` و `cin.tie(0)`. قد يعلق على هذه لتذكير نفسه بضرورتها.
    *   **التعليقات النموذجية**:
        *   `// fast I/O`: فوق أسطر تحسين I/O للتأكد من تضمينها.
        *   `// endl vs \n`: تذكير باستخدام `\n` بدلاً من `endl` لإخراج أسرع.
        *   `// read carefully`: ملاحظة لضمان معالجة تنسيق الإدخال (مثل عدد حالات الاختبار، المسافات البيضاء) بشكل صحيح.

5.  **كود معياري وقابل لإعادة الاستخدام (Modular and Reusable Code)**:
    *   **النصيحة**: غالبًا ما تتضمن ملفات كي مكونات قابلة لإعادة الاستخدام مثل دوال الحساب المعياري، أو خوارزميات الرسم البياني، أو هياكل البيانات (مثل أشجار القطاعات). تساعده التعليقات على تكييف هذه بسرعة لمشكلات محددة.
    *   **التعليقات النموذجية**:
        *   `// mod = 1e9+7`: ملاحظة تحدد المعامل (modulus) للعمليات الحسابية، شائعة في المشكلات التوافقية.
        *   `// precompute`: تذكير لحساب القيم مسبقًا (مثل المضروبات، المعكوسات) للكفاءة.
        *   `// copy-paste from library`: تعليق يشير إلى كتلة كود أعيد استخدامها من مكتبته الشخصية، مما يضمن له التحقق من قابلية تطبيقها.

6.  **الوعي بمدى تعقيد الزمن والمساحة (Time and Space Complexity Awareness)**:
    *   **النصيحة**: يحرص كي بدقة على ضمان أن تلبي حلوله قيود الزمن والمساحة. غالبًا ما تعكس تعليقاته حسابات أو تذكيرات حول المدى التعقيدي.
    *   **التعليقات النموذجية**:
        *   `// O(n log n)`: ملاحظة حول مدى التعقيد الزمني المتوقع للخوارزمية.
        *   `// memory limit`: تذكير للتحقق مما إذا كانت المساحة المستخدمة (مثل المصفوفات الكبيرة) تتلاءم مع قيود المشكلة.
        *   `// bottleneck`: تعليق يحدد الجزء الأبطأ في الكود الذي قد يحتاج إلى تحسين.

7.  **تصحيح الأخطاء والاختبار (Debugging and Testing)**:
    *   **النصيحة**: بينما لا تسمح البرمجة التنافسية بتصحيح أخطاء مكثف أثناء المسابقات، يدرج كي تعليقات لتسهيل الفحوصات السريعة أو لتحديد المناطق التي تحتاج إلى التحقق.
    *   **التعليقات النموذجية**:
        *   `// debug`: فوق جملة طباعة مؤقتة (مثل `cerr`) تُستخدم لفحص قيم المتغيرات.
        *   `// test small cases`: تذكير للتحقق من الكود ذهنيًا أو يدويًا على مدخلات صغيرة.
        *   `// check sample`: ملاحظة لمقارنة المخرجات مع حالات العينة الخاصة بالمشكلة.

### مثال على ملف C++ لبنيامين كي مع التعليقات
فيما يلي مثال افتراضي لكيفية ظهور ملف C++ الخاص بكي لمشكلة برمجة تنافسية، يدمج نصائحه النموذجية وأسلوب التعليق (مستوحى من مستودعه على GitHub ومساهماته في USACO Guide):

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

// fast I/O
#define FOR(i,a,b) for (int i = (a); i < (b); ++i)
const ll MOD = 1e9 + 7; // mod = 1e9+7

int main() {
    ios::sync_with_stdio(0); cin.tie(0); // fast I/O
    int t; cin >> t; // read carefully
    while (t--) {
        int n; cin >> n;
        vector<ll> a(n); // check bounds
        FOR(i,0,n) cin >> a[i];
        
        // init variables
        ll sum = 0;
        FOR(i,0,n) {
            sum = (sum + a[i]) % MOD; // mod arithmetic
            // int overflow?
            if (sum < 0) sum += MOD; // edge cases
        }
        
        // O(n) complexity
        cout << sum << '\n'; // endl vs \n
        // check sample
    }
    return 0;
}
```

### تذكيرات شائعة في التعليقات
تعليقات كي موجزة ومركزة على المشكلة، غالبًا ما تتناول:
*   **الصحة**: `// check if sorted`, `// handle negative numbers`.
*   **الكفاءة**: `// reduce to O(n)`, `// avoid TLE`.
*   **الفروق الدقيقة الخاصة بالمشكلة**: `// multiple test cases?`, `// undirected graph?`.
*   **العادات الشخصية**: `// don’t rush`, `// think before coding`، مما يعكس نهجه المنضبط لتجنب الأخطاء المتسرعة.
