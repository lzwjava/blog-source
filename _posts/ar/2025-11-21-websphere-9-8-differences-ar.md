### الاختلافات الرئيسية بين IBM WebSphere Application Server (WAS) الإصدار 9.0 و 8.0

IBM WebSphere Application Server (WAS) هو خادم تطبيقات Java EE من المستوى المؤسسي. قدم الإصدار 8.0 (الذي صدر في 2011) التوافق مع Java EE 6، بينما تقدم الإصدار 9.0 (الذي صدر في 2017) إلى Java EE 7 مع التركيز على تحديث البروفايل التقليدي ليتوافق مع أنظمة التشغيل الأخف والأكثر ملاءمة للسحابة مثل Liberty. أدناه، سأوضح الاختلافات الرئيسية في جدول من أجل الوضوح، استنادًا إلى الوثائق الرسمية من IBM وملاحظات الإصدار. وتشمل هذه الاختلافات دعم Java والامتثال للمعايير والهندسة المعمارية والنشر.

| الجانب                  | WAS 8.0                                                                 | WAS 9.0                                                                 |
|-------------------------|-------------------------------------------------------------------------|-------------------------------------------------------------------------|
| **دعم Java SE**    | يستخدم Java SE 6 افتراضيًا؛ دعم اختياري لـ Java SE 7 عبر التكوين. | يستخدم Java SE 8 افتراضيًا كمنصة أساسية، باستخدام IBM SDK Java 8 للحصول على توافق كامل مع Oracle Java 8. هذا يمكن من استخدام تعبيرات لامدا، ستريمز، وميزات SE 8 الأخرى. |
| **الامتثال لـ Java EE** | دعم كامل لـ Java EE 6، بما في ذلك JPA 2.0 و JSF 2.0 و Servlet 3.0.    | دعم كامل لـ Java EE 7، مضيفًا ميزات مثل WebSocket 1.0 و JSON-P 1.0 و Batch 1.0 وأدوات التزامن المحسنة. هذا يرفع الإصدار التقليدي ليصبح مكافئًا لقدرات Liberty من الإصدارات السابقة. |
| **دمج بروفايل Liberty** | تم تقديم Liberty في الإصدار 8.5 (وليس في نواة الإصدار 8.0)؛ يركز الإصدار 8.0 على البروفايل الكامل التقليدي فقط. | نظام تشغيل Liberty (الإصدار 16.0.0.2) مدمج بعمق كبديل خفيف الوزن و modular للبروفايل الكامل، مُحسَّن للتطبيقات السحابية الأصلية. Liberty مُضمن ويدعم التوصيل المستمر. |
| **نموذج النشر**   | في المقام الأول داخل الموقع (On-premises)؛ يتم تثبيته عبر Installation Manager مع إصدارات مثل Base و Network Deployment (ND) للتجميع (Clustering). | أول إصدار يتم إصداره في وقت واحد داخل الموقع وكتخدمة على IBM Cloud. يدعم عمليات النشر الهجينة السحابية مع خطاطيف أفضل للتعبئة (Containerization). |
| **الأداء والإدارة** | تحقيق ما يصل إلى 20-26% مكاسب في الإنتاجية مقارنة بـ WAS 7؛ إدارة ذكية في إصدار ND. | يبني على الإصدار 8.0 مع تحسينات إضافية لكفاءة الموارد؛ أدوات إدارية محسنة للهجرة ومقارنة التكوين. |
| **نهاية الدعم**     | انتهى الدعم الموسع في 2019؛ لم يعد يتلقى إصلاحات.              | دعم نشط حتى 2027 على الأقل، مع حزم إصلاح منتظمة (مثل 9.0.5.x) تعالج الأمان والتوافق. |
| **اعتبارات الهجرة** | غير مطبق (خط الأساس).                                                        | هجرة أسهل من الإصدار 8.x عبر أدوات مثل أداة مقارنة التكوين؛ محاذاة منافذ تلقية ومعالجة واجهات برمجة التطبيقات (APIs) المهملة للانتقال إلى Java SE 8. |

#### النقاط الرئيسية
- **القفزة الأكبر**: الانتقال إلى Java EE 7 و Java SE 8 هو الأكثر أهمية، مما يمكن من استخدام ميزات ومعايير Java الحديثة التي تم عرضها مسبقًا في Liberty. إذا كنت تشغل تطبيقات Java EE 6 قديمة، فإن الهجرة إلى الإصدار 9.0 هي عملية مباشرة ولكنها قد تتطلب اختبارًا للتغييرات الخاصة بـ SE 8.
- **متى تقوم بالتحديث**: اختر الإصدار 9.0 للاحتياجات السحابية/الهجينة أو ميزات EE 7؛ التزم باستخدام الإصدار 8.0 (أو قم بتطبيقه إلى الإصدار 8.5.5) فقط للصيانة قصيرة الأجل للتطبيقات الأقدم.
- للحصول على أدلة الهجرة التفصيلية، راجع الوثائق الرسمية من IBM.
