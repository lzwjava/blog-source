لإزالة نموذج في إضافة GitHub Copilot لـ Visual Studio Code (VS Code)، لاحظ أن النماذج المضمنة (مثل نموذج GitHub Copilot الافتراضي) لا يمكن إزالتها، لأنها جزء من الإضافة. ومع ذلك، إذا كنت تشير إلى نموذج مخصص أضفته عبر موفر طرف ثالث (مثل Anthropic أو OpenAI أو Google Gemini أو غيرها) باستخدام مفتاح API الخاص بك، فيمكنك إزالته فعليًا عن طريق حذف مفتاح API المرتبط به وتفاصيل الموفر. هذه الميزة قيد المعاينة وقد لا تكون متاحة بعد لمستخدمي Copilot Business أو Enterprise.

إليك الخطوات بناءً على الوثائق الرسمية:

1.  افتح نافذة الدردشة في VS Code (إذا لم تكن مفتوحة بالفعل، استخدم لوحة الأوامر عبر `Ctrl+Shift+P` أو `Cmd+Shift+P` على نظام macOS، ثم ابحث عن "Copilot: Open Chat").
2.  في نافذة الدردشة، انقر على القائمة المنسدلة لاختيار النموذج (بجوار حقل الإدخال) وحدد **Manage Models**. بدلاً من ذلك، افتح لوحة الأوامر وقم بتنفيذ أمر **GitHub Copilot: Manage Models**.
3.  في واجهة إدارة النماذج، مرر مؤشر الفأرة فوق موفر النموذج الذي تريد إزالته وانقر على أيقونة الترس لتحرير تفاصيله.
4.  احذف مفتاح API (وأي عنوان URL للنقطة الطرفية إذا كان مطبقًا) من الحقول، ثم احفظ التغييرات. يجب أن يزيل هذا توفر النموذج من القائمة المنسدلة.

إذا واجهت مشاكل (مثل عدم تحديث حقل مفتاح API أو مسحه بشكل صحيح، وهو ما تم الإبلاغ عنه كخطأ في بعض الحالات)، جرب هذا الحل أولاً:
*   حدد نموذجًا مختلفًا من القائمة المنسدلة للنماذج في نافذة الدردشة لإلغاء تحديد النموذج الذي تحاول إزالته.
*   عد إلى إدارة النماذج، وقم بتحرير إعدادات الموفر، ثم احذف مفتاح API.

المتطلبات الأساسية والملاحظات:
*   هذا ينطبق فقط على ميزات الدردشة؛ فهو لا يؤثر على إكمالات الكود أو الأدوات الأخرى.
*   تأكد من أن اشتراكك في Copilot يسمح بالوصول إلى نماذج متعددة (مثل Individual أو Pro أو ميزات المعاينة المفعلة للمؤسسات).
*   إذا استمر ظهور النموذج أو واجهت أخطاء، أعد تشغيل VS Code أو تحقق من وجود تحديثات للإضافة.

إذا كان هذا لا يتطابق مع ما تعنيه بـ "إزالة نموذج" (على سبيل المثال، إذا كنت تحاول تعطيل الإضافة بالكامل أو إخفاء ميزات Copilot)، فقم بتوضيح طلبك للحصول على خطوات أكثر تخصيصًا.
