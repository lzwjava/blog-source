يعمل Claude Code، المساعد البرمجي المدعوم بالذكاء الاصطناعي من Anthropic، بشكل أساسي ضمن دليل العمل الذي تم تهيئته من أجل الأمن والتركيز، ولكنه يدعم الآن وظائف المشاريع المتقاطعة بالكامل من خلال ميزة الدلائل المتعددة التي تم تقديمها في التحديثات الأخيرة (مثلاً، الإصدار 1.0.18 وما بعده). هذا يسمح لجلسة واحدة بالوصول إلى الملفات والعمل معها عبر مشاريع أو أدلة متعددة غير مرتبطة خارج الدليل الأصلي، مما يتيح إنجاز مهام مثل الرجوع إلى التكوينات المشتركة، أو دمج كود الواجهة الأمامية/الخلفية، أو التعامل مع إعدادات MonoRepo دون الحاجة إلى تبديل السياق.[1][2][3]

### كيف تعمل وظيفة المشاريع المتقاطعة
- **الآلية الأساسية**: يبدأ Claude Code في دليل جذر واحد (مشروعك "الرئيسي") ولكن يمكنه توسيع الأذونات لقراءة وتحرير وتنفيذ الأوامر في أدلة إضافية. يتم ذلك عبر علم `--add-dir` أو الأمر التفاعلي `/add-dir` أثناء الجلسة. يتم التعامل مع الأدلة المضافة كامتدادات لمساحة العمل، مما يسمح بعمليات ملفات سلسة (مثلاً، يمكنك فحص الكود (lint) لملفات من المشروع A أثناء التحرير في المشروع B).[3][4]
- **نطاق الجلسة**: كل إضافة مشروع مؤقتة ما لم يتم الإبقاء عليها عبر التكوين. يمكن لـ Git worktrees تمكين جلسات متزامنة على أجزاء من مشروع للتعاون الأعمق.[5][6]
- **القيود**: يقيد Claude Code الوصول إلى الملفات على الأدلة المضافة فقط — لن يكتشف تلقائيًا المسارات غير المرتبطة. للإعدادات الدائمة للمشاريع المتعددة (مثلاً، mono repos)، شغّله من دليل أب يحتوي على مجلدات فرعية.[3][7]
- **حالات الاستخدام**:
  - **MonoRepos**: أضف المجلدات الفرعية لتقسيمات الواجهة الأمامية/الخلفية.[3][5][7][8]
  - **الموارد المشتركة**: الرجوع إلى تكوينات أو مكتبات من مشروع منفصل.[3][6]
  - **التعاون عبر المشاريع**: دمج الكود من مكتبات أو أدوات في مستودعات مختلفة.[1][3]

### كيفية توجيه Claude Code لإشراك مشروع آخر
لإضافة مشروع خارج الدليل الحالي (مثلاً، `${another_project_path}`):

1. **ابدأ Claude Code** في دليل مشروعك الرئيسي (مثلاً، `cd /path/to/primary/project && claude`).
2. **أضف الدليل الإضافي تفاعلياً**:
   - أثناء الجلسة، اكتب `/add-dir /full/path/to/another/project` أو مسار نسبي (مثلاً، `../another-project`).
   - سؤكد Claude Code الوصول — أجب بـ "نعم" إذا تمت المطالبة بذلك.[2][3][4]
3. **عند بدء التشغيل عبر علم CLI** (لإعداد متعدد الأدلة فوري):
   - شغّل: `claude --add-dir /path/to/another/project` (أضف عدة أدلة بتكرار العلم).[4][5][7]
4. **وجّه بوتات/وكلاء Claude**: بمجرد الإضافة، أعط تعليمات لغة طبيعية مثل "ارجع إلى ملفات API من الدليل المضاف في `/path/to/another/project`" أو "ادمج هذا التكوين المشترك من المشروع B." تصميم Claude الوكيلي يفهم هذه الطلبات ضمن السياق الموسع.[3][5][6]

### أفضل الممارسات والنصائح
- **تحديد المسار**: استخدم المسارات المطلقة للوضوح عبر البيئات المختلفة. تجنب إضافة الكثير من الأدلة لمنع التأثير على الأداء.[3][7]
- **إعداد مساحة العمل**: للفرق، فكر في ملفات التكوين للإعدادات القابلة لإعادة الاستخدام. اجمعها مع أدوات مثل Apidog لاختبار API عبر الأدلة.[3][8]
- **استكشاف الأخطاء وإصلاحها**: إذا فشل حل المسار، تأكد من أذونات الكتابة. أعد تشغيل الجلسات للتغييرات. بالنسبة للمشاكل المستمرة، تحقق من السجلات أو شغّله من دليل أعلى مستوى.[3][7]
- **سير عمل الفرق**: استخدم هذا للخدمات المصغرة (microservices) أو المكتبات المشتركة؛ زامن عبر Git لإدارة الإصدارات.[5][6][8]

هذه الميزة تحول سير العمل للإعدادات المعقدة، مما يقلل الاحتكاك مقارنة بتشغيل جلسات منفصلة في كل دليل.[3][6] إذا واجهت مشاكل أو تحتاج لأمثلة لأدوات محددة، قدم المزيد من التفاصيل!
