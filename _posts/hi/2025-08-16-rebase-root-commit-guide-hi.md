`git rebase --root` एक विशेष प्रकार का रीबेस है जो आपको अपने रिपॉजिटरी के **बहुत पहले कमिट** को रीराइट करने देता है। आमतौर पर, `git rebेस` को रीबेस करने के लिए एक पैरेंट कमिट की आवश्यकता होती है, लेकिन `--root` विकल्प रूट कमिट (पहली कमिट जिसका कोई पैरेंट नहीं है) के साथ ऐसा व्यवहार करता है जैसे उसे भी रीबेस किया जा रहा है।

यहां बताया गया है कि यह व्यवहार में कैसे काम करता है:

### 1. स्टैंडर्ड रीबेस बनाम रूट रीबेस

* **सामान्य `git rebase`**: आप किसी दूसरी ब्रांच या कमिट के ऊपर कमिट्स को दोबारा एप्लाई करते हैं, जिससे चुने गए बेस के बाद का इतिहास बदल जाता है।
* **`git rebase --root`**: यह रूट के बाद शुरू होने के बजाय, रूट को ही रीराइट कर देता है। आप इसे स्प्लिट कर सकते हैं, बाद की कमिट्स के साथ स्क्वॉश कर सकते हैं, या पूरी तरह से रिप्लेस भी कर सकते हैं।

### 2. विशिष्ट उपयोग के मामले

* **पहली कमिट मैसेज को एडिट करना**
  अगर आप अपने प्रोजेक्ट की सबसे पहली कमिट मैसेज को रीराइट करना चाहते हैं, तो आप चला सकते हैं:

  ```bash
  git rebase --root -i
  ```

  और पहली कमिट के लिए `pick` को `reword` में बदल सकते हैं।

* **रूट को बाद की कमिट्स के साथ स्क्वॉश करना**
  उदाहरण के लिए, अगर आपकी पहली कमिट सिर्फ स्कैफोल्डिंग (जैसे कि एक `README`) थी, तो आप इतिहास को साफ करने के लिए इसे अगली कमिट में स्क्वॉश कर सकते हैं।

* **रूट कमिट को स्प्लिट करना**
  अगर आपकी पहली कमिट में बहुत अधिक कंटेंट था (जैसे, पूरा प्रोजेक्ट कोड डंप), तो आप इंटरएक्टिव रीबेस के दौरान इसे कई लॉजिकल कमिट्स में स्प्लिट कर सकते हैं।

### 3. आंतरिक रूप से क्या होता है

* Git रूट कमिट के लिए एक अस्थायी "नकली" पैरेंट कमिट बनाता है ताकि वह रीबेस ऑपरेशन्स को शुरुआत से एप्लाई कर सके।
* प्रत्येक कमिट को दोबारा चलाया (रीबेस) जाता है, मानो रिपॉजिटरी अलग तरीके से शुरू हुई हो।
* आप अनिवार्य रूप से अपने रिपो के इतिहास की नींव को फिर से लिख रहे हैं।

### 4. उदाहरण वर्कफ़्लो

```bash
git rebase --root -i
```

आपको एक एडिटर इस तरह दिखेगा:

```
pick a1b2c3d Initial commit
pick d4e5f6g Add feature A
pick h7i8j9k Add feature B
```

यहां से आप यह कर सकते हैं:

* रूट कमिट के लिए `pick` को `reword` में बदलें → इसकी मैसेज बदलें।
* `pick` को `squash` में बदलें → रूट को अगली कमिट के साथ मर्ज करें।
* जरूरत के हिसाब से ऑर्डर बदलें या स्प्लिट करें।

### ⚠️ महत्वपूर्ण नोट

`git rebase --root` **इतिहास को रीराइट करता है**, इसलिए इसका उपयोग केवल उन्हीं ब्रांचों पर करना चाहिए जिन्हें पुश/शेयर नहीं किया गया है, या फिर आपको बाद में फोर्स-पुश करना होगा। शेयर्ड ब्रांचों पर, यह सहयोगियों के लिए कन्फ्यूजन और कॉन्फ्लिक्ट्स पैदा कर सकता है।
