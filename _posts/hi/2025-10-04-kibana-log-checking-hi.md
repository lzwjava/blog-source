किबाना एक ओपन-सोर्स डेटा विज़ुअलाइज़ेशन और एक्सप्लोरेशन टूल है जिसे Elastic द्वारा विकसित किया गया है। यह **ELK स्टैक** (Elasticsearch, Logstash, Kibana) का एक प्रमुख घटक है, जिसका उपयोग केंद्रीकृत लॉगिंग, मॉनिटरिंग और एनालिटिक्स के लिए व्यापक रूप से किया जाता है। किबाना Elasticsearch में संग्रहीत डेटा के साथ इंटरैक्ट करने के लिए एक वेब-आधारित इंटरफेस प्रदान करता है, जो आपको लॉग को रीयल-टाइम में खोजने, विज़ुअलाइज़ करने और विश्लेषण करने की अनुमति देता है। यह एप्लिकेशन, सर्वर या क्लाउड सेवाओं के लॉग जांचने के लिए विशेष रूप से शक्तिशाली है।

यह गाइड लॉगों का निरीक्षण और क्वेरी करने के लिए किबाना का उपयोग करने पर केंद्रित है। हम सेटअप, बेसिक उपयोग, लॉग जांच वर्कफ़्लो और उन्नत टिप्स को कवर करेंगे। मान लें कि आप एक बुनियादी ELK सेटअप के साथ काम कर रहे हैं; यदि आप ELK में नए हैं, तो पहले Elasticsearch और Logstash इंस्टॉल करके शुरुआत करें (किबाना के काम करने के लिए Elasticsearch आवश्यक है)।

## 1. आवश्यक शर्तें
किबाना का उपयोग करने से पहले:
- **Elasticsearch**: वर्जन 8.x या बाद का (किबाना Elasticsearch वर्जन के साथ कसकर जुड़ा हुआ है)। [elastic.co](https://www.elastic.co/downloads/elasticsearch) से डाउनलोड करें।
- **Java**: Elasticsearch के लिए JDK 11 या बाद का आवश्यक है।
- **सिस्टम आवश्यकताएँ**: डेवलपमेंट के लिए कम से कम 4GB RAM; प्रोडक्शन के लिए अधिक।
- **डेटा स्रोत**: Logstash, Filebeat के माध्यम से या सीधे Elasticsearch में इन्जेस्ट किए गए लॉग (जैसे, टाइमस्टैम्प के साथ JSON फॉर्मेट)।
- **नेटवर्क एक्सेस**: किबाना डिफ़ॉल्ट रूप से पोर्ट 5601 पर चलता है; सुनिश्चित करें कि यह एक्सेसिबल है।

यदि आपके पास अभी तक लॉग नहीं हैं, तो Elasticsearch में सैंपल लॉग (जैसे, सिस्टम लॉग) शिप करने के लिए Filebeat जैसे टूल्स का उपयोग करें।

## 2. किबाना इंस्टॉल करना
किबाना इंस्टॉलेशन सीधा है और प्लेटफॉर्म-अज्ञेय है। नवीनतम वर्जन [elastic.co/downloads/kibana](https://www.elastic.co/downloads/kibana) से डाउनलोड करें (अपने Elasticsearch वर्जन से मिलान करें)।

### लिनक्स पर (Debian/Ubuntu):
1. Elastic की रिपॉजिटरी जोड़ें:
   ```
   wget -qO - https://artifacts.elastic.co/GPG-KEY-elasticsearch | sudo apt-key add -
   sudo apt-get install apt-transport-https
   echo "deb https://artifacts.elastic.co/packages/8.x/apt stable main" | sudo tee /etc/apt/sources.list.d/elastic-8.x.list
   sudo apt-get update && sudo apt-get install kibana
   ```
2. किबाना स्टार्ट करें:
   ```
   sudo systemctl start kibana
   sudo systemctl enable kibana  # बूट पर ऑटो-स्टार्ट के लिए
   ```

### विंडोज पर:
1. ZIP आर्काइव डाउनलोड करें और इसे `C:\kibana-8.x.x-windows-x86_64` में एक्सट्रैक्ट करें।
2. एडमिनिस्ट्रेटर के रूप में कमांड प्रॉम्प्ट खोलें और एक्सट्रैक्टेड फोल्डर पर नेविगेट करें।
3. रन करें: `bin\kibana.bat`

### macOS पर:
1. Homebrew का उपयोग करें: `brew tap elastic/tap && brew install elastic/tap/kibana-full`।
2. या TAR.GZ डाउनलोड करें, एक्सट्रैक्ट करें, और `./bin/kibana` रन करें।

Docker के लिए: ऑफिशियल इमेज का उपयोग करें:
```
docker run --name kibana -p 5601:5601 -e ELASTICSEARCH_HOSTS=http://elasticsearch:9200 docker.elastic.co/kibana/kibana:8.10.0
```

## 3. बेसिक कॉन्फ़िगरेशन
कॉन्फ़िगरेशन फ़ाइल `kibana.yml` (लिनक्स पर `/etc/kibana/` में स्थित, या अन्य पर `config/` फोल्डर) एडिट करें।

लॉग जांच के लिए मुख्य सेटिंग्स:
```yaml
# Elasticsearch से कनेक्ट करें (डिफ़ॉल्ट localhost:9200 है)
elasticsearch.hosts: ["http://localhost:9200"]

# सर्वर सेटिंग्स
server.port: 5601
server.host: "0.0.0.0"  # रिमोट एक्सेस के लिए सभी इंटरफेस से बाइंड करें

# सुरक्षा (प्रोडक्शन के लिए एनेबल करें)
# elasticsearch.username: "elastic"
# elasticsearch.password: "your_password"

# लॉगिंग
logging.verbose: true  # किबाना को डीबग करने के लिए

# इंडेक्स पैटर्न (ऑप्शनल डिफ़ॉल्ट)
defaultIndex: "logs-*"
```
- बदलावों के बाद किबाना रीस्टार्ट करें: `sudo systemctl restart kibana`।
- यदि सुरक्षा फीचर्स (X-Pack) का उपयोग कर रहे हैं, तो सर्टिफिकेट जेनरेट करें या बेसिक ऑथ का उपयोग करें।

## 4. किबाना स्टार्ट करना और एक्सेस करना
- पहले Elasticsearch स्टार्ट करें (जैसे, `sudo systemctl start elasticsearch`)।
- ऊपर बताए अनुसार किबाना स्टार्ट करें।
- एक वेब ब्राउज़र खोलें और `http://localhost:5601` (या आपके सर्वर का IP:5601) पर जाएँ।
- पहली लॉगिन पर, आपको सेटअप विजार्ड दिखेगा। यदि संकेत दिया जाए तो एक एडमिन यूजर बनाएँ (डिफ़ॉल्ट: elastic/changeme)।

इंटरफेस में **Discover** (लॉग के लिए), **Visualize**, **Dashboard**, **Dev Tools**, और **Management** जैसे ऐप्स शामिल हैं।

## 5. डेटा तैयार करना: इंडेक्स पैटर्न
Elasticsearch में लॉग **इंडेक्स** (जैसे, `logs-2023-10-01`) में संग्रहीत होते हैं। किबाना में उन्हें क्वेरी करने के लिए, एक **इंडेक्स पैटर्न** बनाएँ।

1. **Stack Management** > **Index Patterns** (लेफ्ट साइडबार, हैमबर्गर मेन्यू > Management) पर जाएँ।
2. **Create index pattern** पर क्लिक करें।
3. `logs-*` (सभी लॉग इंडेक्स से मेल खाता है) या `filebeat-*` (Filebeat लॉग के लिए) जैसा पैटर्न दर्ज करें।
4. **Time field** चुनें (जैसे, लॉग टाइमस्टैम्प के लिए `@timestamp`—टाइम-आधारित क्वेरीज़ के लिए महत्वपूर्ण)।
5. **Create index pattern** पर क्लिक करें।
   - यह `message` (लॉग टेक्स्ट), `host.name`, `level` (error/warn/info) आदि जैसे फील्ड्स को मैप करता है।

यदि आपके लॉग बदलते हैं तो फील्ड्स रिफ्रेश करें। प्रीव्यू के लिए **Discover** का उपयोग करें।

## 6. लॉग जांचने के लिए Discover का उपयोग करना
**Discover** ऐप लॉगों का निरीक्षण करने के लिए आपका प्राथमिक टूल है। यह एक सर्च करने योग्य लॉग व्यूअर की तरह है।

### बेसिक नेविगेशन:
1. लेफ्ट साइडबार में **Discover** पर क्लिक करें।
2. ड्रॉपडाउन (टॉप-लेफ्ट) से अपना इंडेक्स पैटर्न चुनें।
3. टाइम रेंज सेट करें (टॉप-राइट): "Last 15 minutes" जैसे क्विक ऑप्शन या कस्टम (जैसे, Last 7 days) का उपयोग करें। यह लॉग को `@timestamp` द्वारा फ़िल्टर करता है।

### लॉग देखना:
- **Hit Count**: कुल मिलान करने वाले लॉग दिखाता है (जैसे, 1,234 hits)।
- **Document Table**: JSON या फॉर्मेटेड टेक्स्ट के रूप में रॉ लॉग एंट्रीज़ दिखाता है।
  - कॉलम: डिफ़ॉल्ट `@timestamp` और `_source` (पूरा लॉग) है। कॉलम जोड़ने के लिए लेफ्ट साइडबार से फील्ड्स (जैसे, `message`, `host.name`) खींचें।
  - पूरा JSON डॉक्यूमेंट देखने के लिए एक पंक्ति को एक्सपैंड करें (तीर पर क्लिक करें)।
- **हिस्टोग्राम**: टॉप चार्ट समय के साथ लॉग वॉल्यूम दिखाता है। ड्रैग करके ज़ूम करें।

### लॉग सर्च करना:
क्वेरीज़ के लिए सर्च बार (टॉप) का उपयोग करें। किबाना डिफ़ॉल्ट रूप से **KQL (Kibana Query Language)** का उपयोग करता है—सरल और सहज।

- **बेसिक सर्च**:
  - कीवर्ड के लिए सर्च करें: `error` ("error" वाले लॉग ढूँढता है)।
  - फील्ड-विशिष्ट: `host.name:webserver AND level:error` ("webserver" से error लेवल वाले लॉग)।
  - वाक्यांश: `"user login failed"` (सटीक मिलान)।

- **फ़िल्टर्स**:
  - साइडबार से जोड़ें: फील्ड वैल्यू (जैसे, `level: ERROR`) पर क्लिक करें > Add filter (इसे क्वेरी में पिन करता है)।
  - बूलियन लॉजिक: `+error -info` ("error" होना चाहिए, "info" को बाहर करें)।
  - रेंज: नंबर/टाइम के लिए, जैसे, `bytes:>1000` (फील्ड > 1000)।

- **एडवांस्ड क्वेरीज़**:
  - जटिल आवश्यकताओं के लिए **Lucene query syntax** पर स्विच करें (क्वेरी लैंग्वेज ड्रॉपडाउन के माध्यम से): `message:(error OR warn) AND host.name:prod*`।
  - Elasticsearch-नेटिव क्वेरीज़ के लिए **Query DSL** का उपयोग करें (जैसे, JSON बॉडी के साथ POST /logs-*/_search)।

### सर्च सेव करना:
- पुन: उपयोग के लिए सर्च स्टोर करने के लिए **Save** (टॉप-राइट) पर क्लिक करें।
- **Share** > CSV/URL के माध्यम से एक्सपोर्ट के लिए शेयर करें।

उदाहरण वर्कफ़्लो: एप्लिकेशन लॉग जांचना
1. लॉग इन्जेस्ट करें (जैसे, Logstash के माध्यम से: इनपुट फ़ाइल > फ़िल्टर grok/parse > आउटपुट Elasticsearch)।
2. Discover में: टाइम रेंज "Last 24 hours"।
3. सर्च: `app.name:myapp AND level:ERROR`।
4. फ़िल्टर्स जोड़ें: `host.name` = विशिष्ट सर्वर।
5. निरीक्षण: स्टैक ट्रेस के लिए `message` देखें, `@timestamp` के साथ सहसंबंध बनाएँ।

## 7. लॉग विज़ुअलाइज़ करना
जबकि Discover रॉ जांच के लिए है, पैटर्न के लिए विज़ुअलाइज़ करें।

### विज़ुअलाइज़ेशन बनाएँ:
1. **Visualize Library** > **Create new visualization** पर जाएँ।
2. प्रकार चुनें:
   - **Lens** (आसान): बकेट्स में फील्ड्स खींचें (जैसे, X-axis: `@timestamp`, Y-axis: एरर्स की काउंट)।
   - **Area/Line Chart**: समय के साथ लॉग वॉल्यूम के लिए (मेट्रिक्स: काउंट, बकेट्स: `@timestamp` पर डेट हिस्टोग्राम)।
   - **Data Table**: सारणीबद्ध लॉग सारांश।
   - **Pie Chart**: `level` द्वारा विभाजन (error 40%, info 60%)।
3. Discover से फ़िल्टर्स/सर्च लागू करें।
4. सेव करें और **Dashboard** में जोड़ें (Analytics > Dashboard > Create new > Add visualization)।

उदाहरण: एरर रेट डैशबोर्ड
- विज़ुअलाइज़: प्रति घंटे एरर लॉग का लाइन चार्ट।
- फ़िल्टर: ग्लोबल टाइम रेंज।
- मॉनिटरिंग के लिए डैशबोर्ड में एम्बेड करें।

## 8. लॉग एनालिसिस के लिए एडवांस्ड फीचर्स
- **अलर्ट्स और मॉनिटरिंग**:
  - लॉग पैटर्न पर नोटिफाई करने के लिए **Alerts** (Stack Management > Rules) का उपयोग करें (जैसे, यदि "critical" >5 बार/घंटा दिखाई दे तो ईमेल करें)।
  - ऐप लॉग के लिए **Uptime Monitoring** या **APM**।

- **मशीन लर्निंग**:
  - लॉग वॉल्यूम में एनोमलियाँ डिटेक्ट करने के लिए ML जॉब्स एनेबल करें (Stack Management > Machine Learning)।

- **Dev Tools**:
  - रॉ Elasticsearch क्वेरीज़ के लिए कंसोल: जैसे,
    ```
    GET logs-*/_search
    {
      "query": { "match": { "message": "error" } },
      "sort": [ { "@timestamp": "desc" } ]
    }
    ```
  - इंडेक्स पैटर्न टेस्ट करें या डेटा इन्जेस्ट करें।

- **रोल्स और सुरक्षा**:
  - प्रोडक्शन में, लॉग व्यूज़ को आइसोलेट करने के लिए **Spaces** का उपयोग करें (जैसे, dev/prod)।
  - रोल-आधारित एक्सेस: यूजर्स को विशिष्ट इंडेक्स तक सीमित करें।

- **एक्सपोर्ट/इम्पोर्ट**:
  - **Stack Management > Saved Objects** के माध्यम से सर्च/डैशबोर्ड को NDJSON के रूप में एक्सपोर्ट करें।
  - **Console** या Beats के माध्यम से लॉग इम्पोर्ट करें।

- **परफॉर्मेंस टिप्स**:
  - कस्टम मैपिंग के लिए **Field Analyzer** (Index Patterns > field) का उपयोग करें।
  - बड़े रिजल्ट्स को पेजिनेट करें: प्रति पेज हिट्स एडजस्ट करें (Discover सेटिंग्स)।
  - बड़े डेटा के लिए, इंडेक्स को शार्ड करें और ILM (Index Lifecycle Management) का उपयोग करें।

## 9. लॉग स्रोतों के साथ इंटीग्रेशन
- **Filebeat/Logstash**: Elasticsearch में लॉग शिप करें।
  - उदाहरण Filebeat कॉन्फ़िग (`filebeat.yml`):
    ```yaml
    filebeat.inputs:
    - type: log
      paths: [/var/log/*.log]
      fields:
        app: myapp
    output.elasticsearch:
      hosts: ["localhost:9200"]
      index: "logs-%{+yyyy.MM.dd}"
    ```
  - रन: `./filebeat -e`।
- **क्लाउड लॉग**: मैनेज्ड सेटअप के लिए AWS S3, Azure, या Elastic Cloud के साथ इंटीग्रेट करें।

## 10. सामान्य समस्याओं का समाधान
- **Discover में कोई डेटा नहीं**:
  - Elasticsearch कनेक्शन चेक करें: `curl localhost:9200/_cat/indices`।
  - सत्यापित करें कि इंडेक्स पैटर्न इंडेक्स से मेल खाता है (जैसे, `*` मिसमैच नहीं)।
  - सुनिश्चित करें कि `@timestamp` डेट के रूप में मैप किया गया है।

- **किबाना स्टार्ट नहीं होगा**:
  - लॉग: `/var/log/kibana/kibana.log` या कंसोल आउटपुट चेक करें।
  - सामान्य: पोर्ट 5601 उपयोग में है, या Elasticsearch नहीं चल रहा/सुरक्षित है।

- **धीमी क्वेरीज़**:
  - ऑप्टिमाइज़ करें: फ़िल्टर्स के लिए कीवर्ड फील्ड्स का उपयोग करें, बड़े टेक्स्ट पर वाइल्डकार्ड से बचें।
  - हीप बढ़ाएँ: Elasticsearch के लिए `jvm.options` एडिट करें (जैसे, `-Xms4g -Xmx4g`)।

- **लॉग में एरर्स**:
  - 403/401: ऑथेंटिकेशन समस्याएँ—`elasticsearch.yml` में सुरक्षा एनेबल करें।
  - वर्जन मिसमैच: सुनिश्चित करें किबाना == Elasticsearch वर्जन।

- **संसाधन**:
  - ऑफिशियल डॉक्स: [elastic.co/guide/en/kibana/current](https://www.elastic.co/guide/en/kibana/current/index.html)।
  - फोरम: Elastic Discuss या Stack Overflow।
  - सैंपल डेटा: किबाना के माध्यम से लोड करें (Stack Management > Kibana > Sample Data)।

यह गाइड आपको लॉग जांच के लिए किबाना से शुरुआत करने में मदद करनी चाहिए। छोटे से शुरू करें: एक लोकल ELK स्टैक सेटअप करें, सैंपल लॉग इन्जेस्ट करें, और Discover में प्रयोग करें। प्रोडक्शन के लिए, आसानी के लिए Elastic Cloud पर विचार करें। यदि आपके पास कोई विशिष्ट सेटअप (जैसे, Docker, AWS) है, तो विशिष्ट सलाह के लिए अधिक विवरण प्रदान करें!
