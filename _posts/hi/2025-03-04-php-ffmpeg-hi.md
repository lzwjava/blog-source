### इंस्टॉलेशन
सबसे पहले, सुनिश्चित करें कि FFMpeg आपके सिस्टम पर इंस्टॉल है:
- Ubuntu पर, `sudo apt-get install ffmpeg` का उपयोग करें।
- macOS पर, `brew install ffmpeg` का उपयोग करें।
- Windows पर, [इस वेबसाइट](https://www.gyan.dev/ffmpeg/builds/) से डाउनलोड करें और निर्देशों का पालन करें।

अगला, Composer के माध्यम से php-FFMpeg लाइब्रेरी इंस्टॉल करें:
```
composer require php-FFMpeg/php-FFMpeg:~0.5
```

### सेटअप और उपयोग
अपनी PHP स्क्रिप्ट में ऑटोलोड फ़ाइल शामिल करें:
```php
require_once 'vendor/autoload.php';
```

`FFMpeg\FFMpeg` का एक इंस्टेंस बनाएं, वैकल्पिक रूप से पथ निर्दिष्ट करें यदि FFMpeg बाइनरीज़ सिस्टम PATH में नहीं हैं:
```php
use FFMpeg\FFMpeg;
$ff = FFMpeg::create(array(
    'binary' => '/path/to/FFMpeg',
    'ffprobe' => '/path/to/FFprobe'
));
```

एक मीडिया फ़ाइल खोलें और ऑपरेशन करें, जैसे:
- ट्रांसकोडिंग: `$video->save('output.mp4', new FFMpeg\Format\Video\X264());`
- एक फ्रेम निकालना: `$frame = $video->frame(FFMpeg\Coordinate\TimeCode::fromSeconds(5)); $frame->save('frame.jpg');`
- क्लिपिंग: `$clip = $video->clip(FFMpeg\Coordinate\TimeCode::fromSeconds(10), FFMpeg\Coordinate\TimeCode::fromSeconds(20)); $clip->save('clip.mp4');`

अधिक विवरण के लिए, लाइब्रेरी के डॉक्यूमेंटेशन को [GitHub](https://github.com/PHP-FFMpeg/PHP-FFMpeg) पर देखें।

### सर्वे नोट: php-FFMpeg/php-FFMpeg संस्करण ~0.5 का उपयोग करने के लिए व्यापक गाइड

यह नोट उपलब्ध जानकारी के आधार पर, विशेष रूप से संस्करण लगभग 0.5, "php-FFMpeg/php-FFMpeg" लाइब्रेरी का उपयोग करने का एक गहन अन्वेषण प्रदान करता है। यह मीडिया मैनिपुलेशन के लिए इस PHP लाइब्रेरी को लागू करने की चाह रखने वाले उपयोगकर्ताओं के लिए पूरी समझ सुनिश्चित करते हुए, शोध से प्राप्त सभी प्रासंगिक विवरणों को शामिल करके सीधे उत्तर का विस्तार करता है।

#### पृष्ठभूमि और संदर्भ
"php-FFMpeg/php-FFMpeg" लाइब्रेरी FFMpeg बाइनरी के लिए एक PHP रैपर है, जो वीडियो और ऑडियो फ़ाइलों के ऑब्जेक्ट-ओरिएंटेड मैनिपुलेशन को सक्षम बनाती है। यह ट्रांसकोडिंग, फ्रेम एक्सट्रैक्शन, क्लिपिंग और अन्य जैसे कार्यों का समर्थन करती है, जिससे यह मीडिया-संबंधित एप्लिकेशन पर काम करने वाले डेवलपर्स के लिए मूल्यवान बन जाती है। संस्करण विशिष्टता "~0.5" 0.5 से बड़े या बराबर और 1.0 से कम किसी भी संस्करण को इंगित करती है, जो पुराने PHP संस्करणों के साथ संगतता का सुझाव देती है, जो संभवतः रिपॉजिटरी की 0.x शाखा में पाई जाती है।

वर्तमान तिथि, 3 मार्च, 2025, और लाइब्रेरी के विकास को देखते हुए, संस्करण 0.5 लीगेसी सपोर्ट का हिस्सा हो सकता है, जहां मुख्य शाखा के लिए अब PHP 8.0 या उच्चतर की आवश्यकता है। यह नोट मानता है कि उपयोगकर्ता इस संस्करण की बाधाओं के भीतर काम कर रहा है, नए रिलीज़ की तुलना में कार्यक्षमता में संभावित अंतरों को स्वीकार करता है।

#### इंस्टॉलेशन प्रक्रिया
शुरुआत करने के लिए, सिस्टम पर FFMpeg इंस्टॉल होना चाहिए, क्योंकि लाइब्रेरी ऑपरेशन्स के लिए इसकी बाइनरीज़ पर निर्भर करती है। इंस्टॉलेशन विधियाँ ऑपरेटिंग सिस्टम के अनुसार भिन्न होती हैं:
- **Ubuntu**: पैकेज मैनेजर के माध्यम से इंस्टॉल करने के लिए `sudo apt-get install ffmpeg` कमांड का उपयोग करें।
- **macOS**: सीधे इंस्टॉलेशन के लिए Homebrew के साथ `brew install ffmpeg` का उपयोग करें।
- **Windows**: [इस वेबसाइट](https://www.gyan.dev/ffmpeg/builds/) से FFMpeg बाइनरीज़ डाउनलोड करें और प्रदान किए गए निर्देशों का पालन करें, यह सुनिश्चित करते हुए कि एक्ज़िक्यूटेबल्स सिस्टम PATH में सुलभ हैं या मैन्युअल रूप से निर्दिष्ट हैं।

FFMpeg इंस्टॉलेशन के बाद, php-FFMpeg लाइब्रेरी को Composer, PHP पैकेज मैनेजर के माध्यम से इंस्टॉल किया जाता है। `composer require php-FFMpeg/php-FFMpeg:~0.5` कमांड सही संस्करण प्राप्त करना सुनिश्चित करती है। यह प्रक्रिया प्रोजेक्ट में एक `vendor` डायरेक्टरी बनाती है, जिसमें लाइब्रेरी और इसकी निर्भरताएं होती हैं, साथ ही Composer सीमलेस इंटीग्रेशन के लिए ऑटोलोडिंग प्रबंधित करता है।

#### सेटअप और आरंभीकरण
इंस्टॉलेशन के बाद, लाइब्रेरी की कक्षाओं तक पहुंच सक्षम करने के लिए अपनी PHP स्क्रिप्ट में ऑटोलोड फ़ाइल शामिल करें:
```php
require_once 'vendor/autoload.php';
```

लाइब्रेरी का उपयोग शुरू करने के लिए `FFMpeg\FFMpeg` का एक इंस्टेंस बनाएं। निर्माण विधि कॉन्फ़िगरेशन की अनुमति देती है, विशेष रूप से महत्वपूर्ण यदि FFMpeg बाइनरीज़ सिस्टम PATH में नहीं हैं:
```php
use FFMpeg\FFMpeg;
$ff = FFMpeg::create(array(
    'timeout' => 0,
    'thread_count' => 12,
    'binary' => '/path/to/your/own/FFMpeg',
    'ffprobe' => '/path/to/your/own/FFprobe'
));
```
यह कॉन्फ़िगरेशन टाइमआउट, थ्रेड काउंट और स्पष्ट बाइनरी पथ सेट करने का समर्थन करती है, जो विभिन्न सिस्टम सेटअप के लिए लचीलापन बढ़ाती है। डिफ़ॉल्ट सेटअप PATH में बाइनरीज़ की तलाश करता है, लेकिन मैन्युअल विशिष्टता विशेष रूप से कस्टम वातावरण पर संगतता सुनिश्चित करती है।

#### मुख्य उपयोग और ऑपरेशन्स
लाइब्रेरी मीडिया मैनिपुलेशन के लिए एक प्रवाही, ऑब्जेक्ट-ओरिएंटेड इंटरफेस प्रदान करती है। एक मीडिया फ़ाइल खोलकर शुरुआत करें:
```php
$video = $ff->open('input.mp4');
```
यह स्थानीय फ़ाइलसिस्टम पथ, HTTP संसाधन और अन्य FFMpeg-समर्थित प्रोटोकॉल का समर्थन करता है, जिसमें समर्थित प्रकारों की सूची `ffmpeg -protocols` कमांड के माध्यम से उपलब्ध है।

##### ट्रांसकोडिंग
ट्रांसकोडिंग में मीडिया को विभिन्न फॉर्मेट में परिवर्तित करना शामिल है। एक फॉर्मेट इंस्टेंस के साथ `save` विधि का उपयोग करें:
```php
$format = new FFMpeg\Format\Video\X264();
$video->save('output.mp4', $format);
```
`X264` फॉर्मेट एक उदाहरण है; लाइब्रेरी विभिन्न वीडियो और ऑडियो फॉर्मेट का समर्थन करती है, जिन्हें `FFMpeg\Format\FormatInterface` के माध्यम से लागू किया जा सकता है, जिसमें संबंधित मीडिया प्रकारों के लिए `VideoInterface` और `AudioInterface` जैसे विशिष्ट इंटरफेस शामिल हैं।

##### फ्रेम एक्सट्रैक्शन
थंबनेल या विश्लेषण के लिए फ्रेम निकालना उपयोगी है। निम्नलिखित कोड 5 सेकंड पर एक फ्रेम निकालता है:
```php
$frame = $video->frame(FFMpeg\Coordinate\TimeCode::fromSeconds(5));
$frame->save('frame.jpg');
```
`TimeCode` क्लास, `FFMpeg\Coordinate` का हिस्सा, सटीक टाइमिंग सुनिश्चित करती है, जिसमें इमेज एक्सट्रैक्शन में सटीकता के लिए विकल्प शामिल हैं।

##### क्लिपिंग
वीडियो के एक हिस्से को क्लिप करने के लिए, प्रारंभ और समाप्ति समय निर्दिष्ट करें:
```php
$clip = $video->clip(FFMpeg\Coordinate\TimeCode::fromSeconds(10), FFMpeg\Coordinate\TimeCode::fromSeconds(20));
$clip->save('clip.mp4');
```
यह मूल गुणवत्ता और फॉर्मेट को बनाए रखते हुए एक नया वीडियो सेगमेंट बनाता है, जिसमें आवश्यकता पड़ने पर अतिरिक्त फ़िल्टर लागू करने की क्षमता शामिल है।

#### उन्नत सुविधाएँ और विचार
लाइब्रेरी अतिरिक्त सुविधाओं का समर्थन करती है, जैसा कि डॉक्यूमेंटेशन में बताया गया है:
- **ऑडियो मैनिपुलेशन**: वीडियो के समान, ऑडियो को `FFMpeg\Media\Audio::save` के साथ ट्रांसकोड किया जा सकता है, फ़िल्टर लागू करके, मेटाडेटा जोड़कर और रीसैंपलिंग करके।
- **GIF निर्माण**: एनिमेटेड GIF को `FFMpeg\Media\Gif::save` का उपयोग करके सेव किया जा सकता है, जिसमें वैकल्पिक अवधि पैरामीटर शामिल हैं।
- **संयोजन**: एक ही कोडेक के लिए `FFMpeg\Media\Concatenate::saveFromSameCodecs` या विभिन्न कोडेक के लिए `saveFromDifferentCodecs` का उपयोग करके कई मीडिया फ़ाइलों को संयोजित करें, आगे पढ़ने के लिए संसाधन [इस लिंक](https://trac.ffmpeg.org/wiki/Concatenate), [इस लिंक](https://ffmpeg.org/ffmpeg-formats.html#concat-1), और [इस लिंक](https://ffmpeg.org/ffmpeg.html#Stream-copy) पर उपलब्ध हैं।
- **उन्नत मीडिया हैंडलिंग**: जटिल फ़िल्टरिंग और मैपिंग के लिए उपयोगी `-filter_complex` के साथ कई इनपुट/आउटपुट का समर्थन करती है, जिसमें बिल्ट-इन फ़िल्टर सपोर्ट शामिल है।
- **मेटाडेटा एक्सट्रैक्शन**: मेटाडेटा के लिए `FFMpeg\FFProbe::create` का उपयोग करें, फ़ाइलों को `FFMpeg\FFProbe::isValid` के साथ वैलिडेट करें (v0.10.0 से उपलब्ध, ध्यान दें कि संस्करण 0.5 में यह कमी हो सकती है)।

निर्देशांक, जैसे `FFMpeg\Coordinate\AspectRatio`, `Dimension`, `FrameRate`, `Point` (v0.10.0 से डायनामिक), और `TimeCode`, मीडिया गुणों पर सटीक नियंत्रण प्रदान करते हैं, हालांकि कुछ सुविधाएँ जैसे डायनामिक पॉइंट संस्करण 0.5 में उपलब्ध नहीं हो सकती हैं।

#### संस्करण-विशिष्ट नोट्स
"~0.5" विशिष्टता को देखते हुए, लाइब्रेरी संभवतः 0.x शाखा के साथ संरेखित होती है, जो पुराने PHP संस्करणों का समर्थन करती है। GitHub रिपॉजिटरी मुख्य शाखा के लिए PHP 8.0 या उच्चतर इंगित करती है, जबकि 0.x लीगेसी सपोर्ट के लिए है। हालाँकि, सटीक संस्करण 0.5 का विवरण रिलीज़ में स्पष्ट रूप से नहीं मिला, जिससे यह संकेत मिलता है कि यह कमिट इतिहास या शाखा कमिट्स का हिस्सा हो सकता है। उपयोगकर्ताओं को संगतता सत्यापित करनी चाहिए, क्योंकि कुछ नई सुविधाएँ जैसे कुछ निर्देशांक प्रकार (जैसे, डायनामिक पॉइंट) के लिए संस्करण 0.5 से परे की आवश्यकता हो सकती है।

| वीडियो ट्रांसकोड करें         | `$video->save('output.mp4', new X264())`           | निर्दिष्ट फॉर्मेट में परिवर्तित करता है                    |
| फ्रेम निकालें           | `$video->frame(TimeCode::fromSeconds(5))->save('frame.jpg')` | निर्दिष्ट समय पर फ्रेम निकालता है, इमेज के रूप में सेव करता है |
| वीडियो क्लिप करें              | `$video->clip(TimeCode::fromSeconds(10), TimeCode::fromSeconds(20))->save('clip.mp4')` | समय के बीच क्लिप बनाता है, नई फ़ाइल के रूप में सेव करता है   |
| इंस्टेंस कॉन्फ़िगर करें      | `FFMpeg::create(array('binary' => '/path/to/FFMpeg'))` | कस्टम बाइनरी पथ और विकल्प सेट करता है            |

यह तालिका मुख्य कार्यक्षमता को समाहित करती है, जो लाइब्रेरी को लागू करते समय उपयोगकर्ताओं को त्वरित संदर्भ में सहायता प्रदान करती है।

#### अप्रत्याशित विवरण: लीगेसी संस्करण के निहितार्थ
एक अप्रत्याशित पहलू संस्करण 0.5 की संभावित सीमा है, जिसे पुराने PHP सपोर्ट के लिए 0.x शाखा में इसके स्थान को देखते हुए। यह आधुनिक सुविधाओं, जैसे v0.10.0 में पेश किए गए डायनामिक कोऑर्डिनेट पॉइंट्स, तक पहुंच को प्रतिबंधित कर सकता है, जिसके लिए उपयोगकर्ताओं को सटीक क्षमताओं के लिए शाखा कमिट्स की जांच करनी पड़ती है, जो नए संस्करणों में तुरंत स्पष्ट नहीं होने वाली जटिलता की एक परत जोड़ता है।

#### निष्कर्ष
"php-FFMpeg/php-FFMpeg" संस्करण ~0.5 का उपयोग करने में FFMpeg इंस्टॉल करना, Composer के माध्यम से सेटअप करना और मीडिया कार्यों के लिए एक ऑब्जेक्ट-ओरिएंटेड API का लाभ उठाना शामिल है। जबकि मूल ऑपरेशन्स जैसे ट्रांसकोडिंग और फ्रेम एक्सट्रैक्शन सीधे हैं, संस्करण-विशिष्ट अंतर [GitHub](https://github.com/PHP-FFMpeg/PHP-FFMpeg) पर 0.x शाखा डॉक्यूमेंटेशन से परामर्श करने की आवश्यकता उत्पन्न कर सकते हैं। यह व्यापक दृष्टिकोण सुनिश्चित करता है कि उपयोगकर्ता लाइब्रेरी का प्रभावी ढंग से उपयोग कर सकें, इसकी शक्ति और संभावित लीगेसी बाधाओं दोनों को स्वीकार करते हुए।
