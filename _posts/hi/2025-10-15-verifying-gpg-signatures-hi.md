### GPG में हस्ताक्षर सत्यापित करना

GnuPG (GPG) आपको डिजिटल हस्ताक्षर सत्यापित करने की अनुमति देता है ताकि यह सुनिश्चित किया जा सके कि कोई फ़ाइल या संदेश छेड़छाड़ नहीं किया गया है और यह दावा किए गए प्रेषक से आया है। सत्यापन के लिए आपके कीरिंग में हस्ताक्षरकर्ता की सार्वजनिक कुंजी होनी आवश्यक है। नीचे सामान्य परिदृश्यों के लिए चरण और कमांड दिए गए हैं।

#### आवश्यक शर्तें
1.  **GPG इंस्टॉल करें**: सुनिश्चित करें कि GnuPG आपके सिस्टम पर इंस्टॉल है (उदाहरण के लिए, डेबियन-आधारित सिस्टम पर `apt install gnupg` या macOS पर `brew install gnupg` के माध्यम से)।
2.  **हस्ताक्षरकर्ता की सार्वजनिक कुंजी इम्पोर्ट करें**: आपको उस व्यक्ति या इकाई की सार्वजनिक कुंजी की आवश्यकता है जिसने फ़ाइल पर हस्ताक्षर किए हैं। इसे कीसर्वर या फ़ाइल से डाउनलोड करें, फिर इसे इम्पोर्ट करें:
    ```
    gpg --import public-key.asc
    ```
    (`public-key.asc` को वास्तविक कुंजी फ़ाइल से बदलें। यदि यह किसी कीसर्वर से है, तो `gpg --keyserver keyserver.ubuntu.com --recv-keys KEYID` का उपयोग करें, जहां `KEYID` कुंजी की फिंगरप्रिंट या ID है।)

#### सत्यापन चरण
सटीक कमांड इस पर निर्भर करता है कि हस्ताक्षर कैसे बनाया गया था (उदाहरण के लिए, फ़ाइल में इनलाइन या अलग किया हुआ)।

1.  **एक अलग किए गए हस्ताक्षर को सत्यापित करें** (सॉफ़्टवेयर रिलीज़ जैसी फ़ाइलों के लिए आम; हस्ताक्षर एक अलग `.sig` फ़ाइल में होता है):
    ```
    gpg --verify signature.sig original-file
    ```
    - एक अच्छे हस्ताक्षर के लिए उदाहरण आउटपुट:
      ```
      gpg: Signature made [date] using [key type] key ID [KEYID]
      gpg: Good signature from "[Name] <email@example.com>"
      ```
    - यदि कुंजी विश्वसनीय नहीं है, तो आपको "This key is not certified with a trusted signature!" जैसी चेतावनी दिखाई दे सकती है। आप `gpg --edit-key KEYID` और फिर `trust` के साथ ट्रस्ट स्तर समायोजित कर सकते हैं।

2.  **एक इनलाइन-साइन की गई फ़ाइल सत्यापित करें** (हस्ताक्षर फ़ाइल में एम्बेडेड होता है, अक्सर बाइनरी):
    - केवल हस्ताक्षर जांचने के लिए:
      ```
      gpg --verify signed-file
      ```
    - सत्यापित करने और मूल सामग्री निकालने के लिए:
      ```
      gpg --output original-file --decrypt signed-file
      ```
      यदि एन्क्रिप्टेड है तो यह किसी भी पासफ्रेज़ के लिए प्रॉम्प्ट करेगा, लेकिन केवल हस्ताक्षर के लिए, यह सिर्फ सत्यापित करता है।

3.  **एक क्लियरसाइन किए गए संदेश को सत्यापित करें** (ASCII-armored हस्ताक्षर के साथ मानव-पठनीय पाठ, ईमेल या README फाइलों में आम):
    ```
    gpg --verify clearsigned-file.txt
    ```
    आउटपुट फ़ाइल को बदले बिना हस्ताक्षर स्थिति दिखाएगा।

#### समस्या निवारण
-   **"No public key" त्रुटि**: ऊपर दिखाए अनुसार कुंजी इम्पोर्ट करें।
-   **खराब हस्ताक्षर**: फ़ाइल में छेड़छाड़ की गई हो सकती है—इसे फिर से डाउनलोड करें।
-   **कुंजी वैधता**: `gpg --list-keys` के साथ कुंजी विवरण जांचें या सत्यापित करें कि फिंगरप्रिंट आधिकारिक स्रोत से मेल खाते हैं।
-   बैच सत्यापन या स्क्रिप्टिंग के लिए, `--batch --exit-on-status-write-error` जोड़ें।

हस्ताक्षर करने या उन्नत विकल्पों के बारे में अधिक जानकारी के लिए, आधिकारिक मैनुअल देखें।
