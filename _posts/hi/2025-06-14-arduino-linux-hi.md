Ubuntu Linux पर Arduino IDE इंस्टॉल करने के लिए, इन चरणों का पालन करें:

1. **Arduino IDE डाउनलोड करें**:
   - आधिकारिक Arduino वेबसाइट [https://www.arduino.cc/en/software](https://www.arduino.cc/en/software) पर जाएं।
   - "Arduino IDE" सेक्शन के तहत, "Linux 64 bits" डाउनलोड विकल्प चुनें (या आपके सिस्टम के लिए उपयुक्त संस्करण, जैसे Raspberry Pi के लिए Linux ARM)।

2. **डाउनलोड की गई फ़ाइल को एक्सट्रैक्ट करें**:
   - डाउनलोड की गई फ़ाइल एक संपीड़ित `.tar.xz` आर्काइव है (उदाहरण के लिए, `arduino-1.8.19-linux64.tar.xz`)।
   - एक टर्मिनल खोलें और डाउनलोड डायरेक्टरी में नेविगेट करें (उदाहरण के लिए, `cd ~/Downloads`)।
   - फ़ाइल को इस कमांड का उपयोग करके एक्सट्रैक्ट करें:
     ```bash
     tar -xf arduino-*.tar.xz
     ```
   - इससे `arduino-1.8.19` जैसा एक फोल्डर बनेगा।

3. **एक उपयुक्त डायरेक्टरी में ले जाएँ (वैकल्पिक)**:
   - सिस्टम-वाइड एक्सेस के लिए, एक्सट्रैक्ट किए गए फोल्डर को `/opt` में ले जाएं (इसके लिए सुपरयूजर विशेषाधिकारों की आवश्यकता होती है):
     ```bash
     sudo mv arduino-1.8.19 /opt/arduino
     ```

4. **इंस्टॉलेशन स्क्रिप्ट चलाएँ**:
   - Arduino फोल्डर में नेविगेट करें:
     ```bash
     cd /opt/arduino
     ```
   - इंस्टॉलेशन स्क्रिप्ट चलाएँ:
     ```bash
     sudo ./install.sh
     ```
   - यह एक डेस्कटॉप शॉर्टकट बनाता है और आवश्यक अनुमतियाँ सेट करता है।

5. **अपने यूज़र को Dialout ग्रुप में जोड़ें**:
   - Arduino बोर्ड को सीरियल पोर्ट के माध्यम से एक्सेस करने के लिए, अपने यूज़र को `dialout` ग्रुप में जोड़ें:
     ```bash
     sudo usermod -a -G dialout $USER
     ```
   - ग्रुप परिवर्तन के प्रभावी होने के लिए लॉग आउट करें और वापस लॉग इन करें, या रीबूट करें।

6. **Arduino IDE लॉन्च करें**:
   - यदि आपने `install.sh` स्क्रिप्ट चलाई है, तो आप एप्लिकेशन मेनू से या "Arduino IDE" खोजकर IDE लॉन्च कर सकते हैं।
   - वैकल्पिक रूप से, इसे टर्मिनल से सीधे चलाएं:
     ```bash
     /opt/arduino/arduino
     ```

7. **Arduino बोर्ड के लिए अनुमतियाँ अपडेट करें (यदि आवश्यक हो)**:
   - अपने Arduino को कनेक्ट करते समय, सुनिश्चित करें कि यह डिटेक्ट हो गया है (उदाहरण के लिए, `/dev/ttyACM0` या `/dev/ttyUSB0`)।
   - यदि आपको अनुमति संबंधी समस्याएँ आती हैं, तो सही अनुमतियाँ सेट करें:
     ```bash
     sudo chmod a+rw /dev/ttyACM0
     ```
   - `/dev/ttyACM0` को उचित पोर्ट से बदलें।

8. **इंस्टॉलेशन सत्यापित करें**:
   - अपने Arduino बोर्ड को USB के माध्यम से कनेक्ट करें।
   - IDE खोलें, **Tools > Board** पर जाएं और अपना Arduino मॉडल चुनें (उदाहरण के लिए, Arduino Uno)।
   - **Tools > Port** पर जाएं और पोर्ट चुनें (उदाहरण के लिए, `/dev/ttyACM0`)।
   - सब कुछ ठीक से काम करने की पुष्टि करने के लिए एक नमूना स्केच (उदाहरण के लिए, **File > Examples > 01.Basics > Blink**) अपलोड करने का प्रयास करें।

**वैकल्पिक: Snap या Flatpak के माध्यम से इंस्टॉल करें**:
- **Snap**:
  ```bash
  sudo snap install arduino
  ```
- **Flatpak**:
  ```bash
  sudo flatpak install flathub cc.arduino.ArduinoIDE
  ```
- ये तरीके इंस्टॉलेशन को सरल बनाते हैं लेकिन अनुमतियों या बोर्ड डिटेक्शन में सीमाएँ हो सकती हैं।

**नोट्स**:
- सुनिश्चित करें कि आपके पास Java इंस्टॉल है, क्योंकि Arduino IDE के लिए इसकी आवश्यकता होती है। यदि आवश्यक हो तो इसे इंस्टॉल करें:
  ```bash
  sudo apt update
  sudo apt install default-jre
  ```
- नवीनतम संस्करण या समस्या निवारण के लिए, Arduino फोरम या [आधिकारिक दस्तावेज़](https://www.arduino.cc/en/Guide/Linux) देखें।

यदि आपको कोई समस्या आती है, तो अपना Ubuntu संस्करण या विशिष्ट त्रुटि साझा करें ताकि आपको विशिष्ट सहायता मिल सके!

चूंकि आपका Arduino Uno Ubuntu पर Arduino IDE में डिटेक्ट हो रहा है और आप बिना ब्रेडबोर्ड या बाहरी घटकों के एक सरल उदाहरण चाहते हैं, यहां एक बुनियादी स्केच दी गई है जो Arduino Uno पर बिल्ट-इन LED (डिजिटल पिन 13 से जुड़ा) को ब्लिंक कराती है। इसके लिए किसी अतिरिक्त हार्डवेयर की आवश्यकता नहीं है।

```cpp
void setup() {
  // put your setup code here, to run once:
  pinMode(LED_BUILTIN, OUTPUT); // Set the built-in LED pin (pin 13) as output
}

void loop() {
  // put your main code here, to run repeatedly:
  digitalWrite(LED_BUILTIN, HIGH); // Turn the LED on
  delay(1000);                     // Wait for 1 second
  digitalWrite(LED_BUILTIN, LOW);  // Turn the LED off
  delay(1000);                     // Wait for 1 second
}
```

### इसे कैसे चलाएं:
1. Arduino IDE खोलें।
2. उपरोक्त कोड को IDE में कॉपी और पेस्ट करें।
3. सुनिश्चित करें कि आपका Arduino Uno USB के माध्यम से कनेक्ट है और डिटेक्ट हो गया है:
   - **Tools > Board** पर जाएं और **Arduino Uno** चुनें।
   - **Tools > Port** पर जाएं और पोर्ट चुनें (उदाहरण के लिए, `/dev/ttyACM0` या `/dev/ttyUSB0`)।
4. स्केच को अपने Arduino Uno पर कंपाइल और अपलोड करने के लिए **Upload** बटन (दायाँ तीर आइकन) पर क्लिक करें।
5. एक बार अपलोड होने के बाद, बिल्ट-इन LED (बोर्ड पर "L" लेबल वाली) 1 सेकंड के लिए चमकनी चाहिए, फिर 1 सेकंड के लिए बंद होनी चाहिए, और यह लगातार दोहराती रहेगी।

### यह क्या करता है:
- `setup()` में, `pinMode` फ़ंक्शन बिल्ट-इन LED पिन को एक आउटपुट के रूप में कॉन्फ़िगर करता है।
- `loop()` में, `digitalWrite` फ़ंक्शन LED को चालू (`HIGH`) और बंद (`LOW`) करता है, जिसके बीच `delay` प्रत्येक स्थिति के बीच 1 सेकंड के लिए रुकता है, जिससे एक ब्लिंकिंग प्रभाव बनता है।

### नोट्स:
- किसी बाहरी घटक की आवश्यकता नहीं है क्योंकि Arduino Uno में पिन 13 पर एक बिल्ट-इन LED होती है।
- यदि LED नहीं ब्लिंक करती है, तो पोर्ट चयन को दोबारा जांचें या सुनिश्चित करें कि बोर्ड ठीक से कनेक्ट है। यदि आप `setup()` में `Serial.begin(9600);` जोड़ते हैं और `loop()` में `Serial.println("Blink");` जैसे संदेश प्रिंट करते हैं, तो आप डीबग करने के लिए **Serial Monitor** (Tools > Serial Monitor) भी खोल सकते हैं।
- यदि आप ब्लिंक की गति को संशोधित करना चाहते हैं, तो `delay(1000)` मानों को समायोजित करें (उदाहरण के लिए, तेज ब्लिंकिंग के लिए `delay(500)`)।

यदि आपको कोई समस्या आती है या कोई अन्य सरल उदाहरण चाहते हैं तो मुझे बताएं!

त्रुटि संदेश Ubuntu पर आपके Arduino Uno पर एक स्केच अपलोड करने का प्रयास करते समय एक अनुमति समस्या का संकेत देता है। Arduino IDE के पास सीरियल पोर्ट `/dev/ttyUSB0` तक पहुंचने के लिए आवश्यक अनुमतियां नहीं हैं। इसे ठीक करने का तरीका यहां बताया गया है:

### समाधान: सीरियल पोर्ट अनुमतियाँ ठीक करें

1. **अपने यूज़र को `dialout` ग्रुप में जोड़ें**:
   - Ubuntu में, `dialout` ग्रुप आमतौर पर सीरियल पोर्ट तक पहुंच को नियंत्रित करता है। अपने यूज़र को इस ग्रुप में जोड़ें:
     ```bash
     sudo usermod -a -G dialout $USER
     ```
   - इसे चलाने के बाद, परिवर्तन के प्रभावी होने के लिए लॉग आउट करें और वापस लॉग इन करें, या अपने सिस्टम को रीबूट करें:
     ```bash
     sudo reboot
     ```

2. **पोर्ट सत्यापित करें**:
   - रीबूट करने के बाद, अपने Arduino Uno को फिर से कनेक्ट करें और Arduino IDE खोलें।
   - **Tools > Port** पर जाएं और सुनिश्चित करें कि `/dev/ttyUSB0` (या कुछ Arduino बोर्डों के लिए `/dev/ttyACM0`) चुना गया है। यदि कोई पोर्ट दिखाई नहीं देता है, तो USB कनेक्शन जांचें या कोई अलग केबल/पोर्ट आज़माएं।

3. **पोर्ट अनुमतियाँ अस्थायी रूप से बदलें (वैकल्पिक)**:
   - यदि अपने यूज़र को `dialout` ग्रुप में जोड़ने के बाद भी समस्या बनी रहती है, तो आप पोर्ट के लिए अनुमतियाँ मैन्युअल रूप से सेट कर सकते हैं (यह एक अस्थायी समाधान है, क्योंकि अनुमतियाँ रीबूट पर रीसेट हो जाती हैं):
     ```bash
     sudo chmod a+rw /dev/ttyUSB0
     ```
   - यदि यह अलग है तो `/dev/ttyUSB0` को सही पोर्ट से बदलें (उदाहरण के लिए, `/dev/ttyACM0`)।

4. **फिर से अपलोड करने का प्रयास करें**:
   - Arduino IDE में, अपना स्केच (जैसे पहले वाला ब्लिंकिंग LED स्केच) अपलोड करने के लिए **Upload** बटन पर क्लिक करें।
   - यदि यह सफलतापूर्वक अपलोड हो जाता है, तो आपके Arduino Uno पर बिल्ट-इन LED ब्लिंक करने लगनी चाहिए।

### अतिरिक्त समस्या निवारण

- **Arduino बोर्ड और केबल जांचें**:
  - सुनिश्चित करें कि Arduino Uno ठीक से USB के माध्यम से कनेक्ट है और Ubuntu द्वारा पहचाना गया है। कनेक्ट किए गए उपकरणों की सूची देखने के लिए यह कमांड चलाएं:
    ```bash
    ls /dev/tty*
    ```
    बोर्ड प्लग इन होने पर `/dev/ttyUSB0` या `/dev/ttyACM0` देखें।
  - यदि बोर्ड डिटेक्ट नहीं होता है तो कोई अलग USB केबल या पोर्ट आज़माएं।

- **बोर्ड और पोर्ट सेटिंग्स सत्यापित करें**:
  - Arduino IDE में, पुष्टि करें कि **Tools > Board** **Arduino Uno** पर सेट है।
  - यदि `/dev/ttyUSB0` दिखाई नहीं देता है, तो बोर्ड को अनप्लग और रीप्लग करने के बाद **Tools > Port** आज़माएं।

- **Arduino IDE को रूट के रूप में चलाएं (अस्थायी वर्कअराउंड)**:
  - यदि उपरोक्त चरण काम नहीं करते हैं, तो आप अनुमति समस्याओं को दरकिनार करने के लिए सुपरयूजर विशेषाधिकारों के साथ IDE चला सकते हैं (दीर्घकालिक अनुशंसित नहीं):
    ```bash
    sudo arduino
    ```
    यदि यह आपके सिस्टम के PATH में नहीं है तो `arduino` को अपने Arduino IDE एक्ज़ीक्यूटेबल के पथ से बदलें (उदाहरण के लिए, `/opt/arduino/arduino`)।

- **संघर्षकारी सॉफ़्टवेयर की जांच करें**:
  - अन्य सॉफ़्टवेयर (जैसे `brltty`, एक ब्रेल डिस्प्ले ड्राइवर) सीरियल पोर्ट में हस्तक्षेप कर सकते हैं। यदि इंस्टॉल है तो इसे हटा दें:
    ```bash
    sudo apt remove brltty
    ```

- **ड्राइवर समस्याएं**:
  - कुछ Arduino क्लोन CH340/CH341 चिप का उपयोग करते हैं, जिसके लिए एक ड्राइवर की आवश्यकता होती है। यदि आवश्यक हो तो इसे इंस्टॉल करें:
    ```bash
    sudo apt install linux-modules-extra-$(uname -r)
    ```
    फिर, बोर्ड को फिर से कनेक्ट करें।

### समाधान की पुष्टि करें
एक बार अनुमतियाँ हल हो जाने के बाद, पिछले ब्लिंक स्केच को फिर से अपलोड करने का प्रयास करें:
```cpp
void setup() {
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_BUILTIN, HIGH);
  delay(1000);
  digitalWrite(LED_BUILTIN, LOW);
  delay(1000);
}
```

यदि आपको अभी भी समस्याएँ आती हैं, तो नया त्रुटि संदेश साझा करें या आगे निदान में सहायता के लिए टर्मिनल में `ls -l /dev/ttyUSB0` चलाएं और आउटपुट प्रदान करें।

---

ऐसा प्रतीत होता है कि आप Ubuntu पर Arduino IDE का उपयोग करके अपने Arduino Uno पर एक स्केच अपलोड करने का प्रयास करते समय एक समस्या का सामना कर रहे हैं। आपके द्वारा प्रदान किया गया त्रुटि संदेश है:

```
avrdude: ser_open(): can't open device "/dev/ttyUSB0": Permission denied
Failed uploading: uploading error: exit status 1
```

यह त्रुटि इंगित करती है कि Arduino IDE के पास सीरियल पोर्ट (`/dev/ttyUSB0`) तक पहुंचने के लिए आवश्यक अनुमतियां नहीं हैं जहां आपका Arduino Uno कनेक्ट है। यह Linux सिस्टम जैसे Ubuntu पर एक सामान्य समस्या है, क्योंकि सीरियल पोर्ट एक्सेस कैसे प्रबंधित किया जाता है। इस समस्या को ठीक करने के लिए एक चरण-दर-चरण मार्गदर्शिका नीचे दी गई है।

### समाधान: सीरियल पोर्ट अनुमति समस्या ठीक करें

#### 1. अपने यूज़र को `dialout` ग्रुप में जोड़ें
Ubuntu में, `dialout` ग्रुप सीरियल पोर्ट तक पहुंच को नियंत्रित करता है। अनुमति समस्या को हल करने के लिए, आपको अपने यूज़र को इस ग्रुप में जोड़ना होगा।

- `Ctrl + Alt + T` दबाकर एक टर्मिनल खोलें।
- निम्नलिखित कमांड चलाएँ:
  ```bash
  sudo usermod -a -G dialout $USER
  ```
- यह कमांड आपके वर्तमान यूज़र को `dialout` ग्रुप में जोड़ता है। `$USER` वेरिएबल स्वचालित रूप से आपके यूज़रनेम का उपयोग करता है।

#### 2. लॉग आउट करें और वापस लॉग इन करें
ग्रुप परिवर्तन के प्रभावी होने के लिए, आपको अपने वर्तमान सत्र से लॉग आउट करना होगा और वापस लॉग इन करना होगा। वैकल्पिक रूप से, आप अपने सिस्टम को रीबूट कर सकते हैं।

- **लॉग आउट और वापस लॉग इन करने के लिए:**
  - कोई भी खुला कार्य सहेजें।
  - Ubuntu डेस्कटॉप के ऊपरी-दाएं कोने में अपने यूज़रनेम पर क्लिक करें और "Log Out" चुनें।
  - अपने यूज़रनेम और पासवर्ड के साथ वापस लॉग इन करें।
- **रीबूट करने के लिए (वैकल्पिक):**
  - टर्मिनल में यह कमांड चलाएँ:
    ```bash
    sudo reboot
    ```
  - अपने सिस्टम के रीस्टार्ट होने की प्रतीक्षा करें।

#### 3. सीरियल पोर्ट सत्यापित करें
वापस लॉग इन करने के बाद, सुनिश्चित करें कि आपका Arduino Uno सिस्टम द्वारा ठीक से पहचाना गया है।

- अपने Arduino Uno को USB पोर्ट से फिर से कनेक्ट करें।
- Arduino IDE खोलें।
- मेनू में **Tools > Port** पर जाएं और जांचें कि क्या `/dev/ttyUSB0` (या कुछ Arduino बोर्डों के लिए `/dev/ttyACM0`) सूचीबद्ध और चयनित है।
- यदि पोर्ट दिखाई नहीं देता है:
  - Arduino Uno को अनप्लग और रीप्लग करें।
  - USB कनेक्शन जांचें या कोई अलग केबल/पोर्ट आज़माएं।

#### 4. स्केच को फिर से अपलोड करें
सही पोर्ट चयनित होने के साथ, अपना स्केच फिर से अपलोड करने का प्रयास करें:

```cpp
void setup() {
  pinMode(LED_BUILTIN, OUTPUT); // Set the built-in LED pin (pin 13) as output
}

void loop() {
  digitalWrite(LED_BUILTIN, HIGH); // Turn the LED on
  delay(1000);                     // Wait for 1 second
  digitalWrite(LED_BUILTIN, LOW);  // Turn the LED off
  delay(1000);                     // Wait for 1 second
}
```

- Arduino IDE में **Upload** बटन (दायाँ तीर आइकन) पर क्लिक करें।
- यदि सफल होता है, तो आपके Arduino Uno पर बिल्ट-इन LED ब्लिंक करने लगनी चाहिए (1 सेकंड के लिए चालू, 1 सेकंड के लिए बंद)।

### अतिरिक्त समस्या निवारण
यदि उपरोक्त चरण समस्या का समाधान नहीं करते हैं, तो इन अतिरिक्त जांचों को आज़माएं:

#### ग्रुप सदस्यता जांचें
सत्यापित करें कि आपका यूज़र `dialout` ग्रुप का हिस्सा है:
- टर्मिनल में यह कमांड चलाएँ:
  ```bash
  groups $USER
  ```
- आउटपुट में `dialout` देखें। यदि यह गायब है, तो चरण 1 दोहराएं और सुनिश्चित करें कि आप लॉग आउट/इन करें या रीबूट करें।

#### अस्थायी अनुमति समाधान (दीर्घकालिक अनुशंसित नहीं)
यदि आपको त्वरित समाधान चाहिए, तो आप सीरियल पोर्ट की अनुमतियाँ मैन्युअल रूप से बदल सकते हैं:
- चलाएँ:
  ```bash
  sudo chmod a+rw /dev/ttyUSB0
  ```
- फिर अपलोड करने का प्रयास करें। नोट: यह अस्थायी है और Arduino को अनप्लग करने या रीबूट करने के बाद रीसेट हो जाएगा।

#### संघर्षकारी सॉफ़्टवेयर की जांच करें
कुछ सॉफ़्टवेयर, जैसे `brltty` (एक ब्रेल डिस्प्ले ड्राइवर), सीरियल पोर्ट में हस्तक्षेप कर सकते हैं। इसे हटाने के लिए:
- चलाएँ:
  ```bash
  sudo apt remove brltty
  ```
- अपने Arduino को फिर से कनेक्ट करें और फिर से प्रयास करें।

#### Arduino कनेक्शन सत्यापित करें
सुनिश्चित करें कि Ubuntu आपके Arduino को पहचानता है:
- चलाएँ:
  ```bash
  ls /dev/tty*
  ```
- Arduino प्लग इन होने पर `/dev/ttyUSB0` या `/dev/ttyACM0` देखें। यदि यह गायब है, तो अपनी USB केबल या पोर्ट जांचें।

#### Arduino IDE को रूट के रूप में चलाएं (अस्थायी वर्कअराउंड)
अंतिम उपाय के रूप में, आप IDE को सुपरयूजर विशेषाधिकारों के साथ चला सकते हैं:
- Arduino IDE बंद करें।
- एक टर्मिनल खोलें और चलाएँ:
  ```bash
  sudo arduino-ide
  ```
- यदि आवश्यक हो तो `arduino-ide` को सही कमांड या पथ से बदलें (उदाहरण के लिए, `/home/lzw/arduino-ide_2.3.6_Linux_64bit/arduino-ide`)।
- फिर से अपलोड करने का प्रयास करें। इस विधि का नियमित रूप से उपयोग करने से बचें, क्योंकि यह कम सुरक्षित है।

### समाधान की पुष्टि करें
इन चरणों का पालन करने के बाद, आपका स्केच सफलतापूर्वक अपलोड हो जाना चाहिए, और आपके Arduino Uno पर LED अपेक्षा के अनुरूप ब्लिंक करनी चाहिए। यदि आप अभी भी समस्याओं का सामना करते हैं, तो कृपया अतिरिक्त विवरण प्रदान करें, जैसे:
- `ls -l /dev/ttyUSB0` का आउटपुट (अनुमतियाँ जांचने के लिए)।
- Arduino IDE से कोई नया त्रुटि संदेश।

यदि आपको आगे सहायता की आवश्यकता हो तो मुझे बताएं!
