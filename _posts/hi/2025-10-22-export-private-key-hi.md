### सर्वर से GPG प्राइवेट की निर्यात करना

अपनी प्राइवेट की निर्यात करने के लिए, आपको SSH या टर्मिनल के माध्यम से सर्वर तक पहुंच की आवश्यकता होगी। GPG प्राइवेट की संवेदनशील होती हैं—उन्हें सुरक्षित रूप से संभालें और ट्रांसफर के बाद निर्यात की गई फ़ाइल को हटा दें।

1.  **अपनी सीक्रेट की सूची बनाएं** (ईमेल या की ID द्वारा की की पहचान करने के लिए):  
    ```
    gpg --list-secret-keys
    ```  
    `sec   rsa4096/ABC123DEF456 2023-01-01 [SC]` जैसी एंट्री देखें। ईमेल या की ID नोट कर लें (जैसे, `user@example.com` या `ABC123DEF456`).

2.  **प्राइवेट की निर्यात करें** (ASCII फॉर्मेट के लिए `--armor` का उपयोग करें, ट्रांसफर करना आसान होता है):  
    ```
    gpg --export-secret-keys --armor user@example.com > private-key.asc
    ```  
    - `user@example.com` को अपने वास्तविक ईमेल या की ID से बदलें।  
    - यह आपके करंट डायरेक्टरी में `private-key.asc` फ़ाइल बनाता है।  
    - यदि आपके पास एक से अधिक की हैं, तो सटीकता के लिए की ID निर्दिष्ट करें: `gpg --export-secret-keys --armor ABC123DEF456 > private-key.asc`.  
    - **सुरक्षा नोट**: यह की को उसके पासफ्रेज *बिना* निर्यात करता है। आयात करते समय यदि संकेत मिले तो आपको पासफ्रेज दर्ज करनी होगी।

3.  **फ़ाइल को सुरक्षित रूप से अपने लोकल लैपटॉप पर ट्रांसफर करें**:  
    - `scp` का उपयोग करें (अपने लोकल मशीन से):  
      ```
      scp user@server-ip:/path/to/private-key.asc ./
      ```  
      `user@server-ip` को अपने सर्वर के विवरण से और `/path/to/` को वास्तविक पथ से बदलें।  
    - विकल्प: बड़ी फ़ाइलों के लिए `rsync` या GUI विकल्प के लिए SFTP। ईमेल या असुरक्षित चैनलों से बचें।

4.  **सर्वर पर सफाई करें**:  
    ```
    rm private-key.asc
    ```  
    सुनिश्चित करने के लिए `ls` के साथ जांचें कि यह हट गई है।

### अपने लोकल लैपटॉप पर GPG प्राइवेट की आयात करना

मान लें कि आपके पास GPG इंस्टॉल है (जैसे, Linux पर `apt install gnupg`, macOS पर `brew install gnupg`, या Windows पर Chocolatey के माध्यम से)।

1.  **की आयात करें**:  
    ```
    gpg --import private-key.asc
    ```  
    - आपको "gpg: key ABC123DEF456: secret key imported" जैसा आउटपुट दिखाई देगा।  
    - आयात के दौरान यदि पासफ्रेज के लिए कहा जाए, तो मूल पासफ्रेज दर्ज करें।

2.  **आयात सत्यापित करें**:  
    ```
    gpg --list-secret-keys
    ```  
    आपकी की अब "sec" के अंतर्गत दिखाई देनी चाहिए।

3.  **वैकल्पिक: ट्रस्ट लेवल सेट करें** (यदि साइनिंग/ईमेल के लिए उपयोग कर रहे हैं):  
    ```
    gpg --edit-key user@example.com
    ```  
    - प्रॉम्प्ट में, `trust` टाइप करें, "ultimate" (विकल्प 5) चुनें, फिर `save` और `quit` करें।  
    - यह आपके लोकल मशीन पर की को पूरी तरह से विश्वसनीय के रूप में चिह्नित करता है।

### सुझाव और चेतावनियाँ
-   **पासफ्रेज**: हमेशा एक मजबूत पासफ्रेज का उपयोग करें। यदि आप इसे भूल जाते हैं, तो की बेकार हो जाती है।
-   **सबकी**: यदि आपकी की में सबकी हैं (जैसे, एन्क्रिप्शन के लिए), तो उन्हें भी `--export-secret-subkeys` के साथ निर्यात करें।
-   **बैकअप**: निर्यात को सुरक्षित वॉल्ट में स्टोर करें (जैसे, एन्क्रिप्टेड ड्राइव)।
-   **वर्जन**: GPG 2.x पर टेस्ट किया गया; 1.x पर भी इसी तरह काम करता है।
-   अधिक जानकारी के लिए, आधिकारिक GPG डॉक्यूमेंटेशन देखें: [GnuPG मैनुअल](https://www.gnupg.org/documentation/manuals/gnupg/)।
