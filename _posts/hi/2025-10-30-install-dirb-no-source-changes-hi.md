### DirB सोर्स कोड में बदलाव के विकल्प

अगर सोर्स कोड (जैसे `variables.h`) में एडिट करना संभव न हो—शायद रीड-ओनली सेटअप, पॉलिसी, या सिर्फ पसंद की वजह से—तो यहाँ कुछ सीधे तरीके हैं DirB को कंपाइल किए बिना चलाने के लिए। ये प्री-बिल्ट पैकेज या री-इम्प्लीमेंटेशन का इस्तेमाल करते हैं। मैं Linux (आपके बिल्ड पाथ के आधार पर) को प्राथमिकता दूंगा, लेकिन OS-स्पेसिफिक ट्वीक्स पर ध्यान दें।

#### 1. **पैकेज मैनेजर के जरिए इंस्टॉल करें (Debian/Ubuntu/Kali के लिए सबसे आसान)**
   DirB कई डिस्ट्रो रेपो में प्री-कंपाइल्ड पैकेज के रूप में उपलब्ध है, इसलिए सोर्स में बदलाव या बिल्डिंग की जरूरत नहीं।
   - **Kali Linux पर** (पेंटेस्टिंग टूल्स के लिए रिकमेंडेड):
     ```
     sudo apt update
     sudo apt install dirb
     ```
     - यह Kali रेपो में ऑफिशियली सपोर्टेड और मेंटेन है। [Kali Tools Page](https://www.kali.org/tools/dirb/)
   - **Ubuntu/Debian पर**:
     ```
     sudo apt update
     sudo apt install dirb
     ```
     - अगर नहीं मिला (पुराने वर्जन में शायद न हो), universe रेपो enable करें: `sudo add-apt-repository universe && sudo apt update`.
   - **वेरिफिकेशन**: इंस्टॉल के बाद `dirb --help` चलाएं। वर्डलिस्ट `/usr/share/dirb/wordlists/` में होती हैं।
   - **यह काम क्यों करता है**: पैकेज अपस्ट्रीम में सभी फिक्सेज (मल्टीपल डेफिनिशन समेत) हैंडल करते हैं।

   अगर आप किसी अलग डिस्ट्रो पर हैं:
   - **Fedora/RHEL**: `sudo dnf install dirb` (अगर EPEL रेपो में हो; EPEL ऐड करें अगर जरूरत हो: `sudo dnf install epel-release`)।
   - **Arch**: `sudo pacman -S dirb`.

#### 2. **Python री-इम्प्लीमेंटेशन इस्तेमाल करें (क्रॉस-प्लेटफॉर्म, C कंपाइल नहीं)**
   ओरिजिनल DirB C-बेस्ड है और बिल्ड करने में झंझट वाली है, लेकिन एक मॉडर्न Python पोर्ट है जो फंक्शनली एक जैसी (या बेहतर) है और pip के जरिए इंस्टॉल होती है—सोर्स ट्वीक की जरूरत नहीं।
   - **Repo**: [ct-Open-Source/dirb on GitHub](https://github.com/ct-Open-Source/dirb)
   - **इंस्टॉल**:
     ```
     pip install git+https://github.com/ct-Open-Source/dirb.git
     ```
     - या क्लोन करें: `git clone https://github.com/ct-Open-Source/dirb.git && cd dirb && pip install .`
   - **यूसेज**: ओरिजिनल जैसी ही, जैसे, `dirb https://example.com /usr/share/wordlists/dirb/common.txt`.
   - **फायदे**: एक्टिवली मेंटेन, कुछ केस में तेज, Mac/Windows/Linux पर काम करती है। रिकर्सिव स्कैनिंग जैसे एक्स्ट्रा शामिल।
   - **नुकसान**: "ऑफिशियल" बाइनरी नहीं, लेकिन आउटपुट करीब-करीब मैच करती है।

#### 3. **प्री-बिल्ट बाइनरी डाउनलोड करें (लिमिटेड उपलब्धता)**
   - ऑफिशियल SourceForge रिलीज ज्यादातर सोर्स टारबॉल (जैसे, dirb222.tar.gz) हैं, जिन्हें अभी भी बिल्डिंग की जरूरत है।
   - यूनिवर्सल बाइनरी नहीं, लेकिन अगर आप Kali/Parrot OS पर हैं, तो पैकेज रूट सबसे अच्छा है।
   - एक हैक के तौर पर: अगर आपके पास Docker है, Kali इमेज पुल करें और अंदर DirB चलाएं:
     ```
     docker run -it kalilinux/kali-rolling dirb http://example.com
     ```
     - इंस्टॉल की जरूरत नहीं; कंटेनर प्री-बिल्ट होते हैं।

#### त्वरित तुलना

| विधि              | OS सपोर्ट          | प्रयास | नोट्स |
|---------------------|---------------------|--------|-------|
| **Apt/DNF/Pacman** | Linux (Debian/Fedora/Arch) | कम | प्री-कंपाइल्ड; वर्डलिस्ट शामिल। |
| **Python पोर्ट**    | कोई भी (Python 3+)    | कम | मॉडर्न विकल्प; सिर्फ pip। |
| **Docker**         | कोई भी (Docker के साथ)    | मध्यम | आइसोलेटेड; होस्ट इंस्टॉल नहीं। |

अगर कोई भी फिट न बैठे (जैसे, रिस्ट्रिक्टेड एनवीरोंमेंट), विकल्पों पर विचार करें जैसे Gobuster (`go install github.com/OJ/gobuster/v3@latest`)—यह Go-बेस्ड है, बिना इश्यू के आसानी से कंपाइल हो जाता है, और समान डायरेक्टरी बस्टिंग करता है।

आप किस OS पर हैं? इससे सही विकल्प चुनने में मदद मिलेगी। अगर कोई दिक्कत आए, तो एरर/आउटपुट शेयर करें।
