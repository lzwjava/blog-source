**Claude Code** में, यह स्निपेट

```json
"cache_control": {
  "type": "ephemeral"
}
```

का मतलब है कि मॉडल द्वारा उत्पन्न प्रतिक्रिया (या आर्टिफैक्ट) **पुन: उपयोग के लिए कैश नहीं की जाती है**।

### इसका अस्तित्व क्यों है

सामान्य रूप से, Claude Code (और अन्य Anthropic SDK) समान प्रॉम्प्ट दोबारा भेजने पर चीजों को तेज करने के लिए प्रतिक्रियाओं को कैश कर सकते हैं। लेकिन जब आप `"cache_control": { "type": "ephemeral" }` सेट करते हैं, तो आप API को यह बता रहे होते हैं:

* **इस परिणाम को कैश में स्थायी रूप से संग्रहीत न करें**।
* यदि बाद में कोई अन्य समान अनुरोध आता है तो इसका **पुन: उपयोग न करें**।
* प्रतिक्रिया को **अल्पकालिक** मानें—केवल इसी कॉल के लिए।

### कब उपयोग करें

यह तब उपयोगी है जब:

* आपके अनुरोध में **संवेदनशील या व्यक्तिगत डेटा** शामिल है जिसे आप कैश नहीं करना चाहते।
* आप **एक-बार के क्वेरी** भेज रहे हैं जिन्हें पुन: उपयोग से कोई लाभ नहीं होगा।
* आप हर बार **ताजा आउटपुट** चाहते हैं, भले ही अनुरोध टेक्स्ट समान हो।

संक्षेप में, `"ephemeral"` Claude के आउटपुट को *क्षणभंगुर* बना देता है—यह इस इंटरैक्शन के लिए निजी होता है, और कॉल्स के बीच कोई कैशिंग नहीं होती।

क्या आप चाहेंगे कि मैं यह भी समझाऊं कि यह Claude Code में डिफ़ॉल्ट `cache_control` व्यवहार (जैसे `type: "default"` बनाम `"none"` बनाम `"ephemeral"`) से कैसे भिन्न है?
