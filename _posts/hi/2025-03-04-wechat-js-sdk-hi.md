### मुख्य बिंदु
- "weixin-js-sdk" संस्करण "^1.2.0" का उपयोग करने के लिए, आप पहले इसे npm के माध्यम से स्थापित करेंगे और फिर इसे अपने JavaScript कोड में आयात करेंगे।
- अनुसंधान सुझाव देता है कि आप इसे अपने WeChat सार्वजनिक खाते के विवरणों, जैसे appId और signature, के साथ सेटअप करें, जो सर्वर-साइड से उत्पन्न होते हैं।
- सबूत इस ओर इशारा करते हैं कि आप उपयोग के लिए आधिकारिक WeChat JS SDK दस्तावेज़ का पालन करें, क्योंकि पैकेज एक व्रैपर है।

### स्थापना
अपने प्रोजेक्ट डायरेक्टरी में पैकेज को npm के साथ स्थापित करने से शुरू करें:
- `npm install weixin-js-sdk` चलाएं ताकि इसे आपकी निर्भरताओं में जोड़ें।

### आयात और उपयोग
JavaScript फ़ाइल में SDK को आयात करें, फिर WeChat APIs के साथ इसे सेटअप और उपयोग करें:
- ES6 के लिए `import wx from 'weixin-js-sdk';` और CommonJS के लिए `const wx = require('weixin-js-sdk');` का उपयोग करें।
- `wx.config({ appId: 'your_app_id', timestamp: your_timestamp, nonceStr: 'your_nonce_str', signature: 'your_signature', jsApiList: ['onMenuShareAppMessage'] })` के साथ सेटअप करें।
- सफलता को `wx.ready()` और त्रुटियों को `wx.error()` के साथ संभालें।

### सर्वर-साइड सेटअप
ध्यान दें कि आपको एक WeChat सार्वजनिक खाता चाहिए, अपने डोमेन को बाइंड करें, और सर्वर पर WeChat के API का उपयोग करके एक सिग्नेचर उत्पन्न करें, क्योंकि इसमें संवेदनशील पात्र होते हैं।

### सर्वेक्षण नोट: "weixin-js-sdk" संस्करण "^1.2.0" का विस्तृत मार्गदर्शन

यह नोट "weixin-js-sdk" पैकेज, विशेष रूप से संस्करण "^1.2.0", का उपयोग करने के लिए एक व्यापक मार्गदर्शन प्रदान करता है, जो WeChat JS SDK का एक व्रैपर है, जिससे वेब डेवलपर्स अपने एप्लिकेशनों में WeChat के मोबाइल क्षमताओं का लाभ उठा सकते हैं। पैकेज CommonJS और TypeScript के साथ एकीकरण को सुविधाजनक बनाता है, जिससे यह आधुनिक वेब डेवलपमेंट पर्यावरणों जैसे browserify और webpack के लिए उपयुक्त है। नीचे, हम प्रक्रिया का विवरण देते हैं, उपलब्ध दस्तावेज़ और उदाहरणों से, जो लागू करने के लिए एक गहन समझ सुनिश्चित करता है।

#### पृष्ठभूमि और संदर्भ
"weixin-js-sdk" पैकेज, जैसा कि इसका npm सूचीकरण से पता चलता है, आधिकारिक WeChat JS SDK, संस्करण 1.6.0, को एन्कैप्सुलेट करने के लिए डिज़ाइन किया गया है, और वर्तमान में npm पर संस्करण 1.6.5 है, जो 3 मार्च, 2025 को प्रकाशित हुआ था। पैकेज विवरण CommonJS और TypeScript का समर्थन उल्लेख करता है, जिससे यह Node.js पर्यावरणों और आधुनिक बंडलर्स के लिए अनुकूलित है। उपयोगकर्ता द्वारा "^1.2.0" का उल्लेख, जो 1.2.0 से 2.0.0 तक के किसी भी संस्करण को अनुमति देता है, और वर्तमान संस्करण 1.6.5 होने के कारण, यह उचित है कि यह मार्गदर्शन के साथ संगत है, हालांकि आधिकारिक दस्तावेज़ में संस्करण-विशिष्ट परिवर्तन की जांच करनी चाहिए।

WeChat JS SDK, आधिकारिक दस्तावेज़ के अनुसार, WeChat Official Accounts Platform द्वारा प्रदान की गई एक वेब डेवलपमेंट टूलकिट है, जो शेयरिंग, QR कोड स्कैनिंग और स्थान सेवा जैसे सुविधाएं प्रदान करती है। पैकेज का GitHub रिपोजिटरी, yanxi123-com द्वारा बनाए गए, यह एक आधिकारिक SDK का सीधा पोर्ट है, जिसमें उपयोग निर्देश WeChat JS SDK दस्तावेज़ की ओर इशारा करते हैं।

#### स्थापना प्रक्रिया
शुरू करने के लिए, पैकेज को npm के माध्यम से स्थापित करें, जो Node.js प्रोजेक्ट्स के लिए मानक पैकेज मैनेजर है। कमांड सीधा है:
- `npm install weixin-js-sdk` अपने प्रोजेक्ट डायरेक्टरी में चलाएं। यह "^1.2.0" के साथ संगत नवीनतम संस्करण को डाउनलोड करेगा, संभवतः 1.6.5, npm रजिस्टर की वर्तमान स्थिति के अनुसार।

yarn का उपयोग करने वाले लोगों के लिए, एक विकल्प होगा `yarn add weixin-js-sdk`, जिससे पैकेज आपकी प्रोजेक्ट की निर्भरताओं में जोड़ा जाएगा। यह कदम महत्वपूर्ण है क्योंकि यह SDK को आपकी प्रोजेक्ट में एकीकृत करता है, जिससे इसे आपकी JavaScript फ़ाइलों में आयात करने के लिए उपलब्ध हो जाता है।

#### आयात और प्रारंभिक सेटअप
स्थापित होने के बाद, अगला कदम SDK को अपने कोड में आयात करना है। पैकेज दोनों ES6 और CommonJS मॉड्यूलों का समर्थन करता है, विभिन्न डेवलपमेंट पसंदों को पूरा करता है:
- ES6 के लिए, अपने JavaScript फ़ाइल के शीर्ष पर `import wx from 'weixin-js-sdk';` का उपयोग करें।
- CommonJS के लिए, `const wx = require('weixin-js-sdk');` का उपयोग करें, जो आमतौर पर Node.js पर्यावरणों में ट्रांसपाइलेशन के बिना होता है।

इस आयात ने `wx` ऑब्जेक्ट को प्रकट किया, जो WeChat के JS APIs के साथ इंटरैक्ट करने के लिए मुख्य इंटरफ़ेस है। ध्यान दें कि, स्क्रिप्ट टैग के माध्यम से SDK को शामिल करने के विपरीत, जो `wx` को ग्लोबल रूप से उपलब्ध बनाता है, npm के माध्यम से एक बंडल्ड पर्यावरण (जैसे webpack) में आयात करने में, कुछ मामलों में `wx` को ग्लोबल window ऑब्जेक्ट से जोड़ने की आवश्यकता हो सकती है, हालांकि पैकेज के डिज़ाइन से पता चलता है कि यह इसे आंतरिक रूप से संभालता है, CommonJS संगतता के कारण।

#### सेटअप और उपयोग
सेटअप प्रक्रिया में `wx.config` को सेटअप करना शामिल है, जो आपके WeChat सार्वजनिक खाते के विवरणों के साथ SDK को प्रारंभ करने के लिए आवश्यक है। इस चरण में, जो आमतौर पर सुरक्षा कारणों से सर्वर-साइड पर उत्पन्न होते हैं, आवश्यक पैरामीटर शामिल हैं:
- **आवश्यक पैरामीटर:** `debug` (बूलियन, डिबगिंग के लिए), `appId` (आपका WeChat app ID), `timestamp` (वर्तमान समय-स्टैम्प सेकंड में), `nonceStr` (रैंडम स्ट्रिंग), `signature` (jsapi_ticket और अन्य पैरामीटर का उपयोग करके उत्पन्न), और `jsApiList` (आपका उपयोग करने वाले APIs का एक एरे, उदाहरण के लिए, `['onMenuShareAppMessage', 'onMenuShareTimeline']`).

एक बुनियादी सेटअप उदाहरण है:
```javascript
wx.config({
    debug: true,
    appId: 'your_app_id',
    timestamp: your_timestamp,
    nonceStr: 'your_nonce_str',
    signature: 'your_signature',
    jsApiList: ['onMenuShareAppMessage', 'onMenuShareTimeline']
});
```

सेटअप के बाद, परिणाम को संभालें:
- `wx.ready(function() { ... });` का उपयोग करें ताकि सेटअप सफलतापूर्वक सत्यापित होने के बाद कोड को कार्यान्वित करें। यह वही जगह है जहां आप WeChat APIs, जैसे शेयरिंग, को कॉल कर सकते हैं:
  ```javascript
  wx.ready(function () {
      wx.onMenuShareAppMessage({
          title: 'Your title',
          desc: 'Your description',
          link: 'Your link',
          imgUrl: 'Your image URL',
          success: function () {
              // सफल शेयर के लिए कॉलबैक
          },
          cancel: function () {
              // रद्द शेयर के लिए कॉलबैक
          }
      });
  });
  ```
- `wx.error(function(res) { ... });` का उपयोग करें ताकि सेटअप त्रुटियों को संभालें, जो सिग्नेचर या डोमेन सेटिंग्स के साथ समस्याओं को इंगित कर सकते हैं।

#### सर्वर-साइड आवश्यकताएं और सिग्नेचर उत्पत्ति
एक महत्वपूर्ण पहलू सर्वर-साइड सेटअप है, क्योंकि सिग्नेचर उत्पत्ति संवेदनशील पात्र और WeChat के सर्वरों पर API कॉल शामिल होती है। सिग्नेचर उत्पन्न करने के लिए:
- पहले, appId और appSecret का उपयोग करके एक एक्सेस टोकन प्राप्त करें WeChat के API के माध्यम से।
- फिर, एक्सेस टोकन का उपयोग करके एक jsapi_ticket प्राप्त करें।
- अंत में, jsapi_ticket, वर्तमान URL, एक nonce स्ट्रिंग और timestamp का उपयोग करके सिग्नेचर उत्पन्न करें, [WeChat JS SDK दस्तावेज़ के अनुप्रयोग 1](https://developers.weixin.qq.com/doc/offiaccount/OA_Web_Apps/JS-SDK.html#62) में वर्णित एल्गोरिथम का पालन करते हुए।

यह प्रक्रिया आमतौर पर PHP, Java, Node.js या Python जैसे भाषाओं में लागू होती है, जिसमें दस्तावेज़ में सांपल कोड प्रदान किया गया है। एक्सेस टोकन और jsapi_ticket को 7200 सेकंड के लिए कश किया जाए ताकि रेट लिमिट्स को हिट करने से बचा जाए, जैसा कि दस्तावेज़ में उल्लेख किया गया है।

अधिक रूप से, सुनिश्चित करें कि आपका डोमेन आपके WeChat सार्वजनिक खाते से बाइंड है:
- WeChat Official Accounts Platform में लॉग इन करें, Official Account Settings > Feature Settings पर जाएं, और JS API Secure Domain Name दर्ज करें। यह कदम सुरक्षा और API एक्सेस के लिए महत्वपूर्ण है।

#### संस्करण विचार
उपयोगकर्ता द्वारा "^1.2.0" का उल्लेख, और पैकेज का नवीनतम संस्करण 1.6.5 होने के कारण, यह उल्लेखनीय है कि पैकेज संस्करण में पैकेजिंग में परिवर्तन हो सकता है, जबकि आधार SDK संस्करण 1.6.0 है, आधिकारिक स्रोत के अनुसार। उपयोग को समान रखने की उम्मीद है, लेकिन संस्करण 1.2.0 के लिए, npm चेंजलॉग या GitHub रिलीज में किसी भी उल्लिखित परिवर्तन की जांच करें, हालांकि सामान्य मार्गदर्शन से पता चलता है कि बुनियादी उपयोग पर कम प्रभाव पड़ता है।

#### उदाहरण और अतिरिक्त संसाधन
प्रैक्टिकल लागू करने के लिए, उदाहरण GitHub रिपोजिटरी में मिल सकते हैं, जैसे [yanxi123-com/weixin-js-sdk](https://github.com/yanxi123-com/weixin-js-sdk), जो स्रोत और उपयोग नोट्स प्रदान करता है। अतिरिक्त रूप से, आधिकारिक दस्तावेज़ में DEMO लिंक शामिल हैं, जैसे [WeChat JS-SDK Examples](https://www.weixinsxy.com/jssdk/), हालांकि खोज में विशेष सामग्री का उल्लेख नहीं किया गया, जिससे पता चलता है कि साइट पर सांपल कोड और ज़िप फ़ाइलें के लिए जांच करें।

#### अपेक्षित विवरण: बंडलर्स के साथ एकीकरण
एक रोचक पहलू पैकेज का बंडलर्स जैसे webpack के साथ संगतता है, जो बुनियादी उपयोग से स्पष्ट नहीं है। यह मोड्यूलर डेवलपमेंट को सुविधाजनक बनाता है, संभवतः बड़े प्रोजेक्टों में निर्भरता प्रबंधन को सरल बनाता है, और TypeScript का समर्थन करता है, जो स्क्रिप्ट टैग के माध्यम से शामिल होने के साथ परिचित उपयोगकर्ताओं के लिए अपेक्षित हो सकता है।

संक्षेप में, "weixin-js-sdk" संस्करण "^1.2.0" का उपयोग npm के माध्यम से स्थापित करना, इसे अपने कोड में आयात करना, सर्वर-उत्पन्न पैरामीटरों के साथ इसे सेटअप करना, और API उपयोग के लिए आधिकारिक WeChat JS SDK दस्तावेज़ का पालन करना शामिल है। सर्वर-साइड सेटअप के लिए सिग्नेचर उत्पत्ति और डोमेन बाइंडिंग सुनिश्चित करें, और आधुनिक वेब डेवलपमेंट के लिए पैकेज के बंडलर संगतता को विचार में लें। विस्तृत उदाहरणों के लिए, प्रदान किए गए GitHub रिपोजिटरी और आधिकारिक दस्तावेज़ लिंक का पता लगाएं।
