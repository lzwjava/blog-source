आपके द्वारा प्रदान की गई डायरेक्टरी संरचना से एक साधारण Rust प्रोग्राम चलाने के लिए, ऐसा लगता है कि आप एक Rust प्रोजेक्ट (`lzwjava.github.io/rust`) में काम कर रहे हैं जिसमें `src` फ़ोल्डर है जिसमें `main.rs` और `simple.rs` फाइलें हैं। यहाँ बताया गया है कि आप एक साधारण Rust प्रोग्राम कैसे चला सकते हैं, यह मानते हुए कि आप `main.rs` या `simple.rs` में कोड चलाना चाहते हैं।

### Rust प्रोग्राम चलाने के चरण

1. **प्रोजेक्ट संरचना की जाँच करें**:
   - आपके प्रोजेक्ट में एक `Cargo.toml` फ़ाइल है, जो दर्शाता है कि यह Cargo (Rust के बिल्ड टूल) द्वारा प्रबंधित एक Rust प्रोजेक्ट है।
   - `src/main.rs` फ़ाइल एक Rust बाइनरी एप्लिकेशन के लिए डिफ़ॉल्ट एंट्री पॉइंट है।
   - `simple.rs` फ़ाइल संभवतः एक मॉड्यूल या कोई अन्य सोर्स फ़ाइल है, लेकिन यह सीधे तब तक नहीं चलेगी जब तक कि इसे `main.rs` में संदर्भित न किया गया हो या `Cargo.toml` में बाइनरी के रूप में कॉन्फ़िगर न किया गया हो।

2. **Cargo का उपयोग करके प्रोग्राम चलाएँ**:
   - यदि आप `src/main.rs` में कोड चलाना चाहते हैं, तो प्रोजेक्ट रूट (`~/projects/lzwjava.github.io/rust`) से निम्न कमांड का उपयोग करें:
     ```bash
     cargo run
     ```
     यह कमांड:
     - प्रोजेक्ट को कंपाइल करती है (डिपेंडेंसीज़ को रिज़ॉल्व करती है और कोड बिल्ड करती है)।
     - `src/main.rs` से जेनरेट की गई बाइनरी को एक्ज़िक्यूट करती है।
     - आउटपुट टर्मिनल में दिखाई देगा।

3. **यदि आप `simple.rs` चलाना चाहते हैं**:
   - डिफ़ॉल्ट रूप से, Cargo मुख्य एंट्री पॉइंट के रूप में `src/main.rs` की अपेक्षा करता है। यदि `simple.rs` एक मॉड्यूल है, तो आपको इसे `main.rs` में संदर्भित करना होगा। उदाहरण के लिए, `src/main.rs` में:
     ```rust
     mod simple; // simple मॉड्यूल घोषित करता है

     fn main() {
         simple::some_function(); // simple.rs से एक फ़ंक्शन को कॉल करता है
     }
     ```
     फिर ऊपर बताए अनुसार `cargo run` चलाएँ।
   - यदि `simple.rs` को एक स्टैंडअलोन बाइनरी के रूप में इरादा किया गया है, तो आपको इसे `Cargo.toml` में कॉन्फ़िगर करना होगा। `Cargo.toml` में निम्नलिखित जोड़ें:
     ```toml
     [[bin]]
     name = "simple"
     path = "src/simple.rs"
     ```
     फिर चलाएँ:
     ```bash
     cargo run --bin simple
     ```

4. **विकल्प: Cargo के बिना चलाएँ**:
   - यदि आप Cargo के बिना एक एकल फ़ाइल (जैसे `simple.rs`) को कंपाइल और चलाना चाहते हैं, तो सुनिश्चित करें कि इसमें एक `main` फ़ंक्शन है और `rustc` कंपाइलर का उपयोग करें:
     ```bash
     rustc src/simple.rs
     ./simple
     ```
     यह करंट डायरेक्टरी में `simple` नाम की एक एक्ज़िक्यूटेबल फ़ाइल जेनरेट करता है और इसे चलाता है। ध्यान दें: यह तरीका डिपेंडेंसीज़ को हैंडल नहीं करता है, इसलिए यह केवल सरल, स्वतंत्र प्रोग्राम्स के लिए उपयुक्त है।

5. **त्रुटियों की जाँच करें**:
   - यदि आपको त्रुटियाँ मिलती हैं, तो सुनिश्चित करें:
     - Rust और Cargo इंस्टॉल हैं (`rustc --version` और `cargo --version` से सत्यापित करें)।
     - `Cargo.toml` आवश्यक डिपेंडेंसीज़ के साथ सही ढंग से कॉन्फ़िगर है।
     - `main.rs` या `simple.rs` में कोड में एक वैध `fn main()` फ़ंक्शन है।
   - बाइनरी बनाए बिना कोड सत्यापित करने के लिए `cargo check` चलाएँ:
     ```bash
     cargo check
     ```

6. **उदाहरण `simple.rs`**:
   यदि `simple.rs` में कुछ इस तरह है:
   ```rust
   fn main() {
       println!("Hello from simple!");
   }
   ```
   और आपने इसे `Cargo.toml` में एक बाइनरी के रूप में जोड़ा है, तो `cargo run --bin simple` चलाने पर `Hello from simple!` प्रिंट होगा।

### नोट्स
- `target/debug` डायरेक्टरी में कंपाइल की गई आर्टिफैक्ट्स होती हैं। आप बाइनरी को सीधे `./target/debug/<project_name>` के साथ चला सकते हैं (`<project_name>` को `Cargo.toml` में नाम से बदलें), लेकिन `cargo run` को प्राथमिकता दी जाती है क्योंकि यह कंपाइलेशन और एक्ज़िक्यूशन को हैंडल करता है।
- यदि `simple.rs` एक मॉड्यूल है, तो सुनिश्चित करें कि यह `mod simple;` के साथ `main.rs` में सही ढंग से एकीकृत है।
- अधिक जटिल सेटअप या `simple.rs` में विशिष्ट कोड के लिए, विशिष्ट मार्गदर्शन के लिए `simple.rs` या `Cargo.toml` की सामग्री साझा करें।

यदि "run simple here" से आपका कोई विशिष्ट मतलब है (जैसे कोई विशिष्ट फ़ंक्शन या कॉन्फ़िगरेशन), तो कृपया अधिक विवरण प्रदान करें, और मैं जवाब को परिष्कृत कर दूंगा!
