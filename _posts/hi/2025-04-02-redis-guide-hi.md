यहाँ Redis का एक व्यापक मार्गदर्शक है, जो एक ओपन-सोर्स, इन-मेमोरी डेटा स्ट्रक्चर स्टोर है, जिसे व्यापक रूप से डेटाबेस, कैश और मैसेज ब्रोकर के रूप में उपयोग किया जाता है। यह मार्गदर्शक इसके मूल सिद्धांतों, विशेषताओं, उपयोग के मामलों, इंस्टालेशन, बुनियादी ऑपरेशन्स और उन्नत अवधारणाओं को कवर करेगा।

---

### Redis क्या है?
Redis (रिमोट डिक्शनरी सर्वर) एक हाई-परफॉर्मेंस, की-वैल्यू स्टोर है जो मुख्य रूप से मेमोरी में काम करता है, जिससे यह अत्यंत तेज़ हो जाता है। यह विभिन्न डेटा स्ट्रक्चर जैसे स्ट्रिंग्स, हैशेस, लिस्ट्स, सेट्स, सॉर्टेड सेट्स, बिटमैप्स, हाइपरलॉगलॉग्स और जियोस्पेशियल इंडेक्सेस को सपोर्ट करता है। साल्वाटोर सैनफिलिप्पो द्वारा 2009 में बनाया गया, Redis अब एक कम्युनिटी द्वारा मेंटेन किया जाता है और Redis Inc. द्वारा प्रायोजित है।

मुख्य विशेषताएँ:
- **इन-मेमोरी**: डेटा रैम में संग्रहीत होता है जिससे कम विलंबता वाली एक्सेस मिलती है।
- **स्थायित्व**: ड्यूरेबिलिटी के लिए वैकल्पिक डिस्क पर्सिस्टेंस प्रदान करता है।
- **बहुमुखी**: सरल की-वैल्यू जोड़े से परे जटिल डेटा स्ट्रक्चर को सपोर्ट करता है।
- **स्केलेबल**: हाई अवेलेबिलिटी के लिए क्लस्टरिंग और रेप्लिकेशन प्रदान करता है।

---

### Redis क्यों उपयोग करें?
Redis अपनी गति और लचीलेपन के लिए लोकप्रिय है। सामान्य उपयोग के मामले:
1. **कैशिंग**: बार-बार एक्सेस किए जाने वाले डेटा (जैसे API रिस्पॉन्सेस, वेब पेज) को संग्रहीत करके एप्लिकेशन की गति बढ़ाता है।
2. **सत्र प्रबंधन**: वेब एप्लिकेशन में यूजर सत्र डेटा संग्रहीत करता है।
3. **रियल-टाइम एनालिटिक्स**: मेट्रिक्स, लीडरबोर्ड या इवेंट काउंटर को ट्रैक करता है।
4. **पब/सब मैसेजिंग**: प्रक्रियाओं या सेवाओं के बीच रियल-टाइम मैसेजिंग सक्षम करता है।
5. **टास्क क्यू**: बैकग्राउंड जॉब्स (जैसे Celery जैसे टूल्स के साथ) प्रबंधित करता है।
6. **जियोस्पेशियल एप्लिकेशन**: लोकेशन-आधारित क्वेरीज (जैसे, आस-पास के पॉइंट्स ऑफ इंटरेस्ट ढूँढना) को हैंडल करता है।

---

### मुख्य विशेषताएँ
1. **डेटा स्ट्रक्चर**:
   - **स्ट्रिंग्स**: सरल की-वैल्यू जोड़े (जैसे, `SET key "value"`)।
   - **लिस्ट्स**: क्रमबद्ध संग्रह (जैसे, `LPUSH mylist "item"`)।
   - **सेट्स**: अक्रमबद्ध, अद्वितीय संग्रह (जैसे, `SADD myset "item"`)।
   - **सॉर्टेड सेट्स**: रैंकिंग के लिए स्कोर वाले सेट (जैसे, `ZADD leaderboard 100 "player1"`)।
   - **हैशेस**: की-वैल्यू मैपिंग (जैसे, `HSET user:1 name "Alice"`)।
   - **बिटमैप्स, हाइपरलॉगलॉग्स, स्ट्रीम्स**: विशेष उपयोग के मामलों जैसे यूनिक यूजर्स गिनने या इवेंट स्ट्रीमिंग के लिए।

2. **स्थायित्व**:
   - **RDB (स्नैपशॉटिंग)**: डेटा को समय-विशेष की स्नैपशॉट के रूप में डिस्क पर समय-समय पर सेव करता है।
   - **AOF (अपेंड-ओनली फाइल)**: ड्यूरेबिलिटी के लिए हर राइट ऑपरेशन को लॉग करता है; डेटासेट को रीबिल्ड करने के लिए रीप्ले किया जा सकता है।

3. **रेप्लिकेशन**: हाई अवेलेबिलिटी और रीड स्केलेबिलिटी के लिए मास्टर-स्लेव रेप्लिकेशन।
4. **क्लस्टरिंग**: हॉरिजॉन्टल स्केलिंग के लिए डेटा को कई नोड्स में वितरित करता है।
5. **एटॉमिक ऑपरेशन्स**: `INCR` या `MULTI` जैसे कमांड्स के साथ सुरक्षित समवर्ती एक्सेस सुनिश्चित करता है।
6. **Lua स्क्रिप्टिंग**: कस्टम सर्वर-साइड लॉजिक की अनुमति देता है।
7. **पब/सब**: रियल-टाइम कम्युनिकेशन के लिए हल्का मैसेजिंग सिस्टम।

### इंस्टालेशन
Redis लिनक्स, macOS और विंडोज (WSL या अनऑफिशियल बिल्ड्स के माध्यम से) पर उपलब्ध है। लिनक्स सिस्टम पर इसे इंस्टॉल करने का तरीका यहाँ बताया गया है:

1. **पैकेज मैनेजर के माध्यम से** (Ubuntu/Debian):
   ```bash
   sudo apt update
   sudo apt install redis-server
   ```

2. **सोर्स से**:
   ```bash
   wget http://download.redis.io/releases/redis-7.0.15.tar.gz
   tar xzf redis-7.0.15.tar.gz
   cd redis-7.0.15
   make
   sudo make install
   ```

3. **Redis स्टार्ट करें**:
   ```bash
   redis-server
   ```

4. **इंस्टालेशन वेरिफाई करें**:
   ```bash
   redis-cli ping
   ```
   आउटपुट: `PONG`

5. **कॉन्फ़िगरेशन**: पर्सिस्टेंस, मेमोरी लिमिट, या विशिष्ट IPs से बाइंडिंग जैसी सेटिंग्स को ट्वीक करने के लिए `/etc/redis/redis.conf` (या समकक्ष) एडिट करें।

### बेसिक ऑपरेशन्स
Redis `redis-cli` या क्लाइंट लाइब्रेरीज़ के माध्यम से एक सरल कमांड-आधारित इंटरफेस का उपयोग करता है। यहाँ कुछ उदाहरण दिए गए हैं:

#### स्ट्रिंग्स
- वैल्यू सेट करें: `SET name "Alice"`
- वैल्यू प्राप्त करें: `GET name` → `"Alice"`
- इंक्रीमेंट: `INCR counter` → `1` (2, 3, आदि तक बढ़ता है)

#### लिस्ट्स
- बाईं ओर जोड़ें: `LPUSH mylist "item1"`
- दाईं ओर जोड़ें: `RPUSH mylist "item2"`
- बाईं ओर से पॉप करें: `LPOP mylist` → `"item1"`

#### सेट्स
- आइटम जोड़ें: `SADD myset "apple" "banana"`
- मेंबर्स लिस्ट करें: `SMEMBERS myset` → `"apple" "banana"`
- मेंबरशिप चेक करें: `SISMEMBER myset "apple"` → `1` (सत्य)

#### हैशेस
- फील्ड्स सेट करें: `HSET user:1 name "Bob" age "30"`
- फील्ड प्राप्त करें: `HGET user:1 name` → `"Bob"`
- सभी फील्ड्स प्राप्त करें: `HGETALL user:1`

#### सॉर्टेड सेट्स
- स्कोर के साथ जोड़ें: `ZADD leaderboard 100 "player1" 200 "player2"`
- टॉप स्कोर प्राप्त करें: `ZRANGE leaderboard 0 1 WITHSCORES` → `"player1" "100" "player2" "200"`

### एडवांस्ड कॉन्सेप्ट्स
1. **पर्सिस्टेंस कॉन्फ़िगरेशन**:
   - RDB सक्षम करें: `redis.conf` में `save 60 1000` सेट करें (हर 60s में सेव करें यदि 1000 कीज़ बदलती हैं)।
   - AOF सक्षम करें: राइट लॉगिंग के लिए `appendonly yes` सेट करें।

2. **रेप्लिकेशन**:
   - एक स्लेव कॉन्फ़िगर करें: `SLAVEOF master_ip master_port`।
   - स्टेटस चेक करें: `INFO REPLICATION`।

3. **क्लस्टरिंग**:
   - `redis.conf` में `cluster-enabled yes` के साथ सक्षम करें।
   - नोड्स सेट अप करने के लिए `redis-cli --cluster create` का उपयोग करें।

4. **एविक्शन पॉलिसीज**:
   - `maxmemory` और `LRU` (least recently used) या `LFU` (least frequently used) जैसी पॉलिसीज के साथ मेमोरी उपयोग नियंत्रित करें।
   - उदाहरण: `maxmemory-policy allkeys-lru`।

5. **ट्रांजैक्शन्स**:
   - कमांड्स ग्रुप करें: `MULTI`, उसके बाद कमांड्स, फिर `EXEC`।
   - उदाहरण:
     ```
     MULTI
     SET key1 "value1"
     SET key2 "value2"
     EXEC
     ```

6. **पब/सब**:
   - सब्सक्राइब: `SUBSCRIBE channel1`
   - पब्लिश: `PUBLISH channel1 "Hello"`

### क्लाइंट लाइब्रेरीज़
Redis कई प्रोग्रामिंग भाषाओं को सपोर्ट करता है। उदाहरण:
- **Python**: `redis-py` (`pip install redis`)
  ```python
  import redis
  r = redis.Redis(host='localhost', port=6379, db=0)
  r.set('key', 'value')
  print(r.get('key'))  # b'value'
  ```
- **Node.js**: `ioredis`
  ```javascript
  const Redis = require('ioredis');
  const redis = new Redis();
  redis.set('key', 'value');
  redis.get('key').then(console.log); // 'value'
  ```

### परफॉर्मेंस टिप्स
1. **पाइपलाइनिंग का उपयोग करें**: राउंड-ट्रिप लेटेंसी कम करने के लिए कमांड्स बैच करें।
2. **डेटा स्ट्रक्चर ऑप्टिमाइज़ करें**: सही स्ट्रक्चर चुनें (जैसे, कई कीज़ के बजाय छोटी ऑब्जेक्ट्स के लिए हैशेस का उपयोग करें)।
3. **मेमोरी मॉनिटर करें**: उपयोग ट्रैक करने के लिए `INFO MEMORY` का उपयोग करें।
4. **पर्सिस्टेंस ट्यून करें**: स्पीड (RDB) और ड्यूरेबिलिटी (AOF) के बीच संतुलन बनाएँ।

---

### सुरक्षा
- **localhost से बाइंड करें**: एक्सेस प्रतिबंधित करने के लिए `redis.conf` में `bind 127.0.0.1` सेट करें।
- **पासवर्ड सेट करें**: `requirepass yourpassword` का उपयोग करें।
- **खतरनाक कमांड्स डिसेबल करें**: `rename-command` का उपयोग करें (जैसे, `rename-command FLUSHALL ""`)।

---

### सीमाएँ
- **मेमोरी-बाउंड**: डेटा का आकार उपलब्ध RAM द्वारा सीमित होता है।
- **सिंगल-थ्रेडेड**: कोर ऑपरेशन्स एक थ्रेड पर चलते हैं (हालाँकि नए वर्जन में I/O और पर्सिस्टेंस मल्टीपल थ्रेड्स का उपयोग कर सकते हैं)।
- **कोई कॉम्प्लेक्स क्वेरीज़ नहीं**: SQL-जैसी क्वेरीइंग का अभाव; की-आधारित एक्सेस पर निर्भर करता है।

---

### Redis बनाम अन्य डेटाबेस
- **Redis बनाम Memcached**: Redis अमीर डेटा स्ट्रक्चर और पर्सिस्टेंस प्रदान करता है; Memcached सरल और शुद्ध रूप से इन-मेमोरी है।
- **Redis बनाम MongoDB**: Redis कैशिंग के लिए तेज़ है लेकिन जटिल, स्थायी डॉक्यूमेंट स्टोरेज के लिए कम उपयुक्त है।
- **Redis बनाम PostgreSQL**: Redis रियल-टाइम, कम-विलंबता वाले कार्यों में उत्कृष्ट है; PostgreSQL रिलेशनल डेटा और ACID कंप्लायंस के लिए बेहतर है।
