### Reveal-In-GitHub क्या है?

Reveal-In-GitHub एक Xcode प्लगइन है जो आपको आपके Xcode वर्कस्पेस से आपके वर्तमान रिपॉजिटरी के लिए GitHub की मुख्य सुविधाओं पर त्वरित रूप से नेविगेट करने देता है। यह GitHub पर होस्ट किए गए Git प्रोजेक्ट्स पर काम करने वाले डेवलपर्स के लिए बनाया गया है, जो एक-क्लिक (या कीबोर्ड शॉर्टकट) के माध्यम से कमिट इतिहास, ब्लेम व्यू, पुल रिक्वेस्ट्स, इशूज़ और नोटिफिकेशन्स जैसी चीज़ों तक पहुंच प्रदान करता है—बिना Xcode छोड़े या URL मैन्युअल रूप से टाइप किए।

### आवश्यक शर्तें
- Xcode इंस्टॉल होना चाहिए (हाल के संस्करणों पर परीक्षण किया गया; कोई विशिष्ट न्यूनतम संस्करण उल्लेखित नहीं)।
- आपका प्रोजेक्ट GitHub पर होस्ट किया गया एक Git रिपॉजिटरी होना चाहिए (प्लगइन रिपो URL और फ़ाइल पथ स्वचालित रूप से पहचान लेता है)।
- यदि आपके प्रोजेक्ट में एकाधिक Git रिमोट हैं, तो पहले उपयोग पर आपसे एक डिफ़ॉल्ट चुनने के लिए कहा जाएगा।

### इंस्टालेशन
इसे इंस्टॉल करने के दो मुख्य तरीके हैं:

#### विकल्प 1: Alcatraz का उपयोग करके (अनुशंसित)
1. यदि आपने पहले से नहीं किया है तो Alcatraz इंस्टॉल करें (Xcode प्लगइन्स के लिए एक पैकेज मैनेजर)। आप ऑनलाइन सेटअप गाइड पा सकते हैं, जैसे [यह एक](http://blog.devtang.com/blog/2014/03/05/use-alcatraz-to-manage-xcode-plugins/) यदि आप हिंदी निर्देश पसंद करते हैं।
2. Xcode में Alcatraz खोलें (मेनू के माध्यम से: `Window > Package Manager`)।
3. "Reveal In GitHub" खोजें।
4. **Install** पर क्लिक करें।
5. Xcode को पुनरारंभ करें।

#### विकल्प 2: मैनुअल इंस्टालेशन
1. रिपॉजिटरी को क्लोन करें:  
   ```
   git clone https://github.com/lzwjava/Reveal-In-GitHub.git
   ```
2. Xcode में `Reveal-In-GitHub.xcodeproj` फ़ाइल खोलें।
3. प्रोजेक्ट बिल्ड करें (Product > Build या ⌘B)। यह `Reveal-In-GitHub.xcplugin` फ़ाइल उत्पन्न करता है।
4. प्लगइन को इस स्थान पर ले जाएँ:  
   `~/Library/Application Support/Developer/Shared/Xcode/Plug-ins/`
5. Xcode को पुनरारंभ करें।

इंस्टालेशन के बाद, प्लगइन Xcode के मेनू बार में **Editor > Reveal In GitHub** के अंतर्गत दिखाई देना चाहिए।

### इसका उपयोग कैसे करें
एक बार इंस्टॉल हो जाने और Xcode पुनरारंभ हो जाने के बाद:
1. Xcode में एक GitHub-होस्टेड प्रोजेक्ट खोलें और एक सोर्स फ़ाइल संपादित करें (उदाहरण के लिए, किसी विशिष्ट लाइन पर नेविगेट करें)।
2. GitHub पर जाने के लिए कीबोर्ड शॉर्टकट या मेनू आइटम्स में से किसी एक का उपयोग करें जो **Editor > Reveal In GitHub** के अंतर्गत हैं। प्लगइन आपके रिपो, वर्तमान फ़ाइल, लाइन नंबर और नवीनतम कमिट हैश को स्वचालित रूप से पहचान लेता है।

यहाँ अंतर्निहित मेनू आइटम्स और शॉर्टकट्स के लिए एक त्वरित संदर्भ दिया गया है (शॉर्टकट ⌃⇧⌘ + शीर्षक के पहले अक्षर के पैटर्न का पालन करते हैं):

| मेनू आइटम      | शॉर्टकट    | यह क्या करता है | उदाहरण GitHub URL (रिपो lzwjava/LZAlbum में कमिट fd7224 पर फ़ाइल LZAlbumManager.m की लाइन 40 के लिए) |
|----------------|-------------|--------------|-----------------------------------------------------------------------------------------------|
| **Setting**    | ⌃⇧⌘S      | अनुकूलन पैनल खोलता है | N/A |
| **Repo**       | ⌃⇧⌘R      | मुख्य रिपो पेज खोलता है | https://github.com/lzwjava/LZAlbum |
| **Issues**     | ⌃⇧⌘I      | इशूज़ सूची खोलता है | https://github.com/lzwjava/LZAlbum/issues |
| **PRs**        | ⌃⇧⌘P      | पुल रिक्वेस्ट्स सूची खोलता है | https://github.com/lzwjava/LZAlbum/pulls |
| **Quick File** | ⌃⇧⌘Q      | वर्तमान लाइन पर फ़ाइल व्यू खोलता है | https://github.com/lzwjava/LZAlbum/blob/fd7224/LZAlbum/manager/LZAlbumManager.m#L40 |
| **List History**| ⌃⇧⌘L     | फ़ाइल के लिए कमिट इतिहास खोलता है | https://github.com/lzwjava/LZAlbum/commits/fd7224/LZAlbum/manager/LZAlbumManager.m |
| **Blame**      | ⌃⇧⌘B      | वर्तमान लाइन के लिए ब्लेम व्यू खोलता है | https://github.com/lzwjava/LZAlbum/blame/fd7224/LZAlbum/manager/LZAlbumManager.m#L40 |
| **Notifications**| ⌃⇧⌘N   | रिपो नोटिफिकेशन्स खोलता है | https://github.com/lzwjava/LZAlbum/notifications?all=1 |

- **टिप्स**: 
  - शॉर्टकट Xcode के डिफ़ॉल्ट से टकराव नहीं करते।
  - यदि आप एक चयनित टेक्स्ट रेंज पर हैं, तो कुछ एक्शन (जैसे Blame) उस रेंज को एंकर करेंगे (उदाहरण के लिए, #L40-L43)।
  - इसका परीक्षण करें: एक प्रोजेक्ट खोलें, एक लाइन पर जाएं, और ⌃⇧⌘B दबाएं—यह आपके ब्राउज़र में ब्लेम पेज खोलना चाहिए।

### अनुकूलन
मेनू आइटम्स को ट्वीक करने या जोड़ने के लिए:
1. ⌃⇧⌘S दबाएं या **Editor > Reveal In GitHub > Setting** पर जाएं।
2. पैनल में:
   - "+" बटन पर क्लिक करके नए आइटम जोड़ें: एक शीर्षक (जैसे, "Wiki"), शॉर्टकट और URL पैटर्न दर्ज करें।
   - URL पैटर्न प्लेसहोल्डर्स का उपयोग करते हैं जैसे:
     - `{git_remote_url}`: पूर्ण रिपो URL (जैसे, https://github.com/user/repo)।
     - `{commit}`: वर्तमान ब्रांच का नवीनतम कमिट हैश।
     - `{file_path}`: फ़ाइल का रिलेटिव पथ।
     - `#{selection}`: वर्तमान लाइन या चयन (जैसे, #L40)।
   - उदाहरण: एक विकि लिंक के लिए, पैटर्न को `{git_remote_url}/wiki` पर सेट करें।
3. यदि आपके पास एकाधिक रिमोट हैं, तो चयन रीसेट करने के लिए "Clear Default Repos" बटन का उपयोग करें।
