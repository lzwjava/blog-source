## 1. Java भाषा का अवलोकन

### 1.1 Java की विशेषताएँ

* **प्लेटफ़ॉर्म स्वतंत्रता:** Java कोड *बाइटकोड* में संकलित होता है, जो *Java वर्चुअल मशीन (JVM)* वाले किसी भी डिवाइस पर चलता है — "एक बार लिखो, कहीं भी चलाओ।"
* **ऑब्जेक्ट-ओरिएंटेड:** *एनकैप्सुलेशन, इनहेरिटेंस,* और *पॉलीमॉर्फिज्म* का समर्थन करता है।
* **सुरक्षित और मजबूत:** स्वचालित मेमोरी प्रबंधन (गार्बेज कलेक्शन) और मजबूत टाइप जाँच त्रुटियों को कम करती है।
* **मल्टीथ्रेडिंग:** एक साथ कई कार्यों के निष्पादन का समर्थन करता है।
* **समृद्ध स्टैंडर्ड लाइब्रेरी (API):** इसमें गणित, स्ट्रिंग्स, फाइलों, नेटवर्किंग आदि के लिए तैयार कक्षाएं शामिल हैं।

### 1.2 Java वर्जन और घटक

* **JDK (Java डेवलपमेंट किट):** डेवलपर्स के लिए — इसमें कंपाइलर (`javac`), JVM, और डेवलपमेंट टूल्स शामिल हैं।
* **JRE (Java रनटाइम एनवायरनमेंट):** एंड-यूजर्स के लिए — इसमें JVM + कोर लाइब्रेरीज़ शामिल हैं।
* **API (एप्लिकेशन प्रोग्रामिंग इंटरफेस):** Java की अंतर्निहित क्लास लाइब्रेरीज़, जैसे `java.lang`, `java.util`, `java.io`, आदि।

## 2. Java डेवलपमेंट टूल्स (IDE और CLI)

### 2.1 सामान्य IDE

परीक्षा के लिए, आपको केवल उनके उद्देश्य को जानने की आवश्यकता है:

* **Eclipse, IntelliJ IDEA, NetBeans:** Java कोड आसानी से लिखने, संकलित करने और चलाने के लिए उपयोग किए जाते हैं।

### 2.2 कमांड-लाइन वर्कफ़्लो

विशिष्ट संकलन और निष्पादन चरण:

1. **लिखें** अपना कोड एक `.java` फ़ाइल में, उदा. `Hello.java`
2. **इसे संकलित करें:**

   ```bash
   javac Hello.java
   ```

   → `Hello.class` (बाइटकोड फ़ाइल) उत्पन्न करता है
3. **इसे चलाएँ:**

   ```bash
   java Hello
   ```

   (चलाते समय `.class` एक्सटेंशन नहीं लगाते)

### 2.3 सरल उदाहरण

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, Java!");
    }
}
```

## 3. प्रोग्रामिंग स्टाइल दिशानिर्देश

### 3.1 नामकरण सम्मेलन

* **कक्षाएँ:** `CamelCase`, पहला अक्षर बड़ा → `StudentInfo`
* **चर और विधियाँ:** `camelCase`, छोटे अक्षर से शुरू → `studentName`, `calculateScore()`
* **स्थिरांक:** सभी बड़े अक्षरों में अंडरस्कोर के साथ → `MAX_SIZE`

### 3.2 इंडेंटेशन और टिप्पणियाँ

* **सुसंगत इंडेंटेशन** (आमतौर पर 4 स्पेस) का उपयोग करें।
* स्पष्ट **टिप्पणियाँ** लिखें:

  ```java
  // यह एकल-पंक्ति टिप्पणी है
  /* यह बहु-पंक्ति टिप्पणी है */
  ```

### 3.3 कोड संरचना

तार्किक समूहीकरण और पठनीयता का पालन करें:

* प्रत्येक फ़ाइल में **एक सार्वजनिक कक्षा** होती है जिसका **नाम फ़ाइल के समान** होता है।
* पंक्तियों को छोटा और सार्थक रखें; प्रति पंक्ति एक स्टेटमेंट।

## 4. सामान्य प्रोग्रामिंग त्रुटियाँ और डीबगिंग की मूल बातें

### 4.1 सिंटैक्स त्रुटियाँ

कंपाइलर द्वारा पकड़ी जाती हैं:

* सेमीकोलन `;` का छूट जाना
* ब्रेस `{ }` का मेल न खाना
* गलत कैपिटलाइज़ेशन (Java **केस-सेंसिटिव** है)

  ```java
  System.out.println("Hi"); // सही
  system.out.println("Hi"); // त्रुटि
  ```

### 4.2 रनटाइम त्रुटियाँ

प्रोग्राम चलाते समय होती हैं:

* शून्य से विभाजन
* सरणी सूचकांक सीमा से बाहर
* नल पॉइंटर एक्सेप्शन

### 4.3 लॉजिक त्रुटियाँ

प्रोग्राम चलता है लेकिन गलत आउटपुट देता है।
→ चर के मानों को चरण दर चरण ट्रेस करने के लिए **प्रिंट स्टेटमेंट** या IDE में **डीबगर टूल्स** का उपयोग करें।

## 5. परीक्षा के लिए मुख्य बिंदु

* **JDK, JRE, JVM** की भूमिकाओं और अंतरों को समझें।
* Java को IDE और कमांड लाइन दोनों से **कंपाइल और रन** करना जानें।
* **बेसिक सिंटैक्स नियम** और **स्टाइल मानकों** को याद रखें।
* **सामान्य त्रुटियों की पहचान** करने और उनके कारणों की व्याख्या करने में सक्षम हों।
* छोटे प्रोग्राम लिखने का अभ्यास करें जो:

  * संदेश प्रिंट करते हों
  * चर और एक्सप्रेशन का उपयोग करते हों
  * सही संरचना और नामकरण का पालन करते हों
