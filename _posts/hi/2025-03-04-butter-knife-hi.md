बटर नाइफ एंड्रॉइड के लिए एक लोकप्रिय व्यू-बाइंडिंग लाइब्रेरी है जो आपके कोड में फील्ड्स से UI एलिमेंट्स को बांधने की प्रक्रिया को सरल बनाती है। हालाँकि, कृपया ध्यान दें कि बटर नाइफ अब डिप्रिकेटेड हो चुकी है, और आधुनिक एंड्रॉइड डेवलपमेंट अब View Binding (जो एंड्रॉइड जेटपैक में पेश किया गया) के उपयोग की सिफारिश करता है। फिर भी, यदि आपको एंड्रॉइड स्टूडियो में 03 मार्च 2025 तक, एक एंड्रॉइड प्रोजेक्ट में `.jar` फाइल के साथ बटर नाइफ 6.1.0 (2015 के आसपास जारी एक पुराना वर्जन) का उपयोग करने की आवश्यकता है, तो यहां बताया गया है कि आप इसे कैसे कर सकते हैं।

### पूर्वापेक्षाएँ
- एंड्रॉइड स्टूडियो इंस्टॉल होना (कोई भी हालिया वर्जन काम करना चाहिए, हालांकि प्रक्रिया आपके वर्जन के आधार पर थोड़ी भिन्न हो सकती है)।
- एक बुनियादी एंड्रॉइड प्रोजेक्ट सेट अप होना।
- `butterknife-6.1.0.jar` फाइल डाउनलोड होना। आप आमतौर पर Maven Central जैसे रिपॉजिटरी पर या आर्काइव्ड स्रोतों के माध्यम से पुराने वर्जन पा सकते हैं यदि आपके पास `.jar` फाइल लोकल में है।

### एंड्रॉइड डेवलपमेंट में `butterknife-6.1.0.jar` का उपयोग करने के चरण

#### चरण 1: अपने प्रोजेक्ट में `.jar` फाइल जोड़ें
1. **`libs` फोल्डर ढूंढें**:
   - अपने एंड्रॉइड स्टूडियो प्रोजेक्ट में, `app` मॉड्यूल पर नेविगेट करें।
   - `app` फोल्डर के अंदर, `libs` नामक एक फोल्डर ढूंढें या बनाएं। यदि यह मौजूद नहीं है, तो `app` फोल्डर पर राइट-क्लिक करें, `New > Directory` चुनें, और इसका नाम `libs` रखें।

2. **`.jar` फाइल कॉपी करें**:
   - `butterknife-6.1.0.jar` फाइल को `libs` फोल्डर में कॉपी करें। आप एंड्रॉइड स्टूडियो में `libs` फोल्डर में फाइल को ड्रैग और ड्रॉप करके या अपने फाइल एक्सप्लोरर के माध्यम से वहां मैन्युअल रूप से रखकर ऐसा कर सकते हैं।

3. **Gradle के साथ `.jar` फाइल सिंक करें**:
   - `app` मॉड्यूल के लिए `build.gradle` फाइल खोलें (जो `app/build.gradle` पर स्थित है)।
   - `libs` फोल्डर में सभी `.jar` फाइलों को शामिल करने के लिए `dependencies` ब्लॉक के अंतर्गत निम्न पंक्ति जोड़ें:
     ```gradle
     dependencies {
         compile fileTree(dir: 'libs', include: ['*.jar'])
     }
     ```
   - एंड्रॉइड स्टूडियो में "Sync Project with Gradle Files" बटन पर क्लिक करके अपने प्रोजेक्ट को सिंक करें।

#### चरण 2: अपने प्रोजेक्ट को कॉन्फ़िगर करें
चूंकि बटर नाइफ 6.1.0 एनोटेशन प्रोसेसिंग का उपयोग करता है, इसलिए आपको इस विशिष्ट वर्जन के लिए (8.x और उसके बाद के वर्जन के विपरीत) एनोटेशन प्रोसेसर डिपेंडेंसी की आवश्यकता नहीं है। `.jar` फाइल में रनटाइम लाइब्रेरी शामिल है, और बटर नाइफ 6.1.0 अपनी अधिकांश कार्यक्षमता के लिए कंपाइल-टाइम कोड जनरेशन के बजाय रनटाइम रिफ्लेक्शन पर निर्भर करता है।

हालाँकि, सुनिश्चित करें कि आपका प्रोजेक्ट Java एनोटेशन का समर्थन करने के लिए सेट अप है:
- अपने `app/build.gradle` में, सुनिश्चित करें कि Java वर्जन संगत है (बटर नाइफ 6.1.0 Java 6+ के साथ काम करता है):
  ```gradle
  android {
      compileOptions {
          sourceCompatibility JavaVersion.VERSION_1_6
          targetCompatibility JavaVersion.VERSION_1_6
      }
  }
  ```

#### चरण 3: अपने कोड में बटर नाइफ का उपयोग करें
1. **बटर नाइफ एनोटेशन जोड़ें**:
   - अपनी Activity या Fragment में, बटर नाइफ इम्पोर्ट करें और `@InjectView` (वर्जन 6.1.0 में उपयोग किया जाने वाला एनोटेशन) के साथ अपने व्यूज़ को एनोटेट करें। उदाहरण के लिए:
     ```java
     import android.os.Bundle;
     import android.widget.Button;
     import android.widget.TextView;
     import butterknife.InjectView;
     import butterknife.ButterKnife;
     import androidx.appcompat.app.AppCompatActivity;

     public class MainActivity extends AppCompatActivity {

         @InjectView(R.id.my_button)
         Button myButton;

         @InjectView(R.id.my_text)
         TextView myText;

         @Override
         protected void onCreate(Bundle savedInstanceState) {
             super.onCreate(savedInstanceState);
             setContentView(R.layout.activity_main);
             ButterKnife.inject(this); // व्यूज़ बांधें

             // उपयोग का उदाहरण
             myButton.setOnClickListener(v -> myText.setText("Button clicked!"));
         }
     }
     ```

2. **XML लेआउट**:
   - सुनिश्चित करें कि आपकी लेआउट फाइल (जैसे, `res/layout/activity_main.xml`) में संबंधित ID वाले व्यूज़ मौजूद हैं:
     ```xml
     <LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
         android:layout_width="match_parent"
         android:layout_height="match_parent"
         android:orientation="vertical">

         <TextView
             android:id="@+id/my_text"
             android:layout_width="wrap_content"
             android:layout_height="wrap_content"
             android:text="Hello World" />

         <Button
             android:id="@+id/my_button"
             android:layout_width="wrap_content"
             android:layout_height="wrap_content"
             android:text="Click Me" />
     </LinearLayout>
     ```

3. **व्यूज़ बांधें**:
   - `onCreate` में `ButterKnife.inject(this)` कॉल लेआउट से व्यूज़ को एनोटेटेड फील्ड्स से बांधती है। ध्यान दें कि वर्जन 6.1.0 में, `bind` (जो बाद के वर्जन जैसे 7.x और 8.x में पेश किया गया) के बजाय `inject` का उपयोग किया जाता है।

#### चरण 4: अपना प्रोजेक्ट रन करें
- अपने प्रोजेक्ट को बिल्ड और रन करें। यदि सब कुछ सही ढंग से सेट अप है, तो बटर नाइफ आपके व्यूज़ को बांध देगा, और आपको UI को अपेक्षा के अनुरूप कार्य करते हुए देखना चाहिए।

### महत्वपूर्ण नोट्स
- **वर्जन सीमा**: बटर नाइफ 6.1.0 काफी पुराना है (2015 में जारी) और इसमें बाद के वर्जन में पेश की गई सुविधाओं का अभाव है, जैसे कि कंपाइल-टाइम कोड जनरेशन (एनोटेशन प्रोसेसर के माध्यम से) और `@InjectView` के बजाय `@BindView` का समर्थन। यह रनटाइम पर रिफ्लेक्शन का उपयोग करता है, जो आधुनिक विकल्पों की तुलना में प्रदर्शन को थोड़ा प्रभावित कर सकता है।
- **डिप्रिकेटेड स्टेटस**: बटर नाइफ का अब रखरखाव नहीं किया जाता है (विकास लगभग 2020 में बंद हो गया), और आधिकारिक सिफारिश है कि नए प्रोजेक्ट्स के लिए View Binding या Jetpack Compose का उपयोग किया जाए।
- **वैकल्पिक सेटअप**: यदि आप एक नए वर्जन (जैसे, 10.2.3) का उपयोग कर रहे थे, तो आपको एक एनोटेशन प्रोसेसर डिपेंडेंसी जोड़ने की आवश्यकता होगी (जैसे, `annotationProcessor 'com.jakewharton:butterknife-compiler:10.2.3'`), लेकिन 6.1.0 के लिए `.jar` फाइल के साथ, यह आवश्यक नहीं है।
- **समस्या निवारण**:
  - यदि आपको बटर नाइफ क्लासेज के लिए "cannot resolve symbol" एरर आती है, तो सुनिश्चित करें कि `.jar` फाइल `libs` फोल्डर में सही ढंग से जोड़ी गई है और Gradle डिपेंडेंसीज में शामिल है।
  - यदि ऐप `NoClassDefFoundError` के साथ क्रैश हो जाता है, तो डबल-चेक करें कि `.jar` फाइल `libs` फोल्डर में है और प्रोजेक्ट सिंक हो गया है।

### सिफारिश
हालाँकि यह गाइड आपको दिखाती है कि `butterknife-6.1.0.jar` का उपयोग कैसे करें, लेकिन एक अधिक आधुनिक, समर्थित दृष्टिकोण के लिए View Binding में माइग्रेट करने पर विचार करें। यहाँ एक त्वरित तुलना दी गई है:
- **बटर नाइफ 6.1.0**: `@InjectView` का उपयोग करता है, रनटाइम रिफ्लेक्शन, मैन्युअल `.jar` इंटीग्रेशन।
- **View Binding**: एंड्रॉइड में बिल्ट-इन, कोई बाहरी डिपेंडेंसी नहीं, कंपाइल टाइम पर बाइंडिंग क्लासेज जनरेट करता है, और null-सुरक्षित है।

क्या आप इसके बजाय View Binding में संक्रमण के लिए मार्गदर्शन चाहेंगे?
