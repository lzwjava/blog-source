जैक्सन (`com.fasterxml.jackson.databind` लाइब्रेरी) में, **डिफ़ॉल्ट टाइपिंग** `ObjectMapper` की एक सुविधा है जो स्वचालित रूप से सीरियलाइज़ किए गए JSON में टाइप मेटाडेटा (जैसे `@class` या `@type`) शामिल करती है, ताकि ऑब्जेक्ट्स को उनके मूल पॉलीमॉर्फिक प्रकारों में वापस डी-सीरियलाइज़ किया जा सके।

उदाहरण के लिए, यदि आप `Dog` और `Cat` दोनों वाली एक `List<Animal>` को सीरियलाइज़ करते हैं, तो डिफ़ॉल्ट टाइपिंग अतिरिक्त JSON फ़ील्ड जोड़ती है ताकि जैक्सन को बाद में पता चल सके कि किस सबक्लास को फिर से इंस्टेंटिएट करना है।

---

### इसे डिसेबल करना कैसे काम करता है

जब आप **डिफ़ॉल्ट टाइपिंग डिसेबल** करते हैं, तो जैक्सन उन अतिरिक्त टाइप मेटाडेटा फ़ील्ड्स को इंजेक्ट करना बंद कर देता है। इसका मतलब है:

* JSON आउटपुट "साफ़" होगा, बिना `@class`/`@type` संकेतों के।
* डी-सीरियलाइज़ेशन पर, जैक्सन केवल घोषित स्टैटिक टाइप्स (आपके द्वारा पास किए गए टार्गेट टाइप) का उपयोग करके ऑब्जेक्ट्स को पुनर्निर्मित कर सकता है, मूल रनटाइम टाइप्स का नहीं।
* कोई भी पॉलीमॉर्फिक उपयोग के मामले तब टूट जाएंगे जब तक कि आप स्पष्ट रूप से `@JsonTypeInfo`, `@JsonSubTypes`, या कस्टम मॉड्यूल कॉन्फ़िगर नहीं करते।

---

### प्रासंगिक मेथड्स

* **एनेबल करना** (2.10 से पहले की शैली):

  ```java
  objectMapper.enableDefaultTyping();
  ```

  (अप्रचलित, क्योंकि यह बहुत अनुमतिपूर्ण था और सुरक्षा जोखिम पैदा करता था।)

* **डिसेबल करना**:

  ```java
  objectMapper.disableDefaultTyping();
  ```

  या नए वर्जन (2.10+) में, आप कॉल करते हैं:

  ```java
  objectMapper.activateDefaultTyping(PolymorphicTypeValidator, ObjectMapper.DefaultTyping.NON_FINAL);
  ```

  इसे *डिसेबल* करने के लिए, आप बस इस मेथड को **कॉल नहीं करते** (जब तक आप इसे एनेबल नहीं करते तब तक कोई डिफ़ॉल्ट टाइपिंग नहीं है), या आप कॉल कर सकते हैं:

  ```java
  objectMapper.deactivateDefaultTyping();
  ```

---

### सुरक्षा और तर्क

* **डिफ़ॉल्ट टाइपिंग आधुनिक जैक्सन वर्जन में डिफ़ॉल्ट रूप से डिसेबल है** क्योंकि इसका दुरुपयोग किया जा सकता है (हमलावर ऐसा JSON बना सकते हैं जो खतरनाक क्लासेस को इंस्टेंटिएट करता है)।
* यदि आप इसे डिसेबल करते हैं, तो जैक्सन एक "सुरक्षित" मोड में व्यवहार करता है, जहां डी-सीरियलाइज़ेशन केवल उन प्रकारों के लिए काम करता है जिन्हें आप स्पष्ट रूप से बाइंड करते हैं, न कि मनमाने रनटाइम प्रकारों के लिए।
* यदि आपको पॉलीमॉर्फिक सपोर्ट की आवश्यकता है, तो आपको ग्लोबल डिफ़ॉल्ट टाइपिंग पर भरोसा करने के बजाय अपनी क्लासेस को `@JsonTypeInfo` के साथ एनोटेट करना चाहिए।

---

✅ **संक्षेप में:**
डिफ़ॉल्ट टाइपिंग को डिसेबल करना जैक्सन को बताता है कि JSON में टाइप मेटाडेटा शामिल **न करें**, जिसका अर्थ है कि यह डी-सीरियलाइज़ेशन पर पॉलीमॉर्फिक सबटाइप्स को ऑटो-रीकंस्ट्रक्ट नहीं करेगा। यह लचीलापन कम करता है लेकिन अधिक सुरक्षित है और JSON को साफ़ रखता है।

क्या आप चाहते हैं कि मैं आपको डिफ़ॉल्ट टाइपिंग एनेबल बनाम डिसेबल के साथ एक **पहले/बाद का JSON उदाहरण** दिखाऊं?
