WebSphere Liberty Profile (WLP) सर्वर के सर्वर डायरेक्टरी में स्थित `jvm.options` फ़ाइल में सेट किए गए JVM विकल्पों को जांचने के लिए, Java Development Kit (JDK) द्वारा प्रदान की गई `jinfo` कमांड का उपयोग किया जा सकता है। यह विधि एक चल रहे सर्वर पर लागू किए गए विकल्पों की पुष्टि करने का एक दक्ष तरीका प्रदान करती है। नीचे दिए गए कदमों का पालन करके यह काम पूरा किया जा सकता है:

### JVM विकल्पों की जांच करने के कदम

1. **संदर्भ समझें**
   सर्वर डायरेक्टरी में स्थित `jvm.options` फ़ाइल (आम तौर पर `<WLP_HOME>/usr/servers/<serverName>/jvm.options`) को उपयोग करके JVM तर्कों, जैसे कि हिप साइज (उदाहरण के लिए, `-Xmx`), गार्बेज कलेक्शन सेटिंग्स (उदाहरण के लिए, `-XX:+UseG1GC`), या सिस्टम गुण (उदाहरण के लिए, `-Dmy.property=value`) को निर्दिष्ट किया जाता है, जो Liberty सर्वर शुरू होने पर लागू होते हैं।

2. **सर्वर शुरू करें**
   निम्न कमांड का उपयोग करके अपने Liberty सर्वर को पृष्ठभूमि में शुरू करें:
   ```
   <WLP_HOME>/bin/server start <serverName>
   ```
   `<WLP_HOME>` को WebSphere Liberty इंस्टॉलेशन के पथ से बदलें और `<serverName>` को अपने सर्वर के नाम से बदलें। यह कमांड सर्वर को एक पृष्ठभूमि प्रक्रिया के रूप में लॉन्च करता है।

3. **प्रक्रिया आईडी (PID) को खोजें**
   सर्वर शुरू करने के बाद, चल रहे Java प्रक्रिया की प्रक्रिया आईडी (PID) की आवश्यकता होगी। Liberty यह कोन्वीनियंस के साथ एक `.pid` फ़ाइल में स्टोर करता है, जो स्थित है:
   ```
   <WLP_HOME>/usr/servers/<serverName>/workarea/<serverName>.pid
   ```
   इस फ़ाइल को खोलें (उदाहरण के लिए, Unix-आधारित सिस्टम पर `cat` का उपयोग करें या एक टेक्स्ट एडिटर) और सर्वर के प्रक्रिया को दर्शाने वाली संख्या प्राप्त करें।

4. **JVM फ्लैग्स की पुष्टि करें**
   `jinfo` कमांड का उपयोग करके चल रहे सर्वर पर लागू किए गए JVM फ्लैग्स की जांच करें। चलाएं:
   ```
   jinfo -flags <pid>
   ```
   `<pid>` को `.pid` फ़ाइल से प्राप्त प्रक्रिया आईडी से बदलें। यह कमांड JVM को पास किए गए कमांड-लाइन फ्लैग्स को प्रदर्शित करता है, जैसे कि `-Xmx1024m` या `-XX:+PrintGCDetails`। आउटपुट में देखें कि आपने `jvm.options` में सेट किए गए फ्लैग्स मौजूद हैं।

5. **सिस्टम गुणों की पुष्टि करें**
   अगर आपके `jvm.options` फ़ाइल में सिस्टम गुण शामिल हैं (उदाहरण के लिए, `-Dmy.property=value`), तो उन्हें अलग से जांचें:
   ```
   jinfo -sysprops <pid>
   ```
   यह सभी सिस्टम गुण प्रदर्शित करता है जो JVM के लिए सेट किए गए हैं। आउटपुट में अपने द्वारा परिभाषित गुणों को खोजें ताकि यह सुनिश्चित हो सके कि वे सही तरीके से लागू किए गए हैं।

### पूर्वापेक्षाएँ
- **JDK इंस्टॉल किया गया**: `jinfo` कमांड JDK का हिस्सा है, नहीं JRE का। सुनिश्चित करें कि एक JDK इंस्टॉल किया गया है और `jinfo` एक्सिक्यूटेबल आपके सिस्टम के PATH में है।
- **अनुमतियाँ**: `jinfo` को उसी उपयोगकर्ता के साथ चलाएं जिसने सर्वर शुरू किया या प्रक्रिया से जोड़ने के लिए पर्याप्त अधिकारों के साथ चलाएं।

### विकल्प विधियाँ
अगर `jinfo` उपलब्ध नहीं है या आप किसी और तरीके का पसंद करते हैं, तो यहाँ दो अतिरिक्त विकल्प हैं:

- **`server dump` का उपयोग करें**:
  1. सर्वर को `server start <serverName>` के साथ शुरू करें।
  2. एक सर्वर डंप जनरेट करें:
     ```
     <WLP_HOME>/bin/server dump <serverName>
     ```
  3. `<WLP_HOME>/usr/servers/<serverName>/` में जनरेट किए गए `.zip` फ़ाइल को खोजें, इसे एक्सट्रैक्ट करें और `javacore.<timestamp>.txt` फ़ाइल को खोलें।
  4. "CMDLINE" खंड को खोजें ताकि पूर्ण Java कमांड लाइन, जिसमें JVM फ्लैग्स शामिल हैं, और "System Properties" खंड में सिस्टम गुणों को देखें।

- **स्टार्टअप स्क्रिप्ट ट्रेसिंग** (Unix-आधारित सिस्टम):
  1. ट्रेसिंग के साथ सर्वर स्क्रिप्ट चलाएं:
     ```
     sh -x <WLP_HOME>/bin/server start <serverName>
     ```
  2. आउटपुट को समीक्षित करें ताकि स्क्रिप्ट द्वारा चलाए गए Java कमांड को खोजें, जिसमें `jvm.options` से सभी विकल्प शामिल हैं।

### टिप्पणियाँ
- **सर्वर नहीं चल रहा**: अगर सर्वर गलत विकल्पों के कारण शुरू नहीं होता, तो `server run <serverName>` का उपयोग करके इसे फॉरग्राउंड में चलाएं और कंसोल आउटपुट में त्रुटियों की जांच करें, या स्क्रिप्ट ट्रेसिंग विधि का उपयोग करें ताकि कमांड देखें।
- **लॉग फ़ाइलें**: `<WLP_HOME>/usr/servers/<serverName>/logs/` में `console.log` फ़ाइल में कुछ स्टार्टअप जानकारी हो सकती है, लेकिन यह सभी JVM विकल्पों को सूचीबद्ध नहीं करती जब तक कि वर्बोज़ लॉगिंग सक्षम नहीं है (उदाहरण के लिए, `-XX:+PrintFlagsFinal`).

### निष्कर्ष
`jinfo` विधि `jvm.options` में सेट किए गए JVM विकल्पों की पुष्टि करने का सबसे सरल तरीका है। ऊपर दिए गए कदमों का पालन करके, आप यह सुनिश्चित कर सकते हैं कि JVM फ्लैग्स और सिस्टम गुणों को जैसा कि इरादा था, लागू किए गए हैं।
