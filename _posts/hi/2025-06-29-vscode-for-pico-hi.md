### Raspberry Pi Pico पर MicroPython के लिए VSCode का उपयोग करने हेतु चरण-दर-चरण मार्गदर्शक

1.  **आवश्यक VSCode एक्सटेंशन इंस्टॉल करें**:
    - VSCode खोलें।
    - एक्सटेंशन व्यू (`Ctrl+Shift+X` या macOS पर `Cmd+Shift+X`) पर जाएं।
    - निम्नलिखित एक्सटेंशन इंस्टॉल करें:
      - **Python** (Microsoft द्वारा): Python और MicroPython सिंटैक्स हाइलाइटिंग और IntelliSense के लिए।
      - **Pico-W-Go** (वैकल्पिक लेकिन अनुशंसित): Raspberry Pi Pico डेवलपमेंट के लिए एक समर्पित एक्सटेंशन। "Pico-W-Go" खोजें और इसे इंस्टॉल करें।
        - नोट: Pico-W-Go फ़ाइल ट्रांसफर और REPL एक्सेस को सरल बनाता है लेकिन इसके लिए अतिरिक्त सेटअप की आवश्यकता होती है (नीचे वर्णित)।
      - वैकल्पिक रूप से, आप मैन्युअल नियंत्रण के लिए **Remote-SSH** या **Serial Monitor** जैसे सामान्य-उद्देश्य वाले एक्सटेंशन का उपयोग कर सकते हैं।

2.  **Pico-W-Go सेट अप करें (अनुशंसित)**:
    - **डिपेंडेंसीज इंस्टॉल करें**: Pico-W-Go के लिए `pyserial` और `esptool` की आवश्यकता होती है। उन्हें pip के माध्यम से इंस्टॉल करें:
      ```bash
      pip3 install pyserial esptool
      ```
    - **Pico-W-Go कॉन्फ़िगर करें**:
      - VSCode का कमांड पैलेट (`Ctrl+Shift+P` या macOS पर `Cmd+Shift+P`) खोलें।
      - टाइप करें और **Pico-W-Go > Configure Project** चुनें।
      - अपना प्रोजेक्ट सेट अप करने के लिए संकेतों का पालन करें:
        - Pico के सीरियल पोर्ट (जैसे, `/dev/ttyACM0`) को चुनें। इसे खोजने के लिए टर्मिनल में `ls /dev/tty*` चलाएं।
        - इंटरप्रेटर के रूप में MicroPython चुनें।
        - एक नया प्रोजेक्ट फ़ोल्डर बनाएं या मौजूदा फ़ोल्डर का उपयोग करें।
      - Pico-W-Go एक वर्कस्पेस बनाता है जिसमें एक `.picowgo` कॉन्फ़िगरेशन फ़ाइल होती है।

3.  **एक साधारण MicroPython प्रोग्राम लिखें**:
    - VSCode में, अपने प्रोजेक्ट फ़ोल्डर में एक नई फ़ाइल बनाएं (जैसे, `main.py`)।
    - एक साधारण प्रोग्राम लिखें, जैसे ऑनबोर्ड LED को ब्लिंक करना:
      ```python
      from machine import Pin
      import time

      led = Pin(25, Pin.OUT)  # Pico W के लिए "LED" का उपयोग करें
      while True:
          led.on()
          time.sleep(0.5)
          led.off()
          time.sleep(0.5)
      ```
    - फ़ाइल को सेव करें (`Ctrl+S` या `Cmd+S`)।

4.  **प्रोग्राम को Pico पर अपलोड करें**:
    - **Pico-W-Go का उपयोग करके**:
      - सुनिश्चित करें कि Pico कनेक्टेड है और सही पोर्ट चुना गया है (यदि आवश्यक हो तो `Pico-W-Go > Configure Project` में जांचें)।
      - कमांड पैलेट (`Ctrl+Shift+P`) खोलें।
      - **Pico-W-Go > Upload Project to Pico** चुनें।
      - यह आपके प्रोजेक्ट फ़ोल्डर की सभी फ़ाइलों (जैसे, `main.py`) को Pico की फ़ाइलसिस्टम पर अपलोड कर देता है।
      - यदि आपने फ़ाइल का नाम `main.py` रखा है, तो यह बूट पर स्वचालित रूप से चल जाएगी।
    - **`rshell` के साथ मैन्युअल अपलोड** (यदि Pico-W-Go का उपयोग नहीं कर रहे हैं):
      - `rshell` इंस्टॉल करें:
        ```bash
        pip3 install rshell
        ```
      - Pico से कनेक्ट करें:
        ```bash
        rshell --port /dev/ttyACM0
        ```
      - फ़ाइल को Pico पर कॉपी करें:
        ```bash
        cp main.py /pyboard/main.py
        ```
      - `rshell` से `exit` के साथ बाहर निकलें।

5.  **प्रोग्राम चलाएं और टेस्ट करें**:
    - **Pico-W-Go का उपयोग करके**:
      - कमांड पैलेट खोलें और **Pico-W-Go > Run** चुनें।
      - यह वर्तमान फ़ाइल को निष्पादित करता है या मैन्युअल कमांड्स के लिए REPL खोलता है।
      - यदि आप उपरोक्त उदाहरण का उपयोग कर रहे हैं तो आपको LED ब्लिंक होती हुई दिखाई देनी चाहिए।
    - **VSCode के टर्मिनल या REPL का उपयोग करके**:
      - Pico-W-Go के साथ REPL खोलें (`Pico-W-Go > Open REPL`) या `rshell` का उपयोग करें:
        ```bash
        rshell --port /dev/ttyACM0 repl
        ```
      - सीधे कमांड्स टेस्ट करें, उदाहरण के लिए:
        ```python
        from machine import Pin
        led = Pin(25, Pin.OUT)
        led.on()
        ```
      - REPL में चल रहे प्रोग्राम को रोकने के लिए `Ctrl+C` दबाएं।
    - यदि फ़ाइल `main.py` है, तो इसे स्वचालित रूप से चलाने के लिए Pico को रीसेट करें (अनप्लग और रीप्लग करें, या RESET बटन दबाएं)।

6.  **डीबग करें और फ़ाइलों का प्रबंधन करें**:
    - **Pico-W-Go**: Pico से फ़ाइलें पुनः प्राप्त करने के लिए **Pico-W-Go > Download Project from Pico** का उपयोग करें या फ़ाइलसिस्टम को साफ़ करने के लिए **Pico-W-Go > Delete All Files** का उपयोग करें।
    - **rshell**: फ़ाइलों को सूचीबद्ध करें:
      ```bash
      rshell ls /pyboard
      ```
      फ़ाइलों को हटाएं:
      ```bash
      rshell rm /pyboard/main.py
      ```
    - VSCode टर्मिनल या REPL में प्रोग्राम आउटपुट जांचें।

### वैकल्पिक: Pico-W-Go के बिना मैन्युअल वर्कफ़्लो
यदि आप Pico-W-Go का उपयोग नहीं करना चाहते हैं, तो आप MicroPython डेवलपमेंट को मैन्युअली प्रबंधित कर सकते हैं:
1. अपना कोड VSCode में लिखें और इसे `main.py` के रूप में सेव करें।
2. अपलोड करने के लिए `ampy` (एक अन्य MicroPython टूल) का उपयोग करें:
   ```bash
   pip3 install adafruit-ampy
   ampy --port /dev/ttyACM0 put main.py
   ```
3. `minicom` या `screen` जैसे टर्मिनल टूल के साथ REPL तक पहुंचें:
   ```bash
   screen /dev/ttyACM0 115200
   ```
   चल रहे प्रोग्राम को रोकने और REPL में प्रवेश करने के लिए `Ctrl+C` दबाएं।

### समस्या निवारण
-   **पोर्ट नहीं मिला**: Pico के पोर्ट (जैसे, `/dev/ttyACM0`) को सत्यापित करने के लिए `ls /dev/tty*` चलाएं। सुनिश्चित करें कि Pico कनेक्टेड है और MicroPython फ्लैश किया गया है।
-   **Pico-W-Go एरर्स**: जांचें कि `pyserial` और `esptool` इंस्टॉल हैं और सही पोर्ट चुना गया है। यदि आवश्यक हो तो VSCode को रीस्टार्ट करें।
-   **प्रोग्राम नहीं चल रहा**: सुनिश्चित करें कि फ़ाइल ऑटो-एक्जिक्यूशन के लिए `main.py` के रूप में सेव है, या इसे मैन्युअल रूप से REPL या `ampy run main.py` के माध्यम से चलाएं।
-   **LED ब्लिंक नहीं हो रही**: GPIO पिन (`25` Pico के लिए, `"LED"` Pico W के लिए) सत्यापित करें। REPL में टेस्ट करें:
    ```python
    from machine import Pin
    led = Pin(25, Pin.OUT)
    led.on()
    ```
