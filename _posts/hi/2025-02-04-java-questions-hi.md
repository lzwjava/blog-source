### Java Core

1. Java में OOP के चार मुख्य सिद्धांत क्या हैं? उत्तर: चार मुख्य सिद्धांत हैं एनकैप्सुलेशन, इनहेरिटेंस, पॉलीमॉर्फिज्म और एब्स्ट्रक्शन। एनकैप्सुलेशन ऑब्जेक्ट की आंतरिक स्थिति को छुपाता है, इनहेरिटेंस क्लास इनहेरिटेंस की अनुमति देता है, पॉलीमॉर्फिज्म मेथड ओवरराइडिंग और ओवरलोडिंग को सक्षम बनाता है, और एब्स्ट्रक्शन बैकग्राउंड विवरण शामिल किए बिना आवश्यक विशेषताओं को प्रस्तुत करने का एक तरीका प्रदान करता है।

2. Java में जेनरिक्स का उद्देश्य समझाएं और एक उदाहरण दें। उत्तर: जेनरिक्स प्रकारों को पैरामीटराइज़ करने की अनुमति देते हैं, जिससे कोड पुन: प्रयोज्यता और टाइप सेफ्टी सक्षम होती है। उदाहरण के लिए, `ArrayList<T>` किसी भी प्रकार के एलिमेंट्स को स्टोर करने के लिए एक टाइप पैरामीटर `T` का उपयोग करता है।

3. Java में आप एक थ्रेड कैसे बनाते हैं, और इसका लाइफसाइकिल क्या है? उत्तर: आप `Thread` को एक्सटेंड करके या `Runnable` को इम्प्लीमेंट करके एक थ्रेड बना सकते हैं। लाइफसाइकिल में न्यू, रननेबल, रनिंग, ब्लॉक्ड, वेटिंग, टाइम्ड वेटिंग और टर्मिनेटेड स्टेट्स शामिल हैं।

4. JVM द्वारा प्रबंधित विभिन्न मेमोरी एरिया का वर्णन करें। उत्तर: JVM हीप, स्टैक, मेथड एरिया, नेटिव मेथड स्टैक और प्रोग्राम काउंटर रजिस्टर को प्रबंधित करता है। हीप ऑब्जेक्ट्स को स्टोर करता है, जबकि प्रत्येक थ्रेड के पास लोकल वेरिएबल्स और मेथड कॉल्स के लिए अपना स्वयं का स्टैक होता है।

5. Java में checked और unchecked एक्सेप्शन में क्या अंतर है? उत्तर: Checked एक्सेप्शन को डिक्लेयर या कॉट किया जाना चाहिए, जबकि unchecked एक्सेप्शन कंपाइल टाइम पर चेक नहीं किए जाते हैं। उदाहरणों में checked के लिए `IOException` और unchecked के लिए `NullPointerException` शामिल हैं।

6. Java में आप सीरियलाइजेशन को कैसे इम्प्लीमेंट करते हैं, और यह महत्वपूर्ण क्यों है? उत्तर: सीरियलाइजेशन `Serializable` इंटरफेस को इम्प्लीमेंट करके किया जाता है। यह किसी ऑब्जेक्ट की स्थिति को सेव और रिस्टोर करने के लिए महत्वपूर्ण है, जो नेटवर्किंग और पर्सिस्टेंस में उपयोगी है।

7. Java कलेक्शन्स फ्रेमवर्क में ArrayList और LinkedList की तुलना करें। उत्तर: `ArrayList` फास्ट एक्सेस और ट्रैवर्सल के लिए उपयुक्त है, जबकि `LinkedList` इंसर्शन और डिलीशन के लिए बेहतर है। `ArrayList` सन्निक्ष मेमोरी का उपयोग करता है, जबकि `LinkedList` पॉइंटर्स वाले नोड्स का उपयोग करता है।

8. Java में लैम्ब्डा एक्सप्रेशन क्या हैं, और वे फंक्शनल इंटरफेस से कैसे संबंधित हैं? उत्तर: लैम्ब्डा एक्सप्रेशन एक मेथड इंटरफेस (फंक्शनल इंटरफेस) को प्रस्तुत करने का एक संक्षिप्त तरीका प्रदान करते हैं। इनका उपयोग `Runnable` या `Comparator` जैसे फंक्शनल इंटरफेस को इम्प्लीमेंट करने के लिए किया जाता है।

9. Java Stream API में उपलब्ध मुख्य ऑपरेशन समझाएं। उत्तर: Stream API में इंटरमीडिएट ऑपरेशन (जैसे, `map`, `filter`) और टर्मिनल ऑपरेशन (जैसे, `forEach`, `collect`) शामिल हैं। ये कलेक्शन्स पर फंक्शनल-स्टाइल ऑपरेशन की अनुमति देते हैं।

10. Java में रिफ्लेक्शन का उपयोग करके रनटाइम पर क्लासेस का निरीक्षण कैसे करते हैं? उत्तर: रिफ्लेक्शन `Class.forName()`, `getMethods()`, और `getField()` का उपयोग करके क्लासेस, मेथड्स और फील्ड्स के निरीक्षण की अनुमति देता है। इसका उपयोग डायनामिक बिहेवियर और फ्रेमवर्क्स के लिए किया जाता है।

### Spring Ecosystem

1. Spring IoC कंटेनर क्या है, और यह कैसे काम करता है? उत्तर: IoC कंटेनर बीन्स और उनके लाइफसाइकिल को प्रबंधित करता है। यह कपलिंग को कम करते हुए डिपेंडेंसी इंजेक्शन का उपयोग करके डिपेंडेंसी को प्रबंधित करता है।

2. Spring Boot ऑटो-कॉन्फ़िगरेशन समझाएं। उत्तर: ऑटो-कॉन्फ़िगरेशन क्लासपाथ डिपेंडेंसी के आधार पर बीन्स को स्वचालित रूप से कॉन्फ़िगर करता है, जिससे सेटअप सरल होता है और बॉयलरप्लेट कोड कम होता है।

3. Spring Data JPA डेटा एक्सेस को कैसे सरल बनाता है? उत्तर: Spring Data JPA CRUD ऑपरेशन और क्वेरी मेथड्स के साथ रिपॉजिटरी प्रदान करता है, जो डेटाबेस इंटरैक्शन को एब्स्ट्रैक्ट करता है।

4. Spring Security का उपयोग किस लिए किया जाता है? उत्तर: Spring Security ऑथेंटिकेशन और ऑथराइजेशन मैकेनिज्म प्रदान करता है, जो एप्लिकेशन को अनधिकृत एक्सेस से सुरक्षित करता है।

5. वेब एप्लिकेशन में Spring MVC की भूमिका का वर्णन करें। उत्तर: Spring MVC वेब रिक्वेस्ट्स को हैंडल करता है, URL को कंट्रोलर्स से मैप करता है और वेब रिस्पांस के लिए व्यूज और मॉडल को प्रबंधित करता है।

6. Spring Cloud क्या है और इसके मुख्य कंपोनेंट क्या हैं? उत्तर: Spring Cloud क्लाउड-नेटिव एप्लिकेशन बनाने के लिए टूल्स प्रदान करता है, जिसमें सर्विस डिस्कवरी (Eureka), सर्किट ब्रेकर (Hystrix), और API गेटवे शामिल हैं।

7. Spring AOP एप्लिकेशन फंक्शनैलिटी को कैसे बढ़ाता है? उत्तर: AOP लॉगिंग और ट्रांजैक्शन मैनेजमेंट जैसे क्रॉस-कटिंग कंसर्न को बिजनेस लॉजिक से अलग करने की अनुमति देता है, जिसमें एस्पेक्ट्स और एडवाइस का उपयोग किया जाता है।

8. Spring Boot Actuator क्या है, और यह क्या करता है? उत्तर: Actuator एप्लिकेशन की मॉनिटरिंग और मैनेजमेंट के लिए एंडपॉइंट प्रदान करता है, जैसे हेल्थ चेक, मेट्रिक्स और एनवायरनमेंट जानकारी।

9. Spring प्रोफाइल्स के उपयोग की व्याख्या करें। उत्तर: प्रोफाइल्स विभिन्न एनवायरनमेंट (जैसे, डेवलपमेंट, प्रोडक्शन) के लिए अलग-अलग कॉन्फ़िगरेशन की अनुमति देते हैं, जिससे एनवायरनमेंट-विशिष्ट सेटिंग्स सक्षम होती हैं।

10. Spring Boot स्टार्टर डिपेंडेंसी मैनेजमेंट को कैसे सरल बनाते हैं? उत्तर: स्टार्टर्स किसी विशिष्ट फंक्शनैलिटी के लिए सभी आवश्यक डिपेंडेंसी शामिल करते हैं, जिससे मैन्युअल रूप से डिपेंडेंसी प्रबंधित करने की आवश्यकता कम हो जाती है।

### Microservices Architecture

1. सर्विस डिस्कवरी क्या है, और यह महत्वपूर्ण क्यों है? उत्तर: सर्विस डिस्कवरी सर्विसेज को लोकेट करने की प्रक्रिया को स्वचालित करती है, जो डायनामिक एनवायरनमेंट और स्केलिंग के लिए आवश्यक है।

2. माइक्रोसर्विसेज में API गेटवे की भूमिका समझाएं। उत्तर: एक API गेटवे सिंगल एंट्री पॉइंट के रूप में कार्य करता है, रिक्वेस्ट्स को उचित सर्विसेज पर रूट करता है, सुरक्षा और प्रोटोकॉल ट्रांसलेशन को हैंडल करता है।

3. सर्किट ब्रेकर पैटर्न क्या है, और यह कैसे मदद करता है? उत्तर: सर्किट ब्रेकर फेलिंग सर्विसेज को रिकवर करने का मौका देते हुए, उन्हें रिक्वेस्ट्स को इंटरप्ट करके कैस्केडिंग फेल्योर को रोकता है।

4. RESTful API डिजाइन सिद्धांतों का वर्णन करें। उत्तर: REST सिद्धांतों में स्टेटलेसनेस, क्लाइंट-सर्वर आर्किटेक्चर, कैशेबिलिटी और यूनिफॉर्म इंटरफेस शामिल हैं, जो स्केलेबल और मेंटेनएबल API सुनिश्चित करते हैं।

5. GraphQL क्या है, और यह REST से कैसे भिन्न है? उत्तर: GraphQL API के लिए एक क्वेरी लैंग्वेज है, जो क्लाइंट्स को ठीक वही रिक्वेस्ट करने की अनुमति देती है जिसकी उन्हें आवश्यकता है, जिससे ओवर-फेचिंग और अंडर-फेचिंग कम होती है।

6. माइक्रोसर्विसेज में आप API वर्जनिंग को कैसे हैंडल करते हैं? उत्तर: वर्जनिंग URL पाथ, हेडर या क्वेरी पैरामीटर के माध्यम से की जा सकती है, जिससे बैकवर्ड कम्पैटिबिलिटी और स्मूद ट्रांजिशन सुनिश्चित होते हैं।

7. माइक्रोसर्विसेज में Saga पैटर्न समझाएं। उत्तर: Saga सर्विसेज में डिस्ट्रीब्यूटेड ट्रांजैक्शन को प्रबंधित करता है, जिसमें लोकल ट्रांजैक्शन की एक श्रृंखला और फेल्योर के लिए कम्पेनसेशन का उपयोग किया जाता है।

8. माइक्रोसर्विसेज में हेल्थ चेक क्या हैं, और वे महत्वपूर्ण क्यों हैं? उत्तर: हेल्थ चेक सर्विसेज की उपलब्धता और परफॉर्मेंस को सत्यापित करते हैं, जो मॉनिटरिंग और सर्विस मेश के प्रबंधन के लिए महत्वपूर्ण हैं।

9. माइक्रोसर्विसेज में कॉन्ट्रैक्ट-फर्स्ट डेवलपमेंट का वर्णन करें। उत्तर: कॉन्ट्रैक्ट-फर्स्ट डेवलपमेंट इम्प्लीमेंटेशन से पहले API को परिभाषित करता है, जिससे सर्विसेज के बीच कम्पैटिबिलिटी और डीकपलिंग सुनिश्चित होती है।

10. माइक्रोसर्विसेज में रेट लिमिटिंग को आप कैसे इम्प्लीमेंट करते हैं? उत्तर: रेट लिमिटिंग को मिडलवेयर या Spring Cloud Gateway जैसी API का उपयोग करके इम्प्लीमेंट किया जा सकता है, जो दुरुपयोग को रोकने के लिए रिक्वेस्ट दरों को नियंत्रित करता है।

### Databases and Caching

1. SQL जॉइन क्या हैं, और उनका उपयोग कब किया जाता है? उत्तर: SQL जॉइन दो या अधिक टेबल के रिकॉर्ड को एक संबंधित कॉलम के आधार पर जोड़ते हैं, जिसका उपयोग संबंधित टेबल में डेटा पुनर्प्राप्त करने के लिए किया जाता है।

2. डेटाबेस ट्रांजैक्शन में ACID प्रॉपर्टीज समझाएं। उत्तर: ACID एटॉमिसिटी, कंसिस्टेंसी, आइसोलेशन और ड्यूरेबिलिटी के लिए खड़ा है, जो विश्वसनीय ट्रांजैक्शन प्रोसेसिंग सुनिश्चित करता है।

3. Redis क्या है, और इसका उपयोग कैशिंग में कैसे किया जाता है? उत्तर: Redis एक इन-मेमोरी की-वैल्यू स्टोर है जिसका उपयोग कैशिंग के लिए किया जाता है, जो अक्सर उपयोग किए जाने वाले डेटा को फास्ट एक्सेस प्रदान करता है।

4. कैशिंग के लिए Redis और Memcached की तुलना करें। उत्तर: Redis डेटा स्ट्रक्चर और पर्सिस्टेंस को सपोर्ट करता है, जबकि Memcached बेसिक कैशिंग के लिए सरल और तेज है।

5. डेटाबेस में शार्डिंग क्या है, और इसका उपयोग क्यों किया जाता है? उत्तर: शार्डिंग डेटा को क्षैतिज रूप से कई डेटाबेस में विभाजित करती है, जिसका उपयोग बड़े सिस्टम में स्केलेबिलिटी और परफॉर्मेंस के लिए किया जाता है।

6. Hibernate डेटाबेस इंटरैक्शन को कैसे सरल बनाता है? उत्तर: Hibernate एक ORM फ्रेमवर्क है जो Java क्लासेस को डेटाबेस टेबल से मैप करता है, जिससे CRUD ऑपरेशन सरल हो जाते हैं।

7. JDBC कनेक्शन पूलिंग समझाएं। उत्तर: कनेक्शन पूलिंग डेटाबेस कनेक्शन को पुन: उपयोग करती है, जिससे कनेक्शन क्रिएशन ओवरहेड कम होकर परफॉर्मेंस में सुधार होता है।

8. टाइम-सीरीज डेटाबेस क्या है, और इसका उपयोग कब किया जाता है? उत्तर: InfluxDB जैसे टाइम-सीरीज डेटाबेस टाइम-स्टैम्प्ड डेटा स्टोर करते हैं, जो मॉनिटरिंग, IoT और सेंसर डेटा के लिए आदर्श हैं।

9. डेटाबेस में ट्रांजैक्शन आइसोलेशन लेवल का वर्णन करें। उत्तर: आइसोलेशन लेवल (रीड अनकमिटेड, रीड कमिटेड, रिपीटेबल रीड, सीरियलाइजेबल) परिभाषित करते हैं कि ट्रांजैक्शन एक-दूसरे के साथ कैसे इंटरैक्ट करते हैं।

10. डेटाबेस में इंडेक्सिंग स्ट्रैटेजी को आप कैसे ऑप्टिमाइज़ करते हैं? उत्तर: क्वेरी पैटर्न के आधार पर इंडेक्स चुनें, ओवर-इंडेक्सिंग से बचें, और मल्टी-कॉलम क्वेरी के लिए कम्पोजिट इंडेक्स का उपयोग करें।

### Concurrency and Multithreading

1. Java में डेडलॉक क्या है, और इसे कैसे टाला जा सकता है? उत्तर: डेडलॉक तब होता है जब थ्रेड्स एक-दूसरे के लिए अनिश्चित काल तक प्रतीक्षा करते हैं। इसे सर्कुलर वेट से बचकर और टाइमआउट का उपयोग करके टाला जा सकता है।

2. Java में Executor फ्रेमवर्क समझाएं। उत्तर: Executor फ्रेमवर्क थ्रेड एक्जिक्यूशन को प्रबंधित करता है, जो थ्रेड पूल और टास्क शेड्यूलिंग प्रदान करता है।

3. Callable और Runnable में क्या अंतर है? उत्तर: Callable एक रिजल्ट वापस कर सकता है और एक्सेप्शन फेंक सकता है, जबकि Runnable नहीं कर सकता, जिससे रिजल्ट वापस करने वाले टास्क के लिए Callable अधिक लचीला होता है।

4. Java मेमोरी मॉडल का वर्णन करें। उत्तर: Java मेमोरी मॉडल परिभाषित करता है कि थ्रेड्स वेरिएबल्स तक कैसे पहुंचते हैं, जिससे प्रोसेसरों में ऑपरेशन की विजिबिलिटी और ऑर्डरिंग सुनिश्चित होती है।

5. Java में volatile कीवर्ड क्या है, और इसका उपयोग कब करना चाहिए? उत्तर: Volatile यह सुनिश्चित करता है कि एक वेरिएबल में परिवर्तन सभी थ्रेड्स को दिखाई दे, जिसका उपयोग कैशिंग समस्याओं को रोकने के लिए मल्टी-थ्रेडेड एनवायरनमेंट में किया जाता है।

6. मल्टी-थ्रेडेड एप्लिकेशन में रेस कंडीशन को आप कैसे रोकते हैं? उत्तर: शेयर्ड रिसोर्सेज तक एक्सक्लूसिव एक्सेस सुनिश्चित करने के लिए सिंक्रोनाइजेशन, लॉक्स या एटॉमिक ऑपरेशन का उपयोग करें।

7. रीड-राइट लॉक की अवधारणा समझाएं। उत्तर: रीड-राइट लॉक मल्टीपल रीडर या एक सिंगल राइटर की अनुमति देते हैं, जिससे शेयर्ड एक्सेस की अनुमति देकर कंकरेंसी में सुधार होता है।

8. CountDownLatch क्या है, और इसका उपयोग कैसे किया जाता है? उत्तर: CountDownLatch एक थ्रेड को थ्रेड्स के एक सेट के पूरा होने की प्रतीक्षा करने की अनुमति देता है, जिसका उपयोग थ्रेड एक्जिक्यूशन को समन्वयित करने के लिए किया जाता है।

9. Java में लॉक स्ट्राइपिंग का वर्णन करें। उत्तर: लॉक स्ट्राइपिंग एक लॉक को कई भागों (स्ट्राइप्स) में विभाजित करती है, जिससे विभिन्न भागों तक कंकरेंट एक्सेस की अनुमति मिलती है और कॉन्टेंशन कम होता है।

10. Java में थ्रेड इंटरप्शन को आप कैसे हैंडल करते हैं? उत्तर: थ्रेड्स इंटरप्टेड स्टेटस चेक कर सकते हैं और `InterruptedException` फेंक सकते हैं, जिससे ग्रेसफुल टर्मिनेशन की अनुमति मिलती है।

### Web Servers and Load Balancing

1. Nginx का आमतौर पर उपयोग किस लिए किया जाता है? उत्तर: Nginx का उपयोग वेब सर्वर, रिवर्स प्रॉक्सी, लोड बैलेंसर और HTTP कैश के रूप में किया जाता है, जो अपनी उच्च परफॉर्मेंस और स्केलेबिलिटी के लिए जाना जाता है।

2. लोड बैलेंसर और रिवर्स प्रॉक्सी के बीच अंतर समझाएं। उत्तर: एक लोड बैलेंसर ट्रैफिक को सर्वर के बीच वितरित करता है, जबकि एक रिवर्स प्रॉक्सी रिक्वेस्ट्स को बैकएंड सर्वर पर फॉरवर्ड करता है, जो अक्सर कैशिंग और सुरक्षा प्रदान करता है।

3. HAProxy क्या है, और इसका उपयोग क्यों किया जाता है? उत्तर: HAProxy एक हाई अवेलेबिलिटी लोड बैलेंसर और प्रॉक्सी सर्वर है, जिसका उपयोग नेटवर्क कनेक्शन को प्रबंधित और वितरित करने के लिए किया जाता है।

4. वेब सर्वर पर SSL/TLS को आप कैसे कॉन्फ़िगर करते हैं? उत्तर: SSL/TLS को सर्टिफिकेट प्राप्त करके और HTTPS लिसनर सेट अप करके कॉन्फ़िगर किया जाता है, जिससे ट्रांजिट में डेटा एन्क्रिप्ट होता है।

5. सर्वर-साइड कैशिंग क्या है, और इसे कैसे इम्प्लीमेंट किया जाता है? उत्तर: सर्वर-साइड कैशिंग अक्सर एक्सेस किए जाने वाले डेटा को मेमोरी में स्टोर करती है, जिसे बेहतर परफॉर्मेंस के लिए Varnish या Redis जैसे टूल का उपयोग करके इम्प्लीमेंट किया जाता है।

6. वेब सर्वर में लॉगिंग का महत्व समझाएं। उत्तर: लॉगिंग सर्वर एक्टिविटी की निगरानी करने, समस्याओं का निवारण करने और सुरक्षा का ऑडिट करने में मदद करती है, जिसमें विश्लेषण के लिए ELK स्टैक जैसे टूल का उपयोग किया जाता है।

7. वेब सर्वर को सुरक्षित करने के लिए सर्वोत्तम प्रथाएं क्या हैं? उत्तर: सर्वोत्तम प्रथाओं में सुरक्षा हेडर का उपयोग, सॉफ्टवेयर को अपडेट रखना और खतरों से सुरक्षा के लिए फायरवॉल कॉन्फ़िगर करना शामिल है।

8. लोड बैलेंसिंग में आप सत्र निरंतरता को कैसे हैंडल करते हैं? उत्तर: सत्र निरंतरता स्टिकी सेशन या सत्र रेप्लिकेशन का उपयोग करके प्राप्त की जा सकती है, जिससे उपयोगकर्ता सत्र सुसंगत बने रहते हैं।

9. SSL ऑफलोडिंग क्या है, और यह फायदेमंद क्यों है? उत्तर: SSL ऑफलोडिंग लोड बैलेंसर पर SSL/TLS ट्रैफिक को डिक्रिप्ट करती है, जिससे सर्वर लोड कम होता है और परफॉर्मेंस में सुधार होता है।

10. वेब सर्वर को क्षैतिज रूप से स्केल करने की प्रक्रिया का वर्णन करें। उत्तर: क्षैतिज स्केलिंग में बढ़े हुए लोड को हैंडल करने के लिए अधिक सर्वर जोड़ना शामिल है, जिसे लोड बैलेंसर और ऑटो-स्केलिंग ग्रुप के माध्यम से प्रबंधित किया जाता है।

### CI/CD and DevOps

1. GitOps क्या है, और यह पारंपरिक CI/CD से कैसे भिन्न है? उत्तर: GitOps इन्फ्रास्ट्रक्चर को कोड के रूप में मानता है, जो कॉन्फ़िगरेशन और डिप्लॉयमेंट को प्रबंधित करने के लिए Git रिपॉजिटरी का उपयोग करता है, जो डिक्लेरेटिव परिभाषाओं पर जोर देता है।

2. ब्लू/ग्रीन डिप्लॉयमेंट स्ट्रैटेजी समझाएं। उत्तर: ब्लू/ग्रीन डिप्लॉयमेंट में दो समान एनवायरनमेंट चलाना शामिल है, जो सफल डिप्लॉयमेंट पर ट्रैफिक को नए एनवायरनमेंट में स्विच करता है।

3. Jenkins पाइपलाइन क्या है, और इसे कैसे कॉन्फ़िगर किया जाता है? उत्तर: एक Jenkins पाइपलाइन सॉफ्टवेयर को बनाने, परीक्षण और डिप्लॉय करने के लिए चरणों की एक श्रृंखला है, जिसे डिक्लेरेटिव या स्क्रिप्टेड सिंटैक्स का उपयोग करके Jenkinsfile में परिभाषित किया जाता है।

4. CI/CD पाइपलाइन में आप कंटीन्यूअस इंटीग्रेशन को कैसे इम्प्लीमेंट करते हैं? उत्तर: कंटीन्यूअस इंटीग्रेशन कमिट पर कोड के निर्माण और परीक्षण को स्वचालित करता है, यह सुनिश्चित करता है कि कोड हमेशा डिप्लॉय करने योग्य स्थिति में हो।

5. CI/CD में Docker की क्या भूमिका है? उत्तर: Docker कंटेनर बनाने, परीक्षण और एप्लिकेशन को डिप्लॉय करने के लिए सुसंगत एनवायरनमेंट प्रदान करते हैं, जिससे स्टेज में समानता सुनिश्चित होती है।

6. इन्फ्रास्ट्रक्चर एज कोड (IaC) की अवधारणा समझाएं। उत्तर: IaC कोड का उपयोग करके इन्फ्रास्ट्रक्चर को प्रबंधित करता है, जिससे एनवायरनमेंट सेटअप में वर्जन कंट्रोल, ऑटोमेशन और स्थिरता की अनुमति मिलती है।

7. CI/CD में Kubernetes का उपयोग करने के क्या लाभ हैं? उत्तर: Kubernetes कंटेनराइज्ड एप्लिकेशन को ऑर्केस्ट्रेट करता है, जो स्केलेबिलिटी, सेल्फ-हीलिंग और डिक्लेरेटिव डिप्लॉयमेंट क्षमताएं प्रदान करता है।
