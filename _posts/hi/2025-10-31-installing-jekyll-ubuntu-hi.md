### Ubuntu पर Jekyll स्थापित और चलाना

आपकी त्रुटि ("Command 'bundle' not found") और आपके द्वारा चलाए जा रहे कमांड्स के आधार पर, ऐसा लगता है कि आपको पहले Ruby, Bundler, और Jekyll को सही तरीके से इंस्टॉल करने की आवश्यकता है। Ubuntu में ये डिफ़ॉल्ट रूप से शामिल नहीं होते हैं, लेकिन आधिकारिक Jekyll गाइड का उपयोग करके यह सीधा-साधा काम है। मैं आपको पूरी सेटअप प्रक्रिया चरण दर चरण समझाऊंगा, और फिर आपके विशिष्ट कमांड्स को चलाने का तरीका बताऊंगा।

#### चरण 1: सिस्टम डिपेंडेंसीज़ इंस्टॉल करें
अपना टर्मिनल खोलें और Ruby और बिल्ड टूल्स इंस्टॉल करने के लिए यह चलाएं (इसमें `ruby-full` के माध्यम से Ruby 3.x शामिल है):

```bash
sudo apt update
sudo apt install ruby-full build-essential zlib1g-dev
```

- यह Ruby सेटअप कर देगा (जांचें `ruby -v` से—इसके परिणाम में 3.0+ जैसा कुछ दिखना चाहिए)।
- यदि आप नए Ubuntu (जैसे, 22.04+) पर हैं, तो यह बिना किसी समस्या के काम करेगा। संभव हो तो Ruby के लिए Snap का उपयोग करने से बचें, क्योंकि इससे path संबंधी समस्याएं हो सकती हैं।

#### चरण 2: एक यूज़र Gem डायरेक्टरी सेटअप करें (सिफारिश की जाती है, Sudo से बचने के लिए)
बिना रूट विशेषाधिकार के gems इंस्टॉल करने के लिए (बाद में परमिशन एरर से बचाता है):

```bash
echo '# Install Ruby Gems to ~/gems' >> ~/.bashrc
echo 'export GEM_HOME="$HOME/gems"' >> ~/.bashrc
echo 'export PATH="$HOME/gems/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

- सत्यापित करें: `gem env` चलाएं—इसमें आपका `GEM_HOME` `~/gems` दिखना चाहिए।

#### चरण 3: Jekyll और Bundler इंस्टॉल करें
अब टूल्स इंस्टॉल करें:

```bash
gem install jekyll bundler
```

- यह `gem`, `jekyll`, और `bundle` कमांड्स को आपके path में जोड़ देगा।
- यदि परमिशन एरर आते हैं, तो चरण 2 को दोबारा जांचें या अस्थायी रूप से `sudo` का उपयोग करें (लेकिन लंबे समय तक इसके उपयोग से बचें)।

#### चरण 4: अपना ब्लॉग बनाएं और चलाएं
अब आप अपने स्निपेट के कमांड्स चला सकते हैं। मैं प्रत्येक की व्याख्या करूंगा:

1. **एक नई Jekyll साइट बनाएं**:
   ```bash
   jekyll new myblog
   ```
   - यह एक `myblog` फोल्डर में एक बेसिक साइट जनरेट करता है।

2. **डायरेक्टरी में जाएं**:
   ```bash
   cd myblog
   ```

3. **डिपेंडेंसीज़ इंस्टॉल करें** (यह वह जगह है जहां आपका मूल `bundle install` फेल हुआ था—अब यह काम करना चाहिए):
   ```bash
   bundle install
   ```
   - Jekyll प्लगइन्स जैसे gems को डाउनलोड करता है।

4. **साइट सर्व करें** (http://127.0.0.1:4000 पर एक लोकल सर्वर शुरू करता है):
   ```bash
   bundle exec jekyll serve
   ```
   - अपना ब्राउज़र उस URL पर खोलें ताकि आपकी साइट दिखाई दे।

5. **इंक्रीमेंटल रीबिल्ड के साथ सर्व करें** (डेवलपमेंट के लिए तेज़, केवल बदले हुए फाइलों को दोबारा बनाता है):
   ```bash
   bundle exec jekyll serve --incremental
   ```

6. **ड्राफ्ट्स सहित सर्व करें** (अप्रकाशित पोस्ट दिखाता है):
   ```bash
   bundle exec jekyll serve --draft
   ```

7. **यदि आपको "webrick" एरर मिलता है** (Ruby 3+ पर आम, क्योंकि यह डिफ़ॉल्ट रूप से बंडल नहीं होता है):
   ```bash
   bundle add webrick
   ```
   - फिर सर्व कमांड को रीस्टार्ट करें: `bundle exec jekyll serve`।
   - वैकल्पिक रूप से, अपनी `Gemfile` में `gem "webrick"` जोड़ें और `bundle install` चलाएं।

8. **Bundler के बिना त्वरित सर्व करें** (यदि आप `bundle exec` छोड़ते हैं):
   ```bash
   jekyll serve
   ```
   - लेकिन प्रोजेक्ट्स में स्थिरता के लिए `bundle exec` का उपयोग करें।

#### समस्या निवारण
- **अभी भी "bundle not found" आ रहा है?** सुनिश्चित करें कि चरण 2 ने आपके `~/.bashrc` को सही तरीके से सोर्स किया है (`source ~/.bashrc`), फिर अपना टर्मिनल रीस्टार्ट करें या `hash -r` चलाएं। `which bundle` से जांचें।
- **Gems के साथ परमिशन समस्याएं?** चरण 2 में बताए गए यूज़र डायरेक्टरी सेटअप पर टिके रहें।
- **Ruby वर्जन बहुत पुराना है?** यदि `ruby -v` <3.0 है, तो नए वर्जन के लिए rbenv का उपयोग करने पर विचार करें (`curl -fsSL https://github.com/rbenv/rbenv-installer/raw/HEAD/bin/rbenv-installer | bash` के माध्यम से इंस्टॉल करें, फिर `rbenv install 3.2.0` चलाएं)।
- **फ़ायरवॉल पोर्ट 4000 को ब्लॉक कर रहा है?** Ubuntu आमतौर पर नहीं करता है, लेकिन `curl http://127.0.0.1:4000` से टेस्ट करें।
- आपका प्रोजेक्ट `~/projects/blog-source` में है—इन कमांड्स को वहां से चलाएं या पाथ एडजस्ट करें।

एक बार सर्व हो जाने के बाद, `_posts/` या `_config.yml` में फाइलों को एडिट करें और सेव करें—Jekyll स्वचालित रूप से रीबिल्ड करेगा। प्रोडक्शन के लिए, `jekyll build` के साथ बिल्ड करें और GitHub Pages या Netlify पर डिप्लॉय करें।
