### मुख्य बिंदु
- ऐसा लगता है कि ये Babel पैकेज आधुनिक JavaScript (ES2015) को ब्राउज़र संगतता के लिए पुराने संस्करणों में संकलित करने के लिए उपयोग किए जाते हैं, खासकर webpack-आधारित फ्रंट-एंड प्रोजेक्ट में।
- शोध से पता चलता है कि सेटअप में विशिष्ट संस्करण स्थापित करना, एक `.babelrc` फ़ाइल बनाना और `babel-loader` का उपयोग करने के लिए webpack को कॉन्फ़िगर करना शामिल है।
- सबूत इन पुराने संस्करणों के लिए संगतता समस्याओं के न्यूनतम होने की ओर इशारा करते हैं, लेकिन यह सुनिश्चित करें कि सभी पैकेज सही ढंग से स्थापित हैं।

### स्थापना
पहले, इन कमांड के साथ पैकेज स्थापित करें:

```bash
npm install --save-dev \
  "babel-core@6.13.0" \
  "babel-loader@6.2.5" \
  "babel-plugin-transform-runtime@6.12.0" \
  "babel-preset-es2015@6.13.2" \
  "babel-runtime@6.11.6"
```

### कॉन्फ़िगरेशन
अपने प्रोजेक्ट रूट में एक `.babelrc` फ़ाइल बनाएं:

```json
{
  "presets": ["es2015"],
  "plugins": ["transform-runtime"]
}
```

फिर, अपना `webpack.config.js` इस तरह से सेट करें:

```javascript
module.exports = {
  entry: './index.js',
  output: {
    filename: 'bundle.js'
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'babel-loader',
        options: {
          configFile: './.babelrc'
        }
      }
    ]
  }
};
```

### उपयोग
अपना ES2015 JavaScript कोड `index.js` जैसी फ़ाइलों में लिखें, फिर इसे `bundle.js` में संकलित करने के लिए `npx webpack` चलाएं, जिसे पुराने ब्राउज़र समझ सकते हैं।

### अप्रत्याशित विवरण
एक अप्रत्याशित पहलू यह है कि ये पुराने संस्करण (Babel 6) हैं, जो अब Babel 7 के पक्ष में अप्रचलित हैं, जो दीर्घकालिक रखरखाव को प्रभावित कर सकता है लेकिन लीगेसी प्रोजेक्ट्स के लिए ठीक है।

### सर्वेक्षण नोट: Babel पैकेजों की विस्तृत सेटअप और उपयोग

यह खंड एक JavaScript प्रोजेक्ट में विशेष रूप से webpack के साथ फ्रंट-एंड डेवलपमेंट के लिए, निर्दिष्ट Babel पैकेजों—`babel-core@6.13.0`, `babel-loader@6.2.5`, `babel-plugin-transform-runtime@6.12.0`, `babel-preset-es2015@6.13.2`, और `babel-runtime@6.11.6`—के उपयोग पर एक व्यापक मार्गदर्शिका प्रदान करता है। संस्करणों को देखते हुए, ये Babel 6 का हिस्सा हैं, जो आधुनिक JavaScript (ES2015) को व्यापक ब्राउज़र संगतता के लिए ES5 में ट्रांसपाइल करने के लिए एक पुराना लेकिन अभी भी कार्यात्मक सुइट है। निम्नलिखित विवरण स्थापना, कॉन्फ़िगरेशन, उपयोग और विचारों को कवर करते हैं, जिससे लीगेसी या विशिष्ट प्रोजेक्ट्स पर काम करने वाले डेवलपर्स के लिए पूरी समझ सुनिश्चित होती है।

#### पृष्ठभूमि और उद्देश्य
Babel एक JavaScript कंपाइलर है जो आधुनिक सिंटैक्स को पुराने, अधिक व्यापक रूप से समर्थित संस्करणों में रूपांतरित करता है। सूचीबद्ध पैकेज Babel 6 के हैं, जो Babel 7 के प्रमुख अपडेट से पहले प्रचलित था। `babel-core@6.13.0` मुख्य संकलन कार्यक्षमता प्रदान करता है, जबकि `babel-loader@6.2.5` फ्रंट-एंड बिल्ड के लिए webpack के साथ एकीकृत होता है। `babel-preset-es2015@6.13.2` ES2015 (ECMAScript 2015) से ES5 रूपांतरण सक्षम करता है, और `babel-plugin-transform-runtime@6.12.0` के साथ `babel-runtime@6.11.6` हेल्पर फ़ंक्शंस को बाहरीकृत करके कोड को अनुकूलित करता है, जिससे डुप्लिकेशन कम होता है।

वर्तमान तिथि, 3 मार्च, 2025 को देखते हुए, ये संस्करण विशेष रूप से पुराने हैं, जिसमें Babel 7 और उसके बाद के मानक हैं। हालाँकि, लीगेसी प्रोजेक्ट्स या विशिष्ट आवश्यकताओं के लिए, ये प्रासंगिक बने हुए हैं। उपयोगकर्ता के प्रश्न से webpack वातावरण में सेटअप की आवश्यकता का संकेत मिलता है, क्योंकि `babel-loader` शामिल है, हालाँकि Babel CLI जैसे विकल्पों पर भी विचार किया जाता है।

#### स्थापना प्रक्रिया
शुरू करने के लिए, npm का उपयोग करके विकास निर्भरताओं के रूप में पैकेज स्थापित करें। कमांड निर्दिष्ट संस्करण सुनिश्चित करती है:

```bash
npm install --save-dev \
  "babel-core@6.13.0" \
  "babel-loader@6.2.5" \
  "babel-plugin-transform-runtime@6.12.0" \
  "babel-preset-es2015@6.13.2" \
  "babel-runtime@6.11.6"
```

यह कदम उन्हें `package.json` में `devDependencies` के तहत रखता है, यह सुनिश्चित करता है कि वे बिल्ड प्रक्रियाओं के लिए उपलब्ध हैं लेकिन प्रोडक्शन में बंडल नहीं हैं। ध्यान दें कि `babel-runtime` आमतौर पर कुछ सेटअप में एक रनटाइम निर्भरता है, लेकिन यहाँ, प्लगइन को देखते हुए, इसे ट्रांसफॉर्म के साथ स्थिरता के लिए एक dev निर्भरता के रूप में माना जाता है।

#### कॉन्फ़िगरेशन विवरण
कॉन्फ़िगरेशन में दो मुख्य फ़ाइलें शामिल हैं: Babel सेटिंग्स के लिए `.babelrc` और webpack एकीकरण के लिए `webpack.config.js`।

##### `.babelrc` फ़ाइल
प्रोजेक्ट रूट में निम्नलिखित सामग्री के साथ `.babelrc` बनाएं या संपादित करें:

```json
{
  "presets": ["es2015"],
  "plugins": ["transform-runtime"]
}
```

- **प्रीसेट**: `"es2015"` `babel-preset-es2015@6.13.2` से मेल खाता है, जो सभी ES2015 रूपांतरणों को ES5 में सक्षम करता है, जैसे एरो फ़ंक्शंस, let/const, और क्लासेस।
- **प्लगइन्स**: `"transform-runtime"` `babel-plugin-transform-runtime@6.12.0` का उपयोग करता है ताकि Babel हेल्पर्स को बाहरीकृत किया जा सके, जो `babel-runtime@6.11.6` को संदर्भित करता है, जो फ़ाइलों में हेल्पर डुप्लिकेशन से बचकर संकलित कोड के आकार को कम करता है।

यह कॉन्फ़िगरेशन सुनिश्चित करता है कि ES2015 कोड ट्रांसपाइल हो और रनटाइम दक्षता के लिए अनुकूलित हो।

##### Webpack कॉन्फ़िगरेशन
Webpack के लिए, निम्न के साथ `webpack.config.js` बनाएं या अपडेट करें:

```javascript
module.exports = {
  entry: './index.js',
  output: {
    filename: 'bundle.js'
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'babel-loader',
        options: {
          configFile: './.babelrc'
        }
      }
    ]
  }
};
```

- **एंट्री और आउटपुट**: `index.js` को एंट्री पॉइंट के रूप में निर्दिष्ट करता है और `bundle.js` आउटपुट करता है।
- **मॉड्यूल नियम**: `.js` फ़ाइलों के लिए `babel-loader@6.2.5` का उपयोग करता है, विकल्पों के लिए `.babelrc` की ओर इशारा करता है, यह सुनिश्चित करता है कि Babel बंडलिंग से पहले फ़ाइलों को प्रोसेस करता है।

यह सेटअप JavaScript फ़ाइलों को संभालने के लिए webpack की मॉड्यूल प्रणाली का लाभ उठाता है, उन्हें Babel के माध्यम से संकलित करता है।

#### उपयोग और वर्कफ़्लो
सेटअप पूरा होने के साथ, अपना ES2015 कोड `index.js` जैसी फ़ाइलों में लिखें। उदाहरण के लिए:

```javascript
const arr = [1, 2, 3];
const sum = arr.reduce((a, b) => a + b, 0);
console.log(sum);
```

बिल्ड कमांड चलाएं:

```bash
npx webpack
```

यह कोड को `bundle.js` में संकलित करता है, जिसे ब्राउज़र निष्पादन के लिए HTML में शामिल किया जा सकता है, जिससे पुराने वातावरण के साथ संगतता सुनिश्चित होती है। `transform-runtime` प्लगइन सुनिश्चित करता है कि हेल्पर्स साझा किए जाते हैं, जिससे बंडल आकार कम होता है, जो विशेष रूप से बड़े प्रोजेक्ट्स के लिए उपयोगी है।

#### वैकल्पिक उपयोग: Babel CLI
जबकि प्रश्न एक webpack सेटअप का सुझाव देता है, पूर्णता के लिए, webpack के बिना मैनुअल संकलन के लिए Babel CLI का उपयोग करने पर विचार करें। पहले, `babel-cli@6.13.0` स्थापित करें:

```bash
npm install --save-dev "babel-cli@6.13.0"
```

फिर, फ़ाइलों को संकलित करें:

```bash
./node_modules/.bin/babel src/index.js -o lib/index.js
```

या, `package.json` स्क्रिप्ट्स में जोड़ें:

```json
"scripts": {
  "build": "babel src --out-dir lib"
}
```

`npm run build` के साथ चलाएं, जो संकलित फ़ाइलों को `lib/` में आउटपुट करता है, Node.js प्रोजेक्ट्स या सर्वर-साइड कोड के लिए उपयोगी।

#### संगतता और विचार
संस्करणों (सभी Babel 6 के आसपास, जो सालों पहले जारी किए गए थे) को देखते हुए, संगतता आम तौर पर एक ही प्रमुख संस्करण के भीतर सुनिश्चित की जाती है। हालाँकि, ध्यान दें:

- `babel-loader@6.2.5` webpack 1 और 2 के साथ काम करता है, इसलिए सुनिश्चित करें कि आपका webpack संस्करण (जैसे, `webpack@1.14.0`) संगत है।
- ये पैकेज Babel 7 के पक्ष में अप्रचलित हैं, जो `@babel/core` जैसे स्कोप्ड पैकेज का उपयोग करता है। दीर्घकालिक प्रोजेक्ट्स के लिए, अपग्रेड पर विचार करें, लेकिन लीगेसी के लिए, ये ठीक काम करते हैं।
- कोई नामकरण संघर्ष न हो, इसकी पुष्टि करें; उपयोगकर्ता की सूची में स्पेस (जैसे, "babel loader") थे, संभवतः टाइपो, जिन्हें हाइफ़न (जैसे, `babel-loader`) में सही किया गया है।

#### संभावित समस्याएं
- सुनिश्चित करें कि `.babelrc` प्रोजेक्ट रूट में है, या webpack विकल्पों में पथ निर्दिष्ट करें।
- सत्यापित करें कि `babel-runtime` स्थापित है, क्योंकि `transform-runtime` हेल्पर्स के लिए इस पर निर्भर करता है।
- इन संस्करणों की उम्र को देखते हुए, स्थापना के दौरान अप्रचलन चेतावनियों की जाँच करें, लेकिए इच्छित उपयोग के लिए उन्हें अपेक्षा के अनुरूप कार्य करना चाहिए।

#### तालिका: पैकेज भूमिकाएँ और संस्करण

| पैकेज नाम                     | संस्करण | भूमिका                                                                |
|-------------------------------|---------|----------------------------------------------------------------------|
| `babel-core`                  | 6.13.0  | Babel के लिए मुख्य संकलन कार्यक्षमता।                            |
| `babel-loader`                | 6.2.5   | Babel के लिए Webpack लोडर, JavaScript फ़ाइलों को संकलित करता है।                  |
| `babel-plugin-transform-runtime` | 6.12.0 | हेल्पर्स को बाहरीकृत करने के लिए प्लगइन, कोड डुप्लिकेशन कम करता है।            |
| `babel-preset-es2015`         | 6.13.2  | ES2015 से ES5 रूपांतरण के लिए प्रीसेट, आधुनिक सिंटैक्स सक्षम करता है।     |
| `babel-runtime`               | 6.11.6  | `transform-runtime` प्लगइन के लिए हेल्पर्स प्रदान करने वाली रनटाइम लाइब्रेरी।    |

यह तालिका सेटअप में उनके उद्देश्य को समझने में सहायता के लिए प्रत्येक पैकेज की भूमिका को सारांशित करती है।

#### अप्रत्याशित विवरण: लीगेसी संदर्भ
एक दिलचस्प पहलू यह है कि ये Babel 6 संस्करण हैं, जो अब अप्रचलित हैं, जिसमें Babel 7 ने स्कोप्ड पैकेज (जैसे, `@babel/core`) पेश किए। यह सेटअप लीगेसी प्रोजेक्ट्स के लिए विशिष्ट है, और जबकि कार्यात्मक है, आधुनिक सुविधाओं जैसे बेहतर TypeScript समर्थन या Babel 7 में प्रदर्शन अनुकूलन का अभाव हो सकता है, जो वर्तमान मानकों के आदी डेवलपर्स के लिए अप्रत्याशित हो सकता है।

#### निष्कर्ष
यह मार्गदर्शिका एक webpack-आधारित फ्रंट-एंड प्रोजेक्ट के लिए निर्दिष्ट Babel पैकेजों को सेट करने को कवर करती है, जिसमें Babel CLI के लिए एक विकल्प शामिल है। संस्करणों को देखते हुए, webpack 1 या 2 के साथ संगतता सुनिश्चित करें, और अप्रचलन से अवगत रहें, लेकिए इच्छित उपयोग के लिए, यह निर्बाध रूप से काम करना चाहिए। अधिक जानकारी के लिए, प्रदान किए गए प्रलेखन लिंक देखें।
