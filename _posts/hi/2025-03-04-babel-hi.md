### मुख्य बिंदु
- यह लगता है कि ये Babel पैकेज आधुनिक JavaScript (ES2015) को पुराने संस्करणों में कम्पाइल करने के लिए उपयोग किए जाते हैं, विशेष रूप से एक webpack आधारित फ्रंट-एंड प्रोजेक्ट में ब्राउज़र संगतता के लिए।
- अनुसंधान सुझाव देता है कि सेटअप में विशेष संस्करणों का इंस्टॉल करना, एक `.babelrc` फ़ाइल बनाना और webpack को `babel-loader` का उपयोग करने के लिए कॉन्फ़िगर करना शामिल है।
- सबूत पुराने संस्करणों के लिए संगतता समस्याओं को कम होने की ओर इशारा करते हैं, लेकिन सभी पैकेज सही तरह से इंस्टॉल हो गए हैं, यह सुनिश्चित करें।

### इंस्टॉलेशन
पहले, इन कमांडों के साथ पैकेज इंस्टॉल करें:

```bash
npm install --save-dev \
  "babel-core@6.13.0" \
  "babel-loader@6.2.5" \
  "babel-plugin-transform-runtime@6.12.0" \
  "babel-preset-es2015@6.13.2" \
  "babel-runtime@6.11.6"
```

### कॉन्फ़िगरेशन
प्रोजेक्ट रूट में एक `.babelrc` फ़ाइल बनाएं:

```json
{
  "presets": ["es2015"],
  "plugins": ["transform-runtime"]
}
```

फिर, `webpack.config.js` को इस तरह से सेटअप करें:

```javascript
module.exports = {
  entry: './index.js',
  output: {
    filename: 'bundle.js'
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'babel-loader',
        options: {
          configFile: './.babelrc'
        }
      }
    ]
  }
};
```

### उपयोग
ES2015 JavaScript कोड को `index.js` जैसे फ़ाइलों में लिखें, फिर `npx webpack` चलाएं ताकि इसे `bundle.js` में कम्पाइल किया जा सके, जिसे पुराने ब्राउज़र समझ सकते हैं।

### अनपेक्षित विवरण
एक अनपेक्षित पहलू यह है कि ये पुराने संस्करण हैं (Babel 6), अब Babel 7 के लिए वर्जित, जो लंबे समय तक रखरखाव पर प्रभाव डाल सकता है, लेकिन पुराने प्रोजेक्टों के लिए ठीक है।

### सर्वेक्षण नोट: Babel पैकेजों का विस्तृत सेटअप और उपयोग

इस खंड में, `babel-core@6.13.0`, `babel-loader@6.2.5`, `babel-plugin-transform-runtime@6.12.0`, `babel-preset-es2015@6.13.2`, और `babel-runtime@6.11.6` जैसे विशेष Babel पैकेजों का उपयोग करने का एक व्यापक मार्गदर्शन प्रदान किया गया है—एक JavaScript प्रोजेक्ट में, विशेष रूप से webpack के साथ फ्रंट-एंड विकास के लिए। इन संस्करणों के आधार पर, ये Babel 6 का हिस्सा हैं, एक पुराना लेकिन अभी भी कार्यक्षम सूट, जो आधुनिक JavaScript (ES2015) को ES5 में ट्रांसपाइल करने के लिए है, ताकि ब्राउज़र संगतता अधिक हो सके। निम्नलिखित विवरण इंस्टॉलेशन, कॉन्फ़िगरेशन, उपयोग, और विचारों को कवर करते हैं, ताकि पुराने या विशेष प्रोजेक्ट पर काम करने वाले डेवलपर्स के लिए एक गहन समझ सुनिश्चित हो सके।

#### पृष्ठभूमि और उद्देश्य
Babel एक JavaScript कम्पाइलर है जो आधुनिक सिंटैक्स को पुराने, अधिक व्यापक रूप से समर्थित संस्करणों में परिवर्तित करता है। सूचीबद्ध पैकेज Babel 6 के हैं, जो Babel 7 के प्रमुख अपडेट से पहले प्रचलित थे। `babel-core@6.13.0` कोर कम्पाइलेशन कार्यक्षमता प्रदान करता है, जबकि `babel-loader@6.2.5` webpack के साथ फ्रंट-एंड बिल्ड के लिए एकीकृत होता है। `babel-preset-es2015@6.13.2` ES2015 (ECMAScript 2015) से ES5 में परिवर्तन को सक्षम बनाता है, और `babel-plugin-transform-runtime@6.12.0` के साथ `babel-runtime@6.11.6` कोड को ऑप्टिमाइज़ करता है, हेल्पर फंक्शंस को बाहरीकरण करके, डुप्लिकेशन को कम करता है।

3 मार्च, 2025 की तारीख के आधार पर, ये संस्करण उल्लेखनीय रूप से पुराने हैं, Babel 7 और उससे आगे का मानक है। फिर भी, पुराने प्रोजेक्टों या विशेष आवश्यकताओं के लिए, ये अभी भी प्रासंगिक हैं। उपयोगकर्ता के प्रश्न से पता चलता है कि एक webpack वातावरण में सेटअप की आवश्यकता है, `babel-loader` के समावेश के कारण, हालांकि Babel CLI जैसे विकल्प भी विचार में लिए जाते हैं।

#### इंस्टॉलेशन प्रक्रिया
शुरू करने के लिए, npm के रूप में विकास निर्भरताओं के रूप में पैकेज इंस्टॉल करें। कमांड सुनिश्चित करता है कि निर्दिष्ट संस्करणों के अनुसार:

```bash
npm install --save-dev \
  "babel-core@6.13.0" \
  "babel-loader@6.2.5" \
  "babel-plugin-transform-runtime@6.12.0" \
  "babel-preset-es2015@6.13.2" \
  "babel-runtime@6.11.6"
```

यह कदम उन्हें `package.json` में `devDependencies` के तहत रखता है, सुनिश्चित करता है कि वे बिल्ड प्रक्रियाओं के लिए उपलब्ध हैं, लेकिन उत्पादन में बंडल नहीं होते हैं। ध्यान दें कि `babel-runtime` आमतौर पर कुछ सेटअप में एक रनटाइम निर्भरता है, लेकिन यहां, प्लगइन के कारण, इसे एक विकास निर्भरता के रूप में सेट किया गया है, ट्रांसफॉर्म के साथ सुसंगतता के लिए।

#### कॉन्फ़िगरेशन विवरण
कोन्फ़िगरेशन में दो मुख्य फ़ाइलें शामिल हैं: `.babelrc` Babel सेटिंग्स के लिए और `webpack.config.js` webpack एकीकरण के लिए।

##### `.babelrc` फ़ाइल
प्रोजेक्ट रूट में `.babelrc` बनाएं या संपादित करें:

```json
{
  "presets": ["es2015"],
  "plugins": ["transform-runtime"]
}
```

- **प्रिसेट्स**: `"es2015"` `babel-preset-es2015@6.13.2` के साथ संबद्ध है, जो सभी ES2015 परिवर्तनों को ES5 में सक्षम बनाता है, जैसे कि तीर फंक्शंस, let/const, और क्लासेस।
- **प्लगइन्स**: `"transform-runtime"` `babel-plugin-transform-runtime@6.12.0` का उपयोग करता है, `babel-runtime@6.11.6` को संदर्भित करता है, जो हेल्पर डुप्लिकेशन को कम करता है, फ़ाइलों के बीच, जो कोड की आकार को कम करता है।

यह कॉन्फ़िगरेशन सुनिश्चित करता है कि ES2015 कोड ट्रांसपाइल और रनटाइम के लिए ऑप्टिमाइज़ किया जाता है।

##### Webpack कॉन्फ़िगरेशन
webpack के लिए, `webpack.config.js` बनाएं या इस तरह से अपडेट करें:

```javascript
module.exports = {
  entry: './index.js',
  output: {
    filename: 'bundle.js'
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'babel-loader',
        options: {
          configFile: './.babelrc'
        }
      }
    ]
  }
};
```

- **इंट्री और आउटपुट**: `index.js` को इंट्री पॉइंट के रूप में और `bundle.js` को आउटपुट के रूप में निर्दिष्ट करता है।
- **मॉड्यूल रूल्स**: `babel-loader@6.2.5` के लिए `.js` फ़ाइलें, `.babelrc` के लिए विकल्पों को संकेत करता है, सुनिश्चित करता है कि Babel फ़ाइलें बंडल करने से पहले प्रोसेस करता है।

यह सेटअप webpack के मॉड्यूल प्रणाली का उपयोग करता है, जो JavaScript फ़ाइलें को कम्पाइल करने के लिए Babel का उपयोग करता है।

#### उपयोग और वर्कफ़्लो
सेटअप पूरा होने के बाद, ES2015 कोड को `index.js` जैसे फ़ाइलों में लिखें। उदाहरण के लिए:

```javascript
const arr = [1, 2, 3];
const sum = arr.reduce((a, b) => a + b, 0);
console.log(sum);
```

बिल्ड कमांड चलाएं:

```bash
npx webpack
```

यह कोड को `bundle.js` में कम्पाइल करता है, जिसे HTML में शामिल किया जा सकता है, ताकि ब्राउज़र द्वारा कार्यान्वित किया जा सके, पुराने वातावरणों के साथ संगतता सुनिश्चित करते हुए। `transform-runtime` प्लगइन सुनिश्चित करता है कि हेल्पर साझा किए जाते हैं, जो कोड आकार को कम करता है, जो बड़े प्रोजेक्टों के लिए विशेष रूप से उपयोगी है।

#### विकल्प उपयोग: Babel CLI
जैसा कि प्रश्न एक webpack सेटअप का सुझाव देता है, पूर्णता के लिए, Babel CLI का उपयोग करके बिना webpack के मैनुअल कम्पाइलेशन का विचार करें। पहले, `babel-cli@6.13.0` इंस्टॉल करें:

```bash
npm install --save-dev "babel-cli@6.13.0"
```

फिर, फ़ाइलें कम्पाइल करें:

```bash
./node_modules/.bin/babel src/index.js -o lib/index.js
```

या, `package.json` स्क्रिप्ट में जोड़ें:

```json
"scripts": {
  "build": "babel src --out-dir lib"
}
```

`npm run build` चलाएं, जो कम्पाइल किए गए फ़ाइलें `lib/` में आउटपुट करता है, जो Node.js प्रोजेक्टों या सर्वर-साइड कोड के लिए उपयोगी है।

#### संगतता और विचार
संस्करणों (सभी Babel 6 के आसपास, वर्षों पहले जारी) के आधार पर, संगतता आमतौर पर उसी मुख्य संस्करण के भीतर सुनिश्चित होती है। फिर भी, ध्यान दें:

- `babel-loader@6.2.5` webpack 1 और 2 के साथ काम करता है, इसलिए सुनिश्चित करें कि आपका webpack संस्करण (जैसे, `webpack@1.14.0`) संगत है।
- इन पैकेजों को Babel 7 के लिए वर्जित कर दिया गया है, जो स्कोप्ड पैकेजों जैसे `@babel/core` का उपयोग करता है। लंबे समय तक के प्रोजेक्ट के लिए अपग्रेड करने का विचार करें, लेकिन पुराने प्रोजेक्टों के लिए ये ठीक काम करते हैं।
- नाम संघर्षों की सुनिश्चितता सुनिश्चित करें; उपयोगकर्ता की सूची में स्पेस थे (जैसे, "babel loader"), संभवतः टाइपो, जो हाइफन (जैसे, `babel-loader`) में सुधार दिया गया है।

#### संभावित गलतियाँ
- सुनिश्चित करें कि `.babelrc` प्रोजेक्ट रूट में है, या webpack विकल्पों में पथ निर्दिष्ट करें।
- `babel-runtime` इंस्टॉल हो गया है, क्योंकि `transform-runtime` इसे हेल्पर के लिए निर्भर करता है।
- इंस्टॉलेशन के दौरान वर्जित चेतावनी की जांच करें, लेकिन इन संस्करणों के उम्र के कारण, वे उम्मीद के अनुसार काम करेंगे।

#### टेबल: पैकेज भूमिकाएँ और संस्करण

| पैकेज नाम                  | संस्करण | भूमिका                                                                 |
|-------------------------------|---------|----------------------------------------------------------------------|
| `babel-core`                  | 6.13.0  | Babel के लिए कोर कम्पाइलेशन कार्यक्षमता।                            |
| `babel-loader`                | 6.2.5   | Webpack लोडर के लिए Babel, JavaScript फ़ाइलें कम्पाइल करता है।                  |
| `babel-plugin-transform-runtime` | 6.12.0 | प्लगइन कोड डुप्लिकेशन को कम करने के लिए हेल्पर को बाहरीकरण करता है।             |
| `babel-preset-es2015`         | 6.13.2  | ES2015 से ES5 में परिवर्तन के लिए प्रिसेट, आधुनिक सिंटैक्स को सक्षम करता है।      |
| `babel-runtime`               | 6.11.6  | `transform-runtime` प्लगइन के लिए हेल्पर प्रदान करने वाला रनटाइम लाइब्रेरी।     |

यह टेबल प्रत्येक पैकेज की भूमिका को सारांशित करता है, सेटअप में उनकी उद्देश्य को समझने में मदद करता है।

#### अनपेक्षित विवरण: पुराने संदर्भ
एक रोचक पहलू यह है कि ये Babel 6 संस्करण हैं, अब वर्जित, Babel 7 में स्कोप्ड पैकेज (जैसे, `@babel/core`) का परिचय देते हैं। यह सेटअप पुराने प्रोजेक्टों के लिए आम है, और जबकि कार्यक्षम, Babel 7 में बेहतर TypeScript समर्थन या प्रदर्शन ऑप्टिमाइज़ेशन जैसे आधुनिक विशेषताओं से वंचित हो सकते हैं, जो वर्तमान मानकों के लिए उपयोगकर्ताओं के लिए अनपेक्षित हो सकते हैं।

#### निष्कर्ष
यह मार्गदर्शन निर्दिष्ट Babel पैकेजों का सेटअप एक webpack आधारित फ्रंट-एंड प्रोजेक्ट के लिए कवर करता है, Babel CLI के लिए एक विकल्प के साथ। संस्करणों के आधार पर, webpack 1 या 2 के साथ संगतता सुनिश्चित करें, और वर्जित होने की चेतावनी के बारे में जानें, लेकिन उद्देश्य के लिए, यह बिना किसी समस्या के काम करेगा। और अधिक विवरण के लिए, दिए गए दस्तावेज़ लिंक पर संदर्भ करें।
