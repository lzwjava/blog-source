एक शक्तिशाली कोड असिस्टेंट जैसे Claude Code में सामान्य **टूल कॉल** या **एक्शन** एक वास्तविक कोडिंग वातावरण के साथ इंटरैक्ट करने और एक मानव डेवलपर जैसे कार्यों को करने पर केंद्रित होते हैं।

ये टूल कॉल मुख्य रूप से कुछ श्रेणियों में आते हैं:

### 1. फ़ाइल सिस्टम और कोडबेस इंटरैक्शन

ये असिस्टेंट के लिए प्रोजेक्ट को समझने, मौजूदा कोड को पढ़ने और बदलाव करने के लिए आवश्यक हैं।

| टूल/एक्शन | सामान्य उपयोग |
| :--- | :--- |
| **`read_file`** (या `@filename` जैसा फ़ाइल रेफरेंस) | किसी विशिष्ट फ़ाइल की लॉजिक, डिपेंडेंसी या कॉन्टेक्स्ट को समझने के लिए उसकी सामग्री प्राप्त करना। |
| **`list_directory`** | फ़ाइल संरचना या डायरेक्टरी में फ़ाइलों की सूची प्राप्त करना ताकि प्रासंगिक मॉड्यूल की पहचान की जा सके या कोई विशिष्ट फ़ाइल ढूंढी जा सके। |
| **`edit_file` / `write_file`** | कोड में सॉल्यूशन इम्प्लीमेंट करने, रिफैक्टरिंग करने, फीचर जोड़ने या बग ठीक करने के लिए मुख्य एक्शन। |
| **`create_file`** | नई फाइलें लिखना, जैसे कि एक नई टेस्ट फ़ाइल, कॉन्फ़िगरेशन फ़ाइल, या एक नया कंपोनेंट। |
| **`search_files`** | कोडबेस में उन सभी फाइलों को ढूंढना जिनमें कोई विशिष्ट स्ट्रिंग हो (जैसे, फंक्शन नाम, क्लास नाम, या एरर मैसेज)। |

### 2. एक्सेक्यूशन और डीबगिंग

अपने काम को सत्यापित करने, एरर ठीक करने और रीयल-टाइम फीडबैक पाने के लिए, असिस्टेंट को कमांड रन करने की आवश्यकता होती है।

| टूल/एक्शन | सामान्य उपयोग |
| :--- | :--- |
| **`bash` / `run_command`** | शेल कमांड एक्सेक्यूट करना जैसे बिल्ड चलाना (`npm build`), टेस्ट चलाना (`pytest`, `npm test`), कोड लिंटिंग करना (`eslint`), या कोई स्क्रिप्ट चलाना। |
| **`code_interpreter`** | किसी चीज़ की त्वरित गणना करने, लाइब्रेरी फंक्शन टेस्ट करने या छोटे डेटासेट का विश्लेषण करने के लिए सैंडबॉक्स्ड वातावरण में कोड का एक छोटा स्निपेट चलाना। |

### 3. वर्जन कंट्रोल (Git)

एक एजेंटिक कोडिंग असिस्टेंट Git के साथ इंटरैक्ट करके डेवलपमेंट वर्कफ़्लो को मैनेज कर सकता है।

| टूल/एक्शन | सामान्य उपयोग |
| :--- | :--- |
| **`git status`** | वर्किंग डायरेक्टरी की वर्तमान स्थिति की जांच करना। |
| **`git diff`** | अपने काम की समीक्षा करने के लिए किए गए बदलावों को देखना। |
| **`git commit`** | अक्सर जेनरेट किए गए कमिट मैसेज के साथ, रिपॉजिटरी में परिवर्तनों को सहेजना। |
| **`git checkout`** | ब्रांच स्विच करना या अनकमिटेड बदलावों को हटाना। |
| **`create_pull_request`** | किसी फीचर या फिक्स के अंतिम चरण को ऑटोमेट करना। |

### 4. एक्सटर्नल इनफॉर्मेशन

अप-टू-डेट जानकारी, डॉक्युमेंटेशन, या बाहरी सेवाओं से कनेक्ट करने के लिए, विशेष टूल्स का उपयोग किया जाता है।

| टूल/एक्शन | सामान्य उपयोग |
| :--- | :--- |
| **`web_search`** | नवीनतम सिंटैक्स देखने, आधिकारिक API डॉक्युमेंटेशन पढ़ने, या किसी विशिष्ट एरर के लिए ट्रबलशूटिंग गाइड ढूंढने के लिए। |
| **कस्टम API / MCP सर्वर** | मॉडल कॉन्टेक्स्ट प्रोटोकॉल (MCP) का उपयोग करके बाहरी सेवाओं से कनेक्ट करना, जैसे कि Google Drive से डेटा फ़ेच करना, डेटाबेस (PostgreSQL) के साथ इंटरैक्ट करना, या कम्युनिकेशन टूल (Slack) को मैसेज भेजना। |

संक्षेप में, ये "टूल कॉल" लार्ज लैंग्वेज मॉडल (LLM) को एक टेक्स्ट जनरेटर से एक पूर्ण **एजेंट** में बदलने की अनुमति देते हैं जो अपने वातावरण का निरीक्षण कर सकता है (फाइलें पढ़ना, कमांड चलाना), सोच सकता है (काम की योजना बनाना) और कार्य कर सकता है (फाइलें एडिट करना, कोड कमिट करना)।

क्या आप एक कोड असिस्टेंट द्वारा एक साधारण बग को ठीक करने के लिए इन टूल्स का उपयोग करने का चरण-दर-चरण उदाहरण देखना चाहेंगे?
