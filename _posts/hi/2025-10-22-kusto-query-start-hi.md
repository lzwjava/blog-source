### KQL (Kusto क्वेरी लैंग्वेज) के साथ Azure में शुरुआत करना

KQL एक शक्तिशाली, रीड-ओनली क्वेरी लैंग्वेज है जिसे Azure Data Explorer, Azure Monitor (Log Analytics), और Microsoft Sentinel जैसी Azure सेवाओं में बड़ी मात्रा में स्ट्रक्चर्ड, सेमी-स्ट्रक्चर्ड और अनस्ट्रक्चर्ड डेटा के एनालिसिस के लिए डिज़ाइन किया गया है। यह इंट्यूटिव, पाइप-आधारित (डेटा फ्लो के लिए `|` का उपयोग करता है) और लॉग एनालिसिस, टेलीमेट्री और थ्रेट डिटेक्शन जैसे बिग डेटा सिनेरियो के लिए ऑप्टिमाइज़्ड है। नीचे इसके उपयोग के लिए एक स्टेप-बाय-स्टेप गाइड दी गई है।

#### पूर्वापेक्षाएँ
- एक एक्टिव Azure सब्सक्रिप्शन जिसमें किसी प्रासंगिक सेवा (जैसे, Azure Data Explorer क्लस्टर, Log Analytics वर्कस्पेस, या Sentinel इंस्टेंस) तक पहुंच हो।
- उचित अनुमतियाँ: न्यूनतम, डेटाबेस, टेबल या वर्कस्पेस तक रीड एक्सेस।
- डेटा कॉन्सेप्ट्स (जैसे टेबल और फ़िल्टरिंग) की बेसिक समझ मददगार है, लेकिन KQL शुरुआत करने वालों के लिए अनुकूल है।
- वैकल्पिक: त्वरित शुरुआत के लिए Azure Data Explorer ऐप इंस्टॉल करें या वेब UI का उपयोग करें—शुरुआत में कोडिंग एनवायरनमेंट की आवश्यकता नहीं है।

#### चरण 1: चुनें कि आपकी क्वेरीज़ कहाँ चलेंगी
KQL कई Azure सेवाओं में चलती है। अपने डेटा स्रोत के अनुकूल एक से शुरुआत करें:
- **Azure Data Explorer**: बिग डेटा एक्सप्लोरेशन के लिए आदर्श। वेब UI पर [dataexplorer.azure.com](https://dataexplorer.azure.com/) पर एक्सेस करें। एक क्लस्टर और डेटाबेस चुनें, फिर क्वेरी एडिटर खोलें।
- **Azure Monitor / Log Analytics**: लॉग्स और मेट्रिक्स के लिए। Azure पोर्टल (portal.azure.com) में, **Monitor > Logs** पर जाएं, एक वर्कस्पेस चुनें, और क्वेरी एडिटर का उपयोग करें।
- **Microsoft Sentinel**: सिक्योरिटी एनालिटिक्स के लिए। Azure पोर्टल में, अपने वर्कस्पेस में **Microsoft Sentinel > Logs** पर नेविगेट करें।
- **अन्य विकल्प**: Microsoft Fabric (KQL क्वेरी एडिटर के माध्यम से) या विज़ुअलाइज़ेशन के लिए Power BI जैसे टूल्स के साथ इंटीग्रेट करें।

डेटा एक हायरार्की में ऑर्गनाइज़ होता है: डेटाबेस > टेबल > कॉलम। क्वेरीज़ रीड-ओनली होती हैं; स्कीमा परिवर्तनों के लिए मैनेजमेंट कमांड्स (`.` से शुरू) का उपयोग करें।

#### चरण 2: बेसिक सिंटैक्स समझें
KQL क्वेरीज़ प्लेन-टेक्स्ट स्टेटमेंट्स होती हैं जो सेमीकोलन (`;`) से अलग की जाती हैं। वे एक डेटा-फ्लो मॉडल का उपयोग करती हैं:
- एक टेबल नाम (जैसे `StormEvents`) से शुरू करें।
- डेटा को फ़िल्टरिंग, एग्रीगेशन आदि के लिए ऑपरेटर्स के माध्यम से पाइप (`|`) करें।
- `count` या `summarize` जैसे आउटपुट के साथ समाप्त करें।
- नाम/ऑपरेटर्स के लिए केस-सेंसिटिव; आवश्यकता पड़ने पर कीवर्ड को `['keyword']` में एनक्लोज़ करें।

एक सरल क्वेरी स्ट्रक्चर:
```
TableName
| where Condition
| summarize Count = count() by GroupByColumn
```

मैनेजमेंट कमांड्स (क्वेरीज़ नहीं) `.` से शुरू होती हैं (जैसे, टेबल्स लिस्ट करने के लिए `.show tables`)।

#### चरण 3: अपनी पहली क्वेरी लिखें और रन करें
1. अपनी चुनी हुई सेवा (जैसे, Azure Data Explorer वेब UI) में क्वेरी एडिटर खोलें।
2. एक बेसिक क्वेरी दर्ज करें। सैंपल डेटा (StormEvents टेबल, अधिकांश एनवायरनमेंट में उपलब्ध) का उपयोग करते हुए उदाहरण:
   ```
   StormEvents
   | where StartTime between (datetime(2007-11-01) .. datetime(2007-12-01))
   | where State == "FLORIDA"
   | count
   ```
   - यह नवंबर 2007 के लिए फ्लोरिडा में तूफानों को फ़िल्टर करता है और काउंट (जैसे, 28) रिटर्न करता है।
3. एक्ज़िक्यूट करने के लिए **Run** पर क्लिक करें। परिणाम एक टेबल के रूप में दिखाई देते हैं; चार्ट के रूप में विज़ुअलाइज़ करने या एक्सपोर्ट करने के लिए UI का उपयोग करें।
4. इटरेट करें: `project` (कॉलम चुनें), `summarize` (एग्रीगेट), या `extend` (कम्प्यूटेड कॉलम जोड़ें) जैसे ऑपरेटर्स जोड़ें।

पहले सीखने के लिए सामान्य ऑपरेटर्स:
- `where`: पंक्तियों को फ़िल्टर करें (जैसे, `| where Level == "Error"`)।
- `summarize`: एग्रीगेट करें (जैसे, `| summarize avg(Duration) by User`)।
- `join`: टेबल्स को कंबाइन करें।
- `search`: टेबल्स में फुल-टेक्स्ट सर्च करें।

#### चरण 4: एडवांस्ड फीचर्स और बेस्ट प्रैक्टिसेज एक्सप्लोर करें
- **टाइम-सीरीज़ और एनालिटिक्स**: ट्रेंड्स के लिए `make-series` या आउटलायर्स के लिए `detect-anomalies` जैसे फंक्शन्स का उपयोग करें।
- **विज़ुअलाइज़ेशन**: एडिटर में, ग्रिड/चार्ट व्यू पर स्विच करें। डैशबोर्ड के लिए Power BI में एक्सपोर्ट करें।
- **परफॉर्मेंस टिप्स**: ऑपरेटर्स को लॉजिकली ऑर्डर करें (जल्दी फ़िल्टर करें), `take 100` के साथ परिणाम सीमित करें, और छोटे डेटासेट पर टेस्ट करें।
- **सामान्य उपयोग के मामले**:
  - **लॉग एनालिसिस**: ऐप्स में एरर क्वेरी करें (`| where Level == "Error" | take 50`)।
  - **सिक्योरिटी हंटिंग**: Sentinel में, थ्रेट्स के लिए सर्च करें (`search "suspicious login"`)।
  - **डेटा एक्सप्लोरेशन**: मेट्रिक्स को सारांशित करें (`| summarize count() by bin(TimeGenerated, 1h)`)।

#### समस्या निवारण
- एरर? एडिटर में सिंटैक्स चेक करें (यह समस्याओं को हाइलाइट करता है)। सुनिश्चित करें कि टेबल/कॉलम नाम बिल्कुल मेल खाते हों।
- कोई डेटा नहीं? सत्यापित करें कि आपके वर्कस्पेस में लॉग इन्जेस्टेड हैं या सैंपल डेटासेट का उपयोग करें।
- Azure पोर्टल या डॉक्स में इंटरैक्टिव ट्यूटोरियल के माध्यम से अधिक सीखें।

हैंड्स-ऑन प्रैक्टिस के लिए, Azure Data Explorer में सैंपल डेटा लोड करें और प्रयोग करें।
