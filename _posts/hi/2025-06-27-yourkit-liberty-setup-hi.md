विंडोज़ पर वेबस्फीयर लिबर्टी सर्वर के साथ योरकिट जावा प्रोफाइलर का उपयोग करने में योरकिट एजेंट को वेबस्फीयर लिबर्टी JVM में एकीकृत करना और प्रोफाइलिंग के लिए इसे कॉन्फ़िगर करना शामिल है। योरकिट और वेबस्फीयर लिबर्टी एकीकरण के लिए उपलब्ध जानकारी और सर्वोत्तम प्रथाओं के आधार पर, इसे सेट अप करने के चरण नीचे दिए गए हैं:

### पूर्वापेक्षाएँ
- **योरकिट जावा प्रोफाइलर**: आपके विंडोज़ मशीन पर इंस्टॉल किया गया। [योरकिट की आधिकारिक साइट](https://www.yourkit.com/) से डाउनलोड करें और सुनिश्चित करें कि आपके पास एक वैध लाइसेंस या ट्रायल कुंजी है।
- **वेबस्फीयर लिबर्टी**: आपके विंडोज़ सिस्टम पर इंस्टॉल और चल रहा है। सुनिश्चित करें कि आपके पास सर्वर कॉन्फ़िगरेशन फ़ाइलों तक प्रशासनिक पहुंच है।
- **जावा JDK**: वेबस्फीयर लिबर्टी एक जावा रनटाइम (IBM JDK या OpenJDK) का उपयोग करता है। पुष्टि करें कि JDK संस्करण योरकिट के साथ संगत है (योरकिट जावा 5 और बाद के संस्करणों को सपोर्ट करता है, लेकिन अपने विशिष्ट संस्करण के साथ संगतता सत्यापित करें)।
- **प्रशासनिक विशेषाधिकार**: वेबस्फीयर लिबर्टी कॉन्फ़िगरेशन फ़ाइलों को संशोधित करने के लिए आवश्यक।

### चरण-दर-चरण मार्गदर्शिका

1. **योरकिट जावा प्रोफाइलर इंस्टॉल करें**
   - [योरकिट की वेबसाइट](https://www.yourkit.com/download) से विंडोज़ के लिए योरकिट जावा प्रोफाइलर डाउनलोड और इंस्टॉल करें।
   - इंस्टॉलेशन डायरेक्टरी नोट करें, क्योंकि आपको योरकिट एजेंट लाइब्रेरी (`yjpagent.dll`) के पथ की आवश्यकता होगी।

2. **योरकिट एजेंट का स्थान ढूंढें**
   - विंडोज़ के लिए योरकिट एजेंट आमतौर पर इस स्थान पर होता है:
     ```
     C:\Program Files\YourKit-Java-Profiler-<version>\bin\win64\yjpagent.dll
     ```
     (यदि 32-बिट JVM चला रहे हैं तो `win64` के बजाय `win32` का उपयोग करें।)
   - सुनिश्चित करें कि एजेंट वेबस्फीयर लिबर्टी द्वारा उपयोग किए जाने वाले JVM आर्किटेक्चर (32-बिट या 64-बिट) से मेल खाता है।

3. **योरकिट एजेंट का उपयोग करने के लिए वेबस्फीयर लिबर्टी को कॉन्फ़िगर करें**
   - **`jvm.options` फ़ाइल का स्थान ढूंढें**:
     - अपनी वेबस्फीयर लिबर्टी सर्वर की कॉन्फ़िगरेशन डायरेक्टरी में नेविगेट करें, आमतौर पर:
       ```
       <LIBERTY_INSTALL_DIR>\usr\servers\<server_name>\jvm.options
       ```
       `<LIBERTY_INSTALL_DIR>` को आपके वेबस्फीयर लिबर्टी इंस्टॉलेशन के पथ (जैसे, `C:\wlp`) से बदलें, और `<server_name>` को आपके सर्वर के नाम (जैसे, `defaultServer`) से बदलें।
     - यदि `jvm.options` फ़ाइल मौजूद नहीं है, तो इसे सर्वर डायरेक्टरी में बनाएं।
   - **योरकिट एजेंट पथ जोड़ें**:
     - प्रशासनिक विशेषाधिकारों के साथ एक टेक्स्ट एडिटर में `jvm.options` खोलें।
     - योरकिट एजेंट को शामिल करने के लिए निम्नलिखित पंक्ति जोड़ें:
       ```
       -agentpath:C:\Program Files\YourKit-Java-Profiler-<version>\bin\win64\yjpagent.dll=disablestacktelemetry,disableexceptiontelemetry,delay=10000,probe_disable=*,sessionname=WebSphereLiberty
       ```
       - `<version>` को आपके योरकिट संस्करण (जैसे, `2023.9`) से बदलें।
       - विकल्प (`disablestacktelemetry`, `disableexceptiontelemetry`, `probe_disable=*`) अनावश्यक टेलीमेट्री को अक्षम करके ओवरहेड कम करते हैं। `delay=10000` सुनिश्चित करता है कि एजेंट सर्वर के इनिशियलाइज़ होने के बाद शुरू हो, और `sessionname=WebSphereLiberty` प्रोफाइलिंग सत्र की पहचान करता है।
       - उदाहरण:
         ```
         -agentpath:C:\Program Files\YourKit-Java-Profiler-2023.9\bin\win64\yjpagent.dll=disablestacktelemetry,disableexceptiontelemetry,delay=10000,probe_disable=*,sessionname=WebSphereLiberty
         ```
   - **फ़ाइल सहेजें**: सुनिश्चित करें कि `jvm.options` फ़ाइल के लिए आपके पास लिखने की अनुमति है।

4. **JVM संगतता सत्यापित करें**
   - वेबस्फीयर लिबर्टी अक्सर IBM JDK या OpenJDK का उपयोग करता है। योरकिट दोनों के साथ संगत है, लेकिन यदि आपको कोई समस्या आती है (जैसे, कुछ IBM JDK मामलों में नोट किया गया `NoSuchMethodError`), तो IBM JDK के साथ संघर्ष करने वाले प्रोब को अक्षम करने के लिए एजेंट पथ में `probe_disable=*` जोड़ें।[](https://www.yourkit.com/forum/viewtopic.php?f=3&t=8042)
   - लिबर्टी द्वारा उपयोग किए जाने वाले जावा संस्करण की जांच करें:
     ```
     <LIBERTY_INSTALL_DIR>\java\bin\java -version
     ```
     सुनिश्चित करें कि यह योरकिट द्वारा समर्थित है (पुराने संस्करणों के लिए जावा 5 या बाद का; आधुनिक संस्करण जावा 8+ का समर्थन करते हैं)।

5. **वेबस्फीयर लिबर्टी शुरू करें**
   - अपने वेबस्फीयर लिबर्टी सर्वर को हमेशा की तरह शुरू करें:
     ```
     <LIBERTY_INSTALL_DIR>\bin\server start <server_name>
     ```
     उदाहरण:
     ```
     C:\wlp\bin\server start defaultServer
     ```
   - योरकिट एजेंट से संबंधित किसी भी त्रुटि के लिए सर्वर लॉग (`<LIBERTY_INSTALL_DIR>\usr\servers\<server_name>\logs\console.log` या `messages.log`) की जांच करें।
   - योरकिट एजेंट लॉग इसमें देखें:
     ```
     %USERPROFILE%\.yjp\log\<session_name>-<pid>.log
     ```
     उदाहरण:
     ```
     C:\Users\<YourUsername>\.yjp\log\WebSphereLiberty-<pid>.log
     ```
     लॉग में संकेत मिलना चाहिए कि एजेंट लोड हो गया है और एक पोर्ट (डिफ़ॉल्ट: 10001) पर सुन रहा है:
     ```
     Profiler agent is listening on port 10001
     ```

6. **योरकिट प्रोफाइलर UI से कनेक्ट करें**
   - अपने विंडोज़ मशीन पर योरकिट जावा प्रोफाइलर UI लॉन्च करें।
   - योरकिट UI में, **Profile | Profile Local Java Server or Application** या **Profile | Profile Remote Java Server or Application** चुनें।
     - स्थानीय प्रोफाइलिंग के लिए (चूंकि योरकिट और लिबर्टी एक ही मशीन पर हैं), **Profile Local Java Server or Application** चुनें।
     - UI को वेबस्फीयर लिबर्टी प्रक्रिया (`sessionname=WebSphereLiberty` द्वारा पहचानी गई) का स्वचालित रूप से पता लगाना चाहिए।
   - यदि स्वचालित रूप से पता नहीं चलता है, तो **Profile Remote Java Server or Application** का उपयोग करें, **Direct Connect** चुनें, और दर्ज करें:
     - **Host**: `localhost`
     - **Port**: `10001` (या एजेंट लॉग में निर्दिष्ट पोर्ट)।
   - सर्वर से कनेक्ट करें। UI CPU, मेमोरी और थ्रेड टेलीमेट्री प्रदर्शित करेगा।

7. **एप्लिकेशन प्रोफाइल करें**
   - योरकिट UI का उपयोग करके:
     - **CPU प्रोफाइलिंग**: प्रदर्शन बॉटलनेक की पहचान करने के लिए CPU सैंपलिंग या ट्रेसिंग सक्षम करें। उच्च-लोड सिस्टम के लिए "Profile J2EE" सक्षम करने से बचें ताकि ओवरहेड कम हो सके।[](https://www.jahia.com/blog/analyzing-system-performance-with-yourkit-java-profiler)
     - **मेमोरी प्रोफाइलिंग**: हीप उपयोग का विश्लेषण करें और ऑब्जेक्ट्स को वेब एप्लिकेशन द्वारा समूहित करके मेमोरी लीक का पता लगाएं (लिबर्टी-होस्टेड ऐप्स के लिए उपयोगी)।[](https://www.yourkit.com/docs/java-profiler/latest/help/webapps.jsp)
     - **थ्रेड विश्लेषण**: थ्रेड्स टैब में डेडलॉक या फ्रोजन थ्रेड्स की जांच करें।[](https://www.yourkit.com/changes/)
   - यदि आवश्यक हो तो ऑफ़लाइन विश्लेषण के लिए स्नैपशॉट लें (File | Save Snapshot)।
   - मेमोरी उपयोग की निगरानी करें, क्योंकि प्रोफाइलिंग मेमोरी खपत बढ़ा सकती है। बिना निगरानी के लंबे प्रोफाइलिंग सत्रों से बचें।[](https://www.jahia.com/blog/analyzing-system-performance-with-yourkit-java-profiler)

8. **समस्या निवारण**
   - **सर्वर शुरू होने में विफल हो जाता है या अनुपलब्ध हो जाता है**:
     - `OutOfMemoryError` या `NoSuchMethodError` जैसी त्रुटियों के लिए लॉग (`console.log`, `messages.log`, और योरकिट एजेंट लॉग) की जांच करें।[](https://www.yourkit.com/forum/viewtopic.php?t=328)[](https://www.yourkit.com/forum/viewtopic.php?t=16205)
     - सुनिश्चित करें कि `-agentpath` सही `jvm.options` फ़ाइल में जोड़ा गया है और लिबर्टी शुरू करने के लिए उपयोग किए गए स्क्रिप्ट से मेल खाता है।[](https://www.yourkit.com/forum/viewtopic.php?t=16205)
     - यदि IBM JDK का उपयोग कर रहे हैं, तो संघर्षों से बचने के लिए एजेंट पथ में `probe_disable=*` जोड़ने का प्रयास करें।[](https://www.yourkit.com/forum/viewtopic.php?f=3&t=8042)
   - **ClassNotFoundException**:
     - यदि आपको `java.lang.ClassNotFoundException` जैसी त्रुटियाँ दिखाई देती हैं (जैसे, `java.util.ServiceLoader` के लिए), सुनिश्चित करें कि योरकिट एजेंट संस्करण आपके JDK के साथ संगत है। पुराने IBM JDKs (जैसे, Java 5) के लिए, योरकिट 8.0 या पहले का उपयोग करें।[](https://www.yourkit.com/forum/viewtopic.php?f=3&t=7149)
   - **कोई प्रोफाइलिंग डेटा नहीं**:
     - सत्यापित करें कि योरकिट एजेंट और UI संस्करण मेल खाते हैं। बेमेल संस्करण कनेक्शन समस्याएं पैदा कर सकते हैं।[](https://www.yourkit.com/forum/viewtopic.php?t=328)
     - सुनिश्चित करें कि सर्वर ब्राउज़र के माध्यम से सुलभ है (जैसे, `https://localhost:9443` यदि SSL का उपयोग कर रहे हैं)। यदि नहीं, तो फ़ायरवॉल सेटिंग्स या SSL कॉन्फ़िगरेशन की जांच करें।[](https://www.yourkit.com/forum/viewtopic.php?t=16205)
   - **लॉग फ़ाइल समस्याएं**:
     - यदि `~/.yjp/log/` में कोई योरकिट लॉग नहीं बनता है, तो सुनिश्चित करें कि प्रक्रिया के पास उपयोगकर्ता की होम डायरेक्टरी में लिखने की अनुमति है।[](https://www.yourkit.com/forum/viewtopic.php?f=3&t=3219)
   - **प्रदर्शन प्रभाव**:
     - प्रोफाइलिंग प्रदर्शन को प्रभावित कर सकती है। प्रोडक्शन-जैसे वातावरण के लिए न्यूनतम सेटिंग्स (जैसे, स्टैक टेलीमेट्री अक्षम करें) का उपयोग करें।[](https://www.jahia.com/blog/analyzing-system-performance-with-yourkit-java-profiler)

9. **वैकल्पिक: योरकिट इंटीग्रेशन विज़ार्ड का उपयोग करें**
   - योरकिट कॉन्फ़िगरेशन को सरल बनाने के लिए एक जावा सर्वर इंटीग्रेशन विज़ार्ड प्रदान करता है:
     - योरकिट UI लॉन्च करें और **Profile | Profile Local Java Server or Application** चुनें।
     - समर्थित सर्वरों की सूची से **WebSphere Liberty** चुनें (या "Other Java application" यदि लिबर्टी सूचीबद्ध नहीं है)।
     - आवश्यक `-agentpath` सेटिंग्स उत्पन्न करने और `jvm.options` को अपडेट करने के लिए विज़ार्ड का पालन करें। सुनिश्चित करें कि आपके पास कॉन्फ़िगरेशन फ़ाइलों के लिए लिखने की अनुमति है।[](https://www.yourkit.com/docs/java-profiler/latest/help/java-server-integration-wizard.jsp)
   - सही पथ और सेटिंग्स सुनिश्चित करने के लिए यह विशेष रूप से उपयोगी है।

10. **प्रोफाइलिंग बंद करें**
    - प्रोफाइलिंग अक्षम करने के लिए, `jvm.options` में `-agentpath` पंक्ति को हटा दें या कमेंट कर दें और सर्वर को पुनरारंभ करें।
    - वैकल्पिक रूप से, सर्वर बंद करें:
      ```
      <LIBERTY_INSTALL_DIR>\bin\server stop <server_name>
      ```

### अतिरिक्त नोट्स
- **लाइसेंस**: सर्वर पर योरकिट एजेंट के लिए किसी लाइसेंस कुंजी की आवश्यकता नहीं है; लाइसेंस योरकिट UI में लागू होता है। किसी अन्य विंडोज़ मशीन से रिमोट प्रोफाइलिंग के लिए, सुनिश्चित करें कि UI के पास एक वैध लाइसेंस है।[](https://www.yourkit.com/forum/viewtopic.php?f=3&t=11385)[](https://www.yourkit.com/forum/viewtopic.php?t=11385)
- **रिमोट प्रोफाइलिंग**: यदि किसी अलग मशीन से प्रोफाइलिंग कर रहे हैं, तो एजेंट पोर्ट (डिफ़ॉल्ट: 10001) तक नेटवर्क कनेक्टिविटी सुनिश्चित करें और सर्वर के होस्टनाम/IP के साथ **Profile Remote Java Server or Application** विकल्प का उपयोग करें।
- **प्रदर्शन विचार**: प्रोफाइलिंग मेमोरी और CPU उपयोग बढ़ा सकती है। प्रोडक्शन सिस्टम के लिए, न्यूनतम प्रोफाइलिंग विकल्पों (जैसे, CPU सैंपलिंग) के साथ शुरू करें और सर्वर स्वास्थ्य की निगरानी करें।[](https://www.jahia.com/blog/analyzing-system-performance-with-yourkit-java-profiler)
- **संगतता**: यदि IBM JDK 1.4.2 या 1.5 के साथ कोई पुराना वेबस्फीयर लिबर्टी संस्करण उपयोग कर रहे हैं, तो आपको पुराने योरकिट संस्करण (जैसे, Java 5 के लिए 8.0) की आवश्यकता हो सकती है।[](https://www.yourkit.com/forum/viewtopic.php?f=3&t=7149)[](https://stackoverflow.com/questions/61795/java-profiler-for-ibm-jvm-1-4-2-websphere-6-0-2)
- **प्रलेखन**: विस्तृत प्रोफाइलिंग विकल्पों के लिए योरकिट के आधिकारिक प्रलेखन देखें: [योरकिट जावा प्रोफाइलर सहायता](https://www.yourkit.com/docs/java/help/)।[](https://www.jahia.com/blog/analyzing-system-performance-with-yourkit-java-profiler)

```
-Xmx4G
-Xms4G
-agentpath:C:\Program Files\YourKit-Java-Profiler-2023.9\bin\win64\yjpagent.dll=disablestacktelemetry,disableexceptiontelemetry,delay=10000,probe_disable=*,sessionname=WebSphereLiberty
```

### सेटअप का परीक्षण
- सर्वर शुरू करने के बाद, सुनिश्चित करने के लिए अपने एप्लिकेशन (जैसे, `https://localhost:9443`) तक पहुंचें कि यह चल रहा है।
- योरकिट UI में, सत्यापित करें कि टेलीमेट्री डेटा (CPU, मेमोरी, थ्रेड्स) प्रदर्शित हो रहा है।
- यदि कोई समस्या उत्पन्न होती है, तो त्रुटियों के लिए योरकिट एजेंट लॉग और लिबर्टी लॉग की जांच करें।

यदि आपको कोई विशिष्ट त्रुटि आती है या किसी विशेष चरण में सहायता की आवश्यकता है, तो मुझे बताएं, और मैं और सहायता प्रदान कर सकता हूँ!
