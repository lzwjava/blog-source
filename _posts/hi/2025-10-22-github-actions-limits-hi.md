### GitHub Actions उपयोग सीमाएँ

GitHub Actions की सीमाएँ आपकी योजना (निजी खाते या संगठन) पर आधारित हैं। इनमें निजी रिपॉजिटरीज में GitHub-होस्टेड रनरों के लिए मुफ्त मासिक मिनट और आर्टिफैक्ट्स के लिए स्टोरेज शामिल हैं। सार्वजनिक रिपॉजिटरीज और सेल्फ-होस्टेड रनर इन मिनटों का उपयोग नहीं करते हैं। मिनटों की सीमा हर महीने रीसेट होती है (स्टोरेज चलता रहता है)। यदि आपके पास एक वैध भुगतान विधि है, तो अधिक उपयोग पर स्वचालित रूप से बिल लग जाता है; अन्यथा, सीमा पार होने पर जॉब्स ब्लॉक हो जाती हैं।

#### योजना के अनुसार शामिल मिनट और स्टोरेज

| योजना                          | स्टोरेज | मिनट (प्रति माह) |
|-------------------------------|---------|---------------------|
| GitHub Free (निजी/संगठन)   | 500 MB | 2,000              |
| GitHub Pro (निजी)        | 1 GB   | 3,000              |
| GitHub Team (संगठन)            | 2 GB   | 3,000              |
| GitHub Enterprise Cloud (संगठन)| 50 GB  | 50,000             |

- **मिनट**: GitHub-होस्टेड रनरों पर कुल जॉब रनटाइम गिना जाता है (विफल जॉब्स का आंशिक समय भी)। मल्टीप्लायर लागू होते हैं: Linux (1x), Windows (2x), macOS (10x)। रिपॉजिटरी में लिखने की पहुंच रखने वाला कोई भी व्यक्ति रिपॉजिटरी मालिक के भत्ते का उपयोग करता है।
- **स्टोरेज**: आर्टिफैक्ट स्टोरेज के GB-घंटों पर आधारित (जैसे, अपलोड/डाउनलोड)। लॉग और सारांश इसमें नहीं गिने जाते।

#### अधिक उपयोग बिलिंग
यदि आप कोटा पार कर जाते हैं:
- **मिनट**: प्रति अतिरिक्त मिनट शुल्क लगता है (मानक Linux रनरों के लिए USD दर: $0.008/मिनट; Windows/macOS के लिए मल्टीप्लायर के कारण अधिक)।
- **स्टोरेज**: GB-महीने के रूप में शुल्क लगता है (कुल GB-घंटे / 744 घंटे प्रति माह, राउंड अप)।
- उदाहरण: Team योजना पर, 5,000 अतिरिक्त मिनट (3,000 Linux + 2,000 Windows) की लागत ~$56 होगी।

भुगतान विधि के बिना, उपयोग कोटा पर रुक जाता है। आपने जो त्रुटि देखी ("हाल के खाते के भुगतान विफल रहे हैं या आपकी खर्च सीमा बढ़ाने की आवश्यकता है") इसका मतलब है कि या तो भुगतान विफल रहे हैं या आपकी **खर्च सीमा (बजट)** अधिक उपयोग को रोक रही है।

#### खर्च सीमाएँ (बजट)
- ये वैकल्पिक सीमाएँ हैं जो Actions (और अन्य मीटर्ड सुविधाओं) पर कुल मासिक खर्च को नियंत्रित करने के लिए होती हैं।
- यदि यह बहुत कम सेट है, तो यह कोटा पहुंचने से पहले ही जॉब्स को ब्लॉक कर देती है।
- **कैसे जांचें/बढ़ाएँ**:
  1. GitHub Settings > Billing & plans पर जाएं।
  2. "Budgets and alerts" के तहत, GitHub Actions के लिए अपना बजट देखें या संपादित करें।
  3. सीमा बढ़ाएँ, एक भुगतान विधि जोड़ें, या बजट को पूरी तरह से हटा दें (असीमित अधिक उपयोग, लेकिन अलर्ट के माध्यम से निगरानी करें)।
- माइग्रेट किए गए खातों में पुरानी सीमाओं से ऑटो-जेनरेटेड बजट हो सकते हैं—अपने उपयोग के अनुरूप उन्हें अपडेट करें।

पूरी जानकारी के लिए, रियल-टाइम उपयोग के लिए अपने खाते के बिलिंग डैशबोर्ड की जांच करें।
