### जैस्मीन-कोर
जैस्मीन-कोर, जैस्मीन (एक लोकप्रिय JavaScript टेस्टिंग फ्रेमवर्क) की कोर लाइब्रेरी है। यह बिहेवियर-ड्रिवन डेवलपमेंट (BDD) टेस्ट लिखने के लिए टूल्स प्रदान करती है, जिसमें टेस्ट सूट (describe ब्लॉक), अलग-अलग टेस्ट (it ब्लॉक), और असेर्शन के लिए मैचर्स (जैसे, expect().toBe()) जैसी विशेषताएं शामिल हैं। यह स्टैंडअलोन है और ब्राउज़र या Node.js एनवायरनमेंट में चल सकती है।

- **उद्देश्य**: डेवलपर्स को पठनीय, स्पेक-जैसे फॉर्मेट में JavaScript एप्लिकेशन के लिए यूनिट टेस्ट लिखने और चलाने में सक्षम बनाता है।
- **इंस्टालेशन**: आमतौर पर npm के माध्यम से (`npm install jasmine-core`)।
- **उपयोग उदाहरण**: आप टेस्ट्स को मैन्युअली सेट अप कर सकते हैं या कर्मा जैसे टूल्स के साथ इंटीग्रेट कर सकते हैं। यह ओपन-सोर्स है और GitHub (https://github.com/jasmine/jasmine) पर मेंटेन किया जाता है, जिसका नवीनतम वर्जन (मेरे अंतिम अपडेट के अनुसार) लगभग 5.x है।
- **प्रासंगिकता का सबूत**: यह कई JavaScript टेस्टिंग सेटअप के लिए एक मूलभूत डिपेंडेंसी है, जिसका उपयोग Angular और React एप्स जैसी प्रोजेक्ट्स में TDD/BDD प्रथाओं के लिए किया जाता है।

### कर्मा-जैस्मीन-एचटीएमएल-रिपोर्टर
कर्मा-जैस्मीन-एचटीएमएल-रिपोर्टर एक NPM पैकेज है जो कर्मा (JavaScript टेस्ट रनर) के लिए एक HTML-आधारित रिपोर्टर प्लगइन प्रदान करता है। यह जैस्मीन टेस्ट्स के साथ एक यूजर-फ्रेंडली वेब इंटरफेस में रिजल्ट्स दिखाकर इंटीग्रेट होता है, जो पास/फेल हुए टेस्ट्स, स्टैक ट्रेस, और टाइमिंग जानकारी दिखाता है, यह सब एक HTML पेज में रेंडर होता है जो टेस्ट एक्सेक्यूशन के दौरान रियल-टाइम में अपडेट होता रहता है।

- **उद्देश्य**: कंसोल आउटपुट के बजाय एक ब्राउज़र-आधारित HTML UI पेश करके टेस्ट फीडबैक को बेहतर बनाना, जिससे डेवलपर्स के लिए डीबगिंग आसान हो जाती है। यह टेस्ट लॉजिक के लिए jasmine-core पर और मल्टीपल ब्राउज़रों में टेस्ट चलाने के लिए karma पर निर्भर करता है।
- **इंस्टालेशन**: npm के माध्यम से (`npm install karma-jasmine-html-reporter --save-dev`)।
- **कॉन्फ़िगरेशन**: कर्मा के कॉन्फ़िग फाइल (जैसे, `karma.conf.js`) में एक प्लगइन और रिपोर्टर के रूप में जोड़ा जाता है। इसके लिए karma और karma-jasmine दोनों के सेट अप होने की आवश्यकता होती है।
- **प्रासंगिकता का सबूत**: यह कर्मा यूजर्स के लिए एक लोकप्रिय विकल्प है, जिसका डॉक्यूमेंटेशन ऑफिशियल कर्मा साइट (https://karma-runner.github.io) पर उपलब्ध है और इसे नियमित रूप से अपडेट किया जाता है। यह टेस्ट एक्सेक्यूशन को नहीं बदलता है बल्कि विज़ुअलाइज़ेशन को बेहतर बनाता है, जैसा कि कई डेव वर्कफ़्लो में देखा जाता है।

दोनों टूल्स अक्सर एक साथ उपयोग किए जाते हैं: jasmine-core टेस्टिंग लॉजिक को हैंडल करती है, जबकि karma-jasmine-html-reporter कर्मा-रन टेस्ट सूट में रिजल्ट्स को विज़ुअलाइज़ करती है। यदि आप JS टेस्टिंग सेट अप कर रहे हैं, तो इंटीग्रेशन के लिए कर्मा के डॉक्स से शुरुआत करें।
