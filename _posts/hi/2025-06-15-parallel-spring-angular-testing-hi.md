ठिक है—आप अपने Spring (Java) + Angular साइट को **Edge और Chrome में समानांतर रूप से**, मुख्य रूप से **मैन्युअल रूप से** (और थोड़ी सी हल्की ऑटोमेशन सहायता के साथ) टेस्ट करने का एक व्यावहारिक तरीका चाहते हैं। यहाँ एक व्यवहार-परीक्षित वर्कफ़्लो है जो चौड़ी टेबलों से बचता है।

---

# 1) स्वच्छ, तुलनीय ब्राउज़र सेट करें

* **नवीनतम स्थिर** Chrome और Edge इंस्टॉल करें। संस्करणों का ध्यान रखें।
* प्रत्येक के लिए टेस्टिंग हेतु **अलग-अलग स्वच्छ प्रोफाइल** बनाएँ:

  * Chrome: `chrome --user-data-dir=/tmp/chrome-qa --no-first-run`
  * Edge: `msedge --user-data-dir=/tmp/edge-qa --no-first-run`
* **ज़ूम (100%)**, **DPR**, **भाषा**, **OS थीम**, **फ़ॉन्ट पैक** मेल करें, और एक्सटेंशन बंद कर दें।
* दोनों ब्राउज़रों को साइड-बाय-साइड पिन करें (यदि संभव हो तो दो मॉनिटर पर)। समान **व्यूपोर्ट** (जैसे, 1440×900) का उपयोग करें।

---

# 2) एक स्थिर बैकएंड + यथार्थवादी डेटा तैयार करें

* अपने Spring बैकएंड को **स्टेजिंग मोड** में डिटरमिनिस्टिक सीड डेटा के साथ चलाएँ।
* **अपरिवर्तनीय टेस्ट अकाउंट** और एक **ज्ञात डेटासेट** (जैसे, db स्नैपशॉट के लिए Testcontainers या Flyway/Liquibase सीड स्क्रिप्ट) प्राथमिकता दें।
* अस्थिर डिपेंडेंसी के लिए, **WireMock** स्टब्स (HTTP) का उपयोग करें ताकि UI व्यवहार दोहराने योग्य हो।

---

# 3) ब्राउज़रों में इंटरैक्शन मिरर करें (मैन्युअल, लेकिन सिंक किया हुआ)

सचमुच समानांतर मैन्युअल टेस्टिंग के लिए, एक ब्राउज़र से दूसरे में क्लिक/स्क्रॉल/टाइपिंग को मिरर करें:

* **इंटरैक्शन सिंक** करने के लिए **Browsersync** को एक लोकल प्रॉक्सी के रूप में उपयोग करें:

  ```bash
  npm i -g browser-sync
  browser-sync start --proxy "http://localhost:4200" --files "dist/**/*" --open "external"
  ```

  प्रॉक्सी किए गए URL को **Chrome** और **Edge** में खोलें; स्क्रॉल, क्लिक और फॉर्म इनपुट मिरर होंगे।
  (लेआउट अंतर, हॉवर/फोकस चेक और त्वरित फ्लो के लिए बढ़िया।)

> यदि आप प्रॉक्सी नहीं कर सकते (ऑथ कंस्ट्रेंट, कॉर्पोरेट नेटवर्क), तो दो विंडो चलाएँ और एक सटीक **स्टेप लिस्ट** (नीचे) प्लस एक स्क्रीन रिकॉर्डर स्प्लिट-व्यू रखें।

# 4) क्रॉस-ब्राउज़र चेकलिस्ट (दोनों को एक साथ चलाएँ)

इसके माध्यम से **समानांतर रूप से** काम करें—अगले स्टेप पर जाने से पहले दोनों ब्राउज़र में एक ही स्टेप।

* **बूटस्ट्रैप और फ़ॉन्ट:** अनस्टाइल कंटेंट का फ्लैश (FOUC), आइकन फ़ॉन्ट, फॉलबैक फ़ॉन्ट।
* **लेआउट:** Flex/grid गैप, स्टिकी हेडर/फुटर, ओवरफ्लो/एलिप्सिस, RTL/L10n टेक्स्ट रैपिंग।
* **फॉर्म:** ऑटोफिल, प्लेसहोल्डर, वैलिडेशन मैसेज, नंबर/डेट इनपुट, IME/चीनी इनपुट, कॉपी/पेस्ट।
* **फोकस/कीबोर्ड:** टैब ऑर्डर, फोकस रिंग विजिबिलिटी, `:focus-visible` बनाम `:focus`, Enter/Esc बिहेवियर, शॉर्टकट।
* **हॉवर/एक्टिव:** मेन्यू, टूलटिप, रिपल इफेक्ट, Angular Material स्टेट क्लासेस।
* **फ़ाइल और डाउनलोड:** फ़ाइल इनपुट एक्सेप्ट फिल्टर, ड्रैग-एंड-ड्रॉप, डाउनलोड प्रॉम्प्ट।
* **ऑथ/सेशन:** कुकीज़, SameSite, टैब में स्टोरेज आइसोलेशन, सत्र टाइमआउट और रिफ्रेश टोकन फ्लो।
* **रूटिंग:** डीप लिंक, नेस्टेड रूट पर हार्ड रिफ्रेश, 404 फॉलबैक।
* **कैशिंग:** सर्विस वर्कर अपडेट साइकिल, स्टेल एसेट्स बस्टिंग, ऑफलाइन पेज बिहेवियर।
* **मीडिया और API:** getUserMedia/clipboard, नोटिफिकेशन परमिशन।
* **एक्सेसिबिलिटी क्विक पास:** लैंडमार्क/रोल, कलर कॉन्ट्रास्ट (DevTools), कीबोर्ड-ओनली नेव।
* **परफॉर्मेंस सैनिटी:** DevTools Performance, लॉन्ग टास्क चेक करें, और **दोनों** ब्राउज़र में Lighthouse।

टिप: **DevTools** को **दोनों** में **खुला रखें** (F12), नीचे डॉक किया हुआ, और **Console** वार्निंग (फ्रेमवर्क + CSP + डिप्रिकेशन मैसेज) की तुलना करें।

# 5) Angular स्पेसिफिक्स जो अक्सर अलग होती हैं

* **चेंज डिटेक्शन और एसिंक:** माइक्रोटास्क टाइमिंग रेस कंडीशन को अलग तरह से सामने ला सकती है; स्पिनर और "Save" बटन को डबल-क्लिक इश्यू के लिए देखें।
* **Zone.js एरर:** एक ब्राउज़र में अनहैंडल्ड प्रॉमिस रिजेक्शन लेकिन दूसरे में नहीं—कंसोल चेक करें।
* **Angular Material थीम:** डार्क/लाइट टोकन, हाई-कॉन्ट्रास्ट मोड, और फोकस आउटलाइन (Edge अक्सर फोकस को थोड़ा अलग रेंडर करता है) वेरिफाई करें।
* **i18n पाइप और डेट फॉर्मेट:** Chromium वेरिएंट में `DatePipe` और `Intl` के साथ लोकेल अंतर।

---

# 6) Spring बैकएंड गोताखोरी

* **CORS और रीडायरेक्ट:** समान नियम लेकिन **Edge कभी-कभी डेव में CORS प्रीफ्लाइट इश्यू को पहले सरफेस** कर देता है; `OPTIONS` रिस्पॉन्स और हेडर वेरिफाई करें।
* **Content-Type और कम्प्रेशन:** `application/json;charset=UTF-8` बनाम `application/json` चेक करें; gzip/br वेरिफाई करें—मिसमैच एक ब्राउज़र में पहले "Failed to load" के रूप में दिख सकते हैं।
* **सिक्योरिटी हेडर:** CSP, HSTS, X-Frame-Options—सख्त पॉलिसी इनलाइन स्क्रिप्ट/स्टाइल को अलग तरह से ब्लॉक कर सकती हैं।

# 7) "मैन्युअल" को एक पतली ऑटोमेशन लेयर के साथ दोहराने योग्य बनाएँ

भले ही आप पूर्ण E2E नहीं चाहते, एक **छोटा, तेज़** ब्राउज़र हार्नेस सेट करें ताकि CI हर PR पर Chrome और Edge दोनों चला सके। आप रिग्रेशन को पहले पकड़ लेंगे और अपने मैन्युअल पास को हल्का करेंगे।

### विकल्प A: Playwright (Angular ऐप्स के लिए मेरी शीर्ष पिक)

* एक टेस्ट रनर, **Chrome Stable** और **Microsoft Edge** चैनल लॉन्च करता है, **समानांतर में चलता है**।
* `npm i -D @playwright/test`
* `npx playwright install chromium`
* `playwright.config.ts` उदाहरण:

  ```ts
  import { defineConfig, devices } from '@playwright/test';

  export default defineConfig({
    testDir: './e2e',
    workers: 4, // parallelism
    use: {
      baseURL: 'http://localhost:4200',
      trace: 'retain-on-failure',
    },
    projects: [
      {
        name: 'Chrome Stable',
        use: { ...devices['Desktop Chrome'], channel: 'chrome' },
      },
      {
        name: 'Microsoft Edge',
        use: { ...devices['Desktop Edge'], channel: 'msedge' },
      },
    ],
  });
  ```

  मिनिमल स्मोक स्पेक (`e2e/smoke.spec.ts`):

  ```ts
  import { test, expect } from '@playwright/test';

  test('home loads and login works', async ({ page }) => {
    await page.goto('/');
    await expect(page.getByRole('heading', { name: 'Welcome' })).toBeVisible();
    await page.getByLabel('Email').fill('user@example.com');
    await page.getByLabel('Password').fill('Password123!');
    await page.getByRole('button', { name: 'Sign in' }).click();
    await expect(page.getByText('Dashboard')).toBeVisible();
  });
  ```

  चलाएँ: `npx playwright test`

### विकल्प B: Cypress (Chromium परिवार, Chrome और Edge चलाता है)

* CI मैट्रिक्स (या Cypress डैशबोर्ड) के माध्यम से समानांतरकरण।
* CI में, चलाएँ:

  * `cypress run --browser chrome`
  * `cypress run --browser edge`
* स्पेक्स को छोटा (केवल स्मोक) रखें ताकि "मैन्युअल-फर्स्ट" बना रहे।

### विकल्प C: Selenium Grid (यदि आप पहले से Selenium उपयोग करते हैं)

* Dockerised Grid **chromium/edge नोड्स** को समवर्ती रूप से चलाता है।

  ```yaml
  # docker-compose.yml
  services:
    selenium-hub:
      image: selenium/hub:4
      ports: ["4444:4444"]
    chrome:
      image: selenium/node-chrome:4
      shm_size: 2gb
      depends_on: [selenium-hub]
      environment: [SE_EVENT_BUS_HOST=selenium-hub]
    edge:
      image: selenium/node-edge:4
      shm_size: 2gb
      depends_on: [selenium-hub]
      environment: [SE_EVENT_BUS_HOST=selenium-hub]
  ```

  अपने WebDriver टेस्ट को `http://localhost:4444/wd/hub` पर पॉइंट करें और सूट को समानांतर में चलाएँ।

# 8) CI "समानांतर by default"

* एक **मैट्रिक्स** जॉब (Chrome/Edge) का उपयोग करें और अपने Playwright/Cypress स्मोक को समानांतर में चलाएँ।
* समय बचाने के लिए फेल-फास्ट; फेल होने वाले ब्राउज़र के लिए **ट्रेस/वीडियो** अटैच करें।
* रोजाना रात में, थोड़ा व्यापक सेट और **Lighthouse** पास दोनों ब्राउज़र पर चलाएँ।

# 9) जब वे अलग हों तो त्वरित ट्राइएज

* प्रत्येक ब्राउज़र से **HAR** लें, रिस्पॉन्स (स्टेटस, हेडर, कैशिंग) को डिफ करें।
* टूटे एलिमेंट के लिए **कंप्यूटेड स्टाइल** की तुलना करें (DevTools → Styles → Computed)।
* **कम मोशन** और **फोर्स्ड कलर्स** टॉगल करके एक्सेसिबिलिटी CSS इश्यू को उजागर करें।
* **साइट डेटा** (Application पैनल) क्लियर करें और स्टेल SW/कैश को रद्द करने के लिए रीटेस्ट करें।

# 10) रखने के लिए न्यूनतम आर्टिफैक्ट्स

* आपकी क्रॉस-ब्राउज़र चेक की एक **साझा चेकलिस्ट** (मार्कडाउन)।
* एक **संस्करण लेजर**: ऐप कमिट, Chrome/Edge संस्करण, OS, महत्वपूर्ण फ्लैग।
* किसी भी विजुअल अंतर के लिए **स्क्रीनशॉट** (समान व्यूपोर्ट) अटैच करें जिसे आप "वोंट फिक्स" के रूप में स्वीकार करते हैं।
