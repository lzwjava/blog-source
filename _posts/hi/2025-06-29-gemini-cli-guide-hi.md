**Google Gemini CLI गाइड**

**Google Gemini CLI** को इंस्टॉल, सेटअप और उपयोग करने के लिए एक संक्षिप्त गाइड नीचे दी गई है। यह एक ओपन-सोर्स AI एजेंट है जो Gemini की क्षमताओं को आपके टर्मिनल में लाता है, जिससे कोडिंग, टास्क ऑटोमेशन और बहुत कुछ संभव होता है। यह गाइड 29 जून, 2025 तक उपलब्ध नवीनतम जानकारी पर आधारित है।

## **Google Gemini CLI गाइड**

### **Gemini CLI क्या है?**
Gemini CLI, Google द्वारा विकसित एक ओपन-सोर्स कमांड-लाइन टूल है जो Gemini 2.5 Pro मॉडल (1 मिलियन टोकन कॉन्टेक्स्ट विंडो के साथ) की शक्ति को आपके टर्मिनल में लाता है। यह कोडिंग, डीबगिंग, कंटेंट जनरेशन, टास्क ऑटोमेशन और मल्टीमॉडल टास्क (जैसे इमेज और वीडियो जनरेशन) का समर्थन करता है। यह एक Google अकाउंट के साथ मुफ्त में उपयोग के लिए उपलब्ध है और Google Search तथा Model Context Protocol (MCP) सर्वर जैसे टूल्स के साथ इंटीग्रेट होता है।

---

### **आवश्यक शर्तें**
- **Node.js**: वर्जन 18 या उच्चतर। `node -v` से जांचें। आवश्यकता होने पर [nodejs.org](https://nodejs.org) से इंस्टॉल करें।
- **Google अकाउंट**: Gemini 2.5 Pro (60 रिक्वेस्ट/मिनट, 1,000 रिक्वेस्ट/दिन) तक मुफ्त पहुंच के लिए आवश्यक।
- (वैकल्पिक) **API Key**: उच्चतर लिमिट या विशिष्ट मॉडल के लिए, [Google AI Studio](https://aistudio.google.com) से एक जनरेट करें।
- (वैकल्पिक) **Docker**: MCP सर्वर इंटीग्रेशन (जैसे GitHub टूल्स) के लिए।

### **इंस्टॉलेशन**
Gemini CLI को शुरू करने के दो तरीके हैं:

1. **ग्लोबली इंस्टॉल करें**:
   ```bash
   npm install -g @google/gemini-cli
   gemini
   ```
   यह CLI को ग्लोबली इंस्टॉल करता है और `gemini` कमांड से रन करता है।

2. **बिना इंस्टॉल किए रन करें**:
   ```bash
   npx https://github.com/google-gemini/gemini-cli
   ```
   यह CLI को बिना इंस्टॉलेशन के सीधे रन करता है, जो टेस्टिंग के लिए आदर्श है।

---

### **सेटअप**
1. **CLI लॉन्च करें**:
   - अपने टर्मिनल में `gemini` रन करें।
   - पहली बार रन करने पर, एक थीम (जैसे, ASCII, डार्क, लाइट) चुनें और Enter दबाएं।

2. **प्रमाणीकरण (Authentication)**:
   - मुफ्त एक्सेस के लिए **Login with Google** चुनें (अधिकांश उपयोगकर्ताओं के लिए सिफारिश)।
   - एक ब्राउज़र विंडो खुलेगी; अपने Google अकाउंट से साइन इन करें।
   - वैकल्पिक रूप से, एक API key का उपयोग करें:
     - [Google AI Studio](https://aistudio.google.com) से एक key जनरेट करें।
     - इसे एनवायरनमेंट वेरिएबल के रूप में सेट करें:
       ```bash
       export GEMINI_API_KEY=YOUR_API_KEY
       ```
     - यह उच्चतर लिमिट या एंटरप्राइज़ उपयोग के लिए उपयोगी है।

3. **अपने प्रोजेक्ट पर नेविगेट करें**:
   - कोड-संबंधित टास्क के लिए कॉन्टेक्स्ट प्रदान करने हेतु `gemini` को अपने प्रोजेक्ट के रूट डायरेक्टरी में रन करें।

### **बेसिक उपयोग**
Gemini CLI एक इंटरैक्टिव Read-Eval-Print Loop (REPL) एनवायरनमेंट में ऑपरेट करता है। Gemini मॉडल के साथ इंटरैक्ट करने के लिए कमांड्स या नेचुरल लैंग्वेज प्रॉम्प्ट टाइप करें। यहां कुछ सामान्य टास्क दिए गए हैं:

1. **कोड एक्सप्लेनेशन**:
   - एक प्रोजेक्ट फोल्डर में नेविगेट करें और रन करें:
     ```bash
     gemini
     ```
   - प्रॉम्प्ट: `Explain the architecture of this project` या `Describe the main function in main.py`.
   - CLI फाइल्स को पढ़ती है और एक संरचित स्पष्टीकरण प्रदान करती है।

2. **कोड जनरेशन**:
   - प्रॉम्प्ट: `Create a simple to-do app in HTML, CSS, and JavaScript`.
   - CLI कोड जनरेट करती है और आवश्यकता पड़ने पर इसे फाइल्स में सेव भी कर सकती है।

3. **डीबगिंग**:
   - एक एरर मैसेज या स्टैक ट्रेस पेस्ट करें और पूछें: `What’s causing this error?`.
   - CLI एरर का विश्लेषण करती है और सुझाव देती है, संभावित रूप से अतिरिक्त कॉन्टेक्स्ट के लिए Google Search का उपयोग करते हुए।

4. **फाइल मैनेजमेंट**:
   - प्रॉम्प्ट: `Organize my PDF invoices by month of expenditure`.
   - CLI फाइल्स को मैनिपुलेट कर सकती है या फॉर्मेट कन्वर्ट कर सकती है (जैसे, इमेजेज़ को PNG में)।

5. **GitHub इंटीग्रेशन**:
   - GitHub टास्क (जैसे, इशू लिस्ट करना) के लिए MCP सर्वर का उपयोग करें:
     - एक GitHub MCP सर्वर को `.gemini/settings.json` में एक Personal Access Token (PAT) के साथ कॉन्फ़िगर करें।
     - प्रॉम्प्ट: `Get all open issues assigned to me in the foo/bar repo`.
   - कॉन्फ़िगर किए गए MCP सर्वर और टूल्स को लिस्ट करने के लिए `/mcp` रन करें।

6. **मल्टीमॉडल टास्क**:
   - Imagen या Veo जैसे टूल्स के साथ मीडिया जनरेट करें:
     - प्रॉम्प्ट: `Create a short video of a cat’s adventures in Australia using Veo`.

### **मुख्य विशेषताएं**
- **कॉन्टेक्स्ट फाइल्स (GEMINI.md)**: अपने प्रोजेक्ट रूट में एक `GEMINI.md` फाइल जोड़ें ताकि कोडिंग स्टाइल, प्रोजेक्ट नियम या प्राथमिकताएं (जैसे, "JavaScript के लिए async/await का उपयोग करें") परिभाषित की जा सकें। CLI टेलर्ड रिस्पॉन्स के लिए इसका उपयोग करती है।
- **बिल्ट-इन टूल्स**:
   - `/tools`: उपलब्ध टूल्स (जैसे, Google Search, फाइल ऑपरेशन) को लिस्ट करें।
   - `/compress`: टोकन बचाने के लिए चैट कॉन्टेक्स्ट को सारांशित करें।
   - `/bug`: Gemini CLI GitHub रेपो में सीधे इशू फाइल करें।
- **नॉन-इंटरैक्टिव मोड**: स्क्रिप्टिंग के लिए, कमांड्स पाइप करें:
   ```bash
   echo "Write a Python script" | gemini
   ```
- **कन्वर्सेशन मेमोरी**: `/save <tag>` के साथ सेशन हिस्ट्री सेव करें और `/restore <tag>` के साथ रिज्यूम करें।
- **कस्टम कॉन्फ़िगरेशन**:
   - ग्लोबल सेटिंग्स के लिए `~/.gemini/settings.json` या लोकल सेटिंग्स के लिए किसी प्रोजेक्ट में `.gemini/settings.json` एडिट करें।
   - उदाहरण: MCP सर्वर या कस्टम थीम सेट करें।

### **टिप्स और ट्रिक्स**
- **प्लान के साथ शुरुआत करें**: कॉम्प्लेक्स टास्क के लिए, पहले एक प्लान मांगें: `Create a detailed implementation plan for a login system`. यह संरचित आउटपुट सुनिश्चित करता है।
- **लोकल कॉन्टेक्स्ट का उपयोग करें**: MCP सर्वर पर निर्भर रहने के बजाय, तेज और विश्वसनीय रिस्पॉन्स के लिए प्रोजेक्ट-विशिष्ट विवरण `GEMINI.md` में एनकोड करें।
- **डीबगिंग**: विस्तृत रिक्वेस्ट/रिस्पॉन्स जानकारी के लिए `DEBUG=true gemini` के साथ वर्बोज़ लॉगिंग एनेबल करें।
- **बदलावों की समीक्षा करें**: अनुमोदन से पहले फाइल मॉडिफिकेशन या कमांड्स की हमेशा समीक्षा करें (पुष्टि करने के लिए `y` टाइप करें)।
- **टूल्स एक्सप्लोर करें**: वेब सर्च या मेमोरी सेविंग जैसी बिल्ट-इन क्षमताओं को खोजने के लिए `/tools` रन करें।

### **समस्या निवारण**
- **प्रमाणीकरण संबंधी समस्याएं**: सुनिश्चित करें कि आपका Google अकाउंट या API key वैध है। मेथड स्विच करने के लिए `/auth` का उपयोग करें।
- **रेट लिमिट**: मुफ्त टियर 60 रिक्वेस्ट/मिनट और 1,000/दिन की अनुमति देता है। उच्चतर लिमिट के लिए, एक API key या Vertex AI का उपयोग करें।
- **एरर्स**: GitHub पर [Troubleshooting Guide](https://github.com/google-gemini/gemini-cli/docs/troubleshooting.md) चेक करें।
- **धीमी प्रतिक्रिया**: CLI प्रीव्यू स्टेज में है और API कॉल के साथ धीमी हो सकती है। GitHub पर फीडबैक फाइल करें।

### **एडवांस्ड उपयोग**
- **MCP सर्वर इंटीग्रेशन**:
  - रिपॉजिटरी इंटरैक्शन के लिए एक GitHub MCP सर्वर सेट अप करें:
    - आवश्यक स्कोप्स के साथ एक GitHub PAT बनाएं।
    - `.gemini/settings.json` में एड करें:
      ```json
      {
        "mcpServers": [
          {
            "name": "github",
            "url": "http://localhost:8080",
            "type": "github"
          }
        ]
      }
      ```
    - MCP सर्वर के लिए एक Docker कंटेनर रन करें (GitHub डॉक्स देखें)।
- **स्क्रिप्टिंग**: Gemini CLI को स्क्रिप्ट्स में इंटीग्रेट करके टास्क्स को ऑटोमेट करें:
  ```bash
  gemini --non-interactive "Generate a bash script to backup my files"
  ```
- **मल्टीमॉडल प्रॉम्प्ट्स**:
  - उदाहरण: `Describe this image: path/to/image.jpg` (एक विजन-कैपेबल मॉडल जैसे `gemini-pro-vision` की आवश्यकता होती है)।
