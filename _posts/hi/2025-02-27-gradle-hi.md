Gradle एक शक्तिशाली बिल्ड ऑटोमेशन टूल है जो आमतौर पर Java, Kotlin, और Android प्रोजेक्ट्स के लिए इस्तेमाल किया जाता है, लेकिन यह अन्य भाषाओं के लिए भी पर्याप्त लचीला है। यहाँ Gradle को प्रभावी ढंग से इस्तेमाल करना शुरू करने के लिए एक शुरुआत-अनुकूल गाइड है:

---

### 1. **Gradle इंस्टॉल करें**
Gradle का उपयोग करने से पहले, आपको इसे इंस्टॉल करना होगा:
- **Mac/Linux (Homebrew के साथ)**:
  - अपने टर्मिनल में `brew install gradle` चलाएँ।
- **Windows (या मैनुअल इंस्टॉलेशन)**:
  - [gradle.org](https://gradle.org/install/) से Gradle बाइनरी डाउनलोड करें।
  - इसे एक फ़ोल्डर में एक्सट्रैक्ट करें (जैसे, `C:\Gradle` या `/opt/gradle`)।
  - Gradle के `bin` डायरेक्टरी (जैसे, `C:\Gradle\gradle-8.x\bin`) को अपने सिस्टम के PATH एनवायरनमेंट वेरिएबल में जोड़ें।
- **इंस्टॉलेशन सत्यापित करें**:
  - एक टर्मिनल खोलें और `gradle -v` चलाएँ। आपको Gradle वर्जन और विवरण दिखना चाहिए।

वैकल्पिक रूप से, यदि आप **Gradle Wrapper** के साथ एक प्रोजेक्ट पर काम कर रहे हैं (साझा प्रोजेक्ट्स में आम), तो आपको ग्लोबली Gradle इंस्टॉल करने की आवश्यकता नहीं है—बस प्रोजेक्ट डायरेक्टरी में `./gradlew` (Linux/Mac) या `gradlew.bat` (Windows) का उपयोग करें।

---

### 2. **Gradle बेसिक्स समझें**
Gradle आपके प्रोजेक्ट को कैसे बनाया जाए यह परिभाषित करने के लिए एक **Groovy** या **Kotlin** स्क्रिप्ट (आमतौर पर `build.gradle` या `build.gradle.kts`) का उपयोग करता है। यहाँ वह जानकारी है जो आपको चाहिए:
- **प्रोजेक्ट्स**: एक Gradle बिल्ड में एक या अधिक प्रोजेक्ट हो सकते हैं (जैसे, एक सिंगल ऐप या मल्टी-मॉड्यूल सेटअप)।
- **टास्क्स**: ये वे एक्शन हैं जो Gradle करता है, जैसे `compileJava`, `test`, या `build`।
- **डिपेंडेंसीज़**: Gradle आपके प्रोजेक्ट द्वारा आवश्यक लाइब्रेरीज़ को मैनेज करता है (जैसे, Maven Central या Google के रिपॉजिटरी से)।

---

### 3. **एक साधारण Gradle प्रोजेक्ट बनाएँ**
आइए Gradle को एक्शन में देखने के लिए एक बेसिक Java प्रोजेक्ट सेट अप करते हैं:
1. **एक प्रोजेक्ट फ़ोल्डर बनाएँ**:
   - एक डायरेक्टरी बनाएँ (जैसे, `my-gradle-project`) और अपने टर्मिनल में उसके अंदर नेविगेट करें।
2. **Gradle इनिशियलाइज़ करें**:
   - `gradle init` चलाएँ।
   - प्रॉम्प्ट का पालन करें: "application", "Java", और बिल्ड स्क्रिप्ट के लिए "Groovy" (या Kotlin) चुनें।
   - यह एक `build.gradle` फ़ाइल और सैंपल कोड के साथ एक बेसिक स्ट्रक्चर बनाता है।
3. **जेनरेट किए गए `build.gradle` को एक्सप्लोर करें**:
   ```groovy
   plugins {
       id 'java'
       id 'application'
   }

   repositories {
       mavenCentral()
   }

   dependencies {
       implementation 'org.slf4j:slf4j-api:1.7.36'
   }

   application {
       mainClass = 'com.example.App'  // अपने पैकेज के आधार पर एडजस्ट करें
   }
   ```
   - `plugins`: Java और ऐप चलाने के लिए सपोर्ट जोड़ता है।
   - `repositories`: वह स्थान जहाँ Gradle डिपेंडेंसीज़ की तलाश करता है (जैसे, Maven Central)।
   - `dependencies`: वे लाइब्रेरीज़ जो आपका प्रोजेक्ट इस्तेमाल करता है।
   - `application`: रन करने के लिए मुख्य क्लास निर्दिष्ट करता है।

4. **टास्क्स चलाएँ**:
   - प्रोजेक्ट बिल्ड करें: `gradle build`।
   - ऐप रन करें: `gradle run`।
   - उपलब्ध टास्क्स की सूची देखें: `gradle tasks`।

---

### 4. **सामान्य Gradle कमांड्स**
यहाँ कुछ कमांड्स दी गई हैं जिनका आप अक्सर उपयोग करेंगे:
- `gradle build`: आपके प्रोजेक्ट को कंपाइल और पैकेज करता है।
- `gradle clean`: ताज़ा शुरुआत करने के लिए `build` डायरेक्टरी को डिलीट करता है।
- `gradle test`: आपके प्रोजेक्ट के टेस्ट चलाता है।
- `gradle dependencies`: एक डिपेंडेंसी ट्री दिखाता है।
- `./gradlew <task>`: ग्लोबल इंस्टॉल के बजाय प्रोजेक्ट के Gradle Wrapper का उपयोग करता है।

---

### 5. **डिपेंडेंसीज़ जोड़ें**
बाहरी लाइब्रेरीज़ का उपयोग करने के लिए, `build.gradle` को एडिट करें। उदाहरण के लिए, Gson (एक JSON पार्सिंग लाइब्रेरी) जोड़ने के लिए:
```groovy
dependencies {
    implementation 'com.google.code.gson:gson:2.10.1'
}
```
फिर इसे डाउनलोड करने के लिए `gradle build` चलाएँ।

---

### 6. **अपना बिल्ड कस्टमाइज़ करें**
आप `build.gradle` में कस्टम टास्क जोड़ सकते हैं या सेटिंग्स ट्वीक कर सकते हैं। एक कस्टम टास्क का उदाहरण:
```groovy
task hello {
    doLast {
        println 'Hello, Gradle!'
    }
}
```
इसे `gradle hello` के साथ चलाएँ।

---

### 7. **Gradle Wrapper का उपयोग करें (टीम्स के लिए रिकमेंडेड)**
Gradle Wrapper यह सुनिश्चित करता है कि सभी लोग एक ही Gradle वर्जन का उपयोग करें:
- इसे जेनरेट करें: `gradle wrapper`।
- इसका उपयोग करें: `./gradlew build` (या Windows पर `gradlew.bat build`)।

---

### 8. **सफलता के लिए टिप्स**
- **डॉक्युमेंटेशन**: गहराई से जानने के लिए [Gradle की ऑफिशियल डॉक्स](https://docs.gradle.org/current/userguide/userguide.html) देखें।
- **कैशिंग**: Gradle डिपेंडेंसीज़ और टास्क आउटपुट को कैश करता है, इसलिए बिल्ड समय के साथ तेज़ होते जाते हैं।
- **प्लगइन्स**: फंक्शनैलिटी बढ़ाने के लिए प्लगइन्स एक्सप्लोर करें (जैसे, Spring Boot, Android, या कोड क्वालिटी के लिए)।

---

### उदाहरण: एक Java ऐप बिल्ड और रन करें
यदि आपका `src/main/java/com/example/App.java` इस तरह दिखता है:
```java
package com.example;
public class App {
    public static void main(String[] args) {
        System.out.println("Hello from Gradle!");
    }
}
```
`gradle run` चलाएँ, और आपको आउटपुट दिखाई देगा।
