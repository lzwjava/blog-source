### Pydantic लाइब्रेरी
Pydantic एक लोकप्रिय ओपन-सोर्स Python लाइब्रेरी है जो डेटा वैलिडेशन और पार्सिंग के लिए उपयोग की जाती है। यह Python के टाइप हिंट्स का लाभ उठाती है ताकि परिभाषित स्कीमा के विरुद्ध इनपुट डेटा (जैसे JSON या डिक्शनरी) का स्वचालित रूप से सत्यापन किया जा सके, जहाँ संभव हो प्रकारों को बदला जा सके और बेमेल होने पर स्पष्ट त्रुटियाँ उत्पन्न की जा सकें। उदाहरण के लिए, यह सुनिश्चित करती है कि एक पूर्णांक फ़ील्ड को स्ट्रिंग के रूप में नहीं पास किया जाए, जब तक कि निर्दिष्ट न किया गया हो। इसका व्यापक रूप से FastAPI जैसे वेब फ्रेमवर्क में API अनुरोध/प्रतिक्रिया हैंडलिंग के लिए उपयोग किया जाता है।

- **मुख्य विशेषताएँ**: जटिल प्रकारों (जैसे, enums, unions), कस्टम वैलिडेटर्स, और क्रमबद्धता/विक्रमबद्धता का समर्थन करती है।
- **इंस्टॉलेशन और उपयोग**: `pip install pydantic` के माध्यम से इंस्टॉल करें। मूल उदाहरण:
  ```python
  from pydantic import BaseModel

  class User(BaseModel):
      id: int
      name: str

  user = User(id='123', name='Alice')  # '123' को int में बदल देता है
  print(user.dict())  # आउटपुट: {'id': 123, 'name': 'Alice'}
  ```

### Pydantic-Core
Pydantic-core, Pydantic का अंतर्निहित हाई-परफॉर्मेंस इंजन है। यह Rust में लिखा गया है (PyO3 बाइंडिंग्स के माध्यम से) ताकि तेज़ डेटा वैलिडेशन प्रदान किया जा सके, जो शुद्ध Python इम्प्लीमेंटेशन की तुलना में कहीं अधिक तेज़ है। इसका उपयोग सीधे उपयोगकर्ताओं द्वारा करने के लिए नहीं है—बल्कि, यह Pydantic द्वारा स्वचालित रूप से कॉल किया जाता है। यह पृथक्करण आसान रखरखाव और ऑप्टिमाइज़ेशन की अनुमति देता है, जैसे कि मुख्य लाइब्रेरी को धीमा किए बिना टाइप कोर्सन में एज केस को हैंडल करना।

- **Pydantic से संबंध**: Pydantic को Pydantic-core के चारों ओर एक उपयोगकर्ता-अनुकूल API रैपर के रूप में समझें। Pydantic-core में अपग्रेड सार्वजनिक API को बदले बिना प्रदर्शन में सुधार करते हैं।
- **यह महत्वपूर्ण क्यों है**: परफॉर्मेंस टेस्ट दिखाते हैं कि Pydantic-core, Marshmallow या Django Forms जैसे विकल्पों की तुलना में वैलिडेशन को 10-100 गुना तेज़ बनाता है।

### Python में टाइपिंग
`typing` मॉड्यूल Python की स्टैंडर्ड लाइब्रेरी का हिस्सा है (Python 3.5+ में PEP 484 के माध्यम से जोड़ा गया) और कोड में टाइप हिंट्स जोड़ने के लिए टूल्स प्रदान करता है। ये हिंट्स रनटाइम पर नियम लागू नहीं करते हैं, लेकिन कोड की पठनीयता में सुधार करते हैं, IDE सपोर्ट (जैसे, ऑटोकम्प्लीट, त्रुटि पहचान) सक्षम करते हैं, और mypy जैसे स्टैटिक टाइप चेकर्स को शुरुआती चरण में बग्स पकड़ने की अनुमति देते हैं।

- **सामान्य घटक**: इसमें स्ट्रिंग्स की सूचियों के लिए `List[str]`, कई प्रकारों के लिए `Union[int, str]`, `int | None` के लिए `Optional[int]`, और जेनरिक्स जैसे `Dict[str, Any]` शामिल हैं।
- **Pydantic से संबंध**: Pydantic सीधे `typing` एनोटेशन्स पर बनता है। एक मॉडल जैसे `class Model(BaseModel): x: int` अपेक्षाओं को परिभाषित करने के लिए `typing` का उपयोग करता है, और Pydantic तदनुसार सत्यापन करता है। `typing` के बिना, Pydantic कार्य नहीं कर सकता क्योंकि इसका टाइप-ड्रिवन वैलिडेशन इन हिंट्स पर निर्भर करता है।
