### Azure में API Management का उपयोग करके API Gateway बनाना

Azure API Management (APIM) एक पूर्णतः प्रबंधित सेवा है जो API गेटवे के रूप में कार्य करती है और API प्रकाशन, सुरक्षा, विश्लेषण और डेवलपर पोर्टल जैसी सुविधाएँ प्रदान करती है। नीचे Azure पोर्टल के माध्यम से एक APIM इंस्टेंस बनाने के लिए चरण-दर-चरण मार्गदर्शिका दी गई है।

#### पूर्वापेक्षाएँ
- एक सक्रिय Azure सब्सक्रिप्शन। यदि आपके पास नहीं है, तो एक [निःशुल्क Azure अकाउंट](https://azure.microsoft.com/hi-in/free/) बनाएँ।

#### API Management इंस्टेंस बनाने के चरण

1. **Azure पोर्टल में साइन इन करें**  
   [Azure पोर्टल](https://portal.azure.com) पर जाएँ और अपने Azure अकाउंट से साइन इन करें।

2. **एक नया रिसोर्स बनाएँ**  
   - Azure पोर्टल मेन्यू से, **रिसोर्स बनाएँ** चुनें। (वैकल्पिक रूप से, Azure होम पेज से **रिसोर्स बनाएँ** चुनें।)  
   - **रिसोर्स बनाएँ** पेज पर, **इंटीग्रेशन** श्रेणी के अंतर्गत **API Management** के लिए खोजें और उसे चुनें।  
   - **बनाएँ** पर क्लिक करें।

3. **मूल सेटिंग्स कॉन्फ़िगर करें**  
   **API Management बनाएँ** पेज पर, **मूल** टैब पर स्विच करें और विवरण भरें:  
   | सेटिंग              | विवरण                                                                 |
   |----------------------|-----------------------------------------------------------------------------|
   | सब्सक्रिप्शन         | इस इंस्टेंस के लिए Azure सब्सक्रिप्शन चुनें।                            |
   | रिसोर्स ग्रुप       | एक मौजूदा रिसोर्स ग्रुप चुनें या एक नया बनाएँ (उदाहरण: "APIM-RG")।    |
   | क्षेत्र               | अपने उपयोगकर्ताओं या बैकएंड सेवाओं के निकट एक क्षेत्र चुनें (उदाहरण: East US)।      |
   | रिसोर्स नाम        | एक अद्वितीय नाम दर्ज करें (उदाहरण: "my-apim-instance")। यह डिफ़ॉल्ट डोमेन का हिस्सा बन जाता है: `<name>.azure-api.net`। इसे बाद में नहीं बदला जा सकता। |
   | संगठन का नाम    | आपके संगठन का नाम (डेवलपर पोर्टल और ईमेल में उपयोग किया जाता है)।             |
   | प्रशासक ईमेल  | सिस्टम सूचनाओं के लिए आपका ईमेल।                                        |
   | मूल्य निर्धारण स्तर         | विकास/परीक्षण के लिए **Basic v2** से शुरुआत करें (त्वरित तैनाती, ~30-40 मिनट)। Developer या Standard जैसे अन्य स्तर अधिक सुविधाएँ प्रदान करते हैं। |
   | यूनिट                | मूल्यांकन के लिए डिफ़ॉल्ट रूप से 1 पर सेट करें।                                                |  
   समीक्षा करें और आगे बढ़ें।

4. **वैकल्पिक: मॉनिटरिंग और सुरक्षा कॉन्फ़िगर करें**  
   **मॉनिटर + सुरक्षित** टैब पर:  
   - लॉगिंग के लिए Log Analytics जैसे वैकल्पिक ऐड-ऑन सक्षम करें (इस पर अतिरिक्त लागत आ सकती है)।  
   - यदि अभी शुरुआत कर रहे हैं तो अभी के लिए छोड़ दें।

5. **समीक्षा करें और बनाएँ**  
   - **समीक्षा + बनाएँ** टैब पर, अपनी सेटिंग्स सत्यापित करें।  
   - **बनाएँ** पर क्लिक करें। तैनाती में 30-40 मिनट या अधिक समय लगता है।  
   - सुझाव: एक बार बन जाने के बाद आसान पहुँच के लिए रिसोर्स को अपने डैशबोर्ड पर पिन कर दें।

6. **इंस्टेंस सत्यापित करें**  
   - Azure पोर्टल में, **API Management services** के लिए खोजें।  
   - अपना नया इंस्टेंस चुनें।  
   - **अवलोकन** पेज पर, पुष्टि करें कि स्थिति "ऑनलाइन" है। अब आप API आयात करने, नीतियाँ कॉन्फ़िगर करने या डेवलपर पोर्टल सेट अप करने के लिए तैयार हैं।

#### अगले कदम
- अपनी पहली API आयात करें: [API आयात और प्रकाशित करने के ट्यूटोरियल](https://learn.microsoft.com/hi-in/azure/api-management/import-and-publish) का पालन करें।  
- इसे सुरक्षित करें: प्रमाणीकरण, दर सीमित करने आदि के लिए नीतियाँ जोड़ें।  
- सफ़ाई करें: यदि परीक्षण कर रहे हैं, तो शुल्क से बचने के लिए पोर्टल के माध्यम से रिसोर्स ग्रुप हटा दें (**रिसोर्स ग्रुप** के लिए खोजें > ग्रुप चुनें > **रिसोर्स ग्रुप हटाएँ**)।  

कस्टम डोमेन या उन्नत कॉन्फ़िगरेशन के लिए, [APIM दस्तावेज़ीकरण](https://learn.microsoft.com/hi-in/azure/api-management/) देखें।
