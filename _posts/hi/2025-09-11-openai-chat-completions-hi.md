आपने Microsoft Learn का लेख **"Work with chat completions models"** (**1 सितंबर, 2025** को प्रकाशित) लिंक किया है। यहाँ इसके मुख्य बिंदुओं का एक स्पष्ट और संक्षिप्त सारांश दिया गया है:

---

### मुख्य बातें

*   **चैट-उन्मुख मॉडल**
    नए चैट मॉडल पुराने GPT-3 मॉडल्स से अलग हैं। "टेक्स्ट-इन, टेक्स्ट-आउट" के बजाय, वे **कन्वर्सेशन-इन, मैसेज-आउट** फॉर्मेट का उपयोग करते हैं। मूल रूप से, इनपुट एक ट्रांसक्रिप्ट की तरह संरचित होता है (जैसे *system*, *user*, *assistant* भूमिकाएँ), और मॉडल वार्तालाप जारी रखने के लिए एक संदेश लौटाता है ([Microsoft Learn][1])।

*   **डायलॉग के लिए डिज़ाइन किया गया**
    यह संरचना मल्टी-टर्न आगे-पीछे वार्तालापों के लिए अनुकूलित है। यह सिंगल-प्रॉम्प्ट परिदृश्यों के लिए भी अच्छी तरह काम करती है, लेकिन वार्तालाप फॉर्मेट सहज इंटरैक्शन को सक्षम बनाता है ([Microsoft Learn][1])।

*   **उन्हें पुराने मॉडल्स की तरह न समझें**
    ये चैट मॉडल अधिक वर्बोज़ हैं और अक्सर कम उपयोगी जवाब देते हैं यदि आप उनके साथ GPT-3 जैसा व्यवहार करते हैं। सर्वोत्तम परिणामों के लिए नए चैट-स्टाइल इंटरफेस का उपयोग करने की सलाह दी जाती है ([Microsoft Learn][1])।

*   **नमूना कोड स्निपेट**
    लेख में Azure के चैट कम्प्लीशन्स API को कॉल करने के लिए Python-स्टाइल सिंटैक्स का उपयोग करके एक बुनियादी कोड नमूना शामिल है:

    ```python
    from openai import OpenAI
    import os

    client = OpenAI(
        api_key=os.getenv("AZURE_OPENAI_API_KEY"),
        base_url="https://YOUR-RESOURCE-NAME.openai.azure.com/openai/v1/"
    )

    response = client.chat.completions.create(
        model="gpt-4o",  # or deployment_name
        messages=[
            {"role": "system", "content": "Assistant is a large language model trained by OpenAI."},
            {"role": "user", "content": "Who were the founders of Microsoft?"}
        ]
    )

    print(response.model_dump_json(indent=2))
    print(response.choices[0].message.content)
    ```

    यह दर्शाता है कि संदेशों को ठीक से कैसे भेजा जाए (भूमिका पृथक्करण के साथ) और मॉडल का जवाब कैसे प्राप्त करें ([Microsoft Learn][1])।

---

### यह महत्वपूर्ण क्यों है

1.  **उन्नत वार्तालाप संदर्भ** — चैट मॉडल मल्टी-टर्न इंटरैक्शन में प्रवाह और संदर्भ को बेहतर ढंग से बनाए रखते हैं।
2.  **बेहतर उपयोगिता** — भूमिकाओं (सिस्टम, यूजर, असिस्टेंट) का उपयोग स्पष्ट निर्देश, व्यक्तित्व सेटिंग, या बाधाओं के लिए अनुमति देता है।
3.  **नुकसान से बचें** — चैट मॉडल्स के साथ पिछली पीढ़ी के मॉडल्स जैसा व्यवहार करने से प्रतिक्रिया की गुणवत्ता खराब हो सकती है; चैट API इंटरफेस का उपयोग उसी तरह से करें जैसे कि इरादा है।
