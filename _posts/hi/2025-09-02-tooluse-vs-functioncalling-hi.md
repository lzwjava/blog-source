आइए LLM APIs के संदर्भ में **टूल यूज़ (tool use)** और **फंक्शन कॉलिंग (function calling)** के अंतर को समझते हैं:

---

## 1. टूल यूज़ (Tool Use)

* **अवधारणा**: टूल यूज़ एक व्यापक विचार है जहां LLM को बाहरी क्षमताओं से सुसज्जित किया जाता है। केवल पाठ (text) जनरेट करने के बजाय, यह जानकारी प्राप्त करने, कोड चलाने, डेटाबेस क्वेरी करने या छवि बनाने के लिए एक "टूल का उपयोग" कर सकता है।
* **कार्यान्वयन**: व्यवहार में, "टूल" बाहरी APIs या सिस्टम होते हैं जो LLM रनटाइम के साथ पंजीकृत होते हैं (जैसे, एक सर्च API, Python एक्सेक्यूशन एनवायरनमेंट, या कैलेंडर API)।
* **LLM की भूमिका**: मॉडल तय करता है कि टूल को कब कॉल करना है, किन आर्ग्युमेंट्स के साथ, और परिणामों को वार्तालाप में वापस एकीकृत करता है।
* **उदाहरण**:

  * उपयोगकर्ता: "गुआंगज़ौ में मौसम कैसा है?"
  * LLM: `weather` टूल को `{city: "Guangzhou"}` के साथ कॉल करता है → `28°C, sunny` प्राप्त करता है।
  * LLM: उत्तर देता है, "यह 28°C और धूप है।"

**टूल यूज़** को एक सामान्य ऑर्केस्ट्रेशन फ्रेमवर्क के रूप में समझें, जहां LLM केवल शब्दों से जवाब नहीं देता—यह बाहरी सिस्टम के साथ कार्यों का समन्वय करता है।

---

## 2. फंक्शन कॉलिंग (Function Calling)

* **अवधारणा**: फंक्शन कॉलिंग कुछ LLM APIs (जैसे OpenAI, Anthropic, आदि) द्वारा प्रदान की गई एक *संरचित* तंत्र है, जहां आप फंक्शन्स को परिभाषित करते हैं (नाम, पैरामीटर, स्कीमा के साथ), और LLM उन्हें कॉल करने के लिए JSON आर्ग्युमेंट्स वापस दे सकता है।
* **कार्यान्वयन**: आप मॉडल को फंक्शन्स का वर्णन करने वाली JSON स्कीमा प्रदान करते हैं। मॉडल का आउटपुट तो या तो सामान्य टेक्स्ट होता है या एक संरचित फंक्शन कॉल ऑब्जेक्ट।
* **LLM की भूमिका**: API उपयोग को "हैलुसिनेट" करने के बजाय, मॉडल पूर्वनिर्धारित फंक्शन्स के लिए वैध, संरचित आर्ग्युमेंट्स आउटपुट करता है।
* **उदाहरण**:

  * आप एक फंक्शन को परिभाषित करते हैं:

    ```json
    {
      "name": "getWeather",
      "description": "Get current weather",
      "parameters": {
        "type": "object",
        "properties": {
          "city": {"type": "string"}
        },
        "required": ["city"]
      }
    }
    ```
  * उपयोगकर्ता: "गुआंगज़ौ में मौसम कैसा है?"
  * LLM: आउटपुट देता है → `{ "name": "getWeather", "arguments": {"city": "Guangzhou"} }`
  * आपकी प्रणाली फंक्शन को एक्जीक्यूट करती है और परिणाम वापस पास करती है।

यह **फंक्शन कॉलिंग** को **टूल यूज़** करने का एक विशिष्ट, सुरक्षित, स्कीमा-संचालित तरीका बनाता है।

---

## 3. मुख्य अंतर

* **दायरा**:

  * टूल यूज़ = सामान्य अवधारणा (बाहरी सिस्टम से सुसज्जित LLM)।
  * फंक्शन कॉलिंग = संरचित कॉल्स के लिए एक मानकीकृत API डिज़ाइन।
* **नियंत्रण**:

  * टूल यूज़ अधिक स्वतंत्र-रूप का हो सकता है (LLM निर्णय लेता है और फॉर्मेट करता है)।
  * फंक्शन कॉलिंग सख्त स्कीमा लागू करती है, अस्पष्टता और हैलुसिनेशन को कम करती है।
* **अंतरसंचालनीयता**:

  * टूल यूज़ में कई बाहरी सिस्टम (सर्च, कोड एक्सेक्यूशन, इमेज जनरेशन) शामिल हो सकते हैं।
  * फंक्शन कॉलिंग तब सबसे अच्छी होती है जब आप सटीक, संरचित आर्ग्युमेंट्स चाहते हैं (जैसे APIs या डेटाबेस क्वेरीज़)।

---

## 4. मानसिक मॉडल

* फंक्शन कॉलिंग, टूल यूज़ का एक **उपसमुच्चय** है।
* यदि टूल यूज़ *सामान्य रणनीति* है, तो फंक्शन कॉलिंग *सुरक्षित, संरचित विधि* है जिसे अधिकांश APIs एक्सपोज़ करते हैं।
