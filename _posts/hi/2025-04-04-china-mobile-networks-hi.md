यह एक सामान्य समस्या है जब चीन में कुछ मोबाइल नेटवर्क का उपयोग किया जाता है, खासकर ऐसी सेवाओं के साथ जैसे कि Git जो विशिष्ट पोर्ट और प्रोटोकॉल पर निर्भर करती हैं। यहां बताया गया है कि ऐसा क्यों होता है और इसे कैसे ठीक करें:

**चीनी मोबाइल नेटवर्क पर Git ऑपरेशन विफल क्यों होते हैं:**

*   **पोर्ट ब्लॉकिंग/थ्रॉटलिंग:**
    * चीन में मोबाइल नेटवर्क ऑपरेटर कभी-कभी ट्रैफिक शेपिंग या पोर्ट ब्लॉकिंग लागू करते हैं, विशेष रूप से Git द्वारा उपयोग किए जाने वाले गैर-मानक पोर्ट (जैसे SSH पोर्ट 22) के लिए। यह अक्सर नेटवर्क बैंडविड्थ प्रबंधन या सुरक्षा कारणों से किया जाता है।
    * भले ही पोर्ट 22 पूरी तरह से ब्लॉक न हो, यह भारी थ्रॉटलिंग का शिकार हो सकता है, जिससे कनेक्शन बेहद धीमा हो जाता है या टाइमआउट हो जाता है, जो Git ऑपरेशन को "अटका हुआ" दिखाता है।
*   **DNS समस्याएं:**
    * जबकि आप GitHub और Google जैसी वेबसाइटों तक पहुंच सकते हैं, Git ऑपरेशन DNS के माध्यम से विशिष्ट होस्टनाम (जैसे, `github.com`) को रिज़ॉल्व करने पर निर्भर करते हैं। यदि DNS रिज़ॉल्यूशन धीमा या अविश्वसनीय है, तो यह Git को हैंग करने का कारण बन सकता है।
*   **पैकेट लॉस/लेटेंसी:**
    * मोबाइल नेटवर्क, खासकर जब हॉटस्पॉट के रूप में उपयोग किए जाते हैं, तो वायर्ड कनेक्शन की तुलना में अधिक लेटेंसी और पैकेट लॉस हो सकता है। यह Git द्वारा उपयोग किए जाने वाले SSH कनेक्शन को बाधित कर सकता है, जिससे विफलताएं होती हैं।
*   **फ़ायरवॉल हस्तक्षेप:**
    * चीन के "ग्रेट फ़ायरवॉल" द्वारा SSH कनेक्शन में हस्तक्षेप किया जा सकता है, भले ही वे स्पष्ट रूप से ब्लॉक न हों। फ़ायरवॉल की डीप पैकेट इंस्पेक्शन कभी-कभी कनेक्शन अस्थिरता का कारण बन सकती है।
*   **MTU समस्याएं:**
    * मैक्सिमम ट्रांसमिशन यूनिट (MTU) की समस्याएं डेटा ट्रांसफर में मुश्किल पैदा कर सकती हैं, और मोबाइल नेटवर्क में कभी-कभी वायर्ड नेटवर्क की तुलना में छोटे MTU मान होते हैं।

**चीनी मोबाइल नेटवर्क पर Git Push/Pull समस्याओं को कैसे ठीक करें:**

1.  **SSH के बजाय HTTPS का उपयोग करें:**
    * HTTPS पर Git पोर्ट 443 का उपयोग करता है, जो आमतौर पर वेब ट्रैफिक के लिए खुला रहता है। यह सबसे विश्वसनीय वर्कअराउंड है।
    * अपने Git रिमोट को HTTPS में बदलने के लिए:
        * अपना टर्मिनल खोलें।
        * अपने Git रिपॉजिटरी में नेविगेट करें।
        * निम्नलिखित कमांड चलाएं, `your_username` और `your_repository` को अपने GitHub विवरण से बदलें:
            ```bash
            git remote set-url origin https://github.com/your_username/your_repository.git
            ```
    * यदि आपको यूजरनेम और पासवर्ड प्रदान करने की आवश्यकता है, तो आप git क्रेडेंशियल हेल्पर का उपयोग कर सकते हैं, या पर्सनल एक्सेस टोकन का उपयोग कर सकते हैं।
2.  **VPN का उपयोग करें:**
    * एक विश्वसनीय VPN नेटवर्क प्रतिबंधों को बायपास कर सकता है और अधिक स्थिर कनेक्शन प्रदान कर सकता है।
    * Git ऑपरेशन करने से पहले चीन के बाहर किसी VPN सर्वर से कनेक्ट हों।
    * ध्यान रखें कि VPN भी अस्थिरता और स्पीड की समस्याओं का अनुभव कर सकते हैं।
3.  **SSH पोर्ट कॉन्फ़िगर करें (यदि SSH आवश्यक है):**
    * यदि आपको SSH का उपयोग करना ही है, तो Git को एक अलग पोर्ट (जैसे 443) उपयोग करने के लिए कॉन्फ़िगर करने का प्रयास करें जो ब्लॉक होने की संभावना कम हो।
    * अपनी `~/.ssh/config` फ़ाइल एडिट करें:
        ```
        Host github.com
            Hostname ssh.github.com
            Port 443
            User git
        ```
    * फिर, अपने git रिमोट URL को ssh.github.com का उपयोग करने के लिए बदलें:
        ```bash
        git remote set-url origin git@ssh.github.com:your_username/your_repository.git
        ```
4.  **MTU सेटिंग्स जांचें:**
    * यदि आपको MTU समस्याओं का संदेह है, तो अपने MTU आकार को कम करने का प्रयास करें।
    * macOS पर, आप MTU एडजस्ट करने के लिए `networksetup` कमांड का उपयोग कर सकते हैं।
    * हालाँकि, MTU बदलना आम तौर पर एक अधिक उन्नत ट्रबलशूटिंग चरण है और इसे सावधानीपूर्वक करना चाहिए।
5.  **DNS सेटिंग्स जांचें:**
    * एक पब्लिक DNS सर्वर, जैसे Google DNS (8.8.8.8 और 8.8.4.4) या Cloudflare DNS (1.1.1.1 और 1.0.0.1) का उपयोग करने का प्रयास करें।
    * आप अपनी macOS नेटवर्क प्राथमिकताओं में DNS सेटिंग्स कॉन्फ़िगर कर सकते हैं।
6.  **Git Proxy का उपयोग करें:**
    * यदि आपके पास प्रॉक्सी सर्वर तक पहुंच है, तो आप Git को इसका उपयोग करने के लिए कॉन्फ़िगर कर सकते हैं।
    * `http.proxy` और `https.proxy` Git कॉन्फ़िगरेशन वेरिएबल सेट करें:
        ```bash
        git config --global http.proxy http://proxy_address:proxy_port
        git config --global https.proxy https://proxy_address:proxy_port
        ```
    * यदि प्रॉक्सी को ऑथेंटिकेशन की आवश्यकता है, तो आप यूजरनेम और पासवर्ड को प्रॉक्सी URL में शामिल कर सकते हैं।
7.  **वेबसॉकेट पर SSH का उपयोग करने का प्रयास करें:**
    * कुछ टूल और सेवाएं आपको वेबसॉकेट पर SSH ट्रैफिक टनल करने की अनुमति देती हैं, जो मददगार हो सकता है यदि स्टैंडर्ड SSH पोर्ट ब्लॉक हैं।
    * यह एक अधिक उन्नत तकनीक है और इसके लिए एक सर्वर सेट अप करने की आवश्यकता होती है जो वेबसॉकेट पर SSH का समर्थन करता हो।

**महत्वपूर्ण विचार:**

*   मोबाइल नेटवर्क की स्थितियां काफी भिन्न हो सकती हैं, इसलिए जो एक स्थान पर या एक समय में काम करता है, वह दूसरे में नहीं कर सकता।
*   अपने मोबाइल नेटवर्क प्रदाता की सेवा की शर्तों और VPN या अन्य वर्कअराउंड के उपयोग पर संभावित प्रतिबंधों से अवगत रहें।
*   अधिकांश उपयोगकर्ताओं के लिए HTTPS आम तौर पर सबसे अच्छा समाधान है।
