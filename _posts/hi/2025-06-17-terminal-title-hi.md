GNOME Terminal के टैब टाइटल को डिफॉल्ट फॉर्मेट (जैसे `lzw@lzw-MS:~/Projects/blog-server`) के बजाय सिर्फ करंट डायरेक्टरी (जैसे `blog-server`) दिखाने के लिए कस्टमाइज़ करने के लिए, आप टर्मिनल के प्रॉम्प्ट या टाइटल सेटिंग्स को मॉडिफाई कर सकते हैं। यहाँ बताया गया है कि इसे कैसे करें:

### GNOME Terminal टैब टाइटल को सिर्फ करंट डायरेक्टरी दिखाने के लिए सेट करने के स्टेप्स

1. **Bash कॉन्फ़िगरेशन फ़ाइल एडिट करें**:
   GNOME Terminal डिफॉल्ट रूप से Bash shell का उपयोग करता है, और टैब टाइटल अक्सर `PS1` (प्रॉम्प्ट) या किसी विशिष्ट टाइटल-सेटिंग कमांड से लिया जाता है। टाइटल को कस्टमाइज़ करने के लिए, आप `~/.bashrc` फ़ाइल को मॉडिफाई कर सकते हैं।

2. **टर्मिनल टाइटल मॉडिफाई करें**:
   अपने `~/.bashrc` में टर्मिनल टाइटल को करंट डायरेक्टरी पर सेट करने के लिए एक कमांड जोड़ें। फ़ाइल को एक टेक्स्ट एडिटर में खोलें:

   ```bash
   nano ~/.bashrc
   ```

   फ़ाइल के अंत में निम्नलिखित लाइनें जोड़ें:

   ```bash
   # टर्मिनल टैब टाइटल को करंट डायरेक्टरी पर सेट करें
   case "$TERM" in
   xterm*|rxvt*)
       PS1="\[\e]0;\W\a\]$PS1"
       ;;
   *)
       ;;
   esac
   ```

   **एक्सप्लेनेशन**:
   - `\e]0;...` टर्मिनल टाइटल सेट करता है।
   - `\W` करंट डायरेक्टरी का बेसनेम रिप्रेजेंट करता है (जैसे फुल पाथ `~/Projects/blog-server` के बजाय `blog-server`)।
   - `\a` टाइटल स्ट्रिंग को टर्मिनेट करने के लिए एक बेल कैरेक्टर है।
   - यह कोड टर्मिनल के `xterm`-कम्पेटिबल (जो GNOME Terminal है) होने की जांच करता है, बदलाव लागू करने से पहले।

3. **बदलाव लागू करें**:
   फ़ाइल सेव करें और Bash कॉन्फ़िगरेशन को रीलोड करें:

   ```bash
   source ~/.bashrc
   ```

   वैकल्पिक रूप से, बदलाव लागू करने के लिए टर्मिनल को बंद करें और फिर से खोलें।

4. **रिजल्ट वेरिफाई करें**:
   किसी डायरेक्टरी में नेविगेट करें (जैसे `cd ~/Projects/blog-server`), और अब टर्मिनल टैब टाइटल में सिर्फ `blog-server` दिखना चाहिए।

### वैकल्पिक: GNOME Terminal प्रोफाइल सेटिंग्स मॉडिफाई करें
अगर आप टाइटल को और कस्टमाइज़ करना चाहते हैं या `~/.bashrc` को एडिट करने से बचना चाहते हैं, तो आप GNOME Terminal की प्रोफाइल सेटिंग्स का उपयोग कर सकते हैं:

1. GNOME Terminal खोलें।
2. **Edit** > **Preferences** (या आपके वर्जन के अनुसार **Terminal** > **Preferences**) पर जाएं।
3. आपके द्वारा उपयोग की जा रही प्रोफाइल चुनें (जैसे “Unnamed” या कोई कस्टम प्रोफाइल)।
4. **Command** टैब पर जाएं।
5. **Title and Command** के अंतर्गत सेट करें:
   - **Initial title**: खाली छोड़ दें या कुछ जेनेरिक सेट करें।
   - **When command exits**: **Keep initial title** या **Replace with command output** चुनें।
6. **Command** सेक्शन में, **Run a custom command instead of my shell** इनेबल करें और कमांड को इस तरह सेट करें:

   ```bash
   bash --rcfile <(echo '. ~/.bashrc; PS1="\[\e]0;\W\a\]$PS1"')
   ```

   यह सुनिश्चित करता है कि टर्मिनल एक कस्टम प्रॉम्प्ट का उपयोग करता है जो टाइटल को करंट डायरेक्टरी पर सेट करता है।

7. डायलॉग बंद करें और नया टैब खोलकर या किसी डायरेक्टरी में नेविगेट करके टेस्ट करें।

### नोट्स
- अगर आप Bash के अलावा कोई और shell उपयोग करते हैं (जैसे Zsh), तो आपको समकक्ष कॉन्फ़िगरेशन फ़ाइल (जैसे `~/.zshrc`) को मॉडिफाई करना होगा और सिंटैक्स को उसी के अनुसार एडजस्ट करना होगा। Zsh के लिए, आप यह जोड़ सकते हैं:

  ```zsh
  precmd() { echo -ne "\e]0;${PWD##*/}\a" }
  ```

- अगर टाइटल अपेक्षा के अनुसार अपडेट नहीं होता है, तो सुनिश्चित करें कि आपका टर्मिनल एमुलेटर `xterm`-कम्पेटिबल है और कोई अन्य स्क्रिप्ट (जैसे `~/.bashrc` में कस्टम `PS1` सेटिंग्स) टाइटल को ओवरराइड नहीं कर रही है।
- सभी टर्मिनल्स में परसिस्टेंट बदलावों के लिए, सुनिश्चित करें कि कॉन्फ़िगरेशन यूजर की shell प्रोफाइल या GNOME Terminal सेटिंग्स में है।

इससे आपको सिर्फ करंट डायरेक्टरी नाम दिखाने वाला एक टैब टाइटल मिलना चाहिए, जैसे `blog-server`। अगर आपको और स्पष्टीकरण की आवश्यकता हो तो मुझे बताएं!
