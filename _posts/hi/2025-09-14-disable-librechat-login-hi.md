**LibreChat** को बार-बार लॉगिन की आवश्यकता के बिना काम कराने के लिए (या लॉगिन को पूरी तरह से अक्षम करने के लिए), आप प्रमाणीकरण सेटिंग्स को समायोजित कर सकते हैं। यहां समस्या को हल करने के चरण दिए गए हैं:

---

### **1. प्रमाणीकरण अक्षम करें (लॉगिन की आवश्यकता नहीं)**
यदि आप चाहते हैं कि **LibreChat पूरी तरह से खुला हो** (कोई लॉगिन नहीं), तो कॉन्फ़िगरेशन संशोधित करें:

#### **विकल्प A: Environment Variables (अनुशंसित)**
अपनी `.env` फ़ाइल (या environment variables) में निम्नलिखित सेट करें:
```env
# प्रमाणीकरण को पूरी तरह से अक्षम करें
AUTHENTICATION=false
```
फिर LibreChat को पुनरारंभ करें:
```bash
docker compose down && docker compose up -d  # Docker उपयोग कर रहे हैं तो
# OR
npm run build && npm run start  # स्थानीय रूप से चला रहे हैं तो
```

#### **विकल्प B: `config.yaml` संशोधित करें**
यदि आप `config.yaml` फ़ाइल का उपयोग कर रहे हैं, तो जोड़ें:
```yaml
auth:
  enabled: false
```

---

### **2. बार-बार लॉगआउट होने की समस्या ठीक करें (यदि आप अभी भी लॉगिन चाहते हैं)**
यदि आप **प्रमाणीकरण रखना चाहते हैं** लेकिन बार-बार लॉगआउट होने की समस्या को ठीक करना चाहते हैं, तो इन समाधानों को आज़माएं:

#### **A. सत्र/कुकी समय सीमा बढ़ाएँ**
`.env` में सत्र सेटिंग्स संशोधित करें:
```env
# सत्र समय सीमा बढ़ाएँ (सेकंड में, उदा., 7 दिन = 604800)
SESSION_EXPIRY=604800

# सुनिश्चित करें कि कुकीज़ सुरक्षित हैं (यदि HTTPS उपयोग कर रहे हैं)
COOKIE_SECURE=true  # HTTP पर परीक्षण कर रहे हैं तो false पर सेट करें
COOKIE_SAME_SITE=lax
```

#### **B. एक लगातार सत्र स्टोर का उपयोग करें**
डिफ़ॉल्ट रूप से, LibreChat मेमोरी-इन-सत्रों का उपयोग कर सकता है (जो पुनरारंभ पर रीसेट हो जाते हैं)। एक **डेटाबेस-बैक्ड सत्र स्टोर** (जैसे, Redis या PostgreSQL) पर स्विच करें:

1. **Redis इंस्टॉल करें** (यदि पहले से इंस्टॉल नहीं है):
   ```bash
   docker run -d --name redis -p 6379:6379 redis
   ```
2. `.env` अपडेट करें:
   ```env
   SESSION_STORE=redis
   REDIS_URL=redis://localhost:6379
   ```

#### **C. रिवर्स प्रॉक्सी सेटिंग्स जांचें (यदि Nginx/Apache उपयोग कर रहे हैं)**
यदि आप प्रॉक्सी के पीछे हैं (जैसे, Nginx), सुनिश्चित करें:
- कुकीज़ ठीक से फॉरवर्ड की जाती हैं।
- `ProxyPass` में `cookie` निर्देश शामिल हैं।

उदाहरण Nginx कॉन्फ़िग:
```nginx
location / {
    proxy_pass http://localhost:3080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_cookie_path / /;  # सुनिश्चित करें कि कुकीज़ गलत तरीके से स्कोप्ड नहीं हैं
}
```

#### **D. ब्राउज़र कैश साफ़ करें और इनकोग्निटो में परीक्षण करें**
कभी-कभी, ब्राउज़र कैश/दूषित कुकीज़ समस्याएं पैदा करती हैं। आज़माएं:
- `librechat.example.com` के लिए कुकीज़ साफ़ करना।
- **इनकोग्निटो मोड** या किसी अलग ब्राउज़र में परीक्षण करना।

---

### **3. कुकी समस्याओं का डीबगिंग**
यदि कुकीज़ अभी भी बनी नहीं रह रही हैं:
1. **ब्राउज़र DevTools जांचें** (`F12` > **Application** > **Cookies**) यह देखने के लिए कि क्या `connect.sid` (या इसी तरह) सेट हो रहा है।
2. **`SameSite` विशेषता सत्यापित करें** (`Lax` या क्रॉस-साइट होने पर `None` होना चाहिए)।
3. **सुनिश्चित करें कि `Domain` सही है** (उदा., सबडोमेन के लिए `.example.com`)।

---

### **4. LibreChat अपडेट करें**
यदि आप पुराने संस्करण पर हैं, तो नए रिलीज़ में बग्स ठीक हो सकते हैं:
```bash
git pull origin main
docker compose down && docker compose up -d --build
```

### **अंतिम नोट्स**
- **सुरक्षा चेतावनी**: प्रमाणीकरण अक्षम करना (`AUTHENTICATION=false`) LibreChat को **सार्वजनिक रूप से सुलभ** बना देता है। ऐसा केवल विश्वसनीय/आंतरिक नेटवर्क के लिए करें।
- **HTTPS आवश्यक**: यदि कुकीज़ उपयोग कर रहे हैं, तो सुनिश्चित करें कि आपकी साइट **HTTPS** है (या परीक्षण के लिए `COOKIE_SECURE=false` सेट करें)।
