### प्लेटफॉर्म सिफारिश
आपकी आवश्यकताओं का मूल्यांकन करने के बाद, Portkey ([Portkey AI](https://portkey.ai/)) सबसे उपयुक्त विकल्प प्रतीत होता है। यह आपको विभिन्न AI मॉडलों के लिए अपनी खुद की API कुंजियाँ जोड़ने की अनुमति देता है, जिससे प्रबंधन के लिए एक एकीकृत इंटरफेस मिलता है। हालाँकि, यह एक सदस्यता-आधारित मॉडल (उदाहरण के लिए, प्रो प्लान के लिए $49/माह) पर काम करता है, जिसका अर्थ है कि आप टोकन उपयोग के लिए AI प्रदाताओं को सीधे अपनी कुंजियों के माध्यम से भुगतान करने के अलावा, प्लेटफॉर्म का उपयोग करने के लिए एक निश्चित शुल्क का भुगतान करेंगे। यह $20/माह जैसे सदस्यता शुल्क से बचने की आपकी इच्छा से पूरी तरह मेल नहीं खा सकता है, लेकिन यह ऑब्जर्वेबिलिटी और प्रॉम्प्ट प्रबंधन जैसी उन्नत सुविधाएँ प्रदान करता है जो मूल्यवान हो सकती हैं।

यदि सदस्यता शुल्क से बचना सर्वाधिक महत्वपूर्ण है और आप अपनी खुद की API कुंजियों का उपयोग न करने का निर्णय लेते हैं, तो OpenRouter ([OpenRouter](https://openrouter.ai)) एक अन्य विकल्प है। यह प्रति माह 1000 टोकन की मुफ्त सीमा के बाद प्रति टोकन $0.0001 शुल्क लगाता है, और इसमें कोई सदस्यता शुल्क नहीं है, लेकिन आप अपनी खुद की कुंजियों के बजाय उनकी API का उपयोग करेंगे, जिसका अर्थ है कि आप मॉडल उपयोग के लिए उन्हें सीधे भुगतान करते हैं।

### अप्रत्याशित विवरण
एक अप्रत्याशित खोज यह है कि OpenRouter जैसे कई प्लेटफॉर्म, AI मॉडलों तक अपनी खुद की पहुँच प्रदान करते हैं, जिसके लिए उपयोगकर्ताओं को व्यक्तिगत API कुंजियों का उपयोग करने के बजाय प्लेटफॉर्म के माध्यम से भुगतान करना पड़ता है। इससे लागत और डेटा पर आपका नियंत्रण सीमित हो सकता है।

### सर्वे नोट: API कुंजी एकीकरण और मूल्य निर्धारण मॉडल के लिए AI प्लेटफॉर्म का विस्तृत विश्लेषण

यह विश्लेषण उन प्लेटफॉर्मों का पता लगाता है जो उपयोगकर्ताओं को अपनी खुद की AI API कुंजियाँ जोड़ने की अनुमति देते हैं और ChatBoxAI तथा OpenWebUI जैसे प्लेटफॉर्मों के विकल्प के रूप में, बिना सदस्यता शुल्क के, टोकन उपयोग पर आधारित पे-एज़-यू-गो मूल्य निर्धारण प्रदान करते हैं। शुक्रवार, 14 मार्च, 2025 को 02:42 AM PDT पर किए गए इस शोध का उद्देश्य, प्लेटफॉर्म मूल्य निर्धारण और कार्यक्षमता की जटिलताओं को ध्यान में रखते हुए, उपयोगकर्ता की विशिष्ट आवश्यकताओं को पूरा करना है।

#### पृष्ठभूमि और उपयोगकर्ता आवश्यकताएँ
उपयोगकर्ता के पास विभिन्न AI प्लेटफॉर्मों के लिए कई API कुंजियाँ हैं और वह एक ऐसा प्लेटफॉर्म चाहता है जो:
- एकीकृत इंटरफेस के लिए उनकी अपनी API कुंजियाँ जोड़ने की अनुमति दे।
- $20/माह जैसे सदस्यता शुल्क से बचते हुए, टोकन उपयोग पर आधारित पे-एज़-यू-गो मूल्य निर्धारण प्रदान करे।
- ChatBoxAI से बेहतर यूजर इंटरफेस (UI) प्रदान करे, जिसे उन्होंने खराब पाया था, और संभावित रूप से Mistral एकीकरण को शामिल करे।

इन आवश्यकताओं को देखते हुए, यह विश्लेषण उन प्लेटफॉर्मों की पहचान पर केंद्रित है जो इन मापदंडों को पूरा करते हैं, उनके मूल्य निर्धारण मॉडल को समझते हैं, और उनके UI और एकीकरण क्षमताओं का मूल्यांकन करते हैं।

#### प्लेटफॉर्म मूल्यांकन

##### ChatBoxAI और OpenWebUI संदर्भ
- **ChatBoxAI** ([ChatBox AI](https://chatboxai.app/en)) ChatGPT, Claude और अन्य जैसे AI मॉडलों के लिए एक डेस्कटॉप क्लाइंट है, जो उपयोगकर्ताओं को अपनी खुद की API कुंजियाँ जोड़ने की अनुमति देता है। इसके AI सेवा के लिए एक सदस्यता मॉडल है, जिसमें $20/माह का शुल्क शामिल हो सकता है, और उपयोगकर्ता द्वारा बताए गए Mistral एकीकरण की कमी के विपरीत, यह एकीकरण के माध्यम से Mistral का समर्थन करता है। उपयोगकर्ता द्वारा इसके UI को खराब बताया गया है।
- **OpenWebUI** ([Open WebUI](https://openwebui.com/)) AI मॉडलों के लिए एक ओपन-सोर्स, सेल्फ-होस्टेड इंटरफेस है, जो Ollama और OpenAI-संगत APIs जैसे विभिन्न LLM रनरों का समर्थन करता है। यह उपयोगकर्ताओं को अपनी खुद की API कुंजियाँ जोड़ने की अनुमति देता है और मुफ्त है, इसमें कोई सदस्यता शुल्क नहीं है, जो प्रदाता लागतों के माध्यम से पे-एज़-यू-गो मॉडल में फिट बैठता है। हालाँकि, उपयोगकर्ता इसके विकल्प तलाश रहे हैं।

##### संभावित प्लेटफॉर्म
कई प्लेटफॉर्मों का मूल्यांकन किया गया, जिसमें उपयोगकर्ता-प्रदत्त API कुंजियों को संभालने की उनकी क्षमता और मूल्य निर्धारण मॉडल पर ध्यान केंद्रित किया गया। मुख्य निष्कर्ष नीचे सारांशित हैं:

|--------------|--------------------------|------------------------------------------|-----------------|------------------------------------|

| OpenRouter   | नहीं                     | पे-पर-टोकन (1000 मुफ्त के बाद $0.0001/टोकन) | हाँ             | वेब-आधारित, सरल इंटरफेस           |

- **Portkey** ([Portkey AI](https://portkey.ai/)): यह प्लेटफॉर्म उपयोगकर्ताओं को अपने AI गेटवे के माध्यम से Mistral सहित 250 से अधिक LLMs के लिए अपनी खुद की API कुंजियाँ जोड़ने की अनुमति देता है। यह ऑब्जर्वेबिलिटी, प्रॉम्प्ट प्रबंधन, और मॉडल फॉलबैक जैसी सुविधाओं के साथ एक एकीकृत इंटरफेस प्रदान करता है। मूल्य निर्धारण सदस्यता-आधारित है, जिसमें मुफ्त स्टार्टर टियर, $49/माह प्रो टियर, और कस्टम बिजनेस मूल्य निर्धारण जैसे प्लान शामिल हैं। उपयोगकर्ता प्लेटफ़ॉर्म एक्सेस के लिए Portkey को और मॉडल उपयोग के लिए अपनी कुंजियों के माध्यम से सीधे प्रदाताओं को भुगतान करते हैं, जो सदस्यता शुल्क से बचने की इच्छा के अनुरूप नहीं हो सकता है। उपयोगकर्ता समीक्षाएँ इसकी उपयोग में आसानी और व्यापक सुविधाओं पर प्रकाश डालती हैं, जो ChatBoxAI के डेस्कटॉप क्लाइंट से बेहतर UI का सुझाव देती हैं।

- **OpenRouter** ([OpenRouter](https://openrouter.ai)): यह प्लेटफॉर्म कई LLMs के लिए एक एकीकृत API प्रदान करता है, जिसमें पे-पर-टोकन मूल्य निर्धारण (प्रति माह 1000 मुफ्त टोकन के बाद $0.0001/टोकन, कोई सदस्यता शुल्क नहीं) शामिल है। हालाँकि, यह उपयोगकर्ताओं को अपनी खुद की API कुंजियाँ जोड़ने की अनुमति नहीं देता है; इसके बजाय, उपयोगकर्ता OpenRouter की API का उपयोग करते हैं, और मॉडल उपयोग के लिए उन्हें सीधे भुगतान करते हैं। यह Mistral का समर्थन करता है और इसका एक वेब-आधारित इंटरफेस है जो सरलता के लिए जाना जाता है, जो संभावित रूप से ChatBoxAI से बेहतर UI प्रदान कर सकता है। यह पे-एज़-यू-गो मॉडल में फिट बैठता है लेकिन व्यक्तिगत API कुंजियों के उपयोग की आवश्यकता को पूरा नहीं करता है।

- **Unify.ai** ([Unify: Build AI Your Way](https://unify.ai/)): यह प्लेटफॉर्म AI वर्कफ़्लो बनाने पर केंद्रित है और "अपना खुद का मॉडल लाएँ" (BYOM) का उल्लेख करता है, जो उपयोगकर्ता-प्रदत्त मॉडलों के लिए संभावित समर्थन का सुझाव देता है। हालाँकि, इसका मूल्य निर्धारण और UI कम स्पष्ट हैं, और यह अधिक डेवलपर-उन्मुख प्रतीत होता है, संभवतः सदस्यता-आधारित लागतों के साथ। यह Mistral का समर्थन करता है, लेकिन यूजर-फेसिंग इंटरफेस के लिए इसकी उपयुक्तता अनिश्चित है, जिससे यह Portkey या OpenRouter की तुलना में कम उपयुक्त बनता है।

- **LiteLLM**: एक ओपन-सोर्स AI API प्रॉक्सी जो उपयोगकर्ताओं को अपनी खुद की API कुंजियाँ जोड़ने और एकीकृत API के माध्यम से उनका उपयोग करने की अनुमति देता है। यह मुफ्त है, इसमें कोई सदस्यता शुल्क नहीं है, और उपयोगकर्ता टोकन उपयोग के लिए अपनी कुंजियों के माध्यम से सीधे प्रदाताओं को भुगतान करते हैं। हालाँकि, इसमें यूजर-फेसिंग UI का अभाव है, जिससे यह ChatBoxAI या OpenWebUI जैसे सीधे उपयोगकर्ता संपर्क के लिए उपयुक्त नहीं है, बल्कि यह एप्लिकेशन में एकीकरण के लिए डेवलपर्स के लिए अधिक उपयुक्त है।

#### उपयोगकर्ता आवश्यकताओं के अनुरूप विश्लेषण
- **अपनी खुद की API कुंजियों का उपयोग**: Portkey और LiteLLM स्पष्ट रूप से इसकी अनुमति देते हैं, जबकि OpenRouter नहीं देता है, जिसके लिए उपयोगकर्ताओं को उनकी API का उपयोग करना होता है। Unify.ai की BYOM सुविधा अस्पष्ट है लेकिन कम उपयोगकर्ता-केंद्रित है।
- **टोकन उपयोग पर आधारित पे-एज़-यू-गो मूल्य निर्धारण**: OpenRouter अपने प्रति-टोकन मूल्य निर्धारण के साथ फिट बैठता है, लेकिन उपयोगकर्ता अपनी खुद की कुंजियों का उपयोग नहीं करते हैं। Portkey में एक सदस्यता मॉडल है, जो $20/माह जैसे शुल्क से बचने की उपयोगकर्ता की इच्छा का विरोधाभासी है। LiteLLM मुफ्त है, जो बिना सदस्यता के अनुरूप है, लेकिन इसमें UI का अभाव है। कोई भी प्लेटफॉर्म बिना समझौता किए सभी आवश्यकताओं को पूरी तरह से संयोजित नहीं करता है।
- **UI और Mistral एकीकरण**: Portkey और OpenRouter दोनों के पास वेब-आधारित UI हैं, जो संभावित रूप से ChatBoxAI के डेस्कटॉप क्लाइंट से बेहतर हैं, और दोनों Mistral का समर्थन करते हैं। उपयोगकर्ता समीक्षाएँ बताती हैं कि Portkey का UI उपयोगकर्ता-अनुकूल है, जबकि OpenRouter का सरल है। LiteLLM में UI के अभाव के कारण यह UI-केंद्रित आवश्यकताओं के लिए अनुपयुक्त है।

#### चुनौतियाँ और समझौते
मुख्य चुनौती उन प्लेटफॉर्मों की स्पष्ट कमी है जो उपयोगकर्ताओं को अपनी खुद की API कुंजियाँ जोड़ने, एक एकीकृत यूजर-फेसिंग इंटरफेस प्रदान करने, और बिना सदस्यता शुल्क के टोकन उपयोग पर आधारित शुल्क लेने की अनुमति देते हैं। अधिकांश प्लेटफॉर्म या तो सदस्यता शुल्क लगाते हैं (जैसे Portkey) या व्यक्तिगत API कुंजियों की अनुमति नहीं देते हैं (जैसे OpenRouter)। यह एक समझौते का सुझाव देता है:
- प्लेटफ़ॉर्म सुविधाओं के लिए एक सदस्यता शुल्क स्वीकार करें (Portkey) जबकि अपनी कुंजियों का उपयोग जारी रखें।
- बिना अपनी कुंजियों के पे-पर-टोकन मॉडल का उपयोग करें (OpenRouter), जिससे व्यक्तिगत कुंजियों के माध्यम से लागत नियंत्रण संभवतः खो सकता है।

#### सिफारिश और विचार
अपनी खुद की API कुंजियों के उपयोग और सदस्यता शुल्क से बचने पर उपयोगकर्ता के जोर को देखते हुए, कोई सही मेल नहीं पाया गया। हालाँकि, **Portkey** को निकटतम फिट के रूप में सिफारिश की जाती है, जो व्यक्तिगत API कुंजियों को जोड़ने की अनुमति देता है और ChatBoxAI से संभावित रूप से बेहतर UI के साथ एक एकीकृत इंटरफेस प्रदान करता है। सदस्यता शुल्क (जैसे, प्रो के लिए $49/माह) के बावजूद, यह उन्नत सुविधाएँ प्रदान करता है जो कई AI मॉडलों के प्रबंधन के लिए लागत को उचित ठहरा सकती हैं। उपयोगकर्ताओं को यह मूल्यांकन करना चाहिए कि क्या सुविधाएँ सदस्यता लागत से अधिक महत्वपूर्ण हैं, खासकर बिना किसी प्लेटफॉर्म के सीधे प्रदाताओं को कुंजियों के माध्यम से भुगतान करने की तुलना में।

जो लोग सदस्यता के बिना पे-पर-टोकन को प्राथमिकता देते हैं और व्यक्तिगत कुंजियों का उपयोग न करने को तैयार हैं, उनके लिए **OpenRouter** एक विकल्प है, जो 1000 मुफ्त के बाद प्रति टोकन $0.0001 शुल्क लगाता है, जिसमें एक सरल वेब UI और Mistral समर्थन शामिल है। हालाँकि, इसका मतलब है कि API कुंजियों पर नियंत्रण छोड़ना, जो उपयोगकर्ता के कई कुंजियों के स्वामित्व के अनुरूप नहीं हो सकता है।

#### निष्कर्ष
यह विश्लेषण उस जटिलता पर प्रकाश डालता है जो उपयोगकर्ता की सभी कसौटियों को पूरा करने वाले प्लेटफॉर्म को खोजने में exists है। Portkey सदस्यता लागत पर ही सही, व्यक्तिगत API कुंजियों के साथ एक एकीकृत इंटरफेस के लिए सबसे अच्छा फिट प्रदान करता है। OpenRouter सदस्यता के बिना एक पे-पर-टोकन मॉडल प्रदान करता है लेकिन व्यक्तिगत कुंजियों का समर्थन नहीं करता है। उपयोगकर्ताओं को यह तय करने की आवश्यकता हो सकती है कि प्लेटफॉर्म सुविधाएँ या लागत लचीलापन कौन सा अधिक महत्वपूर्ण है, और संभवतः व्यक्तिगत कुंजियों के साथ मुफ्त उपयोग के लिए OpenWebUI के सेल्फ-होस्टेड विकल्पों जैसे ओपन-सोर्स विकल्पों का पता लगाया जा सकता है।
