नहीं, LeanCloud अपने सार्वजनिक API को "Uluru API" के नाम से नहीं जानता है। उनके आधिकारिक APIs को LeanCloud REST API (डेटा स्टोरेज, क्वेरीज़ आदि के लिए), Cloud Engine REST API (सर्वर-साइड फ़ंक्शंस और हुक्स के लिए) और पुश नोटिफिकेशन या फ़ाइल स्टोरेज जैसे समान सेवा-विशिष्ट एंडपॉइंट्स के रूप में संदर्भित किया जाता है। इन्हें उनकी डेवलपर साइट (docs.leancloud.cn) पर बिना "Uluru" के किसी संदर्भ के विस्तार से दस्तावेज़ीकृत किया गया है।

हालाँकि, "Uluru" उनके उत्पाद या SDK विकास से जुड़ा एक आंतरिक या प्रारंभिक कोडनेम प्रतीत होता है। यह **uluru-js-sdk** नामक एक संग्रहीत npm पैकेज (संस्करण 0.1.0, लगभग 12 साल पहले "avos" प्रकाशक अकाउंट के तहत प्रकाशित) पर आधारित है। AVOS Cloud, LeanCloud का पुनः ब्रांडिंग से पहले का मूल नाम था, और इस पैकेज को "PAAS" (प्लेटफ़ॉर्म एज़ ए सर्विस) के लिए एक "JavaScript SDK" के रूप में वर्णित किया गया है, जो LeanCloud की बैकएंड सेवाओं के अनुरूप है। यह डेटा ऑपरेशन्स जैसी मुख्य सुविधाओं के लिए ब्राउज़र और Node.js दोनों वातावरणों का समर्थन करता है—जो LeanCloud की प्रारंभिक पेशकशों से मेल खाता है—लेकिन इसे कभी भी मुख्य SDK के रूप में सार्वजनिक रूप से प्रचारित नहीं किया गया। वर्तमान SDKs को "LeanCloud SDK" (जैसे Java, Python, JavaScript आदि के लिए) के रूप में ब्रांडेड किया गया है और GitHub पर leancloud/ के अंतर्गत उपलब्ध हैं।

यदि यह किसी अधिक हालिया या विशिष्ट चीज़ (जैसे कि एक निजी/आंतरिक टूल) को संदर्भित करता है, तो यह सार्वजनिक रूप से दस्तावेज़ीकृत नहीं है। आधिकारिक API विवरण के लिए, [LeanCloud Docs](https://docs.leancloud.cn/en/) देखें।
