Azure DevOps, Microsoft द्वारा प्रदान किए गए क्लाउड-आधारित विकास टूल्स का एक व्यापक सेट है, जो टीमों को सॉफ़्टवेयर को कुशलतापूर्वक योजना बनाने, विकसित करने, परीक्षण करने और डिलीवर करने में मदद करता है। यह अन्य Microsoft सेवाओं जैसे Azure और GitHub के साथ निर्बाध रूप से एकीकृत होता है, और विभिन्न पद्धतियों जैसे कि एजाइल, स्क्रम और DevOps प्रथाओं को सपोर्ट करता है। यह गाइड मूल बातें कवर करती है: यह क्या है, मुख्य घटक, शुरुआत कैसे करें, सर्वोत्तम प्रथाएं, और गहन सीखने के लिए संसाधन।

## Azure DevOps क्या है?
Azure DevOps एंड-टू-एंड DevOps क्षमताएं प्रदान करता है, जो विकास, संचालन और हितधारकों में सहयोग को सक्षम बनाता है। यह प्लेटफ़ॉर्म-अज्ञेयवादी है, और कई भाषाओं, फ्रेमवर्क और टूल्स को सपोर्ट करता है। मुख्य लाभों में शामिल हैं:
- **स्केलेबिलिटी**: किसी भी आकार की परियोजनाओं को संभालता है, छोटी टीमों से लेकर उद्यमों तक।
- **एकीकरण**: Visual Studio, GitHub, Slack, और Jira जैसे IDE के साथ जुड़ता है।
- **सुरक्षा**: रोल-आधारित एक्सेस कंट्रोल (RBAC) और ऑडिट लॉग जैसी अंतर्निहित अनुपालन सुविधाएँ।
- **मूल्य निर्धारण**: 5 उपयोगकर्ताओं तक मुफ्त; अतिरिक्त सुविधाओं के लिए भुगतान योजनाएं $6/उपयोगकर्ता/माह से शुरू।

2025 तक, Azure DevOps ने उन्नत AI एकीकरण (जैसे, GitHub Copilot for Azure) और बेहतर पाइपलाइन एनालिटिक्स के साथ विकसित किया है।

## मुख्य घटक
Azure DevOps में पाँच मुख्य सेवाएँ शामिल हैं, जिनमें से प्रत्येक वेब पोर्टल या API के माध्यम से सुलभ हैं:

### 1. **Boards**
   - **उद्देश्य**: कार्य आइटम्स के लिए दृश्य योजना और ट्रैकिंग टूल।
   - **विशेषताएँ**:
     - वर्कफ़्लो को विज़ुअलाइज़ करने के लिए कानबान बोर्ड।
     - कार्यों को प्राथमिकता देने के लिए बैकलॉग।
     - एजाइल पुनरावृत्तियों के लिए स्प्रिंट।
     - कस्टम रिपोर्टिंग के लिए क्वेरीज़।
   - **उपयोग मामला**: बग्स, फ़ीचर्स और कार्यों को रीयल-टाइम में ट्रैक करें।

### 2. **Repos**
   - **उद्देश्य**: कोड के लिए केंद्रीकृत वर्जन कंट्रोल।
   - **विशेषताएँ**:
     - Git या TFVC रिपॉजिटरी।
     - ब्रांचिंग स्ट्रैटेजी और पुल रिक्वेस्ट।
     - डॉक्युमेंटेशन के लिए विकी एकीकरण।
   - **उपयोग मामला**: कोड रिव्यू पर सहयोग करें और इतिहास बनाए रखें।

### 3. **Pipelines**
   - **उद्देश्य**: CI/CD (निरंतर एकीकरण/निरंतर तैनाती) स्वचालन।
   - **विशेषताएँ**:
     - YAML-आधारित या क्लासिक पाइपलाइन।
     - मल्टी-स्टेज बिल्ड, टेस्ट और डिप्लॉयमेंट।
     - पैकेज मैनेजमेंट के लिए Azure Artifacts के साथ एकीकरण।
     - अनुमोदन और गेट के लिए एनवायरनमेंट।
   - **उपयोग मामला**: प्रत्येक कमिट के लिए बिल्ड स्वचालित करें और क्लाउड या ऑन-प्रिमाइसेस पर डिप्लॉय करें।

### 4. **Test Plans**
   - **उद्देश्य**: मैनुअल और एक्सप्लोरेटरी टेस्टिंग।
   - **विशेषताएँ**:
     - टेस्ट केस मैनेजमेंट।
     - लाइव लॉग और अटैचमेंट।
     - पाइपलाइन से स्वचालित टेस्ट के साथ एकीकरण।
   - **उपयोग मामला**: रिलीज़ से पहले गुणवत्ता सुनिश्चित करें।

### 5. **Artifacts**
   - **उद्देश्य**: पैकेज मैनेजमेंट और निर्भरता हैंडलिंग।
   - **विशेषताएँ**:
     - यूनिवर्सल पैकेज, NuGet, npm, और Maven फ़ीड।
     - बाइनरी के लिए रिटेंशन पॉलिसी।
   - **उपयोग मामला**: टीमों में लाइब्रेरी साझा करें और वर्जन करें।

## शुरुआत कैसे करें
Azure DevOps सेट अप करने के लिए इन चरणों का पालन करें:

1. **एक अकाउंट बनाएँ**:
   - [dev.azure.com](https://dev.azure.com) पर जाएं और एक Microsoft अकाउंट के साथ साइन अप करें (मुफ्त टियर उपलब्ध)।
   - एक नया संगठन बनाएँ (उदाहरण के लिए, "MyProjectOrg")।

2. **एक प्रोजेक्ट सेट अप करें**:
   - अपने संगठन में, "New Project" पर क्लिक करें।
   - दृश्यता (निजी/सार्वजनिक) और वर्जन कंट्रोल (Git/TFVC) चुनें।
   - ईमेल आमंत्रण के माध्यम से टीम के सदस्यों को जोड़ें।

3. **Repos कॉन्फ़िगर करें**:
   - डिफ़ॉल्ट रिपो को क्लोन करें: `git clone https://dev.azure.com/{org}/{project}/_git/{repo}`।
   - अपना प्रारंभिक कोड पुश करें: `git add . && git commit -m "Initial commit" && git push`।

4. **एक सरल पाइपलाइन बनाएँ**:
   - Pipelines > New Pipeline > Select repo > ASP.NET (या आपका फ्रेमवर्क) में जाएं।
   - सरलता के लिए YAML का उपयोग करें:
     ```yaml
     trigger:
     - main
     pool:
       vmImage: 'ubuntu-latest'
     steps:
     - task: DotNetCoreCLI@2
       inputs:
         command: 'build'
         projects: '**/*.csproj'
     ```
   - पाइपलाइन को सेव करें और रन करें।

5. **एक बोर्ड बनाएँ**:
   - Boards > Sprints > New Query पर जाएं।
   - कार्य आइटम प्रकार परिभाषित करें (उदाहरण के लिए, Epic > Feature > Task)।

6. **परीक्षण और तैनाती**:
   - अपनी पाइपलाइन में एक टेस्ट टास्क जोड़ें।
   - Azure App Service पर डिप्लॉय करने के लिए एक रिलीज़ पाइपलाइन सेट अप करें।

हाथों-हाथ ट्यूटोरियल के लिए, आधिकारिक क्विकस्टार्ट से शुरुआत करें।

## सर्वोत्तम प्रथाएं
- **YAML पाइपलाइन अपनाएँ**: ये वर्जन-नियंत्रित और पुन: प्रयोज्य हैं।
- **ब्रांच पॉलिसी का उपयोग करें**: पुल रिक्वेस्ट अनुमोदन और लिंक्ड कार्य आइटम की आवश्यकता होती है।
- **सुरक्षा स्कान लागू करें**: SonarQube या Microsoft Defender जैसे टूल एकीकृत करें।
- **एनालिटिक्स के साथ मॉनिटर करें**: वेलोसिटी और बिल्ड सफलता दर के लिए डैशबोर्ड का उपयोग करें।
- **एक्सटेंशन के साथ स्केल करें**: मार्केटप्लेस में 2,000+ एक्सटेंशन हैं (जैसे, Slack नोटिफिकेशन)।
- **हाइब्रिड सपोर्ट**: ऑन-प्रिमाइसेस के लिए, Azure DevOps Server (सेल्फ-होस्टेड) का उपयोग करें।

सामान्य गलतियाँ: जल्दी अति-अनुकूलन करना—सरल शुरुआत करें और पुनरावृति करें।

## उन्नत विषय
- **एक्सटेंशन और मार्केटप्लेस**: Jenkins या Terraform जैसे टूल्स के लिए प्लगइन्स के साथ कस्टमाइज़ करें।
- **AI/ML एकीकरण**: पाइपलाइन में मॉडल डिप्लॉयमेंट के लिए Azure ML का उपयोग करें।
- **मल्टी-टीम सहयोग**: बड़े संगठनों के लिए एरिया और इटरेशन का लाभ उठाएं।
- **माइग्रेशन**: GitHub या Jira से आयात करने के लिए टूल।
