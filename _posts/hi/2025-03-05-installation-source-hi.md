`@ngx-formly/bootstrap` पैकेज के इंस्टालेशन स्रोत को लोकल `.tgz` फ़ाइल से npm रजिस्ट्री में बदलने के लिए, इन चरणों का पालन करें:

### 1. लोकल इंस्टॉल किए गए पैकेज को अनइंस्टॉल करें
चूंकि आपने पहले `@ngx-formly/bootstrap` को एक लोकल `.tgz` फ़ाइल से इंस्टॉल किया था, यह वर्तमान में आपकी प्रोजेक्ट की `node_modules` डायरेक्टरी में मौजूद है। इसे हटाने के लिए, अपने टर्मिनल में निम्नलिखित कमांड चलाएं:

```bash
npm uninstall @ngx-formly/bootstrap
```

यह कमांड पैकेज को `node_modules` से हटा देती है। यदि पैकेज को आपकी `package.json` फ़ाइल में `dependencies` या `devDependencies` के तहत मैन्युअल रूप से जोड़ा गया था, तो यह उसे वहां से भी हटा देगी। यदि यह `package.json` में सूचीबद्ध नहीं था (आमतौर पर तब होता है जब बिना सेव किए `.tgz` फ़ाइल से इंस्टॉल किया जाता है), तो भी यह आपकी प्रोजेक्ट से हट जाएगा।

### 2. पैकेज को npm रजिस्ट्री से इंस्टॉल करें
अब, पैकेज को npm रजिस्ट्री से इंस्टॉल करने के लिए यह कमांड चलाएं:

```bash
npm install @ngx-formly/bootstrap
```

यह कमांड npm रजिस्ट्री (आमतौर पर `https://registry.npmjs.org/` जब तक कि आपने कोई अलग रजिस्ट्री कॉन्फ़िगर नहीं की है) से `@ngx-formly/bootstrap` का नवीनतम वर्जन लाती है और इसे आपकी `node_modules` डायरेक्टरी में इंस्टॉल कर देती है। यह पैकेज को आपकी `package.json` फ़ाइल में `dependencies` के तहत (हाल के npm वर्जन में) भी जोड़ देगी, यह सुनिश्चित करते हुए कि इसे आपकी प्रोजेक्ट के हिस्से के रूप में ट्रैक किया जाता है।

### वैकल्पिक: रजिस्ट्री सत्यापित करें (यदि आवश्यक हो)
यदि आपने पहले अपनी npm रजिस्ट्री बदली है और यह सुनिश्चित करना चाहते हैं कि यह डिफ़ॉल्ट npm रजिस्ट्री पर सेट है, तो आप इसे इस कमांड से चेक कर सकते हैं:

```bash
npm config get registry
```

यदि आउटपुट `https://registry.npmjs.org/` है, तो आप डिफ़ॉल्ट रजिस्ट्री का उपयोग कर रहे हैं, और `npm install` कमांड अपेक्षा के अनुरूप काम करेगी। यदि यह किसी अलग रजिस्ट्री पर सेट है और आप डिफ़ॉल्ट पर वापस जाना चाहते हैं, तो यह कमांड चलाएं:

```bash
npm config set registry https://registry.npmjs.org/
```

हालांकि, चूंकि आपने उल्लेख किया है कि आपने पहले ही अपनी रजिस्ट्री कॉन्फ़िगर कर ली है, इसलिए यह चरण आवश्यक नहीं हो सकता है।

### नोट्स
- **वर्जन कंट्रोल**: यदि आपको अपनी `.tgz` फ़ाइल में मौजूद वर्जन के समान ही वर्जन चाहिए, तो अनइंस्टॉल करने से पहले `node_modules/@ngx-formly/bootstrap/package.json` में वर्जन नंबर चेक करें। फिर, उस विशिष्ट वर्जन को इंस्टॉल करें:
  ```bash
  npm install @ngx-formly/bootstrap@<version>
  ```
  यदि आप कोई वर्जन निर्दिष्ट नहीं करते हैं, तो रजिस्ट्री से नवीनतम वर्जन इंस्टॉल हो जाएगा।
- **डिपेंडेंसी टाइप**: यदि आप पैकेज को रेगुलर डिपेंडेंसी के बजाय डेवलपमेंट डिपेंडेंसी के रूप में चाहते हैं, तो इसका उपयोग करें:
  ```bash
  npm install @ngx-formly/bootstrap --save-dev
  ```

इन चरणों का पालन करके, आप `@ngx-formly/bootstrap` को लोकल `.tgz` फ़ाइल के माध्यम से इंस्टॉल होने से npm रजिस्ट्री से इंस्टॉल होने में सफलतापूर्वक बदल देंगे।
