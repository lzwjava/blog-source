`@ngx-formly/bootstrap` पैकेज को एक स्थानीय `.tgz` फ़ाइल से npm रजिस्ट्री में बदलने के लिए, निम्नलिखित चरणों का पालन करें:

### 1. स्थानीय रूप से स्थापित पैकेज को अनइंस्टॉल करें
जिस समय आपने पहले `@ngx-formly/bootstrap` को एक स्थानीय `.tgz` फ़ाइल से स्थापित किया था, वह अभी भी आपके प्रोजेक्ट के `node_modules` डायरेक्टरी में है। इसे हटाने के लिए, अपने टर्मिनल में निम्नलिखित कमांड चलाएं:

```bash
npm uninstall @ngx-formly/bootstrap
```

यह कमांड पैकेज को `node_modules` से हटाता है। यदि पैकेज को `package.json` फ़ाइल में `dependencies` या `devDependencies` के तहत मैन्युअल रूप से जोड़ा गया था, तो यह उसे वहाँ से भी हटाएगा। यदि यह `package.json` में सूचीबद्ध नहीं था (जिसे आम तौर पर एक `.tgz` फ़ाइल से स्थापित करने के दौरान बिना बचाए), तो यह भी आपके प्रोजेक्ट से हट जाएगा।

### 2. npm रजिस्ट्री से पैकेज को स्थापित करें
अब, निम्नलिखित कमांड चलाकर npm रजिस्ट्री से पैकेज को स्थापित करें:

```bash
npm install @ngx-formly/bootstrap
```

यह कमांड `@ngx-formly/bootstrap` का नवीनतम संस्करण npm रजिस्ट्री (आम तौर पर `https://registry.npmjs.org/` के अलावा, यदि आपने एक अलग रजिस्ट्री का विन्यास किया है) से प्राप्त करता है और इसे आपके `node_modules` डायरेक्टरी में स्थापित करता है। यह पैकेज को भी आपकी `package.json` फ़ाइल में `dependencies` के तहत डिफ़ॉल्ट रूप से जोड़ता है (नवीनतम npm संस्करणों में), जिससे यह आपके प्रोजेक्ट का हिस्सा के रूप में ट्रैक किया जाता है।

### विकल्प: रजिस्ट्री की पुष्टि करें (यदि आवश्यक हो)
अगर आपने पहले अपना npm रजिस्ट्री बदल दिया है और सुनिश्चित करना चाहते हैं कि यह डिफ़ॉल्ट npm रजिस्ट्री पर सेट है, तो आप इसे निम्नलिखित कमांड के साथ चेक कर सकते हैं:

```bash
npm config get registry
```

अगर आउटपुट `https://registry.npmjs.org/` है, तो आप डिफ़ॉल्ट रजिस्ट्री का उपयोग कर रहे हैं, और `npm install` कमांड अपेक्षित रूप से काम करेगा। अगर यह एक अलग रजिस्ट्री पर सेट है और आप इसे डिफ़ॉल्ट पर वापस करना चाहते हैं, तो निम्नलिखित कमांड चलाएं:

```bash
npm config set registry https://registry.npmjs.org/
```

हालांकि, क्योंकि आपने पहले अपना रजिस्ट्री का विन्यास किया है, यह चरण आवश्यक नहीं हो सकता है।

### नोट्स
- **वर्जन नियंत्रण**: अगर आपको उसी संस्करण की आवश्यकता है जो आपके `.tgz` फ़ाइल में है, तो अनइंस्टॉल करने से पहले `node_modules/@ngx-formly/bootstrap/package.json` में वर्जन नंबर की जांच करें। फिर, निम्नलिखित कमांड के साथ उस विशेष संस्करण को स्थापित करें:
  ```bash
  npm install @ngx-formly/bootstrap@<version>
  ```
  अगर आप एक वर्जन नहीं स्पेसिफाई करते, तो रजिस्ट्री से नवीनतम संस्करण स्थापित होगा।
- **डिपेंडेंसी प्रकार**: अगर आप पैकेज को एक विकास डिपेंडेंसी के रूप में चाहते हैं, बजाय एक सामान्य डिपेंडेंसी के, तो निम्नलिखित कमांड का उपयोग करें:
  ```bash
  npm install @ngx-formly/bootstrap --save-dev
  ```

इन चरणों का पालन करके, आप सफलतापूर्वक `@ngx-formly/bootstrap` को एक स्थानीय `.tgz` फ़ाइल से स्थापित होने से npm रजिस्ट्री से स्थापित होने में बदलेंगे।
