ऐसा लगता है कि आप Eclipse के Liberty Tools प्लगइन से IntelliJ IDEA में रिमोट डीबगिंग के लिए स्विच कर रहे हैं, और आप WebSphere Liberty सर्वर में फ्रंटएंड एक्शन्स द्वारा ट्रिगर किए गए नए लॉग्स को देखने, साफ़ करने और फ़िल्टर करने की क्षमता को दोहराना चाहते हैं, ठीक वैसे ही जैसे आपने Eclipse में कंसोल का उपयोग किया था या VS Code में लॉग्स की निगरानी की थी। आप विशेष रूप से पूछ रहे हैं कि क्या IntelliJ IDEA रिमोट लॉगिंग को सपोर्ट करता है और फ्रंटएंड क्लिक्स द्वारा जेनरेट किए गए नए लॉग्स को कुशलतापूर्वक देखने और प्रबंधित करने का तरीका, जैसा आपने VS Code में लॉग फ़ाइल के अंत में कर्सर रखकर किया था।

नीचे, मैं समझाऊंगा कि कैसे IntelliJ IDEA को WebSphere Liberty सर्वर से रिमोट लॉग्स को देखने और प्रबंधित करने के लिए सेट अप किया जाए, एक रिमोट डीबगिंग सत्र के दौरान, जिसमें लॉग्स साफ़ करना और फ्रंटएंड एक्शन्स द्वारा ट्रिगर किए गए नए लॉग्स पर फोकस करना शामिल है। मैं यह भी बताऊंगा कि नए लॉग्स को सेलेक्ट करने के लिए आपके VS Code सेटअप के समान वर्कफ़्लो को कैसे प्राप्त किया जाए।

### आपकी आवश्यकताओं को समझना
1.  **Eclipse Liberty Tools का व्यवहार**: Eclipse में Liberty Tools प्लगइन के साथ, आपने WebSphere Liberty सर्वर से लॉग्स देखने, उन्हें साफ़ करने और फ्रंटएंड इंटरैक्शन द्वारा ट्रिगर किए गए नए लॉग्स को ऑब्ज़र्व करने के लिए एक कंसोल व्यू का उपयोग किया था।
2.  **VS Code वर्कफ़्लो**: आपने VS Code में WebSphere Liberty (`wlp`) डायरेक्टरी को खोला, एक लॉग फ़ाइल (जैसे `messages.log`) के अंत में कर्सर रखा, और फ्रंटएंड के साथ इंटरैक्ट करते समय ऐपेंड किए गए नए लॉग्स को आसानी से सेलेक्ट या देख सकते थे।
3.  **IntelliJ IDEA का लक्ष्य**: आपने IntelliJ IDEA में रिमोट डीबगिंग सेट अप कर ली है और चाहते हैं:
    - रिमोट WebSphere Liberty सर्वर से लॉग्स को रियल-टाइम में देखना।
    - लॉग्स को साफ़ करना या फ्रंटएंड एक्शन्स द्वारा ट्रिगर किए गए नए लॉग्स पर फोकस करना।
    - VS Code की तरह नए लॉग्स को सेलेक्ट करने की सुविधा को दोहराना।

### क्या IntelliJ IDEA रिमोट लॉगिंग को सपोर्ट करता है?
हां, IntelliJ IDEA एक रिमोट डीबगिंग सत्र के दौरान, WebSphere Liberty सहित, रिमोट सर्वर से लॉग्स देखने को सपोर्ट करता है। हालांकि, Eclipse के Liberty Tools प्लगइन की तरह नहीं, जो Liberty सर्वर लॉग्स के लिए एक समर्पित कंसोल प्रदान करता है, IntelliJ IDEA को रिमोट लॉग्स को **Run** या **Debug** टूल विंडो में दिखाने के लिए मैन्युअल कॉन्फ़िगरेशन की आवश्यकता होती है। आप इसे **Run/Debug Configuration** में **Logs टैब** को कॉन्फ़िगर करके या रिमोट लॉग फ़ाइलों को टेल करने के लिए एक्सटर्नल टूल्स को इंटीग्रेट करके प्राप्त कर सकते हैं। IntelliJ IDEA आपको लॉग्स को साफ़ करने और नए एंट्रीज़ को फ़िल्टर करने की भी अनुमति देता है, हालांकि अनुभव Eclipse या VS Code से अलग है।

### IntelliJ IDEA में रिमोट लॉगिंग सेट अप करना
अपने Eclipse और VS Code वर्कफ़्लो को दोहराने के लिए, आपको IntelliJ IDEA को रिमोट WebSphere Liberty सर्वर की लॉग फ़ाइलों (जैसे `wlp/usr/servers/<serverName>/logs` डायरेक्टरी में `messages.log` या `console.log`) तक पहुंचने और उन्हें डिस्प्ले करने के लिए कॉन्फ़िगर करने की आवश्यकता है। यहां बताया गया है कि इसे कैसे करें:

#### चरण 1: रिमोट डीबगिंग कॉन्फ़िगर करें
चूंकि आपने पहले ही IntelliJ IDEA में रिमोट डीबगिंग सेट अप कर ली है, मैं मानूंगा कि आपके पास एक **Remote JVM Debug** कॉन्फ़िगरेशन है। यदि नहीं, तो यहां एक त्वरित पुनर्कथन है:
1.  **Run > Edit Configurations** पर जाएं।
2.  **+** आइकन पर क्लिक करें और **Remote JVM Debug** सेलेक्ट करें।
3.  निम्नलिखित सेट करें:
    - **Name**: उदा., "Liberty Remote Debug".
    - **Host**: रिमोट सर्वर का पता (उदा., `localhost` या एक IP जैसे `192.168.1.100`).
    - **Port**: डीबग पोर्ट (Liberty के लिए डिफ़ॉल्ट अक्सर `7777` होता है, जब तक कस्टमाइज़ न किया गया हो)।
    - **Command-line arguments for remote JVM**: जेनरेट किए गए आर्ग्युमेंट्स (उदा., `-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:7777`) को कॉपी करें और सुनिश्चित करें कि वे Liberty सर्वर के JVM पर एप्लाई किए गए हैं।
4.  कॉन्फ़िगरेशन को एप्लाई और सेव करें।
5.  Liberty सर्वर को डीबग आर्ग्युमेंट्स के साथ स्टार्ट करें (उदा., `jvm.options` को मॉडिफ़ाई करें या `server debug` कमांड का उपयोग करें)।

#### चरण 2: IntelliJ IDEA में लॉग फ़ाइल डिस्प्ले कॉन्फ़िगर करें
IntelliJ IDEA की Debug टूल विंडो में रिमोट लॉग्स देखने के लिए, आपको Run/Debug Configuration में लॉग फ़ाइल लोकेशन निर्दिष्ट करने की आवश्यकता है। चूंकि लॉग एक रिमोट सर्वर पर हैं, आपको माउंटेड फ़ोल्डर, SSH, या प्लगइन के माध्यम से उन तक पहुंचने की आवश्यकता होगी।

**विकल्प 1: माउंटेड फ़ोल्डर या लोकल कॉपी के माध्यम से लॉग्स तक पहुंचें**
यदि रिमोट सर्वर की लॉग डायरेक्टरी एक्सेसिबल है (उदा., नेटवर्क शेयर, SFTP, या लोकल कॉपी के माध्यम से), तो आप लॉग्स दिखाने के लिए IntelliJ को कॉन्फ़िगर कर सकते हैं:
1.  **लॉग्स को माउंट या कॉपी करें**:
    - रिमोट सर्वर की लॉग डायरेक्टरी (उदा., `wlp/usr/servers/<serverName>/logs`) को SSHFS, NFS, या किसी अन्य विधि का उपयोग करके अपनी लोकल मशीन पर माउंट करें।
    - वैकल्पिक रूप से, अपनी लोकल मशीन पर `messages.log` या `console.log` को समय-समय पर कॉपी करने के लिए `rsync` या `scp` जैसे टूल का उपयोग करें।
2.  **Run/Debug Configuration में लॉग फ़ाइल एड करें**:
    - **Run > Edit Configurations** पर जाएं और अपनी Remote JVM Debug कॉन्फ़िगरेशन को सेलेक्ट करें।
    - **Logs** टैब खोलें।
    - **+** आइकन पर क्लिक करके एक लॉग फ़ाइल एड करें।
    - निर्दिष्ट करें:
      - **Log file location**: लॉग फ़ाइल का पथ (उदा., `/path/to/mounted/wlp/usr/servers/defaultServer/logs/messages.log`).
      - **Alias**: लॉग टैब के लिए एक नाम (उदा., “Liberty Logs”).
      - **Show all files coverable by pattern**: केवल नवीनतम लॉग फ़ाइल दिखाने के लिए इसे अनचेक करें (रोलिंग लॉग्स जैसे `messages.log` के लिए उपयोगी)।
      - **Skip Content**: केवल वर्तमान रन से नए लॉग एंट्रीज़ दिखाने के लिए इसे चेक करें, Eclipse में लॉग्स साफ़ करने के समान।
    - **Apply** और **OK** पर क्लिक करें।
3.  **डीबगर रन करें**:
    - कॉन्फ़िगरेशन को सेलेक्ट करके और **Debug** बटन पर क्लिक करके रिमोट डीबग सत्र स्टार्ट करें।
    - **Debug** टूल विंडो में एक नया टैब (उदा., “Liberty Logs”) दिखाई देगा, जो लॉग फ़ाइल की सामग्री दिखाता है।
    - फ्रंटएंड क्लिक्स द्वारा ट्रिगर किए गए नए लॉग एंट्रीज़ इस टैब में रियल-टाइम में ऐपेंड होंगे यदि फ़ाइल एक्सेसिबल है।

**विकल्प 2: रिमोट लॉग्स को टेल करने के लिए SSH का उपयोग करें**
यदि लॉग्स को माउंट या कॉपी करना संभव नहीं है, तो आप रिमोट लॉग फ़ाइल को सीधे टेल करने के लिए IntelliJ के बिल्ट-इन SSH टर्मिनल या प्लगइन का उपयोग कर सकते हैं:
1.  **SSH एक्सेस एनेबल करें**:
    - सुनिश्चित करें कि आपके पास Liberty होस्ट करने वाले रिमोट सर्वर तक SSH एक्सेस है।
    - **File > Settings > Tools > SSH Configurations** के माध्यम से IntelliJ IDEA में SSH कॉन्फ़िगर करें।
2.  **बिल्ट-इन टर्मिनल का उपयोग करें**:
    - IntelliJ IDEA में **Terminal** टूल विंडो (Alt+F12) खोलें।
    - लॉग फ़ाइल को टेल करने के लिए एक कमांड रन करें:
      ```bash
      ssh user@remote-server tail -f /path/to/wlp/usr/servers/<serverName>/logs/messages.log
      ```
    - यह लॉग फ़ाइल को रियल-टाइम में टर्मिनल में स्ट्रीम करता है, आपके VS Code कर्सर-एट-एंड वर्कफ़्लो के समान।
3.  **लॉग्स साफ़ करें**:
    - IntelliJ के टर्मिनल में Eclipse की कंसोल की तरह कोई डायरेक्ट "clear logs" बटन नहीं है। इसके बजाय, आप कर सकते हैं:
      - व्यू को क्लियर करने के लिए टेल कमांड को स्टॉप (Ctrl+C) करें और रीस्टार्ट करें।
      - टर्मिनल टूलबार में **Clear All** बटन का उपयोग करके टर्मिनल आउटपुट को साफ़ करें।
4.  **नए लॉग्स फ़िल्टर करें**:
    - विशिष्ट फ्रंटएंड-ट्रिगर इवेंट्स के लिए लॉग्स को फ़िल्टर करने के लिए `grep` का उपयोग करें:
      ```bash
      ssh user@remote-server tail -f /path/to/wlp/usr/servers/<serverName>/logs/messages.log | grep "specific-pattern"
      ```
    - उदाहरण के लिए, यदि फ्रंटएंड क्लिक्स किसी विशिष्ट कीवर्ड (उदा., “INFO”) वाले लॉग्स को ट्रिगर करते हैं, तो उनके लिए फ़िल्टर करें।

**विकल्प 3: एन्हांस्ड लॉग व्यूइंग के लिए प्लगइन का उपयोग करें**
**Log4JPlugin** या **Grep Console** प्लगइन IntelliJ IDEA में लॉग व्यूइंग को एन्हांस कर सकते हैं:
1.  **प्लगइन इंस्टॉल करें**:
    - **File > Settings > Plugins** पर जाएं, “Log4JPlugin” या “Grep Console” सर्च करें और इंस्टॉल करें।
    - IntelliJ IDEA को रीस्टार्ट करें।
2.  **Log4JPlugin कॉन्फ़िगर करें**:
    - रिमोट डीबग कॉन्फ़िगरेशन सेट अप करने के बाद, रिमोट लॉग फ़ाइल की ओर इशारा करने के लिए Log4JPlugin का उपयोग करें (इसके लिए SSH या माउंटेड फ़ोल्डर की आवश्यकता होती है)।
    - यह प्लगइन आपको एक समर्पित टैब में लॉग्स देखने और फ़िल्टर करने की अनुमति देता है, Eclipse के Liberty Tools कंसोल के समान।
3.  **Grep Console कॉन्फ़िगर करें**:
    - Grep Console आपको पैटर्न के आधार पर लॉग मैसेजेस को हाइलाइट और फ़िल्टर करने देता है, जिससे फ्रंटएंड एक्शन्स द्वारा ट्रिगर किए गए नए लॉग्स पर फोकस करना आसान हो जाता है।
    - लॉग फ़ाइल को निर्दिष्ट करके और प्लगइन को एनेबल करके इसे **Run/Debug Configurations > Logs** टैब में कॉन्फ़िगर करें।

#### चरण 3: VS Code के "कर्सर एट एंड" वर्कफ़्लो को दोहराएं
लॉग फ़ाइल के अंत में कर्सर रखने और नए लॉग्स को सेलेक्ट करने के VS Code व्यवहार की नकल करने के लिए:
1.  **ऑटो-स्क्रॉल टू एंड**:
    - **Debug** टूल विंडो के लॉग टैब (विकल्प 1 से) में, IntelliJ IDEA नए एंट्रीज़ के ऐड होने पर लॉग फ़ाइल के अंत में स्वचालित रूप से स्क्रॉल करता है, `tail -f` के समान।
    - सुनिश्चित करें कि लॉग टैब के टूलबार में **Scroll to the end** एनेबल है (एक छोटा एरो आइकन नीचे की ओर इशारा करता हुआ)।
2.  **नए लॉग्स सेलेक्ट करें**:
    - कर्सर वहां रखने के लिए लॉग टैब के अंत में क्लिक करें।
    - जैसे ही आप फ्रंटएंड के साथ इंटरैक्ट करते हैं, नए लॉग एंट्रीज़ दिखाई देंगे, और आप माउस ड्रैग करके या कीबोर्ड शॉर्टकट (उदा., Shift+Arrow keys) का उपयोग करके उन्हें सेलेक्ट कर सकते हैं।
    - वैकल्पिक रूप से, कीवर्ड्स या टाइमस्टैम्प्स के आधार पर नए एंट्रीज़ के लिए फ़िल्टर करने के लिए लॉग टैब में **Search** फीचर (मैग्निफाइंग ग्लास आइकन) का उपयोग करें।
3.  **नए एंट्रीज़ के लिए लॉग्स साफ़ करें**:
    - वर्तमान सत्र से केवल नए लॉग एंट्रीज़ दिखाने के लिए Run/Debug Configuration के Logs टैब में **Skip Content** ऑप्शन चेक करें, जो प्रभावी रूप से पुराने लॉग्स को "साफ़" कर देता है।
    - यदि SSH टर्मिनल का उपयोग कर रहे हैं, तो नए लॉग्स के लिए व्यू को रीसेट करने के लिए `tail -f` कमांड को स्टॉप और रीस्टार्ट करें।

#### चरण 4: फ्रंटएंड-ट्रिगर लॉग्स को डीबग और मॉनिटर करें
1.  **ब्रेकपॉइंट्स सेट करें**:
    - IntelliJ IDEA में, संबंधित Java सोर्स फ़ाइलें (उदा., फ्रंटएंड रिक्वेस्ट्स को हैंडल करने वाले बैकएंड कंट्रोलर्स) खोलें।
    - कोड लाइन के बगल वाली गटर में क्लिक करके (या Ctrl+F8 / Cmd+F8 दबाकर) ब्रेकपॉइंट्स सेट करें।
2.  **डीबगिंग स्टार्ट करें**:
    - रिमोट डीबग कॉन्फ़िगरेशन रन करें।
    - Debug टूल विंडो लॉग टैब (यदि कॉन्फ़िगर किया गया है) दिखाएगी और फ्रंटएंड क्लिक्स द्वारा ट्रिगर किए गए ब्रेकपॉइंट्स पर रुकेगी।
3.  **ब्रेकपॉइंट्स के साथ लॉग्स को सहसंबंधित करें**:
    - जब कोई ब्रेकपॉइंट हिट होता है, तो संबंधित लॉग एंट्रीज़ के लिए लॉग टैब या टर्मिनल चेक करें।
    - IntelliJ IDEA SLF4J या Log4J (Liberty एप्लिकेशन में आम) जैसे लॉगिंग फ्रेमवर्क को पहचानता है और लॉग टैब में क्लिक करने योग्य लिंक प्रदान करता है ताकि उस सोर्स कोड पर जाया जा सके जहां लॉग जेनरेट किया गया था।
4.  **फ्रंटएंड एक्शन्स के लिए फ़िल्टर करें**:
    - विशिष्ट लॉग मैसेजेस (उदा., “INFO [frontend]” या “POST /endpoint”) के लिए फ़िल्टर करने के लिए लॉग टैब में सर्च बार का उपयोग करें।
    - यदि Grep Console का उपयोग कर रहे हैं, तो फ्रंटएंड-संबंधित लॉग्स को हाइलाइट करने के लिए पैटर्न कॉन्फ़िगर करें।

### Eclipse और VS Code से अंतर
-   **Eclipse Liberty Tools**: Liberty लॉग्स के लिए एक समर्पित कंसोल प्रदान करता है जिसमें बिल्ट-इन क्लियर और फ़िल्टर ऑप्शंस होते हैं। IntelliJ IDEA को समान कार्यक्षमता प्राप्त करने के लिए मैन्युअल कॉन्फ़िगरेशन या प्लगइन की आवश्यकता होती है।
-   **VS Code**: VS Code में लॉग फ़ाइल को टेल करना हल्का-फुल्का और मैन्युअल होता है, और क्विक लॉग इंस्पेक्शन के लिए कर्सर-एट-एंड एप्रोच सरल होती है। IntelliJ IDEA के लॉग टैब या टर्मिनल अधिक इंटीग्रेटेड हैं लेकिन मैन्युअल कर्सर प्लेसमेंट के लिए कम फ्लेक्सिबल हैं।
-   **लॉग्स साफ़ करना**:
    -   Eclipse: कंसोल में वन-क्लिक क्लियर बटन।
    -   IntelliJ IDEA: **Skip Content** का उपयोग करें या टर्मिनल टेल कमांड को रीस्टार्ट करें।
    -   VS Code: मैन्युअल रूप से टर्मिनल को साफ़ करें या `tail -f` को रीस्टार्ट करें।
-   **रियल-टाइम लॉग व्यूइंग**:
    -   तीनों IDE रियल-टाइम लॉग व्यूइंग को सपोर्ट करते हैं, लेकिन IntelliJ IDEA के लॉग टैब के लिए माउंटेड फ़ाइल या प्लगइन की आवश्यकता होती है, जबकि VS Code टर्मिनल कमांड्स पर निर्भर करता है।

### सिफारिशें
1.  **पसंदीदा तरीका**: Eclipse के कंसोल के निकटतम अनुभव के लिए **विकल्प 1 (माउंटेड फ़ोल्डर)** का उपयोग करें। यह लॉग्स को Debug टूल विंडो में इंटीग्रेट करता है, ऑटो-स्क्रॉलिंग को सपोर्ट करता है और फ़िल्टरिंग की अनुमति देता है। **Skip Content** ऑप्शन लॉग्स को साफ़ करने की नकल करता है।
2.  **VS Code जैसी सरलता के लिए**: एक हल्के, कर्सर-एट-एंड अनुभव के लिए `tail -f` के साथ **विकल्प 2 (SSH टर्मिनल)** का उपयोग करें। फ्रंटएंड-ट्रिगर लॉग्स को फ़िल्टर करने के लिए `grep` के साथ संयोजित करें।
3.  **प्लगइन के साथ एन्हांस करें**: बेहतर लॉग फ़िल्टरिंग और हाइलाइटिंग के लिए, विशेष रूप से फ्रंटएंड-स्पेसिफिक लॉग्स के लिए, **Grep Console** इंस्टॉल करें।
4.  **परफॉर्मेंस नोट**: यदि रिमोट सर्वर में हाई लॉग वॉल्यूम है, तो लॉग्स को माउंट या कॉपी करना SSH के माध्यम से टेल करने की तुलना में धीमा हो सकता है। सबसे अच्छा फिट खोजने के लिए दोनों तरीकों का परीक्षण करें।

### समस्या निवारण
-   **लॉग टैब खाली**: सुनिश्चित करें कि लॉग फ़ाइल पथ सही और एक्सेसिबल है। यदि माउंटेड फ़ोल्डर का उपयोग कर रहे हैं, तो सत्यापित करें कि माउंट एक्टिव है। यदि SSH का उपयोग कर रहे हैं, तो `tail -f` कमांड सिंटैक्स चेक करें।
-   **लॉग्स अपडेट नहीं हो रहे**: पुष्टि करें कि Liberty सर्वर निर्दिष्ट लॉग फ़ाइल (उदा., `messages.log`) में लिख रहा है। फ़ाइल परमिशन या रोलिंग लॉग समस्याओं की जांच करें।
-   **कोई फ्रंटएंड लॉग्स नहीं**: सत्यापित करें कि फ्रंटएंड एक्शन्स बैकएंड तक पहुंच रहे हैं (ब्रेकपॉइंट्स का उपयोग करें) और लॉगिंग फ्रेमवर्क (उदा., SLF4J) संबंधित मैसेजेस आउटपुट करने के लिए कॉन्फ़िगर है।
-   **प्लगइन समस्याएं**: यदि Log4JPlugin या Grep Console काम नहीं करता है, तो सुनिश्चित करें कि वे आपके IntelliJ IDEA वर्जन (उदा., 2024.1 या बाद के) के साथ कंपेटिबल हैं।

### उदाहरण वर्कफ़्लो
1.  लॉग फ़ाइल `/path/to/mounted/wlp/usr/servers/defaultServer/logs/messages.log` के साथ Remote JVM Debug कॉन्फ़िगर करें।
2.  डीबग सत्र स्टार्ट करें।
3.  **Debug** टूल विंडो खोलें, “Liberty Logs” टैब पर स्विच करें, और **Scroll to the end** एनेबल करें।
4.  बैकएंड रिक्वेस्ट को ट्रिगर करने के लिए एक फ्रंटएंड बटन पर क्लिक करें।
5.  नए एंट्रीज़ (उदा., “INFO [frontend] Button clicked”) के लिए लॉग टैब देखें।
6.  निरीक्षण के लिए विशिष्ट कीवर्ड्स के लिए फ़िल्टर करने या नई लॉग लाइनों को सेलेक्ट करने के लिए सर्च बार का उपयोग करें।
7.  यदि आवश्यक हो, तो पुराने लॉग्स को साफ़ करने के लिए **Skip Content** एनेबल करके डीबग सत्र को स्टॉप और रीस्टार्ट करें।

### अतिरिक्त नोट्स
-   **Liberty-स्पेसिफिक लॉग्स**: WebSphere Liberty आमतौर पर `wlp/usr/servers/<serverName>/logs` में `messages.log` या `console.log` में लिखता है। सुनिश्चित करें कि आपकी Run/Debug Configuration सही फ़ाइल की ओर इशारा करती है।
-   **फ्रंटएंड-बैकएंड सहसंबंध**: यदि फ्रंटएंड क्लिक्स अपेक्षित लॉग्स उत्पन्न नहीं करते हैं, तो बैकएंड एंडपॉइंट के लॉगिंग कॉन्फ़िगरेशन की जांच करें (उदा., सुनिश्चित करें कि `logger.info()` स्टेटमेंट मौजूद हैं)।
-   **वैकल्पिक टूल्स**: यदि IntelliJ का लॉग व्यूइंग संतोषजनक नहीं है, तो लॉग मॉनिटरिंग के लिए IntelliJ के साथ-साथ लॉग मॉनिटरिंग के लिए एक अलग टर्मिनल या VS Code में `tail -f` चलाने पर विचार करें, जबकि डीबगिंग के लिए IntelliJ का उपयोग करें।

यदि आपको विशिष्ट कॉन्फ़िगरेशन चरणों, प्लगइन सेटअप, या किसी विशेष फ्रंटएंड एक्शन के लिए लॉग्स फ़िल्टर करने में सहायता की आवश्यकता है, तो कृपया अधिक विवरण प्रदान करें (उदा., Liberty वर्जन, लॉग फ़ाइल पथ, या फ्रंटएंड टेक्नोलॉजी), और मैं निर्देशों को और अधिक टेलर कर सकता हूं!
