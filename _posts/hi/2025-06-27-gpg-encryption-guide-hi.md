## GPG क्या है?
GPG, या GNU प्राइवेसी गार्ड, OpenPGP मानक (प्रिटी गुड प्राइवेसी) का एक ओपन-सोर्स कार्यान्वयन है। यह सुरक्षित संचार के लिए एक उपकरण है, जो एन्क्रिप्शन, डिक्रिप्शन, डिजिटल हस्ताक्षर और कुंजी प्रबंधन प्रदान करता है। GPG का व्यापक रूप से ईमेल, फाइलों और अन्य डेटा को सुरक्षित करने के लिए उपयोग किया जाता है, जो गोपनीयता, अखंडता और प्रामाणिकता सुनिश्चित करता है।

GPG अधिकांश ऑपरेटिंग सिस्टम पर उपलब्ध है, जिसमें Linux, macOS, और Windows शामिल हैं। यह कमांड-लाइन आधारित है लेकिन GUI टूल या ईमेल क्लाइंट जैसे Thunderbird के साथ एकीकृत किया जा सकता है।

## GPG कैसे काम करता है
GPG डेटा को सुरक्षित करने के लिए **सिमेट्रिक-की क्रिप्टोग्राफी** और **पब्लिक-की क्रिप्टोग्राफी** के संयोजन का उपयोग करता है:

1. **सिमेट्रिक-की क्रिप्टोग्राफी**:
   - एन्क्रिप्शन और डिक्रिप्शन दोनों के लिए एक ही कुंजी का उपयोग करती है।
   - GPG वास्तविक डेटा को एन्क्रिप्ट करने के लिए सिमेट्रिक-की एल्गोरिदम (जैसे, AES) का उपयोग करता है क्योंकि ये बड़े डेटासेट के लिए तेज़ होते हैं।
   - प्रत्येक एन्क्रिप्शन ऑपरेशन के लिए एक यादृच्छिक सत्र कुंजी (session key) उत्पन्न की जाती है।

2. **पब्लिक-की क्रिप्टोग्राफी**:
   - कुंजियों की एक जोड़ी का उपयोग करती है: एन्क्रिप्शन के लिए एक **सार्वजनिक कुंजी (public key)** और डिक्रिप्शन के लिए एक **निजी कुंजी (private key)**।
   - GPG कुंजी जोड़े के लिए RSA या ECDSA जैसे एल्गोरिदम का समर्थन करता है।
   - सार्वजनिक कुंजी सत्र कुंजी को एन्क्रिप्ट करती है, जिसका उपयोग फिर डेटा को एन्क्रिप्ट करने के लिए किया जाता है। प्राप्तकर्ता सत्र कुंजी को डिक्रिप्ट करने के लिए अपनी निजी कुंजी का उपयोग करता है, जिसका उपयोग फिर डेटा को डिक्रिप्ट करने के लिए किया जाता है।

3. **डिजिटल हस्ताक्षर**:
   - GPG उपयोगकर्ताओं को प्रामाणिकता और अखंडता साबित करने के लिए अपनी निजी कुंजी का उपयोग करके डेटा पर हस्ताक्षर करने की अनुमति देता है।
   - प्राप्तकर्ता भेजने वाले की सार्वजनिक कुंजी का उपयोग करके हस्ताक्षर को सत्यापित करता है।

4. **कुंजी प्रबंधन**:
   - GPG एक कीरिंग (keyring) में कुंजियों का प्रबंधन करता है, जो सार्वजनिक और निजी कुंजियों को संग्रहीत करता है।
   - कुंजियाँ उत्पन्न, आयात, निर्यात और कीसर्वर (keyservers) पर प्रकाशित की जा सकती हैं।

### GPG एन्क्रिप्शन प्रक्रिया
जब कोई फ़ाइल या संदेश एन्क्रिप्ट करते हैं:
1. GPG सिमेट्रिक एन्क्रिप्शन के लिए एक यादृच्छिक **सत्र कुंजी (session key)** उत्पन्न करता है।
2. डेटा को एक सिमेट्रिक एल्गोरिदम (जैसे, AES-256) का उपयोग करके सत्र कुंजी के साथ एन्क्रिप्ट किया जाता है।
3. सत्र कुंजी को प्राप्तकर्ता की **सार्वजनिक कुंजी (public key)** के साथ एक असममित एल्गोरिदम (जैसे, RSA) का उपयोग करके एन्क्रिप्ट किया जाता है।
4. एन्क्रिप्ट की गई सत्र कुंजी और एन्क्रिप्ट किया गया डेटा एक ही आउटपुट फ़ाइल या संदेश में संयोजित किए जाते हैं।

जब डिक्रिप्ट करते हैं:
1. प्राप्तकर्ता सत्र कुंजी को डिक्रिप्ट करने के लिए अपनी **निजी कुंजी (private key)** का उपयोग करता है।
2. सत्र कुंजी का उपयोग सिमेट्रिक एल्गोरिदम के साथ डेटा को डिक्रिप्ट करने के लिए किया जाता है।

यह संकर दृष्टिकोण (hybrid approach) सिमेट्रिक एन्क्रिप्शन की गति को असममित एन्क्रिप्शन की सुरक्षा के साथ जोड़ता है।

## GPG इंस्टॉल करना
GPG कई Linux वितरणों पर पहले से इंस्टॉल आता है। अन्य सिस्टम के लिए:
- **Linux**: पैकेज मैनेजर के माध्यम से इंस्टॉल करें:
  ```bash
  sudo apt install gnupg  # Debian/Ubuntu
  sudo yum install gnupg  # CentOS/RHEL
  ```
- **macOS**: Homebrew के माध्यम से इंस्टॉल करें:
  ```bash
  brew install gnupg
  ```
- **Windows**: [gpg4win.org](https://gpg4win.org/) से Gpg4win डाउनलोड करें।

इंस्टॉलेशन सत्यापित करें:
```bash
gpg --version
```

## GPG कुंजियाँ जनरेट करना
GPG का उपयोग करने के लिए, आपको एक कुंजी जोड़ी (सार्वजनिक और निजी कुंजी) की आवश्यकता होती है।

### चरण-दर-चरण कुंजी जनरेशन
कुंजी जोड़ी उत्पन्न करने के लिए निम्न कमांड चलाएँ:
```bash
gpg --full-generate-key
```

1. **कुंजी प्रकार चुनें**:
   - डिफ़ॉल्ट RSA और RSA (विकल्प 1) है।
   - अधिकांश उद्देश्यों के लिए RSA व्यापक रूप से उपयोग किया जाता है और सुरक्षित है।

2. **कुंजी आकार**:
   - अनुशंसित: 2048 या 4096 बिट्स (4096 अधिक सुरक्षित है लेकिन धीमी है)।
   - उदाहरण: 4096 चुनें।

3. **कुंजी समाप्ति**:
   - एक समाप्ति तिथि चुनें (जैसे, 1 वर्ष) या कोई समाप्ति नहीं के लिए 0 चुनें।
   - समाप्त होने वाली कुंजियाँ कुंजी के जीवनकाल को सीमित करके सुरक्षा बढ़ाती हैं।

4. **उपयोगकर्ता ID**:
   - अपना नाम, ईमेल और एक वैकल्पिक टिप्पणी दर्ज करें।
   - उदाहरण: `John Doe <john.doe@example.com>`।

5. **पासफ्रेज़**:
   - निजी कुंजी की सुरक्षा के लिए एक मजबूत पासफ्रेज़ सेट करें।
   - डिक्रिप्शन और हस्ताक्षर के लिए इस पासफ्रेज़ की आवश्यकता होती है।

कमांड चलाने के बाद उदाहरण आउटपुट:
```
gpg: key 0x1234567890ABCDEF marked as ultimately trusted
gpg: generated key pair
```

### कुंजियाँ निर्यात करना
- **सार्वजनिक कुंजी निर्यात करें**:
  ```bash
  gpg --armor --output public-key.asc --export john.doe@example.com
  ```
  यह आपकी सार्वजनिक कुंजी को ASCII प्रारूप में शामिल करते हुए एक फ़ाइल (`public-key.asc`) बनाता है।

- **निजी कुंजी निर्यात करें** (सावधान रहें, इसे सुरक्षित रखें):
  ```bash
  gpg --armor --output private-key.asc --export-secret-keys john.doe@example.com
  ```

## फाइलों को एन्क्रिप्ट और डिक्रिप्ट करना
### एक फाइल को एन्क्रिप्ट करना
किसी प्राप्तकर्ता के लिए एक फाइल को एन्क्रिप्ट करने के लिए:
1. सुनिश्चित करें कि प्राप्तकर्ता की सार्वजनिक कुंजी आपकी कीरिंग में है:
   ```bash
   gpg --import recipient-public-key.asc
   ```
2. फाइल को एन्क्रिप्ट करें:
   ```bash
   gpg --encrypt --recipient john.doe@example.com --output encrypted-file.gpg input-file.txt
   ```
   - `--recipient`: प्राप्तकर्ता के ईमेल या कुंजी ID को निर्दिष्ट करता है।
   - `--output`: आउटपुट फाइल को निर्दिष्ट करता है।
   - परिणाम `encrypted-file.gpg` है, जिसे केवल प्राप्तकर्ता ही डिक्रिप्ट कर सकता है।

### एक फाइल को डिक्रिप्ट करना
आपके लिए एन्क्रिप्ट की गई फाइल को डिक्रिप्ट करने के लिए:
```bash
gpg --decrypt --output decrypted-file.txt encrypted-file.gpg
```
- संकेत देने पर अपना पासफ्रेज़ दर्ज करें।
- डिक्रिप्ट की गई सामग्री `decrypted-file.txt` में सहेजी जाती है।

## डेटा पर हस्ताक्षर करना और सत्यापित करना
### एक फाइल पर हस्ताक्षर करना
हस्ताक्षर करना डेटा की प्रामाणिकता और अखंडता साबित करता है।
- **क्लियर-साइन (Clear-sign)** (मानव-पठनीय हस्ताक्षर शामिल करता है):
  ```bash
  gpg --clearsign input-file.txt
  ```
  आउटपुट: फाइल सामग्री और हस्ताक्षर के साथ `input-file.txt.asc`।

- **डिटैच्ड सिग्नेचर (Detached Signature)** (अलग हस्ताक्षर फाइल):
  ```bash
  gpg --detach-sign input-file.txt
  ```
  आउटपुट: `input-file.txt.sig`।

### एक हस्ताक्षर सत्यापित करना
हस्ताक्षरित फाइल को सत्यापित करने के लिए:
```bash
gpg --verify input-file.txt.asc
```
डिटैच्ड सिग्नेचर के लिए:
```bash
gpg --verify input-file.txt.sig input-file.txt
```
आपको हस्ताक्षरकर्ता की सार्वजनिक कुंजी आपकी कीरिंग में चाहिए।

## GPG के साथ पासवर्ड जनरेट करना
GPG यादृच्छिक डेटा उत्पन्न कर सकता है, जिसका उपयोग सुरक्षित पासवर्ड बनाने के लिए किया जा सकता है। हालांकि GPG मुख्य रूप से एक पासवर्ड जनरेटर नहीं है, लेकिन इसकी यादृच्छिक संख्या जनरेशन क्रिप्टोग्राफिक रूप से सुरक्षित है।

### पासवर्ड जनरेट करने की कमांड
```bash
gpg --gen-random --armor 1 32
```
- `--gen-random`: यादृच्छिक बाइट्स उत्पन्न करता है।
- `--armor`: ASCII प्रारूप में आउटपुट देता है।
- `1`: गुणवत्ता स्तर (1 क्रिप्टोग्राफिक उद्देश्यों के लिए उपयुक्त है)।
- `32`: बाइट्स की संख्या (वांछित पासवर्ड लंबाई के लिए समायोजित करें)।

उदाहरण आउटपुट:
```
4eX9j2kPqW8mZ3rT5vY7nL9xF2bC6dA8
```
इसे और अधिक पासवर्ड जैसा बनाने के लिए, इसे base64 या hex कनवर्टर के माध्यम से पाइप करें, या इसे वांछित लंबाई में ट्रिम करें।

### उदाहरण: 20-अक्षर का पासवर्ड जनरेट करें
```bash
gpg --gen-random --armor 1 15 | head -c 20
```
यह 20 अक्षरों की एक यादृच्छिक स्ट्रिंग उत्पन्न करता है।

## कुंजी प्रबंधन
### कुंजियों की सूची बनाना
- सार्वजनिक कुंजियों की सूची बनाएं:
  ```bash
  gpg --list-keys
  ```
- निजी कुंजियों की सूची बनाएं:
  ```bash
  gpg --list-secret-keys
  ```

### सार्वजनिक कुंजियाँ प्रकाशित करना
एक कीसर्वर के माध्यम से अपनी सार्वजनिक कुंजी साझा करें:
```bash
gpg --keyserver hkps://keys.openpgp.org --send-keys 0x1234567890ABCDEF
```
`0x1234567890ABCDEF` को अपनी कुंजी ID से बदलें।

### कुंजियाँ आयात करना
किसी फाइल से सार्वजनिक कुंजी आयात करें:
```bash
gpg --import public-key.asc
```
या किसी कीसर्वर से:
```bash
gpg --keyserver hkps://keys.openpgp.org --recv-keys 0x1234567890ABCDEF
```

### किसी कुंजी को रद्द करना
यदि कोई कुंजी समझौता हो गई है या समाप्त हो गई है:
1. एक निरसन प्रमाणपत्र (revocation certificate) उत्पन्न करें (कुंजी बनाते समय ऐसा करें):
   ```bash
   gpg --output revoke.asc --gen-revoke john.doe@example.com
   ```
2. निरसन को आयात और प्रकाशित करें:
   ```bash
   gpg --import revoke.asc
   gpg --keyserver hkps://keys.openpgp.org --send-keys john.doe@example.com
   ```

## सर्वोत्तम अभ्यास
1. **कुंजियों का बैकअप लें**:
   - निजी कुंजियों और निरसन प्रमाणपत्रों को सुरक्षित रूप से संग्रहीत करें (जैसे, एन्क्रिप्टेड USB ड्राइव)।
   - निजी कुंजियाँ कभी साझा न करें।

2. **मजबूत पासफ्रेज़ का उपयोग करें**:
   - अपनी निजी कुंजी के लिए एक लंबा, अद्वितीय पासफ्रेज़ उपयोग करें।

3. **कुंजियों को नियमित रूप से अपडेट करें**:
   - एक समाप्ति तिथि सेट करें और कुंजियों को समय-समय पर रोटेट करें।

4. **कुंजी फिंगरप्रिंट सत्यापित करें**:
   - किसी सार्वजनिक कुंजी पर भरोसा करने से पहले, मालिक के साथ इसके फिंगरप्रिंट को सत्यापित करें:
     ```bash
     gpg --fingerprint john.doe@example.com
     ```

5. **कीसर्वर का सुरक्षित रूप से उपयोग करें**:
   - विश्वसनीय कीसर्वर जैसे `hkps://keys.openpgp.org` का उपयोग करें।

6. **केवल विश्वसनीय कुंजियों पर हस्ताक्षर करें**:
   - जब किसी और की कुंजी पर हस्ताक्षर करते हैं, तो व्यक्तिगत रूप से या किसी विश्वसनीय चैनल के माध्यम से उनकी पहचान सत्यापित करें।

## सामान्य GPG कमांड्स सारांश
यहाँ सामान्य GPG कमांड्स के लिए एक त्वरित संदर्भ दिया गया है:
- कुंजी जोड़ी उत्पन्न करें: `gpg --full-generate-key`
- फाइल एन्क्रिप्ट करें: `gpg --encrypt --recipient <email> --output <output.gpg> <input.txt>`
- फाइल डिक्रिप्ट करें: `gpg --decrypt --output <output.txt> <input.gpg>`
- फाइल पर हस्ताक्षर करें: `gpg --clearsign <input.txt>` या `gpg --detach-sign <input.txt>`
- हस्ताक्षर सत्यापित करें: `gpg --verify <file.asc>` या `gpg --verify <file.sig> <file>`
- सार्वजनिक कुंजी निर्यात करें: `gpg --armor --output public-key.asc --export <email>`
- कुंजी आयात करें: `gpg --import <key.asc>`
- यादृच्छिक पासवर्ड जनरेट करें: `gpg --gen-random --armor 1 <bytes>`

## समस्या निवारण
- **"No secret key" त्रुटि**: सुनिश्चित करें कि निजी कुंजी आपकी कीरिंग में है (`gpg --list-secret-keys`) और प्राप्तकर्ता की सार्वजनिक कुंजी से मेल खाती है।
- **पासफ्रेज़ भूल गए**: यदि आप अपना पासफ्रेज़ खो देते हैं, तो आपको कुंजी को रद्द करना होगा और एक नई कुंजी उत्पन्न करनी होगी।
- **कुंजी नहीं मिली**: प्राप्तकर्ता की सार्वजनिक कुंजी आयात करें या कीसर्वर जांचें।
- **GPG संस्करण समस्याएं**: सुनिश्चित करें कि सभी पक्ष संगत GPG संस्करणों का उपयोग करते हैं (`gpg --version` से जांचें)।
