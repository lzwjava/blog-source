#### मूल कारण
यह आउटेज सॉफ़्टवेयर लोड बैलेंसर इंफ्रास्ट्रक्चर में संचार के अधिकारीकरण (authorizing communication) के लिए उपयोग किए जाने वाले प्रमाणपत्रों (certificates) में एक नियोजित कॉन्फ़िगरेशन परिवर्तन से उपजी थी। नए प्रमाणपत्रों में से एक का **दोषपूर्ण मान (malformed value)** था जो सत्यापन के दौरान पकड़ा नहीं गया। इस परिवर्तन ने एक त्वरित तैनाती पथ (expedited deployment path) का पालन किया, जिसने अनपेक्षित रूप से इसे स्वास्थ्य सुरक्षा उपायों (health safeguards) को सक्रिय किए बिना कई क्षेत्रों में रोल आउट कर दिया। परिणामस्वरूप:
- लोड बैलेंसर स्टोरेज संसाधनों और नोड्स से कनेक्टिविटी खो बैठे।
- प्रभावित VMs ने डिस्क डिस्कनेक्शन को लंबे समय तक जारी पाया और डेटा दूषित होने से बचने के लिए स्वचालित रूप से शट डाउन हो गईं।
- यह निर्भर सेवाओं (dependent services) में फैल गया, जिससे प्रभाव और बढ़ गया।

#### प्रभावित सेवाएं
इस व्यवधान ने स्विट्ज़रलैंड नॉर्थ में होस्ट की गई Azure सेवाओं की एक विस्तृत श्रृंखला को प्रभावित किया, जिनमें शामिल हैं:
- **कोर इंफ्रास्ट्रक्चर**: Azure Storage, Azure Virtual Machines (VMs), Azure Virtual Machine Scale Sets (VMSS)
- **डेटाबेस**: Azure Cosmos DB, Azure SQL Database, Azure SQL Managed Instance, Azure Database for PostgreSQL
- **कम्प्यूट और ऐप्स**: Azure App Service, Azure API Management, Azure Kubernetes Service (AKS), Azure Databricks
- **नेटवर्किंग और सुरक्षा**: Azure Application Gateway, Azure Firewall, Azure Cache for Redis
- **एनालिटिक्स और मॉनिटरिंग**: Azure Synapse Analytics, Azure Data Factory, Azure Stream Analytics, Azure Data Explorer, Azure Log Analytics, Microsoft Sentinel
- **अन्य**: Azure Backup, Azure Batch, Azure Site Recovery, Azure Application Insights

इन पर निर्भर सेवाएं (जैसे, कस्टम एप्लिकेशन) भी प्रभावित हुईं, जिससे व्यापक अनुपलब्धता या गिरी हुई परफॉर्मेंस देखी गई।
