# `load_dataset` को प्रॉक्सी का उपयोग करने के लिए कैसे बनाएं (चीन-अनुकूल)

## 1. HTTP/HTTPS प्रॉक्सी के लिए एनवायरनमेंट वेरिएबल्स सेट करें

`datasets` आंतरिक रूप से Python की `requests` लाइब्रेरी का उपयोग करता है।
`requests` स्वचालित रूप से मानक प्रॉक्सी env वेरिएबल्स का उपयोग करती है:

### Linux / macOS

```bash
export HTTP_PROXY="http://127.0.0.1:7890"
export HTTPS_PROXY="http://127.0.0.1:7890"
```

यदि आपकी प्रॉक्सी SOCKS5 का उपयोग करती है:

```bash
export HTTP_PROXY="socks5://127.0.0.1:1080"
export HTTPS_PROXY="socks5://127.0.0.1:1080"
```

### Windows (PowerShell)

```powershell
setx HTTP_PROXY "http://127.0.0.1:7890"
setx HTTPS_PROXY "http://127.0.0.1:7890"
```

फिर टर्मिनल को दोबारा खोलें।

---

## 2. यदि आप कोड के अंदर Python चला रहे हैं (जैसे Jupyter), तो इसे Python के अंदर सेट करें

```python
import os

os.environ['HTTP_PROXY'] = 'http://127.0.0.1:7890'
os.environ['HTTPS_PROXY'] = 'http://127.0.0.1:7890'
```

यह विश्वसनीय रूप से काम करता है।

## 3. पुष्टि करें कि प्रॉक्सी काम कर रही है

आप इसके साथ परीक्षण कर सकते हैं:

```python
import requests
print(requests.get("https://api.ipify.org?format=json").text)
```

यदि IP आपके प्रॉक्सी का एक्जिट नोड है, तो Hugging Face डेटासेट डाउनलोड भी उसी प्रॉक्सी के माध्यम से होंगे।

## 4. Hugging Face का `huggingface_hub` भी इन वेरिएबल्स का सम्मान करता है

`datasets` आंतरिक रूप से `huggingface_hub` का उपयोग करता है, और यह **भी** इनका पालन करता है:

* `HTTP_PROXY`
* `HTTPS_PROXY`
* `http_proxy`
* `https_proxy`

इसलिए एक बार env वेरिएबल्स सेट हो जाने के बाद, मॉडल और डेटासेट दोनों का डाउनलोड स्वचालित रूप से आपकी प्रॉक्सी का उपयोग करेगा।
