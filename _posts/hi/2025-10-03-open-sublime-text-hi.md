### कमांड लाइन (CLी) से Sublime Text खोलना

Sublime Text में डिफ़ॉल्ट रूप से macOS (आपके `open -a` रेफरेंस के आधार पर लगता है कि आप इसी का उपयोग कर रहे हैं) पर कोई बिल्ट-इन `sublime` (या `subl`) कमांड शामिल नहीं होता है। हालाँकि, इसे जोड़ने का एक आधिकारिक और सीधा तरीका है, जो Sublime के बंडल किए गए `subl` स्क्रिप्ट का उपयोग करता है। यह `open` कमांड पर निर्भर रहने की तुलना में अधिक साफ-सुथरा तरीका है। मैं आपको नीचे दिए गए विकल्पों के साथ, सबसे पहले अनुशंसित तरीके से चलते हुए, चरण दर चरण बताऊंगा।

#### 1. **आधिकारिक `subl` CLI कमांड इंस्टॉल करें (अनुशंसित)**
Sublime Text अपने ऐप बंडल में एक `subl` एक्जीक्यूटेबल शामिल करता है, जिसे आप अपने PATH में मौजूद एक डायरेक्टरी (जैसे `/usr/local/bin`) में symlink कर सकते हैं। इससे आप टर्मिनल में कहीं से भी `subl .` (वर्तमान डायरेक्टरी खोलने के लिए) या `subl file.txt` चला सकते हैं।

**macOS के लिए चरण (Sublime Text 3 या 4):**
- टर्मिनल खोलें।
- Symlink बनाने के लिए यह कमांड चलाएँ (इसके लिए एडमिन विशेषाधिकारों की आवश्यकता होती है, इसलिए आपसे आपका पासवर्ड माँगा जाएगा):
  ```
  sudo ln -s "/Applications/Sublime Text.app/Contents/SharedSupport/bin/subl" /usr/local/bin/subl
  ```
  - यदि आप Sublime Text 3 का उपयोग कर रहे हैं, तो पथ थोड़ा अलग हो सकता है: `"/Applications/Sublime Text 2.app/Contents/SharedSupport/bin/subl"` (आवश्यकतानुसार वर्जन नंबर एडजस्ट करें)।
  - यदि `/usr/local/bin` आपके PATH में नहीं है, तो इसे अपने शेल प्रोफाइल (जैसे, `~/.zshrc` या `~/.bash_profile`) में एड करें:
    ```
    echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.zshrc
    source ~/.zshrc
    ```

- इसका टेस्ट करें: किसी डायरेक्टरी में नेविगेट करें (जैसे, `cd ~/Desktop`) और चलाएँ:
  ```
  subl .
  ```
  इससे Sublime Text खुल जाना चाहिए जिसमें वर्तमान फोल्डर लोड होगा।

यदि symlink पथ काम नहीं करता है (जैसे, वर्जन अंतर के कारण), तो सटीक लोकेशन चेक करें:
- `find /Applications/Sublime\ Text.app -name subl` चलाकर बाइनरी का पता लगाएँ।

**यह अच्छा क्यों है:**
- यह आधिकारिक और हल्का-फुल्का है—किसी थर्ड-पार्टी टूल की जरूरत नहीं।
- सिस्टम-वाइड काम करता है, जैसे एक रीयल CLI।
- Sublime Text 4 में कंसोल (View > Show Console) में `sublime_installation` या इसी तरह का कोई ऑप्शन भी होता है, लेकिन symlink सबसे विश्वसनीय तरीका है।

**यदि आप Linux या Windows पर हैं:**
- Linux: इसी तरह का symlink प्रोसेस—जैसे, `sudo ln -s /opt/sublime_text/sublime_text /usr/local/bin/subl`।
- Windows: अपने PATH में `C:\Program Files\Sublime Text\sublime_text.exe` को एड करें, या `subl` के लिए एक बैच फाइल का उपयोग करें।

#### 2. **फॉलबैक: `open` कमांड का उपयोग करें (कोई इंस्टॉलेशन जरूरत नहीं)**
यदि आप symlinks या PATH के साथ छेड़छाड़ नहीं करना चाहते हैं, तो macOS का बिल्ट-इन `open` कमांड बेसिक उपयोग के लिए बढ़िया काम करता है। यह पहले से ही हर जगह उपलब्ध है।

- Sublime में वर्तमान डायरेक्टरी खोलने के लिए:
  ```
  open -a "Sublime Text" .
  ```
  - नोट: यदि आपके ऐप का नाम `"Sublime Text"` (स्पेस के साथ) है तो इसका उपयोग करें। यदि यह सिर्फ "Sublime.app" है, तो तदनुसार एडजस्ट करें। आप `/Applications/` में जाकर सटीक नाम चेक कर सकते हैं।

- किसी विशिष्ट फाइल को खोलने के लिए:
  ```
  open -a "Sublime Text" filename.txt
  ```

- सुविधा के लिए इसे alias करें: इसे अपने शेल प्रोफाइल (`~/.zshrc` या `~/.bash_profile`) में एड करें ताकि यह `subl` जैसा व्यवहार करे:
  ```
  alias subl='open -a "Sublime Text"'
  ```
  फिर `source ~/.zshrc` के साथ रीलोड करें और हमेशा की तरह `subl .` का उपयोग करें।

**फायदे/नुकसान:**
- बहुत सरल और कोई सेटअप नहीं।
- लेकिन यह सिर्फ macOS पर काम करता है और कम लचीला है (जैसे, `subl` की तरह लाइन्स या प्रोजेक्ट्स स्पेसिफाई करने का कोई आसान तरीका नहीं)।

#### समस्या निवारण
- **Command not found?** सुनिश्चित करें कि आपके PATH में `/usr/local/bin` शामिल है (चेक करने के लिए `echo $PATH` चलाएँ)।
- **App not found?** पुष्टि करें कि Sublime `/Applications/` में है। यदि यह कहीं और है (जैसे, Homebrew के माध्यम से: `brew install --cask sublime-text`), तो पथ एडजस्ट करें।
- **Permissions issues?** Symlink के लिए sudo की जरूरत होती है, लेकिन एक बार सेट हो जाने पर, यह ठीक रहता है।
- यदि आप Homebrew जैसे पैकेज मैनेजर का उपयोग कर रहे हैं, तो आप Sublime को `brew install --cask sublime-text` के माध्यम से इंस्टॉल कर सकते हैं, और यह आपके लिए CLI को हैंडल भी कर सकता है।

इससे आपको अच्छी तरह से चलने में मदद मिलनी चाहिए! यदि आपको कोई एरर आती है या आप अपना OS/वर्जन स्पेसिफाई करते हैं, तो मैं चरणों को और बेहतर बना सकता हूँ।
