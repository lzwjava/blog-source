**reactor-core** का उपयोग करने के लिए, जो कि Reactive Streams specification पर आधारित नॉन-ब्लॉकिंग, एसिंक्रोनस एप्लिकेशन बनाने के लिए एक Java लाइब्रेरी है, इन चरणों का पालन करें:

---

### 1. reactor-core को एक डिपेंडेंसी के रूप में जोड़ें
सबसे पहले, अपने प्रोजेक्ट में **reactor-core** शामिल करें। आपके बिल्ड टूल के आधार पर:

- **Maven**: अपनी `pom.xml` फाइल में इसे जोड़ें:
  ```xml
  <dependency>
    <groupId>io.projectreactor</groupId>
    <artifactId>reactor-core</artifactId>
    <version>3.7.3</version>
  </dependency>
  ```

- **Gradle**: अपनी `build.gradle` फाइल में इसे जोड़ें:
  ```groovy
  implementation 'io.projectreactor:reactor-core:3.7.3'
  ```

यह सुनिश्चित करता है कि लाइब्रेरी आपके प्रोजेक्ट में उपलब्ध है। यदि आवश्यक हो तो [Maven Central](https://central.sonatype.com/artifact/io.projectreactor/reactor-core) पर नवीनतम संस्करण की जांच करें।

---

### 2. मुख्य घटकों को समझें
Reactor-core रिएक्टिव प्रोग्रामिंग के लिए दो मुख्य क्लास प्रदान करता है:
- **`Flux`**: एक एसिंक्रोनस स्ट्रीम को दर्शाता है जो **0 से N आइटम** emit कर सकती है।
- **`Mono`**: एक एसिंक्रोनस स्ट्रीम को दर्शाता है जो **0 या 1 आइटम** emit करती है।

ये वे बिल्डिंग ब्लॉक्स हैं जिनका उपयोग आप डेटा को रिएक्टिव तरीके से हैंडल करने के लिए करेंगे।

---

### 3. एक Flux या Mono बनाएं
आप अपनी डेटा स्ट्रीम्स को दर्शाने के लिए `Flux` या `Mono` के इंस्टेंस बना सकते हैं।

- **Flux के साथ उदाहरण** (एक से अधिक आइटम):
  ```java
  Flux<Integer> numbers = Flux.just(1, 2, 3, 4, 5);
  ```

- **Mono के साथ उदाहरण** (एकल आइटम):
  ```java
  Mono<String> greeting = Mono.just("Hello, World!");
  ```

`just` मेथड स्टेटिक वैल्यूज से एक स्ट्रीम बनाने का एक सरल तरीका है, लेकिन Reactor कई अन्य क्रिएशन मेथड्स (जैसे, arrays, ranges, या कस्टम स्रोतों से) प्रदान करता है।

---

### 4. डेटा को प्रोसेस करने के लिए सब्सक्राइब करें
Emit किए गए आइटम्स का उपभोग करने के लिए, आपको `Flux` या `Mono` को **सब्सक्राइब** करना होगा। सब्सक्राइब करने से स्ट्रीम डेटा emit करना शुरू कर देती है।

- **Flux को सब्सक्राइब करें**:
  ```java
  numbers.subscribe(System.out::println);  // प्रिंट करेगा: 1, 2, 3, 4, 5
  ```

- **Mono को सब्सक्राइब करें**:
  ```java
  greeting.subscribe(System.out::println); // प्रिंट करेगा: Hello, World!
  ```

`subscribe` मेथड अधिक नियंत्रण के लिए अतिरिक्त आर्गुमेंट्स, जैसे एरर हैंडलर्स या कम्प्लीशन कॉलबैक्स, भी ले सकती है।

---

### 5. ऑपरेटर्स के साथ डेटा को ट्रांसफॉर्म करें
Reactor स्ट्रीम्स को मैनिपुलेट करने के लिए ऑपरेटर्स का एक समृद्ध सेट प्रदान करता है, जैसे `map`, `filter`, और भी बहुत कुछ।

- **Flux और map के साथ उदाहरण**:
  ```java
  numbers.map(n -> n * 2).subscribe(System.out::println);  // प्रिंट करेगा: 2, 4, 6, 8, 10
  ```

- **Mono और map के साथ उदाहरण**:
  ```java
  greeting.map(s -> s.toUpperCase()).subscribe(System.out::println); // प्रिंट करेगा: HELLO, WORLD!
  ```

ये ऑपरेटर्स आपको डेक्लेरेटिव तरीके से डेटा को ट्रांसफॉर्म, फिल्टर, या कॉम्बाइन करने की अनुमति देते हैं।

---

### 6. एरर्स और बैकप्रेशर को हैंडल करें
Reactor-core एरर हैंडलिंग और बैकप्रेशर मैनेजमेंट को सपोर्ट करता है:
- **एरर हैंडलिंग**: एक एरर कंज्यूमर के साथ `subscribe` का उपयोग करें:
  ```java
  Flux.error(new RuntimeException("Oops!"))
      .subscribe(System.out::println, error -> System.err.println(error.getMessage()));
  // प्रिंट करेगा: Oops!
  ```
- **बैकप्रेशर**: `onBackpressureBuffer()` या `onBackpressureDrop()` जैसे ऑपरेटर्स का उपयोग करके सब्सक्राइबर्स को अत्यधिक डेटा स्ट्रीम्स को कैसे हैंडल करना है, इसे नियंत्रित करें।
