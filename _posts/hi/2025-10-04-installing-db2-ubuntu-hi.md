### Ubuntu 22.04 या 24.04 पर IBM Db2 12.1.2 को `ibm_db2_v12.1.2_linuxx64_server_dec.tar.gz` से इंस्टॉल करना

IBM Db2 वर्जन 12.1.2 (फिक्स पैक 2) हाल के Ubuntu वर्जन जैसे 22.04 (जैमी) और 24.04 (नोबल) पर सपोर्टेड है, बशर्ते आपका सिस्टम हार्डवेयर और सॉफ्टवेयर पूर्वापेक्षाओं को पूरा करता हो। यह इंस्टॉलेशन tar.gz फाइल से ग्राफिकल या साइलेंट इंस्टॉलर का उपयोग करती है। मैं नवीनतम विवरणों के लिए आईबीएम Db2 12.1 की आधिकारिक IBM डॉक्यूमेंटेशन (जैसे, IBM नॉलेज सेंटर या Db2 डाउनलोड पेज पर) की समीक्षा करने की सलाह देता हूं, क्योंकि आवश्यकताएं एडिशन (यहां सर्वर एडिशन) के अनुसार भिन्न हो सकती हैं।

**शुरू करने से पहले महत्वपूर्ण नोट्स:**
- **सिस्टम आवश्यकताएँ**:
  - 64-बिट x86_64 आर्किटेक्चर (इंटेल/एएमडी)।
  - कम से कम 4 GB RAM (8 GB अनुशंसित) और 2 GB स्वैप स्पेस।
  - बेस इंस्टॉलेशन के लिए 10 GB खाली डिस्क स्पेस (डेटा के लिए अधिक)।
  - रूट या sudo एक्सेस।
  - कर्नल वर्जन: Ubuntu 22.04/24.04 को काम करना चाहिए, लेकिन सुनिश्चित करें कि आपका कर्नल कम से कम 3.10 है (`uname -r` से चेक करें)।
  - फायरवॉल: अस्थायी रूप से डिसेबल करें या पोर्ट्स खोलें (डिफॉल्ट Db2: 50000 टीसीपी/आईपी के लिए)।
- **Ubuntu पर संभावित समस्याएं**:
  - Db2 मुख्य रूप से RHEL/SUSE पर टेस्ट किया गया है, लेकिन Ubuntu डेबियन पैकेज के माध्यम से सपोर्टेड है। आपको लाइब्रेरी डिपेंडेंसी को रिजॉल्व करने की आवश्यकता हो सकती है।
  - यदि आप Ubuntu 24.04 पर हैं, तो यह बहुत नया है—पहले VM में टेस्ट करें, क्योंकि पूर्ण सर्टिफिकेशन पीछे रह सकता है।
  - यह सर्वर एडिशन इंस्टॉल करता है। अन्य एडिशन (जैसे, एक्सप्रेस-सी) के लिए उचित tar.gz डाउनलोड करें।
- **बैकअप**: आगे बढ़ने से पहले अपने सिस्टम का बैकअप लें।
- फाइल को आधिकारिक IBM पासपोर्ट एडवांटेज या Db2 डाउनलोड साइट से डाउनलोड करें (IBM आईडी की आवश्यकता होती है)।

#### चरण 1: पूर्वापेक्षाएँ इंस्टॉल करें
अपने सिस्टम को अपडेट करें और आवश्यक लाइब्रेरीज इंस्टॉल करें। Db2 को एसिंक्रोनस I/O, PAM, और अन्य रनटाइम लाइब्स की आवश्यकता होती है।

```bash
sudo apt update
sudo apt upgrade -y

# आवश्यक पैकेजेज इंस्टॉल करें (Ubuntu/Debian पर Db2 के लिए कॉमन)
sudo apt install -y libaio1 libpam0g:i386 libncurses5 libstdc++6:i386 \
    unixodbc unixodbc-dev libxml2 libxslt1.1 wget curl

# Ubuntu 24.04 के लिए, आपको इनकी भी आवश्यकता हो सकती है:
sudo apt install -y libc6:i386 libgcc-s1:i386

# glibc कम्पैटिबिलिटी वेरिफाई करें (Db2 12.1 को glibc 2.17+ चाहिए)
ldd --version  # Ubuntu 22.04/24.04 पर glibc 2.35+ दिखाना चाहिए
```

यदि आप गुम 32-बिट लाइब्स (जैसे, Java कंपोनेंट्स के लिए) का सामना करते हैं, तो मल्टीआर्क को एनेबल करें:
```bash
sudo dpkg --add-architecture i386
sudo apt update
sudo apt install -y libc6:i386 libncurses5:i386 libstdc++6:i386
```

#### चरण 2: इंस्टॉलेशन फाइल्स तैयार करें
1. एक्सट्रैक्शन के लिए एक टेम्पररी डायरेक्टरी बनाएं (जैसे, `/tmp/db2_install`):
   ```bash
   sudo mkdir -p /tmp/db2_install
   cd /tmp/db2_install
   ```

2. tar.gz फाइल को इस डायरेक्टरी में कॉपी करें (मान लें कि आपने इसे डाउनलोड कर लिया है, जैसे, `~/Downloads` में):
   ```bash
   cp ~/Downloads/ibm_db2_v12.1.2_linuxx64_server_dec.tar.gz .
   ```

3. आर्काइव को एक्सट्रैक्ट करें:
   ```bash
   tar -xzf ibm_db2_v12.1.2_linuxx64_server_dec.tar.gz
   ```
   - यह `db2` या `sqllib` जैसी एक डायरेक्टरी बनाता है जिसमें इंस्टॉलर फाइल्स (जैसे, `db2setup`) होती हैं।

4. एक्सट्रैक्ट की गई डायरेक्टरी में चेंज करें:
   ```bash
   cd db2  # या जो भी टॉप-लेवल डिर है—`ls` से चेक करें
   ```

#### चरण 3: इंस्टॉलर रन करें
Db2 एक ग्राफिकल इंस्टॉलर (`db2setup`) या साइलेंट इंस्टॉल के लिए एक रिस्पांस फाइल प्रदान करता है। रूट/सुडो के रूप में रन करें।

**विकल्प A: ग्राफिकल इंस्टॉलर (पहली बार सेटअप के लिए अनुशंसित)**
1. सुनिश्चित करें कि आपके पास एक डिस्प्ले है (यदि बिना GUI के सर्वर पर है, तो SSH के साथ X फॉरवर्डिंग का उपयोग करें: `ssh -X user@host`)।
2. इंस्टॉलर रन करें:
   ```bash
   sudo ./db2setup
   ```
   - विजार्ड आपका मार्गदर्शन करेगा:
     - लाइसेंस स्वीकार करें।
     - सर्वर एडिशन के लिए "टाइपिकल" इंस्टॉलेशन चुनें।
     - इंस्टॉल पाथ चुनें (डिफॉल्ट: `/opt/ibm/db2/V12.1`—सुनिश्चित करें कि `/opt/ibm` मौजूद है और लिखने योग्य है; यदि आवश्यक हो तो `sudo mkdir -p /opt/ibm` से बनाएं)।
     - एक Db2 इंस्टेंस बनाएं (जैसे, "db2inst1")—यह डेटाबेस एडमिनिस्ट्रेटर यूजर सेट करता है।
     - ऑथेंटिकेशन सेट करें (जैसे, लोकल या LDAP)।
     - आवश्यकता होने पर SQL प्रोसीजरल लैंग्वेज जैसी फीचर्स को एनेबल करें।
   - इंस्टॉलर इंस्टेंस को कंपाइल और सेट अप करेगा।

**विकल्प B: साइलेंट इंस्टॉलेशन (नॉन-इंटरएक्टिव)**
यदि आप स्क्रिप्टिंग पसंद करते हैं:
1. ड्राई-रन के दौरान एक रिस्पांस फाइल जनरेट करें:
   ```bash
   sudo ./db2setup -g  # करंट डिर में `db2setup.rsp` जनरेट करता है
   ```
   `db2setup.rsp` को एडिट करें (जैसे, `LIC_AGREEMENT=ACCEPT`, `INSTALL_TYPE=TYPICAL`, `CREATE_DB2_INSTANCE=YES`, आदि सेट करें)।

2. साइलेंट इंस्टॉल रन करें:
   ```bash
   sudo ./db2setup -u db2setup.rsp
   ```

- इंस्टॉलेशन में 10-30 मिनट लगते हैं। `/tmp/db2setup.log` में एरर के लिए निगरानी करें।

#### चरण 4: पोस्ट-इंस्टॉलेशन सेटअप
1. **इंस्टॉलेशन वेरिफाई करें**:
   - इंस्टेंस ओनर के रूप में लॉग इन करें (जैसे, `db2inst1`—इंस्टॉल के दौरान बनाया गया):
     ```bash
     su - db2inst1
     ```
   - Db2 वर्जन चेक करें:
     ```bash
     db2level
     ```
   - इंस्टेंस स्टार्ट करें:
     ```bash
     db2start
     ```
   - कनेक्शन टेस्ट करें:
     ```bash
     db2 connect to sample  # यदि कोई डेटाबेस नहीं है तो सैंपल डीबी बनाता है
     db2 "select * from sysibm.sysdummy1"
     db2 disconnect all
     db2stop  # जब पूरा हो जाए
     ```

2. **एक डेटाबेस बनाएं (यदि इंस्टॉल के दौरान नहीं बनाया गया है)**:
   ```bash
   su - db2inst1
   db2sampl  # ऑप्शनल: सैंपल डीबी बनाता है
   # या कस्टम डीबी बनाएं:
   db2 "create database MYDB"
   db2 connect to MYDB
   ```

3. **एनवायरनमेंट सेटअप**:
   - इंस्टेंस यूजर के लिए PATH में Db2 जोड़ें (`~/.bashrc` में जोड़ें):
     ```bash
     export PATH=/opt/ibm/db2/V12.1/bin:$PATH
     export DB2INSTANCE=db2inst1
     ```
   - रीलोड करें: `source ~/.bashrc`.

4. **रिमोट एक्सेस एनेबल करें (ऑप्शनल)**:
   - सर्विसेज अपडेट करें:
     ```bash
     su - db2inst1
     db2 update dbm cfg using SVCENAME db2i  # या आपका पोर्ट
     db2set DB2COMM=TCPIP
     db2start
     ```
   - `/etc/services` (रूट के रूप में) एडिट करके जोड़ें:
     ```
     db2i          50000/tcp
     ```
   - इंस्टेंस रीस्टार्ट करें।

5. **फायरवॉल कॉन्फ़िगरेशन**:
   ```bash
   sudo ufw allow 50000/tcp  # Db2 डिफॉल्ट पोर्ट के लिए
   sudo ufw reload
   ```

#### सामान्य समस्याओं का समाधान
- **"No suitable Java runtime"**: OpenJDK इंस्टॉल करें: `sudo apt install openjdk-11-jre-headless`.
- **लाइब्रेरी एरर (जैसे, libaio.so)**: सुनिश्चित करें कि `libaio1` इंस्टॉल है और `sudo ldconfig` रन करें।
- **परमिशन डिनाइड**: इंस्टॉलर को रूट के रूप में रन करें; `/opt/ibm` की ओनरशिप सुनिश्चित करें: `sudo chown -R db2inst1:db2iadm1 /opt/ibm/db2`.
- **Glibc इनकम्पैटिबिलिटी**: Ubuntu 22.04/24.04 पर असंभावित, लेकिन यदि समस्याएं आती हैं, तो कंटेनर (Docker) इंस्टॉल पर विचार करें।
- **साइलेंट इंस्टॉल फेल**: `db2setup.log` में सुराग के लिए चेक करें।
- **Ubuntu 24.04 स्पेसिफिक**: यदि लाइब डिपेंडेंसी फेल होती है, तो आपको PPA जोड़ने या गुम लाइब्स को कंपाइल करने की आवश्यकता हो सकती है। पहले 22.04 पर टेस्ट करें।
- एरर? IBM के सपोर्ट फोरम में सर्च करें या इंस्टेंस यूजर के रूप में `db2diag` रन करें।

यदि आप विशिष्ट एरर का सामना करते हैं, तो अधिक लक्षित सहायता के लिए विवरण प्रदान करें। प्रोडक्शन उपयोग के लिए, IBM के सर्टिफाइड एनवायरनमेंट या कंसल्टिंग सपोर्ट पर विचार करें। सफलता! आपका Db2 इंस्टेंस अब डेटाबेस क्रिएशन और उपयोग के लिए तैयार होना चाहिए।
