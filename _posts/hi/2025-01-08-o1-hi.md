मुझे हाल ही में GitHub फ्लो में `fontawesome5` पैकेज के साथ एक समस्या का सामना करना पड़ा। 4o-mini द्वारा प्रदान किया गया समाधान (TeX Live 2021 और `tlmgr install fontawesome5` का उपयोग करके) काम नहीं आया, लेकिन 4o ने एक बेहतर तरीका सुझाया (TeX Live 2023 का उपयोग करके और `tlmgr` का उपयोग करके `fontawesome5` इंस्टॉल करके), जिसे मैं अभी आजमा रहा हूँ। इससे समस्या पूरी तरह से ठीक नहीं हुई, लेकिन TeX Live 2021 के बजाय 2023 पर स्विच करने से काफी सुधार हुआ है।

---

## 4o-mini उत्तर

आप जो त्रुटि देख रहे हैं, वह `tlmgr` (TeX Live Manager) से संबंधित है जो उपयोगकर्ता मोड में ठीक से प्रारंभ नहीं हो रहा है। विशेष रूप से, संदेश `tlmgr: user mode not initialized, please read the documentation!` इंगित करता है कि `tlmgr` उपयोगकर्ता वातावरण में सही ढंग से सेटअप नहीं हो रहा है।

यहां बताया गया है कि आप इसे कैसे ठीक कर सकते हैं:

### `tlmgr` यूज़र मोड एरर को ठीक करना

1. **`tlmgr` को यूज़र मोड में सही तरीके से सेट करें**:
   आपको `tlmgr` को स्पष्ट रूप से यूज़र मोड में चलाने के लिए कहना होगा और यह सुनिश्चित करना होगा कि यह ठीक से इनिशियलाइज़ हो। अपने स्क्रिप्ट में इसे करने के लिए, निम्नलिखित चरणों का उपयोग करें:

   - सुनिश्चित करें कि आप सही पर्यावरण चर सेट करके `tlmgr` को उपयोगकर्ता मोड में उपयोग कर रहे हैं।
   - `tlmgr` का उपयोग करने से पहले आप उपयोगकर्ता-विशिष्ट TeX Live डायरेक्टरी को भी प्रारंभ कर सकते हैं।

2. **यूजर मोड में `tlmgr` को इनिशियलाइज़ करें**:
   `tlmgr` कमांड्स को चलाने से पहले, यह सुनिश्चित करें कि `tlmgr` को निम्नलिखित कमांड के साथ इनिशियलाइज़ किया गया है:
   ```bash
   tlmgr init-usertree
   ```

यह उपयोगकर्ता-विशिष्ट TeX Live ट्री को `~/texmf` में बनाएगा। एक बार इनिशियलाइज़ हो जाने के बाद, आप `tlmgr update` और `install` कमांड चला सकते हैं।

3. **अपने GitHub Actions वर्कफ़्लो को संशोधित करें**:
   अपने GitHub Actions वर्कफ़्लो फ़ाइल को आरंभीकरण चरण को शामिल करने के लिए अपडेट करें। यहां एक उदाहरण दिया गया है कि आप वर्कफ़्लो को कैसे संशोधित कर सकते हैं:

```yaml
   - name: tlmgr को यूजर मोड में सेट करें
     run: |
       # यूजर मोड TeX Live डायरेक्टरी को इनिशियलाइज़ करें
       tlmgr init-usertree
       tlmgr update --self
       tlmgr install fontawesome5
   ```

### पूर्ण उदाहरण:

यदि आप इसे एक बड़े GitHub Actions वर्कफ़्लो के हिस्से के रूप में चला रहे हैं, तो आप इन परिवर्तनों को संबंधित जॉब सेक्शन में शामिल कर सकते हैं:

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: TeX Live (xelatex) इंस्टॉल करें
        run: |
          sudo apt-get update
          sudo apt-get install -y texlive-xetex
          tlmgr init-usertree  # TeX Live यूजर ट्री को इनिशियलाइज़ करें
          tlmgr update --self  # tlmgr को अपडेट करें
          tlmgr install fontawesome5  # वांछित पैकेज इंस्टॉल करें
```

### कमांड्स की व्याख्या:

- `tlmgr init-usertree`: यह उपयोगकर्ता-विशिष्ट TeX Live ट्री को प्रारंभ करता है (आमतौर पर `~/texmf` में स्थित)।
- `tlmgr update --self`: `tlmgr` पैकेज को स्वयं नवीनतम संस्करण में अपडेट करता है।
- `tlmgr install fontawesome5`: `fontawesome5` पैकेज को इंस्टॉल करता है।

### अतिरिक्त नोट्स:

- **अनुमतियाँ**: सुनिश्चित करें कि GitHub Actions वर्कफ़्लो चलाने वाले उपयोगकर्ता के पास TeX Live उपयोगकर्ता निर्देशिका में फ़ाइलें स्थापित करने और संशोधित करने की अनुमति हो।
- **पर्यावरण चर**: यदि आप कस्टम पथ के साथ काम कर रहे हैं, तो आपको अपने विशिष्ट निर्देशिकाओं की ओर इशारा करने के लिए `TEXMFSYSVAR` या `TEXMFHOME` को समायोजित करने की आवश्यकता हो सकती है।

`tlmgr` को यूजर मोड में इनिशियलाइज़ करने और इसे अपने GitHub Actions वर्कफ़्लो में सही ढंग से सेट करने से, यह त्रुटि हल हो जानी चाहिए। यदि यह काम करता है या आपको कोई अन्य समस्या आती है, तो मुझे बताएं!

### o1 उत्तर

मैं समझता हूं कि आपको अपने GitHub Actions वर्कफ़्लो में `tlmgr` (TeX Live Manager) के साथ समस्याएं आ रही हैं, विशेष रूप से `fontawesome5` पैकेज को इंस्टॉल करने की कोशिश करते समय। त्रुटि संदेश:

```
TLPDB: डायरेक्टरी नहीं है, लोड नहीं हो रहा: /home/runner/texmf
tlmgr: उपयोगकर्ता मोड प्रारंभ नहीं हुआ है, कृपया दस्तावेज़ीकरण पढ़ें!
त्रुटि: प्रक्रिया ने निकास कोड 2 के साथ पूरा किया।
```

यह संकेत देता है कि `tlmgr` "यूज़र मोड" में काम करने में असमर्थ है क्योंकि आवश्यक डायरेक्टरीज़ और कॉन्फ़िगरेशन सही तरीके से सेट अप नहीं हैं। यह समस्या अक्सर तब उत्पन्न होती है जब `apt-get` जैसे पैकेज मैनेजर द्वारा प्रदान किए गए TeX Live इंस्टॉलेशन का उपयोग किया जाता है, जो TUG (TeX Users Group) से पूर्ण TeX Live इंस्टॉलेशन की तुलना में सीमित कार्यक्षमता प्रदान कर सकते हैं।

इसे हल करने के लिए, मैं आपके GitHub Actions वर्कफ़्लो के भीतर **TeX Live 2023 को मैन्युअल रूप से इंस्टॉल करने** की सलाह देता हूं। यह दृष्टिकोण सुनिश्चित करता है कि `tlmgr` पूरी तरह से कार्यात्मक हो और `fontawesome5` जैसे अतिरिक्त पैकेजों को बिना किसी समस्या के इंस्टॉल कर सके।

### चरण-दर-चरण समाधान:

1. **सिस्टम TeX Live इंस्टॉलेशन को हटाएं:**
   - `apt-get install texlive-xetex` का उपयोग करने से बचें क्योंकि यह एक आंशिक TeX Live इंस्टॉलेशन प्रदान करता है जिसमें `tlmgr` की कार्यक्षमता सीमित होती है।
   
2. **TeX Live 2023 को मैन्युअल रूप से इंस्टॉल करें:**
   - TeX Live को सीधे TUG से डाउनलोड और इंस्टॉल करें।
   
3. **PATH एनवायरनमेंट वेरिएबल को कॉन्फ़िगर करें:**
   - PATH को अपडेट करके यह सुनिश्चित करें कि TeX Live बाइनरीज़ एक्सेसिबल हैं।

4. **आवश्यक पैकेज स्थापित करने के लिए `tlmgr` का उपयोग करें:**
   - `tlmgr` को अपडेट करें और `fontawesome5` स्थापित करें।

5. **कैशिंग के साथ वर्कफ़्लो को अनुकूलित करें (वैकल्पिक):**
   - भविष्य के वर्कफ़्लो रन को तेज़ करने के लिए TeX Live इंस्टॉलेशन को कैश करें।

### विस्तृत व्याख्या:

1. **रिपॉजिटरी चेकआउट करें:**
   - आपके रिपॉजिटरी को क्लोन करने के लिए `actions/checkout@v4` एक्शन का उपयोग करता है।

2. **Make सेट करें:**
   - `apt-get` का उपयोग करके `make` को इंस्टॉल करें, यह सुनिश्चित करते हुए कि यह आपके CV को बनाने के लिए उपलब्ध है।

3. **TeX Live को कैश करें:**
   - `/usr/local/texlive/2023` पर स्थित TeX Live इंस्टॉलेशन को कैश करने के लिए `actions/cache@v3` का उपयोग करता है। यह बाद के वर्कफ़्लो रन के लिए इंस्टॉलेशन समय को काफी कम कर देता है।

4. **TeX Live 2023 इंस्टॉल करें:**
   - **सशर्त निष्पादन:** `if: steps.cache-texlive.outputs.cache-hit != 'true'` स्थिति यह सुनिश्चित करती है कि TeX Live केवल तभी इंस्टॉल हो जब यह पहले से कैश में न हो।
   - **आवश्यक निर्भरताएँ इंस्टॉल करें:** Perl, `wget`, और `xz-utils` जैसी आवश्यक निर्भरताएँ इंस्टॉल करता है।
   - **इंस्टॉलर डाउनलोड और एक्सट्रैक्ट करें:** CTAN से नवीनतम TeX Live इंस्टॉलर डाउनलोड करता है और इसे एक्सट्रैक्ट करता है।
   - **इंस्टॉलेशन प्रोफाइल बनाएँ:** इंस्टॉलेशन प्रक्रिया को स्वचालित करने के लिए एक `texlive.profile` बनाया जाता है। यह प्रोफाइल निम्नलिखित निर्दिष्ट करती है:
     - **`selected_scheme scheme-full`:** पूर्ण TeX Live वितरण इंस्टॉल करता है। यदि चाहें तो इसे एक न्यूनतम स्कीम में समायोजित कर सकते हैं।
     - **निर्देशिकाएँ:** TeX Live और इसकी कॉन्फ़िगरेशन कहाँ इंस्टॉल होगी, यह निर्दिष्ट करता है।
     - **संग्रह:** `basic`, `latex`, और `fontsrecommended` जैसे आवश्यक संग्रहों का चयन करता है।
   - **इंस्टॉलर चलाएँ:** निर्दिष्ट प्रोफाइल के साथ इंस्टॉलर चलाता है, TeX Live 2023 को `/usr/local/texlive/2023` में इंस्टॉल करता है।

5. **TeX Live को PATH में जोड़ें:**
   - `PATH` पर्यावरण चर को TeX Live बाइनरी डायरेक्टरी को जोड़कर अपडेट करता है। यह सुनिश्चित करता है कि `tlmgr` और अन्य TeX Live कमांड बाद के चरणों में उपलब्ध हों।

6. **tlmgr को अपडेट करें और `fontawesome5` इंस्टॉल करें:**
   - **`tlmgr` को अपडेट करें:** यह सुनिश्चित करता है कि `tlmgr` स्वयं अप-टू-डेट है।
   - **`fontawesome5` इंस्टॉल करें:** `tlmgr` का उपयोग करके `fontawesome5` पैकेज को इंस्टॉल करें, जो आपकी प्रारंभिक समस्या को हल करता है।

7. **Make कमांड चलाएं:**
   - `make awesome-cv-copy` कमांड चलाकर, आपके द्वारा इंस्टॉल किए गए TeX Live सेटअप का उपयोग करके अपना CV बनाएं।

8. **बिल्ड और डिप्लॉय जॉब्स:**
   - ये ज्यादातर अपरिवर्तित रहते हैं, लेकिन यह सुनिश्चित करें कि अपडेटेड PATH के कारण TeX Live इंस्टॉलेशन पूरे वर्कफ़्लो में उपलब्ध है।

### अतिरिक्त सिफारिशें:

- **TeX Live इंस्टॉलेशन समय:**
  - **कैशिंग:** `actions/cache@v3` चरण को शामिल करने से पहली बार चलने के बाद TeX Live को कैश करके लंबे इंस्टॉलेशन समय को कम करने में मदद मिलती है। यह सुनिश्चित करें कि कैश कुंजी (`${{ runner.os }}-texlive-2023`) TeX Live संस्करण को सही ढंग से दर्शाती है ताकि कैश मिसमैच से बचा जा सके।
  
- **TeX Live इंस्टॉलेशन को अनुकूलित करना:**
  - यदि आपको पूर्ण TeX Live वितरण की आवश्यकता नहीं है, तो आप `texlive.profile` को संशोधित करके केवल विशिष्ट संग्रहों को इंस्टॉल कर सकते हैं, जिससे इंस्टॉलेशन समय और डिस्क उपयोग कम हो जाता है। उदाहरण के लिए, `scheme-full` को `scheme-basic` से बदलें और केवल आवश्यक संग्रहों को निर्दिष्ट करें।

- **स्थापना की पुष्टि करना:**
  - यह सुनिश्चित करने के लिए कि TeX Live सही ढंग से स्थापित है, आप एक पुष्टिकरण चरण जोड़ सकते हैं:
    ```yaml
    - name: TeX Live स्थापना की पुष्टि करें
      run: |
        tex --version
        tlmgr --version
        kpsewhich fontawesome5.sty
    ```
    यह चरण TeX Live कमांड्स की पहुंच और `fontawesome5.sty` के स्थापित होने की पुष्टि करके किसी भी समस्या का निदान करने में मदद कर सकता है।

- **PATH को सही तरीके से संभालना:**
  - `echo "/usr/local/texlive/2023/bin/x86_64-linux" >> $GITHUB_PATH` का उपयोग करने से यह सुनिश्चित होता है कि TeX Live बाइनरीज़ को जॉब के सभी बाद के चरणों के लिए PATH में जोड़ दिया जाता है। यह `tlmgr` और LaTeX कमांड्स के सही ढंग से काम करने के लिए महत्वपूर्ण है।

### समस्या निवारण युक्तियाँ:

1. **कैश मिस समस्याएँ:**
   - सुनिश्चित करें कि कैश कुंजी TeX Live संस्करण को सही ढंग से दर्शाती है। यदि आप TeX Live को अपडेट करते हैं या इंस्टॉलेशन पथ बदलते हैं, तो कैश कुंजी को तदनुसार अपडेट करें।

2. **इंस्टॉलेशन विफलताएँ:**
   - TeX Live इंस्टॉलेशन के दौरान किसी भी त्रुटि के लिए GitHub Actions लॉग की जाँच करें। सामान्य समस्याओं में इंस्टॉलर डाउनलोड करते समय नेटवर्क समस्याएँ या अपर्याप्त अनुमतियाँ शामिल हैं।

3. **पैकेज इंस्टॉलेशन त्रुटियाँ:**
   - यदि `tlmgr` `fontawesome5` को इंस्टॉल करने में विफल हो जाता है, तो सत्यापित करें कि पैकेज का नाम सही है और यह TeX Live रिपॉजिटरी में उपलब्ध है।

4. **PATH समस्याएँ:**
   - सुनिश्चित करें कि PATH सही ढंग से अपडेट हो गया है। आप डिबगिंग के लिए PATH को प्रिंट करने के लिए एक कदम जोड़ सकते हैं:
     ```yaml
     - name: Print PATH
       run: echo $PATH
     ```

5. **डिस्क स्पेस सीमाएँ:**
   - GitHub Actions रनर्स की डिस्क स्पेस सीमित होती है। एक पूर्ण TeX Live इंस्टॉलेशन काफी जगह ले सकता है। यदि आपको डिस्क स्पेस की समस्या का सामना करना पड़ता है, तो एक न्यूनतम TeX Live सेटअप इंस्टॉल करने पर विचार करें।

### प्रमुख सुधार:

1. **सत्यापन चरण:**
   - **TeX Live स्थापना की जाँच करें:** यह सुनिश्चित करता है कि `tex`, `tlmgr`, और `fontawesome5.sty` उपलब्ध हैं।
   - **`fontawesome5` स्थापना की जाँच करें:** यह पुष्टि करता है कि `fontawesome5.sty` फ़ाइल सही ढंग से स्थापित है।

2. **बेहतर कैशिंग:**
   - **कैश कुंजी सटीकता:** कैश कुंजी OS और TeX Live संस्करण पर आधारित है, यह सुनिश्चित करते हुए कि TeX Live में परिवर्तन होने पर एक नई कैश की आवश्यकता होती है।

3. **त्रुटि प्रबंधन:**
   - **फॉलबैक संदेश:** `kpsewhich fontawesome5.sty || echo "fontawesome5.sty not found"` कमांड एक फॉलबैक संदेश प्रदान करता है यदि पैकेज नहीं मिलता है, जो डिबगिंग में सहायता करता है।

### अतिरिक्त सुझाव:

- **न्यूनतम TeX Live इंस्टालेशन:**
  - यदि पूर्ण TeX Live इंस्टालेशन बहुत समय लेता है या बहुत अधिक स्थान लेता है, तो एक न्यूनतम स्कीम इंस्टॉल करने और केवल आवश्यक पैकेज जोड़ने पर विचार करें।
  - `texlive.profile` को तदनुसार अपडेट करें:
    ```yaml
    selected_scheme scheme-basic
    collection-latexrecommended 1
    collection-fontsrecommended 1
    collection-fontsextra 1
    collection-luatex 1
    ```
  
- **जॉब्स को समानांतर करना:**
  - आपके वर्कफ़्लो की जटिलता के आधार पर, आप रनटाइम को अनुकूलित करने के लिए जॉब्स को समानांतर करने या जॉब्स में चरणों का पुन: उपयोग करने पर विचार कर सकते हैं।

- **TeX Live अपडेट को संभालना:**
  - `tlmgr` और TeX Live को नियमित रूप से अपडेट करना संगतता और नवीनतम पैकेजों तक पहुंच सुनिश्चित करता है।

### निष्कर्ष:

अपने GitHub Actions वर्कफ़्लो में TeX Live 2023 को मैन्युअल रूप से इंस्टॉल करके और पर्यावरण को सही ढंग से कॉन्फ़िगर करके, आप सिस्टम-इंस्टॉल किए गए TeX Live पैकेजों से जुड़ी सीमाओं और त्रुटियों को दूर कर सकते हैं। यह सेटअप पूर्ण `tlmgr` कार्यक्षमता प्रदान करता है, जिससे आप `fontawesome5` जैसे अतिरिक्त LaTeX पैकेजों को सहजता से इंस्टॉल कर सकते हैं।

यदि आपको अभी भी समस्याएँ आ रही हैं, तो कृपया अधिक विस्तृत लॉग या विशिष्ट त्रुटि संदेश प्रदान करें, और मैं आगे सहायता करने में प्रसन्न रहूंगा!
