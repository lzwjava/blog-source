एक विंडोज़ बैच फ़ाइल (.bat या .cmd) एक स्क्रिप्ट फ़ाइल है जिसमें विंडोज़ कमांड प्रॉम्प्ट (cmd.exe) द्वारा निष्पादित कमांडों की एक श्रृंखला होती है। बैच फ़ाइलें दोहराए जाने वाले कार्यों को स्वचालित करती हैं, सिस्टम संचालन का प्रबंधन करती हैं, या प्रोग्राम निष्पादित करती हैं। नीचे विंडोज़ बैच फ़ाइलें लिखने का एक व्यापक परिचय दिया गया है, जिसमें उनका उद्देश्य, सिंटैक्स, सामान्य कमांड और सर्वोत्तम अभ्यास शामिल हैं।

### बैच फ़ाइल क्या है?
एक बैच फ़ाइल एक सादा पाठ फ़ाइल है जिसमें `.bat` या `.cmd` एक्सटेंशन होता है और इसमें विंडोज़ कमांड प्रॉम्प्ट द्वारा व्याख्या किए गए कमांड होते हैं। निष्पादित होने पर, कमांड क्रमिक रूप से चलते हैं, जिससे फ़ाइल प्रबंधन, सिस्टम कॉन्फ़िगरेशन, या सॉफ़्टवेयर इंस्टॉलेशन जैसे कार्यों का स्वचालन संभव होता है।

### बैच फ़ाइलों का उपयोग क्यों करें?
- **स्वचालन**: एक स्क्रिप्ट के साथ कई कमांड निष्पादित करें।
- **सरलता**: उन्नत प्रोग्रामिंग ज्ञान की आवश्यकता नहीं है।
- **सिस्टम प्रबंधन**: बैकअप, उपयोगकर्ता प्रबंधन, या वातावरण सेटअप जैसे कार्य करें।
- **संगतता**: कमांड प्रॉम्प्ट वाले सभी विंडोज़ संस्करणों पर काम करती है।

### बैच फ़ाइल बनाना
1.  **स्क्रिप्ट लिखें**: कमांड लिखने के लिए एक टेक्स्ट एडिटर (जैसे, Notepad, VS Code) का उपयोग करें।
2.  **सही एक्सटेंशन के साथ सहेजें**: फ़ाइल को `.bat` या `.cmd` एक्सटेंशन के साथ सहेजें (उदाहरण के लिए, `script.bat`)।
3.  **निष्पादित करें**: फ़ाइल पर डबल-क्लिक करें या इसे कमांड प्रॉम्प्ट के माध्यम से चलाएं।

### बुनियादी सिंटैक्स और संरचना
- **कमांड**: बैच फ़ाइलें कमांड प्रॉम्प्ट कमांड (जैसे, `dir`, `copy`, `del`) और बैच-विशिष्ट कमांड (जैसे, `echo`, `set`, `goto`) का उपयोग करती हैं।
- **टिप्पणियाँ**: स्पष्टता के लिए टिप्पणियाँ जोड़ने के लिए `REM` या `::` का उपयोग करें।
- **केस इनसेंसिटिविटी**: कमांड और वेरिएबल केस-सेंसिटिव नहीं होते हैं।
- **लाइन निष्पादन**: कमांड लाइन दर लाइन निष्पादित होते हैं, जब तक कि `if`, `for`, या `goto` जैसे फ्लो कमांड द्वारा नियंत्रित नहीं किए जाते।

### सामान्य कमांड और विशेषताएं
#### 1. **बुनियादी कमांड**
- `ECHO`: कमांड इको को नियंत्रित करता है या पाठ प्रदर्शित करता है।
  - उदाहरण: `ECHO Hello, World!` "Hello, World!" प्रदर्शित करता है।
  - `ECHO OFF`: निष्पादन के दौरान कमांड प्रदर्शन को दबा देता है।
- `CLS`: कमांड प्रॉम्प्ट स्क्रीन साफ़ करता है।
- `PAUSE`: निष्पादन को रोकता है, उपयोगकर्ता इनपुट की प्रतीक्षा करता है।
- `EXIT`: स्क्रिप्ट या कमांड प्रॉम्प्ट सत्र को समाप्त करता है।

#### 2. **वेरिएबल**
- **वेरिएबल सेट करें**: वेरिएबल बनाने या संशोधित करने के लिए `SET` का उपयोग करें।
  - उदाहरण: `SET MY_VAR=Hello` एक वेरिएबल `MY_VAR` बनाता है।
- **वेरिएबल का उपयोग करें**: `%VARIABLE_NAME%` के साथ संदर्भित करें।
  - उदाहरण: `ECHO %MY_VAR%` "Hello" प्रदर्शित करता है।
- **एनवायरनमेंट वेरिएबल**: अंतर्निहित वेरिएबल जैसे `%PATH%`, `%USERNAME%`, या `%DATE%`।

#### 3. **इनपुट और आउटपुट**
- **उपयोगकर्ता इनपुट**: इनपुट के लिए संकेत देने के लिए `SET /P` का उपयोग करें।
  - उदाहरण: `SET /P NAME=Enter your name: ` उपयोगकर्ता इनपुट को `NAME` में संग्रहीत करता है।
- **आउटपुट रीडायरेक्ट करें**: आउटपुट को फ़ाइल में लिखने के लिए `>` या जोड़ने के लिए `>>` का उपयोग करें।
  - उदाहरण: `DIR > filelist.txt` डायरेक्टरी लिस्टिंग को `filelist.txt` में सहेजता है।

#### 4. **सशर्त कथन**
- शर्तों के आधार पर कमांड निष्पादित करने के लिए `IF` का उपयोग करें।
  - सिंटैक्स: `IF condition command [ELSE command]`
  - उदाहरण: `IF "%NAME%"=="Admin" ECHO Welcome, Admin! ELSE ECHO Access denied.`

#### 5. **लूप**
- **FOR लूप**: फ़ाइलों, डायरेक्टरियों, या मानों पर पुनरावृति करता है।
  - उदाहरण: `FOR %i IN (*.txt) DO ECHO %i` सभी `.txt` फ़ाइलों को सूचीबद्ध करता है।
  - नोट: बैच फ़ाइलों में, वेरिएबल के लिए `%i` के बजाय `%%i` का उपयोग करें।
- **WHILE-जैसे लूप**: `GOTO` और `IF` के साथ अनुकरण करें।

#### 6. **सबरूटीन और लेबल**
- **लेबल**: कोड के एक खंड को चिह्नित करने के लिए `:label` का उपयोग करें।
- **GOTO**: एक लेबल वाले खंड पर जाता है।
  - उदाहरण: `GOTO :EOF` फ़ाइल के अंत पर जाता है।
- **CALL**: किसी अन्य बैच फ़ाइल या सबरूटीन को कॉल करता है।
  - उदाहरण: `CALL :mySubroutine` एक लेबल सबरूटीन चलाता है।

#### 7. **त्रुटि प्रबंधन**
- `%ERRORLEVEL%` के साथ कमांड सफलता की जाँच करें।
  - उदाहरण: `IF %ERRORLEVEL% NEQ 0 ECHO Command failed.`

### सर्वोत्तम अभ्यास
- **`ECHO OFF` का उपयोग करें**: कमांड आउटपुट छिपाकर अव्यवस्था कम करें।
- **टिप्पणियाँ जोड़ें**: कोड को दस्तावेज़ करने के लिए `REM` या `::` का उपयोग करें।
- **वृद्धिशील परीक्षण करें**: डीबग करने के लिए छोटे खंड चलाएं।
- **त्रुटियों को संभालें**: विफलताओं के लिए `%ERRORLEVEL%` की जाँच करें।
- **पाथ के लिए उद्धरण चिह्नों का उपयोग करें**: रिक्त स्थान को संभालने के लिए फ़ाइल पाथ को उद्धरण चिह्नों में संलग्न करें (उदाहरण के लिए, `"C:\Program Files\"`)।
- **आरक्षित नामों से बचें**: फ़ाइलों या वेरिएबल के लिए `CON`, `NUL`, या `PRN` जैसे नामों का उपयोग न करें।
- **साइलेंस के लिए `@` का उपयोग करें**: व्यक्तिगत कमांड इको को दबाने के लिए कमांड के सामने `@` उपसर्ग लगाएं (उदाहरण के लिए, `@ECHO OFF`)।

### उदाहरण बैच फ़ाइल
नीचे एक नमूना बैच फ़ाइल है जो सामान्य विशेषताओं को प्रदर्शित करती है: उपयोगकर्ता इनपुट के लिए संकेत देना, एक डायरेक्टरी बनाना, और आउटपुट लॉग करना।

```
@echo off
REM नमूना बैच फ़ाइल जो एक डायरेक्टरी बनाती है और कार्यों को लॉग करती है
ECHO Starting script...

:: डायरेक्टरी नाम के लिए संकेत दें
SET /P DIRNAME=Enter directory name:

:: जाँचें कि क्या इनपुट खाली है
IF "%DIRNAME%"=="" (
    ECHO Error: No directory name provided.
    PAUSE
    EXIT /B 1
)

:: डायरेक्टरी बनाएं और परिणाम लॉग करें
MKDIR "%DIRNAME%"
IF %ERRORLEVEL%==0 (
    ECHO Directory "%DIRNAME%" created successfully.
    ECHO %DATE% %TIME%: Created directory "%DIRNAME%" >> log.txt
) ELSE (
    ECHO Failed to create directory "%DIRNAME%".
    ECHO %DATE% %TIME%: Failed to create directory "%DIRNAME%" >> log.txt
)

PAUSE
ECHO Done.
EXIT /B
```

- डबल-क्लिक करें या कमांड प्रॉम्प्ट (`cmd.exe /c sample.bat`) के माध्यम से चलाएं।

### उन्नत सुझाव
- **विलंबित विस्तार**: लूप में अपडेट किए गए वेरिएबल के लिए `SETLOCAL EnableDelayedExpansion` का उपयोग करें (उदाहरण के लिए, `%VAR%` के बजाय `!VAR!`)।
- **एरर लेवल**: विशिष्ट एग्जिट कोड के लिए `IF ERRORLEVEL n` के साथ कमांड का परीक्षण करें।
- **बाहरी उपकरण**: उन्नत कार्यक्षमता के लिए `PowerShell` या `curl` जैसे उपकरणों को एकीकृत करें।
- **सुरक्षा**: दुर्भावनापूर्ण कोड को रोकने के लिए अविश्वसनीय स्रोतों से बैच फ़ाइलें चलाने से बचें।
