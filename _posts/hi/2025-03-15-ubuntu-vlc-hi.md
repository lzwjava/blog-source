अपने Ubuntu सिस्टम पर VLC मीडिया प्लेयर इंस्टॉल करने के लिए, आप या तो टर्मिनल (कमांड लाइन) या Ubuntu सॉफ़्टवेयर सेंटर (ग्राफ़िकल इंटरफ़ेस) का उपयोग कर सकते हैं। नीचे दोनों तरीकों के लिए विस्तृत निर्देश दिए गए हैं।

---

### **विधि 1: टर्मिनल का उपयोग करके VLC इंस्टॉल करें**

यह विधि त्वरित और कुशल है, खासकर यदि आप कमांड लाइन से परिचित हैं। इन चरणों का पालन करें:

1.  **टर्मिनल खोलें**
    - Ubuntu एप्लिकेशन मेन्यू में "Terminal" खोजें, या इसे खोलने के लिए `Ctrl + Alt + T` दबाएं।

2.  **पैकेज सूची अपडेट करें**
    - सॉफ़्टवेयर इंस्टॉल करने से पहले, यह सुनिश्चित करना एक अच्छा अभ्यास है कि आपकी पैकेज सूची अप-टू-डेट है। यह कमांड चलाएं:
      ```bash
      sudo apt update
      ```
    - संकेत मिलने पर अपना पासवर्ड दर्ज करें। यह कमांड उपलब्ध सॉफ़्टवेयर पैकेजों की सूची को ताज़ा करती है।

3.  **VLC इंस्टॉल करें**
    - एक बार पैकेज सूची अपडेट हो जाने के बाद, VLC इंस्टॉल करने के लिए यह कमांड चलाएं:
      ```bash
      sudo apt install vlc
      ```
    - आपको फिर से अपना पासवर्ड दर्ज करना होगा। सिस्टम `y` टाइप करके एंटर दबाकर इंस्टॉलेशन की पुष्टि करने के लिए कह सकता है। यह VLC और उसके आवश्यक डिपेंडेंसीज़ को डाउनलोड और इंस्टॉल कर देगा।

4.  **इंस्टॉलेशन सत्यापित करें**
    - इंस्टॉलेशन के बाद, जांचें कि VLC इंस्टॉल हुआ है या नहीं, यह टाइप करके:
      ```bash
      vlc
      ```
    - इससे VLC मीडिया प्लेयर लॉन्च होना चाहिए। यदि यह खुलता है, तो इंस्टॉलेशन सफल रहा।
    - वैकल्पिक रूप से, आप VLC का वर्जन इस कमांड को चलाकर जांच सकते हैं:
      ```bash
      vlc --version
      ```
    - यह कुछ इस तरह दिखाएगा "VLC media player 3.0.11.1 Vetinari" (वर्जन नंबर भिन्न हो सकता है)।

5.  **वैकल्पिक: VLC का परीक्षण करें**
    - यह सुनिश्चित करने के लिए कि VLC ठीक से काम करता है, एक मीडिया फ़ाइल चलाने का प्रयास करें। उदाहरण के लिए, यदि आपके डेस्कटॉप पर एक MP4 फ़ाइल है, तो उस पर राइट-क्लिक करें और "Open with VLC media player" चुनें। यदि यह चलती है, तो VLC पूरी तरह कार्यात्मक है।

---

### **विधि 2: Ubuntu सॉफ़्टवेयर सेंटर का उपयोग करके VLC इंस्टॉल करें**

यदि आप टर्मिनल के बजाय ग्राफ़िकल इंटरफ़ेस पसंद करते हैं, तो Ubuntu सॉफ़्टवेयर सेंटर का उपयोग करें:

1.  **Ubuntu सॉफ़्टवेयर सेंटर खोलें**
    - एप्लिकेशन मेन्यू में "Ubuntu Software" खोजें और इसे खोलने के लिए क्लिक करें।

2.  **VLC खोजें**
    - सॉफ़्टवेयर सेंटर में, सर्च आइकन (आमतौर पर एक आवर्धक कांच) पर क्लिक करें और सर्च बार में "VLC" टाइप करें।

3.  **VLC इंस्टॉल करें**
    - सर्च रिजल्ट में "VLC media player" ढूंढें, उस पर क्लिक करें, और फिर "Install" बटन पर क्लिक करें। इंस्टॉलेशन को अधिकृत करने के लिए आपको अपना पासवर्ड दर्ज करने की आवश्यकता हो सकती है।

4.  **VLC लॉन्च करें**
    - एक बार इंस्टॉल हो जाने के बाद, आप VLC को सीधे सॉफ़्टवेयर सेंटर से "Open" पर क्लिक करके (यदि उपलब्ध हो) या एप्लिकेशन मेन्यू में "VLC" खोजकर लॉन्च कर सकते हैं।

---

### **अतिरिक्त नोट्स**

-   **VLC को अपडेट रखना**
    - यदि आपने टर्मिनल का उपयोग करके VLC इंस्टॉल किया है, तो आप समय-समय पर यह कमांड चलाकर इसे अपडेट रख सकते हैं:
      ```bash
      sudo apt update
      sudo apt upgrade
      ```
    - यह VLC को अन्य सिस्टम पैकेजों के साथ अपडेट कर देता है। यदि आपने सॉफ़्टवेयर सेंटर का उपयोग किया है, तो अपडेट आमतौर पर उसी इंटरफ़ेस के माध्यम से प्रबंधित किए जाते हैं।

-   **प्लगइन्स और कोडेक्स**
    - `sudo apt install vlc` के माध्यम से डिफ़ॉल्ट VLC इंस्टॉलेशन में आवश्यक प्लगइन्स (जैसे `vlc-plugin-base`) और कोडेक्स शामिल होते हैं, इसलिए इसे अधिकांश सामान्य मीडिया फॉर्मेट (जैसे, MP4, AVI) बिना किसी अतिरिक्त सेटअप के चलाने चाहिए। यदि आपको विशिष्ट फ़ाइल प्रकारों के साथ समस्याएँ आती हैं, तो आप बाद में अतिरिक्त कोडेक्स इंस्टॉल कर सकते हैं, जैसे `ubuntu-restricted-extras`, इस कमांड का उपयोग करके:
      ```bash
      sudo apt install ubuntu-restricted-extras
      ```

-   **Snap के माध्यम से वैकल्पिक इंस्टॉलेशन**
    - VLC एक Snap पैकेज के रूप में भी उपलब्ध है। यदि आप इस विधि को पसंद करते हैं, तो यह कमांड चलाएं:
      ```bash
      sudo snap install vlc
      ```
    - हालाँकि, अधिकांश उपयोगकर्ताओं के लिए `apt` विधि की सिफारिश की जाती है क्योंकि यह सिस्टम के साथ अच्छी तरह एकीकृत होती है।

किसी भी विधि का पालन करके, आपके Ubuntu सिस्टम पर VLC इंस्टॉल और उपयोग के लिए तैयार हो जाएगा। टर्मिनल विधि कमांड-लाइन उपयोगकर्ताओं के लिए तेज़ है, जबकि सॉफ़्टवेयर सेंटर उन लोगों के लिए आदर्श है जो ग्राफ़िकल तरीका पसंद करते हैं। अपने मीडिया प्लेबैक का आनंद लें!

Ubuntu पर VLC में एक MKV फ़ाइल चलाते समय आपके सामने आने वाली त्रुटि को ठीक करने के लिए—विशेष रूप से MPEG-4 AAC और H.265 डिकोडर के गायब होने के संदेश के लिए—नीचे दिए गए चरणों का पालन करें। यह समस्या आमतौर पर तब होती है क्योंकि Ubuntu लाइसेंसिंग प्रतिबंधों के कारण डिफ़ॉल्ट रूप से कुछ मालिकाना कोडेक्स शामिल नहीं करता है। अतिरिक्त कोडेक पैकेज इंस्टॉल करने से समस्या का समाधान होना चाहिए।

---

### **समाधान**

#### **चरण 1: `ubuntu-restricted-extras` पैकेज इंस्टॉल करें**
यह पैकेज विभिन्न प्रकार के कोडेक्स के लिए सपोर्ट प्रदान करता है, जिनमें MPEG-4 AAC (ऑडियो) और H.265 (HEVC) वीडियो शामिल हैं, जो संभवतः आपके VLC इंस्टॉलेशन से गायब हैं।

1.  **टर्मिनल खोलें**
    - Ubuntu एप्लिकेशन मेन्यू में "Terminal" खोजें या `Ctrl + Alt + T` दबाएं।

2.  **पैकेज सूची अपडेट करें**
    - यह सुनिश्चित करने के लिए कि आपकी सिस्टम की पैकेज सूची अप-टू-डेट है, यह कमांड चलाएं:
      ```bash
      sudo apt update
      ```

3.  **`ubuntu-restricted-extras` इंस्टॉल करें**
    - निम्नलिखित कमांड चलाएं:
      ```bash
      sudo apt install ubuntu-restricted-extras
      ```
    - आपको अपना पासवर्ड दर्ज करने की आवश्यकता हो सकती है। इंस्टॉलेशन के दौरान, कुछ घटकों के लिए End User License Agreement (EULA) स्वीकार करने के लिए प्रॉम्प्ट किया जा सकता है—सहमत होने और आगे बढ़ने के लिए ऑन-स्क्रीन निर्देशों का पालन करें।

4.  **VLC रीस्टार्ट करें**
    - VLC को बंद करें यदि यह खुला है, फिर इसे फिर से खोलें और MKV फ़ाइल को फिर से चलाने का प्रयास करें।

---

#### **चरण 2: अतिरिक्त कोडेक पैकेज इंस्टॉल करें (यदि आवश्यक हो)**
यदि चरण 1 के बाद भी त्रुटि बनी रहती है, तो विशिष्ट पैकेज इंस्टॉल करें जो H.265 और अन्य कोडेक्स के लिए अतिरिक्त सपोर्ट प्रदान करते हैं।

1.  **`libde265-0` और `libavcodec-extra` इंस्टॉल करें**
    - H.265 डिकोडिंग और अतिरिक्त कोडेक सपोर्ट के लिए लाइब्रेरीज़ इंस्टॉल करने के लिए यह कमांड चलाएं:
      ```bash
      sudo apt install libde265-0 libavcodec-extra
      ```

2.  **VLC रीस्टार्ट करें**
    - VLC को बंद करें और फिर से खोलें, फिर MKV फ़ाइल को फिर से चलाने का प्रयास करें।

---

#### **चरण 3: अतिरिक्त समस्या निवारण (यदि आवश्यक हो)**
यदि समस्या अभी भी हल नहीं होती है, तो इन अतिरिक्त चरणों को आज़माएं:

-   **VLC वर्जन जांचें**
    - सुनिश्चित करें कि आप नवीनतम VLC वर्जन का उपयोग कर रहे हैं। अपना वर्तमान वर्जन इससे जांचें:
      ```bash
      vlc --version
      ```
    - यदि यह पुराना है, तो इसे अपडेट करें:
      ```bash
      sudo apt update
      sudo apt upgrade vlc
      ```

-   **दूसरी MKV फ़ाइल का परीक्षण करें**
    - यह देखने के लिए कि क्या समस्या एक फ़ाइल तक सीमित है, एक अलग MKV फ़ाइल चलाएं। यदि अन्य फ़ाइलें काम करती हैं, तो समस्याग्रस्त फ़ाइल खराब हो सकती है या असमर्थित एन्कोडिंग का उपयोग कर सकती है।

-   **अधिक विवरण के लिए VLC लॉग देखें**
    - उन्नत समस्या निवारण के लिए, वर्बोज़ आउटपुट के साथ टर्मिनल से VLC चलाएं:
      ```bash
      vlc --verbose 2
      ```
    - फ़ाइल को चलाने का प्रयास करें और गायब कोडेक्स के बारे में विस्तृत त्रुटि संदेशों की तलाश करें।

---

### **अतिरिक्त नोट्स**
-   **यह समस्या क्यों होती है**: MKV एक कंटेनर फॉर्मेट है जो ऑडियो के लिए MPEG-4 AAC और वीडियो के लिए H.265 जैसे विभिन्न कोडेक्स का उपयोग कर सकता है। Ubuntu का डिफ़ॉल्ट VLC इंस्टॉलेशन मालिकाना सॉफ़्टवेयर पर कानूनी प्रतिबंधों के कारण इनको शामिल नहीं कर सकता है।
-   **हार्डवेयर एक्सेलेरेशन**: यदि आपका सिस्टम H.265 हार्डवेयर डिकोडिंग का समर्थन करता है (जैसे, आधुनिक GPU के माध्यम से), सुनिश्चित करें कि आपके ग्राफ़िक्स ड्राइवर अपडेटेड हैं। हालाँकि, यह त्रुटि एक गायब सॉफ़्टवेयर डिकोडर का सुझाव देती है, इसलिए हार्डवेयर एक्सेलेरेशन संभवतः मुख्य मुद्दा नहीं है।
-   **अगले कदम**: यदि उपरोक्त में से कुछ भी काम नहीं करता है, तो VLC को पुनः इंस्टॉल करने पर विचार करें या अपने सिस्टम और विशिष्ट MKV फ़ाइल के विवरण के साथ Ubuntu या VLC फ़ोरम पर सहायता लें।

---

इन चरणों का पालन करके, आप गायब डिकोडर त्रुटियों को ठीक करने और अपनी MKV फ़ाइलों को VLC में सफलतापूर्वक चलाने में सक्षम होने चाहिए। चरण 1 से शुरू करें, क्योंकि यह सबसे सीधा और संभावित समाधान है।
