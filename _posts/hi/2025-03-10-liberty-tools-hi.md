नीचे एक सिंहावलोकन दिया गया है कि आप Visual Studio Code में WebSphere Liberty सर्वर के साथ काम करने के लिए Liberty Tools एक्सटेंशन का उपयोग कैसे कर सकते हैं:

---

### 1. Liberty Tools एक्सटेंशन इंस्टॉल करें
- **ढूंढें और इंस्टॉल करें:**  
  VS Code का एक्सटेंशन व्यू खोलें, **"IBM WebSphere Liberty Tools"** खोजें और इसे इंस्टॉल करें।
- **एक्सटेंशन के लाभ:**  
  यह एक्सटेंशन सर्वर प्रबंधन, एप्लिकेशन डिप्लॉयमेंट और डीबगिंग को VS Code में एकीकृत करता है।

---

### 2. अपने WebSphere Liberty सर्वर को तैयार करें
- **Liberty रनटाइम इंस्टॉल करें या उसकी ओर इशारा करें:**  
  यदि आपके पास पहले से कोई Liberty सर्वर इंस्टॉल नहीं है, तो IBM से एक डाउनलोड करें और इंस्टॉल करें। यदि यह पहले से इंस्टॉल है, तो इसकी इंस्टॉलेशन डायरेक्टरी नोट कर लें।
- **संगतता सुनिश्चित करें:**  
  सत्यापित करें कि आपका Liberty रनटाइम संस्करण एक्सटेंशन के साथ संगत है।

---

### 3. VS Code में अपने Liberty सर्वर को कॉन्फ़िगर करें
- **नया सर्वर इंस्टेंस बनाना:**  
  कमांड पैलेट (`Ctrl+Shift+P` या `Cmd+Shift+P`) खोलें और कमांड चलाएँ:  
  `Liberty: Create Server`  
  प्रॉम्प्ट का पालन करते हुए:
  - रनटाइम इंस्टॉलेशन फ़ोल्डर चुनें।
  - सर्वर कॉन्फ़िगरेशन फ़ाइल (आमतौर पर `server.xml`) निर्दिष्ट करें।
- **मौजूदा प्रोजेक्ट:**  
  यदि आपके पास पहले से ही एक Liberty-आधारित एप्लिकेशन है, तो वर्कस्पेस खोलें ताकि एक्सटेंशन आपकी सर्वर सेटिंग्स का पता लगा सके और उन्हें प्रबंधित करने में मदद कर सके।

---

### 4. अपना एप्लिकेशन जोड़ना
- **ऐप डिप्लॉय करना:**  
  आप अपना एप्लिकेशन सर्वर में दो तरीकों से जोड़ सकते हैं:
  - अपने एप्लिकेशन के कॉन्टेक्स्ट और डिप्लॉयमेंट विवरण शामिल करने के लिए `server.xml` को संपादित करके, या
  - एक्सटेंशन के UI विकल्पों (जो अक्सर Liberty व्यू में उपलब्ध होते हैं) का उपयोग करके "Add Application" या "Deploy Application" चुनकर।
- **बिल्ड एकीकरण:**  
  यदि आप Maven या Gradle का उपयोग कर रहे हैं, तो एक्सटेंशन ऐसे टास्क भी प्रदान कर सकता है जो डिप्लॉयमेंट से पहले आपके एप्लिकेशन को बिल्ड करते हैं।

---

### 5. सर्वर को स्टार्ट, स्टॉप और डीबग करना
- **सर्वर स्टार्ट करना:**  
  Liberty व्यू (जो अक्सर VS Code में एक समर्पित पैनल या ट्री व्यू के रूप में उपलब्ध होता है) में, अपने कॉन्फ़िगर किए गए सर्वर पर राइट-क्लिक करें और **"Start Server"** चुनें।  
  आउटपुट पेन स्टार्टअप लॉग और किसी भी कॉन्फ़िगरेशन संदेश को प्रदर्शित करेगा।
- **सर्वर स्टॉप करना:**  
  इसी तरह, राइट-क्लिक करें और अपने इंस्टेंस को बंद करने के लिए **"Stop Server"** चुनें।
- **डीबगिंग:**  
  यदि आपको अपने एप्लिकेशन को डीबग करने की आवश्यकता है:
  - अपने कोड में ब्रेकपॉइंट सेट करें।
  - अपने चल रहे सर्वर से VS Code डीबगर को जोड़ने के लिए **"Liberty: Debug Server"** चलाने के लिए कमांड पैलेट का उपयोग करें (या Liberty व्यू में प्रदान किए गए डीबग विकल्पों का उपयोग करें)।
  - आपके एप्लिकेशन द्वारा अनुरोधों को प्रोसेस करने पर अब आपके ब्रेकपॉइंट हिट होने चाहिए।

---

### 6. अतिरिक्त सुविधाएँ और सुझाव
- **लाइव लॉग मॉनिटरिंग:**  
  एक्सटेंशन Liberty सर्वर लॉग को वास्तविक समय में देखने का एक तरीका प्रदान करता है, जिससे समस्याओं का निदान करना आसान हो जाता है।
- **सर्वर कॉन्फ़िगरेशन संपादित करना:**  
  VS Code के भीतर सीधे अपनी `server.xml` को संपादित करते समय IntelliSense और सुझावों से लाभ उठाएं।
- **टास्क एकीकरण:**  
  यदि आप अपने डेवलपमेंट वर्कफ़्लो के हिस्से के रूप में स्वचालित बिल्ड या डिप्लॉयमेंट चला रहे हैं, तो VS Code के टास्क का उपयोग करें।
- **कंटेनर और रिमोट सपोर्ट:**  
  आपके सेटअप के आधार पर, आप Liberty को कंटेनर या रिमोट वातावरण के भीतर चलाने के लिए भी कॉन्फ़िगर कर सकते हैं, जिसमें एक्सटेंशन में अतिरिक्त कमांड उपलब्ध हो सकते हैं।

---

### अंतिम नोट्स
Liberty Tools एक्सटेंशन का उपयोग करने से VS Code के भीतर सीधे आपके WebSphere Liberty एप्लिकेशन को डेवलप, डिप्लॉय और डीबग करने की प्रक्रिया सुगम हो जाती है। यह एकीकरण कॉन्टेक्स्ट स्विचिंग को कम करने और आपके डेवलपमेंट साइकिल को तेज करने में मदद करता है।

किसी भी उन्नत कॉन्फ़िगरेशन या समस्या निवारण चरणों के लिए Liberty Tools एक्सटेंशन पर आधिकारिक IBM दस्तावेज़ीकरण देखने के लिए स्वतंत्र महसूस करें।
