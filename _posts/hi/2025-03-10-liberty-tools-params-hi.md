आप VS Code में अपनी Liberty dev mode लॉन्च कॉन्फ़िगरेशन में कस्टम JVM पैरामीटर्स—जैसे `-Dversion=5.1`—जोड़कर पास कर सकते हैं। आमतौर पर, आप ऐसा अपने प्रोजेक्ट में `.vscode/launch.json` फ़ाइल को एडिट करके (या बनाकर) और पैरामीटर को `"vmArgs"` (या `"args"` अगर आपकी कॉन्फ़िगरेशन Maven गोल आर्ग्युमेंट्स की अपेक्षा करती है) जैसी किसी प्रॉपर्टी के अंतर्गत शामिल करके करते हैं।

उदाहरण के लिए, अगर आप dev mode के साथ Liberty Tools एक्सटेंशन का उपयोग कर रहे हैं, तो आपकी कॉन्फ़िगरेशन कुछ इस तरह दिख सकती है:

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "liberty",
      "request": "launch",
      "name": "Liberty Dev Mode",
      "goal": "dev",
      "vmArgs": "-Dversion=5.1"
    }
  ]
}
```

### चरण:
1.  **लॉन्च कॉन्फ़िगरेशन खोलें या बनाएँ:**
    *   VS Code में, अपना प्रोजेक्ट खोलें।
    *   `.vscode` फ़ोल्डर में नेविगेट करें और `launch.json` खोलें (या बनाएँ)।

2.  **अपनी Dev Mode कॉन्फ़िगरेशन जोड़ें:**
    *   ऊपर दिखाए गए अनुसार एक कॉन्फ़िगरेशन एंट्री जोड़ें। `"vmArgs"` कुंजी आपको JVM विकल्प निर्दिष्ट करने देती है जो dev mode शुरू होने पर सर्वर को पास किए जाएंगे।

3.  **सेव करें और लॉन्च करें:**
    *   फ़ाइल को सेव करें।
    *   "Liberty Dev Mode" कॉन्फ़िगरेशन शुरू करने के लिए Run और Debug व्यू का उपयोग करें।

अगर आप सीधे Maven का उपयोग कर रहे हैं, तो आप कमांड लाइन से भी dev mode शुरू कर सकते हैं:

```bash
mvn liberty:dev -Dversion=5.1
```

यह सर्वर शुरू होने पर कस्टम प्रॉपर्टी को JVM को पास करता है।

इस तरह कॉन्फ़िगर करने से, Liberty Tools एक्सटेंशन आपके कस्टम पैरामीटर (`-Dversion=5.1`) को हर बार जब आप dev mode शुरू करेंगे, शामिल करेगा।
