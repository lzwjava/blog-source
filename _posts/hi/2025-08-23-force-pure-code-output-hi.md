ठीक है—आप चाहते हैं कि Claude Code (और आपके वे "राउटर" वर्कफ़्लो जो इसे कॉल करते हैं) **केवल कोड** आउटपुट करें, **बिना इनलाइन कमेंट/डॉकस्ट्रिंग/स्पष्टीकरण के**। यहां तीन विश्वसनीय तरीके दिए गए हैं, जो नरम से सख्त तक हैं। कोई टेबल नहीं—सीधे कदम।

---

# 1) नियम को `CLAUDE.md` में डालें (ग्लोबल या प्रति-प्रोजेक्ट)

Claude Code स्वचालित रूप से "मेमोरी" फ़ाइलें लोड करता है और उन्हें सत्रों में फॉलो करता है। एक छोटा, स्पष्ट नियम जोड़ें:

**ग्लोबल (सभी प्रोजेक्ट):** `~/.claude/CLAUDE.md`
**प्रति प्रोजेक्ट (चेक इन):** `./CLAUDE.md`

```md
# आउटपुट नियम
- कोड जनरेट करते समय, केवल कोड आउटपुट करें।
- इनलाइन कमेंट, ब्लॉक कमेंट, डॉकस्ट्रिंग, या व्याख्यात्मक शब्द शामिल न करें, जब तक कि मैं स्पष्ट रूप से न कहूं।
- संदेश संक्षिप्त रखें; यदि स्पष्टीकरण की आवश्यकता हो, तो उसे कोड ब्लॉक के बाद रखें और केवल तभी जब मैं कहूं।
```

टिप: आप इन मेमोरी को `/memory` के साथ मैनेज कर सकते हैं और Claude Code में `#` से एक संदेश शुरू करके नई मेमोरी जल्दी से जोड़ सकते हैं। ([Anthropic][1])

# 2) एक **कस्टम आउटपुट स्टाइल** बनाएं जो कमेंट पर प्रतिबंध लगाती है (मजबूत)

आउटपुट स्टाइल Claude Code के **सिस्टम प्रॉम्प्ट** को संशोधित करती हैं। एक बनाएं और उस पर स्विच करें:

**बनाएं:**

```
/output-style:new मुझे एक आउटपुट स्टाइल चाहिए जो हमेशा बिना किसी कमेंट या डॉकस्ट्रिंग के कोड आउटपुट करे, कोई स्पष्टीकरण नहीं।
```

यह `~/.claude/output-styles/…` के तहत एक फ़ाइल बनाता है। आप इसे एडिट कर सकते हैं ताकि नियम स्पष्ट हो:

```md
---
name: केवल कोड (कोई कमेंट नहीं)
description: हमेशा बिना कमेंट/डॉकस्ट्रिंग के कोड दें; जब तक न कहा जाए, कोई स्पष्टीकरण नहीं।
---
# निर्देश
- आप "केवल-कोड" मोड में कोडिंग कर रहे हैं।
- कभी भी इनलाइन कमेंट, ब्लॉक कमेंट, या डॉकस्ट्रिंग शामिल न करें।
- जब तक स्पष्ट रूप से न कहा जाए, कोड से पहले या बाद में शब्द न जोड़ें।

## विशिष्ट व्यवहार
- यदि कोई कार्य टेस्ट का संकेत देता है, तो टेस्ट फ़ाइल(ओं) को केवल कोड के रूप में आउटपुट करें।
- यदि स्पष्टीकरण की आवश्यकता है, तो संक्षेप में पूछें, कोई स्पष्टीकरण नहीं।
```

**सक्रिय करें:**

```
/output-style "केवल कोड (कोई कमेंट नहीं)"
```

(आउटपुट स्टाइल सीधे सिस्टम प्रॉम्प्ट बदलती हैं; आप कभी भी `/output-style` के साथ स्विच कर सकते हैं।) ([Anthropic][2])

# 3) रनटाइम में एक अतिरिक्त सिस्टम प्रॉम्प्ट जोड़ें (सबसे मजबूत / प्रति-सत्र)

यदि आप Claude Code को CLI या स्क्रिप्ट से लॉन्च करते हैं, तो जोड़ें:

```bash
claude --append-system-prompt "हमेशा केवल कोड आउटपुट करें। कोई इनलाइन कमेंट नहीं, कोई ब्लॉक कमेंट नहीं, कोई डॉकस्ट्रिंग नहीं, जब तक स्पष्ट रूप से न कहा जाए कोई स्पष्टीकरण नहीं।"
```

आप इसे `/config` के माध्यम से भी सेट कर सकते हैं यदि आप लगातार प्रोजेक्ट सेटिंग्स पसंद करते हैं। (Anthropic नोट करता है कि आप Claude Code के व्यवहार को संशोधित करने के लिए `--append-system-prompt` का उपयोग कर सकते हैं।) ([Anthropic][3])

## "राउटर"-वाइड नियम कहां रखें

* यदि आका "Claude Code राउटर" केवल रेपो में आपके नियमित Claude Code सत्र हैं, तो **ग्लोबल मेमोरी** (`~/.claude/CLAUDE.md`) या एक **कस्टम आउटपुट स्टाइल** को प्राथमिकता दें ताकि नियम हर जगह यात्रा करे। ([Anthropic][1])
* यदि आप स्क्रिप्ट के माध्यम से लॉन्च करते हैं, तो `--append-system-prompt` फ्लैग को बेक कर दें ताकि हर रन नियम को इनहेरिट करे। ([Anthropic][3])

## अतिरिक्त गार्डरेल (वैकल्पिक लेकिन उपयोगी)

* प्रोजेक्ट मेमोरी में, एक **"उल्लंघन"** लाइन जोड़ें ताकि मॉडल स्वयं-जांच करे: "यदि आप कमेंट/डॉकस्ट्रिंग आउटपुट करने वाले हैं, तो रुकें और उन्हें हटा दें।"
* यदि आप कभी-कभी कमेंट *चाहते* हैं, तो अपने प्रॉम्प्ट में एक-शॉट ओवरराइड जोड़ें: "इस बार, संक्षिप्त कमेंट शामिल करें।"

बस इतना ही। यदि आप मुझे बताएं कि आप Claude Code कैसे शुरू करते हैं (डेस्कटॉप, VS Code, JetBrains, या टर्मिनल), तो मैं आपको उस सेटअप के लिए एक ड्रॉप-इन स्निपेट दे सकता हूं।
