`git gc` का मतलब है **Git garbage collection**। यह एक हाउसकीपिंग कमांड है जो आपके लोकल रिपॉजिटरी को साफ़ और ऑप्टिमाइज़ करती है।

### यह क्या करता है

* **फ़ाइल इतिहास को कंप्रेस करता है**: Git ऑब्जेक्ट्स (commits, trees, blobs) को एक डेटाबेस में स्टोर करता है। समय के साथ, बहुत सारे loose ऑब्जेक्ट्स जमा हो जाते हैं। `git gc` उन्हें एक कॉम्पैक्ट बाइनरी फ़ाइल में पैक कर देता है, जिससे डिस्क स्पेस बचती है।
* **अनरीचेबल ऑब्जेक्ट्स को हटाता है**: अगर आपने ब्रांचेज़ डिलीट की हैं या रीबेस किया है, तो कुछ कमिट्स अब किसी भी रेफरेंस से रीचेबल नहीं रह जाते। Git सुरक्षा के लिए उन्हें रखता है, लेकिन `git gc` आखिरकार उन्हें साफ़ कर देता है।
* **रेफरेंसेज़ को ऑप्टिमाइज़ करता है**: यह टैग्स और ब्रांचेज़ जैसे रेफ्स को रीराइट और पैक करता है ताकि लुकअप तेज़ हो सके।
* **रेफ्लॉग को प्रून करता है**: एक्सपायरी पीरियड से पुराने रेफ्लॉग एंट्रीज़ (HEAD बदलावों के रिकॉर्ड) को हटा दिया जाता है।

### यह कभी-कभी अपने आप क्यों चल जाता है

* जब loose ऑब्जेक्ट्स या पैक फ़ाइलों की संख्या एक थ्रेशोल्ड को पार कर जाती है, तो Git **ऑटोमैटिक बैकग्राउंड `gc`** चलाता है। उदाहरण के लिए, अगर आपने बहुत सारी कमिट्स, रीबेस, या मर्ज किए हैं, तो Git इसे ट्रिगर कर सकता है।
* बहुत सारे Git कमांड्स (जैसे `git fetch`, `git merge`, या `git rebase`) चेक करते हैं कि क्या `gc` चलाना उचित है और जरूरत पड़ने पर इसे अपने आप चला देते हैं।
* इससे आपके रेपो की परफॉर्मेंस बनी रहती है और आपको इसके बारे में सोचने की जरूरत नहीं पड़ती।

### इसे मैन्युअली कब चलाएं

* अगर आपका रिपॉजिटरी **स्लो** या **ब्लोटेड** महसूस हो रहा हो (जैसे, बड़ा `.git` फोल्डर)।
* **बड़ी सफाई** करने के बाद (जैसे ब्रांचेज़ डिलीट करना या इतिहास से बड़ी फ़ाइलें हटाना)।
* अगर आप तुरंत सफाई फोर्स करना चाहते हैं:

  ```bash
  git gc --prune=now --aggressive
  ```

  *(सावधान रहें: `--aggressive` बड़े रेपोज़ पर, खासकर, बहुत समय ले सकता है)*
