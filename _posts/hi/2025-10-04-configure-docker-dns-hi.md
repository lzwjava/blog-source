Docker डेमन के लिए DNS सर्वर कॉन्फ़िगर करने के लिए (जो कंटेनर स्तर पर ओवरराइड न किए जाने पर कंटेनरों द्वारा उपयोग किए जाएंगे), आप `/etc/docker/daemon.json` फ़ाइल संपादित कर सकते हैं। यह फ़ाइल आपको Docker के रनटाइम व्यवहार को अनुकूलित करने की अनुमति देती है।

### DNS कॉन्फ़िगर करने के चरण

1. **daemon.json फ़ाइल संपादित करें**:
   - यदि फ़ाइल मौजूद नहीं है, तो इसे root विशेषाधिकारों के साथ बनाएं (उदाहरण के लिए, `sudo nano /etc/docker/daemon.json` या अपने पसंदीदा एडिटर का उपयोग करके)।
   - DNS सर्वरों के IP पतों की एक सरणी के रूप में `dns` कुंजी जोड़ें या अपडेट करें। उदाहरण के लिए, Google के public DNS (8.8.8.8 और 8.8.4.4) का उपयोग करने के लिए:

     ```json
     {
       "dns": ["8.8.8.8", "8.8.4.4"]
     }
     ```

     - यदि आवश्यक हो तो आप अधिक सर्वर जोड़ सकते हैं (उदाहरण के लिए, Cloudflare का 1.1.1.1)।
     - यह सेटिंग डिफ़ॉल्ट नेटवर्क के साथ बनाए गए सभी कंटेनरों पर लागू होती है, जब तक कि अन्यथा निर्दिष्ट न किया गया हो।

2. **JSON वैलिडेट करें**:
   - सुनिश्चित करें कि फ़ाइल वैध JSON है। जाँचने के लिए आप `jq` जैसे टूल का उपयोग कर सकते हैं: `jq . /etc/docker/daemon.json`। यदि यह त्रुटि देता है, तो सिंटैक्स ठीक करें।

3. **Docker सेवा रीस्टार्ट करें**:
   - परिवर्तन लागू करने के लिए Docker को रीस्टार्ट करें:
     ```
     sudo systemctl restart docker
     ```
     - गैर-सिस्टमडी सिस्टम पर (उदाहरण के लिए, पुराने Linux डिस्ट्रो), `sudo service docker restart` का उपयोग करें।
   - स्थिति सत्यापित करें: `sudo systemctl status docker`।

4. **कॉन्फ़िगरेशन टेस्ट करें**:
   - एक टेस्ट कंटेनर चलाएं और उसके DNS रिज़ॉल्यूशन की जाँच करें:
     ```
     docker run --rm busybox nslookup google.com
     ```
     - इसे आपके कॉन्फ़िगर किए गए DNS सर्वरों का उपयोग करके रिज़ॉल्व करना चाहिए।

### नोट्स
- **स्कोप**: यह होस्ट पर Docker डेमन के लिए DNS कॉन्फ़िगर करता है, जो कंटेनरों के लिए आउटबाउंड नेम रिज़ॉल्यूशन को प्रभावित करता है। यह होस्ट के सिस्टम DNS को नहीं बदलता है।
- **ओवरराइड**: यदि किसी कंटेनर को अलग DNS की आवश्यकता होती है (उदाहरण के लिए, `docker run` में `--dns` फ्लैग के माध्यम से), तो यह डेमन-स्तरीय सेटिंग को ओवरराइड कर देगा।
- **सुरक्षा**: अधिकांश मामलों के लिए 8.8.8.8 जैसे public DNS का उपयोग करना ठीक है, लेकिन एंटरप्राइज़ वातावरण के लिए आंतरिक DNS पर विचार करें।
- **डॉक्यूमेंटेशन**: अधिक विकल्पों के लिए (उदाहरण के लिए, DNS खोज डोमेन), [daemon.json पर आधिकारिक Docker डॉक्स](https://docs.docker.com/engine/reference/commandline/dockerd/#daemon-configuration-file) देखें।

यदि आपको त्रुटियाँ मिलती हैं (उदाहरण के लिए, अनुमति संबंधी मुद्दे या JSON पार्सिंग), तो आगे समस्या निवारण के लिए विवरण साझा करें।
