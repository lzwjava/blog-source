## स्थापना

मैंने पहले [कंप्यूटर कैसे बनाएँ](./computer-en) के बारे में लिखा था। आइए कुछ अलग कोशिश करते हैं। आज, मैं एक Lenovo Xiaoxing 14IAH8 लैपटॉप पर, जिसमें पहले से Windows 10 स्थापित है, Ubuntu 24.04 स्थापित कर रहा हूँ।

```bash
sudo dd if=ubuntu-24.04.1-desktop-amd64.iso of=/dev/rdisk6 bs=1m
```

सेटिंग्स में जाकर डिवाइस एन्क्रिप्शन को अक्षम करना आवश्यक है, क्योंकि Ubuntu इंस्टॉलर के लिए इसकी आवश्यकता होती है।

यदि डिवाइस एन्क्रिप्शन सक्षम है और BIOS में सुरक्षित बूट अक्षम है, तो Windows रिकवरी के लिए BitLocker पासफ़्रेज़ के लिए संकेत देगा।

इसलिए, पहले BIOS में सुरक्षित बूट को सक्षम करना, Windows में लॉग इन करना और फिर सेटिंग्स में जाकर डिवाइस एन्क्रिप्शन को अक्षम करना आवश्यक है। उसके बाद, Ubuntu इंस्टॉलर शुरू करने के लिए सुरक्षित बूट को फिर से अक्षम किया जा सकता है।

## अटक जाना

जब Ubuntu `gdm.service` (GNOME डिस्प्ले मैनेजर) शुरू करता है और अटक जाता है, तो यह आमतौर पर ग्राफिकल यूजर इंटरफेस (GUI) या डिस्प्ले मैनेजर में ही किसी समस्या का संकेत देता है। समस्या का निवारण करने और संभावित रूप से इसे हल करने के लिए आप कुछ कदम उठा सकते हैं:

1. **एक वर्चुअल टर्मिनल तक पहुँच:**
   - यदि आपका सिस्टम GDM स्क्रीन पर अटका हुआ है, तो आप समस्या का निदान करने के लिए टर्मिनल तक पहुँचने का प्रयास कर सकते हैं।
   - वर्चुअल टर्मिनल पर स्विच करने के लिए `Ctrl + Alt + F2` (या `F3`, `F4`, आदि) दबाएँ।

2. **लॉग इन करें:**
   - अपने उपयोगकर्ता नाम और पासवर्ड से लॉग इन करें।

3. **सिस्टम लॉग जांचें:**
   - समस्या के कारण का संकेत देने वाली किसी भी त्रुटि या चेतावनी के लिए सिस्टम लॉग की जांच करने के लिए निम्न कमांड का उपयोग करें:
     ```bash
     sudo journalctl -xe
     ```

4. **GDM सेवा पुनः आरंभ करें:**
   - यह देखने के लिए कि क्या इससे समस्या हल हो जाती है, GDM सेवा को पुनः आरंभ करने का प्रयास करें:
     ```bash
     sudo systemctl restart gdm
     ```

5. **सिस्टम को रिबूट करें:**
   - परिवर्तन करने के बाद, उन्हें लागू करने के लिए अपने सिस्टम को रिबूट करें:
     ```bash
     sudo reboot
     ```
