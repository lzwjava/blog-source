## मुख्य विशेषताएं
योरकिट जावा प्रोफाइलर जावा एप्लिकेशन का निदान और ऑप्टिमाइज़ेशन करने के लिए विशेषताओं का एक व्यापक सेट प्रदान करता है। प्राथमिक विशेषताएं नीचे दी गई हैं:

### CPU प्रोफाइलिंग
- **कॉल ट्री और हॉट स्पॉट**: कॉल ट्री या हॉट स्पॉट सूचियों का उपयोग करके मेथड एक्ज़िक्यूशन टाइम को विज़ुअलाइज़ करें और CPU-इंटेंसिव मेथड्स की पहचान करें।
- **फ्लेम ग्राफ**: परफॉर्मेंस बॉटलनेक को जल्दी से पिनपॉइंट करने के लिए कॉल स्टैक का एक विज़ुअल प्रतिनिधित्व प्रदान करते हैं।
- **स्मार्ट व्हाट-इफ एनालिसिस**: एप्लिकेशन को दोबारा प्रोफाइल किए बिना संभावित परफॉर्मेंस सुधारों का मूल्यांकन करें।
- **सैंपलिंग और ट्रेसिंग**: परफॉर्मेंस और सटीकता को संतुलित करने के लिए सैंपलिंग (कम ओवरहेड) या ट्रेसिंग (विस्तृत) के बीच चयन करें।

### मेमोरी प्रोफाइलिंग
- **ऑब्जेक्ट हीप एनालिसिस**: ऑब्जेक्ट ग्राफ को ट्रैवर्स करें, ऑब्जेक्ट प्रॉपर्टीज़ का निरीक्षण करें और मेमोरी लीक की पहचान करें।
- **मेमोरी रिटेंशन पाथ**: समझें कि ऑब्जेक्ट मेमोरी में क्यों बने रहते हैं और ऑब्जेक्ट लाइफसाइकल को ऑप्टिमाइज़ करें।
- **स्नैपशॉट तुलना**: समय के साथ मेमोरी उपयोग में परिवर्तनों को ट्रैक करने के लिए मेमोरी स्नैपशॉट की तुलना करें।
- **डी-ओबफस्केशन सपोर्ट**: ProGuard या Zelix KlassMaster जैसे टूल्स से ओबफस्केटेड एप्लिकेशन के लिए मूल क्लास, मेथड और फील्ड नाम रिस्टोर करें।

### थ्रेड प्रोफाइलिंग
- **थ्रेड एक्टिविटी विज़ुअलाइज़ेशन**: थ्रेड स्टेट्स की निगरानी करें, ब्लॉक्ड थ्रेड्स का पता लगाएं और सिंक्रोनाइज़ेशन समस्याओं का विश्लेषण करें।
- **डेडलॉक डिटेक्शन**: स्वचालित रूप से डेडलॉक की पहचान करें और संबंधित थ्रेड्स और मॉनिटर्स के बारे में विवरण प्रदान करें।
- **फ्रोजन थ्रेड्स व्यू**: लंबे वेट या संभावित डेडलॉक के कारण निष्क्रिय थ्रेड्स की पहचान करें।

### एक्सेप्शन प्रोफाइलिंग
- **एक्सेप्शन एनालिसिस**: एक्ज़िक्यूशन के दौरान फेंके गए एक्सेप्शन का पता लगाएं और विश्लेषण करें, जिसमें अत्यधिक एक्सेप्शन थ्रो करने के कारण होने वाले छिपे हुए परफॉर्मेंस मुद्दे शामिल हैं।
- **एक्सेप्शन फ्लेम ग्राफ**: समस्याग्रस्त क्षेत्रों की पहचान करने के लिए एक्सेप्शन घटनाओं को विज़ुअलाइज़ करें।

### डेटाबेस और I/O प्रोफाइलिंग
- **SQL और NoSQL सपोर्ट**: MongoDB, Cassandra, और HBase जैसे डेटाबेस के लिए धीमे क्वेरी की पहचान करने के लिए क्वेरीज़ प्रोफाइल करें।
- **HTTP रिक्वेस्ट एनालिसिस**: रिक्वेस्ट प्रोसेसिंग परफॉर्मेंस को समझने के लिए थ्रेड स्टेट्स के साथ HTTP रिक्वेस्ट को जोड़ें।
- **I/O ऑपरेशन**: अक्षम I/O ऑपरेशन का पता लगाएं और रिसोर्स उपयोग को ऑप्टिमाइज़ करें।

### परफॉर्मेंस निरीक्षण
- **40+ बिल्ट-इन इंस्पेक्शन**: लीक्ड वेबएप्स, डुप्लिकेटेड ऑब्जेक्ट्स, नॉन-क्लोज्ड SQL स्टेटमेंट्स और अक्षम कलेक्शन जैसे सामान्य मुद्दों को स्वचालित रूप से पहचानें।
- **कस्टम इंस्पेक्शन**: एप्लिकेशन-विशिष्ट परफॉर्मेंस डेटा एकत्र करने के लिए कस्टम प्रोब बनाएं।

### टेलीमेट्री और परफॉर्मेंस चार्ट
- **रियल-टाइम मॉनिटरिंग**: CPU, मेमोरी, गार्बेज कलेक्शन (GC), और अन्य मेट्रिक्स को रियल टाइम में ट्रैक करें।
- **कस्टमाइज़ेबल इंटरफेस**: प्रासंगिक परफॉर्मेंस डेटा पर ध्यान केंद्रित करने के लिए UI को अनुकूलित करें।

### एकीकरण और ऑटोमेशन
- **IDE प्लगइन्स**: वन-क्लिक प्रोफाइलिंग और प्रोफाइलिंग परिणामों से सोर्स कोड पर नेविगेशन के लिए Eclipse, IntelliJ IDEA, और NetBeans के साथ सीमलेस एकीकरण।
- **कमांड-लाइन टूल्स**: प्रोफाइलिंग टास्क्स को ऑटोमेट करें और CI/CD पाइपलाइन (जैसे Jenkins, TeamCity) के साथ एकीकृत करें।
- **API सपोर्ट**: प्रोफाइलिंग मोड को मैनेज करने और स्नैपशॉट को प्रोग्रामेटिक रूप से कैप्चर करने के लिए एक्स्टेंसिबल API का उपयोग करें।

## योरकिट जावा प्रोफाइलर सेटअप करना
### प्रोफाइलिंग सक्षम करना
किसी जावा एप्लिकेशन को प्रोफाइल करने के लिए, आपको योरकिट प्रोफाइलर एजेंट को JVM से अटैच करना होगा। यह मैन्युअल रूप से या IDE इंटीग्रेशन के माध्यम से किया जा सकता है।

#### मैन्युअल सेटअप
1. **एजेंट लाइब्रेरी का पता लगाएं**:
   - एजेंट लाइब्रेरी `<YourKit Home>/bin/<platform>/libyjpagent.so` (Linux) या `libyjpagent.dll` (Windows) में स्थित है।
2. **JVM कॉन्फ़िगर करें**:
   - JVM स्टार्टअप कमांड में एजेंट जोड़ें:
     ```bash
     java -agentpath:<full agent library path> YourMainClass
     ```
   - Linux के लिए उदाहरण:
     ```bash
     java -agentpath:/home/user/yjp-2025.3/bin/linux-x86-64/libyjpagent.so YourMainClass
     ```
   - वैकल्पिक पैरामीटर:
     - `onexit=memory,dir=<path>`: एक्ज़िट पर मेमोरी स्नैपशॉट कैप्चर करें।
     - `usedmem=70`: मेमोरी उपयोग 70% तक पहुंचने पर स्नैपशॉट ट्रिगर करें।
3. **एजेंट लोडिंग सत्यापित करें**:
   - `[YourKit Java Profiler 2025.3] Profiler agent is listening on port 10001` जैसे संदेशों के लिए कंसोल आउटपुट जांचें।

#### IDE एकीकरण
1. अपने IDE (Eclipse, IntelliJ IDEA, या NetBeans) के लिए योरकिट प्लगइन संबंधित प्लगइन मार्केटप्लेस के माध्यम से इंस्टॉल करें।
2. प्लगइन को योरकिट इंस्टालेशन डायरेक्टरी की ओर इशारा करने के लिए कॉन्फ़िगर करें।
3. अपने एप्लिकेशन को योरकिट अटैच किए हुए शुरू करने के लिए IDE के प्रोफाइलिंग विकल्प का उपयोग करें।

#### रिमोट प्रोफाइलिंग
1. **SSH एक्सेस सुनिश्चित करें**: आपको रिमोट सर्वर तक SSH एक्सेस की आवश्यकता है।
2. **एजेंट कॉपी करें**:
   - उपयुक्त एजेंट लाइब्रेरी को रिमोट सर्वर पर कॉपी करें।
   - Docker के लिए उदाहरण:
     ```bash
     docker cp libyjpagent.so <container_id>:/path/to/agent
     ```
3. **एप्लिकेशन शुरू करें**:
   - रिमोट सर्वर पर JVM स्टार्टअप कमांड में एजेंट जोड़ें।
4. **प्रोफाइलर UI कनेक्ट करें**:
   - योरकिट प्रोफाइलर UI खोलें और "Profile remote Java server or application" चुनें।
   - रिमोट होस्ट और पोर्ट (डिफ़ॉल्ट: 10001) दर्ज करें या SSH टनलिंग का उपयोग करें।
   - कनेक्शन का परीक्षण करें और एप्लिकेशन से कनेक्ट करें।

## योरकिट जावा प्रोफाइलर का उपयोग करना
### एक प्रोफाइलिंग सत्र शुरू करना
1. **प्रोफाइलर UI लॉन्च करें**:
   - Windows पर: स्टार्ट मेन्यू से शुरू करें।
   - Linux/Solaris पर: `<YourKit Home>/bin/yjp.sh` चलाएं।
   - macOS पर: योरकिट जावा प्रोफाइलर आइकन पर क्लिक करें।
2. **एप्लिकेशन से कनेक्ट करें**:
   - लोकल एप्लिकेशन "Monitor Applications" सूची में दिखाई देते हैं।
   - रिमोट एप्लिकेशन के लिए, कनेक्शन को ऊपर बताए अनुसार कॉन्फ़िगर करें।
3. **प्रोफाइलिंग मोड चुनें**:
   - टूलबार से CPU, मेमोरी, या एक्सेप्शन प्रोफाइलिंग के बीच चयन करें।
   - कम ओवरहेड CPU प्रोफाइलिंग के लिए सैंपलिंग या विस्तृत विश्लेषण के लिए ट्रेसिंग का उपयोग करें।

### CPU परफॉर्मेंस का विश्लेषण
1. **CPU प्रोफाइलिंग शुरू करें**:
   - टूलबार से वांछित प्रोफाइलिंग मोड (सैंपलिंग या ट्रेसिंग) चुनें।
   - परिणाम कॉल ट्री, फ्लेम ग्राफ, या मेथड लिस्ट जैसे व्यू में दिखाई देते हैं।
2. **परिणामों की व्याख्या करें**:
   - **कॉल ट्री**: मेथड इनवोकेशन चेन और एक्ज़िक्यूशन टाइम दिखाता है।
   - **फ्लेम ग्राफ**: CPU-इंटेंसिव मेथड्स को विज़ुअली हाइलाइट करता है।
   - **हॉट स्पॉट**: सबसे अधिक CPU समय लेने वाली मेथड्स की सूची बनाता है।
3. **ट्रिगर्स का उपयोग करें**: उच्च CPU उपयोग या विशिष्ट मेथड कॉल के आधार पर स्वचालित रूप से प्रोफाइलिंग शुरू करें।

### मेमोरी उपयोग का विश्लेषण
1. **मेमोरी प्रोफाइलिंग शुरू करें**:
   - ऑब्जेक्ट एलोकेशन और गार्बेज कलेक्शन को ट्रैक करने के लिए मेमोरी प्रोफाइलिंग सक्षम करें।
2. **ऑब्जेक्ट हीप का निरीक्षण करें**:
   - रिटेन किए गए ऑब्जेक्ट्स की पहचान करने के लिए ऑब्जेक्ट ग्राफ को ट्रैवर्स करें।
   - मेमोरी लीक ढूंढने के लिए रिटेंशन पाथ का उपयोग करें।
3. **स्नैपशॉट की तुलना करें**:
   - अलग-अलग बिंदुओं पर स्नैपशॉट कैप्चर करें और मेमोरी ग्रोथ की पहचान करने के लिए उनकी तुलना करें।

### थ्रेड और डेडलॉक विश्लेषण
1. **थ्रेड्स की निगरानी करें**:
   - थ्रेड स्टेट्स देखें और ब्लॉक्ड या फ्रोजन थ्रेड्स की पहचान करें।
   - स्वचालित डेडलॉक डिटेक्शन के लिए "Deadlocks" टैब जांचें।
2. **मॉनिटर्स का विश्लेषण करें**:
   - वेटिंग और ब्लॉकिंग इवेंट्स का निरीक्षण करने के लिए मॉनिटर्स टैब का उपयोग करें।
   - मॉनिटर फ्लेम ग्राफ के साथ कॉन्टेंशन को विज़ुअलाइज़ करें।

### एक्सेप्शन और डेटाबेस प्रोफाइलिंग
1. **एक्सेप्शन प्रोफाइलिंग**:
   - फेंके गए एक्सेप्शन को ट्रैक करने के लिए एक्सेप्शन प्रोफाइलिंग सक्षम करें।
   - एक्सेप्शन पैटर्न का विश्लेषण करने के लिए एक्सेप्शन ट्री या फ्लेम ग्राफ का उपयोग करें।
2. **डेटाबेस प्रोफाइलिंग**:
   - धीमी या अक्षम क्वेरीज़ की पहचान करने के लिए SQL/NoSQL क्वेरीज़ की निगरानी करें।
   - डेटाबेस कॉल को एप्लिकेशन परफॉर्मेंस के साथ सहसंबद्ध करने के लिए थ्रेड स्टेट्स के साथ जोड़ें।

### स्नैपशॉट कैप्चर और विश्लेषण
1. **स्नैपशॉट कैप्चर करें**:
   - UI या कमांड-लाइन टूल का उपयोग करें:
     ```bash
     java -jar <YourKit Home>/lib/yjp-controller-api-redist.jar localhost 10001 capture-performance-snapshot
     ```
   - स्नैपशॉट डिफ़ॉल्ट रूप से `<user home>/Snapshots` में सेव होते हैं।
2. **स्नैपशॉट का विश्लेषण करें**:
   - ऑफलाइन विश्लेषण के लिए स्नैपशॉट को योरकिट UI में खोलें।
   - शेयरिंग के लिए HTML, CSV, या XML जैसे फॉर्मेट में रिपोर्ट एक्सपोर्ट करें।

## प्रभावी प्रोफाइलिंग के लिए सर्वोत्तम प्रथाएं
1. **ओवरहेड कम से कम करें**:
   - प्रोडक्शन में ओवरहेड कम करने के लिए CPU प्रोफाइलिंग के लिए सैंपलिंग का उपयोग करें।
   - उच्च लोड के तहत J2EE प्रोफाइलिंग जैसे अनावश्यक फीचर्स को सक्षम करने से बचें।
2. **पर्याप्त अवधि के लिए प्रोफाइल करें**:
   - रुक-रुक कर होने वाले मुद्दों की पहचान करने के लिए पर्याप्त समय तक डेटा कैप्चर करें लेकिन अत्यधिक डेटा संग्रह से बचें।
3. **मुख्य मेट्रिक्स पर ध्यान केंद्रित करें**:
   - CPU-इंटेंसिव मेथड्स, मेमोरी लीक और थ्रेड कॉन्टेंशन को प्राथमिकता दें।
4. **तुलना के लिए स्नैपशॉट का उपयोग करें**:
   - परफॉर्मेंस परिवर्तनों को ट्रैक करने के लिए नियमित रूप से स्नैपशॉट कैप्चर करें और उनकी तुलना करें।
5. **ऑटोमेशन का लाभ उठाएं**:
   - निरंतर परफॉर्मेंस मॉनिटरिंग के लिए कमांड-लाइन टूल्स का उपयोग करके CI/CD पाइपलाइन के साथ एकीकृत करें।
6. **पहले स्टेजिंग में टेस्ट करें**:
   - इसके प्रभाव को समझने के लिए प्रोडक्शन में उपयोग करने से पहले स्टेजिंग एनवायरनमेंट में प्रोफाइलिंग का अभ्यास करें।

## उपयोग के मामले
- **परफॉर्मेंस ऑप्टिमाइज़ेशन**: CPU-इंटेंसिव मेथड्स या धीमी डेटाबेस क्वेरीज़ की पहचान करें और ऑप्टिमाइज़ करें।
- **मेमोरी लीक डिटेक्शन**: अनावश्यक रूप से मेमोरी में बने रहने वाले ऑब्जेक्ट्स ढूंढें और गार्बेज कलेक्शन को ऑप्टिमाइज़ करें।
- **थ्रेड सिंक्रोनाइज़ेशन**: मल्टी-थ्रेडेड एप्लिकेशन में डेडलॉक और कॉन्टेंशन समस्याओं को हल करें।
- **प्रोडक्शन मॉनिटरिंग**: महत्वपूर्ण परफॉर्मेंस प्रभाव के बिना प्रोडक्शन में एप्लिकेशन की निगरानी के लिए लो-ओवरहेड प्रोफाइलिंग का उपयोग करें।
- **CI/CD इंटीग्रेशन**: रिग्रेशन को जल्दी पकड़ने के लिए बिल्ड पाइपलाइन में परफॉर्मेंस टेस्टिंग को ऑटोमेट करें।

## डेवलपमेंट टूल्स के साथ एकीकरण
- **IDE प्लगइन्स**: योरकिट Eclipse, IntelliJ IDEA, और NetBeans के साथ एकीकृत होता है, जो वन-क्लिक प्रोफाइलिंग और प्रोफाइलिंग परिणामों से सोर्स कोड पर नेविगेशन की अनुमति देता है।
