आप चाहते हैं कि macOS पर Nginx उसके डिफ़ॉल्ट पोर्ट (HTTP के लिए 80, HTTPS के लिए 443) से **एक अलग पोर्ट** पर सुनें। यहां एक स्पष्ट चरण-दर-चरण मार्गदर्शिका दी गई है:

---

### 1. अपनी Nginx कॉन्फ़िगरेशन फ़ाइल का पता लगाएं

* यदि आपने **Homebrew** के माध्यम से इंस्टॉल किया है, तो डिफ़ॉल्ट कॉन्फ़िग आमतौर पर यह होती है:

  ```
  /usr/local/etc/nginx/nginx.conf
  ```

  या, Apple Silicon पर:

  ```
  /opt/homebrew/etc/nginx/nginx.conf
  ```
* यदि आपने मैन्युअल रूप से इंस्टॉल किया है, तो यह `/usr/local/nginx/conf/nginx.conf` हो सकती है।

---

### 2. `listen` डायरेक्टिव को एडिट करें

1. कॉन्फ़िग को एक टेक्स्ट एडिटर में खोलें:

   ```bash
   nano /opt/homebrew/etc/nginx/nginx.conf
   ```
2. `server` ब्लॉक ढूंढें:

   ```nginx
   server {
       listen 80;
       server_name localhost;

       location / {
           root   html;
           index  index.html index.htm;
       }
   }
   ```
3. `listen 80;` को अपने वांछित पोर्ट में बदलें, उदाहरण के लिए, 8080:

   ```nginx
   listen 8080;
   ```

---

### 3. कॉन्फ़िगरेशन को टेस्ट करें

चलाएं:

```bash
nginx -t
```

* यदि यह `syntax is ok` और `test is successful` कहता है, तो आपकी कॉन्फ़िग वैध है।
* यदि कोई एरर हैं, तो जिन लाइन नंबरों की यह रिपोर्ट करता है, उन्हें चेक करें।

---

### 4. Nginx को रीस्टार्ट करें

* यदि Homebrew के माध्यम से इंस्टॉल किया गया है:

```bash
brew services restart nginx
```

* या मैन्युअल रूप से:

```bash
sudo nginx -s reload
```

---

### 5. नए पोर्ट के माध्यम से एक्सेस करें

* अपने ब्राउज़र या curl में:

```
http://localhost:8080
```

---

**अतिरिक्त नोट्स:**

* **1024 से कम** पोर्ट्स के लिए macOS पर `sudo` की आवश्यकता होती है। इसलिए 80 जैसे पोर्ट्स का उपयोग करने के लिए रूट की आवश्यकता होगी, लेकिन 8080 या 3000 के लिए नहीं।
* यदि आप एक साथ कई Nginx सर्वर चलाना चाहते हैं, तो प्रत्येक का **एक अद्वितीय पोर्ट होना चाहिए**।

---

मैं **nginx.conf को एडिट किए बिना किसी भी पोर्ट पर अस्थायी रूप से Nginx चलाने की एक त्वरित ट्रिक** भी दे सकता हूं, जो एक कमांड लाइन फ्लैग का उपयोग करती है। यह टेस्टिंग के लिए उपयोगी है। क्या आप चाहते हैं कि मैं इसे दिखाऊं?
