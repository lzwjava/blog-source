Visual Studio Code (VSCode) में WebSphere Liberty Server के लिए रिमोट डीबगिंग करने के लिए, जो डीबग मोड में चल रहा है, आप VSCode की बिल्ट-इन Java डीबगिंग सुविधाओं का उपयोग कर सकते हैं। नीचे इसे सेट अप करने के विस्तृत चरण दिए गए हैं, बिना किसी WebSphere Liberty-विशिष्ट प्लगइन्स को इंस्टॉल किए, यह मानते हुए कि आपके पास स्टैंडर्ड **Java Extension Pack** (जिसमें **Debugger for Java** शामिल है) पहले से ही इंस्टॉल है।

---

### चरण 1: WebSphere Liberty Server को डीबग मोड में शुरू करें
1. एक टर्मिनल या कमांड प्रॉम्प्ट खोलें।
2. अपने WebSphere Liberty इंस्टालेशन डायरेक्टरी में नेविगेट करें।
3. अपने सर्वर को डीबग मोड में शुरू करने के लिए निम्न कमांड चलाएँ:
   ```
   server debug default
   ```
   - यदि आपके सर्वर का नाम अलग है तो `default` को उस नाम से बदलें।
4. सर्वर डीबगिंग सक्षम होकर शुरू हो जाएगा, आमतौर पर पोर्ट **7777** पर सुनता है।
5. सर्वर के कंसोल आउटपुट या लॉग में इस तरह का संदेश देखें:
   ```
   Listening for transport dt_socket at address: 7777
   ```
   - यह डीबग पोर्ट की पुष्टि करता है। यदि यह कोई अलग पोर्ट है (जैसे, कॉन्फ्लिक्ट के कारण), तो प्रदर्शित नंबर नोट कर लें।

---

### चरण 2: VSCode में रिमोट डीबगिंग कॉन्फ़िगर करें
1. **VSCode में अपना प्रोजेक्ट खोलें**:
   - सुनिश्चित करें कि आपका Java प्रोजेक्ट (जिसमें सर्वर पर डिप्लॉय किया गया सोर्स कोड है) VSCode में खुला है। इससे डीबगर को ब्रेकपॉइंट्स को रनिंग कोड से मैप करने में मदद मिलती है।

2. **रन और डीबग व्यू एक्सेस करें**:
   - लेफ्ट साइडबार में **Run and Debug** आइकन (एक प्ले बटन के साथ बग) पर क्लिक करें या `Ctrl+Shift+D` (Windows/Linux) या `Cmd+Shift+D` (Mac) दबाएँ।

3. **`launch.json` फ़ाइल बनाएँ या एडिट करें**:
   - **Run and Debug** व्यू में, कॉन्फ़िगरेशन ड्रॉपडाउन के आगे वाले **गियर आइकन** पर क्लिक करें।
   - यदि एनवायरनमेंट चुनने के लिए कहा जाए, तो **Java** चुनें। यह आपके वर्कस्पेस के `.vscode` फोल्डर में एक `launch.json` फ़ाइल बनाता है।
   - यदि फ़ाइल पहले से मौजूद है, तो वह एडिटिंग के लिए खुल जाएगी।

4. **एक डीबग कॉन्फ़िगरेशन जोड़ें**:
   - `launch.json` फ़ाइल में, सुनिश्चित करें कि इसमें रिमोट JVM से अटैच होने के लिए एक कॉन्फ़िगरेशन है। यहाँ एक उदाहरण है:
     ```json
     {
         "version": "0.2.0",
         "configurations": [
             {
                 "type": "java",
                 "name": "Attach to WebSphere Liberty",
                 "request": "attach",
                 "hostName": "localhost",
                 "port": 7777
             }
         ]
     }
     ```
   - **फ़ील्ड्स की व्याख्या**:
     - `"type": "java"`: Java डीबगर निर्दिष्ट करता है।
     - `"name": "Attach to WebSphere Liberty"`: इस कॉन्फ़िगरेशन के लिए एक वर्णनात्मक नाम।
     - `"request": "attach"`: इंगित करता है कि VSCode एक मौजूदा JVM प्रोसेस से अटैच करेगा।
     - `"hostName": "localhost"`: वह मशीन का होस्टनाम जहाँ सर्वर चल रहा है। यदि सर्वर किसी अलग मशीन पर है तो उसके IP एड्रेस या होस्टनाम का उपयोग करें।
     - `"port": 7777`: चरण 1 से डीबग पोर्ट। यदि सर्वर कोई अलग पोर्ट उपयोग कर रहा है तो इसे अपडेट करें।

5. **फ़ाइल सेव करें**:
   - कॉन्फ़िगरेशन जोड़ने या एडिट करने के बाद `launch.json` फ़ाइल को सेव करें।

### चरण 3: डीबगिंग सेशन शुरू करें
1. **सुनिश्चित करें कि सर्वर चल रहा है**:
   - सत्यापित करें कि WebSphere Liberty Server चरण 1 से डीबग मोड में अभी भी चल रहा है।

2. **कॉन्फ़िगरेशन चुनें**:
   - **Run and Debug** व्यू में, ऊपर के ड्रॉपडाउन मेनू से **"Attach to WebSphere Liberty"** चुनें।

3. **डीबगर लॉन्च करें**:
   - हरे **प्ले बटन** पर क्लिक करें या `F5` दबाएँ। VSCode सर्वर की JVM प्रोसेस से कनेक्ट हो जाएगा।

4. **ब्रेकपॉइंट्स सेट करें**:
   - अपनी Java सोर्स फ़ाइलों को VSCode में खोलें।
   - जहाँ आप एक्जिक्यूशन को रोकना चाहते हैं, वहाँ लाइन नंबरों के बाईं ओर गटर में क्लिक करके ब्रेकपॉइंट्स सेट करें।

5. **एप्लिकेशन के साथ इंटरैक्ट करें**:
   - अपने एप्लिकेशन का उपयोग करें (जैसे, HTTP रिक्वेस्ट भेजें) ताकि ब्रेकपॉइंट्स वाले कोड पाथ ट्रिगर हों। जब कोई ब्रेकपॉइंट हिट होता है, तो VSCode एक्जिक्यूशन को रोक देगा, जिससे आप वेरिएबल्स का निरीक्षण कर सकते हैं, कोड के माध्यम से कदम बढ़ा सकते हैं और डीबग कर सकते हैं।

---

### अतिरिक्त नोट्स
- **लोकल बनाम रिमोट सर्वर**:
  - यदि सर्वर किसी अलग मशीन पर चल रहा है, तो `launch.json` फ़ाइल में `"localhost"` को सर्वर के होस्टनाम या IP एड्रेस से बदलें। सुनिश्चित करें कि डीबग पोर्ट नेटवर्क के माध्यम से एक्सेसिबल है (जैसे, फ़ायरवॉल द्वारा ब्लॉक नहीं है)।

- **डीबग पोर्ट कन्फर्मेशन**:
  - यदि आप पोर्ट के बारे में अनिश्चित हैं, तो हमेशा `server debug default` चलाने के बाद सर्वर के आउटपुट को चेक करें। यह स्पष्ट रूप से बताएगा कि वह किस पोर्ट पर सुन रहा है।

- **सोर्स कोड मिलान**:
  - ब्रेकपॉइंट्स के सही तरीके से काम करने के लिए, VSCode में सोर्स कोड सर्वर पर चल रहे कोड से मेल खाना चाहिए। सुनिश्चित करें कि आपका प्रोजेक्ट अप-टू-डेट है और डीबग इनफॉर्मेशन के साथ कंपाइल किया गया है (अधिकांश Java बिल्ड्स में यह डिफॉल्ट होता है)।

- **सर्वर बिहेवियर**:
  - `server debug default` के साथ, सर्वर तुरंत शुरू हो जाता है (डीबगर के इंतजार में नहीं रुकता)। जब तक यह चल रहा है, आप किसी भी समय VSCode अटैच कर सकते हैं।

- **पूर्वापेक्षाएँ**:
  - VSCode में **Java Extension Pack** (या कम से कम **Debugger for Java** एक्सटेंशन) इंस्टॉल होना चाहिए। यह Java डेवलपमेंट के लिए एक स्टैंडर्ड टूल है और आवश्यक डीबगिंग क्षमताएँ प्रदान करता है।
