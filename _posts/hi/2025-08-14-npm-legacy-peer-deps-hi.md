### अवलोकन
`--legacy-peer-deps` फ्लैग npm (Node Package Manager) के साथ उपयोग किया जाने वाला एक विकल्प है जो पैकेज इंस्टालेशन के दौरान पीयर डिपेंडेंसी को कैसे हैंडल किया जाए, इसे नियंत्रित करता है। पीयर डिपेंडेंसी वे डिपेंडेंसी होती हैं जिन्हें एक पैकेज दूसरे से प्रदान करने की अपेक्षा करता है, लेकिन npm के नए वर्जन में उन्हें स्वचालित रूप से इंस्टॉल नहीं किया जाता है।

### व्यवहार
- **मानक npm (v7+)**: यह डिफ़ॉल्ट रूप से पीयर डिपेंडेंसी आवश्यकताओं को सख्ती से लागू करता है। यदि कोई पैकेज एक पीयर डिपेंडेंसी घोषित करता है जो पूरी नहीं होती है, तो npm संघर्षों को रोकने के लिए इंस्टॉल पर चेतावनी देता है या विफल कर देता है।
- **--legacy-peer-deps के साथ**: यह पुराने व्यवहार (npm v3~v6 से) पर वापस आ जाता है, जहां गायब पीयर डिपेंडेंसी को स्वचालित रूप से इंस्टॉल कर दिया जाता है, भले ही इससे डुप्लिकेट या मिसमैच हो। यह कुछ लीगेसी पैकेज समस्याओं को हल कर सकता है लेकिन बग्स का कारण बन सकता है।

### उदाहरण उपयोग
एक टर्मिनल में, आप यह चलाएंगे:
```
npm install some-package --legacy-peer-deps
```
इसका उपयोग अक्सर उन प्रोजेक्ट्स के लिए किया जाता है जो पुराने लाइब्रेरीज़ पर निर्भर हैं जिन्होंने अपनी पीयर डिपेंडेंसी घोषणाओं को अपडेट नहीं किया है, जैसा कि npm के सख्त पीयर हैंडलिंग से पहले कई React या Angular सेटअप में देखा गया है।

### कब उपयोग करें
इसका उपयोग संयम से करें, क्योंकि यह सुरक्षा उपायों को दरकिनार करता है और डिपेंडेंसी हेल का कारण बन सकता है। बेहतर विकल्पों के लिए पैकेजों को अपडेट करना या `--force`/`--save-dev` का उपयोग करना प्राथमिकता दें। पूरी जानकारी के लिए, पीयर डिपेंडेंसी पर आधिकारिक npm डॉक्स देखें।
