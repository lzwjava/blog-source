जब `ng build` चलाते समय **Node.js 22.9.0** और **npm 10.8.3** के साथ, आप इस त्रुटि से सामना कर सकते हैं:

```
error 0308010C digital envelope routines::unsupported
```

यह त्रुटि आम तौर पर नए Node.js संस्करणों (जैसे 22.9.0, जो OpenSSL 3.0 या बाद के संस्करण का उपयोग करता है) और कुछ क्रिप्टोग्राफिक एल्गोरिथमों के बीच संगतता समस्या के कारण होती है, जिन पर टूल्स जैसे Webpack निर्भर करते हैं, जो Angular के नीचे काम करता है। नीचे इस समस्या को ठीक करने के लिए एक कदम-दर-कदम मार्गदर्शिका है।

---

### **समाधान**

#### कदम 1: OpenSSL Legacy Provider को सक्रिय करें
सबसे सरल और तत्काल समाधान OpenSSL legacy provider को सक्रिय करने के लिए `NODE_OPTIONS` पर्यावरण चर को सेट करना है। इससे Node.js को पुराने क्रिप्टोग्राफिक एल्गोरिथम का उपयोग करने की अनुमति मिलती है, जो OpenSSL 3.0+ में डिफ़ॉल्ट रूप से समर्थित नहीं हो सकते हैं।

- **macOS/Linux के लिए**:
  अपने टर्मिनल खोलें और चलाएं:
  ```bash
  export NODE_OPTIONS=--openssl-legacy-provider
  ng build
  ```

- **Windows (Command Prompt) के लिए**:
  कमांड प्रॉम्प्ट खोलें और चलाएं:
  ```cmd
  set NODE_OPTIONS=--openssl-legacy-provider
  ng build
  ```

- **Windows (PowerShell) के लिए**:
  पावरशेल खोलें और चलाएं:
  ```powershell
  $env:NODE_OPTIONS="--openssl-legacy-provider"
  ng build
  ```

पर्यावरण चर को सेट करने के बाद, फिर से `ng build` चलाएं। यह अधिकांश मामलों में त्रुटि को सुलझाने के लिए Node.js को असमर्थ रूटीन को प्रोसेस करने की अनुमति देगा।

---

#### कदम 2: Angular CLI की जांच और अपडेट करें (अगर आवश्यक हो)
यदि कदम 1 के बाद त्रुटि बनी रहती है, तो आपके Angular CLI का संस्करण Node.js 22.9.0 के साथ पूरी तरह से संगत नहीं हो सकता है। इसे नवीनतम संस्करण में अपडेट करने से मदद मिल सकती है।

- अपने वर्तमान Angular CLI संस्करण की जांच करें:
  ```bash
  ng --version
  ```

- Angular CLI को ग्लोबल रूप से अपडेट करें:
  ```bash
  npm install -g @angular/cli
  ```

- फिर, फिर से `ng build` चलाएं।

---

#### कदम 3: प्रोजेक्ट डिपेंडेंस की जांच और अपडेट करें (वैकल्पिक)
यदि अभी भी समस्या सुलझ नहीं रही है, तो प्रोजेक्ट में पुराने डिपेंडेंस हो सकते हैं जो समस्या का कारण हो सकते हैं। इसके लिए:

- अपने `package.json` फ़ाइल खोलें और अपने डिपेंडेंस (जैसे `@angular/core`, `@angular/cli`, आदि) के संस्करणों की जांच करें।
- उन्हें सावधानीपूर्वक उनके नवीनतम संगत संस्करण में अपडेट करें:
  ```bash
  npm install
  ```
  या, अगर आप सभी डिपेंडेंस अपडेट करना चाहते हैं:
  ```bash
  npm update
  ```

- फिर से `ng build` चलाएं।

*सावधानी*: अपडेट्स में तोड़ने वाले परिवर्तन हो सकते हैं। इसके बाद अपने एप्लिकेशन को पूरी तरह से टेस्ट करें।

---

#### कदम 4: Node.js को डाउनग्रेड करें (अंतिम उपाय)
यदि ऊपर दिए गए किसी भी समाधान काम नहीं करते, तो Node.js 22.9.0 अभी तक आपके प्रोजेक्ट के वर्तमान कॉन्फ़िगरेशन द्वारा पूरी तरह से समर्थित नहीं हो सकता है। इसे Long-Term Support (LTS) संस्करण में डाउनग्रेड करने से समस्या सुलझ सकती है, जैसे **20.x** या **18.x**.

- अपने वर्तमान Node.js संस्करण को अनइंस्टॉल करें।
- [अधिकारिक Node.js वेबसाइट](https://nodejs.org/) से एक LTS संस्करण डाउनलोड और इंस्टॉल करें।
- इंस्टॉल किए गए संस्करण की जांच करें:
  ```bash
  node -v
  ```
- फिर से अपने प्रोजेक्ट डिपेंडेंस इंस्टॉल करें:
  ```bash
  npm install
  ```
- फिर से `ng build` चलाएं।

### **सिफारिश की गई रणनीति**
**कदम 1** (NODE_OPTIONS सेट करने) से शुरू करें, क्योंकि यह सबसे तेज़ समाधान है और आपको Node.js 22.9.0 का उपयोग करने की अनुमति देता है। अगर यह काम नहीं करता, तो **कदम 2** और **कदम 3** पर आगे बढ़ें। **कदम 4** केवल तब उपयोग करें जब आवश्यक हो, क्योंकि Node.js को डाउनग्रेड करने से नए फीचर्स तक पहुंच सीमित हो सकती है।

यह "error 0308010C digital envelope routines::unsupported" त्रुटि को सुलझाने के लिए और फिर से `ng build` कमांड को काम करने के लिए आपको मदद करेगा!
