जब आप **Node.js 22.9.0** और **npm 10.8.3** के साथ `ng build` चलाते हैं, तो आपको यह त्रुटि मिलती है:

```
error 0308010C digital envelope routines::unsupported
```

यह त्रुटि आमतौर पर Node.js के नए संस्करणों (जैसे 22.9.0, जो OpenSSL 3.0 या बाद के संस्करण का उपयोग करता है) और कुछ क्रिप्टोग्राफिक एल्गोरिदम के बीच अनुकूलता की समस्या के कारण उत्पन्न होती है, जिन पर Webpack जैसे टूल निर्भर करते हैं, जिसका उपयोग Angular अंदरूनी तौर पर करता है। इस समस्या को ठीक करने के लिए नीचे चरण-दर-चरण मार्गदर्शिका दी गई है।

---

### समाधान

#### चरण 1: OpenSSL लीगेसी प्रोवाइडर को सक्षम करें
सबसे सरल और तत्काल समाधान `NODE_OPTIONS` पर्यावरण चर सेट करके OpenSSL लीगेसी प्रोवाइडर को सक्षम करना है। यह Node.js को पुराने क्रिप्टोग्राफिक एल्गोरिदम का उपयोग करने की अनुमति देता है जो OpenSSL 3.0+ में डिफ़ॉल्ट रूप से अब समर्थित नहीं हो सकते हैं।

- **macOS/Linux के लिए**:
  अपना टर्मिनल खोलें और चलाएँ:
  ```bash
  export NODE_OPTIONS=--openssl-legacy-provider
  ng build
  ```

- **Windows (Command Prompt) के लिए**:
  Command Prompt खोलें और चलाएँ:
  ```cmd
  set NODE_OPTIONS=--openssl-legacy-provider
  ng build
  ```

- **Windows (PowerShell) के लिए**:
  PowerShell खोलें और चलाएँ:
  ```powershell
  $env:NODE_OPTIONS="--openssl-legacy-provider"
  ng build
  ```

पर्यावरण चर सेट करने के बाद, `ng build` को फिर से चलाएं। यह अधिकांश मामलों में Node.js को असमर्थित रूटीन को प्रोसेस करने की अनुमति देकर त्रुटि को हल कर देगा।

---

#### चरण 2: Angular CLI को सत्यापित और अपडेट करें (यदि आवश्यक हो)
यदि चरण 1 के बाद भी त्रुटि बनी रहती है, तो आपके Angular CLI का संस्करण Node.js 22.9.0 के साथ पूरी तरह से संगत नहीं हो सकता है। इसे नवीनतम संस्करण में अपडेट करने से मदद मिल सकती है।

- अपना वर्तमान Angular CLI संस्करण जांचें:
  ```bash
  ng --version
  ```

- Angular CLI को वैश्विक रूप से अपडेट करें:
  ```bash
  npm install -g @angular/cli
  ```

- फिर, `ng build` को फिर से चलाने का प्रयास करें।

---

#### चरण 3: प्रोजेक्ट निर्भरताओं की जांच करें और अपडेट करें (वैकल्पिक)
यदि समस्या अभी भी हल नहीं होती है, तो आपकी परियोजना में पुरानी निर्भरताएँ समस्या पैदा कर रही हो सकती हैं। इसे हल करने के लिए:

- अपनी `package.json` फ़ाइल खोलें और अपनी निर्भरताओं (जैसे `@angular/core`, `@angular/cli`, आदि) के संस्करणों की समीक्षा करें।
- उन्हें सावधानीपूर्वक उनके नवीनतम संगत संस्करणों में अपडेट करें:
  ```bash
  npm install
  ```
  या, यदि आप सभी निर्भरताओं को अपडेट करना चाहते हैं:
  ```bash
  npm update
  ```

- परीक्षण करने के लिए `ng build` को फिर से चलाएं।

*नोट*: अपडेट के साथ सावधान रहें, क्योंकि वे ब्रेकिंग चेंज ला सकते हैं। बाद में अपने एप्लिकेशन का पूरी तरह से परीक्षण करें।

---

#### चरण 4: Node.js को डाउनग्रेड करें (अंतिम उपाय)
यदि उपरोक्त में से कोई भी समाधान काम नहीं करता है, तो हो सकता है कि आपकी परियोजना की वर्तमान कॉन्फ़िगरेशन द्वारा Node.js 22.9.0 का अभी तक पूरी तरह से समर्थन नहीं किया गया हो। Node.js को **20.x** या **18.x** जैसे लॉन्ग-टर्म सपोर्ट (LTS) संस्करण में डाउनग्रेड करने से समस्या हल हो सकती है।

- अपना वर्तमान Node.js संस्करण अनइंस्टॉल करें।
- [आधिकारिक Node.js वेबसाइट](https://nodejs.org/) से एक LTS संस्करण डाउनलोड करें और इंस्टॉल करें।
- इंस्टॉल किए गए संस्करण को सत्यापित करें:
  ```bash
  node -v
  ```
- अपनी परियोजना निर्भरताओं को पुनः इंस्टॉल करें:
  ```bash
  npm install
  ```
- `ng build` को फिर से चलाएं।

### अनुशंसित दृष्टिकोण
**चरण 1** (`NODE_OPTIONS` सेट करना) से शुरुआत करें, क्योंकि यह सबसे तेज़ समाधान है और आपको Node.js 22.9.0 का उपयोग जारी रखने की अनुमति देता है। यदि यह काम नहीं करता है, तो **चरण 2** और **चरण 3** पर आगे बढ़ें। **चरण 4** का उपयोग केवल तभी करें जब आवश्यक हो, क्योंकि Node.js को डाउनग्रेड करने से नई सुविधाओं तक पहुंच सीमित हो सकती है।

इससे "error 0308010C digital envelope routines::unsupported" त्रुटि हल हो जानी चाहिए और आपकी `ng build` कमांड फिर से काम करने लगेगी!
