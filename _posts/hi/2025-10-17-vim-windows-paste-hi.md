## विंडोज़ पर Git Bash में Vim में पेस्ट करना

आह, मैं समस्या समझ गया/गई—विंडोज़ पर चलने वाले Git Bash में Vim का `p` कमांड चुनौतीपूर्ण हो सकता है, खासकर सिस्टम क्लिपबोर्ड या बाहरी टेक्स्ट के साथ काम करते समय। यह एक आम समस्या है क्योंकि Git Bash टर्मिनल एमुलेटर के रूप में MinTTY का उपयोग करता है, और Vim का डिफ़ॉल्ट व्यवहार हमेशा विंडोज़ के क्लिपबोर्ड के साथ ठीक से काम नहीं करता। `p` कमांड केवल Vim के *आंतरिक* यैंक बफर से पेस्ट करता है (जिसे आपने उसी Vim सत्र के अंदर `y` से कॉपी किया था)। Vim के बाहर से किसी भी चीज़ (जैसे नोटपैड या ब्राउज़र से कॉपी करना) के लिए, आपको टर्मिनल के पेस्ट या क्लिपबोर्ड एकीकरण की आवश्यकता होती है।

चिंता न करें—यहां बताया गया है कि इसे चरण दर चरण कैसे संभालना है, जो आके "सभी चुनें, हटाएं, और नई टेक्स्ट में पेस्ट करें" वर्कफ़्लो के अनुरूप है। मैं आंतरिक पेस्टिंग (जो पहले से ही काम करनी चाहिए) और बाहरी पेस्टिंग को कवर करूंगा/ करूंगी।

### त्वरित जांच: क्या आपका Vim सिस्टम क्लिपबोर्ड को सपोर्ट करता है?
पहले, पुष्टि करें कि क्या आपका Vim सीधे विंडोज़ क्लिपबोर्ड तक पहुंच सकता है (यह बहुत कुछ ठीक कर देता है):
1. Git Bash खोलें।
2. `vim --version | grep clipboard` चलाएं (या Vim को `vim` के साथ खोलें, फिर `:version` टाइप करें और `+clipboard` देखें)।
   - यदि यह `+clipboard` दिखाता है, तो बढ़िया—आप विंडोज़ क्लिपबोर्ड से पेस्ट करने के लिए `"+p` का उपयोग कर सकते हैं।
   - यदि `-clipboard` या कुछ नहीं दिखता, तो यह सपोर्टेड नहीं है (Git Bash के बंडल किए गए Vim में आम)। नीचे दिए गए टर्मिनल पेस्ट विधियों पर आगे बढ़ें।

यदि सपोर्टेड नहीं है, तो विंडोज़ के लिए एक पूर्ण Vim इंस्टॉल करने पर विचार करें (उदाहरण के लिए, Chocolatey के माध्यम से: `choco install vim`) या बेहतर एकीकरण के लिए Neovim/WSL का उपयोग करें।

### 1. आंतरिक पेस्टिंग (Vim के अंदर यैंक करें, उसी सत्र में पेस्ट करें)
यह वही है जिस पर मेरे पिछले निर्देश केंद्रित थे—Vim के *अंदर* कॉपी करना और *उसी* सत्र में एक नई फ़ाइल में पेस्ट करना। यहां `p` ठीक काम करना चाहिए, इसमें Git Bash की कोई समस्या शामिल नहीं है:
- `oldfile.txt` में: `gg` (शीर्ष पर), `yG` (सब कुछ यैंक करें)।
- `:e newfile.txt` (उसी Vim में नई फ़ाइल खोलें)।
- `p` (पेस्ट करें)। यह कर्सर के ठीक बाद कंटेंट डाल देगा।
- सेव करने के लिए `:wq`।

यदि `p` अभी भी विफल हो जाता है (उदाहरण के लिए, कुछ भी पेस्ट नहीं करता या खराब टेक्स्ट पेस्ट करता है), तो यह यैंक समस्या हो सकती है—यदि क्लिपबोर्ड सपोर्टेड है तो `yG` के बजाय `"+yG` आज़माएं, फिर `"+p`।

### 2. Vim में बाहरी टेक्स्ट पेस्ट करना (विंडोज़ ऐप्स से)
यदि आप बाहर से कॉपी कर रहे हैं (उदाहरण के लिए, नोटपैड में सभी चुनें, Ctrl+C, फिर Vim में पेस्ट करना चाहते हैं):
- **विधि 1: Git Bash के बिल्ट-इन पेस्ट का उपयोग करें (सबसे आसान, Vim में कोई बदलाव की आवश्यकता नहीं)**
  1. अपनी फ़ाइल खोलें: `vim newfile.txt`।
  2. इन्सर्ट मोड में प्रवेश करें: `i` दबाएं।
  3. Git Bash विंडो में राइट-क्लिक करें (यह विंडोज़ क्लिपबोर्ड से सीधे टर्मिनल/Vim में पेस्ट कर देता है)।
     - वैकल्पिक शॉर्टकट: `Insert` कुंजी, या Git Bash में क्विक एडिट मोड सक्षम करें (राइट-क्लिक टाइटल बार > Options > Quick Edit) फिर Ctrl+Shift+V का उपयोग करें।
  4. इन्सर्ट मोड से बाहर निकलने के लिए `Esc` दबाएं।
  - *प्रो टिप:* यदि मल्टी-लाइन टेक्स्ट पेस्ट करने पर अजीब दिखता है (अतिरिक्त नई लाइन या इंडेंट), तो यह अक्सर विंडोज़ लाइन एंडिंग (CRLF बनाम LF) के कारण होता है। पेस्ट करने से पहले Vim में `:set ff=unix` से ठीक करें, या बाद में `:%s/\r$//g` का उपयोग करके कैरिज रिटर्न हटाएं।

- **विधि 2: यदि +clipboard सपोर्टेड है (Vim में सीधे)**
  1. टेक्स्ट को बाहरी रूप से कॉपी करें (Ctrl+C)।
  2. Vim कमांड मोड में: `"+p` (सिस्टम क्लिपबोर्ड से कर्सर के बाद पेस्ट करता है) या `"+P` (पहले)।
  3. आपके वर्कफ़्लो के लिए: पुरानी फ़ाइल में डिलीट करने के बाद, नई फ़ाइल पर स्विच करें और `"+p` करें।

### 3. Vim से बाहरी में कॉपी करना (विंडोज़ में बाहर यैंक करना)
- आंतरिक यैंक (`y`) Vim में रहता है—बाहरी के लिए टर्मिनल कॉपी का उपयोग करें।
- Vim में टेक्स्ट चुनें: `v` (विज़ुअल मोड), सभी चुनने के लिए कर्सर घुमाएं (`ggVG`), फिर `y`।
- विंडोज़ में कॉपी करने के लिए: राइट-क्लिक > Edit > Copy (या हाइलाइट करें और राइट-क्लिक > Copy)।
- +clipboard के साथ: `y` के बजाय `"+y`, फिर कहीं और Ctrl+V।

### Git Bash + Vim पेस्ट समस्याओं का समस्या निवारण
- **खराब टेक्स्ट (उदाहरण के लिए, आंशिक लाइनें या अतिरिक्त स्पेस):** Vim में ब्रैकेटेड पेस्ट अक्षम करें (इन्सर्ट मोड में प्रवेश करने से पहले `:set paste`, फिर पेस्ट करें, फिर `:set nopaste`)। या हमेशा चालू रखने के लिए `~/.vimrc` में `set paste` जोड़ें (लेकिन यह ऑटो-इंडेंट जैसी कुछ सुविधाओं को अक्षम कर देता है)।
- **कुछ भी पेस्ट नहीं होता:** सुनिश्चित करें कि आप टर्मिनल पेस्ट के लिए इन्सर्ट मोड में हैं—कमांड मोड में `p` केवल आंतरिक के लिए है।
- **माउस हस्तक्षेप:** यदि राइट-क्लिक करने पर पेस्ट करने के बजाय सिलेक्ट होता है, तो राइट-क्लिक टाइटल बार > Options > Mouse > Quick Edit से टॉगल करें।
- **पेस्ट करने से पहले सब कुछ हटाने के लिए:** अपने वर्कफ़्लो में, यैंक करने के बाद, नई फ़ाइल को साफ़ करने के लिए `:%d` का उपयोग करें, फिर पेस्ट करें।

इससे आपको आसानी से पेस्ट करने में मदद मिलनी चाहिए! यदि यह अभी भी काम नहीं कर रहा है (उदाहरण के लिए, बताएं कि जब आप `p` करने का प्रयास करते हैं तो वास्तव में क्या होता है), या यदि आप विशेष रूप से *बाहरी स्रोतों* से पेस्ट कर रहे हैं, तो और समायोजन के लिए मुझे बताएं।
