विजुअल स्टूडियो कोड (VS Code) में सोर्स कंट्रोल के लिए मजबूत अंतर्निहित समर्थन है, जो डेवलपर्स को संपादक के भीतर ही Git, Mercurial और अन्य जैसी वर्जन कंट्रोल सिस्टम (VCS) प्रबंधित करने की अनुमति देता है। यह गाइड VS Code में सोर्स कंट्रोल का उपयोग करने का एक व्यापक अवलोकन प्रदान करती है, जो Git पर केंद्रित है क्योंकि यह सबसे अधिक उपयोग की जाने वाली VCS है, और इसमें सेटअप, प्रमुख सुविधाएँ, वर्कफ़्लो और उन्नत उपयोग शामिल हैं।

### VS Code में सोर्स कंट्रोल का अवलोकन
VS Code का सोर्स कंट्रोल व्यू वर्जन कंट्रोल सिस्टम के साथ इंटरैक्ट करने के लिए एक सहज इंटरफेस प्रदान करता है। यह डिफ़ॉल्ट रूप से Git के साथ एकीकृत होता है और अन्य VCS के लिए एक्सटेंशन का समर्थन करता है। सोर्स कंट्रोल व्यू परिवर्तनों को प्रदर्शित करता है, स्टेजिंग, कमिटिंग, ब्रांचिंग, मर्जिंग और बहुत कुछ की अनुमति देता है, और यह सब संपादक को छोड़े बिना। सोर्स कंट्रोल का प्रभावी ढंग से लाभ उठाने के लिए नीचे एक चरण-दर-चरण मार्गदर्शिका दी गई है।

### 1. **VS Code में सोर्स कंट्रोल सेट करना**
सोर्स कंट्रोल का उपयोग करने के लिए, आपको Git इंस्टॉल करनी होगी और एक रिपॉजिटरी आरंभ करनी होगी। इसे सेटअप करने का तरीका यहां बताया गया है:

#### आवश्यक शर्तें
- **Git इंस्टॉल करें**: [git-scm.com](https://git-scm.com/) से Git डाउनलोड करें और इंस्टॉल करें। टर्मिनल में `git --version` चलाकर इंस्टॉलेशन सत्यापित करें।
- **Git कॉन्फ़िगर करें**:
  ```bash
  git config --global user.name "Your Name"
  git config --global user.email "your.email@example.com"
  ```
- **VS Code इंस्टॉल करें**: सुनिश्चित करें कि आपके पास [code.visualstudio.com](https://code.visualstudio.com/) से VS Code का नवीनतम संस्करण इंस्टॉल है।

#### एक Git रिपॉजिटरी आरंभ करें
1. VS Code में एक प्रोजेक्ट फ़ोल्डर खोलें।
2. टर्मिनल खोलें (Ctrl+` या macOS पर Cmd+`) और चलाएँ:
   ```bash
   git init
   ```
   यह आपके प्रोजेक्ट में एक `.git` फ़ोल्डर बनाता है, जिससे यह एक Git रिपॉजिटरी के रूप में आरंभ हो जाता है।
3. वैकल्पिक रूप से, एक मौजूदा रिपॉजिटरी को क्लोन करें:
   ```bash
   git clone <repository-url>
   ```
   फिर क्लोन किए गए फ़ोल्डर को VS Code में खोलें।

#### सोर्स कंट्रोल व्यू सक्षम करें
- एक्टिविटी बार में सोर्स कंट्रोल आइकन पर क्लिक करके (शीर्ष से तीसरा आइकन, जो एक शाखा जैसा दिखता है) या `Ctrl+Shift+G` (Windows/Linux) या `Cmd+Shift+G` (macOS) दबाकर सोर्स कंट्रोल व्यू खोलें।
- यदि एक Git रिपॉजिटरी का पता चलता है, तो VS Code परिवर्तनों को प्रबंधित करने के विकल्पों के साथ सोर्स कंट्रोल व्यू प्रदर्शित करता है।

### 2. **सोर्स कंट्रोल व्यू का उपयोग करना**
सोर्स कंट्रोल व्यू वर्जन कंट्रोल कार्यों के लिए केंद्रीय केंद्र है। यह दिखाता है:
- **परिवर्तन**: अंतिम कमिट के बाद से संशोधित, जोड़े गए या हटाए गए फ़ाइलें।
- **स्टेज्ड परिवर्तन**: कमिट के लिए तैयार फ़ाइलें।
- **कमिट मैसेज बॉक्स**: जहां आप कमिट संदेश दर्ज करते हैं।
- **क्रियाएं**: कमिट करने, रीफ्रेश करने और अधिक के लिए बटन।

#### सामान्य वर्कफ़्लो
1. **परिवर्तन करें**: अपने प्रोजेक्ट में फ़ाइलों को संपादित करें। VS Code स्वचालित रूप से परिवर्तनों का पता लगाता है और उन्हें सोर्स कंट्रोल व्यू में "परिवर्तन" के अंतर्गत सूचीबद्ध करता है।
2. **परिवर्तन स्टेज करें**:
   - किसी फ़ाइल के आगे `+` आइकन पर क्लिक करके उसे स्टेज करें, या `Stage All Changes` विकल्प का उपयोग करें (तीन बिंदु मेनू > Stage All Changes)।
   - स्टेजिंग अगली कमिट के लिए परिवर्तनों को तैयार करती है।
3. **एक कमिट संदेश लिखें**:
   - सोर्स कंट्रोल व्यू के शीर्ष पर टेक्स्ट बॉक्स में एक वर्णनात्मक संदेश दर्ज करें।
   - उदाहरण: `Add user authentication feature`।
4. **परिवर्तन कमिट करें**:
   - चेकमार्क आइकन पर क्लिक करें या `Ctrl+Enter` (Windows/Linux) या `Cmd+Enter` (macOS) दबाकर स्टेज किए गए परिवर्तनों को कमिट करें।
   - तीन बिंदु मेनू का उपयोग करके `Commit All`, `Commit Staged`, या `Commit All and Push` के बीच चयन करें।
5. **रिमोट पर पुश करें**:
   - यदि किसी रिमोट रिपॉजिटरी (जैसे, GitHub) से जुड़ा है, तो तीन बिंदु मेनू में `Push` विकल्प का उपयोग करके परिवर्तनों को पुश करें या टर्मिनल में `git push` चलाएँ।

### 3. **VS Code में सोर्स कंट्रोल की प्रमुख विशेषताएं**
VS Code वर्जन कंट्रोल को सुव्यवस्थित करने के लिए कई विशेषताएं प्रदान करता है:

#### डिफ व्यू
- अंतिम कमिट की तुलना में संशोधन दिखाने वाला साइड-बाय-साइड डिफ व्यू खोलने के लिए "परिवर्तन" के अंतर्गत एक फ़ाइल पर क्लिक करें।
- विशिष्ट पंक्तियों को स्टेज या डिस्कार्ड करने के लिए इनलाइन क्रियाओं का उपयोग करें।

#### ब्रांच प्रबंधन
- ब्रांच स्विच करें: नीचे-बाएँ स्टेटस बार में ब्रांच नाम पर क्लिक करें या सोर्स कंट्रोल व्यू के ब्रांच मेनू (तीन बिंदु > Branch > Checkout to...) का उपयोग करें।
- एक नई ब्रांच बनाएँ: ब्रांच मेनू से `Create Branch` चुनें, एक नाम दर्ज करें और पुष्टि करें।
- ब्रांच मर्ज करें: `Branch > Merge Branch` का उपयोग करें और वर्तमान में मर्ज करने के लिए ब्रांच चुनें।

#### पुल और फ़ेच
- **पुल**: तीन बिंदु मेनू में `Pull` विकल्प का उपयोग करके रिमोट रिपॉजिटरी से परिवर्तन सिंक करें।
- **फ़ेच**: `Fetch` का उपयोग करके मर्ज किए बिना रिमोट परिवर्तन प्राप्त करें।

#### संघर्षों को हल करना
- मर्जिंग या पुल करते समय, संघर्ष उत्पन्न हो सकते हैं। VS Code फ़ाइलों में संघर्षों को हाइलाइट करता है और एक इनलाइन संघर्ष समाधान इंटरफेस प्रदान करता है:
  - `Accept Current Change`, `Accept Incoming Change`, `Accept Both Changes` चुनें, या फ़ाइल को मैन्युअल रूप से संपादित करें।
  - हल की गई फ़ाइल को स्टेज और कमिट करें।

#### Git Lens एक्सटेंशन
उन्नत Git सुविधाओं के लिए, **GitLens** एक्सटेंशन इंस्टॉल करें:
- कमिट इतिहास, ब्लेम एनोटेशन और फ़ाइल परिवर्तन देखें।
- हाल की कमिट और स्टैश जैसे रिपॉजिटरी अंतर्दृष्टि तक पहुंचें।
- एक्सटेंशन व्यू (`Ctrl+Shift+X` या `Cmd+Shift+X`) के माध्यम से इंस्टॉल करें।

### 4. **उन्नत उपयोग**
#### परिवर्तन स्टैश करना
- अस्थायी रूप से अकमिट किए गए परिवर्तनों को सहेजें:
  - तीन बिंदु मेनू > Stash > Stash पर जाएँ।
  - बाद में उसी मेनू के माध्यम से स्टैश लागू करें या पॉप करें।
- अधूरे कार्य को कमिट किए बिना ब्रांच स्विच करने के लिए उपयोगी।

#### टर्मिनल में Git कमांड
- UI में सीधे समर्थित नहीं होने वाले कार्यों के लिए, एकीकृत टर्मिनल का उपयोग करें:
  ```bash
  git rebase <branch>
  git cherry-pick <commit>
  git log --oneline
  ```

#### सोर्स कंट्रोल को अनुकूलित करना
- **सेटिंग्स**: VS Code सेटिंग्स (`Ctrl+,` या `Cmd+,`) में सोर्स कंट्रोल व्यवहार समायोजित करें:
  - `git.autoRepositoryDetection`: स्वचालित Git रिपॉजिटरी पहचान सक्षम/अक्षम करें।
  - `git.enableSmartCommit`: जब कोई फ़ाइल स्टेज नहीं होती है तो सभी परिवर्तन कमिट करें।
- **SCM प्रदाता**: Mercurial या SVN जैसी अन्य VCS के लिए एक्सटेंशन इंस्टॉल करें।

#### GitHub एकीकरण
- PRs और इश्यू को सीधे VS Code में प्रबंधित करने के लिए **GitHub Pull Requests and Issues** एक्सटेंशन का उपयोग करें।
- GitHub रिपॉजिटरी से पुश/पुल करने के लिए अकाउंट्स मेनू (नीचे-बाएँ कोने) के माध्यम से GitHub के साथ प्रमाणित करें।

### 5. **नमूना वर्कफ़्लो: एक फीचर ब्रांच बनाना और पुश करना**
यहाँ VS Code में एक सामान्य Git वर्कफ़्लो का एक व्यावहारिक उदाहरण दिया गया है:


# VS Code में नमूना Git वर्कफ़्लो

## फीचर ब्रांच बनाने और पुश करने के चरण

1. **एक नई ब्रांच बनाएँ**:
   - सोर्स कंट्रोल व्यू में, स्टेटस बार में ब्रांच नाम पर क्लिक करें या तीन बिंदु मेनू > Branch > Create Branch का उपयोग करें।
   - ब्रांच का नाम दें, उदा., `feature/add-login`।
   - VS Code नई ब्रांच पर स्विच कर जाता है।

2. **परिवर्तन करें और स्टेज करें**:
   - फ़ाइलें संपादित करें (उदा., `src/Login.js` में एक लॉगिन कंपोनेंट जोड़ें)।
   - सोर्स कंट्रोल व्यू में, फ़ाइलें "परिवर्तन" के अंतर्गत दिखाई देती हैं।
   - परिवर्तनों को स्टेज करने के लिए `+` आइकन पर क्लिक करें या "Stage All Changes" चुनें।

3. **परिवर्तन कमिट करें**:
   - एक कमिट संदेश दर्ज करें, उदा., `Add login component`।
   - चेकमार्क पर क्लिक करें या `Ctrl+Enter` (Windows/Linux) या `Cmd+Enter` (macOS) दबाकर कमिट करें।

4. **ब्रांच पुश करें**:
   - यदि कोई रिमोट मौजूद नहीं है, तो एक जोड़ें:
     ```bash
     git remote add origin <repository-url>
     ```
   - ब्रांच पुश करें: तीन बिंदु मेनू > Push, या चलाएँ:
     ```bash
     git push -u origin feature/add-login
     ```

5. **एक पुल रिक्वेस्ट बनाएँ**:
   - यदि GitHub का उपयोग कर रहे हैं, तो ब्राउज़र में रिपॉजिटरी खोलें या PR बनाने के लिए GitHub Pull Requests एक्सटेंशन का उपयोग करें।
   - PR को `feature/add-login` ब्रांच से लिंक करें।

## सुझाव
- संघर्षों से बचने के लिए मुख्य ब्रांच से नियमित रूप से अपडेट पुल करें।
- बेहतर सहयोग के लिए वर्णनात्मक कमिट संदेशों का उपयोग करें।
- उन्नत कमिट इतिहास और ब्लेम व्यू के लिए GitLens इंस्टॉल करें।


### 6. **सामान्य समस्याओं का समाधान**
- **Git का पता नहीं चलना**: सुनिश्चित करें कि Git इंस्टॉल है और आपके सिस्टम PATH में जोड़ा गया है। इंस्टॉलेशन के बाद VS Code को पुनरारंभ करें।
- **प्रमाणीकरण त्रुटियां**: GitHub/GitLab के लिए SSH कुंजी कॉन्फ़िगर करें या एक पर्सनल एक्सेस टोकन का उपयोग करें। अकाउंट्स मेनू के माध्यम से प्रमाणित करें।
- **मर्ज संघर्ष**: इनलाइन संघर्ष समाधान उपकरणों का उपयोग करें या मैन्युअल रूप से फ़ाइलों को संपादित करें, फिर स्टेज और कमिट करें।
- **धीमा प्रदर्शन**: बड़ी परियोजनाओं के लिए `git.autoRepositoryDetection` अक्षम करें या अनुकूलित Git ऑपरेशन के लिए GitLens का उपयोग करें।

### 7. **सर्वोत्तम अभ्यास**
- **अक्सर कमिट करें**: प्रगति ट्रैक करने के लिए छोटे, तार्किक कमिट करें।
- **वर्णनात्मक संदेशों का उपयोग करें**: `<type>(<scope>): <description>` (उदा., `feat(auth): add login endpoint`) जैसे सम्मेलन का पालन करें।
- **नियमित रूप से पुल करें**: संघर्षों को कम करने के लिए रिमोट रिपॉजिटरी के साथ सिंक में रहें।
- **स्टैश का बैकअप लें**: स्टैश लागू करने से पहले, सुनिश्चित करें कि वे महत्वपूर्ण परिवर्तनों को अधिलेखित नहीं करेंगे।
- **एक्सटेंशन का लाभ उठाएं**: उन्नत कार्यक्षमता के लिए GitLens, Git History, या Git Graph का उपयोग करें।

- **GitHub Pull Requests एक्सटेंशन**: PRs और इश्यू प्रबंधित करने के लिए।

यह गाइड VS Code में सोर्स कंट्रोल के मूल सिद्धांतों और उन्नत सुविधाओं को कवर करती है, जिसमें Git पर ध्यान केंद्रित किया गया है। इन चरणों और सर्वोत्तम अभ्यासों का पालन करके, आप संपादक के भीतर ही वर्जन कंट्रोल वर्कफ़्लो को कुशलतापूर्वक प्रबंधित कर सकते हैं।
