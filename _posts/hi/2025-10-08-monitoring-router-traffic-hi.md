### 360 Wi-Fi 6 राउटर T7 पर HTTP अनुरोधों और TCP कनेक्शनों की निगरानी

360 T7 एक कस्टमाइज्ड Linux-आधारित फर्मवेयर चलाता है (बॉक्स से बाहर पूर्ण OpenWrt नहीं), इसलिए स्टॉक सेटअप में निगरानी क्षमताएं एक पूर्ण Linux शेल की तुलना में सीमित हैं। एडमिन अधिकारों (वेब UI के माध्यम से) के साथ, आप बेसिक लॉग तक पहुंच सकते हैं। रीयल-टाइम HTTP अनुरोध लॉगिंग या TCP कनेक्शन ट्रैकिंग जैसे गहन "हैकिंग" के लिए, सबसे अच्छा नैतिक तरीका OpenWrt फ्लैश करना है (जो आधिकारिक रूप से समर्थित है), SSH सक्षम करना और मानक Linux टूल्स का उपयोग करना है। **चेतावनी: कस्टम फर्मवेयर फ्लैश करने से वारंटी समाप्त हो जाती है और ब्रिक होने का जोखिम होता है—पहले बैकअप लें और गाइडों का सावधानीपूर्वक पालन करें। केवल अपने स्वयं के डिवाइस पर ऐसा करें।**

#### 1. **स्टॉक फर्मवेयर पर एडमिन अधिकार प्राप्त करना**
   - राउटर के Wi-Fi या Ethernet के माध्यम से कनेक्ट करें।
   - एक ब्राउज़र खोलें और `http://ihome.360.cn` या `http://192.168.0.1` (डिफ़ॉल्ट IP) पर जाएँ।
   - लॉगिन: डिफ़ॉल्ट यूज़रनेम `admin`, पासवर्ड राउटर लेबल पर छपा होता है (अक्सर `admin` या एक अद्वितीय स्ट्रिंग जैसे `360XXXXXX`—नीचे के स्टिकर को चेक करें)।
   - एक बार अंदर आने के बाद, बेसिक सिस्टम और ट्रैफ़िक लॉग के लिए **System > Log** या **Security > Log** पर नेविगेट करें। यह फ़ायरवॉल ब्लॉक, कनेक्शन और कुछ HTTP एक्टिविटी (जैसे ब्लॉक की गई साइटें या घुसपैठ) दिखाता है, लेकिन पूर्ण HTTP अनुरोध विवरण नहीं दिखाता।

   **वेब UI के माध्यम से बेसिक मॉनिटरिंग:**
   - **सिस्टम लॉग**: हाल की घटनाएँ देखें, जिसमें TCP कनेक्शन प्रयास और एरर शामिल हैं। लॉग एक्सपोर्ट करें (डिक्रिप्शन के लिए लेबल पासवर्ड की आवश्यकता हो सकती है)।
   - **ट्रैफ़िक स्टैट्स**: **Status > Network** या **Advanced > Traffic Monitor** के तहत, प्रति डिवाइस/IP बैंडविड्थ उपयोग देखें, लेकिन ग्रैन्युलर HTTP/TCP नहीं।
   - सीमाएँ: कोई रीयल-टाइम HTTP पेलोड इंस्पेक्शन नहीं; लॉग हाई-लेवल के होते हैं और बिना ऑथ के एक्सपोर्ट करने योग्य नहीं होते।

#### 2. **एडवांस्ड मॉनिटरिंग: शेल एक्सेस के लिए OpenWrt फ्लैश करें**
   360 T7 (MT7981B चिपसेट) OpenWrt 23.05+ स्नैपशॉट्स में सपोर्टेड है। फ्लैश करने से SSH के माध्यम से पूर्ण रूट शेल एक्सेस मिलती है, जहाँ आप पैकेट कैप्चर के लिए `tcpdump` और लॉग के लिए `logread` जैसे टूल्स चला सकते हैं।

   **OpenWrt फ्लैश करने के चरण (हाई-लेवल; आधिकारिक गाइड का उपयोग करें):**
   1. OpenWrt डाउनलोड से फैक्टरी इमेज डाउनलोड करें ("Qihoo 360T7 sysupgrade.bin" या फैक्टरी इमेज सर्च करें)।
   2. स्टॉक फर्मवेयर का बैकअप लें: वेब UI में, **System > Backup** पर जाएं और कॉन्फ़िग/फर्मवेयर डाउनलोड करें।
   3. वेब UI के माध्यम से अपलोड करें: **System > Firmware Upgrade**, .bin फ़ाइल चुनें, और अप्लाई करें (राउटर OpenWrt में रीबूट होगा)।
   4. पोस्ट-फ्लैश: `http://192.168.1.1` (LuCI इंटरफ़ेस) पर वेब UI एक्सेस करें, यूज़रनेम `root`, शुरुआत में कोई पासवर्ड नहीं—तुरंत SSH या UI के माध्यम से एक सेट करें।
   5. SSH सक्षम करें: यह पोर्ट 22 पर डिफ़ॉल्ट रूप से चालू होता है। अपने PC से कनेक्ट करें: `ssh root@192.168.1.1` (Windows पर PuTTY का उपयोग करें)।

   **जोखिम कम करना**: यदि अटक गए हैं, तो TFTP रिकवरी (बूट के दौरान रीसेट दबाए रखें) या सीरियल कंसोल (UART एडाप्टर की आवश्यकता होती है) का उपयोग करें।

#### 3. **OpenWrt पर निगरानी (SSH शेल के माध्यम से)**
   एक बार रूट के रूप में SSH करने के बाद, राउटर एक मिनिमल Linux सिस्टम की तरह काम करता है। यदि आवश्यक हो तो `opkg update && opkg install tcpdump` के माध्यम से पैकेज इंस्टॉल करें (अंतर्निहित स्टोरेज 128MB है, इसलिए इसे हल्का रखें)।

   - **सभी वर्तमान TCP कनेक्शन सूचीबद्ध करें (स्थैतिक दृश्य):**
     ```
     ss -tunap
     ```
     - स्थापित/सुनने वाले TCP सॉकेट, पोर्ट, प्रक्रियाएँ दिखाता है (उदा. `tcp ESTAB 0 0 192.168.1.1:80 192.168.1.100:54321 users:(("uhttpd",pid=1234,fd=3))`)।
     - रीयल-टाइम के लिए: `watch -n 1 'ss -tunap'`।

   - **रीयल-टाइम TCP ट्रैफ़िक कैप्चर:**
     यदि आवश्यक हो तो इंस्टॉल करें: `opkg update && opkg install tcpdump`.
     ```
     tcpdump -i any tcp -n -v
     ```
     - `-i any`: सभी इंटरफेस (LAN के लिए br-lan, WAN के लिए eth0.2)।
     - HTTP फ़िल्टर करें: `tcpdump -i any tcp port 80 -n -v -A` (`-A` HTTP हेडर/अनुरोधों के लिए ASCII पेलोड दिखाता है)।
     - फ़ाइल में सेव करें: `tcpdump -i any tcp -w /tmp/capture.pcap` (SCP के माध्यम से डाउनलोड करें: `scp root@192.168.1.1:/tmp/capture.pcap .`)।
     - HTTPS (पोर्ट 443) के लिए, ध्यान दें कि पेलोड एन्क्रिप्टेड होते हैं—विश्लेषण के लिए ऑफ़लाइन Wireshark का उपयोग करें।

   - **HTTP अनुरोध लॉग मॉनिटर करें:**
     - सिस्टम लॉग (यदि uhttpd का उपयोग कर रहे हैं तो वेब सर्वर शामिल है): `logread | grep uhttpd` या रीयल-टाइम के लिए `logread -f`।
     - विस्तृत HTTP लॉगिंग सक्षम करें: `/etc/config/uhttpd` (वेब सर्वर कॉन्फ़िग) एडिट करें, `option log_file '/var/log/uhttpd.log'` ऐड करें, फिर `/etc/init.d/uhttpd restart` करें। टेल: `tail -f /var/log/uhttpd.log` (GET/POST अनुरोध, IPs, URLs दिखाता है)।
     - पूर्ण नेटवर्क HTTP के लिए: ऊपर बताए अनुसार `tcpdump` का उपयोग करें, या `ngrep` इंस्टॉल करें (`opkg install ngrep`): `ngrep -d any 'GET|POST|HTTP' -O /tmp/http.pcap`।

   - **फ़ायरवॉल/कनेक्शन लॉग:**
     - नियमों पर लॉगिंग सक्षम करें: `uci set firewall.@rule[0].log='1'; uci commit; /etc/init.d/firewall restart`।
     - देखें: `logread | grep fw` (ड्रॉप/स्वीकृत TCP कनेक्शन दिखाता है)।

#### नैतिक "हैकिंग" के लिए टिप्स
- **सुरक्षा**: सेटअप के बाद, रूट पासवर्ड बदलें (`passwd`), की-ओनली के लिए `/etc/ssh/sshd_config` में पासवर्ड ऑथ डिसेबल करें, और फ़ायरवॉल SSH को केवल LAN तक सीमित करें।
- **प्रदर्शन**: T7 में 256MB RAM है—OOM को रोकने के लिए लॉन्ग-टर्म भारी कैप्चर से बचें।
- **स्टॉक पर वापस जाएं**: 360 की साइट से आधिकारिक फर्मवेयर डाउनलोड करें, OpenWrt के sysupgrade के माध्यम से अपलोड करें (यदि आवश्यक हो तो `--force` के साथ)।
- एक सुरक्षित वातावरण में परीक्षण करें; GUI विश्लेषण के लिए, .pcap को SCP के माध्यम से PC पर ले जाएं और Wireshark में खोलें।

यह व्यापक दृश्यता प्रदान करता है। यदि बेसिक्स के लिए स्टॉक UI पर्याप्त है, तो फ्लैशिंग से बचने के लिए वहीं रहें।
