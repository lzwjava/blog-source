`MultiMark.java` फ़ाइल को सीधे चलाने के लिए, जैसे आप एक Python स्क्रिप्ट चलाते हैं, आपके पास कुछ विकल्प हैं। चूंकि आप एक Unix-जैसी प्रणाली पर हैं (जैसा कि `~/projects` पथ और `ls` कमांड से संकेत मिलता है), मैं Java प्रोग्राम को चलाने को यथासंभव सहज बनाने के लिए कदम प्रदान करूंगा। यहां दो मुख्य दृष्टिकोण हैं:

---

### **विकल्प 1: `java` कमांड के साथ `MultiMark.java` को सीधे चलाएं (Java 11+)**
Java 11 के बाद से, आप एकल Java स्रोत फ़ाइल को बिना स्पष्ट रूप से कंपाइल किए सीधे चला सकते हैं, जो इसे Python स्क्रिप्ट चलाने के करीब महसूस कराता है।

1. **सुनिश्चित करें कि Java इंस्टॉल है**
   जांचें कि क्या आपके पास Java 11 या बाद का संस्करण है:
   ```bash
   java --version
   ```
   यदि Java इंस्टॉल नहीं है या कोई पुराना संस्करण है, तो एक संगत संस्करण इंस्टॉल करें (उदाहरण के लिए, OpenJDK):
   ```bash
   sudo apt update
   sudo apt install openjdk-17-jdk
   ```

2. **Java फ़ाइल को सीधे चलाएं**
   उस डायरेक्टरी पर नेविगेट करें जिसमें `MultiMark.java` है:
   ```bash
   cd ~/projects/lzwjava.github.io/scripts/benchmark
   ```
   Java फ़ाइल को सीधे इसका उपयोग करके चलाएं:
   ```bash
   java MultiMark.java
   ```
   यह कमांड `MultiMark.java` फ़ाइल को एक चरण में कंपाइल और एक्ज़िक्यूट करती है। आउटपुट (जैसे, CPU cores, thread counts और times) टर्मिनल में दिखाई देगा।

### **विकल्प 2: सुविधा के लिए एक Shell स्क्रिप्ट बनाएं**
`MultiMark.java` को चलाना Python स्क्रिप्ट (जैसे, `./multimark`) चलाने जैसा और भी बनाने के लिए, आप एक shell स्क्रिप्ट रैपर बना सकते हैं।

1. **एक Shell स्क्रिप्ट बनाएं**
   `scripts/benchmark` डायरेक्टरी में `multimark` नाम की एक फ़ाइल बनाएं (कोई एक्सटेंशन नहीं):
   ```bash
   nano ~/projects/lzwjava.github.io/scripts/benchmark/multimark
   ```
   निम्नलिखित सामग्री जोड़ें:
   ```bash
   #!/bin/bash
   java MultiMark.java
   ```
   सेव करें और बाहर निकलें (`Ctrl+O`, `Enter`, `Ctrl+X` nano में)।

2. **स्क्रिप्ट को एक्ज़िक्यूटेबल बनाएं**
   एक्ज़िक्यूट परमिशन सेट करें:
   ```bash
   chmod +x multimark
   ```

3. **स्क्रिप्ट चलाएं**
   अब आप प्रोग्राम को Python स्क्रिप्ट की तरह चला सकते हैं:
   ```bash
   ./multimark
   ```
   यह हुड के नीचे `java MultiMark.java` को एक्ज़िक्यूट करेगा।

---

### **विकल्प 3: अलग से कंपाइल और रन करें (पारंपरिक दृष्टिकोण)**
यदि आप Java फ़ाइल को स्पष्ट रूप से कंपाइल करना पसंद करते हैं या करने की आवश्यकता है (उदाहरण के लिए, पुराने Java संस्करणों के लिए या कंपाइलेशन एरर की जांच के लिए), इन चरणों का पालन करें:

1. **Java फ़ाइल को कंपाइल करें**
   डायरेक्टरी पर नेविगेट करें:
   ```bash
   cd ~/projects/lzwjava.github.io/scripts/benchmark
   ```
   Java स्रोत फ़ाइल को कंपाइल करें:
   ```bash
   javac MultiMark.java
   ```
   यह एक `MultiMark.class` फ़ाइल उत्पन्न करता है।

2. **कंपाइल किए गए प्रोग्राम को चलाएं**
   कंपाइल की गई क्लास को एक्ज़िक्यूट करें:
   ```bash
   java MultiMark
   ```
   नोट: `MultiMark` का उपयोग करें (`.java` के बिना) क्योंकि आप कंपाइल की गई क्लास फ़ाइल चला रहे हैं।

### **इसे Python-जैसा बनाना**
अनुभव को Python स्क्रिप्ट चलाने जितना करीबी बनाने के लिए, **विकल्प 1** (सीधा `java MultiMark.java`) सबसे सरल है, क्योंकि यह मैन्युअल कंपाइलेशन को छोड़ देता है। **विकल्प 2** (shell स्क्रिप्ट) आदर्श है यदि आप एक एकल कमांड चाहते हैं जैसे `./multimark` जो `./script.py` चलाने जैसा महसूस हो।

### **नोट्स**
- **Java संस्करण**: कोड मानक Java लाइब्रेरी (`java.util.*` और `System.nanoTime`) का उपयोग करता है, इसलिए यह Java 8 या बाद के संस्करण के साथ काम करना चाहिए। सीधे-चलाने वाली सुविधा (`java MultiMark.java`) के लिए Java 11+ की आवश्यकता होती है।
- **परफॉर्मेंस**: `MultiMark.java` प्रोग्राम मल्टीपल थ्रेड्स के साथ सॉर्टिंग का बेंचमार्क करता है। इसे सीधे या स्क्रिप्ट के माध्यम से चलाने से इसके प्रदर्शन पर कोई प्रभाव नहीं पड़ेगा, लेकिन सुनिश्चित करें कि आपके सिस्टम के पास परीक्षण किए गए थ्रेड काउंट (64 थ्रेड्स तक) के लिए पर्याप्त संसाधन (CPU/मेमोरी) हैं।
- **डायरेक्टरी संदर्भ**: चूंकि आपने `ls scripts/benchmark/` दिखाया, मैंने मान लिया कि `MultiMark.java` उस डायरेक्टरी में है। यदि यह कहीं और है तो पथ समायोजित करें।
- **एरर हैंडलिंग**: यदि आपको कोई एरर मिलते हैं (जैसे, `java: command not found`), सुनिश्चित करें कि Java इंस्टॉल है और आपके `PATH` में जोड़ा गया है। यदि प्रोग्राम फेल हो जाता है (जैसे, मेमोरी या थ्रेड सीमा के कारण), विवरण के लिए स्टैक ट्रेस की जांच करें।

यदि आप इसे अपनी Python स्क्रिप्ट्स (`executormark.py`, `multimark.py`, `threadmark.py`) के साथ एक वर्कफ़्लो में एकीकृत करना चाहते हैं, तो मुझे बताएं, और मैं उन्हें संयोजित करने के तरीके सुझा सकता हूं (जैसे, Python से Java प्रोग्राम को कॉल करना)।
