Java में Lombok का उपयोग करने के लिए, आपको इसे अपने प्रोजेक्ट में सेट अप करना होगा ताकि बॉयलरप्लेट कोड, जैसे कि गेटर्स, सेटर्स और कंस्ट्रक्टर्स, को कम करके अपने कोड को सरल बनाया जा सके। यहां बताया गया है कि आप इसे चरण दर चरण कैसे कर सकते हैं:

सबसे पहले, आपको Lombok को एक डिपेंडेंसी के रूप में जोड़ना होगा। यदि आप Maven का उपयोग कर रहे हैं, तो इसे अपनी `pom.xml` में शामिल करें:

```xml
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <version>1.18.32</version> <!-- नवीनतम संस्करण के लिए जांच करें -->
    <scope>provided</scope>
</dependency>
```

Gradle के लिए, इसे अपने `build.gradle` में जोड़ें:

```gradle
provided 'org.projectlombok:lombok:1.18.32'
// एनोटेशन प्रोसेसिंग सुनिश्चित करने के लिए इसे भी जोड़ें
annotationProcessor 'org.projectlombok:lombok:1.18.32'
```

अगला, यदि आप IntelliJ IDEA या Eclipse जैसे IDE का उपयोग कर रहे हैं, तो आपको एनोटेशन प्रोसेसिंग सक्षम करनी होगी और Lombok प्लगइन इंस्टॉल करना होगा:
- **IntelliJ**: `File > Settings > Plugins` पर जाएं, "Lombok" खोजें, इसे इंस्टॉल करें, और IDE को रीस्टार्ट करें। फिर सुनिश्चित करें कि `Settings > Build, Execution, Deployment > Compiler > Annotation Processors` के अंतर्गत `Enable annotation processing` चेक किया गया है।
- **Eclipse**: आधिकारिक साइट (projectlombok.org) से Lombok JAR डाउनलोड करें, इसे `java -jar lombok.jar` चला कर रन करें, और इंस्टॉलर के निर्देशों का पालन करते हुए इसे अपने Eclipse इंस्टॉलेशन की ओर पॉइंट करें।

अब, आप अपने Java कोड में Lombok एनोटेशन का उपयोग शुरू कर सकते हैं। यहां एक सरल उदाहरण दिया गया है:

```java
import lombok.Getter;
import lombok.Setter;
import lombok.NoArgsConstructor;
import lombok.AllArgsConstructor;

@Getter
@Setter
@NoArgsConstructor
@AllArgsConstructor
public class Person {
    private String name;
    private int age;
}
```

इस उदाहरण में:
- `@Getter` और `@Setter` `name` और `age` के लिए स्वचालित रूप से गेटर और सेटर मेथड जेनरेट करते हैं।
- `@NoArgsConstructor` एक बिना आर्गुमेंट वाला कंस्ट्रक्टर बनाता है।
- `@AllArgsConstructor` सभी फ़ील्ड्स को पैरामीटर के रूप में लेने वाला एक कंस्ट्रक्टर बनाता है।

जब आप इसे कंपाइल या रन करते हैं, तो Lombok इन एनोटेशन्स को प्रोसेस करता है और आपके लिए बॉयलरप्लेट कोड जोड़ देता है। आप इसे इस तरह टेस्ट कर सकते हैं:

```java
public class Main {
    public static void main(String[] args) {
        Person person = new Person("Alice", 30);
        System.out.println(person.getName()); // आउटपुट: Alice
        person.setAge(31);
        System.out.println(person.getAge()); // आउटपुट: 31
    }
}
```

कुछ और लोकप्रिय एनोटेशन जो आपके लिए उपयोगी हो सकते हैं:
- `@Data`: यह `@Getter`, `@Setter`, `@ToString`, `@EqualsAndHashCode`, और आवश्यक फ़ील्ड्स (जैसे final) के लिए एक कंस्ट्रक्टर के लिए एक शॉर्टकट है।
- `@Builder`: आपको ऑब्जेक्ट क्रिएशन के लिए एक बिल्डर पैटर्न देता है।
- `@Slf4j`: आपकी क्लास में एक लॉगर (जैसे, SLF4J से) जोड़ता है।

उदाहरण के लिए, `@Data` और `@Builder` के साथ:

```java
import lombok.Data;
import lombok.Builder;

@Data
@Builder
public class User {
    private String username;
    private String email;
}

// उपयोग
public class Main {
    public static void main(String[] args) {
        User user = User.builder()
                        .username("john_doe")
                        .email("john@example.com")
                        .build();
        System.out.println(user); // आउटपुट: User(username=john_doe, email=john@example.com)
    }
}
```

ध्यान रखें:
- Maven Central या आधिकारिक साइट पर Lombok के नवीनतम संस्करण की हमेशा जांच करें।
- यदि आप एक टीम के साथ काम कर रहे हैं, तो सुनिश्चित करें कि सभी के पास उनके IDE में Lombok सेट अप हो ताकि जेनरेट किए गए कोड के सोर्स में दिखाई न देने से होने वाली उलझन से बचा जा सके।
