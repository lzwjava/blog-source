अपने Raspberry Pi Pico पर MicroPython फ्लैश करने के बाद (जैसा कि बताया गया है `.uf2` फाइल का उपयोग करके), आप Thonny IDE जैसे टूल का उपयोग करके एक साधारण MicroPython प्रोग्राम अपलोड और रन कर सकते हैं। यह शुरुआती लोगों के लिए अनुकूल है और MicroPython डेवलपमेंट के लिए व्यापक रूप से उपयोग किया जाता है। नीचे MicroPython सेट अप करने और अपने Pico पर एक साधारण प्रोग्राम अपलोड करने के चरण दिए गए हैं।

---

### पूर्वापेक्षाएँ
1.  **MicroPython फ्लैश किया हुआ**: आपने पहले ही `RPI_PICO-20250415-v1.25.0.uf2` को `RPI-RP2` ड्राइव पर कॉपी कर दिया है, और Pico रीस्टार्ट हो गया है (`RPI-RP2` ड्राइव अब दिखनी बंद हो जानी चाहिए)।
2.  **USB कनेक्शन**: Pico डेटा ट्रांसफर सपोर्ट करने वाले USB केबल के माध्यम से आपके कंप्यूटर से जुड़ा हुआ है।
3.  **Thonny IDE**: अगर आपने पहले से इंस्टॉल नहीं किया है, तो Thonny इंस्टॉल करें:
    - **Linux**: अपने पैकेज मैनेजर का उपयोग करके Thonny इंस्टॉल करें या [thonny.org](https://thonny.org) से डाउनलोड करें।
      ```bash
      sudo apt update
      sudo apt install thonny
      ```
    - वैकल्पिक रूप से, `pip` का उपयोग करें:
      ```bash
      pip install thonny
      ```
    - Windows/macOS के लिए, [thonny.org](https://thonny.org) से डाउनलोड करके इंस्टॉल करें।

### एक साधारण MicroPython प्रोग्राम अपलोड करने के लिए चरण-दर-चरण मार्गदर्शिका

1.  **Pico को कनेक्ट करें और Thonny खोलें**:
    - अपने Pico को कंप्यूटर के USB पोर्ट में लगाएं।
    - Thonny IDE खोलें।

2.  **MicroPython के लिए Thonny कॉन्फ़िगर करें**:
    - Thonny में, **Tools > Options > Interpreter** (या **Run > Select interpreter**) पर जाएं।
    - इंटरप्रेटर ड्रॉपडाउन से **MicroPython (Raspberry Pi Pico)** चुनें।
    - अगर Pico का सीरियल पोर्ट (जैसे, Linux पर `/dev/ttyACM0`) अपने आप नहीं दिखता है:
      - ड्रॉपडाउन में उपलब्ध पोर्ट्स की जांच करें या Pico के पोर्ट (आमतौर पर `/dev/ttyACM0` या इसी तरह) की पहचान करने के लिए टर्मिनल में `ls /dev/tty*` चलाएं।
      - सही पोर्ट मैन्युअली चुनें।
    - सेव करने के लिए **OK** पर क्लिक करें।

3.  **सत्यापित करें कि MicroPython चल रहा है**:
    - Thonny के **Shell** (नीचे का पैनल) में, आपको MicroPython REPL प्रॉम्प्ट दिखना चाहिए:
      ```
      >>>
      ```
    - इसे एक साधारण कमांड टाइप करके टेस्ट करें, उदाहरण के लिए:
      ```python
      print("Hello, Pico!")
      ```
      एंटर दबाएं, और आपको आउटपुट Shell में दिखना चाहिए।

4.  **एक साधारण MicroPython प्रोग्राम लिखें**:
    - Thonny के मुख्य एडिटर में, एक नई फाइल बनाएं और एक साधारण प्रोग्राम लिखें। उदाहरण के लिए, Pico के ऑनबोर्ड LED को ब्लिंक करने वाला प्रोग्राम (Pico के लिए GPIO 25 पर, या Pico W के लिए "LED"):
      ```python
      from machine import Pin
      import time

      # ऑनबोर्ड LED को इनिशियलाइज़ करें
      led = Pin(25, Pin.OUT)  # Pico W के लिए 25 के बजाय "LED" का उपयोग करें

      # LED को ब्लिंक करें
      while True:
          led.on()           # LED ऑन करें
          time.sleep(0.5)    # 0.5 सेकंड प्रतीक्षा करें
          led.off()          # LED ऑफ करें
          time.sleep(0.5)    # 0.5 सेकंड प्रतीक्षा करें
      ```
    - नोट: अगर Pico W का उपयोग कर रहे हैं, तो `Pin(25, Pin.OUT)` को `Pin("LED", Pin.OUT)` से बदलें।

5.  **प्रोग्राम को Pico पर सेव करें**:
    - **File > Save As** पर क्लिक करें।
    - डायलॉग में, डेस्टिनेशन के रूप में **Raspberry Pi Pico** चुनें (आपके कंप्यूटर को नहीं)।
    - फाइल का नाम `main.py` रखें (MicroPython बूट पर `main.py` को स्वचालित रूप से रन करता है) या कोई दूसरा नाम जैसे `blink.py`।
    - फाइल को Pico की फाइलसिस्टम पर सेव करने के लिए **OK** पर क्लिक करें।

6.  **प्रोग्राम रन करें**:
    - प्रोग्राम को एक्ज़िक्यूट करने के लिए Thonny में हरे **Run** बटन (या **F5** दबाएं) पर क्लिक करें।
    - वैकल्पिक रूप से, अगर आपने इसे `main.py` के रूप में सेव किया है, तो Pico को रीसेट करें (अनप्लग और री-प्लग करें, या अगर उपलब्ध हो तो RESET बटन दबाएं), और प्रोग्राम स्वचालित रूप से रन हो जाएगा।
    - आपको ऑनबोर्ड LED हर 0.5 सेकंड में ब्लिंक होती दिखनी चाहिए।

7.  **प्रोग्राम रोकें** (अगर ज़रूरत हो):
    - प्रोग्राम को रोकने के लिए, चल रही स्क्रिप्ट को इंटरप्ट करने के लिए Thonny की Shell में **Ctrl+C** दबाएं।
    - `main.py` को ऑटो-रन से हटाने के लिए, इसे Pico से डिलीट करें:
      - Thonny में, **View > Files** पर जाएं, Pico की फाइलसिस्टम को सेलेक्ट करें, `main.py` पर राइट-क्लिक करें, और **Delete** चुनें।

### टेस्टिंग और ट्रबलशूटिंग
-   **REPL प्रॉम्प्ट नहीं दिख रहा**: अगर Thonny MicroPython REPL नहीं दिखा रहा:
    - सुनिश्चित करें कि इंटरप्रेटर सेटिंग्स में सही पोर्ट चुना गया है।
    - सत्यापित करें कि MicroPython सही ढंग से फ्लैश हुआ है। अगर नहीं, तो पहले बताए अनुसार `.uf2` फाइल को दोबारा फ्लैश करें।
    - USB केबल (डेटा सपोर्ट करनी चाहिए) की जांच करें और एक अलग पोर्ट आज़माएं।
-   **LED ब्लिंक नहीं हो रही**: सुनिश्चित करें कि सही GPIO पिन का उपयोग किया गया है (Pico के लिए `25`, Pico W के लिए `"LED"`)। अगर फिर भी काम नहीं करती है, तो एक साधारण REPL कमांड से LED को टेस्ट करें:
    ```python
    from machine import Pin
    led = Pin(25, Pin.OUT)
    led.on()
    ```
-   **फाइल सेव नहीं हो रही**: सुनिश्चित करें कि Thonny Pico से कनेक्टेड है और इंटरप्रेटर MicroPython (Raspberry Pi Pico) पर सेट है।

### अतिरिक्त सुझाव
-   **अन्य टूल्स का उपयोग**: Thonny के अलावा, आप Pico पर फाइलों को मैनेज करने के लिए `rshell` या `ampy` का उपयोग कर सकते हैं:
    ```bash
    pip install rshell
    rshell --port /dev/ttyACM0
    cp blink.py /pyboard/main.py
    ```
-   **अधिक उदाहरण प्रोग्राम**:
    - एक बटन प्रेस पढ़ें (उदाहरण के लिए, GPIO 14 पर):
      ```python
      from machine import Pin
      button = Pin(14, Pin.IN, Pin.PULL_UP)
      while True:
          print(button.value())
      ```
    - GPIO पिन नंबर एडजस्ट करके एक एक्सटर्नल LED या अन्य पेरिफेरल्स को कंट्रोल करें।
-   **ऑटो-रन**: किसी फाइल को `main.py` के रूप में सेव करने से वह बूट पर स्वचालित रूप से रन होती है। सेटअप कोड के लिए जो `main.py` से पहले रन हो, उसके लिए `boot.py` का उपयोग करें।

अगर आपको कोई विशिष्ट समस्या आती है (जैसे, पोर्ट डिटेक्शन, Thonny में एरर, या हार्डवेयर समस्याएं), या कोई अलग प्रोग्राम आज़माना चाहते हैं, तो मुझे बताएं, और मैं आपको आगे मार्गदर्शन दूंगा!
