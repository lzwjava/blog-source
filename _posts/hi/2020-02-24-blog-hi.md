# lzwjava.github.io

मेरे व्यक्तिगत ब्लॉग में आपका स्वागत है! यहां, मैं विभिन्न विषयों पर अपने विचारों, अनुभवों और ज्ञान को साझा करता हूँ।

## मुख्य सुधार

इस ब्लॉग में एक साधारण Jekyll ब्लॉग के मुकाबले कई सुधार शामिल हैं, जिसमें jekyll-theme-cayman का उपयोग किया जाता है:

*   **AI-Powered Translation:** प्रौद्योगिकी में उन्नत भाषा मॉडल का उपयोग किया जाता है, जिससे सही और संदर्भ-संबंधित अनुवाद मिलते हैं, जिससे सामग्री के अनुप्रयोग को एक वैश्विक दर्शक तक पहुंचाया जाता है।
*   **XeLaTeX PDF Generation:** XeLaTeX का एकीकरण किया जाता है ताकि उच्च गुणवत्ता वाले, छापने के लिए तैयार PDF बनाएं, जो ऑफ़लाइन पढ़ने और साझा करने के लिए हैं।
*   **Google Cloud Text-to-Speech:** Google Cloud की Text-to-Speech सेवा का उपयोग करते हैं ताकि पोस्टों की ऑडियो वर्सन बनाएं और दृष्टि बाधित उपयोगकर्ताओं और ऑडियो सामग्री को पसंद करने वाले लोगों के लिए पहुंच में सुधार करें।
*   **Enhanced CSS Styling:** एक सुसंस्कृत और कस्टम CSS डिजाइन के साथ, जो एक दृश्य रूप से आकर्षक और उपयोगकर्ता-फ्रेंडली अनुभव प्रदान करता है।
*   **MathJax Support:** MathJax का उपयोग जटिल गणितीय अभिव्यक्तियों और समीकरणों को रेंडर करने के लिए करता है, जिससे तकनीकी सामग्री को अधिक पहुंच्य बना देता है।
*   **Night Mode:** नाइट मोड विकल्प शामिल है ताकि आँखों की थकान कम हो सके और कम रोशनी वातावरण में पढ़ने की गुना बढ़ा सके।
*   **Flexible Post Selection:** विभिन्न पोस्ट चुनाव विकल्पों के साथ, जैसे कि श्रेणी या टैग के अनुसार फ़िल्टर, नाविगेशन में सुधार कर सकते हैं।
*   **Regular Updates:** ब्लॉग के लाइब्रेरी और निर्भरताओं को उपयुक्त प्रदर्शन और सुरक्षा के लिए अपडेट रखे.
*   **`awesome-cv` Integration:** ब्लॉग से `awesome-cv` का उपयोग करके पेशेवर सीवी सृजित करता है।
*   **RSS Feed Support:** `feed.xml` के माध्यम से RSS फीड प्रदान करता है, जिससे उपयोगकर्ता ब्लॉग के लिए सब्सक्राइब कर सकते हैं।
*   **Bilingual Content:** दोनों चीनी और अंग्रेजी सामग्री का समर्थन करता है ताकि विविध दर्शकों को सन्तुष्ट किया जा सके।
*   **GitHub Workflow Automation:** GitHub Actions के साथ स्वचालित निर्माण, परीक्षण और डिजाइन, एक अधिक घनी विकास प्रक्रिया सुनिश्चित करता है।
*   **Automatic Translation Workflow:** GitHub Actions का उपयोग करते हुए नए या अपडेट किए गए पोस्टों को कई भाषाओं में अनुवाद करता है।
*   **EPUB Support:** EPUB के लिए Markdown को परिवर्तित करते हैं, जैसी कि ebook पाठक के लिए.


## शुरुआत करने के लिए

एक स्थानीय Jekyll वातावरण सेटअप करने के लिए, इन चरणों का पालन करें:

```shell
gem install jekyll bundler

jekyll new myblog

cd myblog

bundle install

bundle exec jekyll serve

bundle exec jekyll serve --draft

bundle add webrick

jekyll serve
```

## Markdown Styling

सिंटैक्स हाइलाईटिंग को कस्टमाइज़ करने के लिए, `rougify` का उपयोग करें:

```shell
rougify help style

rougify style github > _sass/syntax.css

rougify style gruvbox.dark > _sass/syntax.css

rougify style base16.monokai.dark > _sass/syntax.css
```

## Audio और PDF पाइपलाइन

ऑडीओ और PDF फाइलें बनाने के लिए, इन आदेशों का उपयोग करें:

```bash
conda activate google-cloud-env

python scripts/audio_pipeline.py --task posts --n 10

python scripts/audio_pipeline.py --task pages

python scripts/audio_pipeline.py --task notes

python scripts/audio_pipeline.py --task notes --n 100

python scripts/pdf_pipeline.py --task pages

python scripts/pdf_pipeline.py --task posts --n 10

python scripts/pdf_pipeline.py --task notes

python scripts/generate_notes_link.py

python scripts/auto-ss-config/upload_configs.py

python scripts/epub_book.py notes

python scripts/epub_book.py _posts/en

python scripts/epub_book.py _posts/ja
```

## स्क्रिप्ट्स

एक नया ड्राफ्ट पोस्ट बनाने के लिए उपयोग करें:

```
python scripts/draft.py <name>
```

## सामग्री दिशा निर्देश

*   `-en`: इंग्लिश पोस्टों को इंगित करता है.
*   `-zh`: चीनी पोस्टों को इंगित करता है।

`_posts` और `pages` डिरेक्टरी दोनों `-en` और `-zh` सफ़ीक्स का समर्थन करती हैं।

`notes` डिरेक्टरी मुख्यत: व्यक्तिगत सीखने के उद्देश्यों के लिए इंग्लिश सामग्री को रखता है और अनुवाद की आवश्यकता नहीं होती है।
