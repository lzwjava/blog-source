### जेनकिंस में एक जॉब कॉन्फ़िगर करना

जेनकिंस विभिन्न प्रकार की जॉब्स (जिन्हें प्रोजेक्ट भी कहा जाता है) को सपोर्ट करता है, जिनमें सबसे आम हैं **फ्रीस्टाइल प्रोजेक्ट्स** (सरल बिल्ड्स के लिए) और **पाइपलाइन प्रोजेक्ट्स** (अधिक जटिल, स्क्रिप्टेड वर्कफ़्लो के लिए Jenkinsfile का उपयोग करके)। मैं नीचे दोनों के लिए चरणों की रूपरेखा दूंगा। यह मानकर चलता है कि आपने जेनकिंस इंस्टॉल कर लिया है और वह चल रहा है (जैसे, Docker के माध्यम से या किसी सर्वर पर)। जेनकिंस डैशबोर्ड पर `http://localhost:8080` (या आपके सर्वर का URL) पर एक्सेस करें और लॉग इन करें।

#### फ्रीस्टाइल प्रोजेक्ट बनाने और कॉन्फ़िगर करने के चरण
फ्रीस्टाइल प्रोजेक्ट सीधे-सादे होते हैं और चरणों को कॉन्फ़िगर करने के लिए GUI का उपयोग करते हैं। ये शुरुआती लोगों या कोड बनाने और टेस्ट करने जैसे सरल कार्यों के लिए बहुत अच्छे हैं।

1.  **एक नई जॉब बनाएँ**:
    *   जेनकिंस डैशबोर्ड से, बाएँ साइडबार पर **New Item** पर क्लिक करें।
    *   अपनी जॉब के लिए एक नाम दर्ज करें (उदाहरण के लिए, "MyFirstBuild")।
    *   **Freestyle project** चुनें और **OK** पर क्लिक करें।

2.  **सामान्य सेटिंग्स**:
    *   जॉब के लिए एक विवरण जोड़ें।
    *   वैकल्पिक रूप से, पुराने बिल्ड्स को हटाने (उदाहरण के लिए, केवल अंतिम 10 बिल्ड्स रखें) या पैरामीटर्स जोड़ने (उदाहरण के लिए, बिल्ड के दौरान यूजर इनपुट के लिए स्ट्रिंग या चॉइस पैरामीटर्स) जैसी सुविधाओं को सक्षम करें।

3.  **सोर्स कोड मैनेजमेंट**:
    *   अपना SCM टूल चुनें, जैसे Git।
    *   रिपॉजिटरी URL दर्ज करें (उदाहरण के लिए, एक GitHub रिपो)।
    *   आवश्यकता होने पर क्रेडेंशियल्स जोड़ें (उदाहरण के लिए, यूजरनेम/पासवर्ड या SSH कुंजी)।
    *   बिल्ड करने के लिए ब्रांच निर्दिष्ट करें (उदाहरण के लिए, `*/main`)।

4.  **बिल्ड ट्रिगर्स**:
    *   चुनें कि जॉब कैसे शुरू होगी, जैसे कि:
        *   **Build periodically** (उदाहरण के लिए, cron सिंटैक्स जैसे `H/5 * * * *` हर 5 मिनट में)।
        *   **Poll SCM** परिवर्तनों की जांच के लिए।
        *   **GitHub hook trigger** GitHub से वेबहुक के लिए।
        *   **Build after other projects** जॉब्स को चेन करने के लिए।

5.  **बिल्ड एनवायरनमेंट**:
    *   एक क्लीन स्लेट के लिए **Delete workspace before build starts** जैसे विकल्प चेक करें।
    *   कंसोल आउटपुट में टाइमस्टैम्प जोड़ें या एनवायरनमेंट वेरिएबल्स सेट करें।

6.  **बिल्ड स्टेप्स**:
    *   **Add build step** पर क्लिक करें और एक्शन चुनें जैसे:
        *   **Execute shell** (Linux/Mac के लिए: उदाहरण के लिए, `echo "Hello World"` या स्क्रिप्ट्स चलाएँ)।
        *   **Invoke top-level Maven targets** Java बिल्ड्स के लिए।
        *   **Execute Windows batch command** Windows के लिए।
    *   आप कई स्टेप्स जोड़ सकते हैं जो क्रमिक रूप से चलते हैं।

7.  **पोस्ट-बिल्ड एक्शन्स**:
    *   एक्शन्स जोड़ें जैसे:
        *   **Archive the artifacts** (उदाहरण के लिए, JAR फ़ाइलें सहेजें)।
        *   **Publish JUnit test result report**।
        *   **Send email notifications** सफलता/विफलता पर।
        *   **Trigger another project**।

8.  **सेव और रन**:
    *   **Save** पर क्लिक करें।
    *   जॉब पेज पर वापस, इसे टेस्ट करने के लिए **Build Now** पर क्लिक करें।
    *   विवरणों के लिए कंसोल आउटपुट देखें।

#### पाइपलाइन प्रोजेक्ट बनाने और कॉन्फ़िगर करने के चरण
पाइपलाइन को कोड के रूप में परिभाषित किया जाता है (डिक्लेरेटिव या स्क्रिप्टेड) और CI/CD वर्कफ़्लो के लिए अधिक लचीली होती हैं।

1.  **एक नई जॉब बनाएँ**:
    *   डैशबोर्ड से, **New Item** पर क्लिक करें।
    *   एक नाम दर्ज करें और **Pipeline** चुनें, फिर **OK** करें।

2.  **सामान्य सेटिंग्स**:
    *   फ्रीस्टाइल के समान: विवरण जोड़ें, पैरामीटर्स, आदि।

3.  **बिल्ड ट्रिगर्स**:
    *   फ्रीस्टाइल के समान विकल्प (उदाहरण के लिए, वेबहुक, शेड्यूल)।

4.  **पाइपलाइन डेफिनिशन**:
    *   इनलाइन कोड के लिए **Pipeline script** चुनें या किसी रिपो से पुल करने के लिए **Pipeline script from SCM** चुनें (उदाहरण के लिए, Git में एक `Jenkinsfile`)।
    *   उदाहरण डिक्लेरेटिव पाइपलाइन स्क्रिप्ट:
        ```
        pipeline {
            agent any
            stages {
                stage('Build') {
                    steps {
                        echo 'Building...'
                        sh 'mvn clean install'  // उदाहरण Maven बिल्ड
                    }
                }
                stage('Test') {
                    steps {
                        echo 'Testing...'
                        sh 'mvn test'
                    }
                }
                stage('Deploy') {
                    steps {
                        echo 'Deploying...'
                    }
                }
            }
            post {
                always {
                    echo 'This runs always'
                }
            }
        }
        ```
    *   यह स्टेप्स के साथ स्टेजेस (बिल्ड, टेस्ट, डिप्लॉय) को परिभाषित करता है।

5.  **सेव और रन**:
    *   जॉब को सेव करें।
    *   इसे बिल्ड करें और स्टेज प्रोग्रेस के लिए पाइपलाइन व्यू मॉनिटर करें।

जेनकिंस में प्रत्येक सेक्शन में कई विकल्प हैं, इसलिए अपनी आवश्यकताओं के आधार पर एक्सप्लोर करें (उदाहरण के लिए, सुरक्षा के लिए, क्रेडेंशियल्स जोड़ें; समानांतरता के लिए, एजेंट्स/नोड्स का उपयोग करें)। यदि आप नए हैं, तो फ्रीस्टाइल से शुरुआत करें और स्केलेबिलिटी के लिए पाइपलाइन पर जाएं।

### जेनकिंस के साथ सॉफ़्टवेयर इंटीग्रेशन और सहयोग

जेनकिंस **प्लगइन्स** (2,000 से अधिक उपलब्ध) के माध्यम से अत्यधिक एक्स्टेंसिबल है, जो इसे DevOps इकोसिस्टम में वर्चुअली किसी भी टूल के साथ इंटीग्रेट होने की अनुमति देता है। ये इंटीग्रेशन बिल्ड्स, डिप्लॉयमेंट्स, टेस्टिंग, नोटिफिकेशन्स और बहुत कुछ ट्रिगर करने में सक्षम बनाते हैं। प्लगइन्स **Manage Jenkins > Manage Plugins** के माध्यम से इंस्टॉल किए जा सकते हैं।

#### श्रेणी के अनुसार सामान्य इंटीग्रेशन
*   **वर्जन कंट्रोल**: Git, GitHub, GitLab, Bitbucket, SVN – कोड पुल करने और कमिट्स/पुश इवेंट्स पर वेबहुक के माध्यम से बिल्ड्स ट्रिगर करने के लिए।
*   **कंटेनराइजेशन और ऑर्केस्ट्रेशन**: Docker (इमेज बनाना/पुश करना), Kubernetes (क्लस्टर में डिप्लॉय करना), Helm – कंटेनर-आधारित वर्कफ़्लो के लिए।
*   **क्लाउड प्रोवाइडर्स**: AWS (EC2, S3, Lambda प्लगइन के माध्यम से), Azure, Google Cloud – क्लाउड इन्फ्रास्ट्रक्चर पर डिप्लॉय करने के लिए।
*   **टेस्टिंग और क्वालिटी**: SonarQube (कोड क्वालिटी स्कैन), Selenium (UI टेस्टिंग), JUnit (यूनिट टेस्ट), Cucumber (BDD) – ऑटोमेटेड टेस्टिंग के लिए बिल्ड स्टेप्स में इंटीग्रेट करें।
*   **डिप्लॉयमेंट और मॉनिटरिंग**: Ansible, Terraform (इन्फ्रास्ट्रक्चर एज़ कोड), Prometheus, Grafana – ऑर्केस्ट्रेशन और पोस्ट-बिल्ड मॉनिटरिंग के लिए।
*   **नोटिफिकेशन और सहयोग**: Slack, Microsoft Teams, Email, Jira, Trello – बिल्ड स्टेटस पर अलर्ट भेजें या टिकट्स अपडेट करें।
*   **बिल्ड टूल्स**: Maven, Gradle (Java), npm (Node.js), pip (Python) – विभिन्न भाषाओं के लिए बिल्ड्स एक्जीक्यूट करें।
*   **सुरक्षा**: OWASP Dependency-Check, Trivy (वल्नरेबिलिटी स्कैनिंग) – CI के दौरान मुद्दों के लिए स्कैन करें।
*   **अन्य**: Artifactory/Nexus (आर्टिफैक्ट रिपॉजिटरी), Zapier (नो-कोड इंटीग्रेशन), Octopus Deploy (एडवांस्ड डिप्लॉयमेंट)।

उदाहरण के लिए, GitHub के साथ, वेबहुक ट्रिगर्स को सक्षम करने के लिए GitHub प्लगइन इंस्टॉल करें। Docker के साथ, पाइपलाइन में इमेज बनाने के लिए Docker Pipeline प्लगइन का उपयोग करें। निच टूल्स के लिए कस्टम प्लगइन भी बनाए जा सकते हैं।

अधिक जानने के लिए, ऑफिशियल प्लगइन इंडेक्स चेक करें। सुरक्षा के लिए प्लगइन्स को नियमित रूप से अपडेट करते रहें।
