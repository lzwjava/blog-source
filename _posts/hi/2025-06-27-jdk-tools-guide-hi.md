डायरेक्टरी `/usr/lib/jvm/java-21-openjdk-amd64/bin/` में OpenJDK 21 Java Development Kit (JDK) द्वारा प्रदान किए गए टूल्स शामिल हैं। ये टूल्स Java डेवलपमेंट, डीबगिंग, मॉनिटरिंग और पैकेजिंग से संबंधित विभिन्न कार्यों के लिए उपयोग किए जाते हैं। नीचे प्रत्येक टूल और उसके सामान्य उपयोग का संक्षिप्त विवरण दिया गया है। विस्तृत उपयोग के लिए, आप टर्मिनल में इनमें से अधिकांश टूल्स के लिए `tool_name --help` या `man tool_name` चला सकते हैं।

### 1. **jar** - Java ARchive (JAR) फ़ाइलें बनाएं और प्रबंधित करें
- **उद्देश्य**: Java क्लासेज और रिसोर्सेज को JAR फ़ाइल में पैकेज करना या JAR से कंटेंट निकालना।
- **सामान्य उपयोग**:
  ```bash
  jar cf myapp.jar *.class  # JAR फ़ाइल बनाएं
  jar xf myapp.jar          # JAR की सामग्री निकालें
  jar tf myapp.jar          # JAR की सामग्री सूचीबद्ध करें
  ```
- **उदाहरण**: `jar cvfm myapp.jar manifest.txt *.class` (मैनिफेस्ट फ़ाइल के साथ JAR बनाता है)।

### 2. **java** - Java एप्लिकेशन लॉन्च करें
- **उद्देश्य**: क्लास फ़ाइल या JAR को एक्ज़िक्यूट करके Java प्रोग्राम चलाना।
- **सामान्य उपयोग**:
  ```bash
  java MyClass              # क्लास फ़ाइल चलाएं
  java -jar myapp.jar       # JAR फ़ाइल चलाएं
  java -cp . MyClass        # विशिष्ट क्लासपाथ के साथ चलाएं
  ```
- **उदाहरण**: `java -Xmx512m -jar myapp.jar` (512MB मैक्स हीप के साथ JAR चलाता है)।

### 3. **javadoc** - API डॉक्यूमेंटेशन जनरेट करें
- **उद्देश्य**: Java सोर्स कोड कमेंट्स से HTML डॉक्यूमेंटेशन बनाना।
- **सामान्य उपयोग**:
  ```bash
  javadoc -d docs MyClass.java  # 'docs' फ़ोल्डर में डॉक्स जनरेट करें
  ```
- **उदाहरण**: `javadoc -d docs -sourcepath src -subpackages com.example` (पैकेज के लिए डॉक्स जनरेट करें)।

### 4. **jcmd** - चल रहे JVM को डायग्नोस्टिक कमांड भेजें
- **उद्देश्य**: डायग्नोस्टिक्स (जैसे थ्रेड डंप, हीप जानकारी) के लिए चल रहे Java प्रोसेस के साथ इंटरैक्ट करना।
- **सामान्य उपयोग**:
  ```bash
  jcmd <pid> help           # JVM प्रोसेस के लिए उपलब्ध कमांड्स की सूची दें
  jcmd <pid> Thread.print   # थ्रेड डंप प्रिंट करें
  ```
- **उदाहरण**: `jcmd 1234 GC.run` (प्रोसेस ID 1234 के लिए गार्बेज कलेक्शन ट्रिगर करें)।

### 5. **jdb** - Java डीबगर
- **उद्देश्य**: Java एप्लिकेशन्स को इंटरैक्टिव तरीके से डीबग करना।
- **सामान्य उपयोग**:
  ```bash
  jdb MyClass               # क्लास को डीबग करना शुरू करें
  java -agentlib:jdwp=transport=dt_socket,server=y,suspend=y,address=*:5005 MyClass  # डीबग एजेंट के साथ चलाएं
  jdb -attach localhost:5005  # चल रहे JVM से अटैच करें
  ```
- **उदाहरण**: `jdb -sourcepath src MyClass` (सोर्स कोड के साथ डीबग करें)।

### 6. **jdeps** - क्लास और JAR डिपेंडेंसीज़ का विश्लेषण करें
- **उद्देश्य**: Java एप्लिकेशन या लाइब्रेरी की डिपेंडेंसीज़ की पहचान करना।
- **सामान्य उपयोग**:
  ```bash
  jdeps myapp.jar           # JAR में डिपेंडेंसीज़ का विश्लेषण करें
  jdeps -s MyClass.class    # डिपेंडेंसीज़ का सारांश
  ```
- **उदाहरण**: `jdeps -v myapp.jar` (वर्बोज़ डिपेंडेंसी विश्लेषण)।

### 7. **jhsdb** - Java HotSpot डीबगर
- **उद्देश्य**: JVM प्रोसेसेज (जैसे कोर डंप) की एडवांस्ड डीबगिंग और विश्लेषण।
- **सामान्य उपयोग**:
  ```bash
  jhsdb jmap --heap --pid <pid>  # चल रहे प्रोसेस के हीप का विश्लेषण करें
  jhsdb hsdb                     # HotSpot डीबगर GUI लॉन्च करें
  ```
- **उदाहरण**: `jhsdb jmap --heap --pid 1234` (प्रोसेस 1234 के लिए हीप जानकारी डंप करें)।

### 8. **jinfo** - JVM कॉन्फ़िगरेशन देखें या संशोधित करें
- **उद्देश्य**: चल रहे प्रोसेस के लिए JVM ऑप्शन्स का निरीक्षण या परिवर्तन करना।
- **सामान्य उपयोग**:
  ```bash
  jinfo <pid>               # JVM फ्लैग्स और प्रॉपर्टीज़ देखें
  jinfo -flag +PrintGC <pid>  # JVM फ्लैग सक्षम करें
  ```
- **उदाहरण**: `jinfo -sysprops 1234` (प्रोसेस 1234 के लिए सिस्टम प्रॉपर्टीज़ प्रदर्शित करें)।

### 9. **jmap** - JVM मेमोरी या हीप जानकारी डंप करें
- **उद्देश्य**: हीप डंप या मेमोरी स्टैटिस्टिक्स जनरेट करना।
- **सामान्य उपयोग**:
  ```bash
  jmap -heap <pid>          # हीप सारांश प्रिंट करें

  ```
- **उदाहरण**: `jmap -dump:live,format=b,file=dump.hprof 1234` (लाइव ऑब्जेक्ट्स डंप करें)।

### 10. **jpackage** - Java एप्लिकेशन्स पैकेज करें

### 15. **jstat** - JVM स्टैटिस्टिक्स मॉनिटर करें

  ```bash

  ```
- **उदाहरण**: `jstat -gcutil 1234 1000 5` (GC स्टैट्स 5 बार दिखाएं, हर 1 सेकंड)।

- **उदाहरण**: `jwebserver -d ./html -p 8000` (पोर्ट 8000 पर ./html से फ़ाइलें सर्व करें)।

### आगे सीखने के लिए सुझाव
- **डॉक्यूमेंटेशन**: `tool_name --help` चलाएं या Oracle के JDK डॉक्यूमेंटेशन (जैसे https://docs.oracle.com/en/java/javase/21/docs/specs/man/) देखें।
- **मैन पेज**: मैनुअल पेज वाले टूल्स के लिए `man tool_name` का उपयोग करें।
- **अभ्यास**: टूल के व्यवहार को समझने के लिए टेस्ट प्रोजेक्ट में प्रयोग करें।
- **मॉनिटरिंग टूल्स**: `jcmd`, `jmap`, `jstack`, आदि के लिए, अपनी Java प्रोसेस की PID खोजने के लिए `jps` का उपयोग करें।
- **डीबगिंग**: एडवांस्ड डीबगिंग के लिए `jdb`, `jhsdb`, या `jconsole` को संयोजित करें।

यदि आप ट्रेडिंग प्रोजेक्ट पर काम कर रहे हैं (जैसा कि आपकी डायरेक्टरी से पता चलता है), तो आपकी एप्लिकेशन को कंपाइल, रन और पैकेज करने के लिए `javac`, `java`, `jar`, और `jpackage` जैसे टूल सबसे प्रासंगिक हैं। परफॉर्मेंस ट्यूनिंग के लिए, `jstat`, `jmap`, या `jfr` का उपयोग करें। यदि आपको किसी विशिष्ट टूल में गहराई से जाने की आवश्यकता है या आपके ट्रेडिंग प्रोजेक्ट में मदद चाहिए, तो मुझे बताएं!
