### मुख्य बिंदु
- ऐसा प्रतीत होता है कि "pili-engineering/pili-sdk-php": "dev-master" का उपयोग करने के लिए, आपको इसे Composer के माध्यम से इंस्टॉल करना होगा और Qiniu क्रेडेंशियल्स सेटअप करने होंगे।
- शोध से पता चलता है कि पैकेज के लिए PHP 5.3.0 या उच्चतर की आवश्यकता होती है और यह Pili स्ट्रीमिंग क्लाउड के लिए है, जो Qiniu से संबंधित है।
- प्रमाण एक Hub ऑब्जेक्ट बनाने और RTMP URL जनरेट करने जैसी स्ट्रीम ऑपरेशन्स का उपयोग करने की ओर इशारा करते हैं, लेकिन सटीक तरीके भिन्न हो सकते हैं।

### इंस्टालेशन
सबसे पहले, सुनिश्चित करें कि आपके पास Composer इंस्टॉल है। अपनी `composer.json` फ़ाइल में पैकेज जोड़ें:
```json
"require": {
    "pili-engineering/pili-sdk-php": "dev-master"
}
```
फिर, `composer install` या `composer update` चलाएं। अपनी PHP स्क्रिप्ट में, शामिल करें:
```php
require 'vendor/autoload.php';
```

### सेटअप और उपयोग
आपको एक Qiniu अकाउंट और एक Pili Hub की आवश्यकता होगी। अपना एक्सेस की, सीक्रेट की, और हब नाम सेट करें, फिर एक Hub ऑब्जेक्ट बनाएं:
```php
use Qiniu\Credentials;
use Pili\Hub;

$credentials = new Credentials('your_access_key', 'your_secret_key');
$hub = new Hub($credentials, 'your_hub_name');
```
एक स्ट्रीम बनाएं या प्राप्त करें, उदाहरण के लिए, `$stream = $hub->createStream('your_stream_key');`, और ऑपरेशन्स के लिए `$stream->rtmpPublishUrl(60)` जैसी विधियों का उपयोग करें।

### अप्रत्याशित विवरण
ध्यान दें कि "dev-master" एक डेवलपमेंट वर्जन है, संभावित रूप से अस्थिर, जिसमें प्रोडक्शन के लिए 1.5.5 जैसे टैग किए गए वर्जन उपलब्ध हैं।

### "pili-engineering/pili-sdk-php": "dev-master" का उपयोग करने पर व्यापक गाइड

यह गाइड "pili-engineering/pili-sdk-php" पैकेज के "dev-master" वर्जन का उपयोग करने के तरीके की उपलब्ध डॉक्यूमेंटेशन और उदाहरणों के आधार पर एक विस्तृत जांच प्रदान करती है। यह इंस्टालेशन, सेटअप, उपयोग और अतिरिक्त विचारों को कवर करती है, जिससे Pili स्ट्रीमिंग क्लाउड सेवाओं के साथ काम करने वाले डेवलपर्स के लिए एक संपूर्ण समझ सुनिश्चित होती है।

#### पृष्ठभूमि और संदर्भ
"pili-engineering/pili-sdk-php" पैकेज PHP के लिए एक सर्वर-साइड लाइब्रेरी है, जिसे Pili स्ट्रीमिंग क्लाउड के साथ इंटरैक्ट करने के लिए डिज़ाइन किया गया है, जो एक क्लाउड स्टोरेज और CDN प्रदाता Qiniu से संबद्ध एक सेवा है। "dev-master" वर्जन नवीनतम डेवलपमेंट ब्रांच को संदर्भित करता है, जिसमें हाल की सुविधाएँ शामिल हो सकती हैं लेकिन टैग किए गए रिलीज़ की तुलना में कम स्थिर हो सकता है। पैकेज के लिए PHP 5.3.0 या उच्चतर की आवश्यकता होती है, जो 3 मार्च, 2025 तक कई PHP वातावरणों के लिए इसे सुलभ बनाता है।

#### इंस्टालेशन प्रक्रिया
शुरुआत करने के लिए, आपके पास Composer इंस्टॉल होना चाहिए, जो PHP के लिए एक डिपेंडेंसी मैनेजर है। इंस्टालेशन में आपके प्रोजेक्ट की `composer.json` फ़ाइल में पैकेज जोड़ना और इसे डाउनलोड करने के लिए एक Composer कमांड चलाना शामिल है। विशेष रूप से:

- अपने `composer.json` में "require" सेक्शन के अंतर्गत निम्नलिखित जोड़ें:
  ```json
  "require": {
      "pili-engineering/pili-sdk-php": "dev-master"
  }
  ```
- पैकेज और उसकी डिपेंडेंसीज़ को फ़ेच करने के लिए अपने टर्मिनल में `composer install` या `composer update` निष्पादित करें। यह आवश्यक फ़ाइलों के साथ एक `vendor` डायरेक्टरी बनाएगा।
- अपनी PHP स्क्रिप्ट में, पैकेज क्लासेज़ तक पहुँचने के लिए ऑटोलोडर को शामिल करें:
  ```php
  require 'vendor/autoload.php';
  ```

यह प्रक्रिया सुनिश्चित करती है कि पैकेज आपके प्रोजेक्ट में एकीकृत हो गया है, जो आसान क्लास एक्सेस के लिए Composer के ऑटोलोडिंग का लाभ उठाता है।

#### पूर्वापेक्षाएँ और सेटअप
SDK का उपयोग करने से पहले, आपको एक Qiniu अकाउंट की आवश्यकता है और आपको एक Pili Hub सेटअप करना होगा, क्योंकि SDK Pili स्ट्रीमिंग क्लाउड सेवाओं के साथ इंटरैक्ट करता है। इसमें Qiniu से एक एक्सेस की और सीक्रेट की प्राप्त करना और उनके प्लेटफ़ॉर्म के भीतर एक हब बनाना शामिल है। डॉक्यूमेंटेशन से पता चलता है कि ये क्रेडेंशियल्स प्रमाणीकरण के लिए आवश्यक हैं।

सेटअप करने के लिए, अपनी PHP स्क्रिप्ट में अपने क्रेडेंशियल्स परिभाषित करें:
- एक्सेस की: आपकी Qiniu एक्सेस की।
- सीक्रेट की: आपकी Qiniu सीक्रेट की।
- हब नाम: आपके Pili Hub का नाम, जिसका उपयोग से पहले अस्तित्व में होना चाहिए।

एक उदाहरण सेटअप इस प्रकार दिखता है:
```php
$accessKey = 'your_access_key';
$secretKey = 'your_secret_key';
$hubName = 'your_hub_name';
```

#### Hub ऑब्जेक्ट बनाना और उपयोग करना
SDK का मूल Hub ऑब्जेक्ट है, जो स्ट्रीम प्रबंधन की सुविधा प्रदान करता है। सबसे पहले, अपनी Qiniu कीज़ का उपयोग करके एक Credentials ऑब्जेक्ट बनाएं:
```php
use Qiniu\Credentials;

$credentials = new Credentials($accessKey, $secretKey);
```
फिर, इन क्रेडेंशियल्स और अपने हब नाम के साथ एक Hub ऑब्जेक्ट इंस्टेंटिएट करें:
```php
use Pili\Hub;

$hub = new Hub($credentials, $hubName);
```
यह Hub ऑब्जेक्ट आपको स्ट्रीम-संबंधित विभिन्न ऑपरेशन्स करने की अनुमति देता है, जैसे स्ट्रीम बनाना, पुनः प्राप्त करना, या सूचीबद्ध करना।

#### स्ट्रीम्स के साथ कार्य करना
स्ट्रीम्स Pili स्ट्रीमिंग क्लाउड के केंद्र में हैं, और SDK Hub ऑब्जेक्ट के माध्यम से उन्हें प्रबंधित करने के लिए विधियाँ प्रदान करता है। एक नई स्ट्रीम बनाने के लिए:
```php
$streamKey = 'your_stream_key'; // हब के भीतर अद्वितीय होना चाहिए
$stream = $hub->createStream($streamKey);
```
एक मौजूदा स्ट्रीम को पुनः प्राप्त करने के लिए:
```php
$stream = $hub->getStream($streamKey);
```
स्ट्रीम ऑब्जेक्ट फिर ऑपरेशन्स के लिए विभिन्न विधियाँ प्रदान करता है, जिन्हें उपलब्ध डॉक्यूमेंटेशन के आधार पर निम्न तालिका में विस्तृत किया गया है:

| **ऑपरेशन**          | **विधि**                     | **विवरण**                                      |
|-------------------------|--------------------------------|------------------------------------------------------|
| स्ट्रीम बनाएं          | `$hub->createStream($key)`     | दिए गए की के साथ एक नई स्ट्रीम बनाता है।             |
| स्ट्रीम प्राप्त करें             | `$hub->getStream($key)`        | की द्वारा एक मौजूदा स्ट्रीम पुनः प्राप्त करता है।                 |
| स्ट्रीम्स सूचीबद्ध करें           | `$hub->listStreams($marker, $limit, $prefix)` | पैजिनेशन विकल्पों के साथ स्ट्रीम्स की सूची बनाता है।               |
| RTMP पब्लिश URL       | `$stream->rtmpPublishUrl($expire)` | एक्सपायरेशन समय के साथ एक RTMP पब्लिश URL जनरेट करता है।  |
| RTMP प्ले URL          | `$stream->rtmpPlayUrl()`       | स्ट्रीम के लिए एक RTMP प्ले URL जनरेट करता है।           |
| HLS प्ले URL           | `$stream->hlsPlayUrl()`        | स्ट्रीमिंग के लिए एक HLS प्ले URL जनरेट करता है।             |
| स्ट्रीम अक्षम करें         | `$stream->disable()`           | स्ट्रीम को अक्षम करता है।                                 |
| स्ट्रीम सक्षम करें          | `$stream->enable()`            | स्ट्रीम को सक्षम करता है।                                  |
| स्ट्रीम स्टेटस प्राप्त करें      | `$stream->status()`            | स्ट्रीम की वर्तमान स्थिति पुनः प्राप्त करता है।          |

उदाहरण के लिए, 60-सेकंड की एक्सपायरेशन के साथ एक RTMP पब्लिश URL जनरेट करने के लिए:
```php
$expire = 60;
$url = $stream->rtmpPublishUrl($expire);
echo $url;
```
इस URL का उपयोग Pili स्ट्रीमिंग क्लाउड पर स्ट्रीम्स पब्लिश करने के लिए किया जा सकता है, जिसमें एक्सपायरेशन अस्थायी एक्सेस सुनिश्चित करता है।

#### अतिरिक्त विचार
- **वर्जन स्थिरता**: "dev-master" वर्जन डेवलपमेंट ब्रांच है, संभावित रूप से अस्थिर। प्रोडक्शन के लिए, एक टैग किए गए वर्जन, जैसे कि 1.5.5, का उपयोग करने पर विचार करें, जो Packagist [pili-engineering/pili-sdk-php versions](https://packagist.org/p/pili-engineering/pili-sdk-php) पर उपलब्ध है। इतिहास API जोड़ और विधि परिष्कार जैसे अपडेट दिखाता है, जिसमें वर्जन 2016 तक के हैं।
- **त्रुटि हैंडलिंग**: डॉक्यूमेंटेशन से पता चलता है कि ऑपरेशन्स के लिए स्ट्रीम निर्माण और पुनर्प्राप्ति के उदाहरणों में संभावित एक्सेप्शन्स को हैंडल करने के लिए try-catch ब्लॉक्स का उपयोग करना चाहिए।
- **डॉक्यूमेंटेशन और उदाहरण**: हालांकि विशिष्ट उदाहरणों का व्यापक रूप से विवरण नहीं दिया गया था, GitHub रिपॉजिटरी [Pili स्ट्रीमिंग क्लाउड सर्वर-साइड लाइब्रेरी फॉर PHP](https://github.com/pili-engineering/pili-sdk-php) इंस्टालेशन निर्देश और सुविधा सूची प्रदान करती है, और अन्य भाषा SDKs (जैसे, Python) के साथ तुलना समान कार्यक्षमता का संकेत देती है, जैसे URL जनरेशन।

#### तुलनात्मक अंतर्दृष्टि
अन्य Pili SDKs, जैसे कि Python वर्जन, के साथ तुलना करने पर समान पैटर्न का पता चलता है: दोनों को क्रेडेंशियल्स, हब सेटअप और स्ट्रीम ऑपरेशन्स की आवश्यकता होती है। हालाँकि, PHP वर्जन Hub और Stream क्लासेज़ के साथ एक ऑब्जेक्ट-ओरिएंटेड दृष्टिकोण का उपयोग करता है, जो Python की अधिक फंक्शनल शैली के विपरीत है। यह PHP की विशिष्ट डेवलपमेंट प्रथाओं के साथ संरेखित होता है, जो टाइप सेफ्टी और IDE सपोर्ट प्रदान करता है।
