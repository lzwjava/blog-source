### मुख्य बिंदु
- लगता है कि "pili-engineering/pili-sdk-php": "dev-master" का उपयोग करने के लिए, आपको इसे Composer के माध्यम से स्थापित करना होगा और Qiniu पात्रता स्थापित करनी होगी।
- अनुसंधान से पता चलता है कि पैकेज PHP 5.3.0 या उससे अधिक की आवश्यकता होती है और यह Qiniu से संबंधित Pili Streaming Cloud के लिए है।
- सबूत इस ओर इशारा करते हैं कि एक Hub ऑब्जेक्ट बनाना होगा और स्ट्रीम ऑपरेशंस जैसे RTMP URLs का जनरेशन करना होगा, लेकिन सही तरीके भिन्न हो सकते हैं।

### स्थापना
पहले, सुनिश्चित करें कि आपके पास Composer स्थापित है। पैकेज को अपने `composer.json` फ़ाइल में जोड़ें:
```json
"require": {
    "pili-engineering/pili-sdk-php": "dev-master"
}
```
फिर, `composer install` या `composer update` चलाएं। अपने PHP स्क्रिप्ट में शामिल करें:
```php
require 'vendor/autoload.php';
```

### सेटअप और उपयोग
आपको एक Qiniu खाता और एक Pili Hub की आवश्यकता होगी। अपने एक्सेस की, सीक्रेट की और हब नाम सेट करें, फिर एक Hub ऑब्जेक्ट बनाएं:
```php
use Qiniu\Credentials;
use Pili\Hub;

$credentials = new Credentials('your_access_key', 'your_secret_key');
$hub = new Hub($credentials, 'your_hub_name');
```
एक स्ट्रीम बनाएं या प्राप्त करें, उदाहरण के लिए, `$stream = $hub->createStream('your_stream_key');`, और `$stream->rtmpPublishUrl(60)` जैसे विधियों का उपयोग करें।

### अनपेक्षित विवरण
ध्यान दें कि "dev-master" एक विकास संस्करण है, जो अस्थिर हो सकता है, और जैसे 1.5.5 जैसे टैग संस्करण उत्पादन के लिए उपलब्ध हैं।

### "pili-engineering/pili-sdk-php": "dev-master" का उपयोग करने का व्यापक मार्गदर्शन

यह मार्गदर्शन "pili-engineering/pili-sdk-php" पैकेज के "dev-master" संस्करण का उपयोग करने के बारे में एक विस्तृत जांच प्रदान करता है, उपलब्ध दस्तावेज़ और उदाहरणों पर आधारित। यह स्थापना, सेटअप, उपयोग और अतिरिक्त विचारों को कवर करता है, Pili Streaming Cloud सेवाओं के साथ काम करने वाले डेवलपर्स के लिए एक गहन समझ सुनिश्चित करता है।

#### पृष्ठभूमि और संदर्भ
"pili-engineering/pili-sdk-php" पैकेज एक सर्वर-साइड लाइब्रेरी है PHP के लिए, जो Qiniu, एक क्लाउड स्टोरेज और CDN प्रोवाइडर से जुड़ी Pili Streaming Cloud के साथ इंटरैक्ट करने के लिए डिज़ाइन किया गया है। "dev-master" संस्करण सबसे हालिया विकास शाखा को संदर्भित करता है, जो हाल के फीचर्स शामिल हो सकते हैं लेकिन टैग संस्करणों से कम स्थिर हो सकते हैं। पैकेज PHP 5.3.0 या उससे अधिक की आवश्यकता होती है, जिससे यह मार्च 3, 2025 तक कई PHP पर्यावरणों के लिए उपलब्ध हो जाता है।

#### स्थापना प्रक्रिया
शुरू करने के लिए, आपको Composer स्थापित होना चाहिए, एक PHP के लिए निर्भरता प्रबंधक। स्थापना में पैकेज को अपने प्रोजेक्ट के `composer.json` फ़ाइल में जोड़ना और एक Composer कमांड चलाना शामिल है ताकि इसे डाउनलोड किया जा सके। विशेष रूप से:

- अपने `composer.json` में "require" खंड के नीचे निम्नलिखित जोड़ें:
  ```json
  "require": {
      "pili-engineering/pili-sdk-php": "dev-master"
  }
  ```
- अपने टर्मिनल में `composer install` या `composer update` चलाएं ताकि पैकेज और उसके निर्भरताओं को प्राप्त किया जा सके। यह एक `vendor` डायरेक्टरी बनाएगा जिसमें आवश्यक फ़ाइलें होंगी।
- अपने PHP स्क्रिप्ट में ऑटोलोडर शामिल करें ताकि पैकेज क्लासेस तक पहुंचा जा सके:
  ```php
  require 'vendor/autoload.php';
  ```

यह प्रक्रिया सुनिश्चित करता है कि पैकेज आपके प्रोजेक्ट में एकीकृत हो जाता है, Composer की ऑटोलोडिंग का उपयोग करके आसान क्लास एक्सेस के साथ।

#### पूर्वापेक्षाएं और सेटअप
SDK का उपयोग करने से पहले, आपको एक Qiniu खाता चाहिए और एक Pili Hub सेटअप करना होगा, क्योंकि SDK Pili Streaming Cloud सेवाओं के साथ इंटरैक्ट करता है। इसमें Qiniu से एक एक्सेस की और सीक्रेट की प्राप्त करना और उनके प्लेटफॉर्म में एक हब बनाना शामिल है। दस्तावेज़ सुझाव देते हैं कि ये पात्रताएं सत्यापन के लिए आवश्यक हैं।

सेटअप करने के लिए, अपने PHP स्क्रिप्ट में अपने पात्रताओं को परिभाषित करें:
- एक्सेस की: आपका Qiniu एक्सेस की।
- सीक्रेट की: आपका Qiniu सीक्रेट की।
- हब नाम: आपके Pili Hub का नाम, जो उपयोग करने से पहले मौजूद होना चाहिए।

एक उदाहरण सेटअप इस प्रकार दिखता है:
```php
$accessKey = 'your_access_key';
$secretKey = 'your_secret_key';
$hubName = 'your_hub_name';
```

#### Hub ऑब्जेक्ट बनाना और उपयोग करना
SDK का कोर Hub ऑब्जेक्ट है, जो स्ट्रीम प्रबंधन को सुविधाजनक बनाता है। पहले, अपने Qiniu कीजों का उपयोग करके एक Credentials ऑब्जेक्ट बनाएं:
```php
use Qiniu\Credentials;

$credentials = new Credentials($accessKey, $secretKey);
```
फिर, इन पात्रताओं और अपने हब नाम के साथ एक Hub ऑब्जेक्ट बनाएं:
```php
use Pili\Hub;

$hub = new Hub($credentials, $hubName);
```
इस Hub ऑब्जेक्ट को विभिन्न स्ट्रीम-संबंधी ऑपरेशंस करने के लिए उपयोग किया जा सकता है, जैसे कि स्ट्रीम बनाना, प्राप्त करना या सूचीबद्ध करना।

#### स्ट्रीमों के साथ काम करना
स्ट्रीम Pili Streaming Cloud के लिए केंद्र हैं, और SDK Hub ऑब्जेक्ट के माध्यम से उन्हें प्रबंधित करने के लिए विधियों को प्रदान करता है। एक नया स्ट्रीम बनाने के लिए:
```php
$streamKey = 'your_stream_key'; // हब के भीतर एकमात्र होना चाहिए
$stream = $hub->createStream($streamKey);
```
एक मौजूदा स्ट्रीम प्राप्त करने के लिए:
```php
$stream = $hub->getStream($streamKey);
```
फिर स्ट्रीम ऑब्जेक्ट विभिन्न ऑपरेशंस के लिए विधियों को प्रदान करता है, जो निम्नलिखित तालिका में उपलब्ध दस्तावेज़ पर आधारित है:

| **ऑपरेशन**          | **विधि**                     | **विवरण**                                      |
|-------------------------|--------------------------------|------------------------------------------------------|
| स्ट्रीम बनाना          | `$hub->createStream($key)`     | दिए गए की के साथ एक नया स्ट्रीम बनाता है।             |
| स्ट्रीम प्राप्त करना     | `$hub->getStream($key)`        | की द्वारा एक मौजूदा स्ट्रीम प्राप्त करता है।                 |
| स्ट्रीम सूचीबद्ध करना   | `$hub->listStreams($marker, $limit, $prefix)` | पेजिनेशन विकल्पों के साथ स्ट्रीम सूचीबद्ध करता है।               |
| RTMP प्रकाशित URL       | `$stream->rtmpPublishUrl($expire)` | एक्सपायरेशन समय के साथ एक RTMP प्रकाशित URL जनरेट करता है।  |
| RTMP प्ले URL          | `$stream->rtmpPlayUrl()`       | स्ट्रीम के लिए एक RTMP प्ले URL जनरेट करता है।           |
| HLS प्ले URL           | `$stream->hlsPlayUrl()`        | स्ट्रीमिंग के लिए एक HLS प्ले URL जनरेट करता है।             |
| स्ट्रीम निष्क्रिय करना         | `$stream->disable()`           | स्ट्रीम को निष्क्रिय करता है।                                 |
| स्ट्रीम सक्रिय करना          | `$stream->enable()`            | स्ट्रीम को सक्रिय करता है।                                  |
| स्ट्रीम स्थिति प्राप्त करना      | `$stream->status()`            | स्ट्रीम की मौजूदा स्थिति प्राप्त करता है।          |

उदाहरण के लिए, एक 60-सेकंड के एक्सपायरेशन के साथ एक RTMP प्रकाशित URL जनरेट करने के लिए:
```php
$expire = 60;
$url = $stream->rtmpPublishUrl($expire);
echo $url;
```
इस URL का उपयोग Pili Streaming Cloud में स्ट्रीम प्रकाशित करने के लिए किया जा सकता है, जहां एक्सपायरेशन अस्थायी एक्सेस सुनिश्चित करता है।

#### अतिरिक्त विचार
- **संस्करण स्थिरता**: "dev-master" विकास शाखा है, जो अस्थिर हो सकता है। उत्पादन के लिए, एक टैग संस्करण, जैसे 1.5.5, का उपयोग करने की सोचें, जो Packagist पर उपलब्ध है [pili-engineering/pili-sdk-php संस्करण](https://packagist.org/p/pili-engineering/pili-sdk-php)। इतिहास में API जोड़ने और विधियों में सुधार जैसे अपडेट शामिल हैं, जिनमें से संस्करण 2016 से पहले के हैं।
- **त्रुटि प्रबंधन**: दस्तावेज़ सुझाव देते हैं कि ऑपरेशंस के लिए try-catch ब्लॉक का उपयोग करें, जैसे कि स्ट्रीम बनाना और प्राप्त करना के उदाहरणों में देखा गया है, ताकि संभावित अपवादों का प्रबंधन किया जा सके।
- **दस्तावेज़ और उदाहरण**: जबकि विशेष उदाहरण विस्तार से विस्तृत नहीं थे, GitHub रिपॉजिटरी [Pili Streaming Cloud server-side library for PHP](https://github.com/pili-engineering/pili-sdk-php) स्थापना निर्देश और फीचर सूची प्रदान करता है, और अन्य भाषा SDKs (जैसे Python) के साथ तुलना में समान कार्यक्षमता, जैसे URL जनरेशन, इंगित करता है।

#### तुलनात्मक दृष्टिकोण
अन्य Pili SDKs, जैसे Python संस्करण के साथ तुलना करने से पता चलता है कि समान पैटर्न हैं: दोनों पात्रताओं, हब सेटअप और स्ट्रीम ऑपरेशंस की आवश्यकता होती हैं। हालांकि, PHP संस्करण एक ऑब्जेक्ट-ओरिएंटेड दृष्टिकोण का उपयोग करता है Hub और Stream क्लासों के साथ, जबकि Python का शैली अधिक कार्यात्मक है। यह PHP के सामान्य विकास अभ्यासों के साथ मेल खाता है, जो प्रकार सुरक्षा और IDE समर्थन प्रदान करता है।
