Nginx कॉन्फ़िगरेशन में "403 Forbidden" एरर को ठीक करने के लिए, आपको दो मुख्य मुद्दों को हल करना होगा: आपके `location` ब्लॉक में `root` डायरेक्टिव का गलत उपयोग और संभावित फ़ाइल सिस्टम परमिशन समस्याएं। नीचे आपकी दी गई कॉन्फ़िगरेशन के आधार पर समस्या को हल करने के लिए एक चरण-दर-चरण मार्गदर्शिका दी गई है:

```nginx
server {
    listen 80;
    server_name localhost;
    location /movies {
        root /path/to/your/movie/folder;
        autoindex on; # डायरेक्टरी लिस्टिंग सक्षम करता है
    }
}
```

जब आप `http://localhost/movies` पर जाते हैं, तो आपको 403 Forbidden एरर मिलता है, जो आमतौर पर इंगित करता है कि Nginx इच्छित डायरेक्टरी तक पहुंच नहीं बना सकता है या उसे सर्व नहीं कर सकता है। इसे ठीक करने का तरीका यहां बताया गया है:

---

### 1. Nginx कॉन्फ़िगरेशन को सही करें
वर्तमान `location /movies` ब्लॉक `root /path/to/your/movie/folder;` का उपयोग करता है। Nginx में, `root` डायरेक्टिव निर्दिष्ट पथ में रिक्वेस्ट URI (जैसे, `/movies`) को जोड़ता है, जिसका अर्थ है कि Nginx `/path/to/your/movie/folder/movies` से फाइलें सर्व करने का प्रयास करता है। यदि यह डायरेक्टरी मौजूद नहीं है, तो Nginx एक 403 एरर लौटाता है।

इसे ठीक करने के लिए, आपको `root` के बजाय `alias` डायरेक्टिव का उपयोग करना चाहिए। `alias` डायरेक्टिव `/movies/` URI को सीधे `/path/to/your/movie/folder/` पर मैप करता है बिना URI को जोड़े, जो संभवतः आपका इच्छित व्यवहार है।

अपनी कॉन्फ़िगरेशन को इस प्रकार अपडेट करें:

```nginx
server {
    listen 80;
    server_name localhost;
    location /movies/ {
        alias /path/to/your/movie/folder/;
        autoindex on; # डायरेक्टरी लिस्टिंग सक्षम करता है
    }
}
```

- **मुख्य बदलाव:**
  - `root` को `alias` में बदला गया।
  - `autoindex` के साथ उचित डायरेक्टरी हैंडलिंग सुनिश्चित करने के लिए `location /movies/` और `alias /path/to/your/movie/folder/` में ट्रेलिंग स्लैश जोड़े गए।

- **बदलाव लागू करें:**
  कॉन्फ़िगरेशन फ़ाइल (जैसे, `/etc/nginx/nginx.conf` या `/etc/nginx/sites-enabled/` में एक फ़ाइल) को अपडेट करने के बाद, बदलाव लागू करने के लिए Nginx को रीस्टार्ट करें:
  - Linux पर: `sudo systemctl restart nginx`
  - Windows पर: Nginx सर्विस को मैन्युअली रोकें और शुरू करें।

- **URL टेस्ट करें:**
  देखने के लिए `http://localhost/movies/` (ट्रेलिंग स्लैश पर ध्यान दें) पर जाएं कि क्या डायरेक्टरी लिस्टिंग दिखाई देती है।

---

### 2. फ़ाइल सिस्टम परमिशन चेक करें
यदि कॉन्फ़िगरेशन परिवर्तन अकेले 403 एरर को हल नहीं करता है, तो समस्या फ़ाइल सिस्टम परमिशन से संबंधित हो सकती है। Nginx को `/path/to/your/movie/folder/` और उसकी सामग्री तक पढ़ने की पहुंच की आवश्यकता होती है, और यह पहुंच उस यूजर पर निर्भर करती है जिसके तहत Nginx चल रहा है (आमतौर पर `nginx` या `www-data`)।

- **Nginx यूजर की पहचान करें:**
  अपनी मुख्य Nginx कॉन्फ़िगरेशन फ़ाइल (जैसे, `/etc/nginx/nginx.conf`) में `user` डायरेक्टिव के लिए चेक करें। यह इस तरह दिख सकता है:
  ```nginx
  user nginx;
  ```
  यदि निर्दिष्ट नहीं है, तो यह आपके सिस्टम के आधार पर `www-data` या किसी अन्य यूजर पर डिफ़ॉल्ट हो सकता है।

- **परमिशन सत्यापित करें:**
  अपनी मूवी फोल्डर की परमिशन का निरीक्षण करने के लिए निम्न कमांड चलाएं:
  ```bash
  ls -l /path/to/your/movie/folder
  ```
  यह मालिक, ग्रुप और परमिशन (जैसे, `drwxr-xr-x`) प्रदर्शित करेगा।

- **आवश्यकता होने पर परमिशन समायोजित करें:**
  सुनिश्चित करें कि Nginx यूजर के पास पढ़ने (और डायरेक्टरी के लिए एक्ज़िक्यूट) की पहुंच है। यहां दो विकल्प हैं:
  - **विकल्प 1: स्वामित्व बदलें (अनुशंसित):**
    फोल्डर के मालिक को Nginx यूजर (जैसे, `nginx`) पर सेट करें:
    ```bash
    sudo chown -R nginx:nginx /path/to/your/movie/folder
    ```
    यदि यह अलग है (जैसे, `www-data`) तो `nginx` को वास्तविक यूजर से बदलें।

  - **विकल्प 2: सभी के लिए पठनीय बनाएं (कम सुरक्षित):**
    यदि आप स्वामित्व नहीं बदलना चाहते हैं, तो फोल्डर को दूसरों के लिए पठनीय बनाएं:
    ```bash
    sudo chmod -R o+r /path/to/your/movie/folder
    ```

- **डायरेक्टरी एक्सेस सुनिश्चित करें:**
  Nginx को इसकी सामग्री तक पहुंचने के लिए डायरेक्टरी को स्वयं एक्ज़िक्यूट परमिशन (`x`) की आवश्यकता होती है:
  ```bash
  sudo chmod o+x /path/to/your/movie/folder
  ```

- **पैरेंट डायरेक्टरीज चेक करें:**
  यदि `/path/to/your/movie/folder` एक प्रतिबंधित पैरेंट डायरेक्टरी (जैसे, `/home/user/`) के भीतर है, तो सुनिश्चित करें कि रूट तक की सभी पैरेंट डायरेक्टरीज में Nginx यूजर के लिए एक्ज़िक्यूट परमिशन हैं:
  ```bash
  sudo chmod o+x /path /path/to /path/to/your /path/to/your/movie
  ```

### 3. सत्यापित करें और ट्रबलशूट करें
इन परिवर्तनों को करने के बाद, `http://localhost/movies/` पर जाकर फिर से टेस्ट करें। यदि 403 एरर बनी रहती है:

- **Nginx एरर लॉग चेक करें:**
  विशिष्ट विवरणों के लिए एरर लॉग की समीक्षा करें (आमतौर पर `/var/log/nginx/error.log` पर स्थित):
  ```bash
  sudo tail -f /var/log/nginx/error.log
  ```
  समस्या का पता लगाने के लिए "permission denied" या "directory index is forbidden" जैसे संदेशों को देखें।

- **अतिरिक्त विचार:**
  - **SELinux (यदि लागू हो):** SELinux सक्षम सिस्टम जैसे CentOS पर, यह एक्सेस को ब्लॉक कर सकता है। टेस्ट करने के लिए अस्थायी रूप से SELinux को permissive मोड में सेट करें:
    ```bash
    sudo setenforce 0
    ```
    यदि इससे समस्या हल हो जाती है, तो आपको SELinux पॉलिसीज को समायोजित करना होगा (जैसे, `chcon` या `audit2allow` का उपयोग करके)।
  - **फ़ायरवॉल:** सुनिश्चित करें कि पोर्ट 80 खुला है यदि किसी अन्य मशीन से एक्सेस कर रहे हैं (हालांकि `localhost` के माध्यम से स्थानीय रूप से टेस्ट करने पर संभावना नहीं है)।

### अंतिम कॉन्फ़िगरेशन
यहां सही और पूर्ण कॉन्फ़िगरेशन दिया गया है:

```nginx
server {
    listen 80;
    server_name localhost;
    location /movies/ {
        alias /path/to/your/movie/folder/;
        autoindex on; # डायरेक्टरी लिस्टिंग सक्षम करता है
    }
}
```

- `/path/to/your/movie/folder/` को अपनी मूवी डायरेक्टरी के वास्तविक absolute पथ से बदलें।
- Nginx को रीस्टार्ट करें: `sudo systemctl restart nginx`।
- यह पुष्टि करने के लिए `http://localhost/movies/` पर जाएं कि डायरेक्टरी लिस्टिंग काम करती है।

`alias` डायरेक्टिव को सही करके और उचित फ़ाइल परमिशन सुनिश्चित करके, 403 Forbidden एरर हल हो जानी चाहिए, जिससे आप Nginx के माध्यम से अपनी मूवी फोल्डर तक सफलतापूर्वक पहुंच सकते हैं।

---

अपने Nginx 403 Forbidden एरर को ठीक करने और यह सुनिश्चित करने के लिए कि आपकी एकल `.mkv` फ़ाइल एक्सेसिबल है, आइए आपकी स्थिति के आधार पर सेटअप को परिष्कृत करें। आपके पास एक `.mkv` फ़ाइल है, और आप जानना चाहते हैं कि इसे कहाँ सेव करें ताकि Nginx यूजर इस तक पहुंच सके। नीचे, मैं आपको प्रक्रिया के माध्यम से चरण-दर-चरण मार्गदर्शन करूंगा, जिसमें एक उपयुक्त डायरेक्टरी चुनना और Nginx को कॉन्फ़िगर करना शामिल है।

### चरण 1: अपनी `.mkv` फ़ाइल के लिए एक डायरेक्टरी चुनें
जब आपकी `.mkv` फ़ाइल को Nginx द्वारा सर्व करने के लिए स्टोर करने का निर्णय लेते हैं, तो इन कारकों पर विचार करें:
- **एक्सेसिबिलिटी**: Nginx यूजर (जैसे, `nginx` या `www-data`) को पढ़ने की पहुंच की आवश्यकता होती है।
- **कन्वेंशन्स**: वेब सर्वर आमतौर पर मानक डायरेक्टरीज जैसे `/var/www/` या `/srv/www/` से फाइलें सर्व करते हैं।
- **सुरक्षा**: अत्यधिक अनुमति वाले या संवेदनशील स्थानों (जैसे, `/home/user/` जब तक आवश्यक न हो) में फाइलें रखने से बचें।

आपके मामले के लिए, चूंकि यह सिर्फ एक `.mkv` फ़ाइल है और आप स्थानीय रूप से (`localhost`) टेस्ट कर रहे हैं, मैं एक सरल, मानक स्थान के रूप में `/var/www/html/movies/` का उपयोग करने की सलाह देता हूं। इसके कारण यहां बताए गए हैं:
- `/var/www/html/` Nginx और Apache के लिए एक सामान्य डिफ़ॉल्ट वेब रूट है।
- यह आमतौर पर वेब सर्वर यूजर या ग्रुप के स्वामित्व में होता है, जिससे परमिशन प्रबंधित करना आसान हो जाता है।
- `/movies/` सबडायरेक्टरी जोड़ने से चीजें व्यवस्थित रहती हैं।

यदि `/var/www/html/` आपके सिस्टम पर मौजूद नहीं है या उपयुक्त नहीं है, तो विकल्पों में शामिल हैं:
- `/srv/www/movies/` (एक और मानक वेब डायरेक्टरी)।
- `/usr/share/nginx/html/movies/` (कभी-कभी डिफ़ॉल्ट Nginx डॉक्यूमेंट रूट)।

इस उदाहरण के लिए, आइए `/var/www/html/movies/` का उपयोग करें।

### चरण 2: डायरेक्टरी और फ़ाइल सेट अप करें
मान लें कि आप Linux सिस्टम पर हैं, इन चरणों का पालन करें:

1. **डायरेक्टरी बनाएं**:
   ```bash
   sudo mkdir -p /var/www/html/movies
   ```

2. **अपनी `.mkv` फ़ाइल मूव करें**:
   `yourfile.mkv` को आपकी फ़ाइल के वास्तविक नाम से बदलें और इसे डायरेक्टरी में मूव करें:
   ```bash
   sudo mv /path/to/yourfile.mkv /var/www/html/movies/yourfile.mkv
   ```

3. **परमिशन सेट करें**:
   Nginx यूजर (आमतौर पर `nginx` या `www-data`) को फ़ाइल तक पढ़ने की पहुंच और डायरेक्टरी तक एक्ज़िक्यूट एक्सेस की आवश्यकता होती है। सबसे पहले, `/etc/nginx/nginx.conf` चेक करके Nginx यूजर की पहचान करें:
   ```bash
   grep user /etc/nginx/nginx.conf
   ```
   `user nginx;` या `user www-data;` जैसी लाइन देखें। यदि यह निर्दिष्ट नहीं है, तो यह `www-data` (Ubuntu/Debian) या `nginx` (CentOS/RHEL) पर डिफ़ॉल्ट हो सकता है।

   फिर, स्वामित्व समायोजित करें:
   ```bash
   sudo chown -R nginx:nginx /var/www/html/movies
   ```
   यदि अलग है तो `nginx` को `www-data` या वास्तविक यूजर से बदलें।

   उचित परमिशन सुनिश्चित करें:
   ```bash
   sudo chmod -R 755 /var/www/html/movies
   ```
   - `755` का मतलब है कि मालिक (Nginx) के पास पूर्ण पहुंच है, और अन्य (वेब सर्वर प्रक्रिया सहित) डायरेक्टरी को पढ़ और एक्ज़िक्यूट (नेविगेट) कर सकते हैं।

### चरण 3: Nginx कॉन्फ़िगर करें
अपनी Nginx कॉन्फ़िगरेशन को `/var/www/html/movies/` से `.mkv` फ़ाइल सर्व करने के लिए अपडेट करें। यहां एक न्यूनतम कार्यशील कॉन्फ़िग है:

```nginx
server {
    listen 80;
    server_name localhost;

    location /movies/ {
        alias /var/www/html/movies/;
        autoindex on; # यदि आप फाइलें ब्राउज़ करना चाहते हैं तो डायरेक्टरी लिस्टिंग सक्षम करता है
    }
}
```

- **नोट्स**:
  - `/movies/` को सीधे `/var/www/html/movies/` पर मैप करने के लिए `root` के बजाय `alias` का उपयोग करें।
  - `autoindex on;` वैकल्पिक है। यदि आप इसे अक्षम करते हैं (`autoindex off;`), तो इसे एक्सेस करने के लिए आपको सटीक फ़ाइल URL (जैसे, `http://localhost/movies/yourfile.mkv`) निर्दिष्ट करना होगा।

इस कॉन्फ़िग को सेव करें (जैसे, `/etc/nginx/sites-enabled/default` में या एक कस्टम फ़ाइल जैसे `/etc/nginx/conf.d/movies.conf` में), फिर टेस्ट करें और Nginx को रीस्टार्ट करें:
```bash
sudo nginx -t  # सिंटैक्स एरर के लिए कॉन्फ़िग टेस्ट करें
sudo systemctl restart nginx  # बदलाव लागू करें
```

### चरण 4: एक्सेस टेस्ट करें
- अपना ब्राउज़र खोलें और यहां जाएं:
  - `http://localhost/movies/` (यदि `autoindex` ऑन है, तो आप `yourfile.mkv` के साथ एक डायरेक्टरी लिस्टिंग देखेंगे)।
  - `http://localhost/movies/yourfile.mkv` (फ़ाइल को सीधे एक्सेस करने के लिए)।
- यदि यह काम करता है, तो फ़ाइल या तो ब्राउज़र में प्रदर्शित होनी चाहिए (यदि समर्थित है) या डाउनलोड के लिए प्रॉम्प्ट करना चाहिए।

### चरण 5: आवश्यकता होने पर ट्रबलशूट करें
यदि आपको अभी भी 403 एरर मिलती है:
1. **परमिशन चेक करें**:
   ```bash
   ls -l /var/www/html/movies/
   ```
   सुनिश्चित करें कि `yourfile.mkv` पठनीय है (जैसे, `-rwxr-xr-x` या कम से कम `-rw-r--r--`)।

2. **Nginx लॉग्स चेक करें**:
   ```bash
   sudo tail -f /var/log/nginx/error.log
   ```
   "permission denied" जैसे सुराग के लिए देखें।

3. **SELinux (यदि लागू हो)**:
   यदि SELinux वाले सिस्टम पर (जैसे, CentOS), चलाएं:
   ```bash
   sudo setenforce 0  # अस्थायी रूप से अक्षम करें
   ```
   फिर से टेस्ट करें। यदि यह काम करता है, तो SELinux कॉन्टेक्स्ट्स को समायोजित करें:
   ```bash
   sudo chcon -R -t httpd_sys_content_t /var/www/html/movies/
   sudo setenforce 1  # SELinux को फिर से सक्षम करें
   ```

### अंतिम विचार
अपनी `.mkv` फ़ाइल को `/var/www/html/movies/` में सेव करके, सही परमिशन सेट करके, और Nginx में `alias` डायरेक्टिव का उपयोग करके, आपको 403 एरर हल हो जानी चाहिए। यह सेटअप सरल है और वेब सर्वर कन्वेंशन्स का पालन करता है, जिससे Nginx यूजर के लिए फ़ाइल तक पहुंचना आसान हो जाता है। यदि आपको कोई समस्या आती है तो मुझे बताएं!

---

यदि आपको `sudo chown -R nginx:nginx /var/www/movies` के साथ "invalid user" एरर मिलती है, तो इसका मतलब है कि `nginx` यूजर आपके सिस्टम पर मौजूद नहीं है। ऐसा इसलिए होता है क्योंकि Nginx प्रक्रिया एक अलग यूजर के तहत चल सकती है, जैसे `www-data` (Debian/Ubuntu पर आम) या आपके OS के आधार पर कोई अन्य डिफ़ॉल्ट।

### फिक्स: सही यूजर की पहचान करें और उसका उपयोग करें
1. **Nginx यूजर चेक करें**:
   अपनी Nginx कॉन्फ़िगरेशन फ़ाइल (आमतौर पर `/etc/nginx/nginx.conf`) में देखें:
   ```bash
   grep user /etc/nginx/nginx.conf
   ```
   - यदि यह `user www-data;` कहता है, तो `www-data` का उपयोग करें।
   - यदि यह `user nginx;` कहता है, तो `nginx` यूजर मौजूद होना चाहिए (संभवतः मिसकॉन्फ़िगरेशन का संकेत देता है)।
   - यदि कोई `user` डायरेक्टिव नहीं मिलता है, तो डिफ़ॉल्ट यूजर आपके सिस्टम पर निर्भर करता है:
     - Ubuntu/Debian: `www-data`
     - CentOS/RHEL: `nginx`
     - अन्य: `http`, `apache`, या इसी तरह का हो सकता है।

2. **विकल्प: चल रही प्रक्रिया चेक करें**:
   यदि कॉन्फ़िग स्पष्ट नहीं है, तो देखें कि Nginx किस यूजर के रूप में चल रहा है:
   ```bash
   ps aux | grep nginx
   ```
   वर्कर प्रक्रियाओं के लिए पहले कॉलम (जैसे, `www-data` या `nginx`) को देखें।

3. **सही यूजर लागू करें**:
   मान लें कि यह `www-data` है (Ubuntu/Debian पर आम), कमांड अपडेट करें:
   ```bash
   sudo chown -R www-data:www-data /var/www/movies
   ```
   यदि यह कोई अलग यूजर है (जैसे, `http`), तो उपयोग करें:
   ```bash
   sudo chown -R http:http /var/www/movies
   ```

4. **परमिशन सेट करें**:
   स्वामित्व ठीक करने के बाद, उचित परमिशन सुनिश्चित करें:
   ```bash
   sudo chmod -R 755 /var/www/movies
   ```

5. **रीस्टार्ट करें और टेस्ट करें**:
   ```bash
   sudo systemctl restart nginx
   ```
   `http://localhost/movies/yourfile.mkv` पर जाएं।

### यदि अभी भी अटके हैं
यदि न तो `nginx` और न ही `www-data` काम करता है और प्रक्रिया चेक मदद नहीं करता है, तो मुझे अपना OS (जैसे, Ubuntu, CentOS) या `ps aux | grep nginx` का आउटपुट बताएं, और मैं आपके लिए सटीक यूजर का पता लगा दूंगा!

---

यदि `autoindex on;` सेट है लेकिन `http://localhost/movies/` अभी भी फाइलों की सूची नहीं दे रहा है, तो आइए इसे चरण-दर-चरण ट्रबलशूट और ठीक करें। चूंकि आपको डायरेक्टरी लिस्टिंग नहीं दिख रही है, यह कॉन्फ़िग समस्या, परमिशन समस्या, या कुछ और ब्लॉक कर सकता है। इसे हल करने का तरीका यहां बताया गया है:

### 1. अपनी कॉन्फ़िग सत्यापित करें
सुनिश्चित करें कि आपकी Nginx कॉन्फ़िगरेशन सही और लागू है। यह इस तरह दिखनी चाहिए:

```nginx
server {
    listen 80;
    server_name localhost;

    location /movies/ {
        alias /var/www/movies/;
        autoindex on;
    }
}
```

- **सामान्य गलतियाँ**:
  - `alias` पथ में टाइपो (जैसे, `/var/www/movie/` के बजाय `/var/www/movies/`)।
  - `autoindex on;` गलत ब्लॉक में है या कमेंट आउट है।
  - कॉन्फ़िग फ़ाइल लोड नहीं हो रही है (जैसे, गलत स्थान पर सेव की गई)।

चेक करें कि आपकी कॉन्फ़िग कहां है:
- यदि यह `/etc/nginx/sites-enabled/` में है, तो सुनिश्चित करें कि यह ठीक से लिंक है (जैसे, `ls -l /etc/nginx/sites-enabled/`)।
- यदि यह `/etc/nginx/conf.d/` में है, तो सुनिश्चित करें कि यह `.conf` में समाप्त होती है (जैसे, `movies.conf`)।

टेस्ट करें और रीलोड करें:
```bash
sudo nginx -t
sudo systemctl reload nginx  # डाउनटाइम से बचने के लिए रीस्टार्ट के बजाय रीलोड करें
```

### 2. पुष्टि करें कि फाइलें मौजूद हैं
चेक करें कि `/var/www/movies/` में आपकी `.mkv` फ़ाइल है:
```bash
ls -l /var/www/movies/
```
- यदि यह खाली है, तो अपनी फ़ाइल वहां मूव करें:
  ```bash
  sudo mv /path/to/yourfile.mkv /var/www/movies/
  ```
- यदि यह खाली नहीं है, तो टेस्टिंग के लिए फ़ाइल नाम नोट करें।

### 3. परमिशन चेक करें
Nginx को डायरेक्टरी और फाइलों तक पढ़ने (`r`) और एक्ज़िक्यूट (`x`) एक्सेस की आवश्यकता होती है। सत्यापित करें:
```bash
ls -ld /var/www/movies/
ls -l /var/www/movies/
```
- आउटपुट इस तरह दिखना चाहिए:
  ```
  drwxr-xr-x 2 www-data www-data 4096 Mar 15 14:00 /var/www/movies/
  -rw-r--r-- 1 www-data www-data 123456 Mar 15 14:00 yourfile.mkv
  ```
- आवश्यकता होने पर ठीक करें (`www-data` को अपने Nginx यूजर से बदलें):
  ```bash
  sudo chown -R www-data:www-data /var/www/movies/
  sudo chmod -R 755 /var/www/movies/
  ```

### 4. लॉग्स चेक करें
सुराग के लिए Nginx एरर लॉग देखें:
```bash
sudo tail -n 20 /var/log/nginx/error.log
```
- **"permission denied"**: परमिशन समस्या का संकेत देता है—चरण 3 को दोबारा चेक करें।
- **"directory index forbidden"**: सुझाव देता है कि `autoindex` काम नहीं कर रहा है—कॉन्फ़िग को दोबारा चेक करें।
- कोई प्रासंगिक एरर नहीं: इसका मतलब हो सकता है कि Nginx सही लोकेशन ब्लॉक को हिट नहीं कर रहा है।

### 5. डायरेक्ट एक्सेस टेस्ट करें
एक विशिष्ट फ़ाइल तक पहुंचने का प्रयास करें:
- `http://localhost/movies/yourfile.mkv` पर जाएं।
- यदि यह काम करता है लेकिन `/movies/` नहीं करता है, तो `autoindex` या डायरेक्टरी सेटअप समस्या है।

### 6. सामान्य फिक्सेस
- **ट्रेलिंग स्लैश**: सुनिश्चित करें कि `alias` `/` (`/var/www/movies/`) के साथ समाप्त होता है, और `http://localhost/movies` और `http://localhost/movies/` दोनों का टेस्ट करें।
- **SELinux (यदि CentOS/RHEL पर)**:
  ```bash
  sudo setenforce 0  # अस्थायी रूप से अक्षम करें
  ```
  फिर से टेस्ट करें। यदि यह काम करता है, तो लागू करें:
  ```bash
  sudo chcon -R -t httpd_sys_content_t /var/www/movies/
  sudo setenforce 1
  ```
- **कन्फ्लिक्टिंग लोकेशन ब्लॉक्स**: यदि कोई अन्य `location` ब्लॉक (जैसे, `location /`) `/movies/` को ओवरराइड करता है, तो इस ब्लॉक को कॉन्फ़िग में उच्चतर ले जाएं या इसे और अधिक विशिष्ट बनाएं।

### 7. न्यूनतम टेस्ट कॉन्फ़िग
यदि यह अभी भी काम नहीं कर रहा है, तो समस्या को अलग करने के लिए सरल बनाएं। अपनी कॉन्फ़िग को इससे बदलें:

```nginx
server {
    listen 80;
    server_name localhost;

    location /movies/ {
        alias /var/www/movies/;
        autoindex on;
    }
}
```
