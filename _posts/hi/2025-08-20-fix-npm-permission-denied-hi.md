आपको जो त्रुटि मिल रही है, `EACCES: permission denied`, का मतलब है कि आपके वर्तमान उपयोगकर्ता के पास `/usr/local/lib/node_modules` डायरेक्टरी में **लिखने की अनुमति नहीं है**। यह Linux या macOS पर global npm पैकेज इंस्टॉल करते समय एक आम समस्या है।

### **इस समस्या को कैसे ठीक करें**

#### **विकल्प 1: `sudo` का उपयोग करें (नियमित उपयोग के लिए अनुशंसित नहीं)**
आप कमांड को root उपयोगकर्ता के रूप में चलाने के लिए `sudo` का उपयोग कर सकते हैं, लेकिन यह नियमित उपयोग के लिए **अनुशंसित नहीं** है, क्योंकि इससे बाद में npm पैकेज के साथ अनुमति संबंधी समस्याएं हो सकती हैं।

```bash
sudo npm install -g @anthropic-ai/claude-code
```

#### **विकल्प 2: npm की डिफ़ॉल्ट डायरेक्टरी बदलें (अनुशंसित)**
सबसे अच्छा तरीका यह है कि npm को आपके होम फोल्डर में एक ऐसी डायरेक्टरी का उपयोग करने के लिए कॉन्फ़िगर करें जहां आपके पास लिखने की अनुमति हो।

1. **अपने होम फोल्डर में एक global npm डायरेक्टरी बनाएँ:**
   ```bash
   mkdir ~/.npm-global
   ```
2. **npm को इस डायरेक्टरी का उपयोग करने के लिए कॉन्फ़िगर करें:**
   ```bash
   npm config set prefix '~/.npm-global'
   ```
3. **नई डायरेक्टरी को अपने `PATH` में जोड़ें:**
   अपनी shell कॉन्फ़िगरेशन फ़ाइल (जैसे, `~/.bashrc`, `~/.zshrc`, या `~/.profile`) खोलें और यह लाइन जोड़ें:
   ```bash
   export PATH=~/.npm-global/bin:$PATH
   ```
   फिर, अपनी shell को रीलोड करें:
   ```bash
   source ~/.bashrc  # or ~/.zshrc, ~/.profile
   ```
4. **इंस्टालेशन फिर से आज़माएँ:**
   ```bash
   npm install -g @anthropic-ai/claude-code
   ```

#### **विकल्प 3: `/usr/local` के लिए अनुमतियाँ ठीक करें (उन्नत)**
यदि आप `/usr/local` का उपयोग जारी रखना चाहते हैं, तो आप इसका स्वामित्व अपने उपयोगकर्ता को दे सकते हैं:
```bash
sudo chown -R $(whoami) /usr/local/lib/node_modules
```
फिर इंस्टालेशन फिर से आज़माएँ।
