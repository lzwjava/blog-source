## FMDB की परिचय

FMDB (Flying Meat Database) एक Objective-C wrapper है जो SQLite के आसपास है और database operations को आसान और अधिक efficient बनाता है। यह SQLite databases के साथ interact करने का एक convenient तरीका प्रदान करता है बिना raw SQLite APIs के complexities से deal करने के. FMDB iOS development में simplicity और performance के लिए widely used है.

## FMDB क्यों use करें?

- **Simplicity**: FMDB database operations को easy-to-use methods के साथ simplify करता है.
- **Performance**: यह performance के लिए optimize किया गया है, making it suitable for mobile applications.
- **Thread Safety**: FMDB thread-safe database operations प्रदान करता है.
- **Active Community**: Being widely used, FMDB एक active community और plenty of resources ke साथ है.

## FMDB के साथ शुरू करना

### Step 1: FMDB install करें

आप FMDB ko CocoaPods, Carthage, या Swift Package Manager ke साथ install कर सकते हैं. यहाँ, हम installation के लिए CocoaPods use करेंगे.

1. **FMDB ko आपke Podfile में add करें**:
   ```ruby
   platform :ios, '10.0'
   use_frameworks!

   target 'YourAppTarget' do
     pod 'FMDB', '~> 2.7'
   end
   ```

2. **Pods install करें**:
   ```bash
   pod install
   ```

### Step 2: FMDB import करें

FMDB ko आपke Swift या Objective-C files में import करें:

```swift
import FMDB
```

### Step 3: Basic Database Operations

#### Database खोलना

```swift
let fileURL = try! FileManager.default
    .url(for: .documentDirectory, in: .userDomainMask, appropriateFor: nil, create: false)
    .appendingPathComponent("database.sqlite")

let database = FMDatabase(url: fileURL)

if database.open() {
    print("Database खोलने में सफल")
} else {
    print("Database खोलने में असमर्थ")
}
```

#### Table बनाना

```swift
let createTableQuery = """
CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT,
    age INTEGER
)
"""

if database.executeStatements(createTableQuery) {
    print("Table बनने में सफल")
} else {
    print("Table बनने में असमर्थ")
}
```

#### Data insert करना

```swift
let insertQuery = "INSERT INTO users (name, age) VALUES (?, ?)"

if database.executeUpdate(insertQuery, withArgumentsIn: ["Alice", 30]) {
    print("Data insert करने में सफल")
} else {
    print("Data insert करने में असमर्थ")
}
```

#### Data query करना

```swift
let selectQuery = "SELECT * FROM users"

if let results = database.executeQuery(selectQuery, withArgumentsIn: []) {
    while results.next() {
        let id = results.int(forColumn: "id")
        let name = results.string(forColumn: "name")
        let age = results.int(forColumn: "age")
        print("User ID: \(id), Name: \(name ?? ""), Age: \(age)")
    }
} else {
    print("Data fetch करने में असमर्थ")
}
```

#### Data update करना

```swift
let updateQuery = "UPDATE users SET age = ? WHERE name = ?"

if database.executeUpdate(updateQuery, withArgumentsIn: [31, "Alice"]) {
    print("Data update करने में सफल")
} else {
    print("Data update करने में असमर्थ")
}
```

#### Data delete करना

```swift
let deleteQuery = "DELETE FROM users WHERE name = ?"

if database.executeUpdate(deleteQuery, withArgumentsIn: ["Alice"]) {
    print("Data delete करने में सफल")
} else {
    print("Data delete करने में असमर्थ")
}
```

### Step 4: Database बंद करना

Database को जब आपका use हो jaye तो resources ko free करने के लिए हमेशा database ko band करें.

## Best Practices

- **Thread Safety**: Thread-safe operations के लिए `FMDatabaseQueue` use करें.
- **Error Handling**: हमेशा errors ko check करें aur unhe appropriately handle करें.
- **Resource Management**: Database ko band करें जब वो zaroori nahi rahe.
- **Performance**: Batch operations ke लिए transactions use करें performance ko improve करने ke लिए.
