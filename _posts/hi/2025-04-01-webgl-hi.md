आइए WebGL (वेब ग्राफिक्स लाइब्रेरी) की एक व्यापक गाइड में गोता लगाएँ। इसमें शामिल होगा कि यह क्या है, यह कैसे काम करता है, यह उपयोगी क्यों है, इसे शुरू कैसे करें, और इसका व्यापक संदर्भ—यह Figma जैसे टूल्स में इसकी भूमिका को समझने के लिए एकदम सही है। तैयार हो जाइए!

---

### **1. WebGL क्या है?**
WebGL एक JavaScript API है जो हार्डवेयर-एक्सेलेरेटेड 2D और 3D ग्राफिक्स को वेब पर लाता है। यह OpenGL ES (एम्बेडेड सिस्टम्स) पर आधारित है, जो नेटिव ऐप्स में इस्तेमाल होने वाले OpenGL स्टैंडर्ड का एक सबसेट है, जिसे ब्राउज़रों के लिए अनुकूलित किया गया है। WebGL के साथ, आप पारंपरिक HTML/CSS के स्लो CPU-आधारित रेंडरिंग को बायपास करते हुए, GPU का उपयोग करके सीधे एक `<canvas>` एलिमेंट पर ग्राफिक्स रेंडर कर सकते हैं।

- **मुख्य विशेषताएँ:**
  - **हार्डवेयर एक्सेलेरेशन:** तेज रेंडरिंग के लिए GPU का लाभ उठाता है।
  - **क्रॉस-प्लेटफॉर्म:** सभी प्रमुख ब्राउज़रों (Chrome, Firefox, Safari, Edge) में चलता है।
  - **प्रोग्राम करने योग्य:** चीजों के दिखने और हिलने-डुलने का तरीका परिभाषित करने के लिए शेडर्स (छोटे प्रोग्राम) का उपयोग करता है।
  - **कोई प्लगइन नहीं:** Flash या पुरानी टेक्नोलॉजी के विपरीत, ब्राउज़र में नेटिव।

- **इतिहास:**
  - 2011 में Khronos Group (OpenGL के पीछे के लोग) द्वारा पेश किया गया।
  - WebGL 1.0, OpenGL ES 2.0 पर आधारित है; WebGL 2.0 (2017) OpenGL ES 3.0 पर बनता है।

- **उपयोग के मामले:**
  - गेम्स (जैसे, ब्राउज़र-आधारित 3D गेम)।
  - डेटा विज़ुअलाइज़ेशन (जैसे, इंटरैक्टिव चार्ट या मैप)।
  - क्रिएटिव टूल्स (जैसे, Figma की वेक्टर रेंडरिंग)।
  - सिमुलेशन (जैसे, फिजिक्स इंजन या 3D मॉडलिंग)।

### **2. WebGL कैसे काम करता है?**
WebGL एक JavaScript API के माध्यम से GPU को लो-लेवल एक्सेस देकर काम करता है। यह एक **रेंडरिंग पाइपलाइन** के इर्द-गिर्द बना है—कोड को स्क्रीन पर पिक्सेल में बदलने वाले चरणों की एक श्रृंखला।

1. **सेटअप:**
   - आप HTML में एक `<canvas>` एलिमेंट बनाते हैं।
   - एक WebGL कॉन्टेक्स्ट प्राप्त करें: `canvas.getContext('webgl')` (या 2.0 के लिए `'webgl2'`)।

2. **शेडर्स:**
   - **वर्टेक्स शेडर:** 3D स्पेस में पॉइंट्स (वर्टिसीज़) की पोजीशन को परिभाषित करता है।
   - **फ्रैगमेंट शेडर:** वर्टिसीज़ के बीच प्रत्येक पिक्सेल को रंग देता है।
   - GLSL (OpenGL शेडिंग लैंग्वेज) में लिखे जाते हैं, रनटाइम पर कंपाइल किए जाते हैं।

3. **बफ़र्स:**
   - डेटा (जैसे, वर्टेक्स पोजीशन, रंग) GPU मेमोरी में बफ़र्स के माध्यम से स्टोर किया जाता है।
   - उदाहरण: एक त्रिकोण के तीन कोनों को निर्देशांक की एक ऐरे के रूप में भेजा जाता है।

4. **रेंडरिंग:**
   - आप बफ़र्स को बाइंड करते हैं, शेडर्स सेट अप करते हैं, और एक ड्रॉ कॉल जारी करते हैं (जैसे, `gl.drawArrays()`)।
   - GPU इसे समानांतर रूप से प्रोसेस करता है, आउटपुट कैनवास पर देता है।

- **कोऑर्डिनेट सिस्टम:** WebGL एक सामान्यीकृत 3D स्पेस (x, y, z एक्सेस पर -1 से 1) का उपयोग करता है, जिसे आप मैट्रिक्स (जैसे, परिप्रेक्ष्य या रोटेशन के लिए) के साथ ट्रांसफॉर्म करते हैं।

- **स्टेट मशीन:** WebGL स्टेटलेस है—आप प्रत्येक ड्रॉ कॉल से पहले पैरामीटर (जैसे, टेक्सचर्स, ब्लेंडिंग) सेट करते हैं।

### **3. WebGL क्यों Use करें?**
- **परफॉर्मेंस:** जटिल ग्राफिक्स के लिए GPU एक्सेलेरेशन CPU रेंडरिंग को मात देता है।
- **लचीलापन:** शेडर्स आपको दृश्यों को पिक्सेल स्तर तक कस्टमाइज़ करने देते हैं।
- **वेब इंटीग्रेशन:** JavaScript, DOM इवेंट्स, और अन्य APIs के साथ सहजता से काम करता है।
- **कोई इंस्टॉलेशन नहीं:** जहाँ भी ब्राउज़र चलता है, वहाँ चलता है।

**ट्रेड-ऑफ:**
- सीखने की अवस्था कठिन—HTML5 Canvas 2D की तुलना में लो-लेवल।
- डीबगिंग मुश्किल है (GLSL एरर रहस्यमय होते हैं)।
- ब्राउज़र कंपैटिबिलिटी अलग-अलग हो सकती है (खासकर WebGL 2.0)।

### **4. WebGL के साथ शुरुआत करना**
आइए एक साधारण रंगीन त्रिकोण रेंडर करके WebGL को एक्शन में देखें।

#### **स्टेप 1: HTML सेटअप**
```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
</head>
<body>
    <canvas id="glCanvas" width="400" height="400"></canvas>
    <script>
        // JavaScript यहाँ जाएगी
    </script>
</body>
</html>
```

#### **स्टेप 2: JavaScript बेसिक्स**
`<script>` टैग के अंदर यह जोड़ें:
```javascript
// कैनवास और कॉन्टेक्स्ट प्राप्त करें
const canvas = document.getElementById('glCanvas');
const gl = canvas.getContext('webgl');
if (!gl) {
    alert('WebGL सपोर्टेड नहीं है!');
}

// वर्टेक्स शेडर सोर्स (GLSL)
const vsSource = `
    attribute vec2 a_position;
    void main() {
        gl_Position = vec4(a_position, 0.0, 1.0);
    }
`;

// फ्रैगमेंट शेडर सोर्स (GLSL)
const fsSource = `
    void main() {
        gl_FragColor = vec4(1.0, 0.0, 0.0, 1.0); // लाल
    }
`;

// शेडर्स कंपाइल करें
function createShader(gl, type, source) {
    const shader = gl.createShader(type);
    gl.shaderSource(shader, source);
    gl.compileShader(shader);
    if (!gl.getShaderParameter(shader, gl.COMPILE_STATUS)) {
        console.error(gl.getShaderInfoLog(shader));
        gl.deleteShader(shader);
        return null;
    }
    return shader;
}

const vertexShader = createShader(gl, gl.VERTEX_SHADER, vsSource);
const fragmentShader = createShader(gl, gl.FRAGMENT_SHADER, fsSource);

// एक प्रोग्राम में लिंक करें
const program = gl.createProgram();
gl.attachShader(program, vertexShader);
gl.attachShader(program, fragmentShader);
gl.linkProgram(program);
if (!gl.getProgramParameter(program, gl.LINK_STATUS)) {
    console.error(gl.getProgramInfoLog(program));
}

// वर्टेक्स डेटा (एक त्रिकोण)
const positions = new Float32Array([
    0.0,  0.5,  // ऊपर
   -0.5, -0.5,  // नीचे-बाएँ
    0.5, -0.5   // नीचे-दाएँ
]);

// बफ़र सेटअप
const positionBuffer = gl.createBuffer();
gl.bindBuffer(gl.ARRAY_BUFFER, positionBuffer);
gl.bufferData(gl.ARRAY_BUFFER, positions, gl.STATIC_DRAW);

// WebGL को बताएं कि बफ़र को कैसे पढ़ना है
const positionLocation = gl.getAttribLocation(program, 'a_position');
gl.enableVertexAttribArray(positionLocation);
gl.vertexAttribPointer(positionLocation, 2, gl.FLOAT, false, 0, 0);

// क्लियर करें और ड्रॉ करें
gl.clearColor(0.0, 0.0, 0.0, 1.0); // काली बैकग्राउंड
gl.clear(gl.COLOR_BUFFER_BIT);
gl.useProgram(program);
gl.drawArrays(gl.TRIANGLES, 0, 3); // 3 वर्टिसीज़ ड्रॉ करें
```

#### **स्टेप 3: इसे टेस्ट करें**
- इसे `index.html` के रूप में सेव करें, इसे सर्व करें (जैसे, `python -m http.server 8080`), और ब्राउज़र में खोलें।
- आपको काले बैकग्राउंड पर एक लाल त्रिकोण दिखाई देगा।

### **5. कोर कॉन्सेप्ट्स**
- **शेडर्स:** वर्टेक्स शेडर ज्योमेट्री की पोजीशन तय करते हैं; फ्रैगमेंट शेडर इसे रंगते हैं। GLSL, C जैसी है लेकिन GPU-स्पेसिफिक है।
- **बफ़र्स:** वर्टेक्स डेटा (पोजीशन, नॉर्मल्स, रंग) को GPU मेमोरी में स्टोर करते हैं।
- **टेक्सचर्स:** 3D ऑब्जेक्ट्स पर मैप की गई 2D इमेजेस (जैसे, `gl.texImage2D()`)।
- **मैट्रिक्स:** ट्रांसफॉर्मेशन (ट्रांसलेशन, रोटेशन, स्केलिंग) के लिए `glMatrix` जैसी लाइब्रेरीज़ के माध्यम से उपयोग किए जाते हैं।
- **ड्रॉ कॉल:** साधारण आकृतियों के लिए `gl.drawArrays()`; इंडेक्स्ड ज्योमेट्री के लिए `gl.drawElements()`।

**WebGL 2.0 में जोड़:**
- बेहतर टेक्सचर्स (जैसे, 3D टेक्सचर्स)।
- मल्टीपल रेंडर टारगेट्स।
- एन्हांस्ड शेडर फीचर्स (जैसे, यूनिफ़ॉर्म बफ़र ऑब्जेक्ट्स)।

### **6. इकोसिस्टम और टूल्स**
- **लाइब्रेरीज़:**
  - **Three.js:** हाई-लेवल 3D फ्रेमवर्क (raw WebGL कॉम्प्लेक्सिटी को छुपाता है)।
  - **Babylon.js:** गेम-फोकस्ड 3D इंजन।
  - **PixiJS:** WebGL फॉलबैक के साथ 2D रेंडरिंग।
  - **glMatrix:** ट्रांसफॉर्मेशन के लिए मैथ यूटिलिटीज़।

- **ब्राउज़र सपोर्ट:**
  - WebGL 1.0: लगभग सार्वभौमिक।
  - WebGL 2.0: अधिकांश मॉडर्न ब्राउज़र (caniuse.com चेक करें)।

- **डीबगिंग:**
  - ब्राउज़र डेव टूल्स (जैसे, Chrome का WebGL इंस्पेक्टर)।
  - Spector.js जैसे एक्सटेंशन।

### **7. WebGL व्यवहार में**
- **Figma:** वेक्टर ग्राफिक्स रेंडरिंग के लिए WebGL का उपयोग करता है, जो कंप्यूटेशन के लिए WebAssembly के साथ जोड़ा जाता है।
- **गेम्स:** Unity और Unreal Engine ब्राउज़र प्ले के लिए WebGL को एक्सपोर्ट करते हैं।
- **मैप्स:** Google Maps और Mapbox स्मूद ज़ूमिंग और 3D टेरेन के लिए इसका उपयोग करते हैं।

### **8. भविष्य**
- **WebGPU:** नेक्स्ट-जनरेशन API (Vulkan/Metal पर आधारित) उभर रहा है, जो बेहतर परफॉर्मेंस और मॉडर्न फीचर्स का वादा करता है। यह अभी तक रिप्लेसमेंट नहीं है बल्कि एक पूरक है।
- **व्यापक अपनाना:** WebGL अभी भी प्रमुख है, खासकर WebGL 2.0 के अपग्रेड्स के साथ।
