आपके सामने आने वाले त्रुटि संदेश Clash VPN एप्लिकेशन के साथ एक **मिश्रित सामग्री (mixed content)** की समस्या को इंगित करते हैं, विशेष रूप से जब पृष्ठ `https://clash.razord.top/` एक असुरक्षित `http://127.0.0.1:9090/` एंडपॉइंट (जैसे `http://127.0.0.1:9090/version`) से संसाधन लोड करने का प्रयास करता है। आधुनिक ब्राउज़र ऐसे अनुरोधों को ब्लॉक कर देते हैं क्योंकि वे एक सुरक्षित HTTPS पृष्ठ पर असुरक्षित HTTP सामग्री लोड करने का प्रयास करके सुरक्षा नीतियों का उल्लंघन करते हैं। इसके अतिरिक्त, एक्सेस कंट्रोल प्रतिबंधों के कारण CORS (क्रॉस-ओरिजिन रिसोर्स शेयरिंग) त्रुटियां भी हैं। इस समस्या को हल करने के लिए एक चरण-दर-चरण मार्गदर्शिका नीचे दी गई है।

---

### यह समस्या क्यों होती है
- **मिश्रित सामग्री त्रुटि**: वेबपेज HTTPS पर परोसा जाता है, लेकिन यह `http://127.0.0.1:9090` से संसाधन (जैसे वर्जन चेक) प्राप्त करने का प्रयास कर रहा है, जो असुरक्षित है। संभावित सुरक्षा कमजोरियों, जैसे मैन-इन-द-मिडिल अटैक, को रोकने के लिए ब्राउज़र इन अनुरोधों को ब्लॉक कर देते हैं।
- **CORS त्रुटि**: ब्राउज़र CORS नीति के कारण `http://127.0.0.1:9090` के अनुरोध को ब्लॉक कर रहा है, जो सर्वर द्वारा स्पष्ट रूप से अनुमति दिए जाने तक क्रॉस-ओरिजिन अनुरोधों को प्रतिबंधित करती है।
- **Clash संदर्भ**: Clash (या Clash for Windows) एक प्रॉक्सी एप्लिकेशन है जो संभवतः अपने डैशबोर्ड या API के लिए एक स्थानीय सर्वर (`127.0.0.1:9090`) का उपयोग करता है। यदि यह स्थानीय सर्वर HTTPS का समर्थन नहीं करता है या सही ढंग से कॉन्फ़िगर नहीं है, तो जब एक HTTPS वेबपेज के माध्यम से इस तक पहुंचा जाता है, तो ये त्रुटियां ट्रिगर होती हैं।

### समस्या को ठीक करने के चरण

#### 1. **Clash Core कॉन्फ़िगरेशन सत्यापित करें**
   - **जांचें कि क्या Clash Core चल रहा है**: सुनिश्चित करें कि Clash core (बैकएंड सेवा) आपके मशीन पर चल रही है और `127.0.0.1:9090` पर सुन रही है। आप इसे इस प्रकार सत्यापित कर सकते हैं:
     - एक टर्मिनल या कमांड प्रॉम्प्ट खोलें।
     - `curl http://127.0.0.1:9090/version` चलाकर यह जांचें कि एंडपॉइंट Clash वर्जन के साथ प्रतिक्रिया देता है या नहीं।
     - यदि यह प्रतिक्रिया नहीं देता है, तो सुनिश्चित करें कि Clash सेवा सक्रिय है। Clash for Windows या Clash core प्रक्रिया को पुनरारंभ करें।
   - **Clash Core के लिए HTTPS सक्षम करें** (यदि संभव हो):
     - Clash के कुछ संस्करण (जैसे Clash Premium या Clash Meta) स्थानीय API के लिए HTTPS का समर्थन करते हैं। बाहरी नियंत्रक (API एंडपॉइंट) के लिए HTTPS सक्षम करने के विकल्प के लिए Clash दस्तावेज़ीकरण या कॉन्फ़िगरेशन फ़ाइल (आमतौर पर `config.yaml`) जांचें।
     - कॉन्फ़िगरेशन फ़ाइल में `external-controller` या `external-ui` जैसी सेटिंग देखें। उदाहरण के लिए:
       ```yaml
       external-controller: 127.0.0.1:9090
       external-ui: <path-to-ui>
       ```
       यदि HTTPS समर्थित है, तो आपको स्थानीय सर्वर के लिए एक प्रमाणपत्र कॉन्फ़िगर करने की आवश्यकता हो सकती है। यह उन्नत है और एक स्व-हस्ताक्षरित प्रमाणपत्र (self-signed certificate) बनाने की आवश्यकता हो सकती है (नीचे चरण 4 देखें)।

#### 2. **डैशबोर्ड को HTTP के माध्यम से एक्सेस करें (अस्थायी समाधान)**
   - यदि Clash डैशबोर्ड को HTTP के माध्यम से एक्सेस किया जा सकता है (जैसे `http://clash.razord.top/` HTTPS के बजाय), तो मिश्रित सामग्री समस्याओं से बचने के लिए इसे HTTPS के बिना लोड करने का प्रयास करें:
     - अपना ब्राउज़र खोलें और `http://clash.razord.top/` पर नेविगेट करें।
     - ध्यान दें: यह प्रोडक्शन उपयोग के लिए अनुशंसित नहीं है, क्योंकि HTTP असुरक्षित है। इसका उपयोग केवल परीक्षण के लिए या यदि डैशबोर्ड केवल स्थानीय रूप से एक्सेस किया जाता है तो करें।
   - यदि डैशबोर्ड के लिए HTTPS आवश्यक है, तो मूल कारण को संबोधित करने के लिए अगले चरणों पर आगे बढ़ें।

#### 3. **URL को HTTPS में अपडेट करें**
   - त्रुटि बताती है कि Clash डैशबोर्ड `http://127.0.0.1:9090` से संसाधन प्राप्त करने का प्रयास कर रहा है। यदि आपके पास Clash डैशबोर्ड के स्रोत कोड या कॉन्फ़िगरेशन तक पहुंच है:
     - फ्रंटएंड कोड (जैसे `index-5e90ca00.js` या `vendor-827b5617.js`) में हार्डकोडेड `http://127.0.0.1:9090` संदर्भों की जांच करें।
     - यदि Clash core HTTPS का समर्थन करता है तो इन्हें `https://127.0.0.1:9090` में अपडेट करें, या एक रिलेटिव URL (जैसे `/version`) का उपयोग करें ताकि ब्राउज़र पृष्ठ के समान प्रोटोकॉल का उपयोग कर सके।
     - यदि आपके पास स्रोत कोड तक पहुंच नहीं है, तो आपको एक रिवर्स प्रॉक्सी कॉन्फ़िगर करने की आवश्यकता हो सकती है (चरण 4 देखें)।

#### 4. **HTTPS के साथ एक रिवर्स प्रॉक्सी सेट अप करें**
   - मिश्रित सामग्री समस्या को हल करने के लिए, आप Clash core API (`http://127.0.0.1:9090`) को HTTPS पर परोसने के लिए एक रिवर्स प्रॉक्सी (जैसे Nginx या Caddy का उपयोग करके) सेट कर सकते हैं। यह डैशबोर्ड को कोर के साथ सुरक्षित रूप से संचार करने की अनुमति देता है।
   - **Nginx के लिए चरण**:
     1. अपने सिस्टम पर Nginx इंस्टॉल करें (यदि पहले से इंस्टॉल नहीं है)।
     2. `127.0.0.1` के लिए एक स्व-हस्ताक्षरित SSL प्रमाणपत्र (self-signed SSL certificate) जनरेट करें:
        ```bash
        openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout localhost.key -out localhost.crt -subj "/CN=localhost"
        ```
     3. Nginx को `http://127.0.0.1:9090` के अनुरोधों को HTTPS पर प्रॉक्सी करने के लिए कॉन्फ़िगर करें। एक कॉन्फ़िगरेशन फ़ाइल बनाएं (जैसे `/etc/nginx/sites-available/clash`):
        ```nginx
        server {
            listen 443 ssl;
            server_name localhost;

            ssl_certificate /path/to/localhost.crt;
            ssl_certificate_key /path/to/localhost.key;

            location / {
                proxy_pass http://127.0.0.1:9090;
                proxy_set_header Host $host;
                proxy_set_header X-Real-IP $remote_addr;
                proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
                proxy_set_header X-Forwarded-Proto $scheme;
            }
        }
        ```
     4. कॉन्फ़िगरेशन को सक्षम करें और Nginx को पुनरारंभ करें:
        ```bash
        sudo ln -s /etc/nginx/sites-available/clash /etc/nginx/sites-enabled/
        sudo nginx -t
        sudo systemctl restart nginx
        ```
     5. API अनुरोधों के लिए Clash डैशबोर्ड को `http://127.0.0.1:9090` के बजाय `https://localhost:443` का उपयोग करने के लिए अपडेट करें।
     6. अपने ब्राउज़र में, संकेत मिलने पर स्व-हस्ताक्षरित प्रमाणपत्र (self-signed certificate) को स्वीकार करें।

   - **Caddy के साथ विकल्प**: Caddy कॉन्फ़िगर करने में सरल है और स्वचालित रूप से HTTPS को संभालता है:
     1. Caddy इंस्टॉल करें।
     2. एक `Caddyfile` बनाएं:
        ```caddy
        localhost:443 {
            reverse_proxy http://127.0.0.1:9090
        }
        ```
     3. Caddy चलाएं: `caddy run`.
     4. Clash डैशबोर्ड को `https://localhost:443` का उपयोग करने के लिए अपडेट करें।

#### 5. **CORS प्रतिबंधों को बायपास करें (उन्नत)**
   - CORS त्रुटि (`XMLHttpRequest cannot load http://127.0.0.1:9090/version due to access control checks`) इंगित करती है कि Clash core सर्वर उचित CORS हेडर नहीं भेज रहा है। यदि आप Clash core को नियंत्रित करते हैं:
     - CORS हेडर शामिल करने के लिए Clash core कॉन्फ़िगरेशन को संशोधित करें, जैसे:
       ```yaml
       external-controller: 127.0.0.1:9090
       allow-cors: true
       ```
       (सटीक सिंटैक्स के लिए Clash दस्तावेज़ीकरण जांचें, क्योंकि यह Clash वर्जन पर निर्भर करता है।)
     - वैकल्पिक रूप से, चरण 4 में रिवर्स प्रॉक्सी सेटअप हेडर जोड़कर CORS को संभाल सकता है:
       ```nginx
       add_header Access-Control-Allow-Origin "https://clash.razord.top";
       add_header Access-Control-Allow-Methods "GET, POST, OPTIONS";
       add_header Access-Control-Allow-Headers "*";
       ```
   - यदि आप core को नियंत्रित नहीं करते हैं, तो आप अस्थायी रूप से CORS को बायपास करने के लिए एक ब्राउज़र एक्सटेंशन (जैसे Chrome के लिए "CORS Unblock") का उपयोग कर सकते हैं, लेकिन सुरक्षा कारणों से यह अनुशंसित नहीं है।

#### 6. **Clash अपडेट करें या एक संगत संस्करण पर स्विच करें**
   - सुनिश्चित करें कि आप Clash for Windows या Clash Verge के नवीनतम संस्करण का उपयोग कर रहे हैं, क्योंकि पुराने संस्करणों में बग हो सकते हैं या बाहरी नियंत्रक के लिए HTTPS समर्थन का अभाव हो सकता है।
   - मिश्रित सामग्री या CORS से संबंधित रिपोर्ट की गई समस्याओं के लिए Clash GitHub रिपॉजिटरी (`github.com/Dreamacro/clash` या `github.com/Fndroid/clash_for_windows_pkg`) जांचें।
   - **Clash Verge** या **Clash Meta** पर स्विच करने पर विचार करें, जिनमें HTTPS और आधुनिक ब्राउज़र सुरक्षा नीतियों के लिए बेहतर समर्थन हो सकता है।[](https://clashverge.net/en/tutorial-en/)

#### 7. **ब्राउज़र में असुरक्षित सामग्री की अनुमति दें (अनुशंसित नहीं)**
   - अंतिम उपाय के रूप में, आप अपने ब्राउज़र में `https://clash.razord.top/` के लिए असुरक्षित सामग्री की अनुमति दे सकते हैं:
     - **Chrome**:
       1. एड्रेस बार में लॉक आइकन पर क्लिक करें।
       2. "Site settings" > "Insecure content" > "Allow" पर सेट करें।
     - **Firefox**:
       1. लॉक आइकन पर क्लिक करें और "Connection settings" पर जाएं।
       2. "Block dangerous and deceptive content" को अस्थायी रूप से अक्षम करें।
     - **चेतावनी**: यह सुरक्षा सुरक्षा को बायपास करता है और इसका उपयोग केवल विश्वसनीय नेटवर्क पर स्थानीय परीक्षण के लिए किया जाना चाहिए।
   - वैकल्पिक रूप से, Chrome को `--disable-web-security` फ्लैग के साथ लॉन्च करें (केवल विकास के लिए):
     ```bash
     google-chrome --disable-web-security --user-data-dir="/tmp/chrome_dev"
     ```

#### 8. **विरोधाभासी एक्सटेंशन या फ़ायरवॉल की जांच करें**
   - ब्राउज़र एक्सटेंशन (जैसे एड ब्लॉकर्स, प्राइवेसी टूल्स) या फ़ायरवॉल सेटिंग्स Clash के स्थानीय सर्वर में हस्तक्षेप कर सकती हैं। एक्सटेंशन को अस्थायी रूप से अक्षम करें या अपने फ़ायरवॉल की जांच करें ताकि सुनिश्चित हो सके कि `127.0.0.1:9090` एक्सेस करने योग्य है।[](https://vpncentral.com/reddit-blocked-by-network-security/)
   - Windows पर, सुनिश्चित करें कि Clash ऐप के पास फ़ायरवॉल अनुमतियां हैं:
     - Settings > Network & Internet > Windows Firewall > Allow an app through firewall पर जाएं।
     - Clash for Windows को जोड़ें या सक्षम करें।

#### 9. **Clash सपोर्ट या कम्युनिटी से संपर्क करें**
   - यदि उपरोक्त चरण समस्या का समाधान नहीं करते हैं, तो Clash for Windows GitHub इश्यू पेज (`github.com/Fndroid/clash_for_windows_pkg/issues`) या Clash कम्युनिटी फोरम पर समस्या की रिपोर्ट करें। अपने सेटअप के विवरण प्रदान करें, जिसमें शामिल हैं:
     - Clash वर्जन।
     - ऑपरेटिंग सिस्टम।
     - कॉन्फ़िगरेशन फ़ाइल (संवेदनशील जानकारी हटाएं)।
     - ब्राउज़र और वर्जन।
   - कम्युनिटी में समान समस्याओं की जांच करें, क्योंकि यह त्रुटि कुछ विशिष्ट Clash संस्करणों या कॉन्फ़िगरेशन से संबंधित हो सकती है।[](https://superuser.com/questions/1769163/could-not-switch-to-this-profile-proxy-0-filed-ws-opts-headershost-invalid-c)

### अनुशंसित दृष्टिकोण
सबसे सुरक्षित और मजबूत समाधान **HTTPS के साथ एक रिवर्स प्रॉक्सी सेट अप करना** (चरण 4) है ताकि यह सुनिश्चित हो सके कि सभी अनुरोध HTTPS पर परोसे जाते हैं, जिससे मिश्रित सामग्री और CORS दोनों समस्याओं का समाधान होता है। यदि आप Clash core या डैशबोर्ड को संशोधित करने में असमर्थ हैं, तो Clash के नए संस्करण (जैसे Clash Verge) का उपयोग करना या डैशबोर्ड को HTTP के माध्यम से अस्थायी रूप से एक्सेस करना (चरण 2) त्वरित समाधान हो सकते हैं।

### अतिरिक्त नोट्स
- यदि आप कस्टम Clash डैशबोर्ड या तृतीय-पक्ष सेवा जैसे `clash.razord.top` का उपयोग कर रहे हैं, तो सत्यापित करें कि यह आपके Clash core वर्जन के साथ संगत है।
- परिवर्तन करने से पहले हमेशा अपनी Clash कॉन्फ़िगरेशन (`config.yaml`) का बैकअप लें।
- यदि समस्या बनी रहती है, तो आप वैकल्पिक प्रॉक्सी क्लाइंट जैसे v2rayN या OpenVPN पर स्विच करने का प्रयास कर सकते हैं, जिनमें स्थानीय API के लिए बेहतर HTTPS समर्थन हो सकता है।[](https://stackoverflow.com/questions/67822697/how-to-route-internet-traffic-via-clash-for-windows-ping-from-python-code-is)

मुझे बताएं यदि आपको विशिष्ट चरणों, जैसे Nginx सेट अप करने या Clash कॉन्फ़िगरेशन को डीबग करने में सहायता की आवश्यकता है!
