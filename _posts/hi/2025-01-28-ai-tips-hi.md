* Cursor में उदाहरण कोड को कॉपी करें, और टिप्पणियाँ प्रदान करने के लिए संदर्भ प्रदान करें।

* हांगकांग क्षेत्र में, DeepSeek या Mistral सेवाओं का उपयोग स्वीकार्य है, जबकि ChatGPT, Claude, या Gemini का उपयोग स्वीकार्य नहीं है।

* एक क्षेत्र के एपीआई नीति को लगभग उसी के ऐप नीति जैसा समझा जाता है।

* Visual Studio Code के बजाय Cursor का उपयोग करें।

* Visual Studio Code का उपयोग करने की कुछ स्थितियां अभी भी हैं, जैसे कि git merge editor की स्थितियाँ, जहां मैं अभी भी `git config --global core.editor "code --wait"` का उपयोग करती हूँ।

* Deepseek V3 रिलीज के दिन से, हमें कोई एआई टूल्स सब्सक्राइब करने की जरूरत नहीं है।

* Gemini या Grok के साथ Pinch के जश्न की तस्वीरें बनाने के लिए प्रॉम्प जैसा "Generate a happy Lunar Snake New Year image with text names included" का उपयोग करें।

* कुछ मामलों में, जब आप AI models को मूल पाठ प्रदान करते हैं ताकि एक टेबल बनाया जा सके, कुछ जगहें output में input से अलग हो सकती हैं। उदाहरण के लिए, जब Deepseek V3 model को Cursor में pip list के एक टेबल का निर्माण करने के लिए उपयोग किया जाता है, तो यह `1.极狐0` जैसी संस्करणें शामिल कर सकता है। यहां, `极狐` चीनी GitLab प्लेटफॉर्म को संदर्भित करता है।

* Deepseek या Mistral API का उपयोग करने पर, प्रॉम्प जैसे `You are a professional translator. You are translating a markdown file for a Jekyll blog post from English to Chinese. {text}` के साथ शीर्षकों की अनुवाद करने से गलत अनुवाद हो सकते हैं। आपने दिए गए पाठ के अलावा, output में अक्सर अत्यधिक अनुवाद शामिल होता है।

* कभी-कभी, Cursor में AI models सही पाठ के कुछ हिस्से दे सकते हैं, हम उन्हें स्वीकार कर सकते हैं, क्योंकि हमने इसके बाद निर्देश जोड़ सकते हैं जो AI models को सही हिस्सों को पुनः पैदा करने के लिए करेंगे।

* यदि यह मददगार होने की संभावना नहीं है, तो बड़े भाषा मॉडलों को अतिरिक्त संदर्भ प्रदान करने से बचें। उदाहरण के लिए, जब आप वार्तालाप डायलॉग लाइंस बनाते हैं, तो किसी विषय पर 100 अंक प्रदान करने से बचें। बड़े भाषा मॉडलों में पहले ही विशाल मात्रा में डेटा शामिल है।
