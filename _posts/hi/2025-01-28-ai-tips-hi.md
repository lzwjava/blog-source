* Cursor में उदाहरण कोड को कॉपी करें और टिप्पणियों का उपयोग करके संदर्भ प्रदान करें।

* हांगकांग क्षेत्र में, DeepSeek या Mistral सेवाओं का उपयोग करने में स्वीकार्य है, जबकि ChatGPT, Claude, या Gemini का उपयोग करने में स्वीकार्य नहीं है।

* एक क्षेत्र की एपीआई नीति लगभग उसके ऐप नीति की तरह होती है।

* Cursor का उपयोग Visual Studio Code के बजाय करें।

* कुछ मामलों में, आप Visual Studio Code का उपयोग करना पड़ेगा, जैसे कि git merge editor के मामलों में, जहां मैं अभी भी `git config --global core.editor "code --wait"` का उपयोग करता हूँ।

* Deepseek V3 रिलीज़ के दिन से, हमें किसी भी एआई टूल्स का सब्सक्राइब करने की आवश्यकता नहीं है।

* Gemini या Grok का उपयोग करके प्रॉम्प्ट जैसे "Generate a happy Lunar Snake New Year image with text names included" के साथ त्योहार मनाने के चित्र बनाएं।

* कुछ मामलों में, जब आप मूल पाठ को एआई मॉडल्स को टेबल बनाने के लिए प्रदान करते हैं, तो कुछ स्थानों में आउटपुट इनपुट से अलग हो सकते हैं। उदाहरण के लिए, जब आप Deepseek V3 मॉडल को Cursor में pip list का टेबल बनाने के लिए उपयोग करते हैं, तो इसमें `1.极狐0` जैसे संस्करण शामिल हो सकते हैं। यहाँ, `极狐` चीनी GitLab प्लेटफॉर्म का संदर्भ देता है।

* Deepseek या Mistral API का उपयोग करके प्रॉम्प्ट जैसे `You are a professional translator. You are translating a markdown file for a Jekyll blog post from English to Chinese. {text}` के साथ शीर्षक अनुवाद करने पर गलत अनुवाद हो सकते हैं। आपने प्रदान किया गया पाठ के अलावा, आउटपुट में अक्सर अतिरिक्त अनुवाद शामिल होता है।

* कभी-कभी, Cursor में एआई मॉडल्स कुछ हद तक सही पाठ देते हैं, हम उन्हें स्वीकार कर सकते हैं, क्योंकि हम उन्हें फिर से सही हिस्सों को पुनः उत्पन्न करने के लिए फॉलो-अप निर्देश जोड़ सकते हैं।

* अगर यह संभव नहीं है कि यह उपयोगी हो, तो बड़े भाषा मॉडल्स को अतिरिक्त संदर्भ प्रदान करने से बचें। उदाहरण के लिए, जब आप संवाद डायलॉग लाइनों को उत्पन्न करते हैं, तो किसी विषय पर 100 बिंदुओं को प्रदान करने से बचें। बड़े भाषा मॉडल्स में पहले से ही विशाल मात्रा में डेटा शामिल है।

* अनुवाद या संवाद गीत उत्पन्न करने जैसे कार्य के लिए पर्याप्त संदर्भ प्रदान करते समय, चेन-ऑफ़-थॉट फीचर्स का उपयोग करने से बचें, क्योंकि यह धीमा हो सकता है और विस्तृत या उपयोगी नहीं होने वाले जवाबों को जन्म दे सकता है।

* एक चैटबॉट को उपयोगकर्ता के निर्देशों का पालन करने की क्षमता का परीक्षण करने का एक तरीका यह है कि उसे कुछ अंग्रेजी में समझाने के लिए कहा जाए और फिर चीनी में इनपुट जारी रखें, और देखें कि चैटबॉट अपने आउटपुट को अंग्रेजी में बनाए रखता है।

* बड़े भाषा मॉडल्स को संदर्भ प्रदान करने के बजाय, एक बड़े डेटासेट के आधार पर एक मॉडल को फाइनट्यून करें और फिर फाइनट्यून मॉडल का उपयोग करें।
