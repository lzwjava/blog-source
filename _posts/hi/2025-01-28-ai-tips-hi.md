* उदाहरण कोड को कर्सर में कॉपी करें, और संदर्भ प्रदान करने के लिए टिप्पणियों का उपयोग करें।

* हांगकांग क्षेत्र में, DeepSeek या Mistral सेवाओं का उपयोग स्वीकार्य है, जबकि ChatGPT, Claude, या Gemini का उपयोग स्वीकार्य नहीं है।

* किसी क्षेत्र की API नीति उसकी ऐप नीति के लगभग समान है।

* Visual Studio Code के बजाय Cursor का उपयोग करें।

* अभी भी ऐसे मामले हैं जहाँ आपको Visual Studio Code का उपयोग करने की आवश्यकता है, जैसे कि git merge editor परिदृश्यों के लिए, जहाँ मैं अभी भी `git config --global core.editor "code --wait"` का उपयोग करता हूँ।

* Deepseek V3 रिलीज़ से दिन की शुरुआत करते हुए, हमें किसी भी AI टूल को सब्सक्राइब करने की आवश्यकता नहीं है।

* "Generate a happy Lunar Snake New Year image with text names included" जैसे प्रॉम्प्ट के साथ त्योहार समारोह की छवियां उत्पन्न करने के लिए Gemini या Grok का उपयोग करें।

* कुछ मामलों में, AI मॉडल को टेबल बनाने के लिए मूल पाठ प्रदान करते समय भी, आउटपुट में कुछ स्थान इनपुट से भिन्न हो सकते हैं। उदाहरण के लिए, कर्सर में Deepseek V3 मॉडल का उपयोग करके pip सूची की तालिका उत्पन्न करते समय, इसमें `1.极狐0` जैसे संस्करण शामिल हो सकते हैं। यहाँ, `极狐` चीनी GitLab प्लेटफ़ॉर्म को संदर्भित करता है।

* DeepSeek या Mistral API का उपयोग करके `You are a professional translator. You are translating a markdown file for a Jekyll blog post from English to Chinese. {text}` जैसे प्रॉम्प्ट के साथ शीर्षकों का अनुवाद करने पर, गलत अनुवाद हो सकते हैं। आपके द्वारा प्रदान किए गए पाठ के अलावा, आउटपुट में अक्सर अत्यधिक अनुवाद शामिल होता है।

* हालाँकि कभी-कभी Cursor में AI मॉडल आंशिक रूप से सही पाठ देते हैं, हम उन्हें स्वीकार कर सकते हैं, क्योंकि हम अनुवर्ती निर्देश जोड़ सकते हैं जो AI मॉडल को सही भागों को पुन: उत्पन्न करेंगे।

* यदि यह सहायक होने की संभावना नहीं है, तो बड़े भाषा मॉडल को अत्यधिक संदर्भ प्रदान करने से बचें। उदाहरण के लिए, संवादी संवाद लाइनों को उत्पन्न करते समय, किसी विषय पर 100 बिंदु प्रदान करने से बचें। बड़े भाषा मॉडल में पहले से ही बड़ी मात्रा में डेटा होता है।

* अनुवाद या संवाद गीत उत्पन्न करने जैसे कार्यों के लिए पर्याप्त संदर्भ प्रदान करते समय, विचार-श्रृंखला सुविधाओं का उपयोग करने से बचें, क्योंकि यह धीमा हो सकता है और विस्तृत या असहाय प्रतिक्रियाएँ दे सकता है।

* यह परीक्षण करने का एक तरीका है कि क्या कोई चैटबॉट उपयोगकर्ता के निर्देशों का पालन कर सकता है, यह पूछना है कि वह किसी चीज़ को अंग्रेजी में समझाए और फिर चीनी में इनपुट जारी रखे, यह देखते हुए कि क्या चैटबॉट अपना आउटपुट अंग्रेजी में बनाए रखता है।
