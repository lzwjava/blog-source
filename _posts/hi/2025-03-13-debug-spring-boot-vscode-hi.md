Visual Studio Code (VSCode) में Maven Spring Boot प्रोजेक्ट को डीबग करने के लिए, आप डीबग मोड में अपना एप्लिकेशन सेट अप और रन करने के लिए इन सीधे चरणों का पालन कर सकते हैं। यह गाइड मानती है कि आपके पास एक बुनियादी Maven Spring Boot प्रोजेक्ट है जिसमें एक `pom.xml` फ़ाइल और `@SpringBootApplication` एनोटेशन वाला एक मुख्य वर्ग है। यहां बताया गया है कि इसे कैसे करें:

### चरण 1: आवश्यक एक्सटेंशन इंस्टॉल करें
शुरू करने से पहले, सुनिश्चित करें कि VSCode Java और Spring Boot डेवलपमेंट के लिए आवश्यक एक्सटेंशन से सुसज्जित है:
- **Java Extension Pack**: इसमें Red Hat द्वारा Java के लिए Language Support जैसे आवश्यक टूल शामिल हैं, जो डीबगिंग क्षमताएं प्रदान करते हैं, साथ ही Java डेवलपमेंट के लिए अन्य यूटिलिटीज भी।
- **Spring Boot Extension Pack**: यह पैक Spring Boot डैशबोर्ड, Spring Boot टूल्स, और अन्य सुविधाओं के साथ Spring Boot डेवलपमेंट को बढ़ाता है।

इन्हें इंस्टॉल करने के लिए:
1. VSCode खोलें।
2. एक्सटेंशन व्यू (`Ctrl+Shift+X` या macOS पर `Cmd+Shift+X`) पर जाएं।
3. "Java Extension Pack" और "Spring Boot Extension Pack" खोजें, फिर प्रत्येक के लिए **Install** पर क्लिक करें।

### चरण 2: अपना Maven Spring Boot प्रोजेक्ट खोलें
- VSCode लॉन्च करें और **File > Open Folder** का चयन करके अपना प्रोजेक्ट फ़ोल्डर खोलें और वह डायरेक्टरी चुनें जिसमें आपकी `pom.xml` फ़ाइल है।
- VSCode `pom.xml` का पता लगा लेगा, और Java Extension Pack स्वचालित रूप से प्रोजेक्ट को इंडेक्स करेगा और निर्भरताओं को हल करेगा। इसमें कुछ पल लग सकते हैं, इसलिए प्रक्रिया के पूरा होने तक प्रतीक्षा करें (आपको नीचे दाईं ओर स्थित स्टेटस बार में प्रगति दिखाई देगी)।

### चरण 3: `launch.json` फ़ाइल बनाएं या संपादित करें
डीबगिंग कॉन्फ़िगर करने के लिए, आपको VSCode में एक `launch.json` फ़ाइल सेट अप करनी होगी:
1. साइडबार में बग-एंड-प्ले आइकन पर क्लिक करके या `Ctrl+Shift+D` दबाकर **Run and Debug** व्यू खोलें।
2. यदि कोई डीबग कॉन्फ़िगरेशन मौजूद नहीं है, तो **"create a launch.json file"** पर क्लिक करें। यदि पहले से मौजूद है, तो इसे संपादित करने के लिए गियर आइकन पर क्लिक करें।
3. जब संकेत दिया जाए, तो परिवेश के रूप में **Java** का चयन करें। VSCode आपके प्रोजेक्ट के भीतर एक `.vscode` फ़ोल्डर में एक डिफ़ॉल्ट `launch.json` फ़ाइल जनरेट करेगा।
4. अपने Spring Boot एप्लिकेशन के लिए एक डीबग कॉन्फ़िगरेशन जोड़ें या संशोधित करें। यहां एक उदाहरण कॉन्फ़िगरेशन दिया गया है:

    ```json
    {
        "type": "java",
        "name": "Debug Spring Boot",
        "request": "launch",
        "mainClass": "com.example.demo.DemoApplication",
        "projectName": "demo"
    }
    ```

    - `"com.example.demo.DemoApplication"` को अपने मुख्य वर्ग के पूरी तरह से योग्य नाम से बदलें (उदाहरण के लिए, `com.yourcompany.yourapp.YourApplication`)।
    - `"demo"` को अपने प्रोजेक्ट के नाम से बदलें, जो आमतौर पर आपकी `pom.xml` से `<artifactId>` होता है।

5. `launch.json` फ़ाइल सेव करें।

#### वैकल्पिक: तर्क जोड़ें
यदि आपके एप्लिकेशन को विशिष्ट तर्कों (जैसे, Spring प्रोफाइल) की आवश्यकता है, तो आप उन्हें शामिल कर सकते हैं:
```json
{
    "type": "java",
    "name": "Debug Spring Boot",
    "request": "launch",
    "mainClass": "com.example.demo.DemoApplication",
    "projectName": "demo",
    "args": "--spring.profiles.active=dev"
}
```

### चरण 4: डीबगिंग शुरू करें
- **Run and Debug** व्यू में, शीर्ष पर ड्रॉपडाउन मेनू से **"Debug Spring Boot"** का चयन करें।
- एप्लिकेशन को डीबग मोड में लॉन्च करने के लिए हरे प्ले बटन पर क्लिक करें (या `F5` दबाएं)।
- VSCode Maven का उपयोग करके प्रोजेक्ट को कंपाइल करेगा, Spring Boot एप्लिकेशन शुरू करेगा और डीबगर को स्वचालित रूप से अटैच कर देगा।

### चरण 5: ब्रेकपॉइंट सेट करें और डीबग करें
- अपने प्रोजेक्ट में एक Java फ़ाइल खोलें (उदाहरण के लिए, एक कंट्रोलर या सर्विस क्लास)।
- लाइन नंबरों के बाईं ओर गटर में क्लिक करके ब्रेकपॉइंट सेट करें, जहां एक लाल बिंदु दिखाई देगा।
- अपने एप्लिकेशन के साथ इंटरैक्ट करें (उदाहरण के लिए, ब्राउज़र या API क्लाइंट के माध्यम से)। जब एक्जिक्यूशन किसी ब्रेकपॉइंट से टकराता है, तो VSCode रुक जाएगा, जिससे आप यह कर सकते हैं:
  - **Variables** पैनल में वेरिएबल्स का निरीक्षण करें।
  - **Step Over** (`F10`), **Step Into** (`F11`), या **Continue** (`F5`) जैसे कंट्रोल्स का उपयोग करके कोड के माध्यम से कदम बढ़ाएं।
  - डीबग साइडबार में कॉल स्टैक और अधिक देखें।

### वैकल्पिक तरीके
जबकि `launch.json` विधि अपनी लचीलापन के लिए अनुशंसित है, आप इसका उपयोग करके भी डीबग कर सकते हैं:
- **Maven View**:
  1. साइडबार में Maven व्यू खोलें ("M" आइकन देखें या Extensions के माध्यम से इसे सक्षम करें)।
  2. अपने प्रोजेक्ट को विस्तारित करें, Plugins के तहत `spring-boot` पर नेविगेट करें, `spring-boot:run` पर राइट-क्लिक करें और **Debug** का चयन करें।
  3. एप्लिकेशन शुरू हो जाएगा, और डीबगर अटैच हो जाएगा।
- **Spring Boot Dashboard**:
  1. Spring Boot डैशबोर्ड खोलें (आमतौर पर Explorer व्यू में या कमांड पैलेट के माध्यम से: `Spring Boot Dashboard`)।
  2. अपने एप्लिकेशन को ढूंढें, उस पर राइट-क्लिक करें और **Debug** का चयन करें (यदि उपलब्ध हो)।

### पूर्वापेक्षाएँ और नोट्स
- सुनिश्चित करें कि **Maven** आपके सिस्टम पर इंस्टॉल है और कमांड लाइन के माध्यम से एक्सेस करने योग्य है, या VSCode के एम्बेडेड Maven पर भरोसा करें (यदि आवश्यक हो तो सेटिंग्स के माध्यम से कॉन्फ़िगर किया गया)।
- डीबगिंग के दौरान रनटाइम एरर से बचने के लिए सत्यापित करें कि आपका प्रोजेक्ट सफलतापूर्वक बिल्ड होता है (`mvn clean install`)।
- अपनी `pom.xml` में निर्दिष्ट अनुसार एक संगत Java संस्करण का उपयोग करें (उदाहरण के लिए, `<java.version>` के माध्यम से)।

इन चरणों का पालन करके, आप VSCode में अपने Maven Spring Boot प्रोजेक्ट को प्रभावी ढंग से डीबग कर सकते हैं, अपने कोड में मुद्दों की पहचान करने और हल करने के लिए इसके शक्तिशाली डीबगिंग टूल्स का लाभ उठा सकते हैं।
