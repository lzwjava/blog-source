मैं मान रहा हूं कि आप DBeaver का उपयोग करके यह जांचने के तरीके के बारे में पूछ रहे हैं कि कौन सी Oracle डेटाबेस टेबल्स में डेटा है, आपके Oracle DB में डेटा वाली टेबल्स ढूंढने के पिछले प्रश्न के आधार पर। DBeaver एक लोकप्रिय, मुफ्त, मल्टी-प्लेटफॉर्म डेटाबेस टूल है जो Oracle और अन्य डेटाबेस के साथ अच्छी तरह काम करता है। नीचे, मैं आपको दिखाऊंगा कि कैसे DBeaver का उपयोग करके अपने Oracle स्कीमा का निरीक्षण करें और डेटा वाली टेबल्स की पहचान करें।

---

#### 1. **अपने Oracle डेटाबेस से कनेक्शन सेट अप करें**
   - **DBeaver इंस्टॉल करें**: यदि आपने पहले से नहीं किया है, तो [dbeaver.io](https://dbeaver.io/) से DBeaver (कम्युनिटी एडिशन ठीक है) डाउनलोड और इंस्टॉल करें।
   - **एक नया कनेक्शन बनाएँ**:
     1. DBeaver खोलें और मेनू में **Database** > **New Database Connection** पर क्लिक करें।
     2. सूची से **Oracle** चुनें और **Next** पर क्लिक करें।
     3. अपना कनेक्शन विवरण दर्ज करें:
        - **Host**: आपके Oracle सर्वर का होस्टनाम या IP।
        - **Port**: आमतौर पर 1521 (Oracle के लिए डिफॉल्ट)।
        - **Database/SID or Service Name**: आपके सेटअप के आधार पर (जैसे, एक्सप्रेस एडिशन के लिए SID = `XE` या एक सर्विस नाम)।
        - **Username** और **Password**: आपके Oracle क्रेडेंशियल्स।
     4. यह सत्यापित करने के लिए कि यह काम करता है, **Test Connection** पर क्लिक करें। यदि संकेत दिया जाए तो आपको Oracle JDBC ड्राइवर डाउनलोड करने की आवश्यकता हो सकती है (DBeaver यह स्वचालित रूप से कर सकता है)।
     5. कनेक्शन को सेव करने के लिए **Finish** पर क्लिक करें।

#### 2. **डेटाबेस नेविगेटर में स्कीमा एक्सप्लोर करें**
   - **Database Navigator** (बाएं हाथ का पेन) में, अपने Oracle कनेक्शन को विस्तारित करें।
   - आपको स्कीमा की एक सूची दिखाई देगी (जैसे, आपका उपयोगकर्ता नाम या अन्य जिन तक आपकी पहुंच है)। उस स्कीमा को विस्तारित करें जिसका आप निरीक्षण करना चाहते हैं।
   - प्रत्येक स्कीमा के अंतर्गत, सभी टेबल्स देखने के लिए **Tables** नोड को विस्तारित करें।

#### 3. **GUI का उपयोग करके डेटा के लिए टेबल्स जांचें**
   - **टेबल डेटा देखें**:
     1. एक टेबल नाम पर डबल-क्लिक करें या उस पर राइट-क्लिक करें और **Edit Table** चुनें।
     2. जो एडिटर खुलेगा उसमें **Data** टैब पर स्विच करें।
     3. यदि टेबल में डेटा है, तो आपको पंक्तियाँ दिखाई देंगी। यदि यह खाली है, तो आपको कोई पंक्ति नहीं दिखेगी (या "No data" जैसा संदेश)।
     - डिफॉल्ट रूप से, DBeaver अधिकतम 200 पंक्तियाँ फ़ेच करता है। सभी पंक्तियाँ फ़ेच करने के लिए, Data टैब के निचले टूलबार में **Fetch All Rows** बटन (एक छोटा एरो आइकन) पर क्लिक करें।
   - **पंक्तियों की संख्या त्वरित गिनें**:
     1. Database Navigator में टेबल पर राइट-क्लिक करें।
     2. **Navigate** > **Row Count** चुनें।
     3. DBeaver एक `SELECT COUNT(*)` क्वेरी चलाता है और परिणाम एक पॉप-अप में दिखाता है। यदि यह 0 है, तो टेबल खाली है।

#### 4. **एकाधिक टेबल्स जांचने के लिए SQL क्वेरीज़ चलाएँ**
   - यदि आप एक साथ कई टेबल्स जांचना चाहते हैं (प्रत्येक पर क्लिक करने से अधिक कुशल), SQL एडिटर का उपयोग करें:
     1. **SQL Editor** > **New SQL Script** पर क्लिक करें (या `Ctrl + [` दबाएं)।
     2. स्कीमा संदर्भ सेट करें:
        - SQL एडिटर के शीर्ष पर ड्रॉपडाउन का उपयोग करके अपना स्कीमा चुनें, या यह चलाएँ:
          ```sql
          ALTER SESSION SET CURRENT_SCHEMA = schema_name;
          ```
          `schema_name` को अपने लक्षित स्कीमा (जैसे, `HR`) से बदलें।
     3. सभी टेबल्स के लिए पंक्ति गिनती की क्वेरी करें:
        - डेटा वाली टेबल्स की सूची प्राप्त करने के लिए इस SQL का उपयोग करें (मेरे पिछले उत्तर के समान):
          ```sql
          SELECT table_name, num_rows
          FROM user_tables
          WHERE num_rows > 0
          ORDER BY num_rows DESC;
          ```
        - इसे `Ctrl + Enter` दबाकर रन करें। परिणाम नीचे **Results** टैब में दिखाई देंगे।
        - `USER_TABLES` आपके स्कीमा में टेबल्स दिखाता है। विभिन्न स्कीमा में आपकी पहुंच वाली टेबल्स के लिए `ALL_TABLES` का उपयोग करें:
          ```sql
          SELECT owner, table_name, num_rows
          FROM all_tables
          WHERE num_rows > 0
          AND owner = UPPER('schema_name');
          ```
     4. **नोट**: `NUM_ROWS` अंतिम सांख्यिकी अपडेट से एक अनुमान है। यदि यह गलत है, तो स्टैट्स को इसके साथ अपडेट करें:
        ```sql
        EXEC DBMS_STATS.GATHER_SCHEMA_STATS('schema_name');
        ```
