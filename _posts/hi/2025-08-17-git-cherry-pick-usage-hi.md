### Git Cherry-Pick क्या है?

Git cherry-pick एक कमांड है जो आपको किसी विशिष्ट कमिट (या कमिट्स के समूह) में किए गए परिवर्तनों को आपकी वर्तमान ब्रांच पर लागू करने की अनुमति देती है, जिससे उन परिवर्तनों के साथ एक नई कमिट बनती है। यह तब उपयोगी होता है जब आप किसी ब्रांच को पूरी तरह से मर्ज किए बिना, उसमें से किसी विशेष फिक्स या फीचर को चुनना चाहते हैं। मर्ज के विपरीत, जो पूरे इतिहास को एकीकृत करता है, cherry-pick परिवर्तनों को चुनिंदा रूप से कॉपी करता है।

### Git Cherry-Pick आंतरिक रूप से कैसे काम करता है

1.  **कमिट का चयन करें**: आप एक कमिट हैश निर्दिष्ट करते हैं (उदाहरण के लिए, इसे खोजने के लिए `git log` के माध्यम से) जिसमें वे परिवर्तन हैं जिन्हें आप चाहते हैं।

2.  **परिवर्तन लागू करें**: Git उस कमिट से डिफ (वास्तविक कोड परिवर्तन) लेता है और इसे आपकी वर्तमान ब्रांच के वर्किंग डायरेक्टरी में लागू करने का प्रयास करता है, जहां संभव हो Git के थ्री-वे मर्ज लॉजिक का उपयोग करते हुए।

3.  **नई कमिट बनाएं**: यदि सफल होता है, तो आपकी ब्रांच पर एक नई कमिट बनाई जाती है जिसमें शामिल होते हैं:
    - मूल कमिट के समान परिवर्तन।
    - एक नया कमिट हैश (क्योंकि यह एक नई कमिट है)।
    - एक कमिट संदेश जो मूल कमिट को संदर्भित करता है (उदाहरण के लिए, "-x" फ्लैग का उपयोग करके "cherry picked from commit <hash>" जोड़ने के लिए)।

    यदि मर्ज कॉन्फ्लिक्ट्स होते हैं (उदाहरण के लिए, टार्गेट ब्रांच में असंगत परिवर्तन हैं), तो Git रुक जाता है और आपको कमिट करने से पहले उन्हें मैन्युअल रूप से हल करने देता है।

यह प्रक्रिया मूल कमिट के इतिहास को नहीं बदलती है—यह सिर्फ परिवर्तनों को कॉपी करती है। यह cherry-pick को बग फिक्स को बैकपोर्ट करने या फीचर्स को चुनिंदा रूप से लागू करने के लिए बेहतरीन बनाता है।

### बेसिक उपयोग के चरण

1.  सुनिश्चित करें कि आप टार्गेट ब्रांच पर हैं (`git checkout <branch>` के साथ स्विच करें)।
2.  वह कमिट हैश ढूंढें जिसे आप चाहते हैं (उदाहरण के लिए, `git log --oneline`)।
3.  रन करें: `git cherry-pick <commit-hash>`
    - एक से अधिक के लिए: `git cherry-pick <hash1> <hash2>`
    - विकल्पों के साथ जैसे `-n` (कमिट नहीं, स्टेजिंग के लिए) या `-x` (मूल को संदर्भित करने के लिए)।
4.  यदि कॉन्फ्लिक्ट उत्पन्न होते हैं, तो फाइलों को एडिट करें, `git add` के साथ स्टेज करें, फिर `git cherry-pick --continue` चलाएं।
5.  यदि आप बीच में ही रोकना चाहते हैं: `git cherry-pick --abort` चलाएं।

### उदाहरण परिदृश्य

मान लीजिए आपके पास एक फीचर ब्रांच है जिसमें कमिट A -> B -> C हैं, और आप कमिट B को मुख्य ब्रांच पर cherry-pick करना चाहते हैं।

- मुख्य ब्रांच पर: `git checkout main`
- `git cherry-pick B` (B को वास्तविक हैश से बदलकर)

यह B के परिवर्तनों को मुख्य ब्रांच पर एक नई कमिट D के रूप में जोड़ देता है, बिना A या C को शामिल किए। मूल कमिट B फीचर ब्रांच में ही रहती है।

### चेतावनियाँ और सुझाव

- मर्ज कमिट्स (एक से अधिक पैरेंट वाली कमिट्स) को cherry-pick करने से बचें, क्योंकि इससे समस्याएं हो सकती हैं—एक पैरेंट निर्दिष्ट करने के लिए `git cherry-pick -m 1 <merge-commit>` का उपयोग करें।
- यह कमिट की मेटाडेटा जैसे लेखक की तारीख को संरक्षित नहीं करता है, लेकिन `git cherry-pick --edit` आपको संदेश को संशोधित करने देता है।
- क्रमबद्ध परिवर्तनों के लिए, निर्भरताओं को छूटने से बचाने के लिए कई कमिट्स को क्रम में cherry-pick करें।
- Cherry-pick करने के बाद हमेशा टेस्ट करें, क्योंकि यदि संदर्भ अलग-अलग होते हैं तो यह बग्स पैदा कर सकता है।

अधिक जानकारी के लिए, आधिकारिक Git डॉक्स व्यापक मार्गदर्शन प्रदान करते हैं: https://git-scm.com/docs/git-cherry-pick। यदि आपको कोई विशिष्ट समस्या आती है, तो अपना Git लॉग आउटपुट साझा करें!
