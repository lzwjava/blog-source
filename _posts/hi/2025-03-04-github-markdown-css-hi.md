### उपयोग
इंस्टॉलेशन के बाद, CSS को अपने प्रोजेक्ट में इंटीग्रेट करें:
- CSS फ़ाइल को इम्पोर्ट करें, उदाहरण के लिए, JavaScript/React में `import 'github-markdown-css';`.
- अपने रेंडर्ड मार्कडाउन कंटेंट को `<div class="markdown-body">...</div>` में लपेटें ताकि स्टाइल्स लागू हों।
- विकल्पिक रूप से, GitHub की दिखावट को नकल करने के लिए चौड़ाई और पैडिंग के लिए CSS जोड़ें:
  ```css
  .markdown-body {
    box-sizing: border-box;
    min-width: 200px;
    max-width: 980px;
    margin: 0 auto;
    padding: 45px;
  }
  @media (max-width: 767px) {
    .markdown-body {
      padding: 15px;
    }
  }
  ```
- सुनिश्चित करें कि आपके HTML में एक DOctype (उदाहरण के लिए, `<!DOCTYPE html>`) शामिल है ताकि क्विर्क्स मोड समस्याएं, जो स्टाइलिंग को प्रभावित कर सकती हैं, से बचा जा सके।

#### इंस्टॉलेशन प्रक्रिया
शुरू करने के लिए, आपको npm, Node.js पैकेज मैनेजर के माध्यम से पैकेज इंस्टॉल करना होगा। कमांड सीधा है:
- प्रोजेक्ट डायरेक्टरी में `npm install github-markdown-css` चलाएं। यह पैकेज को आपके `node_modules` फ़ोल्डर में जोड़ता है और `package.json` में निर्भरता को अपडेट करता है।

पैकेज का नवीनतम संस्करण, हाल के चेक के अनुसार, 5.8.1 है, जो लगभग तीन महीने पहले प्रकाशित हुआ था, जो सक्रिय रखरखाव और अपडेट का संकेत देता है। यह आधुनिक वेब विकास अभ्यासों और फ्रेमवर्कों के साथ संगतता सुनिश्चित करता है।

#### इंटीग्रेशन और उपयोग
इंस्टॉलेशन के बाद, अगला कदम है CSS को अपने प्रोजेक्ट में इंटीग्रेट करना। पैकेज एक फ़ाइल प्रदान करता है जिसका नाम `github-markdown.css` है, जिसे आप अपने प्रोजेक्ट सेटअप के अनुसार इम्पोर्ट कर सकते हैं:

- **JavaScript/आधुनिक फ्रेमवर्क (उदाहरण के लिए, React, Vue) के लिए:**
  - JavaScript या TypeScript फ़ाइलों में एक इम्पोर्ट स्टेटमेंट का उपयोग करें, जैसे `import 'github-markdown-css';`. यह Webpack या Vite जैसे बंडलरों के साथ अच्छी तरह से काम करता है, जो CSS इम्पोर्ट को सीधे संभालते हैं।
  - React में, आप देख सकते हैं कि डेवलपर्स इसे एक कॉम्पोनेंट फ़ाइल में इम्पोर्ट करते हैं, सुनिश्चित करते हुए कि स्टाइल्स ग्लोबल या आवश्यकता के अनुसार स्कोप्ड हों।

- **सादा HTML के लिए:**
  - CSS फ़ाइल को सीधे HTML head सेक्शन में लिंक करें:
    ```html
    <link rel="stylesheet" href="node_modules/github-markdown-css/github-markdown.css">
    ```
  - ध्यान रखें कि पथ आपके प्रोजेक्ट संरचना पर आधारित हो सकता है; सुनिश्चित करें कि रिलेटिव पथ सही तरीके से `node_modules` डायरेक्टरी को इंगित करता है।

इम्पोर्ट करने के बाद, स्टाइल्स लागू करने के लिए अपने रेंडर्ड मार्कडाउन कंटेंट को एक `<div>` में लपेटें जिसमें "markdown-body" क्लास है। उदाहरण के लिए:
```html
<div class="markdown-body">
  <h1>यूनिकॉर्न</h1>
  <p>सभी चीजें</p>
</div>
```
यह क्लास महत्वपूर्ण है क्योंकि CSS `.markdown-body` के अंदर तत्वों को लक्षित करता है ताकि GitHub जैसी स्टाइलिंग लागू हो सके, जिसमें टाइपोग्राफी, कोड ब्लॉक्स, टेबल और अधिक शामिल हैं।

#### स्टाइलिंग विचार
GitHub की मार्कडाउन दिखावट को पूरी तरह से नकल करने के लिए, `.markdown-body` क्लास के लिए चौड़ाई और पैडिंग सेट करने की विचार करें। दस्तावेज़ का सुझाव है:
- 980px का अधिकतम चौड़ाई, 45px पैडिंग बड़े स्क्रीन पर, और 15px पैडिंग मोबाइल डिवाइसों पर (स्क्रीन ≤ 767px)।
- आप इसे निम्नलिखित CSS के साथ प्राप्त कर सकते हैं:
  ```css
  .markdown-body {
    box-sizing: border-box;
    min-width: 200px;
    max-width: 980px;
    margin: 0 auto;
    padding: 45px;
  }
  @media (max-width: 767px) {
    .markdown-body {
      padding: 15px;
    }
  }
  ```
यह प्रतिक्रिया सुनिश्चित करता है और GitHub की डिजाइन के साथ मेल खाता है, पढ़ने और उपयोगकर्ता अनुभव को बढ़ाता है।

#### तकनीकी नोट और सर्वोत्तम प्रथाएँ
- **DOctype आवश्यकता:** दस्तावेज़ में पेश किए गए स्टाइलिंग समस्याओं, जैसे कि डार्क मोड में टेबल गलत रूप से रेंडर हो सकते हैं, यदि ब्राउज़र क्विर्क्स मोड में प्रवेश करता है, को उजागर करता है। इस से बचने के लिए, हमेशा HTML के शीर्ष पर एक DOctype शामिल करें, जैसे `<!DOCTYPE html>`. यह मानक-संगत रेंडरिंग सुनिश्चित करता है और अप्रत्याशित व्यवहार से बचता है।
- **मार्कडाउन पर्सिंग:** जबकि पैकेज CSS प्रदान करता है, यह मार्कडाउन को HTML में परिवर्तित नहीं करता। आपको एक मार्कडाउन पर्सर जैसे [marked.js](https://marked.js.org/) या [react-markdown](https://github.com/remarkjs/react-markdown) के लिए React प्रोजेक्टों के लिए, जो मार्कडाउन टेक्स्ट को HTML में परिवर्तित करता है, की आवश्यकता होगी, जिसे फिर इस CSS के साथ स्टाइल किया जा सकता है।
- **कस्टम CSS उत्पन्न:** उन्नत उपयोगकर्ताओं के लिए, संबंधित पैकेज [generate-github-markdown-css](https://www.npmjs.com/package/generate-github-markdown-css) कस्टम CSS उत्पन्न करने की अनुमति देता है, जो विशेष थीमिंग या संशोधनों के लिए उपयोगी हो सकता है। यह एक अपेक्षित विवरण है जिसके लिए आप सोच सकते हैं कि पैकेज केवल सीधे उपयोग के लिए है।

#### विशेष संदर्भों में उपयोग
- **React प्रोजेक्ट:** React में, `github-markdown-css` और `react-markdown` को एक साथ उपयोग करना आम है। दोनों इंस्टॉल करने के बाद, CSS को इम्पोर्ट करें और कॉम्पोनेंट का उपयोग करें:
  ```javascript
  import React from 'react';
  import ReactMarkdown from 'react-markdown';
  import 'github-markdown-css';

  const MarkdownComponent = () => (
    <div className="markdown-body">
      <ReactMarkdown># हेलो, वर्ल्ड!</ReactMarkdown>
    </div>
  );
  ```
  सुनिश्चित करें कि आप पहले दिखाए गए CSS के साथ चौड़ाई और पैडिंग भी सेट करते हैं ताकि पूरी GitHub स्टाइलिंग हो सके।

- **सादा HTML के साथ CDN:** जल्दी प्रोटोटाइपिंग के लिए, आप एक CDN संस्करण का उपयोग कर सकते हैं, जो [cdnjs](https://cdnjs.com/libraries/github-markdown-css) पर उपलब्ध है, सीधे लिंक करके:
  ```html
  <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/github-markdown-css/5.8.1/github-markdown.min.css">
  ```
  फिर `.markdown-body` क्लास को पहले की तरह लागू करें।

#### संभावित समस्याएं और समाधान
- **स्टाइलिंग संघर्ष:** अगर आपका प्रोजेक्ट अन्य CSS फ्रेमवर्क (उदाहरण के लिए, Tailwind, Bootstrap) का उपयोग करता है, तो सुनिश्चित करें कि `.markdown-body` क्लास अधिकांश स्टाइल्स को ओवरराइड करता है, लेकिन पूरी तरह से परीक्षण करें।
- **डार्क मोड समर्थन:** पैकेज डार्क मोड समर्थन प्रदान करता है, लेकिन सुनिश्चित करें कि आपका मार्कडाउन पर्सर और प्रोजेक्ट सेटअप, विशेष रूप से कोड ब्लॉक्स और टेबल के लिए, थीम स्विचिंग को सही तरह से संभालता है।
- **ब्राउज़र संगतता:** पैकेज के व्यापक उपयोग के कारण, संगतता आम तौर पर अच्छी होती है, लेकिन हमेशा मुख्य ब्राउज़रों (Chrome, Firefox, Safari) पर परीक्षण करें ताकि एकसमान रेंडरिंग सुनिश्चित हो सके।
