Spring Boot CLI (कमांड लाइन इंटरफेस) एक शक्तिशाली टूल है जो आपको सीधे कमांड लाइन से Spring Boot एप्लिकेशन्स को तेजी से डेवलप, टेस्ट और प्रोटोटाइप करने की सुविधा देता है। यह Groovy स्क्रिप्ट्स को पूर्ण प्रोजेक्ट स्ट्रक्चर सेट अप किए बिना चलाने, नए प्रोजेक्ट जनरेट करने, या Spring Boot फीचर्स के साथ प्रयोग करने के लिए विशेष रूप से उपयोगी है। नीचे Spring Boot CLI को प्रभावी ढंग से इंस्टॉल और उपयोग करने का एक संपूर्ण गाइड दिया गया है।

---

## इंस्टॉलेशन
Spring Boot CLI का उपयोग करने से पहले, आपको इसे इंस्टॉल करना होगा। आपके ऑपरेटिंग सिस्टम के आधार पर मुख्य रूप से दो तरीके हैं:

### 1. SDKMAN! का उपयोग करना (Linux या macOS जैसे Unix-आधारित सिस्टम्स के लिए अनुशंसित)
SDKMAN! सॉफ्टवेयर डेवलपमेंट किट्स को मैनेज करने का एक टूल है, जो Spring Boot CLI को इंस्टॉल करने का एक आसान तरीका प्रदान करता है।

- **चरण 1: SDKMAN! इंस्टॉल करें**
  अपना टर्मिनल खोलें और निम्न कमांड रन करें:
  ```bash
  curl -s "https://get.sdkman.io" | bash
  ```
  स्क्रिप्ट को सोर्स करके SDKMAN! को इनिशियलाइज़ करने के लिए प्रॉम्प्ट का पालन करें:
  ```bash
  source "$HOME/.sdkman/bin/sdkman-init.sh"
  ```

- **चरण 2: Spring Boot CLI इंस्टॉल करें**
  निम्न कमांड रन करें:
  ```bash
  sdk install springboot
  ```

### 2. मैनुअल इंस्टॉलेशन (Windows या मैनुअल सेटअप के लिए)
यदि आप Windows पर हैं या मैनुअल इंस्टॉलेशन पसंद करते हैं:
- [आधिकारिक Spring वेबसाइट](https://spring.io/projects/spring-boot) से Spring Boot CLI ZIP फ़ाइल डाउनलोड करें।
- ZIP फ़ाइल को अपनी पसंद के किसी डायरेक्टरी में एक्सट्रैक्ट करें।
- एक्सट्रैक्ट की गई फोल्डर से `bin` डायरेक्टरी को अपने सिस्टम के PATH एनवायरनमेंट वेरिएबल में ऐड करें।

### इंस्टॉलेशन वेरिफाई करें
यह पुष्टि करने के लिए कि Spring Boot CLI सही ढंग से इंस्टॉल हो गया है, अपने टर्मिनल में यह कमांड रन करें:
```bash
spring --version
```
आपको Spring Boot CLI की इंस्टॉल की गई वर्जन दिखाई देनी चाहिए (उदाहरण के लिए, `Spring CLI v3.3.0`)। यदि यह काम करता है, तो आप इसका उपयोग शुरू करने के लिए तैयार हैं!

## Spring Boot CLI के उपयोग के प्रमुख तरीके
Spring Boot CLI कई फीचर्स प्रदान करता है जो इसे रैपिड डेवलपमेंट और प्रोटोटाइपिंग के लिए आदर्श बनाते हैं। यहां इसके उपयोग के मुख्य तरीके दिए गए हैं:

### 1. Groovy स्क्रिप्ट्स चलाना
Spring Boot CLI की एक उल्लेखनीय विशेषता यह है कि यह बिना पूर्ण प्रोजेक्ट सेटअप के सीधे Groovy स्क्रिप्ट्स चला सकता है। यह त्वरित प्रोटोटाइपिंग या Spring Boot के साथ प्रयोग करने के लिए परफेक्ट है।

- **उदाहरण: एक साधारण वेब एप्लिकेशन बनाना**
  `hello.groovy` नाम की एक फाइल बनाएं जिसमें निम्न कंटेंट हो:
  ```groovy
  @RestController
  class HelloController {
      @RequestMapping("/")
      String home() {
          "Hello, World!"
      }
  }
  ```

- **स्क्रिप्ट रन करें**
  अपने टर्मिनल में, `hello.groovy` वाली डायरेक्टरी में नेविगेट करें और रन करें:
  ```bash
  spring run hello.groovy
  ```
  यह पोर्ट 8080 पर एक वेब सर्वर शुरू करता है। "Hello, World!" देखने के लिए एक ब्राउज़र खोलें और `http://localhost:8080` पर जाएं।

- **डिपेंडेंसीज ऐड करना**
  आप `@Grab` एनोटेशन का उपयोग करके सीधे स्क्रिप्ट में डिपेंडेंसीज शामिल कर सकते हैं। उदाहरण के लिए:
  ```groovy
  @Grab('org.springframework.boot:spring-boot-starter-data-jpa')
  @RestController
  class HelloController {
      @RequestMapping("/")
      String home() {
          "Hello, World!"
      }
  }
  ```
  यह बिल्ड फाइल की आवश्यकता के बिना आपकी स्क्रिप्ट में Spring Data JPA ऐड कर देता है।

- **एकाधिक स्क्रिप्ट्स चलाना**
  करंट डायरेक्टरी में सभी Groovy स्क्रिप्ट्स को चलाने के लिए, उपयोग करें:
  ```bash
  spring run *.groovy
  ```

### 2. नए Spring Boot प्रोजेक्ट्स बनाना
Spring Boot CLI आपकी इच्छित डिपेंडेंसीज के साथ एक नया प्रोजेक्ट स्ट्रक्चर जनरेट कर सकता है, जो पूर्ण एप्लिकेशन शुरू करते समय आपका समय बचाता है।

- **उदाहरण: एक प्रोजेक्ट जनरेट करना**
  वेब और data-jpa डिपेंडेंसीज के साथ एक नया प्रोजेक्ट बनाने के लिए यह कमांड रन करें:
  ```bash
  spring init --dependencies=web,data-jpa my-project
  ```
  यह `my-project` नामक एक डायरेक्टरी बनाता है जिसमें Spring Web और Spring Data JPA के साथ कॉन्फ़िगर किया गया एक Spring Boot एप्लिकेशन होता है।

- **कस्टमाइज़ेशन ऑप्शन्स**
  आप अतिरिक्त ऑप्शन्स निर्दिष्ट कर सकते हैं जैसे:
  - बिल्ड टूल: `--build=maven` या `--build=gradle`
  - भाषा: `--language=java`, `--language=groovy`, या `--language=kotlin`
  - पैकेजिंग: `--packaging=jar` या `--packaging=war`

  उदाहरण के लिए:
  ```bash
  spring init --dependencies=web --build=gradle --language=kotlin my-kotlin-project
  ```

### 3. एप्लिकेशन्स पैकेजिंग करना
Spring Boot CLI आपको डिप्लॉयमेंट के लिए अपनी स्क्रिप्ट्स को एक्जिक्यूटेबल JAR या WAR फाइल्स में पैकेज करने की अनुमति देता है।

- **एक JAR फाइल बनाएं**
  ```bash
  spring jar my-app.jar *.groovy
  ```
  यह करंट डायरेक्टरी में सभी Groovy स्क्रिप्ट्स को `my-app.jar` में पैकेज करता है।

- **एक WAR फाइल बनाएं**
  ```bash
  spring war my-app.war *.groovy
  ```
  यह एक `my-app.war` फाइल जनरेट करता है जो एक सर्वलेट कंटेनर पर डिप्लॉयमेंट के लिए उपयुक्त है।

### 4. टेस्ट्स चलाना
यदि आपके पास Groovy टेस्ट स्क्रिप्ट्स हैं, तो आप उन्हें निम्न कमांड से एक्जिक्यूट कर सकते हैं:
```bash
  spring test *.groovy
  ```
  यह करंट डायरेक्टरी में सभी टेस्ट स्क्रिप्ट्स को रन करता है।

### 5. इंटरएक्टिव शेल का उपयोग करना
इंटरएक्टिव अनुभव के लिए, Spring Boot CLI शेल लॉन्च करें:
```bash
spring shell
```
शेल के अंदर, आप `run`, `grab`, या `jar` जैसे कमांड्स को इंटरएक्टिव रूप से चला सकते हैं, जो प्रयोग के लिए बहुत अच्छा है।

## सारांश
Spring Boot CLI डेवलपर्स के लिए एक उत्कृष्ट टूल है जो कमांड लाइन से Spring Boot के साथ त्वरित और कुशलता से काम करना चाहते हैं। संक्षेप में इसका उपयोग कैसे करें:

1.  **इसे इंस्टॉल करें** SDKMAN! (`sdk install springboot`) का उपयोग करके या मैन्युअल रूप से ZIP डाउनलोड करके और अपने PATH को अपडेट करके।
2.  **Groovy स्क्रिप्ट्स चलाएं** `spring run <script.groovy>` के साथ त्वरित प्रोटोटाइपिंग के लिए।
3.  **नए प्रोजेक्ट बनाएं** `spring init --dependencies=<deps> <project-name>` के साथ।
4.  **एप्लिकेशन्स को पैकेज करें** `spring jar` या `spring war` का उपयोग करके JAR या WAR फाइल्स में।
5.  **इंटरएक्टिव रूप से प्रयोग करें** `spring shell` के साथ।

चाहे आप किसी आइडिया का प्रोटोटाइप बना रहे हों, Spring Boot सीख रहे हों, या टास्क्स को ऑटोमेट कर रहे हों, Spring Boot CLI पूर्ण IDE या प्रोजेक्ट सेटअप के ओवरहेड के बिना शुरुआत करने का एक हल्का और लचीला तरीका प्रदान करता है।
