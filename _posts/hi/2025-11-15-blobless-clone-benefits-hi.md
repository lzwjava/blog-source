Git कमांड:

```bash
git clone --filter=blob:none <repository-url>
```

एक **आंशिक क्लोन (partial clone)** है जो चेकआउट के समय **लगभग कोई फ़ाइल सामग्री (blobs)** डाउनलोड नहीं करती है। यहाँ बताया गया है कि यह वास्तव में क्या करती है और यह उपयोगी क्यों है:

### `--filter=blob:none` का क्या अर्थ है
- `--filter=blob:none` Git को **ब्लॉबलेस क्लोनिंग (blobless cloning)** का उपयोग करने के लिए कहता है।
- Git फिर भी डाउनलोड करता है:
  - सभी कमिट (इतिहास)
  - सभी ट्री ऑब्जेक्ट (डायरेक्टरी संरचना)
  - सभी रेफ्स/ब्रांचेज़/टैग्स
- लेकिन यह प्रारंभिक क्लोन के दौरान **शून्य फ़ाइल सामग्री (blobs)** डाउनलोड करता है।
- जब आप बाद में `git checkout` करते हैं या ब्रांच स्विच करते हैं, तो Git केवल उन ब्लॉब्स (फ़ाइल सामग्री) को डाउनलोड करता है जिनकी आपके वर्किंग ट्री में उस समय वास्तव में आवश्यकता होती है।

### मुख्य लाभ
- अत्यंत तीव्र और छोटा प्रारंभिक क्लोन (Linux kernel, Android, या Windows सोर्स जैसे विशाल रेपो के लिए भी अक्सर केवल कुछ MB)।
- पहले बहुत कम नेटवर्क बैंडविड्थ और डिस्क स्पेस का उपयोग करता है।
- CI/CD, त्वरित देखने, या जब आपको फाइलें चेकआउट किए बिना केवल इतिहास (जैसे `git log`, blame, कमिट में अंतर) की आवश्यकता होती है, उसके लिए बिल्कुल सही।

### आकार तुलना का उदाहरण (वास्तविक दुनिया)

| रिपॉजिटरी       | सामान्य क्लोन | `blob:none` क्लोन | गति लाभ |
|------------------|--------------|-------------------|------------|
| torvalds/linux   | ~5+ GB       | ~300–400 MB       | 10–20× तेज़ |
| microsoft/vscode | ~2–3 GB      | ~200 MB           | बहुत तेज़   |

### blob:none क्लोन के बाद सामान्य फॉलो-अप कमांड
```bash
# पहली बार जब आपको वर्किंग ट्री की आवश्यकता हो:
git checkout <branch>        # केवल आवश्यक ब्लॉब्स डाउनलोड करता है

# या यदि आप बाद में सब कुछ चाहते हैं:
git fetch --unshallow        # ब्लॉबलेस के लिए आवश्यक नहीं है
# गायब ब्लॉब्स को स्थायी रूप से भरने के लिए:
git repack -a -d             # वैकल्पिक, सभी ब्लॉब्स को डाउनलोड करने के लिए मजबूर करता है
```

### संबंधित फ़िल्टर जो आप देख सकते हैं
- `--filter=blob:none` → प्रारंभ में बिल्कुल भी ब्लॉब्स नहीं (सबसे आक्रामक)
- `--filter=tree:0`    → ट्री डाउनलोड करता है लेकिन ब्लॉब्स नहीं (समान प्रभाव)
- `--filter=sparse:oid=<oid>` → आंशिक ब्लॉब्स के साथ स्पार्स-चेकआउट के लिए

### सारांश
```bash
git clone --filter=blob:none https://github.com/example/repo.git
```
