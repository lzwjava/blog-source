### AuthenticationCanceled त्रुटि का समाधान

यह त्रुटि आमतौर पर Logitech ब्लूटूथ कीबोर्ड (आपका MAC `F4:73:35:78:28:4E` Logitech के OUI उपसर्ग से मेल खाता है) के साथ Ubuntu में पेयरिंग के दौरान प्रमाणीकरण टाइमआउट, गुम PIN प्रॉम्प्ट, या BlueZ में एजेंट हैंडलिंग की समस्या के कारण होती है। आपके लॉग में संक्षिप्त कनेक्ट/डिस्कनेक्ट संकेत देता है कि कीबोर्ड प्रतिक्रिया दे रहा है लेकिन प्रक्रिया पूरा होने से पहले टाइमआउट हो जाती है। अन्य डिवाइस (जैसे `54:2B:FC:F1:1C:D8`) का दिखना हस्तक्षेप का कारण बन सकता है—पहले उन्हें हटा दें।

#### त्वरित तैयारी के चरण
1.  **सभी पेयर्ड डिवाइस हटाएं** टकराव से बचने के लिए:  
    `bluetoothctl` में, उन्हें सूचीबद्ध करने के लिए `devices` चलाएं, फिर प्रत्येक के लिए `remove <MAC>` चलाएं (उदाहरण के लिए, `remove 54:2B:FC:F1:1C:D8`)। `exit` से बाहर निकलें।

2.  **Bluetooth सेवा पुनरारंभ करें**:  
    ```
    sudo systemctl restart bluetooth
    sudo systemctl status bluetooth  # सत्यापित करें कि यह सक्रिय है
    ```

3.  **कीबोर्ड को पेयरिंग मोड में डालें**: पेयरिंग बटन (जैसे, Easy-Switch चैनल या Bluetooth बटन) को तब तक दबाए रखें जब तक LED तेजी से न ब्लिंक करने लगे। प्रत्येक प्रयास से पहले ताज़ा करके ऐसा करें।

#### bluetoothctl में उन्नत पेयरिंग चरण
`bluetoothctl` को फिर से खोलें और इन चरणों का **बिल्कुल** पालन करें—एजेंट सेटअप महत्वपूर्ण है, और पहले ट्रस्ट करने से कुछ टाइमआउट से बचा जा सकता है। Logitech मॉडल (जैसे, K380, K480, MX Keys) के लिए, `pair` कमांड के तुरंत बाद फिजिकल कीबोर्ड पर कोई भी PIN **बिना देखे** टाइप करें (कोई ऑन-स्क्रीन इको नहीं)।

1.  **bluetoothctl में प्रवेश करें**:  
    ```
    bluetoothctl
    ```

2.  **पावर ऑन करें और एजेंट सेट करें**:  
    ```
    power on
    agent on
    default-agent
    ```

3.  **स्कैन करें और डिवाइस की पुष्टि करें**:  
    ```
    scan on
    ```  
    `F4:73:35:78:28:4E` के दिखने की प्रतीक्षा करें (यदि आवश्यक हो तो Enter दबाकर रिफ्रेश करें)। फिर:  
    ```
    scan off  # फोकस करने के लिए स्कैनिंग बंद करें
    ```

4.  **डिवाइस को ट्रस्ट करें** (रीकनेक्ट पर ऑटो-एक्सेप्ट में मदद करता है):  
    ```
    trust F4:73:35:78:28:4E
    ```

5.  **पेयर**:  
    ```
    pair F4:73:35:78:28:4E
    ```  
    - यदि कोई PIN/पासकी दिखाई दे (जैसे, `[agent] PIN code: 390412` या `[agent] Passkey: 299595`), **तुरंत उसे फिजिकल कीबोर्ड पर टाइप करें** और Enter दबाएं। यह बिना देखे करें—भले ही कोई प्रॉम्प्ट सक्रिय न दिखाई दे।  
    - यदि कोई PIN न दिखे (कुछ Logitech डिवाइस में आम), कीबोर्ड पर `0000` या `1234` टाइप करें और वैसे भी Enter दबाएं।  
    - `[CHG] Paired: yes` के लिए निगरानी रखें। यदि यह कुछ देर कनेक्ट होकर फिर डिस्कनेक्ट हो जाता है, तो तुरंत बाद `connect F4:73:35:78:28:4E` फिर से चलाएं।

6.  **कनेक्ट करें और ट्रस्ट करें**:  
    ```
    connect F4:73:35:78:28:4E
    trust F4:73:35:78:28:4E  # यदि पहले नहीं किया गया हो
    ```

7.  **बाहर निकलें और परीक्षण करें**:  
    ```
    exit
    ```  
    एक टेक्स्ट एडिटर या टर्मिनल खोलें—इनपुट सत्यापित करने के लिए टाइप करें। यदि यह काम करता है, तो वेक अप पर यह स्वचालित रूप से रीकनेक्ट होना चाहिए।

#### यदि फिर भी विफल रहता है
-   **GUI फॉलबैक**: Ubuntu Settings > Bluetooth का उपयोग करें (टॉगल ऑन करें, डिवाइस चुनें)। यह अक्सर एक दृश्यमान PIN प्रॉम्प्ट दिखाता है। यदि CLI एजेंट विफल रहता है, तो GUI का pinentry इसे बेहतर तरीके से संभालता है।
-   **बिना देखे PIN पुनः प्रयास**: प्रदर्शित कोड के बिना भी, कुछ उपयोगकर्ता `pair` के तुरंत बाद `0000` + Enter टाइप करके सफल होते हैं—कीबोर्ड चुपचाप प्रमाणीकरण करता है।
-   **सुराग के लिए लॉग**: पेयरिंग के दौरान दूसरे टर्मिनल में `sudo journalctl -u bluetooth -f` चलाएं। "PIN code" या टाइमआउट संदेशों की तलाश करें।
-   **BlueZ अपडेट**: नवीनतम सुनिश्चित करें (Ubuntu 24.04+ में BlueZ 5.72+ है जिसमें बेहतर Logitech सपोर्ट है):  
    ```
    sudo apt update && sudo apt upgrade bluez
    ```
-   **USB डोंगल क्वर्क**: USB 3.0 पर आपका BT 5.3 एमिटर ठीक है, लेकिन USB 2.0 पोर्ट पर परीक्षण करें—दुर्लभ हस्तक्षेप की सूचना मिली है।
-   **मॉडल-विशिष्ट**: यदि यह K480/MX Keys है, तो पेयरिंग बटन को 5+ सेकंड तक दबाए रखें। मल्टी-डिवाइस के लिए, पहले एक अप्रयुक्त चैनल पर साइकिल करें।

इससे इसे हल होना चाहिए—अधिकांश Logitech मामले बिना देखे PIN चरण से उत्पन्न होते हैं। यदि आप अपना सटीक मॉडल या पूर्ण लॉग साझा करते हैं, तो मैं और परिष्कृत कर सकता हूं।
