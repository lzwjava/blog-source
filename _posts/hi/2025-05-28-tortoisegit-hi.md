TortoiseGit एक विंडोज शेल एक्सटेंशन है जो Git, लोकप्रिय वर्जन कंट्रोल सिस्टम के लिए एक यूजर-फ्रेंडली इंटरफेस प्रदान करता है। यह सीधे विंडोज एक्सप्लोरर में इंटीग्रेट होता है, जिससे आप कॉन्टेक्स्ट मेन्यू के माध्यम से Git ऑपरेशन कर सकते हैं। यह गाइड विंडोज पर TortoiseGit को इंस्टॉल, कॉन्फ़िगर और उपयोग करने के साथ-साथ सामान्य कार्यों और बेस्ट प्रैक्टिसेज को कवर करती है।

### TortoiseGit का अवलोकन
TortoiseGit, msysGit (Git for Windows) के ऊपर बनाया गया है और Git कमांड्स के लिए एक ग्राफिकल इंटरफेस प्रदान करता है। यह उन यूजर्स के लिए आदर्श है जो कमांड-लाइन Git ऑपरेशन्स पर GUI को प्राथमिकता देते हैं। मुख्य विशेषताओं में शामिल हैं:
- विंडोज एक्सप्लोरर में कॉन्टेक्स्ट मेन्यू इंटीग्रेशन।
- विजुअल डिफ और मर्ज टूल्स।
- सामान्य Git वर्कफ़्लो के लिए सपोर्ट (कमिट, पुश, पुल, ब्रांच, मर्ज, आदि)।
- लोकप्रिय इश्यू ट्रैकर्स और एक्सटर्नल टूल्स के साथ इंटीग्रेशन।

### आवश्यक शर्तें
TortoiseGit इंस्टॉल करने से पहले, सुनिश्चित करें कि आपके पास है:
- एक विंडोज ऑपरेटिंग सिस्टम (विंडोज 10 या बाद वाला संस्करण अनुशंसित)।
- Git for Windows इंस्टॉल (TortoiseGit इस पर निर्भर करता है)।
- Git अवधारणाओं की बुनियादी समझ (रिपॉजिटरी, कमिट, ब्रांचेज, आदि)।

### इंस्टॉलेशन
1. **Git for Windows इंस्टॉल करें**:
   - नवीनतम संस्करण [Git for Windows](https://gitforwindows.org/) या [Git SCM](https://git-scm.com/downloads) से डाउनलोड करें।
   - इंस्टॉलर चलाएं और प्रॉम्प्ट का पालन करें। अनुशंसित सेटिंग्स:
     - डिफ़ॉल्ट एडिटर (जैसे, नोटपैड) का उपयोग करें या VS Code जैसा कोई एडिटर चुनें।
     - एक्सेसिबिलिटी के लिए "Use Git from the Windows Command Prompt" चुनें।
     - HTTPS ट्रांसपोर्ट के लिए "OpenSSL" चुनें।
     - लाइन एंडिंग्स के लिए "Checkout as-is, commit as-is" चुनें (जब तक कि क्रॉस-प्लेटफ़ॉर्म टीमों के साथ काम न कर रहे हों)।
   - इंस्टॉलेशन पूरी करें।

2. **TortoiseGit इंस्टॉल करें**:
   - नवीनतम संस्करण [TortoiseGit की आधिकारिक वेबसाइट](https://tortoisegit.org/download/) से डाउनलोड करें।
   - इंस्टॉलर चलाएं:
     - डिफ़ॉल्ट भाषा और कंपोनेंट्स चुनें।
     - सुनिश्चित करें कि Git for Windows डिटेक्ट हो गया है (अगर यह गायब है तो TortoiseGit प्रॉम्प्ट करेगा)।
     - अगर ज़रूरत हो तो TortoiseGitPlink इंस्टॉल करें (SSH के लिए अनुशंसित)।
   - अगर प्रॉम्प्ट किया जाए तो अपने कंप्यूटर को रीस्टार्ट करें।

3. **इंस्टॉलेशन सत्यापित करें**:
   - विंडोज एक्सप्लोरर में किसी भी फ़ोल्डर पर राइट-क्लिक करें। आपको TortoiseGit विकल्प दिखने चाहिए जैसे "Git Clone," "Git Create Repository here," आदि।

### प्रारंभिक कॉन्फ़िगरेशन
इंस्टॉलेशन के बाद, अपने यूजर विवरण और प्राथमिकताओं के लिए TortoiseGit को कॉन्फ़िगर करें:
1. **यूजर जानकारी सेट करें**:
   - किसी फ़ोल्डर में राइट-क्लिक करें, **TortoiseGit > Settings** चुनें।
   - सेटिंग्स विंडो में, **Git > Config** पर नेविगेट करें।
   - अपना नाम और ईमेल दर्ज करें (वही जो GitHub, GitLab, आदि में उपयोग किया गया है):
     ```
     Name: आपका नाम
     Email: your.email@example.com
     ```
   - **Apply** और **OK** क्लिक करें।

2. **SSH कॉन्फ़िगर करें (वैकल्पिक)**:
   - अगर रिमोट रिपॉजिटरी के लिए SSH का उपयोग कर रहे हैं, तो एक SSH कुंजी सेट अप करें:
     - **PuTTYgen** खोलें (TortoiseGit के साथ इंस्टॉल होता है)।
     - एक नया SSH key pair जनरेट करें, प्राइवेट की सेव करें, और पब्लिक की कॉपी करें।
     - पब्लिक की को अपनी Git होस्टिंग सर्विस (जैसे, GitHub, GitLab) में एड करें।
     - TortoiseGit सेटिंग्स में, **Git > Remote** के अंदर, SSH क्लाइंट के रूप में TortoiseGitPlink चुनें।

3. **डिफ और मर्ज टूल्स सेट करें**:
   - **TortoiseGit > Settings > Diff Viewer** में, TortoiseGitMerge (डिफ़ॉल्ट) जैसे टूल या Beyond Compare जैसे एक्सटर्नल टूल को चुनें।
   - मर्जिंग के लिए, **Merge Tool** के अंदर कॉन्फ़िगर करें (शुरुआती लोगों के लिए TortoiseGitMerge अनुशंसित है)।

### बेसिक उपयोग
नीचे सामान्य TortoiseGit ऑपरेशन्स दिए गए हैं, जो विंडोज एक्सप्लोरर में राइट-क्लिक कॉन्टेक्स्ट मेन्यू के माध्यम से एक्सेस किए जा सकते हैं।

#### 1. **रिपॉजिटरी क्लोन करना**
   - किसी फ़ोल्डर में राइट-क्लिक करें और **Git Clone** चुनें।
   - डायलॉग में:
     - रिपॉजिटरी URL दर्ज करें (जैसे, `https://github.com/username/repo.git`)।
     - रिपॉजिटरी के लिए लोकल डायरेक्टरी निर्दिष्ट करें।
     - वैकल्पिक रूप से, कोई ब्रांच चुनें या SSH keys लोड करें।
   - रिपॉजिटरी को क्लोन करने के लिए **OK** क्लिक करें।

#### 2. **नई रिपॉजिटरी बनाना**
   - किसी फ़ोल्डर पर नेविगेट करें, राइट-क्लिक करें, और **Git Create Repository here** चुनें।
   - अगर सर्वर-साइड रिपॉजिटरी बना रहे हैं तो "Make it Bare" चेक करें (लोकल उपयोग के लिए दुर्लभ)।
   - **OK** क्लिक करें। एक `.git` फ़ोल्डर बन जाता है, जो रिपॉजिटरी को इनिशियलाइज़ करता है।

#### 3. **परिवर्तन कमिट करना**
   - अपनी रिपॉजिटरी फ़ोल्डर में फाइल्स एड करें।
   - फ़ोल्डर या चयनित फाइल्स पर राइट-क्लिक करें, फिर **Git Commit -> "main"** (या वर्तमान ब्रांच) चुनें।
   - कमिट डायलॉग में:
     - परिवर्तनों का वर्णन करते हुए एक कमिट मैसेज दर्ज करें।
     - स्टेज करने के लिए फाइल्स चुनें (चेक बॉक्स)।
     - रिमोट रिपॉजिटरी में परिवर्तन पुश करने के लिए **OK** या **Commit & Push** क्लिक करें।

#### 4. **परिवर्तन पुश करना**
   - कमिट करने के बाद, राइट-क्लिक करें और **TortoiseGit > Push** चुनें।
   - रिमोट रिपॉजिटरी और ब्रांच चुनें।
   - अगर प्रॉम्प्ट किया जाए तो ऑथेंटिकेट करें (HTTPS के लिए यूजरनेम/पासवर्ड या SSH key)।
   - पुश करने के लिए **OK** क्लिक करें।

#### 5. **परिवर्तन पुल करना**
   - अपनी लोकल रिपॉजिटरी को रिमोट परिवर्तनों के साथ अपडेट करने के लिए, राइट-क्लिक करें और **TortoiseGit > Pull** चुनें।
   - रिमोट ब्रांच चुनें और **OK** क्लिक करें।
   - अगर प्रॉम्प्ट किया जाए तो कॉन्फ्लिक्ट्स रिज़ॉल्व करें (मर्ज टूल का उपयोग करें)।

#### 6. **ब्रांचेज बनाना और स्विच करना**
   - राइट-क्लिक करें और **TortoiseGit > Create Branch** चुनें।
   - एक ब्रांच नाम दर्ज करें और **OK** क्लिक करें।
   - ब्रांच स्विच करने के लिए, राइट-क्लिक करें और **TortoiseGit > Switch/Checkout** चुनें, फिर ब्रांच चुनें।

#### 7. **हिस्ट्री देखना**
   - राइट-क्लिक करें और **TortoiseGit > Show Log** चुनें।
   - कमिट हिस्ट्री देखें, जिसमें लेखक, तारीख और संदेश शामिल हैं।
   - परिवर्तन देखने, रिवर्ट करने, या चेरी-पिक करने के लिए किसी कमिट पर राइट-क्लिक करें।

#### 8. **मर्ज कॉन्फ्लिक्ट्स रिज़ॉल्व करना**
   - पुल या मर्ज के दौरान, अगर कॉन्फ्लिक्ट होते हैं, तो TortoiseGit आपको सूचित करेगा।
   - कॉन्फ्लिक्ट करने वाली फाइल्स पर राइट-क्लिक करें और **TortoiseGit > Resolve** चुनें।
   - कॉन्फ्लिक्ट्स को मैन्युअली एडिट करने के लिए मर्ज टूल का उपयोग करें, फिर रिज़ॉल्व के रूप में मार्क करें।
   - रिज़ॉल्व किए गए परिवर्तनों को कमिट करें।

### एडवांस्ड फीचर्स
1. **परिवर्तन स्टैश करना**:
   - अनकमिटेड परिवर्तनों को अस्थायी रूप से सेव करने के लिए, राइट-क्लिक करें और **TortoiseGit > Stash Save** चुनें।
   - स्टैश किए गए परिवर्तनों को पुनः प्राप्त करने के लिए, **TortoiseGit > Stash Pop** चुनें।

2. **रिबेसिंग**:
   - राइट-क्लिक करें और **TortoiseGit > Rebase** चुनें।
   - रिबेस करने के लिए ब्रांच चुनें और कमिट्स को रीऑर्डर या स्क्वैश करने के लिए प्रॉम्प्ट का पालन करें।

3. **सबमॉड्यूल्स**:
   - सबमॉड्यूल्स को मैनेज करने के लिए, राइट-क्लिक करें और **TortoiseGit > Submodule Update** या **Add** चुनें।
   - TortoiseGit सेटिंग्स में सबमॉड्यूल सेटिंग्स कॉन्फ़िगर करें।

4. **बाइसेक्टिंग**:
   - बग पैदा करने वाली कमिट ढूंढने के लिए, **TortoiseGit > Bisect Start** का उपयोग करें।
   - समस्याग्रस्त कमिट को निर्धारित करने के लिए कमिट्स को "good" या "bad" के रूप में मार्क करें।

### बेस्ट प्रैक्टिसेज
- **अक्सर कमिट करें**: छोटे, लगातार कमिट स्पष्ट संदेशों के साथ बनाएं।
- **नियमित रूप से पुल करें**: कॉन्फ्लिक्ट्स से बचने के लिए अपनी लोकल रिपॉजिटरी को अपडेटेड रखें।
- **ब्रांचेज का उपयोग करें**: मुख्य ब्रांच को स्थिर रखने के लिए नए काम के लिए फीचर ब्रांचेज बनाएं।
- **SSH Keys का बैकअप लें**: SSH keys को सुरक्षित रूप से स्टोर करें और उनका बैकअप लें।
- **परिवर्तनों की समीक्षा करें**: कमिट करने से पहले डिफ व्यूअर का उपयोग करके परिवर्तनों की समीक्षा करें।

### समस्या निवारण
- **ऑथेंटिकेशन समस्याएं**: सुनिश्चित करें कि SSH keys या क्रेडेंशियल्स आपकी Git होस्टिंग सर्विस में सही ढंग से कॉन्फ़िगर हैं।
- **मर्ज कॉन्फ्लिक्ट्स**: कॉन्फ्लिक्ट्स को विजुअली रिज़ॉल्व करने के लिए TortoiseGitMerge का उपयोग करें। मर्ज करने से पहले फाइल्स का बैकअप लें।
- **कॉन्टेक्स्ट मेन्यू गायब**: जांचें कि क्या TortoiseGit सही ढंग से इंस्टॉल है और एक्सप्लोरर इंटीग्रेशन सेटिंग्स में सक्षम है।
- **धीमा प्रदर्शन**: **TortoiseGit > Settings > Icon Overlays** में अनावश्यक ओवरले अक्षम करें।

### उदाहरण वर्कफ़्लो
नीचे किसी प्रोजेक्ट में योगदान देने के लिए एक सैंपल वर्कफ़्लो दिया गया है:

# TortoiseGit सैंपल वर्कफ़्लो

## 1. रिपॉजिटरी क्लोन करें
- किसी फ़ोल्डर में राइट-क्लिक करें, **Git Clone** चुनें।
- रिपॉजिटरी URL और लोकल डायरेक्टरी दर्ज करें।
- **OK** क्लिक करें।

## 2. फीचर ब्रांच बनाएं
- राइट-क्लिक करें, **TortoiseGit > Create Branch** चुनें।
- ब्रांच का नाम दें (जैसे, `feature/add-login`)।
- **OK** क्लिक करें और नई ब्रांच पर स्विच करें।

## 3. परिवर्तन करें
- रिपॉजिटरी फ़ोल्डर में फाइल्स एडिट करें।
- राइट-क्लिक करें, **Git Commit -> "feature/add-login"** चुनें।
- एक कमिट मैसेज लिखें और फाइल्स स्टेज करें।
- **Commit** क्लिक करें।

## 4. परिवर्तन पुश करें
- राइट-क्लिक करें, **TortoiseGit > Push** चुनें।
- रिमोट ब्रांच चुनें (जैसे, `origin/feature/add-login`)।
- **OK** क्लिक करें।

## 5. पुल रिक्वेस्ट बनाएं
- अपनी Git होस्टिंग सर्विस (जैसे, GitHub) पर, अपनी ब्रांच से मुख्य ब्रांच में एक पुल रिक्वेस्ट बनाएं।
- समीक्षा और मर्ज होने की प्रतीक्षा करें।

## 6. लोकल रिपॉजिटरी अपडेट करें
- मुख्य ब्रांच पर स्विच करें (**TortoiseGit > Switch/Checkout**)।
- अपडेट्स पुल करें (**TortoiseGit > Pull**)।
