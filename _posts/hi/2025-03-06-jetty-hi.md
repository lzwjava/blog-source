Java प्रोजेक्ट में `org.eclipse.jetty` का उपयोग करने के लिए, आप आमतौर पर Jetty लाइब्रेरीज़ का लाभ उठाकर अपने एप्लिकेशन के भीतर एक हल्का वेब सर्वर या सर्वलेट कंटेनर एम्बेड करना चाहते हैं। Jetty, जिसे Eclipse Foundation के तहत मेंटेन किया जाता है, एक ओपन-सोर्स प्रोजेक्ट है जो आपको वेब कंटेंट सर्व करने, HTTP रिक्वेस्ट्स को हैंडल करने और सर्वलेट्स और वेब एप्लिकेशन्स जैसी सुविधाओं को सपोर्ट करने की अनुमति देता है। नीचे Maven को बिल्ड टूल के रूप में उपयोग करते हुए, एक Java एप्लिकेशन में बेसिक वेब सर्वर सेट अप करने के लिए `org.eclipse.jetty` का उपयोग शुरू करने के लिए एक चरण-दर-चरण गाइड दी गई है।

---

### **चरण 1: अपने प्रोजेक्ट में Jetty डिपेंडेंसीज़ जोड़ें**
Jetty का उपयोग करने के लिए, आपको अपने प्रोजेक्ट में इसकी लाइब्रेरीज़ शामिल करनी होंगी। यदि आप Maven का उपयोग कर रहे हैं, तो अपनी `pom.xml` फ़ाइल में आवश्यक डिपेंडेंसीज़ जोड़ें। सर्वलेट सपोर्ट के साथ एक बेसिक वेब सर्वर सेटअप के लिए, `jetty-server` और `jetty-servlet` शामिल करें। यहाँ एक उदाहरण दिया गया है:

```xml
<project>
    <dependencies>
        <!-- Jetty Server -->
        <dependency>
            <groupId>org.eclipse.jetty</groupId>
            <artifactId>jetty-server</artifactId>
            <version>9.4.31.v20200723</version>
        </dependency>
        <!-- Jetty Servlet Support -->
        <dependency>
            <groupId>org.eclipse.jetty</groupId>
            <artifactId>jetty-servlet</artifactId>
            <version>9.4.31.v20200723</version>
        </dependency>
        <!-- Servlet API (optional, often included transitively) -->
        <dependency>
            <groupId>javax.servlet</groupId>
            <artifactId>javax.servlet-api</artifactId>
            <version>3.1.0</version>
            <scope>provided</scope>
        </dependency>
    </dependencies>
</project>
```

- **`jetty-server`**: HTTP रिक्वेस्ट्स को हैंडल करने के लिए कोर सर्वर फंक्शनैलिटी प्रदान करता है।
- **`jetty-servlet`**: सर्वलेट्स के लिए सपोर्ट जोड़ता है, जिससे आप Java Servlet API का उपयोग कर सकते हैं।
- **`javax.servlet-api`**: सर्वलेट इंटरफेस को परिभाषित करता है; यहाँ स्पष्टता के लिए एक्सप्लिसिटली शामिल किया गया है, हालाँकि यह अक्सर `jetty-servlet` द्वारा ट्रांजिटिवली आ जाता है।

**नोट**: नवीनतम Jetty वर्जन के लिए [Maven Central Repository](https://mvnrepository.com/artifact/org.eclipse.jetty) चेक करें और `<version>` टैग को तदनुसार अपडेट करें।

### **चरण 2: एक साधारण Jetty सर्वर बनाएँ**
आप `org.eclipse.jetty.server` पैकेज से `Server` क्लास का एक इंस्टेंस बनाकर Jetty का प्रोग्रामेटिक रूप से उपयोग कर सकते हैं। नीचे दो सामान्य तरीके दिए गए हैं: स्टेटिक रिस्पॉन्स के लिए एक बेसिक हैंडलर का उपयोग करना या डायनामिक कंटेंट के लिए एक सर्वलेट का उपयोग करना।

#### **विकल्प 1: एक बेसिक हैंडलर का उपयोग करना**
यह उदाहरण एक ऐसा सर्वर सेट अप करता है जो सभी रिक्वेस्ट्स का जवाब "Hello, World!" देता है।

```java
import org.eclipse.jetty.server.Server;
import org.eclipse.jetty.server.Request;
import org.eclipse.jetty.server.handler.AbstractHandler;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;

public class SimpleJettyServer {
    public static void main(String[] args) throws Exception {
        // पोर्ट 8080 पर लिसन करते हुए एक सर्वर इंस्टेंस बनाएँ
        Server server = new Server(8080);

        // रिक्वेस्ट्स को प्रोसेस करने के लिए एक हैंडलर सेट करें
        server.setHandler(new AbstractHandler() {
            @Override
            public void handle(String target, Request baseRequest, HttpServletRequest request, HttpServletResponse response)
                    throws IOException {
                // रिस्पॉन्स प्रॉपर्टीज़ सेट करें
                response.setContentType("text/plain");
                response.setStatus(HttpServletResponse.SC_OK);
                response.getWriter().println("Hello, World!");

                // रिक्वेस्ट को हैंडल किया गया चिह्नित करें
                baseRequest.setHandled(true);
            }
        });

        // सर्वर स्टार्ट करें
        server.start();
        server.join(); // सर्वर को रुकने तक चलता रहने देता है
    }
}
```

- **`Server server = new Server(8080)`**: पोर्ट 8080 पर एक Jetty सर्वर बनाता है।
- **`AbstractHandler`**: एक साधारण हैंडलर जहाँ आप `handle` मेथड में रिक्वेस्ट्स को कैसे प्रोसेस करना है, यह परिभाषित करते हैं।
- **`baseRequest.setHandled(true)`**: इंगित करता है कि इस हैंडलर ने रिक्वेस्ट को प्रोसेस कर लिया है, जिससे आगे की प्रोसेसिंग रुक जाती है।

इस कोड को रन करें, फिर ब्राउज़र खोलें और `http://localhost:8080/` पर नेविगेट करके "Hello, World!" देखें।

#### **विकल्प 2: एक सर्वलेट का उपयोग करना**
अधिक जटिल एप्लिकेशन्स के लिए, आप `ServletContextHandler` के साथ सर्वलेट्स का उपयोग कर सकते हैं।

```java
import org.eclipse.jetty.server.Server;
import org.eclipse.jetty.servlet.ServletContextHandler;
import org.eclipse.jetty.servlet.ServletHolder;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;

public class ServletJettyServer {
    public static void main(String[] args) throws Exception {
        // पोर्ट 8080 पर एक सर्वर इंस्टेंस बनाएँ
        Server server = new Server(8080);

        // एक सर्वलेट कॉन्टेक्स्ट हैंडलर बनाएँ
        ServletContextHandler context = new ServletContextHandler();
        context.setContextPath("/"); // रूट कॉन्टेक्स्ट पाथ

        // सर्वर के लिए हैंडलर सेट करें
        server.setHandler(context);

        // रिक्वेस्ट्स को हैंडल करने के लिए एक सर्वलेट जोड़ें
        context.addServlet(new ServletHolder(new HelloServlet()), "/*");

        // सर्वर स्टार्ट करें
        server.start();
        server.join();
    }
}

// एक साधारण सर्वलेट को परिभाषित करें
class HelloServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws IOException {
        response.setContentType("text/plain");
        response.setStatus(HttpServletResponse.SC_OK);
        response.getWriter().println("Hello from Jetty Servlet!");
    }
}
```

- **`ServletContextHandler`**: सर्वलेट कॉन्टेक्स्ट्स और मैपिंग को मैनेज करता है।
- **`context.addServlet`**: `HelloServlet` को सभी रिक्वेस्ट्स (`/*`) को हैंडल करने के लिए रजिस्टर करता है।
- **`doGet`**: GET रिक्वेस्ट्स के लिए सर्वलेट के व्यवहार को परिभाषित करता है।

इस कोड को रन करें और `http://localhost:8080/` पर विजिट करके "Hello from Jetty Servlet!" देखें।

### **चरण 3: कंपाइल और रन करें**
1. सुनिश्चित करें कि आपका प्रोजेक्ट Maven के साथ सेट अप है (उदाहरण के लिए, डिपेंडेंसीज़ डाउनलोड करने के लिए `mvn compile` रन करें)।
2. Java क्लास को कंपाइल और रन करें (उदाहरण के लिए, `java SimpleJettyServer` या अपने IDE की रन सुविधा का उपयोग करें)।
3. ब्राउज़र में `http://localhost:8080/` एक्सेस करके या `curl` जैसे टूल का उपयोग करके सर्वर को टेस्ट करें।

### **अतिरिक्त उपयोग के मामले**
- **WAR फ़ाइल डिप्लॉय करना**: वेब एप्लिकेशन आर्काइव (WAR) फ़ाइल या डायरेक्टरी को डिप्लॉय करने के लिए `WebAppContext` का उपयोग करें:
  ```java
  import org.eclipse.jetty.server.Server;
  import org.eclipse.jetty.webapp.WebAppContext;

  public class WarJettyServer {
      public static void main(String[] args) throws Exception {
          Server server = new Server(8080);
          WebAppContext webapp = new WebAppContext();
          webapp.setContextPath("/");
          webapp.setWar("path/to/your.war"); // आपकी WAR फ़ाइल का पथ
          server.setHandler(webapp);
          server.start();
          server.join();
      }
  }
  ```
  इस सुविधा के लिए अपनी `pom.xml` में `jetty-webapp` डिपेंडेंसी जोड़ें।

- **Jetty कॉन्फ़िगर करना**: पोर्ट, थ्रेड पूल, या HTTPS जैसी सेटिंग्स को प्रोग्रामेटिक रूप से या XML कॉन्फ़िगरेशन फ़ाइलों के माध्यम से एडजस्ट करें (एडवांस्ड सेटअप्स के लिए Jetty डॉक्यूमेंटेशन देखें)।

### **मुख्य बिंदु**
- **इम्पोर्ट्स**: `org.eclipse.jetty.server`, `org.eclipse.jetty.servlet`, आदि से क्लासेज़ का उपयोग करें, जैसी आवश्यकता हो उन्हें इम्पोर्ट करके।
- **लचीलापन**: Jetty स्टेटिक कंटेंट सर्व कर सकता है, सर्वलेट्स को हैंडल कर सकता है, WAR फ़ाइलें डिप्लॉय कर सकता है, या अतिरिक्त मॉड्यूल्स के साथ WebSockets और HTTP/2 को भी सपोर्ट कर सकता है।
- **डॉक्यूमेंटेशन**: अधिक जानकारी के लिए, [आधिकारिक Jetty डॉक्यूमेंटेशन](https://www.eclipse.org/jetty/documentation/) देखें।

इन चरणों का पालन करके, आप अपनी विशिष्ट आवश्यकताओं के अनुरूप, अपने Java एप्लिकेशन में एक वेब सर्वर एम्बेड करने के लिए `org.eclipse.jetty` का प्रभावी ढंग से उपयोग कर सकते हैं!
