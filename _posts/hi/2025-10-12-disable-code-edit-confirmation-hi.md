Claude Code में एडिट कन्फर्मेशन पॉपअप को डिसेबल करने के लिए (ताकि यह `sync_githook.py` जैसी फाइलों में बिना पूछे सीधे एडिट्स अप्लाई करे), आपको अपनी `settings.json` फाइल में permissions को कॉन्फ़िगर करना होगा। यही वह "Claude Code JSON" है जिसकी आप बात कर रहे हैं—यह यूजर-लेवल कॉन्फ़िग्स के लिए Claude Code की मुख्य सेटिंग्स फाइल है।

### इसे सेट अप करने के Steps

1. **फाइल को ढूंढें या बनाएं**:
   - फाइल `~/.claude/settings.json` पर है (macOS/Linux पर) या `%USERPROFILE%\.claude\settings.json` पर (Windows पर)। `.claude` फोल्डर हिडन होता है, इसलिए वहां नेविगेट करने के लिए अपने terminal/file explorer का उपयोग करें।
   - अगर फाइल या फोल्डर मौजूद नहीं है, तो उन्हें बनाएं। आप यह Claude Code के `/config` कमांड को इसके इंटरएक्टिव REPL में चलाकर कर सकते हैं (यह आपको सेटिंग्स इंटरफेस खोलने के लिए गाइड करेगा)।

2. **JSON को एडिट करें**:
   - `settings.json` को किसी भी टेक्स्ट एडिटर (जैसे, VS Code) में खोलें।
   - सभी एडिट एक्शन्स के लिए प्रॉम्प्ट्स को डिसेबल करने के लिए `permissions` सेक्शन को एड या अपडेट करें। यहां एक सिंपल कॉन्फ़िग है:

     ```json:disable-run
     {
       "permissions": {
         "defaultMode": "acceptEdits",
         "allow": [
           "Edit(*)",
           "Write(*)",
           "MultiEdit(*)"
         ],
         "ask": []
       }
     }
     ```

     - **यह क्या करता है**:
       - `"defaultMode": "acceptEdits"`: ग्लोबल डिफॉल्ट को बिना प्रॉम्प्ट किए एडिट्स स्वीकार करने के लिए सेट करता है।
       - `"allow"`: किसी भी फाइल (`*` वाइल्डकार्ड) के लिए `Edit` (सिंगल-फाइल एडिट्स के लिए), `Write` (कंटेंट लिखने के लिए), और `MultiEdit` (बैच चेंजेस के लिए) जैसे स्पेसिफिक टूल्स को प्री-अप्रूव करता है।
       - `"ask": []`: यह सुनिश्चित करता है कि कोई भी एक्शन प्रॉम्प्ट न ट्रिगर करे, जब तक कि एक्सप्लिसिटली कहीं और डिनाइ न किया गया हो।
     - फाइल सेव करें। बदलाव आपके अगले Claude Code सेशन में तुरंत लागू हो जाते हैं।

3. **इसे टेस्ट करें**:
   - Claude Code को रीस्टार्ट करें या सेशन को रीलोड करें।
   - इसे किसी फाइल को एडिट करने के लिए कहें (उदाहरण के लिए, "एक कमेंट जोड़ने के लिए sync_githook.py को एडिट करें")। इसे बिना पॉपअप के बदलाव अप्लाई करना चाहिए।
   - अगर आप प्रोजेक्ट-स्पेसिफिक सेटिंग्स चाहते हैं (उदाहरण के लिए, सिर्फ इस repo के लिए), अपने प्रोजेक्ट के रूट फोल्डर में `.claude/settings.json` बनाएं।

4. **ऑप्शनल ट्वीक्स**:
   - अधिक ग्रेन्युलर कंट्रोल के लिए (उदाहरण के लिए, सिर्फ कुछ डायरेक्टरीज में एडिट्स की अनुमति दें): `"Edit(*)"` को `"Edit(./src/**)"` में बदलें।
   - अगर प्रॉम्प्ट्स बनी रहती हैं, तो करंट रूल्स चेक करने के लिए REPL में `/allowed-tools` रन करें, या UI के जरिए ट्वीक करने के लिए `/config` का उपयोग करें।
   - सिक्योरिटी के लिए, अत्यधिक ब्रॉड `allow` रूल्स से बचें—सिंपल शुरू करें और एक्सपैंड करें।

यह आपके वर्कफ्लो को बिना रुकावटों के तेज बना देगा। अगर आप एंटरप्राइज सेटअप में हैं, तो अपने एडमिन से चेक करें, क्योंकि मैनेज्ड पॉलिसीज यूजर सेटिंग्स को ओवरराइड कर सकती हैं।
