Microsoft Entra ID एक क्लाउड-आधारित पहचान और पहुंच प्रबंधन सेवा है जो एप्लिकेशन, डेटा और डिवाइस जैसे संसाधनों तक सुरक्षित पहुंच में मदद करती है। यह प्रमाणीकरण (जैसे साइन-इन विधियां, MFA), प्राधिकरण (जैसे भूमिका-आधारित पहुंच), और Azure सेवाओं के साथ एकीकरण का समर्थन करती है। आरंभ करने के लिए, आपको एक Azure सदस्यता और उचित अनुमतियों (जैसे वैश्विक व्यवस्थापक भूमिका) की आवश्यकता होगी।

## आरंभ करना
1. **Azure पोर्टल में साइन इन करें**: [portal.azure.com](https://portal.azure.com) पर जाएं और अपने Microsoft अकाउंट से साइन इन करें।
2. **Microsoft Entra ID पर नेविगेट करें**: शीर्ष सर्च बार में "Microsoft Entra ID" खोजें या इसे "Azure सेवाएं" के अंतर्गत ढूंढें।
3. **डैशबोर्ड एक्सप्लोर करें**: अपने टेनेंट अवलोकन की समीक्षा करें, जिसमें उपयोगकर्ता, समूह और ऐप शामिल हैं। आवश्यकता पड़ने पर कस्टम डोमेन जैसी बुनियादी सेटिंग्स को सेट अप करें।
4. **मुख्य सुविधाएं सक्षम करें**:
   - **प्रमाणीकरण**: "प्रमाणीकरण विधियां" के अंतर्गत सेल्फ-सर्विस पासवर्ड रीसेट या मल्टीफैक्टर प्रमाणीकरण (MFA) कॉन्फ़िगर करें।
   - **सशर्त पहुंच**: "सुरक्षा" > "सशर्त पहुंच" के अंतर्गत नीतियां बनाएं ताकि उपयोगकर्ता, डिवाइस या स्थान के आधार पर नियम लागू किए जा सकें।

## उपयोगकर्ताओं और समूहों का प्रबंधन
- **उपयोगकर्ता जोड़ें**: "उपयोगकर्ता" > "नया उपयोगकर्ता" पर जाएं। नाम, उपयोगकर्तानाम (जैसे user@yourdomain.com) जैसे विवरण दर्ज करें और भूमिकाएं या लाइसेंस असाइन करें।
- **समूह बनाएं**: "समूह" > "नया समूह" के अंतर्गत, सुरक्षा या Microsoft 365 प्रकार चुनें, सदस्य जोड़ें, और पहुंच असाइनमेंट के लिए उपयोग करें।
- **लाइसेंस असाइन करें**: उपयोगकर्ता/समूह विवरण में, "लाइसेंस" पर जाएं और उन्नत सुविधाओं (जैसे Privileged Identity Management - PIM) के लिए Entra ID P1/P2 असाइन करें।
- **सर्वोत्तम अभ्यास**: न्यूनतम विशेषाधिकार के सिद्धांत का पालन करें—न्यूनतम अनुमतियां असाइन करें और बल्क प्रबंधन के लिए समूहों का उपयोग करें।

## एप्लिकेशन प्रबंधन
- **एक ऐप पंजीकृत करें**: "ऐप पंजीकरण" > "नया पंजीकरण" के अंतर्गत, नाम, रीडायरेक्ट URI, और समर्थित अकाउंट प्रकार (सिंगल-टेनेंट, मल्टी-टेनेंट, आदि) प्रदान करें।
- **एंटरप्राइज़ ऐप जोड़ें**: तृतीय-पक्ष ऐप्स के लिए, "एंटरप्राइज़ एप्लिकेशन" > "नया एप्लिकेशन" पर जाएं ताकि गैलरी ब्राउज़ कर सकें या नॉन-गैलरी ऐप्स बना सकें।
- **पहुंच कॉन्फ़िगर करें**: "उपयोगकर्ता और समूह" के अंतर्गत ऐप को उपयोगकर्ताओं/समूहों को असाइन करें, और SAML या OAuth के माध्यम से सिंगल साइन-ऑन (SSO) सेट अप करें।
- **पहचान प्रोविजन करें**: जस्ट-इन-टाइम एक्सेस के लिए ऐप्स में उपयोगकर्ता सिंक को "प्रोविजनिंग" के अंतर्गत स्वचालित करें।

हाइब्रिड सेटअप (ऑन-प्रिमाइसेस AD) के लिए, पहचानों को सिंक करने के लिए Microsoft Entra Connect का उपयोग करें। "मॉनिटरिंग" > "साइन-इन लॉग" के अंतर्गत लॉग के माध्यम से उपयोग की निगरानी करें।

# डेटाबेस, Kubernetes (AKS), या अन्य संसाधन तक पहुंच कैसे जांचें

Azure में पहुंच Role-Based Access Control (RBAC) के माध्यम से प्रबंधित की जाती है, जो Entra ID के साथ एकीकृत होती है। उपयोगकर्ता Entra क्रेडेंशियल्स के साथ प्रमाणित होते हैं, और भूमिकाएं अनुमतियां परिभाषित करती हैं। पहुंच जांचने के लिए, Azure पोर्टल के IAM (Identity and Access Management) टूल्स का उपयोग करें। यह सीधे असाइनमेंट, पैरेंट स्कोप (जैसे सदस्यता) से प्राप्त असाइनमेंट, और अस्वीकार असाइनमेंट सूचीबद्ध करता है।

## किसी भी Azure संसाधन के लिए सामान्य चरण
1. **संसाधन खोलें**: Azure पोर्टल में, संसाधन (जैसे संसाधन समूह, VM, स्टोरेज अकाउंट) पर नेविगेट करें।
2. **एक्सेस कंट्रोल (IAM) पर जाएं**: बाएं मेनू से "एक्सेस कंट्रोल (IAM)" चुनें।
3. **पहुंच जांचें**:
   - अपनी स्वयं की पहुंच के लिए: "चेक एक्सेस" > "व्यू माई एक्सेस" पर क्लिक करें ताकि इस स्कोप और प्राप्त असाइनमेंट देख सकें।
   - किसी विशिष्ट उपयोगकर्ता/समूह/सर्विस प्रिंसिपल के लिए:
     - "चेक एक्सेस" > "चेक एक्सेस" पर क्लिक करें।
     - "उपयोगकर्ता, समूह, या सर्विस प्रिंसिपल" चुनें।
     - नाम या ईमेल से खोजें।
     - भूमिका असाइनमेंट (जैसे मालिक, योगदानकर्ता) और प्रभावी अनुमतियों के लिए परिणाम पेन देखें।
4. **योग्य असाइनमेंट देखें** (यदि PIM उपयोग कर रहे हैं): जस्ट-इन-टाइम भूमिकाओं के लिए "योग्य असाइनमेंट" टैब पर स्विच करें।
5. **PowerShell/CLI विकल्प**: स्क्रिप्टेड जांच के लिए `az role assignment list --assignee <user> --scope <resource-id>` का उपयोग करें।

नोट: इसमें चाइल्ड-स्कोप असाइनमेंट शामिल नहीं हैं; आवश्यकता पड़ने पर ड्रिल डाउन करें।

## Azure SQL डेटाबेस तक पहुंच जांचना
Azure SQL कंटेन्ड डेटाबेस उपयोगकर्ताओं (Entra पहचानों से बंधे, SQL लॉगिन नहीं) के लिए Entra प्रमाणीकरण का उपयोग करता है।
1. **Entra एडमिन कॉन्फ़िगर करें (यदि सेट नहीं है)**: SQL सर्वर अवलोकन > सेटिंग्स के अंतर्गत "Microsoft Entra ID" > "एडमिन सेट करें" में। एक उपयोगकर्ता/समूह खोजें और चुनें, फिर सेव करें। यह क्लस्टर-वाइड Entra ऑथ सक्षम करता है।
2. **सर्वर-लेवल एक्सेस जांचें**:
   - SQL सर्वर पेन > "Microsoft Entra ID" में, एडमिन फील्ड देखें ताकि असाइन की गई पहचान देख सकें।
   - `master` डेटाबेस क्वेरी करें: `SELECT name, type_desc FROM sys.database_principals WHERE type IN ('E', 'X');` (E एक्सटर्नल उपयोगकर्ताओं के लिए, X एक्सटर्नल समूहों के लिए)।
3. **डेटाबेस-लेवल एक्सेस जांचें**:
   - Entra ऑथ का उपयोग करके SSMS के साथ डेटाबेस से कनेक्ट करें (कनेक्शन डायलॉग में "Microsoft Entra - यूनिवर्सल विद MFA" चुनें)।
   - उपयोगकर्ताओं और भूमिकाओं की सूची देखने के लिए `SELECT * FROM sys.database_principals;` या `EXEC sp_helprolemember;` रन करें।
4. **समस्या निवारण**: यदि लॉगिन विफल होता है (जैसे त्रुटि 33134), जांचें कि Entra सशर्त पहुंच नीतियां Microsoft Graph API एक्सेस की अनुमति देती हैं या नहीं।

उपयोगकर्ताओं को डिफ़ॉल्ट रूप से `CONNECT` मिलता है; T-SQL के माध्यम से `db_datareader` जैसी भूमिकाएं प्रदान करें: `ALTER ROLE db_datareader ADD MEMBER [user@domain.com];`.

## AKS (Kubernetes क्लस्टर) तक पहुंच जांचना
AKS, प्रमाणीकरण के लिए Entra ID और प्राधिकरण के लिए Azure RBAC या Kubernetes RBAC का उपयोग करता है।
1. **Azure-लेवल एक्सेस (AKS संसाधन तक)**:
   - AKS क्लस्टर संसाधन पर ऊपर दिए गए सामान्य चरणों का पालन करें।
   - सामान्य भूमिकाएं: पूर्ण kubeconfig एक्सेस के लिए "Azure Kubernetes Service क्लस्टर एडमिन"; व्यू-ओनली के लिए "रीडर"।
2. **Kubernetes API एक्सेस**:
   - `az login` के माध्यम से प्रमाणित करें (Entra टोकन का उपयोग करता है)।
   - kubeconfig प्राप्त करें: `az aks get-credentials --resource-group <rg> --name <cluster>`।
   - बाइंडिंग जांचें: `kubectl auth can-i <verb> <resource> --as <user@domain.com> -n <namespace>`।
   - क्लस्टर भूमिकाएं सूचीबद्ध करें: `kubectl get clusterrolebindings | grep <user-or-group>`।
3. **Entra समूहों का उपयोग करना**: स्केलेबिलिटी के लिए Kubernetes भूमिकाओं/बाइंडिंग को समूहों को असाइन करें। एक नेमस्पेस भूमिका के लिए उदाहरण YAML (`kubectl apply -f file.yaml` के साथ लागू करें):
   ```
   apiVersion: rbac.authorization.k8s.io/v1
   kind: RoleBinding
   metadata:
     name: example-binding
     namespace: default
   subjects:
   - kind: Group
     name: finance-group  # Entra समूह
     apiGroup: rbac.authorization.k8s.io
   roleRef:
     kind: Role
     name: example-role
     apiGroup: rbac.authorization.k8s.io
   ```
4. **सर्वोत्तम अभ्यास**: न्यूनतम विशेषाधिकार भूमिकाओं का उपयोग करें, जस्ट-इन-टाइम एलिवेशन के लिए PIM के साथ एकीकृत करें, और फिक्स्ड क्रेडेंशियल्स से बचें—पॉड्स के लिए वर्कलोड आइडेंटिटीज का उपयोग करें।

संसाधनों (जैसे AKS से SQL) तक पहुंचने वाले पॉड्स के लिए, मैनेज्ड आइडेंटिटीज असाइन करें और उन आइडेंटिटीज पर Azure RBAC के माध्यम से जांचें।
