नीचे **सबसे अधिक उपयोग किए जाने वाले Java भाषा फीचर्स** का एक विस्तृत और विस्तारित स्पष्टीकरण दिया गया है, जो प्रत्येक अवधारणा की गहन समझ प्रदान करने के लिए डिज़ाइन किया गया है। मैं समझाऊंगा कि प्रत्येक फीचर क्या है, यह महत्वपूर्ण क्यों है, इसे वास्तविक दुनिया की प्रोग्रामिंग में कैसे लागू किया जाता है, और इसके उपयोग को दर्शाने के लिए व्यावहारिक उदाहरण शामिल करूंगा। यह प्रतिक्रिया मूल विचारों पर आधारित है और अतिरिक्त संदर्भ, बारीकियों और अनुप्रयोगों के साथ और अधिक गहराई जोड़ती है।

## 1. **क्लासेस (Classes)**
- **यह क्या है**: एक क्लास Java का मौलिक बिल्डिंग ब्लॉक है, जो ऑब्जेक्ट्स बनाने के लिए एक टेम्पलेट या ब्लूप्रिंट के रूप में कार्य करती है। यह डेटा (फ़ील्ड्स) और व्यवहार (मेथड्स) को समाहित करती है जो ऑब्जेक्ट्स के गुणों और कार्यों को परिभाषित करते हैं।
- **यह महत्वपूर्ण क्यों है**: क्लासेस Java के ऑब्जेक्ट-ओरिएंटेड प्रोग्रामिंग (OOP) प्रतिमान की आधारशिला हैं। वे एनकैप्सुलेशन (डेटा और मेथड्स को बंडल करना), एब्स्ट्रक्शन (इम्प्लीमेंटेशन विवरण छुपाना), और मॉड्यूलैरिटी को बढ़ावा देती हैं, जिससे कोड पुन: प्रयोज्य और रखरखाव में आसान हो जाता है।
- **इसका उपयोग कैसे किया जाता है**: क्लासेस प्रोग्राम में इकाइयों को मॉडल करती हैं, जैसे कि `Person`, `Vehicle`, या `BankAccount`। इनमें कंस्ट्रक्टर्स, एक्सेस मॉडिफायर्स (`public`, `private`) वाली फ़ील्ड्स, और ऑब्जेक्ट की स्थिति में हेरफेर करने के लिए मेथड्स शामिल हो सकते हैं।
- **गहन जानकारी**:
  - क्लासेस नेस्टेड (आंतरिक क्लासेस) या अमूर्त (abstract) हो सकती हैं (सीधे इंस्टेंटिएट नहीं की जा सकतीं)।
  - वे इनहेरिटेंस का समर्थन करती हैं, जिससे एक क्लास किसी अन्य क्लास को एक्सटेंड कर सकती है और उसके गुणों और मेथड्स को इनहेरिट कर सकती है।
- **उदाहरण**:
  ```java
  public class Student {
      private String name;  // इंस्टेंस फ़ील्ड
      private int age;
      
      // कंस्ट्रक्टर
      public Student(String name, int age) {
          this.name = name;
          this.age = age;
      }
      
      // मेथड
      public void displayInfo() {
          System.out.println("Name: " + name + ", Age: " + age);
      }
  }
  ```
- **वास्तविक दुनिया में उपयोग**: एक `Student` क्लास एक स्कूल प्रबंधन प्रणाली का हिस्सा हो सकती है, जिसमें ग्रेड की गणना करने या उपस्थिति ट्रैक करने के लिए मेथड्स हों।

## 2. **ऑब्जेक्ट्स (Objects)**
- **यह क्या है**: एक ऑब्जेक्ट क्लास का एक इंस्टेंस होता है, जो `new` कीवर्ड का उपयोग करके बनाया जाता है। यह अपनी स्वयं की स्थिति के साथ क्लास ब्लूप्रिंट के एक विशिष्ट अवतार का प्रतिनिधित्व करता है।
- **यह महत्वपूर्ण क्यों है**: ऑब्जेक्ट्स क्लासेस को जीवंत करते हैं, जिससे अद्वितीय डेटा वाले कई इंस्टेंस संभव होते हैं। वे वास्तविक दुनिया की इकाइयों का प्रतिनिधित्व करके जटिल सिस्टम के मॉडलिंग को सक्षम करते हैं।
- **इसका उपयोग कैसे किया जाता है**: ऑब्जेक्ट्स को उनकी मेथड्स और फ़ील्ड्स के माध्यम से इंस्टेंटिएट और मैनिपुलेट किया जाता है। उदाहरण के लिए, `Student student1 = new Student("Alice", 20);` एक `Student` ऑब्जेक्ट बनाता है।
- **गहन जानकारी**:
  - ऑब्जेक्ट्स हीप मेमोरी में संग्रहीत होते हैं, और उनके संदर्भ (references) वेरिएबल्स में संग्रहीत होते हैं।
  - Java ऑब्जेक्ट्स के लिए पास-बाय-रेफरेंस का उपयोग करती है, जिसका अर्थ है कि ऑब्जेक्ट की स्थिति में परिवर्तन सभी संदर्भों में परिलक्षित होते हैं।
- **उदाहरण**:
  ```java
  Student student1 = new Student("Alice", 20);
  student1.displayInfo();  // आउटपुट: Name: Alice, Age: 20
  ```
- **वास्तविक दुनिया में उपयोग**: एक ई-कॉमर्स सिस्टम में, `Order` या `Product` जैसे ऑब्जेक्ट व्यक्तिगत खरीदारी या बिक्री के लिए आइटम्स का प्रतिनिधित्व करते हैं।

## 3. **मेथड्स (Methods)**
- **यह क्या है**: मेथड्स एक क्लास के भीतर कोड के ब्लॉक होते हैं जो ऑब्जेक्ट्स के व्यवहार को परिभाषित करते हैं। वे पैरामीटर्स ले सकते हैं, मान वापस कर सकते हैं, या क्रियाएं कर सकते हैं।
- **यह महत्वपूर्ण क्यों है**: मेथड्स लॉजिक को समाहित करते हैं, अतिरेक को कम करते हैं, और कोड की पठनीयता में सुधार करते हैं। वे ऑब्जेक्ट की स्थिति के साथ इंटरैक्ट करने का प्राथमिक तरीका हैं।
- **इसका उपयोग कैसे किया जाता है**: मेथड्स को ऑब्जेक्ट्स पर या स्टैटिक रूप से क्लासेस पर इनवोक किया जाता है। प्रत्येक Java एप्लिकेशन `public static void main(String[] args)` मेथड से शुरू होता है।
- **गहन जानकारी**:
  - मेथड्स को ओवरलोड किया जा सकता है (एक ही नाम, अलग-अलग पैरामीटर्स) या ओवरराइड किया जा सकता है (सबक्लास में पुनर्परिभाषित)।
  - वे `static` (क्लास-लेवल) या इंस्टेंस-आधारित (ऑब्जेक्ट-लेवल) हो सकते हैं।
- **उदाहरण**:
  ```java
  public class MathUtils {
      public int add(int a, int b) {
          return a + b;
      }
      
      public double add(double a, double b) {  // मेथड ओवरलोडिंग
          return a + b;
      }
  }
  // उपयोग
  MathUtils utils = new MathUtils();
  System.out.println(utils.add(5, 3));      // आउटपुट: 8
  System.out.println(utils.add(5.5, 3.2));  // आउटपुट: 8.7
  ```
- **वास्तविक दुनिया में उपयोग**: एक `BankAccount` क्लास में `withdraw` मेथड खाते की बैलेंस को अपडेट कर सकती है और लेन-देन लॉग कर सकती है।

## 4. **वेरिएबल्स (Variables)**
- **यह क्या है**: वेरिएबल्स डेटा मान संग्रहीत करते हैं और इन्हें एक विशिष्ट प्रकार (जैसे, `int`, `String`, `double`) के साथ घोषित किया जाना चाहिए।
- **यह महत्वपूर्ण क्यों है**: वेरिएबल्स किसी प्रोग्राम के डेटा के लिए मेमोरी प्लेसहोल्डर हैं, जो स्टेट मैनेजमेंट और कम्प्यूटेशन को सक्षम करते हैं।
- **इसका उपयोग कैसे किया जाता है**: Java में कई वेरिएबल प्रकार होते हैं:
  - **लोकल वेरिएबल्स**: मेथड्स के अंदर घोषित, जिनका स्कोप उस मेथड तक सीमित होता है।
  - **इंस्टेंस वेरिएबल्स**: एक क्लास में घोषित, प्रत्येक ऑब्जेक्ट से बंधे होते हैं।
  - **स्टैटिक वेरिएबल्स**: `static` के साथ घोषित, एक क्लास के सभी इंस्टेंसेस में साझा होते हैं।
- **गहन जानकारी**:
  - वेरिएबल्स के डिफॉल्ट मान होते हैं (जैसे, `int` के लिए `0`, ऑब्जेक्ट्स के लिए `null`) यदि वे इनिशियलाइज़ नहीं हैं (केवल इंस्टेंस/स्टैटिक वेरिएबल्स के लिए)।
  - Java स्ट्रॉंग टाइपिंग लागू करती है, जो स्पष्ट कास्टिंग के बिना असंगत असाइनमेंट्स को रोकती है।
- **उदाहरण**:
  ```java
  public class Counter {
      static int totalCount = 0;  // स्टैटिक वेरिएबल
      int instanceCount;          // इंस्टेंस वेरिएबल
      
      public void increment() {
          int localCount = 1;     // लोकल वेरिएबल
          instanceCount += localCount;
          totalCount += localCount;
      }
  }
  ```
- **वास्तविक दुनिया में उपयोग**: लॉग-इन उपयोगकर्ताओं की संख्या (स्टैटिक) बनाम व्यक्तिगत सत्र समय (इंस्टेंस) को ट्रैक करना।

## 5. **कंट्रोल फ्लो स्टेटमेंट्स (Control Flow Statements)**
- **यह क्या है**: कंट्रोल फ्लो स्टेटमेंट्स किसी प्रोग्राम के एक्सेक्यूशन पथ को निर्धारित करते हैं, जिसमें कंडीशनल्स (`if`, `else`, `switch`) और लूप्स (`for`, `while`, `do-while`) शामिल हैं।
- **यह महत्वपूर्ण क्यों है**: वे निर्णय लेने और पुनरावृत्ति को सक्षम करते हैं, जो जटिल लॉजिक को लागू करने के लिए आवश्यक हैं।
- **इसका उपयोग कैसे किया जाता है**:
  - **कंडीशनल्स**: बूलियन शर्तों के आधार पर कोड निष्पादित करते हैं।
  - **लूप्स**: डेटा पर पुनरावृति करते हैं या किसी शर्त के पूरी होने तक क्रियाओं को दोहराते हैं।
- **गहन जानकारी**:
  - `switch` स्टेटमेंट `String` (Java 7 से) और एनम्स का समर्थन करता है, प्रिमिटिव प्रकारों के अतिरिक्त।
  - लूप्स नेस्टेड हो सकते हैं, और `break`/`continue` कीवर्ड्स उनके व्यवहार को संशोधित करते हैं।
- **उदाहरण**:
  ```java
  int score = 85;
  if (score >= 90) {
      System.out.println("A");
  } else if (score >= 80) {
      System.out.println("B");
  } else {
      System.out.println("C");
  }
  
  for (int i = 0; i < 3; i++) {
      System.out.println("Loop iteration: " + i);
  }
  ```
- **वास्तविक दुनिया में उपयोग**: ऑर्डर्स की एक सूची को प्रोसेस करना (`for` लूप) और कुल राशि के आधार पर छूट लागू करना (`if`)।

## 6. **इंटरफेसेस (Interfaces)**
- **यह क्या है**: एक इंटरफेस एक अनुबंध है जो उन मेथड्स को निर्दिष्ट करता है जिन्हें इम्प्लीमेंटिंग क्लासेस को परिभाषित करना होगा। यह एब्स्ट्रक्शन और मल्टीपल इनहेरिटेंस का समर्थन करता है।
- **यह महत्वपूर्ण क्यों है**: इंटरफेसेस ढीले युग्मन (loose coupling) और पॉलीमॉर्फिज्म को सक्षम करते हैं, जिससे अलग-अलग क्लासेस एक सामान्य API साझा कर सकती हैं।
- **इसका उपयोग कैसे किया जाता है**: क्लासेस `implements` कीवर्ड का उपयोग करके इंटरफेसेस को इम्प्लीमेंट करती हैं। Java 8 के बाद से, इंटरफेसेस में डिफॉल्ट और स्टैटिक मेथड्स इम्प्लीमेंटेशन के साथ शामिल हो सकते हैं।
- **गहन जानकारी**:
  - डिफॉल्ट मेथड्स इंटरफेसेस के बैकवर्ड-कम्पेटिबल विकास की अनुमति देते हैं।
  - फंक्शनल इंटरफेसेस (एक अमूर्त मेथड वाले) लैम्ब्डा एक्सप्रेशन्स के लिए महत्वपूर्ण हैं।
- **उदाहरण**:
  ```java
  public interface Vehicle {
      void start();
      default void stop() {  // डिफॉल्ट मेथड
          System.out.println("Vehicle stopped");
      }
  }
  
  public class Bike implements Vehicle {
      public void start() {
          System.out.println("Bike started");
      }
  }
  // उपयोग
  Bike bike = new Bike();
  bike.start();  // आउटपुट: Bike started
  bike.stop();   // आउटपुट: Vehicle stopped
  ```
- **वास्तविक दुनिया में उपयोग**: एक पेमेंट गेटवे सिस्टम में `CreditCard` और `PayPal` क्लासेस के लिए एक `Payment` इंटरफेस।

## 7. **एक्सेप्शन हैंडलिंग (Exception Handling)**
- **यह क्या है**: एक्सेप्शन हैंडलिंग `try`, `catch`, `finally`, `throw`, और `throws` का उपयोग करके रनटाइम एरर्स को प्रबंधित करती है।
- **यह महत्वपूर्ण क्यों है**: यह क्रैश को रोककर और फ़ाइल न मिलने या शून्य से विभाजन जैसी त्रुटियों से उबरने की अनुमति देकर मजबूती सुनिश्चित करती है।
- **इसका उपयोग कैसे किया जाता है**: जोखिम भरा कोड एक `try` ब्लॉक में जाता है, विशिष्ट एक्सेप्शन्स को `catch` ब्लॉक्स में पकड़ा जाता है, और `finally` क्लीनअप कोड निष्पादित करता है।
- **गहन जानकारी**:
  - एक्सेप्शन्स `Throwable` (`Error` या `Exception`) से प्राप्त ऑब्जेक्ट्स होते हैं।
  - कस्टम एक्सेप्शन्स `Exception` को एक्सटेंड करके बनाए जा सकते हैं।
- **उदाहरण**:
  ```java
  try {
      int[] arr = new int[2];
      arr[5] = 10;  // ArrayIndexOutOfBoundsException
  } catch (ArrayIndexOutOfBoundsException e) {
      System.out.println("Index out of bounds: " + e.getMessage());
  } finally {
      System.out.println("Cleanup done");
  }
  ```
- **वास्तविक दुनिया में उपयोग**: एक वेब एप्लिकेशन में नेटवर्क टाइमआउट्स को हैंडल करना।

## 8. **जेनरिक्स (Generics)**
- **यह क्या है**: जेनरिक्स क्लासेस, इंटरफेसेस और मेथड्स को प्रकारों के साथ पैरामीटराइज़ करके टाइप-सुरक्षित, पुन: प्रयोज्य कोड की अनुमति देते हैं।
- **यह महत्वपूर्ण क्यों है**: वे कम्पाइल टाइम पर टाइप एरर्स को पकड़ते हैं, रनटाइम बग्स को कम करते हैं और कास्टिंग की आवश्यकता को समाप्त करते हैं।
- **इसका उपयोग कैसे किया जाता है**: कलेक्शन्स (जैसे, `List<String>`) और कस्टम जेनरिक क्लासेस/मेथड्स में आम।
- **गहन जानकारी**:
  - वाइल्डकार्ड्स (`? extends T`, `? super T`) टाइप वेरिएंस को हैंडल करते हैं।
  - टाइप इरेज़र बैकवर्ड कम्पेटिबिलिटी के लिए रनटाइम पर जेनरिक टाइप जानकारी को हटा देता है।
- **उदाहरण**:
  ```java
  public class Box<T> {
      private T content;
      public void set(T content) { this.content = content; }
      public T get() { return content; }
  }
  // उपयोग
  Box<Integer> intBox = new Box<>();
  intBox.set(42);
  System.out.println(intBox.get());  // आउटपुट: 42
  ```
- **वास्तविक दुनिया में उपयोग**: की-वैल्यू स्टोरेज के लिए एक जेनरिक `Cache<K, V>` क्लास।

## 9. **लैम्ब्डा एक्सप्रेशन्स (Lambda Expressions)**
- **यह क्या है**: लैम्ब्डा एक्सप्रेशन्स (Java 8+) अनाम फ़ंक्शन्स के संक्षिप्त प्रतिनिधित्व हैं, जो आमतौर पर फंक्शनल इंटरफेसेस के साथ उपयोग किए जाते हैं।
- **यह महत्वपूर्ण क्यों है**: वे इवेंट हैंडलिंग, कलेक्शन्स प्रोसेसिंग और फंक्शनल प्रोग्रामिंग के लिए कोड को सरल बनाते हैं।
- **इसका उपयोग कैसे किया जाता है**: `Runnable`, `Comparator`, या एकल अमूर्त मेथड वाले कस्टम इंटरफेसेस के साथ जोड़े जाते हैं।
- **गहन जानकारी**:
  - सिंटैक्स: `(parameters) -> expression` या `(parameters) -> { statements; }`।
  - वे फंक्शनल-स्टाइल डेटा प्रोसेसिंग के लिए स्ट्रीम्स API को सक्षम करते हैं।
- **उदाहरण**:
  ```java
  List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
  names.forEach(name -> System.out.println(name.toUpperCase()));
  ```
- **वास्तविक दुनिया में उपयोग**: `Collections.sort(products, (p1, p2) -> p1.getPrice() - p2.getPrice())` का उपयोग करके उत्पादों की एक सूची को कीमत के आधार पर क्रमबद्ध करना।

## 10. **एनोटेशन्स (Annotations)**
- **यह क्या है**: एनोटेशन्स मेटाडेटा टैग (जैसे, `@Override`, `@Deprecated`) होते हैं जो कोड एलिमेंट्स पर लागू होते हैं, जिन्हें कम्पाइल टाइम या रनटाइम पर प्रोसेस किया जाता है।
- **यह महत्वपूर्ण क्यों है**: वे कम्पाइलर्स, फ्रेमवर्क्स, या टूल्स को निर्देश प्रदान करते हैं, जिससे ऑटोमेशन बढ़ता है और बॉयलरप्लेट कोड कम होता है।
- **इसका उपयोग कैसे किया जाता है**: कॉन्फ़िगरेशन (जैसे, JPA में `@Entity`), डॉक्यूमेंटेशन, या नियमों को लागू करने के लिए उपयोग किया जाता है।
- **गहन जानकारी**:
  - कस्टम एनोटेशन्स `@interface` के साथ परिभाषित किए जा सकते हैं।
  - रिटेंशन पॉलिसीज़ (`SOURCE`, `CLASS`, `RUNTIME`) उनके जीवनकाल को निर्धारित करती हैं।
- **उदाहरण**:
  ```java
  public class MyClass {
      @Override
      public String toString() {
          return "Custom string";
      }
      
      @Deprecated
      public void oldMethod() {
          System.out.println("Old way");
      }
  }
  ```
- **वास्तविक दुनिया में उपयोग**: Spring में `@Autowired` का उपयोग करके डिपेंडेंसीज़ को स्वचालित रूप से इंजेक्ट करना।

## अतिरिक्त मुख्य विशेषताएं

आपकी समझ को गहरा करने के लिए, यहां अधिक विस्तृत स्पष्टीकरण के साथ और अधिक व्यापक रूप से उपयोग की जाने वाली Java विशेषताएं दी गई हैं:

### 11. **ऐरेज़ (Arrays)**
- **यह क्या है**: ऐरेज़ एक ही प्रकार के तत्वों के निश्चित आकार, क्रमबद्ध संग्रह होते हैं।
- **यह महत्वपूर्ण क्यों है**: वे कई मानों को संग्रहीत करने और एक्सेस करने का एक सरल, कुशल तरीका प्रदान करते हैं।
- **इसका उपयोग कैसे किया जाता है**: `type[] name = new type[size];` के रूप में घोषित या सीधे इनिशियलाइज़ किए जाते हैं।
- **उदाहरण**:
  ```java
  int[] numbers = {1, 2, 3, 4};
  System.out.println(numbers[2]);  // आउटपुट: 3
  ```
- **वास्तविक दुनिया में उपयोग**: एक सप्ताह के लिए तापमान की एक सूची संग्रहीत करना।

### 12. **एनम्स (Enums)**
- **यह क्या है**: एनम्स नामित स्थिरांकों (constants) का एक निश्चित सेट परिभाषित करते हैं, जो अक्सर संबद्ध मानों या मेथड्स के साथ होते हैं।
- **यह महत्वपूर्ण क्यों है**: वे कच्चे स्थिरांकों (raw constants) पर टाइप सुरक्षा और पठनीयता में सुधार करते हैं।
- **इसका उपयोग कैसे किया जाता है**: पूर्वनिर्धारित श्रेणियों जैसे दिन, राज्य, या स्थितियों के लिए उपयोग किया जाता है।
- **उदाहरण**:
  ```java
  public enum Status {
      PENDING("In progress"), APPROVED("Done"), REJECTED("Failed");
      private String desc;
      Status(String desc) { this.desc = desc; }
      public String getDesc() { return desc; }
  }
  // उपयोग
  System.out.println(Status.APPROVED.getDesc());  // आउटपुट: Done
  ```
- **वास्तविक दुनिया में उपयोग**: एक ई-कॉमर्स सिस्टम में ऑर्डर स्थितियों का प्रतिनिधित्व करना।

### 13. **स्ट्रीम्स (Java 8+)**
- **यह क्या है**: स्ट्रीम्स कलेक्शन्स को प्रोसेस करने के लिए एक फंक्शनल दृष्टिकोण प्रदान करती हैं, जो `filter`, `map`, और `reduce` जैसे ऑपरेशन्स का समर्थन करती हैं।
- **यह महत्वपूर्ण क्यों है**: वे डेटा मैनिपुलेशन को सरल बनाती हैं, समानांतरता (parallelism) का समर्थन करती हैं, और कोड एक्सप्रेसिवनेस में सुधार करती हैं।
- **इसका उपयोग कैसे किया जाता है**: `.stream()` का उपयोग करके कलेक्शन्स से बनाई जाती हैं और ऑपरेशन्स के साथ चेन की जाती हैं।
- **उदाहरण**:
  ```java
  List<Integer> nums = Arrays.asList(1, 2, 3, 4, 5);
  int sum = nums.stream()
                .filter(n -> n % 2 == 0)
                .mapToInt(n -> n * 2)
                .sum();
  System.out.println(sum);  // आउटपुट: 12 (2*2 + 4*2)
  ```
- **वास्तविक दुनिया में उपयोग**: क्षेत्र द्वारा बिक्री डेटा एकत्रित करना।

### 14. **कंस्ट्रक्टर्स (Constructors)**
- **यह क्या है**: कंस्ट्रक्टर्स विशेष मेथड्स होते हैं जो एक ऑब्जेक्ट के बनाए जाने पर इनवोक होते हैं, जिनका उपयोग उसकी स्थिति को इनिशियलाइज़ करने के लिए किया जाता है।
- **यह महत्वपूर्ण क्यों है**: वे यह सुनिश्चित करते हैं कि ऑब्जेक्ट्स वैध डेटा के साथ शुरू हों और इनिशियलाइज़ेशन एरर्स को कम करें।
- **इसका उपयोग कैसे किया जाता है**: क्लास के समान नाम के साथ परिभाषित, वैकल्पिक रूप से पैरामीटर्स के साथ।
- **उदाहरण**:
  ```java
  public class Book {
      String title;
      public Book(String title) {
          this.title = title;
      }
  }
  ```
- **वास्तविक दुनिया में उपयोग**: यूज़रनेम और पासवर्ड के साथ एक `User` ऑब्जेक्ट को इनिशियलाइज़ करना।

### 15. **इनहेरिटेंस (Inheritance)**
- **यह क्या है**: इनहेरिटेंस एक क्लास (सबक्लास) को `extends` का उपयोग करके किसी अन्य क्लास (सुपरक्लास) से फ़ील्ड्स और मेथड्स इनहेरिट करने की अनुमति देता है।
- **यह महत्वपूर्ण क्यों है**: यह कोड पुन: उपयोग को बढ़ावा देती है और क्लासेस के बीच एक पदानुक्रमित संबंध स्थापित करती है।
- **इसका उपयोग कैसे किया जाता है**: एक सामान्य क्लास के विशेष संस्करण बनाने के लिए उपयोग किया जाता है।
- **उदाहरण**:
  ```java
  public class Animal {
      void eat() { System.out.println("Eating"); }
  }
  public class Dog extends Animal {
      void bark() { System.out.println("Barking"); }
  }
  // उपयोग
  Dog dog = new Dog();
  dog.eat();  // आउटपुट: Eating
  dog.bark(); // आउटपुट: Barking
  ```
- **वास्तविक दुनिया में उपयोग**: `BankAccount` से इनहेरिट करने वाली एक `SavingsAccount` क्लास।

- **अन्वेषण करें**: Java API डॉक्यूमेंटेशन (जैसे, `java.util`, `
