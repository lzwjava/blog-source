`java.configuration.runtimes` का उपयोग Visual Studio Code (VS Code) सेटिंग्स में करने के लिए, आपको इसे अपनी `settings.json` फ़ाइल में कॉन्फ़िगर करना होगा। यह सेटिंग आपको कई Java Development Kit (JDK) वर्जन निर्दिष्ट करने की अनुमति देती है जिन्हें VS Code आपकी Java प्रोजेक्ट्स के लिए उपयोग कर सकता है। यह तब विशेष रूप से उपयोगी होती है जब आप विभिन्न प्रोजेक्ट्स पर काम कर रहे होते हैं जिनके लिए विशिष्ट Java वर्जन की आवश्यकता होती है। इसे सेट अप करने के लिए यहां एक चरण-दर-चरण मार्गदर्शिका दी गई है:

### `java.configuration.runtimes` को कॉन्फ़िगर करने के चरण

1. **VS Code सेटिंग्स खोलें**:
   - Windows/Linux पर: `File > Preferences > Settings` पर जाएं या `Ctrl + ,` दबाएं।
   - macOS पर: `Code > Preferences > Settings` पर जाएं या `Cmd + ,` दबाएं।

2. **JSON सेटिंग्स फ़ाइल तक पहुंचें**:
   - सेटिंग्स UI में, `java.configuration.runtimes` खोजें।
   - आपको "Java: Configuration: Runtimes" जैसा एक विकल्प दिखाई देगा। `settings.json` फ़ाइल को खोलने के लिए "Edit in settings.json" पर क्लिक करें (आमतौर पर सेटिंग विवरण के नीचे एक लिंक होता है)।

3. **`settings.json` एडिट करें**:
   - `settings.json` फ़ाइल में, `java.configuration.runtimes` ऐरे को जोड़ें या संशोधित करें। इस ऐरे में ऑब्जेक्ट्स होते हैं, जिनमें से प्रत्येक एक JDK वर्जन का प्रतिनिधित्व करता है जिसे VS Code को पहचानना है।
   - प्रत्येक ऑब्जेक्ट में आमतौर पर शामिल होते हैं:
     - `name`: Java वर्जन पहचानकर्ता (उदाहरण के लिए, `JavaSE-1.8`, `JavaSE-11`, `JavaSE-17`)।
     - `path`: आपके सिस्टम पर JDK इंस्टॉलेशन डायरेक्टरी का पूर्ण पथ।
     - `default` (वैकल्पिक): अनमैनेज्ड फ़ोल्डर्स (बिना बिल्ड टूल्स वाली प्रोजेक्ट्स, जैसे Maven या Gradle) के लिए इसे डिफ़ॉल्ट JDK बनाने के लिए `true` सेट करें।

   यहां एक उदाहरण कॉन्फ़िगरेशन दिया गया है:

   ```json
   {
       "java.configuration.runtimes": [
           {
               "name": "JavaSE-1.8",
               "path": "C:/Program Files/Java/jdk1.8.0_351",
               "default": true
           },
           {
               "name": "JavaSE-11",
               "path": "C:/Program Files/Java/jdk-11.0.15"
           },
           {
               "name": "JavaSE-17",
               "path": "C:/Program Files/Java/jdk-17.0.6"
           }
       ]
   }
   ```

4. **JDK पथ सत्यापित करें**:
   - सुनिश्चित करें कि `path` आपकी JDK इंस्टॉलेशन की रूट डायरेक्टरी की ओर इशारा करता है (उदाहरण के लिए, वह स्थान जहां `java.exe` या `java` वाला `bin` फ़ोल्डर स्थित है)।
   - Windows पर, पथ में फॉरवर्ड स्लैश (`/`) का उपयोग करें या बैकस्लैश को एस्केप करें (`\\`)।
   - macOS/Linux पर, उपयुक्त फ़ाइल सिस्टम पथ का उपयोग करें (उदाहरण के लिए, `/usr/lib/jvm/java-17-openjdk`)।

5. **सेव करें और रीलोड करें**:
   - `settings.json` फ़ाइल को सेव करें।
   - परिवर्तनों को लागू करने के लिए VS Code को पुनरारंभ करें या विंडो को रीलोड करें (`Ctrl + R` या `Cmd + R`)।

6. **कॉन्फ़िगरेशन जांचें**:
   - कमांड पैलेट खोलें (`Ctrl + Shift + P` या `Cmd + Shift + P`) और `Java: Configure Java Runtime` कमांड चलाएं।
   - यह एक व्यू खोलता है जो आपकी प्रोजेक्ट्स के लिए उपलब्ध JDKs दिखाता है। सत्यापित करें कि आपके कॉन्फ़िगर किए गए रनटाइम "Project JDKs" टैब के अंतर्गत दिखाई देते हैं।

### यह कैसे काम करता है
- **अनमैनेज्ड फ़ोल्डर्स**: बिना बिल्ड टूल वाली प्रोजेक्ट्स (उदाहरण के लिए, सादी Java फ़ाइलें) के लिए, VS Code `java.configuration.runtimes` में निर्दिष्ट `default` JDK का उपयोग करता है।
- **मैनेज्ड प्रोजेक्ट्स (Maven/Gradle)**: बिल्ड टूल वाली प्रोजेक्ट्स के लिए, JDK वर्जन बिल्ड कॉन्फ़िगरेशन (उदाहरण के लिए, `pom.xml` या `build.gradle`) द्वारा निर्धारित होता है, लेकिन VS Code अभी भी यहां सूचीबद्ध रनटाइम्स को संगतता के लिए पहचानेगा।
- **लैंग्वेज सर्वर**: Java लैंग्वेज सर्वर (Java एक्सटेंशन का हिस्सा) को चलाने के लिए JDK 17+ की आवश्यकता होती है। आप इसे अलग से `java.jdt.ls.java.home` के साथ निर्दिष्ट कर सकते हैं यदि आवश्यक हो (उदाहरण के लिए, `"java.jdt.ls.java.home": "C:/Program Files/Java/jdk-17.0.6"`)।

### एकाधिक JDKs के साथ उदाहरण
यदि आप Mac पर हैं और Homebrew के माध्यम से JDK 8 और 17 इंस्टॉल किए हैं, तो आपका कॉन्फ़िगरेशन कुछ इस तरह दिख सकता है:

```json
{
    "java.configuration.runtimes": [
        {
            "name": "JavaSE-1.8",
            "path": "/Library/Java/JavaVirtualMachines/jdk1.8.0_351.jdk/Contents/Home",
            "default": true
        },
        {
            "name": "JavaSE-17",
            "path": "/Library/Java/JavaVirtualMachines/jdk-17.0.6.jdk/Contents/Home"
        }
    ]
}
```

### सुझाव
- सही `name` फॉर्मेट (`JavaSE-<version>`) का उपयोग करें, Java वर्जन से मेल खाता हुआ (उदाहरण के लिए, Java 11 के लिए `JavaSE-11`)।
- यदि आपके पास केवल एक JDK है, तो भी आप इसे सूचीबद्ध कर सकते हैं ताकि VS Code इसका स्पष्ट रूप से उपयोग करे।
- अपना JDK पथ खोजने के लिए, टर्मिनल में `java -version` चलाएं और इंस्टॉलेशन डायरेक्टरी जांचें (उदाहरण के लिए, Windows पर `where java` या Unix-जैसी प्रणालियों पर `which java` के माध्यम से, फिर JDK रूट पर वापस ट्रेस करें)।

यह सेटअप आपको VS Code में कई Java वर्जन के साथ निर्बाध रूप से काम करने की सुविधा देता है! यदि आपको अपने JDK पथ खोजने या समस्या निवारण में सहायता की आवश्यकता हो तो मुझे बताएं।
