नीचे, मैं आपके द्वारा प्रदान किए गए Checkstyle कॉन्फ़िगरेशन फ़ाइल के प्रत्येक नियम की व्याख्या करूंगा, साथ ही कोड का एक छोटा सा उदाहरण दूंगा जो नियम का **उल्लंघन** करेगा और यह बताऊंगा कि इसे क्यों फ्लैग किया गया है। नियमों को आपके विवरण में दिए गए श्रेणियों के अनुसार समूहीकृत किया गया है, और मैं स्पष्टता सुनिश्चित करते हुए व्याख्याओं को संक्षिप्त रखूंगा। प्रत्येक उदाहरण मानता है कि नियम डिफ़ॉल्ट सेटिंग्स के साथ सक्रिय है, जब तक कि अन्यथा निर्दिष्ट न हो। कॉन्फ़िगर करने योग्य गुणों वाले नियमों (जैसे, अधिकतम लाइन लंबाई) के लिए, मैं सामान्य डिफ़ॉल्ट का उपयोग करूंगा (जैसे, LineLength के लिए 80 वर्ण) जब तक कि अन्यथा नोट न किया गया हो।

---

### रूट मॉड्यूल: `<module name="Checker">`
`Checker` मॉड्यूल सभी जांचों और फ़िल्टर के लिए शीर्ष-स्तरीय ऑर्केस्ट्रेटर है। यह `.java`, `.properties`, और `.xml` फ़ाइलों पर लागू होता है और गंभीरता को `error` पर सेट करता है।

#### गुण
- **severity="error"**: सभी उल्लंघनों को त्रुटियों के रूप में माना जाता है।
- **fileExtensions="java, properties, xml"**: जांचें इन फ़ाइल प्रकारों पर लागू होती हैं।

---

### फ़ाइल फ़िल्टर
ये फ़िल्टर करते हैं कि किन फ़ाइलों की जांच की जाएगी।

1.  **BeforeExecutionExclusionFileFilter**
    - **उद्देश्य**: एक regex से मेल खाने वाली फ़ाइलों को बाहर करता है (जैसे, `module-info.java`)।
    - **उदाहरण उल्लंघन**:
      ```java
      // module-info.java
      module com.example {
          requires java.base;
      }
      ```
    - **क्यों फ्लैग किया गया**: यह फ़ाइल regex `module\-info\.java$` से मेल खाती है और जांचों से बाहर कर दी जाती है। इस फ़ाइल के लिए कोई उल्लंघन नहीं होता, लेकिन अन्य फ़ाइलों की अभी भी जांच की जाती है।

2.  **SuppressionFilter**
    - **उद्देश्य**: किसी फ़ाइल (जैसे, `checkstyle-suppressions.xml`) में नियमों के आधार पर जांचों को दबाता है।
    - **उदाहरण उल्लंघन**: यदि `checkstyle-suppressions.xml` किसी विशिष्ट फ़ाइल के लिए `LineLength` को दबाता है, तो उस फ़ाइल में एक लंबी लाइन को फ्लैग नहीं किया जाएगा। दमन के बिना:
      ```java
      public class MyClass { // यह लाइन बहुत लंबी है और 80 वर्णों की डिफ़ॉल्ट अधिकतम लंबाई से अधिक है, जिससे एक त्रुटि होती है।
      }
      ```
    - **क्यों फ्लैग किया गया**: दमन नियम के बिना, लंबी लाइन `LineLength` का उल्लंघन करती है।

3.  **SuppressWarningsFilter**
    - **उद्देश्य**: `@SuppressWarnings("checkstyle:<check-name>")` का उपयोग करके जांचों को दबाने की अनुमति देता है।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {
          int my_field; // MemberName का उल्लंघन (camelCase नहीं)
      }
      ```
      ```java
      @SuppressWarnings("checkstyle:MemberName")
      public class MyClass {
          int my_field; // दमन के कारण कोई उल्लंघन नहीं
      }
      ```
    - **क्यों फ्लैग किया गया**: दमन के बिना, `my_field` `MemberName` का उल्लंघन करता है (camelCase की अपेक्षा, जैसे `myField`)।

### विविध जांचें
ये सामान्य फ़ाइल गुणों पर लागू होती हैं।

4.  **JavadocPackage**
    - **उद्देश्य**: सुनिश्चित करता है कि प्रत्येक पैकेज में Javadoc के साथ एक `package-info.java` फ़ाइल हो।
    - **उदाहरण उल्लंघन**:
      ```java
      // com/example/package-info.java (गायब या कोई Javadoc नहीं)
      package com.example;
      ```
    - **क्यों फ्लैग किया गया**: Javadoc टिप्पणी गायब है (जैसे, `/** पैकेज विवरण */`)।

5.  **NewlineAtEndOfFile**
    - **उद्देश्य**: सुनिश्चित करता है कि फ़ाइलें एक न्यूलाइन के साथ समाप्त हों।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {} // अंत में न्यूलाइन नहीं
      ```
    - **क्यों फ्लैग किया गया**: फ़ाइल न्यूलाइन वर्ण के बिना समाप्त होती है।

6.  **Translation**
    - **उद्देश्य**: सत्यापित करता है कि अंतर्राष्ट्रीयकरण के लिए `.properties` फ़ाइलों में सुसंगत कुंजियाँ हों।
    - **उदाहरण उल्लंघन**:
      ```properties
      # messages.properties
      key1=Hello
      key2=World
      ```
      ```properties
      # messages_fr.properties
      key1=Bonjour
      # key2 गायब है
      ```
    - **क्यों फ्लैग किया गया**: `messages_fr.properties` में `key2` की कमी है, जो `messages.properties` में मौजूद है।

### आकार जांचें
ये फ़ाइल और लाइन की लंबाई पर सीमा लागू करती हैं।

7.  **FileLength**
    - **उद्देश्य**: एक फ़ाइल में कुल पंक्तियों को सीमित करता है (डिफ़ॉल्ट आमतौर पर 2000 पंक्तियाँ)।
    - **उदाहरण उल्लंघन**: 2001-पंक्ति वाली Java फ़ाइल।
    - **क्यों फ्लैग किया गया**: डिफ़ॉल्ट पंक्ति सीमा से अधिक।

8.  **LineLength**
    - **उद्देश्य**: सुनिश्चित करता है कि पंक्तियाँ अधिकतम लंबाई (डिफ़ॉल्ट 80 वर्ण) से अधिक न हों।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass { public void myMethodWithVeryLongNameToExceedEightyCharactersInALine() {} }
      ```
    - **क्यों फ्लैग किया गया**: पंक्ति 80 वर्णों से अधिक है।

### व्हाइटस्पेस जांचें
ये सुसंगत व्हाइटस्पेस उपयोग लागू करती हैं।

9.  **FileTabCharacter**
    - **उद्देश्य**: स्रोत फ़ाइलों में टैब वर्णों (`\t`) पर प्रतिबंध लगाता है।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {
      →    int x; // इंडेंटेशन के लिए टैब वर्ण का उपयोग किया गया
      }
      ```
    - **क्यों फ्लैग किया गया**: स्थानों के बजाय टैब का उपयोग किया गया है।

10. **RegexpSingleline**
    - **उद्देश्य**: ट्रेलिंग व्हाइटस्पेस (पंक्तियाँ जो `\s+$` के साथ समाप्त होती हैं) का पता लगाता है।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {   // ट्रेलिंग स्पेस
      }
      ```
    - **क्यों फ्लैग किया गया**: पंक्ति व्हाइटस्पेस के साथ समाप्त होती है।

### हैडर चेक (टिप्पणी रहित)
11. **Header**
    - **उद्देश्य**: एक विशिष्ट फ़ाइल हैडर (जैसे, कॉपीराइट नोटिस) को `checkstyle.header.file` से लागू करता है।
    - **उदाहरण उल्लंघन** (यदि सक्षम है):
      ```java
      // हैडर गायब
      public class MyClass {}
      ```
    - **क्यों फ्लैग किया गया**: आवश्यक हैडर की कमी (जैसे, `// Copyright 2025 Example Inc.`)।

#### Javadoc जांचें
ये उचित Javadoc टिप्पणियों को लागू करती हैं।

12. **InvalidJavadocPosition**
    - **उद्देश्य**: सुनिश्चित करता है कि Javadoc टिप्पणियाँ कक्षाओं/विधियों से पहले हों, कहीं और नहीं।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {
          /** यह गलत स्थान पर है Javadoc */
          int x;
      }
      ```
    - **क्यों फ्लैग किया गया**: Javadoc कक्षा/विधि घोषणा से पहले नहीं है।

13. **JavadocMethod**
    - **उद्देश्य**: विधियों की उचित Javadoc (पैरामीटर, रिटर्न, अपवाद) के लिए जांचता है।
    - **उदाहरण उल्लंघन**:
      ```java
      public int add(int a, int b) { return a + b; }
      ```
    - **क्यों फ्लैग किया गया**: सार्वजनिक विधि के लिए Javadoc गायब है।

14. **JavadocType**
    - **उद्देश्य**: सुनिश्चित करता है कि कक्षाओं/इंटरफेस/एनमों में Javadoc हो।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {}
      ```
    - **क्यों फ्लैग किया गया**: कक्षा के लिए Javadoc गायब है।

15. **JavadocVariable**
    - **उद्देश्य**: सार्वजनिक/संरक्षित फ़ील्ड्स के लिए Javadoc की आवश्यकता होती है।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {
          public int x;
      }
      ```
    - **क्यों फ्लैग किया गया**: सार्वजनिक फ़ील्ड के लिए Javadoc गायब है।

16. **JavadocStyle**
    - **उद्देश्य**: Javadoc शैली लागू करता है (जैसे, वैध HTML, कोई दोषपूर्ण टिप्पणियाँ नहीं)।
    - **उदाहरण उल्लंघन**:
      ```java
      /** अंत में अवधि गायब */
      public class MyClass {}
      ```
    - **क्यों फ्लैग किया गया**: Javadoc के अंत में अवधि की कमी है।

17. **MissingJavadocMethod**
    - **उद्देश्य**: Javadoc गायब होने वाली विधियों को फ्लैग करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      public void myMethod() {}
      ```
    - **क्यों फ्लैग किया गया**: सार्वजनिक विधि में Javadoc की कमी है।

#### नामकरण सम्मेलन
ये नामकरण पैटर्न लागू करते हैं।

18. **ConstantName**
    - **उद्देश्य**: स्थिरांक (`static final`) `UPPER_CASE` में होने चाहिए।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {
          static final int myConstant = 42;
      }
      ```
    - **क्यों फ्लैग किया गया**: `myConstant` को `MY_CONSTANT` होना चाहिए।

19. **LocalFinalVariableName**
    - **उद्देश्य**: स्थानीय `final` चरों को `camelCase` में होना चाहिए।
    - **उदाहरण उल्लंघन**:
      ```java
      public void myMethod() {
          final int MY_VAR = 1;
      }
      ```
    - **क्यों फ्लैग किया गया**: `MY_VAR` को `myVar` होना चाहिए।

20. **LocalVariableName**
    - **उद्देश्य**: स्थानीय चरों को `camelCase` में होना चाहिए।
    - **उदाहरण उल्लंघन**:
      ```java
      public void myMethod() {
          int MY_VAR = 1;
      }
      ```
    - **क्यों फ्लैग किया गया**: `MY_VAR` को `myVar` होना चाहिए।

21. **MemberName**
    - **उद्देश्य**: इंस्टेंस फ़ील्ड्स को `camelCase` में होना चाहिए।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {
          int my_field;
      }
      ```
    - **क्यों फ्लैग किया गया**: `my_field` को `myField` होना चाहिए।

22. **MethodName**
    - **उद्देश्य**: विधियों को `camelCase` में होना चाहिए।
    - **उदाहरण उल्लंघन**:
      ```java
      public void MyMethod() {}
      ```
    - **क्यों फ्लैग किया गया**: `MyMethod` को `myMethod` होना चाहिए।

23. **PackageName**
    - **उद्देश्य**: पैकेजों को डॉट्स के साथ लोअरकेस में होना चाहिए (जैसे, `com.example`)।
    - **उदाहरण उल्लंघन**:
      ```java
      package com.Example;
      ```
    - **क्यों फ्लैग किया गया**: `Example` को `example` होना चाहिए।

24. **ParameterName**
    - **उद्देश्य**: विधि पैरामीटरों को `camelCase` में होना चाहिए।
    - **उदाहरण उल्लंघन**:
      ```java
      public void myMethod(int MY_PARAM) {}
      ```
    - **क्यों फ्लैग किया गया**: `MY_PARAM` को `myParam` होना चाहिए।

25. **StaticVariableName**
    - **उद्देश्य**: स्थिर (गैर-अंतिम) फ़ील्ड्स को एक नामकरण पैटर्न का पालन करना चाहिए।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {
          static int MY_FIELD;
      }
      ```
    - **क्यों फ्लैग किया गया**: `MY_FIELD` को `myField` होना चाहिए (camelCase मानते हुए)।

26. **TypeName**
    - **उद्देश्य**: कक्षा/इंटरफेस/एनम नामों को `UpperCamelCase` में होना चाहिए।
    - **उदाहरण उल्लंघन**:
      ```java
      public class myClass {}
      ```
    - **क्यों फ्लैग किया गया**: `myClass` को `MyClass` होना चाहिए।

#### इम्पोर्ट जांचें
ये इम्पोर्ट स्टेटमेंट को नियंत्रित करती हैं।

27. **AvoidStarImport**
    - **उद्देश्य**: वाइल्डकार्ड इम्पोर्ट (जैसे, `import java.util.*`) पर प्रतिबंध लगाता है।
    - **उदाहरण उल्लंघन**:
      ```java
      import java.util.*;
      ```
    - **क्यों फ्लैग किया गया**: विशिष्ट इम्पोर्ट (जैसे, `import java.util.List`) के बजाय `*` का उपयोग करता है।

28. **IllegalImport**
    - **उद्देश्य**: प्रतिबंधित पैकेजों (जैसे, `sun.*`) से इम्पोर्ट को ब्लॉक करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      import sun.misc.Unsafe;
      ```
    - **क्यों फ्लैग किया गया**: `sun.misc.Unsafe` एक प्रतिबंधित पैकेज में है।

29. **RedundantImport**
    - **उद्देश्य**: डुप्लिकेट या अनावश्यक इम्पोर्ट को फ्लैग करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      import java.util.List;
      import java.util.List;
      ```
    - **क्यों फ्लैग किया गया**: `List` का डुप्लिकेट इम्पोर्ट।

30. **UnusedImports**
    - **उद्देश्य**: अप्रयुक्त इम्पोर्ट का पता लगाता है।
    - **उदाहरण उल्लंघन**:
      ```java
      import java.util.List;
      public class MyClass {}
      ```
    - **क्यों फ्लैग किया गया**: `List` इम्पोर्ट किया गया है लेकिन उपयोग नहीं किया गया है।

#### आकार जांचें
ये विधि और पैरामीटर गणना को सीमित करती हैं।

31. **MethodLength**
    - **उद्देश्य**: विधि की लंबाई को सीमित करता है (डिफ़ॉल्ट आमतौर पर 150 पंक्तियाँ)।
    - **उदाहरण उल्लंघन**: 151 पंक्तियों वाली एक विधि।
    - **क्यों फ्लैग किया गया**: डिफ़ॉल्ट पंक्ति सीमा से अधिक।

32. **ParameterNumber**
    - **उद्देश्य**: विधि पैरामीटरों को सीमित करता है (डिफ़ॉल्ट आमतौर पर 7)।
    - **उदाहरण उल्लंघन**:
      ```java
      public void myMethod(int a, int b, int c, int d, int e, int f, int g, int h) {}
      ```
    - **क्यों फ्लैग किया गया**: 8 पैरामीटर डिफ़ॉल्ट सीमा 7 से अधिक हैं।

#### व्हाइटस्पेस जांचें
ये कोड में सुसंगत व्हाइटस्पेस लागू करती हैं।

33. **EmptyForIteratorPad**
    - **उद्देश्य**: खाली `for` लूप इटरेटर में पैडिंग की जांच करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      for (int i = 0; ; i++) {}
      ```
    - **क्यों फ्लैग किया गया**: खाली इटरेटर सेक्शन में स्थान होना चाहिए (जैसे, `for (int i = 0; ; i++)`)।

34. **GenericWhitespace**
    - **उद्देश्य**: जेनेरिक प्रकारों (जैसे, `List<String>`) के आसपास स्पेसिंग सुनिश्चित करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      List<String>list;
      ```
    - **क्यों फ्लैग किया गया**: `>` और `list` के बीच कोई स्थान नहीं है।

35. **MethodParamPad**
    - **उद्देश्य**: विधि पैरामीटर सूचियों से पहले स्पेसिंग की जांच करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      public void myMethod (int x) {}
      ```
    - **क्यों फ्लैग किया गया**: `(int x)` से पहले स्थान गलत है।

36. **NoWhitespaceAfter**
    - **उद्देश्य**: कुछ टोकन (जैसे, `++`) के बाद व्हाइटस्पेस पर प्रतिबंध लगाता है।
    - **उदाहरण उल्लंघन**:
      ```java
      int x = y ++ ;
      ```
    - **क्यों फ्लैग किया गया**: `++` के बाद स्थान।

37. **NoWhitespaceBefore**
    - **उद्देश्य**: कुछ टोकन (जैसे, `;`) से पहले व्हाइटस्पेस पर प्रतिबंध लगाता है।
    - **उदाहरण उल्लंघन**:
      ```java
      int x = 1 ;
      ```
    - **क्यों फ्लैग किया गया**: `;` से पहले स्थान।

38. **OperatorWrap**
    - **उद्देश्य**: सुनिश्चित करता है कि ऑपरेटर सही पंक्ति पर हों।
    - **उदाहरण उल्लंघन**:
      ```java
      int x = 1 +
          2;
      ```
    - **क्यों फ्लैग किया गया**: `+` पहली पंक्ति के अंत में होना चाहिए।

39. **ParenPad**
    - **उद्देश्य**: कोष्ठकों के अंदर स्पेसिंग की जांच करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      if ( x == y ) {}
      ```
    - **क्यों फ्लैग किया गया**: `(` और `)` के अंदर स्थान गलत हैं।

40. **TypecastParenPad**
    - **उद्देश्य**: टाइपकास्ट में स्पेसिंग सुनिश्चित करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      Object o = ( String ) obj;
      ```
    - **क्यों फ्लैग किया गया**: `( String )` के अंदर स्थान गलत हैं।

41. **WhitespaceAfter**
    - **उद्देश्य**: कुछ टोकन (जैसे, कॉमा) के बाद व्हाइटस्पेस की आवश्यकता होती है।
    - **उदाहरण उल्लंघन**:
      ```java
      int[] arr = {1,2,3};
      ```
    - **क्यों फ्लैग किया गया**: कॉमा के बाद स्थान गायब है।

42. **WhitespaceAround**
    - **उद्देश्य**: ऑपरेटर/कीवर्ड के आसपास व्हाइटस्पेस सुनिश्चित करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      if(x==y) {}
      ```
    - **क्यों फ्लैग किया गया**: `==` और `if` के आसपास स्थान गायब हैं।

#### मॉडिफायर जांचें
ये Java मॉडिफायर को नियंत्रित करती हैं।

43. **ModifierOrder**
    - **उद्देश्य**: सुनिश्चित करता है कि मॉडिफायर सही क्रम में हों (JLS के अनुसार)।
    - **उदाहरण उल्लंघन**:
      ```java
      static public final int x = 1;
      ```
    - **क्यों फ्लैग किया गया**: गलत क्रम; `public static final` होना चाहिए।

44. **RedundantModifier**
    - **उद्देश्य**: अनावश्यक मॉडिफायर को फ्लैग करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      public final class MyClass {
          public final void myMethod() {}
      }
      ```
    - **क्यों फ्लैग किया गया**: `final` कक्षा में विधि पर `final` अनावश्यक है।

#### ब्लॉक जांचें
ये कोड ब्लॉक के उचित उपयोग को लागू करती हैं।

45. **AvoidNestedBlocks**
    - **उद्देश्य**: अनावश्यक नेस्टेड ब्लॉक पर प्रतिबंध लगाता है।
    - **उदाहरण उल्लंघन**:
      ```java
      public void myMethod() {
          { int x = 1; }
      }
      ```
    - **क्यों फ्लैग किया गया**: अनावश्यक नेस्टेड ब्लॉक।

46. **EmptyBlock**
    - **उद्देश्य**: खाली ब्लॉक को फ्लैग करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      if (x == 1) {}
      ```
    - **क्यों फ्लैग किया गया**: खाली `if` ब्लॉक।

47. **LeftCurly**
    - **उद्देश्य**: सुनिश्चित करता है कि ओपनिंग ब्रेसिज़ सही ढंग से रखे गए हैं।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass
      {
      }
      ```
    - **क्यों फ्लैग किया गया**: `{` को `class` के समान पंक्ति पर होना चाहिए।

48. **NeedBraces**
    - **उद्देश्य**: सिंगल-स्टेटमेंट ब्लॉक के लिए ब्रेसिज़ की आवश्यकता होती है।
    - **उदाहरण उल्लंघन**:
      ```java
      if (x == 1) y = 2;
      ```
    - **क्यों फ्लैग किया गया**: ब्रेसिज़ गायब हैं; `{ y = 2; }` होना चाहिए।

49. **RightCurly**
    - **उद्देश्य**: सुनिश्चित करता है कि क्लोजिंग ब्रेसिज़ सही ढंग से रखे गए हैं।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {
      }
      ```
    - **क्यों फ्लैग किया गया**: `}` को एक नई पंक्ति पर होना चाहिए (शैली के आधार पर)।

#### कोडिंग समस्या जांचें
ये सामान्य कोडिंग मुद्दों की पहचान करती हैं।

50. **EmptyStatement**
    - **उद्देश्य**: खाली स्टेटमेंट को फ्लैग करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      int x = 1;; // अतिरिक्त सेमीकोलन
      ```
    - **क्यों फ्लैग किया गया**: अतिरिक्त `;` एक खाली स्टेटमेंट बनाता है।

51. **EqualsHashCode**
    - **उद्देश्य**: सुनिश्चित करता है कि `equals()` और `hashCode()` दोनों ओवरराइड किए गए हैं।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {
          @Override
          public boolean equals(Object o) { return true; }
      }
      ```
    - **क्यों फ्लैग किया गया**: `hashCode()` ओवरराइड गायब है।

52. **HiddenField**
    - **उद्देश्य**: स्थानीय चर/पैरामीटर द्वारा छायांकित फ़ील्ड्स का पता लगाता है।
    - **उदाहरण उल्लंघन**:
      ```java
      public class MyClass {
          int x;
          public void setX(int x) { this.x = x; }
      }
      ```
    - **क्यों फ्लैग किया गया**: पैरामीटर `x` फ़ील्ड `x` को छाया देता है।

53. **IllegalInstantiation**
    - **उद्देश्य**: कुछ वर्गों के इंस्टेंटिएशन पर प्रतिबंध लगाता है।
    - **उदाहरण उल्लंघन**:
      ```java
      String s = new String("test");
      ```
    - **क्यों फ्लैग किया गया**: `String` का अनावश्यक इंस्टेंटिएशन।

54. **InnerAssignment**
    - **उद्देश्य**: एक्सप्रेशन में असाइनमेंट की अनुमति नहीं देता है।
    - **उदाहरण उल्लंघन**:
      ```java
      if (x = 1) {}
      ```
    - **क्यों फ्लैग किया गया**: एक्सप्रेशन में असाइनमेंट `x = 1`।

55. **MagicNumber**
    - **उद्देश्य**: हार्डकोडेड संख्यात्मक लिटरल को फ्लैग करता है।
    - **उदाहरण उल्लंघन**:
      ```java
      int x = 42;
      ```
    - **क्यों फ्लैग किया गया**: `42` एक नामित स्थिरांक होना चाहिए (जैसे, `static final int MY_CONST = 42;`)।

56. **MissingSwitchDefault**
    - **उद्देश्य**: `switch` स्टेटमेंट में एक `default` केस की आवश्यकता होती है।
    - **उदाहरण उल्लंघन**:
      ```java
      switch (x) {
          case 1: break;
      }
      ```
    - **क्यों फ्लैग किया गया**: `default` केस गायब है।

57. **MultipleVariableDeclarations**
