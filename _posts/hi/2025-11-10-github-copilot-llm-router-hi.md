एंटरप्राइज़ सेटिंग में अपने LLM राउटर को विशेष रूप से GitHub Copilot के LLM API का उपयोग करने के लिए कॉन्फ़िगर करने के लिए, इसमें एक समर्पित प्रदाता जोड़ें और राउटर सेटिंग्स को समायोजित करें ताकि सभी ट्रैफ़िक उस प्रदाता के माध्यम से रूट हो।

- **GitHub Copilot प्रदाता जोड़ें**: "प्रदाताओं" सरणी में एक नई प्रविष्टि शामिल करें, जिसमें उचित API आधार URL, आपकी GitHub व्यक्तिगत पहुंच टोकन (PAT) API कुंजी के रूप में, और समर्थित मॉडल शामिल हों।
- **राउटर डिफ़ॉल्ट अपडेट करें**: सभी राउटर फ़ील्ड (जैसे, "डिफ़ॉल्ट", "थिंक") को केवल नए प्रदाता नाम की ओर इंगित करने के लिए बदलें, यह सुनिश्चित करते हुए कि कोई अन्य प्रदाता नहीं बुलाया जाता है।
- **एंटरप्राइज़ प्रतिबंधों को संभालें**: अपने एंटरप्राइज़ GitHub खाते की PAT का उपयोग करें जिसमें आवश्यक स्कोप हों, और यदि आपके वातावरण को अनुपालन के लिए प्रॉक्सी रूटिंग की आवश्यकता है तो मौजूदा "PROXY_URL" का लाभ उठाएं।
- **परीक्षण और सत्यापन**: अपडेट के बाद, यह सत्यापित करें कि सभी API कॉल केवल Copilot एंडपॉइंट की ओर निर्देशित हैं, ताकि केवल Copilot API इंटरैक्शन की अनुमति देने वाली नीतियों के अनुरूप हो सके।

### चरण-दर-चरण कॉन्फ़िगरेशन
1. **GitHub PAT जनरेट करें**: अपने GitHub एंटरप्राइज़ खाते में लॉग इन करें और एक व्यक्तिगत पहुंच टोकन बनाएं जिसमें `copilot` (चैट एक्सेस के लिए) या `models:read` (व्यापक मॉडल अनुमान के लिए) जैसे स्कोप हों। यह व्यापक अनुमतियों को उजागर किए बिना सुरक्षित प्रमाणीकरण सुनिश्चित करता है।
2. **प्रदाताओं की सरणी संशोधित करें**: अपने कॉन्फ़िग JSON में "प्रदाताओं" सूची में एक नई ऑब्जेक्ट जोड़ें। "नाम" को कुछ वर्णनात्मक सेट करें जैसे "github_copilot", "api_base_url" को "https://api.githubcopilot.com/chat/completions" (Copilot एजेंटों के लिए) या "https://models.github.ai/inference/chat/completions" (सामान्य GitHub मॉडल्स अनुमान के लिए), "api_key" को आपकी PAT पर सेट करें, और संगत मॉडलों की सूची बनाएं।
3. **राउटर सेक्शन समायोजित करें**: "राउटर" ऑब्जेक्ट में सभी मानों को अपने नए प्रदाता नाम (जैसे, "github_copilot") से बदलें ताकि विशेष उपयोग लागू हो सके। यह OpenRouter जैसे अन्य प्रदाताओं पर फ़ॉलबैक को रोकता है।
4. **एंटरप्राइज़ विचार**: प्रतिबंधित वातावरण में, पुष्टि करें कि आपकी नेटवर्क नीतियां GitHub के डोमेन को आउटबाउंड कॉल की अनुमति देती हैं। यदि आवश्यक हो, तो "PROXY_URL" को एक अनुमोदित एंटरप्राइज़ प्रॉक्सी के माध्यम से रूट करने के लिए अपडेट करें। कॉल का ऑडिट करने और अनुपालन सुनिश्चित करने के लिए लॉगिंग ("LOG": true) सक्षम करें।

एंटरप्राइज़ वातावरण में, GitHub Copilot जैसे LLM API को एकीकृत करने के लिए सुरक्षा नीतियों का पालन करने हेतु सावधानीपूर्वक कॉन्फ़िगरेशन की आवश्यकता होती है, जो अक्सर आउटबाउंड कॉल को विशिष्ट अनुमोदित सेवाओं तक सीमित करती है। प्रदान किया गया राउटर कॉन्फ़िग LLM अनुरोधों को प्रदाताओं के पार रूट करने के लिए एक कस्टम सेटअप प्रतीत होता है, जो OpenRouter या LiteLLM जैसे टूल्स के समान है, जहां "प्रदाता" API एंडपॉइंट्स और मॉडल्स को परिभाषित करते हैं, और "राउटर" फ़ॉलबैक या श्रेणी-विशिष्ट रूटिंग निर्धारित करता है। GitHub Copilot के LLM API के विशेष उपयोग के लिए इसे अनुकूलित करने के लिए—यह सुनिश्चित करते हुए कि कोई अन्य बाहरी सेवा आमंत्रित नहीं की जाती है—आपको Copilot को एक प्रदाता के रूप में शामिल करना होगा और इसे सभी राउटर पथों में लागू करना होगा। यह दृष्टिकोण उन परिदृश्यों का समर्थन करता है जहां एंटरप्राइज़ फ़ायरवॉल या अनुपालन नियम केवल GitHub-होस्टेड API की अनुमति देते हैं, Copilot के OpenAI-संगत इंटरफेस का लाभ उठाते हुए चैट पूर्णता के लिए।

GitHub Copilot मुख्य रूप से दो माध्यमों से LLM पहुंच प्रदान करता है: एजेंटों और एक्सटेंशन बनाने के लिए समर्पित Copilot LLM एंडपॉइंट, और सामान्य अनुमान के लिए व्यापक GitHub मॉडल्स API। `https://api.githubcopilot.com/chat/completions` पर Copilot-विशिष्ट एंडपॉइंट एंटरप्राइज़-ग्रेड एजेंट विकास के लिए तैयार किया गया है, जो OpenAI चैट पूर्णता प्रारूप में POST अनुरोधों का समर्थन करता है। प्रमाणीकरण GitHub व्यक्तिगत पहुंच टोकन (PAT) से प्राप्त Bearer टोकन का उपयोग करता है, जो आमतौर पर `Authorization` हेडर के माध्यम से पारित किया जाता है। उदाहरण के लिए, एक नमूना अनुरोध में `Authorization: Bearer <your-pat>` और `Content-Type: application/json` जैसे हेडर शामिल हो सकते हैं, और एक बॉडी जिसमें `messages` (उपयोगकर्ता/सिस्टम प्रॉम्प्ट्स की एक सरणी) और वैकल्पिक पैरामीटर जैसे रीयल-टाइम प्रतिक्रियाओं के लिए `stream: true` शामिल होते हैं। मॉडल डॉक्स में स्पष्ट रूप से सूचीबद्ध नहीं हैं लेकिन Copilot के अंतर्निहित प्रदाताओं, जैसे GPT-4 वेरिएंट और Claude मॉडल्स के साथ संरेखित होते हैं, जिसमें दुरुपयोग को रोकने के लिए तृतीय-पक्ष एजेंटों पर सख्त दर सीमा लागू होती है।

वैकल्पिक रूप से, `https://models.github.ai/inference/chat/completions` पर GitHub मॉडल्स API एक अधिक बहुमुखी अनुमान सेवा प्रदान करता है, जो केवल GitHub क्रेडेंशियल्स का उपयोग करके मॉडल्स की एक कैटलॉग तक पहुंच की अनुमति देता है। यह प्रोटोटाइपिंग और GitHub Actions जैसे वर्कफ़्लो में एकीकरण के लिए आदर्श है। प्रमाणीकरण के लिए `models:read` स्कोप वाले PAT की आवश्यकता होती है, जो आपकी GitHub सेटिंग्स (https://github.com/settings/tokens) के माध्यम से बनाया जाता है। एंटरप्राइज़ सेटअप में, इसे संगठन-स्तरीय टोकन तक बढ़ाया जा सकता है या वर्कफ़्लो YAML फ़ाइलों में `permissions: models: read` जोड़कर CI/CD पाइपलाइनों में उपयोग किया जा सकता है। उपलब्ध मॉडल्स में उद्योग मानक शामिल हैं जैसे `openai/gpt-4o`, `openai/gpt-4o-mini`, `anthropic/claude-3-5-sonnet-20240620`, Meta का Llama 3.1 सीरीज़, और Mistral वेरिएंट्स, जिन सभी को एक ही OpenAI-संगत API प्रारूप के माध्यम से आमंत्रित किया जा सकता है। यह संगतता इसे डाउनस्ट्रीम कोड में बड़े बदलाव किए बिना आपके राउटर कॉन्फ़िग में सीधे शामिल करना सीधा बनाती है।

एंटरप्राइज़-विशिष्ट कॉन्फ़िगरेशन के लिए, GitHub Copilot Enterprise मानक Copilot को संगठन-व्यापी नियंत्रणों के साथ बढ़ाता है, जैसे आपके कोडबेस पर आधारित फाइन-ट्यून किए गए मॉडल, लेकिन API एक्सेस समान पैटर्न का पालन करता है। नेटवर्क प्रबंधन महत्वपूर्ण है: आप सब्सक्रिप्शन-आधारित रूटिंग कॉन्फ़िगर कर सकते हैं ताकि यह सुनिश्चित हो सके कि Copilot ट्रैफ़िक अनुमोदित पथों का उपयोग करता है, जिसके लिए उपयोगकर्ताओं को इसका समर्थन करने वाले न्यूनतम संस्करणों के लिए अपने IDE एक्सटेंशन (जैसे, VS Code) को अपडेट करने की आवश्यकता होती है। यदि आपका वातावरण प्रॉक्सी अनिवार्य करता है, तो कॉन्फ़िग के "PROXY_URL" को अपने एंटरप्राइज़ प्रॉक्सी सर्वर की ओर इंगित करने के लिए अपडेट करें, और SSL निरीक्षण के लिए कस्टम प्रमाणपत्रों पर विचार करें। LiteLLM जैसे टूल आगे के नियंत्रण के लिए एक मध्यवर्ती प्रॉक्सी के रूप में कार्य कर सकते हैं—`pip install litellm[proxy]` के माध्यम से इंस्टॉल करें, YAML कॉन्फ़िग में मॉडल्स को परिभाषित करें, सर्वर को एक स्थानीय पोर्ट पर शुरू करें, और लॉगिंग, दर सीमित करने और फ़ॉलबैक हैंडलिंग के लिए Copilot अनुरोधों को इसके माध्यम से पुनर्निर्देशित करें। हालाँकि, आपके मामले में, चूंकि लक्ष्य विशिष्टता है, "केवल Copilot को कॉल करना ठीक है" नीतियों का अनुपालन करने के लिए राउटर में फ़ॉलबैक से बचें।

इसे अपने कॉन्फ़िग में लागू करने के लिए, एक नया प्रदाता ऑब्जेक्ट जोड़कर शुरू करें। अपने उपयोग के मामले के आधार पर एंडपॉइंट चुनें: एक्सटेंशन बनाने के लिए Copilot एजेंट एंडपॉइंट का उपयोग करें, या सामान्य LLM रूटिंग के लिए GitHub मॉडल्स का उपयोग करें। उन मॉडल्स की सूची बनाएं जो आपके मौजूदा मॉडल्स (जैसे, Claude और GPT वेरिएंट्स) के साथ ओवरलैप करते हैं ताकि संगतता बनी रहे। फिर, सभी "राउटर" फ़ील्ड्स को केवल इस नए प्रदाता को संदर्भित करने के लिए अधिलेखित करें, ",minimax/minimax-m2" जैसे अल्पविराम या फ़ॉलबैक को समाप्त करें। अनुपालन की निगरानी के लिए लॉगिंग सक्षम करें, और यह सत्यापित करने के लिए परीक्षण करें कि कोई अनधिकृत एंडपॉइंट्स नहीं मारा जाता है। यदि VS Code या अन्य IDE के साथ एकीकृत कर रहे हैं, तो सेटिंग्स जैसे `github.copilot.advanced.debug.overrideProxyUrl` को समायोजित करें ताकि यदि आवश्यक हो तो आपके कॉन्फ़िगर प्रॉक्सी के माध्यम से रूट हो सके।

आपके प्रदाता कॉन्फ़िग में किस एंडपॉइंट का उपयोग करना है, यह तय करने में मदद के लिए दो मुख्य GitHub LLM API विकल्पों की तुलना तालिका यहां दी गई है:

| पहलू                   | GitHub Copilot LLM API (एजेंटों के लिए)              | GitHub मॉडल्स API                                   |
|-------------------------|-----------------------------------------------------|-----------------------------------------------------|
| एंडपॉइंट               | https://api.githubcopilot.com/chat/completions      | https://models.github.ai/inference/chat/completions |
| प्राथमिक उपयोग         | Copilot एक्सटेंशन और एजेंट बनाना                   | सामान्य प्रोटोटाइपिंग, अनुमान, और वर्कफ़्लो       |
| प्रमाणीकरण             | Bearer PAT (Authorization हेडर के माध्यम से)        | `models:read` स्कोप वाली PAT                        |
| समर्थित मॉडल           | अंतर्निहित (जैसे, GPT-4, Claude वेरिएंट्स)         | स्पष्ट कैटलॉग: gpt-4o, claude-3-5-sonnet, Llama 3.1, आदि। |
| एंटरप्राइज़ सुविधाएँ   | तृतीय-पक्षों के लिए दर सीमा; Copilot Enterprise के साथ एकीकृत | GitHub Actions में उपयोग योग्य; अपनी-अपनी-कुंजी समर्थन |
| दर सीमा/कोटा           | गैर-GitHub एजेंटों के लिए सख्त                     | प्रोटोटाइपिंग के लिए मुफ्त टियर; एंटरप्राइज़ के लिए स्केलेबल |
| संगतता                 | OpenAI चैट प्रारूप                                 | OpenAI-संगत; आसान राउटर एकीकरण                     |

यह तालिका उजागर करती है कि व्यापक राउटर उपयोग के लिए GitHub मॉडल्स क्यों बेहतर हो सकते हैं, क्योंकि यह अधिक स्पष्ट मॉडल चयन और एंटरप्राइज़ वर्कफ़्लो एकीकरण प्रदान करता है।

संभावित चुनौतियों में टोकन समाप्ति शामिल है (PAT आमतौर पर 30-90 दिनों तक चलते हैं), इसलिए एंटरप्राइज़ स्क्रिप्ट्स में नवीनीकरण को स्वचालित करें। यदि आपका वातावरण GitHub Enterprise Cloud का उपयोग करता है, तो सीमलेस एक्सेस के लिए संगठन स्तर पर Copilot सक्षम करें। उन्नत सेटअप के लिए, GitHub के इकोसिस्टम के भीतर रहते हुए पसंदीदा प्रदाताओं के माध्यम से रूट करने के लिए GitHub मॉडल्स में अपनी स्वयं की API कुंजी लाने (BYOK) का अन्वेषण करें। सुविधाओं के तेजी से विकसित होने के कारण नवीनतम स्कोप्स और एंडपॉइंट्स के लिए हमेशा प्राथमिक GitHub प्रलेखन को प्राथमिकता दें। यदि नेटवर्क प्रतिबंधों के कारण सीधे API कॉल विफल होते हैं, तो GitHub के प्रॉक्सी कॉन्फ़िगरेशन विकल्पों का उपयोग करने पर विचार करें या हाइब्रिड एंटरप्राइज़ तैनाती के लिए Azure के साथ एकीकरण करें।

संक्षेप में, यह कॉन्फ़िगरेशन आपके राउटर को एक Copilot-विशिष्ट प्रणाली में बदल देता है, जो कोड-संबंधित या सामान्य LLM कार्यों के लिए कार्यक्षमता बनाए रखते हुए एंटरप्राइज़ नीतियों के साथ संरेखित करता है।
