### Git में पैटर्न के साथ फाइलों को स्टेज करना

Git का `git add` कमांड फाइल पथ और ग्लोब पैटर्न (जैसे `*.java`) स्वीकार करता है ताकि मेल खाने वाली फाइलों को स्टेज किया जा सके। आप विभिन्न सेट की फाइलों को स्टेज करने के लिए कई `git add` कमांड चला सकते हैं, और फिर उन्हें अलग-अलग कमिट कर सकते हैं। यह समूहबद्ध कमिट (उदाहरण के लिए, एक Java फाइलों के लिए, और दूसरा बाकी फाइलों के लिए) के लिए चयनात्मक स्टेजिंग की अनुमति देता है।

#### बुनियादी उदाहरण: पहले Java फाइलों को स्टेज और कमिट करें
1. **सभी .java फाइलों को स्टेज करें**:
   ```bash
   git add "*.java"
   ```
   - यह करंट डायरेक्टरी और उसकी सब-डायरेक्टरीज़ में `.java` अंत वाली सभी अनट्रैक्ड या मॉडिफाइड फाइलों को स्टेज करता है।
   - कई सिस्टम (जैसे, Bash) पर शेल एक्सपेंशन इश्यूज़ से बचने के लिए डबल कोट्स (`"`) का उपयोग करें।

2. **स्टेज की गई Java फाइलों को कमिट करें**:
   ```bash
   git commit -m "Add Java files"
   ```
   - यह केवल स्टेज की गई फाइलों (Java वाली) को कमिट करता है।

3. **बाकी सब कुछ स्टेज करें**:
   ```bash
   git add .
   ```
   - यह बची हुई सभी अनट्रैक्ड/मॉडिफाइड फाइलों (करंट डायरेक्टरी की सामग्री सहित) को स्टेज करता है।

4. **बाकी फाइलों को कमिट करें**:
   ```bash
   git commit -m "Add other files"
   ```

- **नोट्स**:
  - पहली कमिट के बाद, `git add .` पहले से कमिट की गई फाइलों को दोबारा स्टेज नहीं करेगा; यह केवल अनस्टेज्ड बदलावों को प्रभावित करता है।
  - हर `add` के बाद `git status` चेक करके सत्यापित करें कि क्या स्टेज हुआ है।
  - यदि आपके पास नेस्टेड डायरेक्टरीज़ हैं, तो `*.java` किसी भी डेप्थ की फाइलों को पकड़ लेता है।

#### एक्सक्लूज़न या अधिक पैटर्न को हैंडल करना
यदि आपका मतलब कुछ ऐसा था जैसे `*.java` को शामिल करना लेकिन दूसरों को एक ही स्टेप में बाहर करना, तो Git का `git add` सीधे नेगेशन (जैसे `"!*.java"`) को सपोर्ट नहीं करता है। (आपके उदाहरण सिंटैक्स `"!*.java"` Git कमांड्स में वैलिड नहीं है।) इसके बजाय, ऊपर बताए अनुसार कई `git add` कॉल्स का उपयोग करें, या:

- विशिष्ट पथों का उपयोग करें: मैन्युअल चयन के लिए `git add file1.txt file2.md`।
- एडवांस्ड एक्सक्लूज़न के लिए: फाइलों को छुपाने के लिए अस्थायी रूप से `.gitignore` का उपयोग करें, जो आप चाहते हैं उसे ऐड करें, फिर `.gitignore` को रीस्टोर करें। या `git add -A` (सब कुछ ऐड करता है) का उपयोग करें लेकिन इसे चयनात्मक स्टेजिंग के साथ जोड़ें।
- इंटरएक्टिव ऐड: हंक-बाय-हंक स्टेजिंग के लिए `git add -p`।
- बदलावों का प्रिव्यू देखने के लिए: `git add --dry-run "*.java"` बिना किए यह दिखाता है कि क्या स्टेज होगा।

#### अलग-अलग कमिट के लिए टिप्स
- कमिट करने से पहले हमेशा `git diff --cached` के साथ रिव्यू करें।
- बड़े रेपो के लिए, एक साथ सभी बदलावों को स्टेज करने के लिए `git add --all` या `git add -A` पर विचार करें, फिर जरूरत पड़ने पर `git reset HEAD <file>` के साथ चयनात्मक रूप से अनस्टेज करें।
- यदि पैटर्न काम नहीं करते हैं, तो सुनिश्चित करें कि कोट्स का उपयोग किया गया है और आपका शेल ग्लोब्स को सपोर्ट करता है (उदाहरण के लिए, Windows में, Git Bash का उपयोग करें)। केस-इनसेंसिटिविटी के लिए, Git आपकी सिस्टम की सेटिंग का सम्मान करता है (`core.ignorecase` देखें)।

यदि यह आपके इरादे से मेल नहीं खाता (उदाहरण के लिए, यदि आपका मतलब `.gitignore` जैसे पैटर्न को बाहर करना था), तो अधिक जानकारी प्रदान करें!
