अपने Jekyll ब्लॉग में Kramdown के बजाय Marked का उपयोग करने के लिए, आपको अपने Jekyll कॉन्फ़िगरेशन और सेटअप में कुछ बदलाव करने होंगे। Marked, Markdown को पार्स और कंपाइल करने के लिए एक JavaScript लाइब्रेरी है, और इसे बेहतर गणित रेंडरिंग के लिए MathJax के साथ इस्तेमाल किया जा सकता है। यहाँ स्विच करने के लिए एक चरण-दर-चरण मार्गदर्शिका दी गई है:

### चरण 1: Marked इंस्टॉल करें
सबसे पहले, आपको अपने Jekyll प्रोजेक्ट में Marked लाइब्रेरी को शामिल करना होगा। आप इसे अपनी साइट के `<head>` सेक्शन में अपनी लेआउट फ़ाइल (जैसे, `default.html`) में जोड़कर ऐसा कर सकते हैं।

```html
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>
```

### चरण 2: Kramdown को अक्षम करें
अपनी `_config.yml` फ़ाइल में, `markdown` विकल्प को `none` पर सेट करके Kramdown को अक्षम करें, या बस इसे हटा दें अगर यह Kramdown पर सेट है।

```yaml
markdown: none
```

### चरण 3: Marked को MathJax के साथ कॉन्फ़िगर करें
आपको Marked को MathJax के साथ काम करने के लिए कॉन्फ़िगर करना होगा। अपने Jekyll प्रोजेक्ट में एक JavaScript फ़ाइल (जैसे, `marked-mathjax.js`) बनाएं और निम्नलिखित कोड शामिल करें:

```javascript
document.addEventListener("DOMContentLoaded", function() {
  // Marked का उपयोग करके Markdown कंटेंट को रेंडर करें
  const elements = document.querySelectorAll('.markdown-content');
  elements.forEach(element => {
    const html = marked(element.textContent);
    element.innerHTML = html;
  });

  // MathJax को रेंडर करें
  MathJax.typesetPromise().then(() => {
    console.log('MathJax rendering complete!');
  }).catch((err) => console.log('Typeset failed:', err));
});
```

Marked और MathJax को शामिल करने के बाद इस स्क्रिप्ट को अपनी लेआउट फ़ाइल में शामिल करें:

```html
<script src="https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-mml-chtml.js"></script>
<script src="{{ '/assets/js/marked-mathjax.js' | relative_url }}"></script>
```

### चरण 4: अपने Markdown कंटेंट को अपडेट करें
सुनिश्चित करें कि आपका Markdown कंटेंट `markdown-content` क्लास वाले कंटेनर में लपेटा गया है ताकि JavaScript रेंडरिंग के लिए उसे टारगेट कर सके।

```html
<div class="markdown-content">
  <!-- आपका Markdown कंटेंट यहाँ -->
</div>
```

### चरण 5: अपने सेटअप का परीक्षण करें
यह सुनिश्चित करने के लिए कि Marked आपके Markdown को सही ढंग से रेंडर कर रहा है और MathJax गणितीय एक्सप्रेशन को ठीक से प्रदर्शित कर रहा है, अपनी Jekyll साइट को स्थानीय रूप से बिल्ड और सर्व करें।

```bash
bundle exec jekyll serve
```

### अतिरिक्त नोट्स
- सुनिश्चित करें कि आपका MathJax कॉन्फ़िगरेशन आपके Markdown कंटेंट के साथ संगत है। आपको अपनी विशिष्ट आवश्यकताओं के आधार पर कॉन्फ़िगरेशन को समायोजित करने की आवश्यकता हो सकती है।
- यदि आपको कोई समस्या आती है, तो त्रुटियों के लिए ब्राउज़र कंसोल जांचें और अपना JavaScript कोड तदनुसार समायोजित करें।

इन चरणों का पालन करके, आप अपने Jekyll ब्लॉग में Kramdown से Marked पर स्विच करने में सक्षम होने चाहिए और गणितीय एक्सप्रेशन के लिए बेहतर MathJax सपोर्ट का लाभ उठा सकते हैं।
