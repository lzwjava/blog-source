आपके प्रश्न के संदर्भ में कि बीन्स XML कैसे काम करता है, जिसमें props और keys के साथ बीन गुण शामिल हैं, कौन सी क्लास इन XML फ़ाइलों का उपयोग करती है, और वे कैसे लोड होती हैं, आइए इसे स्पष्ट और व्यवस्थित रूप से समझते हैं। यह स्पष्टीकरण Spring जैसे फ्रेमवर्क के संदर्भ में है, जहाँ "बीन्स XML" बीन्स को कॉन्फ़िगर करने के लिए एक सामान्य शब्द है।

### बीन्स XML कैसे काम करता है?

Spring जैसे फ्रेमवर्क में, एक **बीन्स XML फ़ाइल** एक कॉन्फ़िगरेशन फ़ाइल होती है जिसका उपयोग *बीन्स* नामक ऑब्जेक्ट्स को परिभाषित और प्रबंधित करने के लिए किया जाता है। ये बीन्स आमतौर पर Java ऑब्जेक्ट्स होते हैं जिन्हें Spring फ्रेमवर्क इंस्टेंटिएट, कॉन्फ़िगर और प्रबंधित करता है। XML फ़ाइल निम्नलिखित निर्दिष्ट करती है:

- **बीन्स**: प्रत्येक बीन को एक `<bean>` टैग का उपयोग करके परिभाषित किया जाता है, जिसमें एक `id` (एक अद्वितीय पहचानकर्ता) और एक `class` (इंस्टेंटिएट करने के लिए Java क्लास का पूरी तरह से योग्य नाम) शामिल होता है।
- **गुण**: बीन्स में गुण हो सकते हैं, जो मान या संदर्भ होते हैं जिन्हें उसके व्यवहार को कॉन्फ़िगर करने के लिए बीन में सेट किया जाता है। गुणों को `<property>` टैग का उपयोग करके परिभाषित किया जाता है।
- **Props और Keys**: एक `<property>` टैग के भीतर, आप key-value जोड़े के एक सेट को परिभाषित करने के लिए एक `<props>` एलिमेंट का उपयोग कर सकते हैं। यह तब उपयोगी होता है जब एक बीन `java.util.Properties` ऑब्जेक्ट या `Map` जैसी समान संरचना की अपेक्षा करता है। `<props>` एलिमेंट में कई `<prop>` टैग होते हैं, जिनमें से प्रत्येक में एक `key` विशेषता और एक संबंधित मान होता है।

यहाँ एक उदाहरण दिया गया है कि यह एक बीन्स XML फ़ाइल में कैसा दिखता है:

```xml
<bean id="myBean" class="com.example.MyBean">
    <property name="someProperty">
        <props>
            <prop key="key1">value1</prop>
            <prop key="key2">value2</prop>
        </props>
    </property>
</bean>
```

इस उदाहरण में:
- `myBean` ID वाला एक बीन `com.example.MyBean` क्लास से बनाया गया है।
- बीन में `someProperty` नामक एक गुण है।
- `<props>` एलिमेंट key-value जोड़े (`key1=value1` और `key2=value2`) के एक सेट को परिभाषित करता है, जिसे Spring एक `Properties` ऑब्जेक्ट में परिवर्तित करता है और `myBean` में `setSomeProperty(Properties props)` जैसे सेटर मेथड के माध्यम से इंजेक्ट करता है।

आपके प्रश्न में "it puts in resources" वाक्यांश थोड़ा अस्पष्ट है, लेकिन इसका संभावित अर्थ यह है कि XML फ़ाइल एक *resource* (एप्लिकेशन की classpath या filesystem में एक फ़ाइल) है जिसका उपयोग एप्लिकेशन करता है, या इसका मतलब यह हो सकता है कि XML में परिभाषित बीन्स (जैसे डेटा सोर्स) एप्लिकेशन द्वारा उपयोग किए जाने वाले संसाधनों का प्रतिनिधित्व करते हैं। अभी के लिए, मान लेते हैं कि यह XML फ़ाइल के बारे में है जो एप्लिकेशन द्वारा लोड की जाने वाली एक resource है।

### कौन सी क्लास इन XML फ़ाइलों का उपयोग करेगी?

Spring में, बीन्स XML फ़ाइल का उपयोग (यानी लोड और प्रोसेस करने) के लिए जिम्मेदार क्लास **`ApplicationContext`** है। अधिक सटीक रूप से, यह `ApplicationContext` इंटरफेस का एक कार्यान्वयन है, जैसे:

- **`ClassPathXmlApplicationContext`**: XML फ़ाइल को classpath से लोड करता है।
- **`FileSystemXmlApplicationContext`**: XML फ़ाइल को filesystem से लोड करता है।

`ApplicationContext` एप्लिकेशन को कॉन्फ़िगरेशन जानकारी प्रदान करने के लिए Spring का केंद्रीय इंटरफेस है। यह बीन्स XML फ़ाइल को पढ़ता है, उसका पार्स करता है, और बीन्स को बनाने और प्रबंधित करने के लिए परिभाषाओं का उपयोग करता है। जबकि बीन्स स्वयं (जैसे, `com.example.MyBean`) XML में परिभाषित गुणों का उपयोग करते हैं, `ApplicationContext` वह क्लास है जो सीधे तौर पर XML फ़ाइल को प्रोसेस करती है ताकि यह संभव हो सके।

### यह कैसे लोड होगा?

बीन्स XML फ़ाइल को एप्लिकेशन में `ApplicationContext` के एक इंस्टेंस को बनाकर और XML फ़ाइल का स्थान निर्दिष्ट करके लोड किया जाता है। यहाँ बताया गया है कि यह चरण-दर-चरण कैसे काम करता है:

1. **XML फ़ाइल स्थान निर्दिष्ट करें**: जब आप `ApplicationContext` को इंस्टेंटिएट करते हैं, तो आप XML फ़ाइल का नाम या पथ प्रदान करते हैं। उदाहरण के लिए:
   ```java
   ApplicationContext context = new ClassPathXmlApplicationContext("beans.xml");
   ```
   यहाँ, `"beans.xml"` classpath में होने की अपेक्षा की जाती है (जैसे, एक सामान्य Java प्रोजेक्ट में `src/main/resources` डायरेक्टरी में)।

2. **एक Resource के रूप में लोडिंग**: `ApplicationContext` XML फ़ाइल को एक `Resource` के रूप में ढूंढने और लोड करने के लिए एक `ResourceLoader` का उपयोग करता है। इस मामले में, `ClassPathXmlApplicationContext` classpath में फ़ाइल को ढूंढता है।

3. **XML को पार्स करना**: आंतरिक रूप से, Spring XML फ़ाइल को पार्स करने के लिए एक `XmlBeanDefinitionReader` का उपयोग करता है। यह रीडर `<bean>` टैग, गुणों और `<props>` एलिमेंट्स की व्याख्या करता है, `BeanDefinition` ऑब्जेक्ट्स बनाता है जो बताते हैं कि प्रत्येक बीन को कैसे बनाया जाना चाहिए।

4. **बीन निर्माण**: `ApplicationContext` (जो Spring के `BeanFactory` का विस्तार करता है) बीन्स को इंस्टेंटिएट करने, उनके गुणों को सेट करने (जिसमें `<props>` से `Properties` ऑब्जेक्ट को इंजेक्ट करना भी शामिल है) और उनके lifecycle को प्रबंधित करने के लिए इन बीन परिभाषाओं का उपयोग करता है।

एक बार लोड हो जाने के बाद, आप `ApplicationContext` से बीन्स को इस तरह प्राप्त कर सकते हैं:
```java
MyBean myBean = context.getBean("myBean", MyBean.class);
```

### सब कुछ एक साथ रखते हुए

- **बीन्स XML कैसे काम करता है**: XML बीन्स और उनके गुणों को परिभाषित करता है, जिसमें `<props>` के माध्यम से key-value जोड़े शामिल होते हैं। Spring इस XML को पार्स करके बीन्स को कॉन्फ़िगर और इंस्टेंटिएट करता है।
- **XML का उपयोग करने वाली क्लास**: `ApplicationContext` (जैसे, `ClassPathXmlApplicationContext`) एप्लिकेशन को सेट अप करने के लिए XML फ़ाइल का उपयोग करती है।
- **यह कैसे लोड होता है**: XML को classpath या filesystem से एक resource के रूप में लोड किया जाता है जब आप फ़ाइल के स्थान को निर्दिष्ट करते हुए एक `ApplicationContext` इंस्टेंस बनाते हैं।

उदाहरण के लिए, यदि आपकी `beans.xml` classpath में है, तो आप इसे इस तरह लोड करेंगे:
```java
ApplicationContext context = new ClassPathXmlApplicationContext("beans.xml");
```
Spring फिर बाकी सब संभालता है, जिससे कॉन्फ़िगर किए गए बीन्स आपके एप्लिकेशन के लिए उपलब्ध हो जाते हैं।

यह तंत्र शक्तिशाली और लचीला है, जो आपको XML में जटिल कॉन्फ़िगरेशन परिभाषित करने की अनुमति देता है जिन्हें Spring रनटाइम पर जीवंत करता है।
