### अपनी GPG कुंजी से एक फ़ाइल को एन्क्रिप्ट करना

आपकी GPG कुंजी सूची एक सार्वजनिक कुंजी (pub) दिखाती है जिसकी फिंगरप्रिंट `982740390B5D33E90B6B6133892BC5A4789B71CD` है और यह आपके ईमेल `lzwjava@gmail.com` से संबद्ध है। इस कुंजी का उपयोग करके एक फ़ाइल को एन्क्रिप्ट करने के लिए (ताकि केवल आप ही इसे अपनी निजी कुंजी से डिक्रिप्ट कर सकें), अपने टर्मिनल में इन चरणों का पालन करें:

1.  **उस डायरेक्टरी में नेविगेट करें** जिसमें वह फ़ाइल है जिसे आप एन्क्रिप्ट करना चाहते हैं (उदाहरण के लिए, यदि आपकी फ़ाइल `secret.txt` है):
    ```
    cd ~/path/to/your/file
    ```

2.  **फ़ाइल को एन्क्रिप्ट करें** `--encrypt` विकल्प का उपयोग करके और प्राप्तकर्ता (आपका ईमेल या कुंजी ID) निर्दिष्ट करें। इससे एक एन्क्रिप्टेड `.gpg` फ़ाइल बनती है।

    - अपने ईमेल का उपयोग करके (सरल):
      ```
      gpg --encrypt --recipient lzwjava@gmail.com secret.txt
      ```

    - कुंजी ID (पूर्ण फिंगरप्रिंट) का उपयोग करके:
      ```
      gpg --encrypt --recipient 982740390B5D33E90B6B6133892BC5A4789B71CD secret.txt
      ```

    इसका आउटपुट `secret.txt.gpg` होगा, जो एन्क्रिप्टेड वर्जन है। मूल `secret.txt` फ़ाइल अपरिवर्तित रहती है।

3.  **एन्क्रिप्शन सत्यापित करें** (वैकल्पिक):
    ```
    gpg --list-packets secret.txt.gpg
    ```
    यह एन्क्रिप्टेड पैकेट के बारे में विवरण दिखाता है।

### फ़ाइल को डिक्रिप्ट करना (बाद के संदर्भ के लिए)

इसे वापस डिक्रिप्ट करने के लिए (केवल आपकी निजी कुंजी और पासफ्रेज से ही संभव):
```
gpg --decrypt secret.txt.gpg > secret.txt.decrypted
```
जब संकेत दिया जाए तो अपना पासफ्रेज दर्ज करें।

### सुझाव

-   **सिमेट्रिक एन्क्रिप्शन** (यदि आपको प्राप्तकर्ता कुंजी की आवश्यकता नहीं है, केवल एक पासफ्रेज चाहिए): इसके बजाय `gpg -c secret.txt` का उपयोग करें।
-   **एन्क्रिप्ट और साइन करें** (यह साबित करने के लिए कि यह आपसे है): एन्क्रिप्ट कमांड में `--sign` जोड़ें, उदाहरण के लिए, `gpg --encrypt --sign --recipient lzwjava@gmail.com secret.txt`।
-   यदि आपको त्रुटियाँ आती हैं (उदाहरण के लिए, कुंजी विश्वसनीय नहीं है), तो `gpg --edit-key lzwjava@gmail.com` चलाएं और इसे अंतिम (ultimate) सेट करने के लिए `trust` का उपयोग करें।
-   कुंजी सुरक्षा के लिए हमेशा अपनी `~/.gnupg` डायरेक्टरी का बैकअप लें।
