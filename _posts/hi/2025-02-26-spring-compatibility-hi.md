### त्रुटि को समझना

जब आप Spring Boot 3.4.3 के साथ `mvn spring-boot:run` चलाते हैं और त्रुटि "class file has wrong version 61.0, should be 52.0" आती है, तो यह एक Java संस्करण संगतता समस्या का संकेत देता है। Java में, क्लास फ़ाइलों का एक संस्करण नंबर होता है जो उस Java संस्करण के अनुरूप होता है जिसके साथ उन्हें संकलित किया गया था:

- **संस्करण 52.0** Java 8 के अनुरूप है।
- **संस्करण 61.0** Java 17 के अनुरूप है।

त्रुटि बताती है कि आपकी Spring Boot एप्लिकेशन की क्लास फ़ाइलें Java 17 (संस्करण 61.0) के साथ संकलित की गई थीं, लेकिन Java Virtual Machine (JVM) जो उन्हें चलाने का प्रयास कर रही है, वह Java 8 (संस्करण 52.0) के साथ संगत क्लास फ़ाइलों की अपेक्षा करती है। चूंकि Java 8 की JVM, Java 17 के साथ संकलित क्लास फ़ाइलों को निष्पादित नहीं कर सकती (क्योंकि Java इस संदर्भ में पिछड़े संगत है लेकिन आगे की ओर संगत नहीं है), इसलिए आपको यह त्रुटि मिलती है।

### ऐसा क्यों होता है

Spring Boot 3.4.3 को अपने न्यूनतम समर्थित संस्करण के रूप में **Java 17 या बाद के संस्करण** की आवश्यकता होती है। इसका मतलब है कि आपके प्रोजेक्ट को कम से कम Java 17 का उपयोग करके संकलित और चलाया जाना चाहिए। यह बेमेल संभवतः इसलिए होता है क्योंकि:

1.  **संकलन**: आपका प्रोजेक्ट Java 17 के साथ संकलित होने के लिए कॉन्फ़िगर किया गया है, जिससे संस्करण 61.0 वाली क्लास फ़ाइलें उत्पन्न होती हैं। यह आमतौर पर आपके `pom.xml` में `maven-compiler-plugin` या `<java.version>` गुण के साथ सेट किया जाता है।
2.  **रनटाइम**: जब आप `mvn spring-boot:run` निष्पादित करते हैं, तो Spring Boot Maven प्लगइन एप्लिकेशन को चलाने के लिए एक नई Java प्रक्रिया शुरू करता है। हालाँकि, यह जिस `java` निष्पादन योग्य का उपयोग करता है, वह Java 8 इंस्टॉलेशन (संस्करण 52.0) से है, जो Java 17 से संकलित क्लास फ़ाइलों को हैंडल नहीं कर सकता।

### समस्या को ठीक करने के चरण

इसे हल करने के लिए, आपको यह सुनिश्चित करने की आवश्यकता है कि संकलन और रनटाइम दोनों वातावरण Java 17 का उपयोग करें। इसे ठीक करने का तरीका यहां बताया गया है:

#### 1. अपने प्रोजेक्ट के Java संस्करण को सत्यापित करें
सबसे पहले, पुष्टि करें कि आपका प्रोजेक्ट Java 17 का उपयोग करने के लिए सेट है। अपने `pom.xml` में, निम्नलिखित की जाँच करें:

```xml
<properties>
    <java.version>17</java.version>
</properties>
```

यह गुण `maven-compiler-plugin` को आपके कोड को Java 17 के साथ संकलित करने के लिए कहता है। Spring Boot 3.4.3 इसे डिफ़ॉल्ट रूप से सेट करता है, लेकिन इसकी पुष्टि करना अच्छा है। यदि यह गायब है या किसी भिन्न संस्करण (जैसे, 8) पर सेट है, तो इसे 17 पर अपडेट करें।

#### 2. Java 17 इंस्टॉल करें
सुनिश्चित करें कि Java 17 आपके सिस्टम पर इंस्टॉल है। आप इसे यहाँ से डाउनलोड कर सकते हैं:

- [Adoptium (Eclipse Temurin)](https://adoptium.net/)
- [Oracle JDK](https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html) (यदि आप लाइसेंस शर्तों को स्वीकार करते हैं)

यह जांचने के लिए कि क्या Java 17 इंस्टॉल है, एक टर्मिनल खोलें और चलाएँ:

```bash
java -version
```

यदि यह Java 17 (जैसे, `openjdk 17.x.x` या इसी तरह) नहीं दिखाता है, तो इसे इंस्टॉल करें और आगे बढ़ें।

#### 3. Java 17 का उपयोग करने के लिए अपने वातावरण को अपडेट करें
Spring Boot Maven प्लगइन आपके वातावरण से `java` निष्पादन योग्य का उपयोग करता है, जो आमतौर पर `JAVA_HOME` पर्यावरण चर या आपके सिस्टम के PATH में `java` कमांड द्वारा निर्धारित होता है। यदि आपकी वर्तमान `java` कमांड Java 8 की ओर इशारा करती है, तो आपको इसे अपडेट करने की आवश्यकता है।

##### विकल्प A: JAVA_HOME और PATH सेट करें
अपने `JAVA_HOME` पर्यावरण चर को Java 17 इंस्टॉलेशन की ओर इशारा करने के लिए सेट करें और सुनिश्चित करें कि आपके PATH में `java` कमांड इसका उपयोग करता है।

- **Linux/Mac पर**:
  1.  अपना Java 17 इंस्टॉलेशन डायरेक्टरी ढूंढें (उदाहरण के लिए, `/usr/lib/jvm/java-17-openjdk` या जहाँ भी आपने इसे इंस्टॉल किया है)।
  2.  अपने टर्मिनल में `JAVA_HOME` सेट करें और `PATH` अपडेट करें:
      ```bash
      export JAVA_HOME=/path/to/java-17
      export PATH=$JAVA_HOME/bin:$PATH
      ```
  3.  सत्यापित करें:
      ```bash
      java -version
      ```
      अब इसे Java 17 दिखाना चाहिए।

  इसे स्थायी बनाने के लिए, `export` लाइनों को अपनी शेल कॉन्फ़िगरेशन फ़ाइल (जैसे, `~/.bashrc`, `~/.zshrc`) में जोड़ें।

- **Windows पर**:
  1.  Java 17 इंस्टॉलेशन डायरेक्टरी ढूंढें (उदाहरण के लिए, `C:\Program Files\Java\jdk-17`)।
  2.  `JAVA_HOME` सेट करें:
      - सिस्टम गुण विंडो खोलें (स्टार्ट मेन्यू में "Environment Variables" खोजें)।
      - "System Variables" के तहत, `JAVA_HOME` को `C:\Program Files\Java\jdk-17` पर सेट करें या अपडेट करें।
  3.  `Path` अपडेट करें:
      - "System Variables" में, `Path` को संपादित करें और `%JAVA_HOME%\bin` जोड़ें (सुनिश्चित करें कि यह शीर्ष पर है या Java 8 जैसे पुराने Java पथों को हटा दें)।
  4.  एक नया कमांड प्रॉम्प्ट खोलें और सत्यापित करें:
      ```cmd
      java -version
      ```
      इसे Java 17 दिखाना चाहिए।

##### विकल्प B: Maven के लिए एक विशिष्ट Java संस्करण का उपयोग करें
यदि आपके पास कई Java संस्करण इंस्टॉल हैं और आप अपने सिस्टम-वाइड `JAVA_HOME` को बदलना नहीं चाहते हैं, तो आप एक विशिष्ट Java संस्करण के साथ Maven चला सकते हैं:

- `mvn spring-boot:run` चलाने से पहले, उस कमांड के लिए `JAVA_HOME` सेट करें:
  ```bash
  JAVA_HOME=/path/to/java-17 mvn spring-boot:run
  ```
  यह सुनिश्चित करता है कि Spring Boot Maven प्लगइन आपके सिस्टम सेटिंग्स को बदले बिना Java 17 का उपयोग करता है।

#### 4. एप्लिकेशन को फिर से चलाएँ
Java 17 को ठीक से सेट करने के बाद, चलाएँ:

```bash
mvn spring-boot:run
```

प्लगइन अब आपकी एप्लिकेशन को निष्पादित करने के लिए Java 17 का उपयोग करेगा, जो संकलित क्लास फ़ाइल संस्करण (61.0) से मेल खाएगा, और त्रुटि गायब हो जानी चाहिए।

### अतिरिक्त जाँच
- **Maven का Java संस्करण**: यह देखने के लिए `mvn -version` चलाएँ कि Maven किस Java संस्करण का उपयोग करता है। यह आपके `JAVA_HOME` से मेल खाना चाहिए। यदि नहीं, तो सुनिश्चित करें कि आपका PATH और `JAVA_HOME` सही ढंग से कॉन्फ़िगर किया गया है।
- **निर्भरताएँ**: यदि त्रुटि बनी रहती है, तो हो सकता है कि एक निर्भरता Java 17 के साथ संकलित की गई हो जबकि आपका रनटाइम अभी भी गलत तरीके से कॉन्फ़िगर है। चूंकि Spring Boot 3.4.3 Java 17 को लागू करता है, इसलिए यह सुनिश्चित करना कि आपका रनटाइम Java 17 है, इसे हल कर देगा।

### Java 8 क्यों नहीं?
Spring Boot 3.4.3 Java 8 का समर्थन नहीं करता है। यदि आपको Java 8 का उपयोग करने की आवश्यकता है, तो आपको एक पुराने Spring Boot संस्करण (जैसे, 2.x) पर डाउनग्रेड करना होगा, लेकिन यह तब तक अनुशंसित नहीं है जब तक कि आपके पास विशिष्ट बाधाएँ न हों। चूंकि आपकी क्लास फ़ाइलें संस्करण 61.0 की हैं, इसलिए Java 17 के साथ बने रहना Spring Boot 3.4.3 की आवश्यकताओं के अनुरूप है।

### अंतिम समाधान सारांश
Spring Boot 3.4.3 के साथ `mvn spring-boot:run` चलाते समय "class file has wrong version 61.0, should be 52.0" को ठीक करने के लिए:

1.  सुनिश्चित करें कि `<java.version>17</java.version>` आपके `pom.xml` में है।
2.  यदि पहले से मौजूद नहीं है तो Java 17 इंस्टॉल करें।
3.  `JAVA_HOME` को अपने Java 17 इंस्टॉलेशन डायरेक्टरी पर सेट करें।
4.  अपने PATH को अपडेट करें ताकि `java -version` Java 17 दिखाए।
5.  `mvn spring-boot:run` को फिर से चलाएँ।

यह सुनिश्चित करता है कि आपका रनटाइम आपकी संकलित Java 17 क्लास फ़ाइलों से मेल खाता है, जिससे संगतता समस्या हल हो जाती है।
