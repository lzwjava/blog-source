Voici quelques invites fréquemment utilisées pour diverses tâches de programmation, organisées par catégorie. Ces invites sont couramment employées lors de l'écriture de code, du refactoring, du changement de noms de variables et d'autres activités connexes, en particulier dans des contextes comme les IDE, les éditeurs de code ou lors d'interactions avec des assistants de programmation IA.

### Écriture de Code
Ces invites sont utilisées pour générer de nouveaux snippets de code ou des implémentations :
- "Écris une fonction pour [tâche spécifique]."  
  *Exemple : "Écris une fonction pour calculer la somme d'un tableau."*
- "Crée une classe pour [objectif spécifique]."  
  *Exemple : "Crée une classe pour gérer un profil utilisateur."*
- "Génère du code pour [obtenir une certaine fonctionnalité]."  
  *Exemple : "Génère du code pour récupérer des données depuis une API."*

### Refactoring
Ces invites se concentrent sur l'amélioration de la structure, de la lisibilité ou de l'efficacité du code existant sans modifier son comportement :
- "Refactorise ce code pour améliorer sa lisibilité."  
  *Exemple : "Refactorise ce code pour utiliser des noms de variables plus clairs et une mise en forme cohérente."*
- "Simplifie cette fonction en la divisant en fonctions plus petites."  
  *Exemple : "Simplifie cette fonction en extrayant la logique de validation dans une méthode séparée."*
- "Optimise ce code pour de meilleures performances."  
  *Exemple : "Optimise ce code en réduisant les boucles imbriquées."*

### Changement de Noms de Variables
Ces invites visent à améliorer la clarté du code en mettant à jour les identifiants pour qu'ils soient plus descriptifs ou cohérents :
- "Renomme la variable '[nom_variable]' avec un nom plus descriptif."  
  *Exemple : "Renomme la variable 'x' en 'ageUtilisateur'."*
- "Change le nom de la fonction '[nom_fonction]' pour mieux refléter son objectif."  
  *Exemple : "Change le nom de la fonction 'func' en 'calculerPrixTotal'."*
- "Met à jour tous les noms de variables pour suivre [convention de nommage spécifique]."  
  *Exemple : "Met à jour tous les noms de variables pour suivre la convention camelCase."*

### Débogage
Ces invites aident à identifier et résoudre les problèmes dans le code :
- "Trouve et corrige l'erreur dans ce code."  
  *Exemple : "Trouve et corrige l'erreur causant cette exception de pointeur nul."*
- "Débogue cette fonction pour gérer [cas particuliers spécifiques]."  
  *Exemple : "Débogue cette fonction pour gérer les tableaux d'entrée vides."*

### Ajout de Commentaires/Documentation
Ces invites améliorent la maintenabilité du code en ajoutant des explications ou une documentation formelle :
- "Ajoute des commentaires à ce code pour expliquer sa fonctionnalité."  
  *Exemple : "Ajoute des commentaires à ce code pour décrire l'algorithme de tri utilisé."*
- "Génère la documentation pour cette classe/méthode."  
  *Exemple : "Génère la documentation pour cette classe User."*

### Traduction de Code
Ces invites convertissent le code entre différents langages de programmation :
- "Traduis ce code [langage source] en [langage cible]."  
  *Exemple : "Traduis ce code Python en Java."*
- "Convertis cette fonction [langage] en [autre langage]."  
  *Exemple : "Convertis cette fonction JavaScript en TypeScript."*

### Revue de Code
Ces invites sollicitent des retours ou identifient des améliorations potentielles :
- "Revoyez ce code et suggérez des améliorations."  
  *Exemple : "Revoyez ce code et suggérez des moyens de réduire sa complexité."*
- "Vérifiez ce code pour détecter d'éventuels problèmes de sécurité."  
  *Exemple : "Vérifiez ce code pour détecter des vulnérabilités d'injection SQL."*

### Génération de Tests
Ces invites créent des cas de test pour vérifier l'exactitude du code :
- "Écris des tests unitaires pour cette fonction."  
  *Exemple : "Écris des tests unitaires pour cette fonction calculerRemise."*
- "Crée un cas de test pour vérifier cette méthode."  
  *Exemple : "Crée un cas de test pour vérifier que cette méthode gère les entrées négatives."*

### Invites Conversationnelles
Il s'agit d'invites plus orientées vers le langage naturel, souvent utilisées avec des assistants IA :
- "Pouvez-vous m'aider à écrire un script pour [effectuer une tâche] ?"  
  *Exemple : "Pouvez-vous m'aider à écrire un script pour renommer des fichiers dans un répertoire ?"*
- "J'ai besoin d'un code qui [fait des choses spécifiques]."  
  *Exemple : "J'ai besoin d'un code qui lit un fichier CSV et trie les données."*
- "Montrez-moi un exemple d'[utilisation d'une fonctionnalité ou API particulière]."  
  *Exemple : "Montrez-moi un exemple d'utilisation de l'API Fetch en JavaScript."*

Ces invites fournissent une base polyvalente pour les tâches de programmation courantes. Elles peuvent être adaptées en renseignant des détails spécifiques (par exemple, des noms de fonctions, des langages ou des contraintes) en fonction du contexte ou des exigences de votre projet. Que vous écriviez du nouveau code, que vous amélioriez une base de code existante ou que vous cherchiez de l'aide auprès d'un outil IA, ces modèles devraient couvrir un large éventail de scénarios.
