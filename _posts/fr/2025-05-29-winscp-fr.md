## **Aperçu de WinSCP**
WinSCP (Windows Secure Copy) est un client de transfert de fichiers gratuit et open source conçu pour Windows. Il permet aux utilisateurs de transférer des fichiers de manière sécurisée entre une machine Windows locale et un serveur distant ou entre deux serveurs distants. Son interface utilisateur graphique (GUI) le rend accessible aux débutants, tandis que ses capacités de script et d'automatisation répondent aux besoins des utilisateurs avancés. Il est largement utilisé pour gérer les fichiers sur les serveurs web, le stockage en cloud ou d'autres systèmes distants.

---

## **Fonctionnalités principales**
- **Support des protocoles** : SFTP, FTP, FTPS, SCP, WebDAV et Amazon S3.
- **Interface utilisateur graphique** : Gestionnaire de fichiers à deux panneaux avec support du glisser-déposer.
- **Sécurité** : Prend en charge l'authentification par clé SSH, le chiffrement et les transferts de fichiers sécurisés.
- **Automatisation** : Scripts et interface en ligne de commande pour les transferts de fichiers par lots.
- **Synchronisation** : Synchronisation automatique des répertoires locaux et distants.
- **Intégration** : Intégration transparente avec PuTTY pour l'accès terminal SSH.
- **Personnalisation** : Options étendues pour les paramètres de transfert, les permissions des fichiers, etc.
- **Mode portable** : Peut être exécuté depuis une clé USB sans installation.
- **Multilingue** : Prend en charge plusieurs langues pour une accessibilité mondiale.

---

## **Configuration système requise**
- **Système d'exploitation** : Windows XP ou ultérieur (Windows 10/11 recommandé).
- **Espace disque** : Environ 20 Mo pour l'installation.
- **Dépendances** : Aucune pour les fonctionnalités de base ; PuTTY recommandé pour la gestion des clés SSH.
- **Internet** : Requis pour se connecter aux serveurs distants.

## **Installation**

### **Télécharger et installer WinSCP**
1. **Télécharger WinSCP** :
   - Visitez le site officiel de WinSCP : [https://winscp.net/](https://winscp.net/).
   - Téléchargez la dernière version (par exemple, WinSCP 6.3.4 en mai 2025).
   - Choisissez entre **l'Installeur** (recommandé pour la plupart des utilisateurs) ou l'**exécutable Portable** (pour les clés USB).

2. **Exécuter l'installeur** :
   - Double-cliquez sur le fichier `.exe` téléchargé.
   - Suivez l'assistant de configuration :
     - Sélectionnez la langue (anglais par défaut).
     - Choisissez le type d'installation : **Standard** (installe dans Program Files) ou **Portable** (aucune modification du registre).
     - Sélectionnez le style d'interface : **Commander** (deux panneaux, style Norton Commander) ou **Explorer** (style Explorateur Windows).
   - Terminez l'installation.

3. **Vérifier l'installation** :
   - Lancez WinSCP depuis le menu Démarrer ou le raccourci sur le bureau.
   - Assurez-vous que l'application s'ouvre sans erreurs.

## **Pour commencer**

### **Configuration initiale**
1. **Lancer WinSCP** :
   - Ouvrez WinSCP pour accéder à la boîte de dialogue **Connexion**.

2. **Configurer une nouvelle session** :
   - **Protocole de fichier** : Choisissez SFTP (le plus sécurisé), FTP ou un autre protocole.
   - **Nom de l'hôte** : Entrez l'adresse IP ou le domaine du serveur (par exemple, `example.com`).
   - **Numéro de port** : Par défaut 22 pour SFTP/SCP, 21 pour FTP.
   - **Nom d'utilisateur** : Entrez votre nom d'utilisateur sur le serveur.
   - **Mot de passe** : Entrez votre mot de passe ou laissez vide pour une authentification par clé.
   - **Sauvegarder la session** : Cliquez sur "Sauvegarder" pour enregistrer les détails de connexion pour une utilisation future.

3. **Paramètres avancés** (Optionnel) :
   - Cliquez sur **Avancé** dans la boîte de dialogue Connexion pour configurer :
     - **Clés SSH** : Importez des clés privées pour une authentification sécurisée.
     - **Paramètres de transfert** : Définissez le mode de transfert par défaut (binaire, texte ou auto).
     - **Paramètres de répertoire** : Spécifiez les répertoires locaux et distants par défaut.

4. **Se connecter** :
   - Cliquez sur **Connexion** pour vous connecter au serveur.
   - Acceptez la clé hôte du serveur si vous y êtes invité (vérifiez auprès de votre administrateur serveur en cas de doute).

## **Utiliser WinSCP comme gestionnaire de fichiers**

### **Aperçu de l'interface**
WinSCP propose deux styles d'interface :
- **Interface Commander** : Vue à deux panneaux avec les fichiers locaux à gauche et les fichiers distants à droite. Idéal pour les utilisateurs avancés.
- **Interface Explorer** : Vue à un seul panneau affichant uniquement le serveur distant, ressemblant à l'Explorateur Windows.

**Éléments clés de l'interface** :
- **Barre d'outils** : Accès rapide aux tâches courantes (connexion, déconnexion, nouvelle session).
- **Panneaux de fichiers** : Affichent les répertoires locaux et distants.
- **Barre d'état** : Affiche le statut de la connexion, la progression du transfert et les erreurs.
- **Menus contextuels** : Clic droit sur les fichiers/dossiers pour les options comme téléverser, télécharger, supprimer ou renommer.

### **Connexion à un serveur distant**
1. Ouvrez WinSCP et sélectionnez une session sauvegardée ou créez-en une nouvelle.
2. Entrez les identifiants ou sélectionnez une clé SSH.
3. Cliquez sur **Connexion** pour établir la connexion.
4. Si la connexion réussit, le système de fichiers du serveur distant apparaît dans l'interface.

### **Navigation et gestion des fichiers**
- **Navigation** :
  - Utilisez les panneaux de fichiers pour parcourir les répertoires locaux et distants.
  - Double-cliquez sur les dossiers pour les ouvrir.
  - Utilisez les boutons **Retour** et **Avancer** ou la navigation par fil d'Ariane.
- **Opérations sur les fichiers** :
  - **Copier/Déplacer** : Glissez-déposez les fichiers entre les panneaux ou utilisez les options du menu contextuel.
  - **Supprimer** : Clic droit et sélectionnez **Supprimer** ou appuyez sur `Suppr`.
  - **Renommer** : Clic droit et sélectionnez **Renommer** ou appuyez sur `F2`.
  - **Créer un dossier** : Clic droit et sélectionnez **Nouveau > Répertoire**.
  - **Permissions** : Clic droit et sélectionnez **Propriétés** pour modifier les permissions des fichiers (par exemple, 755 pour les répertoires, 644 pour les fichiers sur les serveurs Linux).
- **Rechercher** : Utilisez `Ctrl+F` pour trouver des fichiers dans le répertoire courant.

### **Transfert de fichiers**
1. **Téléverser** :
   - Glissez-déposez les fichiers du panneau local vers le panneau distant.
   - Ou, faites un clic droit sur un fichier et sélectionnez **Téléverser**.
   - Choisissez le mode de transfert :
     - **Binaire** : Pour les images, exécutables ou archives.
     - **Texte** : Pour les fichiers texte afin de gérer les conversions de fins de ligne.
     - **Automatique** : WinSCP décide en fonction du type de fichier.
2. **Télécharger** :
   - Glissez-déposez les fichiers du panneau distant vers le panneau local.
   - Ou, faites un clic droit et sélectionnez **Télécharger**.
3. **Paramètres de transfert** :
   - Configurez dans **Préférences > Transfert** :
     - Définissez des limites de vitesse.
     - Activez la reprise pour les transferts interrompus.
     - Préservez les horodatages ou les permissions.

### **Synchronisation de fichiers**
La fonction de synchronisation de WinSCP permet de garder les répertoires locaux et distants synchronisés.
1. Allez dans **Commandes > Synchroniser**.
2. Choisissez :
   - **Local** : Mettre à jour le répertoire local pour qu'il corresponde au distant.
   - **Distant** : Mettre à jour le répertoire distant pour qu'il corresponde au local.
   - **Miroir** : Garder les deux répertoires identiques.
3. Sélectionnez les répertoires à comparer.
4. Prévisualisez les modifications et confirmez pour synchroniser.

## **Fonctionnalités avancées**

### **Automatisation et scripts**
WinSCP prend en charge l'automatisation pour les tâches répétitives :
- **Scripts par lots** :
  - Écrivez des scripts en utilisant l'interface en ligne de commande de WinSCP (par exemple, `winscp.com`).
  - Exemple : Automatiser les sauvegardes nocturnes.
    ```bash
    open sftp://user:password@host
    get /remote/path/* C:\local\path\
    exit
    ```
  - Sauvegardez en tant que fichier `.txt` et exécutez avec `winscp.com /script=script.txt`.
- **Assembly .NET** :
  - Utilisez l'assembly .NET de WinSCP pour l'intégration avec PowerShell ou C#.
  - Exemple : Automatiser les transferts de fichiers dans un script PowerShell.

### **Commandes personnalisées**
- Créez des commandes personnalisées pour les tâches répétitives (par exemple, compresser des fichiers sur le serveur).
- Allez dans **Commandes > Commandes personnalisées > Ajouter**.
- Exemple : Exécutez `tar -czf archive.tar.gz !` pour compresser les fichiers sélectionnés.

### **Intégration avec PuTTY**
- Liez WinSCP avec PuTTY pour l'accès terminal SSH :
  - Dans **Préférences > Intégration**, spécifiez le chemin vers `putty.exe`.
  - Utilisez **Ouvrir dans PuTTY** depuis la barre d'outils pour lancer une session terminal avec les mêmes identifiants.

### **Chiffrement et sécurité**
- **Clés SSH** :
  - Générez des clés en utilisant PuTTYgen ou importez des clés existantes.
  - Configurez dans **Avancé > SSH > Authentification**.
- **Chiffrement** : Chiffrez les fichiers avant le transfert en utilisant le chiffrement intégré de WinSCP.
- **Mot de passe principal** : Protégez les mots de passe des sessions stockées dans **Préférences > Sécurité**.

## **Conseils et bonnes pratiques**
- **Sauvegarder les sessions** : Enregistrez les connexions fréquemment utilisées pour gagner du temps.
- **Utiliser le mode binaire** : Pour les fichiers non textuels pour éviter la corruption.
- **Sauvegarder avant la suppression** : Vérifiez toujours les fichiers avant de les supprimer sur le serveur.
- **Vérifier les permissions** : Assurez-vous des permissions correctes pour les serveurs web (par exemple, 644 pour les fichiers, 755 pour les répertoires).
- **Mettre à jour régulièrement** : Maintenez WinSCP à jour pour les correctifs de sécurité et les nouvelles fonctionnalités.
- **Utiliser les filtres** : Excluez les fichiers inutiles (par exemple, `.git`) pendant les transferts.

## **Résolution des problèmes courants**
1. **Connexion refusée** :
   - Vérifiez l'hôte, le port et les identifiants.
   - Vérifiez si le pare-feu du serveur autorise les connexions.
2. **Échec de l'authentification** :
   - Assurez-vous que le nom d'utilisateur/mot de passe ou la clé SSH est correct.
   - Vérifiez le format de la clé (WinSCP supporte le format `.ppk` de PuTTY).
3. **Transferts lents** :
   - Vérifiez la stabilité du réseau.
   - Ajustez les paramètres de transfert (par exemple, désactivez la compression pour des transferts plus rapides).
4. **Permission refusée** :
   - Confirmez les permissions de l'utilisateur sur le serveur.
   - Utilisez **Propriétés** pour ajuster les permissions des fichiers/dossiers.
5. **Vérification de la clé hôte** :
   - Acceptez la clé hôte seulement si vous faites confiance au serveur.
   - Contactez l'administrateur du serveur pour vérifier l'empreinte de la clé.

## **Alternatives à WinSCP**
- **FileZilla** : Client FTP/SFTP open source avec une interface similaire.
- **Cyberduck** : Prend en charge plusieurs protocoles et le stockage en cloud (macOS/Windows).
- **PuTTY PSCP** : Outil SCP en ligne de commande pour les transferts basiques.
- **MobaXterm** : Outil tout-en-un avec transfert de fichiers et capacités terminal.
