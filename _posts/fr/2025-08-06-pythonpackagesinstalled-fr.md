Ce sont tous des packages Python (wheels) qui sont installés — voici un très bref aperçu de chacun :

* **safetensors** : un format de fichier rapide et sécurisé pour sauvegarder/charger des données tensorielles sans les problèmes de sécurité de pickle.
* **typer** : une bibliothèque moderne pour créer des applications CLI avec les indications de type Python.
* **shellingham** : utilitaire pour détecter dans quel shell (bash, zsh, etc.) vous exécutez un programme.
* **uvicorn** : un serveur ASGI extrêmement rapide pour héberger des applications web Python (par ex. FastAPI).
* **httptools** : bibliothèque d'analyse syntaxique HTTP bas niveau, utilisée par des serveurs comme Uvicorn pour la vitesse.
* **uvloop** : boucle d'événements asyncio ultra-rapide implémentée en Cython, remplacement direct pour asyncio.
* **watchfiles** : bibliothèque de surveillance de fichiers qui peut déclencher des actions (par ex. rechargement automatique) lors de changements dans le système de fichiers.
* **airportsdata** : tables de recherche simples pour les codes, noms et localisations des aéroports.
* **astor** : outils pour reconvertir les AST Python en code source (et vice versa).
* **blake3** : binding Python pour l'algorithme de hachage cryptographique BLAKE3.
* **cloudpickle** : module pickle amélioré pour les objets Python, gère plus de types que le pickle de la bibliothèque standard.
* **cupy\_cuda12x** : bibliothèque de tableaux GPU compatible NumPy pour CUDA 12.x.
* **fastrlock** : une implémentation de verrou réentrant très rapide pour Python.
* **diskcache** : une bibliothèque de cache sur disque pour persister efficacement les objets Python.
* **einops** : opérations expressives et lisibles de remodelage et de réorganisation de tenseurs.
* **importlib\_metadata** : accès aux métadonnées des packages (version, points d'entrée) à l'exécution (rétroportage pour les anciennes versions de Python).
* **zipp** : interface de style pathlib pour les archives ZIP, utilisée par importlib\_metadata.
* **msgspec** : sérialisation/désérialisation de messages haute performance (JSON, Msgpack, etc.).
* **nest\_asyncio** : permet l'imbrication des boucles d'événements asyncio (par ex. pour exécuter des notebooks).
* **ninja** : bindings Python pour le système de build Ninja (builds rapides et incrémentiels).
* **partial\_json\_parser** : analyseur JSON en flux continu/partiel pour JSON volumineux ou mal formé.
* **py\_cpuinfo** : recueille des informations détaillées sur le CPU (modèle, cœurs, fréquence).
* **python\_json\_logger** : formateur JSON pour le module de journalisation de Python.
* **pyzmq** : bindings Python pour la bibliothèque de messagerie haute performance ZeroMQ.
* **scipy** : bibliothèque principale de calcul scientifique (optimisation, intégration, statistiques, etc.).
* **outlines\_core** : utilitaires de base pour la bibliothèque « outlines » (gère les structures/contours de documents).
* **xformers** : implémentations efficaces et modulaires de transformers (attention) de Meta.
* **sentencepiece** : tokeniseur/détokeniseur de texte non supervisé (couramment utilisé en NLP).
