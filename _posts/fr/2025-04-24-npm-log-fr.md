NPM (Node Package Manager) propose plusieurs niveaux de journalisation qui contrôlent la verbosité de la sortie lors de l'exécution des commandes npm. Les voici, du moins au plus verbeux :

- **silent** : N'affiche aucune sortie
- **error** : Affiche uniquement les erreurs (par défaut)
- **warn** : Affiche les erreurs et les avertissements
- **notice** : Affiche les erreurs, les avertissements et les notifications
- **http** : Affiche les requêtes HTTP
- **timing** : Affiche les informations de temporisation
- **info** : Affiche les messages d'information
- **verbose** : Affiche des informations détaillées
- **silly** : Affiche toutes les informations de débogage

Vous pouvez définir le niveau de journalisation en utilisant l'option `--loglevel` ou la variable d'environnement `npm_config_loglevel` :

```bash
npm install --loglevel verbose
```

Ou vous pouvez le définir dans votre configuration npm :

```bash
npm config set loglevel verbose
```

Le niveau de journalisation par défaut est "error" pour la plupart des commandes, ce qui signifie que vous ne verrez que les messages d'erreur à moins de demander explicitement une sortie plus verbeuse.
