## Installation

J'ai précédemment écrit sur [Comment construire un ordinateur](./computer-en). Essayons quelque chose de différent. Aujourd'hui, j'installe Ubuntu 24.04 sur un ordinateur portable Lenovo Xiaoxing 14IAH8 livré avec Windows 10 préinstallé.

```bash
sudo dd if=ubuntu-24.04.1-desktop-amd64.iso of=/dev/rdisk6 bs=1m
```

Il est nécessaire d'aller dans les paramètres et de désactiver le chiffrement du périphérique, car l'installateur Ubuntu l'exige.

Si le chiffrement du périphérique est activé et que le démarrage sécurisé est désactivé dans le BIOS, Windows demandera la phrase secrète BitLocker pour la récupération.

Par conséquent, il est nécessaire d'activer d'abord le démarrage sécurisé dans le BIOS, de se connecter à Windows, puis d'aller dans les paramètres pour désactiver le chiffrement du périphérique. Ensuite, le démarrage sécurisé peut être désactivé à nouveau pour démarrer l'installateur Ubuntu.

## Blocage

Lorsque Ubuntu démarre le service `gdm.service` (GNOME Display Manager) et se bloque, cela indique généralement un problème avec l'interface utilisateur graphique (GUI) ou le gestionnaire d'affichage lui-même. Voici quelques étapes que vous pouvez suivre pour dépanner et potentiellement résoudre le problème :

1. **Accéder à un terminal virtuel :**
   - Si votre système est bloqué à l'écran GDM, vous pouvez essayer d'accéder à un terminal pour diagnostiquer le problème.
   - Appuyez sur `Ctrl + Alt + F2` (ou `F3`, `F4`, etc.) pour passer à un terminal virtuel.

2. **Se connecter :**
   - Connectez-vous avec votre nom d'utilisateur et votre mot de passe.

3. **Vérifier les journaux système :**
   - Utilisez la commande suivante pour vérifier les journaux système afin de détecter toute erreur ou avertissement pouvant indiquer la cause du problème :
     ```bash
     sudo journalctl -xe
     ```

4. **Redémarrer le service GDM :**
   - Essayez de redémarrer le service GDM pour voir si cela résout le problème :
     ```bash
     sudo systemctl restart gdm
     ```

5. **Installer et configurer LightDM :**
   - Si le problème persiste, vous pouvez essayer d'installer un gestionnaire d'affichage alternatif :
     ```bash
     sudo apt-get install lightdm
     ```
   - Après l'installation de lightdm, le système vous demandera de définir le gestionnaire d'affichage par défaut.

6. **Redémarrer le système :**
   - Après avoir effectué des modifications, redémarrez votre système pour les appliquer :
     ```bash
     sudo reboot
     ```
