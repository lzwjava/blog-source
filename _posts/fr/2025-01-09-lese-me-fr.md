> "resume-en.txt" n'est pas autorisé à être analysé. Il doit être renommé en "resume.txt".

> "resume-en.txt" n'est pas autorisé à être analysé. Je dois le renommer en "resume.txt".

Lequel est meilleur ? Apparemment, la première version est préférable car elle implique que ce n'est pas seulement moi, ou lui, ou elle, mais tout le monde qui doit renommer le fichier dans cette situation.

Elle suggère également que le travail humain pourrait ne pas être nécessaire—cette tâche pourrait potentiellement être gérée par du code ou des machines.

La deuxième version est plus directe, exprimant les choses de manière simple. Cependant, nous devrions viser à exprimer les choses d'un point de vue objectif plutôt que de notre propre perspective. Ajuster nos expressions dans chaque situation peut être difficile.

Concernant ce qui précède, j'ai utilisé "nous" ici. Cependant, ce n'est pas seulement "nous"—le code ou la machine peut également gérer cette tâche. Par conséquent, il est préférable d'écrire : "Notre expression devrait être ajustée de notre point de vue à une perspective objective."

Ne vous concentrez pas sur les êtres humains. Observez simplement comment les choses devraient fonctionner objectivement.

Lorsque vous faites référence à des personnes, il est souvent préférable de mentionner un individu spécifique plutôt qu'un groupe, une communauté ou une nationalité. Évitez d'étiqueter les gens ; décrivez plutôt leurs actions ou comportements.

Dans des contextes professionnels, utilisez des titres plutôt que des noms pour mettre l'accent sur les rôles et les responsabilités. Les actions et l'état d'esprit doivent refléter la position, et non l'individu.
