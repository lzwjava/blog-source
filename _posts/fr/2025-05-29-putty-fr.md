## Qu'est-ce que PuTTY ?

PuTTY est un émulateur de terminal polyvalent et un client réseau développé par Simon Tatham, principalement pour Windows mais également disponible pour Linux et macOS. Il prend en charge plusieurs protocoles, notamment :

- **SSH (Secure Shell)** : Pour un accès distant sécurisé aux serveurs.
- **Telnet** : Pour les connexions non chiffrées (moins courantes en raison des problèmes de sécurité).
- **Rlogin** : Pour la connexion à distance aux systèmes Unix.
- **SCP et SFTP** : Pour les transferts de fichiers sécurisés.
- **Connexions par socket brut** : Pour la communication réseau directe.
- **Connexions série** : Pour interagir avec du matériel comme les routeurs.

PuTTY est léger, hautement configurable et inclut des outils comme PuTTYgen (pour générer des clés SSH) et PSFTP/PSCP (pour les transferts de fichiers). Il est largement utilisé par les administrateurs système, les développeurs et les professionnels de l'informatique pour gérer les serveurs distants.[](https://en.wikipedia.org/wiki/PuTTY)

## Téléchargement et installation de PuTTY

### Étape 1 : Télécharger PuTTY
1. Visitez le site officiel de PuTTY à [www.putty.org](https://www.putty.org/) ou la page de téléchargement à [chiark.greenend.org.uk/~sgtatham/putty/latest.html](https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html).[](https://www.putty.org/)
2. Choisissez le programme d'installation approprié :
   - **Programme d'installation MSI 32 bits ou 64 bits** : Sélectionnez `putty-<version>-installer.msi` en fonction de votre système (64 bits est recommandé pour les PC modernes).[](https://its.gmu.edu/knowledge-base/how-to-install-putty-ssh-for-windows/)
   - **Binaires individuels** : Téléchargez `putty.exe`, `puttygen.exe`, `psftp.exe`, etc., si vous préférez ne pas utiliser de programme d'installation.
3. Si vous n'êtes pas sûr du type de votre système, la version 32 bits est universellement compatible.[](https://www.ssh.com/academy/ssh/putty/windows/install)

### Étape 2 : Installer PuTTY
1. Double-cliquez sur le fichier `.msi` téléchargé pour lancer l'assistant d'installation de PuTTY.
2. Cliquez sur **Suivant** sur l'écran de bienvenue.
3. Acceptez le dossier de destination par défaut (généralement `C:\Program Files\PuTTY\`) ou choisissez un emplacement personnalisé, puis cliquez sur **Suivant**.[](https://www.techtarget.com/searchsecurity/tutorial/How-to-use-PuTTY-for-SSH-key-based-authentication)
4. Sélectionnez les fonctionnalités souhaitées (par exemple, créer un raccourci sur le bureau) et cliquez sur **Installer**.[](https://www.ssh.com/academy/ssh/putty/windows/install)
5. Si vous y êtes invité par le Contrôle de compte d'utilisateur Windows, cliquez sur **Oui**.
6. Une fois l'installation terminée, cliquez sur **Terminer**.[](https://www.ssh.com/academy/ssh/putty/windows/install)

### Étape 3 : Vérifier l'installation
- Si vous avez créé un raccourci sur le bureau, double-cliquez sur l'icône PuTTY.
- Sinon, recherchez "PuTTY" dans le menu Démarrer de Windows ou naviguez vers `C:\Program Files\PuTTY\putty.exe`.[](https://its.gmu.edu/knowledge-base/how-to-install-putty-ssh-for-windows/)

## Configuration de base et connexion à un serveur

### Lancement de PuTTY
1. Ouvrez PuTTY à partir du menu Démarrer ou du raccourci sur le bureau. Cela ouvre la fenêtre **Configuration de PuTTY**, qui comporte :
   - Un volet gauche pour les catégories de configuration.
   - Une section centrale pour saisir les détails de connexion.
   - Une section en bas à droite pour enregistrer les profils de session.[](https://www.ssh.com/academy/ssh/putty/windows)

### Connexion à un serveur
1. **Saisir les détails de l'hôte** :
   - Dans le champ **Nom d'hôte (ou adresse IP)**, entrez le nom de domaine du serveur (par exemple, `students.example.edu`) ou l'adresse IP (par exemple, `192.168.1.1`).
   - Assurez-vous que le **Port** est défini sur `22` pour SSH (par défaut) ou sur le port spécifié par votre administrateur de serveur.[](https://www.wikihow.com/Use-Putty-in-Windows)
2. **Sélectionner le type de connexion** :
   - Choisissez **SSH** (par défaut) pour les connexions sécurisées. Les autres options incluent Telnet, Rlogin, Raw ou Serial.[](https://www.wikihow.com/Use-Putty-in-Windows)
3. **Enregistrer une session** (optionnel) :
   - Entrez un nom dans le champ **Sessions enregistrées** (par exemple, `MonServeur`).
   - Cliquez sur **Sauvegarder** pour stocker la configuration pour une utilisation future.[](https://www.cuit.columbia.edu/putty)
4. **Se connecter** :
   - Cliquez sur **Ouvrir** pour initier la connexion.
   - Si vous vous connectez pour la première fois, une **Alerte de sécurité PuTTY** peut apparaître, avertissant que la clé d'hôte du serveur n'est pas mise en cache. Cliquez sur **Oui** pour faire confiance au serveur et continuer.[](https://www.ssh.com/academy/ssh/putty/windows)

### Connexion
- Saisissez votre nom d'utilisateur et votre mot de passe lorsqu'ils sont demandés dans la fenêtre du terminal.
- Si une authentification par clé est requise, configurez-la comme décrit dans la section [Utilisation de PuTTYgen](#utilisation-de-puttygen-pour-lauthentification-par-clé-ssh).[](https://www.wikihow.com/Use-Putty-in-Windows)

## Options de configuration avancées

La fenêtre de configuration de PuTTY offre des options étendues de personnalisation. Voici les catégories et paramètres clés :

### Session
- **Sessions enregistrées** : Chargez, sauvegardez ou supprimez des profils de session pour un accès rapide.
- **Fermer la fenêtre à la sortie** : Choisissez si le terminal se ferme automatiquement après la déconnexion (`Toujours`, `Jamais` ou `Seulement en sortie propre`).[](https://www.ssh.com/academy/ssh/putty/windows)

### Terminal
- **Mode Auto Wrap** : Activez-le pour renvoyer les lignes de texte longues à la ligne suivante. Si désactivé, le texte dépassant la largeur de la fenêtre peut ne pas s'afficher.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
- **Lignes de défilement** : Augmentez le nombre de lignes stockées dans le tampon (200 par défaut) pour voir plus de sortie de terminal.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
- **Keepalives** : Définissez une valeur (en secondes) pour envoyer des paquets nuls et empêcher les déconnexions dues à l'inactivité.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)

### Fenêtre
- **Apparence** :
   - Modifiez les paramètres de police (par exemple, taille, style) sous **Fenêtre > Apparence**.
   - Modifiez les couleurs sous **Fenêtre > Couleurs** pour améliorer la lisibilité (assurez un contraste suffisant entre l'avant-plan et l'arrière-plan).[](https://www.cuit.columbia.edu/putty)
- **Comportement au redimensionnement** : Ajustez la façon dont le terminal répond au redimensionnement de la fenêtre (par exemple, mettre à l'échelle la taille de la police ou ajuster les dimensions du terminal).[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)

### Connexion
- **Paramètres du proxy** : Configurez les proxies si vous vous connectez via un réseau d'entreprise.
- **Connexion automatique** : Spécifiez un nom d'utilisateur pour contourner l'invite de connexion.[](https://www.puttygen.com/download-putty)
- **Paramètres SSH** :
  - Sélectionnez la version préférée du protocole SSH (SSH-2 est recommandé pour la sécurité).
  - Choisissez les chiffrements de chiffrement (par exemple, AES, 3DES) et les algorithmes d'échange de clés.[](https://en.wikipedia.org/wiki/PuTTY)

### Sauvegarde et chargement des sessions
1. Après avoir configuré les paramètres, revenez à la catégorie **Session**.
2. Entrez un nom dans le champ **Sessions enregistrées** et cliquez sur **Sauvegarder**.
3. Pour charger une session sauvegardée, sélectionnez-la dans la liste, cliquez sur **Charger**, puis sur **Ouvrir**.[](https://www.cuit.columbia.edu/putty)

## Utilisation de PuTTYgen pour l'authentification par clé SSH

L'authentification par clé SSH est plus sécurisée que l'authentification par mot de passe. PuTTYgen génère des paires de clés au format `.ppk` de PuTTY.

### Étape 1 : Générer une paire de clés
1. Ouvrez **PuTTYgen** à partir du menu Démarrer ou de `C:\Program Files\PuTTY\puttygen.exe`.
2. Sélectionnez un type de clé (par exemple, **RSA**, par défaut, ou **ECDSA**) et une longueur en bits (par exemple, 2048 ou 4096 pour RSA).
3. Cliquez sur **Générer** et déplacez votre souris pour créer de l'aléatoire jusqu'à ce que la barre de progression soit complète.
4. Optionnellement, définissez une **phrase secrète** pour une sécurité accrue.
5. Sauvegardez la clé privée (fichier `.ppk`) dans un emplacement sécurisé.
6. Copiez la clé publique affichée dans le champ "Public key for pasting into OpenSSH authorized_keys file".[](https://www.wikihow.com/Use-Putty-in-Windows)

### Étape 2 : Configurer le serveur
1. Connectez-vous au serveur distant en utilisant l'authentification par mot de passe (ou une autre méthode).
2. Créez ou modifiez le fichier `~/.ssh/authorized_keys` :
   - Exécutez `mkdir ~/.ssh && chmod 700 ~/.ssh` pour créer le répertoire `.ssh`.
   - Ouvrez le fichier `authorized_keys` avec un éditeur de texte (par exemple, `nano ~/.ssh/authorized_keys`).
   - Collez la clé publique et sauvegardez le fichier.
   - Définissez les permissions avec `chmod 600 ~/.ssh/authorized_keys`.[](https://www.techtarget.com/searchsecurity/tutorial/How-to-use-PuTTY-for-SSH-key-based-authentication)
3. Déconnectez-vous du serveur.

### Étape 3 : Configurer PuTTY pour l'authentification par clé
1. Ouvrez PuTTY et chargez votre session sauvegardée.
2. Naviguez vers **Connexion > SSH > Auth > Credentials**.
3. Cliquez sur **Parcourir** à côté de "Private key file for authentication" et sélectionnez votre fichier `.ppk`.
4. Revenez à la catégorie **Session**, sauvegardez la session mise à jour et cliquez sur **Ouvrir**.
5. Saisissez votre phrase secrète (si définie) lorsqu'elle est demandée. Vous devriez maintenant vous connecter sans mot de passe.[](https://www.wikihow.com/Use-Putty-in-Windows)

## Transferts de fichiers avec PSFTP et PSCP

PuTTY inclut **PSFTP** (pour SFTP) et **PSCP** (pour SCP) pour les transferts de fichiers sécurisés.

### Utilisation de PSFTP
1. Ouvrez **PSFTP** à partir du menu Démarrer ou de `C:\Program Files\PuTTY\psftp.exe`.
2. Tapez `open <nom_hôte>` (par exemple, `open students.example.edu`) et appuyez sur Entrée.
3. Connectez-vous avec vos identifiants ou votre clé privée.
4. Utilisez des commandes comme :
   - `ls` ou `dir` : Lister les fichiers.
   - `get <nom_fichier>` : Télécharger un fichier.
   - `put <nom_fichier>` : Téléverser un fichier.
   - `cd <répertoire>` : Changer de répertoire.
   - `quit` : Quitter PSFTP.[](https://www.wikihow.com/Use-Putty-in-Windows)

### Utilisation de PSCP
1. Ouvrez une invite de commande ou PowerShell.
2. Naviguez vers le répertoire d'installation de PuTTY (par exemple, `cd C:\Program Files\PuTTY`).
3. Exécutez des commandes comme :
   - `pscp <fichier_local> <nom_utilisateur>@<nom_hôte>:<chemin_distant>` pour téléverser.
   - `pscp <nom_utilisateur>@<nom_hôte>:<fichier_distant> <chemin_local>` pour télécharger.
   - Ajoutez `-i <clé_privée.ppk>` pour l'authentification par clé.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)

### Note
- PSFTP est interactif, tandis que PSCP est basé sur la ligne de commande.
- Pour les transferts de fichiers graphiques, envisagez des alternatives comme WinSCP, qui peut importer les sessions PuTTY.[](https://www.ssh.com/academy/ssh/putty)

## Port Forwarding et Tunneling

PuTTY prend en charge le **tunneling SSH** (port forwarding) pour accéder de manière sécurisée aux services sur des serveurs distants.

### Port Forwarding Local
1. Dans PuTTY, allez dans **Connexion > SSH > Tunnels**.
2. Entrez un **Port source** (par exemple, `8080`) sur votre machine locale.
3. Entrez la **Destination** (par exemple, `remote_host:80`) pour le service distant.
4. Sélectionnez **Local** et cliquez sur **Ajouter**.
5. Sauvegardez la session et connectez-vous.
6. Accédez au service localement (par exemple, `http://localhost:8080`).[](https://www.ssh.com/academy/ssh/putty/windows)

### Port Forwarding Distant
- Similaire au forwarding local, mais sélectionnez **Remote** et spécifiez un port sur le serveur distant à rediriger vers votre machine locale.

### Port Forwarding Dynamique
- Sélectionnez **Dynamic** et spécifiez un port local (par exemple, `8080`). Cela crée un proxy SOCKS pour le routage dynamique (utile pour naviguer via le serveur).[](https://rushax.com/what-is-putty-a-comprehensive-guide/)

## X11 Forwarding pour les applications graphiques

PuTTY prend en charge le **X11 forwarding** pour exécuter des applications graphiques sur un serveur distant et les afficher localement.

1. Installez un serveur X sur Windows (par exemple, **XMing**, une option gratuite).
2. Dans PuTTY, allez dans **Connexion > SSH > X11**.
3. Cochez **Enable X11 forwarding**.
4. Définissez **X display location** sur `localhost:0.0`.
5. Sauvegardez la session et connectez-vous.
6. Exécutez des commandes graphiques (par exemple, `xterm`) sur le serveur, et l'interface s'affichera localement via XMing.[](https://www.ssh.com/academy/ssh/putty/windows)

## Dépannage des problèmes courants

1. **Erreur "Accès refusé"** :
   - Vérifiez le nom d'utilisateur, le mot de passe ou la clé privée.
   - Assurez-vous que l'accès SSH est activé sur le serveur (par exemple, vérifiez les paramètres du panneau de contrôle d'hébergement).[](https://www.hostinger.com/uk/tutorials/how-to-use-putty-ssh)
   - Vérifiez si le serveur nécessite uniquement une authentification par clé.

2. **"Connection Reset by Peer"** :
   - Activez **Keepalives** dans les paramètres de **Connexion** pour éviter les délais d'expiration.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
   - Vérifiez les paramètres du pare-feu ou des périphériques réseau qui peuvent fermer les connexions inactives.

3. **"Server's Host Key Not Cached"** :
   - Ceci est normal pour les premières connexions. Cliquez sur **Oui** pour mettre la clé en cache. Si cela apparaît de manière répétée, vérifiez l'authenticité du serveur pour exclure une attaque de l'homme du milieu.[](https://www.ssh.com/academy/ssh/putty/windows)

4. **Avertissement d'échecs de connexion** :
   - Réinitialisez votre mot de passe ou vérifiez la protection contre les attaques par force brute sur le serveur.
   - Contactez votre administrateur de serveur si vous êtes bloqué.[](https://www.hostinger.com/uk/tutorials/how-to-use-putty-ssh)

5. **La fenêtre PuTTY se ferme de manière inattendue** :
   - Vérifiez les journaux du serveur pour des erreurs (par exemple, permissions incorrectes sur `~/.ssh/authorized_keys`).
   - Assurez-vous que la bonne clé privée est chargée.[](https://www.hostinger.com/uk/tutorials/how-to-use-putty-ssh)

## Conseils et bonnes pratiques

- **Sauvegardez les sessions** : Sauvegardez toujours les profils de session pour éviter de ressaisir les paramètres.[](https://www.cuit.columbia.edu/putty)
- **Utilisez l'authentification par clé** : Elle est plus sécurisée que les mots de passe et empêche les attaques par force brute.[](https://www.techtarget.com/searchsecurity/tutorial/How-to-use-PuTTY-for-SSH-key-based-authentication)
- **Sécurisez les clés privées** : Stockez les fichiers `.ppk` dans un endroit sûr et utilisez une phrase secrète.[](https://www.ssh.com/academy/ssh/putty)
- **Personnalisez l'apparence** : Ajustez les polices et les couleurs pour une meilleure lisibilité, surtout pour les sessions longues.[](https://www.cuit.columbia.edu/putty)
- **Activez les Keepalives** : Empêchez les déconnexions pendant les périodes d'inactivité.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
- **Sauvegardez les configurations** : Exportez les sessions sauvegardées depuis le Registre Windows (`HKEY_CURRENT_USER\Software\SimonTatham\PuTTY\Sessions`) pour la sauvegarde ou la migration.
- **Mettez à jour régulièrement** : Téléchargez la dernière version de PuTTY pour garantir les correctifs de sécurité et les nouvelles fonctionnalités (par exemple, la cryptographie à courbe elliptique dans les versions récentes).[](https://www.ssh.com/academy/ssh/putty)

## Alternatives à PuTTY

Bien que PuTTY soit fiable, les alternatives modernes offrent des fonctionnalités améliorées :
- **OpenSSH** : Intégré à Windows 10/11, Linux et macOS. Basé sur la ligne de commande mais robuste.[](https://phoenixnap.com/kb/putty-alternative)
- **WinSCP** : Un client de transfert de fichiers graphique qui s'intègre avec PuTTY.[](https://www.ssh.com/academy/ssh/putty)
- **MobaXterm** : Offre une interface à onglets, un serveur X intégré et une prise en charge SFTP.[](https://rushax.com/what-is-putty-a-comprehensive-guide/)
- **Bitvise SSH Client** : Prend en charge l'interface graphique, le script et le tunneling en un clic.[](https://phoenixnap.com/kb/putty-alternative)
- **Termius** : Multiplateforme avec gestion des clés SSH et une interface moderne.[](https://phoenixnap.com/kb/putty-alternative)
