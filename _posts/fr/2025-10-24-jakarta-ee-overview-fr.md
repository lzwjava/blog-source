Jakarta EE est une plateforme open source pour développer des applications Java de niveau entreprise. Elle fournit un ensemble de spécifications, d'API et d'outils qui étendent le noyau Java SE (Standard Edition) pour gérer des tâches complexes telles que les services web, l'informatique distribuée, les déploiements cloud-native et les applications côté serveur évolutives.

### Historique succinct
- Initialement connue sous le nom de Java 2 Platform, Enterprise Edition (J2EE) et plus tard Java EE, elle a été développée par Oracle/Sun Microsystems.
- En 2017, Oracle a fait don des technologies à la Eclipse Foundation, conduisant au changement de marque pour Jakarta EE (nommé d'après Jakarta, en Indonésie, où la première conférence Eclipse Java s'est tenue).
- Ce changement l'a rendue entièrement open source et pilotée par la communauté, favorisant une innovation plus rapide et une adoption plus large.

### Caractéristiques principales et objectif
- **Composants** : Inclut des API pour les servlets, JPA (Java Persistence API) pour l'accès aux données, CDI (Contexts and Dependency Injection) pour la gestion des dépendances, JAX-RS pour les services web RESTful, et plus encore.
- **Cas d'utilisation** : Idéal pour construire des logiciels d'entreprise robustes, sécurisés et portables, tels que des plateformes de commerce électronique, des systèmes financiers et des architectures de microservices.
- **Statut actuel** : En 2025, Jakarta EE 11 est la dernière version majeure, mettant l'accent sur la compatibilité cloud-native, le traitement JSON et une sécurité améliorée. Elle est prise en charge par des serveurs comme Open Liberty, Payara et WildFly.
