```javascript
        options: {

      }
    ]
  }
```

#### Détails de Configuration
La configuration implique deux fichiers principaux : `.babelrc` pour les paramètres Babel et `webpack.config.js` pour l'intégration avec webpack.

##### Fichier `.babelrc`
Créez ou éditez `.babelrc` à la racine du projet avec le contenu suivant :

```json
{
  "presets": ["es2015"],
  "plugins": ["transform-runtime"]
}
```

- **Presets** : `"es2015"` correspond à `babel-preset-es2015@6.13.2`, permettant toutes les transformations d'ES2015 en ES5, telles que les fonctions fléchées, let/const et les classes.
- **Plugins** : `"transform-runtime"` utilise `babel-plugin-transform-runtime@6.12.0` pour externaliser les aides Babel, faisant référence à `babel-runtime@6.11.6`, ce qui réduit la taille du code compilé en évitant la duplication des aides entre les fichiers.

Cette configuration assure que le code ES2015 est transpilé et optimisé pour l'efficacité d'exécution.

##### Configuration Webpack
Pour webpack, créez ou mettez à jour `webpack.config.js` avec :

```javascript
module.exports = {
  entry: './index.js',
  output: {
    filename: 'bundle.js'
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'babel-loader',
        options: {
          configFile: './.babelrc'
        }
      }
    ]
  }
};
```

- **Entrée et Sortie** : Spécifie `index.js` comme point d'entrée et produit `bundle.js`.
- **Règles de Module** : Utilise `babel-loader@6.2.5` pour les fichiers `.js`, pointant vers `.babelrc` pour les options, assurant que Babel traite les fichiers avant le regroupement.

Cette configuration tire parti du système de modules de webpack pour gérer les fichiers JavaScript, les compilant via Babel.

#### Utilisation et Flux de Travail
Avec la configuration terminée, écrivez votre code ES2015 dans des fichiers comme `index.js`. Par exemple :

Le plugin `transform-runtime` assure que les aides sont partagées, réduisant la taille du bundle, ce qui est particulièrement utile pour les grands projets.

#### Utilisation Alternative : Babel CLI
Bien que la requête suggère une configuration webpack, pour l'exhaustivité, envisagez d'utiliser Babel CLI pour la compilation manuelle sans webpack. Tout d'abord, installez `babel-cli@6.13.0` :

```bash

```

Ensuite, compilez les fichiers avec :


Ou, ajoutez à `package.json` scripts :

Exécutez avec `npm run build`, qui produit les fichiers compilés dans `lib/`, utile pour les projets Node.js ou le code côté serveur.
