## **Descripción General de WinSCP**
WinSCP (Windows Secure Copy) es un cliente de transferencia de archivos gratuito y de código abierto diseñado para Windows. Permite a los usuarios transferir archivos de forma segura entre una máquina Windows local y un servidor remoto o entre dos servidores remotos. Su interfaz gráfica de usuario (GUI) lo hace accesible para principiantes, mientras que sus capacidades de scripting y automatización atienden a usuarios avanzados. Es ampliamente utilizado para gestionar archivos en servidores web, almacenamiento en la nube u otros sistemas remotos.

---

## **Características Principales**
- **Compatibilidad con Protocolos**: SFTP, FTP, FTPS, SCP, WebDAV y Amazon S3.
- **Interfaz Gráfica de Usuario**: Administrador de archivos de doble panel con soporte para arrastrar y soltar.
- **Seguridad**: Admite autenticación con clave SSH, cifrado y transferencias seguras de archivos.
- **Automatización**: Scripting e interfaz de línea de comandos para transferencias de archivos por lotes.
- **Sincronización**: Sincronización automática de directorios locales y remotos.
- **Integración**: Integración perfecta con PuTTY para acceso por terminal SSH.
- **Personalización**: Amplias opciones para configuraciones de transferencia, permisos de archivos y más.
- **Modo Portátil**: Puede ejecutarse desde una unidad USB sin instalación.
- **Multilingüe**: Admite múltiples idiomas para accesibilidad global.

---

## **Requisitos del Sistema**
- **Sistema Operativo**: Windows XP o posterior (se recomienda Windows 10/11).
- **Espacio en Disco**: Aproximadamente 20 MB para la instalación.
- **Dependencias**: Ninguna para la funcionalidad básica; se recomienda PuTTY para la gestión de claves SSH.
- **Internet**: Requerido para conectarse a servidores remotos.

## **Instalación**

### **Descargar e Instalar WinSCP**
1. **Descargar WinSCP**:
   - Visite el sitio web oficial de WinSCP: [https://winscp.net/](https://winscp.net/).
   - Descargue la última versión (por ejemplo, WinSCP 6.3.4 a mayo de 2025).
   - Elija entre el **Instalador** (recomendado para la mayoría de usuarios) o el **Ejecutable Portátil** (para unidades USB).

2. **Ejecutar el Instalador**:
   - Haga doble clic en el archivo `.exe` descargado.
   - Siga el asistente de configuración:
     - Seleccione el idioma (inglés por defecto).
     - Elija el tipo de instalación: **Estándar** (instala en Archivos de Programa) o **Portátil** (sin cambios en el registro).
     - Seleccione el estilo de interfaz: **Commander** (doble panel, similar a Norton Commander) o **Explorer** (similar al Explorador de Windows).
   - Complete la instalación.

3. **Verificar la Instalación**:
   - Inicie WinSCP desde el menú Inicio o el acceso directo del escritorio.
   - Asegúrese de que la aplicación se abra sin errores.

## **Primeros Pasos**

### **Configuración Inicial**
1. **Iniciar WinSCP**:
   - Abra WinSCP para acceder al cuadro de diálogo **Inicio de sesión**.

2. **Configurar una Nueva Sesión**:
   - **Protocolo de Archivo**: Elija SFTP (más seguro), FTP u otro protocolo.
   - **Nombre del Host**: Ingrese la dirección IP o el dominio del servidor (ej., `ejemplo.com`).
   - **Número de Puerto**: Por defecto es 22 para SFTP/SCP, 21 para FTP.
   - **Nombre de Usuario**: Ingrese su nombre de usuario del servidor.
   - **Contraseña**: Ingrese su contraseña o déjela en blanco para autenticación basada en claves.
   - **Guardar Sesión**: Haga clic en "Guardar" para almacenar los detalles de conexión para uso futuro.

3. **Configuración Avanzada** (Opcional):
   - Haga clic en **Avanzado** en el cuadro de diálogo Inicio de sesión para configurar:
     - **Claves SSH**: Importe claves privadas para autenticación segura.
     - **Configuración de Transferencia**: Establezca el modo de transferencia por defecto (binario, texto o automático).
     - **Configuración de Directorios**: Especifique los directorios local y remoto por defecto.

4. **Conectar**:
   - Haga clic en **Iniciar sesión** para conectarse al servidor.
   - Acepte la clave de host del servidor si se le solicita (verifique con su administrador de servidor si no está seguro).

## **Usar WinSCP como Administrador de Archivos**

### **Descripción General de la Interfaz**
WinSCP ofrece dos estilos de interfaz:
- **Interfaz Commander**: Vista de doble panel con archivos locales a la izquierda y archivos remotos a la derecha. Ideal para usuarios avanzados.
- **Interfaz Explorer**: Vista de un solo panel que muestra solo el servidor remoto, similar al Explorador de Windows.

**Elementos Clave de la Interfaz**:
- **Barra de Herramientas**: Acceso rápido a tareas comunes (conectar, desconectar, nueva sesión).
- **Paneles de Archivos**: Muestran directorios locales y remotos.
- **Barra de Estado**: Muestra el estado de la conexión, el progreso de la transferencia y los errores.
- **Menús Contextuales**: Haga clic derecho en archivos/carpetas para opciones como cargar, descargar, eliminar o renombrar.

### **Conectarse a un Servidor Remoto**
1. Abra WinSCP y seleccione una sesión guardada o cree una nueva.
2. Ingrese las credenciales o seleccione una clave SSH.
3. Haga clic en **Iniciar sesión** para establecer una conexión.
4. Si es exitosa, el sistema de archivos del servidor remoto aparece en la interfaz.

### **Navegar y Gestionar Archivos**
- **Navegación**:
  - Use los paneles de archivos para navegar por directorios locales y remotos.
  - Haga doble clic en las carpetas para abrirlas.
  - Use los botones **Atrás** y **Adelante** o la navegación por migas de pan.
- **Operaciones con Archivos**:
  - **Copiar/Mover**: Arrastre archivos entre paneles o use las opciones del menú contextual.
  - **Eliminar**: Haga clic derecho y seleccione **Eliminar** o presione `Supr`.
  - **Renombrar**: Haga clic derecho y seleccione **Renombrar** o presione `F2`.
  - **Crear Carpeta**: Haga clic derecho y seleccione **Nuevo > Directorio**.
  - **Permisos**: Haga clic derecho y seleccione **Propiedades** para cambiar los permisos de archivo (ej., 755 para directorios, 644 para archivos en servidores Linux).
- **Búsqueda**: Use `Ctrl+F` para buscar archivos en el directorio actual.

### **Transferir Archivos**
1. **Cargar**:
   - Arrastre archivos desde el panel local al panel remoto.
   - O, haga clic derecho en un archivo y seleccione **Cargar**.
   - Elija el modo de transferencia:
     - **Binario**: Para imágenes, ejecutables o archivos comprimidos.
     - **Texto**: Para archivos de texto para manejar conversiones de fin de línea.
     - **Automático**: WinSCP decide basándose en el tipo de archivo.
2. **Descargar**:
   - Arrastre archivos desde el panel remoto al panel local.
   - O, haga clic derecho y seleccione **Descargar**.
3. **Configuración de Transferencia**:
   - Configure en **Preferencias > Transferencia**:
     - Establezca límites de velocidad.
     - Habilite soporte para reanudar transferencias interrumpidas.
     - Preserve marcas de tiempo o permisos.

### **Sincronizar Archivos**
La función de sincronización de WinSCP mantiene sincronizados los directorios locales y remotos.
1. Vaya a **Comandos > Sincronizar**.
2. Elija:
   - **Local**: Actualizar el directorio local para que coincida con el remoto.
   - **Remoto**: Actualizar el directorio remoto para que coincida con el local.
   - **Espejo**: Mantener ambos directorios idénticos.
3. Seleccione los directorios a comparar.
4. Previsualice los cambios y confirme para sincronizar.

## **Características Avanzadas**

### **Automatización y Scripting**
WinSCP admite automatización para tareas repetitivas:
- **Scripts por Lotes**:
  - Escriba scripts usando la interfaz de línea de comandos de WinSCP (ej., `winscp.com`).
  - Ejemplo: Automatizar copias de seguridad nocturnas.
    ```bash
    open sftp://usuario:contraseña@host
    get /ruta/remota/* C:\ruta\local\
    exit
    ```
  - Guarde como un archivo `.txt` y ejecute con `winscp.com /script=script.txt`.
- **Ensamblado .NET**:
  - Use el ensamblado .NET de WinSCP para integración con PowerShell o C#.
  - Ejemplo: Automatizar transferencias de archivos en un script de PowerShell.

### **Comandos Personalizados**
- Cree comandos personalizados para tareas repetitivas (ej., comprimir archivos en el servidor).
- Vaya a **Comandos > Comandos Personalizados > Agregar**.
- Ejemplo: Ejecute `tar -czf archivo.tar.gz !` para comprimir los archivos seleccionados.

### **Integración con PuTTY**
- Vincule WinSCP con PuTTY para acceso por terminal SSH:
  - En **Preferencias > Integración**, especifique la ruta a `putty.exe`.
  - Use **Abrir en PuTTY** desde la barra de herramientas para iniciar una sesión de terminal con las mismas credenciales.

### **Cifrado y Seguridad**
- **Claves SSH**:
  - Genere claves usando PuTTYgen o importe claves existentes.
  - Configure en **Avanzado > SSH > Autenticación**.
- **Cifrado**: Cifre archivos antes de la transferencia usando el cifrado incorporado de WinSCP.
- **Contraseña Maestra**: Proteja las contraseñas de sesión almacenadas en **Preferencias > Seguridad**.

## **Consejos y Mejores Prácticas**
- **Guardar Sesiones**: Almacene conexiones de uso frecuente para ahorrar tiempo.
- **Usar Modo Binario**: Para archivos que no son de texto para evitar corrupción.
- **Hacer Copia de Seguridad Antes de Eliminar**: Siempre verifique los archivos antes de eliminarlos en el servidor.
- **Verificar Permisos**: Asegure los permisos de archivo correctos para servidores web (ej., 644 para archivos, 755 para directorios).
- **Actualizar Regularmente**: Mantenga WinSCP actualizado para parches de seguridad y nuevas características.
- **Usar Filtros**: Excluya archivos innecesarios (ej., `.git`) durante las transferencias.

---

## **Solución de Problemas Comunes**
1. **Conexión Rechazada**:
   - Verifique el host, puerto y credenciales.
   - Compruebe si el firewall del servidor permite conexiones.
2. **Autenticación Fallida**:
   - Asegúrese de que el nombre de usuario/contraseña o la clave SSH sean correctos.
   - Verifique el formato de la clave (WinSCP admite el formato `.ppk` de PuTTY).
3. **Transferencias Lentas**:
   - Compruebe la estabilidad de la red.
   - Ajuste la configuración de transferencia (ej., deshabilite la compresión para transferencias más rápidas).
4. **Permiso Denegado**:
   - Confirme los permisos del usuario en el servidor.
   - Use **Propiedades** para ajustar los permisos de archivos/carpetas.
5. **Verificación de Clave de Host**:
   - Acepte la clave de host solo si confía en el servidor.
   - Contacte al administrador del servidor para verificar la huella digital de la clave.

## **Alternativas a WinSCP**
- **FileZilla**: Cliente FTP/SFTP de código abierto con una interfaz similar.

- **MobaXterm**: Herramienta todo en uno con capacidades de transferencia de archivos y terminal.
