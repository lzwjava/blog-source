### Instalación
Primero, instala los paquetes con estos comandos:

```bash
npm install --save-dev \
  "babel-core@6.13.0" \
  "babel-loader@6.2.5" \
  "babel-plugin-transform-runtime@6.12.0" \
  "babel-preset-es2015@6.13.2" \
  "babel-runtime@6.11.6"
```

### Configuración
Crea un archivo `.babelrc` en la raíz de tu proyecto con:

```json
{
  "presets": ["es2015"],
  "plugins": ["transform-runtime"]
}
```

Luego, configura tu `webpack.config.js` de la siguiente manera:

```javascript
module.exports = {
  entry: './index.js',
  output: {
    filename: 'bundle.js'
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'babel-loader',
        options: {
          configFile: './.babelrc'
        }
      }
    ]
  }
};
```

### Uso
Escribe tu código JavaScript ES2015 en archivos como `index.js`, luego ejecuta `npx webpack` para compilarlo en `bundle.js`, que los navegadores más antiguos pueden entender.

#### Detalles de Configuración
La configuración implica dos archivos principales: `.babelrc` para la configuración de Babel y `webpack.config.js` para la integración de webpack.

##### Archivo `.babelrc`
Crea o edita `.babelrc` en la raíz del proyecto con el siguiente contenido:

```json
{
  "presets": ["es2015"],
  "plugins": ["transform-runtime"]
}
```

- **Presets**: `"es2015"` corresponde a `babel-preset-es2015@6.13.2`, habilitando todas las transformaciones de ES2015 a ES5, como funciones de flecha, let/const y clases.
- **Plugins**: `"transform-runtime"` usa `babel-plugin-transform-runtime@6.12.0` para externalizar ayudantes de Babel, haciendo referencia a `babel-runtime@6.11.6`, lo que reduce el tamaño del código compilado evitando la duplicación de ayudantes entre archivos.

Esta configuración asegura que el código ES2015 se transpile y se optimice para la eficiencia en tiempo de ejecución.

##### Configuración de Webpack
Para webpack, crea o actualiza `webpack.config.js` con:

```javascript
module.exports = {
  entry: './index.js',
  output: {
    filename: 'bundle.js'
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'babel-loader',
        options: {
          configFile: './.babelrc'
        }
      }
    ]
  }
};
```

- **Entrada y Salida**: Especifica `index.js` como el punto de entrada y produce `bundle.js`.
- **Reglas de Módulo**: Usa `babel-loader@6.2.5` para archivos `.js`, apuntando a `.babelrc` para opciones, asegurando que Babel procese los archivos antes de agruparlos.

Esta configuración aprovecha el sistema de módulos de webpack para manejar archivos JavaScript, compilándolos a través de Babel.

#### Uso Alternativo: Babel CLI
Aunque la consulta sugiere una configuración de webpack, para completitud, considera usar Babel CLI para la compilación manual sin webpack. Primero, instala `babel-cli@6.13.0`:

```bash
npm install --save-dev "babel-cli@6.13.0"
```

Luego, compila archivos con:

```bash

```

```json
}
```

Ejecuta con `npm run build`, que produce archivos compilados en `lib/`, útil para proyectos de Node.js o código del lado del servidor.
