Jakarta EE es una plataforma de código abierto para desarrollar aplicaciones Java a nivel empresarial. Proporciona un conjunto de especificaciones, APIs y herramientas que extienden el núcleo de Java SE (Standard Edition) para manejar tareas complejas como servicios web, computación distribuida, despliegues cloud-native y aplicaciones escalables del lado del servidor.

### Historia Breve
- Originalmente conocido como Java 2 Platform, Enterprise Edition (J2EE) y posteriormente Java EE, fue desarrollado por Oracle/Sun Microsystems.
- En 2017, Oracle donó las tecnologías a la Eclipse Foundation, lo que llevó al cambio de marca como Jakarta EE (nombrado así por Yakarta, Indonesia, donde se celebró la primera conferencia Eclipse Java).
- Este cambio lo convirtió en completamente de código abierto y dirigido por la comunidad, fomentando una innovación más rápida y una adopción más amplia.

### Características Clave y Propósito
- **Componentes**: Incluye APIs para servlets, JPA (Java Persistence API) para acceso a datos, CDI (Contexts and Dependency Injection) para gestionar dependencias, JAX-RS para servicios web RESTful, y más.
- **Casos de Uso**: Ideal para construir software empresarial robusto, seguro y portable, como plataformas de comercio electrónico, sistemas financieros y arquitecturas de microservicios.
- **Estado Actual**: A partir de 2025, Jakarta EE 11 es la última versión principal, que hace hincapié en la compatibilidad cloud-native, el procesamiento JSON y una seguridad mejorada. Es compatible con servidores como Open Liberty, Payara y WildFly.
