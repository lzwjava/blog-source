Este esquema se basa en el plan de estudios estándar del Examen Nacional de Autodidactas de Educación Superior de China, específicamente adaptado para la programación introductoria en Java. Cubre los conceptos centrales, la estructura y el enfoque de evaluación para el curso en la provincia de Guangdong. El curso enfatiza las habilidades fundamentales de programación usando Java, incluyendo sintaxis, principios de orientación a objetos y desarrollo básico de aplicaciones.

## I. Naturaleza y Objetivos del Curso
- **Naturaleza**: Este es un curso de programación fundamental que se centra en la programación orientada a objetos (POO) usando Java. Java es un lenguaje independiente de la plataforma, seguro y versátil, ideal para el desarrollo de redes y aplicaciones.
- **Objetivos**:
  - Comprender los conceptos básicos de programación y las características de Java.
  - Dominar la sintaxis de Java, tipos de datos, estructuras de control y métodos.
  - Desarrollar habilidades para crear programas simples, manejar estructuras de datos como arrays e implementar principios de POO.
  - Preparar a los estudiantes para la resolución práctica de problemas mediante codificación y depuración.

## II. Objetivos de Evaluación
- **Conocimiento**: Comprender el entorno de desarrollo de Java, la sintaxis básica y las bibliotecas centrales.
- **Habilidades**: Escribir, compilar y ejecutar programas Java; diseñar algoritmos simples; usar POO para la organización del código.
- **Formato del Examen**: Libro cerrado, 100 puntos, 90 minutos. Incluye preguntas de opción múltiple, verdadero/falso, completar espacios, lectura de programas, respuestas cortas y preguntas de programación.
- **Ponderación**: Conocimiento teórico (40%), codificación práctica (60%).

## III. Contenido y Requisitos del Curso

### Primera Parte: Descripción General de Java
- Especificaciones del lenguaje Java, API, JDK y herramientas de IDE.
- Crear, compilar y ejecutar programas Java.
- Pautas de estilo de programación.
- Errores de programación comunes y conceptos básicos de depuración.

### Segunda Parte: Programación Básica
- Leer la entrada desde la consola.
- Identificadores, variables, expresiones, sentencias de asignación y constantes.
- Tipos de datos numéricos y operaciones; conversiones de tipo.
- Evaluación de expresiones y precedencia de operadores.
- Asignación aumentada, operadores de incremento/decremento.

### Tercera Parte: Estructuras de Selección
- Sentencias if-else y anidamiento.
- Sentencias switch.
- Expresiones condicionales.
- Reglas de precedencia y asociatividad de operadores.

### Cuarta Parte: Funciones Matemáticas, Caracteres y Cadenas
- Funciones matemáticas comunes (por ejemplo, la clase Math).
- Tipos de datos de caracteres y operaciones.
- Tipo String: creación, manipulación y métodos.
- Formateo de la salida de consola (por ejemplo, printf).

### Quinta Parte: Bucles
- Bucles while.
- Bucles do-while.
- Bucles for.
- Bucles anidados.
- Palabras clave break y continue.

### Sexta Parte: Métodos
- Definir y llamar métodos.
- Paso de parámetros por valor.
- Sobrecarga de métodos.
- Alcance y tiempo de vida de las variables.

### Séptima Parte: Arrays
- Fundamentos de arrays (declaración, inicialización).
- Pasar arrays a métodos.
- Devolver arrays desde métodos.
- Varargs (argumentos de longitud variable).
- Arrays multidimensionales.

### Octava Parte: Objetos y Clases
- Definir clases y crear objetos.
- Acceder a objetos mediante variables de referencia.
- Usar clases incorporadas (por ejemplo, Date, Random).
- Variables y métodos estáticos.
- Modificadores de visibilidad (public, private, etc.).
- Encapsulación de campos de datos.
- Pasar objetos como parámetros.
- Arrays de objetos.
- Alcance de variables de clase.
- La referencia 'this'.

### Novena Parte: Herencia y Polimorfismo
- Superclases y subclases.
- Palabra clave super.
- Sobrescritura de métodos.
- Clase Object y método toString().
- Polimorfismo y enlace dinámico.
- Conversión de objetos y operador instanceof.
- Acceso protected.
- Palabra clave final para prevenir herencia/sobrescritura.

### Décima Parte: Clases Abstractas e Interfaces
- Clases y métodos abstractos.
- Interfaces: definición e implementación.
- Interface Comparable (por ejemplo, para ordenar).
