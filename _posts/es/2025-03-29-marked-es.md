Para cambiar de usar Kramdown a Marked en tu blog de Jekyll, necesitarás realizar algunos cambios en la configuración y configuración de Jekyll. Marked es una librería JavaScript para analizar y compilar Markdown, y puede usarse junto con MathJax para una mejor representación de fórmulas matemáticas. Aquí tienes una guía paso a paso para hacer el cambio:

### Paso 1: Instalar Marked
Primero, necesitas incluir la librería Marked en tu proyecto de Jekyll. Puedes hacerlo agregándola a la sección `<head>` de tu archivo de diseño (por ejemplo, `default.html`).

```html
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>
```

### Paso 2: Deshabilitar Kramdown
En tu archivo `_config.yml`, deshabilita Kramdown estableciendo la opción `markdown` en `none` o simplemente elimínala si está configurada para Kramdown.

```yaml
markdown: none
```

### Paso 3: Configurar Marked con MathJax
Necesitas configurar Marked para que funcione con MathJax. Crea un archivo JavaScript (por ejemplo, `marked-mathjax.js`) en tu proyecto de Jekyll e incluye el siguiente código:

```javascript
document.addEventListener("DOMContentLoaded", function() {
  // Renderizar el contenido Markdown usando Marked
  const elements = document.querySelectorAll('.markdown-content');
  elements.forEach(element => {
    const html = marked(element.textContent);
    element.innerHTML = html;
  });

  // Renderizar MathJax
  MathJax.typesetPromise().then(() => {
    console.log('MathJax rendering complete!');
  }).catch((err) => console.log('Typeset failed:', err));
});
```

Incluye este script en tu archivo de diseño después de incluir Marked y MathJax:

```html
<script src="https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-mml-chtml.js"></script>
<script src="{{ '/assets/js/marked-mathjax.js' | relative_url }}"></script>
```

### Paso 4: Actualizar tu Contenido Markdown
Asegúrate de que tu contenido Markdown esté envuelto en un contenedor con la clase `markdown-content` para que el JavaScript pueda identificarlo y renderizarlo.

```html
<div class="markdown-content">
  <!-- Tu contenido Markdown aquí -->
</div>
```

### Paso 5: Probar tu Configuración
Construye y sirve tu sitio de Jekyll localmente para asegurarte de que Marked está renderizando correctamente tu Markdown y que MathJax está mostrando correctamente las expresiones matemáticas.

```bash
bundle exec jekyll serve
```

### Notas Adicionales
- Asegúrate de que tu configuración de MathJax sea compatible con tu contenido Markdown. Es posible que necesites ajustar la configuración según tus necesidades específicas.
- Si encuentras algún problema, revisa la consola del navegador en busca de errores y ajusta tu código JavaScript en consecuencia.

Siguiendo estos pasos, deberías poder cambiar de Kramdown a Marked en tu blog de Jekyll y aprovechar el mejor soporte de MathJax para expresiones matemáticas.
