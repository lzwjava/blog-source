## ¿Qué es PuTTY?

PuTTY es un emulador de terminal versátil y un cliente de red desarrollado por Simon Tatham, principalmente para Windows pero también disponible para Linux y macOS. Soporta múltiples protocolos, incluyendo:

- **SSH (Secure Shell)**: Para acceso remoto seguro a servidores.
- **Telnet**: Para conexiones no cifradas (menos común debido a problemas de seguridad).
- **Rlogin**: Para inicio de sesión remoto en sistemas Unix.
- **SCP y SFTP**: Para transferencias seguras de archivos.
- **Conexiones Raw socket**: Para comunicación directa de red.
- **Conexiones Serie**: Para interactuar con hardware como routers.

PuTTY es ligero, altamente configurable e incluye herramientas como PuTTYgen (para generar claves SSH) y PSFTP/PSCP (para transferencias de archivos). Es ampliamente utilizado por administradores de sistemas, desarrolladores y profesionales de TI para gestionar servidores remotos.[](https://en.wikipedia.org/wiki/PuTTY)

## Descargar e Instalar PuTTY

### Paso 1: Descargar PuTTY
1. Visita el sitio web oficial de PuTTY en [www.putty.org](https://www.putty.org/) o la página de descargas en [chiark.greenend.org.uk/~sgtatham/putty/latest.html](https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html).[](https://www.putty.org/)
2. Elige el instalador apropiado:
   - **Instalador MSI de 32 o 64 bits**: Selecciona `putty-<version>-installer.msi` según tu sistema (se recomienda 64-bit para PCs modernos).[](https://its.gmu.edu/knowledge-base/how-to-install-putty-ssh-for-windows/)
   - **Binarios individuales**: Descarga `putty.exe`, `puttygen.exe`, `psftp.exe`, etc., si prefieres no usar un instalador.
3. Si no estás seguro del tipo de tu sistema, la versión de 32 bits es universalmente compatible.[](https://www.ssh.com/academy/ssh/putty/windows/install)

### Paso 2: Instalar PuTTY
1. Haz doble clic en el archivo `.msi` descargado para iniciar el Asistente de Configuración de PuTTY.
2. Haz clic en **Siguiente** en la pantalla de Bienvenida.
3. Acepta la carpeta de destino predeterminada (normalmente `C:\Program Files\PuTTY\`) o elige una ubicación personalizada, luego haz clic en **Siguiente**.[](https://www.techtarget.com/searchsecurity/tutorial/How-to-use-PuTTY-for-SSH-key-based-authentication)
4. Selecciona las funciones deseadas (por ejemplo, crear un acceso directo en el escritorio) y haz clic en **Instalar**.[](https://www.ssh.com/academy/ssh/putty/windows/install)
5. Si lo solicita el Control de Cuentas de Usuario de Windows, haz clic en **Sí**.
6. Una vez completada la instalación, haz clic en **Finalizar**.[](https://www.ssh.com/academy/ssh/putty/windows/install)

### Paso 3: Verificar la Instalación
- Si creaste un acceso directo en el escritorio, haz doble clic en el icono de PuTTY.
- Alternativamente, busca "PuTTY" en el menú Inicio de Windows o navega a `C:\Program Files\PuTTY\putty.exe`.[](https://its.gmu.edu/knowledge-base/how-to-install-putty-ssh-for-windows/)

## Configuración Básica y Conexión a un Servidor

### Iniciar PuTTY
1. Abre PuTTY desde el menú Inicio o el acceso directo del escritorio. Esto abre la ventana **Configuración de PuTTY**, que tiene:
   - Un panel izquierdo para categorías de configuración.
   - Una sección central para ingresar detalles de conexión.
   - Una sección inferior derecha para guardar perfiles de sesión.[](https://www.ssh.com/academy/ssh/putty/windows)

### Conectarse a un Servidor
1. **Ingresar Detalles del Host**:
   - En el campo **Nombre de Host (o dirección IP)**, ingresa el nombre de dominio del servidor (por ejemplo, `students.example.edu`) o la dirección IP (por ejemplo, `192.168.1.1`).
   - Asegúrate de que el **Puerto** esté establecido en `22` para SSH (predeterminado) o el puerto especificado por tu administrador de servidor.[](https://www.wikihow.com/Use-Putty-in-Windows)
2. **Seleccionar Tipo de Conexión**:
   - Elige **SSH** (predeterminado) para conexiones seguras. Otras opciones incluyen Telnet, Rlogin, Raw o Serial.[](https://www.wikihow.com/Use-Putty-in-Windows)
3. **Guardar una Sesión** (opcional):
   - Ingresa un nombre en el campo **Sesiones Guardadas** (por ejemplo, `MiServidor`).
   - Haz clic en **Guardar** para almacenar la configuración para uso futuro.[](https://www.cuit.columbia.edu/putty)
4. **Conectar**:
   - Haz clic en **Abrir** para iniciar la conexión.
   - Si es la primera vez que te conectas, puede aparecer una **Alerta de Seguridad de PuTTY**, advirtiendo que la clave de host del servidor no está en caché. Haz clic en **Sí** para confiar en el servidor y proceder.[](https://www.ssh.com/academy/ssh/putty/windows)

### Iniciar Sesión
- Ingresa tu nombre de usuario y contraseña cuando se solicite en la ventana de terminal.
- Si se requiere autenticación basada en clave, configúrala como se describe en la sección [Usar PuTTYgen](#usar-puttygen-para-autenticación-con-clave-ssh).[](https://www.wikihow.com/Use-Putty-in-Windows)

## Opciones de Configuración Avanzada

La ventana de configuración de PuTTY ofrece opciones extensas para personalización. A continuación se presentan categorías y configuraciones clave:

### Sesión
- **Sesiones Guardadas**: Cargar, guardar o eliminar perfiles de sesión para acceso rápido.
- **Cerrar Ventana al Salir**: Elegir si el terminal se cierra automáticamente después de la desconexión (`Siempre`, `Nunca`, o `Solo en salida limpia`).[](https://www.ssh.com/academy/ssh/putty/windows)

### Terminal
- **Modo Auto Wrap**: Activar para ajustar líneas largas de texto a la siguiente línea. Si está desactivado, el texto que exceda el ancho de la ventana puede no mostrarse.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
- **Líneas de Historial**: Aumentar el número de líneas almacenadas en el búfer (por defecto es 200) para ver más salida del terminal.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
- **Keepalives**: Establecer un valor (en segundos) para enviar paquetes nulos y prevenir caídas de conexión por inactividad.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)

### Ventana
- **Apariencia**:
   - Cambiar la configuración de fuente (por ejemplo, tamaño, estilo) en **Ventana > Apariencia**.
   - Modificar colores en **Ventana > Colores** para mejorar la legibilidad (asegurar contraste suficiente entre primer plano y fondo).[](https://www.cuit.columbia.edu/putty)
- **Comportamiento al Redimensionar**: Ajustar cómo responde el terminal al redimensionar la ventana (por ejemplo, escalar el tamaño de fuente o ajustar las dimensiones del terminal).[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)

### Conexión
- **Configuración de Proxy**: Configurar proxies si te conectas a través de una red corporativa.
- **Auto-login**: Especificar un nombre de usuario para omitir la solicitud de inicio de sesión.[](https://www.puttygen.com/download-putty)
- **Configuración SSH**:
  - Seleccionar la versión preferida del protocolo SSH (se recomienda SSH-2 por seguridad).
  - Elegir cifrados de encriptación (por ejemplo, AES, 3DES) y algoritmos de intercambio de claves.[](https://en.wikipedia.org/wiki/PuTTY)

### Guardar y Cargar Sesiones
1. Después de configurar los ajustes, regresa a la categoría **Sesión**.
2. Ingresa un nombre en el campo **Sesiones Guardadas** y haz clic en **Guardar**.
3. Para cargar una sesión guardada, selecciónala de la lista, haz clic en **Cargar**, luego en **Abrir**.[](https://www.cuit.columbia.edu/putty)

## Usar PuTTYgen para Autenticación con Clave SSH

La autenticación basada en clave SSH es más segura que la basada en contraseña. PuTTYgen genera pares de claves en el formato `.ppk` de PuTTY.

### Paso 1: Generar un Par de Claves
1. Abre **PuTTYgen** desde el menú Inicio o `C:\Program Files\PuTTY\puttygen.exe`.
2. Selecciona un tipo de clave (por ejemplo, **RSA**, predeterminado, o **ECDSA**) y longitud en bits (por ejemplo, 2048 o 4096 para RSA).
3. Haz clic en **Generar** y mueve el mouse para crear aleatoriedad hasta que se complete la barra de progreso.
4. Opcionalmente, establece una **frase de contraseña** para mayor seguridad.
5. Guarda la clave privada (archivo `.ppk`) en una ubicación segura.
6. Copia la clave pública mostrada en el campo “Public key for pasting into OpenSSH authorized_keys file”.[](https://www.wikihow.com/Use-Putty-in-Windows)

### Paso 2: Configurar el Servidor
1. Conéctate al servidor remoto usando autenticación basada en contraseña (u otro método).
2. Crea o edita el archivo `~/.ssh/authorized_keys`:
   - Ejecuta `mkdir ~/.ssh && chmod 700 ~/.ssh` para crear el directorio `.ssh`.
   - Abre el archivo `authorized_keys` con un editor de texto (por ejemplo, `nano ~/.ssh/authorized_keys`).
   - Pega la clave pública y guarda el archivo.
   - Establece los permisos con `chmod 600 ~/.ssh/authorized_keys`.[](https://www.techtarget.com/searchsecurity/tutorial/How-to-use-PuTTY-for-SSH-key-based-authentication)
3. Cierra sesión en el servidor.

### Paso 3: Configurar PuTTY para Autenticación por Clave
1. Abre PuTTY y carga tu sesión guardada.
2. Navega a **Conexión > SSH > Auth > Credentials**.
3. Haz clic en **Examinar** junto a “Private key file for authentication” y selecciona tu archivo `.ppk`.
4. Regresa a la categoría **Sesión**, guarda la sesión actualizada y haz clic en **Abrir**.
5. Ingresa tu frase de contraseña (si se estableció) cuando se solicite. Ahora deberías conectarte sin contraseña.[](https://www.wikihow.com/Use-Putty-in-Windows)

## Transferencia de Archivos con PSFTP y PSCP

PuTTY incluye **PSFTP** (para SFTP) y **PSCP** (para SCP) para transferencias seguras de archivos.

### Usar PSFTP
1. Abre **PSFTP** desde el menú Inicio o `C:\Program Files\PuTTY\psftp.exe`.
2. Escribe `open <nombre_de_host>` (por ejemplo, `open students.example.edu`) y presiona Enter.
3. Inicia sesión con tus credenciales o clave privada.
4. Usa comandos como:
   - `ls` o `dir`: Listar archivos.
   - `get <nombre_de_archivo>`: Descargar un archivo.
   - `put <nombre_de_archivo>`: Subir un archivo.
   - `cd <directorio>`: Cambiar directorio.
   - `quit`: Salir de PSFTP.[](https://www.wikihow.com/Use-Putty-in-Windows)

### Usar PSCP
1. Abre un Símbolo del sistema o PowerShell.
2. Navega al directorio de instalación de PuTTY (por ejemplo, `cd C:\Program Files\PuTTY`).
3. Ejecuta comandos como:
   - `pscp <archivo_local> <usuario>@<nombre_de_host>:<ruta_remota>` para subir.
   - `pscp <usuario>@<nombre_de_host>:<archivo_remoto> <ruta_local>` para descargar.
   - Agrega `-i <clave_privada.ppk>` para autenticación basada en clave.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)

### Nota
- PSFTP es interactivo, mientras que PSCP se basa en línea de comandos.
- Para transferencias de archivos gráficas, considera alternativas como WinSCP, que puede importar sesiones de PuTTY.[](https://www.ssh.com/academy/ssh/putty)

## Reenvío de Puertos y Tunneling

PuTTY soporta **SSH tunneling** (reenvío de puertos) para acceder de forma segura a servicios en servidores remotos.

### Reenvío de Puerto Local
1. En PuTTY, ve a **Conexión > SSH > Tunnels**.
2. Ingresa un **Puerto de origen** (por ejemplo, `8080`) en tu máquina local.
3. Ingresa el **Destino** (por ejemplo, `remote_host:80`) para el servicio remoto.
4. Selecciona **Local** y haz clic en **Agregar**.
5. Guarda la sesión y conéctate.
6. Accede al servicio localmente (por ejemplo, `http://localhost:8080`).[](https://www.ssh.com/academy/ssh/putty/windows)

### Reenvío de Puerto Remoto
- Similar al reenvío local, pero selecciona **Remote** y especifica un puerto en el servidor remoto para reenviar a tu máquina local.

### Reenvío de Puerto Dinámico
- Selecciona **Dynamic** y especifica un puerto local (por ejemplo, `8080`). Esto crea un proxy SOCKS para enrutamiento dinámico (útil para navegar a través del servidor).[](https://rushax.com/what-is-putty-a-comprehensive-guide/)

## X11 Forwarding para Aplicaciones Gráficas

PuTTY soporta **X11 forwarding** para ejecutar aplicaciones gráficas en un servidor remoto y mostrarlas localmente.

1. Instala un servidor X en Windows (por ejemplo, **XMing**, una opción gratuita).
2. En PuTTY, ve a **Conexión > SSH > X11**.
3. Marca **Enable X11 forwarding**.
4. Establece **X display location** en `localhost:0.0`.
5. Guarda la sesión y conéctate.
6. Ejecuta comandos gráficos (por ejemplo, `xterm`) en el servidor, y la interfaz se mostrará localmente a través de XMing.[](https://www.ssh.com/academy/ssh/putty/windows)

## Solución de Problemas Comunes

1. **Error “Acceso Denegado”**:
   - Verifica el nombre de usuario, contraseña o clave privada.
   - Asegúrate de que el acceso SSH esté habilitado en el servidor (por ejemplo, revisa la configuración del panel de control de hosting).[](https://www.hostinger.com/uk/tutorials/how-to-use-putty-ssh)
   - Comprueba si el servidor requiere solo autenticación basada en clave.

2. **“Conexión Reiniciada por el Par”**:
   - Habilita **Keepalives** en la configuración de **Conexión** para prevenir tiempos de espera.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
   - Revisa la configuración del firewall o dispositivos de red que puedan cerrar conexiones inactivas.

3. **“Clave de Host del Servidor No Encontrada en Caché”**:
   - Esto es normal para las primeras conexiones. Haz clic en **Sí** para almacenar la clave en caché. Si aparece repetidamente, verifica la autenticidad del servidor para descartar un ataque de intermediario.[](https://www.ssh.com/academy/ssh/putty/windows)

4. **Advertencia de Intentos de Inicio de Sesión Fallidos**:
   - Restablece tu contraseña o busca protección contra fuerza bruta en el servidor.
   - Contacta a tu administrador de servidor si estás bloqueado.[](https://www.hostinger.com/uk/tutorials/how-to-use-putty-ssh)

5. **La Ventana de PuTTY se Cierra Inesperadamente**:
   - Revisa los registros del servidor en busca de errores (por ejemplo, permisos incorrectos en `~/.ssh/authorized_keys`).
   - Asegúrate de que la clave privada correcta esté cargada.[](https://www.hostinger.com/uk/tutorials/how-to-use-putty-ssh)

## Consejos y Mejores Prácticas

- **Guardar Sesiones**: Siempre guarda perfiles de sesión para evitar volver a ingresar configuraciones.[](https://www.cuit.columbia.edu/putty)
- **Usar Autenticación Basada en Clave**: Es más segura que las contraseñas y previene ataques de fuerza bruta.[](https://www.techtarget.com/searchsecurity/tutorial/How-to-use-PuTTY-for-SSH-key-based-authentication)
- **Proteger Claves Privadas**: Almacena archivos `.ppk` en una ubicación segura y usa una frase de contraseña.[](https://www.ssh.com/academy/ssh/putty)
- **Personalizar la Apariencia**: Ajusta fuentes y colores para una mejor legibilidad, especialmente para sesiones largas.[](https://www.cuit.columbia.edu/putty)
- **Habilitar Keepalives**: Previene caídas de conexión durante períodos de inactividad.[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
- **Respaldar Configuraciones**: Exporta las sesiones guardadas del Registro de Windows (`HKEY_CURRENT_USER\Software\SimonTatham\PuTTY\Sessions`) para respaldo o migración.
- **Actualizar Regularmente**: Descarga la última versión de PuTTY para asegurar parches de seguridad y nuevas funciones (por ejemplo, criptografía de curva elíptica en versiones recientes).[](https://www.ssh.com/academy/ssh/putty)

## Alternativas a PuTTY

Si bien PuTTY es confiable, las alternativas modernas ofrecen funciones mejoradas:
- **OpenSSH**: Integrado en Windows 10/11, Linux y macOS. Basado en línea de comandos pero robusto.[](https://phoenixnap.com/kb/putty-alternative)
- **WinSCP**: Un cliente gráfico de transferencia de archivos que se integra con PuTTY.[](https://www.ssh.com/academy/ssh/putty)
- **MobaXterm**: Ofrece una interfaz con pestañas, servidor X integrado y soporte SFTP.[](https://rushax.com/what-is-putty-a-comprehensive-guide/)
- **Bitvise SSH Client**: Soporta GUI, scripting y tunneling con un clic.[](https://phoenixnap.com/kb/putty-alternative)
- **Termius**: Multiplataforma con gestión de claves SSH y una interfaz moderna.[](https://phoenixnap.com/kb/putty-alternative)
