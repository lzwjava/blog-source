Gmail puede mejorarse enormemente en la era de la IA. Envié algunos archivos epub al servicio Send-to-Kindle, y reportó un error de conversión de documentos. Estas acciones son desencadenadas por el usuario y deberían categorizarse como primarias para mí. Sin embargo, este tipo de filtros deberían ser diferentes para cada usuario. Sería mejor aprender del comportamiento de los usuarios, como si abren, interactúan o activan estas acciones. La información está destinada a ser leída o revisada. Además, es probable que una aplicación de correo impulsada por RAG o LLM se vuelva popular en el futuro.

En una publicación anterior, escribí: "A veces, después de enviar documentos a Kindle por correo electrónico, reporta un problema con el documento enviado. Un problema es que el documento carece de título, y para solucionarlo, necesitamos pasar el parámetro --metadata". Sin embargo, la condición no es realmente "a veces". Sin importar cuándo, si el documento carece de título, el servicio actual de send-to-Kindle reportará este error. Podemos usar "a veces" si el error está relacionado con el tiempo o es altamente relevante para el tiempo.
