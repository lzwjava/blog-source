NPM（Node Package Manager）提供了多个日志级别，用于控制运行 npm 命令时输出的详细程度。以下是各级别说明，按详细程度从低到高排列：

- **silent**：不显示任何输出
- **error**：仅显示错误信息（默认级别）
- **warn**：显示错误和警告信息
- **notice**：显示错误、警告和通知信息
- **http**：显示 HTTP 请求信息
- **timing**：显示时间统计信息
- **info**：显示常规提示信息
- **verbose**：显示详细信息
- **silly**：显示所有调试信息

您可以通过 `--loglevel` 参数或 `npm_config_loglevel` 环境变量设置日志级别：

```bash
npm install --loglevel verbose
```

也可以在 npm 配置中永久设置：

```bash
npm config set loglevel verbose
```

大多数命令的默认日志级别为 "error"，这意味着除非特别指定更详细的输出级别，否则您只会看到错误信息。
