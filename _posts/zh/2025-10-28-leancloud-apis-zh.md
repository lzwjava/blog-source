不，LeanCloud 并未将其公开 API 命名为 "Uluru API"。他们的官方 API 被称作 LeanCloud REST API（用于数据存储、查询等）、云引擎 REST API（用于服务端函数和钩子），以及类似推送通知或文件存储等特定服务的接口。这些都在其开发者网站 (docs.leancloud.cn) 上有详细文档，且从未提及 "Uluru"。

不过，"Uluru" 确实曾作为其产品或 SDK 开发的内部或早期代号出现。这基于一个名为 **uluru-js-sdk** 的归档 npm 包（版本 0.1.0，约 12 年前由 "avos" 发布者账户发布）。AVOS Cloud 是 LeanCloud 在品牌重塑前的原名，该包被描述为 "用于 PAAS（平台即服务）的 JavaScript SDK"，这与 LeanCloud 的后端服务定位相符。它支持浏览器和 Node.js 环境的核心功能（如数据操作），与 LeanCloud 早期提供的服务相匹配，但从未作为主推 SDK 公开宣传。当前的 SDK 均以 "LeanCloud SDK" 为品牌（例如 Java、Python、JavaScript 等版本），并在 GitHub 的 leancloud/ 组织下提供。

若此处指的是更近期或更具体的功能（例如私有/内部工具），则未见公开文档记载。如需官方 API 详情，请查阅 [LeanCloud 文档](https://docs.leancloud.cn/en/)。
