# lzwjava.github.io

欢迎来到我的个人博客！这里，我分享我的思考、经历和各种话题的知识。

## 主要改进

与标准 Jekyll 博客使用的 jekyll-theme-cayman 相比，这个博客包含了以下几个改进：

*   **AI 翻译：** 利用高级语言模型进行准确和上下文相关的翻译，扩大内容的可及性，面向全球受众。
*   **XeLaTeX PDF 生成：** 集成 XeLaTeX 生成高质量、适合打印阅读和分享的 PDF。
*   **Google Cloud 文本转语音：** 利用 Google Cloud 的文本转语音服务生成文章的音频版本，提升视障用户和喜欢音频内容的用户的可访问性。
*   **增强的 CSS 样式：** 提供精细且定制的 CSS 设计，提供视觉上吸引人且用户友好的体验。
*   **MathJax 支持：** 实现 MathJax 以渲染复杂的数学表达式和方程，使技术内容更具可访问性。
*   **夜间模式：** 包含夜间模式选项，减少眼疲劳，提高低光条件下的可读性。
*   **灵活的帖子选择：** 提供多种帖子选择选项，如按类别或标签过滤，以增强导航。
*   **定期更新：** 确保博客的库和依赖项是最新的，以获得最佳性能和安全性。
*   `awesome-cv` 集成：** 使用 `awesome-cv` 直接从博客生成专业简历。
*   **RSS 订阅支持：** 通过 `feed.xml` 提供 RSS 订阅，让用户订阅博客。
*   **双语内容：** 支持中文和英文内容以迎合多样化的受众。
*   **GitHub 工作流自动化：** 实施 GitHub Actions 进行自动构建、测试和部署，确保流畅的开发流程。
*   **自动翻译工作流：** 使用 GitHub Actions 自动将新的或更新后的帖子翻译成多种语言。
*   **EPUB 支持：** 将 Markdown 转换为 EPUB 以供电子书阅读器使用。

## 快速开始

要设置本地 Jekyll 环境，请按照以下步骤进行：

```shell
gem install jekyll bundler

jekyll new myblog

cd myblog

bundle install

bundle exec jekyll serve

bundle exec jekyll serve --draft

bundle add webrick

jekyll serve
```

## Markdown 样式

要定制语法高亮，请使用 `rougify`：

```shell
rougify help style

rougify style github > _sass/syntax.css

rougify style gruvbox.dark > _sass/syntax.css

rougify style base16.monokai.dark > _sass/syntax.css
```

## 音频和 PDF 流水线

要生成音频和 PDF 文件，请使用以下命令：

```bash
conda activate google-cloud-env

python scripts/audio_pipeline.py --task posts --n 10

python scripts/audio_pipeline.py --task pages

python scripts/audio_pipeline.py --task notes

python scripts/audio_pipeline.py --task notes --n 100

python scripts/pdf_pipeline.py --task pages

python scripts/pdf_pipeline.py --task posts --n 10

python scripts/pdf_pipeline.py --task notes

python scripts/generate_notes_link.py

python scripts/auto-ss-config/upload_configs.py

python scripts/epub_book.py notes

python scripts/epub_book.py _posts/en

python scripts/epub_book.py _posts/ja
```

## 脚本

要创建新的草稿帖子，请使用：

```shell
python scripts/draft.py <name>
```

## 内容指南

*   `-en`：表示英文文章。
*   `-zh`：表示中文文章。

`_posts` 和 `pages` 目录支持 `-en` 和 `-zh` 后缀。

`notes` 目录主要包含英文内容，用于个人学习目的，不需要翻译。
