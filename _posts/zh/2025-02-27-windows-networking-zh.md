下面介绍一些实用的 Windows 网络命令，你可以在命令提示符（CMD）中运行它们来排查故障、收集信息或管理网络。我会解释几个常用命令（比如 `getmac`），并补充其他可能派上用场的命令。你无需精通技术——这些命令一旦上手就会变得非常简单。

首先，要使用这些命令，请打开命令提示符：按 `Windows 键 + R`，输入 `cmd`，然后按回车键。很简单对吧？

### 1. `getmac` - 查找 MAC 地址
`getmac` 命令可以显示网络适配器的媒体访问控制（MAC）地址——这些是设备在网络中的唯一标识符。使用方法如下：

- 输入 `getmac` 后按回车。
- 你会看到每个网络适配器（如以太网或 Wi-Fi）的 MAC 地址列表。
- 添加 `-v` 参数（`getmac -v`）可启用详细模式，显示适配器名称和传输类型（如以太网或无线）等额外信息。

示例输出可能如下：
```
物理地址          传输名称
=================== ==========================================================
00-14-22-01-23-45   \Device\Tcpip_{12345678-ABCD-1234-EF56-7890ABCDEF12}
```
其中“物理地址”就是你的 MAC 地址。这个命令在排查网络问题或在路由器上设置 MAC 过滤时非常有用。

### 2. `ipconfig` - 检查 IP 配置
这是查看网络信息的首选命令：
- 输入 `ipconfig` 并按回车，可查看 IP 地址、子网掩码和默认网关等基本信息。
- 使用 `ipconfig /all` 可查看完整详情，包括 DNS 服务器、DHCP 状态，以及——没错——你的 MAC 地址。

这个命令非常适合判断设备是否正常连接或是否存在 IP 冲突。

### 3. `ping` - 测试网络连通性
想检查是否能连接到其他设备或网站？
- 输入 `ping [地址]`（例如 `ping google.com` 或 `ping 8.8.8.8`）。
- 它会发送几个数据包并告诉你是否收到回复，以及往返耗时（以毫秒为单位）。

如果显示“请求超时”，说明存在连接阻塞——可能是防火墙、服务器宕机或自身网络问题。

### 4. `tracert` - 追踪路由路径
这是“跟踪路由”的缩写，可以显示数据到达目的地的路径：
- 输入 `tracert [地址]`（例如 `tracert google.com`）。
- 它会列出途径的每一跳（路由器或服务器）及其响应时间。

这个命令便于定位连接速度慢或中断的具体环节。

### 5. `netstat` - 查看网络活动
这个命令用于实时检查网络状态：
- 输入 `netstat -a` 查看所有活动连接和监听端口。
- 添加 `-n` 参数（`netstat -an`）可显示 IP 地址而非主机名——速度更快且更原始。

它就像一扇窗口，让你看到计算机正在与哪些对象通信——有助于发现可疑活动。

### 6. `nslookup` - 解析 DNS 记录
如果网站无法加载，这个命令可以检查域名系统（DNS）解析：
- 输入 `nslookup [域名]`（例如 `nslookup google.com`）。
- 它会返回该域名对应的 IP 地址，从而判断 DNS 是否正常工作。

### 实用技巧
- 如果命令需要更高权限，请以管理员身份运行 CMD（右键点击命令提示符 >“以管理员身份运行”）。
- 使用 `|`（管道符）组合命令进行筛选——例如 `getmac | find "00-14"` 可搜索特定 MAC 地址。
- 如果遇到困难，输入 `[命令] /?`（例如 `ipconfig /?`）可查看该命令的帮助信息。
