我正在尝试使用 IPv6 连接到 Hetzner Cloud 中的一台机器。`ssh 2a01:4f8:c17:2000::/64` 不起作用，但 `ssh root@2a01:4f8:c17:2000::1` 可以。

IPv6 地址是从 Hetzner Cloud 控制台复制的。

可以配置 `~/.ssh/config` 文件以对 IPv4 和 IPv6 地址应用不同的代理规则。此设置允许您为 IPv4 地址指定代理命令，同时以不同的方式处理 IPv6 地址。

```bash
Host 192.168.1.*
    UseKeychain yes
    AddKeysToAgent yes
    IdentityFile ~/.ssh/id_rsa 
Host *.*.*.*
    UseKeychain yes
    AddKeysToAgent yes
    IdentityFile ~/.ssh/id_rsa
    ProxyCommand corkscrew localhost 7890 %h %p
Host *
    UseKeychain yes
    AddKeysToAgent yes
    IdentityFile ~/.ssh/id_rsa   
```

运行 `ssh root@192.168.1.3` 时，以下输出显示 SSH 客户端正在应用 `~/.ssh/config` 文件中的配置选项：

SSH 连接速度明显很慢，所以我恢复到以下更简单的配置：

```bash
Host 192.168.1.*
    UseKeychain yes
    AddKeysToAgent yes
    IdentityFile ~/.ssh/id_rsa 
Host *
    UseKeychain yes
    AddKeysToAgent yes
    IdentityFile ~/.ssh/id_rsa  
    ProxyCommand corkscrew localhost 7890 %h %p 
```

当使用 `ProxyCommand corkscrew localhost 7890 %h %p` 指令与 IPv6 地址一起使用时，问题就出现了，因为此代理命令可能无法正确处理 IPv6 地址。
