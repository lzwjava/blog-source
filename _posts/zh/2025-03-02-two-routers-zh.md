要将两台 TP-Link AX3000 (TL-XDR 3050) 路由器与调制解调器配置为 **调制解调器 → 电缆 → 路由器1 → 无线 → 路由器2** 结构，请按照以下步骤进行操作。您的调制解调器已经具有 WiFi 功能和 IP 地址 `192.168.1.1`。我们将配置路由器1 作为主路由器，通过电缆连接到调制解调器，并将路由器2 无线连接到路由器1 以扩展网络。

以下是实现此配置的方法：

---

### **步骤 1：了解配置**
- **调制解调器**：提供互联网访问并具有自己的 WiFi（IP：`192.168.1.1`）。
- **路由器1**：将通过电缆连接到调制解调器，并作为网络的主路由器。
- **路由器2**：将无线连接到路由器1 以扩展网络覆盖范围。

您提到的几种模式（有线 AP 桥接、无线 AP 桥接、DHCP、宽带连接）。我们将为路由器1 使用 **DHCP** 从调制解调器获取互联网连接，并为路由器2 使用 **无线 AP 桥接**（或类似模式如 WDS/Repeater）无线连接到路由器1。

---

### **步骤 2：设置路由器1**
1. **将路由器1 连接到调制解调器**：
   - 取一根以太网电缆，将一端插入调制解调器的 **LAN 端口**。
   - 将另一端插入路由器1 的 **WAN（互联网）端口**。

2. **访问路由器1 的 Web 接口**：
   - 将计算机或智能手机连接到路由器1 的默认 WiFi 网络（检查路由器上的标签以获取默认 SSID 和密码）或使用以太网电缆。
   - 打开浏览器并输入 `http://tplinkwifi.net` 或 `192.168.0.1`（TP-Link 路由器的默认 IP）。
   - 使用默认凭据登录（通常是 `admin` 为用户名和密码），除非您已更改。

3. **配置路由器1**：
   - **互联网连接**：
     - 转到 **快速设置** 或 **互联网** 设置部分。
     - 选择 **DHCP** 模式。这允许路由器1 自动从调制解调器获取 IP 地址（可能在 `192.168.1.x` 范围内）。
   - **WiFi 设置**：
     - 为路由器1 的 WiFi 设置唯一的 **SSID**（网络名称）和强 **密码**。
     - 保存这些详细信息，路由器2 将需要它们无线连接。
   - **LAN 设置**：
     - 确保路由器1 的 LAN IP 与调制解调器的 IP 不同。默认情况下，它可能是 `192.168.0.1`，这很好，因为调制解调器是 `192.168.1.1`。
     - 确认路由器1 上启用了 **DHCP**。这允许路由器1 为连接到它的设备（包括路由器2）分配 IP 地址（例如 `192.168.0.x`）。
   - 保存设置并重启路由器1（如果提示）。

---

### **步骤 3：将路由器2 设置为无线桥接**
1. **访问路由器2 的 Web 接口**：
   - 将计算机或智能手机连接到路由器2 的默认 WiFi 网络或通过以太网。
   - 打开浏览器并输入 `http://tplinkwifi.net` 或 `192.168.0.1`。
   - 使用默认凭据（或您的自定义凭据）登录。

2. **将路由器2 配置为无线桥接模式**：
   - 在设置中查找类似 **无线 AP 桥接**、**WDS** 或 **Repeater** 的模式（可能在 **操作模式** 或 **无线** 设置下）。
   - 选择 **无线 AP 桥接**（或如果可用，WDS/Repeater）。
   - **连接到路由器1 的 WiFi**：
     - 扫描可用网络并选择路由器1 的 SSID。
     - 输入路由器1 的 WiFi 密码。
     - 确保路由器2 使用与路由器1 相同的无线信道以确保兼容性（例如，如果路由器1 在信道 6，则将路由器2 设置为信道 6）。
   - **LAN IP 设置**：
     - 更改路由器2 的 LAN IP 以避免与路由器1 发生冲突。例如，将其设置为 `192.168.0.2`（因为路由器1 可能是 `192.168.0.1`）。
     - **在路由器2 上禁用 DHCP**。路由器1 将为所有设备分配 IP 地址。
   - 保存设置并重启路由器2。它现在应该无线连接到路由器1。

---

### **步骤 4：测试配置**
1. **验证路由器2 的连接**：
   - 重启后，检查路由器2 的界面以确认它已连接到路由器1 的 WiFi。
2. **将设备连接到路由器2**：
   - 使用智能手机、笔记本电脑或其他设备连接到路由器2 的 WiFi（它可能使用与路由器1 相同的 SSID，具体取决于模式）。
   - 检查设备是否从路由器1 获取 IP 地址（例如 `192.168.0.x`）。
   - 通过浏览网站测试互联网访问。

---

### **最终配置摘要**
- **调制解调器**：IP `192.168.1.1`，提供互联网和 WiFi。
- **路由器1**：
  - WAN：通过电缆连接到调制解调器，设置为 **DHCP**（从调制解调器获取 IP，例如 `192.168.1.x`）。
  - LAN：IP `192.168.0.1`，**DHCP 启用**以为设备分配 IP。
  - WiFi：自定义 SSID 和密码。
- **路由器2**：
  - 模式：**无线 AP 桥接**（或 WDS/Repeater），无线连接到路由器1 的 WiFi。
  - LAN IP：`192.168.0.2`，**DHCP 禁用**。

---

### **故障排除技巧**
- **路由器2 无法无线连接**：
  - 双重检查路由器1 的 SSID、密码和信道设置在路由器2 的配置中。
  - 如果 **无线 AP 桥接** 无法使用，请尝试 **WDS** 或 **Repeater** 模式。参考 TP-Link AX3000 手册以获取支持的模式。
- **路由器2 无互联网**：
  - 确保路由器1 的 DHCP 正常工作，路由器2 的 DHCP 关闭。
  - 验证路由器2 是否成功桥接到路由器1。
- **双重 NAT 问题**：
  - 由于您的调制解调器具有 WiFi，并且路由器1 作为路由器，您可能会遇到双重 NAT（调制解调器和路由器1 都分配 IP）。这对于基本使用通常是可以的，但可能会影响某些应用程序（例如游戏）。如果需要，将调制解调器设置为 **桥接模式**（参考调制解调器的手册），但这可能会禁用其 WiFi。

通过此配置，路由器2 将无线扩展路由器1 的网络，并且连接到任何路由器的设备都将通过调制解调器访问互联网。如果需要进一步帮助，请告诉我！

如果您决定为第一个路由器（路由器1）使用固定（静态）IP 地址而不是依赖 DHCP，您需要手动配置其 WAN 设置以与调制解调器通信。以下是您应该为 IP 地址、DNS、子网掩码和网关配置的内容：

---

### **假设**
- 您的调制解调器的 IP 地址为 `192.168.1.1`，使用子网 `192.168.1.0/24`（子网掩码 `255.255.255.0`）。
- 调制解调器具有 DHCP 范围（例如 `192.168.1.2` 到 `192.168.1.100`），为连接的设备分配 IP 地址。

如果您的调制解调器使用不同的 IP 或子网，请相应调整值。

---

### **路由器1 的 WAN 设置**
这些设置配置了路由器1 如何连接到调制解调器：

1. **IP 地址**
   - 在调制解调器的子网（`192.168.1.0/24`）中选择一个静态 IP，但 **在调制解调器的 DHCP 范围之外** 以避免冲突。
   - 示例：如果调制解调器的 DHCP 范围是 `192.168.1.2` 到 `192.168.1.100`，您可以使用 **`192.168.1.101`**。
   - 这确保路由器1 在调制解调器的网络上具有唯一的固定地址。

2. **子网掩码**
   - 将其设置为与调制解调器的子网掩码匹配，通常是 **`255.255.255.0`**。
   - 这定义了网络范围为 `192.168.1.0` 到 `192.168.1.255`。

3. **默认网关**
   - 将其设置为调制解调器的 IP 地址，即 **`192.168.1.1`**。
   - 网关是路由器1 使用的设备（您的调制解调器）以访问互联网。

4. **DNS 服务器**
   - 您有两个选项：
     - 使用调制解调器的 IP： **`192.168.1.1`**。这让调制解调器处理 DNS 请求。
     - 使用公共 DNS 服务器： **`8.8.8.8`**（Google）和 **`8.8.4.4`**（Google 备用）。这些服务器可靠且通常更快。
   - 建议：从 `192.168.1.1` 开始。如果遇到 DNS 问题，切换到 `8.8.8.8` 和 `8.8.4.4`。

---

### **示例配置**
以下是路由器1 WAN 设置的示例：

- **IP 地址**：`192.168.1.101`
- **子网掩码**：`255.255.255.0`
- **默认网关**：`192.168.1.1`
- **DNS 服务器**：`192.168.1.1`（或 `8.8.8.8` 和 `8.8.4.4`）

---

### **附加说明**
- **检查调制解调器设置**：登录到调制解调器（通常在 `192.168.1.1`）以确认其 IP、子网掩码和 DHCP 范围。根据需要调整路由器1 的静态 IP。
- **路由器1 的 LAN 设置**：为了避免冲突，将路由器1 的 LAN 网络配置为不同的子网，例如 `192.168.0.1`，子网掩码为 `255.255.255.0`。在路由器1 上启用 DHCP 以为连接到它的设备分配 IP（例如 `192.168.0.2` 到 `192.168.0.254`）。
- **测试**：应用这些设置后，确保路由器1 可以访问互联网。如果不能，双重检查 IP（无冲突）、网关和 DNS 值。

此配置为路由器1 在调制解调器的网络上提供了固定 IP，确保一致的连接。如果需要帮助进一步配置路由器或特定调制解调器详细信息，请告诉我！

听起来您已经将路由器1 配置为静态 WAN IP，并为其 LAN 配置了 DHCP，但您的笔记本电脑仍然无法连接到网络。让我们逐步排查问题，找出原因并解决。

---

### 步骤 1：验证路由器1 的 WAN 连接
您的 WAN 设置是：

- **IP 地址**：`192.168.1.101`
- **子网掩码**：`255.255.255.0`
- **默认网关**：`192.168.1.1`
- **DNS 服务器**：`192.168.1.1`（或 `8.8.8.8` 和 `8.8.4.4`）

这些设置表明路由器1 连接到 IP 为 `192.168.1.1` 的调制解调器或另一个路由器。以下是要检查的内容：

- **路由器1 的互联网访问**：
  - 登录到路由器1 的 Web 接口（可能在 `192.168.0.1`，根据您的 LAN 设置）。
  - 转到 **状态** 或 **WAN** 部分，查看路由器1 是否显示有效的互联网连接。如果没有，问题可能出在路由器1 和调制解调器之间。

- **IP 冲突**：
  - 确保调制解调器网络上没有其他设备使用 `192.168.1.101`。如果调制解调器的 DHCP 范围包括此 IP，可能会导致冲突。登录到调制解调器（在 `192.168.1.1`）并检查其 DHCP 设置或连接的设备列表。如果 `192.168.1.101` 在调制解调器的 DHCP 范围内，请将路由器1 的 WAN IP 更改为该范围之外的值（例如 `192.168.1.200`）或从调制解调器的 DHCP 池中排除它。

- **调制解调器连接性**：
  - 确认以太网电缆插入调制解调器的 **LAN 端口** 和路由器1 的 **WAN 端口**。如果未正确设置，路由器1 将无法连接到互联网。

---

### 步骤 2：检查路由器1 的 LAN 和 DHCP 设置
您的 LAN 和 DHCP 设置是：

- **LAN IP**：`192.168.0.1`
- **子网掩码**：`255.255.255.0`
- **DHCP 启用**：是
- **IP 地址范围**：`192.168.0.2` 到 `192.168.0.254`
- **网关**：`192.168.0.1`
- **DNS 服务器**：`192.168.0.1`

这些看起来很合理，但让我们确保它们正常工作：

- **DHCP 功能**：
  - 启用 DHCP 后，您的笔记本电脑应该自动获取 `192.168.0.2` 到 `192.168.0.254` 范围内的 IP 地址，网关为 `192.168.0.1`。如果没有，DHCP 可能未正常工作。

- **DNS 配置**：
  - 将 DNS 服务器设置为 `192.168.0.1` 表示路由器1 处理笔记本电脑的 DNS 请求。确保路由器1 配置为将这些请求转发到上游 DNS 服务器（例如 `192.168.1.1` 或 `8.8.8.8`）。这通常是自动的，但请在路由器1 的设置中双重检查。或者，您可以直接将 DHCP DNS 设置为 `8.8.8.8` 和 `8.8.4.4`，以绕过路由器1 进行 DNS。

---

### 步骤 3：测试笔记本电脑的连接
由于您的笔记本电脑无法连接，让我们进行诊断：

- **连接类型**：
  - 您是使用 WiFi 还是以太网？如果是 WiFi，请确保连接到路由器1 的 SSID（而不是调制解调器的）。如果是以太网，请确认电缆插入路由器1 的 LAN 端口之一。

- **检查笔记本电脑的 IP 地址**：
  - 在笔记本电脑上打开 **命令提示符**（Windows）或 **终端**（macOS/Linux）：
    - Windows：输入 `ipconfig` 并按 Enter。
    - macOS/Linux：输入 `ifconfig` 或 `ip addr` 并按 Enter。
  - 查找网络适配器的 IP 地址。它应该是类似 `192.168.0.x`（其中 `x` 在 2 到 254 之间），子网掩码为 `255.255.255.0`，网关为 `192.168.0.1`。
  - **如果没有 IP 地址**：
    - 您的笔记本电脑可能未从 DHCP 获取 IP。确保其网络设置设置为“自动获取 IP 地址”。尝试刷新 IP：
      - Windows：运行 `ipconfig /release` 然后 `ipconfig /renew`。
    - 如果仍然失败，DHCP 可能在路由器1 或连接到它的问题。

- **Ping 路由器1**：
  - 从笔记本电脑运行 `ping 192.168.0.1`。如果收到回复，笔记本电脑已连接到路由器1 但可能没有互联网访问。如果超时，笔记本电脑未到达路由器1。

---

### 步骤 4：常见问题及解决方法
以下是可能的原因：

- **电缆问题**：
  - 如果使用以太网，请尝试不同的电缆或路由器1 的 LAN 端口。故障电缆或端口可能会阻止连接。

- **WiFi 问题**：
  - 如果在 WiFi 上，请双重检查密码并确保路由器1 的 WiFi 已启用。信号干扰或配置错误的 SSID 也可能是问题。

- **路由器1-调制解调器链接**：
  - 如果路由器1 未获取互联网连接，调制解调器可能需要特定设置（例如 MAC 地址克隆）。检查调制解调器的手册或设置以获取此信息。

- **双重 NAT**：
  - 您的配置具有调制解调器（在 `192.168.1.1`）和路由器1（在 `192.168.0.1`）都作为路由器。这通常不会阻止基本连接，但将调制解调器设置为 **桥接模式**（如果可能）可以简化事情。

- **防火墙**：
  - 路由器1 的防火墙可能阻止您的笔记本电脑。暂时禁用任何自定义防火墙规则进行测试。

---

### 步骤 5：如果需要，重置
如果没有效果，重置路由器1 并重新配置：

- **出厂重置**：
  - 按住路由器1 的重置按钮 10 秒，然后重新设置 WAN 和 DHCP 设置。

- **尝试动态 WAN**：
  - 而不是静态 WAN IP，将路由器1 的 WAN 设置为 **DHCP**，以查看它是否可以从调制解调器自动获取 IP。如果成功，静态 IP（`192.168.1.101`）可能是问题。

---

### 最终建议
- **调制解调器 DHCP 范围**：确认 `192.168.1.101` 在调制解调器的 DHCP 范围之外以避免冲突。
- **固件**：检查路由器1 的 Web 接口以获取固件更新。
- **笔记本电脑驱动程序**：确保笔记本电脑的网络适配器驱动程序是最新的。
