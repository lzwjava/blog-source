要将两台 TP-Link AX3000 (TL-XDR 3050) 路由器与您的调制解调器按 **调制解调器 → 网线 → 路由器1 → 无线 → 路由器2** 的配置进行设置，请按照以下分步指南操作。您的调制解调器已具备 WiFi 功能，IP 地址为 `192.168.1.1`。我们将把路由器1配置为通过网线连接到调制解调器的主路由器，并配置路由器2以无线方式扩展网络。

具体操作如下：

---

### **步骤 1：了解设置**
- **调制解调器**：提供互联网接入，并拥有自己的 WiFi（IP：`192.168.1.1`）。
- **路由器1**：将通过网线连接到调制解调器，并作为您网络的主路由器。
- **路由器2**：将通过无线方式连接到路由器1，以扩展网络覆盖范围。

您提到了几种模式（有线AP桥接、无线AP桥接、DHCP、宽带连接）。我们将对路由器1使用 **DHCP** 模式以从调制解调器获取互联网连接，并对路由器2使用 **无线AP桥接**（或类似的模式，如 WDS/中继器）以无线方式连接到路由器1。

---

### **步骤 2：设置路由器1**
1. **将路由器1连接到调制解调器**：
   - 取一根以太网电缆，一端插入调制解调器的 **LAN 端口**。
   - 另一端插入路由器1的 **WAN（互联网）端口**。

2. **访问路由器1的 Web 界面**：
   - 将计算机或智能手机连接到路由器1的默认 WiFi 网络（查看路由器标签上的默认 SSID 和密码）或使用以太网电缆连接。
   - 打开 Web 浏览器，输入 `http://tplinkwifi.net` 或 `192.168.0.1`（TP-Link 路由器的默认 IP）。
   - 使用默认凭据登录（通常用户名和密码均为 `admin`），除非您已更改。

3. **配置路由器1**：
   - **互联网连接**：
     - 转到 **快速设置** 或 **互联网** 设置部分。
     - 选择 **DHCP** 模式。这允许路由器1自动从调制解调器获取 IP 地址（可能在 `192.168.1.x` 范围内）。
   - **WiFi 设置**：
     - 为路由器1的 WiFi 设置一个唯一的 **SSID**（网络名称）和一个强 **密码**。
     - 保存这些详细信息，因为路由器2需要它们来进行无线连接。
   - **LAN 设置**：
     - 确保路由器1的 LAN IP 与调制解调器的 IP 不同。默认情况下，它可能是 `192.168.0.1`，这没问题，因为调制解调器的 IP 是 `192.168.1.1`。
     - 确认路由器1上启用了 **DHCP**。这允许路由器1为连接到它的设备（包括路由器2）分配 IP 地址（例如 `192.168.0.x`）。
   - 保存设置，并在提示时重新启动路由器1。

---

### **步骤 3：将路由器2设置为无线网桥**
1. **访问路由器2的 Web 界面**：
   - 将计算机或智能手机连接到路由器2的默认 WiFi 网络或通过以太网连接。
   - 打开 Web 浏览器，输入 `http://tplinkwifi.net` 或 `192.168.0.1`。
   - 使用默认凭据（或您的自定义凭据）登录。

2. **在无线网桥模式下配置路由器2**：
   - 在设置中查找类似 **无线AP桥接**、**WDS** 或 **中继器** 的模式（可能在 **操作模式** 或 **无线** 设置下）。
   - 选择 **无线AP桥接**（或者如果可用，选择 WDS/中继器）。
   - **连接到路由器1的 WiFi**：
     - 扫描可用的网络并选择路由器1的 SSID。
     - 输入路由器1的 WiFi 密码。
     - 为确保兼容性，请确保路由器2使用与路由器1相同的无线信道（例如，如果路由器1使用信道 6，则将路由器2也设置为信道 6）。
   - **LAN IP 设置**：
     - 更改路由器2的 LAN IP 以避免与路由器1冲突。例如，将其设置为 `192.168.0.2`（因为路由器1可能是 `192.168.0.1`）。
     - 在路由器2上 **禁用 DHCP**。路由器1将处理所有设备的 IP 分配。
   - 保存设置并重新启动路由器2。它现在应该无线连接到路由器1。

---

### **步骤 4：测试设置**
1. **验证路由器2的连接**：
   - 重新启动后，检查路由器2的界面以确认它已连接到路由器1的 WiFi。
2. **将设备连接到路由器2**：
   - 使用智能手机、笔记本电脑或其他设备连接到路由器2的 WiFi（根据模式，它可能使用与路由器1相同的 SSID）。
   - 检查设备是否从路由器1获取了 IP 地址（例如 `192.168.0.x`）。
   - 通过浏览网站测试互联网访问。

---

### **最终配置摘要**
- **调制解调器**：IP `192.168.1.1`，提供互联网和 WiFi。
- **路由器1**：
  - WAN：通过网线连接到调制解调器，设置为 **DHCP**（从调制解调器获取 IP，例如 `192.168.1.x`）。
  - LAN：IP `192.168.0.1`，**启用 DHCP** 为设备分配 IP。
  - WiFi：自定义 SSID 和密码。
- **路由器2**：
  - 模式：**无线AP桥接**（或 WDS/中继器），无线连接到路由器1的 WiFi。
  - LAN IP：`192.168.0.2`，**禁用 DHCP**。

---

### **故障排除提示**
- **路由器2无法无线连接**：
  - 在路由器2的配置中仔细检查路由器1的 SSID、密码和信道设置。
  - 如果 **无线AP桥接** 不可用，请尝试 **WDS** 或 **中继器** 模式。请参阅 TP-Link AX3000 手册了解支持的模式。
- **路由器2无法上网**：
  - 确保路由器1的 DHCP 正常工作，并且路由器2的 DHCP 已关闭。
  - 验证路由器2是否成功桥接到路由器1。
- **双重 NAT 问题**：
  - 由于您的调制解调器具有 WiFi 功能且路由器1充当路由器，您可能会遇到双重 NAT（调制解调器和路由器1都分配 IP）。这对于基本使用通常没问题，但可能会影响某些应用程序（例如游戏）。如果需要，将调制解调器设置为 **桥接模式**（请参阅调制解调器手册），但这可能会禁用其 WiFi。

通过此设置，路由器2将从路由器1无线扩展您的网络，连接到任一路由器的设备都应能通过调制解调器访问互联网。如果您需要进一步帮助，请告诉我！

如果您决定为第一个路由器（路由器1）使用固定（静态）IP 地址，而不是依赖 DHCP，您需要手动配置其 WAN 设置以与调制解调器通信。以下是您需要为 IP 地址、DNS、子网掩码和网关配置的内容：

---

### **假设**
- 您的调制解调器 IP 地址为 `192.168.1.1`，并使用子网 `192.168.1.0/24`（子网掩码 `255.255.255.0`）。
- 调制解调器有一个 DHCP 范围（例如 `192.168.1.2` 到 `192.168.1.100`），用于为连接的设备分配 IP 地址。

如果您的调制解调器使用不同的 IP 或子网，请相应调整值。

---

### **路由器1的 WAN 设置**
这些设置配置路由器1如何连接到调制解调器：

1. **IP 地址**
   - 在调制解调器的子网（`192.168.1.0/24`）内选择一个静态 IP，但**要在调制解调器的 DHCP 范围之外**以避免冲突。
   - 示例：如果调制解调器的 DHCP 范围是 `192.168.1.2` 到 `192.168.1.100`，您可以使用 **`192.168.1.101`**。
   - 这确保路由器1在调制解调器的网络上拥有唯一的固定地址。

2. **子网掩码**
   - 将其设置为与调制解调器的子网掩码匹配，通常为 **`255.255.255.0`**。
   - 这将网络范围定义为 `192.168.1.0` 到 `192.168.1.255`。

3. **默认网关**
   - 将其设置为调制解调器的 IP 地址，即 **`192.168.1.1`**。
   - 网关是路由器1用于访问互联网的设备（您的调制解调器）。

4. **DNS 服务器**
   - 您有两个选择：
     - 使用调制解调器的 IP：**`192.168.1.1`**。这让调制解调器处理 DNS 请求。
     - 使用公共 DNS 服务器：**`8.8.8.8`**（Google）和 **`8.8.4.4`**（Google 备用）。这些服务器可靠且通常更快。
   - 建议：从 `192.168.1.1` 开始。如果遇到 DNS 问题，请切换到 `8.8.8.8` 和 `8.8.4.4`。

---

### **配置示例**
您的路由器1 WAN 设置可能如下所示：

- **IP 地址**：`192.168.1.101`
- **子网掩码**：`255.255.255.0`
- **默认网关**：`192.168.1.1`
- **DNS 服务器**：`192.168.1.1`（或 `8.8.8.8` 和 `8.8.4.4`）

---

### **补充说明**
- **检查调制解调器设置**：登录到您的调制解调器（通常在 `192.168.1.1`）以确认其 IP、子网掩码和 DHCP 范围。如果需要，调整路由器1的静态 IP。
- **路由器1的 LAN 设置**：为避免冲突，将路由器1的 LAN 网络配置在不同的子网上，例如 `192.168.0.1`，子网掩码为 `255.255.255.0`。在路由器1上启用 DHCP 以分配 IP（例如 `192.168.0.2` 到 `192.168.0.254`）给连接到它的设备。
- **测试**：应用这些设置后，确保路由器1可以访问互联网。如果不能，请仔细检查 IP（无冲突）、网关和 DNS 值。

此配置为路由器1在调制解调器的网络上提供了一个固定 IP，确保持续连接。如果您需要其他路由器或特定调制解调器详细信息的帮助，请告诉我！

听起来您已经为路由器1设置了静态 WAN IP 并配置了其 LAN 的 DHCP，但您的笔记本电脑仍然无法连接到网络。让我们逐步排查问题，找出原因并解决。

---

### 步骤 1：验证路由器1的 WAN 连接
您的 WAN 设置是：

- **IP 地址**：`192.168.1.101`
- **子网掩码**：`255.255.255.0`
- **默认网关**：`192.168.1.1`
- **DNS 服务器**：`192.168.1.1`（或 `8.8.8.8` 和 `8.8.4.4`）

这些设置表明路由器1连接到 IP 为 `192.168.1.1` 的调制解调器或另一个路由器。以下是需要检查的内容：

- **路由器1的互联网访问**：
  - 登录路由器1的 Web 界面（根据您下面的 LAN 设置，可能是在 `192.168.0.1`）。
  - 转到 **状态** 或 **WAN** 部分，查看路由器1是否显示活跃的互联网连接。如果没有，问题可能出在路由器1和调制解调器之间。

- **IP 冲突**：
  - 确保调制解调器网络上没有其他设备使用 `192.168.1.101`。如果调制解调器的 DHCP 范围包含此 IP，可能会导致冲突。登录调制解调器（在 `192.168.1.1`）并检查其 DHCP 设置或连接的设备列表。如果 `192.168.1.101` 在调制解调器的 DHCP 范围内，要么将路由器1的 WAN IP 更改为该范围之外的 IP（例如 `192.168.1.200`），要么将其从调制解调器的 DHCP 池中排除。

- **调制解调器连接性**：
  - 确认以太网电缆已插入调制解调器的 **LAN 端口** 和路由器1的 **WAN 端口**。如果设置不正确，路由器1将无法连接到互联网。

---

### 步骤 2：检查路由器1的 LAN 和 DHCP 设置
您的 LAN 和 DHCP 设置是：

- **LAN IP**：`192.168.0.1`
- **子网掩码**：`255.255.255.0`
- **启用 DHCP**：是
- **IP 地址范围**：`192.168.0.2` 到 `192.168.0.254`
- **网关**：`192.168.0.1`
- **DNS 服务器**：`192.168.0.1`

这些设置看起来不错，但让我们确保它们正常工作：

- **DHCP 功能**：
  - 启用 DHCP 后，您的笔记本电脑应自动获取一个介于 `192.168.0.2` 和 `192.168.0.254` 之间的 IP 地址，网关为 `192.168.0.1`。如果没有，DHCP 可能无法正常工作。

- **DNS 配置**：
  - 将 DNS 服务器设置为 `192.168.0.1` 意味着路由器1为您的笔记本电脑处理 DNS 请求。确保路由器1配置为将这些请求转发到上游 DNS 服务器（如 `192.168.1.1` 或 `8.8.8.8`）。这通常是自动的，但请在路由器1的设置中仔细检查。或者，您可以直接将 DHCP DNS 设置为 `8.8.8.8` 和 `8.8.4.4`，以绕过路由器1进行 DNS 解析。

---

### 步骤 3：测试笔记本电脑的连接
既然您的笔记本电脑无法连接，让我们诊断一下：

- **连接类型**：
  - 您使用的是 WiFi 还是以太网？如果是 WiFi，请确保您连接到的是路由器1的 SSID（而不是调制解调器的）。如果是以太网，请确认电缆已插入路由器1的 LAN 端口之一。

- **检查笔记本电脑的 IP 地址**：
  - 在您的笔记本电脑上，打开 **命令提示符**（Windows）或 **终端**（macOS/Linux）：
    - Windows：输入 `ipconfig` 并按 Enter。
    - macOS/Linux：输入 `ifconfig` 或 `ip addr` 并按 Enter。
  - 查找网络适配器的 IP 地址。它应该是类似 `192.168.0.x` 的地址（其中 `x` 在 2 到 254 之间），子网掩码为 `255.255.255.0`，网关为 `192.168.0.1`。
  - **如果没有 IP 地址**：
    - 您的笔记本电脑可能没有从 DHCP 获取 IP。确保其网络设置设置为“自动获取 IP 地址”。尝试续订 IP：
      - Windows：运行 `ipconfig /release`，然后运行 `ipconfig /renew`。
    - 如果仍然失败，可能是路由器1上的 DHCP 或与它的连接出了问题。

- **Ping 路由器1**：
  - 从您的笔记本电脑运行 `ping 192.168.0.1`。如果您收到回复，说明您的笔记本电脑已连接到路由器1，但可能无法访问互联网。如果超时，则笔记本电脑无法到达路由器1。

---

### 步骤 4：常见问题及修复方法
以下是一些可能的原因：

- **电缆问题**：
  - 如果使用以太网，请尝试更换电缆或路由器1上的 LAN 端口。故障电缆或端口可能会阻止连接。

- **WiFi 问题**：
  - 如果使用 WiFi，请仔细检查密码并确保路由器1的 WiFi 已启用。信号干扰或配置错误的 SSID 也可能导致问题。

- **路由器1-调制解调器链路**：
  - 如果路由器1无法获取互联网连接，调制解调器可能需要特定设置（例如 MAC 地址克隆）。请查看调制解调器的手册或设置。

- **双重 NAT**：
  - 您的设置中，调制解调器（位于 `192.168.1.1`）和路由器1（位于 `192.168.0.1`）都充当路由器。这不应阻止基本连接，但将调制解调器设置为 **桥接模式**（如果可能）可以简化设置。

- **防火墙**：
  - 路由器1的防火墙可能阻止了您的笔记本电脑。临时禁用任何自定义防火墙规则进行测试。

---

### 步骤 5：如果需要，进行重置
如果所有方法都不起作用，重置路由器1并重新配置：

- **恢复出厂设置**：
  - 按住路由器1的重置按钮 10 秒钟，然后按照您列出的设置重新配置 WAN 和 DHCP 设置。

- **尝试动态 WAN**：
  - 不要使用静态 WAN IP，将路由器1的 WAN 设置为 **DHCP**，看看它是否能从调制解调器自动获取 IP。如果这可行，则静态 IP（`192.168.1.101`）可能是问题所在。

---

### 最后提示
- **调制解调器 DHCP 范围**：确认 `192.168.1.101` 在调制解调器的 DHCP 范围之外以避免冲突。
- **固件**：检查路由器1的 Web 界面是否有固件更新。
- **笔记本电脑驱动程序**：确保您的笔记本电脑网络适配器驱动程序是最新的。

按照这些步骤操作，您应该能找出笔记本电脑无法连接的原因。如果您仍然卡住，请告诉我您在每一步的发现——我会帮您进一步缩小范围！
