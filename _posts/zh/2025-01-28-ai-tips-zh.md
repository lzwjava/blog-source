* 将示例代码复制到 Cursor 中，并使用注释提供上下文。

* 在香港地区，可以使用 DeepSeek 或 Mistral 服务，但不能使用 ChatGPT、Claude 或 Gemini。

* 某个地区的 API 策略与其应用策略大致相同。

* 使用 Cursor 代替 Visual Studio Code。

* 仍然有一些情况需要使用 Visual Studio Code，例如 git merge editor 场景，我仍然使用 `git config --global core.editor "code --wait"`。

* 从 Deepseek V3 发布之日起，我们不需要订阅任何 AI 工具。

* 使用 Gemini 或 Grok 生成节日庆祝图片，提示语例如“生成一张包含姓名文字的快乐农历蛇年图片”。

* 在某些情况下，即使向 AI 模型提供原文来创建表格，输出中的某些地方也可能与输入不同。例如，在 Cursor 中使用 Deepseek V3 模型生成 pip list 表格时，可能会包含 `1.极狐0` 之类的版本。这里，“极狐”指的是中国 GitLab 平台。

* 使用 DeepSeek 或 Mistral API 翻译标题，提示语例如 `You are a professional translator. You are translating a markdown file for a Jekyll blog post from English to Chinese. {text}`，可能会导致翻译不准确。除了你提供的文本之外，输出通常还会包含过多的翻译内容。

* 虽然有时 Cursor 中的 AI 模型会给出部分正确的文本，但我们可以接受它们，因为我们可以添加后续指令，让 AI 模型重新生成正确的部分。

* 如果过多的上下文不太可能有帮助，请避免将其提供给大型语言模型。例如，在生成对话对话行时，避免提供关于某个主题的 100 个要点。大型语言模型已经包含大量数据。

* 在为翻译或生成对话歌词等任务提供充足的上下文时，避免使用思维链功能，因为它可能会很慢，并导致冗长或无用的响应。

* 测试聊天机器人是否能遵循用户指令的一种方法是，要求它用英语解释某些内容，然后用中文继续输入，观察聊天机器人是否保持用英语输出。
