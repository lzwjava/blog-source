## 什么是 PuTTY？

PuTTY 是由 Simon Tatham 开发的多功能终端模拟器和网络客户端，主要面向 Windows 系统，同时支持 Linux 和 macOS。它支持多种协议，包括：

- **SSH（安全外壳协议）**：用于安全远程访问服务器
- **Telnet**：用于非加密连接（因安全问题较少使用）
- **Rlogin**：用于 Unix 系统远程登录
- **SCP 与 SFTP**：用于安全文件传输
- **原始套接字连接**：用于直接网络通信
- **串行连接**：用于与路由器等硬件设备交互

PuTTY 具有轻量级、高度可配置的特点，并包含 PuTTYgen（用于生成 SSH 密钥）和 PSFTP/PSCP（用于文件传输）等工具，被系统管理员、开发人员和 IT 专业人员广泛用于远程服务器管理。[](https://en.wikipedia.org/wiki/PuTTY)

## 下载与安装 PuTTY

### 步骤 1：下载 PuTTY
1. 访问 PuTTY 官网 [www.putty.org](https://www.putty.org/) 或下载页面 [chiark.greenend.org.uk/~sgtatham/putty/latest.html](https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html)。[](https://www.putty.org/)
2. 选择适合的安装包：
   - **32 位或 64 位 MSI 安装包**：根据系统类型选择 `putty-<版本>-installer.msi`（现代 PC 建议选择 64 位）。[](https://its.gmu.edu/knowledge-base/how-to-install-putty-ssh-for-windows/)
   - **独立二进制文件**：如果不想使用安装包，可单独下载 `putty.exe`、`puttygen.exe`、`psftp.exe` 等文件
3. 若不确定系统类型，32 位版本具有通用兼容性。[](https://www.ssh.com/academy/ssh/putty/windows/install)

### 步骤 2：安装 PuTTY
1. 双击下载的 `.msi` 文件启动 PuTTY 安装向导
2. 在欢迎界面点击 **Next**
3. 接受默认目标文件夹（通常为 `C:\Program Files\PuTTY\`）或选择自定义位置，点击 **Next**。[](https://www.techtarget.com/searchsecurity/tutorial/How-to-use-PuTTY-for-SSH-key-based-authentication)
4. 选择所需功能（如创建桌面快捷方式），点击 **Install**。[](https://www.ssh.com/academy/ssh/putty/windows/install)
5. 若出现 Windows 用户账户控制提示，点击 **Yes**
6. 安装完成后点击 **Finish**。[](https://www.ssh.com/academy/ssh/putty/windows/install)

### 步骤 3：验证安装
- 若创建了桌面快捷方式，双击 PuTTY 图标
- 或在 Windows 开始菜单搜索 "PuTTY"，也可前往 `C:\Program Files\PuTTY\putty.exe` 打开。[](https://its.gmu.edu/knowledge-base/how-to-install-putty-ssh-for-windows/)

## 基础配置与服务器连接

### 启动 PuTTY
1. 从开始菜单或桌面快捷方式打开 PuTTY，将显示 **PuTTY 配置** 窗口，包含：
   - 左侧配置分类面板
   - 中部连接详细信息输入区
   - 右下角会话配置文件保存区。[](https://www.ssh.com/academy/ssh/putty/windows)

### 连接服务器
1. **输入主机信息**：
   - 在 **主机名称（或 IP 地址）** 字段输入服务器域名（如 `students.example.edu`）或 IP 地址（如 `192.168.1.1`）
   - 确保 **端口** 设置为 SSH 默认端口 `22` 或服务器管理员指定的端口。[](https://www.wikihow.com/Use-Putty-in-Windows)
2. **选择连接类型**：
   - 选择 **SSH**（默认）建立安全连接，其他选项包括 Telnet、Rlogin、Raw 或 Serial。[](https://www.wikihow.com/Use-Putty-in-Windows)
3. **保存会话**（可选）：
   - 在 **已保存会话** 字段输入名称（如 `MyServer`）
   - 点击 **Save** 存储配置供后续使用。[](https://www.cuit.columbia.edu/putty)
4. **建立连接**：
   - 点击 **Open** 开始连接
   - 首次连接时可能出现 **PuTTY 安全警报**，提示服务器主机密钥未缓存，点击 **Yes** 信任服务器并继续。[](https://www.ssh.com/academy/ssh/putty/windows)

### 登录操作
- 在终端窗口提示时输入用户名和密码
- 若需密钥认证，请参照 [使用 PuTTYgen](#使用-puttygen-实现-ssh-密钥认证) 章节进行配置。[](https://www.wikihow.com/Use-Putty-in-Windows)

## 高级配置选项

PuTTY 配置窗口提供丰富的自定义选项，以下是关键分类和设置：

### 会话
- **已保存会话**：加载、保存或删除会话配置文件以实现快速访问
- **退出时关闭窗口**：选择断开连接后终端是否自动关闭（`总是`、`从不` 或 `仅正常退出时`）。[](https://www.ssh.com/academy/ssh/putty/windows)

### 终端
- **自动换行模式**：启用后将长文本行自动换行显示，禁用时超出窗口宽度的文本可能无法显示。[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
- **回滚行数**：增加缓冲区存储的行数（默认 200 行）以查看更多终端输出。[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
- **保活数据包**：设置数值（秒）发送空数据包，防止因空闲导致连接断开。[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)

### 窗口
- **外观**：
   - 在 **窗口 > 外观** 下更改字体设置（如大小、样式）
   - 在 **窗口 > 颜色** 下调整颜色提升可读性（确保前景色与背景色对比度充足）。[](https://www.cuit.columbia.edu/putty)
- **调整大小行为**：设置终端响应窗口调整大小的方式（如缩放字体大小或调整终端尺寸）。[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)

### 连接
- **代理设置**：通过企业网络连接时配置代理
- **自动登录**：指定用户名跳过登录提示。[](https://www.puttygen.com/download-putty)
- **SSH 设置**：
  - 选择首选 SSH 协议版本（推荐 SSH-2 以确保安全）
  - 选择加密算法（如 AES、3DES）和密钥交换算法。[](https://en.wikipedia.org/wiki/PuTTY)

### 保存与加载会话
1. 完成配置后返回 **会话** 分类
2. 在 **已保存会话** 字段输入名称并点击 **Save**
3. 加载已保存会话时，从列表选择会话，点击 **Load**，然后 **Open**。[](https://www.cuit.columbia.edu/putty)

## 使用 PuTTYgen 实现 SSH 密钥认证

SSH 密钥认证比密码认证更安全。PuTTYgen 可生成 PuTTY 专用格式 `.ppk` 的密钥对。

### 步骤 1：生成密钥对
1. 从开始菜单或 `C:\Program Files\PuTTY\puttygen.exe` 打开 **PuTTYgen**
2. 选择密钥类型（如 **RSA**、默认或 **ECDSA**）和位长度（RSA 推荐 2048 或 4096）
3. 点击 **Generate** 并移动鼠标生成随机性，直到进度条完成
4. 可选设置 **通行短语** 增强安全性
5. 将私钥（`.ppk` 文件）保存至安全位置
6. 复制 "用于 OpenSSH authorized_keys 文件的公钥" 字段中显示的公钥。[](https://www.wikihow.com/Use-Putty-in-Windows)

### 步骤 2：配置服务器
1. 使用密码认证（或其他方式）连接远程服务器
2. 创建或编辑 `~/.ssh/authorized_keys` 文件：
   - 运行 `mkdir ~/.ssh && chmod 700 ~/.ssh` 创建 `.ssh` 目录
   - 用文本编辑器打开 `authorized_keys` 文件（如 `nano ~/.ssh/authorized_keys`）
   - 粘贴公钥并保存文件
   - 使用 `chmod 600 ~/.ssh/authorized_keys` 设置权限。[](https://www.techtarget.com/searchsecurity/tutorial/How-to-use-PuTTY-for-SSH-key-based-authentication)
3. 退出服务器登录

### 步骤 3：配置 PuTTY 密钥认证
1. 打开 PuTTY 加载已保存会话
2. 进入 **连接 > SSH > 认证 > 凭证**
3. 点击 "认证用私钥文件" 旁的 **Browse**，选择您的 `.ppk` 文件
4. 返回 **会话** 分类，保存更新后的会话，点击 **Open**
5. 提示时输入通行短语（若设置）。现在应无需密码即可连接。[](https://www.wikihow.com/Use-Putty-in-Windows)

## 通过 PSFTP 和 PSCP 进行文件传输

PuTTY 包含 **PSFTP**（用于 SFTP）和 **PSCP**（用于 SCP）以实现安全文件传输。

### 使用 PSFTP
1. 从开始菜单或 `C:\Program Files\PuTTY\psftp.exe` 打开 **PSFTP**
2. 输入 `open <主机名>`（如 `open students.example.edu`）并按回车
3. 使用凭证或私钥登录
4. 使用以下命令：
   - `ls` 或 `dir`：列出文件
   - `get <文件名>`：下载文件
   - `put <文件名>`：上传文件
   - `cd <目录>`：切换目录
   - `quit`：退出 PSFTP。[](https://www.wikihow.com/Use-Putty-in-Windows)

### 使用 PSCP
1. 打开命令提示符或 PowerShell
2. 进入 PuTTY 安装目录（如 `cd C:\Program Files\PuTTY`）
3. 运行命令：
   - `pscp <本地文件> <用户名>@<主机名>:<远程路径>` 上传文件
   - `pscp <用户名>@<主机名>:<远程文件> <本地路径>` 下载文件
   - 添加 `-i <私钥.ppk>` 参数使用密钥认证。[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)

### 注意事项
- PSFTP 为交互式操作，PSCP 为命令行操作
- 如需图形化文件传输，可考虑 WinSCP 等替代方案，支持导入 PuTTY 会话。[](https://www.ssh.com/academy/ssh/putty)

## 端口转发与隧道技术

PuTTY 支持 **SSH 隧道**（端口转发）以安全访问远程服务器上的服务。

### 本地端口转发
1. 在 PuTTY 中进入 **连接 > SSH > 隧道**
2. 输入本地机器的 **源端口**（如 `8080`）
3. 输入远程服务的 **目标**（如 `remote_host:80`）
4. 选择 **Local** 并点击 **Add**
5. 保存会话并连接
6. 在本地访问服务（如 `http://localhost:8080`）。[](https://www.ssh.com/academy/ssh/putty/windows)

### 远程端口转发
- 与本地转发类似，但选择 **Remote** 并指定远程服务器端口转发至本地机器

### 动态端口转发
- 选择 **Dynamic** 并指定本地端口（如 `8080`），这将创建用于动态路由的 SOCKS 代理（适用于通过服务器浏览）。[](https://rushax.com/what-is-putty-a-comprehensive-guide/)

## 图形化应用的 X11 转发

PuTTY 支持 **X11 转发**，可在远程服务器运行图形应用程序并在本地显示。

1. 在 Windows 上安装 X 服务器（如免费选项 **XMing**）
2. 在 PuTTY 中进入 **连接 > SSH > X11**
3. 勾选 **Enable X11 forwarding**
4. 设置 **X display location** 为 `localhost:0.0`
5. 保存会话并连接
6. 在服务器运行图形命令（如 `xterm`），界面将通过 XMing 在本地显示。[](https://www.ssh.com/academy/ssh/putty/windows)

## 常见问题排查

1. **“访问被拒绝”错误**：
   - 验证用户名、密码或私钥
   - 确保服务器启用 SSH 访问（如检查主机控制面板设置）。[](https://www.hostinger.com/uk/tutorials/how-to-use-putty-ssh)
   - 检查服务器是否仅允许密钥认证

2. **“连接被对等端重置”**：
   - 在 **连接** 设置中启用 **保活数据包** 防止超时。[](https://www.tecmint.com/putty-configuration-tips-and-tricks/)
   - 检查可能关闭空闲连接的防火墙或网络设备设置

3. **“服务器主机密钥未缓存”**：
   - 首次连接属正常现象，点击 **Yes** 缓存密钥。若重复出现，请验证服务器真实性以排除中间人攻击。[](https://www.ssh.com/academy/ssh/putty/windows)

4. **登录尝试失败警告**：
   - 重置密码或检查服务器上的暴力破解保护
   - 若被锁定请联系服务器管理员。[](https://www.hostinger.com/uk/tutorials/how-to-use-putty-ssh)

5. **PuTTY 窗口意外关闭**：
   - 检查服务器日志中的错误（如 `~/.ssh/authorized_keys` 权限错误）
   - 确保加载了正确的私钥。[](https://www.hostinger.com/uk/tutorials/how-to-use-putty-ssh)

## 技巧与最佳实践

- **定期更新**：下载最新 PuTTY 版本确保安全补丁和新功能（如新版支持的椭圆曲线加密）。[](https://www.ssh.com/academy/ssh/putty)

## PuTTY 替代方案

虽然 PuTTY 稳定可靠，但现代替代方案提供增强功能：
- **OpenSSH**：内置于 Windows 10/11、Linux 和 macOS，基于命令行但功能强大。[](https://phoenixnap.com/kb/putty-alternative)
- **WinSCP**：图形化文件传输客户端，可与 PuTTY 集成。[](https://www.ssh.com/academy/ssh/putty)
- **MobaXterm**：提供标签式界面、集成 X 服务器和 SFTP 支持。[](https://rushax.com/what-is-putty-a-comprehensive-guide/)
- **Bitvise SSH Client**：支持 GUI、脚本编写和单键隧道功能。[](https://phoenixnap.com/kb/putty-alternative)
- **Termius**：跨平台工具，具有 SSH 密钥管理和现代界面。[](https://phoenixnap.com/kb/putty-alternative)
