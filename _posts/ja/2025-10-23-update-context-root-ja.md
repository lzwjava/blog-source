### IBM WebSphere Application Server (WAS) での Web コンテキストルートの更新

Web コンテキストルートは、Web アプリケーションのベース URL パス (例: `/myapp`) を定義します。IBM WebSphere Application Server では、管理コンソールを使用して、初期デプロイ時または既存のアプリケーションに対してこれを更新できます。以下は、既存のアプリケーションに対する手順です。WAS コンソールへの管理者アクセス権があることを確認してください。

#### 前提条件
- WebSphere 管理コンソールへのアクセス (通常は `https://your-server:9043/ibm/console`)。
- アプリケーションはインストール済みであり、変更を行う前に停止されていること (推奨)。これにより競合を回避できます。

#### コンテキストルートを更新する手順
1.  **管理コンソールにログインする**:
    - Web ブラウザを開き、WAS コンソールの URL に移動します。
    - 管理者資格情報を入力します。

2.  **アプリケーションに移動する**:
    - 左側のナビゲーションペインで、**アプリケーション** > **アプリケーション・タイプ** > **WebSphere エンタープライズ・アプリケーション** を展開します。
    - リストからデプロイ済みのアプリケーションを探して選択します。

3.  **コンテキストルート設定にアクセスする**:
    - アプリケーションの詳細ページで、**Web モジュール・プロパティー** セクションまでスクロールダウンします。
    - **Web モジュールのコンテキスト・ルート** をクリックします。

4.  **コンテキストルートを編集する**:
    - 表示されたテーブルで、Web モジュール (例: WAR ファイル名) を見つけます。
    - **コンテキスト・ルート** フィールドを新しい値 (例: `/oldapp` から `/newapp` へ変更) で更新します。必要がなければ先頭のスラッシュは避けますが、`/myapp` のようなパスの場合はスラッシュを含めてください。
    - **OK** をクリックして変更を保存します。

5.  **構成を保存して同期する**:
    - コンソールで **保存** をクリックします (または、プロンプトが表示された場合は **マスター構成に直接保存**)。
    - クラスター環境またはネットワーク・デプロイメント環境にある場合:
        - **システム管理** > **ノード** に移動します。
        - 関連するすべてのノードを選択し、**完全再同期** をクリックします。

6.  **アプリケーションを再始動する**:
    - **アプリケーション** > **WebSphere エンタープライズ・アプリケーション** に戻ります。
    - アプリケーションを選択し、**停止**、次に **開始** をクリックします。
    - または、**サーバー** > **サーバー・タイプ** > **WebSphere アプリケーション・サーバー** > サーバーを選択 > **再始動** からサーバー全体を再始動します。

7.  **変更を確認する**:
    - ブラウザでアプリケーション URL (例: `http://your-server:9080/newapp`) にアクセスします。
    - サーバーログ (例: `SystemOut.log`) でデプロイメントに関連するエラーがないか確認します。

#### 追加の注意点
-   **初期デプロイ時**: 新しいアプリケーションをデプロイする場合、**インストール・オプションの選択** ステップで、各 Web モジュールの **コンテキスト・ルート** にコンテキストルートを指定します。
-   **EAR ファイル**: EAR にバンドルされたアプリケーションの場合、`application.xml` がコンテキストルートをハードコードしていないことを確認してください。コンソールでのオーバーライドが優先されます。
-   **一般的な問題**:
    - アプリケーションが実行中の場合は変更が適用されない可能性があります。必ず事前に停止してください。
    - WebSphere Liberty または Open Liberty では、代わりに `server.xml` 設定を使用します。
    - 仮想ホストまたはプラグインを使用する場合は、変更後に Web サーバープラグイン構成を更新してください。
-   アプリケーションがセキュリティまたは共有ライブラリを使用する場合は、更新後に十分にテストしてください。

詳細については、公式の IBM ドキュメントまたはコミュニティガイドを参照してください。
