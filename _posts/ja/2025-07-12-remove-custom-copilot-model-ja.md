GitHub Copilot 拡張機能 for Visual Studio Code (VS Code) でモデルを削除するには、組み込みモデル（デフォルトの GitHub Copilot モデルなど）は拡張機能の一部であるため削除できないことに注意してください。ただし、サードパーティプロバイダー (例: Anthropic, OpenAI, Google Gemini など) 経由で独自の API キーを使用して追加したカスタムモデルを参照している場合、関連する API キーとプロバイダーの詳細を削除することで事実上削除できます。この機能はプレビュー段階であり、Copilot Business または Enterprise ユーザーにはまだ利用できない場合があります。

公式ドキュメントに基づく手順は以下の通りです：

1.  VS Code でチャットビューを開きます (まだ開いていない場合は、コマンドパレットを `Ctrl+Shift+P` (macOS では `Cmd+Shift+P`) で開き、「Copilot: Open Chat」を検索します)。
2.  チャットビューで、モデル選択ドロップダウン (入力フィールドの横) をクリックし、**Manage Models** を選択します。または、コマンドパレットを開き、**GitHub Copilot: Manage Models** コマンドを実行します。
3.  Manage Models インターフェースで、削除したいモデルのプロバイダーにマウスを合わせ、歯車アイコンをクリックして詳細を編集します。
4.  フィールドから API キー (および該当する場合はエンドポイント URL) を削除し、変更を保存します。これにより、モデルはピッカーから利用できなくなります。

問題が発生した場合 (例：API キーフィールドが適切に更新またはクリアされない。これはバグとして報告されている場合があります)、まず次の回避策を試してください：
-   チャットビューのモデルピッカーで、別のモデルを選択し、削除しようとしているモデルの選択を解除します。
-   Manage Models に戻り、プロバイダーを編集してから API キーを削除します。

前提条件と注意点：
-   これはチャット機能にのみ適用されます。コード補完やその他のツールには影響しません。
-   Copilot サブスクリプションが複数のモデルへのアクセスを許可していること (例：Individual, Pro、または組織でプレビュー機能が有効になっていること) を確認してください。
-   モデルが残存する場合やエラーが発生する場合は、VS Code を再起動するか、拡張機能の更新を確認してください。

これが「モデルを削除する」という意味に合致しない場合 (例：拡張機能全体を無効化しようとしている、または Copilot 機能を非表示にしようとしている場合)、より適切な手順を得るために詳細を明記してください。
