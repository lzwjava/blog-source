## 1. Java言語の概要

### 1.1 Javaの特徴

*   **プラットフォーム独立性:** Javaコードは*バイトコード*にコンパイルされ、*Java仮想マシン（JVM）* を搭載した任意のデバイスで実行されます — 「一度書けば、どこでも実行できる」。
*   **オブジェクト指向:** *カプセル化、継承、ポリモーフィズム*をサポートします。
*   **安全性と堅牢性:** 自動メモリ管理（ガベージコレクション）と強力な型チェックにより、エラーを削減します。
*   **マルチスレッド:** 複数タスクの同時実行をサポートします。
*   **豊富な標準ライブラリ（API）:** 数学、文字列、ファイル、ネットワーキングなどのための既製のクラスを含みます。

### 1.2 Javaのバージョンと構成要素

*   **JDK (Java Development Kit):** 開発者向け — コンパイラ(`javac`)、JVM、開発ツールを含みます。
*   **JRE (Java Runtime Environment):** エンドユーザー向け — JVM + コアライブラリを含みます。
*   **API (Application Programming Interface):** Javaの組み込みクラスライブラリ。例: `java.lang`, `java.util`, `java.io` など。

## 2. Java開発ツール (IDEとCLI)

### 2.1 一般的なIDE

試験では、その目的を知っているだけで十分です:

*   **Eclipse, IntelliJ IDEA, NetBeans:** Javaコードを簡単に記述、コンパイル、実行するために使用されます。

### 2.2 コマンドラインでのワークフロー

典型的なコンパイルと実行の手順:

1.  `.java`ファイル（例: `Hello.java`）にコードを**記述**します。
2.  コードを**コンパイル**します:

    ```bash
    javac Hello.java
    ```

    → `Hello.class`（バイトコードファイル）が生成されます。
3.  コードを**実行**します:

    ```bash
    java Hello
    ```

    (実行時は `.class` 拡張子は付けません)

### 2.3 簡単な例

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, Java!");
    }
}
```

## 3. プログラミングスタイルガイドライン

### 3.1 命名規則

*   **クラス:** `CamelCase`、最初の文字は大文字 → `StudentInfo`
*   **変数とメソッド:** `camelCase`、最初の文字は小文字 → `studentName`, `calculateScore()`
*   **定数:** すべて大文字でアンダースコアを使用 → `MAX_SIZE`

### 3.2 インデントとコメント

*   **一貫したインデント**を使用します（通常は4スペース）。
*   明確な**コメント**を記述します:

    ```java
    // これは単一行コメントです
    /* これは
       複数行コメントです */
    ```

### 3.3 コード構造

論理的なグループ化と可読性に従います:

*   各ファイルには、ファイル名と**同じ名前**の**publicクラスが1つ**含まれます。
*   行は短く意味のあるものに保ち、1行に1つのステートメントを記述します。

## 4. 一般的なプログラミングエラーとデバッグの基礎

### 4.1 構文エラー

コンパイラによって検出されます:

*   セミコロン `;` の欠落
*   括弧 `{ }` の不一致
*   大文字小文字の誤り (Javaは**大文字と小文字を区別**します)

    ```java
    System.out.println("Hi"); // 正しい
    system.out.println("Hi"); // エラー
    ```

### 4.2 実行時エラー

プログラムの実行中に発生します:

*   ゼロ除算
*   配列のインデックスが範囲外
*   Nullポインタ例外

### 4.3 論理エラー

プログラムは実行されますが、誤った出力を生成します。
→ **print文**やIDEの**デバッガツール**を使用して、変数の値を段階的にトレースします。

## 5. 試験の重要なポイント

*   **JDK, JRE, JVM** の役割と違いを理解する。
*   IDEとコマンドラインの両方からJavaを**コンパイルおよび実行**する方法を知っている。
*   **基本的な構文規則**と**スタイル標準**を覚えておく。
*   **一般的なエラーを特定**し、その原因を説明できる。
*   以下のような小さなプログラムを書く練習をする:
    *   メッセージを表示する
    *   変数と式を使用する
    *   正しい構造と命名規則に従う

## 6. 試験形式のサンプル問題

**例1:**

> Javaについて*誤っている*記述はどれか？
> A. Javaはバイトコードを通じてプラットフォーム独立性をサポートする
> B. Javaプログラムは常にネイティブマシンコードにコンパイルされなければならない
> C. JVMはコンパイルされたバイトコードを実行する
> D. JavaプログラムはJVMがインストールされた任意のシステムで実行できる

**解答:** B

**例2:**

> `Test.java` という名前のJavaソースファイルをコンパイルするコマンドはどれか？
> A. java Test
> B. javac Test.java
> C. run Test
> D. compile Test

**解答:** B
