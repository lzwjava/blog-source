### Sublime Text での `subl` コマンドの使用

朗報です — `subl` CLI は既にインストールされて動作しています (`--help` の出力で確認できますね)! これは Sublime Text ビルド 4200 (最近のバージョン) であり、`subl` はファイルやディレクトリを開くための公式コマンドラインインターフェースです。開発者やパワーユーザーにとって非常に多用途に使えます。

実用的な例とともに説明します。これらをターミナルの任意のディレクトリから実行してください。基本的な構文は次のとおりです:
```
subl [引数] [ファイルまたはディレクトリ]
```

#### 1. **基本的な使用法: ファイルやディレクトリを開く**
- **カレントディレクトリを開く** (プロジェクト/フォルダとして Sublime に読み込む):
  ```
  subl .
  ```
  - これにより、カレントフォルダの内容を含む新しいウィンドウが開きます。

- **特定のファイルを開く**:
  ```
  subl myfile.txt
  ```
  - `myfile.txt` をデフォルトのウィンドウ (または必要に応じて新しいウィンドウ) で開きます。

- **複数のファイル/ディレクトリを開く**:
  ```
  subl file1.txt file2.js ~/Documents/myproject/
  ```
  - これらすべてを Sublime で開きます。

- **特定の行/列で開く** (エラーへのジャンプに便利):
  ```
  subl myfile.py:42          # 42行目で開く
  subl myfile.py:42:5        # 42行目、5列目で開く
  ```

#### 2. **一般的な引数 (ヘルプより)**
以下は最も便利なフラグと例です。必要に応じて組み合わせてください (例: `subl -n file.txt`)。

- **`-n` または `--new-window`**: 常に新しいウィンドウで開く。
  ```
  subl -n myfile.txt
  ```
  - 既存の Sublime セッションを分離しておきたい場合に便利です。

- **`-a` または `--add`**: ファイル/フォルダを*現在の* Sublime ウィンドウに追加する (既に開いている場合)。
  ```
  subl -a newfolder/
  ```
  - 新しいウィンドウは作成しません。ワークスペースを構築するのに最適です。

- **`-w` または `--wait`**: ターミナルコマンドが終了する前に、Sublime でファイルを閉じるのを待つ。
  ```
  subl -w myfile.txt
  ```
  - スクリプト内で有用 (例: ビルド実行後、開いてレビューを待つ)。標準入力から読み取る時は暗黙的に適用されます。

- **`-b` または `--background`**: Sublime を前面に持ってこずに開く (ターミナルにフォーカスを維持)。
  ```
  subl -b myfile.txt
  ```

- **`-s` または `--stay`**: ファイルを閉じた後も Sublime にフォーカスを維持する (`-w` と組み合わせた場合のみ関連)。
  ```
  subl -w -s myfile.txt
  ```
  - 自動的にターミナルに切り替わるのを防ぎます。

- **`--project <project>`**: 特定の Sublime プロジェクトファイル (`.sublime-project`) を開く。
  ```
  subl --project MyProject.sublime-project
  ```
  - プロジェクトはワークスペース、設定などを保存します。Sublime で File > Save Project から作成できます。

- **`--command <command>`**: ファイルを開かずに Sublime コマンド (例: プラグインアクション) を実行する。
  ```
  subl --command "build"    # ビルドシステムが設定されていれば、ビルドコマンドをトリガーする
  ```
  - 利用可能なコマンドは Sublime のコンソール (View > Show Console) で確認してください。

- **`--launch-or-new-window`**: Sublime が既に実行されていない場合にのみ、新しいウィンドウを開く。
  ```
  subl --launch-or-new-window .
  ```
  - 画面を散らかさずに素早く確認するのに効率的です。

- **標準入力サポート** (パイプ入力の編集):
  ```
  echo "Hello World" | subl -   # 標準入力を Sublime で編集用に開く
  subl - > output.txt          # 標準入力を編集し、変更を output.txt に保存
  ```
  - Sublime で編集後、保存して閉じると続行します。

- **`--safe-mode`**: プラグインやカスタム設定なしで Sublime を起動 (トラブルシューティング用)。
  ```
  subl --safe-mode myfile.txt
  ```

- **その他のフラグ**:
  - `-h` または `--help`: 既にご覧になったヘルプを表示します。
  - `-v` または `--version`: バージョンを表示します (例: `subl -v`)。

#### 3. **日常使用のためのヒント**
- **`-` で始まるファイル**: ファイル名がフラグのように見える場合 (例: `-config`)、区切りに `--` を使用:
  ```
  subl -- -config.json
  ```
- **便利なエイリアス**: シェルプロファイル (`~/.zshrc` for zsh, または `~/.bash_profile` for bash) にショートカットを追加:
  ```
  alias st='subl'     # 短いエイリアス
  alias stt='subl -n' # 新しいタブ/ウィンドウ用
  ```
  `source ~/.zshrc` でリロードします。
- **スクリプト/GIT 内で**: フックやエイリアスで使用します。例: `git diff | subl -` で変更をレビュー。
- **`open` コマンドとの比較**: `subl` が設定された今では、`open -a "Sublime Text"` よりも優れています。なぜなら、行指定、プロジェクト、そしてより高度な機能をサポートしているからです。
- **トラブルシューティング**:
  - `subl` が動作しなくなった場合は、以前の回答で説明したシンボリックリンクのインストールを再実行してください。
  - 大規模なプロジェクトでは、複数ウィンドウを避けるために `-a` を使用してください。
  - macOS でフォーカスに関する問題が発生した場合は、まず `-b` なしで試してください。
