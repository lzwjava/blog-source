NPM（Node Package Manager）は、npmコマンド実行時の出力詳細度を制御するいくつかのログレベルを提供しています。以下に、出力が少ない順から多い順に示します：

- **silent**: 出力を表示しない
- **error**: エラーのみ表示（デフォルト）
- **warn**: エラーと警告を表示
- **notice**: エラー、警告、通知を表示
- **http**: HTTPリクエストを表示
- **timing**: タイミング情報を表示
- **info**: 情報メッセージを表示
- **verbose**: 詳細な情報を表示
- **silly**: すべてのデバッグ情報を表示

ログレベルは`--loglevel`フラグまたは`npm_config_loglevel`環境変数を使用して設定できます：

```bash
npm install --loglevel verbose
```

または、npm設定で設定することもできます：

```bash
npm config set loglevel verbose
```

ほとんどのコマンドのデフォルトログレベルは「error」です。つまり、より詳細な出力を明示的に要求しない限り、エラーメッセージのみが表示されます。
