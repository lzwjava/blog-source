### Azure API Management を使用した API ゲートウェイの作成

Azure API Management (APIM) は、API ゲートウェイとして機能するフルマネージドサービスであり、API の発行、セキュリティ、分析、開発者ポータルなどの機能を提供します。以下は、Azure ポータル経由で APIM インスタンスを作成するステップバイステップガイドです。

#### 前提条件
- アクティブな Azure サブスクリプション。お持ちでない場合は、[無料の Azure アカウント](https://azure.microsoft.com/ja-jp/free/)を作成してください。

#### API Management インスタンスの作成手順

1. **Azure ポータルにサインインする**  
   [Azure ポータル](https://portal.azure.com) に移動し、Azure アカウントでサインインします。

2. **新しいリソースを作成する**  
   - Azure ポータルメニューから **[リソースの作成]** を選択します（または、Azure ホームページから **[リソースの作成]** を選択します）。  
   - **[リソースの作成]** ページで、**[統合]** カテゴリの下の **API Management** を検索して選択します。  
   - **[作成]** をクリックします。

3. **基本設定を構成する**  
   **[API Management の作成]** ページで、**[基本]** タブに切り替え、詳細を入力します：  
   | 設定              | 説明                                                                         |
   |-------------------|-----------------------------------------------------------------------------|
   | サブスクリプション         | このインスタンスの Azure サブスクリプションを選択します。                            |
   | リソース グループ       | 既存のリソース グループを選択するか、新しいものを作成します（例: "APIM-RG"）。    |
   | リージョン               | ユーザーまたはバックエンド サービスに近いリージョンを選択します（例: East US）。      |
   | リソース名        | 一意の名前を入力します（例: "my-apim-instance"）。これはデフォルト ドメインの一部になります: `<名前>.azure-api.net`。後で変更することはできません。 |
   | 組織名    | 開発者ポータルやメールで使用される組織の名前。             |
   | 管理者メール  | システム通知用のメールアドレス。                                        |
   | 価格レベル         | 開発/テストには **Basic v2** から始めます（迅速なデプロイ、約30〜40分）。Developer や Standard などの他のレベルでは、より多くの機能を利用できます。 |
   | ユニット                | 評価用にはデフォルトで 1 に設定します。                                                |  
   設定を確認し、次に進みます。

4. **オプション: 監視とセキュリティを構成する**  
   **[監視 + セキュリティ]** タブで：  
   - ログ記録用の Log Analytics などのオプションのアドオンを有効にします（追加費用が発生する場合があります）。  
   - 使い始めの段階では、今はスキップしても構いません。

5. **確認と作成**  
   - **[確認 + 作成]** タブで、設定を検証します。  
   - **[作成]** をクリックします。デプロイには 30〜40 分以上かかります。  
   - ヒント: 作成後、リソースをダッシュボードにピン留めすると簡単にアクセスできます。

6. **インスタンスを確認する**  
   - Azure ポータルで **API Management サービス** を検索します。  
   - 新しいインスタンスを選択します。  
   - **[概要]** ページで、ステータスが「オンライン」であることを確認します。これで API のインポート、ポリシーの構成、開発者ポータルの設定を行う準備が整いました。

#### 次のステップ
- 最初の API をインポートする: [API をインポートして発行するチュートリアル](https://learn.microsoft.com/ja-jp/azure/api-management/import-and-publish) に従ってください。  
- セキュリティ保護: 認証、レート制限などのポリシーを追加します。  
- クリーンアップ: テスト中の場合は、課金を避けるためにポータルからリソース グループを削除します（**リソース グループ** を検索 > グループを選択 > **[リソース グループの削除]**）。  

カスタム ドメインや高度な構成については、[APIM ドキュメント](https://learn.microsoft.com/ja-jp/azure/api-management/)を参照してください。
