### 360 Wi-Fi ルーター T7 の DHCP ログエントリの理解

この 360 T7 ルーターからのログスニペット（おそらく Web UI の **システム > ログ** セクションから抽出されたもの）は、ルーターの WAN インターフェースが DHCP クライアントとして機能し、ISP の DHCP サーバーから公開 IP アドレスを取得および更新していることを示しています。DHCP (Dynamic Host Configuration Protocol) は、ネットワーク上で IP アドレスを自動的に割り当てるための標準プロトコルです。ルーターはインターネット接続を維持するために、定期的に IP リースの要求または更新を行います。これは通常、ISP のリース時間に依存して 12～24 時間ごとに行われます。

これらのエントリは **正常な動作を示しており、エラーではありません**。これらは、初期 IP 割り当てのための標準的な DORA プロセス (Discover, Offer, Request, ACK)、およびその後のリース更新を表しています。以下に、時系列順（注：ログ内のタイムスタンプは逆順ですが、明確にするために並べ替えています。日付形式は YYYY/MM/DD-HH:MM:SS です）に各行の内訳を示します：

#### これが発生する理由
- **リースサイクル**: ISP はリース時間（例：24時間）を設定します。経過時間の 50% で、クライアントはダウンタイムを避けるために更新を要求します。87.5% で更新に失敗した場合、リバインド（ブロードキャスト Request）を行います。100% で、完全な DORA プロセスを再開します。
- **360 T7 のコンテキスト**: これは WAN 接続イベントとして記録されます。T7 は、そのファームウェア内で軽量な DHCP クライアント（おそらく udhcpc ベース）を使用しており、これらとまったく同じメッセージを生成します。"DHCP NAK received"（IP 拒否）や ACK のない Discover の繰り返し（ISP の問題またはケーブルの問題を示す可能性がある）などの失敗が見られない限り、問題はありません。
- **トラブルシューティングのヒント**:
  - これらの時間帯にインターネットが切断される場合は、UI の WAN ステータス (**ステータス > WAN**) をチェックして、IP 変更やエラーがないか確認してください。
  - パターンを確認するために、UI 経由で完全なログをエクスポートしてください。
  - 強制的に更新するには： **ネットワーク > WAN** で WAN を解放/更新するか、再起動してください。

これは通常の動作です。ルーターは単に「インターネットのチケット」を有効に保っています。
