私はブログ記事の翻訳を自動化するためにGitHub Actionsを使用しています。最初は、すべての記事を一つのジョブで翻訳し、すべての翻訳が完了してから変更をリポジトリにコミットするように試みました。

楽観的に考え、プロセスが完了することを期待して眠りにつきました。しかし、8時間後、目を覚ますと次のエラーが発生していました：

> GitHub Actions 12のランナーで実行中のジョブが、360分の最大実行時間を超えました。

これは、6時間の翻訳作業が失われたことを意味します。なぜなら、コミットは最後にしか行われなかったからです。

この問題に対処するために、ワークフローを変更し、10ファイルごとに変更をコミットするようにしました。

さらに、マルチスレッドプログラミングを実装して、総翻訳時間を6時間から約1時間に短縮しました。
