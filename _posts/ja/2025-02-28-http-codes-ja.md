Web サーフィンをしているときに「404 Not Found」エラーに出くわし、それが何なのか気になったことはありませんか？あるいは「500 Internal Server Error」に遭遇し、なぜ表示されたのか首をかしげたことは？これらの数字は **HTTP レスポンスコード** であり、インターネットが通信を行う上で重要な役割を果たしています。このブログでは、これらのコードが何であるかを解説し、最も一般的なコードを探求し、あなたが開発者であれ単に好奇心旺盛な Web ユーザーであれ、なぜそれらを理解する価値があるのかを説明します。

## HTTP とは？

まずは基本から始めましょう。**HTTP**、つまり *Hypertext Transfer Protocol* は、World Wide Web 上のデータ交換を支えるシステムです。ブラウザに URL を入力してエンターキーを押すと、ブラウザはその Web サイトをホストしているサーバーに対して **HTTP リクエスト** を送信します。サーバーはその後、**HTTP レスポンス** で返信します。このレスポンスには 3 桁の **ステータスコード** が含まれており、このコードはリクエストが成功したかどうか、そして失敗した場合は何が問題だったのかを伝えます。

---

## HTTP レスポンスコードの 5 つのクラス

HTTP レスポンスコードは、それぞれ特定の目的を持つ 5 つのカテゴリに分類されます。

- **1xx (情報)**: サーバーはリクエストを受け取り、まだ処理中である。
- **2xx (成功)**: リクエストが正常に受信され、理解され、完了した。
- **3xx (リダイレクト)**: 目的のものを得るには、新しい URL へ移動するなどの追加の手順が必要である。
- **4xx (クライアントエラー)**: タイプミスや認証情報の不足など、クライアント側に問題がある。
- **5xx (サーバーエラー)**: サーバー側で問題が発生し、有効なリクエストを処理できなかった。

それでは、遭遇する可能性が高いコードについて詳しく見ていきましょう。

## よくある HTTP レスポンスコードの解説

最も一般的な HTTP レスポンスコードを、明確な例とともにご紹介します。

### 200 OK
- **意味**: リクエストは完全に成功しました。サーバーはリクエストを処理し、要求されたデータを返しました。
- **例**: `www.example.com` のような Web ページが問題なく読み込まれること。それが 200 OK です。

### 201 Created
- **意味**: リクエストが成功し、その結果として新しいリソースが作成されました。
- **例**: ニュースレターに登録するためのフォームを送信し、サーバーがアカウントが作成されたことを確認する。

### 301 Moved Permanently
- **意味**: 要求したリソースは新しい URL に恒久的に移動しており、今後はその新しいアドレスを使用すべきです。
- **例**: ブログ投稿が `oldblog.com/post1` から `newblog.com/post1` に移行し、サーバーがあなたをリダイレクトする。

### 302 Found
- **意味**: リソースは一時的に別の URL にありますが、今後のリクエストでは元の URL を使用し続けてください。
- **例**: サイトのホームページが、一時的にホリデーセールのページにリダイレクトされる。

### 404 Not Found
- **意味**: サーバーは要求したものを見つけることができません。ページが削除されたか、URLが間違っている可能性があります。
- **例**: `www.example.com/oops` と入力し、「oops」が存在しないためエラーページが表示される。

### 403 Forbidden
- **意味**: サーバーは要求内容を理解していますが、権限がないためアクセスを許可しません。
- **例**: ログインせずに非公開の管理画面にアクセスしようとする。

### 401 Unauthorized
- **意味**: 先に進む前に（ログインなどの）認証が必要です。
- **例**: ログインせずに会員制のフォーラムにアクセスしようとする。

### 400 Bad Request
- **意味**: 構文が不正なリクエストや無効なデータのため、サーバーはリクエストを理解できません。
- **例**: メールアドレス欄に「@#$%」のような意味不明な文字列を入力してフォームを送信する。

### 500 Internal Server Error
- **意味**: サーバー側で何かが壊れましたが、具体的に何が原因かは教えてくれません。
- **例**: 開発者が気づかなかったバグにより Web サイトがクラッシュする。

### 503 Service Unavailable
- **意味**: サーバーがダウンしています。メンテナンス中か、過負荷状態である可能性があります。
- **例**: 大規模セール中にオンラインショッピングをしようとすると、「しばらくしてからもう一度試してください」というメッセージが表示される。

## 知っておくと役立つその他のコード

これらのコードはそれほど一般的ではありませんが、十分な頻度で登場するので、言及する価値があります。

- **100 Continue**: サーバーは大きなリクエストの送信に問題ないので、どうぞ送信してください。
- **204 No Content**: リクエストは成功しましたが、返すコンテンツがありません（例：何かを削除した後）。
- **304 Not Modified**: リソースは変更されていないので、キャッシュしているバージョンを使用してください。
- **429 Too Many Requests**: サーバーへのリクエストが多すぎるため、しばらく控えるように言われています（API でよく見られます）。
- **502 Bad Gateway**: 仲介役のサーバーが、アクセスしようとしているメインサーバーから不正なレスポンスを受け取りました。

## HTTP コードを日常の例えで理解する

これらのコードを、現実世界の比較で親しみやすいものにしてみましょう。

- **200 OK**: コーヒーを注文し、好み通りのものが手渡される。
- **201 Created**: オリジナル T シャツを依頼すると、店が「作成中です！」と言う。
- **301 Moved Permanently**: いつものダイナーが街の反対側に移転し、新しい住所を教えてくれる。
- **302 Found**: ダイナーが修理で閉まっているが、近くのフードトラックを案内してくれる。
- **404 Not Found**: 図書館で本を探すが、カタログに載っていない。
- **403 Forbidden**: 招待状なしでプライベートパーティーに押しかけようとする。
- **401 Unauthorized**: ジムを利用しようとするが、会員証を忘れてしまった。
- **400 Bad Request**: ウェイターが理解できない言語で料理を注文する。
- **500 Internal Server Error**: シェフにスープを頼むと、キッチンで火事が発生する。
- **503 Service Unavailable**: ホットラインに電話するが、すべての回線がビジーである。

## なぜ HTTP コードを気にする必要があるのか？

**開発者** にとって、これらのコードは貴重な情報です。問題のデバッグ、エラーの適切な処理、ユーザーを待たせないアプリの構築に役立ちます。例えば、400 エラーと 500 エラーのどちらが発生したかを知ることで、問題がユーザー側にあるのかサーバー側にあるのかを特定できます。

**一般ユーザー** にとって、これらのコードを基本的に理解することで、Web 上の問題の原因がわかるようになります。404 はページがないことを意味し、503 はしばらく待つべきであることを示唆します。それはインターネット用のチートシートを持つようなものです。

さらに、**検索エンジン** は SEO のためにこれらのコードに依存しています。301 リダイレクトはコンテンツを移動する際にサイトのランキングを維持し、404 は Google に対して行き止まりであることを示す可能性があります。

## まとめ

HTTP レスポンスコードは、私たちのリクエストが成功したのか失敗したのかを、Web が伝える方法です。勝利の **200 OK** から恐れられる **500 Internal Server Error** まで、それぞれのコードが物語を語っています。サイトをコーディングしている場合でも、単に閲覧している場合でも、これらの一般的なコードを理解することは、オンラインでの生活をよりスムーズに、そしてより謎の少ないものにすることができます。次に 404 を見かけたときは、それはあなたのせいではなく、Web が「そのページは見つけられません！」と言っているだけだとわかるでしょう。
