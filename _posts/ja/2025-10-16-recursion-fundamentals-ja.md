### 再帰とは何か？

再帰とは、コンピュータサイエンスと数学における基本的な概念であり、関数やプロセスが自身を呼び出すことで、問題をより小さく類似した部分問題に分解して解決する手法です。その核心は、以下の2つの重要な要素に依存しています：

1. **基底ケース**: 再帰を停止させ、無限ループを防ぐ単純な条件です。これは直接解決できる「最小」の問題バージョンです。
2. **再帰ケース**: 関数が修正された（通常はより小さな）入力で自身を呼び出し、基底ケースに向かって進行する部分です。

例えば、5の階乗（5! = 5 × 4 × 3 × 2 × 1 = 120）を計算することを想像してみてください。擬似コードでの再帰的アプローチは以下のようになります：

```
function factorial(n):
    if n == 0 or n == 1:  # 基底ケース
        return 1
    else:                  # 再帰ケース
        return n * factorial(n - 1)
```

`factorial(5)` を呼び出すと、`factorial(4)` を呼び出し、それが `factorial(3)` を呼び出し、というように `factorial(1)` で基底ケースに達するまで続き、その後、戻りながら乗算を行います。この自己参照的な構造は、木の走査、整列されたデータの検索、式の解析といった問題がエレガントに解決できる方法を反映しています。

再帰は分割統治のシナリオ（例：クイックソートアルゴリズム）で威力を発揮しますが、繰り返し呼び出しやスタック空間の使用により、管理が不十分な場合は非効率になる可能性があります。このため、反復（ループベース）バージョンに最適化されることがよくあります。
