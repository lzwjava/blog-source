WindowsでGitリポジトリの複数のブランチを可視化するには、TortoiseGitやその他のツールを使用して選択したブランチのグラフを表示できます。TortoiseGitのRevision Graphは強力ですが、すべてのブランチを表示すると見づらくなる場合があります。以下では、TortoiseGitを使用して特定のブランチのグラフを表示する手順と、より柔軟な代替ツールを紹介します。

### TortoiseGitを使用して複数のブランチを表示する
TortoiseGitのRevision Graphは複数のブランチを表示できますが、インターフェースで特定のブランチを直接選択することはできません。ただし、表示をフィルタリングして関連するブランチに焦点を当てることができます。

1. **Revision Graphを開く**:
   - Windowsエクスプローラーでリポジトリフォルダに移動します。
   - フォルダを右クリックし、**TortoiseGit** > **Revision Graph**を選択します。
   - これにより、すべての参照（ブランチ、タグなど）のグラフがデフォルトで表示されますが、ブランチが多いと煩雑になる可能性があります。

2. **特定のブランチをフィルタリングする**:
   - Revision Graphウィンドウで**フィルターオプション**を使用して表示を整理します：
     - **View**メニューに移動し、**Show branchings and mergings**を選択してブランチの関係を強調表示します。
     - 特定のブランチに焦点を当てるには、コミットを右クリックして**Show Log**を選択し、ログダイアログを表示します。そこで**View > Labels > Local branches**または**Remote branches**を切り替えて関連する参照のみを表示できます。
   - または、ログダイアログで**Walk Behavior > Compressed Graph**オプションを使用してグラフを簡素化し、マージポイントと参照（ブランチ先端など）を持つコミットのみを表示します。

3. **グラフを操作する**:
   - **概要ウィンドウ**を使用して、強調表示された領域をドラッグすることで大きなグラフ内を移動できます。
   - リビジョンノードにマウスを重ねると、日付、作成者、コメントなどの詳細が表示されます。
   - 2つのリビジョンをCtrlキーを押しながらクリックし、コンテキストメニュー（例：**Compare Revisions**）からそれらを比較できます。

4. **制限事項**:
   - TortoiseGitのRevision Graphは、フィルタリングしない限りすべてのブランチを表示し、グラフビューで特定のブランチのみを選択する直接的なオプションはありません。
   - よりクリーンな表示が必要な場合は、以下の代替ツールを検討してください。

### 複数のブランチを表示する代替ツール
TortoiseGitのインターフェースが特定のブランチの選択に制限がある場合、これらのツールを試してみてください。これらはブランチの可視化をより細かく制御できます。

#### 1. **Visual Studio CodeとGit Graph拡張機能**
   - **インストール**: Visual Studio Codeをダウンロードし、**Git Graph**拡張機能をインストールします。
   - **使用方法**:
     - VS Codeでリポジトリを開きます。
     - ソース管理タブまたはコマンドパレット（`Ctrl+Shift+P`を押し、「Git Graph」と入力）からGit Graphビューにアクセスします。
     - インターフェースでブランチ名をクリックして、グラフに表示する特定のブランチを選択します。
     - グラフはコミット、ブランチ、マージを色分けされた線で明確に表示します。
   - **利点**: 軽量、無料、かつ複数のブランチを対話的に選択できます。コミットの比較や基本的なGit操作をサポートします。

#### 2. **SourceTree**
   - **インストール**: Windows用のSourceTree（無料）をダウンロードします。
   - **使用方法**:
     - SourceTreeでリポジトリを開きます。
     - **履歴**ビューはブランチとコミットのグラフィカルな表現を表示します。
     - 左側のブランチリストを使用して特定のブランチの表示を切り替え、見たいブランチのみに焦点を当てます。
     - ブランチやコミットを右クリックしてマージや比較などのアクションを実行します。
   - **利点**: 明確なブランチの可視化と一貫した色分け、ドラッグアンドドロップでのマージなどの対話機能を提供します。

#### 3. **GitKraken**
   - **インストール**: GitKrakenをダウンロードします（オープンソースプロジェクトは無料、プライベートリポジトリは有料）。
   - **使用方法**:
     - GitKrakenでリポジトリを開きます。
     - 中央のグラフはすべてのブランチを表示し、ブランチリストを使用して特定のブランチの表示/非表示を切り替えるオプションがあります。
     - ブランチラベルをクリックして特定のブランチに焦点を当てるか、検索を使用してコミットをフィルタリングします。
   - **利点**: 直感的で視覚的に魅力的、一貫したブランチの色分けと競合解決などの高度な機能を備えています。

#### 4. **`git log`を使用したコマンドライン**
   - ターミナルベースのソリューションを好む場合は、Gitの組み込みグラフビューを使用します：
     ```bash
     git log --graph --oneline --decorate --branches=<branch1> --branches=<branch2>
     ```
     `<branch1>`と`<branch2>`を可視化したいブランチの名前に置き換えます（例：`feature1`、`feature2`）。すべてのブランチには`--branches=*`を、または複数のブランチを明示的に指定します。
   - **便利なエイリアス**:
     簡単にアクセスするためにGitエイリアスを作成します：
     ```bash
     git config --global alias.tree "log --graph --oneline --decorate --branches="
     ```
     その後、`git tree feature1 feature2`を実行してそれらのブランチのみのグラフを表示します。
   - **利点**: 追加のソフトウェアが不要で、正確なブランチを指定できます。出力はテキストベースですが、小さなリポジトリでは明確です。

### 推奨事項
- **シンプルさと統合性のために**: 軽量で無料のソリューションとして**VS CodeとGit Graph**を使用し、特定のブランチを選択できます。
- **高度な可視化のために**: より洗練されたGUIと対話的なブランチ選択、一貫した色分けを提供する**GitKraken**または**SourceTree**を使用します。
- **TortoiseGitユーザーの場合**: Revision Graphを使用し、**Compressed Graph**や**Show branchings and mergings**などのフィルターを使用して主要なブランチに焦点を当てますが、特定のブランチを選択する際の制限に注意してください。
- **ターミナルユーザーの場合**: 追加のツールなしで迅速かつ正確な可視化のために、特定のブランチフィルターを指定した`git log`コマンドを使用します。

ブランチの数が多い場合は、直感的なブランチの切り替えを提供するGitKrakenまたはSourceTreeが、関心のあるブランチのみを選択的に可視化するのに最適なオプションである可能性が高いです。これらのツールのセットアップや詳細な説明が必要な場合はお知らせください！
