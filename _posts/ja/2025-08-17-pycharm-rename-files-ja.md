はい — **PyCharm のリファクタリング機能**（ファイルを右クリック → Refactor → Rename、またはファイル名で `Shift+F6` を押す）を使用すると、ディスク上のファイル名を変更するだけでなく、プロジェクト内で解決可能な**すべての参照と import 文**が更新されます。

いくつか注意すべき重要な詳細があります：

* **リファクタリングの範囲**: PyCharm はプロジェクト全体を分析するため、プロジェクト内の他の Python ファイル内の import 文は自動的に更新されます。
* **解決できない / 動的インポート**: 一部のインポートが動的に構築されている場合（例えば `__import__`、`importlib`、または文字列ベースのモジュールパスを使用している場合）、PyCharm はそれらを検出しないため、更新されません。
* **複数のルート / 外部スクリプト**: 他のスクリプトが PyCharm プロジェクトの外部にある場合、またはインデックス化されていない場合（例：プロジェクトのソースルートにない場合）、PyCharm はそれらも検出しません。
* **テストと相対インポート**: 相対インポート（`from . import something`）を使用していて、フォルダ構造が変更された場合、個別に確認する必要があるかもしれません。

したがって、プロジェクト内の通常の `import mymodule.something` のようなケースでは、PyCharm のリファクタリングを使用して名前を変更することで、すべてが一貫性を保ちます。

名前変更後に**安全にテストする方法**についても説明しましょうか？何も壊れていないことを確認できるように。
