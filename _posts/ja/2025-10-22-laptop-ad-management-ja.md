### Windows の Active Directory (AD) とは？

Active Directory (AD) は、Microsoft が Windows ドメインネットワーク向けに開発したディレクトリサービスです。ユーザーアカウント、コンピューターアカウント、プリンター、共有フォルダーなどのネットワークオブジェクトに関する情報を集中管理するデータベースとして機能します。この階層構造により、管理者は組織内のこれらのリソースへのアクセスを効率的に管理および保護できます。

中核となるコンポーネントは **Active Directory Domain Services (AD DS)** であり、ディレクトリデータの保存と、ユーザーおよび管理者への提供を担当します。主な機能は次のとおりです：
- **セキュリティ統合**: ネットワーク全体での認証とアクセス制御に単一のユーザー名とパスワードを使用します。
- **スキーマ**: オブジェクトタイプ（ユーザー、コンピューターなど）とその属性のルールを定義します。
- **グローバルカタログ**: すべてのディレクトリオブジェクトの検索可能なインデックスで、場所に関係なく迅速な検索を可能にします。
- **レプリケーション**: ドメインコントローラー間で変更を自動的に同期し、データの一貫性を維持します。
- **クエリおよびインデックス機構**: ユーザーやアプリケーションがディレクトリ情報を検索および取得できるようにします。

**AD アカウント** は通常、AD で作成および保存されるユーザーアカウント（またはコンピューターアカウント）を指します。これらのアカウントには、ユーザー名、パスワード、メールアドレス、グループメンバーシップなどの詳細が含まれており、安全なログインとリソースへのアクセスを可能にします。

要するに、AD は Windows 環境における ID と権限の単一の管理ポイントを提供することで IT 管理を簡素化し、個々のマシン上の分散したローカルアカウントに取って代わります。

### Active Directory を使用して従業員のノートパソコンアクセス権を管理する方法

AD はユーザー ID とポリシーを一元管理するため、ノートパソコンのアクセス管理に強力であり、リモートやモバイルデバイスに対しても一貫した適用を保証します。これにより、従業員が完全なローカル管理者権限を持つことを防ぎ（セキュリティリスクを低減）、必要なツールへの制御されたアクセスを許可します。以下にステップバイステップガイドを示します：

1. **AD ドメインをセットアップする**:
   - Windows Server（ドメインコントローラーとして機能）に AD DS をインストールします。
   - Server Manager または PowerShell を使用してドメイン（例: company.local）を作成します。

2. **ノートパソコンをドメインに参加させる**:
   - 各従業員のノートパソコン（Windows 10/11 Pro または Enterprise を実行）で、**設定 > システム > 詳細情報 > ドメインに参加**（または「ファイル名を指定して実行」ダイアログで `sysdm.cpl` を使用）に移動します。
   - ドメイン名を入力し、ドメイン管理者の資格情報を提供して参加させます。
   - ノートパソコンを再起動します。参加後、ノートパソコンはローカルアカウントではなく AD に対して認証を行うようになり、ドメイン全体の管理が可能になります。

3. **ユーザーアカウントを作成および整理する**:
   - ドメインコントローラーで **Active Directory Users and Computers** (dsa.msc) を使用して、従業員のユーザーアカウントを作成します。
   - 権限管理を容易にするために、ユーザーを**セキュリティグループ**（例: "Sales Team" や "Remote Workers"）に割り当てます。ユーザーのプロパティの「所属するグループ」タブからグループを追加します。

4. **アクセス制御のためのグループポリシーを適用する**:
   - **グループポリシーオブジェクト (GPO)** — AD のポリシーエンジン — を使用して、ドメイン参加済みノートパソコンに設定を適用します。
     - ドメインコントローラーで **グループポリシーの管理** (gpmc.msc) を開きます。
     - 新しい GPO（例: "Laptop User Restrictions"）を作成し、ノートパソコンを含む組織単位（OU）にリンクします（AD で "Employee Laptops" などの OU を作成してデバイスをグループ化します）。
     - 設定する一般的なポリシー：
       - **ユーザー権利**: コンピューターの構成 > ポリシー > Windows の設定 > セキュリティの設定 > ローカル ポリシー > ユーザー権利の割り当てで、標準ユーザーから「Administrators」を削除し、ローカル管理者権限の昇格を防ぎます。
       - **ソフトウェア制限**: ソフトウェア制限のポリシーを使用して、許可されていないアプリのインストールをブロックします。
       - **フォルダー/プリンターアクセス**: グループメンバーシップに基づいて NTFS/共有権限を付与します（例: 営業グループは共有ドライブへの読み書き権限を取得）。
       - **リモートアクセス**: ネットワーク外のノートパソコン向けに VPN または条件付きアクセスを構成します。
     - GPO を適用します — 自動的に伝達されます（またはノートパソコンで `gpupdate /force` を実行して強制します）。
   - 高度な制御には、**Windows Admin Center** を使用した **ロールベースのアクセス制御 (RBAC)** を統合します：
     - サーバーに Windows Admin Center をインストールします。
     - ノートパソコンで RBAC を有効にします（Just Enough Administration エンドポイントをインストール）。
     - AD グループを「Administrators」（フルアクセス）や「Readers」（表示のみ）などのロールに割り当てます。

5. **監視および維持する**:
   - Event Viewer や PowerShell（例: `Get-ADUser` や `Get-GPResultantSetOfPolicy`）などのツールを使用してアクセスを監査します。
   - 自動化には、PowerShell で一括変更をスクリプト化します（例: ユーザーをグループに追加）。
   - ハイブリッド設定（例: Microsoft Entra ID との併用）を使用する場合は、AD を同期して多要素認証などのクラウドポリシーを適用します。

この設定により、従業員は AD 資格情報でログインし、グループベースの権限を継承し、制限を回避できなくなり、コンプライアンスとセキュリティに理想的です。混乱を避けるために、テスト用 OU から小規模に始めてください。
