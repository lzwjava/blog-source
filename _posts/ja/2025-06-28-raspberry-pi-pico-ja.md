**Raspberry Pi Pico**は、Raspberry Pi Foundationによって開発された、組み込みシステム、プロトタイピング、DIYエレクトロニクスプロジェクト向けに設計された、多機能で低コストのマイクロコントローラボードです。Raspberry Pi 4のようなシングルボードコンピュータ（SBC）とは異なり、Picoはマイクロコントローラであり、ハードウェアの制御、軽量なプログラムの実行、センサー、モーター、その他のコンポーネントとのインターフェースに特化していることを意味します。2021年1月に発売され、Foundationのマイクロコントローラ市場への参入を果たし、ホビイスト、教育者、プロフェッショナル向けに手頃な価格でありながら強力なプラットフォームを提供しました。

以下は、Raspberry Pi Picoの機能、仕様、プログラミング能力、アプリケーションなどを網羅した包括的な紹介です。

---

### **概要**
Raspberry Pi Picoは、Raspberry Pi Foundationが独自設計したマイクロコントローラチップ**RP2040**を中心に構築されています。約4米ドルという価格帯で、ArduinoやESP32などのプラットフォームと競合しますが、その高性能、低コスト、そして広範なコミュニティサポートにより際立っています。Picoはコンパクトで（サイズはわずか51mm x 21mm）、単純なLED点滅から複雑なIoT、ロボットアプリケーションまで、初心者から上級ユーザーまでを対象に設計されています。

---

### **主な特徴**
1. **RP2040 マイクロコントローラ**:
   - 最大**133 MHz**（オーバークロック可能）で動作するデュアルコアの**Arm Cortex-M0+**プロセッサ。
   - **264 KB SRAM**とプログラム保存用の**2 MBオンボードQSPIフラッシュメモリ**。
   - バッテリー駆動アプリケーション向けのスリープおよび休止モードによる低消費電力。
   - パフォーマンス最適化のための柔軟なクロック設定。

2. **GPIOピン**:
   - 26本の多機能**汎用入出力（GPIO）**ピン。
   - 周辺機器接続のための**I2C**、**SPI**、**UART**、**PWM**インターフェースをサポート。
   - 2x UART、2x SPIコントローラ、2x I2Cコントローラ、16x PWMチャネル。
   - アナログセンサー入力用の3x 12ビットADC。
   - カスタムプロトコル用の8x プログラム可能I/O（PIO）ブロック（例：WS2812 LED制御、VGA出力）。

3. **電源と接続性**:
   - **USB micro-B**（5V）または外部電源（1.8–5.5V）経由で給電。
   - GPIOピン用の**3.3V ロジックレベル**。
   - RP2040上の内蔵**温度センサー**。
   - プログラミングとデバッグに使用されるUSB 1.1コントローラ（デバイス/ホストモード対応）。

4. **物理的設計**:
   - コンパクトサイズ: 51mm x 21mm。
   - 40ピンDIPスタイルのレイアウトと**カステレーション加工エッジ**により、基板にはんだ付けしたりブレッドボードで使用したり可能。
   - 片面実装により、はんだ付けが容易。

5. **低コスト**:
   - 約4米ドルと、利用可能な最も手頃な価格のマイクロコントローラの一つ。

### **バリエーション**
発売以来、Raspberry Pi FoundationとパートナーはPicoのバリエーションをリリースしています：
- **Raspberry Pi Pico W** (2022): Infineon CYW43439チップによる**Wi-Fi**（2.4 GHz 802.11n）と**Bluetooth 5.2**を追加し、ワイヤレスIoTアプリケーションを可能に。価格は約6米ドル。
- **Raspberry Pi Pico H**: プロトタイピングを容易にするための、事前はんだ付け済みの40ピンヘッダーを付属。
- **Raspberry Pi Pico WH**: Pico Wのワイヤレス機能と事前はんだ付け済みヘッダーを組み合わせ。
- **Pico 2** (2024): **RP2350**マイクロコントローラを搭載。これはRP2040のアップグレード版で、デュアルコアの**Arm Cortex-M33**または**RISC-V Hazard3**（ユーザー選択可能）、520 KB SRAM、改善された電力効率、および強化されたセキュリティ機能（Arm TrustZone、SHA-256アクセラレーションなど）を特徴とする。

### **Raspberry Pi Picoのプログラミング**
Picoは複数のプログラミング言語と環境をサポートしており、幅広いユーザーが利用できます：

1. **MicroPython**:
   - 初心者と迅速なプロトタイピングにおける最も人気な選択肢。
   - Raspberry Pi Foundationにより提供される公式MicroPythonファームウェア。
   - GPIO、I2C、SPI、PWM、ADC、PIO用のライブラリをサポート。
   - USB経由の対話型REPLによるリアルタイムコーディング。

2. **C/C++**:
   - 公式**Pico SDK**を使用してRP2040の機能を完全に制御。
   - パフォーマンスがクリティカルなアプリケーションと低レベルハードウェア制御に適している。
   - PIOプログラミングやマルチコア処理などの高度な機能をサポート。
   - コンパイルにはCMakeやGCCなどのツールを使用。

3. **その他の言語**:
   - **CircuitPython**: AdafruitによるMicroPythonのフォーク。教育と使いやすさに最適化。
   - **Rust**: RP2040でのRustプログラミングのためのコミュニティ駆動のサポート。
   - **Arduino**: 公式RP2040ボードパッケージを使用してArduino IDEでプログラミング可能。
   - JavaScript（Espruino経由）やLuaなどの他の言語の実験的サポート。

4. **開発ツール**:
   - **ドラッグ＆ドロッププログラミング**: BOOTSELボタンを押しながら接続することで、MicroPythonやCircuitPythonの.uf2ファームウェアファイルをUSB経由でアップロード。
   - **デバッグ**: Raspberry Pi Debug Probeなどのツールを使った高度なデバッグのためにSWDをサポート。
   - **Thonny**（Python用）や**Visual Studio Code**（C/C++用）などの統合開発環境が一般的に使用される。

### **アプリケーション**
Raspberry Pi Picoの柔軟性は、以下のような幅広いプロジェクトに適しています：
- **プロトタイピングと教育**: 組み込みシステム、プログラミング、電子工学の学習に理想的。
- **IoTプロジェクト**: Pico Wを使用して、スマートホームコントローラや気象観測所などのWi-Fi対応デバイスを作成可能。
- **ロボティクス**: ロボットアプリケーション向けにモーター、サーボ、センサーを制御。
- **カスタムインターフェース**: PIOを使用して、WS2812（NeoPixel）LED制御、VGA、DVI出力などのプロトコルを実装。
- **データロギング**: 環境モニタリングのためのセンサー（温度、湿度、光など）とインターフェース。
- **ウェアラブルと組み込みシステム**: コンパクトサイズと低消費電力が、ウェアラブル技術やバッテリー駆動デバイスに適している。

### **エコシステムとコミュニティ**
Raspberry Pi Picoは堅牢なエコシステムの恩恵を受けています：
- **公式ドキュメント**: Raspberry Pi Foundationが、*Pico Getting Started*ガイド、RP2040データシート、ハードウェア設計ファイルを含む詳細なガイドを提供。
- **コミュニティサポート**: X、Reddit、Raspberry Piフォーラムなどのプラットフォームで、プロジェクト、チュートリアル、トラブルシューティングのヒントを共有する大規模なコミュニティが存在。
- **サードパーティ製アクセサリ**: Adafruit、SparkFun、Pimoroniなどの企業から、センサーブレイクアウトボード、ディスプレイ、シールドなど多数のアドオンが利用可能。
- **オープンソースハードウェア**: RP2040の設計は十分に文書化されており、カスタムボードの開発を促進。

### **代替品との比較**
- **Arduino**: Picoはより高速（デュアルコア、133 MHz vs. Arduino Unoの16 MHz）で安価、かつより多くのGPIOとPIOのような高度な機能を備える。ただし、Arduinoはシールドとライブラリのより大きなエコシステムを持つ。
- **ESP32**: ESP32は内蔵Wi-FiとBluetoothを提供するが、Pico Wはより低コストでこれを実現。PicoのPIOはカスタムプロトコルにおいてユニーク。
- **STM32**: Picoは初心者にとって（MicroPythonなどで）プログラムが容易で、多くのSTM32ボードよりも手頃な価格。

---

### **制限事項**
- **内蔵ワイヤレスなし（基本モデル）**: オリジナルのPicoはWi-Fi/Bluetoothを欠いている（Pico Wで対応）。
- **オンボードストレージの制限**: 2MBフラッシュはほとんどのプロジェクトに十分だが、複雑なアプリケーションでは制限となる可能性がある。
- **3.3V ロジック**: 5V周辺機器にはレベルシフタが必要。
- **内蔵リセットボタンなし**: 一部のユーザーには少し不便。

### **はじめに**
Raspberry Pi Picoを始めるには：
1. **ハードウェアセットアップ**: PicoをUSB経由でコンピュータに接続するか、外部電源で給電。
2. **ファームウェアのインストール**: Raspberry PiのウェブサイトまたはAdafruitからMicroPythonやCircuitPythonのファームウェアをダウンロードし、BOOTSELボタンを押しながらPicoを接続してアップロード。
3. **プログラミング**: PythonにはThonnyを、C/C++開発にはPico SDKをセットアップして使用。
4. **プロジェクト**: LEDの点滅やセンサーの読み取りなどの簡単なプロジェクトから始め、IoTやロボティクスなどの高度なアプリケーションを探求。
