### IBM WebSphere Application Server (WAS) 管理コンソール: 左側ナビゲーションペイン

IBM WebSphere Application Server (WAS) 管理コンソール (統合ソリューションコンソールとも呼ばれる) の左側ビューは、設定と管理タスクを整理する階層型ナビゲーションツリーです。サーバー環境、アプリケーション、リソースを管理するための主要な領域へのクイックアクセスを提供します。正確な構造は WAS のバージョン (例: 8.5, 9.0) やエディション (Base と Network Deployment) によって若干異なりますが、コアとなる最上位カテゴリは一貫しています。

以下は、ナビゲーションツリーの主要な最上位セクションの一覧と、その主な目的の簡単な説明です。サブセクションは (+/− アイコンを使用して) 展開可能で、より詳細なタスクにアクセスできます。

#### 主な最上位セクション
- **アプリケーション**  
  アプリケーション (例: EAR/WAR ファイル) のデプロイ、インストール、更新、開始/停止、および管理に使用されます。  
  *主なサブセクション*: エンタープライズアプリケーション、WebSphere エンタープライズアプリケーション、Web モジュール、共有ライブラリ。  
  *一般的なタスク*: 新しいアプリのインストール、モジュールのサーバーへのマッピング、クラスローダーの設定。

- **リソース**  
  アプリケーションが使用できるデータベース、メッセージング、接続プールなどの共有リソースを設定します。  
  *主なサブセクション*: JDBC (データソース/プロバイダー)、JMS (キュー/トピック)、JavaMail セッション、URL プロバイダー。  
  *一般的なタスク*: JDBC データソースの設定、JMS 接続ファクトリーの作成。

- **サービス**  
  セキュリティ、メッセージング、通信プロトコルなどのサーバー全体のサービスを管理します。  
  *主なサブセクション*: セキュリティ (グローバルセキュリティ、ユーザー/グループ、認証)、メールプロバイダー、ポート、ORB サービス、トランザクションサービス。  
  *一般的なタスク*: SSL の有効化、ユーザーレジストリの設定、ポート割り当ての調整。

- **サーバー**  
  サーバーインスタンス、クラスタリング、および Web サーバー定義を扱います。  
  *主なサブセクション*: サーバータイプ (WebSphere アプリケーションサーバー、WebSphere プロキシーサーバー)、クラスター、Web サーバー。  
  *一般的なタスク*: サーバーの開始/停止、JVM 設定の構成、高可用性のためのクラスターの作成。

- **システム管理**  
  ノード、セル、コンソール設定を含む全体のトポロジーを管理します。  
  *主なサブセクション*: ノード、セル、デプロイメントマネージャー、コンソール設定。  
  *一般的なタスク*: ノードの同期、デプロイメントマネージャーの管理、コンソールのテーマ/スコープの設定。

- **環境**  
  環境固有の変数とエイリアスを定義します。  
  *主なサブセクション*: WebSphere 変数、ホストエイリアス。  
  *一般的なタスク*: パスやプロパティのためのシステム変数の設定。

- **モニタリングとチューニング**  
  パフォーマンスを監視し、ランタイムの動作を調整します。  
  *主なサブセクション*: パフォーマンスモニタリングインフラストラクチャー (PMI)、カスタムサービス、ロギングとトレース。  
  *一般的なタスク*: メトリクスの有効化、スレッドプールの調整、リクエストメトリクスの表示。

- **トラブルシューティング**  
  診断ツールとログにアクセスします。  
  *主なサブセクション*: ログとトレース、ファーストステップ。  
  *一般的なタスク*: ログのダウンロード、トレース仕様の有効化。

#### 追加の注意点
- **スコープ選択**: 多くのページの上部で、スコープ (セル、ノード、サーバー) でフィルタリングして表示を制限できます。
- **ガイド付きアクティビティ**: ウィザード (例: 環境のセキュア化) のために、最上位またはコンテキスト依存のセクションとして表示されることがあります。
- コンソールへのアクセス方法: 通常は `https://<ホスト>:<ポート>/ibm/console` から (管理用デフォルトポート 9060、セキュア用 9043)。
- バージョン固有の詳細 (例: WAS 9.0) については、マイナーな UI 変更が更新で行われるため、公式 IBM 文書を参照してください。
