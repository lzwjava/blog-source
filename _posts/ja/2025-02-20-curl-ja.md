以下は複数の `curl` の使用例をまとめた bash スクリプトです：

```bash
#!/bin/bash

# 基本的な GET リクエスト
echo "基本的な GET リクエスト："
curl http://example.com

# カスタムヘッダー付き GET
echo "\nカスタムヘッダー付き GET リクエスト："
curl -H "User-Agent: MyUserAgent/1.0" http://example.com

# POST リクエスト
echo "\nPOST リクエスト："
curl -X POST -d "key1=value1&key2=value2" http://example.com/post

# JSON データでの POST
echo "\nJSON データでの POST："
curl -X POST -H "Content-Type: application/json" -d "{\"key1\":\"value1\",\"key2\":\"value2\"}" http://example.com/post

# プロキシの使用
echo "\nプロキシの使用："
curl -x http://proxy.example.com:8080 http://example.com

# 認証
echo "\n基本認証："
curl -u username:password http://example.com/auth

# ファイルのダウンロード
echo "\nファイルのダウンロード："
curl -O http://example.com/file.zip

# リダイレクトの追従
echo "\nリダイレクトの追従："
curl -L http://example.com/redirect

# 出力をファイルに保存
echo "\n出力をファイルに保存："
curl http://example.com -o output.txt

# カスタム HTTP メソッド
echo "\nカスタム HTTP メソッド (DELETE)："
curl -X DELETE http://example.com/delete/resource

# 詳細出力
echo "\n詳細出力："
curl -v http://example.com

# 接続タイムアウト
echo "\n接続タイムアウト："
curl --connect-timeout 5 http://example.com

echo "\nスクリプトの実行が完了しました。"
```

このスクリプトを `.sh` 拡張子で保存してください（例：`curl_examples.sh`）。Unix 系環境または Windows Subsystem for Linux (WSL) で実行する場合は、実行可能ファイルに設定してください：

```sh
chmod +x curl_examples.sh
```

その後、実行できます：

```sh
./curl_examples.sh
```

**Windows での注意事項：**
- Windows Cmd または PowerShell で実行する場合、WSL がインストールされている場合は `bash` または `wsl` を使用するか、直接実行するために `curl` が PATH にあることを確認してください：
  - `bash curl_examples.sh` または `wsl ./curl_examples.sh`
- このスクリプトは bash 構文を使用しているため、WSL または bash 環境を介して実行しない限り、Windows コマンドプロンプトや PowerShell とはネイティブに互換性がありません。ただし、`curl` コマンド自体は Windows 版の `curl` と互換性があります。
