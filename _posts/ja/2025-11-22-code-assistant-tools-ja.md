強力なコードアシスタント（Claude Codeなど）における典型的な **ツール呼び出し** または **アクション** は、実際のコーディング環境と対話し、人間の開発者が行うタスクを実行できるようにすることを中心に構成されています。

これらのツール呼び出しは、いくつかの主要なカテゴリに分類されます：

### 1. ファイルシステムとコードベースの対話

アシスタントがプロジェクトを理解し、既存のコードを読み、変更を加えるために不可欠です。

| ツール/アクション | 典型的な使用例 |
| :--- | :--- |
| **`read_file`**（または`@filename`のようなファイル参照） | 特定のファイルの内容を取得して、そのロジック、依存関係、またはコンテキストを理解する。 |
| **`list_directory`** | ファイル構造やディレクトリ内のファイルリストを取得して、関連するモジュールを特定したり、特定のファイルを見つけたりする。 |
| **`edit_file` / `write_file`** | ソリューションの実装、リファクタリング、機能の追加、バグの修正を行うためのコアアクション。 |
| **`create_file`** | 新しいテストファイル、設定ファイル、新しいコンポーネントなどの新しいファイルを作成する。 |
| **`search_files`** | 特定の文字列（関数名、クラス名、エラーメッセージなど）を含むコードベース内のすべてのファイルを検索する。 |

### 2. 実行とデバッグ

作業を検証し、エラーを修正し、リアルタイムのフィードバックを得るために、アシスタントはコマンドを実行する必要があります。

| ツール/アクション | 典型的な使用例 |
| :--- | :--- |
| **`bash` / `run_command`** | ビルドの実行（`npm build`）、テストの実行（`pytest`、`npm test`）、コードのリント（`eslint`）、スクリプトの実行などのシェルコマンドを実行する。 |
| **`code_interpreter`** | サンドボックス環境で小さなコードスニペットを実行して、何かを素早く計算したり、ライブラリ関数をテストしたり、小さなデータセットを分析したりする。 |

### 3. バージョン管理（Git）

自律的なコーディングアシスタントは、Gitと対話することで開発ワークフローを管理できます。

| ツール/アクション | 典型的な使用例 |
| :--- | :--- |
| **`git status`** | 作業ディレクトリの現在の状態を確認する。 |
| **`git diff`** | 行われた変更を確認して、自身の作業をレビューする。 |
| **`git commit`** | 生成されたコミットメッセージとともに変更をリポジトリに保存する。 |
| **`git checkout`** | ブランチの切り替えや、コミットされていない変更の破棄を行う。 |
| **`create_pull_request`** | 機能や修正の最終ステップを自動化する。 |

### 4. 外部情報

最新の情報、ドキュメント、または外部サービスへの接続のために、特殊なツールが使用されます。

| ツール/アクション | 典型的な使用例 |
| :--- | :--- |
| **`web_search`** | 最新の構文を調べたり、公式APIドキュメントを読んだり、特定のエラーのトラブルシューティングガイドを見つけたりする。 |
| **Custom APIs / MCP Servers** | Model Context Protocol（MCP）を使用して外部サービスに接続する。Google Driveからデータを取得したり、データベース（PostgreSQL）と対話したり、コミュニケーションツール（Slack）にメッセージを送信したりする。 |

本質的に、これらの「ツール呼び出し」により、大規模言語モデル（LLM）は、テキスト生成器から、環境を観察（ファイルの読み取り、コマンドの実行）、思考（作業の計画）、行動（ファイルの編集、コードのコミット）できる完全な**エージェント**へと変貌することができます。
