#### YouTube：プログラムによるアップロードとポッドキャスト統合

YouTubeは、特に動画アップロードをサポートする堅牢なAPI、具体的にはYouTube Data APIを提供しています。ただし、YouTubeは主に動画コンテンツを扱うため、音声ファイルをアップロードするには動画形式に変換する必要があります。このプロセスでは、FFmpegなどのツールを使用して音声ファイルと静止画を結合し、YouTubeが処理できる動画ファイル（例：MP4）を作成します。この方法は、ポッドキャストの各エピソードをポッドキャストの番組アートのような静止画付きの動画として表現できるポッドキャストアップロードに特に関連性があります。

YouTube Data APIの `videos.insert` メソッドはプログラムによるアップロードを可能にし、バッチ処理の自動化を実現します。例えば、スクリプトが100個の音声ファイルを繰り返し処理し、それぞれを動画に変換し、APIを使用してアップロードすることができます。ドキュメントによると、アップロードされるファイルはファイルサイズや形式などの特定の制約に準拠する必要があり、アクセスにはOAuth 2.0による認証が必要です ([Upload a Video | YouTube Data API | Google for Developers](https://developers.google.com/youtube/v3/guides/uploading_a_video))。このアプローチは、YouTubeがプレイリストをポッドキャストとして自動的に分類し、エピソードを動画として扱うため、ポッドキャストに実行可能です。

ポッドキャスト作成者の場合、YouTubeにRSSフィードを送信することでプロセスを自動化できます。この場合、YouTubeは番組アートを使用して音声エピソードから動画を作成します。ただし、直接APIアップロードの場合、変換ステップが必要であり、APIはタイトル、説明、プライバシー状況などのメタデータの設定をサポートしており、バッチアップロードの使いやすさを向上させています。

#### Spotify：アップロード用の公開APIの欠如

対照的に、Spotifyは、音楽またはポッドキャストエピソードのいずれであっても、音声ファイルをアップロードするための公開APIを提供していません。Spotify Web APIは、メタデータの取得、プレイリストの管理、再生の制御を目的として設計されており、コンテンツの投稿用ではありません ([Web API | Spotify for Developers](https://developer.spotify.com/documentation/web-api))。ポッドキャスト配信者の場合、Spotify for Creatorsはエピソードをアップロードするためのユーザーインターフェースを提供し、MP3、M4A、WAVなどの形式をサポートしていますが、これは手動であり、スクリプト化できません ([Publishing audio episodes - Spotify](https://support.spotify.com/us/creators/article/publishing-audio-episodes/))。

ミュージシャンの場合、DistroKid、TuneCore、Record Unionなどの配信サービスがSpotifyへのアップロードを容易にしますが、これらのサービスは外部開発者向けの公開APIを提供していません。DistroKidのドキュメントとサポートセンターの調査では、バッチアップロードのためのAPIについて言及されておらず、自動化がサポートされていないことが示されています ([DistroKid Help Center](https://support.distrokid.com/hc/en-us))。この制限はバッチアップロードにとって重要であり、ユーザーはプラットフォームのWebインターフェースに依存する必要があり、100本の音声には非現実的です。

興味深い観察として、GitHub上のDistroKid用のGolangラッパーのような非公式のAPIラッパーの存在は、リバースエンジニアリングの努力を示唆しています。ただし、これらは公式ではなく、利用規約に違反する可能性があり、実際の使用には信頼性がありません ([distrokid · GitHub Topics · GitHub](https://github.com/topics/distrokid))。
