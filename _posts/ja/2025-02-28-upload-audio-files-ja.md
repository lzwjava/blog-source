#### YouTube：プログラムによるアップロードとポッドキャストの統合

YouTubeは、YouTube Data APIを提供しており、動画のアップロードをサポートしています。しかし、YouTubeは主に動画コンテンツを扱うため、音声ファイルは動画形式に変換してアップロードする必要があります。このプロセスには、FFmpegなどのツールを使用して音声ファイルを静的画像と組み合わせて動画ファイル（例：MP4）を作成することが含まれます。この方法は、各エピソードを静的画像（例：ポッドキャストのショーアート）を使用して動画として表示するポッドキャストに特に関連します。

YouTube Data APIの`videos.insert`メソッドを使用してプログラムによるアップロードが可能であり、一括処理の自動化が可能です。例えば、スクリプトは100の音声ファイルを反復処理し、各ファイルを動画に変換し、APIを使用してアップロードできます。ドキュメントには、アップロードされるファイルが特定の制約（例：ファイルサイズと形式）に準拠し、OAuth 2.0を使用してアクセスを承認する必要があることが記載されています（[Upload a Video | YouTube Data API | Google for Developers](https://developers.google.com/youtube/v3/guides/uploading_a_video)）。このアプローチはポッドキャストに適しており、YouTubeはプレイリストをポッドキャストとして自動的に分類し、エピソードを動画として扱います。

ポッドキャストクリエイターは、YouTubeにRSSフィードを送信することでプロセスを自動化でき、YouTubeは音声エピソードから動画を作成し、ショーアートを使用します。しかし、直接のAPIアップロードの場合、変換ステップが必要であり、APIはタイトル、説明、プライバシー状態などのメタデータを設定することができ、一括アップロードの使用性を向上させます。

#### Spotify：アップロード用の公式APIがない

一方、Spotifyには、音楽やポッドキャストエピソードの音声ファイルをアップロードするための公式APIはありません。Spotify Web APIは、メタデータの取得、プレイリストの管理、再生の制御に設計されており、コンテンツの提出には使用されません（[Web API | Spotify for Developers](https://developer.spotify.com/documentation/web-api)）。ポッドキャスターの場合、Spotify for Creatorsは、MP3、M4A、WAVなどの形式をサポートするエピソードのアップロード用のユーザーインターフェースを提供しますが、これは手動で行う必要があり、スクリプト化できません（[Publishing audio episodes - Spotify](https://support.spotify.com/us/creators/article/publishing-audio-episodes/)）。

ミュージシャンの場合、DistroKid、TuneCore、Record Unionなどの配布サービスはSpotifyへのアップロードを促進しますが、これらのサービスは外部開発者向けの公式APIを提供していません。DistroKidのドキュメントとサポートセンターの調査では、一括アップロード用のAPIに関する記載が見つかりませんでした。これにより、自動化はサポートされていないことが示されています（[DistroKid Help Center](https://support.distrokid.com/hc/en-us)）。この制限は、ユーザーがプラットフォームのWebインターフェースに依存するため、100の音声ファイルの一括アップロードには実用的ではありません。

興味深い観察点として、非公式のAPIラッパー（例：GitHubのDistroKid用Golangラッパー）が存在することが挙げられます。これらのラッパーは公式ではなく、サービス条項に違反する可能性があるため、生産用には信頼性がありません（[distrokid · GitHub Topics · GitHub](https://github.com/topics/distrokid)）。
