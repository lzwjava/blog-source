IntelliJ IDEA には、ワイルドカードの使用を防止するなど、インポートの処理方法を制御する設定があります。常に単一クラスのインポートを使用するように設定する方法は以下の通りです：

1.  **設定/環境設定を開く：**
    * **Windows/Linux:** `Ctrl + Alt + S`
    * **macOS:** `Cmd + ,` (コンマ)

2.  **Java のコードスタイルに移動：**
    * 設定ダイアログの左側のペインで、以下に移動します：
        `Editor` -> `Code Style` -> `Java`

3.  **「Imports」タブに移動：**
    * 右側で、`Imports` タブを選択します。

4.  **インポート設定を構成：**
    ここにはいくつかの重要な設定があります：

    * **`Use single class import`**: **このチェックボックスをオンにします。** これは、IntelliJ にワイルドカードインポートよりも個々のクラスのインポートを優先するように指示する主要な設定です。

    * **`Class count to use import with '*'`**: この設定は、IntelliJ が自動的にワイルドカードインポート（例: `java.util.*`）に切り替わる前に、単一のパッケージからインポートする必要があるクラスの数を決定します。ワイルドカードインポートを効果的に無効にするには、**この値を非常に高い数値（`999` や `9999` など）に設定します。** これにより、ワイルドカードインポートの閾値に達することはほぼなくなります。

    * **`Names count to use static import with '*'`**: これは上記と同様ですが、静的インポート用です。ワイルドカードの静的インポートを防ぐため、これも高い数値（例: `999` や `9999`）に設定します。

    * **`Packages to Use Imports with '*'`**: このテーブルは、他の設定に関係なく IntelliJ が*常に*ワイルドカードインポートを使用するパッケージをリストします。ワイルドカードインポートを完全に回避したい場合は、**このテーブルが空であることを確認してください。** `java.util.*` や `java.awt.*` のようなエントリが表示された場合は、それらを削除します。

5.  **変更を適用：**
    * `Apply` をクリックし、次に `OK` をクリックして変更を保存します。

**これらの変更を行った後：**

* 「自動インポート」機能（例：未解決のクラスで `Alt + Enter` を押す）を使用すると、IntelliJ はワイルドカードではなく個々のクラスインポートを挿入するようになります。
* ワイルドカードインポートが含まれる既存のファイルがある場合、「インポートの最適化」アクション（Windows/Linux では `Ctrl + Alt + O`、macOS では `Cmd + Option + O`）を使用して、新しい設定に従ってそれらを単一クラスのインポートに変換できます。

これらの手順に従うことで、Java ファイルに対して IntelliJ IDEA がワイルドカードインポートを使用するのを効果的に防止できます。
