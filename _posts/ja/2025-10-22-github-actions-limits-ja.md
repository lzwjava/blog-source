### GitHub Actions の利用制限

GitHub Actions には、ご利用のプラン（個人アカウントまたは Organization）に応じた段階的制限があります。これには、プライベートリポジトリでの GitHub ホストランナーの無料月間分数と、アーティファクトのストレージが含まれます。パブリックリポジトリとセルフホストランナーはこれらの分数を消費しません。分数の制限は毎月リセットされます（ストレージは継続的）。有効な支払い方法が登録されている場合、制限を超えると自動的に超過料金が請求されます。支払い方法がない場合、制限に達するとジョブはブロックされます。

#### プラン別の付与分数とストレージ

| プラン                          | ストレージ | 分数 (月間) |
|-------------------------------|---------|---------------------|
| GitHub Free (個人/Organization) | 500 MB | 2,000              |
| GitHub Pro (個人)              | 1 GB   | 3,000              |
| GitHub Team (Organization)    | 2 GB   | 3,000              |
| GitHub Enterprise Cloud (Organization)| 50 GB  | 50,000             |

- **分数**: GitHub ホストランナーでのジョブ総実行時間をカウントします（失敗したジョブの部分的な時間も含む）。乗数が適用されます: Linux (1倍), Windows (2倍), macOS (10倍)。リポジトリへの書き込み権限を持つユーザーは、リポジトリ所有者の割り当てを使用します。
- **ストレージ**: アーティファクトストレージの GB-時間に基づきます（例: アップロード/ダウンロード）。ログとサマリーはカウントされません。

#### 超過料金
クォータを超えた場合:
- **分数**: 追加分数ごとに課金されます（標準 Linux ランナーの USD レート: $0.008/分; Windows/macOS は乗数により高額）。
- **ストレージ**: GB-月単位で課金されます（総 GB-時間 / 月 744 時間、切り上げ）。
- 例: Team プランで、5,000 超過分数 (3,000 Linux + 2,000 Windows) の場合、コストは約 $56。

支払い方法がない場合、利用はクォータで停止します。表示されたエラー（"最近のアカウント支払いに失敗したか、支出制限を増やす必要があります"）は、支払いが拒否されたか、**支出制限（予算）** が超過料金を制限していることを意味します。

#### 支出制限（予算）
- これらは、Actions（および他の従量制機能）に対する月間総支出を管理するための任意のしきい値です。
- 低く設定しすぎると、クォータに達する前にジョブがブロックされます。
- **確認/増やす方法**:
  1. GitHub 設定 > 請求とプラン に移動します。
  2. 「予算とアラート」で、GitHub Actions の予算を表示または編集します。
  3. 制限を増やす、支払い方法を追加する、または予算を完全に削除します（無制限の超過が可能ですが、アラートで監視してください）。
- 移行されたアカウントでは、古い制限から自動生成された予算が設定されている場合があります。利用状況に合わせて更新してください。

詳細については、アカウントの請求ダッシュボードでリアルタイムの利用状況を確認してください。
