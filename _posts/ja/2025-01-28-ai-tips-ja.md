* コード例を Cursor にコピーし、コメントを使用してコンテキストを提供します。

* 香港地域では、DeepSeek または Mistral サービスを使用することは適切ですが、ChatGPT、Claude、または Gemini を使用することは適切ではありません。

* リージョンの API ポリシーは、そのアプリのポリシーとほぼ同じです。

* Cursor を Visual Studio Code の代わりに使用します。

* Visual Studio Code を使用する必要がある場合もあります。例えば、git マージ エディターのシナリオです。この場合、私はまだ `git config --global core.editor "code --wait"` を使用しています。

* Deepseek V3 リリースの日から、AI ツールにサブスクライブする必要はありません。

* Gemini または Grok を使用して、プロンプト「文言を含む楽しい農暦新年イメージを生成してください」などのプロンプトを使用して、祭りの祝賀イメージを生成します。

* ときには、AI モデルにオリジナルのテキストを提供して表を作成しても、出力のいくつかの場所が入力と異なる場合があります。例えば、Cursor で Deepseek V3 モデルを使用して pip list の表を生成すると、`1.极狐0` などのバージョンを含むことがあります。ここで、`极狐` は中国の GitLab プラットフォームを意味します。

* Deepseek または Mistral API を使用して、プロンプト「あなたは専門の翻訳者です。英語から中国語に Jekyll ブログ記事の Markdown ファイルを翻訳しています。 {text}」などのプロンプトを使用してタイトルを翻訳すると、誤った翻訳につながることがあります。提供するテキスト以外にも、出力には過剰な翻訳が含まれることが多いです。

* Cursor の AI モデルが一部正しいテキストを提供することがありますが、追加の指示を与えて正しい部分を再生成させることで、受け入れられることがあります。
