2台目のラップトップ（Nginxサーバーを搭載したUbuntuを実行）を使ってモニターで映画を視聴するには、手間をかけずに済む方法が最適です。Fire TV StickやApple TVのような追加デバイスの購入は避けたいところです。長いHDMIケーブルも使えますが、動画の選択や再生のためにラップトップを操作し続けるのは避けたいでしょう。既存のセットアップを使ってシンプルで便利なソリューションを設定しましょう。

映画がUbuntuラップトップに保存されており、すでにNginxサーバーが実行されているので、最も簡単な方法は、そのラップトップをHDMIでモニターに直接接続し、再生コントロールを簡単にすることです。以下の手順で設定できます。

### 解決策：ラップトップをモニターに接続し、VLCをリモートコントロールで使用する

#### 必要なもの
- **HDMIケーブル**: Ubuntuラップトップをモニターに接続するため。
- **VLCメディアプレイヤー**: Ubuntuに既にインストールされている可能性が高い。または簡単にインストール可能。
- **スマートフォンまたは別のデバイス**: 自宅のネットワーク経由で再生をリモートコントロールするため。

#### 設定手順
1. **ラップトップをモニターに接続する**
   - HDMIケーブルの一方をUbuntuラップトップのHDMIポートに、もう一方をモニターのHDMI入力に差し込みます。
   - モニターの電源を入れ、正しいHDMI入力を選択します。ラップトップのディスプレイがモニターに表示されるはずです（ビデオとオーディオ両方。モニターにスピーカーがない場合は、ラップトップのスピーカーまたは外部スピーカーを使用してください）。

2. **VLCをインストールする（まだインストールされていない場合）**
   - Ubuntuラップトップでターミナルを開き、以下を実行します：
     ```
     sudo apt update
     sudo apt install vlc
     ```
   - これにより、多機能なメディアプレイヤーであるVLCを使用できるようになります。

3. **リモートコントロールのためにVLCのWebインターフェースを有効にする**
   - UbuntuラップトップでVLCを開きます。
   - **ツール > 設定**に移動します。
   - 左下の **「すべて」** をクリックして詳細設定を表示します。
   - **インターフェース > メインインターフェース**に移動し、**「Web」** ボックスにチェックを入れてHTTPインターフェースを有効にします。
   - **インターフェース > メインインターフェース > Lua**に移動し、**Lua HTTP パスワード**フィールドにパスワード（例: "mypassword"）を設定します。
   - **保存**をクリックし、VLCを再起動します。

4. **VLCに映画を読み込む**
   - VLCで、**再生 > プレイリスト**に移動します。
   - ラップトップに保存されている映画ファイルをフォルダからVLCプレイリストにドラッグ＆ドロップするか、**メディア > ファイルを開く**を使って一つずつ追加します。
   - プレイリスト（例: "My Movies"）を**Ctrl+Y**で保存し、後で簡単にアクセスできるようにします。

5. **ラップトップのIPアドレスを確認する**
   - ターミナルで以下を入力します：
     ```
     ip addr show
     ```
   - ネットワーク接続（例：Wi-Fiの場合は`wlan0`の下の`192.168.1.100`）の下にあるIPアドレスを探します。これがスマートフォンがラップトップに接続する方法です。

6. **スマートフォンから再生をコントロールする**
   - スマートフォンとラップトップが同じWi-Fiネットワーク上にあることを確認します。
   - スマートフォンのWebブラウザを開き、`http://<laptop-ip>:8080`（例: `http://192.168.1.100:8080`）と入力します。
   - プロンプトが表示されたら、ユーザー名は空白のまま、設定したパスワード（例: "mypassword"）を入力します。
   - VLCのWebインターフェースが表示されます。これを使用して、再生、一時停止、停止、またはプレイリストから次の映画を選択できます。

7. **視聴を開始する**
   - ラップトップで、VLCで映画の再生を開始します（プレイリストの項目をダブルクリック）。
   - VLCを全画面モードに切り替えます（**表示 > 全画面**または`F`キーを押す）。
   - くつろいで、スマートフォンを使ってラップトップに触れることなく再生をコントロールします。

#### この方法があなたに適している理由
- **追加コストなし**: 既存のラップトップ、モニター、スマートフォンを使用するため、新しいものを購入する必要はありません。
- **手間が最小限**: 初期設定後は、ラップトップとモニターの電源を入れ、VLCを開き、すべてをスマートフォンからコントロールするだけです。
- **ローカル再生**: 映画はモニターに接続されたラップトップ上にあるため、ネットワーク経由でストリーミングする必要はありません（Nginxは設定されていますが、ここでは必要ありません）。VLCがファイルを直接再生するため、スムーズなパフォーマンスが確保されます。

#### オプションの改善点
- **ワイヤレスコントロール**: ワイヤレスマウスやキーボードを持っている場合は、スマートフォンの代わりにそれを使用できますが、距離がある場合にはスマートフォンの方が便利です。
- **自動起動**: さらに簡単にするために、ラップトップの起動時にVLCが自動的にプレイリストを起動するように設定します：
  - Ubuntuの**スタートアップアプリケーション**にプレイリスト付きのVLCを追加します（Ubuntuで「スタートアップアプリケーション」を検索し、「追加」をクリックして、`vlc /path/to/your/playlist.m3u`と入力します）。
- **オーディオ確認**: モニターにスピーカーがない場合は、ラップトップからオーディオが出力されることを確認するか、ラップトップのオーディオジャック経由で外部スピーカーを接続してください。

### 代替案：Nginxとブラウザを使用する（希望の場合）
Nginxサーバーを使用したい場合は：
- Nginxディレクトリ（例: `/var/www/html/movies`）に、映画ファイルへのリンク（例: `<a href="movie1.mp4">Movie 1</a>`）を含むシンプルなHTMLページを設定します。
- ラップトップをモニターに接続し、ブラウザを開いて`http://localhost/movies`にアクセスします。
- リンクをクリックしてブラウザで映画を再生し、全画面モードを使用します。
- ただし、スマートフォンから同じページにアクセスしてキャストしない限り、スマートフォンでのコントロールは容易ではなく、追加の設定やハードウェアが必要になる可能性があります。

### 最終的な推奨事項
VLCメソッドが最適です。これは straightforward で、現在のセットアップを活用し、スマートフォンからすべてをコントロールすることで「モニターを開き、その後面倒なく視聴する」ことを可能にします。一度設定すれば、VLCを起動し、スマートフォンを使って映画を選ぶだけです。シンプルで効果的です！
