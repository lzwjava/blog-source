TortoiseGitは、人気のあるバージョン管理システムであるGitのためのユーザーフレンドリーなインターフェースを提供するWindowsシェル拡張です。Windowsエクスプローラーに直接統合され、コンテキストメニューからGit操作を実行できます。このガイドでは、WindowsでのTortoiseGitのインストール、設定、使用方法と、一般的なタスクおよびベストプラクティスについて説明します。

### TortoiseGitの概要
TortoiseGitはmsysGit（Windows用Git）上に構築されており、Gitコマンドのグラフィカルインターフェースを提供します。コマンドラインのGit操作よりもGUIを好むユーザーに理想的です。主な機能は次のとおりです：
- Windowsエクスプローラーへのコンテキストメニュー統合。
- ビジュアルなdiffおよびマージツール。
- 一般的なGitワークフロー（コミット、プッシュ、プル、ブランチ、マージなど）のサポート。
- 人気のある課題追跡システムおよび外部ツールとの統合。

### 前提条件
TortoiseGitをインストールする前に、以下を確認してください：
- Windowsオペレーティングシステム（Windows 10以降を推奨）。
- Git for Windowsがインストールされていること（TortoiseGitはこれに依存します）。
- Gitの概念（リポジトリ、コミット、ブランチなど）の基本的な理解。

### インストール
1. **Git for Windowsのインストール**：
   - [Git for Windows](https://gitforwindows.org/) または [Git SCM](https://git-scm.com/downloads) から最新バージョンをダウンロードします。
   - インストーラーを実行し、プロンプトに従います。推奨設定：
     - デフォルトのエディター（例：Notepad）を使用するか、VS Codeなどのエディターを選択します。
     - アクセシビリティのために「Use Git from the Windows Command Prompt」を選択します。
     - HTTPS転送のために「OpenSSL」を選択します。
     - クロスプラットフォームのチームで作業する場合を除き、行末文字には「Checkout as-is, commit as-is」を選択します。
   - インストールを完了します。

2. **TortoiseGitのインストール**：
   - [TortoiseGit公式ウェブサイト](https://tortoisegit.org/download/) から最新バージョンをダウンロードします。
   - インストーラーを実行します：
     - デフォルトの言語とコンポーネントを選択します。
     - Git for Windowsが検出されていることを確認します（見つからない場合、TortoiseGitがプロンプトを表示します）。
     - 必要に応じてTortoiseGitPlinkをインストールします（SSH用に推奨）。
   - プロンプトが表示された場合はコンピューターを再起動します。

3. **インストールの確認**：
   - Windowsエクスプローラーで任意のフォルダーを右クリックします。「Git Clone」、「Git Create Repository here」などのTortoiseGitオプションが表示されるはずです。

### 初期設定
インストール後、ユーザー詳細と設定をTortoiseGit用に構成します：
1. **ユーザー情報の設定**：
   - フォルダー内で右クリックし、**TortoiseGit > Settings** を選択します。
   - 設定ウィンドウで、**Git > Config** に移動します。
   - 名前とメールアドレスを入力します（GitHub、GitLabなどで使用するものと同じ）：
     ```
     Name: Your Name
     Email: your.email@example.com
     ```
   - **Apply** および **OK** をクリックします。

2. **SSHの設定（オプション）**：
   - リモートリポジトリにSSHを使用する場合、SSHキーを設定します：
     - **PuTTYgen**（TortoiseGitとともにインストール）を開きます。
     - 新しいSSHキーペアを生成し、秘密キーを保存し、公開キーをコピーします。
     - 公開キーをGitホスティングサービス（例：GitHub、GitLab）に追加します。
     - TortoiseGit設定で、**Git > Remote** の下でSSHクライアントとしてTortoiseGitPlinkを選択します。

3. **Diffおよびマージツールの設定**：
   - **TortoiseGit > Settings > Diff Viewer** で、TortoiseGitMerge（デフォルト）やBeyond Compareなどの外部ツールを選択します。
   - マージについては、**Merge Tool** で設定します（初心者にはTortoiseGitMergeが推奨されます）。

### 基本的な使用方法
以下は、Windowsエクスプローラーの右クリックコンテキストメニューからアクセスできる一般的なTortoiseGit操作です。

#### 1. **リポジトリのクローン**
   - フォルダー内で右クリックし、**Git Clone** を選択します。
   - ダイアログで：
     - リポジトリURLを入力します（例：`https://github.com/username/repo.git`）。
     - リポジトリのローカルディレクトリを指定します。
     - オプションで、ブランチを選択するかSSHキーをロードします。
   - **OK** をクリックしてリポジトリをクローンします。

#### 2. **新しいリポジトリの作成**
   - フォルダーに移動し、右クリックして **Git Create Repository here** を選択します。
   - サーバーサイドリポジトリを作成する場合は「Make it Bare」をチェックします（ローカル使用では稀）。
   - **OK** をクリックします。`.git` フォルダーが作成され、リポジトリが初期化されます。

#### 3. **変更のコミット**
   - ファイルをリポジトリフォルダーに追加します。
   - フォルダーまたは選択したファイルを右クリックし、**Git Commit -> "main"**（または現在のブランチ）を選択します。
   - コミットダイアログで：
     - 変更内容を説明するコミットメッセージを入力します。
     - ステージするファイルを選択します（チェックボックス）。
     - **OK** または **Commit & Push** をクリックして変更をリモートリポジトリにプッシュします。

#### 4. **変更のプッシュ**
   - コミット後、右クリックして **TortoiseGit > Push** を選択します。
   - リモートリポジトリとブランチを選択します。
   - プロンプトが表示されたら認証します（HTTPSの場合はユーザー名/パスワード、SSHキーの場合はSSHキー）。
   - **OK** をクリックしてプッシュします。

#### 5. **変更のプル**
   - ローカルリポジトリをリモートの変更で更新するには、右クリックして **TortoiseGit > Pull** を選択します。
   - リモートブランチを選択し、**OK** をクリックします。
   - 競合が発生した場合は解決します（マージツールを使用）。

#### 6. **ブランチの作成と切り替え**
   - 右クリックして **TortoiseGit > Create Branch** を選択します。
   - ブランチ名を入力し、**OK** をクリックします。
   - ブランチを切り替えるには、右クリックして **TortoiseGit > Switch/Checkout** を選択し、ブランチを選択します。

#### 7. **履歴の表示**
   - 右クリックして **TortoiseGit > Show Log** を選択します。
   - コミット履歴を表示します。作成者、日付、メッセージが含まれます。
   - コミットを右クリックして変更を表示、元に戻す、またはチェリーピックします。

#### 8. **マージ競合の解決**
   - プルまたはマージ中に競合が発生した場合、TortoiseGitが通知します。
   - 競合ファイルを右クリックし、**TortoiseGit > Resolve** を選択します。
   - マージツールを使用して競合を手動で編集し、解決済みとしてマークします。
   - 解決された変更をコミットします。

### 高度な機能
1. **変更のスタッシュ**：
   - 未コミットの変更を一時的に保存するには、右クリックして **TortoiseGit > Stash Save** を選択します。
   - スタッシュされた変更を取得するには、**TortoiseGit > Stash Pop** を選択します。

2. **リベース**：
   - 右クリックして **TortoiseGit > Rebase** を選択します。
   - リベース先のブランチを選択し、プロンプトに従ってコミットを並べ替えたりスカッシュしたりします。

3. **サブモジュール**：
   - サブモジュールを管理するには、右クリックして **TortoiseGit > Submodule Update** または **Add** を選択します。
   - TortoiseGit設定でサブモジュール設定を構成します。

4. **二分探索**：
   - バグを導入したコミットを見つけるには、**TortoiseGit > Bisect Start** を使用します。
   - コミットを「良い」または「悪い」とマークして問題のあるコミットを絞り込みます。

### ベストプラクティス
- **頻繁にコミットする**：明確なメッセージで小さく頻繁にコミットします。
- **定期的にプルする**：競合を避けるためにローカルリポジトリを更新し続けます。
- **ブランチを使用する**：新規作業には機能ブランチを作成し、メインブランチを安定させます。
- **SSHキーのバックアップ**：SSHキーは安全に保存し、バックアップします。
- **変更の確認**：コミット前にdiffビューアーを使用して変更を確認します。

### トラブルシューティング
- **認証の問題**：SSHキーまたは資格情報がGitホスティングサービスで正しく設定されていることを確認します。
- **マージ競合**：TortoiseGitMergeを使用して視覚的に競合を解決します。マージ前にファイルをバックアップします。
- **コンテキストメニューが表示されない**：TortoiseGitが正しくインストールされ、設定でエクスプローラー統合が有効になっているか確認します。
- **パフォーマンスの低下**：**TortoiseGit > Settings > Icon Overlays** で不要なオーバーレイを無効にします。

### ワークフローの例
以下は、プロジェクトに貢献するためのサンプルワークフローです：

# TortoiseGitサンプルワークフロー

## 1. リポジトリのクローン
- フォルダー内で右クリックし、**Git Clone** を選択します。
- リポジトリURLとローカルディレクトリを入力します。
- **OK** をクリックします。

## 2. 機能ブランチの作成
- 右クリックし、**TortoiseGit > Create Branch** を選択します。
- ブランチに名前を付けます（例：`feature/add-login`）。
- **OK** をクリックし、新しいブランチに切り替えます。

## 3. 変更の作成
- リポジトリフォルダー内のファイルを編集します。
- 右クリックし、**Git Commit -> "feature/add-login"** を選択します。
- コミットメッセージを書き、ファイルをステージします。
- **Commit** をクリックします。

## 4. 変更のプッシュ
- 右クリックし、**TortoiseGit > Push** を選択します。
- リモートブランチを選択します（例：`origin/feature/add-login`）。
- **OK** をクリックします。

## 5. プルリクエストの作成
- Gitホスティングサービス（例：GitHub）で、ブランチからメインブランチへのプルリクエストを作成します。
- レビューとマージを待ちます。

## 6. ローカルリポジトリの更新
- メインブランチに切り替えます（**TortoiseGit > Switch/Checkout**）。
- 更新をプルします（**TortoiseGit > Pull**）。
