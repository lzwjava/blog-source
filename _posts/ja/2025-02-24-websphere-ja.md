WebSphere Liberty サーバーを管理する際には、サーバーの作成、開始、停止、監視などの基本的なタスクを実行するために頻繁に使用されるいくつかのコマンドがあります。以下に、最も一般的に使用されるコマンドとその簡単な説明を示します：

- **`./server create <server_name>`**
  指定された名前で新しいサーバーインスタンスを作成します。このコマンドは、サーバーの必要なディレクトリ構造とデフォルトの設定ファイル（例：`server.xml`）を設定します。

- **`./server start <server_name>`**
  サーバーをバックグラウンドで開始します。サーバーはバックグラウンドプロセスとして実行され、ログはサーバーのログファイルに書き込まれます。

- **`./server stop <server_name>`**
  実行中のサーバーインスタンスを停止します。このコマンドは、サーバーを優雅にシャットダウンします。

- **`./server status <server_name>`**
  指定されたサーバーが実行中かどうかを確認します。現在の状態（例：開始または停止）を返します。

- **`./server run <server_name>`**
  サーバーをフォアグラウンドで開始します。これは、サーバーの出力がコンソールに直接表示されるため、デバッグまたは監視に便利です。

- **`./server package <server_name>`**
  サーバー、その設定、アプリケーションを含むzipファイルにパッケージ化します。これは、サーバーを他の環境にデプロイするか、バックアップのために便利です。

- **`./server debug <server_name>`**
  サーバーをデバッグモードで開始し、トラブルシューティングのためにデバッガをサーバープロセスにアタッチできるようにします。

これらのコマンドは、WebSphere Liberty サーバーを効果的に管理するために必要な基本的な操作をカバーしています。より高度なタスク、例えば機能のインストールやセキュリティの管理については、`featureUtility`や`securityUtility`のような追加のユーティリティが利用可能ですが、日常的な操作ではあまり頻繁に使用されません。
