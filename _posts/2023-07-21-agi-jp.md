1. JiraとConfluenceは時代遅れです。
2. AIパワードワークセンター = Jira + Confluence + GitHub + Teams。
3. 全ての知識労働者はAIパワードのフルスタック労働者になる。
4. 全ての労働者は作成した情報に応じて自動的に報酬を受け取るべきです。
5. 誰もがコードを作成できる限り、自分の好きな言語を使用して作業できます。

ここでの"10x"は10倍の意味です。AGIは"Artificial General Intelligence（人工汎用知能）"の略称です。この未来の世界を表現するために、3つの形容詞を使っています。世界の約1600万人が未来がAGIの世界になることを認識していると考えられています。2023年7月時点で、Midjourneyには約1600万のユーザーアカウントがあります。一方、ChatGPTは2023年1月時点で1億人のユーザーを持っています。したがって、AIツール愛好者の数を表すために1600万という数字を選ぶことが妥当でしょう。ChatGPTはテキスト会話ツールであり、シンプルかつ直接的なものです。一方、MidjourneyはAIアルゴリズムによる画像生成ツールです。真剣な愛好者が試すかもしれません。

ここでの"10x"は非常に高速であり、通常の10倍以上の速さを意味します。そして、インターネットユーザーの数が51億6千万人に達していることから、この世界は情報の世界であることは疑いようがありません。

努力し、考え抜く人々は未来に対してより楽観的です。それは、自分の知恵を使って世界に影響を与える感覚があるからかもしれません。さらに、学ぶにつれて、人間社会がまだ技術の章で始まったばかりであることがわかります。第三の理由は、世界中のほとんどの人が知らない何かを証明できると思う大胆さがあるからかもしれません。そのため、未来は興味深いと感じるかもしれません。

私もそのような人々の一人です。28年間この世界で学んだ後、自分の考える未来の世界を共有したいと思います。

## 仕事の進め方

私は10x AGI情報世界では、JiraとConfluenceは時代遅れであると考えています。これらのツールは完全にAGIとGPTの力を統合するべきです。多くのJiraの課題はプロダクトマネージャーによって作成されるべきではないと言えるでしょう。ユーザーにAIツールを使って作成させるべきです。そして、AIツールの助けを借りてユーザーが作成した課題は、自動的に担当者を計算します。AIパワードのJiraは課題の説明と内部情報（ドキュメントやコードなど）を使用して、課題をどのような担当者に割り当てるべきかを判断します。

ここでの担当者は、AIパワードの開発者、テスター、またはAIパワードのフルスタック労働者を指します。もはやプロダクトマネージャーや開発者、テスターなどの役割を区別する必要はないかもしれません。彼らはみなAIパワードのフルスタック労働者なのです。

そして、それを考えると、なぜユーザーにバグや問題を報告させる必要があるのでしょうか。なぜ私たちが多くのAIパワードのテストユーザーを作成して、彼らに報告させないのでしょうか？完璧なアプリを作るのは難しいです。さまざまな背景を持つ多くのユーザーが使用する多くのシナリオがあるため、ユーザーは自分の好きな言語をサポートするアプリを必要とします。ChatGPTのように、サポートされている言語を持つ非常に基本的でシンプルな会話インターフェースを好みます。ただし、テキストを入力する必要があり、モバイルフォンでの入力は不便です。モバイルフォンからの入力では、多くのユーザーがメッセージを高速かつ便利に送信するために、一つの完全なメッセージではなく3つの短いフレーズを送信することがよくあります。また、入力メッセージボックスは長いテキストをうまくサポートしていません。

私はChatGPT iOSアプリを確認しましたが、以下のように入力ボックスが拡大することを確認しました。しかし、ユーザーの習慣を変えて、一文ではなく短いフレーズを送信することは難しいです。

<div align="center"><img src="/assets/images/agi/input-box.jpg" width="100%" /><img/></div>

しかしながら、ChatGPTでは音声入力もサポートされていることに気付きました。したがって、長いテキストの入力が簡単になります。これについては既に知っているようなので、このディスカッションは終了します。

では、未来の仕事の進め方に戻りましょう。PayGPTをAIパワーとChatGPTインターフェースを持つ支払いアプリとして使用するとします。製品のイテレーションはどのように行うべきでしょうか？ChatPTとPayGPTの間にはどのような差があるのでしょうか？これらのデータ集約型アプリケーションの基盤にはデータベースがあります。したがって、テキストとデータベースの間にはどのような差があるのでしょうか？

データベースをテキストにエクスポートし、言語モデルにトレーニングしてからクエリを実行するという方法はどうでしょうか？ユーザー権限の問題をどのように扱うべきでしょうか？ユーザーは自分のデータ、友人の一部のデータ、およびアプリの公開データにのみアクセスできるべきです。ユーザーごとに言語モデルをトレーニングする必要があるのでしょうか？それとも、事前にトレーニングされた大規模な言語モデルがあり、ユーザーのコンテキストを入力してからクエリを開始するべきでしょうか？

ユーザーが課題を作成する方法に戻りましょう。ユーザーAがPayGPTを使ってユーザーBにお金を支払う場合を考えてみましょう。ユーザーAは音声入力を使ってPayGPTに支払いを指示します。そして支払い後、ユーザーBはPayGPTから通知を受け取ります。ユーザーBはアプリを開いて、PayGPTのボットがユーザーAが自分にお金を支払ったと伝えるのを見ます。

もしユーザーBがお金を受け取れなかった場合、PayGPTの会社に報告したいと思うかもしれません。そこで、ユーザーBはPayGPTに確認を依頼します。その結果、ユーザーAとユーザーBの会話メッセージから課題が作成されます。

この課題はPayGPTの会社のAIパワードのフルスタック労働者に委任されます。AIパワードのJiraは、課題をどのような担当者に割り当てるべきかを判断するため、ユーザーBの問題報告に含まれる言葉などの会話を分析します。AI時代では、すべての知識労働者がフルスタック労働者になるかもしれません。ただし、フルスタックの能力レベルは異なるかもしれません。AIパワードのJiraは、Jiraが会社内のすべての情報とその作成者を知っているため、これを理解しています。

したがって、JiraとConfluenceはもはや分離される必要はありません。企業内のJiraとConfluenceでは情報が重複していると思われることが多いです。単一の情報源と情報を持つべきです。そうしないと、すべての情報のうち一つだけを更新した場合に矛盾が生じる可能性があります。

AIパワードのフルスタック労働者は、PayGPTの会社のすべての情報、ドキュメント、コード、テストケースを処理できます。彼は機能を提供するために他の誰かと協力する必要がないかもしれません。コードはCopilotのようなツールによってほとんど作成されています。ドキュメントはコーダーと非コーダーの協力のためのものです。そして、AI時代には、みながコーダーです。したがって、ドキュメントは少なくなるでしょう。コードほど正確ではないですが。

このような自動的な作業ルーチンでは、もはやリーダーは必要ありません。なぜなら、一人がどれだけの情報を生成しているかは明確だからです。

これらの機能を含んだプラットフォームに、新たに「Work Center」という名前を付けましょう。Work Center = Jira + Confluence + GitHub + Teamsです。Microsoft Teamsはもはや必要ないかもしれません。AIパワードのフルスタック労働者はすべての情報を持ち、WorkGPTの助けを借りてすべての情報を理解できます。そして既存の情報と同様の情報を作成することもできるでしょう。なぜ彼らはまだ他の人と議論して協力する必要があるのでしょうか？

そして今、人々は運転中にChatGPTに質問することができます。では、なぜ私たちがそのように仕事をしないのでしょうか？運転中にChatGPTに質問することで、ニューラルネットワークを理解できるなら、なぜ運転中にニューラルネットワークのコードを書くことはできないのでしょうか？人間の声はテキストの言葉やコードに簡単に変換できます。したがって、誰もが世界中を楽しみながら旅行し、運転中に仕事をすることができるでしょう。

もちろん、皆が好きな言語を使ってコードを作成できる限り、仕事のほとんどはその言語で行うことができます。

そして給与はAIツールによって生成された情報をAIパワードのフルスタック労働者の監督下で自動的に計算されるでしょう。タイトルや企業の構造はもはや必要ないかもしれません。なぜなら、協力する必要がないからです。


## 私たちの楽しみ方

10x AGI情報世界は非常に楽しい場所になるでしょう。すべてがクラウド上に存在します。新しい家に引っ越す際には1週間で設置できます。AIパワードのロボットが私たちの荷物を手伝い、適切な場所に整理してくれます。基本的に、私たちは労働をする必要がありません。ChatGPTの背後にある変形はその位置が正確でないかもしれませんが、家の家具の位置が完璧でなくても受け入れられるでしょう。

家の温度と湿度だけでなく、TVOC（揮発性有機化合物）やホルムアルデヒドの濃度も家のどこでも可能な限り低くなるようにしなければなりません。家のすべてのハードウェアや家電製品はクラウドに接続され、常にその周りの温度、湿度、および空気の品質を報告します。

想像すべき1つの方向は、どこからでも家電製品を操作できるようにすることです。そして、できるだけ私たちの利益に適した形で動作するべきです。

これは家のシナリオですが、他の場所も同じです。自分から1メートルのスペースのすべての側面をコントロールできるようにすべきです。オフィスには小さな部屋があるべきで、自宅と同じようにすべてをコントロールできるようにすべきです。例えば、私は何度かオフィスでコードを書くためにアウトレットを持参していなかったことにかなりイライラしました。

公共の場所では、自宅のAGIのようにすべてをコントロールすることは難しいように思えるかもしれません。しかし、それは考えるべきことではありません。私たちは四輪の上半透明の部屋が必要です。上半分が透明であれば、周囲の状況を知るために部屋の上半分から外を見るだけで済みます。

それは私たちがどこにでも歩いたり移動したりするのを助けます。階段は四輪の部屋にとって難しいかもしれません。それを適応させるために階段を変更する必要があるかもしれません。そのため、以下のような大きな車を持って、移動可能な部屋を持ち運ぶ必要があるかもしれません。

私の想像では、理想的な移動可能な部屋は、私たちだけを収容できる程度に小さくするべきです。なぜなら、まだ多くの建物が移動可能な部屋に対応していないからです。それらは単に移動可能な人々のために建てられたものです。

この場合、人々はどんな季節でも、どんな天候でも外に出かけることが好きになるでしょう。移動可能な部屋は確かに才能のあるコンピューターでもあります。周囲の温度、湿度、および空気の品質を報告し、どこにでも行く方法を知っています。私たちは単に口を使ってそれに伝える必要があります。

したがって、誰もが移動可能な部屋を持ち、どこにでも持参またはレンタルできるようになったら、世界を探検することがより一層増えるでしょう。実際には、車は移動可能な部屋のようなものです。ただし、車を使用して建物の内部のどこにでも行くことはできません。これは非常に良いアイデアだと思います。世界は完全に建物を移動可能な小さな部屋のために再設計すべきです。人々は自分を取り巻くすべてのものをコントロールできるようにすべきです。

そのため、ドアはより大きく、エレベーターもより大きくする必要があります。そして、私たちの小さな移動可能な部屋を持ち運べるようにしたいのです。例えば、以下のような携帯式衣類乾燥機のように。

ですから、AIパワードの電気自動車とAIパワードの電気移動可能な部屋を持つことで、世界を旅行し、家族や友人とより良い時間を過ごすことができるようになります。

## 終わりに

ここでは、10x AGI情報世界での私たちの仕事と楽しみ方のいくつかの側面をカバーしました。その世界は10倍のデータ、10倍のコード、そして10倍の処理速度についてです。これらのアイデアを以前に思いついてから記述し、新しいアイデアもいくつか考えた後、私は未来に対してより楽観的に感じています。私たちはまだ人間がより良い生活を送るために試みていない可能性が多くあります。これが人間にいくつかのアイデアを想像し、議論することができるようになればと願っています。1000年前に生きていた人々は電気、コンピューター、機械を持っていなかったため苦しんでいるように見えるかもしれません。そして今、技術は急速に発展しています。100年後に生きる人々は私たちが苦しんでいると考えるでしょう。終わり。
