n8n（発音は「n-eight-n」）は、技術チームや開発者向けに設計されたオープンソースのワークフロー自動化プラットフォームです。様々なアプリ、サービス、APIを接続して、反復的なタスク、データフロー、複雑なプロセスを自動化できます。カスタムコードへの依存を大幅に減らせますが、必要に応じてコードの使用もサポートしています。ZapierやMakeのようなツールのセルフホスト型代替品と考えてください。ただし、より柔軟性が高く、マルチステップのAIエージェントの構築、任意の大規模言語モデル(LLM)との統合、そしてデータプライバシーと制御を強化するための自社インフラでの実行が可能です。

中核となるのは、ビジュアルでノードベースのインターフェースです。ワークフローは「ノード」（トリガー、アクション、変換を表すビルディングブロック）をドラッグ＆接続して構築します。フェアコードライセンス（GitHubでソースコード利用可能）で、400以上の事前構築済みインテグレーション（Google Sheets、Slack、OpenAI、GitHubなど）をサポートし、単純な通知から、チケットの要約やコンテンツ生成といった高度なAI駆動の自動化まで、あらゆることを処理できます。

### 主な機能
- **ビジュアルワークフロービルダー**: ノーコード設定のためのドラッグ＆ドロップのノード。カスタムロジックのためにJavaScript、Python、さらにはnpm/Pythonライブラリを埋め込むオプションあり。
- **AI統合**: LangChainのようなツールでマルチステップエージェントを構築、任意のLLM（ローカルまたはクラウドベース）に接続、Slack、SMS、音声を介してデータを照会またはタスクを実行するチャットインターフェースを作成。
- **セルフホスティングとセキュリティ**: Dockerまたはnpmを介した完全なオンプレミスデプロイ。SSO、暗号化されたシークレット、RBAC、監査ログをサポート。ベンダーロックインなし—自身のAIモデルもホスト可能。
- **ハイブリッド開発**: UIとコードを混合。テスト用のデータ再生、デバッグ用のインラインログ、クイックスタート用の1,700以上のテンプレート。
- **スケーラビリティ**: ワークフロー履歴、Gitバージョン管理、分離された環境、顧客向け自動化のための埋め込みなどのエンタープライズ機能。
- **パフォーマンス例**: Delivery Heroのような企業は月200時間以上を節約。StepStoneは数週間の作業を数時間に短縮。

Zapierと比較して、n8nはより開発者フレンドリー（コードアクセス、セルフホスティング）、コスト効率が良い（コアは無料、タスクごとの課金なし）、プライバシー重視（データが第三者を経由しない）です。金融、医療、または内部業務で機密データを扱うチームに理想的です。

# n8nの使い方：包括的ガイド

このガイドでは、セットアップから高度な使用法までを実践的な例を用いて説明します。例：毎日新しい記事をメールするRSSフィードモニター（AI要約への拡張も可能）。基本的な技術的知識を前提とします。n8nはNode.js上で動作します。

## 1. インストールとセットアップ

n8nは軽量で素早く開始できます。前提条件：ローカルインストールにはNode.js（v18以上推奨）。コンテナにはDocker。本番環境では、DigitalOceanやAWSなどのVPSを使用してください。

### クイックローカルスタート（開発/テスト）
1.  ターミナルを開く。
2.  実行: `npx n8n`
    -   これによりn8nが一時的にダウンロード、起動されます。
3.  ブラウザで `http://localhost:5678` にアクセスしてエディターを開く。
    -   デフォルトログイン：最初は資格情報不要（後でセキュリティのために設定）。

### 永続的なローカルインストール (npm)
1.  グローバルインストール: `npm install n8n -g`
2.  起動: `n8n start`
3.  `http://localhost:5678` でアクセス。

### Docker (本番環境推奨)
1.  イメージをプル: `docker run -it --rm --name n8n -p 5678:5678 -v ~/.n8n:/home/node/.n8n n8nio/n8n`
    -   データ永続化のためのボリュームをマップします。
2.  高度なセットアップ（例：PostgreSQL DB）の場合：ドキュメントの `docker-compose.yml` を使用。
3.  `http://localhost:5678` でアクセス。

### クラウドオプション
- **n8n Cloud**: n8n.ioでのマネージドホスティング—サインアップ後、数分でデプロイ、制限付きで無料から開始。
- **サードパーティPaaS**: Render、Railway、Sevalla（ワンクリックテンプレート）を使用。Sevallaでの例：
    1.  sevalla.com でサインアップ。
    2.  「n8n」テンプレートを選択、リソース（例：1 CPU, 1GB RAM）をデプロイ。
    3.  `https://your-n8n.sevalla.app` のようなURLを取得。

**ヒント**: セルフホスティングの場合、HTTPSで保護（Nginxのようなリバースプロキシ経由）、環境変数を設定（例: `N8N_BASIC_AUTH_ACTIVE=true`）、`~/.n8n` フォルダをバックアップ。高ボリュームのワークフローにはキューモードでスケール。

## 2. UI概要

起動後:
- **キャンバス**: ワークフローのための空白のワークスペース。「+」をクリックしてノードを追加。
- **ノードパネル**: 400以上のノードの検索可能なライブラリ（例: "Schedule Trigger"）。
- **実行パネル**: テスト中のデータフローをリアルタイム表示。
- **サイドバー**: ワークフロー設定、実行履歴、テンプレート。
- **トップバー**: 保存、有効/無効トグル、共有/エクスポートオプション。

ワークフローは自動保存。チームでのバージョン管理にはGitを使用。

## 3. コアコンセプト

- **ワークフロー**: 自動化ロジックを定義する、接続されたノードのシーケンス。アクティブなワークフローはトリガーで実行。非アクティブはテスト用。
- **ノード**: モジュール式ブロック:
    - **トリガー**: ワークフローを開始（例: cronジョブ用のSchedule、HTTPイベント用のWebhook、フィード用のRSS Read）。
    - **アクション**: 作業を実行（例: メール送信、API用のHTTP Request、カスタムコード用のFunction）。
    - **コアノード**: IF（条件分岐）、Merge（データ結合）、Set（変数操作）。
- **接続**: ノード間の矢印はデータフロー（JSON形式）を示す。あるノードからのデータが次のノードに供給される。
- **式**: `{{ $json.title }}` のような動的プレースホルダーで、フィールドにデータ（例: 記事タイトル）を引き出す。タイムスタンプには `$now`、バッチには `$input.all()` を使用。
- **資格情報**: APIキー/OAuthの安全な保存。サービスごとに一度設定（例: Gmail OAuth）し、ノード間で再利用。
- **実行**: ワークフローの実行回数。ログの表示、データの再生、エラーのデバッグが可能。

## 4. 初めてのワークフロー作成：ステップバイステップ

「日次RSSダイジェストメール」を構築しましょう。

1.  **新規ワークフロー作成**:
    -   「New」> 名前を「RSS Digest」に設定。
    -   キャンバスが開く。

2.  **トリガーノードの追加**:
    -   「+」> 「Schedule Trigger」を検索。
    -   設定: トリガーを「Every Day」の午前9:00に（cron: `0 9 * * *`）。
    -   テスト: 「Execute Node」をクリック（今すぐ1回実行）。

3.  **データ取得ノードの追加**:
    -   トリガーの後ろで「+」> 「RSS Read」。
    -   URL: `https://blog.cloudflare.com/rss/`。
    -   実行: アイテムを取得（例: title, link, pubDateを含むJSON）。

4.  **データ変換（オプションのFunctionノード）**:
    -   「+」> 「Function」。
    -   コード:
        ```
        // 上位3アイテムに制限
        return items.slice(0, 3);
        ```
    -   これは受信データに対してJavaScriptを実行します。

5.  **アクションノードの追加**:
    -   「+」> 「Gmail」（またはSMTPの場合は「Email Send」）。
    -   資格情報: 「Create New」> Gmail用OAuth（Google認証フローに従う）。
    -   To: 自身のメールアドレス。
    -   Subject: `Daily Digest: {{ $input.first().json.title }}`
    -   Message: 式を使用してアイテムをループ:
        ```
        {{#each $input.all()}}
        - {{ $json.title }}: {{ $json.link }} ({{ $json.pubDate }})
        {{/each}}
        ```
    -   （ループにHandlebarsライクな構文を使用。）

6.  **接続とテスト**:
    -   矢印をドラッグ: Trigger → RSS → Function → Email。
    -   「Execute Workflow」: データフローを確認。受信ボックスをチェック。
    -   エラー修正: 赤いノードは問題（例: 無効な資格情報）を強調表示。

7.  **有効化**:
    -   「Active」トグルをオンに。これで毎日実行されるようになります。

共有用にJSONとして保存およびエクスポート。

## 5. より複雑なワークフローの構築

- **条件分岐**: RSSの後に「IF」ノードを追加: `{{ $json.pubDate }} > {{ $now.minus({days:1}) }}` で新しいアイテムをフィルタリング。
- **ループとバッチ**: 大規模データセットの処理に「Split In Batches」を使用。
- **エラーハンドリング**: 「Error Trigger」ワークフローまたはリトライ用の「IF」を追加。「Set」でエラーをログに記録。
- **API統合**: カスタムエンドポイント用の「HTTP Request」ノード（例: Slack webhookへのPOST）。
- **データ操作**: JSON調整用の「Edit Fields」またはFunctionノード。
- **テスト**: 特定の実行を再生。ノード内でデータをモック。

例: Twitterモニター
1.  トリガー: メンション時の「Twitter Trigger」。
2.  AIノード: 感情を分類する「OpenAI」。
3.  IF: ポジティブ → CRM追加。ネガティブ → Slackアラート。

## 6. 高度な使用法とベストプラクティス

- **AIワークフロー**: 「AI Agent」ノードを追加 > ツール（例: 検索、要約）と連鎖。「Ollama」ノード経由でローカルLLMを統合。
- **カスタムノード**: JSで構築（ドキュメントチュートリアル）。コミュニティに公開。
- **スケーリング**: キューモード (`N8N_WORKER=1`)、外部DB (PostgreSQL)、リアルタイム用のWebhookを使用。
- **セキュリティ**: シークレットを暗号化、チーム用にRBACを使用、監査ログ。
- **デバッグ**: Function内のインラインログ (`console.log(items)`)。Gitでバージョン管理。
- **よくある落とし穴**: データ型（全てJSON）に注意。ノード内のレート制限を処理。AIの前にシンプルに開始。
- **拡張**: アプリに埋め込み、テンプレート（1700以上利用可能）、またはnpmでエクストラを使用。

本番環境では、外部ツールで監視。セルフホストはコスト削減（Zapierの$20+/月と比較して）。
