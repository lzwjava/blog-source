## 1. Java言語仕様、API、JDK、IDEツール

### **Java言語仕様**
Java言語仕様は、Javaの構文と意味を定義しています。理解すべき重要な概念：

- **プラットフォーム独立性**：「一度書けば、どこでも実行できる」（WORA） - Javaコードはバイトコードにコンパイルされ、JVM（Java仮想マシン）がある任意のプラットフォームで実行可能
- **オブジェクト指向**：Javaではすべてがオブジェクトとクラスを中心に構成される
- **強い型付け**：変数は特定のデータ型で宣言する必要がある
- **自動メモリ管理**：ガベージコレクションがメモリの解放を処理

### **Java API（アプリケーション・プログラミング・インターフェース）**
Java APIは、パッケージに編成された事前に書かれたクラスの大規模なコレクションです：

- **主要パッケージ**：`java.lang`（自動的にインポート）、`java.util`、`java.io`
- **目的**：すぐに使用できる機能を提供（コレクション、ファイルI/O、ネットワーキングなど）
- **ドキュメント**：Oracleの公式Javaドキュメントサイトで利用可能
- **使用方法**：`import`文を使用してパッケージをインポート

### **JDK（Java Development Kit）**
JDKの必須コンポーネント：

- **javac**：Javaコンパイラ（.javaファイルを.classバイトコードファイルに変換）
- **java**：Javaランタイム環境ランチャー
- **javadoc**：ドキュメント生成ツール
- **jar**：Javaアーカイブツール
- **JRE含む**：プログラムを実行するためのJavaランタイム環境
- **標準ライブラリ**：完全なJava APIの実装

**インストールとセットアップ**：
- Oracleからダウンロードするか、OpenJDKを使用
- JAVA_HOME環境変数を設定
- システムのPATHにJDKのbinディレクトリを追加

### **IDE（統合開発環境）ツール**
Java開発で人気のIDE：

1. **Eclipse** - 無料、オープンソース、教育機関で広く使用
2. **IntelliJ IDEA** - 強力な機能、無料版と有料版の両方あり
3. **NetBeans** - Oracle公式サポートのIDE
4. **VS Code** - 軽量でJava拡張機能対応

**IDEの利点**：
- シンタックスハイライトとエラー検出
- コード補完と提案
- 統合デバッグツール
- プロジェクト管理
- バージョン管理統合

## 2. Javaプログラムの作成、コンパイル、実行

### **基本的なJavaプログラム構造**

```java
// すべてのJavaアプリケーションにはメインクラスが必要
public class HelloWorld {
    // mainメソッド - プログラムのエントリーポイント
    public static void main(String[] args) {
        // ここにコードを記述
        System.out.println("Hello, World!");
    }
}
```

### **ステップバイステップのプロセス**

**ステップ1：Javaプログラムの作成**
- `.java`拡張子のテキストファイルを作成
- ファイル名は公開クラス名と一致する必要あり（大文字小文字を区別）
- 例：`HelloWorld`クラスの場合は`HelloWorld.java`

**ステップ2：コンパイル**
```bash
javac HelloWorld.java
```
- これにより`HelloWorld.class`（バイトコードファイル）が作成される
- コンパイラは構文エラーをチェック
- エラーがある場合、コンパイルはエラーメッセージとともに失敗

**ステップ3：実行**
```bash
java HelloWorld
```
- 注意：クラス名は`.class`拡張子なしで使用
- JVMがクラスをロードし、mainメソッドを実行

### **コマンドライン vs IDEワークフロー**

**コマンドライン**：
- ターミナル/コマンドプロンプトを開く
- .javaファイルを含むディレクトリに移動
- コンパイルに`javac`、実行に`java`を使用
- 基礎的なプロセスを理解するのに適している

**IDEワークフロー**：
- 新しいJavaプロジェクトを作成
- 新しいクラスを作成
- エディタでコードを記述
- 「実行」ボタンをクリック（IDEが自動的にコンパイルを処理）
- 大規模プロジェクトにより便利

## 3. プログラミングスタイルガイドライン

良いプログラミングスタイルは、コードの可読性と保守性を高めます。以下の規則に従ってください：

### **命名規則**

- **クラス**：PascalCase（各単語の最初の文字を大文字）
  - 例：`StudentRecord`、`BankAccount`、`HelloWorld`

- **メソッドと変数**：camelCase（最初の単語は小文字、以降の単語の最初を大文字）
  - 例：`calculateTotal()`、`firstName`、`studentAge`

- **定数**：すべて大文字、アンダースコア使用
  - 例：`MAX_SIZE`、`PI`、`DEFAULT_VALUE`

- **パッケージ**：すべて小文字、多くの場合逆ドメイン名
  - 例：`com.company.project`、`java.util`

### **コードフォーマット**

**インデント**：
```java
public class Example {
    public static void main(String[] args) {
        if (condition) {
            // 4スペースまたは1タブでインデント
            statement;
        }
    }
}
```

**ブレース**：
- 開始ブレースは同じ行（Javaの規則）
- 終了ブレースは単独の行、文と揃える

**スペース**：
```java
// 良いスペースの使用
int sum = a + b;
if (x > 0) {

// 悪いスペースの使用
int sum=a+b;
if(x>0){
```

### **コメント**

**一行コメント**：
```java
// これは一行コメントです
int age = 20; // コードの後のコメント
```

**複数行コメント**：
```java
/*
 * これは複数行コメントです
 * 長い説明に使用されます
 */
```

**Javadocコメント**（ドキュメント用）：
```java
/**
 * 2つの数値の合計を計算します。
 * @param a 最初の数値
 * @param b 2番目の数値
 * @return aとbの合計
 */
public int add(int a, int b) {
    return a + b;
}
```

### **ベストプラクティス**

1. **意味のある名前**：説明的な変数名とメソッド名を使用
   - 良い例：`studentCount`、`calculateAverage()`
   - 悪い例：`x`、`doStuff()`

2. **1行に1つの文**：1行に複数の文を詰め込まない

3. **一貫したスタイル**：コード全体で同じ規則に従う

4. **空白行**：論理的なセクションを分けるために空白行を使用

5. **メソッドを短く保つ**：各メソッドは1つのことを適切に行うべき

## 4. 一般的なプログラミングエラーとデバッグの基礎

### **エラーの種類**

#### **A. 構文エラー（コンパイル時エラー）**
これらはコンパイルを妨げ、実行前に修正する必要があります：

**一般的な構文エラー**：
```java
// セミコロン欠落
int x = 5  // エラー：;が欠落

// ブレースの不一致
public class Test {
    public static void main(String[] args) {
        System.out.println("Hello");
    // 終了ブレース}が欠落

// 大文字小文字の区別
String name = "John";
system.out.println(name); // エラー：'System'であるべき

// ファイル名の不一致
// ファイル：Test.java
public class MyClass { // エラー：クラス名はファイル名と一致する必要あり
```

#### **B. ランタイムエラー**
プログラムはコンパイルされるが、実行中にクラッシュ：

```java
// ゼロ除算
int result = 10 / 0; // ArithmeticException

// ヌルポインタ
String str = null;
int length = str.length(); // NullPointerException

// 配列インデックス範囲外
int[] arr = {1, 2, 3};
int value = arr[5]; // ArrayIndexOutOfBoundsException
```

#### **C. 論理エラー**
プログラムは実行されるが、誤った結果を生成：

```java
// 誤った演算子
int average = (a + b) * 2; // *ではなく/であるべき

// オフバイワンエラー
for (int i = 0; i <= arr.length; i++) { // i < arr.lengthであるべき

// 誤った条件
if (age > 18) { // 「18歳以上」の場合は>=であるべき
```

### **デバッグ技術**

#### **1. エラーメッセージを注意深く読む**
```
HelloWorld.java:5: error: ';' expected
        int x = 5
                 ^
1 error
```
- **行番号**：エラーが発生した場所を示す（5行目）
- **エラータイプ**：何が悪いかを伝える（セミコロン欠落）
- **ポインタ**：正確な位置を示す

#### **2. プリント文デバッグ**
```java
public static int calculateSum(int a, int b) {
    System.out.println("Debug: a = " + a + ", b = " + b);
    int sum = a + b;
    System.out.println("Debug: sum = " + sum);
    return sum;
}
```

#### **3. IDEデバッガの使用**
- **ブレークポイント**：特定の行で実行を一時停止
- **ステップオーバー**：現在の行を実行し次に移動
- **ステップイン**：メソッド呼び出しに入り内部実行を確認
- **変数の監視**：リアルタイムで変数値を監視
- **コールスタック**：メソッド呼び出しのシーケンスを確認

#### **4. 分割統治法**
- 問題を分離するためにコードのセクションをコメントアウト
- 小さな部分を独立してテスト
- エラーが再現するまで徐々にコードを戻す

#### **5. ラバーダックデバッグ**
- 誰か（または何か）にコードを一行ずつ説明する
- 多くの場合、自分自身で問題を見つけるのに役立つ

### **初心者によくある間違い**

1. **変更後のコンパイル忘れ**
   - 実行前には常に再コンパイル

2. **クラス名がファイル名と一致しない**
   - `public class Student`は`Student.java`にある必要あり

3. **mainメソッドのシグネチャ欠落**
   - 正確に以下である必要あり：`public static void main(String[] args)`

4. **パッケージのインポート忘れ**
   ```java
   import java.util.Scanner; // これを忘れずに！
   ```

5. **誤った大文字小文字の使用**
   - `string`ではなく`String`、`system`ではなく`System`

6. **条件文で==の代わりに=を使用**
   ```java
   if (x = 5) { // エラー：比較ではなく代入
   if (x == 5) { // 正しい
   ```

### **一般的な試験問題タイプ**

1. **多肢選択**：「正しいmainメソッドのシグネチャはどれですか？」
2. **エラー特定**：「このコードの構文エラーを見つけてください」
3. **コードトレース**：「このプログラムの出力は何ですか？」
4. **短答問題**：「JDKとJREの違いを説明してください」
5. **コード記述**：「'Hello, China!'と出力するプログラムを書いてください」

### **練習問題**

このコードのすべてのエラーを特定して修正してみてください：

```java
public class test {
    Public Static Void Main(string[] args) {
        int x = 5
        int y = 10;
        System.Out.Println(x + y)
    }
}
```

**回答**：
- クラス名は`Test`（大文字のT）でファイル名と一致するべき
- `Public` → `public`、`Static` → `static`、`Void` → `void`
- `string` → `String`
- `int x = 5`の後にセミコロン欠落
- `System.Out` → `System.out`
- println文の後にセミコロン欠落
