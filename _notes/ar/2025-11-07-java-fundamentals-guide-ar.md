## امتحان التعليم العالي الوطني الذاتي

يغطي هذا الدليل المفاهيم الأساسية التي تحتاج إلى إتقانها للجزء الأول من دورة برمجة جافا الخاصة بك.

## 1. مواصفات لغة جافا، واجهة برمجة التطبيقات (API)، مجموعة تطوير جافا (JDK)، وأدوات بيئة التطوير المتكاملة (IDE)

### **مواصفات لغة جافا**
تحدد مواصفات لغة جافا بناء الجملة ودلالات لغة جافا. المفاهيم الرئيسية لفهمها:

- **الاستقلالية عن المنصة**: "اكتب مرة واحدة، شغل في أي مكان" (WORA) - يترجم كود جافا إلى bytecode يعمل على أي منصة بها آلة جافا الافتراضية (JVM)
- **البرمجة كائنية التوجه**: كل شيء في جافا منظم حول الكائنات والكلاسات
- **كتابة قوية**: يجب تعريف المتغيرات بأنواع بيانات محددة
- **إدارة الذاكرة التلقائية**: يتولى جامع القمامة (Garbage collection) تحرير الذاكرة

### **واجهة برمجة تطبيقات جافا (API)**
واجهة برمجة تطبيقات جافا هي مجموعة كبيرة من الكلاسات المكتوبة مسبقًا والمنظمة في حزم:

- **الحزم الأساسية**: `java.lang` (يتم استيرادها تلقائيًا)، `java.util`، `java.io`
- **الغرض**: يوفر وظائف جاهزة للاستخدام (المجموعات، إدخال/إخراج الملفات، الشبكات، إلخ)
- **التوثيق**: متوفر في موقع وثائق جافا الرسمي من أوراكل
- **كيفية الاستخدام**: استيراد الحزم باستخدام عبارات `import`

### **مجموعة تطوير جافا (JDK)**
المكونات الأساسية لمجموعة تطوير جافا:

- **javac**: مترجم جافا (يحول ملفات .java إلى ملفات bytecode .class)
- **java**: مشغل بيئة وقت تشغيل جافا
- **javadoc**: أداة إنشاء التوثيق
- **jar**: أداة أرشيف جافا
- **يشمل JRE**: بيئة وقت تشغيل جافا لتنفيذ البرامج
- **المكتبات القياسية**: التنفيذ الكامل لواجهة برمجة تطبيقات جافا

**التثبيت والإعداد**:
- التنزيل من أوراكل أو استخدام OpenJDK
- تعيين متغير البيئة JAVA_HOME
- إضافة دليل bin لمجموعة تطوير جافا إلى مسار النظام (PATH)

### **أدوات بيئة التطوير المتكاملة (IDE)**
بيئات التطوير المتكاملة الشائعة لتطوير جافا:

1. **Eclipse** - مجاني، مفتوح المصدر، مستخدم على نطاق واسع في التعليم
2. **IntelliJ IDEA** - ميزات قوية، إصدارات مجانية ومدفوعة
3. **NetBeans** - بيئة التطوير المتكاملة المدعومة رسميًا من أوراكل
4. **VS Code** - خفيف الوزن مع إضافات جافا

**فوائد بيئة التطوير المتكاملة**:
- تمييز بناء الجملة وكشف الأخطاء
- إكمال الكود والاقتراحات
- أدوات تصحيح متكاملة
- إدارة المشاريع
- تكامل التحكم في الإصدار

## 2. إنشاء وترجمة وتشغيل برامج جافا

### **الهيكل الأساسي لبرنامج جافا**

```java
// كل تطبيق جافا يحتاج إلى كلاس رئيسي
public class HelloWorld {
    // main method - نقطة دخول البرنامج
    public static void main(String[] args) {
        // كودك يذهب هنا
        System.out.println("Hello, World!");
    }
}
```

### **الخطوات خطوة بخطوة**

**الخطوة 1: إنشاء برنامج جافا**
- إنشاء ملف نصي بامتداد `.java`
- يجب أن يتطابق اسم الملف مع اسم الكلاس العام (حساس لحالة الأحرف)
- مثال: `HelloWorld.java` للكلاس `HelloWorld`

**الخطوة 2: الترجمة**
```bash
javac HelloWorld.java
```
- هذا ينشئ `HelloWorld.class` (ملف bytecode)
- يتحقق المترجم من أخطاء بناء الجملة
- إذا كانت هناك أخطاء، تفشل الترجمة مع رسائل خطأ

**الخطوة 3: التشغيل**
```bash
java HelloWorld
```
- ملاحظة: استخدم اسم الكلاس بدون امتداد `.class`
- تقوم آلة جافا الافتراضية (JVM) بتحميل الكلاس وتنفيذ الـ main method

### **سير العمل عبر سطر الأوامر مقابل بيئة التطوير المتكاملة**

**سطر الأوامر**:
- افتح الطرفية/موجه الأوامر
- انتقل إلى الدليل الذي يحتوي على ملف .java الخاص بك
- استخدم `javac` للترجمة، `java` للتشغيل
- جيد لفهم العملية الأساسية

**سير عمل بيئة التطوير المتكاملة**:
- إنشاء مشروع جافا جديد
- إنشاء كلاس جديد
- كتابة الكود في المحرر
- انقر على زر "Run" (تقوم بيئة التطوير المتكاملة بالتعامل مع الترجمة تلقائيًا)
- أكثر ملاءمة للمشاريع الأكبر

## 3. إرشادات نمط البرمجة

نمط البرمجة الجيد يجعل الكود مقروءًا وقابلًا للصيانة. اتبع هذه الاتفاقيات:

### **اتفاقيات التسمية**

- **الكلاسات**: PascalCase (اكتب الحرف الأول من كل كلمة بحرف كبير)
  - أمثلة: `StudentRecord`، `BankAccount`، `HelloWorld`

- **الأساليب والمتغيرات**: camelCase (الكلمة الأولى بحرف صغير، اكتب الحرف الأول من الكلمات اللاحقة بحرف كبير)
  - أمثلة: `calculateTotal()`، `firstName`، `studentAge`

- **الثوابت**: أحرف كبيرة كلها مع شرطات سفلية
  - أمثلة: `MAX_SIZE`، `PI`، `DEFAULT_VALUE`

- **الحزم**: أحرف صغيرة كلها، غالبًا اسم نطاق عكسي
  - أمثلة: `com.company.project`، `java.util`

### **تنسيق الكود**

**المسافة البادئة**:
```java
public class Example {
    public static void main(String[] args) {
        if (condition) {
            // مسافة بادئة 4 مسافات أو 1 tab
            statement;
        }
    }
}
```

**الأقواس المعقوفة**:
- القوس الافتتاحي على نفس السطر (اتفاقية جافا)
- القوس الختامي على سطره الخاص، محاذٍ مع العبارة

**المسافات**:
```java

int sum = a + b;
if (x > 0) {

// تباعد سيء
int sum=a+b;

```

### **التعليقات**

**تعليقات سطر واحد**:
```java

```

**تعليقات متعددة الأسطر**:
```java

 * هذا تعليق متعدد الأسطر
 * يستخدم للشروحات الأطول
 */
```

**تعليقات Javadoc** (للتوثيق):
```java
/**

 * @param a الرقم الأول
 * @param b الرقم الثاني

```

### **أفضل الممارسات**

1. **أسماء ذات معنى**: استخدم أسماء متغيرات وأساليب وصفيّة
   - جيد: `studentCount`، `calculateAverage()`
   - سيء: `x`، `doStuff()`

2. **عبارة واحدة لكل سطر**: تجنب حشر عبارات متعددة في سطر واحد

3. **نمط متسق**: اتبع نفس الاتفاقيات في جميع أنحاء كودك

4. **المساحة البيضاء**: استخدم أسطرًا فارغة لفصل الأقسام المنطقية

5. **اجعل الأساليب قصيرة**: يجب أن يقوم كل أسلوب بعمل واحد بشكل جيد

## 4. أخطاء البرمجة الشائعة وأساسيات التصحيح

### **أنواع الأخطاء**

#### **أ. أخطاء بناء الجملة (أخطاء وقت الترجمة)**
هذه تمنع الترجمة ويجب إصلاحها قبل التشغيل:

**أخطاء بناء الجملة الشائعة**:
```java
// فاصلة منقوطة مفقودة
int x = 5  // خطأ: ; مفقودة

// أقواس معقوفة غير متطابقة
public class Test {
    public static void main(String[] args) {
        System.out.println("Hello");
    // قوس إغلاق } مفقود

// حساسية حالة الأحرف
String name = "John";
system.out.println(name); // خطأ: يجب أن تكون 'System'

// عدم تطابق اسم الملف
// الملف: Test.java
public class MyClass { // خطأ: يجب أن يتطابق اسم الكلاس مع اسم الملف
```

#### **ب. أخطاء وقت التشغيل**
البرنامج يترجم لكنه يتعطل أثناء التنفيذ:

```java
// القسمة على صفر
int result = 10 / 0; // ArithmeticException

// مؤشر null
String str = null;
int length = str.length(); // NullPointerException

// فهرس المصفوفة خارج الحدود
int[] arr = {1, 2, 3};
int value = arr[5]; // ArrayIndexOutOfBoundsException
```

#### **ج. أخطاء منطقية**
البرنامج يعمل لكنه ينتج نتائج خاطئة:

```java
// عامل تشغيل خاطئ
int average = (a + b) * 2; // يجب أن يكون / وليس *

// خطأ off-by-one
for (int i = 0; i <= arr.length; i++) { // يجب أن يكون i < arr.length

// شرط خاطئ
if (age > 18) { // يجب أن يكون >= لـ "18 عامًا فأكثر"
```

### **تقنيات التصحيح**

#### **1. اقرأ رسائل الخطأ بعناية**
```
HelloWorld.java:5: error: ';' expected
        int x = 5
                 ^
1 error
```
- **رقم السطر**: يظهر مكان حدوث الخطأ (السطر 5)
- **نوع الخطأ**: يخبرك ما الخطأ (فاصلة منقوطة مفقودة)
- **المؤشر**: يظهر الموقع الدقيق

#### **2. تصحيح باستخدام عبارات الطباعة**
```java
public static int calculateSum(int a, int b) {
    System.out.println("Debug: a = " + a + ", b = " + b);
    int sum = a + b;
    System.out.println("Debug: sum = " + sum);
    return sum;
}
```

#### **3. استخدم مصحح بيئة التطوير المتكاملة**
- **نقاط التوقف**: أوقف التنفيذ عند أسطر محددة
- **Step Over**: نفذ السطر الحالي وانتقل إلى التالي
- **Step Into**: ادخل إلى استدعاءات الأساليب لرؤية التنفيذ الداخلي
- **مراقبة المتغيرات**: راقب قيم المتغيرات في الوقت الفعلي
- **كومة الاستدعاءات**: انظر تسلسل استدعاءات الأساليب

#### **4. فرق تسد**
- علّق أقسامًا من الكود لعزل المشكلة
- اختبر الأجزاء الصغيرة بشكل مستقل
- أضف الكود تدريجيًا حتى تظهر الخطأ مرة أخرى

#### **5. تصحيح البطة المطاطية**
- اشرح كودك سطرًا بسطر لشخص ما (أو شيء ما)
- غالبًا ما يساعدك هذا على اكتشاف المشكلة بنفسك

### **أخطاء المبتدئين الشائعة**

1. **نسيان الترجمة بعد التغييرات**
   - أعد الترجمة دائمًا قبل التشغيل

2. **اسم الكلاس لا يتطابق مع اسم الملف**
   - `public class Student` يجب أن تكون في `Student.java`

3. **توقيع الـ main method مفقود**
   - يجب أن يكون بالضبط: `public static void main(String[] args)`

4. **نسيان استيراد الحزم**
   ```java
   import java.util.Scanner; // لا تنسى هذا!
   ```

5. **حالة الأحرف غير صحيحة**
   - `String` وليس `string`، `System` وليس `system`

6. **استخدام = بدلاً من == في الشروط**
   ```java
   if (x = 5) { // خطأ: تعيين، ليس مقارنة
   if (x == 5) { // صحيح
   ```

### **تمرين عملي**

حاول تحديد وإصلاح جميع الأخطاء في هذا الكود:

```java
public class test {
    Public Static Void Main(string[] args) {
        int x = 5

    }
}
```

**الإجابات**:
- يجب أن يكون اسم الكلاس `Test` (بحرف T كبير) ويتطابق مع اسم الملف
- `Public` → `public`، `Static` → `static`، `Void` → `void`
- `string` → `String`
- فاصلة منقوطة مفقودة بعد `int x = 5`
- `System.Out` → `System.out`
- فاصلة منقوطة مفقودة بعد عبارة println
