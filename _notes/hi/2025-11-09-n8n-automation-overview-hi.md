## 1. इंस्टालेशन और सेटअप

n8n हल्का और तेजी से शुरू होने वाला है। आवश्यकताएँ: स्थानीय इंस्टॉलेशन के लिए Node.js (v18+ अनुशंसित); कंटेनरों के लिए Docker। प्रोडक्शन के लिए, DigitalOcean या AWS जैसे VPS का उपयोग करें।

### त्वरित स्थानीय प्रारंभ (विकास/परीक्षण)
1. अपना टर्मिनल खोलें।
2. चलाएँ: `npx n8n`
   - यह n8n को अस्थायी रूप से डाउनलोड और लॉन्च करता है।
3. अपने ब्राउज़र में `http://localhost:5678` पर एडिटर तक पहुँचें।
   - डिफ़ॉल्ट लॉगिन: शुरुआत में कोई क्रेडेंशियल्स आवश्यक नहीं (सुरक्षा के लिए बाद में सेट करें)।

### स्थायी स्थानीय इंस्टॉलेशन (npm)
1. वैश्विक रूप से इंस्टॉल करें: `npm install n8n -g`
2. प्रारंभ करें: `n8n start`
3. `http://localhost:5678` पर एक्सेस करें।

### Docker (प्रोडक्शन के लिए अनुशंसित)
1. इमेज पुल करें: `docker run -it --rm --name n8n -p 5678:5678 -v ~/.n8n:/home/node/.n8n n8nio/n8n`
   - यह डेटा दृढ़ता के लिए एक वॉल्यूम मैप करता है।
2. उन्नत सेटअप के लिए (जैसे, PostgreSQL DB): डॉक्स से `docker-compose.yml` का उपयोग करें।
3. `http://localhost:5678` पर एक्सेस करें।

### क्लाउड विकल्प
- **n8n क्लाउड**: n8n.io पर प्रबंधित होस्टिंग—साइन अप करें, मिनटों में डिप्लॉय करें, सीमाओं के साथ मुफ्त शुरू होता है।
- **थर्ड-पार्टी PaaS**: Render, Railway, या Sevalla (वन-क्लिक टेम्पलेट) का उपयोग करें। Sevalla पर उदाहरण:

**सुझाव**: सेल्फ-होस्टिंग के लिए, HTTPS के साथ सुरक्षित करें (रिवर्स प्रॉक्सी जैसे Nginx के माध्यम से), env वेरिएबल सेट करें (जैसे, `N8N_BASIC_AUTH_ACTIVE=true`), और `~/.n8n` फ़ोल्डर का बैकअप लें। उच्च-वॉल्यूम वर्कफ़्लो के लिए क्यू मोड के साथ स्केल करें।

## 3. मूल अवधारणाएं

- **वर्कफ़्लो**: कनेक्टेड नोड्स का एक क्रम जो ऑटोमेशन लॉजिक को परिभाषित करता है। सक्रिय वर्कफ़्लो ट्रिगर पर चलते हैं; निष्क्रिय परीक्षण के लिए होते हैं।
- **नोड्स**: मॉड्यूलर ब्लॉक:
  - **ट्रिगर**: वर्कफ़्लो शुरू करते हैं (जैसे, क्रॉन जॉब्स के लिए Schedule, HTTP इवेंट्स के लिए Webhook, फ़ीड्स के लिए RSS Read)।
  - **क्रियाएं**: काम करती हैं (जैसे, ईमेल भेजें, API के लिए HTTP Request, कस्टम कोड के लिए Function)।
  - **कोर नोड्स**: IF (सशर्त), Merge (डेटा संयोजन), Set (वेरिएबल्स में हेरफेर)।
- **कनेक्शन**: नोड्स के बीच तीर डेटा फ़्लो दिखाते हैं (JSON फॉर्मेट)। एक नोड से डेटा अगले नोड को फीड करता है।
- **एक्सप्रेशन**: डायनामिक प्लेसहोल्डर जैसे `{{ $json.title }}` फील्ड्स में डेटा खींचने के लिए (जैसे, लेख शीर्षक)। टाइमस्टैम्प के लिए `$now` या बैचों के लिए `$input.all()` का उपयोग करें।
- **क्रेडेंशियल्स**: API कुंजियों/OAuth के लिए सुरक्षित संग्रहण। प्रति सेवा एक बार सेट करें (जैसे, Gmail OAuth) और नोड्स में पुन: उपयोग करें।
- **एक्ज़िक्यूशन**: एक वर्कफ़्लो के रन; लॉग देखें, डेटा दोबारा चलाएं, या त्रुटियों को डीबग करें।

## 4. अपना पहला वर्कफ़्लो बनाना: चरण-दर-चरण

आइए "डेली RSS डाइजेस्ट ईमेल" बनाएं।

1. **नया वर्कफ़्लो बनाएं**:
   - "New" > क्लिक करें > इसका नाम "RSS Digest" रखें।
   - कैनवास खुलता है।

2. **ट्रिगर नोड जोड़ें**:
   - "+" > क्लिक करें > "Schedule Trigger" खोजें।
   - कॉन्फ़िगर करें: "Every Day" सुबह 9:00 बजे ट्रिगर करें (क्रॉन: `0 9 * * *`)।
   - परीक्षण: "Execute Node" क्लिक करें (अभी एक बार चलता है)।

3. **डेटा फ़ेच नोड जोड़ें**:
   - ट्रिगर के बाद "+" > "RSS Read" क्लिक करें।
   - URL: `https://blog.cloudflare.com/rss/`।
   - एक्ज़िक्यूट: आइटम फ़ेच करता है (जैसे, शीर्षक, लिंक, pubDate के साथ JSON)।

4. **डेटा ट्रांसफ़ॉर्म करें (ऑप्शनल फ़ंक्शन नोड)**:
   - "+" > "Function"।
   - कोड:
     ```
     // शीर्ष 3 आइटम्स तक सीमित करें
     return items.slice(0, 3);
     ```
   - यह आने वाले डेटा पर JS चलाता है।

5. **एक्शन नोड जोड़ें**:
   - "+" > "Gmail" (या SMTP के लिए "Email Send")।
   - क्रेडेंशियल्स: "Create New" > क्लिक करें > Gmail के लिए OAuth (Google ऑथ फ़्लो का पालन करें)।
   - To: आपका ईमेल।
   - Subject: `Daily Digest: {{ $input.first().json.title }}`
   - Message: एक्सप्रेशन के साथ आइटम्स पर लूप करें:
     ```
     {{#each $input.all()}}
     - {{ $json.title }}: {{ $json.link }} ({{ $json.pubDate }})
     {{/each}}
     ```
   - (लूप्स के लिए Handlebars-जैसे सिंटैक्स का उपयोग करता है।)

6. **कनेक्ट करें और परीक्षण करें**:
   - तीर खींचें: Trigger → RSS → Function → Email।
   - "Execute Workflow": डेटा फ़्लो देखें; इनबॉक्स चेक करें।
   - त्रुटियाँ ठीक करें: लाल नोड्स समस्याओं को हाइलाइट करते हैं (जैसे, अमान्य क्रेडेंशियल्स)।

7. **सक्रिय करें**:
   - "Active" टॉगल ऑन करें। अब यह दैनिक चलता है।

साझा करने के लिए JSON के रूप में सेव और एक्सपोर्ट करें।

## 5. अधिक जटिल वर्कफ़्लो बनाना

- **सशर्त**: RSS के बाद "IF" नोड जोड़ें: `{{ $json.pubDate }} > {{ $now.minus({days:1}) }}` नए आइटम्स को फ़िल्टर करने के लिए।
- **लूप और बैच**: बड़े डेटासेट को प्रोसेस करने के लिए "Split In Batches" का उपयोग करें।
- **त्रुटि प्रबंधन**: "Error Trigger" वर्कफ़्लो जोड़ें या पुनः प्रयास के लिए "IF" का उपयोग करें। त्रुटियों को लॉग करने के लिए "Set" का उपयोग करें।
- **API एकीकरण**: कस्टम एंडपॉइंट्स के लिए "HTTP Request" नोड (जैसे, Slack वेबहुक पर POST)।
- **डेटा हेरफेर**: JSON ट्वीक्स के लिए "Edit Fields" या Function नोड्स।
- **परीक्षण**: विशिष्ट एक्ज़िक्यूशन दोबारा चलाएं; नोड्स में मॉक डेटा।

उदाहरण: Twitter मॉनिटर
1. ट्रिगर: "Twitter Trigger" मेंशन पर।
2. AI नोड: सेंटीमेंट वर्गीकृत करने के लिए "OpenAI"।
3. IF: पॉजिटिव → CRM में जोड़ें; नेगेटिव → Slack अलर्ट।

## 6. उन्नत उपयोग और सर्वोत्तम अभ्यास

- **AI वर्कफ़्लो**: "AI Agent" नोड जोड़ें > टूल्स के साथ चेन करें (जैसे, खोज, सारांश)। "Ollama" नोड के माध्यम से लोकल LLM एकीकृत करें।
- **कस्टम नोड्स**: JS के माध्यम से बनाएं (डॉक्स ट्यूटोरियल); कम्युनिटी में प्रकाशित करें।
- **स्केलिंग**: क्यू मोड (`N8N_WORKER=1`), एक्सटर्नल DB (PostgreSQL), और रियल-टाइम के लिए वेबहुक का उपयोग करें।
- **सुरक्षा**: सीक्रेट्स एन्क्रिप्ट करें, टीमों के लिए RBAC का उपयोग करें, ऑडिट लॉग।
- **डीबगिंग**: फ़ंक्शन में इनलाइन लॉग (`console.log(items)`); Git के साथ वर्जन करें।
- **सामान्य गलतियाँ**: डेटा प्रकार देखें (सभी JSON); नोड्स में रेट लिमिट हैंडल करें; AI से पहले सरल शुरुआत करें।
- **एक्सटेंशन**: ऐप्स में एम्बेड करें, टेम्पलेट्स (1700+ उपलब्ध) का उपयोग करें, या एक्स्ट्रा के लिए npm।

प्रोडक्शन के लिए, बाहरी टूल्स के माध्यम से मॉनिटर करें; सेल्फ-होस्ट लागत बचाता है (बनाम Zapier के $20+/माह)।
