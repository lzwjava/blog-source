## 1. Java भाषा विनिर्देश, API, JDK, और IDE टूल्स

### **Java भाषा विनिर्देश**
Java भाषा विनिर्देश Java के सिंटैक्स और शब्दार्थ को परिभाषित करता है। समझने योग्य प्रमुख अवधारणाएँ:

- **प्लेटफ़ॉर्म स्वतंत्रता**: "एक बार लिखो, कहीं भी चलाओ" (WORA) - Java कोड बाइटकोड में संकलित होता है जो JVM (Java वर्चुअल मशीन) वाले किसी भी प्लेटफ़ॉर्म पर चलता है
- **ऑब्जेक्ट-ओरिएंटेड**: Java में सब कुछ ऑब्जेक्ट्स और क्लासेस के आसपास संगठित होता है
- **सख्ती से टाइप की गई**: वेरिएबल्स को विशिष्ट डेटा टाइप के साथ घोषित किया जाना चाहिए
- **स्वचालित मेमोरी प्रबंधन**: गार्बेज कलेक्शन मेमोरी डीलोकेशन को संभालता है

### **Java API (एप्लिकेशन प्रोग्रामिंग इंटरफेस)**
Java API प्री-रिटन क्लासेस का एक बड़ा संग्रह है जो पैकेजों में संगठित है:

- **कोर पैकेज**: `java.lang` (स्वचालित रूप से इम्पोर्ट किया गया), `java.util`, `java.io`
- **उद्देश्य**: तैयार-से-उपयोग कार्यक्षमता प्रदान करता है (कलेक्शन, फ़ाइल I/O, नेटवर्किंग, आदि)
- **डॉक्यूमेंटेशन**: Oracle की आधिकारिक Java डॉक्यूमेंटेशन साइट पर उपलब्ध
- **उपयोग कैसे करें**: `import` स्टेटमेंट का उपयोग करके पैकेज इम्पोर्ट करें

### **JDK (Java डेवलपमेंट किट)**
JDK के आवश्यक घटक:

- **javac**: Java कंपाइलर (.java फ़ाइलों को .class बाइटकोड फ़ाइलों में परिवर्तित करता है)
- **java**: Java रनटाइम एनवायरनमेंट लॉन्चर
- **javadoc**: डॉक्यूमेंटेशन जनरेटर
- **jar**: Java आर्काइव टूल
- **JRE शामिल**: प्रोग्राम निष्पादित करने के लिए Java रनटाइम एनवायरनमेंट
- **मानक लाइब्रेरीज़**: संपूर्ण Java API इम्प्लीमेंटेशन

**इंस्टालेशन और सेटअप**:
- Oracle से डाउनलोड करें या OpenJDK का उपयोग करें
- JAVA_HOME एनवायरनमेंट वेरिएबल सेट करें
- सिस्टम PATH में JDK bin डायरेक्टरी जोड़ें

### **IDE (इंटीग्रेटेड डेवलपमेंट एनवायरनमेंट) टूल्स**
Java डेवलपमेंट के लिए लोकप्रिय IDE:

1. **Eclipse** - मुफ्त, ओपन-सोर्स, शिक्षा में व्यापक रूप से उपयोग किया जाता है
2. **IntelliJ IDEA** - शक्तिशाली सुविधाएँ, मुफ्त और भुगतान दोनों संस्करण
3. **NetBeans** - आधिकारिक Oracle-समर्थित IDE
4. **VS Code** - Java एक्सटेंशन के साथ हल्का

**IDE लाभ**:
- सिंटैक्स हाइलाइटिंग और त्रुटि पहचान
- कोड पूर्णता और सुझाव
- एकीकृत डिबगिंग टूल
- प्रोजेक्ट प्रबंधन
- वर्जन कंट्रोल एकीकरण

## 2. Java प्रोग्राम बनाना, संकलित करना और चलाना

### **बेसिक Java प्रोग्राम स्ट्रक्चर**

```java
// प्रत्येक Java एप्लिकेशन को एक मुख्य क्लास की आवश्यकता होती है
public class HelloWorld {
    // main मेथड - प्रोग्राम का एंट्री पॉइंट
    public static void main(String[] args) {
        // आपका कोड यहाँ जाता है
        System.out.println("Hello, World!");
    }
}
```

### **चरण-दर-चरण प्रक्रिया**

**चरण 1: Java प्रोग्राम बनाना**
- `.java` एक्सटेंशन के साथ एक टेक्स्ट फ़ाइल बनाएँ
- फ़ाइलनाम MUST पब्लिक क्लास नाम से मेल खाना चाहिए (केस-संवेदी)
- उदाहरण: क्लास `HelloWorld` के लिए `HelloWorld.java`

**चरण 2: संकलित करना**
```bash
javac HelloWorld.java
```
- यह `HelloWorld.class` (बाइटकोड फ़ाइल) बनाता है
- कंपाइलर सिंटैक्स त्रुटियों की जांच करता है
- यदि त्रुटियाँ मौजूद हैं, तो संकलन त्रुटि संदेशों के साथ विफल हो जाता है

**चरण 3: चलाना**
```bash
java HelloWorld
```
- नोट: `.class` एक्सटेंशन के बिना क्लास नाम का उपयोग करें
- JVM क्लास लोड करता है और main मेथड निष्पादित करता है

### **कमांड लाइन बनाम IDE वर्कफ़्लो**

**कमांड लाइन**:
- टर्मिनल/कमांड प्रॉम्प्ट खोलें
- अपनी .java फ़ाइल वाली डायरेक्टरी पर नेविगेट करें
- संकलित करने के लिए `javac` का उपयोग करें, चलाने के लिए `java` का उपयोग करें
- अंतर्निहित प्रक्रिया को समझने के लिए अच्छा है

**IDE वर्कफ़्लो**:
- एक नया Java प्रोजेक्ट बनाएँ
- एक नई क्लास बनाएँ
- एडिटर में कोड लिखें
- "रन" बटन क्लिक करें (IDE स्वचालित रूप से संकलन संभालता है)
- बड़े प्रोजेक्ट्स के लिए अधिक सुविधाजनक

## 3. प्रोग्रामिंग स्टाइल दिशानिर्देश

अच्छी प्रोग्रामिंग स्टाइल कोड को पठनीय और रखरखाव योग्य बनाती है। इन परंपराओं का पालन करें:

### **नामकरण परंपराएँ**

- **क्लासेस**: PascalCase (प्रत्येक शब्द का पहला अक्षर बड़ा करें)
  - उदाहरण: `StudentRecord`, `BankAccount`, `HelloWorld`

- **मेथड्स और वेरिएबल्स**: camelCase (पहला शब्द लोअरकेस, बाद के शब्दों को कैपिटलाइज़ करें)
  - उदाहरण: `calculateTotal()`, `firstName`, `studentAge`

- **कॉन्स्टेंट्स**: ALL_CAPS अंडरस्कोर के साथ
  - उदाहरण: `MAX_SIZE`, `PI`, `DEFAULT_VALUE`

- **पैकेजेस**: सभी लोअरकेस, अक्सर रिवर्स डोमेन नाम
  - उदाहरण: `com.company.project`, `java.util`

### **कोड फॉर्मेटिंग**

**इंडेंटेशन**:
```java

    public static void main(String[] args) {
        if (condition) {

        }
    }
}
```

**ब्रेसेस**:
- उसी लाइन पर ओपनिंग ब्रेस (Java परंपरा)
- क्लोजिंग ब्रेस अपनी लाइन पर, स्टेटमेंट के साथ संरेखित

**स्पेसिंग**:

**सिंगल-लाइन टिप्पणियाँ**:

4. **व्हाइट स्पेस**: तार्किक खंडों को अलग करने के लिए रिक्त पंक्तियों का उपयोग करें

5. **मेथड्स को छोटा रखें**: प्रत्येक मेथड को एक काम अच्छी तरह से करना चाहिए

```java

public class MyClass { // त्रुटि: क्लास नाम फ़ाइलनाम से मेल खाना चाहिए
```

#### **B. रनटाइम त्रुटियाँ**
प्रोग्राम संकलित होता है लेकिन निष्पादन के दौरान क्रैश हो जाता है:

```java
// शून्य से विभाजन
int result = 10 / 0; // ArithmeticException

// नल पॉइंटर
String str = null;
int length = str.length(); // NullPointerException

// सरणी सूचकांक सीमा से बाहर

```

प्रोग्राम चलता है लेकिन गलत परिणाम देता है:

```java
// गलत ऑपरेटर
int average = (a + b) * 2; // / होना चाहिए, * नहीं

// ऑफ-बाय-वन त्रुटि
for (int i = 0; i <= arr.length; i++) { // i < arr.length होना चाहिए

// गलत कंडीशन

```

### **अभ्यास अभ्यास**

इस कोड में सभी त्रुटियों को पहचानने और ठीक करने का प्रयास करें:

```java
public class test {
    Public Static Void Main(string[] args) {
        int x = 5
        int y = 10;
        System.Out.Println(x + y)
    }
}
```

**उत्तर**:
- क्लास नाम `Test` (कैपिटल T) होना चाहिए और फ़ाइलनाम से मेल खाना चाहिए
- `Public` → `public`, `Static` → `static`, `Void` → `void`
- `string` → `String`
- `int x = 5` के बाद अर्धविराम गुम है
- `System.Out` → `System.out`
- println स्टेटमेंट के बाद अर्धविराम गुम है
